W&B requires a valid signed SSL/TLS certificate for secure communication between clients and the server. The ingress or load balancer must terminate SSL/TLS. The W&B Server application doesn't terminate SSL or TLS connections.

**Important**: W&B doesn't support self-signed certificates or custom certificate authorities (CAs). Self-signed certificates cause problems for users.

If possible, use a service like [Let's Encrypt](https://letsencrypt.org) to provide trusted certificates to your load balancer. Services like Caddy and Cloudflare manage SSL for you.

If your security policies require SSL communication within your trusted networks, consider a tool like Istio with [sidecar containers](https://istio.io/latest/docs/reference/config/networking/sidecar/).
