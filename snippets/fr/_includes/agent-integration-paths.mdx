<div id="agent-harnesses">
  ## Harness d’agents
</div>

Les harness d’agents sont des environnements d’exécution pour agents destinés aux utilisateurs finaux, comme les agents de code et les outils de développement. Installez le plugin ou l’extension, puis démarrez une session dans le harness pour activer le traçage :

* [plugin Claude Code](/fr/weave/guides/integrations/agents/claude-code-harness)
* [plugin Codex](/fr/weave/guides/integrations/agents/codex-harness)
* [plugin OpenClaw](/fr/weave/guides/integrations/agents/openclaw-harness)
* [extension Pi](/fr/weave/guides/integrations/agents/pi-dev-harness)

<div id="agent-sdks">
  ## SDK Agents
</div>

Les SDK Agents sont des bibliothèques permettant de créer des agents et des flux de travail multi-agent dans le code de votre propre application. Ajoutez `weave.init("[YOUR-TEAM]/[YOUR-PROJECT]")` à votre script, et Weave applique automatiquement un patch au SDK :

* [SDK OpenAI Agents](/fr/weave/guides/integrations/agents/openai-agents-sdk)
* [Google Agent Development Kit (ADK)](/fr/weave/guides/integrations/agents/google-adk)
* [Claude Agent SDK](/fr/weave/guides/integrations/agents/claude-agents-sdk)

<div id="custom-agents-and-opentelemetry">
  ## Agents personnalisés et OpenTelemetry
</div>

Si votre agent n’est pas développé avec l’un des SDK ci-dessus, ou s’il émet déjà des spans OpenTelemetry, utilisez directement le SDK Weave. Weave accepte n’importe quel span OTel et applique un traitement spécifique aux [attributs de convention sémantique GenAI](https://opentelemetry.io/docs/specs/semconv/gen-ai/gen-ai-agent-spans/) afin que les spans s’affichent dans la vue **Agents** :

* [Démarrage rapide : instrumenter manuellement un agent](/fr/weave/custom-agents-quickstart) — un guide pas à pas pour un petit agent sur plusieurs tours de conversation.
* [Tracer vos agents](/fr/weave/guides/tracking/trace-agents) — la référence complète des fonctions utilitaires pour les agents et des concepts associés.
