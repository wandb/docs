---
title: Wandb Docker
---

Exécutez votre code dans un conteneur Docker.

W&amp;B docker vous permet d’exécuter votre code dans une image Docker tout en garantissant que wandb est configuré. Il ajoute les variables d’environnement WANDB&#95;DOCKER et WANDB&#95;API&#95;KEY à votre conteneur et monte, par défaut, le répertoire actuel dans /app. Vous pouvez également transmettre des arguments supplémentaires, qui seront ajoutés à `docker run` avant le nom de l’image ; si aucune image n’est fournie, nous en choisirons une par défaut :

```sh
wandb docker -v /mnt/dataset:/app/data
wandb docker gcr.io/kubeflow-images-public/tensorflow-1.12.0-notebook-cpu:v0.4.0 --jupyter
wandb docker wandb/deepo:keras-gpu --no-tty --cmd "python train.py --epochs=5"
```

Par défaut, nous remplaçons le point d’entrée afin de vérifier si wandb est présent et de l’installer dans le cas contraire. Si vous passez l’indicateur --jupyter, nous nous assurons que jupyter est installé et démarrons jupyter lab sur le port 8888. Si nous détectons nvidia-docker sur votre système, nous utiliserons le runtime nvidia. Si vous voulez simplement que wandb définisse la variable d’environnement pour une commande docker run existante, voir la commande wandb docker-run.
