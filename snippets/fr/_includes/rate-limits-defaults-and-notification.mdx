Les limites par défaut suivantes contribuent à maintenir la stabilité de la plateforme :

| Limite                                  | Par défaut | Portée |
| --------------------------------------- | ---------- | ------ |
| Requêtes Filestream par seconde         | 500        | projet |
| Ingestion Filestream par seconde        | 120 MB     | projet |
| Requêtes Filestream par seconde         | 2          | Run    |
| Requêtes de création de Run par seconde | 80         | projet |

Lorsqu&#39;une limite est dépassée, le SDK W&amp;B renvoie la réponse HTTP `429`, et le message `HTTP 429: rate limited exceeded` apparaît dans les journaux du SDK.

* Les limites de débit de Filestream ne provoquent jamais de plantage ni d&#39;échec de la journalisation. Lorsque le SDK reçoit une réponse `429` à une requête Filestream, il ralentit puis réessaie la requête limitée telle quelle, tandis que les mises à jour suivantes s&#39;accumulent.

* Les limites de débit de création de Run bloquent la poursuite de l&#39;entraînement.

```
HTTP 429: rate limit exceeded
```
