다음 기본 제한은 플랫폼 안정성을 유지하는 데 도움이 됩니다:

| Limit              | Default | Scope |
| ------------------ | ------- | ----- |
| Filestream 초당 요청 수 | 500     | 프로젝트  |
| Filestream 초당 수집량  | 120 MB  | 프로젝트  |
| Filestream 초당 요청 수 | 2       | run   |
| run 생성 초당 요청 수     | 80      | 프로젝트  |

제한을 초과하면 W&amp;B SDK는 HTTP 응답 `429`를 반환하고, SDK 로그에 `HTTP 429: rate limited exceeded` 메시지가 표시됩니다.

* 파일 시스템 속도 제한으로 인해 로깅이 중단되거나 실패하지는 않습니다. SDK가 filestream 요청에 대해 `429` 응답을 받으면, 이후 업데이트가 누적되는 동안 해당 요청에 대해 백오프한 뒤 원래 요청 그대로 재시도합니다.

* run 생성 속도 제한은 이후 트레이닝을 차단합니다.

```
HTTP 429: rate limit exceeded
```
