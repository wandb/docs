---
title: 요청 속도 제한 기본값 및 알림
---

다음 기본 제한은 플랫폼 안정성을 유지하는 데 도움이 됩니다:

| Limit              | Default | Scope |
| ------------------ | ------- | ----- |
| 초당 Filestream 요청 수 | 500     | 프로젝트  |
| 초당 Filestream 수집량  | 120 MB  | 프로젝트  |
| 초당 Filestream 요청 수 | 2       | run   |
| 초당 run 생성 요청 수     | 80      | 프로젝트  |

제한을 초과하면 W&amp;B SDK는 HTTP 응답 `429`를 반환하며, SDK 로그에 `HTTP 429: rate limited exceeded` 메시지가 표시됩니다.

* 파일시스템 요청 속도 제한 때문에 로깅이 중단되거나 실패하는 일은 절대 없습니다. SDK가 filestream 요청에서 `429` 응답을 받으면, 후속 업데이트가 누적되는 동안 잠시 대기한 뒤 요청 속도 제한에 걸린 요청을 그대로 재시도합니다.

* run 생성 요청 속도 제한은 이후 트레이닝을 차단합니다.

```
HTTP 429: rate limit exceeded
```
