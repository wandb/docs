<div id="review-evaluation-results">
  ## 평가 결과 검토
</div>

대상 프로젝트 워크스페이스의 W&amp;B Models에서 Evaluation 작업 결과를 검토하세요.

1. 페이지 상단의 원형 화살표 아이콘을 클릭하여 최근 실행 모달 창을 엽니다. 여기에서 Evaluation 작업은 프로젝트의 다른 실행들과 함께 표시됩니다. Evaluation 작업에 리더보드가 있는 경우 **Leaderboard**를 클릭하여 전체 화면으로 리더보드를 열거나, 실행 이름을 클릭하여 프로젝트의 단일 실행 보기에서 엽니다.
2. 워크스페이스의 **Evaluations** 섹션 또는 **Weave** 사이드바 패널의 **Traces** 탭에서 Evaluation 작업의 트레이스를 확인합니다.
3. **Overview** 탭을 클릭하여 구성 및 요약 지표를 포함한 Evaluation 작업의 상세 정보를 확인합니다.
4. **Logs** 탭을 클릭하여 Evaluation 작업의 디버그 로그를 조회, 검색 또는 다운로드합니다.
5. **Files** 탭을 클릭하여 코드, 로그, 구성 파일 및 기타 출력 파일을 포함한 Evaluation 작업의 파일을 탐색, 조회 또는 다운로드합니다.

<div id="customize-a-leaderboard">
  ## 리더보드 사용자 지정
</div>

리더보드는 특정 프로젝트로 전송된 모든 평가 작업의 결과를 표시하며, 평가 작업당 각 벤치마크마다 한 행이 표시됩니다. 열에는 평가 작업에 대한 trace, 입력 값, 출력 값 등의 세부 정보가 표시됩니다. 리더보드에 대한 자세한 내용은 [Weave의 리더보드](/ko/weave/guides/core-types/leaderboards)를 참조하세요.

<Tip>리더보드에서 특정 결과에 대해 피드백을 남기려면 **Feedback** 열의 이모지 아이콘 또는 채팅 아이콘을 클릭하세요.</Tip>

* 기본적으로 모든 평가 작업이 표시됩니다. 왼쪽의 실행 선택기를 사용해 평가 작업을 필터링하거나 검색하세요.
* 기본적으로 평가 작업은 그룹화되어 있지 않습니다. 하나 이상의 열을 기준으로 그룹화하려면 **Group** 아이콘을 클릭하세요. 그룹을 표시하거나 숨기고, 그룹을 펼쳐 해당 실행을 볼 수 있습니다.
* 기본적으로 모든 operation이 표시됩니다. 단일 operation만 표시하려면 **All ops**를 클릭한 다음 operation을 선택하세요.
* 특정 열을 기준으로 정렬하려면 열 제목을 클릭하세요. 열 표시를 사용자 지정하려면 **Columns**를 클릭하세요.
  * 기본적으로 헤더는 단일 수준으로 구성됩니다. 관련 헤더를 함께 묶어 구성할 수 있도록 헤더 깊이를 늘릴 수 있습니다.
  * 개별 열을 선택하거나 선택 해제하여 표시하거나 숨길 수 있고, 한 번의 클릭으로 모든 열을 표시하거나 숨길 수도 있습니다.
  * 열을 고정하여 고정되지 않은 열보다 앞에 표시되도록 할 수 있습니다.

<div id="export-a-leaderboard">
  ## 리더보드 내보내기
</div>

리더보드를 내보내려면 다음을 수행하세요:

1. **Columns** 버튼 근처에 있는 다운로드 아이콘을 클릭합니다.
2. 내보내기 파일 크기를 최적화하기 위해 기본적으로 trace 루트만 내보내집니다. 전체 trace를 내보내려면 **Trace roots only**를 끕니다.
3. 내보내기 파일 크기를 최적화하기 위해 기본적으로 피드백과 비용은 내보내지 않습니다. 내보내기에 포함하려면 **Feedback** 또는 **Costs**를 토글합니다.
4. 기본적으로 내보내기 형식은 JSONL입니다. 형식을 변경하려면 **Export to file**을 클릭하고 형식을 선택합니다.
5. 브라우저에서 리더보드를 내보내려면 **Export**를 클릭합니다.
6. 코드를 통해 리더보드를 내보내려면 **Python** 또는 **cURL**을 선택한 다음 **Copy**를 클릭하고 스크립트나 명령을 실행합니다.
