---
title: 셀프 매니지드 Redis 요구사항
---

W&B는 작업 대기열 처리와 데이터 캐싱을 위해 W&B의 컴포넌트에서 사용하는 단일 노드 Redis 7.x 배포에 의존합니다. 개념 증명(PoC)의 테스트 및 개발을 편리하게 하기 위해, W&B Self-Managed에는 프로덕션 환경에 적합하지 않은 로컬 Redis 배포가 포함되어 있습니다.

프로덕션 배포의 경우, W&B는 다음 환경의 Redis 인스턴스에 연결할 수 있습니다:

- [AWS Elasticache](https://aws.amazon.com/pm/elasticache/)
- [Google Cloud Memory Store](https://cloud.google.com/memorystore?hl=en)
- [Azure Cache for Redis](https://azure.microsoft.com/en-us/products/cache)
- 클라우드 또는 온프레미스 인프라에 호스팅된 Redis 배포
