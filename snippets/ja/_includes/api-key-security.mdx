<div id="store-and-handle-api-keys-securely">
  ## APIキーを安全に保管・取り扱う
</div>

APIキーは W&amp;B アカウントへの액세스を提供するものなので、パスワード同様に保護する必要があります。以下のベストプラクティスに従ってください。

<div id="recommended-storage-methods">
  ### 推奨される保管方法
</div>

* **Secrets manager**: [AWS Secrets Manager](https://aws.amazon.com/secrets-manager/)、[HashiCorp Vault](https://developer.hashicorp.com/vault)、[Azure Key Vault](https://azure.microsoft.com/en-us/products/key-vault)、[Google Secret Manager](https://cloud.google.com/security/products/secret-manager) などの専用のシークレット管理システムを使用してください。
* **Password manager**: 信頼できるパスワードマネージャーアプリケーションを使用してください。
* **OS レベルのキーチェーン**: macOS Keychain、Windows Credential Manager、Linux Secret Service にキーを保存してください。本番環境での使用は推奨されません。

<div id="what-to-avoid">
  ### 避けるべきこと
</div>

* Git などのバージョン管理システムに APIキーをコミットしないでください。
* APIキーをプレーンテキストの設定ファイルに保存しないでください。
* OS の `ps` などのコマンド出力に表示されてしまうため、コマンドラインで APIキーを渡さないでください。
* メールやチャットなどの暗号化されていないチャネルで APIキーを共有することは避けてください。
* ソースコードに APIキーをハードコードしないでください。

APIキーが漏えいした場合は、直ちに W&amp;B アカウントからその APIキーを削除し、[サポート](mailto:support@wandb.ai) または担当の AISE に連絡してください。

<div id="environment-variables">
  ### 環境変数
</div>

コード内でAPIキーを使用する場合は、環境変数として渡してください。

```bash
export WANDB_API_KEY="your-api-key-here"
```

この方法を使うと、キーをソースコード内に埋め込まずに済み、必要になったときにローテーションしやすくなります。

<Warning>
  コマンドと同じ行で環境変数を設定するのは避けてください。その方法だと、`ps` のような OS コマンドの出力に表示されてしまいます。

  ```bash
  # このパターンは避けてください。プロセス管理ツールで APIキー が漏洩するおそれがあります
  export WANDB_API_KEY="your-api-key-here" ./my-script.sh
  ```
</Warning>


<div id="sdk-version-compatibility">
  ### SDK バージョンの互換性
</div>

新しいAPIキーは従来のキーより長くなっています。古いバージョンの`wandb`または`weave` SDKで認証する場合、APIキーの長さに関するエラーが発生することがあります。

**解決策**: 新しいSDKバージョンに更新してください。

* `wandb` SDK v0.22.3+

  ```bash
  pip install --upgrade wandb==0.22.3
  ```
* `weave` SDK v0.52.17+
  ```bash
  pip install --upgrade weave==0.52.17
  ```

SDKをすぐにアップグレードできない場合は、回避策として`WANDB_API_KEY`環境変数を使用してAPIキーを設定してください。
