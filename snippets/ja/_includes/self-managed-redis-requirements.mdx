---
title: セルフマネージド Redis の要件
---

W&amp;B は、W&amp;B のコンポーネントがジョブキューイングとデータキャッシュに使用する単一ノードの Redis 7.x デプロイメントに依存しています。テストや概念実証 (PoC) の開発を簡単に行えるように、W&amp;B Self-Managed にはローカル Redis デプロイメントが含まれていますが、これは本番デプロイメントには適していません。

本番デプロイメントでは、W&amp;B は次の環境にある Redis インスタンスに接続できます。

* [AWS Elasticache](https://aws.amazon.com/elasticache/)
* [Google Cloud Memory Store](https://cloud.google.com/memorystore?hl=en)
* [Azure Cache for Redis](https://azure.microsoft.com/en-us/products/cache)
* お使いのクラウドまたはオンプレミスのインフラストラクチャでホストされる Redis デプロイメント
