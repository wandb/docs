---
title: Wandb Docker
---

Dockerコンテナーでコードを実行します。

W&amp;B Docker を使うと、wandb が設定された状態で Dockerイメージ内でコードを実行できます。コンテナーには WANDB&#95;DOCKER と WANDB&#95;API&#95;KEY の環境変数が追加され、デフォルトでは現在のディレクトリーが /app にマウントされます。追加の引数を渡すこともでき、それらはイメージ名の前に `docker run` に追加されます。イメージを指定しない場合は、デフォルトのイメージが選択されます:

```sh
wandb docker -v /mnt/dataset:/app/data
wandb docker gcr.io/kubeflow-images-public/tensorflow-1.12.0-notebook-cpu:v0.4.0 --jupyter
wandb docker wandb/deepo:keras-gpu --no-tty --cmd "python train.py --epochs=5"
```

デフォルトでは、`wandb` が存在するかどうかを確認し、存在しない場合はインストールするようにエントリポイントを上書きします。`--jupyter` フラグを指定すると、`jupyter` がインストールされていることを確認し、ポート 8888 で `jupyter lab` を起動します。システム上で `nvidia-docker` を検出した場合は、`nvidia` ランタイムを使用します。既存の `docker run` コマンドに対して `wandb` に環境変数だけ設定させたい場合は、`wandb docker-run` コマンドを参照してください。
