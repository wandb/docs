プラットフォームの安定性を維持するために、次のデフォルト上限が設けられています。

| Limit                      | Default | Scope  |
| -------------------------- | ------- | ------ |
| Filestream の 1 秒あたりのリクエスト数 | 500     | プロジェクト |
| Filestream の 1 秒あたりの取り込み量  | 120 MB  | プロジェクト |
| Filestream の 1 秒あたりのリクエスト数 | 2       | run    |
| run 作成の 1 秒あたりのリクエスト数      | 80      | プロジェクト |

上限を超えると、W&amp;B SDK は HTTP レスポンス `429` を返し、SDK ログに `HTTP 429: rate limited exceeded` というメッセージが表示されます。

* Filestream のレート制限によって、ログ記録がクラッシュしたり失敗したりすることはありません。SDK が filestream リクエストで `429` レスポンスを受け取ると、後続の更新を蓄積しながら、レート制限されたリクエストをそのままバックオフして再試行します。

* run 作成のレート制限に達すると、それ以上のトレーニングはブロックされます。

```
HTTP 429: rate limit exceeded
```
