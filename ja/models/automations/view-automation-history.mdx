title: オートメーション の履歴を表示する
description: "ステータス、トリガーイベント、アクション結果を確認するために、W&B Automations の実行履歴を表示します。"
---

<Note>
オートメーションの実行履歴は、W&B Multi-tenant Cloud、W&B Dedicated Cloud、および W&B Self-Managed v0.75.0 以降で利用できます。
</Note>

このページでは、W&amp;B の [automations](/ja/models/automations) の実行履歴を表示して確認する方法について説明します。どのイベントが オートメーション をトリガーしたか、どのようなアクションが実行されたか、またその結果が成功したか失敗したかを確認できます。

実行された オートメーション ごとに、次の情報を含むレコードが生成されます。

* **実行 Timestamp**: オートメーション がトリガーされた日時。
* **トリガーイベント**: オートメーション をトリガーした特定のイベント。
* **ステータス**: 実行のステータス。[実行ステータス](#execution-status)を参照してください。
* **アクションの詳細**: Slack チャンネルへの通知や webhook の実行など、実行されたアクションに関する情報。
* **結果の詳細**: オートメーション の実行結果に関する追加情報 (ある場合) 。失敗した実行のエラー情報も含まれます。

ユースケースに応じて、オートメーション の履歴を表示する詳細な手順について **Registry** または **Project** タブを選択してください。

<Tabs>
  <Tab title="Registry">
    1. プロジェクトのサイドバーで **Registry** をクリックし、レジストリにアクセスします。
    2. リストからレジストリを選択します。
    3. **Automations** タブでレジストリの automations を表示します。**Last execution** の Timestamp をクリックすると、実行履歴の詳細を表示できます。検索バーを使用して オートメーション 名でフィルターしたり、最後にトリガーされた日付で並べ替えて最近実行された automations を見つけたりできます。
    4. **Automations history** タブでは、イベント、アクション、ステータスを含むレジストリの オートメーション 実行履歴を新しい順に表示できます。実行 Timestamp をクリックすると、特定の実行の詳細を表示できます。

    <Tip>collection に関連付けられた オートメーション 実行がある場合は、アイコン ![Collection automation execution symbol, a circle with a right-pointing arrow](/images/automations/collection-automation-execution-icon.png) と関連する実行数が表示されます。</Tip>
  </Tab>

  <Tab title="Project">
    1. プロジェクトのサイドバーで **Automations** タブをクリックします。プロジェクトの automations が表示されます。**Last execution** の Timestamp をクリックすると、実行履歴の詳細を表示できます。検索バーを使用して オートメーション 名でフィルターしたり、最後にトリガーされた日付で並べ替えて最近実行された automations を見つけたりできます。
    2. **History** タブでは、プロジェクトの automations のすべての実行を、最新の実行から順に新しい順で表示できます。各実行のメタデータ (イベント、アクション、ステータスを含む) が表示されます。実行 Timestamp をクリックすると、特定の実行の詳細を表示できます。
  </Tab>
</Tabs>

<div id="understanding-execution-details">
  ## 実行の詳細を確認する
</div>

各 オートメーション の実行は、次のいずれかのステータスになります。

* **Finished**: オートメーション はすべての action を正常に完了しました。
* **Failed**: オートメーション の実行中にエラーが発生し、正常に完了しませんでした。
* **Pending**: オートメーション は実行待ちとしてキューに入っています。

履歴内の任意の実行をクリックすると、詳細を確認できます。

* **イベントの詳細**: オートメーション をトリガーした具体的なイベント。内容は次のとおりです。
  * イベントタイプ (例: &quot;新しい artifact バージョン&quot;、&quot;run の完了&quot;)
  * entity 情報 (run ID、artifact 名など)
  * イベントをトリガーしたユーザー (該当する場合)

* **アクション の詳細**: オートメーション が実行しようとした内容に関する情報。
  * アクション タイプ (Slack 通知または webhook)
  * ターゲット (Slack チャンネルまたは webhook URL)
  * 送信されたペイロード (webhook の場合)

* **結果情報**:
  * Response ステータス (webhook の場合)
  * エラーメッセージまたはスタックトレース (失敗した実行の場合)
