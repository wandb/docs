<Note>
  オートメーションの実行履歴は、W&amp;B Multi-tenant Cloud、W&amp;B Dedicated Cloud、および W&amp;B Self-Managed v0.75.0 以降で利用できます。
</Note>

このページでは、W&amp;B の [automations](/ja/models/automations) の実行履歴を表示して理解する方法を説明します。実行履歴では、何がオートメーションをトリガーしたのか、どのアクションが実行されたのか、その結果が成功か失敗かを確認できます。オートメーション履歴を確認することで、オートメーションが想定どおりに実行されているかを確認し、失敗のトラブルシューティングを行い、どのイベントが後続のアクションをトリガーしたかを監査できます。

実行された各オートメーションでは、次の内容を含むレコードが生成されます。

* **実行タイムスタンプ**: オートメーションがトリガーされた時刻。
* **トリガーイベント**: オートメーションをトリガーした特定のイベント。
* **ステータス**: 実行のステータス。詳しくは [実行ステータス](#execution-status) を参照してください。
* **アクションの詳細**: Slack チャンネルへの通知や webhook の実行など、どのアクションが実行されたかに関する情報。
* **結果の詳細**: オートメーションの最終結果に関する追加情報 (ある場合) 。失敗した実行のエラーも含まれます。

オートメーション履歴は、Registry スコープと project スコープの両方のオートメーションで利用できます。詳しい手順については、**Registry** または **Project** タブを選択してください。

<Tabs>
  <Tab title="Registry">
    1. プロジェクトのサイドバーで **Registry** をクリックします。
    2. 一覧から対象の Registry を選択します。
    3. **Automations** タブにアクセスします。このタブには Registry のオートメーションが一覧表示されます。実行履歴の詳細を表示するには、**Last execution** のタイムスタンプをクリックします。検索バーを使用してオートメーション名でフィルターし、最後にトリガーされた日時で並べ替えると、最近実行されたオートメーションを検索できます。
    4. **Automations history** タブでは、Registry のオートメーション実行が新しい順に表示され、イベント、アクション、ステータスも確認できます。特定の実行の詳細を表示するには、実行タイムスタンプをクリックします。

    <Tip>コレクションに関連付けられたオートメーション実行がある場合は、アイコン ![コレクションのオートメーション実行を示す記号。右向き矢印のある円](/images/automations/collection-automation-execution-icon.png) と関連する実行数が表示されます。</Tip>
  </Tab>

  <Tab title="Project">
    1. project にアクセスします。
    2. プロジェクトのサイドバーで **Automations** タブをクリックします。このタブには project のオートメーションが一覧表示されます。実行履歴の詳細を表示するには、**Last execution** のタイムスタンプをクリックします。検索バーを使用してオートメーション名でフィルターし、最後にトリガーされた日時で並べ替えると、最近実行されたオートメーションを検索できます。
    3. **History** タブでは、project のオートメーションのすべての実行が新しい順に表示されます。各実行のメタデータには、イベント、アクション、ステータスが含まれます。特定の実行の詳細を表示するには、実行タイムスタンプをクリックします。
  </Tab>
</Tabs>

<div id="execution-details">
  ## 実行の詳細
</div>

履歴内で実行を見つけたら、次の情報を使用してそのステータスと結果を判断してください。

<div id="execution-status">
  ### 実行ステータス
</div>

各 オートメーション の実行は、次のいずれかのステータスになります。

* **Finished**: オートメーション はすべての アクション を正常に完了しました。
* **Failed**: オートメーション の実行中にエラーが発生し、正常に完了しませんでした。
* **Pending**: オートメーション は実行待ちとしてキューに入っています。

<div id="execution-metadata">
  ### 実行メタデータ
</div>

履歴内の任意の実行をクリックすると、次の詳細を確認できます。

* **イベントの詳細**: オートメーションをトリガーした特定のイベントに関する情報。内容は次のとおりです。
  * イベント タイプ (例: &quot;新しい artifact version&quot;、&quot;run の完了&quot;) 。
  * run ID や artifact 名などの entity 情報。
  * イベントをトリガーしたユーザー (該当する場合) 。
* **アクションの詳細**: オートメーションが実行しようとした内容に関する情報。内容は次のとおりです。
  * アクション タイプ (Slack 通知 または webhook) 。
  * ターゲット (Slack チャンネルまたは webhook URL) 。
  * 送信されたペイロード (webhook の場合) 。
* **結果情報**:
  * レスポンス ステータス (webhook の場合) 。
  * エラー メッセージまたはスタック トレース (実行が失敗した場合) 。
