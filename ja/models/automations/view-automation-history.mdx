---
title: オートメーションの履歴を表示する
description: "W&B Automations の実行履歴を表示して、ステータス、トリガーイベント、액션の結果を確認します。"
---

<Note>
  This feature requires an Enterprise license. It is available only for:

  * W&amp;B Multi-tenant Cloud
  * W&amp;B Dedicated Cloud
  * W&amp;B Self-Managed v0.75.0+
</Note>

このページでは、W&amp;B [automations](/ja/models/automations) の実行履歴を表示して内容を理解する方法を説明します。これには、オートメーションをトリガーした要因、実行された액션、その実行が成功したか失敗したかの確認が含まれます。

実行された各オートメーションでは、次の内容を含むレコードが生成されます。

* **Execution timestamp**: オートメーションがトリガーされた時刻。
* **Triggering event**: オートメーションをトリガーした特定のイベント。
* **Status**: 実行のステータス。[Execution status](#execution-status) を参照してください。
* **Action details**: Slack チャンネルへの通知や webhook の実行など、実行された액ションに関する情報。
* **Result details**: オートメーション実行の最終結果に関する追加情報 (ある場合) 。失敗した実行のエラーも含まれます。

ユースケースに応じて、オートメーション履歴の表示方法に関する詳細な手順を確認するには、**Registry** タブまたは **Project** タブを選択してください。

<Tabs>
  <Tab title="Registry">
    1. プロジェクトのサイドバーで **Registry** をクリックして、Registry にアクセスします。
    2. 一覧から対象の Registry を選択します。
    3. **Automations** タブで Registry のオートメーションを表示します。**Last execution** のタイムスタンプをクリックすると、実行履歴の詳細を表示できます。検索バーを使用してオートメーション名でフィルターしたり、最後にトリガーされた日付で並べ替えて最近実行されたオートメーションを探したりできます。
    4. **Automations history** タブでは、Registry のオートメーション実行が新しい順に表示され、イベント、액션、ステータスを確認できます。特定の実行の詳細を表示するには、その実行のタイムスタンプをクリックします。

    <Tip>コレクションに関連付けられたオートメーション実行がある場合は、関連する実行数とともにアイコン ![コレクションのオートメーション実行を表す記号（右向き矢印が描かれた円）](/images/automations/collection-automation-execution-icon.png) が表示されます。</Tip>
  </Tab>

  <Tab title="Project">
    1. プロジェクトのサイドバーで **Automations** タブをクリックします。プロジェクトのオートメーションが表示されます。**Last execution** のタイムスタンプをクリックすると、実行履歴の詳細を表示できます。検索バーを使用してオートメーション名でフィルターしたり、最後にトリガーされた日付で並べ替えて最近実行されたオートメーションを探したりできます。
    2. **History** タブでは、プロジェクトのオートメーションのすべての実行が最新のものから順に表示されます。各実行について、イベント、액ション、ステータスなどのメタデータが表示されます。特定の実行の詳細を表示するには、その実行のタイムスタンプをクリックします。
  </Tab>
</Tabs>

<div id="understanding-execution-details">
  ## 実行の詳細を理解する
</div>

各オートメーションの実行には、次のいずれかのステータスがあります。

* **Finished**: オートメーションがすべての 액션 を正常に完了しました。
* **Failed**: オートメーションでエラーが発生し、正常に終了しませんでした。
* **Pending**: オートメーションは実行待ちのキューに入っています。

履歴内の任意の実行をクリックすると、詳細を表示できます。

* **イベントの詳細**: オートメーションをトリガーした特定のイベント。内容は次のとおりです。
  * イベントタイプ (例: 「新しいアーティファクトバージョン」、「run 完了」)
  * entity 情報 (run ID、アーティファクト名など)
  * イベントをトリガーしたユーザー (該当する場合)

* **액션 の詳細**: オートメーションが実行しようとした内容に関する情報。
  * 액션 タイプ (Slack 通知または webhook)
  * ターゲット (Slack チャンネルまたは webhook URL)
  * 送信されたペイロード (webhook の場合)

* **結果情報**:
  * Response ステータス (webhook の場合)
  * エラーメッセージまたはスタックトレース (実行が失敗した場合)
