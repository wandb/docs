W&amp;B はデフォルトで、米国内にある非公開の Google Cloud Storage バケットに artifact ファイルを保存します。すべてのファイルは、保存時および転送時に暗号化されます。

機密性の高いファイルについては、[Private Hosting](/ja/platform/hosting/) を設定するか、[reference artifacts](/ja/models/artifacts/track-external-files/) を使用することを推奨します。

トレーニング中、W&amp;B はログ、artifact、設定ファイルをローカルの以下のディレクトリに保存します。

| ファイル                       | デフォルトの保存場所             | デフォルトの保存場所を変更するには次を設定します                    |
| -------------------------- | ---------------------- | ------------------------------------------- |
| ログ                         | `./wandb`              | `wandb.init()` の `dir` または `WANDB_DIR` 環境変数 |
| artifact                   | `~/.cache/wandb`       | `WANDB_CACHE_DIR` 環境変数                      |
| 設定                         | `~/.config/wandb`      | `WANDB_CONFIG_DIR` 環境変数                     |
| アップロード用にステージングされた artifact | `~/.cache/wandb-data/` | `WANDB_DATA_DIR` 環境変数                       |
| ダウンロードした artifact          | `./artifacts`          | `WANDB_ARTIFACT_DIR` 環境変数                   |

環境変数を使用して W&amp;B を設定する方法の詳細については、[environment variables reference](/ja/models/track/environment-variables/) を参照してください。

<Warning>`wandb` を初期化するマシンによっては、これらのデフォルトフォルダがファイルシステム上の書き込み可能な場所に存在しない場合があります。その結果、エラーが発生することがあります。</Warning>

<div id="clean-up-local-artifact-cache">
  ### ローカルのartifactキャッシュをクリーンアップする
</div>

W&amp;B は、共通ファイルを持つ複数バージョン間でのダウンロードを高速化するために、artifact ファイルをキャッシュします。時間が経つと、このキャッシュディレクトリが大きくなることがあります。キャッシュを整理し、最近使用されていないファイルを削除するには、[`wandb artifact cache cleanup`](/ja/models/ref/cli/wandb-artifact/wandb-artifact-cache/) コマンドを実行します。

次のコード例は、キャッシュサイズを 1GB に制限する方法を示しています。コード例をターミナルにコピー＆ペーストしてください。

```bash
$ wandb artifact cache cleanup 1GB
```
