description: external cloud storage bucket、HTTP ファイルサーバー、または NFS 共有に保存されたファイルを追跡します。
title: 外部ファイルを追跡する
---

*reference artifacts* を使用すると、W&amp;B サーバーの外部に保存されたファイルを追跡して利用できます。一般的な外部ストレージソリューションには、CoreWeave AI Object Storage、Amazon Simple Storage Service (Amazon S3) 버킷、GCS 버킷、Azure blob、HTTP ファイルサーバー、NFS 共有などがあります。

reference artifacts は、非参照アーティファクトと同様に動作します。主な違いは、reference artifacts がファイルサイズや MD5 チェックサムなど、ファイルに関するメタデータのみで構成されている点です。ファイル自体がシステムの外部に出ることはありません。

reference artifacts は、非参照アーティファクトと同様に操作できます。W&amp;B App では、ファイルブラウザーを使って reference artifacts の内容を閲覧し、完全な依存関係グラフを確認し、アーティファクトのバージョン管理された履歴を確認できます。ただし、データ自体はアーティファクトに含まれていないため、UI では画像やオーディオなどのリッチメディアを表示できません。

<Note>
  外部ファイルを追跡しないアーティファクトをログすると、W&amp;B はアーティファクトのファイルを W&amp;B サーバーに保存します。これは、W&amp;B Python SDK でアーティファクトをログするときのデフォルトの動作です。

  外部ファイルを追跡するアーティファクトをログすると、W&amp;B はオブジェクトの ETag やサイズなど、オブジェクトに関するメタデータをログします。버킷でオブジェクトのバージョン管理が有効になっている場合は、バージョン ID もログされます。
</Note>

以下のセクションでは、外部 reference artifacts を追跡する方法について説明します。

<div id="track-an-artifact-in-an-external-bucket">
  ## external cloud storage bucket 内のアーティファクトをトラッキングする
</div>

W&amp;B Python SDK を使用して、W&amp;B の外部に保存されているファイルへの参照をトラッキングします。

1. `wandb.init()` で run を初期化します。
2. `wandb.Artifact()` でアーティファクト object を作成します。
3. アーティファクト object の `wandb.Artifact.add_reference()` method を使用して、버킷 パスへの参照を指定します。
4. `run.log_artifact()` でアーティファクトのメタデータをログします。

```python
import wandb

# W&B の run を初期化する
with wandb.init(project="my-project") as run:

  # アーティファクトオブジェクトを作成する
  artifact = wandb.Artifact(name="name", type="type")

  # バケットパスへの参照を追加する
  artifact.add_reference(uri = "uri/to/your/bucket/path")

  # アーティファクトのメタデータをログする
  run.log_artifact(artifact)
```

例として、使用している버킷に次のディレクトリ構造があるとします:

```text
s3://my-bucket

|datasets/
  |-- mnist/
|models/
  |-- cnn/
```

`datasets/mnist/` ディレクトリには、画像のコレクションが含まれています。画像の `datasets/mnist/` ディレクトリをデータセットのアーティファクトとしてトラッキングするには、次のように指定します。

1. `"mnist"` など、アーティファクトの名を指定します。
2. アーティファクト object を構築する際に、`type` パラメーターを `"dataset"` に設定します (`wandb.Artifact(type="dataset")`) 。
3. `wandb.Artifact.add_reference()` を呼び出すときに、`datasets/mnist/` ディレクトリへのパスを Amazon S3 URI (`s3://my-bucket/datasets/mnist/`) として指定します。
4. `run.log_artifact()` でアーティファクトをログします。

次のコードサンプルは、reference artifact `mnist:latest` を作成します。

```python
import wandb

with wandb.init(project="my-project") as run:
  artifact = wandb.Artifact(name="mnist", type="dataset")
  artifact.add_reference(uri="s3://my-bucket/datasets/mnist")
  run.log_artifact(artifact)
```

W&amp;B App 内では、ファイルブラウザーを使用してreference artifactの内容を確認したり、[完全な依存関係グラフを探索したり](/ja/models/artifacts/explore-and-traverse-an-artifact-graph/)、アーティファクトのバージョン管理された履歴を確認したりできます。W&amp;B App では、データ自体がアーティファクト内に含まれていないため、画像やオーディオなどのリッチメディアは表示されません。

<Note>
  W&amp;B Artifacts は、CoreWeave Storage や MinIO を含む、Amazon S3 互換のあらゆるインターフェースをサポートしています。以下で説明するスクリプトは、`AWS_S3_ENDPOINT_URL` 環境変数で CoreWeave Storage または MinIO サーバーを指すように設定すると、どちらの提供元でもそのまま動作します。
</Note>

<Warning>
  デフォルトでは、W&amp;B はオブジェクトプレフィックスを追加するときに 10,000 オブジェクトの制限を課します。`wandb.Artifact.add_reference()` を呼び出す際に `max_objects=` を指定することで、この制限を調整できます。
</Warning>

<div id="download-an-artifact-from-an-external-bucket">
  ## external cloud storage bucketからアーティファクトをダウンロードする
</div>

W&amp;B は、アーティファクトがログされた際に記録されたメタデータを使用して reference artifact をダウンロードするとき、基盤となる버킷からファイルを取得します。버킷でオブジェクトのバージョン管理が有効になっている場合、W&amp;B は、アーティファクトがログされた時点でのファイルの状態に対応するオブジェクトバージョンを取得します。버킷の内容を更新し続けても、アーティファクトはトレーニング run 中の버킷のスナップショットとして機能するため、特定のモデルがトレーニングされたデータの正確なバージョンを常に参照できます。

次のコードサンプルは、reference artifact をダウンロードする方法を示しています。アーティファクトをダウンロードするための API は、reference artifact と非 reference artifact の両方で共通です。

```python
import wandb

with wandb.init(project="my-project") as run:
  artifact = run.use_artifact("mnist:latest", type="dataset")
  artifact_dir = artifact.download()
```

<Note>
  ワークフローの一環としてファイルを上書きする場合は、ストレージ 버킷で「Object Versioning」を有効にすることを W&amp;B は推奨します。

  バージョニングが有効になっていれば、アーティファクトをダウンロードする際に、そのアーティファクトをログしてからファイルが上書きされていたとしても、W&amp;B は常に正しいバージョンのファイルを取得できます。

  ユースケースに応じて、オブジェクトのバージョン管理を有効にするための手順については、[AWS](https://docs.aws.amazon.com/AmazonS3/latest/userguide/manage-versioning-examples.html)、[Google Cloud](https://cloud.google.com/storage/docs/using-object-versioning#set)、[Azure](https://learn.microsoft.com/azure/storage/blobs/versioning-enable) のドキュメントを参照してください。
</Note>

<div id="add-and-download-an-external-from-a-bucket">
  ## 버킷 から external を追加してダウンロードする
</div>

次のコードサンプルでは、データセットを Amazon S3 버킷 にアップロードし、reference artifactでトラッキングしてから、それをダウンロードします。

```python
import boto3
import wandb

with wandb.init() as run:
  # トレーニングはここで...

  s3_client = boto3.client("s3")
  s3_client.upload_file(file_name="my_model.h5", bucket="my-bucket", object_name="models/cnn/my_model.h5")

  # モデルアーティファクトをログする
  model_artifact = wandb.Artifact("cnn", type="model")
  model_artifact.add_reference("s3://my-bucket/models/cnn/")
  run.log_artifact(model_artifact)
```

後で、モデル アーティファクトをダウンロードできます。アーティファクトの名とそのタイプを指定してください:

```python
import wandb

with wandb.init() as run:
  artifact = run.use_artifact(artifact_or_name = "cnn", type="model")
  datadir = artifact.download()
```

<Note>
  Google Cloud または Azure で参照によってアーティファクトをトラッキングする方法のエンドツーエンドのウォークスルーについては、以下のReportsをご覧ください。

  * [Google Cloudで参照によってArtifactsをトラッキングするためのガイド](https://wandb.ai/stacey/artifacts/reports/Tracking-Artifacts-by-Reference--Vmlldzo1NDMwOTE)
  * [Microsoft Azureでreference artifactを扱う](https://wandb.ai/andrea0/azure-2023/reports/Efficiently-Harnessing-Microsoft-Azure-Blob-Storage-with-Weights-Biases--Vmlldzo0NDA2NDgw)
</Note>

<div id="cloud-storage-credentials">
  ## クラウドストレージの認証情報
</div>

W&amp;B は、使用しているクラウドプロバイダーに応じたデフォルトの方法で認証情報を検出します。使用される認証情報の詳細については、各クラウドプロバイダーのドキュメントを参照してください。

| クラウドプロバイダー                  | 認証情報のドキュメント                                                                                                                                                 |
| --------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| CoreWeave AI Object Storage | [CoreWeave AI Object Storage documentation](https://docs.coreweave.com/docs/products/storage/object-storage/how-to/manage-access-keys/cloud-console-tokens) |
| AWS                         | [Boto3 documentation](https://boto3.amazonaws.com/v1/documentation/api/latest/guide/credentials.html#configuring-credentials)                               |
| Google Cloud                | [Google Cloud documentation](https://cloud.google.com/docs/authentication/provide-credentials-adc)                                                          |
| Azure                       | [Azure documentation](https://learn.microsoft.com/python/api/azure-identity/azure.identity.defaultazurecredential?view=azure-python)                        |

AWS の場合、버킷が設定済みユーザーのデフォルトリージョンとは異なるリージョンにあるときは、버킷のリージョンに合わせて `AWS_REGION` 環境変数を設定する必要があります。

<Warning>
  画像、オーディオ、動画、ポイントクラウドなどのリッチメディアは、버킷の CORS 設定によっては App UI に表示されない場合があります。버킷の CORS 設定で **app.wandb.ai** を許可リストに追加すると、W&amp;B App でこのようなリッチメディアを正しく表示できるようになります。

  画像、オーディオ、動画、ポイントクラウドなどのリッチメディアが App UI に表示されない場合は、버킷の CORS ポリシーで `app.wandb.ai` が許可リストに追加されていることを確認してください。
</Warning>

<div id="track-an-artifact-in-a-filesystem">
  ## ファイルシステムでアーティファクトをトラッキングする
</div>

データセットに액세스する一般的なパターンは、トレーニング ジョブを実行しているすべてのマシン上で、リモート ファイルシステムへの NFS マウント ポイントを公開することです。これはクラウドストレージ 버킷 の代替ソリューションになり得ます。トレーニング スクリプトの観点では、ファイルがファイルシステム上のローカル ファイルとして見えるためです。

{/* マウントされているかどうかにかかわらず、W&B Artifacts を使用してファイルシステムへの参照をトラッキングできます。 */}

ファイルシステム内のアーティファクトをトラッキングするには:

1. `wandb.init()` で run を初期化します。
2. `wandb.Artifact()` でアーティファクト オブジェクトを作成します。
3. アーティファクト オブジェクトの `wandb.Artifact.add_reference()` method を使用して、ファイルシステム パスへの参照を指定します。
4. `run.log_artifact()` を使用してアーティファクトのメタデータをログします。

マウントされたファイルシステム内のファイルをトラッキングするには、次の 코드 스니펫 をコピー＆ペーストしてください。山かっこ (`< >`) で囲まれた値は、ご自身の値に置き換えてください。

```python
import wandb

# runを初期化する
with wandb.init(entity="<entity>", project="<project>") as run:

  # アーティファクトオブジェクトを作成する
  artifact = wandb.Artifact(name="<name>", type="<type>")

  # ファイルシステムパスへの参照を追加する
  artifact.add_reference("file:///path/to/dataset/")

  # アーティファクトをログする（メタデータのみ）
  run.log_artifact(artifact)
```

URL にスラッシュが 3 つあることに注意してください。最初のコンポーネントは、ファイルシステム参照を使用することを示す `file://` プレフィックスです。2 番目のコンポーネントは、ファイルシステムのルート `/` です。残りのコンポーネントは、トラッキングしたいディレクトリまたはファイルへのパスです。

예시として、`/mount` に次の構造のファイルシステムがマウントされているとします。

```text
mount
|datasets/
  |-- mnist/
|models/
  |-- cnn/
```

`datasets/mnist/` ディレクトリをデータセット アーティファクトとしてトラッキングするには、次の코드 스니펫を使用できます。

```python
import wandb

with wandb.init() as run:
  artifact = wandb.Artifact("mnist", type="dataset")
  artifact.add_reference("file:///mount/datasets/mnist/")
  run.log_artifact(artifact)
```

これにより、`/mount/datasets/mnist/` の下に保存されているファイルを指すreference artifact `mnist:latest` が作成されます。

<Warning>
  デフォルトでは、W&amp;B はディレクトリへの参照を追加する際に 10,000 ファイルの上限を設けています。`wandb.Artifact.add_reference()` を呼び出すときに `max_objects=` を指定することで、この上限を調整できます。
</Warning>

同様に、`models/cnn/my_model.h5` に保存されているモデルをトラッキングするには、次の코드 스니펫を使用できます。

```python
import wandb

with wandb.init() as run:

  # ここでトレーニング...

  # モデルをディスクに書き込む

  # アーティファクトオブジェクトを作成する
  model_artifact = wandb.Artifact("cnn", type="model")

  # モデルファイルパスへの参照を追加する
  model_artifact.add_reference("file:///mount/cnn/my_model.h5")

  # アーティファクトをW&Bにログする
  run.log_artifact(model_artifact)
```

<div id="download-an-artifact-from-an-external-filesystem">
  ## 外部ファイルシステムからアーティファクトをダウンロードする
</div>

参照先ファイルシステムからファイルをダウンロードするには、参照ではないアーティファクトと同じ API を使用します。

1. `wandb.init()` で run を初期化します。
2. ダウンロードするアーティファクトを指定するには、`wandb.Run.use_artifact()` method を使用します。
3. アーティファクトの `wandb.Artifact.download()` method を呼び出して、参照先ファイルシステムからファイルをダウンロードします

```python
with wandb.init() as run:
  artifact = run.use_artifact("entity/project/mnist:latest", type="dataset")
  artifact_dir = artifact.download()
```

W&amp;B は `/mount/datasets/mnist` の内容を `artifacts/mnist:v0/` ディレクトリにコピーします。

<Info>
  アーティファクトを再構築できない場合、`Artifact.download()` はエラーをスローします。たとえば、アーティファクトに上書きされたファイルへの参照が含まれていると、アーティファクトを再構築できなくなるため、`Artifact.download()` はエラーをスローします。
</Info>

{/* ### run の外でアーティファクトをログする

  W&B では、run の外でアーティファクトをログすると、run が作成されます。各アーティファクトは run に属し、その run はプロジェクトに属します。アーティファクト（バージョン）はコレクションにも属し、タイプを持ちます。

  W&B run の外でアーティファクトを W&B Server にアップロードするには、[`wandb artifact put`](/models/ref/cli/wandb-artifact/wandb-artifact-put) コマンドを使用します。アーティファクトを所属させるプロジェクトの名と、アーティファクトの名（`project/artifact_name`）を指定してください。必要に応じて、タイプ（`TYPE`）も指定できます。以下の코드 스니펫内の `PATH` は、アップロードするアーティファクトのファイルパスに置き換えてください。

  ```bash
  $ wandb artifact put --name project/artifact_name --type TYPE PATH
  ```

  指定したプロジェクトが存在しない場合、W&B は新しいプロジェクトを作成します。アーティファクトのダウンロード方法については、[Download and use artifacts](/models/artifacts/download-and-use-an-artifact) を参照してください。 */}
