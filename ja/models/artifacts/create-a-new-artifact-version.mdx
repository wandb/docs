---
description: 単一の run または分散プロセスから新しい artifact バージョンを作成します。
title: artifact バージョンを作成する
---

単一の [run](/ja/models/runs/) から、または分散 run と連携して、新しい artifact バージョンを作成できます。必要に応じて、[段階的 artifact](#create-a-new-artifact-version-from-an-existing-version) と呼ばれる既存のバージョンから新しい artifact バージョンを作成することもできます。

<Note>
  artifact 内の一部のファイルに変更を適用する必要があり、元の artifact のサイズが大幅に大きい場合は、段階的 artifact を作成することをおすすめします。
</Note>

<div id="create-new-artifact-versions-from-scratch">
  ## 新しいartifactバージョンを最初から作成する
</div>

新しいartifactバージョンを作成する方法は 2 つあります。単一の run から作成する方法と、分散 run から作成する方法です。それぞれ次のとおりです。

* **Single run**: 1 つの run が、新しいバージョンに必要なデータをすべて提供します。これは最も一般的なケースで、run が必要なデータを完全に再現する場合に最適です。たとえば、保存済みモデルを出力したり、分析用にモデルの予測を表として出力したりする場合です。
* **Distributed runs**: 複数の run の集合が、まとめて新しいバージョンに必要なデータをすべて提供します。これは、複数の run が多くの場合並列でデータを生成する分散ジョブに最適です。たとえば、分散方式でモデルを評価し、予測を出力する場合です。

`wandb.Artifact` API にプロジェクト内に存在しない名を渡すと、W&amp;B は新しいartifactを作成し、`v0` alias を割り当てます。同じartifactに対して再度ログすると、W&amp;B は内容のチェックサムを計算します。artifactが変更されていた場合、W&amp;B は新しいバージョン `v1` を保存します。

`wandb.Artifact` API に、プロジェクト内の既存のartifactに一致する名と artifact type を渡すと、W&amp;B は既存のartifactを取得します。取得されたartifactのバージョンは 1 より大きくなります。

<div id="single-run">
  ### 単一の run
</div>

artifact 内のすべてのファイルを生成する 1 つの run で、Artifact の新しいバージョンをログします。これは、1 つの run が artifact 内のすべてのファイルを生成する場合に該当します。

ユースケースに応じて、以下のタブから 1 つ選択し、run の内側または外側で新しい artifact バージョンを作成します。

<Tabs>
  <Tab title="run 内">
    W&amp;B run 内で artifact バージョンを作成します。

    1. `wandb.init` で run を作成します。
    2. `wandb.Artifact` を使用して、新しい artifact を作成するか既存の artifact を取得します。
    3. `.add_file` で artifact にファイルを追加します。
    4. `.log_artifact` で artifact を run にログします。

    ```python
    with wandb.init() as run:
        artifact = wandb.Artifact("artifact_name", "artifact_type")

        # `.add`、`.add_file`、`.add_dir`、`.add_reference` を使用して
        # artifact にファイルとアセットを追加
        artifact.add_file("image1.png")
        run.log_artifact(artifact)
    ```
  </Tab>

  <Tab title="run 外">
    W&amp;B run の外で artifact バージョンを作成します。

    1. `wanb.Artifact` を使用して、新しい artifact を作成するか既存の artifact を取得します。
    2. `.add_file` で artifact にファイルを追加します。
    3. `.save` で artifact を保存します。

    ```python
    artifact = wandb.Artifact("artifact_name", "artifact_type")
    # `.add`、`.add_file`、`.add_dir`、`.add_reference` を使用して
    # artifact にファイルとアセットを追加
    artifact.add_file("image1.png")
    artifact.save()
    ```
  </Tab>
</Tabs>

<div id="distributed-runs">
  ### 分散 run
</div>

複数の run が、コミット前に同じバージョンに対して共同で作業できるようにします。これは、前述の 単一の run モード (1 つの run が新しいバージョンのすべてのデータを提供する方式) とは対照的です。

<Note>
  1. collection 内の各 run は、同じバージョンで共同作業するために、同じ一意の ID (`distributed_id`) を認識している必要があります。デフォルトでは、存在する場合、W&amp;B は `wandb.init(group=GROUP)` で設定した run の `group` を `distributed_id` として使用します。
  2. バージョンを「コミット」し、その状態を永続的に固定する最後の run が必要です。
  3. 共同の artifact への追加には `upsert_artifact` を使用し、コミットを確定するには `finish_artifact` を使用します。
</Note>

次の例を見てみましょう。異なる run (以下では **Run 1**、**Run 2**、**Run 3** と表記) が、`upsert_artifact` を使って同じ artifact にそれぞれ異なる画像ファイルを追加します。

<div id="run-1">
  #### Run 1
</div>

```python
with wandb.init() as run:
    artifact = wandb.Artifact("artifact_name", "artifact_type")
    # `.add`、`.add_file`、`.add_dir`、`.add_reference` を使用して
    # ファイルとアセットをartifactに追加する
    artifact.add_file("image1.png")
    run.upsert_artifact(artifact, distributed_id="my_dist_artifact")
```

<div id="run-2">
  #### Run 2
</div>

```python
with wandb.init() as run:
    artifact = wandb.Artifact("artifact_name", "artifact_type")
    # `.add`、`.add_file`、`.add_dir`、`.add_reference` を使用して
    # ファイルとアセットをartifactに追加する
    artifact.add_file("image2.png")
    run.upsert_artifact(artifact, distributed_id="my_dist_artifact")
```

<div id="run-3">
  #### Run 3
</div>

Run 1 と Run 2 の完了後に実行する必要があります。`finish_artifact` を呼び出す Run では artifact にファイルを含めることもできますが、必須ではありません。

```python
with wandb.init() as run:
    artifact = wandb.Artifact("artifact_name", "artifact_type")
    # ファイルとアセットをartifactに追加する
    # `.add`, `.add_file`, `.add_dir`, `.add_reference`
    artifact.add_file("image3.png")
    run.finish_artifact(artifact, distributed_id="my_dist_artifact")
```

<div id="create-a-new-artifact-version-from-an-existing-version">
  ## 既存のバージョンから新しいartifactバージョンを作成する
</div>

変更のないファイルを再インデックスせずに、以前のartifactバージョンのファイルの一部を追加、変更、または削除できます。以前のartifactバージョンのファイルの一部を追加、変更、または削除すると、*段階的artifact* と呼ばれる新しいartifactバージョンが作成されます。

<Frame>
  <img src="/images/artifacts/incremental_artifacts.png" alt="アーティファクトの段階的なバージョン管理" />
</Frame>

発生し得る各タイプの段階的な変更について、代表的なシナリオを紹介します。

* 追加: 新しいバッチを収集するたびに、データセットに新しいファイルのサブセットを定期的に追加します。
* 削除: 重複するファイルがいくつか見つかり、それらをアーティファクトから削除したい場合。
* 更新: 一部のファイルのアノテーションを修正し、古いファイルを正しいファイルに置き換えたい場合。

段階的なartifactと同じ機能を実現するために、artifactをゼロから作成することもできます。ただし、artifactをゼロから作成する場合は、artifactのすべての内容をローカルディスク上に用意する必要があります。段階的な変更を行う場合は、以前のartifactバージョンのファイルを変更せずに、単一のファイルを追加、削除、または変更できます。

<Note>
  単一のrun内で、または複数のrun (分散モード) を使用して、段階的なartifactを作成できます。
</Note>

以下の手順に従って、artifactを段階的に変更してください。

1. 段階的な変更を加える対象のartifactバージョンを取得します:

<Tabs>
  <Tab title="run 内">
    ```python
    saved_artifact = run.use_artifact("my_artifact:latest")
    ```
  </Tab>

  <Tab title="run の外">
    ```python
    client = wandb.Api()
    saved_artifact = client.artifact("my_artifact:latest")
    ```
  </Tab>
</Tabs>

2. 以下の内容でドラフトを作成します：

```python
draft_artifact = saved_artifact.new_draft()
```

3. 次のバージョンに反映したい段階的な変更を行います。エントリは追加、削除、または変更できます。

各変更の実行例については、いずれかのタブを選択してください。

<Tabs>
  <Tab title="追加">
    `add_file` method を使用して、既存の artifact バージョンにファイルを追加します。

    ```python
    draft_artifact.add_file("file_to_add.txt")
    ```

    <Note>
      `add_dir` method でディレクトリを追加すると、複数のファイルを追加することもできます。
    </Note>
  </Tab>

  <Tab title="削除">
    `remove` method を使用して、既存の artifact バージョンからファイルを削除します。

    ```python
    draft_artifact.remove("file_to_remove.txt")
    ```

    <Note>
      ディレクトリパスを渡すことで、`remove` method を使用して複数のファイルを削除することもできます。
    </Note>
  </Tab>

  <Tab title="変更">
    ドラフトから古い内容を削除し、新しい内容を追加し直すことで、内容を変更または置き換えます。

    ```python
    draft_artifact.remove("modified_file.txt")
    draft_artifact.add_file("modified_file.txt")
    ```
  </Tab>
</Tabs>

{/* <Note>
  artifact の追加と変更で使用する method は同じです。すでに存在するエントリに対してファイル名を渡すと、エントリは複製されず、置き換えられます。
  </Note> */}

4. 最後に、変更内容をログするか保存します。以下のタブでは、W&amp;B run 内と run 外で変更内容を保存する方法を示しています。ユースケースに適したタブを選択してください。

<Tabs>
  <Tab title="run 内で">
    ```python
    run.log_artifact(draft_artifact)
    ```
  </Tab>

  <Tab title="run 外で">
    ```python
    draft_artifact.save()
    ```
  </Tab>
</Tabs>

以上をまとめると、上記のコード例は次のようになります。

<Tabs>
  <Tab title="run 内">
    ```python
    with wandb.init(job_type="modify dataset") as run:
        saved_artifact = run.use_artifact(
            "my_artifact:latest"
        )  # artifact を取得し、run への入力として使用します
        draft_artifact = saved_artifact.new_draft()  # ドラフト版を作成します

        # ドラフト版のファイルの一部を変更します
        draft_artifact.add_file("file_to_add.txt")
        draft_artifact.remove("dir_to_remove/")
        run.log_artifact(
            draft_artifact
        )  # 変更をログして新しいバージョンを作成し、run の出力としてマークします
    ```
  </Tab>

  <Tab title="run 外">
    ```python
    client = wandb.Api()
    saved_artifact = client.artifact("my_artifact:latest")  # artifact を読み込みます
    draft_artifact = saved_artifact.new_draft()  # ドラフト版を作成します

    # ドラフト版のファイルの一部を変更します
    draft_artifact.remove("deleted_file.txt")
    draft_artifact.add_file("modified_file.txt")
    draft_artifact.save()  # ドラフトへの変更をコミットします
    ```
  </Tab>
</Tabs>
