---
description: 単一の run または分散プロセスから新しいアーティファクトバージョンを新規作成します。
title: アーティファクトバージョンの作成
---

単一の [run](/ja/models/runs/) を使用するか、分散された run を共同で使用して、新しいアーティファクトバージョンを新規作成します。オプションとして、[段階的アーティファクト](#create-a-new-artifact-version-from-an-existing-version) と呼ばれる前のバージョンから、新しいアーティファクトバージョンを新規作成することもできます。

<Note>
  アーティファクト内のファイルのサブセットに変更を適用する必要があり、元の アーティファクトのサイズが大幅に大きい場合に、段階的アーティファクトを作成することをおすすめします。
</Note>

<div id="create-new-artifact-versions-from-scratch">
  ## 新しいアーティファクト バージョンを一から作成する
</div>

新しいアーティファクト バージョンを作成する方法は2つあります。単一の runからと、分散 runからです。それぞれ次のとおりです。

* **単一の run**: 1つのrunが、新しいバージョンに必要なすべてのデータを提供します。これは最も一般的なケースで、runが必要なデータを完全に再作成する場合に最適です。たとえば、保存したモデルやモデルの予測結果を分析用の表として出力する場合です。
* **分散 run**: 複数のrunの集合が、まとめて新しいバージョンに必要なすべてのデータを提供します。これは、複数のrunが並列にデータを生成する分散ジョブに最適です。たとえば、分散方式でモデルを評価し、予測結果を出力する場合です。

`wandb.Artifact` APIに、プロジェクト内に存在しない名前を渡すと、W&amp;Bは新しいアーティファクトを作成し、`v0`のエイリアスを割り当てます。同じアーティファクトに再度ログすると、W&amp;Bは内容のチェックサムを計算します。アーティファクトに変更があれば、W&amp;Bは新しいバージョン`v1`を保存します。

`wandb.Artifact` APIに、プロジェクト内の既存のアーティファクトと一致する名前とアーティファクトタイプを渡した場合、W&amp;Bは既存のアーティファクトを取得します。取得されたアーティファクトのバージョンは1より大きくなります。

<div id="single-run">
  ### 単一の run
</div>

アーティファクト内のすべてのファイルを生成する単一の run で、アーティファクトの新しいバージョンをログします。このケースは、1 つの run がアーティファクト内のすべてのファイルを生成する場合に発生します。

使用ケースに応じて、以下のタブのいずれかを選択して、run の内側または外側で新しいアーティファクト バージョンを生成してください：

<Tabs>
  <Tab title="run 内で">
    W&amp;B run 内でアーティファクト バージョンを生成します：

    1. `wandb.init` で run を作成します。
    2. `wandb.Artifact` で新しいアーティファクトを作成するか、既存のものを取得します。
    3. `.add_file` でファイルをアーティファクトに追加します。
    4. `.log_artifact` でアーティファクトを run にログします。

    ```python
    with wandb.init() as run:
        artifact = wandb.Artifact("artifact_name", "artifact_type")

        # Add Files and Assets to the artifact using
        # `.add`, `.add_file`, `.add_dir`, and `.add_reference`
        artifact.add_file("image1.png")
        run.log_artifact(artifact)
    ```
  </Tab>

  <Tab title="run の外で">
    W&amp;B run の外でアーティファクト バージョンを生成します：

    1. `wandb.Artifact` で新しいアーティファクトを作成するか、既存のものを取得します。
    2. `.add_file` でファイルをアーティファクトに追加します。
    3. `.save` でアーティファクトを保存します。

    ```python
    artifact = wandb.Artifact("artifact_name", "artifact_type")
    # Add Files and Assets to the artifact using
    # `.add`, `.add_file`, `.add_dir`, and `.add_reference`
    artifact.add_file("image1.png")
    artifact.save()
    ```
  </Tab>
</Tabs>

<div id="distributed-runs">
  ### 分散 run
</div>

このモードでは、コミット前のバージョンに対して複数の run のコレクションで共同作業できるようにします。これは、1 つの run が新しいバージョン用のすべてのデータを提供する、上で説明した単一の run モードとは対照的です。

<Note>
  1. コレクション内の各 run は、同じ一意の ID (`distributed_id` と呼ばれます) を認識している必要があり、これにより同じバージョンで共同作業できます。デフォルトでは、その値が存在する場合、W&amp;B は run の `group` (`wandb.init(group=GROUP)` で設定) を `distributed_id` として使用します。
  2. 最終的にバージョンを「コミット」して、その状態を永久にロックする run が 1 つ必要です。
  3. 共同作業用のアーティファクトに追加するには `upsert_artifact` を使用し、コミットを確定するには `finish_artifact` を使用します。
</Note>

次の例を見てみましょう。異なる run (以下で **Run 1**、**Run 2**、**Run 3** と表記) が、`upsert_artifact` を使って同じアーティファクトにそれぞれ異なる画像ファイルを追加します。

<div id="run-1">
  #### Run 1
</div>

```python
with wandb.init() as run:
    artifact = wandb.Artifact("artifact_name", "artifact_type")
    # アーティファクトにファイルとアセットを追加するには
    # `.add`、`.add_file`、`.add_dir`、`.add_reference` を使用します
    artifact.add_file("image1.png")
    run.upsert_artifact(artifact, distributed_id="my_dist_artifact")
```

<div id="run-2">
  #### Run 2
</div>

```python
with wandb.init() as run:
    artifact = wandb.Artifact("artifact_name", "artifact_type")
    # `.add`、`.add_file`、`.add_dir`、`.add_reference` を使用して
    # アーティファクトにファイルとアセットを追加する
    artifact.add_file("image2.png")
    run.upsert_artifact(artifact, distributed_id="my_dist_artifact")
```

<div id="run-3">
  #### Run 3
</div>

Run 1 と Run 2 が完了した後に実行する必要があります。`finish_artifact` を呼び出す run では、アーティファクトにファイルを含めることができますが、含める必要はありません。

```python
with wandb.init() as run:
    artifact = wandb.Artifact("artifact_name", "artifact_type")
    # アーティファクトにファイルとアセットを追加する
    # `.add`、`.add_file`、`.add_dir`、`.add_reference`
    artifact.add_file("image3.png")
    run.finish_artifact(artifact, distributed_id="my_dist_artifact")
```

<div id="create-a-new-artifact-version-from-an-existing-version">
  ## 既存のバージョンから新しいアーティファクトのバージョンを新規作成する
</div>

変更されていないファイルを再インデックスすることなく、以前のアーティファクト バージョンに含まれる一部のファイルを追加、変更、または削除できます。以前のアーティファクト バージョンに含まれる一部のファイルを追加、変更、または削除すると、&#95;段階的アーティファクト&#95;と呼ばれる新しいアーティファクト バージョンが作成されます。

<Frame>
  <img src="/images/artifacts/incremental_artifacts.png" alt="段階的アーティファクトのバージョン管理" />
</Frame>

発生する可能性のある段階的な変更の各タイプのシナリオを以下に示します。

* add: 新しいバッチを収集するたびに、データセットにファイルの新しいサブセットを追加します。
* remove: いくつかの重複ファイルを見つけ、それらをアーティファクトから削除したい場合。
* update: 一部のファイルのアノテーションを修正したので、古いファイルを正しいものに置き換えたい場合。

段階的な Artifacts と同じ機能を実現するために、Artifacts をゼロから作成することもできます。ただし、ゼロから作成する場合は、Artifacts のすべての内容をローカルディスク上に用意する必要があります。段階的な変更を行う場合は、以前の Artifacts バージョンのファイルを変更せずに、1 つのファイルを追加、削除、または変更できます。

<Note>
  単一の run 内で、または複数の run を使用した分散モードで、段階的なアーティファクトを作成できます。
</Note>

アーティファクトを段階的に変更するには、以下の手順に従ってください：

1. 段階的な変更を加えるアーティファクトバージョンを取得します：

<Tabs>
  <Tab title="run 内で">
    ```python
    saved_artifact = run.use_artifact("my_artifact:latest")
    ```
  </Tab>

  <Tab title="run の外で">
    ```python
    client = wandb.Api()
    saved_artifact = client.artifact("my_artifact:latest")
    ```
  </Tab>
</Tabs>

2. 次のようにドラフトを作成します：

```python
draft_artifact = saved_artifact.new_draft()
```

3. 次のバージョンに反映したい段階的な変更を行います。既存のエントリの追加、削除、または修正を行えます。

これらの各変更の実行方法の例については、いずれかのタブを選択してください：

<Tabs>
  <Tab title="追加">
    既存のアーティファクトのバージョンにファイルを追加するには、`add_file` method を使用します：

    ```python
    draft_artifact.add_file("file_to_add.txt")
    ```

    <Note>
      また、`add_dir` method でディレクトリを追加すると、複数のファイルを一度に追加することもできます。
    </Note>
  </Tab>

  <Tab title="削除">
    既存のアーティファクトのバージョンからファイルを削除するには、`remove` method を使用します：

    ```python
    draft_artifact.remove("file_to_remove.txt")
    ```

    <Note>
      `remove` method にディレクトリのパスを渡すと、複数のファイルを一度に削除することもできます。
    </Note>
  </Tab>

  <Tab title="変更">
    ドラフトから古い内容を削除して、新しい内容を再度追加することで、内容を変更または置き換えられます：

    ```python
    draft_artifact.remove("modified_file.txt")
    draft_artifact.add_file("modified_file.txt")
    ```
  </Tab>
</Tabs>

{/* <Note>
  アーティファクトを追加または変更するための method は同じです。既存のエントリ用のファイル名を渡すと、エントリは複製されるのではなく置き換えられます。
  </Note> */}

4. 最後に、変更内容をログするか保存してください。以下のタブは、W&amp;B run 内外で変更内容を保存する方法を示しています。ユースケースに適したタブを選択してください：

<Tabs>
  <Tab title="run 内で">
    ```python
    run.log_artifact(draft_artifact)
    ```
  </Tab>

  <Tab title="run の外で">
    ```python
    draft_artifact.save()
    ```
  </Tab>
</Tabs>

これらをすべてまとめると、上記のコード例は次のようになります：

<Tabs>
  <Tab title="run 内で">
    ```python
    with wandb.init(job_type="modify dataset") as run:
        saved_artifact = run.use_artifact(
            "my_artifact:latest"
        )  # アーティファクトを取得し、それを run の入力として使用する
        draft_artifact = saved_artifact.new_draft()  # ドラフト版を作成する

        # ドラフト版内の一部のファイルを変更する
        draft_artifact.add_file("file_to_add.txt")
        draft_artifact.remove("dir_to_remove/")
        run.log_artifact(
            draft_artifact
        )  # 変更をログして新しいバージョンを作成し、run の出力としてマークする
    ```
  </Tab>

  <Tab title="run の外で">
    ```python
    client = wandb.Api()
    saved_artifact = client.artifact("my_artifact:latest")  # アーティファクトを読み込む
    draft_artifact = saved_artifact.new_draft()  # ドラフト版を作成する

    # ドラフト版内の一部のファイルを変更する
    draft_artifact.remove("deleted_file.txt")
    draft_artifact.add_file("modified_file.txt")
    draft_artifact.save()  # 変更をドラフトにコミットする
    ```
  </Tab>
</Tabs>
