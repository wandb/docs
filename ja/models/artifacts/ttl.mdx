W&amp;B アーティファクト の有効期間 (TTL) ポリシーを使用して、W&amp;B でアーティファクトが削除されるタイミングをスケジュールします。アーティファクトを削除すると、W&amp;B はそのアーティファクトを *soft-delete* としてマークします。つまり、そのアーティファクトは削除対象としてマークされますが、ファイルが storage からすぐに削除されるわけではありません。W&amp;B がアーティファクトをどのように削除するかについて詳しくは、[Delete artifacts](./delete-artifacts) ページを参照してください。

W&amp;B App で Artifacts TTL を使用してデータ保持を管理する方法については、[Managing data retention with Artifacts TTL](https://www.youtube.com/watch?v=hQ9J6BoVmnc) のビデオチュートリアルをご覧ください。

<Note>
  W&amp;B は、Registry にリンクされたアーティファクトに対して TTL ポリシーを設定するオプションを無効にします。これは、本番 workflow で使用されるリンク済みアーティファクトが誤って期限切れにならないようにするためです。
</Note>

<Note>
  * [チームの設定](/ja/platform/app/settings-page/teams) を表示し、(1) 誰が TTL ポリシーを設定または編集できるかを許可する、または (2) チームのデフォルト TTL を設定するといったチームレベルの TTL 設定に액세スできるのは、チーム管理者のみです。
  * W&amp;B App UI のアーティファクト詳細で TTL ポリシーを設定または編集するオプションが表示されない場合、またはプログラムで TTL を設定してもアーティファクトの TTL プロパティが正常に変更されない場合は、チーム管理者からその権限が付与されていません。
</Note>

<div id="auto-generated-artifacts">
  ## 自動生成された Artifacts
</div>

TTL ポリシーを使用できるのは、ユーザーが生成したアーティファクトのみです。W&amp;B によって自動生成されたアーティファクトには、TTL ポリシーを設定できません。

次のアーティファクト 유형 は、自動生成されたアーティファクトであることを示します:

* `run_table`
* `code`
* `job`
* 次で始まる任意のアーティファクト 유형: `wandb-*`

[W&amp;B プラットフォーム](/ja/models/artifacts/explore-and-traverse-an-artifact-graph/) またはプログラムから、アーティファクトの 유형 を確認できます:

```python
import wandb

with wandb.init(project="<my-project-name>") as run:
    artifact = run.use_artifact(artifact_or_name="<my-artifact-name>")
    print(artifact.type)
```

`<>` で囲まれた値は、ご自身の値に置き換えてください。

<div id="define-who-can-edit-and-set-ttl-policies">
  ## TTL ポリシーを編集および設定できるユーザーを定義する
</div>

チーム内で TTL ポリシーを設定および編集できるユーザーを定義します。TTL 権限はチーム管理者のみに付与することも、チーム管理者とチームメンバーの両方に付与することもできます。

<Note>
  TTL ポリシーを設定または編集できるユーザーを定義できるのは、チーム管理者のみです。
</Note>

1. チームのプロフィールページにアクセスします。
2. **Settings** タブを選択します。
3. **Artifacts time-to-live (TTL) section** にアクセスします。
4. **TTL permissions dropdown** から、TTL ポリシーを設定および編集できるユーザーを選択します。
5. **Review and save settings** をクリックします。
6. 変更内容を確認し、**Save settings** を選択します。

<div id="create-a-ttl-policy">
  ## TTL ポリシーを作成する
</div>

アーティファクトの TTL ポリシーは、アーティファクトの作成時、またはアーティファクトの作成後にさかのぼって設定できます。

以下のすべての 코드 스니펫 では、`<>` で囲まれた内容を、ご自身の情報に置き換えて 코드 스니펫 を使用してください。

<div id="set-a-ttl-policy-when-you-create-an-artifact">
  ### アーティファクトの作成時に TTL ポリシーを設定する
</div>

アーティファクトの作成時に TTL ポリシーを定義するには、W&amp;B Python SDK を使用します。TTL ポリシーは通常、日数で定義します。

<Note>
  アーティファクトの作成時に TTL ポリシーを定義する方法は、通常の[アーティファクトの作成](/ja/models/artifacts/construct-an-artifact/)と似ています。異なる点は、アーティファクトの `ttl` 属性に time delta を渡すことです。
</Note>

手順は次のとおりです。

1. [アーティファクトを作成](/ja/models/artifacts/construct-an-artifact/)します。
2. ファイル、ディレクトリ、または参照などの[コンテンツをアーティファクトに追加](/ja/models/artifacts/construct-an-artifact/#add-files-to-an-artifact)します。
3. Python の標準ライブラリに含まれる [`datetime.timedelta`](https://docs.python.org/3/library/datetime.html) データ型を使用して、TTL の制限時間を定義します。
4. [アーティファクトをログする](/ja/models/artifacts/construct-an-artifact/#3-save-your-artifact-to-the-wb-server)。

次のコード 스니펫は、アーティファクトを作成して TTL ポリシーを設定する方法を示しています。

```python
import wandb
from datetime import timedelta

with wandb.init(project="<my-project-name>", entity="<my-entity>") as run:
    artifact = wandb.Artifact(name="<artifact-name>", type="<type>")
    artifact.add_file("<my_file>")

    artifact.ttl = timedelta(days=30)  # TTL ポリシーを設定する
    run.log_artifact(artifact)
```

前述の코드 스니펫では、アーティファクトの TTL ポリシーを 30 日に設定しています。つまり、W&amp;B は 30 日後にアーティファクトを削除します。

<div id="set-or-edit-a-ttl-policy-after-you-create-an-artifact">
  ### アーティファクトを作成した後に TTL ポリシーを設定または編集する
</div>

W&amp;B App UI または W&amp;B Python SDK を使用して、すでに存在するアーティファクトの TTL ポリシーを定義します。

<Note>
  アーティファクトの TTL を変更しても、アーティファクトが期限切れになるまでの時間は、引き続きアーティファクトの `createdAt` Timestamp を使用して計算されます。
</Note>

<Tabs>
  <Tab title="Python SDK">
    1. [アーティファクトを取得します](/ja/models/artifacts/download-and-use-an-artifact/)。
    2. アーティファクトの `ttl` 属性に time delta を渡します。
    3. [`save`](/ja/models/ref/python/experiments/run#save) method でアーティファクトを更新します。

    次のコード 스니펫 は、アーティファクトの TTL ポリシーを設定する方法を示しています。

    ```python
    import wandb
    from datetime import timedelta

    artifact = run.use_artifact("<my-entity/my-project/my-artifact:alias>")
    artifact.ttl = timedelta(days=365 * 2)  # 2 年後に削除
    artifact.save()
    ```

    前のコード例では、TTL ポリシーを 2 年に設定しています。
  </Tab>

  <Tab title="W&B App">
    1. W&amp;B App UI で W&amp;B プロジェクトにアクセスします。
    2. プロジェクトのサイドバーでアーティファクト アイコンを選択します。
    3. アーティファクトのリストから、対象の 유형 を展開します
    4. TTL ポリシーを編集するアーティファクト バージョンを選択します。
    5. **Version** タブをクリックします。
    6. ドロップダウンから **Edit TTL policy** を選択します。
    7. 表示されるモーダル内で、TTL ポリシーのドロップダウンから **Custom** を選択します。
    8. **TTL duration** フィールドで、日単位で TTL ポリシーを設定します。
    9. 変更を保存するには、**Update TTL** ボタンを選択します。

      <img src="/images/artifacts/edit_ttl_ui.gif" alt="TTL ポリシーの編集" />

<div id="set-default-ttl-policies-for-a-team">
  ### チームのデフォルト TTL ポリシーを設定する
</div>

<Note>
  チームのデフォルト TTL ポリシーを設定できるのは、チーム管理者のみです。
</Note>

チームのデフォルト TTL ポリシーを設定します。デフォルト TTL ポリシーは、それぞれの作成日に기준으로、既存および今後作成されるすべてのアーティファクトに適用されます。既存のバージョン レベル TTL ポリシーを持つ Artifacts は、チームのデフォルト TTL の影響を受けません。

1. チームのプロフィールページにアクセスします。
2. **Settings** タブを選択します。
3. **Artifacts time-to-live (TTL) section** にアクセスします。
4. **Set team&#39;s default TTL policy** をクリックします。
5. **Duration** フィールドで、TTL ポリシーを日単位で設定します。
6. **Review and save settings** をクリックします。
   7/ 変更を確認してから、**Save settings** を選択します。

<div id="set-a-ttl-policy-outside-of-a-run">
  ### run の外で TTL ポリシーを設定する
</div>

Public API を使用して、run を取得せずにアーティファクトを取得し、TTL ポリシーを設定します。TTL ポリシーは通常、日数で定義します。

次のコード例は、Public API を使用してアーティファクトを取得し、TTL ポリシーを設定する方法を示しています。

```python
api = wandb.Api()

artifact = api.artifact("entity/project/artifact:alias")

artifact.ttl = timedelta(days=365)  # 1年後に削除

artifact.save()
```

  </Note> */}

<Tabs>
  <Tab title="Python SDK">
    1. [アーティファクトを取得します](/ja/models/artifacts/download-and-use-an-artifact/).
    2. アーティファクトの `ttl` 属性を `None` に設定します。
    3. [`save`](/ja/models/ref/python/experiments/run#save) method でアーティファクトを更新します。

    次の코드 스니펫は、アーティファクトの TTL ポリシーを無効にする方法を示しています。


  </Tab>

  <Tab title="W&B App">

    3. アーティファクトの list から、対象の 유형 を展開します

    8. 表示されるモーダル内で、TTL ポリシーのドロップダウンから **Deactivate** を Select します。
    9. **Update TTL** ボタンを Select して変更を保存します。

<div id="view-ttl-policies">
  ## TTL ポリシーを表示

<Tabs>
  <Tab title="Python SDK">
    `print` 文を使用して、アーティファクトの TTL ポリシーを表示します。次の例は、アーティファクトを取得し、その TTL ポリシーを表示する方法を示しています。

    ```python
    artifact = run.use_artifact("<my-entity/my-project/my-artifact:alias>")

    ```
  </Tab>

  <Tab title="W&B App">
    W&amp;B App UI を使用して、アーティファクトの TTL ポリシーを表示します。

    2. W&amp;B プロジェクトにアクセスします。
    3. プロジェクト内で、プロジェクトのサイドバーにある Artifacts タブを選択します。
    4. コレクションをクリックします。

    コレクションビューでは、選択したコレクション内のすべてのアーティファクトを確認できます。`Time to Live` 列には、そのアーティファクトに割り当てられている TTL ポリシーが表示されます。
