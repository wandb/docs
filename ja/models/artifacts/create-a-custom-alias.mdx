description: "best や production のような意味のある名前で特定の W&B アーティファクト バージョンを参照するためのカスタムエイリアスを作成・管理します。"
title: アーティファクトのエイリアスを作成する
---

エイリアスは特定のバージョンを指すポインターとして使用します。デフォルトでは、`wandb.Run.log_artifact()` はログされたバージョンに `latest` エイリアスを追加します。

W&amp;B は、アーティファクトを初めてログしたときにアーティファクト バージョン `v0` を作成し、そのアーティファクトに関連付けます。W&amp;B は同じアーティファクトに再度ログするときに内容のチェックサムを計算します。アーティファクトが変更されている場合、W&amp;B は新しいバージョン `v1` を保存します。

たとえば、トレーニングスクリプトでデータセットの最新バージョンを取得したい場合は、そのアーティファクトを使用するときに `latest` を指定します。次のコード例は、`latest` というエイリアスを持つ `bike-dataset` という名前の最新のデータセット アーティファクトをダウンロードする方法を示しています。

```python
import wandb

with wandb.init(project="<project>") as run:
    artifact = run.use_artifact("bike-dataset:latest")
    artifact.download()
```

アーティファクト バージョンにカスタム エイリアスを適用することもできます。たとえば、あるモデルのチェックポイントがメトリクス AP-50 で最高であることを示したい場合、モデル アーティファクトをログするときに、エイリアスとして文字列 `'best-ap50'` を追加できます。

```python
with wandb.init(project="<project>") as run:
    artifact = wandb.Artifact("run-3nq3ctyy-bike-model", type="model")
    artifact.add_file("model.h5")
    run.log_artifact(artifact, aliases=["latest", "best-ap50"])
```
