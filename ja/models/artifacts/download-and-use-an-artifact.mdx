title: アーティファクト をダウンロードして使用する
---

W&amp;B サーバー にすでに保存されている アーティファクト をダウンロードして使用するか、必要に応じて アーティファクト オブジェクトを構築し、重複排除のために渡します。

<Note>
  View-Only シートのチームメンバーは アーティファクト をダウンロードできません。
</Note>

<div id="download-and-use-an-artifact-stored-on-wb">
  ### W&amp;B に保存されたアーティファクトをダウンロードして使用する
</div>

W&amp;B Run の実行中でも実行外でも、W&amp;B に保存されたアーティファクトをダウンロードして使用できます。Public API ([`wandb.Api`](/ja/models/ref/python/public-api/api)) を使用して、W&amp;B にすでに保存されているデータをエクスポート (または更新) できます。

<Tabs>
  <Tab title="run 中">
    まず、W&amp;B Python SDK をインポートします。次に、W&amp;B の [Run](/ja/models/ref/python/experiments/run) を作成します:

    ```python
    import wandb

    with wandb.init(project="<example>", job_type="<job-type>") as run:
        # 次のステップを参照
    ```

    [`wandb.Run.use_artifact()`](/ja/models/ref/python/experiments/run#use_artifact) メソッドで、使用するアーティファクトを指定します。このメソッドは run オブジェクトを返します。次のコードスニペットでは、`'bike-dataset'` という名前で、エイリアスが `'latest'` のアーティファクトを指定しています:

    ```python
    # 使用するアーティファクトを指定します。形式は "name:alias"
    artifact = run.use_artifact("bike-dataset:latest")
    ```

    返されたオブジェクトを使用して、アーティファクトの内容全体をダウンロードします:

    ```python
    # アーティファクト全体をダウンロード
    datadir = artifact.download()
    ```

    必要に応じて、`root` パラメーターにパスを渡すと、アーティファクトの内容を特定のディレクトリにダウンロードできます。

    [`wandb.Artifact.get_entry()`](/ja/models/ref/python/experiments/artifact#get_entry) メソッドを使用すると、ファイルの一部だけをダウンロードできます:

    ```python
    # 特定のファイルをダウンロード
    entry = artifact.get_entry(name)
    ```

    これらをまとめると、完全なコード例は次のようになります:

    ```python
    import wandb    

    with wandb.init(project="<example>", job_type="<job-type>") as run:
        # 使用するアーティファクトを指定します。形式は "name:alias"
        artifact = run.use_artifact("bike-dataset:latest")

        # アーティファクト全体をダウンロード
        datadir = artifact.download()

        # 特定のファイルをダウンロード
        entry = artifact.get_entry("bike.png")
    ```

    これは、パス `name` にあるファイルだけを取得します。戻り値は、次のメソッドを持つ `Entry` オブジェクトです:

    * `Entry.download`: パス `name` にあるアーティファクト内のファイルをダウンロードします
    * `Entry.ref`: `add_reference` によってエントリが参照として保存されている場合、その URI を返します

    {/* W&B が処理方法を認識しているスキームを持つ参照は、アーティファクト ファイルと同様にダウンロードされます。詳細は [外部ファイルのトラッキング](/models/artifacts/track-external-files/) を参照してください。 */}
  </Tab>

  <Tab title="run の外で">
    まず、W&amp;B SDK をインポートします。次に、Public API クラスからアーティファクトオブジェクトを作成します。そのアーティファクトに関連付けられている entity、プロジェクト、artifact、alias を指定します:

    ```python
    import wandb

    api = wandb.Api()
    artifact = api.artifact("entity/project/artifact:alias")
    ```

    返されたオブジェクトを使用して、アーティファクトの内容をダウンロードします:

    ```python
    artifact.download()
    ```

    必要に応じて、`root` パラメーターにパスを渡すと、アーティファクトの内容を特定のディレクトリにダウンロードできます。詳細は [Python SDK Reference Guide](/ja/models/ref/python/experiments/artifact#download) を参照してください。
  </Tab>

  <Tab title="W&B CLI">
    `wandb artifact get` コマンドを使用して、W&amp;B サーバーからアーティファクトをダウンロードします。

    ```
    $ wandb artifact get project/artifact:alias --root mnist/
    ```
  </Tab>
</Tabs>

<div id="partially-download-an-artifact">
  ### アーティファクトを部分的にダウンロードする
</div>

プレフィックスを指定すると、アーティファクトの一部だけを必要に応じてダウンロードできます。`path_prefix=` (`wandb.Artifact.download(path_prefix=)`) パラメーターを使用して、単一のファイルまたはサブフォルダー内の内容をダウンロードします。

```python
with wandb.init(project="<example>", job_type="<job-type>") as run:
    # 使用するアーティファクトを指定します。形式は "name:alias" です
    artifact = run.use_artifact("bike-dataset:latest")

    # 特定のファイルまたはサブフォルダーをダウンロードする
    artifact.download(path_prefix="bike.png") # bike.png のみをダウンロード
```

別の方法として、特定のディレクトリ内のファイルをダウンロードすることもできます。その場合は、`path_prefix=` パラメーターにディレクトリを指定します。前のコードスニペットに続けて、次のように指定します。

```python
# images/bikes ディレクトリ内のファイルをダウンロードする
artifact.download(path_prefix="images/bikes/") 
```

<div id="use-an-artifact-from-a-different-project">
  ### 別のプロジェクトの アーティファクト を使用する
</div>

アーティファクト を参照するには、アーティファクト 名とあわせてそのプロジェクト名を指定します。Entities をまたいで アーティファクト を参照する場合は、アーティファクト 名とあわせてその entity 名を指定します。

次のコード例は、別のプロジェクトの アーティファクト を、現在の W&amp;B run の入力として取得する方法を示しています。

```python
with wandb.init(project="<example>", job_type="<job-type>") as run:
    # 別のプロジェクトのアーティファクトを W&B に照会し、
    # この run への入力としてマークする。
    artifact = run.use_artifact("my-project/artifact:alias")

    # 別の entity のアーティファクトを使用し、この run への
    # 入力としてマークする。
    artifact = run.use_artifact("my-entity/my-project/artifact:alias")

```

<div id="construct-and-use-an-artifact-simultaneously">
  ### アーティファクトを同時に構築して使用する
</div>

アーティファクトを同時に構築して使用します。アーティファクト オブジェクトを作成し、`use&#95;artifact` に渡します。これにより、まだ存在しない場合は W&amp;B にアーティファクトが作成されます。[`wandb.Run.use_artifact()`](/ja/models/ref/python/experiments/run#use_artifact) API は冪等であるため、必要に応じて何度でも呼び出せます。

```python
import wandb

with wandb.init(project="<example>", job_type="<job-type>") as run:
    artifact = wandb.Artifact("reference model")
    artifact.add_file("model.h5")
    run.use_artifact(artifact)
```

詳細は、[アーティファクトを構築する](/ja/models/artifacts/construct-an-artifact/)をご覧ください。
