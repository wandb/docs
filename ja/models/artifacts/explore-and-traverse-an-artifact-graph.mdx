description: "リネージ グラフを表示してたどり、W&B の run の入力と出力を有向非巡回グラフとして追跡します。"
title: リネージ グラフを探索する
---

W&amp;B は、*リネージ グラフ* と呼ばれる有向非巡回グラフ (DAG) を使用して、run の入力と出力を追跡します。リネージ グラフは、ML 実験における artifacts と run の関係を視覚的に表現したものです。生データの取り込みからモデルのトレーニングや評価に至るまで、データやモデルが ML ライフサイクルのさまざまな段階をどのように流れていくかを示します。

artifact のリネージを追跡することには、次のような主な利点があります。

* **再現性**: デバッグ、実験、検証のために、実験、モデル、結果を再現できるようにします。
* **バージョン管理**: artifact の経時的な変更を追跡し、必要に応じて過去のデータやモデルのバージョンに戻せるようにします。
* **監査**: コンプライアンスとガバナンスを支援するために、artifact とその変換の詳細な記録を維持します。
* **コラボレーション**: 実験履歴を可視化することでチームワークの向上に役立ち、重複作業を減らし、開発を加速します。

<div id="view-an-artifacts-lineage-graph">
  ## Artifacts のリネージ グラフを表示する
</div>

Artifacts のリネージ グラフを表示するには、次の手順に従います。

1. W&amp;B App で対象のプロジェクトのワークスペースにアクセスします。
2. プロジェクト サイドバーで **Artifacts** タブをクリックします。
3. アーティファクトを選択し、**Lineage** タブをクリックします。

<div id="navigate-lineage-graphs">
  ## リネージ グラフをたどる
</div>

リネージ グラフは、ML 実験におけるアーティファクトと run の関係を視覚的に表したものです。W&amp;B App UI または Python SDK を使用して、アーティファクトのリネージ グラフを探索し、たどることができます。

<Tabs>
  <Tab title="W&B App UI">
    緑色のアイコンのノードは run を表します。青色のアイコンのノードはアーティファクトを表します。ノード間の矢印は、run またはアーティファクトの入力と出力を示します。

    アーティファクト ノードには、`<artifact_name>:<version>` の形式でアーティファクトの名とバージョンが表示されます。アーティファクトのタイプは、アーティファクト名の上に表示されます。

    <Note>
      アーティファクトのタイプと名は、左サイドバーとリネージ グラフのノードの両方で確認できます。
    </Note>

    run ノードには、run の名が表示されます。

    <Frame>
      <img src="/images/artifacts/lineage2.png" alt="run とアーティファクトのノード" />
    </Frame>

    個々の run をクリックすると、その run の詳細情報 (開始時刻、実行時間、作成者、ジョブ タイプなど) を確認できます。個々のアーティファクトをクリックすると、そのアーティファクトの詳細情報 (alias、作成時刻、タイプ、バージョン、説明、そのアーティファクトをログした run、ファイル サイズなど) を確認できます。

    <Frame>
      <img src="/images/artifacts/lineage3a.gif" alt="run のプレビュー" />
    </Frame>

    同じアーティファクトの複数バージョンを作成する Runs は、クラスターとしてまとめて表示されます。クラスター内に表示されている特定のアーティファクト バージョンをクリックすると、そのバージョン固有の情報を確認できます。

    <Frame>
      <img src="/images/artifacts/lineage_multiple_artifact_versions.png" alt="リネージ グラフ内のアーティファクト バージョンのクラスター" />
    </Frame>

    ノードをクリックしてドラッグすると、グラフを並べ替えて Layout を custom できます。ノードとその関係を見やすくするために、グラフをズーム インまたはズーム アウトすることもできます。

    <Frame>
      <img src="/images/artifacts/lineage_rearrange_nodes.gif" alt="リネージ グラフ内でノードを並べ替える" />
    </Frame>

    ノードにマウス カーソルを合わせて目のアイコンをクリックすると、グラフ内でそのノードの表示と非表示を切り替えられます。これは、特定のノードとその関係に注目できるように、グラフの煩雑さを減らすのに役立ちます。
  </Tab>

  <Tab title="W&B Python SDK">
    W&amp;B Python SDK を使用して、プログラムからグラフをたどることができます。アーティファクト オブジェクトの
    [`logged_by()`](/ja/models/ref/python/experiments/artifact#method-artifact-logged-by) と [`used_by()`](/ja/models/ref/python/experiments/artifact#method-artifact-used-by) method を使用して、グラフをたどります。

    ```python
    with wandb.init() as run:
        artifact = run.use_artifact("artifact_name:latest")

        # アーティファクトからグラフを上下にたどる:
        producer_run = artifact.logged_by()
        consumer_runs = artifact.used_by()
    ```
  </Tab>
</Tabs>

<div id="enable-lineage-graph-tracking">
  ## リネージ グラフの追跡を有効化する
</div>

リネージ グラフの追跡を有効化するには、W&amp;B Python SDK を使用して Artifacts を run の [入力](/ja/models/artifacts/explore-and-traverse-an-artifact-graph)または
[出力](/ja/models/artifacts/explore-and-traverse-an-artifact-graph#track-the-output-of-a-run)として指定する必要があります。

<div id="track-the-input-of-a-run">
  ### run の入力を追跡する
</div>

[`wandb.Run.use_artifact()`](/ja/models/ref/python/experiments/run#method-runuse_artifact)
メソッドを使って、 Artifacts を run の入力 (または依存関係) として設定します。 Artifacts 名と、その Artifacts の特定のバージョンを参照するための任意のエイリアスを指定します。 Artifacts の名前は `<artifact_name>:<version>` または `<artifact_name>:<alias>` という形式です。

山かっこ (`< >`) で囲まれた値を、自分の値に置き換えてください。

```python
import wandb

# run を初期化する
with wandb.init(entity="<entity>", project="<project>") as run:
  # Artifacts を取得し、依存関係としてマークする
  artifact = run.use_artifact(artifact_or_name="<name>", aliases="<alias>")
```

<div id="track-the-output-of-a-run">
  ### run の出力を追跡する
</div>

[`wandb.Run.log_artifact()`](/ja/models/ref/python/experiments/run#log_artifact) を使用して、ある run の出力として Artifacts を指定します。まず、
[`wandb.Artifact()`](/ja/models/ref/python/experiments/artifact#wandb.Artifact) コンストラクターで Artifacts を作成します。次に、`wandb.Run.log_artifact()` を使って
その Artifacts を run の出力として記録します。

山かっこ (`< >`) で囲まれた値を、実際の値に置き換えてください。

```python
import wandb

# run を初期化する
with wandb.init(entity="<entity>", project="<project>") as run:
  
  # Artifacts を作成する
  artifact = wandb.Artifact(name = "<artifact_name>", type = "<artifact_type>")
  artifact.add_file(local_path = "<local_filepath>", name="<optional-name>")

  # Artifacts を run の出力としてログに記録する
  run.log_artifact(artifact_or_path = artifact)
```

<div id="artifact-clusters">
  ## Artifact クラスター
</div>

グラフのあるレベルに run または Artifacts が 5 つ以上ある場合、そのレベルにはクラスターが作成されます。クラスターには、特定のバージョンの run や Artifacts を検索できる検索バーがあり、クラスターから個々のノードを取り出して、そのノードの lineage を引き続き調査できます。

ノードをクリックすると、そのノードの概要を示すプレビューが開きます。矢印をクリックすると、個々の run または Artifacts がクラスターから取り出され、取り出したノードの lineage を確認できます。
