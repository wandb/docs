description: "W&B Runs の入力と出力を有向非巡回グラフとしてトラッキングするために、Artifacts のリネージ グラフを表示してたどります。"
title: アーティファクトのリネージ グラフを確認する
---

W&amp;B は、*リネージ グラフ* と呼ばれる有向非巡回グラフ (DAG) を使用して、Runs の入力と出力をトラッキングします。リネージ グラフは、ML の experiment における Artifacts と Runs の関係を視覚的に表したものです。これにより、生データの取り込みからモデル トレーニング、評価に至るまで、ML lifecycle のさまざまな段階をデータやモデルがどのように
流れていくかを確認できます。

Artifacts のリネージをトラッキングすると、主に次の利点があります。

* **再現性**: デバッグ、実験、検証のために、チームが Experiments、モデル、結果を再現できるようにします。
* **バージョン管理**: Artifacts の change を経時的に追跡することで、必要に応じて以前のデータや model versions に戻せるようにします。
* **監査**: Artifacts と変換の詳細な記録を維持し、Compliance とガバナンスをサポートします。
* **コラボレーション**: experiment の history を可視化し、重複作業を減らし、開発を加速することで、チームワークの向上に役立ちます。

<div id="view-an-artifacts-lineage-graph">
  ## アーティファクトのリネージ グラフを表示する
</div>

アーティファクトのリネージ グラフを表示するには、次の手順に従います。

1. W&amp;B App でプロジェクトの Workspace にアクセスします。
2. プロジェクトのサイドバーで **Artifacts** タブをクリックします。
3. アーティファクトを選択し、**Lineage** タブをクリックします。

<div id="navigate-lineage-graphs">
  ## リネージ グラフを移動する
</div>

リネージ グラフは、ML experiment における Artifacts と Runs の関係を視覚的に表したものです。W&amp;B App UI または Python SDK を使用して、Artifacts のリネージ グラフを探索し、たどることができます。

<Tabs>
  <Tab title="W&B App UI">
    緑色のアイコンが付いたノードは Runs を表します。青色のアイコンが付いたノードは Artifacts を表します。ノード間の矢印は、run またはアーティファクトの入力と出力を示します。

    アーティファクトのノードには、`<artifact_name>:<version>` の形式でアーティファクト名とそのバージョンが表示されます。アーティファクトのタイプは、アーティファクト名の上に表示されます。

    <Note>
      アーティファクトのタイプと名は、左サイドバーとリネージ グラフのノードの両方で確認できます。
    </Note>

    run のノードには、run 名が表示されます。

    <Frame>
      <img src="/images/artifacts/lineage_multiple_artifact_versions.png" alt="リネージ グラフ内のアーティファクトバージョンのクラスター" />

    ノードをクリックしてドラッグすると、グラフを並べ替えて Layout をカスタマイズできます。また、グラフを拡大・縮小して、ノードとその関係を見やすくすることもできます。

  <Tab title="W&B Python SDK">
    W&amp;B Python SDK を使用して、プログラムからグラフを移動します。アーティファクト object の
    [`logged_by()`](/ja/models/ref/python/experiments/artifact#method-artifact-logged-by) と [`used_by()`](/ja/models/ref/python/experiments/artifact#method-artifact-used-by) method を使用してグラフをたどります。

    ```python
    with wandb.init() as run:
        artifact = run.use_artifact("artifact_name:latest")

        # アーティファクトからグラフを上下にたどる:
        producer_run = artifact.logged_by()
        consumer_runs = artifact.used_by()
    ```
  </Tab>
</Tabs>

<div id="enable-lineage-graph-tracking">
  ## リネージ グラフのトラッキングを有効にする
</div>

リネージ グラフのトラッキングを有効にするには、W&amp;B Python SDK を使用して、Artifacts を run の [inputs](/ja/models/artifacts/explore-and-traverse-an-artifact-graph) または
[outputs](/ja/models/artifacts/explore-and-traverse-an-artifact-graph#track-the-output-of-a-run) としてマークする必要があります。

<div id="track-the-input-of-a-run">
  ### run の入力をトラッキングする
</div>

[`wandb.Run.use_artifact()`](/ja/models/ref/python/experiments/run#method-runuse_artifact)
メソッドを使用して、アーティファクトを run の入力 (または依存関係) としてマークします。そのアーティファクトの特定のバージョンを参照するには、アーティファクトの名と、省略可能なエイリアスを指定します。アーティファクトの名の形式は
`<artifact_name>:<version>` または `<artifact_name>:<alias>` です。

山かっこ (`< >`) で囲まれた値を、実際の値に置き換えてください。

```python
import wandb

# runを初期化する
with wandb.init(entity="<entity>", project="<project>") as run:
  # アーティファクトを取得し、依存関係としてマークする
  artifact = run.use_artifact(artifact_or_name="<name>", aliases="<alias>")
```

<div id="track-the-output-of-a-run">
  ### run の出力をトラッキングする
</div>

[`wandb.Run.log_artifact()`](/ja/models/ref/python/experiments/run#log_artifact) を使用して、アーティファクトを run の出力として宣言します。まず、
[`wandb.Artifact()`](/ja/models/ref/python/experiments/artifact#wandb.Artifact) コンストラクターを使用してアーティファクトを作成します。次に、
`wandb.Run.log_artifact()` を使用して、そのアーティファクトを run の出力としてログします。

山かっこ (`< >`) で囲まれた値は、ご自身の値に置き換えてください。

```python
import wandb

# runを初期化する
with wandb.init(entity="<entity>", project="<project>") as run:
  
  # アーティファクトを作成する
  artifact = wandb.Artifact(name = "<artifact_name>", type = "<artifact_type>")
  artifact.add_file(local_path = "<local_filepath>", name="<optional-name>")

  # アーティファクトをrunの出力としてログする
  run.log_artifact(artifact_or_path = artifact)
```

<div id="artifact-clusters">
  ## アーティファクト クラスター
</div>

グラフのあるレベルに 5 つ以上の run または Artifacts がある場合、クラスターが作成されます。クラスターには、run または Artifacts の特定のバージョンを検索するための検索バーがあり、クラスターから個別のノードを取り出して、クラスター内のノードのリネージを引き続き調査できます。

ノードをクリックすると、ノードの概要を示すプレビューが開きます。矢印をクリックすると、個別の run または Artifacts が取り出され、取り出したノードのリネージを確認できます。
