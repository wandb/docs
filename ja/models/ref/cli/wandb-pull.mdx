<div id="usage">
  ## 使用方法
</div>

```bash
pull [OPTIONS] RUN
```

<div id="description">
  ## 説明
</div>

W&amp;B の run からファイルをダウンロードします。

指定した run に関連付けられているすべてのファイルを取得します。内容が同一のファイルが
すでにローカルに存在する場合は、そのファイルをスキップします。必要に応じて
サブディレクトリを作成し、W&amp;B 内のファイル構造を再現します。

run ID を使用して run を参照し、run 引数に含まれていない場合は、
必要に応じて project と entity を指定します。

たとえば、&quot;foobar&quot; project の
&quot;team-awesome&quot; entity にある run ID &quot;abcd1234&quot; の run からファイルをダウンロードするには:

```bash
wandb pull -p foobar -e team-awesome abcd1234
```

<div id="arguments">
  ## 引数
</div>

| 名     | デフォルト | タイプ  |
| ----- | ----- | ---- |
| `run` | STR   | True |

<div id="options">
  ## オプション
</div>

| Flag            | Type | Description                                                                                                     |
| --------------- | ---- | --------------------------------------------------------------------------------------------------------------- |
| `--project, -p` | STR  | ファイルの取得元となる run を含む project。**デフォルト**: なし                                                                       |
| `--entity, -e`  | STR  | project を所有する entity。設定されている場合は WANDB&#95;ENTITY 環境変数の値が使用され、設定されていない場合はデフォルトの entity が使用されます。**デフォルト**: models |
