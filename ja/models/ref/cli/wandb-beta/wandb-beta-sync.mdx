PATHS で指定した .wandb ファイルをアップロードします。

これは `wandb sync` のベータ版の再実装です。機能はまだ完全ではなく、動作も保証されていません。また、wandb のどの version でも、後方互換性のない変更が行われる可能性があります。

PATHS には、.wandb ファイル、.wandb ファイルを含む run ディレクトリ、または run ディレクトリを含む &quot;wandb&quot; ディレクトリを指定できます。

たとえば、ディレクトリ内のすべての run を Sync するには:

```bash
wandb beta sync ./wandb
```

特定の run をSyncするには:

```bash
wandb beta sync ./wandb/run-20250813_124246-n67z9ude
```

あるいは次のようにも書けます：

```bash
wandb beta sync ./wandb/run-20250813_124246-n67z9ude/run-n67z9ude.wandb
```

<div id="usage">
  ## 使用法
</div>

```bash

```

  ## 引数

| 引数      | 説明       | 必須  |
| :------ | :------- | :-- |
| `PATHS` | 説明はありません | いいえ |

<div id="options">
  ## オプション
</div>

| オプション             | 説明                                                                                                                                                                      |
| :---------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--live`          | まだログ中の run を Sync します。run を生成しているプロセスが異常終了した場合、ハングすることがあります。 (デフォルト: False)                                                                                             |
| `-e`, `--entity`  | Sync するすべての run に使用する entity のオーバーライドです。 (デフォルト: )                                                                                                                      |
| `-p`, `--project` | Sync するすべての run に使用するプロジェクトのオーバーライドです。 (デフォルト: )                                                                                                                        |
| `--id`            | Sync するすべての run に使用する run ID のオーバーライドです。これを設定して複数のファイルを (同じ entity とプロジェクトで) Sync する場合、ファイルは開始時刻順に Sync されます。これは、同じ run の再開後の複数のフラグメントを Sync するケースを想定したものです。 (デフォルト: )  |
| `--job-type`      | Sync するすべての run に使用するジョブタイプのオーバーライドです。 (デフォルト: )                                                                                                                        |
| `--replace-tags`  | `'old1=new1,old2=new2'` の形式で tags 名を変更します。 (デフォルト: )                                                                                                                    |
| `--skip-synced`   | この command ですでに Sync 済みの run をスキップします。 (デフォルト: True)                                                                                                                    |
| `--dry-run`       | 何も upload せずに、実行された場合の内容を表示します。 (デフォルト: False)                                                                                                                          |
| `-v`, `--verbose` | より詳細な情報を表示します。 (デフォルト: False)                                                                                                                                           |
| `-n`              | 一度に Sync する run の最大数です。同じ run の一部である複数のファイルを Sync する場合、ファイルはこの設定に関係なく、開始時刻順に順次 Sync されます。これは、run を再開した場合や `--id` パラメーターを使用した場合に発生します。 (デフォルト: 5)                        |
