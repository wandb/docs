W&amp;B Job を Launch するか、キューに追加します。詳しくは https://wandb.me/launch を参照してください.


<div id="usage">
  ## 使用方法
</div>

```bash
wandb launch [OPTIONS]
```


<div id="options">
  ## オプション
</div>

| Option                  | Description                                                                                                                                                                                                                                     |
| :---------------------- | :---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--uri`, `-u`           | Launch するローカルパスまたは git リポジトリの uri。指定すると、このコマンドは指定した uri から job を作成します。                                                                                                                                                                          |
| `--job`, `-j`           | Launch する job の名前。これを指定した場合、uri は不要です。                                                                                                                                                                                                          |
| `--entry-point`, `-E`   | プロジェクト内のエントリーポイント。[デフォルト: main]。エントリーポイントが見つからない場合は、指定した名前のプロジェクトファイルをスクリプトとして実行しようとします。.py ファイルは &#39;python&#39; を使用して実行し、.sh ファイルはデフォルトのシェル (環境変数 $SHELL で指定) を使用して実行します。指定した場合、設定ファイルで指定した entrypoint の値を上書きします。                       |
| `--build-context`       | ソースコード内の build context へのパス。デフォルトはソースコードのルートです。-u と併用する場合にのみ使用できます。                                                                                                                                                                             |
| `--name`                | run を Launch する際の run 名。指定しない場合は、ランダムな run 名で run が Launch されます。指定した場合、設定ファイルで指定した name を上書きします。                                                                                                                                           |
| `--entity`, `-e`        | 新しい run の送信先となる entity の名前。デフォルトでは、ローカルの wandb/settings フォルダで設定された entity を使用します。指定した場合、設定ファイルで指定した entity の値を上書きします。                                                                                                                      |
| `--project`, `-p`       | 新しい run の送信先となるプロジェクト名。デフォルトでは、source uri から与えられたプロジェクト名、または GitHub run の場合は git リポジトリ名を使用します。指定した場合、設定ファイルで指定した プロジェクト の値を上書きします。                                                                                                        |
| `--resource`, `-r`      | run の実行に使用する実行リソース。サポートされる値: &#39;local-process&#39;, &#39;local-container&#39;, &#39;kubernetes&#39;, &#39;sagemaker&#39;, &#39;gcp-vertex&#39;。resource 設定のない キュー に push する場合、これは必須パラメーターになりました。指定した場合、設定ファイルで指定した resource の値を上書きします。 |
| `--docker-image`, `-d`  | 使用する特定の Docker image。形式は name:tag です。指定した場合、設定ファイルで指定した docker image の値を上書きします。                                                                                                                                                            |
| `--base-image`, `-B`    | job コードを実行する Docker image。--docker-image とは併用できません。                                                                                                                                                                                             |
| `--config`, `-c`        | JSON ファイル (拡張子は必ず &#39;.json&#39;) へのパス、または Launch 設定として渡される JSON 文字列。Launch された run をどのように設定するかを指定します。                                                                                                                                         |
| `--set-var`, `-v`       | allow listing が有効な キュー に対して、テンプレート変数の値を key-value ペアとして設定します。例: `--set-var key1=value1 --set-var key2=value2`                                                                                                                                 |
| `--queue`, `-q`         | push 先の run キュー 名。指定しない場合は、単一の run を直接 Launch します。引数なし (`--queue`) で指定した場合、デフォルトで キュー &#39;default&#39; が使用されます。それ以外で名前を指定した場合、指定した run キュー は、指定した プロジェクト および entity の下に存在している必要があります。                                                   |
| `--async`               | job を非同期に実行するフラグ。デフォルトは false です。つまり、--async を指定しない限り、wandb launch は job が終了するまで待機します。このオプションは --queue とは互換性がありません。agent とともに実行する場合の非同期オプションは wandb launch-agent で設定する必要があります。(default: False)                                                  |
| `--resource-args`, `-R` | コンピュートリソースに resource args として渡される JSON ファイル (拡張子は必ず &#39;.json&#39;) へのパス、または JSON 文字列。指定する内容は実行バックエンドごとに異なります。このファイルのレイアウトについてはドキュメントを参照してください。                                                                                                |
| `--dockerfile`, `-D`    | job のビルドに使用する Dockerfile へのパス。job のルートからの相対パスです。                                                                                                                                                                                                |
| `--priority`, `-P`      | --queue を指定した場合、job の優先度を設定します。優先度の高い Launch job から先に処理されます。優先度は高い順に critical、high、medium、low です。                                                                                                                                               |
