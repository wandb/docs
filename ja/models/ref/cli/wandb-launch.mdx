W&amp;B ジョブを起動するか、キューに入れます。詳細は https://wandb.me/launch を参照してください


<div id="usage">
  ## 使い方
</div>

```bash
wandb launch [OPTIONS]
```


<div id="options">
  ## オプション
</div>

| オプション                   | 説明                                                                                                                                                                                                     |
| :---------------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--uri`, `-u`           | 起動元のローカルパスまたは git リポジトリ URI。指定すると、この command は指定した URI からジョブを作成します。                                                                                                                                    |
| `--job`, `-j`           | 起動するジョブの名。これを指定すると、Launch に URI は不要です。                                                                                                                                                                 |
| `--entry-point`, `-E`   | プロジェクト内のエントリポイント。[default: main]。エントリポイントが見つからない場合は、指定した名のプロジェクトファイルをスクリプトとして実行しようとします。`.py` ファイルは `python`、`.sh` ファイルはデフォルトのシェル (環境変数 `$SHELL` で指定) で実行します。これを指定すると、設定ファイルで指定した entrypoint の値を上書きします。 |
| `--build-context`       | ソースコード内のビルドコンテキストのパス。デフォルトはソースコードのルートです。`-u` とのみ併用できます。                                                                                                                                                |
| `--name`                | 起動する run に関連付ける run の名。指定しない場合は、ランダムな run 名で run を起動します。これを指定すると、設定ファイルで指定した名を上書きします。                                                                                                                  |
| `--entity`, `-e`        | 新しい run の送信先となる entity の名。デフォルトでは、ローカルの wandb/settings フォルダーで設定された entity を使用します。これを指定すると、設定ファイルで指定した entity の値を上書きします。                                                                                |
| `--project`, `-p`       | 新しい run の送信先となるプロジェクトの名。デフォルトでは、ソース URI で指定されたプロジェクト名、または GitHub の run では git リポジトリ名を使用します。これを指定すると、設定ファイルで指定したプロジェクトの値を上書きします。                                                                     |
| `--resource`, `-r`      | run に使用する実行リソース。サポートされる値: `'local-process'`, `'local-container'`, `'kubernetes'`, `'sagemaker'`, `'gcp-vertex'`。リソース設定のないキューにプッシュする場合、これは必須パラメーターです。これを指定すると、設定ファイルで指定した resource の値を上書きします。           |
| `--docker-image`, `-d`  | 使用する特定の Docker image。形式は `name:tag` です。これを指定すると、設定ファイルで指定した docker image の値を上書きします。                                                                                                                    |
| `--base-image`, `-B`    | ジョブコードの実行に使用する Docker image。`--docker-image` とは併用できません。                                                                                                                                                |
| `--config`, `-c`        | Launch 設定として渡される JSON ファイル (末尾が `.json` である必要があります) のパス、または JSON 文字列。起動される run の設定方法を指定します。                                                                                                            |
| `--set-var`, `-v`       | 許可リストが有効なキューに対して、テンプレート変数の値をキーと値のペアとして設定します。例: `--set-var key1=value1 --set-var key2=value2`                                                                                                           |
| `--queue`, `-q`         | プッシュ先の run キューの名。指定しない場合は、single run を直接起動します。引数なしで指定した場合 (`--queue`) 、デフォルトでキュー `'default'` が使用されます。名前を指定した場合は、指定したプロジェクト および entity 配下にその run キューが存在している必要があります。                                   |
| `--async`               | ジョブを非同期で実行するためのフラグ。デフォルトは false です。つまり、`--async` を設定しない限り、`wandb launch` はジョブの完了を待機します。このオプションは `--queue` と併用できません。agent とともに実行する場合の非同期オプションは `wandb launch-agent` で設定してください。(default: False)          |
| `--resource-args`, `-R` | コンピュートリソースに resource args として渡される JSON ファイル (末尾が `.json` である必要があります) のパス、または JSON 文字列。指定すべき正確な内容は実行バックエンドごとに異なります。このファイルの Layout についてはドキュメントを参照してください。                                                 |
| `--dockerfile`, `-D`    | ジョブのビルドに使用する Dockerfile のパス。ジョブのルートからの相対パスです                                                                                                                                                           |
| `--priority`, `-P`      | `--queue` を指定した場合に、ジョブの優先度を設定します。優先度の高い Launch ジョブほど先に処理されます。優先度は高い順に critical、high、medium、low です                                                                                                      |
