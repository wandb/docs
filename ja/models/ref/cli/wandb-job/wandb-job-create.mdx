wandb run を使わずに、ソースからジョブを作成します。

ジョブには、git、code、image の 3 つのタイプがあります。

git: git のソースです。エントリポイントはパス内にあるか、メインの Python 実行ファイルを指すものとして明示的に指定します。code: requirements.txt ファイルを含む code パスです。image: Docker image です.


<div id="usage">
  ## 使い方
</div>

```bash
wandb job create JOB_TYPE PATH [OPTIONS]
```


<div id="arguments">
  ## 引数
</div>

| 引数         | 説明            | 必須 |
| :--------- | :------------ | :- |
| `JOB_TYPE` | 利用可能な説明はありません | はい |
| `PATH`     | 利用可能な説明はありません | はい |

<div id="options">
  ## オプション
</div>

| オプション                   | 説明                                                                                                                                  |
| :---------------------- | :---------------------------------------------------------------------------------------------------------------------------------- |
| `--project`, `-p`       | ジョブを一覧表示する対象のプロジェクト。                                                                                                                |
| `--entity`, `-e`        | ジョブが属する entity。                                                                                                                     |
| `--name`, `-n`          | ジョブ名。                                                                                                                               |
| `--description`, `-d`   | ジョブの説明。                                                                                                                             |
| `--alias`, `-a`         | ジョブのエイリアス。                                                                                                                          |
| `--entry-point`, `-E`   | スクリプトのエントリポイント。実行可能ファイルとエントリポイントファイルを含みます。code ジョブまたはリポジトリジョブでは必須です。`--build-context` を指定した場合、エントリポイントコマンド内のパスはビルドコンテキストからの相対パスになります。 |
| `--git-hash`, `-g`      | git ジョブのソースとして使用するコミット参照。                                                                                                           |
| `--runtime`, `-r`       | ジョブの実行に使用する Python ランタイム。                                                                                                           |
| `--build-context`, `-b` | ジョブのソースコードのルートから見たビルドコンテキストへのパス。指定した場合、Dockerfile とエントリポイントの基準パスとして使用されます。                                                          |
| `--base-image`, `-B`    | ジョブに使用するベースimage。image ジョブでは使用できません。                                                                                                   |
| `--dockerfile`, `-D`    | ジョブの Dockerfile へのパス。`--build-context` を指定した場合、Dockerfile のパスはビルドコンテキストからの相対パスになります。                                                |
