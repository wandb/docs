## <kbd>クラス</kbd> `ArtifactType`

指定されたタイプを条件とするクエリに一致するアーティファクト オブジェクトです。

**引数:**

* `client`:  W&amp;B をクエリするために使用するクライアント インスタンス。
* `entity`:  プロジェクトを所有する entity (ユーザーまたはチーム) 。
* `project`:  アーティファクト 유형をクエリする対象のプロジェクト名。
* `type_name`:  アーティファクト 유형の名。
* `attrs`:  ArtifactType を初期化するためのオプション属性です。省略した場合、オブジェクトは初期化時に W&amp;B からその属性を読み込みます。 

### <kbd>property</kbd> ArtifactType.id

アーティファクト 유형の一意の ID。

**戻り値:**

* `str`: id プロパティの値。

***

### <kbd>property</kbd> ArtifactType.name

유형の名。

**戻り値:**

* `str`: `name` プロパティの値。

***

### <kbd>method</kbd> `ArtifactType.collection`

```python
collection(name: 'str') → ArtifactCollection
```

指定した名のアーティファクト コレクションを取得します。

**引数:**

* `name` (str):  取得するアーティファクト コレクションの名。

***


### <kbd>method</kbd> `ArtifactType.collections`

```python
collections(
    filters: 'Mapping[str, Any] | None' = None,
    order: 'str | None' = None,
    per_page: 'int' = 50
) → ArtifactCollections
```

この유형に関連付けられているすべてのアーティファクト コレクションを取得します。

**引数:**

* `filters` (dict):  クエリに適用するフィルターのマッピング (省略可能) 。
* `order` (str):  結果の並び順を指定する文字列 (省略可能) 。`order` の先頭に `+` を付けると昇順 (デフォルト) になり、`-` を付けると降順になります。デフォルトの並び順は、コレクション ID の降順です。
* `per_page` (int):  1 ページあたりに取得するアーティファクト コレクションの数。デフォルトは 50 です。

***
