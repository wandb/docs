## <kbd>class</kbd> wandb.apis.public.File

W&amp;B に保存されたファイル。

W&amp;B に保存された単一のファイルを表します。ファイルのメタデータにもアクセスできます。ファイルは特定の run に関連付けられており、テキストファイル、モデルの重み、データセット、可視化、その他の Artifacts などを含めることができます。ファイルのダウンロード、削除、プロパティへのアクセスが可能です。

特定の run にログされた特定のファイルを絞り込むには、辞書で 1 つ以上の属性を指定します。次のキーを使用して検索できます。

* id (str): ファイルを含む run の ID
* name (str): ファイル名
* url (str): ファイルへのパス
* direct&#95;url (str): バケット内のファイルへのパス
* sizeBytes (int): ファイルサイズ (バイト単位)
* md5 (str): ファイルの md5
* mimetype (str): ファイルの MIME タイプ
* updated&#95;at (str): 最終更新のタイムスタンプ
* path&#95;uri (str): バケット内のファイルへのパス。現在は S3 オブジェクトと参照ファイルでのみ利用可能です

<div id="properties">
  ## プロパティ
</div>

### <kbd>プロパティ</kbd> path_uri

ストレージバケット内のファイルの URI パスを返します。

### <kbd>プロパティ</kbd> size

ファイルのサイズをバイト単位で返します。

### <kbd>method</kbd> File.download()

```python
self,
root: 'str' = '.',
replace: 'bool' = False,
exist_ok: 'bool' = False,
api: 'Api | None' = None
```

wandb サーバーから、run が以前に保存したファイルをダウンロードします。

<div id="arguments">
  ##### 引数
</div>

* `root`: ファイルの保存先となるローカルディレクトリ。デフォルトは現在の作業ディレクトリ (`"."`) です。
* `replace`: `True` の場合、ローカルファイルが存在するとダウンロード時に上書きします。デフォルトは `False` です。
* `exist_ok`: `True` の場合、ファイルがすでに存在していても `ValueError` を発生させず、`replace=True` でない限り再ダウンロードもしません。デフォルトは `False` です。
* `api`: 指定した場合、ファイルのダウンロードに使用する `Api` インスタンス。
