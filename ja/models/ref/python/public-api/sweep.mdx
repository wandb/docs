## <kbd>class</kbd> `Sweep`

sweep に関連付けられた run の集合。

**属性:**

* `runs` (Runs):  run のリスト
* `id` (str):  Sweep ID
* `project` (str):  sweep が属するプロジェクトの名前
* `config` (dict):  sweep の設定を含む辞書
* `state` (str):  sweep の状態。&quot;Finished&quot;、&quot;Failed&quot;、&quot;Crashed&quot;、&quot;Running&quot; のいずれか
* `expected_run_count` (int):  sweep に対して想定される run の数

### <kbd>プロパティ</kbd> Sweep.config

この sweep に対して使用される設定。

***

### <kbd>property</kbd> Sweep.entity

この sweep に関連付けられている entity。

**Returns:**

* `str`: entity プロパティの値。

***

### <kbd>property</kbd> Sweep.expected_run_count

sweep 内で想定される run の数を返します。run が無限に実行される場合は None を返します。

**戻り値:**

* `int | None`: expected&#95;run&#95;count プロパティの値。

***

### <kbd>property</kbd> Sweep.name

sweep の名前。

以下の優先順位で、存在するもののうち最初の名前を返します:

1. ユーザーが編集した表示名 2. 作成時に設定された名前 3. sweep ID

***

### <kbd>プロパティ</kbd> Sweep.order

sweep の順序キーを返します。

***

### <kbd>property</kbd> Sweep.path

プロジェクトのパスを返します。

このパスは entity、プロジェクト名、および sweep ID から成るリストです。

***

### <kbd>property</kbd> Sweep.url

sweep の URL。

sweep の URL は、entity、プロジェクト、&quot;sweeps&quot; という語、および sweep ID.run&#95;id から生成されます。SaaS ユーザーの場合、形式は `https://wandb.ai/entity/project/sweeps/sweeps_ID` となります。

***

### <kbd>property</kbd> Sweep.username

非推奨です。代わりに `Sweep.entity` を使用してください。

**戻り値:**

* `str`: username プロパティの値。

***

### <kbd>method</kbd> `Sweep.best_run`

```python
best_run(order=None)
```

設定で定義されたメトリクス、または引数で指定された並び順でソートした結果から、最良の run を返します。

### <kbd>classmethod</kbd> `Sweep.get`

```python
get(
    client: 'RetryingClient',
    entity: 'str | None' = None,
    project: 'str | None' = None,
    sid: 'str | None' = None,
    order: 'str | None' = None,
    query: 'Document | None' = None,

```

クラウドバックエンドに対してクエリを実行します。

**引数:**

* `client`:  クエリを実行する際に使用するクライアント。
* `entity`:  プロジェクトの所有者である entity  (ユーザー名またはチーム) 。
* `project`:  sweep を取得するプロジェクト名。
* `sid`:  クエリ対象の sweep ID。
* `order`:  sweep に含まれる runs が返される順序。
* `query`:  クエリを実行するために使用するクエリ。
* `**kwargs`:  クエリに渡す追加のキーワード引数。
