### <kbd>function</kbd> `setup`

```python
setup(settings: 'Settings | None' = None) → _WandbSetup
```

現在のプロセスとその子プロセスで使用するために W&amp;B を準備します。

これは `wandb.init()` によって暗黙的に呼び出されるため、通常は無視しても問題ありません。

複数のプロセスで wandb を使用する場合、子プロセスを起動する前に親プロセスで `wandb.setup()` を呼び出すと、パフォーマンスとリソースの利用効率が向上する場合があります。

`wandb.setup()` は `os.environ` を変更するため、子プロセスが変更された環境変数を継承することが重要です。

`wandb.teardown()` も参照してください。

**引数:**

* `settings`:  グローバルに適用する設定。これらは後続の `wandb.init()` 呼び出しによって上書きできます。

**例:**

```python
import multiprocessing

import wandb


def run_experiment(params):
    with wandb.init(config=params):
         # 実験を実行する
         pass


if __name__ == "__main__":
    # バックエンドを起動してグローバル設定を行う
    wandb.setup(settings={"project": "my_project"})

    # 実験パラメーターを定義する
    experiment_params = [
         {"learning_rate": 0.01, "epochs": 10},
         {"learning_rate": 0.001, "epochs": 20},
    ]

    # 複数のプロセスを起動し、それぞれ個別の実験を実行する
    processes = []
    for params in experiment_params:
         p = multiprocessing.Process(target=run_experiment, args=(params,))
         p.start()
         processes.append(p)

    # すべてのプロセスが完了するまで待機する
    for p in processes:
         p.join()

    # 省略可能: バックエンドを明示的にシャットダウンする
    wandb.teardown()
```
