## <kbd>class</kbd> `Settings`

W&B SDK の設定。

このクラスは W&B SDK の設定を管理し、すべての設定の型安全性と検証を保証します。設定には属性としてアクセスでき、プログラムによる設定、環境変数（`WANDB_` プレフィックス）、および設定ファイルを通じて指定できます。

設定は次の 3 つのカテゴリに分類されます。

1. Public settings: ユーザーが安全に変更して、自身のニーズに合わせて W&B の動作をカスタマイズできるコアな設定オプション。
2. Internal settings: `x_` プレフィックスが付いた、SDK の低レベルな動作を扱う設定。これらの設定は主に内部利用やデバッグのためのもので、変更は可能ですが、public API の一部とは見なされず、将来のバージョンで予告なく変更されることがあります。
3. Computed settings: 他の設定や環境から自動的に導出される読み取り専用の設定。

**引数:**

* `allow_offline_artifacts` (bool): オフライン モードでテーブル アーティファクトの同期を許可するかどうかを制御するフラグです。
  以前の挙動に戻すには、この値を False に設定します。

* `allow_val_change` (bool): 一度設定された後でも `Config` の値を変更できるようにするフラグ。

* `anonymous` (オブジェクト): 非推奨であり、今後削除されます。

* `api_key` (Optional[str]): W&amp;B の APIキー。

* `azure_account_url_to_access_key` (Optional[Dict[str, str]]): Azure アカウント URL と、それに対応するアクセスキーとのマッピングを表す Azure インテグレーション用の設定です。

* `app_url_override` (Optional[str]): W&amp;B UI の `app` URL を上書きするための引数です。
  通常は `app_url` は `base_url` に基づいて計算されますが、これを使うと明示的に設定できます。
  対応する環境変数は WANDB&#95;APP&#95;URL です。

* `base_url` (str): データ同期用の W&amp;B バックエンドの URL。

* `code_dir` (Optional[str]): W&amp;B で追跡するコードを含むディレクトリ。

* `config_paths` (Optional[Sequence]): 設定を `Config` オブジェクトに読み込むためのファイルパス。

* `console` (Literal[&#39;auto&#39;, &#39;off&#39;, &#39;wrap&#39;, &#39;redirect&#39;, &#39;wrap&#95;raw&#39;, &#39;wrap&#95;emu&#39;]): 適用するコンソール出力キャプチャの種類。
  指定可能な値は次のとおりです:
  * &quot;auto&quot; - システム環境と設定に基づいてコンソールキャプチャの方法を自動的に選択します。
  * &quot;off&quot; - コンソールキャプチャを無効にします。
  * &quot;redirect&quot; - 出力をキャプチャするために低レベルのファイルディスクリプターをリダイレクトします。
  * &quot;wrap&quot; - sys.stdout/sys.stderr の write メソッドをオーバーライドします。システムの状態に応じて &quot;wrap&#95;raw&quot; または &quot;wrap&#95;emu&quot; のいずれかにマッピングされます。
  * &quot;wrap&#95;raw&quot; - &quot;wrap&quot; と同様ですが、エミュレーターを介さずに生の出力を直接キャプチャします。`wrap` 設定から自動的に決定されるものであり、手動で設定すべきではありません。
  * &quot;wrap&#95;emu&quot; - &quot;wrap&quot; と同様ですが、エミュレーターを介して出力をキャプチャします。`wrap` 設定から自動的に決定されるものであり、手動で設定すべきではありません。

* `console_multipart` (bool): マルチパートコンソールロギングを有効にします。
  True の場合、SDK はコンソール出力を 1 つの `output.log` ではなく、
  `logs/` ディレクトリ配下のタイムスタンプ付きファイルに書き込みます。
  各パートはクローズされるとすぐにアップロードされるため、run がアクティブな間、
  ユーザーはログへリアルタイムにアクセスできます。ロールオーバーのタイミングは
  `console_chunk_max_bytes` および/または `console_chunk_max_seconds` によって制御されます。
  両方の上限が `0` の場合、すべてのログは run の終了時に一括でアップロードされます。
  注: アップロードされたチャンクは変更できません。以前の行を変更する端末の制御シーケンス
  （キャリッジリターンを使うプログレスバーなど）は、現在のチャンクにのみ影響します。

* `console_chunk_max_bytes` (int): マルチパートコンソールログのサイズに基づくロールオーバーのしきい値（バイト単位）。現在のパートがこのサイズに達したときに、新しいコンソールログファイルを開始します。`console_multipart` が `True` の場合にのみ有効です。`console_chunk_max_seconds` と組み合わせて使用でき、いずれかの制限に先に達した時点でロールオーバーが発生します。`0` を指定すると、サイズに基づく制限は無効になります。

* `console_chunk_max_seconds` (int): マルチパートコンソールログに対する時間ベースのロールオーバーのしきい値（秒単位）。
  現在のパートが開始してから指定した秒数が経過すると、新しいコンソールログファイルを開始します。`console_multipart` が
  `True` である必要があります。`console_chunk_max_bytes` と併用できます。先に到達した制限によってパートが閉じられます。
  `0` を指定すると、時間ベースの制限は無効になります。

* `credentials_file` (str): 一時アクセス トークンを書き込むファイルのパス。

* `disable_code` (bool): コードのキャプチャを無効にするかどうかを指定します。

* `disable_git` (bool): git の状態の取得を無効化するかどうか。

* `disable_job_creation` (bool): W&amp;B Launch の job アーティファクトの作成を無効にするかどうかを指定します。

* `docker` (Optional[str]): スクリプトを実行するために使用する Docker イメージ。

* `email` (Optional[str]): ユーザーのメールアドレス。

* `entity` (Optional[str]): ユーザー やチームなどの W&amp;B entity。

* `organization` (Optional[str]): W&amp;B の組織。

* `force` (bool): `wandb.login()` に `force` フラグを渡すかどうかを指定します。

* `fork_from` (Optional[RunMoment]): 以前の run の実行において、どの時点から fork するかを指定します。
  その時点は run ID、メトリクス、およびその値によって定義されます。
  現在は、メトリクス &#39;&#95;step&#39; のみがサポートされています。

* `git_commit` (Optional[str]): run に関連付ける git のコミットハッシュ。

* `git_remote` (str): run と関連付ける git リモート。

* `git_remote_url` (Optional[str]): Git リモートリポジトリの URL。

* `git_root` (Optional[str]): git リポジトリのルートディレクトリ。

* `host` (Optional[str]): スクリプトを実行しているマシンのホスト名。

* `http_proxy` (Optional[str]): W&amp;B への HTTP リクエストに使用するカスタム プロキシサーバー。

* `https_proxy` (Optional[str]): W&amp;B への HTTPS リクエスト用のカスタムプロキシサーバー。

* `identity_token_file` (Optional[str]): 認証に使用する ID トークン (JWT) が格納されているファイルへのパス。

* `ignore_globs` (Sequence): アップロード対象から除外するファイルを指定するための、`files_dir` を基準とした Unix のグロブパターン。

* `init_timeout` (float): タイムアウトになる前に `wandb.init` 呼び出しの完了を待機する最大時間（秒）。

* `insecure_disable_ssl` (bool): SSL 検証を安全性を無視して無効化するかどうか。

* `job_name` (Optional[str]): スクリプトを実行する Launch ジョブの名前。

* `job_source` (Optional[Literal[&#39;repo&#39;, &#39;artifact&#39;, &#39;image&#39;]]): Launch のソース種別。

* `label_disable` (bool): 自動ラベリング機能を無効にするかどうかを指定します。

* `launch_config_path` (Optional[str]): Launch の設定ファイルへのパス。

* `login_timeout` (Optional[float]): login 操作の完了を待機する最大時間（秒）。

* `mode` (Literal[&#39;online&#39;, &#39;offline&#39;, &#39;shared&#39;, &#39;disabled&#39;, &#39;dryrun&#39;, &#39;run&#39;]): W&amp;B のロギングと同期の動作モード。

* `notebook_name` (Optional[str]): Jupyter などのノートブック環境で実行している場合のノートブック名。

* `program` (Optional[str]): 存在する場合は、run を生成したスクリプトへのパス。

* `program_abspath` (Optional[str]): run を作成したスクリプトへの、ルートリポジトリディレクトリからの絶対パス。
  ルートリポジトリディレクトリは、存在する場合は .git ディレクトリを含むディレクトリとして定義されます。存在しない場合は、現在のカレントディレクトリになります。

* `program_relpath` (Optional[str]): run を作成したスクリプトへの相対パス。

* `project` (Optional[str]): W&amp;B のプロジェクト ID。

* `quiet` (bool): 不要な出力を抑制するフラグ。

* `reinit` (Union[Literal[&#39;default&#39;, &#39;return&#95;previous&#39;, &#39;finish&#95;previous&#39;, &#39;create&#95;new&#39;], bool]): 実行中の run があるときに `wandb.init()` が呼び出された場合の動作を指定します。
  オプション:
  * &quot;default&quot;: ノートブックでは &quot;finish&#95;previous&quot; を、それ以外では
    &quot;return&#95;previous&quot; を使用します。
  * &quot;return&#95;previous&quot;: まだ終了していない最新の run を返します。
    これは `wandb.run` を更新しません。&quot;create&#95;new&quot; オプションを参照してください。
  * &quot;finish&#95;previous&quot;: すべてのアクティブな run を終了し、その後新しい run を返します。
  * &quot;create&#95;new&quot;: 他のアクティブな run を変更せずに新しい run を作成します。
    `wandb.run` や `wandb.log` のようなトップレベル関数は更新されません。
    このため、グローバルな run に依存している一部の古いインテグレーションは動作しない場合があります。
    ブール値を指定することもできますが、これは非推奨です。False は
    &quot;return&#95;previous&quot; と同じで、True は &quot;finish&#95;previous&quot; と同じ意味になります。

* `relogin` (bool): 再ログインを強制するフラグ。

* `resume` (Optional[Literal[&#39;allow&#39;, &#39;must&#39;, &#39;never&#39;, &#39;auto&#39;]]): run の再開時の挙動を指定します。
  Options:
  * &quot;must&quot;: 同じ ID を持つ既存の run から再開します。そのような run が存在しない場合はエラーになります。
  * &quot;allow&quot;: 同じ ID を持つ既存の run からの再開を試みます。見つからない場合は新しい run が作成されます。
  * &quot;never&quot;: 常に新しい run を開始します。同じ ID を持つ run がすでに存在する場合はエラーになります。
  * &quot;auto&quot;: 同じマシン上で直近に失敗した run から自動的に再開します。

* `resume_from` (Optional[RunMoment]): 以前の run 実行のどの時点から再開するかを指定します。
  この時点は、run ID、メトリクス、およびその値によって定義されます。
  現在は、メトリクス &#39;&#95;step&#39; のみがサポートされています。

* `root_dir` (str): すべての run 関連パスの基準とするルートディレクトリ。
  特に、wandb ディレクトリおよび run ディレクトリのパスを決定するために使用されます。

* `run_group` (Optional[str]): 関連する run をまとめるためのグループ識別子。
  UI 上で run をグループ化するために使用します。

* `run_id` (Optional[str]): run の ID。

* `run_job_type` (Optional[str]): 実行するジョブの種類（例: トレーニング、評価）。

* `run_name` (Optional[str]): run の人間が判読可能な名前。

* `run_notes` (Optional[str]): run に関する追加のメモや説明。

* `run_tags` (Optional[Tuple[str, Ellipsis]]): run の整理およびフィルタリングのために関連付けるタグ。

* `sagemaker_disable` (bool): SageMaker 固有の機能を無効にするフラグ。

* `save_code` (Optional[bool]): run に関連付けられたコードを保存するかどうか。

* `settings_system` (Optional[str]): システム全体用設定ファイルへのパス。

* `max_end_of_run_history_metrics` (int): run の終了時に表示する履歴メトリクスのスパークラインの最大数。

* `max_end_of_run_summary_metrics` (int): run の終了時に表示されるサマリー メトリクスの最大数。

* `show_errors` (bool): エラーメッセージを表示するかどうかを指定します。

* `show_info` (bool): 情報メッセージを表示するかどうかを指定します。

* `show_warnings` (bool): 警告メッセージを表示するかどうかを指定します。

* `silent` (bool): すべての出力を抑制するフラグ。

* `strict` (Optional[bool]): 検証およびエラー チェックのための strict モードを有効にするかどうか。

* `summary_timeout` (int): サマリー処理がタイムアウトするまでの待機時間（秒）。

* `sweep_id` (Optional[str]): この run が属している sweep の識別子。

* `sweep_param_path` (Optional[str]): sweep パラメーター設定のパス。

* `symlink` (bool): シンボリックリンクを使用するかどうか (デフォルトは Windows 以外で True)。

* `sync_tensorboard` (Optional[bool]): TensorBoard ログを W&amp;B と同期するかどうか。

* `table_raise_on_max_row_limit_exceeded` (bool): テーブルの行数制限を超えたときに例外を発生させるかどうかを指定します。

* `use_dot_wandb` (Optional[bool]): run データ用に、隠しディレクトリ `.wandb` を使用するか、表示される `wandb` ディレクトリを使用するかを指定します。
  `True` の場合、SDK は `.wandb` を使用します。`False` の場合は `wandb` を使用します。
  設定されていない場合、既に `.wandb` が存在していれば `.wandb`、存在しなければ `wandb` がデフォルトになります。

* `username` (Optional[str]): ユーザー名。

* `x_disable_meta` (bool): システム メタデータの収集を無効化するフラグ。

* `x_disable_stats` (bool): システム メトリクスの収集を無効化するフラグ。

* `x_extra_http_headers` (Optional[Dict[str, str]]): すべての送信 HTTP リクエストに追加するヘッダー。

* `x_label` (Optional[str]): run 用に収集されるシステム メトリクスおよびコンソールログに割り当てるラベル。
  これはフロントエンド でデータをグループ化するために使用され、分散トレーニング ジョブ内の異なるプロセスからのデータを区別するために利用できます。

* `x_primary` (bool): 内部の wandb ファイルおよびメタデータを保存するかどうかを制御します。
  分散環境では、メインのロギングをプライマリ プロセスが担当し、システム メトリクスとログだけが必要な場合に、
  セカンダリ プロセスによるファイルの上書きを防ぐのに役立ちます。

* `x_save_requirements` (bool): requirements ファイルを保存するかどうかを指定するフラグ。

* `x_server_side_derived_summary` (bool): サーバー側で history から summary を自動計算するように委譲するためのフラグです。
  これは、ユーザーが提供する summary の更新を無効化するものではありません。

* `x_service_wait` (float): wandb-core の内部サービスが起動するまで待機する時間（秒）。

* `x_skip_transaction_log` (bool): run のイベントをトランザクションログへの保存からスキップするかどうかを指定する設定です。
  これはオンラインの run にのみ適用されます。ディスクに書き込まれる
  データ量を減らすために使用できます。
  ただし、復旧に関する保証がなくなるため、
  注意して使用してください。

* `x_stats_sampling_interval` (float): システムモニターのサンプリング間隔（秒単位で指定）。

* `x_stats_dcgm_exporter` (Optional[str]): Nvidia DCGM メトリクスを取得するためのエンドポイント。
  オプション:
  * Prometheus の `/api/v1/query` エンドポイントへのクエリから DCGM 関連のメトリクスを抽出します。
    クラスター内の異なるノード上で動作している DCGM Exporter のインスタンスが報告するメトリクスを、Prometheus を使って集約するのは一般的な運用方法です。
  * TODO: DCGM Exporter の `/metrics` エンドポイントからメトリクスを直接パースする。
    例:
  * `http://localhost:9400/api/v1/query?query=DCGM_FI_DEV_GPU_TEMP{node="l1337", cluster="globular"}`.

* `x_stats_open_metrics_endpoints` (Optional[Dict[str, str]]): システムのメトリクスを監視するための OpenMetrics の `/metrics` エンドポイント。

* `x_stats_open_metrics_filters` (Union[Dict[str, Dict[str, str]], Sequence, None]): OpenMetrics の `/metrics` エンドポイントから収集されたメトリクスに適用するフィルター。
  2 つの形式をサポートします。
  * `{"metric regex pattern, including endpoint name as prefix": {"label": "label value regex pattern"}}`
  * `("metric regex pattern 1", "metric regex pattern 2", ...)`

* `x_stats_open_metrics_http_headers` (Optional[Dict[str, str]]): OpenMetrics リクエストに追加する HTTP ヘッダーを指定します。

* `x_stats_disk_paths` (Optional[Sequence]): ディスク使用量を監視するために使用するシステムパス。

* `x_stats_cpu_count` (Optional[int]): システムの CPU 数。
  この値を設定すると、run メタデータで自動検出される値を上書きします。

* `x_stats_cpu_logical_count` (Optional[int]): 論理 CPU の数。
  この値を設定すると、run メタデータで自動検出された値を上書きします。

* `x_stats_gpu_count` (Optional[int]): GPU デバイス数。
  設定された場合、この値は run メタデータで自動検出された値を上書きします。

* `x_stats_gpu_type` (Optional[str]): GPU デバイスの種類。
  指定すると、run メタデータ内で自動検出された値を上書きします。

* `x_stats_gpu_device_ids` (Optional[Sequence]): 監視対象とする GPU デバイスのインデックス。
  設定しない場合、システムモニターはすべての GPU のメトリクスを収集します。
  CUDA/ROCm のデバイス列挙と一致する 0 から始まるインデックスであることを前提とします。

* `x_stats_track_process_tree` (bool): `x_stats_pid` を起点として、プロセスツリー全体のリソース使用状況を監視します。
  `True` の場合、システムモニターは PID `x_stats_pid` を持つプロセスと、そのすべての子孫プロセスから RSS、CPU%、スレッド数を集約します。
  これはパフォーマンスにオーバーヘッドが生じる可能性があるため、デフォルトでは無効になっています。

* `x_sync_dir_suffix` (str): run のディレクトリ名 (sync&#95;dir) に追加するサフィックス。
  これは名前の競合を避けるために wandb.init() で設定されます。
  設定されている場合、デフォルト名の末尾にハイフンで連結されます。

* `x_update_finish_state` (bool): このプロセスがサーバー上で run の最終状態を更新してよいかどうかを示すフラグ。
  分散トレーニングでは、最終状態をメインプロセスだけが決定するようにしたい場合は False に設定します。
