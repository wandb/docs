このページでは、W&amp;B SDK によってトラッキングされるシステムメトリクスの詳細情報を提供します。

<Note>
  `wandb` は 15 秒ごとにシステムメトリクスを自動的にログします。
</Note>

<div id="view-system-metrics">
  ## システムメトリクスの表示
</div>

W&amp;B App または `wandb beta leet` ターミナル UI を使用してシステムメトリクスを表示および監視できます。

<Tabs>
  <Tab title="App">
    W&amp;B App でシステムメトリクスを表示するには、次の手順を実行します。

    1. W&amp;B App で自分のプロジェクトにアクセスします。
    2. **Runs** 表から run を 1 つ選択します。
    3. Workspace 内で、次のチャートを表示する **System** セクションを探します。
       * GPU 利用率とメモリ
       * CPU 使用率
       * メモリ使用量
       * ディスク I/O
       * ネットワークトラフィック

    Workspace にパネルを追加することで、表示するシステムメトリクスをカスタマイズできます。可視化の作成とカスタマイズの詳細は、[Panels](/ja/models/app/features/panels/) を参照してください。
  </Tab>

  <Tab title="LEET">
    `wandb beta leet` ターミナル UI を使用して run のシステムメトリクスをターミナルで表示するには、次の手順を実行します。

    1. run をローカルのスクリプトから開始した場合は、コードを実行したディレクトリに移動します。そこには、run ごとのサブディレクトリと `latest-run/` シンボリックリンクを含む `wandb/` ディレクトリがあります。各 run ディレクトリには、`run-<run-ID>.wandb` という形式のトランザクションログが含まれています。

       run をローカルで開始せず、代わりに `.wandb` トランザクションログファイルをダウンロードした場合は、その場所をメモしておきます。
    2. 次のいずれかのコマンドで `wandb beta leet` を起動します。

       ```bash
       # ./wandb/latest-run/ に保存されている最新の run を表示
       wandb beta leet

       # run ディレクトリを指定
       wandb beta leet ./wandb/run-20250813_124246-n67z9ude

       # .wandb ファイルを指定
       wandb beta leet ./wandb/run-20250813_124246-n67z9ude/run-n67z9ude.wandb
       ```

    ![LEET ターミナル UI で run を表示している様子を示すスクリーンショット](/images/leet/leet-tui.png)

    LEET はシステムメトリクスを **右サイドバー** に表示し、次の情報を示します。

    * GPU 利用率 (%) とメモリ使用量 (GB)
    * CPU 使用率
    * RAM 使用量 (GB)
    * ディスク I/O
    * ネットワークアクティビティ

    次のキーボードショートカットを使用できます。

    * `h` または `?` - すべてのキーボードショートカットを表示
    * `/` - パターンでメトリクスをフィルター
    * `[` / `]` - 左/右サイドバーの表示を切り替え
    * `n` / `N` - メトリクス ページ間を移動
    * `q` / `CMD+C` - 終了

    詳細は [`wandb beta leet`](/ja/models/ref/cli/wandb-beta/wandb-beta-leet) を参照してください。
  </Tab>
</Tabs>

<div id="cpu">
  ## CPU
</div>

<div id="process-cpu-percent-cpu">
  ### プロセス CPU 使用率 (CPU)
</div>

プロセスによる CPU 使用率の割合で、使用可能な CPU 数で正規化されています。

W&amp;B はこのメトリクスに `cpu` タグを付与します。

<div id="process-cpu-threads">
  ### プロセス CPU スレッド数
</div>

プロセスが利用するスレッド数。

W&amp;B はこのメトリクスに `proc.cpu.threads` というタグを付与します。

<div id="disk">
  ## ディスク
</div>

デフォルトでは、`/` パスの使用量メトリクスが収集されます。監視するパスを設定するには、次の設定を使用します。

```python
run = wandb.init(
    settings=wandb.Settings(
        x_stats_disk_paths=("/System/Volumes/Data", "/home", "/mnt/data"),
    ),
)
```

<div id="disk-usage-percent">
  ### ディスク使用率
</div>

指定したパスにおけるシステム全体のディスク使用率 (パーセンテージ) を表します。

W&amp;B はこのメトリクスに `disk.{path}.usagePercent` タグを付与します。

<div id="disk-usage">
  ### ディスク使用量
</div>

指定されたパスにおけるシステム全体のディスク使用量をギガバイト (GB) 単位で表します。
액세스可能なパスがサンプリングされ、各パスのディスク使用量 (GB) がサンプルに追加されます。

W&amp;B はこのメトリクスに `disk.{path}.usageGB` タグを부여します。

<div id="disk-in">
  ### ディスク In
</div>

システム全体のディスク読み取り量の合計をメガバイト (MB) 単位で示します。
最初のサンプル取得時に、初期のディスク読み取りバイト数が記録されます。以降のサンプルでは、現在の読み取りバイト数と初期値との差分が計算されます。

W&amp;B はこのメトリクスに `disk.in` タグを付与します。

<div id="disk-out">
  ### ディスク Out
</div>

システム全体のディスク書き込み量の合計をメガバイト (MB) 単位で表します。
[ディスク In]() と同様に、最初のサンプル取得時に初期のディスク書き込みバイト数が記録されます。以降のサンプルでは、現在の書き込みバイト数と初期値との差分が計算されます。

W&amp;B はこのメトリクスに `disk.out` タグを付与します。

<div id="memory">
  ## メモリ
</div>

<div id="process-memory-rss">
  ### プロセス メモリ RSS
</div>

プロセスのメモリ常駐セットサイズ (RSS) をメガバイト (MB) で表します。RSS は、プロセスが占有するメモリのうち、主メモリ (RAM) に保持されている部分です。

W&amp;B は、このメトリクスに `proc.memory.rssMB` というタグを付与します。

<div id="process-memory-percent">
  ### プロセスのメモリ使用率 (パーセント)
</div>

사용 가능하다なメモリ全体に対する、プロセスのメモリ使用量のパーセンテージを示します。

W&amp;B はこのメトリクスに `proc.memory.percent` タグを부여します。

<div id="memory-percent">
  ### メモリ使用率
</div>

利用可能なメモリの合計に対するシステムのメモリ使用量の割合を表します。

W&amp;Bはこのメトリクスに `memory_percent` タグを割り当てます。

<div id="memory-available">
  ### 利用可能メモリ
</div>

システムの利用可能メモリの合計をメガバイト (MB) で示します。

W&amp;B はこのメトリクスに `proc.memory.availableMB` タグを割り当てます。

<div id="network">
  ## ネットワーク
</div>

<div id="network-sent">
  ### ネットワーク Sent
</div>

ネットワーク経由で送信された合計バイト数を表します。
メトリクスが最初に初期化された際に、初期の送信バイト数が記録されます。以降のサンプルでは、現在の送信バイト数と初期値の差を計算します。

W&amp;B はこのメトリクスに `network.sent` タグを부여します。

<div id="network-received">
  ### ネットワーク受信
</div>

ネットワーク経由で受信した合計バイト数を示します。
[ネットワーク送信]() と同様に、初期の受信バイト数はメトリクスが最初に初期化されたときに記録されます。以降のサンプルでは、現在の受信バイト数と初期値との差分を計算します。

W&amp;B はこのメトリクスに `network.recv` タグを 부여します。

<div id="nvidia-gpu">
  ## NVIDIA GPU
</div>

以下で説明するメトリクスに加えて、プロセスおよび／またはその子プロセスが特定の GPU を使用している場合、W&amp;B は対応するメトリクスを `gpu.process.{gpu_index}.{metric_name}` として取得します。

<div id="gpu-memory-utilization">
  ### GPU メモリ使用率
</div>

各 GPU のメモリ使用率をパーセントで表します。

W&amp;B はこのメトリクスに `gpu.{gpu_index}.memory` タグを割り当てます。

<div id="gpu-memory-allocated">
  ### GPU 割り当て済みメモリ
</div>

各 GPU について、使用可能な総メモリに対する割り当て済み GPU メモリの割合を示します。

W&amp;B はこのメトリクスに `gpu.{gpu_index}.memoryAllocated` タグを付与します。

<div id="gpu-memory-allocated-bytes">
  ### GPU メモリ割り当てバイト数
</div>

各 GPU ごとに、バイト単位で割り当てられた GPU メモリを示します。

W&amp;B はこのメトリクスに `gpu.{gpu_index}.memoryAllocatedBytes` タグを割り当てます。

<div id="gpu-utilization">
  ### GPU 使用率
</div>

各 GPU の使用率をパーセントで示します。

W&amp;B はこのメトリクスに `gpu.{gpu_index}.gpu` タグを付与します。

<div id="gpu-temperature">
  ### GPU 温度
</div>

各 GPU の温度 (摂氏) です。

W&amp;B はこのメトリクスに `gpu.{gpu_index}.temp` タグを 부여します。

<div id="gpu-power-usage-watts">
  ### GPU 電力使用量 (ワット)
</div>

各 GPU の電力使用量 (ワット) を示します。

W&amp;B はこのメトリクスに `gpu.{gpu_index}.powerWatts` タグを割り当てます。

<div id="gpu-power-usage-percent">
  ### GPU 電力使用率 (パーセント)
</div>

各 GPU について、その電力容量に対する GPU の電力使用量をパーセントで示します。

W&amp;B はこのメトリクスに `gpu.{gpu_index}.powerPercent` タグを부여します。

<div id="gpu-sm-clock-speed">
  ### GPU SM クロック速度
</div>

GPU 上の Streaming Multiprocessor (SM) のクロック速度を MHz 単位で表します。このメトリクスは、計算タスクを担う GPU コア内の処理速度を示す指標です。

W&amp;B はこのメトリクスに `gpu.{gpu_index}.smClock` タグを付与します。

<div id="gpu-memory-clock-speed">
  ### GPU メモリ クロック速度
</div>

GPU メモリのクロック速度を MHz 単位で表します。これは、GPU メモリと処理コア間のデータ転送速度に影響します。

W&amp;B はこのメトリクスに `gpu.{gpu_index}.memoryClock` タグを부여します。

<div id="gpu-graphics-clock-speed">
  ### GPU Graphics クロック速度
</div>

GPUでのグラフィックス表示処理におけるベースクロック速度を表し、MHz単位で示されます。このメトリクスは、可視化や表示タスク中のパフォーマンスを反映することがよくあります。

W&amp;B はこのメトリクスに `gpu.{gpu_index}.graphicsClock` タグを付与します。

<div id="gpu-corrected-memory-errors">
  ### GPU の修正済みメモリエラー
</div>

GPU で発生したメモリエラーのうち、エラーチェックプロトコルによって W&amp;B が自動的に修正した件数をトラッキングします。これは、回復可能なハードウェアの問題を示します。

W&amp;B はこのメトリクスに `gpu.{gpu_index}.correctedMemoryErrors` タグを付与します。

<div id="gpu-uncorrected-memory-errors">
  ### GPU の未修正メモリエラー
</div>

GPU 上で W&amp;B が未修正のままのメモリエラー数をトラッキングします。これは、処理の信頼性に影響する可能性がある、回復不能なエラーを示します。

W&amp;B はこのメトリクスに `gpu.{gpu_index}.unCorrectedMemoryErrors` タグを부여します。

<div id="gpu-encoder-utilization">
  ### GPU Encoder の使用率
</div>

GPU のビデオエンコーダーの使用率 (パーセンテージ) を表します。エンコード タスク (たとえば、動画のレンダリング) の実行時に、エンコーダーにかかっている負荷を示します。

W&amp;B は、このメトリクスに `gpu.{gpu_index}.encoderUtilization` タグを付与します。

<div id="graphcore-ipu">
  ## Graphcore IPU
</div>

Graphcore IPU (Intelligence Processing Unit) は、機械インテリジェンス向けの処理に特化して設計された独自のハードウェアアクセラレーターです。

<div id="ipu-device-metrics">
  ### IPU デバイスのメトリクス
</div>

これらのメトリクスは、特定の IPU デバイスに関する各種統計を表します。各メトリクスは、デバイス ID (`device_id`) とメトリクス キー (`metric_key`) によって識別されます。W&amp;B は、このメトリクスに `ipu.{device_id}.{metric_key}` タグを付与します。

メトリクスは、Graphcore の `gcipuinfo` バイナリと連携する独自の `gcipuinfo` ライブラリを使用して抽出されます。`sample` method は、プロセス ID (`pid`) に関連付けられた各 IPU デバイスのこれらのメトリクスを取得します。冗長なデータのログを避けるため、時間の経過とともに変化するメトリクス、またはデバイスのメトリクスを初めて取得したときのみログされます。

各メトリクスでは、method `parse_metric` を使用して、生の文字列表現からメトリクス値を抽出します。その後、メトリクスは `aggregate` method を使用して複数のサンプルにわたって集計されます。

以下に、利用可能なメトリクスとその単位を示します。

* **平均ボード温度** (`average board temp (C)`): IPU ボードの温度 (摂氏) 。
* **平均ダイ温度** (`average die temp (C)`): IPU ダイの温度 (摂氏) 。
* **クロック速度** (`clock (MHz)`): IPU のクロック速度 (MHz) 。
* **IPU 電力** (`ipu power (W)`): IPU の消費電力 (ワット) 。
* **IPU 使用率** (`ipu utilisation (%)`): IPU 使用率の割合。
* **IPU セッション使用率** (`ipu utilisation (session) (%)`): 現在のセッションに固有の IPU 使用率の割合。
* **データリンク速度** (`speed (GT/s)`): 1 秒あたりのギガトランスファー数で表したデータ転送速度。

<div id="google-cloud-tpu">
  ## Google Cloud TPU
</div>

Tensor Processing Units (TPU) は、機械学習ワークロードを高速化するために用いられる、Google が独自に開発した ASIC (特定用途向け集積回路) です。

<div id="tpu-memory-usage">
  ### TPU メモリ使用量
</div>

TPU コアごとの現在の High Bandwidth Memory 使用量をバイト単位で示します。

W&amp;B はこのメトリクスに `tpu.{tpu_index}.memoryUsageBytes` タグを부여します。

<div id="tpu-memory-usage-percentage">
  ### TPU メモリ使用率
</div>

現在の TPU コアごとの High Bandwidth Memory 使用率 (%) です。

W&amp;B はこのメトリクスに `tpu.{tpu_index}.memoryUsageBytes` タグを付与します。

<div id="tpu-duty-cycle">
  ### TPU デューティサイクル
</div>

TPU デバイスごとの TensorCore デューティサイクルの割合です。アクセラレータの TensorCore がサンプル期間中にアクティブに処理を行っていた時間の割合をトラッキングします。値が大きいほど、TensorCore の使用率が高いことを意味します。

W&amp;B はこのメトリクスに `tpu.{tpu_index}.dutyCycle` タグを부여します。

<div id="aws-trainium">
  ## AWS Trainium
</div>

[AWS Trainium](https://aws.amazon.com/machine-learning/trainium/) は、機械学習ワークロードの高速化に重点を置いた、AWS が提供する専用ハードウェアプラットフォームです。AWS の `neuron-monitor` を使用して、AWS Trainium のメトリクスを取得します。

<div id="trainium-neuron-core-utilization">
  ### Trainium Neuron Core 使用率
</div>

各 NeuronCore の使用率をコア単位で示した割合です。

W&amp;B はこのメトリクスに `trn.{core_index}.neuroncore_utilization` タグを付与します。

<div id="trainium-host-memory-usage-total">
  ### Trainium ホストの合計メモリ使用量
</div>

ホスト上の合計メモリ消費量をバイト単位で示します。

W&amp;B はこのメトリクスに `trn.host_total_memory_usage` タグを부여します。

<div id="trainium-neuron-device-total-memory-usage">
  ### Trainium Neuron デバイスの合計メモリ使用量
</div>

Neuron デバイス上の合計メモリ使用量をバイト単位で示します。

W&amp;B は、このメトリクスに `trn.neuron_device_total_memory_usage)` タグを 부여します。

<div id="trainium-host-memory-usage-breakdown">
  ### Trainium ホストのメモリ使用状況の内訳:
</div>

以下は、ホスト上のメモリ使用状況の内訳です。

* **アプリケーション メモリ** (`trn.host_total_memory_usage.application_memory`): アプリケーションで使用されるメモリ。
* **定数** (`trn.host_total_memory_usage.constants`): 定数に使用されるメモリ。
* **DMA バッファ** (`trn.host_total_memory_usage.dma_buffers`): Direct Memory Access バッファに使用されるメモリ。
* **テンソル** (`trn.host_total_memory_usage.tensors`): テンソルに使用されるメモリ。

<div id="trainium-neuron-core-memory-usage-breakdown">
  ### Trainium Neuron Core メモリ使用量の内訳
</div>

各 NeuronCore のメモリ使用量の詳細:

* **定数** (`trn.{core_index}.neuroncore_memory_usage.constants`)
* **モデルコード** (`trn.{core_index}.neuroncore_memory_usage.model_code`)
* **モデル共有スクラッチパッド** (`trn.{core_index}.neuroncore_memory_usage.model_shared_scratchpad`)
* **ランタイムメモリ** (`trn.{core_index}.neuroncore_memory_usage.runtime_memory`)
* **テンソル** (`trn.{core_index}.neuroncore_memory_usage.tensors`)

  ## OpenMetrics
