W&amp;B Public API は、W&amp;B に保存されたデータにプログラムからアクセスし、クエリ、エクスポート、更新するための機能を提供します。この API は、事後分析、データのエクスポート、および Runs、アーティファクト、Sweeps のプログラムによる管理に使用します。メインの SDK はトレーニング中のリアルタイム logging を処理しますが、Public API を使用すると、履歴データを取得し、メタデータを更新し、アーティファクトを管理し、完了した Experiments に対して分析を実行できます。主要な `Api` クラスは、ほとんどの機能のエントリ ポイントです。

| [`User`](/ja/models/ref/python/public-api/user)          | ユーザープロフィールおよびユーザー固有のデータにアクセスします。                              |
| [`Files`](/ja/models/ref/python/public-api/files/)       | Runs に関連付けられたファイルをダウンロードおよび管理します。                              |
| `History`                                             | トレーニング中にログされた詳細な時系列メトリクスにアクセスします (`Run.history` を参照してください)。   |
| [`Automations`](./automations/)                       | 自動化されたワークフローとアクションを管理します。                                     |
| [`Integrations`](/ja/models/integrations)                | サードパーティーインテグレーションを構成および管理します。                                 |

<div id="common-use-cases">
  ## 一般的なユースケース
</div>

<div id="data-export-and-analysis">
  ### データのエクスポートと分析
</div>

* 分析のために、run の履歴を Jupyter notebook 用の DataFrames としてエクスポートします
* カスタムの可視化やレポート作成のためにメトリクスをダウンロードします
* 複数の実験にまたがる結果を集計します

<div id="post-hoc-updates">
  ### 事後更新
</div>

* 完了後にrunのメタデータを更新
* 完了したExperimentsにtagsまたはメモを追加
* runの設定またはサマリーを変更

<div id="artifact-management">
  ### アーティファクト管理
</div>

* バージョンまたは별칭でアーティファクトをクエリする
* プログラム的にモデル チェックポイントをダウンロードする
* アーティファクトのリネージと依存関係をトラッキングする

<div id="sweep-analysis">
  ### sweep の分析
</div>

* sweep の結果と最高のパフォーマンスを示した Runs に 액세ス
* ハイパーパラメーター探索の結果をエクスポート
* パラメーターの重要度を分析

<div id="authentication">
  ## 認証
</div>

Public API は Python SDK と同じ認証メカニズムを使用します。認証する方法はいくつかあります。

`WANDB_API_KEY` 環境変数を使用して APIキーを設定します。

```bash
export WANDB_API_KEY=your_api_key
```

`Api` クラスの初期化時に、APIキーを直接渡します。

```python
api = Api(api_key="your_api_key")
```

または、`wandb.login()` を使用して現在のセッションを認証します：

```python
import wandb

wandb.login()
api = Api()
```

<div id="example-usage">
  ## 使用例
</div>

<div id="download-an-artifact-by-name-and-alias">
  ### 名と별칭でアーティファクトをダウンロードする
</div>

次の例では、W&amp;B にログされたアーティファクトをその名と별칭で取得し、その内容をダウンロードする方法を示します。

```python
import wandb

api = wandb.Api()
artifact = api.artifact("entity/project/artifact:alias")
artifact.download()
```

<div id="download-an-artifact-from-a-registry">
  ### Registry からアーティファクトをダウンロードする
</div>

次の例では、W&amp;B Registry からリンクされたアーティファクトを取得する方法を示します

```python
import wandb

REGISTRY = "<registry_name>"
COLLECTION = "<collection_name>"
VERSION = "<version>"

api = wandb.Api()
artifact_name = f"wandb-registry-{REGISTRY}/{COLLECTION}:{VERSION}"

# アーティファクトを取得する
fetched_artifact = api.artifact(name = artifact_name)

# アーティファクトをダウンロードする。ダウンロードされたコンテンツへのパスを返す
downloaded_path = fetched_artifact.download()
```

<div id="query-wb-registry">
  ### W&amp;B Registry をクエリする
</div>

Mongo 風のフィルターを使用して、W&amp;B Registry、コレクション、および Artifacts をクエリします。次の例では、正規表現を使用してコレクション名でフィルタリングする方法を示します。

```python
import wandb

# wandb API を初期化する
api = wandb.Api()

# registry に関係なく、コレクション名に文字列 `yolo` を
# 含むすべてのコレクションをフィルタリングする
collection_filters = {
    "name": {"$regex": "yolo"}
}

# フィルターに一致するすべてのコレクションのイテラブルを返す
collections = api.registries().collections(filter=collection_filters)
```

Registry、コレクション、またはアーティファクトをクエリする方法の詳細については、[Find registry items](/ja/models/registry/search_registry) を参照してください。
