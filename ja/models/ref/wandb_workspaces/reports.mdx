W&amp;B レポート API をプログラム的に操作するための Python ライブラリです。

```python
import wandb_workspaces.reports.v2 as wr

report = wr.Report(
     entity="entity",
     project="project",
     title="驚くべきタイトル",
     description="詳細な説明。",
)

blocks = [
     wr.PanelGrid(
         panels=[
             wr.LinePlot(x="time", y="velocity"),
             wr.ScatterPlot(x="time", y="acceleration"),
         ]
     )
]

report.blocks = blocks
report.save()
```


## <kbd>class</kbd> `BarPlot`

2D 棒グラフを表示するパネルオブジェクトです。

**属性：**

* `title` (Optional[str]): プロットの上部に表示されるテキストです。
* `metrics` (LList[MetricType]):
* `orientation` (Literal[&quot;v&quot;, &quot;h&quot;]): 棒グラフの向きです。縦 (&quot;v&quot;) または横 (&quot;h&quot;) に設定します。デフォルトは横 (&quot;h&quot;) です。
* `range_x` (Tuple[float | None, float | None]): x 軸の範囲を指定するタプルです。
* `title_x` (Optional[str]): x 軸のラベルです。
* `title_y` (Optional[str]): y 軸のラベルです。
* `groupby` (Optional[str]): レポートが情報を取得する W&amp;B プロジェクトにログされたメトリクスに基づいて run をグループ化します。
* `groupby_aggfunc` (Optional[GroupAgg]): 指定された関数で run を集計します。オプションには &quot;mean&quot;、&quot;min&quot;、&quot;max&quot;、&quot;median&quot;、&quot;sum&quot;、&quot;samples&quot;、または `None` が含まれます。
* `groupby_rangefunc` (Optional[GroupArea]): 範囲に基づいて run をグループ化します。オプションには &quot;minmax&quot;、&quot;stddev&quot;、&quot;stderr&quot;、&quot;none&quot;、&quot;samples&quot;、または `None` が含まれます。
* `max_runs_to_show` (Optional[int]): プロットに表示する run の最大数です。
* `max_bars_to_show` (Optional[int]): 棒グラフに表示するバーの最大数です。
* `custom_expressions` (Optional[LList[str]]): 棒グラフで使用するカスタム式のリストです。
* `legend_template` (Optional[str]): 凡例のテンプレートです。
* `font_size` (Optional[FontSize]): 折れ線グラフのフォントサイズです。オプションには &quot;small&quot;、&quot;medium&quot;、&quot;large&quot;、&quot;auto&quot;、または `None` が含まれます。
* `line_titles` (Optional[dict]): ラインのタイトルです。キーはライン名で、値はタイトルです。
* `line_colors` (Optional[dict]): ラインの色です。キーはライン名で、値は色です。
* `aggregate` (Optional[bool]): `True` に設定すると、データを集計します。

## <kbd>class</kbd> `LinePlot`

2D 折れ線グラフを持つパネルオブジェクトです。

**属性：**

* `title` (Optional[str]):  プロットのトップに表示されるテキストです。
* `x` (Optional[MetricType]):  レポートが情報を取得する W&amp;B プロジェクトにログされたメトリクスの名前です。指定されたメトリクスは x 軸に使用されます。
* `y` (LList[MetricType]):  レポートが情報を取得する W&amp;B プロジェクトにログされた 1 つ以上のメトリクスです。指定されたメトリクスは y 軸に使用されます。
* `range_x` (Tuple[float | `None`, float | `None`]):  x 軸の範囲を指定するタプルです。
* `range_y` (Tuple[float | `None`, float | `None`]):  y 軸の範囲を指定するタプルです。
* `log_x` (Optional[bool]):  x 座標を底が 10 の対数スケールでプロットします。
* `log_y` (Optional[bool]):  y 座標を底が 10 の対数スケールでプロットします。
* `title_x` (Optional[str]):  x 軸のラベルです。
* `title_y` (Optional[str]):  y 軸のラベルです。
* `ignore_outliers` (Optional[bool]):  `True` に設定した場合、外れ値をプロットしません。
* `groupby` (Optional[str]):  レポートが情報を取得する W&amp;B プロジェクトにログされたメトリクスに基づいて run をグループ化します。
* `groupby_aggfunc` (Optional[GroupAgg]):  指定された関数で run を集計します。オプションには &quot;mean&quot;、&quot;min&quot;、&quot;max&quot;、&quot;median&quot;、&quot;sum&quot;、&quot;samples&quot;、または `None` が含まれます。
* `groupby_rangefunc` (Optional[GroupArea]):   範囲に基づいて run をグループ化します。オプションには &quot;minmax&quot;、&quot;stddev&quot;、&quot;stderr&quot;、&quot;none&quot;、&quot;samples&quot;、または `None` が含まれます。
* `smoothing_factor` (Optional[float]):  スムージングタイプに適用するスムージングファクターです。許容値は 0 から 1 の範囲です。
* `smoothing_type Optional[SmoothingType]`:  指定されたスムージングタイプに基づいてフィルターを適用します。オプションには &quot;exponentialTimeWeighted&quot;、&quot;exponential&quot;、&quot;gaussian&quot;、&quot;average&quot;、または &quot;none&quot; が含まれます。
* `smoothing_show_original` (Optional[bool]):    `True` に設定した場合、元のデータを表示します。
* `max_runs_to_show` (Optional[int]):  折れ線グラフに表示する run の最大数です。
* `custom_expressions` (Optional[LList[str]]):  データに適用するカスタム式です。
* `plot_type Optional[LinePlotStyle]`:  生成する折れ線グラフの種類です。オプションには &quot;line&quot;、&quot;stacked-area&quot;、または &quot;pct-area&quot; が含まれます。
* `font_size Optional[FontSize]`:  折れ線グラフのフォントのサイズです。オプションには &quot;small&quot;、&quot;medium&quot;、&quot;large&quot;、&quot;auto&quot;、または `None` が含まれます。
* `legend_position Optional[LegendPosition]`:  凡例を配置する場所です。オプションには &quot;north&quot;、&quot;south&quot;、&quot;east&quot;、&quot;west&quot;、または `None` が含まれます。
* `legend_template` (Optional[str]):  凡例のテンプレートです。
* `aggregate` (Optional[bool]):  `True` に設定した場合、データを集計します。
* `xaxis_expression` (Optional[str]):  x 軸の式です。
* `xaxis_format` (Optional[str]):  x 軸のフォーマットです。このオプションはカスタムメトリクスを定義した場合に表示されます。たとえば、x 軸を日付と時刻としてフォーマットするために &#39;datetime&#39; を指定できます。
* `legend_fields` (Optional[LList[str]]):  凡例に含めるフィールドです。
* `metric_regex` (Optional[str]):  y 軸のメトリクスに一致する正規表現パターンです。バックエンドはこのパターンを使用して一致するメトリクスを選択します。
* `point_visualization_method` (Optional[PointVizMethod]):  表示するポイントが多すぎる場合にポイントを集計するために使用されるメソッドです。オプションには &quot;bucketing-gorilla&quot; (データポイントをバケット化し、外れ値とスパイクを保持するためにバケットごとの min、max、avg を表示) または &quot;sampling&quot; (高速レンダリングのためにポイントをランダムにサンプリングしますが、外れ値を逃す可能性があります) が含まれます。

## <kbd>class</kbd> `PanelGrid`

* `custom_run_colors` (dict):  キーが run の名、値が 16 進数値で指定された色であるキーと値のペアです。

## <kbd>クラス</kbd> `Report`

W&amp;B Report を表すオブジェクトです。返されたオブジェクトの `blocks` 属性を使用して、レポートをカスタマイズします。Report オブジェクトは自動では保存されません。`save()` method を使用して変更を永続化します。

**属性:**

* `project` (str):  読み込む W&amp;B プロジェクトの名です。指定したプロジェクトはレポートの URL に表示されます。
* `entity` (str):  レポートを所有する W&amp;B entity です。entity はレポートの URL に表示されます。
* `title` (str):  レポートのタイトルです。タイトルはレポートの上部に H1 見出しとして表示されます。
* `description` (str):  レポートの説明です。説明はレポートのタイトルの下に表示されます。
* `blocks` (LList[BlockTypes]):  1 つ以上の HTML タグ、プロット、グリッド、runset などのリストです。
* `width` (Literal[&#39;readable&#39;, &#39;fixed&#39;, &#39;fluid&#39;]):  レポートの幅です。オプションには &#39;readable&#39;、&#39;fixed&#39;、&#39;fluid&#39; があります。

### <kbd>プロパティ</kbd> url

レポートがホストされている URL です。レポートの URL は `https://wandb.ai/{entity}/{project_name}/reports/` で構成されます。ここで、`{entity}` はレポートが属する entity、`{project_name}` はプロジェクトの名です。

## <kbd>class</kbd> `Runset`

  * 設定パラメーターでグループ化するには&quot;config.param&quot;

* `order` (LList[OrderBy]):  並べ替えに使用する`OrderBy` objectのlist。
* `custom_run_colors` (dict):  run ID (またはグループ化されたrunの場合は`RunsetGroup`キー) をカラーに対応付けるdict。単純なrunごとのカラーには、`run_settings`の使用を推奨します。
* `run_settings` (Dict[str, RunSettings]):  run IDをキーとするrunごとの表示設定。 これを使用してカラーの設定や個々のrunの非表示を行います。`RunSettings`を参照してください。

**例:**

```python

    wr.Runset(
         entity="my-entity",
         project="my-project",
         filters="Config('learning_rate') = 0.001 and State = 'finished'"
    )

    # FilterExpr リストを使用する場合
    wr.Runset(
         entity="my-entity",
         project="my-project",
         filters=[expr.Config("learning_rate") == 0.001]
    )
```

## <kbd>class</kbd> `WeavePanelSummaryTable`

W&amp;B Table、pandas DataFrame、プロット、または W&amp;B にログされたその他の値を表示するパネルです。クエリは次の形式を取ります

```python
runs.summary['value']
```

API 名に含まれる &quot;Weave&quot; という用語は、LLM のトラッキングと評価に使用される W&amp;B Weave toolkit を指すものではありません。

**属性:**

* `table_name` (str):  表、DataFrame、プロット、または値の名。
* `layout` (Layout):  位置とサイズを含む、パネルの Layout 設定です。  Panel 基底クラスから継承されます。幅 (w) と高さ (h) を調整するために使用します。

**例:**
カスタム寸法の表を表示する summary table パネルを含むレポートを作成します。

```python
    import wandb_workspaces.reports as wr

    report = wr.Report(
         project="my-project",
         entity="my-entity",
         title="Summary Table Report"
    )

    report.blocks = [
         wr.PanelGrid(
             runsets=[wr.Runset(project="my-project")],
             panels=[
                 wr.WeavePanelSummaryTable(
                     table_name="my-table-name",
                     layout=wr.Layout(w=24, h=20)
                 )
             ]
         )
    ]

    report.save()
```

Layout パラメーターはパネルの寸法を制御します。

* w (width): グリッド単位での幅 (デフォルト: 8、最大: 24) 
  * h (height): グリッド単位での高さ (デフォルト: 6) 
  * x (x-position): グリッド内の水平位置 (デフォルト: 0) 
  * y (y-position): グリッド内の垂直位置 (デフォルト: 0)
