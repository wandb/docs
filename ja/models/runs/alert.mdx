run がクラッシュしたときやカスタムトリガーに基づいて、Slack またはメールでアラートを作成できます。たとえば、トレーニング ループの勾配が発散し始めたとき (NaN を報告したとき) や、ML パイプライン内のある step が完了したときにアラートを作成できます。アラートは、個人プロジェクトとチーム プロジェクトの両方を含め、run を初期化するすべてのプロジェクトに適用されます。

その後、Slack (またはメール) で W&amp;B Alerts のメッセージを確認できます。

<Note>
  W&amp;B Alerts を使用するには、コードに `run.alert()` を追加する必要があります。コードを変更しなくても、[Automations](/ja/models/automations/) を使えば、[artifact](/ja/models/artifacts/) の artifact バージョンが作成されたときや、[run metric](/ja/models/runs/) がしきい値に達したとき、またはしきい値を超えて変化したときなど、W&amp;B 内のイベントに基づいて Slack に通知できます。

  たとえば、automation を使って、新しいバージョンが作成されたときに Slack チャンネルへ通知したり、artifact に `production` alias が追加されたときに自動テスト用の webhook を実行したり、run の `loss` が許容範囲内にある場合にのみ検証ジョブを開始したりできます。

  [Automations overview](/ja/models/automations/) を参照するか、[automation を作成](/ja/models/automations/create-automations/) してください。
</Note>

<Note>
  以下のガイドは、Multi-tenant Cloud のアラートにのみ適用されます。

  Private Cloud または W&amp;B 専用クラウド上で [W&amp;B Server](/ja/platform/hosting/) を使用している場合は、Slack アラートを設定するには [Configure Slack alerts in W&amp;B Server](/ja/platform/hosting/monitoring-usage/slack-alerts) を参照してください。
</Note>

アラートを設定するには、次の手順を実行します。詳細は以下のセクションで説明します。

1. W&amp;B の [User Settings](https://wandb.ai/settings) で Alerts を有効にする。
2. コードに `run.alert()` を追加する。
3. 設定をテストする。

<div id="1-turn-on-alerts-in-your-wb-user-settings">
  ### 1. W&amp;B User Settings でアラートを有効にする
</div>

[User Settings](https://wandb.ai/settings) で次を行います。

* **Alerts** セクションまでスクロールします
* `run.alert()` からのアラートを受け取るには、**Scriptable run alerts** をオンにします
* アラートの投稿先となる Slack チャンネルを選択するには、**Connect Slack** を使用します。アラートを非公開に保てるため、**Slackbot** チャンネルを推奨します。
* **Email** は、W&amp;B へのサインアップ時に使用したメールアドレスに送信されます。これらのアラートで受信トレイが埋まらないように、すべて専用フォルダーに振り分けるメールフィルターを設定することを推奨します。

この設定が必要なのは、W&amp;B Alerts を初めて設定するとき、またはアラートの受信方法を変更したいときだけです。

  <img src="/images/track/demo_connect_slack.png" alt="W&B User Settings の Alerts 設定" />

<div id="2-add-runalert-to-your-code">
  ### 2. コードに `run.alert()` を追加する
</div>

Notebook または Python スクリプトで、`run.alert()` を送信したい箇所に追加します

```python
import wandb

with wandb.init() as run:
    run.alert(title="High Loss", text="Loss is increasing rapidly")
```

<div id="3-test-the-configuration">
  ### 3. 設定をテストする
</div>

Slack またはメールでアラート メッセージが届いているか確認してください。もし受信していない場合は、[User Settings](https://wandb.ai/settings) で **Scriptable Alerts** 向けにメールまたは Slack が有効になっていることを確認してください。

<div id="example">
  ## 例
</div>

このシンプルなアラートは、accuracy がしきい値を下回ったときに警告を送信します。この例では、アラートは少なくとも 5 分間隔でのみ送信されます。

```python
import wandb
from wandb import AlertLevel

with wandb.init() as run:

    if acc < threshold:
        run.alert(
            title="Low accuracy",
            text=f"Accuracy {acc} is below the acceptable threshold {threshold}",
            level=AlertLevel.WARN,
            wait_duration=300,
        )
```

<div id="tag-or-mention-users">
  ## ユーザーをタグ付けまたはメンションする
</div>

アットマーク `@` の後に Slack ユーザー ID を付けて、アラートのタイトルまたは本文で自分や同僚をタグ付け／メンションします。Slack ユーザー ID は、そのユーザーの Slack プロフィール ページから確認できます。

```python
run.alert(title="Loss is NaN", text=f"Hey <@U1234ABCD> loss has gone to NaN")
```

<div id="configure-team-alerts">
  ## チームアラートを設定する
</div>

チーム管理者は、チーム設定ページ `wandb.ai/teams/your-team` でチーム向けのアラートを設定できます。

チームアラートは、チーム内の全員に適用されます。W&amp;B は、アラートを非公開に保てるため、**Slackbot** チャンネルの使用を推奨します。

<div id="change-slack-channel-to-send-alerts-to">
  ## 送信先の Slack チャンネルを変更する
</div>

送信先の Slack チャンネルを変更するには、**Disconnect Slack** をクリックして一度接続を解除し、再度接続し直してください。再接続後に、別の Slack チャンネルを選択します。
