run がクラッシュしたときやカスタムトリガーに基づいて、Slack またはメールでアラートを作成できます。たとえば、トレーニング ループの勾配が暴走し始めたとき（NaN を出力したとき）や、ML パイプライン内のあるステップが完了したときにアラートを作成できます。アラートは、run を初期化するすべてのプロジェクト（個人プロジェクトとチーム プロジェクトの両方）に適用されます。

その後、Slack（またはメール）で W&amp;B Alerts のメッセージを確認できます。

<Note>
  W&amp;B Alerts を使用するには、コードに `run.alert()` を追加する必要があります。コードを変更せずに、[Automations](/ja/models/automations/) を使って、W&amp;B 内のイベントに基づいて Slack に通知する別の方法も利用できます。たとえば、[artifact](/ja/models/artifacts/) のバージョンが作成されたときや、[run metric](/ja/models/runs/) がしきい値に達したり、しきい値を超えて変化したときなどです。

  たとえば、オートメーションを使って、新しいバージョンが作成されたときに Slack チャンネルへ通知したり、`production` エイリアスが Artifacts に追加されたときに自動テスト用の webhook を実行したり、run の `loss` が許容範囲内にある場合にのみ検証ジョブを開始したりできます。

  [Automations の概要](/ja/models/automations/)を読むか、[オートメーションを作成](/ja/models/automations/create-automations/)してください。
</Note>

<Note>
  次のガイドは、Multi-tenant Cloud のアラートにのみ適用されます。

  Private Cloud または W&amp;B Dedicated Cloud 上で [W&amp;B Server](/ja/platform/hosting/) を使用している場合は、Slack アラートを設定するために、[Configure Slack alerts in W&amp;B Server](/ja/platform/hosting/monitoring-usage/slack-alerts) を参照してください。
</Note>

アラートをセットアップするには、次の手順を実行します。詳細は後述のセクションで説明します。

1. W&amp;B の [User Settings](https://wandb.ai/settings) で Alerts を有効にする。
2. コードに `run.alert()` を追加する。
3. 設定をテストする。

<div id="1-turn-on-alerts-in-your-wb-user-settings">
  ### 1. W&amp;B User Settings でアラートを有効にする
</div>

[User Settings](https://wandb.ai/settings) で次を行います:

* **Alerts** セクションまでスクロールします
* `run.alert()` からのアラートを受け取るには **Scriptable run alerts** をオンにします
* **Connect Slack** を使って、アラートを投稿する Slack チャンネルを選びます。アラートを非公開に保てるため、**Slackbot** チャンネルをおすすめします。
* **Email** は、W&amp;B にサインアップしたときに使用したメールアドレス宛に送信されます。これらのアラート専用のフォルダーに振り分けて受信トレイがいっぱいにならないよう、メールにフィルターを設定することをおすすめします。

これは W&amp;B Alerts を最初にセットアップするとき、またはアラートの受け取り方を変更したいときに一度だけ行えば十分です。

<div id="2-add-runalert-to-your-code">
  ### 2. コードに `run.alert()` を追加する
</div>

Notebook または Python スクリプト内の、トリガーさせたい任意の場所に `run.alert()` を追加します

```python
import wandb

with wandb.init() as run:
    run.alert(title="High Loss", text="Loss is increasing rapidly")
```

<div id="3-test-the-configuration">
  ### 3. 設定をテストする
</div>

Slack またはメールでアラートメッセージが届いているか確認してください。受信していない場合は、[User Settings](https://wandb.ai/settings) で **Scriptable Alerts** 向けのメールまたは Slack が有効になっていることを確認してください。

<div id="example">
  ## 例
</div>

このシンプルなアラートは、accuracy がしきい値を下回ったときに警告を送信します。この例では、アラートは少なくとも 5 分以上の間隔をあけてのみ送信されます。

```python
import wandb
from wandb import AlertLevel

with wandb.init() as run:

    if acc < threshold:
        run.alert(
            title="Low accuracy",
            text=f"Accuracy {acc} is below the acceptable threshold {threshold}",
            level=AlertLevel.WARN,
            wait_duration=300,
        )
```

<div id="tag-or-mention-users">
  ## ユーザーをタグ付けまたはメンションする
</div>

アットマーク `@` の後に Slack ユーザー ID を付けて、アラートのタイトルまたは本文で自分や同僚をタグ付け／メンションします。Slack ユーザー ID は、そのユーザーの Slack プロフィール ページから確認できます。

```python
run.alert(title="Loss is NaN", text=f"Hey <@U1234ABCD> loss has gone to NaN")
```

<div id="configure-team-alerts">
  ## チームアラートを設定する
</div>

Team の管理者は、チーム設定ページ `wandb.ai/teams/your-team` でチーム向けのアラートを設定できます。

チームアラートは、チーム内の全員に適用されます。W&amp;B では、アラートを非公開に保てるため、**Slackbot** チャンネルの利用を推奨します。

<div id="change-slack-channel-to-send-alerts-to">
  ## アラートの送信先 Slack チャンネルを変更する
</div>

アラートの送信先の Slack チャンネルを変更するには、**Disconnect Slack** をクリックして一度接続を解除し、再度接続し直してください。再接続後に、別の Slack チャンネルを選択します。
