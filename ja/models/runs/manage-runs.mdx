---
title: Move a run to a different project or team
description: W&B App を使用して、プロジェクトまたはチーム間で run を移動します。
---

<Note>
  作業を始める前に、プロジェクトまたはチーム間で run を移動するために必要な権限があることを確認してください。現在の場所と新しい場所の両方で、その run へのアクセス権が必要です。
</Note>

run をあるプロジェクトから別のプロジェクト、またはチーム間で移動するには、次の手順を実行します。

1. 移動したい run を含むプロジェクトにアクセスします。
2. プロジェクトのサイドバーから **Runs** タブを選択します。
3. 移動したい run の横にあるチェックボックスを選択します。
4. テーブル上部の **Move to project** ボタンをクリックします。
5. ドロップダウンから移動先のチームとプロジェクトを選択します。

<Note>
  run を移動しても、それに関連付けられた過去の Artifacts は移動されません。 Artifacts を手動で移動するには、[`wandb artifact get`](/ja/models/ref/cli/wandb-artifact/wandb-artifact-get/) SDK コマンドまたは [`Api.artifact` API](/ja/models/ref/python/public-api/api/#artifact) を使用して Artifacts をダウンロードし、その後 [`wandb artifact put`](/ja/models/ref/cli/wandb-artifact/wandb-artifact-put/) または `Api.artifact` API を使用して、その run の新しい場所にアップロードします。
</Note>

{/* ## run をグループに移動する

  既存のグループに run を移動するか、新しいグループを作成して移動することで、run をグループ化できます。

  1. 移動したい run を含むプロジェクトに移動します。
  2. プロジェクトサイドバーから **Runs** タブを選択します。
  3. 移動したい run の横にあるチェックボックスを選択します。
  4. テーブル上部の **Move to group** ボタンを選択します。
  5. モーダルで、対象のグループを選択するか、新しいグループを作成します。
  6. **Move** をクリックします。 */}

{/* #### Manage runs in bulk
  run を一括管理する

  このセクションでは、1 つ以上の run グループにまたがる複数の run をまとめて管理する方法を説明します。

  1. 必要に応じて、[列ごとのグループ化を切り替え](#toggle-grouping-by-column-in-the-ui)、**Group** 列でグループ化します。
  1. グループを展開して、そのグループ内の run を表示します。1 回の一括操作で、異なるグループにある run を同じ対象グループに移動できます。該当するグループをすべて展開します。
  1. グループ内のすべての run を選択するには、グループ名の横にあるチェックボックスをクリックします。
  1. 個別の run を選択するには、それぞれのチェックボックスをクリックします。
  1. すべてのグループ内のすべての run を選択するには、**Name** 列の見出しの横にあるチェックボックスをクリックします。
  1. テーブル上部で、一括操作として **Tag**、**Move to group**、**Move to project**、または **New sweep** を選択します。 */}
