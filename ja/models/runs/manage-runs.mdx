---
title: run を別のプロジェクトまたはチームに移動する
description: W&B App を使用して、プロジェクトまたはチーム間で run を移動します。
---

<Note>
  始める前に、プロジェクトまたはチーム間で run を移動するために必要な権限があることを確認してください。現在の場所と新しい場所の両方で、その run に액세스できる必要があります。
</Note>

あるプロジェクトから別のプロジェクトへ、またはチーム間で run を移動するには、次の手順に従います。

1. 移動したい run が含まれているプロジェクトにアクセスします。
2. プロジェクトのサイドバーから **Runs** タブを選択します。
3. 移動したい run の横にあるチェックボックスを選択します。
4. 表の上にある **Move to project** ボタンをクリックします。
5. ドロップダウンから移動先のチームとプロジェクトを選択します。

<Note>
  run を移動しても、それに関連付けられた履歴アーティファクトは移動されません。アーティファクトを手動で移動するには、[`wandb artifact get`](/ja/models/ref/cli/wandb-artifact/wandb-artifact-get/) SDK コマンドまたは [`Api.artifact` API](/ja/models/ref/python/public-api/api/#artifact) を使用してアーティファクトをダウンロードし、その後 [`wandb artifact put`](/ja/models/ref/cli/wandb-artifact/wandb-artifact-put/) または `Api.artifact` API を使用して、run&#39;s new location にアップロードできます。
</Note>

{/* ## run をグループに移動する

  既存のグループに移動するか、新しいグループを作成すると、run をグループ化できます。

  1. 移動する run を含むプロジェクトにアクセスします。
  2. プロジェクトのサイドバーから **Runs** タブを選択します。
  3. 移動する run の横にあるチェックボックスを選択します。
  4. 表の上部にある **Move to group** ボタンを選択します。
  5. モーダルで、移動先のグループを選択するか、新しいグループを作成します。
  6. **Move** をクリックします。 */}

{/* #### 複数の run を一括管理する
  このセクションでは、1 つ以上の run グループにまたがる複数の run を一度に管理する方法を説明します。

  1. 必要に応じて、[UI で列ごとのグループ化を切り替え](#toggle-grouping-by-column-in-the-ui)、**Group** 列でグループ化します。
  1. グループを展開して、その run を表示します。1 回の一括操作で、異なるグループの run を同じ対象グループに移動できます。該当するグループをすべて展開します。
  1. グループ内のすべての run を選択するには、グループ名の横にあるチェックボックスをクリックします。
  1. 個々の run を選択するには、それぞれのチェックボックスをクリックします。
  1. すべてのグループ内のすべての run を選択するには、**Name** 列見出しの横にあるチェックボックスをクリックします。
  1. 表の上部で、一括操作を選択します: **Tag**、**Move to group**、**Move to project**、または **New sweep**。 */}
