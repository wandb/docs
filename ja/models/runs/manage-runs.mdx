---
title: Run を別の Project または Team に移動する
description: W&B App を使用して、 Runs を Projects や Teams 間で移動します。
---

<Note>
開始する前に、Run を Project 間または Team 間で移動するために必要な権限があることを確認してください。現在の場所と移動先の両方の場所にアクセスできる必要があります。
</Note>

ある Project から別の Project へ、あるいは Team 間で Run を移動するには：

1. 移動したい Run が含まれている Project に移動します。
2. Project のサイドバーから **Runs** タブを選択します。
3. 移動したい Run の横にあるチェックボックスを選択します。
4. テーブルの上にある **Move to project** ボタンをクリックします。
5. ドロップダウンから移動先の Team と Project を選択します。

<Note>
Run を移動しても、それに関連付けられた過去の Artifacts は移動されません。Artifact を手動で移動するには、[`wandb artifact get`](/models/ref/cli/wandb-artifact/wandb-artifact-get/) SDK コマンドまたは [`Api.artifact` API](/models/ref/python/public-api/api/#artifact) を使用して Artifact をダウンロードし、次に [`wandb artifact put`](/models/ref/cli/wandb-artifact/wandb-artifact-put/) または `Api.artifact` API を使用して、Run の新しい場所にアップロードします。
</Note>

{/* ## Run をグループに移動する

既存のグループに移動するか、新しいグループを作成することで、Run をグループ化できます。

1. 移動したい Run が含まれている Project に移動します。
2. Project のサイドバーから **Runs** タブを選択します。
3. 移動したい Run の横にあるチェックボックスを選択します。
4. テーブルの上にある **Move to group** ボタンを選択します。
5. モーダルで、移動先のグループを選択するか、新しいグループを作成します。
6. **Move** をクリックします。 */}

{/* #### Run を一括で管理する
このセクションでは、1つ以上の Run グループにわたって、複数の Run を一度に管理する方法を説明します。

1. 必要に応じて、**Group** 列で [列によるグループ化を切り替えます](#toggle-grouping-by-column-in-the-ui)。
1. グループを展開してその Run を表示します。一括操作では、異なるグループの Run を同じターゲットグループに移動できます。関連するすべてのグループを展開します。
1. グループ内のすべての Run を選択するには、グループ名の横にあるチェックボックスをクリックします。
1. 個別の Run を選択するには、それらのチェックボックスをクリックします。
1. すべてのグループのすべての Run を選択するには、**Name** 列の見出しの横にあるチェックボックスをクリックします。
1. テーブルの上で、一括操作を選択します： **Tag**、**Move to group**、**Move to project**、または **New sweep**。 */}
