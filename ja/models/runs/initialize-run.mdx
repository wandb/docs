---
title: run の初期化
---

[`wandb.init()`](/ja/models/ref/python/functions/init) を使って W&amp;B の run を初期化します。

デフォルトでは、`wandb.init()` を呼び出したときに、各 Python プロセスには同時に 1 つのアクティブな run しかないと W&amp;B はみなします。`wandb.init()` を再度呼び出すと、W&amp;B は同じ run を返すか、古い run を終了してから新しい run を開始します。同一プロセス内で `wandb.init()` を複数回呼び出した場合の挙動は、環境 (notebook か非 notebook か) と `reinit` 設定によって異なります。

同じプロセス内で複数のアクティブな run を管理する方法については、[Multiple runs in one process](/ja/models/runs/initialize-run#multiple-runs-in-one-process) を参照してください。

<Note>
  W&amp;B は、`wandb.init()` を呼び出す際に `with` ブロックを使用することを推奨します。これにより、ブロックの終了時に W&amp;B が run を正しく終了処理し、すべてのデータをアップロードできるようになります。
</Note>

<div id="single-run-per-process">
  ## 1 プロセスにつき 1 つの run
</div>

次のコードスニペットは、W&amp;B Python SDK をインポートして run を初期化する方法を示します。

```python title="basic.py"
import wandb

with wandb.init(entity="nico", project="awesome-project") as run:
    # トレーニングロジックをここに記述
```

このコード スニペットを実行すると、次のような出力になります：

```bash
🚀 View run exalted-darkness-6 at: 
https://wandb.ai/nico/awesome-project/runs/pgbn9y21
Find logs at: wandb/run-20241106_090747-pgbn9y21/logs
```

この出力から、W&amp;B が entity `nico` の配下にある project `awesome-project` に run `exalted-darkness-6` を記録していることがわかります。`pgbn9y21` は、この run に対して W&amp;B が生成した一意の run ID です。

<div id="multiple-runs-in-one-process">
  ## 1 つのプロセス内で複数の run を扱う
</div>

1 つの Python プロセス内で複数の run を管理します。これは、メインとなるプロセスを動かし続けながら、副次的なサブタスク用に短時間だけ動作するセカンダリプロセスを作成したいワークフローで有用です。ユースケースの例としては次のようなものがあります。

* スクリプト全体を通して 1 つの「プライマリ」run をアクティブに保ちながら、評価やサブタスク用に短時間だけ動作する「セカンダリ」run を立ち上げる。
* 1 つのファイル内でサブ実験をオーケストレーションする。
* 1 つの「メイン」プロセスから、異なるタスクや時間帯を表す複数の run へログを記録する。

デフォルトでは、`wandb.init()` を呼び出すと、W&amp;B は各 Python プロセス内に同時にアクティブな run が 1 つだけ存在すると想定します。`wandb.init()` を再度呼び出すと、W&amp;B は同じ run を返すか、設定に応じて古い run を終了してから新しい run を開始します。

このガイドでは、1 つの Python プロセス内で複数の run を有効にするために、`reinit` を使用して `wandb.init()` の動作を変更する方法について説明します。

<Note>
  **要件**

  1 つの Python プロセス内で複数の run を管理するには、W&amp;B Python SDK バージョン `v0.19.10` 以降が必要です。
</Note>

<div id="reinit-options">
  ### `reinit` オプション
</div>

`reinit` パラメーターを使用して、W&amp;B が `wandb.init()` を複数回呼び出したときの挙動を制御します。次の表は、有効な引数とその効果を示します:

| | 説明 | run を作成するか? | 利用例 |
|----------------|----------------|----------------| -----------------|
| `create_new` | 既存のアクティブな run を終了せずに、`wandb.init()` で新しい run を作成します。W&amp;B はグローバルな `wandb.Run` を自動的に新しい run に切り替えません。各 run オブジェクトは自分で保持する必要があります。詳細は、下記の [1 つのプロセスで複数の run を扱う例](/ja/models/runs/initialize-run/#example-multiple-runs-in-one-process) を参照してください。 | Yes | 同時に動作する複数のプロセスを作成・管理する場合に最適です。たとえば、「プライマリ」run をアクティブなまま保ちながら、「セカンダリ」run を開始・終了するようなケースです。|
| `finish_previous` | 新しい run を `wandb.init()` で作成する前に、`run.finish()` を使ってすべてのアクティブな run を終了します。ノートブック以外の環境でのデフォルトの挙動です。 | Yes | 逐次的なサブプロセスを、個別の run に分割したい場合に最適です。 |
| `return_previous` | 直近の未終了の run を返します。ノートブック環境でのデフォルトの挙動です。 | No | |

<Note>
  W&amp;B は、Hugging Face Trainer、Keras のコールバック、PyTorch Lightning など、単一のグローバル run を前提とする [W&amp;B Integrations](/ja/models/integrations) に対して `create_new` モードをサポートしていません。これらのインテグレーションを使用する場合は、各サブ実験を個別のプロセスで実行してください。
</Note>

<div id="specifying-reinit">
  ### `reinit` の指定方法
</div>

{/* 単一の Python プロセス内で複数の run を作成・管理する方法はいくつかあります。 */}

* `reinit` 引数を指定して、直接 `wandb.init()` を使用します:
  ```python
  import wandb
  with wandb.init(reinit="<create_new|finish_previous|return_previous>") as run:
      # ここにコードを記述
  ```

* `wandb.init()` を使用し、`settings` パラメーターに `wandb.Settings` オブジェクトを渡します。`Settings` オブジェクト内で `reinit` を指定します:

  ```python
  import wandb
  with wandb.init(settings=wandb.Settings(reinit="<create_new|finish_previous|return_previous>")) as run:
      # ここにコードを記述
  ```

* `wandb.setup()` を使用して、現在のプロセス内のすべての run に対して `reinit` オプションをグローバルに設定します。これは、一度だけ動作を設定し、そのプロセス内の後続のすべての `wandb.init()` 呼び出しに適用したい場合に便利です。

  ```python
  import wandb
  with wandb.setup(wandb.Settings(reinit="<create_new|finish_previous|return_previous>")) as run:
       # ここにコードを記述
  ```

* 環境変数 `WANDB_REINIT` に `reinit` の希望する値を設定します。環境変数を定義すると、`wandb.init()` の呼び出しに `reinit` オプションが適用されます。

  ```bash
  export WANDB_REINIT="<create_new|finish_previous|return_previous>"
  ```

次のコードスニペットでは、`wandb.init()` を呼び出すたびに新しい run を作成するように W&amp;B を設定する方法の概要を示しています:

```python
import wandb

wandb.setup(wandb.Settings(reinit="create_new"))

with wandb.init() as experiment_results_run:
    # この run は各実験の結果をログするために使用されます。
    # これを結果を収集する親 run と考えることができます。
      with wandb.init() as run:
         # do_experiment() 関数は詳細なメトリクスを指定された run にログし、
         # 個別に追跡したい結果のメトリクスを返します。
         experiment_results = do_experiment(run)

         # 各実験の後、その結果を親 run にログします。
         # 親 run のチャートの各ポイントは
         # 1 つの実験の結果に対応します。
         experiment_results_run.log(experiment_results)
```

<div id="example-concurrent-processes">
  ### 例: 並行プロセス
</div>

スクリプトの実行全体を通して開いたままにしておくメインプロセスを 1 つ用意しつつ、その間に短時間だけ動作するセカンダリプロセスを定期的に生成し、メインプロセス自体は終了させないままにしておきたいとします。たとえば、このパターンはメインの run でモデルをトレーニングしつつ、評価の計算やその他の作業を別の run で実行したい場合に有用です。

これを実現するには、`reinit="create_new"` を使って複数の run を初期化します。この例では、「Run A」がスクリプト全体を通して開いたままのメインプロセスで、「Run B1」「Run B2」が評価などのタスク用に短時間だけ動作するセカンダリ run だとします。

全体のワークフローは次のようになります:

1. メインプロセス Run A を `wandb.init()` で初期化し、トレーニング メトリクスをログします。
2. Run B1 を初期化し (`wandb.init()` を使用) 、データをログしてから終了します。
3. Run A にさらにデータをログします。
4. Run B2 を初期化し、データをログしてから終了します。
5. Run A へのログを継続します。
6. 最後に、スクリプトの終わりで Run A を終了します。

次の Python コード例は、このワークフローを示したものです:

```python
import wandb

def train(name: str) -> None:
    """独自の W&B run で1回のトレーニングイテレーションを実行する。

    `reinit="create_new"` を指定した 'with wandb.init()' ブロックを使用することで、
    別の run（プライマリのトラッキング run など）がすでにアクティブな場合でも、
    このトレーニングサブ run を作成できる。
    """
    with wandb.init(
        project="my_project",
        name=name,
        reinit="create_new"
    ) as run:
        # 実際のスクリプトでは、このブロック内でトレーニングステップを実行する。
        run.log({"train_loss": 0.42})  # 実際のメトリクスに置き換えること

def evaluate_loss_accuracy() -> (float, float):
    """現在のモデルの loss と accuracy を返す。
    
    このプレースホルダーを実際の評価ロジックに置き換えること。
    """
    return 0.27, 0.91  # メトリクスの値の例

# 複数のトレーニング/評価ステップを通じてアクティブであり続ける「プライマリ」run を作成する。
with wandb.init(
    project="my_project",
    name="tracking_run",
    reinit="create_new"
) as tracking_run:
    # 1) 'training_1' という名前のサブ run でトレーニングを1回実行する
    train("training_1")
    loss, accuracy = evaluate_loss_accuracy()
    tracking_run.log({"eval_loss": loss, "eval_accuracy": accuracy})

    # 2) 'training_2' という名前のサブ run で再度トレーニングを実行する
    train("training_2")
    loss, accuracy = evaluate_loss_accuracy()
    tracking_run.log({"eval_loss": loss, "eval_accuracy": accuracy})
    
    # この 'with' ブロックが終了すると、'tracking_run' は自動的に終了する。
```

前の例から次の 3 つの重要なポイントに注意してください。

1. `reinit="create_new"` は、`wandb.init()` を呼び出すたびに新しい run を作成します。
2. 各 run への参照を保持しておく必要があります。`wandb.run` は、`reinit="create_new"` で作成された新しい run を自動的には指しません。新しい run は `run_a`、`run_b1` などの変数に保存し、必要に応じてそれらのオブジェクトに対して `.log()` や `.finish()` を呼び出してください。
3. メインの run を開いたままにしておきつつ、必要なタイミングでサブ run を終了できます。
4. 対象へのロギングが完了したら、`run.finish()` で run を終了してください。これにより、すべてのデータがアップロードされ、run が正しく終了することが保証されます。
