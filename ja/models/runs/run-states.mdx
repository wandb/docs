---
title: run の状態
description: W&B run が取り得るさまざまな状態について説明します。
---

[Run state](/ja/models/runs/run-states#run-states) は、W&amp;B run の現在のステータスを示します。run の[状態を表示](/ja/models/runs/run-states#view-the-state-of-a-run)するには、W&amp;B App または W&amp;B Python SDK をプログラムで使用します。

<div id="run-states">
  ## run の状態
</div>

次の表は、run が取り得る状態を示しています。

| State      | 説明                                                          |
| ---------- | ----------------------------------------------------------- |
| `Crashed`  | マシンのクラッシュなどにより、内部プロセスで run からのハートビート送信が停止した状態。              |
| `Failed`   | run が非ゼロの終了ステータスで終了した状態。                                    |
| `Finished` | run が終了し、データが完全に同期された、または `wandb.Run.finish()` が呼び出された状態。   |
| `Killed`   | run が完了する前に強制的に停止された状態。                                     |
| `Running`  | run がまだ実行中で、最近ハートビートを送信した状態。                                |
| `Pending`  | run がスケジュールされているが、まだ開始されていない状態 (sweep や Launch ジョブで一般的です) 。 |

<div id="run-states-in-sweeps">
  ### sweep における run の状態
</div>

run が [sweep](/ja/models/sweeps/) の一部の場合、run の状態は sweep のステータスとは独立して動作します：

* **個々の run の状態** は、各 run の実行状態 (Running、Finished、Failed など) を反映します
* **sweep のステータス** は、新しい run の作成を制御するものであり、既存の run の実行方法には影響しません
* sweep を一時停止または停止しても、実行中の run には影響しません
* sweep をキャンセルした場合にのみ、実行中の run を強制的に終了します (状態が `Killed` に変更されます)

sweep と run のステータスがどのように相互作用するかの詳細な説明については、[Understanding sweep and run statuses](/ja/models/sweeps/pause-resume-and-cancel-sweeps#understanding-sweep-and-run-statuses) を参照してください。

<div id="view-the-state-of-a-run">
  ## run の状態を確認する
</div>

Python SDK または W&amp;B App を使用して、プログラムから、またはインタラクティブに run の状態を確認できます。

<Tabs>
  <Tab title="Python SDK">
    [`wandb.Api.Run`](/ja/models/ref/python/public-api/runs) オブジェクトの `state` プロパティを使用して、run の現在の状態にアクセスします。

    次のコードスニペットは、指定したプロジェクト内のすべての run の状態を取得して出力します。以下のコードスニペットを Python 環境にコピー＆ペーストし、山かっこ (`< >`) で囲まれた値を自分の値に置き換えてください。

    ```python
    import wandb

    api = wandb.Api()

    runs = api.runs(path="<entity>/<project>")

    # Access run object's properties
    for run in runs:
        print(f"Run: {run.name}")
        print(f"Run state: {run.state}")
        print()
    ```

    さまざまな条件に基づいてフィルターを適用し、プロジェクトから run を取得できます。[Filter runs](/ja/models/runs/filter-runs) を参照して、プログラムから run をフィルタリングする方法の詳細を確認してください。
  </Tab>

  <Tab title="W&B App">
    W&amp;B App から run の状態を確認するには、次の手順に従います。

    1. 自分の W&amp;B プロジェクトにアクセスします。
    2. プロジェクトのサイドバーから **Workspace** または **Runs** タブを選択します。
    3. 表示したい run を検索またはスクロールして見つけます。
    4. run を選択して run の概要ページを開きます。
    5. **Overview** タブを選択します。

    **State** フィールドの横に、run の現在の状態が表示されます。
  </Tab>
</Tabs>
