runを*グループ*に整理します。グループは、特定のモデルのトレーニング runや特定のデータセットの評価 runなど、共通の目的を持つrunのコレクションです。

*ジョブタイプ*など、その他のプロパティでもrunを整理できます。[Job types](/ja/models/runs/grouping#organize-runs-by-job-type)は、`preprocessing`、`training`、`evaluation` など、runの機能を示します。

<div id="organize-runs-into-groups">
  ## run をグループにまとめる
</div>

W&amp;B Python SDK を使用してプログラム的に、または W&amp;B App でインタラクティブに、run をグループに追加できます。

<Note>
  W&amp;B はグループ名を run の [`wandb.Run.group`](/ja/models/ref/python/experiments/run#property-run-group) プロパティとして保存します。
</Note>

<Tabs>
  <Tab title="Python SDK">
    W&amp;B Python SDK を使用して、1 つ以上の Runs をプログラム的にグループに追加できます。`wandb.init(group="")` で run を初期化する際に、`group` パラメーターにグループ名を指定します。グループ名を使うと、W&amp;B App で Runs を整理およびフィルタリングできます。

    次の例では、`A`、`B`、`C` という 3 つのグループを作成します。各グループには 3 つの Runs が含まれます。

    ```python
    import wandb

    entity = "<entity>"
    project = "<project>"

    for group in ["A", "B", "C"]:
        for i in range(3):
            with wandb.init(entity=entity, project=project, group=group, name=f"{group}_run_{i}") as run:
                # Simulate some training
                for step in range(100):
                    run.log({
                        "acc": 0.5 + (step / 100) * 0.3 + (i * 0.05),
                        "loss": 1.0 - (step / 100) * 0.5
                    })
    ```

    プロジェクトの Workspace で、グループごとに整理された Runs を確認できます。次の画像は、Runs table をグループ名で整理した例を示しています。`A`、`B`、`C` という 3 つのグループが Runs table に表示され、それぞれに 3 つの Runs が含まれています。

    <Frame>
      <img src="/images/runs/group_name_3_groups.png" alt="グループ名でグループ化された Runs table" />
    </Frame>
  </Tab>

  <Tab title="W&B App">
    1. 対象の W&amp;B プロジェクトにアクセスします。
    2. プロジェクトのサイドバーから **Runs** タブを選択します。
    3. run の一覧の上にある **Group** ボタンをクリックします。
    4. グループ化したい 1 つ以上の Runs の横にあるチェックボックスをクリックします。
    5. **Move to group** を選択します。
    6. ドロワー で既存のグループを選択するか、新しいグループを作成します。
  </Tab>
</Tabs>

<div id="view-groups">
  ### グループを表示
</div>

W&amp;B App でグループごとに整理された Runs を表示するには、次の手順に従います：

1. プロジェクトのサイドバーで **Runs** タブを選択します。
2. Runs の一覧の上にある **Group** ボタンをクリックします。
3. ドロップダウンから **Group** を選択します。

<div id="move-runs-between-groups">
  ### グループ間で run を移動する
</div>

Runs をあるグループから別のグループに移動します：

1. W&amp;B プロジェクトにアクセスします。
2. プロジェクトのサイドバーから **Runs** タブを選択します。
3. チェックボックスをクリックして、1 つ以上の Runs を選択します。
4. 表の上部にある **Move to group** をクリックします。
5. ドロワー内で、移動先のグループを選択するか、新しいグループを作成します。
6. **Move** をクリックします。

<div id="remove-runs-from-a-group">
  ### グループから run を削除する
</div>

1. 対象の W&amp;B プロジェクトにアクセスします。
2. プロジェクトのサイドバーから **Runs** タブを選択します。
3. run の一覧の上にある **Group** ボタンをクリックします。
4. ドロップダウンで、削除したいグループの名の横にある **X** を選択します。

<div id="delete-a-group">
  ### グループを削除する
</div>

グループを削除するには、そのグループに含まれるすべての Runs を削除します。これによりグループは自動的に削除されます。

<div id="organize-runs-by-job-type">
  ## ジョブタイプごとに run を整理する
</div>

Runs を *ジョブタイプ* ごとに整理します。ジョブタイプは、`preprocessing`、`training`、`evaluation` など、run の機能を示します。

<Note>
  run のジョブタイプは、run の [`wandb.Run.job_type`](/ja/models/ref/python/experiments/run#property-run-job-type) プロパティにアクセスして確認できます。
</Note>

`wandb.init(job_type="")` に `job_type` パラメーターを渡すことで、run にジョブタイプを追加できます。たとえば、次のコードスニペットは、`training` または `evaluation` のジョブタイプを持つ Runs を作成します。

```python
import wandb

entity = "<entity>"
project = "<project>"

for job_type in ["training", "evaluation"]:
    for i in range(2):
        with wandb.init(entity=entity, project=project, job_type=job_type, name=f"{job_type}_run_{i}") as run:
            # 何らかの処理をシミュレートする
            for step in range(50):
                run.log({
                    "metric1": 0.2 + (step / 50) * 0.4 + (i * 0.03),
                    "metric2": 0.8 - (step / 50) * 0.3
                })

```

<div id="view-runs-organized-by-job-type">
  ### ジョブタイプごとに整理された run を表示する
</div>

W&amp;B App で Runs をグループごとに表示するには、次の手順に従います：

1. プロジェクトのサイドバーで **Runs** タブを選択します。
2. run の一覧の上にある **Group** ボタンをクリックします。
3. ドロップダウンから **Job Type** を選択します。
