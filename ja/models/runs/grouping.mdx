自分の run を *グループ* にまとめて整理します。グループとは、共通の目的を持つ run の集合です。たとえば、特定のモデルのトレーニング run や、特定のデータセットの評価 run などです。

また、*ジョブタイプ* などの他のプロパティごとに run を整理することもできます。[ジョブタイプ](/ja/models/runs/grouping#organize-runs-by-job-type) は、その run の役割を `preprocessing`、`training`、`evaluation` などのように示します。

<div id="organize-runs-into-groups">
  ## run をグループにまとめる
</div>

W&amp;B Python SDK を使用してプログラム的に、または W&amp;B App 上でインタラクティブに、run をグループに追加できます。

<Note>
  W&amp;B はグループ名を run の [`wandb.Run.group`](/ja/models/ref/python/experiments/run#property-run-group) プロパティとして保存します。
</Note>

<Tabs>
  <Tab title="Python SDK">
    W&amp;B Python SDK を使用して、1 つ以上の run をプログラム的にグループに追加できます。`wandb.init(group="")` で run を初期化するときに、`group` パラメーターの引数としてグループ名を渡します。グループ名を使うと、W&amp;B App で run を整理およびフィルタリングできます。

    次の例では、`A`、`B`、`C` という 3 つのグループを作成します。各グループには 3 つの run が含まれます。

    ```python
    import wandb

    entity = "<entity>"
    project = "<project>"

    for group in ["A", "B", "C"]:
        for i in range(3):
            with wandb.init(entity=entity, project=project, group=group, name=f"{group}_run_{i}") as run:
                # Simulate some training
                for step in range(100):
                    run.log({
                        "acc": 0.5 + (step / 100) * 0.3 + (i * 0.05),
                        "loss": 1.0 - (step / 100) * 0.5
                    })
    ```

    プロジェクトのワークスペースで、グループごとに整理された run を表示できます。次の画像は、run テーブルをグループ名で整理した様子を示しています。`A`、`B`、`C` という 3 つのグループが run テーブルに表示され、それぞれに 3 つの run が含まれています。

    <Frame>
      <img src="/images/runs/group_name_3_groups.png" alt="グループ名でグループ化された Runs テーブル" />
    </Frame>
  </Tab>

  <Tab title="W&B App">
    1. 対象の W&amp;B プロジェクトに移動します。
    2. プロジェクトのサイドバーから **Runs** タブを選択します。
    3. run の一覧の上にある **Group** ボタンをクリックします。
    4. グループ化したい 1 つ以上の run の横にあるチェックボックスをクリックします。
    5. **Move to group** を選択します。
    6. ドロワーで既存のグループを選択するか、新しいグループを作成します。
  </Tab>
</Tabs>

<div id="view-groups">
  ### グループを表示する
</div>

W&amp;B App でグループ別に整理された run を表示するには、次の操作を行います：

1. プロジェクトのサイドバーで **Runs** タブを選択します。
2. run の一覧の上にある **Group** ボタンをクリックします。
3. ドロップダウンから **Group** を選択します。

<div id="move-runs-between-groups">
  ### グループ間で run を移動する
</div>

run をあるグループから別のグループに移動します。

1. W&amp;B プロジェクトに移動します。
2. プロジェクト サイドバーから **Runs** タブを選択します。
3. チェックボックスをクリックして、1 つ以上の run を選択します。
4. テーブルの上部にある **Move to group** をクリックします。
5. ドロワー内で、移動先のグループを選択するか、新しいグループを作成します。
6. **Move** をクリックします。

<div id="remove-runs-from-a-group">
  ### グループから run を削除する
</div>

1. 対象の W&amp;B プロジェクトを開きます。
2. プロジェクトのサイドバーから **Runs** タブを選択します。
3. run の一覧の上にある **Group** ボタンをクリックします。
4. ドロップダウンで、削除したいグループ名の横にある **X** をクリックします。

<div id="delete-a-group">
  ### グループを削除する
</div>

グループを削除するには、そのグループに含まれるすべての run を削除します。これによりグループは自動的に削除されます。

<div id="organize-runs-by-job-type">
  ## ジョブタイプごとに run を整理する
</div>

run をその *ジョブタイプ* ごとに整理します。ジョブタイプは、`preprocessing`、`training`、`evaluation` など、run の種類や役割を表します。

<Note>
  run のジョブタイプは、run の [`wandb.Run.job_type`](/ja/models/ref/python/experiments/run#property-run-job-type) プロパティにアクセスして確認できます。
</Note>

`wandb.init(job_type="")` に `job_type` パラメーターを渡すことで、run にジョブタイプを追加できます。たとえば、次のコードスニペットは、`training` または `evaluation` のいずれかのジョブタイプを持つ run を作成します。

```python
import wandb

entity = "<entity>"
project = "<project>"

for job_type in ["training", "evaluation"]:
    for i in range(2):
        with wandb.init(entity=entity, project=project, job_type=job_type, name=f"{job_type}_run_{i}") as run:
            # 何らかの処理をシミュレートする
            for step in range(50):
                run.log({
                    "metric1": 0.2 + (step / 50) * 0.4 + (i * 0.03),
                    "metric2": 0.8 - (step / 50) * 0.3
                })

```

<div id="view-runs-organized-by-job-type">
  ### ジョブタイプごとに整理された run を表示する
</div>

W&amp;B App で run をグループごとに表示するには、次の手順に従います：

1. プロジェクトのサイドバーで **Runs** タブを選択します。
2. run の一覧の上にある **Group** ボタンをクリックします。
3. ドロップダウンから **Job Type** を選択します。
