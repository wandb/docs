run を *グループ* 単位で整理できます。グループとは、特定のモデル向けの学習 run や、特定のデータセット向けの評価 run など、共通の目的を持つ run の集合です。

また、*ジョブタイプ* などの他のプロパティで run を整理することもできます。[ジョブタイプ](/ja/models/runs/grouping#organize-runs-by-job-type) は、`preprocessing`、`training`、`evaluation` など、run が果たす役割を示します。

  `WANDB_RUN_GROUP` を環境変数として使用して、run のグループを指定します。詳細については、[Environment Variables](/models/track/environment-variables/) のドキュメントを参照してください。**Group** は、プロジェクト内で一意であり、そのグループ内のすべての run で共有される必要があります。すべてのプロセスで使用する一意の 8 文字の文字列を生成するには、`wandb.util.generate_id()` を使用できます。たとえば、`os.environ["WANDB_RUN_GROUP"] = "experiment-" + wandb.util.generate_id()` のように指定します。 */}

<div id="organize-runs-into-groups">
  ## run をグループに整理する
</div>

run をグループに追加するには、W&amp;B Python SDK を使ってプログラムで追加するか、W&amp;B App から手動で追加できます。

<Note>
  W&amp;B はグループ名を run の [`wandb.Run.group`](/ja/models/ref/python/experiments/run#property-run-group) プロパティとして保存します。
</Note>

<Tabs>
  <Tab title="Python SDK">
    W&amp;B Python SDK を使って、1 つ以上の run をプログラムでグループに追加できます。`wandb.init(group="")` で run を初期化するときに、`group` パラメータの引数としてグループ名を渡します。グループ名を使って、W&amp;B App 内で run を整理したり、フィルタリングしたりできます。

    次の例では、`A`、`B`、`C` という 3 つのグループを作成します。各グループには 3 つの run が含まれます。

    ```python
    import wandb

    entity = "<entity>"
    project = "<project>"

    for group in ["A", "B", "C"]:
        for i in range(3):
            with wandb.init(entity=entity, project=project, group=group, name=f"{group}_run_{i}") as run:
                # 学習をシミュレートする
                for step in range(100):
                    run.log({
                        "acc": 0.5 + (step / 100) * 0.3 + (i * 0.05),
                        "loss": 1.0 - (step / 100) * 0.5
                    })
    ```

    プロジェクトのワークスペースで、グループごとに整理された run を確認できます。次の画像は、run テーブルをグループ名で整理した様子を示しています。`A`、`B`、`C` という名前の 3 つのグループが run テーブルに表示され、それぞれ 3 つの run を含みます。

    <Frame>
      <img src="/images/runs/group_name_3_groups.png" alt="グループ名でグループ化された Runs テーブル" />
    </Frame>
  </Tab>

  <Tab title="W&B App">
    1. W&amp;B プロジェクトに移動します。
    2. プロジェクトのサイドバーから **Runs** タブを選択します。
    3. run の一覧の上にある **Group** ボタンをクリックします。
    4. グループ化したい 1 つ以上の run の横にあるチェックボックスをクリックします。
    5. **Move to group** を選択します。
    6. ドロワーで既存のグループを選択するか、新しいグループを作成します。
  </Tab>
</Tabs>

<div id="view-groups">
  ### グループを表示
</div>

W&amp;B App でグループごとに整理された run を表示します：

1. プロジェクトのサイドバーで **Runs** タブを選択します。
2. run 一覧の上にある **Group** ボタンをクリックします。
3. ドロップダウンメニューから **Group** を選択します。

<div id="move-runs-between-groups">
  ### グループ間でrunを移動する
</div>

runをあるグループから別のグループに移動します。

1. 対象のW&amp;Bプロジェクトを開きます。
2. プロジェクトのサイドバーから **Runs** タブを選択します。
3. チェックボックスをクリックして、1つ以上のrunを選択します。
4. テーブル上部の **Move to group** をクリックします。
5. ドロワー内で、移動先のグループを選択するか、新しいグループを作成します。
6. **Move** をクリックします。

<div id="remove-runs-from-a-group">
  ### グループから run を削除する
</div>

1. 対象の W&amp;B プロジェクトに移動します。
2. プロジェクトのサイドバーから **Runs** タブを選択します。
3. run の一覧の上にある **Group** ボタンをクリックします。
4. ドロップダウンメニューで、削除したいグループ名の横にある **X** をクリックします。

<div id="delete-a-group">
  ### グループを削除する
</div>

グループを削除するには、そのグループに含まれるすべての run を削除します。すると、グループは自動的に削除されます。

<div id="organize-runs-by-job-type">
  ## run をジョブタイプ別に整理する
</div>

run を *ジョブタイプ* 別に整理します。ジョブタイプは、`preprocessing`、`training`、`evaluation` など、その run の役割を示します。

<Note>
  run の [`wandb.Run.job_type`](/ja/models/ref/python/experiments/run#property-run-job-type) プロパティにアクセスして、run のジョブタイプを確認できます。
</Note>

`wandb.init(job_type="")` に `job_type` パラメータを渡すことで、run にジョブタイプを設定できます。たとえば、次のコードスニペットは、`training` または `evaluation` のいずれかのジョブタイプを持つ run を作成します。

```python
import wandb

entity = "<entity>"
project = "<project>"

for job_type in ["training", "evaluation"]:
    for i in range(2):
        with wandb.init(entity=entity, project=project, job_type=job_type, name=f"{job_type}_run_{i}") as run:
            # 何らかの処理をシミュレートする
            for step in range(50):
                run.log({
                    "metric1": 0.2 + (step / 50) * 0.4 + (i * 0.03),
                    "metric2": 0.8 - (step / 50) * 0.3
                })

```

<div id="view-runs-organized-by-job-type">
  ### ジョブタイプ別に整理された run を表示する
</div>

W&amp;B App でグループごとに整理された run を表示するには、次の手順に従います:

1. プロジェクトのサイドバーで **Runs** タブを選択します。
2. run の一覧の上にある **Group** ボタンをクリックします。
3. ドロップダウンメニューから **Job Type** を選択します。
