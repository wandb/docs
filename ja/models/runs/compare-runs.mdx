多数の run があるワークスペースでは、高い性能を示す run、本番 モデル、失敗した実験、重要な比較基準を把握しておくのが難しいことがあります。W&amp;B App には、run の整理や比較に役立つ機能があります。

* **ピン留めされた run**: 最大 6 つの run をピン留めして、ワークスペース内と run リストの先頭に常に表示できます。ベースライン run がある場合は、ベースラインが暗黙的にピン留めされるため、ピン留めできる run は最大 5 つです。
* **ベースライン run**: 比較の基準点としてベースライン run を指定します。ベースライン run は、ワークスペース内と run リストの先頭に常に表示されます。runs table では、サマリー メトリクスの差分により、各 run がベースラインと比べてどうかを確認できます。折れ線グラフでは、比較しやすいように、ベースラインは視覚的に区別しやすいスタイルで表示されます。

![ベースラインとピン留めされた run を含む折れ線グラフ](/images/models/pinned-and-baseline-runs/baseline-run-visual-distinction.png)

これらの機能は、特に次のような場合に役立ちます。

* 新しい実験を本番 モデルと比較する。
* 実験中に複数の候補モデルをトラッキングする。
* 新しい run が最良の結果を上回っているかどうかを評価する。

[制限事項](#limitations) を参照してください.

<div id="manage-the-baseline-run">
  ## ベースライン run を管理する
</div>

ワークスペース内の他の run を評価する際の参照点として使用できるように、1 つの run をベースラインとして指定できます。

runs セレクターと runs table では、ベースライン run はピン留めされた run と並んで上部に表示され、ピンではなくブックマーク アイコンが付きます。

折れ線グラフでは、ベースライン run の線は他の線よりも太く表示されます。プロットまたは凡例にカーソルを合わせると、ベースライン run の線は破線になります。

<Frame>
  ![ベースラインと別の run を比較するデモ](/images/models/pinned-and-baseline-runs/line-plot-baseline-comparison.png)
</Frame>

<div id="set-a-baseline-run">
  ### ベースライン run を設定する
</div>

ベースライン run を設定するには、次の手順に従います。

1. ワークスペースにアクセスします。
2. run selector または runs table で、ベースラインとして使用する run を検索します。
3. アクション メニュー `...` をクリックし、**Set as baseline** を選択します。

ベースライン run は run selector の上部に表示され、他の run とは区切り線で分けられます。ベースライン run には、円ではなくブックマーク アイコンが表示されます。

<Frame>
  ![ベースライン run とピン留めされた run が表示された runs table](/images/models/pinned-and-baseline-runs/runs-table-with-pinned-and-baseline-runs.png)
</Frame>

<div id="change-the-baseline-run">
  ### ベースライン run を変更する
</div>

同時にベースラインに設定できる run は 1 つだけです。どの run をベースラインにするかを変更するには、次の手順に従います。

1. ワークスペースにアクセスします。
2. run selector または runs table で、新しいベースラインとして使用する run を探します。
3. `...` アクションメニューをクリックし、**Replace baseline** を選択します。 <Note>メニュー項目が無効な場合は、使用可能なピン留めスロットが少なくとも 1 つあることを確認してください。必要に応じて、ピン留めされた run の横にある円形のピンアイコンをクリックして、その run のピン留めを解除します。</Note>
4. 新しい run がベースラインになり、以前のベースラインは自動的にピン留めされるため、後で見つけやすくなります。必要に応じて、そのピンアイコンをクリックしてピン留めを解除します。

<div id="remove-the-baseline-designation">
  ### ベースライン指定を解除する
</div>

ベースライン指定を解除するには、次の手順を実行します。

1. ワークスペースにアクセスします。
2. run セレクターまたは runs table で、現在のベースライン run を見つけます。
3. アクション `...` メニューをクリックし、**ベースラインを削除** を選択します。 <Note>メニュー項目が無効になっている場合は、利用可能なピン留めスロットが少なくとも 1 つあることを確認してください。必要に応じて、ピン留めされている run の横にある円形のピンアイコンをクリックして、run のピン留めを解除してください。</Note>
4. 以前のベースラインは、自動的にピン留めされるため簡単に見つけられます。必要に応じて、ピンアイコンをクリックしてピン留めを解除します。

<div id="compare-runs-to-the-baseline">
  ## run をベースラインと比較する
</div>

ベースライン run は、その run がログしたメトリクスの折れ線グラフに常に表示されます。折れ線グラフでは、ベースライン run の線は他の線よりも太く表示されます。

* プロット上の任意の箇所にカーソルを合わせると、ベースライン run とピン留めされた run を含む、表示中のすべての Runs の値を示すツールチップが表示されます。
  <Frame>
    ![ある時点で表示中のすべての Runs の詳細を示すデモ](/images/models/pinned-and-baseline-runs/line-plot-point-details.png)
  </Frame>
* ベースライン run の凡例ラベルにカーソルを合わせると、その線が強調表示されます。太い破線として表示され、表示中の他の Runs の線は彩度が下がって表示されます。
  <Frame>
    ![ベースライン run の詳細を示すデモ](/images/models/pinned-and-baseline-runs/line-plot-baseline-run-foreground.png)
  </Frame>
* 別の run の凡例ラベルにカーソルを合わせると、その run の線が強調表示され、太い破線で表示されるベースラインと比較できます。表示中の他の Runs の線は彩度が下がって表示されます。
  <Frame>
    ![別の run とベースラインを比較するデモ](/images/models/pinned-and-baseline-runs/line-plot-baseline-comparison.png)

<div id="summary-metric-deltas">
  ### サマリー メトリクスの差分
</div>

ある run がベースラインとして設定されると、デフォルトでは、ベースライン run と同じサマリー メトリクスをログする他のすべての run に、そのメトリクスのベースラインからの差分 (変化量) が表示されます。この差分は、Runs table の各 run の行で、そのメトリクス値の右側に表示されます。

デフォルトでは、差分は濃いグレーのテキストが濃いグレーの背景で表示されます。すばやく視覚的に把握できるように意味に基づく色分けを有効にするには、列の **Metric directionality** を設定します。directionality を設定すると、次のようになります。

* 他の run がベースラインより **優れている**  (方向性としてより良い) 場合、差分は濃い赤のテキストが薄い赤の背景で表示されます。
* 他の run がベースラインより **劣っている**  (方向性としてより悪い) 場合、差分は濃いティールのテキストが薄いティールの背景で表示されます。

メトリクスの directionality を設定するには、次の手順に従います。

1. Runs table で、そのメトリクスの列見出しにカーソルを合わせます。
2. 表示される `...` アクション メニューをクリックします。
3. **Metric directionality** を **Higher values are best** または **Lower values are best** に設定します。

次のスクリーンショットは、runs `nanochat-train-base` と `nanochat-train-mid` を、ベースライン run `nanochat-train` と比較したものです。`TOTAL_TRAINING_TIME`、`TRAIN/DT`、`TRAIN/GRAD_NORM` の差分メトリクスが表示されています。
![ベースライン run からのサマリー メトリクス差分を比較したスクリーンショット](/images/models/pinned-and-baseline-runs/baseline_run_deltas.png)

<div id="hide-summary-metric-deltas-in-a-workspace">
  ## ワークスペースでサマリー メトリクスの差分を非表示にする
</div>

デフォルトでは、ベースライン run があるワークスペースでは、サマリー メトリクスの差分が常に表示されます。これらを非表示にするには、次の手順に従います。

1. ワークスペースで **Settings** をクリックします。
2. 表示されたドロワーで **Runs** をクリックします。
3. **Baseline** タブで、**Show value deltas in the runs table** をオフにします。
4. ワークスペース設定ドロワーを閉じます。

<div id="use-cases">
  ## ユースケース
</div>

このセクションでは、ピン留めした run とベースライン run が実験の方向性を判断するのに役立つ代表的なシナリオを紹介します。

* **本番モデルをトラッキングする**: 新しいモデルがデプロイ前に品質基準を満たしていることを確認します。
  1. 本番モデルをベースラインとして設定します。
  2. デプロイ済みモデルを基準にすべての実験を比較し、本番を上回る候補を特定します。
* **ハイパーパラメーター実験を比較する**: ハイパーパラメーター sweep や手動で行った実験を、既知の最良の設定と比較して評価します。
  1. 既知の最良の設定をベースラインとして設定します。
  2. 有望な候補を見つけたらピン留めします。
  3. 折れ線グラフを使用して、ベースラインに対する run を視覚的に比較します。
  4. より良い設定が見つかるたびに、ベースラインを更新します。

<div id="example-workflow">
  ## 例のワークフロー
</div>

このセクションでは、ピン留めした run と ベースライン run を使って、run 同士を比較する方法を説明します。

1. 一連の run によるハイパーパラメーター調整のシナリオをシミュレートする次のコードを実行します。山かっこ (`<>`) で囲まれたプレースホルダーは、自分の値に置き換えてください。

   ```python
   import wandb
   import random
   import math

   def train_model(learning_rate, batch_size, run_name, tags=None):
       """指定したハイパーパラメーターでモデルをトレーニングする様子をシミュレートします。"""
       config = {
           "learning_rate": learning_rate,
           "batch_size": batch_size,
           "optimizer": "adam",
           "architecture": "resnet50"
       }
       
       with wandb.init(
         # 自分のチーム名とプロジェクト名に置き換えてください
           project="hyperparameter-tuning",
           entity="<team>",
           name=run_name,
           config=config,
           tags=tags or []
       ) as run:
           # トレーニング ループをシミュレート
           for epoch in range(50):
               # シミュレートしたメトリクス
               accuracy = 0.6 + 0.3 * (1 - math.exp(-learning_rate * epoch / 10))
               loss = 1.0 * math.exp(-learning_rate * epoch / 10)
               
               run.log({
                   "epoch": epoch,
                   "accuracy": accuracy,
                   "loss": loss
               })

   # 標準設定で ベースライン run を作成
   train_model(
       learning_rate=0.001,
       batch_size=64,
       run_name="baseline-config",
       tags=["baseline", "production"]
   )

   # 異なる学習率で実験
   train_model(
       learning_rate=0.003,
       batch_size=64,
       run_name="lr-experiment-0.003",
       tags=["experiment"]
   )

   train_model(
       learning_rate=0.0001,
       batch_size=64,
       run_name="lr-experiment-0.0001",
       tags=["experiment"]
   )
   ```

   このコードを実行すると、ワークスペースに 3 つの run が作成されます。
2. `baseline-config` を ベースライン run に設定します。
3. `baseline-config` をピン留めして表示したままにします。
4. 実験用の run を baseline と比較します。

* runs table で、各 run の値の横に表示される サマリー メトリクスの差分を確認し、baseline と比較します。
  * 折れ線グラフでは、常に表示される baseline と 1 つ以上の run のパフォーマンスを比較できます。

5. さらに調査するために、有望な実験をピン留めします。この例では、50 エポック後の `lr-experiment-0.003` が最も高い accuracy (`~0.64`) と最も低い 損失 (`~0.86`) を示します。

<div id="limitations">
  ## 制限事項
</div>

以下の機能は、ピン留めされた run とベースライン run ではまだサポートされていません。

* **グループ化**: run selector または runs table で [run を表示](/ja/models/runs#view-logged-runs) しているとき、run が列ごとにグループ化されている場合、ピン留めされた run とベースライン run は、ほかの run と見た目で区別されません。
* **Reports**: [W&amp;B Report](/ja/models/reports) 内の run set では、ピン留めされた run とベースライン run は、ほかの run と見た目で区別されません。
* **Workspace view のみ**: 単一の run のワークスペースを表示している場合、ベースラインは表示されません。
* **折れ線グラフのみ**: ベースライン比較は折れ線グラフでのみ利用でき、棒グラフやメディアパネルなど、ほかのパネルではまだ利用できません。
