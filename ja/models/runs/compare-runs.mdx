多数の run が存在するワークスペースでは、最良のパフォーマンスを出したものや本番用モデル、失敗した実験、重要な参照ポイントを追跡するのが難しくなることがあります。W&amp;B App には、run を整理して比較しやすくするための機能があります。

* **Pinned runs**: 最大 6 個の run をピン留めして、ワークスペース内および run リストの先頭に常に表示しておくことができます。ベースライン run がある場合、ベースラインは暗黙的にピン留めされるため、ピン留めできる run は最大 5 個です。
* **Baseline run**: 比較のための参照ポイントとしてベースライン run を指定します。ベースライン run は常にワークスペース内および run リストの先頭に表示されます。折れ線グラフでは、ベースラインは比較しやすいように視覚的に区別されたスタイルで表示されます。

これらの機能は次のような場合に特に有用です。

* 新しい実験を本番用モデルと比較する。
* 実験中に複数の候補モデルを追跡する。
* 新しい run がこれまでの最良の結果を上回っているかどうかを評価する。

<div id="manage-the-baseline-run">
  ## ベースライン run を管理する
</div>

ワークスペースの基準点として使用するために、1 つの run を Workspace のベースラインとして指定できます。これにより、ワークスペース内の他の run を評価する際の比較対象として利用できます。

Runs セレクターと Runs テーブルでは、ベースライン run はピン留めされた run と並んで最上部に表示され、ピンアイコンの代わりにブックマークアイコンが表示されます。

折れ線プロットでは、ベースライン run の線は他の線より太く表示されます。プロットまたは凡例にカーソルを合わせると、ベースライン run の線は破線になります。

<Frame>
  ![別の run をベースラインと比較するデモ](/images/models/pinned-and-baseline-runs/line-plot-baseline-comparison.png)
</Frame>

<div id="set-a-baseline-run">
  ### ベースライン run を設定する
</div>

ベースライン run を設定するには、次の手順を実行します。

1. ワークスペースに移動します。
2. run セレクターまたは Runs テーブルで、ベースラインとして使用したい run を探します。
3. アクション `...` メニューをクリックし、**Set as baseline**（ベースラインに設定）を選択します。

ベースライン run は run セレクターの最上部に表示され、他の run とは視覚的な区切り線で分けられます。ベースライン run には、円の代わりにブックマークのアイコンが表示されます。

<Frame>
  ![ベースライン run とピン留めされた run が表示された Runs テーブル](/images/models/pinned-and-baseline-runs/runs-table-with-pinned-and-baseline-runs.png)
</Frame>

<div id="change-the-baseline-run">
  ### ベースライン run を変更する
</div>

同時にベースラインにできる run は 1 つだけです。ベースラインにする run を変更するには、次の手順を実行します。

1. ワークスペースに移動します。
2. run セレクターまたは Runs テーブルで、新しいベースラインとして使用したい run を探します。
3. アクション `...` メニューをクリックし、**Replace baseline** を選択します。 <Note>メニュー項目が選択できない場合は、ピン留めスロットに空きが少なくとも 1 つあることを確認してください。必要に応じて、ピン留めされた run の横にある円形のピンアイコンをクリックして、その run のピンを外します。</Note>
4. 新しい run がベースラインになり、以前のベースラインは自動的にピン留めされるため、簡単に見つけられます。必要に応じて、そのピンアイコンをクリックしてピン留めを解除します。

<div id="remove-the-baseline-designation">
  ### ベースライン指定を解除する
</div>

ベースライン指定を解除するには、次の手順を実行します。

1. ワークスペースに移動します。
2. run セレクターまたは Runs テーブルで、現在のベースライン run を探します。
3. アクション `...` メニューをクリックし、**Remove baseline** を選択します。 <Note>メニューアイテムが非アクティブな場合は、ピン留めスロットが少なくとも 1 つ空いていることを確認してください。必要に応じて、ピン留めされた run の横にある円形のピンアイコンをクリックして、その run のピン留めを解除します。</Note>
4. 以前のベースライン run は自動的にピン留めされるため、簡単に見つけることができます。必要に応じて、ピンアイコンをクリックしてピン留めを解除します。

<div id="compare-runs-to-the-baseline">
  ## ベースライン run との比較
</div>

ベースライン run は、その run がログしたメトリクスの折れ線グラフで常に表示されます。折れ線グラフでは、ベースライン run の線は他の線より太く表示されます。

* プロット上の任意の箇所にカーソルを合わせると、ベースライン run やピン留めされた run を含む、表示中のすべての run の値がツールチップで表示されます。

* ベースライン run の凡例ラベルにカーソルを合わせると、その線が強調表示されます。太い破線として表示され、他の表示中の run の線は彩度を落として表示されます。
  <Frame>
    ![Demo showing details for the baseline run](/images/models/pinned-and-baseline-runs/line-plot-baseline-run-foreground.png)

* 別の run の凡例ラベルにカーソルを合わせると、その run の線が強調表示され、太い破線として表示されるベースラインと比較できます。他の表示中の run の線は彩度を落として表示されます。

<div id="use-cases">
  ## ユースケース
</div>

このセクションでは、ピン留めされた run とベースライン run が Experiments を進める際にどのように役立つかを、いくつかのシナリオを通して説明します。

* **本番モデルの追跡**: 新しいモデルがデプロイ前に品質基準を満たしていることを確認します。
  1. 本番モデルをベースラインとして設定します。
  2. すべての Experiments をデプロイ済みモデルと比較し、本番より優れた候補を特定します。
* **ハイパーパラメーター実験の比較**: ハイパーパラメーター sweep または手動の Experiments を、既知の最良の設定と比較して評価します。
  1. 既知の最良の設定をベースラインとして設定します。
  2. 見つけた有望な候補を順次ピン留めします。
  3. 折れ線グラフを使用して、ベースラインと各 run を視覚的に比較します。
  4. より良い設定が見つかったら、そのたびにベースラインを更新し続けます。

<div id="example-workflow">
  ## 例となるワークフロー
</div>

このセクションでは、ピン留めした run とベースライン run が run の比較にどのように役立つかを示します。

1. 一連の run によるハイパーパラメーター調整シナリオをシミュレートする次のサンプルコードを実行します。山かっこ（`<>`）で囲まれたプレースホルダーを自分の値に置き換えてください。

   ```python
   import wandb
   import random
   import math

   def train_model(learning_rate, batch_size, run_name, tags=None):
       """Simulate training a model with given hyperparameters."""
       config = {
           "learning_rate": learning_rate,
           "batch_size": batch_size,
           "optimizer": "adam",
           "architecture": "resnet50"
       }
       
       with wandb.init(
         # 自分の team と project 名に置き換えてください
           project="hyperparameter-tuning",
           entity="<team>",
           name=run_name,
           config=config,
           tags=tags or []
       ) as run:
           # トレーニングループをシミュレート
           for epoch in range(50):
               # シミュレートされたメトリクス
               accuracy = 0.6 + 0.3 * (1 - math.exp(-learning_rate * epoch / 10))
               loss = 1.0 * math.exp(-learning_rate * epoch / 10)
               
               run.log({
                   "epoch": epoch,
                   "accuracy": accuracy,
                   "loss": loss
               })

   # 標準的な設定でベースライン run を作成
   train_model(
       learning_rate=0.001,
       batch_size=64,
       run_name="baseline-config",
       tags=["baseline", "production"]
   )

   # 異なる learning rate で実験
   train_model(
       learning_rate=0.003,
       batch_size=64,
       run_name="lr-experiment-0.003",
       tags=["experiment"]
   )

   train_model(
       learning_rate=0.0001,
       batch_size=64,
       run_name="lr-experiment-0.0001",
       tags=["experiment"]
   )
   ```

   このコードを実行すると、ワークスペースには 3 つの run が作成されます。
2. `baseline-config` をベースライン run として設定します。
3. `baseline-config` をピン留めして、常に表示されるようにします。
4. ワークスペースの折れ線グラフを使って、ベースラインと比較する実験 run を確認します。
5. 有望な実験をさらに調査するためにピン留めします。この例では、50 エポック後に `lr-experiment-0.003` が最も高い accuracy（`~0.64`）と最も低い loss（`~0.86`）になっています。

<div id="limitations">
  ## 制限事項
</div>

<CloudOnly />

ピン留めされた run とベースライン run については、現在次の機能はサポートされていません。

* **グルーピング**: [run を表示](/ja/models/runs#view-logged-runs)する際に、run セレクターや runs テーブルで列ごとに run をグループ化している場合、ピン留めされた run とベースライン run は他の run と視覚的に区別されません。
* **Reports**: [W&amp;B Report](/ja/models/reports) 内の run セットでは、ピン留めされた run とベースライン run は他の run と視覚的に区別されません。
* **Workspace ビューのみ**: 単一の run の Workspace を表示している場合、ベースラインは表示されません。
* **折れ線グラフのみ**: ベースラインとの比較は折れ線グラフでのみ利用でき、棒グラフやメディアパネルなどの他のパネルではまだ利用できません。
