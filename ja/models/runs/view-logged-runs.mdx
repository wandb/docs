特定のrunの現在の状態、アーティファクト、メトリクスなどの情報を表示します。

<div id="view-a-logged-run">
  ## ログされた run を表示する
</div>

W&amp;B App または `wandb beta leet` ターミナル UI を使用して run を表示および監視できます。

<Tabs>
  <Tab title="WA&B App">
    W&amp;B App で特定の run を表示するには:

    1. [W&amp;B App](https://wandb.ai/home) に移動します。
    2. run を初期化したときに指定した W&amp;B プロジェクトに移動します。
    3. プロジェクトのサイドバー内で **Workspace** タブを選択します。
    4. run セレクタ内で、表示したい run をクリックするか、run 名の一部を入力して一致する run を絞り込みます。

    または、ブラウザに URL を入力して、特定の run のワークスペースに直接アクセスすることもできます。特定の run の URL パスは次の形式です:

    ```text
    https://wandb.ai/<team-name>/<project-name>/runs/<run-id>
    ```

    山かっこ（`< >`）で囲まれた部分を、チーム名、プロジェクト名、および [run ID](/ja/models/runs/run-identifiers#run-id) の実際の値に置き換えてください。

    タブを切り替えて [Overview](/ja/models/runs/view-logged-runs#overview)、[Logs](/ja/models/runs/view-logged-runs#logs)、[Files](/ja/models/runs/view-logged-runs#files)、[Code](/ja/models/runs/view-logged-runs#code)、[Artifacts](/ja/models/runs/view-logged-runs#artifacts) を参照し、run の各種プロパティを確認します。
  </Tab>

  <Tab title="LEET">
    `wandb beta leet` ターミナル UI を使用して、ターミナル上でローカルの run を表示するには:

    1. スクリプトからローカルで run を開始した場合は、コードを実行したディレクトリに移動します。そこには、run ごとのサブディレクトリと `latest-run/` シンボリックリンクを含む `wandb/` ディレクトリがあります。各 run ディレクトリには、`run-<run-ID>.wandb` という形式の名前のトランザクションログが含まれています。

       run をローカルで開始しておらず、代わりに `.wandb` トランザクションログファイルをダウンロードした場合は、その場所を控えておきます。
    2. 次のいずれかのコマンドで `wandb beta leet` を起動します:

       ```bash
       # ./wandb/latest-run/ に保存されている最新の run を表示
       wandb beta leet

       # run ディレクトリを指定
       wandb beta leet ./wandb/run-20250813_124246-n67z9ude

       # .wandb ファイルを指定
       wandb beta leet ./wandb/run-20250813_124246-n67z9ude/run-n67z9ude.wandb
       ```

    ![LEET ターミナル UI で run を表示している様子を示すスクリーンショット](/images/leet/leet-tui.png)

    LEET には 3 つのペインからなるインターフェースが表示されます:

    * **左ペイン**: 環境変数、設定、サマリ統計を含む run の概要
    * **中央ペイン**: ログされたメトリクスを表示する、ブライユ（点字）スタイルの折れ線グラフ付きメトリクスグリッド
    * **右ペイン**: GPU/CPU/RAM 使用率を含むシステムメトリクス

    次のキーボードショートカットを利用できます:

    * `h` または `?` - すべてのキーボードショートカットを表示
    * `/` - パターンでメトリクスをフィルタリング
    * `[` / `]` - 左/右ペインを切り替え
    * `n` / `N` - メトリクスページ間を移動
    * `q` / `CMD+C` - 終了

    詳しくは [`wandb beta leet`](/ja/models/ref/cli/wandb-beta/wandb-beta-leet) リファレンスを参照してください。
  </Tab>
</Tabs>

<div id="overview">
  ### 概要
</div>

**Overview** タブを使って、プロジェクト内の特定の run について次の情報を確認できます:

* **Author**: run を作成した W&amp;B エンティティ。
* **Command**: run を初期化するコマンド。
* **Description**: ユーザーが入力した run の説明。run の作成時に説明を指定しなかった場合、このフィールドは空のままです。W&amp;B App から、または Python SDK を使ってプログラムから run に説明を追加できます。
* **Tracked Hours**: 一時停止や待機時間を除き、run が実際に計算やデータのログ記録を行っている時間の総量。この指標は、run に費やされた実際の計算時間を把握するのに役立ちます。
* **Runtime**: run の開始から終了までの総時間を測定します。run の一時停止中やリソース待ちの時間も含めた、run のウォールクロック時間です。この指標は、run の経過時間全体を示します。
* **Git repository**: run に関連付けられた Git リポジトリ。[Git を有効化](/ja/platform/app/settings-page/user-settings/#personal-github-integration)すると、このフィールドを表示できます。
* **Host name**: W&amp;B が run を実行する場所。ローカルマシンで run を初期化した場合は、そのマシン名が表示されます。
* **Name**: run の名前。
* **OS**: run を初期化したオペレーティングシステム。
* **Python executable**: run を開始する Python 実行ファイル。
* **Python version**: run を作成した Python のバージョン。
* **Run path**: `entity/project/run-ID` の形式で表される一意の run 識別子。
* **Start time**: run を初期化したときのタイムスタンプ。
* **State**: [run の状態](/ja/models/runs/run-states)。
* **System hardware**: W&amp;B が run の計算に使用するハードウェア。
* **Tags**: 文字列のリスト。タグは、関連する run をまとめて整理したり、`baseline` や `production` のような一時的なラベルを付けたりするのに役立ちます。
* **W&amp;B CLI version**: run コマンドを実行したマシンにインストールされている W&amp;B CLI のバージョン。
* **Git state**: run が初期化されたリポジトリまたは作業ディレクトリの最新の Git コミット SHA。run の作成時に Git を有効化していない、または Git 情報が利用できない場合、このフィールドは空のままです。

W&amp;B は、概要セクションの下に次の情報も保存します:

* **Artifact Outputs**: run によって生成されたアーティファクトの出力。
* **Config**: [`wandb.Run.config`](/ja/models/track/config/) で保存された Config パラメータのリスト。
* **Summary**: [`wandb.Run.log()`](/ja/models/track/log/) で保存された Summary パラメータのリスト。デフォルトでは、W&amp;B はこの値を最後にログされた値に設定します。

プロジェクト概要の例は[こちら](https://wandb.ai/stacey/deep-drive/overview)から確認できます。

<div id="logs">
  ### ログ
</div>

**Log** タブでは、標準出力 (`stdout`) や標準エラー (`stderr`) など、コマンドラインに出力された内容が表示されます。

<Frame>
  <img src="/images/runs/run_logs_info.png" alt="Run logs tab" />
</Frame>

ログファイルをダウンロードするには、右上の **Download** ボタンをクリックします。

ログタブの例は[こちら](https://app.wandb.ai/stacey/deep-drive/runs/pr0os44x/logs)から確認できます。

<div id="files">
  ### ファイル
</div>

特定の run に関連付けられたファイル（モデルのチェックポイント、検証用データセットのサンプルなど）を表示するには、**Files** タブを使用します。

<Frame>
  <img src="/images/runs/runs_files_info.png" alt="Run files tab" />
</Frame>

**Files** タブの例は[こちら](https://app.wandb.ai/stacey/deep-drive/runs/pr0os44x/files/media/images)から確認できます。

<div id="code">
  ### Code
</div>

**Code** タブには、特定のrunに関連するコードファイルが表示されます。ここには、実行されたメインスクリプトだけでなく、そのrunの環境を構成していた追加のコードファイルも含まれます。

<div id="artifacts">
  ### Artifacts
</div>

**Artifacts** タブには、指定した run の入出力[アーティファクト](/ja/models/artifacts/)が一覧表示されます。

<Frame>
  <img src="/images/runs/runs_artifacts_info.png" alt="Run artifacts tab" />
</Frame>
