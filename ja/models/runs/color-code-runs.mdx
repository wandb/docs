メトリクスまたは設定パラメーターに基づいて W&amp;B Runs を色分けすることで、視覚的に意味のある折れ線グラフとプロット凡例を作成できます。パフォーマンス メトリクス (最高値、最低値、または最新の値) に応じて Runs に色を付けることで、Experiments 全体のパターンや傾向を特定します。W&amp;B は、選択したパラメーターの値に基づいて、Runs を自動的に色分けされたバケットにグループ化します。

Runs にメトリクスベースまたは設定ベースの色を使用するには、2 つの設定を構成する必要があります:&quot;

<div id="turn-on-key-based-colors">
  ### キーに基づく色分けを有効にする
</div>

1. W&amp;B プロジェクトにアクセスします。
2. プロジェクトのサイドバーから **Workspace** タブを選択します。
3. 画面右上の **Settings** アイコンをクリックします。
4. ドロワーから **Runs** を選択します。
5. **Run colors** セクションで **Key-based colors** を選択します。
6. 以下のオプションを設定します:
   * **Key** ドロップダウンから、Runs の色分けに使用するメトリクスを選択します。
   * **Y value** ドロップダウンから、Runs の色分けに使用する y 値を選択します。
   * バケット数を 2〜8 の値に設定します。

<Note>
  キーに基づく色分けを使用している場合、[Runs の色をカスタマイズする](/ja/models/runs/run-colors) オプションは利用できません。
</Note>

以下のセクションでは、メトリクスと y 値の設定方法、および Runs に色を割り当てるために使用するバケットのカスタマイズ方法について説明します。

<div id="example-key-based-coloring-with-loss-metric">
  ### 例: lossメトリクスを使用したキーに基づくカラー分け
</div>

この例のプロットでは、Runs はグラデーションで色分けされており、濃い色ほど loss 値が高く、薄い色ほど loss 値が低いことを表します。Y value は、各 Runs の最新の loss 値を使用するために `latest` に設定されています。

<div id="set-a-metric">
  ## メトリクスを設定する
</div>

**Key** ドロップダウンで選択できるメトリクスは、[W&amp;B にログするキーと値のペア](/ja/models/runs/color-code-runs/#custom-metrics) と [W&amp;B が定義するデフォルト メトリクス](/ja/models/runs/color-code-runs/#default-metrics) に基づいています。

<div id="default-metrics">
  ### デフォルトのメトリクス
</div>

* `Relative Time (Process)`: Runs の相対時間で、Runs の開始からの経過秒数で表されます。
* `Relative Time (Wall)`: Runs の相対時間で、Runs の開始からの経過秒数をウォールクロック時間に基づいて調整したものです。
* `Wall Time`: Runs のウォールクロック時間で、エポックからの経過秒数で表されます。
* `Step`: Runs の step 番号で、通常はトレーニングや評価の進行状況をトラッキングするために使用されます。

<div id="custom-metrics">
  ### カスタム メトリクス
</div>

トレーニングまたは評価スクリプトでログしたカスタム メトリクスに基づいてRunsに色を付け、意味のあるプロットの凡例を作成できます。カスタム メトリクスはキーと値のペアとしてログされ、キーはメトリクスの名、値はメトリクス値です。

たとえば、次のコード スニペットは、トレーニング ループ中にaccuracy (`"acc"` キー) とloss (`"loss"` キー) をログします。

```python
import wandb
import random

epochs = 10

with wandb.init(project="basic-intro") as run:
  # トレーニングループでメトリクスをログするブロックのシミュレーション
  offset = random.random() / 5
  for epoch in range(2, epochs):
      acc = 1 - 2 ** -epoch - random.random() / epoch - offset
      loss = 2 ** -epoch + random.random() / epoch + offset

      # スクリプトからW&Bにメトリクスをログする
      run.log({"acc": acc, "loss": loss})
```

**Key** ドロップダウンでは、`"acc"` と `"loss"` の両方を選択できます。

<div id="set-a-configuration-key">
  ## 設定キーを設定する
</div>

**Key** ドロップダウンの設定オプションは、W&amp;B run を初期化するときに `config` パラメーターに渡すキーと値のペアから取得されます。通常、設定キーは、トレーニングまたは評価スクリプトで使用するハイパーパラメーターやその他の設定をログするために使用されます。

```python
import wandb

config = {
  "learning_rate": 0.01,
  "batch_size": 32,
  "optimizer": "adam"
}

with wandb.init(project="basic-intro", config=config) as run:
  # トレーニングコードをここに記述
  pass
```

**Key** ドロップダウンでは、`"learning_rate"`、`"batch_size"`、および `"optimizer"` を選択できます.

<div id="set-a-y-value">
  ## Y value を設定する
</div>

次のオプションから選択できます。

* **Latest**: 各線の最後にログされた step の Y value を基準に色を決定します。
* **Max**: メトリクスに対してログされた最も高い Y value を基準に色を決定します。
* **Min**: メトリクスに対してログされた最も低い Y value を基準に色を決定します。

<div id="customize-buckets">
  ## バケットをカスタマイズする
</div>

バケットは、選択したメトリクスまたは設定キー기준으로 W&amp;B が Runs を分類するために使用する値の範囲です。バケットは、指定したメトリクスまたは設定キーの値の範囲全体に均等に分布し、それぞれのバケットには一意の色が부여されます。そのバケットの範囲内に入る Runs は、その色で表示されます。

次の例を見てみましょう：

<Frame>
  <img src="/images/track/color-coding-runs.png" alt="色分けされた Runs" />
</Frame>

* **Key** は `"Accuracy"` (短縮形は `"acc"`) に設定されています。
* **Y value** は `"Max"` に設定されています。

この設定では、W&amp;B は各 Runs を accuracy の値기준으로色分けします。色は淡い黄色から濃い色まで変化します。色が薄いほど accuracy の値が低く、色が濃いほど accuracy の値が高いことを表します。

このメトリクスには 6 つのバケットが定義されており、それぞれのバケットは accuracy の値の範囲を表します。**Buckets** セクション内では、次の範囲のバケットが定義されています：

* Bucket 1: (Min - 0.7629)
* Bucket 2: (0.7629 - 0.7824)
* Bucket 3: (0.7824 - 0.8019)
* Bucket 4: (0.8019 - 0.8214)
* Bucket 5: (0.8214 - 0.8409)
* Bucket 6: (0.8409 - Max)

下の折れ線グラフでは、最も高い accuracy (0.8232) の Runs は濃い紫色 (Bucket 5) で表示され、最も低い accuracy (0.7684) の Runs は薄いオレンジ色 (Bucket 2) で表示されています。他の Runs も accuracy の値기준으로色分けされ、色のグラデーションによって相対的なパフォーマンスが示されます。
