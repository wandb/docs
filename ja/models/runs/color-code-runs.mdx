description: "視覚的に意味のあるチャート凡例を作成するために、メトリクスまたは設定パラメーターに基づいて W&B run を色分けします。"
title: セマンティックな run プロット凡例
---

W&amp;B run をメトリクスまたは設定パラメーターに基づいて色分けすることで、視覚的に意味のある線グラフと凡例を作成できます。パフォーマンス メトリクス (最大値、最小値、または最新値) に応じて run に色を付けることで、実験全体のパターンや傾向を把握できます。W&amp;B は、選択したパラメーターの値に基づいて run を自動的に色分けされたバケットにグループ化します。

run にメトリクスまたは設定に基づく色を使用するには、2 つの設定を行う必要があります。

<div id="turn-on-key-based-colors">
  ### キーに基づく色分けを有効にする
</div>

1. W&amp;B プロジェクトに移動します。
2. プロジェクトのサイドバーから **Workspace** タブを選択します。
3. 右上隅の **Settings** アイコンをクリックします。
4. ドロワーから **Runs** を選択します。
5. **Run colors** セクションで **Key-based colors** を選択します。
6. 次のオプションを設定します:
   * **Key** ドロップダウンから、run の色を割り当てるために使用するメトリクスを選択します。
   * **Y value** ドロップダウンから、run の色を割り当てるために使用する y 値を選択します。
   * バケット数を 2 から 8 の範囲で設定します。

<Note>
  キーに基づく色分けを使用している場合は、[run colors をカスタマイズする](/ja/models/runs/run-colors) オプションは利用できません。
</Note>

次のセクションでは、メトリクスと y 値の設定方法、および run の色を割り当てるために使用するバケットをカスタマイズする方法について説明します。

<div id="example-key-based-coloring-with-loss-metric">
  ### 例：loss メトリクスを用いたキー ベースの色付け
</div>

このプロット例では、各 run がグラデーションで色分けされており、濃い色は高い loss 値を、薄い色は低い loss 値を表します。Y 値は `latest` に設定されており、各 run の最新の loss 値が使用されます。

<div id="set-a-metric">
  ## メトリクスを設定する
</div>

**Key** ドロップダウンで選択できるメトリクスのオプションは、[W&amp;B にログしたキーと値のペア](/ja/models/runs/color-code-runs/#custom-metrics)および W&amp;B によって定義された[デフォルトメトリクス](/ja/models/runs/color-code-runs/#default-metrics)から取得されます。

<div id="default-metrics">
  ### デフォルトメトリクス
</div>

* `Relative Time (Process)`: run の開始からの経過秒数を表す相対時間 (プロセス時間) です。
* `Relative Time (Wall)`: run の開始からの経過秒数を表す相対時間 (ウォールクロック時間で補正された値) です。
* `Wall Time`: run のウォールクロック時間で、エポックからの経過秒数で測定されます。
* `Step`: run のステップ番号で、通常はトレーニングや評価の進行状況を追跡するために使用されます。

<div id="custom-metrics">
  ### カスタムメトリクス
</div>

run を色分けしたり、トレーニングまたは評価用スクリプトでログしたカスタムメトリクスに基づいて意味のあるプロットの凡例を作成できます。カスタムメトリクスはキーと値のペアとしてログされ、キーはメトリクス名、値はそのメトリクスの値です。

たとえば、次のコードスニペットでは、トレーニングループ中に精度 (`"acc"` キー) と損失 (`"loss"` キー) をログします。

```python
import wandb
import random

epochs = 10

with wandb.init(project="basic-intro") as run:
  # トレーニングループでメトリクスをログするブロックのシミュレーション
  offset = random.random() / 5
  for epoch in range(2, epochs):
      acc = 1 - 2 ** -epoch - random.random() / epoch - offset
      loss = 2 ** -epoch + random.random() / epoch + offset

      # スクリプトから W&B にメトリクスをログする
      run.log({"acc": acc, "loss": loss})
```

**Key** ドロップダウンでは、`"acc"` と `"loss"` のいずれも選択できます。

<div id="set-a-configuration-key">
  ## 設定キーを指定する
</div>

**Key** ドロップダウンに表示される設定オプションは、W&amp;B run を初期化する際に `config` パラメーターへ渡したキーと値のペアから決まります。設定キーは通常、トレーニングや評価スクリプトで使用されるハイパーパラメーターやその他の設定をログするために使用されます。

```python
import wandb

config = {
  "learning_rate": 0.01,
  "batch_size": 32,
  "optimizer": "adam"
}

with wandb.init(project="basic-intro", config=config) as run:
  # トレーニングコードをここに記述
  pass
```

**Key** ドロップダウンでは、`"learning_rate"`、`"batch_size"`、`"optimizer"` が選択肢として利用できます。

<div id="set-a-y-value">
  ## y 値を設定する
</div>

次のオプションから選択できます。

* **Latest**: 各ラインについて、最後にログされたステップ時点の Y 値に基づいて色を決定します。
* **Max**: メトリクスに対してログされた最大の Y 値に基づいて色を決定します。
* **Min**: メトリクスに対してログされた最小の Y 値に基づいて色を決定します。

<div id="customize-buckets">
  ## バケットをカスタマイズする
</div>

バケットは、選択したメトリクスまたは設定キーに基づいて W&amp;B が run を分類するために使用する値の範囲です。バケットは、指定したメトリクスまたは設定キーの値の範囲全体に均等に分布し、それぞれのバケットには一意の色が割り当てられます。そのバケットの範囲内に入る run は、その色で表示されます。

次の例を見てみましょう：

<Frame>
  <img src="/images/track/color-coding-runs.png" alt="色分けされた run" />
</Frame>

* **Key** は `"Accuracy"` (短縮形は `"acc"`) に設定されています。
* **Y value** は `"Max"` に設定されています。

この設定では、W&amp;B は各 run を accuracy の値に基づいて色分けします。色は淡い黄色から濃い色まで変化します。色が薄いほど accuracy の値が低く、色が濃いほど accuracy の値が高いことを表します。

このメトリクスには 6 つのバケットが定義されており、それぞれのバケットは accuracy の値の範囲を表します。**Buckets** セクション内では、次の範囲のバケットが定義されています：

* Bucket 1: (Min - 0.7629)
* Bucket 2: (0.7629 - 0.7824)
* Bucket 3: (0.7824 - 0.8019)
* Bucket 4: (0.8019 - 0.8214)
* Bucket 5: (0.8214 - 0.8409)
* Bucket 6: (0.8409 - Max)

下の折れ線プロットでは、最も高い accuracy (0.8232) の run は濃い紫色 (Bucket 5) で表示され、最も低い accuracy (0.7684) の run は薄いオレンジ色 (Bucket 2) で表示されています。他の run も accuracy の値に基づいて色分けされ、色のグラデーションによって相対的な性能が示されます。
