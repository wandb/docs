description: ある実験の特定のポイントから、元の run に影響を与えずに、異なるパラメーターやモデルを探索します。
title: run をフォークする
---

<Warning>
  run をフォークする機能は現在活発に開発中で、プライベートプレビュー段階です。この機能へのアクセスをリクエストするには、support@wandb.com まで W&amp;B Support にお問い合わせください。
</Warning>

実験の特定のポイントから、元の run に影響を与えずに、異なるハイパーパラメーターやモデルを探索できます。そのためには、既存の W&amp;B run からフォークします。

run からフォークすると、W&amp;B はフォーク元の run の [一意の ID](/ja/models/runs/run-identifiers#unique-run-identifiers) と指定した step を使って新しい run を作成します。フォーク元 run のサマリー メトリクスはフォークされた run にコピーされます。フォークされた run は、指定した step までのフォーク元 run のすべての履歴とファイルを共有します。

フォークした step 以降は、元の run とは独立してフォークされた run に新しいデータをログできます。

<Note>
  * run のフォークには [`wandb`](https://pypi.org/project/wandb/) SDK バージョン &gt;= 0.16.5 が必要です
  * run のフォークには単調増加する step が必要です。[`define_metric()`](/ja/models/ref/python/experiments/run#define_metric) で定義された非単調な step を使う run からはフォークできません。非単調な step は run history とシステム メトリクスの時間順序を崩します。
</Note>

フォーク元の run の一意の `run ID` と、フォークした run を開始したい `step` を、[`wandb.init()`](/ja/models/ref/python/functions/init) の `fork_from` に引数として指定します。

<div id="fork-from-a-previously-logged-run">
  ## 以前にログした run からフォークする
</div>

次のコードスニペットは、以前に W&amp;B にログした run からフォークする方法を示します。

まず、フォーク元にしたい run の run ID を取得します。次に、[`wandb.init()`](/ja/models/ref/python/functions/init) の `fork_from` に、run ID とフォーク元にしたいステップを引数として指定します。

次のコードを Python スクリプトまたはノートブックのセルにコピー＆ペーストします。`<source-run-id>`、`<project>`、`<entity>` をご自身の値に置き換えてください。

```python
import wandb

# フォーク元のソース run の一意の ID
source_run_id = "<source-run-id>"

# フォーク元のステップを指定する
fork_step = 200

# run をフォークする
with wandb.init(
    project="<project>",
    entity="<entity>",
    fork_from=f"{source_run_id}?_step={fork_step}",
) as forked_run:
    pass
```

<div id="fork-from-a-run-in-the-same-script">
  ## 同じスクリプト内の run からフォークする
</div>

次のコードスニペットは、同じスクリプト内で run を作成し、その run からフォークする方法を示しています。これは、W&amp;B App で run ID を調べることなく、作成したばかりの run からフォークしたい場合に該当します。

まず、run を初期化してデータをいくつかログに記録します。次に、元の run オブジェクトの `id` プロパティを使用して、その run の ID を取得します。最後に、新しい run を初期化し、元の run の ID とフォーク元にしたいステップを [`wandb.init()`](/ja/models/ref/python/functions/init) の `fork_from` に引数として渡します。

```python
import wandb

# Initialize a run
with wandb.init(
    project="<project>",
    entity="<entity>"
) as original_run:
    # ...training logic goes here ...
    pass

# Specify the step to fork from
fork_step = int("<num>")

# Use the original run's ID and specify the step to fork from
with wandb.init(
    project="<project>",
    entity="<entity>",
    fork_from=f"{original_run.id}?_step={fork_step}",
) as forked_run:
    # ...training logic goes here ...
    pass
```

<Tip>
  元の run の一意の ID を取得するには、`original_run.id` プロパティを使用します。
</Tip>

<div id="example-script">
  ### スクリプト例
</div>

次のコード例は、まず run を fork し、その後、トレーニングステップ 200 から fork した run にメトリクスをログする方法を示します。

次のコードを Python スクリプトまたはノートブック セルにコピーしてペーストし、`<project>` と `<entity>` をご自身の値に置き換えてください。

```python
import wandb
import math

# 最初の run を初期化してメトリクスをログする
with wandb.init(
    project="<project>",
    entity="<entity>"
) as run1:
    for i in range(300):
        run1.log({"metric": i})

# 最初の run の特定のステップから fork し、
# ステップ 200 からメトリクスをログする
with wandb.init(
    project="<project>", 
    entity="<entity>", 
    fork_from=f"{run1.id}?_step=200"
) as run2:
    # 新しい run でログを継続する
    # 最初の数ステップは run1 のメトリクスをそのままログする
    # ステップ 250 以降はスパイクパターンのログを開始する
    for i in range(200, 300):
        if i < 250:
            # スパイクなしで run1 からログを継続する
            metric_value = i
        else:
            # ステップ 250 からスパイク動作を導入する
            metric_value = i + (2 * math.sin(i / 3.0))  # 緩やかなスパイクパターンを適用する

        # 同じステップでログされるよう、両方のメトリクスを
        # 1 回の呼び出しでログする
        run2.log({
            "metric": metric_value,
            "additional_metric": i * 1.1
        })
```

<Note>
  **rewind と forking の互換性**

  Forking は、run の管理や実験をより柔軟に行えるようにすることで、[`rewind`](/ja/models/runs/rewind/) を補完します。

  run から fork すると、W&amp;B は特定の時点でその run から分岐した新しいブランチを作成し、異なるパラメーターや モデル を試せるようにします。

  run を rewind すると、W&amp;B では run の履歴そのものをさかのぼって修正・変更できるようになります。
</Note>
