Runs Table の式エディタを使用すると、名前、状態、[タグ](#filter-runs-with-tags)、またはその他のプロパティに基づいて Runs をフィルタリングできます。

フィルタを追加する際は、まずフィールド（例：tags、timestamp、entity）を選択します。各フィールドには、テキスト、時間、ID などの基礎となるタイプがあります。

表示される演算子のリスト（例：is、in、&ge;、within last）は、このタイプによって異なります。フィールドを選択すると、UI にはそのフィールドのタイプに対して有効な演算子のみが表示されます。

## タイプ別の一般的な演算子

| フィルタタイプ | フィールド例 | 一般的な演算子 | 使用例 |
| ----------- | -------------- | ---------------- | ------------- |
| Tags        | `tags`           | is, is not, in, not in  | `tags is "baseline"` |
| Time        | `created timestamp`      | &le;, &ge;, within last | `created timestamp` &ge; `01/16/2026` |
| String      | `state`         | &equals;, &ne;, IN, NOT IN | `state = "finished"` |

<Note>
上の表は、利用可能なフィールドと演算子の サブセット のみを示しています。式エディタには、利用可能なすべてのフィールドと演算子が表示されます。
</Note>

## フィルタ式の作成

1. プロジェクト のサイドバーから **Runs** タブに移動します。
2. Runs Table の上部にある、漏斗の形をした **Filter** ボタンを選択します。
3. 左から順に、カラム名、論理演算子、フィルタの 値 を選択して、フィルタ式を作成します。
4. 必要に応じて **New Filter** を選択し、追加のフィルタを適用します。

### 例：タグで Runs をフィルタリングする

タグに基づいて Runs をフィルタリングします：

1. プロジェクト のサイドバーから **Runs** タブをクリックします。
2. Runs Table の上部にある、漏斗の形をした **Filter** ボタンを選択します。
3. 一番左のドロップダウンメニューから `"Tags"` を選択します。
4. 二番目のドロップダウンメニューから is、is not、in、または not in を選択します。
5. 三番目のドロップダウンメニューから、フィルタリングしたいタグ名を入力または選択します。

## デフォルトフィルタ

デフォルトで、W&B は以下のフィルタを提供しています：

* **Show only my works**: 現在の ユーザー が作成した Runs のみを表示します。
* **Hide crashed runs**: `crashed` 状態の Runs を非表示にします。

デフォルトフィルタは、フィルタ式エディタの **New filter** ボタンの下にトグルとして表示されます。

## フィルタの削除

Runs Table からフィルタを削除するには：
1. Runs Table の上部にある、漏斗の形をした **Filter** ボタンをクリックします。
2. 削除したいフィルタの隣にある `x` アイコンを選択します。
