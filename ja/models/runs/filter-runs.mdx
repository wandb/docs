---

Runs table の式エディターを使用して、名、状態、[tags](#filter-runs-with-tags)、またはその他のプロパティに基づいて Runs をフィルターします。

フィルターを追加するときは、まずフィールド (たとえば、tags、timestamp、または entity) を選択します。各フィールドには、テキスト、時刻、ID などの基になるタイプがあります。

表示される演算子のリスト (たとえば、is、in、≥、within last) は、このタイプによって決まります。フィールドを選択すると、UI にはそのフィールドのタイプで有効な演算子のみが表示されます。

<div id="common-operators-by-type">
  ## タイプ別の一般的な演算子
</div>

| フィルタータイプ | フィールド例              | 一般的な演算子                | 使用例                                |
| -------- | ------------------- | ---------------------- | ---------------------------------- |
| タグ       | `tags`              | is, is not, in, not in | `tags is "baseline"`               |
| 時間       | `created timestamp` | ≤, ≥, within last      | `created timestamp` ≥ `01/16/2026` |
| 文字列      | `state`             | =, ≠, IN, NOT IN       | `state = "finished"`               |

<Note>
  この表には、使用可能なフィールドと演算子の一部のみを示しています。式エディターには、使用可能なすべてのフィールドと演算子が表示されます。
</Note>

<div id="create-a-filter-expression">
  ## フィルター式を作成する
</div>

1. プロジェクトのサイドバーから **Runs** タブにアクセスします。
2. Runs table の上にある、漏斗の形をした **フィルター** ボタンを選択します。
3. 左から順に列名、論理演算子、フィルター値を選択して、フィルター式を作成します。
4. 必要に応じて **新しいフィルター** を選択し、追加のフィルターを適用します。

<div id="example-filter-runs-with-tags">
  ### 例: タグで Runs をフィルターする
</div>

タグに基づいて Runs をフィルターします。

1. プロジェクトのサイドバーで **Runs** タブをクリックします。
2. Runs table の上にある、漏斗の形をした **フィルター** ボタンを選択します。
3. 左から順に、ドロップダウンメニューから `"Tags"` を選択し、論理演算子を選択します。
4. 2 番目のドロップダウンメニューから、is、is not、in、または not in を選択します。
5. 3 番目のドロップダウンメニューに、フィルターするタグ名を入力します。

<div id="default-filters">
  ## デフォルトのフィルター
</div>

デフォルトでは、W&amp;B には次のフィルターがあります。

* **自分の Runs のみを表示**: 現在のユーザーが作成した Runs のみを表示します。
* **クラッシュした Runs を非表示**: `crashed` 状態の Runs を非表示にします。

デフォルトのフィルターは、フィルター式エディターの **新しいフィルター** ボタンの下にトグルとして表示されます。

<div id="remove-a-filter">
  ## フィルターを削除
</div>

Runs table からフィルターを削除するには:

1. Runs table の上にある、じょうごの形をした**フィルター**ボタンをクリックします。
2. 削除したいフィルターの横にある `x` アイコンを選択します。
