---
title: コンソールログ
description: "W&B の experiment runs からの情報、警告、エラーを含むコンソールのログメッセージを表示してデバッグします。"
---

experiment を実行すると、コンソールにさまざまなメッセージが出力されることがあります。W&amp;B はコンソールログを取得し、W&amp;B App に表示します。これらのメッセージを使用して、experiment の動作をデバッグおよび監視します。

<div id="view-console-logs">
  ## コンソールログ を表示
</div>

W&amp;B App で run の コンソールログ にアクセスします:

1. W&amp;B App のプロジェクトにアクセスします。
2. **Runs** 表で run を選択します。
3. プロジェクトのサイドバーで **Logs** タブをクリックします。

<Note>
  W&amp;B では、1 つの run について最大 100,000 行のログを保存します。W&amp;B App では、一度に表示されるログは最大 10,000 行です。保存されているすべてのログ行を表示するには、ログをスクロールして古い行を表示してください。
</Note>

{/* ## コンソールログのフィールド

  W&B App では、コンソールログの表に表示するフィールドを変更できます。 */}


<div id="types-of-console-logs">
  ## コンソールログ の種類
</div>

W&amp;B は、コンソールログ の重大度を示す接頭辞とともに、情報メッセージ、警告、エラーなど、複数の種類の コンソールログ を取得します。

<div id="informational-messages">
  ### 情報メッセージ
</div>

情報メッセージは、run の進行状況やステータスに関する更新を知らせます。通常は `wandb:` で始まります。

```text
wandb: Starting Run: abc123
wandb: Run data is saved locally in ./wandb/run-20240125_120000-abc123
```


<div id="warning-messages">
  ### 警告メッセージ
</div>

実行は停止しないものの、潜在的な問題に関する警告の先頭には `WARNING:` が付きます

```text
WARNING Found .wandb file, not streaming tensorboard metrics.
WARNING These runs were logged with a previous version of wandb.
```


<div id="error-messages">
  ### エラーメッセージ
</div>

重大な問題に対するエラーメッセージには、接頭辞として `ERROR:` が付きます。これは、run が正常に完了できなくなる可能性がある問題を示します。

```text
ERROR Unable to save notebook session history.
ERROR Failed to save notebook.
```


<div id="console-log-settings">
  ## コンソールログの設定
</div>

コード内で、`wandb.Settings` オブジェクトを `wandb.init()` に渡して、W&amp;B がコンソールログをどのように処理するかを設定します。`wandb.Settings` では、次のパラメーターを設定してコンソールログの動作を制御できます。

* `show_errors`: `True` に設定すると、エラーメッセージが W&amp;B App に表示されます。`False` に設定すると、エラーメッセージは表示されません。
* `silent`: `True` に設定すると、すべての W&amp;B のコンソール出力が抑制されます。これは、コンソールのノイズを最小限に抑えたい本番環境で便利です。
* `show_warnings`: `True` に設定すると、警告メッセージが W&amp;B App に表示されます。`False` に設定すると、警告メッセージは表示されません。
* `show_info`: `True` に設定すると、情報メッセージが W&amp;B App に表示されます。`False` に設定すると、情報メッセージは表示されません。

次の例は、これらの設定を構成する方法を示しています。

```python
import wandb

settings = wandb.Settings(
    show_errors=True,  # W&B App にエラーメッセージを表示する
    silent=False,      # すべての W&B コンソール出力を無効にする
    show_warnings=True # W&B App に警告メッセージを表示する
)

with wandb.init(settings=settings) as run:
    # トレーニングコードをここに記述する
    run.log({"accuracy": 0.95})
```


<div id="custom-logging">
  ## カスタム logging
</div>

W&amp;B はアプリケーションからコンソールログを取得しますが、お使いの独自の logging 設定には干渉しません。Python に組み込まれている `print()` 関数または `logging` モジュールを使用して、メッセージをログできます。

```python
import wandb

with wandb.init(project="my-project") as run:
    for i in range(100, 1000, 100):
        # W&B にログし、コンソールに出力します
        run.log({"epoch": i, "loss": 0.1 * i})
        print(f"epoch: {i} loss: {0.1 * i}")
```

コンソールログは次のようになります。

```text
1 epoch:  100 loss: 1.3191105127334595
2 epoch:  200 loss: 0.8664389848709106
3 epoch:  300 loss: 0.6157898902893066
4 epoch:  400 loss: 0.4961796700954437
5 epoch:  500 loss: 0.42592573165893555
6 epoch:  600 loss: 0.3771176040172577
7 epoch:  700 loss: 0.3393910825252533
8 epoch:  800 loss: 0.3082585036754608
9 epoch:  900 loss: 0.28154927492141724
```


<div id="time-stamps">
  ## タイムスタンプ
</div>

タイムスタンプは、各 コンソールログ のエントリに自動的に追加されます。これにより、各ログメッセージがいつ生成されたかを確認できます。

コンソールログ のタイムスタンプは、オンまたはオフに切り替えられます。コンソールページの左上にある **Timestamp visible** ドロップダウンを選択します。タイムスタンプを表示または非表示にできます。

<div id="search-console-logs">
  ## コンソールログを検索
</div>

コンソールログページの上部にある検索バーを使用して、キーワードでログをフィルタリングします。特定の用語、ラベル、またはエラーメッセージを検索できます。

<div id="filter-with-custom-labels">
  ## カスタムラベルでフィルターする
</div>

<Warning>
  先頭に `x_` が付いたパラメーター (`x_label` など) はパブリックプレビュー中です。フィードバックを送るには、[W&amp;B repository の GitHub issue](https://github.com/wandb/wandb) を作成してください。
</Warning>

コンソールログページ上部の検索バーで、`wandb.Settings` の `x_label` 引数に渡したラベルに基づいてコンソールログをフィルターできます。

```python
import wandb

# プライマリノードでrunを初期化する
with wandb.init(
    entity="entity",
    project="project",
    settings=wandb.Settings(
        x_label="custom_label"  # (オプション) ログのフィルタリング用カスタムラベル
    )
) as run:
    # ここにコードを記述する
```


<div id="download-console-logs">
  ## コンソールログをダウンロードする
</div>

W&amp;B App で run のコンソールログをダウンロードします。

1. W&amp;B App でご利用のプロジェクトにアクセスします。
2. **Runs** 表内の run を選択します。
3. プロジェクトのサイドバーで **Logs** タブをクリックします。
4. コンソールログページの画面右上にあるダウンロードボタンをクリックします。

<div id="copy-console-logs">
  ## コンソールログをコピー
</div>

W&amp;B App で run のコンソールログをコピーするには、次の手順に従います。

1. W&amp;B App でプロジェクトにアクセスします。
2. **Runs** 表で run を選択します。
3. プロジェクトのサイドバーで **Logs** タブをクリックします。
4. コンソールログページの画面右上にあるコピーボタンをクリックします。
