カスタムチャートを使用すると、パネルに読み込むデータとその可視化を制御できます。

<div id="1-log-data-to-wb">
  ## 1. データを W&amp;B にログする
</div>

まず、スクリプトでデータをログします。ハイパーパラメーターのように、トレーニングの開始時に設定する単一の値には [wandb.Run.config](/ja/models/track/config/) を使用します。時間の経過に伴う複数の値には [wandb.Run.log()](/ja/models/track/log/) を使用し、カスタムの 2D 配列は `wandb.Table()` でログします。ログしたキーごとに、データポイントは最大 10,000 個までにすることをお勧めします。

```python
with wandb.init() as run: 

  # カスタムデータの表をログする
  my_custom_data = [[x1, y1, z1], [x2, y2, z2]]
  run.log(
    {"custom_data_table": wandb.Table(data=my_custom_data, columns=["x", "y", "z"])}
  )
```

[簡単な例示ノートブックを試してみてください](https://bit.ly/custom-charts-colab)。データ表をログし、次のstepではカスタムチャートを設定します。結果のチャートがどのように見えるかは、[ライブレポート](https://app.wandb.ai/demo-team/custom-charts/reports/Custom-Charts--VmlldzoyMTk5MDc)で確認してください。

<div id="2-create-a-query">
  ## 2. クエリを作成する
</div>

可視化するデータをログしたら、プロジェクトページに移動し、新しいパネルを追加するために **`+`** ボタンをクリックしてから、**Custom Chart** を選択します。[custom charts demo workspace](https://app.wandb.ai/demo-team/custom-charts) を見ながら進めることもできます。

<Frame>
  <img src="/images/app_ui/create_a_query.png" alt="空のカスタムチャート" />
</Frame>

<div id="add-a-query">
  ### クエリを追加する
</div>

1. `summary` をクリックし、`historyTable` を選択して、run の履歴からデータを取得する新しいクエリを設定します。
2. `wandb.Table()` をログしたキーを入力します。上記の 코드 스니펫 では、これは `my_custom_table` でした。[例示ノートブック](https://bit.ly/custom-charts-colab) では、キーは `pr_curve` と `roc_curve` です。

<div id="set-vega-fields">
  ### Vega フィールドを設定する
</div>

クエリがこれらの列に読み込まれると、Vega フィールドのドロップダウン メニューで選択できるオプションとして表示されます。

<Frame>
  <img src="/images/app_ui/set_vega_fields.png" alt="Vega フィールドを設定するためにクエリ結果から列を取り込む" />
</Frame>

* **x-axis:** runSets&#95;historyTable&#95;r (recall)
* **y-axis:** runSets&#95;historyTable&#95;p (precision)
* **色:** runSets&#95;historyTable&#95;c (class label)

<div id="3-customize-the-chart">
  ## 3. チャートをカスタマイズする
</div>

これでかなり見栄えがよくなりましたが、散布図から折れ線グラフに切り替えたいと思います。この組み込みチャートの Vega spec を変更するには、**Edit** をクリックします。[custom charts demo workspace](https://app.wandb.ai/demo-team/custom-charts) を見ながら進めてください。

<Frame>
  <img src="/images/general/custom-charts-1.png" alt="カスタム チャートの選択" />
</Frame>

可視化をカスタマイズするために、Vega spec を更新しました。

* プロット、凡例、x 軸、Y 軸のタイトルを追加する (各フィールドに “title” を設定します)
* “mark” の値を “point” から “line” に変更する
* 使用していない “size” フィールドを削除する

  <img src="/images/app_ui/customize_vega_spec_for_pr_curve.png" alt="PR 曲線の Vega 仕様" />

<div id="bonus-composite-histograms">
  ## おまけ: 複合ヒストグラム
</div>

ヒストグラムは数値分布を可視化し、より大きなデータセットを理解するのに役立ちます。複合ヒストグラムは同じビンにまたがる複数の分布を表示し、異なるモデル間、またはモデル内の異なるクラス間で2つ以上のメトリクスを比較できます。運転シーン内の物体を検出するセマンティックセグメンテーションモデルでは、accuracy と intersection over union (IOU) のどちらを最適化する方が効果的かを比較したり、車 (データ内で大きく一般的な領域) と交通標識 (はるかに小さく、あまり一般的ではない領域) を異なるモデルがどの程度うまく検出できるかを知りたい場合があります。[デモ Colab](https://bit.ly/custom-charts-colab)では、生物の10クラスのうち2クラスについて confidence score を比較できます。

<Frame>
  <img src="/images/app_ui/composite_histograms.png" alt="複合ヒストグラム" />
</Frame>

独自のカスタム複合ヒストグラムパネルを作成するには、次の手順に従います。

1. Workspace またはレポートで新しい Custom Chart パネルを作成します (“Custom Chart” 可視化を追加します) 。右上の “Edit” ボタンを押すと、任意の組み込みパネルタイプを起点に Vega spec を変更できます。
2. その組み込み Vega spec を、私の [Vega による複合ヒストグラム用 MVP コード](https://gist.github.com/staceysv/9bed36a2c0c2a427365991403611ce21)に置き換えます。この Vega spec では、メインタイトル、軸タイトル、入力ドメイン、そのほかの詳細を [Vega 構文を使って](https://vega.github.io/) 直接変更できます (色を変えたり、3つ目のヒストグラムを追加したりすることもできます :)) 。
3. 右側のクエリを変更して、wandb ログから正しいデータを読み込みます。フィールド `summaryTable` を追加し、対応する `tableKey` を `class_scores` に設定して、run によってログされた `wandb.Table` を取得します。これにより、`class_scores` としてログされた `wandb.Table` の列を使って、ドロップダウンメニュー経由で2つのヒストグラムのビンセット (`red_bins` と `blue_bins`) を設定できるようになります。私の例では、赤いビンには `animal` クラスの prediction score を、青いビンには `plant` を選びました。
4. プレビュー表示で見えるプロットに満足するまで、Vega spec とクエリを引き続き変更できます。完了したら、上部の **Save as** をクリックし、再利用できるようにカスタムプロットに名前を付けます。次に **Apply from panel library** をクリックして、プロットを完成させます。

ごく短時間の experiment で得られた結果は次のようになりました。1,000 件の例だけで 1 エポック学習すると、ほとんどの画像は植物ではないと非常に強く確信しつつ、どの画像が動物かについては非常に不確かなモデルになります。
