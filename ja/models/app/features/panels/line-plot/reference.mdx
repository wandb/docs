このページでは、折れ線グラフの設定について包括的な詳細を説明します。折れ線グラフの操作の詳細については、[Line plots overview](/ja/models/app/features/panels/line-plot) を参照してください。

<div id="data-settings">
  ## データ設定
</div>

<div id="x-axis">
  ### X 軸
</div>

<Frame>
  <img src="/images/app_ui/reference_x_axis.png" alt="X 軸 を選択" />
</Frame>

`wandb.Run.log()` でログした任意の整数値または浮動小数点値に X 軸 の範囲を設定できます。

時間ベースの X 軸 オプション:

* **Step**: `wandb.Run.log()` が呼び出されるたびに増加します。モデルからログされたトレーニング step 数を反映します。 (デフォルト)
* **Relative Time (Wall)**: プロセスの開始からの時計時間です。run を開始し、1 日停止してから Resume してログすると、その点は 24 時間の位置に表示されます。
* **Relative Time (Process)**: 実行中のプロセス内の時間です。run を開始して 10 秒間実行し、1 日停止してから Resume すると、その点は 10 秒の位置に表示されます。
* **Wall Time**: グラフ上の最初の run の開始からの経過時間 (分) です。
* **X range**: デフォルトでは X 軸 の最小値から最大値までです。最小値と最大値はカスタマイズできます。

<div id="y-axis">
  ### Y 軸
</div>

Y 軸の変数には、`wandb.Run.log()`でログした任意の整数値または浮動小数点値を設定できます。単一の値、値の配列、または値のヒストグラムを指定します。ある変数について 1500 点を超えてログした場合、W&amp;B は 1500 点にサンプリングダウンします。

<Note>
  Runs table で run の色を変更すると、Y 軸の線の色をカスタマイズできます。
</Note>

使用可能な Y 軸オプション:

* **Y range**: デフォルトでは、メトリクスの最小の正の値 (0 を含む) からメトリクスの最大値までの範囲です。最小値と最大値はカスタマイズできます。

<div id="point-aggregation-method">
  ### 点の集約method
</div>

データポイントの表示に使用するサンプリングモードを選択します：

* **Random sampling** (デフォルト): [Random sampling](/ja/models/app/features/panels/line-plot/sampling/#random-sampling) を参照してください。
* **フルフィデリティ**: [フルフィデリティ](/ja/models/app/features/panels/line-plot/sampling/#full-fidelity) を参照してください。

<div id="smoothing">
  ### スムージング
</div>

[smoothing coefficient](/ja/models/support/formula_smoothing_algorithm/) を 0 から 1 の間で設定します。0 はスムージングなし、1 は最大のスムージングです。

使用可能なスムージング method:

* **Time weighted EMA** (デフォルト) : 以前の点の重みを指数関数的に減衰させることで、時系列データをスムージングする手法です。
* **Running average**: 指定した x 値の前後のウィンドウ内にある点の平均値で、その点を置き換えます。
* **Gaussian**: 点の加重平均を計算します。重みはガウス分布に対応し、標準偏差はスムージング パラメーターとして指定します。
* **No smoothing**

詳細については、[折れ線グラフのスムージング](/ja/models/app/features/panels/line-plot/smoothing) を参照してください。

<div id="ignore-outliers">
  ### 外れ値を無視
</div>

デフォルトのプロットの最小値および最大値のスケールから外れ値を除外するようにプロットを再スケーリングします。この設定の影響は、プロットのサンプリングモードによって異なります。

* **Random サンプリングモード**: 外れ値を無視すると、プロットから 5% 未満および 95% を超える点が除外されます。
* **フルフィデリティモード**: 外れ値を無視すると、すべての点が表示され、各버킷の最後の値に集約されます。また、5% 未満および 95% を超える領域が陰影表示されます。

<div id="max-runs-or-groups">
  ### run またはグループ数の上限
</div>

デフォルトでは、プロットには run リストまたは run セット内の先頭 10 件の run または run のグループのみが含まれます。どの run やグループを表示するかは、ソート順を変更して制御できます。

<Note>
  Workspace では、設定に関係なく、表示できる run の最大数は 1000 件に制限されています。
</Note>

<div id="grouping-settings">
  ## グループ化の設定
</div>

グループ化をオンにしてすべての run を集約することも、個別の変数でグループ化することもできます。Runs table でグループ化をオンにすると、グループが自動的にグラフに反映されます。

* **Group runs**: プロットで run のグループ化をオンにします。以下のプロット内の網掛け範囲を設定するには必須です。
* **Group by**: 必要に応じて列を選択します。その列で同じ値を持つすべての run が同じグループにまとめられます。
* **集約**: グラフ上の線の値です。オプションは、グループの mean、median、min、max です。
* **Range**: full-fidelity の折れ線グラフの網掛け領域を設定します。オプションは Min/Max、Std Dev、Std Err、None です。

<div id="chart-settings">
  ## チャート設定
</div>

タイトルと凡例の表示を設定します。

* **パネルタイトル**: パネル上部に表示されるタイトル。
* **X 軸タイトル**: X 軸のラベル。
* **Y 軸タイトル**: Y 軸のラベル。
* **凡例**: 凡例の表示 / 非表示や位置を設定します。

<div id="legend-settings">
  ## 凡例設定
</div>

凡例をカスタマイズして、ログされた任意の設定値や、作成時刻、run を作成したユーザーなどの run のメタデータを表示できます。

<div id="legend-template">
  ### 凡例テンプレート
</div>

凡例名のテンプレートを定義します。

1. 歯車アイコンをクリックしてプロット設定を開きます。
2. **Display preferences** タブを開きます。
3. **Advanced legend** を展開し、凡例テンプレートを指定します。
4. **Apply** をクリックします。

例:

`${run:displayName} - ${config:dropout}` と指定すると、`royal-sweep - 0.5` のような凡例名になります。このとき `royal-sweep` は run 名で、`0.5` は `dropout` という名の設定パラメーターです。

<div id="point-specific-values">
  ### 点ごとの値
</div>

`[[ ]]` 内に値を設定すると、チャートにカーソルを合わせたときに、クロスヘアに点ごとの値を表示できます。

1. 歯車アイコンをクリックして、プロット設定を開きます。
2. **Display preferences** タブに移動します。
3. タブの下部で、プロットの1つ以上のメトリクスに対して点ごとの値を設定します。
4. **Apply** をクリックします。

例: `[[ $x: $y ($original) ]]` は、&quot;2: 3 (2.9)&quot; のように表示します

`[[ ]]` 内でサポートされる値:

| Value         | Meaning                |
| ------------- | ---------------------- |
| `${x}`        | X 値                    |
| `${y}`        | Y 値 (スムージング調整を含む)      |
| `${original}` | スムージング調整を含まない Y 値      |
| `${mean}`     | グループ化された Runs の平均      |
| `${stddev}`   | グループ化された Runs の標準偏差    |
| `${min}`      | グループ化された Runs の最小値     |
| `${max}`      | グループ化された Runs の最大値     |
| `${percent}`  | 合計に対する割合 (積み上げ面チャート用)  |

<div id="expressions">
  ## 式
</div>

カスタムの計算式を追加して、派生メトリクスを作成します。

<div id="y-axis-expressions">
  ### Y 軸の式
</div>

メトリクスから導出した値をプロットします。たとえば、`1-accuracy` やその他の算術式を計算できます。現在は、単一のメトリクスをプロットする場合にのみ機能します。

サポートされる演算子: `+`, `-`, `*`, `/`, `%`, `**` (べき乗) 。

ログされたメトリクスと、ハイパーパラメーターなどの設定値の両方を使用して、カスタムの線を計算できます。

<div id="x-axis-expressions">
  ### X 軸 の式
</div>

カスタム式を使用して、計算された値に基づいて X 軸 を再スケールします。

使用可能な変数:

* `_step`: デフォルトの X 軸 の値。
* `${summary:value}`: summary の値を参照します。

<div id="multi-metric-panel-expressions">
  ### 複数メトリクス パネル式
</div>

正規表現を使用すると、複数のメトリクスをまとめて表示する 1 つの折れ線グラフを作成できます (今後ログされる一致メトリクスも含みます) 。詳しい手順は、[折れ線グラフを追加する](/ja/models/app/features/panels/line-plot#multi-metric-line-plot) を参照してください。

例えば、次のように使用できます。

* 各レイヤーのメトリクスごとに個別のパネルを作成する代わりに、1 つのパネルにまとめて表示できます。例えば、`layer_0_loss`、`layer_1_loss`、`layer_2_loss` のように一貫した命名でメトリクスをログしている場合は、`layer_\d+_loss` のような正規表現を使用して、すべてのレイヤーの loss を 1 つのプロットに表示できます。
* 共通の命名パターンを持つすべてのメトリクスに一致させます。例えば:
  * `train_.*` は、`train_loss`、`train_accuracy`、`train_f1_score` などのすべてのトレーニング メトリクスに一致します
  * `.*_accuracy` は、`train_accuracy`、`val_accuracy`、`test_accuracy` など、異なるデータセットにまたがる accuracy メトリクスに一致します
* オルタネーションを使用して、必要なメトリクスだけに一致させます。例えば、非キャプチャグループ `(?:layer_0|layer_10)_loss` は、中間のレイヤーを除外し、最初と 10 番目のレイヤーの loss のみに一致します。
