<div id="create-a-query-panel">
  ## クエリパネルを作成する
</div>

クエリを Workspace または レポート 内に追加します。

<Tabs>
  <Tab title="ワークスペース">
    1. プロジェクトの Workspace にアクセスします。
    2. 右上隅の `Add panel` をクリックします。
    3. ドロップダウンから `Query panel` を選択します。
  </Tab>

  <Tab title="W&B Report">
    `/Query panel` と入力して選択します。

    <Frame>
      <img src="/images/weave/add_weave_panel_report_1.png" alt="クエリパネルのオプション" />
    </Frame>

    または、クエリを一連の run に関連付けることもできます。

    1. レポート内で `/Panel grid` と入力して選択します。
    2. `Add panel` ボタンをクリックします。
    3. ドロップダウンから `Query panel` を選択します。
  </Tab>
</Tabs>

  ## クエリする컴포넌트

<div id="expressions">
  ### Expressions
</div>

W&amp;B に保存されている runs、アーティファクト、モデル、表などのデータをクエリするには、クエリ式を使用します。

<div id="example-query-a-table">
  #### 例: 表をクエリする
</div>

W&amp;B の表をクエリする場合、トレーニングコードで `"cifar10_sample_table"` という名前の表をログします:

```python
import wandb
with wandb.init() as run:
  run.log({"cifar10_sample_table":<MY_TABLE>})
```

クエリパネル では、次を使用して表をクエリできます:

```python
runs.summary["cifar10_sample_table"]
```

<Frame>
  <img src="/images/weave/basic_weave_expression.png" alt="表のクエリ式" />
</Frame>

これを分解すると、次のようになります。

* `runs` は、クエリパネル が Workspace 内にある場合に Query Panel Expressions に自動的に挿入される変数です。その「値」は、その特定の Workspace で表示される Runs のリストです。[run 内で使用できるさまざまな属性についてはこちらを参照してください](/ja/models/track/public-api-guide/#understanding-the-different-attributes)。
* `summary` は、Run の Summary object を返す op です。Ops は *mapped* されます。つまり、この op はリスト内の各 Run に適用され、その結果、Summary objects のリストが返されます。
* `["cifar10_sample_table"]` は Pick op (角かっこで示されます) で、`predictions` をパラメーターとして取ります。Summary objects は辞書やマップのように動作するため、この操作では各 Summary object から `predictions` フィールドを取り出します。

独自のクエリをインタラクティブに記述する方法については、[クエリパネル demo](https://wandb.ai/luis_team_test/weave_example_queries/reports/Weave-queries---Vmlldzo1NzIxOTY2?accessToken=bvzq5hwooare9zy790yfl3oitutbvno2i6c2s81gk91750m53m2hdclj0jvryhcr) を参照してください.

<div id="concat">
  ### Concat
</div>

concat 操作を使用すると、2 つの表を連結でき、Panel settings から連結や結合を行うこともできます。

<div id="join">
  ### Join
</div>

クエリ内で表を直接結合することもできます。次のクエリ式を考えてみましょう。

```python
project("luis_team_test", "weave_example_queries").runs.summary["short_table_0"].table.rows.concat.join(\
project("luis_team_test", "weave_example_queries").runs.summary["short_table_1"].table.rows.concat,\
(row) => row["Label"],(row) => row["Label"], "Table1", "Table2",\
"false", "false")
```

<Frame>
  <img src="/images/weave/weave_join.png" alt="表の結合操作" />
</Frame>

左側の表は次のように生成されます。

```python
project("luis_team_test", "weave_example_queries").\
runs.summary["short_table_0"].table.rows.concat.join
```

右側の表は、次のコードから生成されています：

```python
project("luis_team_test", "weave_example_queries").\
runs.summary["short_table_1"].table.rows.concat
```

ここでは、次の意味です。

* `(row) => row["Label"]` は各表のセレクターで、どの列をキーとして結合するかを決定します
* `"Table1"` と `"Table2"` は、結合時の各表の名です
* `true` と `false` は、左側と右側の内部結合 / 外部結合の設定です

<div id="runs-object">
  ## Runs オブジェクト
</div>

クエリパネルを使用して `runs` オブジェクトにアクセスします。`run` オブジェクトは実験の記録を保存します。詳細は [Accessing runs object](https://wandb.ai/luis_team_test/weave_example_queries/reports/Weave-queries---Vmlldzo1NzIxOTY2?accessToken=bvzq5hwooare9zy790yfl3oitutbvno2i6c2s81gk91750m53m2hdclj0jvryhcr#3.-accessing-runs-object) を参照してください。ここでは概要として、`runs` オブジェクトでは次のものを利用できます。

* `summary`: run の結果を要約した情報の辞書です。精度や損失のようなスカラー値や、大きなファイルを含めることができます。デフォルトでは、`wandb.Run.log()` はログされた時系列の最終値を summary に設定します。summary の内容を直接設定することもできます。summary は run の出力だと考えてください。
* `history`: 損失のように、モデルのトレーニング中に変化する値を保存することを目的とした辞書のリストです。`wandb.Run.log()` コマンドはこのオブジェクトに追記します。
* `config`: トレーニング run のハイパーパラメーターや、データセット アーティファクトを作成する run における前処理手法など、run の設定情報の辞書です。これらは run の「入力」だと考えてください。

<div id="access-artifacts">
  ## Artifacts への액세스
</div>

Artifacts は W&amp;B の中核となる概念です。Artifacts は、バージョン管理された名前付きのファイルやディレクトリのコレクションです。Artifacts を使用して、モデルの重み、データセット、そのほかのあらゆるファイルやディレクトリをトラッキングできます。Artifacts は W&amp;B に保存され、ダウンロードしたり、ほかの run で使用したりできます。詳細と例については、[Artifacts への액세ス](https://wandb.ai/luis_team_test/weave_example_queries/reports/Weave-queries---Vmlldzo1NzIxOTY2?accessToken=bvzq5hwooare9zy790yfl3oitutbvno2i6c2s81gk91750m53m2hdclj0jvryhcr#4.-accessing-artifacts) を参照してください。Artifacts には通常、`project` オブジェクトから액세스します。

* `project.artifactVersion()`: プロジェクト内で、指定した名とバージョンに対応する特定のアーティファクト バージョンを返します
* `project.artifact("")`: プロジェクト内で、指定した名に対応するアーティファクトを返します。その後 `.versions` を使用して、このアーティファクトのすべてのバージョンのリストを取得できます
* `project.artifactType()`: プロジェクト内で、指定した名に対応する `artifactType` を返します。その後 `.artifacts` を使用して、このタイプのすべてのアーティファクトのリストを取得できます
* `project.artifactTypes`: プロジェクト配下のすべての 유형 のリストを返します
