---
title: パネル
sidebartitle: 概要
description: ログされたデータを可視化するために、Workspace パネルを使用およびカスタマイズします
---

Workspace のパネル可視化を使用すると、キーごとに[ログされたデータ](/ja/models/ref/python/experiments/run.md/#method-runlog)を探索し、ハイパーパラメーターと出力メトリクスの関係を可視化するなどの操作ができます。

<div id="workspace-modes">
  ## Workspace モード
</div>

W&amp;B のプロジェクトは、2 つの異なる Workspace モードをサポートしています。Workspace 名の横にあるアイコンは、そのモードを示します。

| Icon                                                                                      | Workspace mode                                                                                                                                                                                                                                                                                        |
| ----------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| <img src="/images/app_ui/automated_workspace.svg" alt="自動 Workspace アイコン" width="32px" /> | **自動 Workspace** では、プロジェクトにログされたすべてのキーに対してパネルが自動的に生成されます。自動 Workspace を選択するのは、次のような場合です。<ul><li>プロジェクトで使用可能なすべてのデータを可視化して、すばやく始めたい場合。</li><li>ログするキーが少ない小規模なプロジェクトの場合。</li><li>より広範な分析を行う場合。</li></ul>自動 Workspace からパネルを削除した場合は、[Quick add](#quick-add) を使用して再作成できます。                                |
| <img src="/images/app_ui/manual_workspace.svg" alt="手動 Workspace アイコン" width="32px" />    | **手動 Workspace** は空の状態から始まり、ユーザーが意図的に追加したパネルのみを表示します。手動 Workspace を選択するのは、次のような場合です。<ul><li>主に、プロジェクトにログされたキーの一部だけが重要な場合。</li><li>より焦点を絞った分析を行う場合。</li><li>自分にとってあまり有用でないパネルの読み込みを避けて、Workspace のパフォーマンスを向上させる場合。</li></ul>[Quick add](#quick-add) を使用すると、手動 Workspace とそのセクションに有用な可視化をすばやく簡単に追加できます。 |

Workspace がパネルを生成する方法を変更するには、[Workspace をリセット](#reset-a-workspace)します。

<Note>
  **Workspace への変更を元に戻す**

  Workspace への変更を元に戻すには、Undo ボタン (左向きの矢印) をクリックするか、**CMD + Z** (macOS) または **CTRL + Z** (Windows / Linux) を入力します。
</Note>

<div id="reset-a-workspace">
  ## Workspaceをリセットする
</div>

Workspaceをリセットするには:

1. Workspace の上部で、작업 메뉴 `...` をクリックします。
2. **Reset workspace** をクリックします。

<div id="configure-the-workspace-layout">
  ## Workspace layout を設定する
</div>

Workspace layout を設定するには、Workspace の上部付近にある **Settings** をクリックし、次に **Workspace layout** をクリックします。

* **検索時に空のセクションを非表示にする** (デフォルトでオン)
* **パネルをアルファベット順に並べ替える** (デフォルトでオフ)
* **セクションの組織化** (デフォルトでは最初のプレフィックスでグループ化) 。この設定を変更するには:
  1. 南京錠アイコンをクリックします。
  2. セクション内でパネルをどのようにグループ化するかを選択します。

Workspace の折れ線グラフのデフォルトを設定するには、[Line plots](/ja/models/app/features/panels/line-plot/)を参照してください。

<div id="configure-a-sections-layout">
  ### セクションの Layout を設定する
</div>

セクションの Layout を設定するには、その歯車アイコンをクリックし、**Display preferences** をクリックします。

* **ツールチップで色付きの run 名の表示をオンまたはオフにする** (既定ではオン)
* **コンパニオン チャートのツールチップではハイライトされた run のみを表示する** (既定ではオフ)
* **ツールチップに表示する Runs の数** (single run、all runs、または **Default**)
* **プライマリ チャートのツールチップに完全な run 名を表示する** (既定ではオフ)

<div id="view-a-panel-in-full-screen-mode">
  ## 全画面モードでパネルを表示する
</div>

全画面モードでは、run selector が表示され、パネルでは通常の 1000 버킷 ではなく、10,000 버킷 の full-fidelity サンプリング モードのプロットが使用されます。

パネルを全画面モードで表示するには、次の手順に従います。

1. パネルにカーソルを合わせます。
2. パネルの 작업 메뉴 `...` をクリックし、次に全画面ボタンをクリックします。これは、ファインダーのような形、または正方形の四隅を示すアウトラインのような形をしています。
   <Frame>
     <img src="/images/app_ui/panel_fullscreen.png" alt="全画面パネル" />
   </Frame>
3. 全画面モードで表示中に[パネルを共有](#share-a-panel)すると、生成されるリンクは自動的に全画面モードで開きます。

* 全画面モードからパネルの Workspace に戻るには、ページ上部の左向き矢印をクリックします。
* 全画面モードを終了せずにセクション内のパネル間を移動するには、パネルの下にある **Previous** ボタンと **Next** ボタン、または左右の矢印キーを使用します。
* パネル用のスペースをさらに確保するには、**Cmd+.** (macOS) または **Ctrl+.** (Windows/Linux) で run selector を最小化します。
* [メディアパネル](/ja/models/app/features/panels/media)の image を全画面モードで表示している場合は、キーボード ショートカットを使用してズームイン、ズームアウト、ズームのリセット、または画面に合わせたズームができます。詳細は、[キーボード ショートカット](/ja/models/app/keyboard-shortcuts#media-panels)を参照してください。

その他の全画面表示およびパネルのショートカットについては、[キーボード ショートカット](/ja/models/app/keyboard-shortcuts)を参照してください。

<div id="add-panels">
  ## Add panels
</div>

このセクションでは、Workspace にパネルを追加するさまざまな方法を説明します。

<div id="add-a-panel-manually">
  ### パネルを手動で追加する
</div>

Workspace にパネルを 1 つずつ追加します。Workspace 全体に追加することも、セクション レベルで追加することもできます。

1. パネルを Workspace 全体に追加するには、パネル検索フィールドの近くにあるコントロール バーで **Add panels** をクリックします。
2. 代わりにセクションに直接パネルを追加するには、セクションの작업 메뉴 `...` をクリックし、**+ Add panels** をクリックします。
3. チャートなど、追加するパネルのタイプを選択します。パネルの設定の詳細が表示され、デフォルトが選択されています。
4. 必要に応じて、パネルとその表示設定をカスタマイズします。設定オプションは、選択したパネルのタイプによって異なります。各パネル タイプのオプションの詳細については、以下の該当するセクション ([折れ線グラフ](/ja/models/app/features/panels/line-plot/) や [棒グラフ](/ja/models/app/features/panels/bar-plot/) など) を参照してください。
5. **Apply** をクリックします。

<Frame>
  <img src="/images/app_ui/add_single_panel.gif" alt="パネルを追加するデモ" />
</Frame>

<div id="quick-add-panels">
  ### パネルのクイック追加
</div>

**Quick add** を使用すると、選択したキーごとに、Workspace 全体またはセクションごとにパネルを自動的に追加できます。

<Note>
  パネルが削除されていない自動 Workspace の場合、すべてのログされたキーに対するパネルがすでに含まれているため、**Quick add** オプションは表示されません。削除したパネルを再追加するには、**Quick add** を使用できます。
</Note>

1. **Quick add** を使用してパネルを Workspace 全体に追加するには、パネル検索フィールド付近のコントロールバーで **Add panels** をクリックし、次に **Quick add** をクリックします。
2. **Quick add** を使用してパネルをセクションに直接追加するには、セクションの작업 메뉴 `...` をクリックし、**Add panels** をクリックしてから **Quick add** をクリックします。
3. パネルの一覧が表示されます。チェックマークの付いたパネルは、すでに Workspace に含まれています。
   * すべての追加可能パネルを追加するには、リスト上部の **Add `<N>` panels** ボタンをクリックします。**Quick add** リストが閉じ、新しいパネルが Workspace に表示されます。
   * リストから個別の 1 つを追加するには、パネルの行にカーソルを合わせて **Add** をクリックします。追加したいパネルごとにこの手順を繰り返し、右上の **X** をクリックして **Quick add** リストを閉じます。新しく追加したパネルが Workspace に表示されます。
4. 必要に応じて、パネルの設定をカスタマイズします。

<div id="share-a-panel">
  ## パネルを共有する
</div>

このセクションでは、リンクを使用してパネルを共有する方法を示します。

リンクを使用してパネルを共有するには、次のいずれかの方法を使用できます。

* パネルを全画面モードで表示している間に、ブラウザーから URL をコピーします。
* 작업 메뉴 `...` をクリックし、**パネル URL をコピー** を選択します。

リンクをユーザーまたはチームと共有します。相手がそのリンクに 액세스 すると、パネルは[全画面モード](#view-a-panel-in-full-screen-mode)で開きます。

全画面モードからパネルの Workspace に戻るには、ページ上部の左向き矢印をクリックします。

<div id="compose-a-panels-full-screen-link-programmatically">
  ### パネルの全画面リンクをプログラムで作成する
</div>

[オートメーションを作成する](/ja/models/automations/) などの特定の状況では、パネルの全画面 URL を含めると便利です。このセクションでは、パネルの全画面 URL の形式を示します。次の例では、角括弧内の entity、プロジェクト、パネル、およびセクションの名を置き換えてください。

```text
https://wandb.ai/<ENTITY_NAME>/<PROJECT_NAME>?panelDisplayName=<PANEL_NAME>&panelSectionName=<SECTON_NAME>
```

同じセクション内の複数のパネルが同じ名を持つ場合、このURLではその名を持つ最初のパネルが開きます。

<div id="embed-or-share-a-panel-on-social-media">
  ### パネルをウェブサイトに埋め込む、またはソーシャルメディアで共有する
</div>

パネルをウェブサイトに埋め込むかソーシャルメディアで共有するには、リンクを持っている誰でもパネルを表示できる状態にしておく必要があります。プロジェクトが非公開の場合、パネルを表示できるのはそのプロジェクトのメンバーのみです。プロジェクトが公開の場合、リンクを持っている誰でもパネルを表示できます。

パネルを埋め込むかソーシャルメディアで共有するためのコードを取得するには：

1. Workspace から、パネルにマウスを合わせ、その작업 메뉴 `...` をクリックします。
2. **Share** タブをクリックします。
3. **Only those who are invited have access** を **Anyone with the link can view** に変更します。そうしないと、次の手順の選択肢が利用できません。
4. **Share on Twitter**、**Share on Reddit**、**Share on LinkedIn**、または **Copy embed link** を選択します。

<div id="email-a-panel-report">
  ### パネル レポートをメールで送信する
</div>

単一のパネルを独立したレポートとしてメールで送信するには：

1. パネルにマウスを合わせ、パネルの작업 메뉴 `...` をクリックします。
2. **Share panel in report** をクリックします。
3. **Invite** タブを選択します。
4. メールアドレスまたはユーザー名を入力します。
5. 必要に応じて **can view** を **can edit** に変更します。
6. **Invite** をクリックします。W&amp;B は、共有するパネルだけを含むレポートへのクリック可能なリンクを記載したメールをユーザーに送信します。

[パネルを共有](#share-a-panel) する場合とは異なり、受信者はこのレポートから Workspace にアクセスすることはできません。

<div id="manage-panels">
  ## パネルの管理
</div>

<div id="edit-a-panel">
  ### パネルを編集する
</div>

パネルを編集するには、次の手順に従います。

1. パネルの鉛筆アイコンをクリックします。
2. パネルの設定を変更します。
3. パネルを別のタイプに変更する場合は、タイプを選択してから設定を行います。
4. **Apply** をクリックします。

<div id="move-a-panel">
  ### パネルを移動する
</div>

パネルを別のセクションに移動するには、パネルのドラッグハンドルを使用できます。代わりに、リストから新しいセクションを選択するには、次の手順に従います。

1. 必要に応じて、最後のセクションの後にある **Add section** をクリックして、新しいセクションを作成します。
2. パネルのアクションメニューをクリックします。
3. **Move** をクリックし、新しいセクションを選択します。

ドラッグハンドルを使用して、セクション内のパネルを並べ替えることもできます。

<div id="duplicate-a-panel">
  ### パネルを複製する
</div>

パネルを複製するには、次の手順を実行します。

1. パネルの上部で、アクションメニューをクリックします。
2. **Duplicate** をクリックします。

必要に応じて、複製したパネルを[カスタマイズ](#edit-a-panel)または[移動](#move-a-panel)できます。

<div id="remove-panels">
  ### パネルを削除する
</div>

パネルを削除するには、次の手順を実行します。

1. パネルにマウスポインタを合わせます。
2. アクションメニュー `...` を選択します。
3. **Delete** をクリックします。

手動 Workspace からすべてのパネルを削除するには、そのアクションメニュー `...` をクリックしてから、**Clear all panels** をクリックします。

自動または手動 Workspace からすべてのパネルを削除するには、[Workspace をリセット](#reset-a-workspace)できます。デフォルトのパネルセットから開始するには **Automatic** を選択し、パネルのない空の Workspace から開始するには **Manual** を選択します。

<div id="manage-sections">
  ## セクションを管理
</div>

デフォルトでは、Workspace 内のセクションはキーのログ階層を反映します。ただし、手動 Workspace では、パネルの追加を開始した後にのみセクションが表示されます。

<div id="add-a-section">
  ### セクションを追加する
</div>

セクションを追加するには、最後のセクションの後にある **Add section** をクリックします。

既存のセクションの前後に新しいセクションを追加するには、代わりにそのセクションのアクションメニュー `...` をクリックし、**New section below** または **New section above** をクリックします。

<Warning>
  セクション名を「Section」にしないでください。既知の制限により、セクション名を変更するまで、そのセクション内のパネルは表示されません。
</Warning>

<div id="manage-a-sections-panels">
  ### セクションのパネルを管理する
</div>

多数のパネルを含むセクションは、デフォルトでページ分割されます。1 ページに表示されるパネル数のデフォルト値は、パネルの設定とセクション内のパネルのサイズによって決まります。

1. パネルのサイズを変更するには、パネルにカーソルを合わせ、ドラッグハンドルをクリックしてドラッグし、サイズを調整します。1 つのパネルのサイズを変更すると、そのセクション内のすべてのパネルのサイズも変更されます。
2. セクションがページ分割されている場合は、1 ページに表示するパネル数をカスタマイズできます:
3. セクションの上部にある **1 to `<X>` of `<Y>`** をクリックします。ここで `<X>` は現在表示されているパネル数、`<Y>` はパネルの総数です。
4. 1 ページあたりに表示するパネル数を、最大 100 まで選択します。
5. セクションからパネルを削除するには:
6. パネルにカーソルを合わせて、アクションメニュー `...` をクリックします。
7. **Delete** をクリックします。

Workspace を自動 Workspace にリセットすると、削除したパネルがすべて再度表示されます。

<div id="rename-a-section">
  ### セクション名を変更する
</div>

セクション名を変更するには、その アクションメニューをクリックし、**セクション名を変更** をクリックします。

<Warning>
  セクション名を &quot;Section&quot; にしないでください。既知の制限により、このセクション内のパネルはセクション名を変更するまで表示されません。
</Warning>

<div id="delete-a-section">
  ### セクションを削除
</div>

セクションを削除するには、その `...` メニューをクリックし、**Delete section** をクリックします。これにより、セクションとそのパネルが削除されます。
