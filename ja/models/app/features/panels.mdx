ワークスペースのパネルを使って、キーごとに[ログしたデータ](/ja/models/ref/python/experiments/run.md/#method-runlog)を探索したり、ハイパーパラメータと出力メトリクスの関係を可視化したりできます。

<div id="workspace-modes">
  ## Workspace モード
</div>

W&amp;B のプロジェクトは 2 種類の Workspace モードをサポートしています。ワークスペース名の横に表示されるアイコンでモードを確認できます。

| Icon | Workspace モード |
| --- | --- |
| <img src="/images/app_ui/automated_workspace.svg" alt="自動ワークスペースのアイコン" width="32px" /> | **Automated workspaces** は、そのプロジェクトでログされたすべてのキーに対してパネルを自動生成します。自動ワークスペースを選択するのは次のような場合です。<ul><li>プロジェクトで利用可能なすべてのデータを可視化して、すばやく使い始めたいとき。</li><li>ログするキーが少ない小規模なプロジェクトの場合。</li><li>より広範な分析を行いたい場合。</li></ul>自動ワークスペースからパネルを削除した場合は、[Quick add](#quick-add) を使って再作成できます。 |
| <img src="/images/app_ui/manual_workspace.svg" alt="手動ワークスペースのアイコン" width="32px" /> | **Manual workspaces** は空の状態から始まり、ユーザーが意図的に追加したパネルのみを表示します。手動ワークスペースを選択するのは次のような場合です。<ul><li>プロジェクトでログされたキーの一部だけに主に関心があるとき。</li><li>よりフォーカスした分析を行いたい場合。</li><li>自分にとって有用性の低いパネルの読み込みを避けて、ワークスペースのパフォーマンスを改善したい場合。</li></ul>[Quick add](#quick-add) を使うと、有用な可視化で手動ワークスペースとそのセクションをすばやく埋めることができます。 |

ワークスペースがパネルを生成する方法を変更するには、[ワークスペースをリセット](#reset-a-workspace)します。

<Note>
  **ワークスペースへの変更を元に戻す**

  ワークスペースへの変更を元に戻すには、Undo ボタン（左向きの矢印）をクリックするか、**CMD + Z**（macOS）または **CTRL + Z**（Windows / Linux）を押します。
</Note>

<div id="reset-a-workspace">
  ## Workspace をリセットする
</div>

Workspace をリセットするには、次の手順を実行します。

1. Workspace 上部にあるアクションメニュー `...` をクリックします。
2. **Reset workspace** をクリックします。

<div id="configure-the-workspace-layout">
  ## ワークスペース レイアウトを設定する
</div>

ワークスペース レイアウトを設定するには、ワークスペース上部付近の **Settings** をクリックし、**Workspace layout** をクリックします。

* **Hide empty sections during search**（デフォルトでオン）
* **Sort panels alphabetically**（デフォルトでオフ）
* **Section organization**（デフォルトでは先頭のプレフィックスでグループ化）。この設定を変更するには:
  1. 鍵アイコンをクリックします。
  2. セクション内でパネルをどのようにグループ化するかを選択します。

ワークスペースの折れ線グラフのデフォルト設定を行うには、[Line plots](/ja/models/app/features/panels/line-plot/) を参照してください。

<div id="configure-a-sections-layout">
  ### セクションのレイアウトを設定する
</div>

セクションのレイアウトを設定するには、そのセクションの歯車アイコンをクリックし、**Display preferences** をクリックします。

* **ツールチップ内の run 名の色分け表示をオンまたはオフにする**（デフォルトでオン）
* **コンパニオン チャートのツールチップにはハイライトされている run のみを表示する**（デフォルトでオフ）
* **ツールチップに表示する run の数**（単一の run、すべての run、または **Default**）
* **プライマリ チャートのツールチップに完全な run 名を表示する**（デフォルトでオフ）

<div id="view-a-panel-in-full-screen-mode">
  ## パネルを全画面表示で見る
</div>

全画面表示モードでは、run セレクタが表示され、パネルは通常の 1000 バケットではなく 10,000 バケットのフルフィデリティ サンプリングモードのプロットを使用します。

パネルを全画面表示モードで表示するには:

1. パネルにカーソルを重ねます。
2. パネルのアクションメニュー `...` をクリックし、続いて全画面ボタンをクリックします。このボタンは、ビューファインダーや正方形の四隅のアウトラインのようなアイコンです。
   <Frame>
     <img src="/images/app_ui/panel_fullscreen.png" alt="全画面表示パネル" />
   </Frame>
3. 全画面表示モードでパネルを表示しているときに[パネルを共有](#share-a-panel)すると、そのリンクを開いたときは自動的に全画面表示モードになります。

全画面表示モードからパネルのワークスペースに戻るには、ページ上部の左向き矢印をクリックします。全画面表示モードを終了せずにセクション内のパネルを切り替えるには、パネル下部の **Previous** と **Next** ボタン、または左右の矢印キーを使用します。

<div id="add-panels">
  ## パネルを追加する
</div>

ここでは、ワークスペースにパネルを追加するさまざまな方法を紹介します。

<div id="add-a-panel-manually">
  ### パネルを手動で追加する
</div>

パネルは、ワークスペースに一度に 1 つずつ、グローバルまたはセクション単位で追加できます。

1. パネルをグローバルに追加するには、パネル検索フィールド付近のコントロールバーで **Add panels** をクリックします。
2. パネルをセクションに直接追加するには、そのセクションのアクション `...` メニューをクリックし、**+ Add panels** をクリックします。
3. 追加するパネルの種類（チャートなど）を選択します。デフォルト値が選択された状態で、そのパネルの設定内容が表示されます。
4. 必要に応じて、パネルおよびその表示設定をカスタマイズします。利用可能な設定オプションは、選択したパネルの種類によって異なります。各パネルタイプのオプションについて詳しくは、[Line plots](/ja/models/app/features/panels/line-plot/) や [Bar plots](/ja/models/app/features/panels/bar-plot/) など、以下の該当セクションを参照してください。
5. **Apply** をクリックします。

<Frame>
  <img src="/images/app_ui/add_single_panel.gif" alt="パネルを追加するデモ" />
</Frame>

<div id="quick-add-panels">
  ### クイック追加パネル
</div>

**Quick add** を使用すると、選択した各キーに対してパネルを自動的に追加できます。これはグローバルにもセクションレベルにも適用できます。

<Note>
  削除されたパネルがない自動生成されたワークスペースでは、すでにすべてのログ済みキー用のパネルが含まれているため、**Quick add** オプションは表示されません。削除したパネルを再度追加する場合に **Quick add** を使用できます。
</Note>

1. **Quick add** を使ってパネルをグローバルに追加するには、パネル検索フィールド付近のコントロールバーで **Add panels** をクリックし、続いて **Quick add** をクリックします。
2. **Quick add** を使ってパネルをセクションに直接追加するには、そのセクションのアクション `...` メニューをクリックし、**Add panels** をクリックしてから **Quick add** をクリックします。
3. パネルの一覧が表示されます。チェックマークが付いているパネルは、すでにワークスペースに含まれています。
   * すべての利用可能なパネルを追加するには、一覧の上部にある **Add `<N>` panels** ボタンをクリックします。**Quick Add** リストが閉じられ、新しいパネルがワークスペースに表示されます。
   * 一覧から個々のパネルを追加するには、そのパネルの行にカーソルを合わせてから **Add** をクリックします。追加したい各パネルについてこの手順を繰り返し、完了したら右上の **X** をクリックして **Quick Add** リストを閉じます。新しいパネルがワークスペースに表示されます。
4. 必要に応じて、パネルの設定をカスタマイズします。

<div id="share-a-panel">
  ## パネルを共有する
</div>

このセクションでは、リンクを使ってパネルを共有する方法を説明します。

リンクを使ってパネルを共有するには、次のいずれかを実行します。

* パネルを全画面表示モードで表示しているときに、ブラウザからURLをコピーします。
* アクションメニュー `...` をクリックして、**Copy panel URL** を選択します。

リンクをユーザーやチームと共有してください。リンクにアクセスすると、パネルは[全画面表示モード](#view-a-panel-in-full-screen-mode)で開きます。

全画面表示モードからパネルのワークスペースに戻るには、ページ上部の左向き矢印をクリックします。

<div id="compose-a-panels-full-screen-link-programmatically">
  ### パネルのフルスクリーンリンクをプログラムから生成する
</div>

[オートメーションの作成](/ja/models/automations/) などの状況では、パネルのフルスクリーン URL を含めると便利な場合があります。このセクションでは、パネルのフルスクリーン URL の形式を示します。次の例では、角括弧内の entity 名、プロジェクト名、パネル名、およびセクション名を自分の環境に合わせて置き換えてください。

```text
https://wandb.ai/<ENTITY_NAME>/<PROJECT_NAME>?panelDisplayName=<PANEL_NAME>&panelSectionName=<SECTON_NAME>
```

同じセクション内に同名のパネルが複数ある場合、このURLではその名前の最初のパネルが開きます。

<div id="embed-or-share-a-panel-on-social-media">
  ### パネルを埋め込む、またはソーシャルメディアで共有する
</div>

パネルをウェブサイトに埋め込むかソーシャルメディアで共有するには、リンクを持っている人なら誰でもそのパネルを表示できる必要があります。プロジェクトが非公開の場合、パネルを表示できるのはプロジェクトのメンバーのみです。プロジェクトが公開の場合、リンクを持っている人なら誰でもパネルを表示できます。

パネルを埋め込むコードを取得するか、ソーシャルメディアで共有するには、次の手順を実行します。

1. Workspace でパネルの上にカーソルを合わせ、アクションメニュー `...` をクリックします。
2. **Share** タブをクリックします。
3. **Only those who are invited have access** を **Anyone with the link can view** に変更します。これを行わないと、次の手順の選択肢は使用できません。
4. **Share on Twitter**、**Share on Reddit**、**Share on LinkedIn**、または **Copy embed link** を選択します。

<div id="email-a-panel-report">
  ### パネルレポートをメールで送信する
</div>

単一のパネルを単独のレポートとしてメール送信するには、次の手順を実行します。

1. パネルにマウスカーソルを合わせ、パネルのアクションメニュー `...` をクリックします。
2. **Share panel in report** をクリックします。
3. **Invite** タブを選択します。
4. メールアドレスまたはユーザー名を入力します。
5. 必要に応じて **can view** を **can edit** に変更します。
6. **Invite** をクリックします。W&amp;B は、共有しているパネルのみを含むレポートへのリンクが含まれたメールをユーザーに送信します。

[パネルを共有](#share-a-panel) する場合とは異なり、受信者はこのレポートからワークスペースにアクセスすることはできません。

<div id="manage-panels">
  ## パネルの管理
</div>

<div id="edit-a-panel">
  ### パネルを編集する
</div>

パネルを編集するには、次の手順に従います。

1. パネルの鉛筆アイコンをクリックします。
2. パネルの設定を変更します。
3. パネルのタイプを変更する場合は、変更先のタイプを選択し、その設定を行います。
4. **Apply** をクリックします。

<div id="move-a-panel">
  ### パネルを移動する
</div>

パネルを別のセクションに移動するには、パネル上のドラッグハンドルを使用します。リストから新しいセクションを選択したい場合は、次の手順を実行します。

1. 必要であれば、最後のセクションの後にある **Add section** をクリックして新しいセクションを作成します。
2. パネルの `...` アクションメニューをクリックします。
3. **Move** をクリックし、新しいセクションを選択します。

ドラッグハンドルを使って、同じセクション内でパネルの並び順を変更することもできます。

<div id="duplicate-a-panel">
  ### パネルを複製する
</div>

パネルを複製するには、次の手順を実行します。

1. パネル上部のアクション `...` メニューをクリックします。
2. **複製** をクリックします。

必要に応じて、複製したパネルを[カスタマイズ](#edit-a-panel)したり、[移動](#move-a-panel)したりできます。

<div id="remove-panels">
  ### パネルを削除する
</div>

パネルを削除するには：

1. パネルの上にマウスポインタを合わせます。
2. アクション `...` メニューを選択します。
3. **Delete** をクリックします。

手動ワークスペースからすべてのパネルを削除するには、そのアクション `...` メニューをクリックし、**Clear all panels** をクリックします。

自動または手動のワークスペースからすべてのパネルを削除するには、[ワークスペースをリセット](#reset-a-workspace)します。デフォルトのパネルセットから開始するには **Automatic** を選択し、パネルがない空のワークスペースから開始するには **Manual** を選択します。

<div id="manage-sections">
  ## セクションを管理する
</div>

デフォルトでは、ワークスペース内のセクションはキーのロギングの階層構造を反映します。ただし、手動管理のワークスペースでは、パネルを追加し始めてから初めてセクションが表示されます。

<div id="add-a-section">
  ### セクションを追加する
</div>

セクションを追加するには、最後のセクションの後にある **Add section** をクリックします。

既存のセクションの前または後に新しいセクションを追加するには、そのセクションのアクション `...` メニューをクリックし、**New section below** または **New section above** をクリックします。

<Warning>
  セクション名を「Section」としないでください。既知の制限により、そのセクションの名前を変更するまで、そのセクション内のパネルは表示されません。
</Warning>

<div id="manage-a-sections-panels">
  ### セクション内のパネルを管理する
</div>

多数のパネルを含むセクションは、デフォルトで複数ページに分割されます。1ページあたりのデフォルトのパネル数は、パネルの設定と、そのセクション内のパネルのサイズに依存します。

1. パネルのサイズを変更するには、そのパネルにカーソルを合わせてドラッグハンドルをクリックし、そのままドラッグしてパネルのサイズを調整します。1つのパネルのサイズを変更すると、そのセクション内のすべてのパネルのサイズも変更されます。
2. セクションがページ分割されている場合、ページに表示するパネル数をカスタマイズできます。
3. セクション上部で **1 to `<X>` of `<Y>`** をクリックします。`<X>` は表示中のパネル数、`<Y>` はパネルの総数です。
4. 1ページあたりに表示するパネル数を、最大100まで選択します。
5. セクションからパネルを削除するには:
6. パネルにカーソルを合わせて、そのパネルのアクション `...` メニューをクリックします。
7. **Delete** をクリックします。

ワークスペースを自動 Workspace にリセットすると、削除されたすべてのパネルが再び表示されます。

<div id="rename-a-section">
  ### セクション名を変更する
</div>

セクション名を変更するには、そのセクションのアクション `...` メニューをクリックし、**Rename section** をクリックします。

<Warning>
  セクション名を &quot;Section&quot; にしないでください。既知の制限により、このセクション内のパネルは、セクション名を変更するまで表示されません。
</Warning>

<div id="delete-a-section">
  ### セクションを削除する
</div>

セクションを削除するには、そのセクションの `...` メニューをクリックし、**Delete section** を選択します。これにより、そのセクションとその中のパネルが削除されます。
