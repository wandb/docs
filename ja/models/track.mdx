<div id="how-it-works">
  ## 動作の仕組み
</div>

数行のコードで機械学習の実験をトラッキングできます:

1. [W&amp;B Run](/ja/models/runs/) を作成します。
2. 学習率やモデルのタイプなどのハイパーパラメーターの辞書を、設定 ([`wandb.Run.config`](/ja/models/track/config/)) に保存します。
3. トレーニングループ内で、accuracy や loss などのメトリクスを時間の経過に沿ってログします ([`wandb.Run.log()`](/ja/models/track/log/))。
4. モデルの重みや予測の表など、run の出力を保存します。

以下のコードは、一般的な W&amp;B の実験管理ワークフローを示しています:

```python
# runを開始します。
#
# このブロックを抜けると、ログされたデータのアップロードが完了するまで待機します。
# 例外が発生した場合、runは失敗としてマークされます。
with wandb.init(entity="", project="my-project-name") as run:
  # モデルの入力とハイパーパラメーターを保存します。
  run.config.learning_rate = 0.01

  # 実験コードを実行します。
  for epoch in range(num_epochs):
    # トレーニングを実行します...

    # モデル性能を可視化するために、メトリクスを継続的にログします。
    run.log({"loss": loss})

  # モデルの出力をアーティファクトとしてアップロードします。
  run.log_artifact(model)
```
