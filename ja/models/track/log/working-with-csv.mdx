W&amp;B Python Libraryを使用してCSVファイルをログし、[W&amp;B ダッシュボード](/ja/models/track/workspaces/)で可視化します。W&amp;B ダッシュボードは、機械学習モデルの結果を整理して可視化するための中心的な場所です。これは特に、[過去の機械学習experimentsの情報を含むCSVファイル](#import-and-log-your-csv-of-experiments)があり、それらがW&amp;Bにログされていない場合や、[データセットを含むCSVファイル](#import-and-log-your-dataset-csv-file)がある場合に便利です。

<div id="import-and-log-your-dataset-csv-file">
  ## データセットのCSVファイルをインポートしてログする
</div>

{/* {% embed url="https://drive.google.com/file/d/1jBG3M4VnaMgeclRzowYZEYvFxvwb9SXF/view?usp=sharing" %} */}

CSV ファイルの内容を再利用しやすくするために、W&amp;B アーティファクト の使用をおすすめします。

1. まず、CSV ファイルをインポートしてください。次の 코드 스니펫 内の `iris.csv` というファイル名を、ご使用の CSV ファイル名に置き換えてください。

```python
import wandb
import pandas as pd

# CSV を新しい DataFrame に読み込む
new_iris_dataframe = pd.read_csv("iris.csv")
```

2. CSV ファイルを W&amp;B Table に変換して、[W&amp;B ダッシュボード](/ja/models/track/workspaces/) を活用します。

```python
# DataFrame を W&B Table に変換する
iris_table = wandb.Table(dataframe=new_iris_dataframe)
```

3. 次に、W&amp;B アーティファクトを作成し、そのアーティファクトに表を追加します。

```python
# テーブルを Artifact に追加して行数の上限を
# 200000 に増やし、再利用しやすくする
iris_table_artifact = wandb.Artifact("iris_artifact", type="dataset")
iris_table_artifact.add(iris_table, "iris_table")

# データを保持するために Artifact 内に生の CSV ファイルをログする
iris_table_artifact.add_file("iris.csv")
```

W&amp;B アーティファクト の詳細については、[アーティファクト の章](/ja/models/artifacts/) を参照してください。

4. 最後に、`wandb.init` を使用して新しい W&amp;B run を開始し、W&amp;B へのトラッキングとログ記録を行います。

```python
# データをログするために W&B の run を開始する
with wandb.init(project="tables-walkthrough") as run:

    # run で可視化するためにテーブルをログする...
    run.log({"iris": iris_table})

    # Artifact としてログして、利用可能な行数の上限を増やす！
    run.log_artifact(iris_table_artifact)
```

`wandb.init()` API は、新しいバックグラウンドプロセスを起動してデータを run にログし、デフォルトではデータを wandb.ai に同期します。W&amp;B Workspace ダッシュボードでリアルタイムの可視化を確認できます。次の画像は、このコードスニペットを実行した際の出力例を示しています。

<Frame>
  <img src="/images/track/import_csv_tutorial.png" alt="W&B ダッシュボードにインポートされた CSV ファイル" />
</Frame>

前述のコードスニペットをすべて含んだ完全なスクリプトは、以下に示します。

```python
import wandb
import pandas as pd

# CSV を新しい DataFrame に読み込む
new_iris_dataframe = pd.read_csv("iris.csv")

# DataFrame を W&B Table に変換する
iris_table = wandb.Table(dataframe=new_iris_dataframe)

# テーブルを Artifact に追加して行数の上限を
# 200000 に増やし、再利用しやすくする
iris_table_artifact = wandb.Artifact("iris_artifact", type="dataset")
iris_table_artifact.add(iris_table, "iris_table")

# データを保持するために Artifact 内に生の CSV ファイルをログする
iris_table_artifact.add_file("iris.csv")

# データをログするために W&B の run を開始する
with wandb.init(project="tables-walkthrough") as run:

    # run でテーブルをログして可視化する...
    run.log({"iris": iris_table})

    # Artifact としてログして利用可能な行数の上限を増やす！
    run.log_artifact(iris_table_artifact)
```

場合によっては、実験の詳細を CSV ファイルに保存していることがあります。このような CSV ファイルによく含まれる情報は次のとおりです。

* 実験の run 名
* 初期の[メモ](/ja/models/runs/#add-a-note-to-a-run)
* 実験を区別するための[Tags](/ja/models/runs/tags/)
* 実験に必要な設定 ([Sweeps ハイパーパラメーター チューニング](/ja/models/sweeps/)を活用できるという利点もあります) 。

| 実験   | モデル名             | メモ                      | Tags         | レイヤー数 | 最終トレーニング精度 | 最終検証精度 | トレーニング損失                             |
| ---- | ---------------- | ----------------------- | ------------ | ----- | ---------- | ------ | ------------------------------------ |
| 実験 1 | mnist-300-layers | トレーニングデータに過剰適合しすぎた      | [latest]     | 300   | 0.99       | 0.90   | [0.55, 0.45, 0.44, 0.42, 0.40, 0.39] |
| 実験 2 | mnist-250-layers | 現在の 최고 성능 モデル           | [prod, best] | 250   | 0.95       | 0.96   | [0.55, 0.45, 0.44, 0.42, 0.40, 0.39] |
| 実験 3 | mnist-200-layers | ベースラインモデルより悪かった。デバッグが必要 | [debug]      | 200   | 0.76       | 0.70   | [0.55, 0.45, 0.44, 0.42, 0.40, 0.39] |
| ...  | ...              | ...                     | ...          | ...   | ...        | ...    |                                      |
| 実験 N | mnist-X-layers   | メモ                      | ...          | ...   | ...        | ...    | [..., ...]                           |

W&amp;B は実験の CSV ファイルを受け取り、それを W&amp;B Experiment Run に変換できます。以下の코드 스니펫とコードスクリプトは、実験の CSV ファイルを import してログする方法を示しています。

1. まず、CSV ファイルを Read して Pandas DataFrame に変換します。`"experiments.csv"` をご使用の CSV ファイル名に置き換えてください。

```python
import wandb
import pandas as pd

FILENAME = "experiments.csv"
loaded_experiment_df = pd.read_csv(FILENAME)

PROJECT_NAME = "Converted Experiments"

EXPERIMENT_NAME_COL = "Experiment"
NOTES_COL = "Notes"
TAGS_COL = "Tags"
CONFIG_COLS = ["Num Layers"]
SUMMARY_COLS = ["Final Train Acc", "Final Val Acc"]
METRIC_COLS = ["Training Losses"]

# Pandas DataFrameを扱いやすい形式に整形する
for i, row in loaded_experiment_df.iterrows():
    run_name = row[EXPERIMENT_NAME_COL]
    notes = row[NOTES_COL]
    tags = row[TAGS_COL]

    config = {}
    for config_col in CONFIG_COLS:
        config[config_col] = row[config_col]

    metrics = {}
    for metric_col in METRIC_COLS:
        metrics[metric_col] = row[metric_col]

    summaries = {}
    for summary_col in SUMMARY_COLS:
        summaries[summary_col] = row[summary_col]
```

2. 次に、新しい W&amp;B Run を開始して、[`wandb.init()`](/ja/models/ref/python/functions/init) を使用して W&amp;B へのトラッキングとログ記録を行います。

   ```python
   with wandb.init(
       project=PROJECT_NAME, name=run_name, tags=tags, notes=notes, config=config
   ) as run:
   ```

experiment の実行中は、W&amp;B で表示、クエリ、分析できるように、メトリクスの各インスタンスをログしたい場合があります。これを行うには、[`run.log()`](/ja/models/ref/python/experiments/run/#method-runlog) コマンドを使用します。

```python
run.log({key: val})
```

必要に応じて、[`define_metric`](/ja/models/ref/python/experiments/run#define_metric) API を使用して、run の結果を定義する最終的な summary メトリクスをログできます。この例では、`run.summary.update()` を使用して summary メトリクスを run に追加します:

```python
run.summary.update(summaries)
```

summary メトリクスの詳細については、[summary メトリクスをログする](./log-summary)を参照してください。

以下は、上記のサンプル表を [W&amp;B ダッシュボード](/ja/models/track/workspaces/) に変換する完全なサンプルスクリプトです。

```python
FILENAME = "experiments.csv"
loaded_experiment_df = pd.read_csv(FILENAME)

PROJECT_NAME = "Converted Experiments"

EXPERIMENT_NAME_COL = "Experiment"
NOTES_COL = "Notes"
TAGS_COL = "Tags"
CONFIG_COLS = ["Num Layers"]
SUMMARY_COLS = ["Final Train Acc", "Final Val Acc"]
METRIC_COLS = ["Training Losses"]

for i, row in loaded_experiment_df.iterrows():
    run_name = row[EXPERIMENT_NAME_COL]
    notes = row[NOTES_COL]
    tags = row[TAGS_COL]

    config = {}
    for config_col in CONFIG_COLS:
        config[config_col] = row[config_col]

    metrics = {}
    for metric_col in METRIC_COLS:
        metrics[metric_col] = row[metric_col]

    summaries = {}
    for summary_col in SUMMARY_COLS:
        summaries[summary_col] = row[summary_col]

    with  wandb.init(
        project=PROJECT_NAME, name=run_name, tags=tags, notes=notes, config=config
    ) as run:

        for key, val in metrics.items():
            if isinstance(val, list):
                for _val in val:
                    run.log({key: _val})
            else:
                run.log({key: val})

        run.summary.update(summaries)
```
