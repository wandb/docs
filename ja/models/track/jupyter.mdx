ノートブックを離れることなく、W&amp;B を Jupyter と使用してインタラクティブな可視化を取得します。カスタム分析、実験、プロトタイプを組み合わせ、すべてを完全にログできます。

<div id="use-cases-for-wb-with-jupyter-notebooks">
  ## Jupyter notebook で W&amp;B を使用するユースケース
</div>

1. **反復的な実験**: パラメーターを調整しながら実験を実行して繰り返し実行し、その過程で手動でメモを取らなくても、行ったすべての Runs が自動的に W&amp;B に保存されます。
2. **コードの保存**: モデルを再現するとき、notebook 内のどのセルがどの順序で実行されたかを把握するのは困難です。[設定ページ](/ja/platform/app/settings-page/)でコードの保存を有効にすると、各実験についてセル実行の記録を保存できます。
3. **独自の分析**: Runs が W&amp;B にログされると、API からデータフレームを取得して独自の分析を行い、その結果を W&amp;B にログして保存し、Reports で共有することが簡単になります。

<div id="getting-started-in-a-notebook">
  ## ノートブックでの Getting Started
</div>

以下のコードでノートブックを開始し、W&amp;B をインストールして Account をリンクします。

```notebook
!pip install wandb -qqq
import wandb
wandb.login()
```

次に、実験を設定し、ハイパーパラメーターを保存します:

```python
wandb.init(
    project="jupyter-projo",
    config={
        "batch_size": 128,
        "learning_rate": 0.01,
        "dataset": "CIFAR-100",
    },
)
```

`wandb.init()` を実行した後、ノートブックでライブグラフを表示するには、`%%wandb` を付けて新しいセルを開始します。このセルを複数回実行すると、データは run に追記されます。

```notebook
%%wandb

# トレーニングループをここに記述
```

この[例示ノートブック](https://wandb.me/jupyter-interact-colab)で実際に試してみてください。

<div id="rendering-live-wb-interfaces-directly-in-your-notebooks">
  ### ノートブックにライブのW&amp;Bインターフェイスを直接表示する
</div>

また、`%wandb` マジックを使用して、既存のダッシュボード、sweep、またはレポートをノートブックに直接表示できます。

```notebook
# プロジェクトのワークスペースを表示する
%wandb USERNAME/PROJECT
# 単一のrunを表示する
%wandb USERNAME/PROJECT/runs/RUN_ID
# sweepを表示する
%wandb USERNAME/PROJECT/sweeps/SWEEP_ID
# レポートを表示する
%wandb USERNAME/PROJECT/reports/REPORT_ID
# 埋め込みiframeの高さを指定する
%wandb USERNAME/PROJECT -h 2048
```

`%%wandb` または `%wandb` マジックの代替として、`wandb.init()` を実行した後、任意のセルの末尾に `wandb.Run.finish()` を追加するとインライングラフを表示できます。または、API から返される任意のレポート、sweep、または run object に対して `ipython.display(...)` を呼び出すこともできます。

```python
import wandb
from IPython.display import display
# runを初期化する
run = wandb.init()

# セルがrun.finish()を出力すると、ライブグラフが表示されます
run.finish()
```

<div id="additional-jupyter-features-in-wb">
  ## W&amp;B の Jupyter の追加機能
</div>

1. **Colab での簡単な認証**: Colab で初めて `wandb.init` を呼び出すと、ブラウザーで現在 W&amp;B にログインしている場合は、ランタイムを自動的に認証します。run ページの Overview タブには、Colab へのリンクが表示されます。
2. **Jupyter マジック:** ダッシュボード、Sweeps、レポートをノートブックに直接表示します。`%wandb` マジック は、プロジェクト、Sweeps、またはレポートへのパスを受け入れ、ノートブック内に W&amp;B インターフェイスを直接表示します。
3. **Docker 化された Jupyter を起動する**: `wandb docker --jupyter` を呼び出して Docker コンテナーを起動し、その中にコードをマウントし、Jupyter がインストールされていることを確認して、ポート 8888 で起動します。
4. **任意の順序で安心してセルを実行する**: デフォルトでは、run を `finished` としてマークするのは、次に `wandb.init` が呼び出されるときまで待機します。これにより、複数のセル (たとえば、1 つはデータを設定するため、1 つはトレーニングするため、1 つはテストするため) を好きな順序で実行し、それらすべてを同じ run にログできます。[User Settings](https://wandb.ai/settings) でコードの保存を有効にすると、実行されたセルも、順序どおりに、実行時の状態のままでログされるため、どれほど非線形なパイプラインでも再現できます。Jupyter notebook で run を手動で完了としてマークするには、`run.finish` を呼び出します。

```python
import wandb

run = wandb.init()

# トレーニングスクリプトとログするをここに記述

run.finish()
```
