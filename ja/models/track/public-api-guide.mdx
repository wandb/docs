<div id="import-data-from-mlflow">
  ## MLFlow からデータをインポートする
</div>

W&amp;B は MLFlow からのデータのインポートをサポートしており、実験、Runs、 Artifacts 、メトリクス、その他のメタデータをインポートできます。

依存パッケージをインストールします:

```shell
# 注意: py38以上が必要です
pip install wandb[importers]
```

W&amp;B にログインします。まだログインしたことがない場合は、画面の指示に従ってください。

```shell
wandb login
```

既存の MLFlow サーバーからすべての Runs をインポートします：

```py
from wandb.apis.importers.mlflow import MlflowImporter

importer = MlflowImporter(mlflow_tracking_uri="...")

runs = importer.collect_runs()
importer.import_runs(runs)
```

デフォルトでは、`importer.collect_runs()` は MLFlow サーバー上のすべての Runs を収集します。特定のサブセットのみをアップロードしたい場合は、独自の Runs イテラブルを作成し、それを importer に渡せます。

```py
import mlflow
from wandb.apis.importers.mlflow import MlflowRun

client = mlflow.tracking.MlflowClient(mlflow_tracking_uri)

runs: Iterable[MlflowRun] = []
for run in mlflow_client.search_runs(...):
    runs.append(MlflowRun(run, client))

importer.import_runs(runs)
```

<Note>
  Databricks MLFlow からインポートする場合は、先に [Databricks CLI を設定する必要があることがあります](https://docs.databricks.com/dev-tools/cli/index.html)。

  前の step で `mlflow-tracking-uri="databricks"` を設定してください。
</Note>

Artifacts のインポートをスキップするには、`artifacts=False` を渡します：

```py
importer.import_runs(runs, artifacts=False)
```

特定の W&amp;B の entity とプロジェクトにインポートするには、`Namespace` を渡せます。

```py
from wandb.apis.importers import Namespace

importer.import_runs(runs, namespace=Namespace(entity, project))
```

<div id="export-data">
  ## データをエクスポート
</div>

Public API を使用して、W&amp;B に保存したデータをエクスポートまたは更新できます。この API を使用する前に、スクリプトからデータをログしてください。詳細については、[クイックスタート](/ja/models/quickstart/) を参照してください。

**Public API のユースケース**

* **データをエクスポート**: Jupyter Notebook でカスタム分析を行うために、データフレームを取得します。データを調査した後は、新しい analysis run を作成して結果をログすることで、調査結果を同期できます。例: `wandb.init(job_type="analysis")`
* **既存の Runs を更新**: W&amp;B run に関連付けてログしたデータを更新できます。たとえば、複数の run の設定を更新して、アーキテクチャや、もともとログしていなかったハイパーパラメーターなどの追加情報を含めたい場合があります。

利用可能な関数の詳細については、[Generated Reference Docs](/ja/models/ref/python/public-api/) を参照してください。

<div id="find-the-run-path">
  ### run パスを見つける

Public API を使用するには、多くの場合、`<entity>/<project>/<run_id>` 形式の run パスが必要です。アプリの UI で run ページを開き、[Overview タブ ](/ja/models/track/public-api-guide/#overview-tab)をクリックして run パスを取得します。

<div id="export-run-data">
  ### run データのエクスポート
</div>

完了済みまたは実行中の run からデータをダウンロードします。一般的な使用例としては、Jupyter Notebook でカスタム分析を行うためにデータフレームをダウンロードしたり、自動化された環境でカスタムロジックに利用したりすることが挙げられます。

```python
import wandb

api = wandb.Api()
run = api.run("<entity>/<project>/<run_id>")
```

run オブジェクトで最もよく使用される属性は次のとおりです:

| Attribute       | Meaning                                                                                                                                                                      |
| --------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `run.config`    | run の設定情報を保持する辞書です。たとえば、トレーニング run のハイパーパラメーターや、データセット アーティファクトを作成する run の前処理方法などが含まれます。これらは run の入力と考えるとわかりやすいです。                                                           |
| `run.history()` | 損失のように、モデルのトレーニング中に変化する値を保存するための辞書のリストです。`run.log()` コマンドは、このオブジェクトに値を追加します。                                                                                                 |
| `run.summary`   | run の結果を要約した情報を保持する辞書です。精度や損失などのスカラー値だけでなく、大きなファイルも含めることができます。デフォルトでは、`run.log()` はログされた時系列データの最終値を summary に設定します。summary の内容は直接設定することもできます。summary は run の出力と考えるとわかりやすいです。 |

過去の run のデータを変更または更新することもできます。デフォルトでは、1 つの API オブジェクト インスタンスがすべてのネットワークリクエストをキャッシュします。実行中のスクリプトでリアルタイムの情報が必要な場合は、`api.flush()` を呼び出して更新された値を取得してください。

<div id="understanding-different-run-attributes">
  ### さまざまな run 属性の理解
</div>

次のコードスニペットは、run を作成していくつかのデータをログし、その run の属性にアクセスする方法を示しています。

```python
import wandb
import random

with wandb.init(project="public-api-example") as run:
    n_epochs = 5
    config = {"n_epochs": n_epochs}
    run.config.update(config)
    for n in range(run.config.get("n_epochs")):
        run.log(
            {"val": random.randint(0, 1000), "loss": (random.randint(0, 1000) / 1000.00)}
        )
```

<div id="sampling">
  ### サンプリング
</div>

デフォルトの history method は、メトリクスを一定数のサンプル (デフォルトは 500。`samples` &#95;&#95; 인수で変更可能) にサンプリングします。大規模な run のすべてのデータをエクスポートしたい場合は、`run.scan_history()` method を使用できます。詳細については、[API Reference](/ja/models/ref/python/public-api) を参照してください。

<div id="querying-multiple-runs">
  ### 複数の Runs をクエリする
</div>

<Tabs>
  <Tab title="DataFrame と CSV">
    このサンプルスクリプトはプロジェクトを検索し、Runs の名前、設定、およびサマリー統計量を含む CSV を出力します。`<entity>` と `<project>` を、それぞれあなたの W&amp;B entity とプロジェクト名に置き換えてください。

    ```python
    import pandas as pd
    import wandb

    api = wandb.Api()
    entity, project = "<entity>", "<project>"
    runs = api.runs(entity + "/" + project)

    summary_list, config_list, name_list = [], [], []
    for run in runs:
        # .summary には精度などの
        # メトリクスの出力キーと値が含まれます。
        #  大きなファイルを除外するために ._json_dict を呼び出します
        summary_list.append(run.summary._json_dict)

        # .config にはハイパーパラメーターが含まれます。
        #  _ で始まる特殊な値を削除します。
        config_list.append({k: v for k, v in run.config.items() if not k.startswith("_")})

        # .name は run の人間が読める名前です。
        name_list.append(run.name)

    runs_df = pd.DataFrame(
        {"summary": summary_list, "config": config_list, "name": name_list}
    )

    runs_df.to_csv("project.csv")

    run.finish()
    ```
  </Tab>

  <Tab title="MongoDB スタイル">
    W&amp;B API は、`api.runs()` を使ってプロジェクト内の Runs を横断的にクエリする方法も提供します。最も一般的なユースケースは、Runs データをエクスポートしてカスタム分析を行うことです。クエリインターフェースは [MongoDB が使用しているもの](https://www.mongodb.com/docs/manual/reference/mql/query-predicates/) と同じです。

    ```python
    runs = api.runs(
        "username/project",
        {"$or": [{"config.experiment_name": "foo"}, {"config.experiment_name": "bar"}]},
    )
    print(f"Found {len(runs)} runs")
    ```
  </Tab>
</Tabs>

`api.runs` を呼び出すと、反復可能でリストのように動作する `Runs` オブジェクトが返されます。デフォルトでは、このオブジェクトは必要に応じて一度に 50 件の Runs を順番に読み込みますが、`per_page` キーワード引数を使って 1 ページあたりに読み込む数を変更できます。

`api.runs` は `order` キーワード引数も受け取ります。デフォルトの順序は `-created_at` です。昇順で並べ替えるには `+created_at` を指定します。設定や summary の値でソートすることもできます。たとえば `summary.val_acc` や `config.experiment_name` などです。

<div id="get-a-runs-name-and-id-during-a-run">
  ### run実行中に run 名と ID を取得する
</div>

`wandb.init()` を呼び出した後、スクリプトからランダムに生成された run ID や人間が読みやすい run 名に、次のようにアクセスできます:

* 一意の run ID (8 文字のハッシュ): `run.id`
* ランダムな run 名 (人間が読みやすい): `run.name`

Runs の有用な識別子を設定する方法を考えている場合、以下をおすすめします:

* **Run ID**: 生成されたハッシュのままにしておきます。これはプロジェクト内の Runs で一意である必要があります。
* **Run name**: チャート上の異なる線を区別できるように、短く読みやすく、可能であれば一意のものにします。
* **Run notes**: run で何を行っているかの簡単な説明を記入するのに最適な場所です。`wandb.init(notes="your notes here")` で設定できます。
* **Run tags**: run タグで動的に情報を追跡し、UI のフィルターを使って関心のある Runs のみを表に絞り込みます。タグはスクリプトから設定でき、その後 UI の Runs 表および run ページの Overview タブの両方で編集できます。詳細な手順は[こちら](/ja/models/runs/tags/)を参照してください。

<div id="read-metrics-from-a-run">
  ### run からメトリクスを読み取る
</div>

この例では、`"<entity>/<project>/<run_id>"` に保存された run について、`run.log({"accuracy": acc})` で保存された Timestamp と accuracy を出力します。

```python
import wandb

api = wandb.Api()

run = api.run("<entity>/<project>/<run_id>")
if run.state == "finished":
    for i, row in run.history().iterrows():
        print(row["_timestamp"], row["accuracy"])
```

<div id="read-specific-metrics-from-a-run">
  ### run から特定のメトリクスを読み取る
</div>

run から特定のメトリクスを取得するには、`keys` 인수を使用します。`run.history()` を使用する場合のデフォルトのサンプル数は 500 です。特定のメトリクスを含まないログ済みの step は、出力データフレームでは `NaN` として表示されます。`keys` 인수を指定すると、API は指定したメトリクスキーを含む step をより高い頻度でサンプリングします。

```python
import wandb

api = wandb.Api()

run = api.run("<entity>/<project>/<run_id>")
if run.state == "finished":
    for i, row in run.history(keys=["accuracy"]).iterrows():
        print(row["_timestamp"], row["accuracy"])
```

<div id="update-metrics-for-a-run-after-the-run-has-finished">
  ### 完了した run のメトリクスを更新する

<div id="get-unsampled-metric-data">
  ### 未サンプリングのメトリクスデータを取得する
</div>

history からデータを取得すると、デフォルトでは 500 ポイントにサンプリングされます。`run.scan_history()` を使用して、ログされたすべてのデータポイントを取得できます。以下は、history にログされたすべての `loss` データポイントをダウンロードする例です。

```python
import wandb

api = wandb.Api()

run = api.run("<entity>/<project>/<run_id>")
history = run.scan_history()
losses = [row["loss"] for row in history]
```

<div id="get-paginated-data-from-history">
  ### 履歴からページ単位でデータを取得する
</div>

バックエンドでのメトリクス取得が遅い場合や、API リクエストがタイムアウトする場合は、個々のリクエストがタイムアウトしないように、`scan_history` のページサイズを小さくしてみてください。デフォルトのページサイズは 500 です。どの値が最適かを確認するために、いくつかのサイズを試してみてください：

```python
import wandb

api = wandb.Api()

run = api.run("<entity>/<project>/<run_id>")
run.scan_history(keys=sorted(cols), page_size=100)
```

<div id="export-metrics-from-all-runs-in-a-project-to-a-csv-file">
  ### プロジェクト内のすべての Runs のメトリクスを CSV ファイルにエクスポートする
</div>

このスクリプトは、プロジェクト内の Runs を取得し、各 Runs の名前、設定、およびサマリー統計量を含むデータフレーム と CSV ファイルを生成します。`<entity>` と `<project>` は、それぞれ W&amp;B の entity とプロジェクト名に置き換えてください。

```python
import pandas as pd
import wandb

api = wandb.Api()
entity, project = "<entity>", "<project>"
runs = api.runs(entity + "/" + project)

summary_list, config_list, name_list = [], [], []
for run in runs:
    # .summary には、accuracy などのメトリクスの出力キーと値が含まれます。
    #  大きなファイルを省略するために ._json_dict を呼び出します。
    summary_list.append(run.summary._json_dict)

    # .config にはハイパーパラメーターが含まれます。
    #  _ で始まる特殊な値は除外します。
    config_list.append({k: v for k, v in run.config.items() if not k.startswith("_")})

    # .name は run の人間が読める名前です。
    name_list.append(run.name)

runs_df = pd.DataFrame(
    {"summary": summary_list, "config": config_list, "name": name_list}
)

runs_df.to_csv("project.csv")
```

<div id="download-the-best-model-file-from-a-sweep">
  ### sweep から最良のモデルファイルをダウンロードする
</div>

このスニペットは、モデルファイルを `model.h5` として保存した run を含む sweep から、検証精度が最も高いモデルファイルをダウンロードします。

```python
import wandb

api = wandb.Api()

sweep = api.sweep("<entity>/<project>/<sweep_id>")
runs = sorted(sweep.runs, key=lambda run: run.summary.get("val_acc", 0), reverse=True)
val_acc = runs[0].summary.get("val_acc", 0)
print(f"Best run {runs[0].name} with {val_acc}% val accuracy")

runs[0].file("model.h5").download(replace=True)
print("Best model saved to model-best.h5")
```

<div id="delete-all-files-with-a-given-extension-from-a-run">
  ### 指定した拡張子を持つファイルを run からすべて削除する
</div>

このスニペットは、指定した拡張子を持つファイルを run から削除します。

```python
import wandb

api = wandb.Api()

run = api.run("<entity>/<project>/<run_id>")

extension = ".png"
files = run.files()
for file in files:
    if file.name.endswith(extension):
        file.delete()
```
