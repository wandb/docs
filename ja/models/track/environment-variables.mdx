---
description: "認証、プロジェクト設定、ログモードなどを環境変数で設定して、W&B SDK の動作を制御します。"
title: 環境変数
---

自動化された環境でスクリプトを実行する場合は、スクリプトの実行前またはスクリプト内で設定した環境変数を使って W&amp;B を制御できます。

<div id="optional-environment-variables">
  ## 任意の環境変数
</div>

これらの任意の環境変数を使用すると、リモートマシンでの認証設定などを行えます。

| 変数名                     | 用途                                                                                                                                                                                                              |
| ----------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `WANDB_API_KEY`         | アカウントに紐づく認証キーを設定します。APIキーは [User Settings](https://wandb.ai/settings) で作成してください。リモートマシンで `wandb login` を実行していない場合は、これを設定する必要があります。                                                                              |
| `WANDB_BASE_URL`        | [wandb/local](/ja/platform/hosting/) を利用している場合は、この環境変数を `http://YOUR_IP:YOUR_PORT` に設定してください                                                                                                                       |
| `WANDB_CACHE_DIR`       | デフォルトでは ~/.cache/wandb が使用されます。この環境変数で保存場所を上書きできます                                                                                                                                                              |
| `WANDB_CONFIG_DIR`      | デフォルトでは ~/.config/wandb が使用されます。この環境変数で保存場所を上書きできます                                                                                                                                                             |
| `WANDB_CONFIG_PATHS`    | wandb.config に読み込む YAML ファイルのカンマ区切りの一覧。[設定](./config#file-based-configs)を参照してください。                                                                                                                              |
| `WANDB_CONSOLE`         | stdout / stderr のログを無効にするには、これを &quot;off&quot; に設定します。デフォルトは、対応する環境では &quot;on&quot; です。                                                                                                                       |
| `WANDB_DATA_DIR`        | ステージング用Artifactsのアップロード先です。デフォルトの場所は、`platformdirs` Python パッケージの `user_data_dir` の値を使用するため、プラットフォームによって異なります。このディレクトリが存在し、実行ユーザーに書き込み権限があることを確認してください。                                                         |
| `WANDB_DIR`             | 生成されたすべてのファイルの保存先です。未設定の場合、トレーニングスクリプトを基準とした相対パス上の `wandb` ディレクトリがデフォルトになります。このディレクトリが存在し、実行するユーザーに書き込み権限があることを確認してください。これはダウンロードした artifacts の保存先は制御しません。artifacts の保存先は、`WANDB_ARTIFACT_DIR` 環境変数を使用して設定できます。 |
| `WANDB_ARTIFACT_DIR`    | ダウンロードした artifact をすべて保存する場所です。未設定の場合、トレーニングスクリプトからの相対パスにある `artifacts` ディレクトリがデフォルトで使用されます。このディレクトリが存在し、実行中のユーザーに書き込み権限があることを確認してください。これは生成されるメタデータファイルの保存場所は制御しません。保存場所は `WANDB_DIR` 環境変数で設定できます。             |
| `WANDB_DISABLE_GIT`     | wandb が git リポジトリを検出したり、最新の commit / diff を取得したりしないようにします。                                                                                                                                                      |
| `WANDB_DISABLE_CODE`    | wandb がノートブックや git の差分を保存しないようにするには、これを true に設定します。git リポジトリ内にいる場合でも、現在のコミットは保存されます。                                                                                                                           |
| `WANDB_DOCKER`          | これを Docker イメージのダイジェストに設定すると、run を復元できるようになります。`wandb docker` コマンドを使うと自動的に設定されます。イメージのダイジェストは、`wandb docker my/image/name:tag --digest` を実行すると取得できます                                                            |
| `WANDB_ENTITY`          | あなたのrunに関連付けられたentityです。トレーニングスクリプトのディレクトリで `wandb init` を実行すると、*wandb* という名前のディレクトリが作成され、ソース管理にチェックインできるデフォルトのentityが保存されます。そのファイルを作成したくない場合や、そのファイルを上書きしたい場合は、環境変数を使用できます。                                    |
| `WANDB_ERROR_REPORTING` | wandb が致命的なエラーをエラー追跡システムにログしないようにするには、これを false に設定します。                                                                                                                                                         |
| `WANDB_HOST`            | システムで提供されるホスト名を使用したくない場合は、wandb インターフェースに表示したいホスト名をこれに設定します。                                                                                                                                                    |
| `WANDB_IGNORE_GLOBS`    | 無視するファイルグロブのカンマ区切りリストをこれに設定します。これらのファイルはクラウドに Sync されません。                                                                                                                                                       |
| `WANDB_JOB_NAME`        | `wandb` で作成されるジョブ名を指定します。                                                                                                                                                                                       |
| `WANDB_JOB_TYPE`        | 異なるタイプのrunを示すために、&quot;training&quot; や &quot;evaluation&quot; などのジョブタイプを指定します。詳しくは [grouping](/ja/models/runs/grouping/) を参照してください。                                                                               |
| `WANDB_MODE`            | これを &quot;offline&quot; に設定すると、wandb は run のメタデータをローカルに保存し、サーバーとは Sync しません。これを `disabled` に設定すると、wandb は完全に無効化されます。                                                                                            |
| `WANDB_NAME`            | run の人が読みやすい名です。設定しない場合は、自動的にランダム生成されます                                                                                                                                                                         |
| `WANDB_NOTEBOOK_NAME`   | Jupyter で実行している場合は、この変数でノートブックの名を設定できます。これは自動検出されるようになっています。                                                                                                                                                    |
| `WANDB_NOTES`           | run に関する詳細なメモです。Markdown を使用でき、後から UI 上で編集することもできます。                                                                                                                                                            |
| `WANDB_PROJECT`         | あなたのrunに紐づくプロジェクトです。`wandb init`でも設定できますが、環境変数を設定するとその値が優先されます。                                                                                                                                                 |
| `WANDB_RESUME`          | デフォルトでは *never* に設定されています。*auto* に設定すると、wandb は失敗した run を自動的に再開します。*must* に設定すると、起動時にその run が存在している必要があります。独自の一意の ID を常に生成したい場合は、これを *allow* に設定し、`WANDB_RUN_ID` も必ず設定してください。                                   |
| `WANDB_RUN_GROUP`       | run を自動的に同じグループにまとめるための実験名を指定します。詳細は [grouping](/ja/models/runs/grouping/) を参照してください。                                                                                                                              |
| `WANDB_RUN_ID`          | これを、スクリプトの 1 回の実行に対応するグローバルに一意な文字列 (プロジェクトごと) に設定します。64 文字以内である必要があります。単語文字以外の文字はすべてダッシュに変換されます。障害発生時に既存の run を再開するために使用できます。                                                                                   |
| `WANDB_QUIET`           | これを `true` に設定すると、標準出力に出力されるメッセージは重大なもののみに制限されます。この設定を有効にすると、すべてのログは `$WANDB_DIR/debug.log` に書き込まれます。                                                                                                           |
| `WANDB_SILENT`          | これを `true` に設定すると、wandb のログ出力を無効にします。これはスクリプトで実行するコマンドで役立ちます。これを設定すると、すべてのログは `$WANDB_DIR/debug.log` に書き込まれます。                                                                                                  |
| `WANDB_SHOW_RUN`        | お使いのオペレーティングシステムが対応している場合、これを `true` に設定すると、run URL がブラウザーで自動的に開きます。                                                                                                                                            |
| `WANDB_SWEEP_ID`        | sweep ID のトラッキングを `Run` オブジェクトと関連クラスに追加し、UI に表示します。                                                                                                                                                             |
| `WANDB_TAGS`            | run に適用するタグのカンマ区切りリスト。                                                                                                                                                                                          |
| `WANDB_USERNAME`        | run に関連付けられた、チームメンバーのユーザー名。サービスアカウントのAPIキーとあわせて使用すると、自動化された run の実行者としてチームメンバーを割り当てられます。                                                                                                                        |
| `WANDB_USER_EMAIL`      | run に関連付けられた、チームメンバーのメールアドレス。サービスアカウントのAPIキーとあわせて使用すると、自動化された run の実行者としてチームメンバーを割り当てられます。                                                                                                                      |

<div id="singularity-environments">
  ## Singularity 環境
</div>

[Singularity](https://singularity.lbl.gov/index.html) でコンテナーを実行している場合は、上記の変数名の先頭に `SINGULARITYENV_` を付けることで、環境変数を渡せます。Singularity の環境変数の詳細は、[こちら](https://singularity.lbl.gov/docs-environment-metadata#environment)を参照してください。

<div id="running-on-aws">
  ## AWS で実行する
</div>

AWS でバッチジョブを実行している場合は、W&amp;B の認証情報を使ってマシンを簡単に認証できます。[User Settings](https://wandb.ai/settings) で APIキー を作成し、[AWS Batch ジョブ仕様](https://docs.aws.amazon.com/batch/latest/userguide/job_definition_parameters.html#parameters) で `WANDB_API_KEY` 環境変数を設定します。
