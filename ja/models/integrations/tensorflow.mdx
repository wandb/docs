<div id="get-started">
  ## はじめに
</div>

すでに TensorBoard を利用しているのであれば、簡単に wandb と統合できます。

```python
import tensorflow as tf
import wandb
```


<div id="log-custom-metrics">
  ## カスタムメトリクスをログする
</div>

TensorBoard にログされていない追加のカスタムメトリクスをログする必要がある場合は、コード内で `run.log()` を呼び出します。例えば、`run.log({"custom": 0.8})` のように記述します。

TensorBoard を同期している場合、`run.log()` の step 인수は無効になります。別の step カウントを設定したい場合は、次のように step を指定してメトリクスをログできます。

```python
with wandb.init(config=tf.flags.FLAGS, sync_tensorboard=True) as run:
    run.log({"custom": 0.8, "global_step":global_step}, step=global_step)
```


<div id="tensorflow-estimators-hook">
  ## TensorFlow Estimator フック
</div>

ログする内容をより細かく制御したい場合は、wandb では TensorFlow Estimator 用のフックも提供しています。このフックはグラフ内のすべての `tf.summary` の値をログします。

```python
import tensorflow as tf
import wandb

run = wandb.init(config=tf.FLAGS)

estimator.train(hooks=[wandb.tensorflow.WandbHook(steps_per_log=1000)])
run.finish()
```

<div id="log-manually">
  ## 手動でログする
</div>

TensorFlow でメトリクスをログする最も簡単な方法は、TensorFlow のロガーを使用して `tf.summary` をログすることです。

```python
import wandb
run = wandb.init(config=tf.flags.FLAGS, sync_tensorboard=True)
with tf.Session() as sess:
    # ...
    wandb.tensorflow.log(tf.summary.merge_all())
```

TensorFlow 2 では、カスタムループを使ってモデルをトレーニングするには、`tf.GradientTape` を使用する方法が推奨されています。詳細については [TensorFlow custom training walkthrough](https://www.tensorflow.org/tutorials/customization/custom_training_walkthrough) を参照してください。カスタム TensorFlow トレーニングループでメトリクスをログするために `wandb` を組み込む場合は、次のスニペットに従ってください。

```python
    with tf.GradientTape() as tape:
        # 確率を取得する
        predictions = model(features)
        # 損失を計算する
        loss = loss_func(labels, predictions)

    # メトリクスをログする
    run.log("loss": loss.numpy())
    # 勾配を取得する
    gradients = tape.gradient(loss, model.trainable_variables)
    # 重みを更新する
    optimizer.apply_gradients(zip(gradients, model.trainable_variables))
```

[TensorFlow 2 でトレーニングループをカスタマイズする完全な例](https://www.wandb.com/articles/wandb-customizing-training-loops-in-tensorflow-2) を利用できます。


<div id="how-is-wb-different-from-tensorboard">
  ## W&amp;B は TensorBoard とどう違いますか？
</div>

共同創業者たちが W&amp;B の開発を始めた当初、OpenAI で TensorBoard に不満を感じていたユーザーのためのツールを作ろうと考えました。ここでは、特に改善に注力してきた点をいくつかご紹介します。

1. **モデルの再現**: W&amp;B は、実験、探索、そして後からのモデル再現に適しています。メトリクスだけでなく、ハイパーパラメーターやコードのバージョンも記録し、バージョン管理の状態やモデル チェックポイントも保存できるため、プロジェクトの再現性を保てます。
2. **自動整理**: 共同作業者からプロジェクトを引き継ぐときでも、休暇明けに作業へ戻るときでも、昔のプロジェクトを再開するときでも、W&amp;B を使えば試したモデルをすべて簡単に確認できます。そのため、実験のやり直しに何時間も費やしたり、GPU リソースやカーボンフットプリントを無駄にしたりせずに済みます。
3. **高速で柔軟なインテグレーション**: W&amp;B は 5 分でプロジェクトに追加できます。無料のオープンソース Python パッケージをインストールして、コードに数行追加するだけで、モデルを実行するたびにメトリクスや記録がきれいにログされます。
4. **永続的で一元化されたダッシュボード**: ローカルマシン、共有ラボのクラスター、クラウド上のスポットインスタンスなど、どこでモデルをトレーニングしていても、結果はすべて同じ一元化されたダッシュボードに集約されます。異なるマシンから TensorBoard のファイルをコピーして整理するのに時間をかける必要はありません。
5. **強力なテーブル機能**: 異なるモデルの結果を検索、フィルター、ソート、グループ化できます。何千ものモデル バージョンを確認し、タスクごとに最も高性能なモデルを簡単に見つけられます。TensorBoard は大規模なプロジェクトでの利用に適した設計にはなっていません。
6. **コラボレーションのためのツール**: W&amp;B を使用して複雑な機械学習プロジェクトを整理できます。W&amp;B へのリンクは簡単に共有でき、プライベートな Teams を使って全員の結果を共有プロジェクトに送れます。また、Reports を使ったコラボレーションにも対応しており、インタラクティブな可視化を追加し、Markdown で作業内容を記述できます。これは、作業ログを残したり、指導教員や上司と知見を共有したり、ラボやチームに成果を発表したりするのに最適な方法です。
