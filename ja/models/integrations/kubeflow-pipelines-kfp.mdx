[Kubeflow Pipelines (kfp) ](https://www.kubeflow.org/docs/components/pipelines/overview/)は、Docker コンテナに基づいてポータブルでスケーラブルな機械学習 (ML) ワークフローを構築およびデプロイするためのプラットフォームです。

このインテグレーションを使うと、ユーザーは kfp の Python の関数型コンポーネントにデコレータを適用して、パラメータとアーティファクトを自動的に W&amp;B に記録できます。

この機能は `wandb==0.12.11` から利用可能で、`kfp<2.0.0` が必要です。

  ## サインアップしてAPIキーを作成する
</div>

APIキーは、マシンをW&amp;Bに対して認証するためのものです。APIキーは、ユーザープロフィールから生成できます。

<div id="decorate-your-components">
  ## コンポーネントにデコレータを追加する
</div>

`@wandb_log` デコレータを追加し、通常どおりコンポーネントを作成します。これにより、パイプラインを実行するたびに、入出力パラメータとアーティファクトが自動的に W&amp;B にログされます。

```python
from kfp import components
from wandb.integration.kfp import wandb_log


@wandb_log
def add(a: float, b: float) -> float:
    return a + b


add = components.create_component_from_func(add)
```

<div id="pass-environment-variables-to-containers">
  ## コンテナに環境変数を渡す
</div>

コンテナに[環境変数](/ja/models/track/environment-variables/)を明示的に渡す必要がある場合があります。双方向リンクを行うには、Kubeflow Pipelines インスタンスのベース URL を環境変数 `WANDB_KUBEFLOW_URL` に設定してください。たとえば `https://kubeflow.mysite.com` のような形式です。

```python
import os
from kubernetes.client.models import V1EnvVar


def add_wandb_env_variables(op):
    env = {
        "WANDB_API_KEY": os.getenv("WANDB_API_KEY"),
        "WANDB_BASE_URL": os.getenv("WANDB_BASE_URL"),
    }

    for name, value in env.items():
        op = op.add_env_variable(V1EnvVar(name, value))
    return op


@dsl.pipeline(name="example-pipeline")
def example_pipeline(param1: str, param2: int):
    conf = dsl.get_pipeline_conf()
    conf.add_op_transformer(add_wandb_env_variables)
```

<div id="concept-mapping-from-kubeflow-pipelines-to-wb">
  ### Kubeflow Pipelines から W&amp;B への概念マッピング
</div>

Kubeflow Pipelines の概念と W&amp;B との対応は次のとおりです。

| Kubeflow Pipelines | W&amp;B | W&amp;B 内での場所 |
| ------------------ | --- | --------------- |
| Input Scalar | [`config`](/ja/models/) | [Overview タブ](/ja/models/runs/#overview-tab) |
| Output Scalar | [`summary`](/ja/models/) | [Overview タブ](/ja/models/runs/#overview-tab) |
| Input Artifact | 入力アーティファクト | [Artifacts タブ](/ja/models/runs/#artifacts-tab) |
| Output Artifact | 出力アーティファクト | [Artifacts タブ](/ja/models/runs/#artifacts-tab) |

<div id="fine-grain-logging">
  ## 細かなログ記録
</div>

ログ記録をより細かく制御したい場合は、コンポーネント内の任意の箇所に `wandb.log` や `wandb.log_artifact` の呼び出しを挿入してください。

<div id="with-explicit-wandblog_artifacts-calls">
  ### 明示的な `wandb.log_artifacts` 呼び出しを使う場合
</div>

次の例では、モデルを学習しています。`@wandb_log` デコレータは、関連する入力と出力を自動的に追跡します。学習プロセスもログに記録したい場合は、次のように明示的にそのためのログ処理を追加できます：

```python
@wandb_log
def train_model(
    train_dataloader_path: components.InputPath("dataloader"),
    test_dataloader_path: components.InputPath("dataloader"),
    model_path: components.OutputPath("pytorch_model"),
):
    with wandb.init() as run:
        ...
        for epoch in epochs:
            for batch_idx, (data, target) in enumerate(train_dataloader):
                ...
                if batch_idx % log_interval == 0:
                    run.log(
                        {"epoch": epoch, "step": batch_idx * len(data), "loss": loss.item()}
                    )
            ...
            run.log_artifact(model_artifact)
```

<div id="with-implicit-wandb-integrations">
  ### 暗黙的な wandb インテグレーションを使用する場合
</div>

サポートされている[フレームワークのインテグレーション](/ja/models/integrations)を使用している場合は、コールバックを直接渡すこともできます。

```python
@wandb_log
def train_model(
    train_dataloader_path: components.InputPath("dataloader"),
    test_dataloader_path: components.InputPath("dataloader"),
    model_path: components.OutputPath("pytorch_model"),
):
    from pytorch_lightning.loggers import WandbLogger
    from pytorch_lightning import Trainer

    trainer = Trainer(logger=WandbLogger())
    ...  # 学習を実行する
```
