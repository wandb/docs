W&amp;B と DSPy を併用して、言語モデル プログラムをトラッキングおよび最適化します。W&amp;B は [Weave DSPy integration](/ja/weave/guides/integrations/dspy) を補完し、次の機能を提供します:

* 評価メトリクスの時系列トラッキング
* プログラムシグネチャの変化の追跡のための W&amp;B Tables
* MIPROv2 のような DSPy オプティマイザとのインテグレーション

DSPy モジュールを最適化する際に包括的な観測性を得るには、W&amp;B と Weave の両方でインテグレーションを有効にします。

<Info>
  **Note**

  `wandb==0.21.2` および `weave==0.52.5` 以降、W&amp;B と併用する場合は Weave が自動的に初期化されます:

  * `weave` をインポートしてから `wandb.init()` を呼び出した場合（スクリプトの場合）
  * `wandb.init()` を呼び出した後に `weave` をインポートした場合（notebook/Jupyter の場合）

  明示的な `weave.init(...)` 呼び出しは不要です。
</Info>

<div id="track-program-optimization-experimental">
  ## プログラム最適化の追跡（実験的）
</div>

`dspy.Evaluate`（MIPROv2 など）を使用する DSPy optimizer では、`WandbDSPyCallback` を使うと、時間経過に伴う評価メトリクスを記録し、W&amp;B Tables 上でプログラムシグネチャの変化を追跡できます。

```python
import dspy
from dspy.datasets import MATH

import weave
import wandb
from wandb.integration.dspy import WandbDSPyCallback

# W&Bを初期化（weaveをインポートするだけで十分。weave.initの明示的な呼び出しは不要）
project_name = "dspy-optimization"
with wandb.init(project=project_name) as run:
    # W&BコールバックをDSPyに追加
    dspy.settings.callbacks.append(
        WandbDSPyCallback(run=run)
    )

    # 言語モデルを設定
    teacher_lm = dspy.LM('openai/gpt-4o', max_tokens=2000, cache=True)
    student_lm = dspy.LM('openai/gpt-4o-mini', max_tokens=2000)
    dspy.configure(lm=student_lm)

    # データセットを読み込み、プログラムを定義
    dataset = MATH(subset='algebra')
    program = dspy.ChainOfThought("question -> answer")

    # オプティマイザーを設定して実行
    optimizer = dspy.MIPROv2(
        metric=dataset.metric,
        auto="light",
        num_threads=24,
        teacher_settings=dict(lm=teacher_lm),
        prompt_model=student_lm
    )

    optimized_program = optimizer.compile(
        program,
        trainset=dataset.train,
        max_bootstrapped_demos=2,
        max_labeled_demos=2
    )
```

このコードを実行すると、W&amp;B の run URL と Weave URL の両方が得られます。W&amp;B では、時間経過に伴う評価メトリクスに加えて、プログラムシグネチャの変化を示す Tables も表示されます。run の **Overview** タブには、詳細な確認用の Weave トレースへのリンクが含まれます。

`run` オブジェクトが `WandbDSPyCallback` に渡されない場合は、グローバルな `run` オブジェクトが使用されます。

<div id="log-predictions-to-wb-tables">
  ## 予測を W&amp;B Tables に記録する
</div>

詳細な予測ログを有効にして、最適化中に各サンプルを確認できるようにします。コールバックは各評価ステップごとに W&amp;B Tables を作成し、個々の成功例や失敗例を分析するのに役立ちます。

```python
from wandb.integration.dspy import WandbDSPyCallback

# 予測ログを有効化（デフォルトで有効）
callback = WandbDSPyCallback(log_results=True)
dspy.settings.callbacks.append(callback)

# 最適化を実行
optimized_program = optimizer.compile(program, trainset=train_data)

# 必要に応じて予測ログを無効化
# callback = WandbDSPyCallback(log_results=False)
```

<div id="access-prediction-data">
  ### 予測データにアクセスする
</div>

最適化が完了したら、W&amp;B で予測データを確認します：

1. 対象の run の **Overview** ページに移動します。
2. `predictions_0`、`predictions_1` などのパターンの名前が付いた Table パネルを探します。
3. `is_correct` でフィルタして失敗ケースを分析します。
4. プロジェクトのワークスペース内で、run 間のテーブルを比較します。

各テーブルには次の列が含まれます：

* `example`: 入力データ
* `prediction`: モデル出力
* `is_correct`: 評価結果

詳細については [W&amp;B Tables ガイド](/ja/models/tables/visualize-tables/) を参照してください。

<div id="save-and-version-dspy-programs">
  ## DSPy プログラムの保存とバージョン管理
</div>

最良の DSPy プログラムを再現およびバージョン管理するには、W&amp;B Artifacts として保存してください。プログラム全体を保存するか、状態のみを保存するかを選択できます。

```python
from wandb.integration.dspy import WandbDSPyCallback

# コールバックインスタンスを作成
callback = WandbDSPyCallback()
dspy.settings.callbacks.append(callback)

# 最適化を実行
optimized_program = optimizer.compile(program, trainset=train_data)

# 保存オプション:

# 1. 完全なプログラム（推奨）- アーキテクチャと状態を含む
callback.log_best_model(optimized_program, save_program=True)

# 2. JSONとして状態のみ - 軽量で人間が読みやすい
callback.log_best_model(optimized_program, save_program=False, filetype="json")

# 3. pickleとして状態のみ - Pythonオブジェクトを保持
callback.log_best_model(optimized_program, save_program=False, filetype="pkl")

# バージョン管理用のカスタムエイリアスを追加
callback.log_best_model(
    optimized_program,
    save_program=True,
    aliases=["best", "production", "v2.0"]
)
```
