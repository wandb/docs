<div id="overview">
  ## 概要
</div>

[Metaflow](https://docs.metaflow.org) は、Netflix が開発した、ML ワークフローを作成して実行するためのフレームワークです。

このインテグレーションを使用すると、Metaflow の [steps と フロー](https://docs.metaflow.org/metaflow/basics) にデコレーターを適用して、パラメーターと Artifacts を W&amp;B に自動的にログできます。

* step をデコレートすると、その step 内の特定のタイプについてログのオン/オフを切り替えられます。
* フロー をデコレートすると、その フロー 内のすべての step についてログのオン/オフを切り替えられます。

<div id="decorate-your-flows-and-steps">
  ### フローやstepをデコレートする
</div>

<Tabs>
  <Tab title="step">
    stepをデコレートすると、そのstep内で特定のタイプに対するログをオンまたはオフにできます。

    この例では、`start` 内のすべてのデータセットとモデルがログされます。

    ```python
    from wandb.integration.metaflow import wandb_log

    class WandbExampleFlow(FlowSpec):
        @wandb_log(datasets=True, models=True, settings=wandb.Settings(...))
        @step
        def start(self):
            self.raw_df = pd.read_csv(...).    # pd.DataFrame -> データセットとしてアップロード
            self.model_file = torch.load(...)  # nn.Module    -> モデルとしてアップロード
            self.next(self.transform)
    ```
  </Tab>

  <Tab title="フロー">
    フローをデコレートすることは、そのフローを構成するすべてのstepをデフォルト設定でデコレートするのと同等です。

    この場合、`WandbExampleFlow` 内のすべてのstepは、各stepを `@wandb_log(datasets=True, models=True)` でデコレートした場合と同様に、デフォルトでデータセットとモデルをログします。

    ```python
    from wandb.integration.metaflow import wandb_log

    @wandb_log(datasets=True, models=True)  # すべての @step をデコレート
    class WandbExampleFlow(FlowSpec):
        @step
        def start(self):
            self.raw_df = pd.read_csv(...).    # pd.DataFrame -> データセットとしてアップロード
            self.model_file = torch.load(...)  # nn.Module    -> モデルとしてアップロード
            self.next(self.transform)
    ```
  </Tab>

  <Tab title="フローとstep">
    フローをデコレートすることは、すべてのstepをデフォルト設定でデコレートするのと同等です。つまり、後から特定のstepに別の `@wandb_log` を付けると、フローレベルのデコレーションを上書きします。

    この例では:

    * `start` と `mid` はデータセットとモデルの両方をログします。
    * `end` はデータセットもモデルもログしません。

    ```python
    from wandb.integration.metaflow import wandb_log

    @wandb_log(datasets=True, models=True)  # start と mid をデコレートするのと同じ
    class WandbExampleFlow(FlowSpec):
      # このstepはデータセットとモデルをログします
      @step
      def start(self):
        self.raw_df = pd.read_csv(...).    # pd.DataFrame -> データセットとしてアップロード
        self.model_file = torch.load(...)  # nn.Module    -> モデルとしてアップロード
        self.next(self.mid)

      # このstepもデータセットとモデルをログします
      @step
      def mid(self):
        self.raw_df = pd.read_csv(...).    # pd.DataFrame -> データセットとしてアップロード
        self.model_file = torch.load(...)  # nn.Module    -> モデルとしてアップロード
        self.next(self.end)

      # このstepの設定は上書きされ、データセットもモデルもログしません
      @wandb_log(datasets=False, models=False)
      @step
      def end(self):
        self.raw_df = pd.read_csv(...).    
        self.model_file = torch.load(...)
    ```
  </Tab>
</Tabs>

<div id="access-your-data-programmatically">
  ## データにプログラムで액세スする
</div>

取得した情報には、3 つの方法でアクセスできます。ログされている元の Python プロセス内で [`wandb` client library](/ja/models/ref/python/) を使用する方法、[web app UI](/ja/models/track/workspaces/) を使用する方法、または [our Public API](/ja/models/ref/python/public-api/) を使用してプログラムからアクセスする方法です。`パラメーター` は W&amp;B の [`設定`](/ja/models/) に保存され、[Overview タブ](/ja/models/runs/#overview-tab) で確認できます。`データセット`、`モデル`、`others` は [W&amp;B Artifacts](/ja/models/artifacts/) に保存され、[Artifacts タブ](/ja/models/runs/#artifacts-tab) で確認できます。基本的な python の型は W&amp;B の [`summary`](/ja/models/) dict に保存され、Overview タブで確認できます。この情報を外部からプログラムで取得するために API を使用する方法の詳細については、[guide to the Public API](/ja/models/track/public-api-guide/) を参照してください。

</div>

  ### `wandb_log` kwargs
</div>

| kwarg      | Options                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| ---------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `datasets` | <ul><li><code>True</code>: データセットであるインスタンス変数をログする</li><li><code>False</code></li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                   |
| `models`   | <ul><li><code>True</code>: モデルであるインスタンス変数をログする</li><li><code>False</code></li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                      |
| `others`   | <ul><li><code>True</code>: その他のうち、pickle としてシリアライズ可能なものをすべてログする</li><li><code>False</code></li></ul>                                                                                                                                                                                                                                                                                                                                                                                                     |
| `settings` | <ul><li><code>wandb.Settings(...)</code>: この step または フロー 用に独自の <code>wandb</code> 設定を指定する</li><li><code>None</code>: <code>wandb.Settings()</code> を渡す場合と同等</li></ul><p>デフォルトでは、以下の場合:</p><ul><li><code>settings.run&#95;group</code> が <code>None</code> の場合、<code>&#123;flow&#95;name&#125;/&#123;run&#95;id&#125;</code> に設定される</li><li><code>settings.run&#95;job&#95;type</code> が <code>None</code> の場合、<code>&#123;run&#95;job&#95;type&#125;/&#123;step&#95;name&#125;</code> に設定される</li></ul> |

  ## よくある質問

<div id="what-exactly-do-you-log-do-you-log-all-instance-and-local-variables">
  ### 具体的に何がログされますか？インスタンス変数とローカル変数はすべてログされますか？
</div>

`wandb_log` はインスタンス変数のみをログします。ローカル変数がログされることは一切ありません。これは不要なデータがログに記録されるのを避けるのに役立ちます。

<div id="which-data-types-get-logged">
  ### どのデータタイプがログ対象ですか？
</div>

現在、以下のタイプをサポートしています。

| ログ設定           | タイプ                                                                                                              |
| -------------- | ---------------------------------------------------------------------------------------------------------------- |
| default (常に有効) | <ul><li><code>dict, list, set, str, int, float, bool</code></li></ul>                                            |
| `datasets`     | <ul><li><code>pd.DataFrame</code></li><li><code>pathlib.Path</code></li></ul>                                    |
| `models`       | <ul><li><code>nn.Module</code></li><li><code>sklearn.base.BaseEstimator</code></li></ul>                         |
| `others`       | <ul><li><a href="https://wiki.python.org/moin/UsingPickle">pickle 化可能</a> で、かつ JSON シリアライズ可能な任意のオブジェクト</li></ul> |

<div id="how-can-i-configure-logging-behavior">
  ### ロギングの動作はどのように設定できますか？
</div>

| 変数の種類    | 動作                             | 例               | データタイプ         |
| -------- | ------------------------------ | --------------- | -------------- |
| Instance | 自動的にログに記録される                   | `self.accuracy` | `float`        |
| Instance | `datasets=True` の場合にログに記録される   | `self.df`       | `pd.DataFrame` |
| Instance | `datasets=False` の場合はログに記録されない | `self.df`       | `pd.DataFrame` |
| Local    | ログに記録されない                      | `accuracy`      | `float`        |
| Local    | ログに記録されない                      | `df`            | `pd.DataFrame` |

<div id="is-artifact-lineage-tracked">
  ### Artifacts のリネージはトラッキングされますか？
</div>

はい。step A の出力であり、step B の入力でもある Artifacts がある場合、リネージ DAG が自動的に構築されます。
