[Meta AI の MMF](https://github.com/facebookresearch/mmf) ライブラリの `WandbLogger` クラスを使うと、学習/検証メトリクス、システム（GPU および CPU）のメトリクス、モデルのチェックポイント、および設定パラメータを W&B に記録できるようになります。

<div id="current-features">
  ## 現在の機能
</div>

次の機能が MMF の `WandbLogger` によって現在サポートされています:

* 学習および検証メトリクス
* 時間経過に伴う学習率
* モデルのチェックポイントを W&B Artifacts に保存
* GPU および CPU のシステムメトリクス
* 学習設定のパラメータ

<div id="config-parameters">
  ## Config parameters
</div>

MMF の config で wandb へのログ出力を有効化し、カスタマイズするために、次のオプションが用意されています。

```
training:
    wandb:
        enabled: true
        
        # エンティティとは、runの送信先となるユーザー名またはチーム名です。
        # デフォルトでは、ユーザーアカウントにrunが記録されます。
        entity: null
        
        # wandbで実験を記録する際に使用するプロジェクト名
        project: mmf
        
        # wandbでプロジェクト配下の実験を記録する際に使用する
        # 実験名／run名。デフォルトの実験名:
        # ${training.experiment_name}
        name: ${training.experiment_name}
        
        # モデルのチェックポイント保存を有効にし、チェックポイントをW&B Artifactsに保存する
        log_model_checkpoint: true
        
        # wandb.init()に渡す追加の引数値（例）:
        # job_type: 'train'
        # tags: ['tag1', 'tag2']
        
env:
    # wandbメタデータを保存するディレクトリのパスを変更する場合
    # （デフォルト: env.log_dir）:
    wandb_logdir: ${env:MMF_WANDB_LOGDIR,}
```
