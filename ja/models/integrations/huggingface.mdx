<div id="why-should-i-use-wb">
  ## なぜ W&amp;B を使うべきですか？
</div>

<Frame>
  <img src="/images/tutorials/huggingface-why.png" alt="W&B を使う利点" />
</Frame>

* **Unified ダッシュボード**: すべてのモデルのメトリクスと予測を一元的に管理するリポジトリ
* **軽量**: Hugging Face とのインテグレーションにコード変更は不要
* **利用しやすい**: 個人および学術チームは無料
* **安全**: すべてのプロジェクトはデフォルトで非公開
* **信頼性**: OpenAI、Toyota、Lyft などの機械学習チームで利用されています

W&amp;B は、機械学習モデル向けの GitHub のようなものです。機械学習の実験を、非公開のホスト型ダッシュボードに保存できます。スクリプトをどこで実行していても、モデルのすべてのバージョンが保存されるため、安心してすばやく実験できます。

W&amp;B の軽量なインテグレーションは、あらゆる Python スクリプトで動作します。必要なのは、無料の W&amp;B アカウントにサインアップすることだけで、すぐにモデルのトラッキングと可視化を開始できます。

Hugging Face Transformers リポジトリでは、`Trainer` に計装を追加しており、各ログステップでトレーニングおよび評価のメトリクスを自動的に W&amp;B にログします。

インテグレーションの仕組みについて詳しくは、こちらをご覧ください: [Hugging Face + W&amp;B Report](https://app.wandb.ai/jxmorris12/huggingface-demo/reports/Train-a-model-with-Hugging-Face-and-Weights-%26-Biases--VmlldzoxMDE2MTU)。

<div id="train-the-model">
  ## モデルをトレーニングする
</div>

次に、ダウンロードしたトレーニングスクリプト [run&#95;glue.py](https://huggingface.co/transformers/examples.html#glue) を実行し、トレーニングが W&amp;B ダッシュボードに自動的にトラッキングされることを確認します。このスクリプトは、Microsoft Research Paraphrase Corpus (文のペアに、それらが意味的に等価かどうかを示す人によるアノテーションが付いたコーパス) で BERT をファインチューニングします。

```python
%env WANDB_PROJECT=huggingface-demo
%env TASK_NAME=MRPC

!python run_glue.py \
  --model_name_or_path bert-base-uncased \
  --task_name $TASK_NAME \
  --do_train \
  --do_eval \
  --max_seq_length 256 \
  --per_device_train_batch_size 32 \
  --learning_rate 2e-4 \
  --num_train_epochs 3 \
  --output_dir /tmp/$TASK_NAME/ \
  --overwrite_output_dir \
  --logging_steps 50
```

<div id="track-key-information-effortlessly-by-default">
  ## デフォルトで重要な情報を自動的にトラッキングする
</div>

W&amp;B は、実験ごとに新しい run を保存します。デフォルトで保存される情報は以下の通りです：

* **Hyperparameters**: モデルの設定が Config に保存されます
* **Model Metrics**: ストリーミングされるメトリクスの時系列データが Log に保存されます
* **Terminal Logs**: コマンドラインの出力が保存され、タブで確認できます
* **System Metrics**: GPU と CPU の使用率、メモリ、温度などが保存されます
