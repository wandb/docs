<div id="docker-integration">
  ## Docker インテグレーション
</div>

W&amp;B は、コードが実行された Docker イメージへのポインタを保存できるため、以前の実験を、実行されたのとまったく同じ環境に復元できます。wandb ライブラリは、この状態を永続化するために **WANDB&#95;DOCKER** 環境変数を参照します。W&amp;B では、この状態を自動的に設定するいくつかのヘルパーを提供しています。

<div id="local-development">
  ### ローカル開発
</div>

`wandb docker` は、Docker コンテナーを起動し、wandb の環境変数を渡し、コードをマウントして、wandb がインストールされていることを確認するコマンドです。デフォルトでは、このコマンドは TensorFlow、PyTorch、Keras、Jupyter がインストールされた Docker イメージを使用します。同じコマンドで独自の Docker イメージを起動することもできます：`wandb docker my/image:latest`。このコマンドは、現在のディレクトリをコンテナーの &quot;/app&quot; ディレクトリにマウントします。変更するには `--dir` フラグを使用してください。

<div id="production">
  ### 本番
</div>

`wandb docker-run` コマンドは本番ワークロード向けに提供されています。`nvidia-docker` のドロップイン代替として設計されており、認証情報と **WANDB&#95;DOCKER** 環境変数を Call に追加する `docker run` コマンドのシンプルなラッパーです。&quot;--runtime&quot; フラグを指定せず、マシン上で `nvidia-docker` が利用可能な場合は、ランタイムが nvidia に設定されることも保証されます。

<div id="kubernetes">
  ### Kubernetes
</div>

Kubernetes でトレーニング ワークロードを実行しており、k8s API が pod に公開されている場合 (デフォルトです) 、wandb は Docker イメージのダイジェストを取得するために API をクエリし、自動的に **WANDB&#95;DOCKER** 環境変数を設定します。

<div id="restoring">
  ## 復元
</div>

runに **WANDB&#95;DOCKER** 環境変数が設定されている場合、`wandb restore username/project:run_id` を呼び出すと、新しいブランチをチェックアウトしてコードを復元し、元のコマンドが事前に入力された状態で、トレーニングに使用した正確なDocker imageを起動します。
