title: 概要
---

*run* は、W&amp;B によってログされる 1 回分の計算処理です。W&amp;B の run は、プロジェクト全体を構成する最小単位の要素と考えることができます。言い換えると、それぞれの run は「特定の計算処理」の記録であり、たとえばモデルの学習とその結果のログ、ハイパーパラメータスイープなどが該当します。

run を初期化してログを記録する、代表的なユースケースは次のとおりです。

* モデルを学習し、精度や損失などの[メトリクスを記録する](/ja/models/ref/python/experiments/run#method-run-log)
* [ハイパーパラメータチューニング](/ja/models/sweeps/)を実行して新しい実験を走らせる
* 異なるモデルを用いた新しい機械学習実験を行う
* データセットやモデルを [W&amp;B Artifacts](/ja/models/artifacts/) として追跡および保存する
* チームの他のメンバーが W&amp;B Artifacts として保存しているデータセットやモデルを[ダウンロードして利用する](/ja/models/artifacts/download-and-use-an-artifact/)

W&amp;B の run を初期化するには、W&amp;B Python SDK から [`wandb.init()`](/ja/models/ref/python/functions/init) メソッドを呼び出します。これにより新しい run が開始され、メトリクスやアーティファクト、その他の情報をその run にログするために使用できる `wandb.Run` オブジェクトが返されます。run の初期化について詳しくは、[Initialize runs](/ja/models/runs/initialize-run) を参照してください。

各 run オブジェクトには、[*run ID* と呼ばれる一意の識別子](/ja/models/runs/run-identifiers#unique-run-identifiers)が付与されます。[一意の ID を自分で指定してもよいですし](/ja/models/runs/run-identifiers#unique-run-identifiers)、[W&amp;B にランダムに生成させることもできます](/ja/models/runs/run-identifiers#autogenerated-run-ids)。各 run オブジェクトには、ユーザーが読みやすい非一意の [run name](/ja/models/runs/run-identifiers#run-name) もあります。run の名前も、自分で指定するか、W&amp;B にランダムに生成させることができます。run の初期化後に名前を変更することも可能です。

W&amp;B は、run を[*プロジェクト*](/ja/models/track/project-page/)にログします。`wandb.init(project="")` で run を初期化する際に、プロジェクトを指定します。指定したプロジェクトが存在しない場合、W&amp;B は新しいプロジェクトを作成します。すでに存在する場合は、そのプロジェクトに run をログします。

<Note>
  プロジェクト名を指定しない場合、W&amp;B は run を `Uncategorized` というプロジェクトに保存します。
</Note>

`wandb.init()` は、その ID、名前、設定、状態など、run のプロパティを含む `wandb.Run` オブジェクトを返します。`wandb.Run.log()`、`wandb.Run.log_code()`、`wandb.Run.use_artifact()` などのメソッドを使って、この run オブジェクトに対してメトリクス、アーティファクト、その他の情報をログします。

各 run には、その時点の状態を表すステートがあります。取りうる run のステート一覧については、[Run states](/ja/models/runs/run-states) を参照してください。

W&amp;B App では、run の属するプロジェクトの Workspace 内で、[run とそのプロパティを確認できます](/ja/models/runs/view-logged-runs)。また、[`wandb.Api.Run`](/ja/models/ref/python/experiments/run) オブジェクトを使用して、プログラムから run のプロパティにアクセスすることもできます。

次のコードスニペットは、W&amp;B の run を初期化し、いくつかのメトリクスをログする例です。

<Note>
  以降のコードスニペットを実際に試したい場合は、`entity` 変数にあなたの W&amp;B エンティティを指定してください。エンティティはあなたの W&amp;B ユーザー名またはチーム名で、W&amp;B App のワークスペース URL に含まれています。たとえばワークスペースの URL が `https://wandb.ai/nico/awesome-project` の場合、エンティティは `nico` です。
</Note>

```python
import wandb

entity = "nico"  # あなたの W&B エンティティに置き換えてください
project = "awesome-project"

with wandb.init(entity=entity, project=project) as run:
    run.log({"accuracy": 0.9, "loss": 0.1})
```

最初の行では、W&amp;B の Python SDK をインポートしています。2 行目では、エンティティ `nico` 配下のプロジェクト `awesome-project` で run を初期化しています。3 行目では、その run にモデルの正解率 (accuracy) と損失 (loss) を記録しています。

単一の時点でメトリクスを 1 回だけ記録しても、それほど有用とは限りません。識別モデルを学習する場合の、より現実的な例としては、メトリクスを一定間隔で記録することが挙げられます。例えば、次のコードスニペットを見てください。

```python
import wandb
import random

config = {
    "epochs": 10,
    "learning_rate": 0.01,
}

with wandb.init(project="awesome-project", config=config) as run:
    print(f"lr: {config['learning_rate']}")
      
    # 学習runをシミュレートする
    for epoch in range(config['epochs']):
      offset = random.random() / 5
      acc = 1 - 2**-epoch - random.random() / (epoch + 1) - offset
      loss = 2**-epoch + random.random() / (epoch + 1) + offset
      print(f"epoch={epoch}, accuracy={acc}, loss={loss}")
      run.log({"accuracy": acc, "loss": loss})
```

学習スクリプトは `wandb.Run.log()` を 10 回呼び出します。スクリプトが `wandb.Run.log()` を呼び出すたびに、W&amp;B はそのエポックの精度と損失をログに記録します。
