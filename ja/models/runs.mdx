*run* は、W&amp;B によってログされる単一の計算単位です。W&amp;B Run は、プロジェクト全体の原子的な要素と考えることができます。言い換えると、各 run は、モデルのトレーニングと結果のログ、ハイパーパラメータースイープなど、特定の計算の記録です。

run を初期化してログする一般的なユースケースには、次のようなものがあります。

* モデルをトレーニングし、精度や損失などの[メトリクスを記録する](/ja/models/ref/python/experiments/run#method-run-log)
* [ハイパーパラメーターチューニング](/ja/models/sweeps/)を実施し、新しい Experiments を実行する
* 異なるモデルで新しい機械学習 experiment を実施する
* データセットとモデルを [W&amp;B Artifacts](/ja/models/artifacts/) としてトラッキングして保存する
* チームの他のメンバーが W&amp;B Artifacts として使用したデータセットやモデルを[ダウンロードして使用する](/ja/models/artifacts/download-and-use-an-artifact/)

W&amp;B run を初期化するには、W&amp;B Python SDK の [`wandb.init()`](/ja/models/ref/python/functions/init) method を呼び出します。これにより新しい run が開始され、メトリクス、アーティファクト、その他の情報を run にログするために使用できる `wandb.Run` object が返されます。run の初期化の詳細については、[Initialize runs](/ja/models/runs/initialize-run) を参照してください。

各 run object には、[*run ID*](/ja/models/runs/run-identifiers#unique-run-identifiers) として知られる[一意の識別子](/ja/models/runs/run-identifiers#unique-run-identifiers)があります。[一意の ID を指定する](/ja/models/runs/run-identifiers#unique-run-identifiers)ことも、[W&amp;B にランダムに生成させる](/ja/models/runs/run-identifiers#autogenerated-run-ids)こともできます。各 run object には、人間が読める一意ではない [run 名](/ja/models/runs/run-identifiers#run-name) もあります。run の名を指定することも、W&amp;B にランダムに生成させることもできます。初期化後に run の名を変更することもできます。

W&amp;B は run を [*プロジェクト*](/ja/models/track/project-page/) にログします。`wandb.init(project="")` で run を初期化するときにプロジェクトを指定します。プロジェクトが存在しない場合、W&amp;B は新しいプロジェクトを作成します。プロジェクトが存在する場合、W&amp;B は指定したプロジェクトに run をログします。

<Note>
  プロジェクト名を指定しない場合、W&amp;B は `Uncategorized` という名前のプロジェクトに run を保存します。
</Note>

`wandb.init()` は、ID、名、設定、state など、run のプロパティを含む `wandb.Run` object を返します。`wandb.Run.log()`、`wandb.Run.log_code()`、`wandb.Run.use_artifact()` などの methods を使用して、run object からメトリクス、アーティファクト、その他の情報を run にログします。

各 run には、run の現在のステータスを表す state があります。可能な run state の一覧については、[Run states](/ja/models/runs/run-states) を参照してください。

W&amp;B App の run のワークスペース内で、[runs とそのプロパティを表示](/ja/models/runs/view-logged-runs)できます。また、[`wandb.Api.Run`](/ja/models/ref/python/experiments/run) object を使用して、プログラムから run のプロパティに 액세スすることもできます。

例として、W&amp;B run を初期化し、いくつかのメトリクスをその run にログする次の 코드 스니펫を考えてみましょう。

<Note>
  一緒に進めたい場合は、以下の코드 스니펫内の `entity` 変数に W&amp;B entity を渡してください。entity は W&amp;B のユーザー名またはチーム名です。これは W&amp;B App Workspace の URL で確認できます。たとえば、Workspace の URL が `https://wandb.ai/nico/awesome-project` の場合、entity は `nico` です。
</Note>

```python
import wandb

entity = "nico"  # W&B の entity に置き換えてください
project = "awesome-project"

with wandb.init(entity=entity, project=project) as run:
    run.log({"accuracy": 0.9, "loss": 0.1})
```

1 行目では W&amp;B Python SDK をインポートします。2 行目では、entity `nico` のプロジェクト `awesome-project` で run を初期化します。3 行目では、その run にモデルの accuracy と loss をログします。

単一の時点でメトリクスをログしても、それほど有用ではない場合があります。より現実的な例として、識別モデルのトレーニングでは、一定の間隔でメトリクスをログします。たとえば、次の코드 스니펫を考えてみてください。

```python
import wandb
import random

config = {
    "epochs": 10,
    "learning_rate": 0.01,
}

with wandb.init(project="awesome-project", config=config) as run:
    print(f"lr: {config['learning_rate']}")
      
    # トレーニング run をシミュレート
    for epoch in range(config['epochs']):
      offset = random.random() / 5
      acc = 1 - 2**-epoch - random.random() / (epoch + 1) - offset
      loss = 2**-epoch + random.random() / (epoch + 1) + offset
      print(f"epoch={epoch}, accuracy={acc}, loss={loss}")
      run.log({"accuracy": acc, "loss": loss})
```

トレーニングスクリプトは `wandb.Run.log()` を 10 回呼び出します。スクリプトが `wandb.Run.log()` を呼び出すたびに、W&amp;B はそのエポックの accuracy と loss をログします。
