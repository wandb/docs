---

*run* とは、W&amp;B によってログされる 1 回分の計算処理の単位です。W&amp;B の run は、プロジェクト全体を構成する原子的な要素と考えることができます。つまり、各 run は「モデルをトレーニングして結果をログする」「ハイパーパラメーター sweep を実行する」など、特定の計算処理 1 回分の記録です。

run を初期化し、そこにログを記録する代表的なユースケースは次のとおりです。

* モデルをトレーニングし、精度や損失などの [メトリクスを記録する](/ja/models/ref/python/experiments/run#method-run-log)
* [ハイパーパラメーター チューニング](/ja/models/sweeps/) を行い、新しい実験を実行する
* 異なるモデルを用いた新しい機械学習実験を行う
* データセットやモデルを [W&amp;B Artifacts](/ja/models/artifacts/) として追跡および保存する
* チームの他のメンバーが W&amp;B Artifacts として保存しているデータセットやモデルを [ダウンロードして使用する](/ja/models/artifacts/download-and-use-an-artifact/)

W&amp;B run を初期化するには、W&amp;B Python SDK から [`wandb.init()`](/ja/models/ref/python/functions/init) メソッドを呼び出します。これにより新しい run が開始され、メトリクスや Artifacts 、その他の情報をその run にログするために使用できる `wandb.Run` オブジェクトが返されます。run の初期化の詳細については、[Initialize runs](/ja/models/runs/initialize-run) を参照してください。

各 run オブジェクトには、[*run ID* と呼ばれる一意の識別子](/ja/models/runs/run-identifiers#unique-run-identifiers) が付与されます。[一意の ID を自分で指定する](/ja/models/runs/run-identifiers#unique-run-identifiers) ことも、[W&amp;B にランダムに生成させる](/ja/models/runs/run-identifiers#autogenerated-run-ids) こともできます。各 run オブジェクトには、人間が読める形式だが一意ではない [run name](/ja/models/runs/run-identifiers#run-name) もあります。run に任意の名前を付けることも、W&amp;B にランダムに生成させることもできます。run の初期化後に、run の名前を変更することも可能です。

W&amp;B は run を [*project*](/ja/models/track/project-page/) にログします。`wandb.init(project="")` を使って run を初期化する際に、どの project にログするかを指定します。指定した project が存在しない場合、W&amp;B は新しい project を作成します。既に存在する場合は、その project に run をログします。

<Note>
  プロジェクト名を指定しない場合、W&amp;B は run を `Uncategorized` という名前のプロジェクトに保存します。
</Note>

`wandb.init()` は、その run の ID、名前、設定、状態といったプロパティを含む `wandb.Run` オブジェクトを返します。`wandb.Run.log()`、`wandb.Run.log_code()`、`wandb.Run.use_artifact()` などのメソッドを使って、この run オブジェクトにメトリクスや Artifacts 、その他の情報をログします。

各 run には、その run の現在の状況を表す状態が設定されています。取り得る run の状態の一覧については、[Run states](/ja/models/runs/run-states) を参照してください。

[run とそのプロパティは](/ja/models/runs/view-logged-runs)、W&amp;B App 上のその run の project Workspace 内で確認できます。[`wandb.Api.Run`](/ja/models/ref/python/experiments/run) オブジェクトを使って、プログラムから run のプロパティにアクセスすることもできます。

次のコードスニペットは、W&amp;B run を初期化し、その run にいくつかのメトリクスをログする例です。

<Note>
  以降のコードスニペットで実際に試してみたい場合は、`entity` 変数に自分の W&amp;B entity を渡してください。entity は、あなたの W&amp;B ユーザー名またはチーム名です。自分の entity は、W&amp;B App の Workspace の URL から確認できます。たとえば、Workspace の URL が `https://wandb.ai/nico/awesome-project` の場合、entity は `nico` です。
</Note>

```python
import wandb

entity = "nico"  # W&B の entity を置き換えてください
project = "awesome-project"

with wandb.init(entity=entity, project=project) as run:
    run.log({"accuracy": 0.9, "loss": 0.1})
```

最初の行では W&amp;B Python SDK をインポートします。2 行目では、エンティティ `nico` 配下の project `awesome-project` 内で run を初期化します。3 行目では、その run に対してモデルの accuracy と loss をログします。

ある一時点だけで 1 つのメトリクスを記録しても、それほど有用ではないかもしれません。識別モデルをトレーニングする場合の、より現実的な例としては、一定間隔でメトリクスを記録することが挙げられます。例えば、次のコードスニペットを考えてみましょう。

```python
import wandb
import random

config = {
    "epochs": 10,
    "learning_rate": 0.01,
}

with wandb.init(project="awesome-project", config=config) as run:
    print(f"lr: {config['learning_rate']}")
      
    # トレーニング run をシミュレート
    for epoch in range(config['epochs']):
      offset = random.random() / 5
      acc = 1 - 2**-epoch - random.random() / (epoch + 1) - offset
      loss = 2**-epoch + random.random() / (epoch + 1) + offset
      print(f"epoch={epoch}, accuracy={acc}, loss={loss}")
      run.log({"accuracy": acc, "loss": loss})
```

トレーニングスクリプトは `wandb.Run.log()` を 10 回呼び出します。スクリプトが `wandb.Run.log()` を呼び出すたびに、W&amp;B はそのエポックの精度と損失をログに記録します。
