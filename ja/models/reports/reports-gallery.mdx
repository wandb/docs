## Notes: クイックサマリーで可視化を追加する

重要な観察事項、将来の作業へのアイデア、またはプロジェクト開発で達成されたマイルストーンを記録します。 Reports 内のすべての実験 run は、その パラメータ 、 メトリクス 、 ログ 、および コード にリンクされるため、作業の完全なコンテキストを保存できます。

テキストを書き留め、関連するチャートを取り込んで、あなたの洞察を説明しましょう。

トレーニング時間の比較を共有する方法の例については、W&B Report の [What To Do When Inception-ResNet-V2 Is Too Slow](https://wandb.ai/stacey/estuary/reports/When-Inception-ResNet-V2-is-too-slow--Vmlldzo3MDcxMA) をご覧ください。

複雑なコードベース から最良の例を保存して、簡単に参照したり、将来的に活用したりできるようにします。 Lyft データセット からの LIDAR ポイントクラウドを可視化し、3D バウンディングボックスでアノテーションを付ける方法の例については、W&B Report の [LIDAR point clouds](https://wandb.ai/stacey/lyft/reports/LIDAR-Point-Clouds-of-Driving-Scenes--Vmlldzo2MzA5Mg) をご覧ください。

## Collaboration: 同僚と 学び を共有する

プロジェクトの 開始方法 を説明し、これまでに観察した内容を共有し、最新の 学び を統合します。同僚は、任意の パネル や Reports の最後にあるコメントを使用して、提案を行ったり詳細を議論したりできます。

同僚が自分で探索し、追加の洞察を得て、次のステップ をより適切に計画できるように、動的な 設定 を含めます。この例では、3種類の実験を個別に可視化したり、比較したり、平均化したりできます。

ベンチマークの最初の run と観察結果を共有する方法の例については、W&B Report の [SafeLife benchmark experiments](https://wandb.ai/stacey/saferlife/reports/SafeLife-Benchmark-Experiments--Vmlldzo0NjE4MzM) を参照してください。

## Work log: 試したことを追跡し、 次のステップ を計画する

プロジェクトを進める中で、実験に関する考え、 学び 、注意点、 次のステップ を書き留め、すべてを1か所に整理して保管します。これにより、 スクリプト 以外のすべての重要な要素を「ドキュメント化」できます。 学び を報告する方法の例については、W&B Report の [Who Is Them? Text Disambiguation With Transformers](https://wandb.ai/stacey/winograd/reports/Who-is-Them-Text-Disambiguation-with-Transformers--VmlldzoxMDU1NTc) をご覧ください。

<Frame>
    <img src="/images/reports/intro_work_log_1.png" alt="Text disambiguation report"  />
</Frame>

プロジェクトのストーリーを伝えることで、あなたや他の人が後で モデル がどのように、そしてなぜ開発されたのかを理解するために参照できるようになります。 学び を報告する方法については、W&B Report の [The View from the Driver's Seat](https://wandb.ai/stacey/deep-drive/reports/The-View-from-the-Driver-s-Seat--Vmlldzo1MTg5NQ) をご覧ください。
