<Note>
  [プロジェクト間でRunsを比較する方法を示す動画](https://www.youtube.com/watch?v=uD4if_nGrs4) (2分) をご覧ください。
</Note>

クロスプロジェクトレポートを使用して、2つの異なるプロジェクトのRunsを比較します。run set 表のプロジェクトセレクターを使用して、プロジェクトを選択してください。

<Frame>
  <img src="/images/reports/howto_pick_a_different_project_to_draw_runs_from.gif" alt="異なるプロジェクト間でRunsを比較する" />
</Frame>

このセクションの可視化は、最初のアクティブなrun setから列を取得します。折れ線グラフで探しているメトリクスが表示されない場合は、セクション内で最初にチェックされているrun setでその列が使用可能であることを確認してください。

この機能は時系列の線におけるhistoryデータをサポートしますが、異なるプロジェクトから異なるsummaryメトリクスを取得することはサポートしていません。つまり、別のプロジェクトでのみログされている列から散布図を作成することはできません。

2つのプロジェクトのRunsを比較する必要があり、列が機能しない場合は、一方のプロジェクトのRunsにタグを追加してから、それらのRunsをもう一方のプロジェクトに移動してください。各プロジェクトのRunsだけを引き続きフィルターできますが、レポートには両方のRunsセットのすべての列が含まれます。

<div id="view-only-report-links">
  ## View-Only レポートリンク
</div>

プライベートなプロジェクトまたはチームプロジェクト内のレポートに、View-Only リンクを共有します。

<Frame>
  <img src="/images/reports/magic-links.gif" alt="View-only report links" />
</Frame>

View-Only レポートリンクは URL に secret 액세스 token を追加するため、リンクを開いた人は誰でもページを表示できます。誰でも、最初にログインしなくても、このマジックリンクを使用してレポートを表示できます。[W&amp;B Local](/ja/platform/hosting/) のプライベートクラウド環境をご利用のお客様の場合、これらのリンクは引き続きファイアウォールの内側に留まるため、プライベートインスタンスへのアクセス権限 *and* View-Only リンクへのアクセス権限の両方を持つチームメンバーのみがレポートを表示できます。

**View-Only モード**では、ログインしていないユーザーでもチャートを確認し、マウスオーバーして値のツールチップを表示し、チャートをズームインおよびズームアウトし、表内の列をスクロールできます。view モードでは、データを explore するための新しいチャートや新しい表クエリは作成できません。レポートリンクの View-Only 訪問者は、run をクリックして run ページに移動することはできません。また、View-Only 訪問者には共有モーダルは表示されず、代わりにホバー時に `Sharing not available for view only access` と表示されるツールチップが表示されます。

<Info>
  マジックリンクが 사용 가능하다 のは、“Private” および “Team” プロジェクトのみです。“Public”  (誰でも表示可能) または “Open”  (誰でも表示および Runs への投稿が可能) プロジェクトでは、このプロジェクトは公開されており、すでにリンクを知っている誰でも利用できることを意味するため、リンクのオン/オフは切り替えられません。
</Info>

<div id="send-a-graph-to-a-report">
  ## グラフをレポートに送信する
</div>

Workspace からレポートにグラフを送信して、進捗を追跡します。レポートにコピーしたいチャートまたはパネルのドロップダウンメニューをクリックし、**Add to report** をクリックして送信先のレポートを選択します。
