<Note>
  W&amp;B Report and Workspace API は Public Preview です。
</Note>

App UI を使用してインタラクティブに、または W&amp;B SDK を使用してプログラムでレポートを編集します。

レポート は *blocks* で構成されます。Blocks はレポートの本文を構成します。これらの blocks 内には、テキスト、画像、埋め込みビジュアライゼーション、Experiments や run のプロット、パネルグリッドを追加できます。

*Panel grids* は、パネルと *run sets* を保持する特定の種類の block です。Run sets は、W&amp;B のプロジェクトにログされた Runs のコレクションです。パネルは run set データのビジュアライゼーションです。

<Note>
  保存済み Workspace ビューを作成およびカスタマイズする方法について、ステップバイステップの例は [Programmatic workspaces notebook](https://colab.research.google.com/github/wandb/wandb-workspaces/blob/Update-wandb-workspaces-tuturial/Workspace_tutorial.ipynb) を参照してください。
</Note>

<Note>
  レポートをプログラムで編集する場合は、W&amp;B Python SDK に加えて W&amp;B Report and Workspace API `wandb-workspaces` がインストールされていることを確認してください。

  ```pip
  pip install wandb wandb-workspaces
  ```
</Note>

<div id="add-plots">
  ## プロットを追加
</div>

各パネルグリッドには、run セット一式とパネル一式があります。セクションの下部にある run セットによって、グリッド内のパネルにどのデータを表示するかが制御されます。別の runs 一式からデータを取得するチャートを追加する場合は、新しいパネルグリッドを作成します。

<Tabs>
  <Tab title="W&B App">
    レポート内でスラッシュ (`/`) を入力すると、ドロップダウンメニューが表示されます。**Add panel** を選択してパネルを追加します。折れ線グラフ、散布図、Parallel coordinates チャートなど、W&amp;B でサポートされる任意のパネルを追加できます。

    <Frame>
      <img src="/images/reports/demo_report_add_panel_grid.gif" alt="レポートにチャートを追加" />
    </Frame>
  </Tab>

  <Tab title="Report and Workspace API">
    SDK を使用してレポートにプロットをプログラムで追加します。1 つ以上のプロットまたはチャートの object の list を、`PanelGrid` Public API クラスの `panels` パラメーターに渡します。関連付けられた Python クラスを使用して、プロットまたはチャートの object を作成します。

    次の例は、折れ線グラフと散布図の作成方法を示しています。

    ```python
    import wandb
    import wandb_workspaces.reports.v2 as wr

    report = wr.Report(
        project="report-editing",
        title="An amazing title",
        description="A descriptive description.",
    )

    blocks = [
        wr.PanelGrid(
            panels=[
                wr.LinePlot(x="time", y="velocity"),
                wr.ScatterPlot(x="time", y="acceleration"),
            ]
        )
    ]

    report.blocks = blocks
    report.save()
    ```

    レポートにプログラムで追加できる사용 가능하다プロットとチャートの詳細については、`wr.panels` を参照してください。
  </Tab>
</Tabs>

<div id="add-run-sets">
  ## run セットを追加
</div>

App UI または W&amp;B SDK を使用して、プロジェクトからインタラクティブに run セットを追加します。

<Tabs>
  <Tab title="W&B App">
    レポート内でスラッシュ (`/`) を入力すると、ドロップダウン メニューが表示されます。ドロップダウンから **Panel Grid** を選択します。これにより、レポートの作成元であるプロジェクトから run セットが自動的にインポートされます。

    レポートにパネルをインポートすると、run 名はプロジェクトから引き継がれます。レポートでは、必要に応じて [run の名前を変更](/ja/models/runs/#rename-a-run) して、読者により多くのコンテキストを提供できます。run 名が変更されるのは個々のパネル内だけです。同じレポート内でパネルを複製すると、複製されたパネル内でも run 名が変更されます。

    1. レポートで鉛筆アイコンをクリックして、レポート エディターを開きます。

    2. run セットで、名前を変更する run を検索します。レポート名にカーソルを合わせ、縦に並んだ 3 つの点をクリックします。次のいずれかを選択して、フォームを送信します。

       * **Rename run for project**: プロジェクト全体で run 名を変更します。新しいランダム名を生成するには、フィールドを空白のままにします。
       * **Rename run for panel grid** run 名をレポート内でのみ変更し、他のコンテキストでは既存の名前を保持します。新しいランダム名の生成はサポートされていません。

    3. **Publish report** をクリックします。
  </Tab>

  <Tab title="Report and Workspace API">
    `wr.Runset()` および `wr.PanelGrid` クラスを使って、プロジェクトから run セットを追加します。以下の手順では、runset を追加する方法を説明します。

    1. `wr.Runset()` オブジェクト インスタンスを作成します。`project` パラメーターには run セットを含むプロジェクトの名前を、`entity` パラメーターにはそのプロジェクトを所有する entity を指定します。
    2. `wr.PanelGrid()` オブジェクト インスタンスを作成します。`run sets` パラメーターに、1 つ以上の runset オブジェクトの list を渡します。
    3. 1 つ以上の `wr.PanelGrid()` オブジェクト インスタンスを list に格納します。
    4. レポート インスタンスの blocks 属性を、panel grid インスタンスの list で更新します。

    ```python
    import wandb
    import wandb_workspaces.reports.v2 as wr

    report = wr.Report(
        project="report-editing",
        title="An amazing title",
        description="A descriptive description.",
    )

    panel_grids = wr.PanelGrid(
        runsets=[wr.RunSet(project="<project-name>", entity="<entity-name>")]
    )

    report.blocks = [panel_grids]
    report.save()
    ```

    必要に応じて、SDK への 1 回の Call で runset とパネルを追加することもできます。

    ```python
    import wandb

    report = wr.Report(
        project="report-editing",
        title="An amazing title",
        description="A descriptive description.",
    )

    panel_grids = wr.PanelGrid(
        panels=[
            wr.LinePlot(
                title="line title",
                x="x",
                y=["y"],
                range_x=[0, 100],
                range_y=[0, 100],
                log_x=True,
                log_y=True,
                title_x="x axis title",
                title_y="y axis title",
                ignore_outliers=True,
                groupby="hyperparam1",
                groupby_aggfunc="mean",
                groupby_rangefunc="minmax",
                smoothing_factor=0.5,
                smoothing_type="gaussian",
                smoothing_show_original=True,
                max_runs_to_show=10,
                plot_type="stacked-area",
                font_size="large",
                legend_position="west",
            ),
            wr.ScatterPlot(
                title="scatter title",
                x="y",
                y="y",
                # z='x',
                range_x=[0, 0.0005],
                range_y=[0, 0.0005],
                # range_z=[0,1],
                log_x=False,
                log_y=False,
                # log_z=True,
                running_ymin=True,
                running_ymean=True,
                running_ymax=True,
                font_size="small",
                regression=True,
            ),
        ],
        runsets=[wr.RunSet(project="<project-name>", entity="<entity-name>")],
    )


    report.blocks = [panel_grids]
    report.save()
    ```
  </Tab>
</Tabs>

<div id="freeze-a-run-set">
  ## run セット を固定する
</div>

レポートでは、プロジェクトの最新データが表示されるように run セット が自動的に更新されます。レポート内の run セット は、その run セット を*固定*することで保持できます。run セット を固定すると、ある時点での run セット の状態がレポート内に保持されます。

レポートの表示中に run セット を固定するには、パネルグリッド内の **フィルター** ボタンの近くにある雪の結晶アイコンをクリックします。

<div id="group-a-run-set-programmatically">
  ## run セットをプログラムでグループ化する
</div>

[Workspace and Reports API](/ja/models/ref/wandb_workspaces/reports) を使用して、run セット内の Runs をプログラムでグループ化できます。

設定値、run メタデータ、またはサマリーメトリクスによって、run セット内の Runs をグループ化できます。次の表は、使用可能なグループ化 method と、そのグループ化 method で使用できるキーを示します。

| Grouping Method | Description               | Available keys                        |
| --------------- | ------------------------- | ------------------------------------- |
| 設定値             | 設定値で Runs をグループ化します       | `wandb.init(config=)` の設定パラメーターで指定した値 |
| run メタデータ       | run メタデータで Runs をグループ化します | `State`, `Name`, `JobType`            |
| サマリーメトリクス       | サマリーメトリクスで Runs をグループ化します | `wandb.Run.log()` で run にログした値        |

{/* レポートでrunをグループ化する場合とWorkspaceでrunをグループ化する場合の主な違い:
  1. 文字列パスとobject: Reports では、`ws.Config("group")` のようなタイプ object ではなく、`"config.group"` のような文字列パスを使用します。
  2. ドット表記: ネストされた値に액세스するには、`"config.model_type"` のようにドットを使用します。
  3. 定義済みフィールド: `"Name"`、`"Tags"`、`"State"` などの一部のフィールドは、特別なrunメタデータフィールドです。 */}

<div id="group-runs-by-config-values">
  ### 設定値で run をグループ化する
</div>

類似した設定を持つ run を比較するには、設定値で run をグループ化します。設定値は、run の設定 `(wandb.init(config=))` で指定するパラメーターです。設定値で run をグループ化するには、`config.<key>` 構文を使用します。ここで、`<key>` はグループ化に使用する設定値の名です。

たとえば、次のコード 스니펫では、最初に `group` の設定値を持つ run を初期化し、その後 `group` 設定値を 기준으로 レポート内の run をグループ化します。`<entity>` と `<project>` の値は、ご使用の W&amp;B entity 名およびプロジェクト名に置き換えてください。

```python
import wandb
import wandb_workspaces.reports.v2 as wr

entity = "<entity>"
project = "<project>"

for group in ["control", "experiment_a", "experiment_b"]:
    for i in range(3):
        with wandb.init(entity=entity, project=project, group=group, config={"group": group, "run": i}, name=f"{group}_run_{i}") as run:
            # トレーニングをシミュレート
            for step in range(100):
                run.log({
                    "acc": 0.5 + (step / 100) * 0.3 + (i * 0.05),
                    "loss": 1.0 - (step / 100) * 0.5
                })
```

Python script or notebook 内で、`config.group` の値を使って run をグループ化できます:

```python
runset = wr.Runset(
  project=project,
  entity=entity,
  groupby=["config.group"]  # "group" の設定値でグループ化
)
```

前の例に続いて、グループ化されたrun セットを使ってレポートを作成できます。

```python
report = wr.Report(
  entity=entity,
  project=project,
  title="Grouped Runs Example",
)

report.blocks = [
  wr.PanelGrid(
      runsets=[runset],
          )
      ]

report.save()
```

<div id="group-runs-by-run-metadata">
  ### run メタデータで run をグループ化する
</div>

run の名 (`Name`)、状態 (`State`)、またはジョブ タイプ (`JobType`) で run をグループ化します。

前の例に続いて、次の코드 스니펫を使用して run を名でグループ化できます：

```python
runset = wr.Runset(
  project=project,
  entity=entity,
  groupby=["Name"]  # run 名ごとにグループ化
)
```

<Note>
  run の名は、`wandb.init(name=)` パラメーターで指定する名です。名を指定しない場合、W&amp;B はその run に対してランダムな名を生成します。

  run の名は、W&amp;B App の run の **Overview** ページ、または `Api.runs().run.name` を使ってプログラムから検索できます。
</Note>

<div id="group-runs-by-summary-metrics">
  ### サマリーメトリクスでRunsをグループ化する
</div>

以下の例では、サマリーメトリクスでRunsをグループ化する方法を示します。サマリーメトリクスは、`wandb.Run.log()` を使って run にログする値です。run をログした後、サマリーメトリクスの名は W&amp;B App の run の **Overview** ページにある **Summary** セクションで確認できます。

サマリーメトリクスでRunsをグループ化する構文は `summary.<key>` です。ここで、`<key>` はグループ化に使用するサマリーメトリクスの名です。

たとえば、`acc` という名のサマリーメトリクスをログするとします。

```python
import wandb
import wandb_workspaces.reports.v2 as wr

entity = "<entity>"
project = "<project>"

for group in ["control", "experiment_a", "experiment_b"]:
    for i in range(3):
        with wandb.init(entity=entity, project=project, group=group, config={"group": group, "run": i}, name=f"{group}_run_{i}") as run:
            # トレーニングをシミュレートする
            for step in range(100):
                run.log({
                    "acc": 0.5 + (step / 100) * 0.3 + (i * 0.05),
                    "loss": 1.0 - (step / 100) * 0.5
                })

```

次に、`summary.acc` の summary メトリクスで Runs をグループ化できます:

```python
runset = wr.Runset(
  project=project,
  entity=entity,
  groupby=["summary.acc"]  # summary の値でグループ化 
)
```

<div id="filter-a-run-set-programmatically">
  ## run セットをプログラムでフィルターする
</div>

run セットをプログラムでフィルターし、[Workspace and Reports API](/ja/models/ref/wandb_workspaces/reports) を使用してレポートに追加します。

フィルター式の一般的な構文は次のとおりです。

```text
Filter('key') operation <value>
```

ここで、`key` はフィルター名、`operation` は比較演算子 (例: `>`, `<`, `==`, `in`, `not in`, `or`, `and`) 、`<value>` は比較対象の値です。`Filter` は、適用するフィルターのタイプを表すプレースホルダーです。次の表は、利用可能なフィルターとその説明を示します。

| Filter                 | Description      | Available keys                                 |
| ---------------------- | ---------------- | ---------------------------------------------- |
| `Config('key')`        | 設定値でフィルター        | `wandb.init(config=)` の `config` パラメーターで指定した値。 |
| `SummaryMetric('key')` | サマリーメトリクスでフィルター  | `wandb.Run.log()` を使用して run にログした値。            |
| `Tags('key')`          | tags でフィルター      | run に追加したタグの値 (プログラムから、または W&amp;B App から追加) 。 |
| `Metric('key')`        | run のプロパティでフィルター | `tags`, `state`, `displayName`, `jobType`      |

フィルターを定義したら、レポートを作成し、フィルター済みの run セット を `wr.PanelGrid(runsets=)` に渡せます。レポートにさまざまな要素をプログラムで追加する方法の詳細については、このページ全体にある **Report and Workspace API** タブを参照してください。

次の例では、レポート内で run セット をフィルターする方法を示します。`<>` で囲まれた値はご自身の値に置き換えてください。

<div id="config-filters">
  ### 設定フィルター
</div>

1 つ以上の設定値で runset をフィルターします。設定値は、run 設定 (`wandb.init(config=)`) で指定するパラメーターです。

たとえば、次の코드 スニペット では、まず `learning_rate` と `batch_size` の設定値を含む run を初期化し、その後 `learning_rate` の設定値を 기준으로 レポート内の Runs をフィルターします。

```python
import wandb

config = {
    "learning_rate": 0.01,
    "batch_size": 32,
}

with wandb.init(project="<project>", entity="<entity>", config=config) as run:
    # トレーニングコードをここに記述
    pass
```

Python script or notebook 内で、学習率が `0.01` を超える Runs をプログラムでフィルターできます。

```python
import wandb_workspaces.reports.v2 as wr

runset = wr.Runset(
  entity="<entity>",
  project="<project>",
  filters="Config('learning_rate') > 0.01"
)
```

`and` 演算子を使用して、複数の設定値でフィルターすることもできます:

```python
runset = wr.Runset(
  entity="<entity>",
  project="<project>",
  filters="Config('learning_rate') > 0.01 and Config('batch_size') == 32"
)
```

前の例の続きとして、フィルターした runset を使って次のようにレポートを作成できます。

```python
report = wr.Report(
  entity="<entity>",
  project="<project>",
  title="My Report"
)

report.blocks = [
  wr.PanelGrid(
      runsets=[runset],
      panels=[
          wr.LinePlot(
              x="Step",
              y=["accuracy"],
          )
      ]
  )
]

report.save()
```

<div id="metric-filters">
  ### メトリクス フィルター
</div>

run セットは、run の tag (`tags`)、run state (`state`)、run 名 (`displayName`)、またはジョブ タイプ (`jobType`) に基づいてフィルターします。

<Note>
  `Metric` フィルターは構文が異なります。値のリストはリストとして渡します。

  ```text
  Metric('key') operation [<value>]
  ```
</Note>

たとえば、次の Python スニペットでは 3 つの run を作成し、それぞれに名を付けます。

```python
import wandb

with wandb.init(project="<project>", entity="<entity>") as run:
    for i in range(3):
        run.name = f"run{i+1}"
        # トレーニングコードをここに記述
        pass
```

レポートを作成するときは、run を Display name でフィルターできます。たとえば、名が `run1`、`run2`、`run3` の run をフィルターするには、次のコードを使用できます。

```python
runset = wr.Runset(
  entity="<entity>",
  project="<project>",
  filters="Metric('displayName') in ['run1', 'run2', 'run3']"
)
```

<Note>
  W&amp;B App の run の **Overview** ページ、または `Api.runs().run.name` を使用してプログラムから run の名を確認できます。
</Note>

次の例では、run の状態 (`finished`、`crashed`、または `running`) で runset をフィルターする方法を示します。

```python
runset = wr.Runset(
  entity="<entity>",
  project="<project>",
  filters="Metric('state') in ['finished']"
)
```

```python
runset = wr.Runset(
  entity="<entity>",
  project="<project>",
  filters="Metric('state') not in ['crashed']"
)
```

<div id="summarymetric-filters">
  ### SummaryMetric フィルター
</div>

次の例は、サマリーメトリクスで run セットをフィルターする方法を示しています。サマリーメトリクスは、`wandb.Run.log()` を使用して run にログする値です。run をログした後、W&amp;B App の run の **Overview** ページにある **Summary** セクションで、サマリーメトリクスの名前を検索できます。

```python
runset = wr.Runset(
  entity="<entity>",
  project="<project>",
  filters="SummaryMetric('accuracy') > 0.9"
)
```

```python
runset = wr.Runset(
  entity="<entity>",
  project="<project>",
  filters="Metric('state') in ['finished'] and SummaryMetric('train/train_loss') < 0.5"
)
```

<div id="tags-filters">
  ### Tags フィルター
</div>

以下の코드 スニペットは、Runs set をその tags でフィルターする方法を示しています。Tags は、run に追加する値です (プログラムから、または W&amp;B App で追加します) 。

```python
runset = wr.Runset(
  entity="<entity>",
  project="<project>",
  filters="Tags('training') == 'training'"
)
```

<div id="add-code-blocks">
  ## コードブロックを追加する
</div>

App UI または W&amp;B SDK を使用して、レポートにコードブロックをインタラクティブに追加します。

<Tabs>
  <Tab title="App UI">
    レポート内でスラッシュ (`/`) を入力すると、ドロップダウンメニューが表示されます。ドロップダウンから **Code** を選択します。

    コードブロックの右側にあるプログラミング言語の名を選択します。これによりドロップダウンが展開されます。ドロップダウンから、プログラミング言語の構文を選択します。Javascript、Python、CSS、JSON、HTML、Markdown、YAML から選択できます。
  </Tab>

  <Tab title="Report and Workspace API">
    `wr.CodeBlock` Class を使用して、プログラムでコードブロックを作成します。language パラメーターと code パラメーターに、それぞれ言語の名と表示したいコードを指定します。

    たとえば、次の例は YAML ファイル内のリストを示しています。

    ```python
    import wandb
    import wandb_workspaces.reports.v2 as wr

    report = wr.Report(project="report-editing")

    report.blocks = [
        wr.CodeBlock(
            code=["this:", "- is", "- a", "cool:", "- yaml", "- file"], language="yaml"
        )
    ]

    report.save()
    ```

    これにより、次のようなコードブロックが表示されます。

  ## 複数の次元にまたがる関係を可視化する
</div>

複数の次元にまたがる関係を効果的に可視化するには、変数の 1 つを表すために色のグラデーションを使用します。これにより明瞭さが向上し、パターンを解釈しやすくなります。

1. 色のグラデーションで表す変数を選択します (例: penalty scores、learning rates など) 。これにより、トレーニング時間 (x-axis) にわたって、penalty (色) が reward/side effects (Y 軸) とどのように相互作用するかを、より明確に理解できます。
2. 主要な傾向を強調表示します。特定の runs グループにカーソルを合わせると、可視化内でそれらが強調表示されます。
