リネージグラフを使用して、リンクされたアーティファクトの履歴を可視化します。コレクションの履歴を監査して、そのコレクション内の アーティファクト に加えられた変更をトラッキングします。

<div id="lineage-graphs">
  ## リネージグラフ
</div>

W&amp;B Registry のコレクション内では、ML 実験で使用されるアーティファクトの履歴を表示できます。この履歴を *リネージグラフ* と呼びます。

リネージグラフには、次の内容が表示されます。

* [run への入力として使用されたアーティファクト](/ja/models/artifacts/explore-and-traverse-an-artifact-graph#track-the-input-of-a-run)。
* [run からの出力として作成されたアーティファクト](/ja/models/artifacts/explore-and-traverse-an-artifact-graph#track-the-output-of-a-run)。

つまり、リネージグラフは run の入力と出力を示します。

たとえば、次の画像は、ML 実験全体を通じて作成および使用されるアーティファクトの典型的なリネージグラフを示しています。

<Frame>
  <img src="/images/registry/registry_lineage.png" alt="Registry のリネージ" />
</Frame>

この画像は左から右に向かって、次の内容を示しています。

1. 複数の run が `split_zoo_dataset:v4` アーティファクトをログします。
2. &quot;rural-feather-20&quot; run は、`split_zoo_dataset:v4` アーティファクトをトレーニングに使用します。
3. &quot;rural-feather-20&quot; run の出力は、`zoo-ylbchv20:v0` という名前のモデルアーティファクトです。
4. &quot;northern-lake-21&quot; という名前の run が、モデルアーティファクト `zoo-ylbchv20:v0` を使用してモデルを評価します。

コレクション内のアーティファクトのリネージグラフを表示するには、次の手順を実行します。

1. W&amp;B Registry にアクセスします。
2. アーティファクトを含むコレクションを選択します。
3. ドロップダウンから、リネージグラフを表示するアーティファクトのバージョンを選択します。
4. **Lineage** タブを選択します。
5. ノードを選択すると、run またはアーティファクトの詳細情報を表示できます。

<Info>
  [Enable lineage graph tracking](/ja/models/artifacts/explore-and-traverse-an-artifact-graph#enable-lineage-graph-tracking) を参照して、W&amp;B Python SDK を使用して run の入力と出力をトラッキングする方法を確認してください。
</Info>

<div id="audit-a-collections-history">
  ## コレクションの履歴を監査する
</div>

組織のメンバーがそのコレクションに対して実行した 액ションを確認できます。確認できる内容は次のとおりです。

* アーティファクトバージョンに 별칭 が追加または削除されたかどうか。
* アーティファクトバージョンがコレクションに追加または削除されたかどうか。

どちらの 액ションについても、その 액ションを実行したユーザーと、액ションが発生した日付を確認できます。

コレクションの 액ション履歴を表示するには、次の手順に従います。

1. W&amp;B Registry にアクセスします。
2. 액ション履歴を表示するコレクションを選択します。
3. コレクション名の横にあるドロップダウンメニューを選択します。
4. **Action History** オプションを選択します。
