Lineage グラフを使用して、リンクされたアーティファクトの履歴を可視化します。コレクションの履歴を監査して、そのコレクション内のアーティファクトに加えられた変更を追跡します。

<div id="lineage-graphs">
  ## ライネージグラフ
</div>

W&amp;B Registry 内のコレクションでは、ML 実験で使用されるアーティファクトの履歴を表示できます。この履歴は *lineage graph*（ライネージグラフ）と呼ばれます。

ライネージグラフには、次の情報が表示されます。

* [run の入力として使用されたアーティファクト](/ja/models/artifacts/explore-and-traverse-an-artifact-graph#track-the-input-of-a-run)
* [run の出力として作成されたアーティファクト](/ja/models/artifacts/explore-and-traverse-an-artifact-graph#track-the-output-of-a-run)

言い換えると、ライネージグラフは run の入力と出力を表示します。

たとえば、次の画像は、ML 実験全体で作成および使用されるアーティファクトの典型的なライネージグラフを示しています。

<Frame>
  <img src="/images/registry/registry_lineage.png" alt="Registry のライネージ" />
</Frame>

この画像は左から右にかけて、次の内容を示しています。

1. 複数の run が `split_zoo_dataset:v4` アーティファクトをログに記録します。
2. &quot;rural-feather-20&quot; run は `split_zoo_dataset:v4` アーティファクトをトレーニングに使用します。
3. &quot;rural-feather-20&quot; run の出力は、`zoo-ylbchv20:v0` という名前のモデルアーティファクトです。
4. &quot;northern-lake-21&quot; という run が、モデルアーティファクト `zoo-ylbchv20:v0` を使用してモデルを評価します。

コレクション内のアーティファクトに対してライネージグラフを表示するには、次の手順を実行します。

1. W&amp;B Registry に移動します。
2. アーティファクトを含むコレクションを選択します。
3. ドロップダウンから、ライネージグラフを表示したいアーティファクトバージョンを選択します。
4. **Lineage** タブを選択します。
5. ノードを選択して、run またはアーティファクトに関する詳細情報を表示します。

<Info>
  W&amp;B Python SDK を使用して run の入力と出力を追跡する方法については、[Enable lineage graph tracking](/ja/models/artifacts/explore-and-traverse-an-artifact-graph#enable-lineage-graph-tracking) を参照してください。
</Info>

<div id="audit-a-collections-history">
  ## コレクションの履歴を監査する
</div>

組織のメンバーがそのコレクションに対して実行した操作を表示できます。次の内容を確認できます。

* エイリアスがアーティファクト バージョンに追加または削除されたかどうか。
* アーティファクト バージョンがコレクションに追加または削除されたかどうか。

どちらの操作についても、その操作を実行したユーザーと、操作が行われた日付を表示できます。

コレクションの操作履歴を表示するには、次の手順を実行します。

1. W&amp;B Registry に移動します。
2. 操作履歴を表示したいコレクションを選択します。
3. コレクション名の横にあるドロップダウン メニューを選択します。
4. **Action History** オプションを選択します。

  アーティファクト ノードを選択すると、そのアーティファクトの完全な名前、タイプ、作成時刻、関連付けられたエイリアスなどの詳細を表示できます。 */}
