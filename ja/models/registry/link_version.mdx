---
title: コレクションにアーティファクト バージョンをリンクする
description: "W&B Registry のコレクションにアーティファクト バージョンをリンクして、組織全体で共有します。"
---

アーティファクト バージョンを組織で利用できるようにするには、W&amp;B Registry の
[コレクション](/ja/models/registry/create_collection)に*リンク*します。リンクすると、バージョンは[非公開のプロジェクト レベルのスコープから、共有された組織レベルのスコープへ](/ja/models/registry/create_registry#visibility-types)移動します。アーティファクト バージョンは、[W&amp;B Python SDK を使用してプログラムから、または W&amp;B App でインタラクティブに](/ja/models/registry/link_version#link-an-artifact-to-a-collection)リンクできます。

アーティファクトをリンクすると、W&amp;B はソース アーティファクトとコレクション エントリの間に参照を作成します。リンクされたバージョンは、プロジェクト内の run にログされたソース アーティファクト バージョンを参照します。コレクション内のリンクされたバージョンと、ログされたプロジェクト内のソース バージョンの両方を表示できます。

<div id="link-an-artifact-to-a-collection">
  ## コレクションに アーティファクト をリンクする
</div>

ユースケースに応じて、以下のタブに記載された手順に従って アーティファクト バージョンをリンクします。

<Note>
  作業を始める前に、次の点を確認してください。

  * コレクションで許可されている アーティファクト の種類。コレクション タイプの詳細については、[Create a collection](./create_collection) 内の「Collection types」を参照してください。
  * コレクションが属する Registry がすでに存在していること。Registry が存在するか確認するには、[Registry App にアクセスして](/ja/models/registry/search_registry) Registry 名を検索します。
</Note>

  アーティファクト バージョンがメトリクスをログしている場合（`wandb.Run.log_artifact()` を使用するなど）、そのバージョンの詳細ページでメトリクスを表示でき、アーティファクトのページでアーティファクト バージョン間のメトリクスを比較できます。[Registry でリンクされたアーティファクトを表示する](#view-linked-artifacts-in-a-registry) を参照してください。
  </Note> */}

    アーティファクト バージョンをプログラムでコレクションにリンクするには、[`wandb.Run.link_artifact()`](/ja/models/ref/python/experiments/run#link_artifact) または [`wandb.Artifact.link()`](/ja/models/ref/python/experiments/artifact#method-artifactlink) を使用します。

    <Note>
      [run のコンテキスト内でアーティファクト バージョンをリンクする](#link-an-artifact-version-within-the-context-of-a-run)には、`wandb.Run.link_artifact()` を使用します。[run のコンテキスト外でアーティファクト バージョンをリンクする](#link-an-artifact-version-outside-the-context-of-a-run)には、`wandb.Artifact.link()` を使用して、*既存のアーティファクト バージョン* をリンクします。
    </Note>

    {/* <Note>
            `wandb.Artifact.link()` を使用する場合、`wandb.init()` で run を初期化する必要はありません。`wandb.Run.link_artifact()` を使用するには、`wandb.init()` で run を初期化しておく必要があります。
            </Note> */}

    どちらのアプローチでも、アーティファクトの名前 (`wandb.Artifact(name="<name>"`) 、アーティファクトのタイプ (`wandb.Artifact(type="<type>"`) 、およびアーティファクト バージョンをリンクしたいコレクションと Registry の `target_path` (`wandb.Artifact(target_path="<target_path>"`) を指定します。

    ターゲットパスは、接頭辞 `"wandb-registry"`、Registry 名、コレクション名をスラッシュ (`/`) で区切った形式で構成されます：

    {/* `target_path` パラメーターを使用して、アーティファクト バージョンのリンク先のコレクションと Registry を指定します。ターゲット パスは、接頭辞 "wandb-registry"、Registry 名、コレクション名をスラッシュ (`/`) で区切った形式で構成されます： */}

    ```text
    wandb-registry-{REGISTRY_NAME}/{COLLECTION_NAME}
    ```

    ### run のコンテキスト内でアーティファクト バージョンをリンクする

    run のコンテキスト内でアーティファクト バージョンをリンクするには `wandb.Run.link_artifact()` を使用します。まず `wandb.init()` で run を初期化します。次に、アーティファクトオブジェクトを作成してファイルを追加します。最後に、`wandb.Run.link_artifact()` メソッドを使用して、アーティファクト バージョンをコレクションにリンクします。

    このアプローチを使用すると、W&amp;B プロジェクトに run が作成されます。アーティファクト バージョンはコレクションにリンクされ、その run に関連付けられます。

    以下のコードスニペットをコピー＆ペーストしてください。`<>` で囲まれた値は実際の値に置き換えてください：

    ```python
    import wandb

    entity = "<team_entity>"          # チーム entity
    project = "<project_name>"        # アーティファクトを含むプロジェクトの名前

    # run を初期化する
    with wandb.init(entity = entity, project = project) as run:

      # アーティファクトオブジェクトを作成する
      # type パラメーターはアーティファクトオブジェクトのタイプと
      # コレクションのタイプの両方を指定する
      artifact = wandb.Artifact(name = "<name>", type = "<type>")

      # アーティファクトオブジェクトにファイルを追加する。
      # ローカルマシン上のファイルのパスを指定する。
      artifact.add_file(local_path = "<local_path_to_artifact>")

      # アーティファクトのリンク先となるコレクションと Registry を指定する
      REGISTRY_NAME = "<registry_name>"  
      COLLECTION_NAME = "<collection_name>"
      target_path=f"wandb-registry-{REGISTRY_NAME}/{COLLECTION_NAME}"

      # アーティファクトをコレクションにリンクする
      run.link_artifact(artifact = artifact, target_path = target_path)
    ```

    ### run のコンテキスト外でアーティファクト バージョンをリンクする

    `wandb.Artifact.link()` を使用すると、run のコンテキスト外で既存のアーティファクト バージョンをリンクできます。この方法では、`wandb.init()` で run を初期化する必要はありません。つまり、W&amp;B プロジェクトに run は作成されません。言い換えると、アーティファクト バージョンは run に関連付けられることなく、コレクションにリンクされます。

    まず、アーティファクト オブジェクトを作成してファイルを追加します。次に、`wandb.Artifact.link()` メソッドを使用して、アーティファクト バージョンをコレクションにリンクします。

    以下のコードスニペットをコピー＆ペーストしてください。`<>` で囲まれた値は実際の値に置き換えてください：

    ```python
    import wandb

    # アーティファクトオブジェクトを作成する
    # type パラメーターはアーティファクトオブジェクトと
    # コレクションの両方のタイプを指定する
    artifact = wandb.Artifact(name = "<name>", type = "<type>")

    # アーティファクトオブジェクトにファイルを追加する。
    # ローカルマシン上のファイルのパスを指定する。
    artifact.add_file(local_path = "<local_path_to_artifact>")

    # アーティファクトのリンク先となるコレクションと Registry を指定する
    REGISTRY_NAME = "<registry_name>"  
    COLLECTION_NAME = "<collection_name>"
    target_path=f"wandb-registry-{REGISTRY_NAME}/{COLLECTION_NAME}"

    # アーティファクトをコレクションにリンクする
    artifact.link(target_path = target_path)
    ```

{/* <Note>
  **リンクされたアーティファクト バージョンとソース アーティファクト バージョンの違い**

  * ソース バージョン: チームのプロジェクト内で [run](/models/track/runs/) にログされたアーティファクト バージョンです。
  * リンクされたバージョン: Registry に公開されたアーティファクト バージョンです。これはソース アーティファクトへのポインタであり、Registry のスコープで利用できるようにしただけの、同一のアーティファクト バージョンです。
  </Note> */}

Registry App では、[リンクされたアーティファクトのメタデータ、バージョン データ、利用状況、リネージ情報](/ja/models/registry/link_version#view-linked-artifacts-in-a-registry")などを確認できます。

<div id="view-linked-artifacts-in-a-registry">
  ## Registry でリンクされた アーティファクト を表示する
</div>

W&amp;B Registry で、メタデータ、リネージ、利用状況など、リンクされた アーティファクト に関する情報を表示します。

1. W&amp;B Registry にアクセスします。
2. アーティファクト をリンクした Registry の名前を選択します。
3. コレクションの名前を選択します。
4. コレクションの アーティファクト がメトリクスをログしている場合は、**Show metrics** をクリックしてバージョン間でメトリクスを比較します。
5. アーティファクト バージョンの一覧から、アクセスするバージョンを選択します。バージョン番号は `v0` から始まり、リンクされた各 アーティファクト バージョンに順番に割り当てられます。
6. アーティファクト バージョンの詳細を表示するには、そのバージョンをクリックします。このページのタブから、そのバージョンのメタデータ (ログされたメトリクスを含む)、リネージ、利用状況を表示できます。

**Version** タブ内の **Full Name** フィールドに注目してください。リンクされた アーティファクト の完全名は、Registry 名、コレクション名、および アーティファクト バージョンのエイリアスまたはインデックスで構成されます。

```text title="Full name of a linked artifact"
wandb-registry-{REGISTRY_NAME}/{COLLECTION_NAME}:v{INTEGER}
```

リンクされた アーティファクト のバージョンにプログラムから 액세スするには、その アーティファクト の完全名が必要です.

<div id="troubleshooting">
  ## トラブルシューティング
</div>

以下では、アーティファクトをリンクできない場合に確認すべき一般的なポイントをいくつか紹介します。

<div id="logging-artifacts-from-a-personal-account">
  ### 個人アカウントからのアーティファクトのログ
</div>

個人のentityでW&amp;Bにログされたアーティファクトは、Registryにリンクできません。組織内のチーム entityを使用してアーティファクトをログするようにしてください。組織のチーム内でログされたアーティファクトのみを、組織のRegistryにリンクできます。

<Note>
  アーティファクトをRegistryにリンクする場合は、そのアーティファクトをチーム entityでログしてください。
</Note>

<div id="find-your-team-entity">
  #### チーム entity を確認する
</div>

W&amp;B はチームの名をそのチームの entity 名として使用します。たとえば、チームの名が **team-awesome** の場合、そのチームの entity は `team-awesome` です。

チームの名は次の手順で確認できます。

1. チームの W&amp;B プロフィールページにアクセスします。
2. サイトの URL をコピーします。URL は `https://wandb.ai/<team>` の形式です。ここで `<team>` はチームの名であり、チームの entity 名でもあります。

<div id="log-from-a-team-entity">
  #### チーム entity からログを記録する
</div>

1. [`wandb.init()`](/ja/models/ref/python/functions/init) で run を初期化する際に、`entity` としてチームを指定します。run を初期化する際に `entity` を指定しない場合、run ではデフォルトの entity が使用されますが、これは必ずしもチーム entity とは限りません。

   ```python
   import wandb   

   with wandb.init(entity='<team_entity>', project='<project_name>') as run:
       # ここでアーティファクトをログする
   ```

2. `wandb.Run.log_artifact()` を使用するか、Artifact オブジェクトを作成してファイルを追加することで、run にアーティファクトをログします：

   ```python
   artifact = wandb.Artifact(name="<artifact_name>", type="<type>")
   ```

   アーティファクトをログするには、[アーティファクトを構築する](/ja/models/artifacts/construct-an-artifact/) を参照してください。

3. アーティファクトが個人の entity にログされた場合、組織内の entity に再度ログする必要があります。

<div id="confirm-the-path-of-a-registry-in-the-wb-app-ui">
  ### W&amp;B App UI でRegistryのパスを確認する
</div>

UI でRegistryのパスを確認するには、2 つの方法があります。空のコレクションを作成してその詳細を表示するか、コレクションのホームページにある自動生成コードをコピー＆ペーストします。

<div id="copy-and-paste-autogenerated-code">
  #### 自動生成されたコードをコピー＆ペーストする
</div>

1. https://wandb.ai/registry/ にある W&amp;B Registry にアクセスします。
2. アーティファクトをリンクしたい Registry をクリックします。
3. ページ上部に自動生成されたコードブロックが表示されます。
4. これを自分のコードにコピー＆ペーストし、パスの最後の部分をコレクション名に置き換えてください。

<Frame>
  <img src="/images/registry/get_autogenerated_code.gif" alt="Auto-generated 코드 스니펫" />
</Frame>

<div id="create-an-empty-collection">
  #### 空のコレクションを作成する
</div>

1. https://wandb.ai/registry/ にある W&amp;B Registry にアクセスします。
2. アーティファクトをリンクしたいRegistryをクリックします。
3. 空のコレクションをクリックします。空のコレクションが存在しない場合は、新しいコレクションを作成します。
4. 表示された코드 스니펫内で、`.link_artifact()` の `target_path` フィールドを特定します。
5. (任意) コレクションを削除します。

<Frame>
  <img src="/images/registry/check_empty_collection.gif" alt="空のコレクションを作成する" />
</Frame>

たとえば、上記のstepを完了すると、`target_path` パラメーターを含むコードブロックを検索できます。

```python
target_path = 
      "smle-registries-bug-bash/wandb-registry-Golden Datasets/raw_images"
```

これを構成要素ごとに分解すると、プログラムでアーティファクトをリンクするためのパスを作成する際に使用すべき要素がわかります：

```python
ORG_ENTITY_NAME = "smle-registries-bug-bash"
REGISTRY_NAME = "Golden Datasets"
COLLECTION_NAME = "raw_images"
```

<Note>
  一時コレクションの名を、アーティファクトをリンクしたいコレクションの名に置き換えてください。
</Note>
