---
title: 별칭で アーティファクト バージョンを参照する
description: "デフォルト、カスタム、保護された별칭を使用して、W&B Registry 内の特定の アーティファクト バージョン を参照します。"
---

1 つ以上の별칭を使って、特定の [アーティファクト バージョン](/ja/models/artifacts/create-a-new-artifact-version/) を参照できます。同じ名前でリンクした各 Artifacts には、[W&amp;B が自動的に별칭を割り当て](/ja/models/registry/aliases/#default-aliases) ます。また、特定の Artifacts バージョンを参照するために、[カスタム 별칭を 1 つ以上作成](/ja/models/registry/aliases/#custom-aliases) することもできます。

별칭は Registry UI では、その별칭名が書かれた四角形として表示されます。[별칭が保護されている](/ja/models/registry/aliases/#protected-aliases) 場合、鍵アイコン付きの灰色の四角形として表示されます。それ以外の場合、별칭はオレンジ色の四角形として表示されます。별칭は Registry 間で共有されません。

<Note>
  **별칭とタグの使い分け**

  별칭は、特定の Artifacts バージョンを参照するために使用します。コレクション内の各별칭は一意であり、同じ별칭を持てる Artifacts バージョンは同時に 1 つだけです。

  タグは、共通のテーマに基づいて Artifacts バージョンやコレクションを整理・グループ化するために使用します。複数の Artifacts バージョンやコレクションが同じタグを共有できます。
</Note>

Artifacts バージョンに별칭を追加するときに、オプションで Slack チャンネルへの通知や webhook のトリガーを行うための [Registry automation](/ja/models/automations/automation-events/#registry) を開始できます。

<div id="default-aliases">
  ## デフォルトの별칭
</div>

W&amp;B は、同じ名でリンクする各アーティファクト バージョンに、自動的に次の별칭を부여します。

* コレクションにリンクした最新のアーティファクト バージョンには、`latest` 별칭を부여します。
* 一意のバージョン番号。W&amp;B は、リンクした各アーティファクト バージョンを 0 から順にカウントし、そのカウント番号を使って、そのアーティファクトに一意のバージョン番号を부여します。

たとえば、`zoo_model` という名のアーティファクトを 3 回リンクすると、W&amp;B はそれぞれ `v0`、`v1`、`v2` の 3 つの별칭を作成します。`v2` には `latest` 별칭も付きます。

<div id="custom-aliases">
  ## カスタム별칭
</div>

特定のアーティファクトバージョンに対して、ユースケースに応じた1つ以上のカスタム별칭を作成できます。例えば：

* `dataset_version_v0`、`dataset_version_v1`、`dataset_version_v2` などの별칭を使用して、モデルがどのデータセットでトレーニングされたかを識別できます。
* `best_model` 별칭を使用して、最も高いパフォーマンスを示すアーティファクトモデルバージョンを追跡できます。

Registry で [**Member** または **Admin** Registry ロール](/ja/models/registry/configure_registry/#registry-roles) を持つすべてのユーザーは、その Registry 内のリンクされたアーティファクトに対してカスタム별칭を追加または削除できます。[**Restricted Viewer** または **Viewer** ロール](/ja/models/registry/configure_registry/#registry-roles) を持つ Users は별칭の追加または削除ができません。

<Note>
  [保護された별칭](/ja/models/registry/aliases/#protected-aliases) は、変更や削除から保護するアーティファクトバージョンにラベルを付けて識別するための方法です。
</Note>

カスタム별칭は W&amp;B Registry または Python SDK を使用して作成できます。ユースケースに応じて、以下のタブからニーズに最も適したものをクリックしてください。

<Tabs>
  <Tab title="W&B Registry">
    1. W&amp;B Registry にアクセスします。
    2. コレクション内の **View details** ボタンをクリックします。
    3. **Versions** セクション内で、特定のアーティファクトバージョンの **View** ボタンをクリックします。
    4. **Aliases** フィールドの横にある **+** ボタンをクリックして、1つ以上の별칭を追加します。
  </Tab>

  <Tab title="Python SDK">
    Python SDK を使用してアーティファクトバージョンをコレクションにリンクする際、[`link_artifact()`](/ja/models/ref/python/experiments/run.md/#link_artifact) の `alias` パラメーターの引数として、オプションで1つ以上の별칭のリストを指定できます。指定した별칭がまだ存在しない場合、W&amp;B は별칭 ([保護されていない별칭](#custom-aliases)) を自動的に作成します。

    以下のコードスニペットは、Python SDK を使用してアーティファクトバージョンをコレクションにリンクし、そのアーティファクトバージョンに별칭を追加する方法を示しています。`<>` 内の値はご自身のものに置き換えてください：

    ```python
    import wandb

    # run を初期化する
    with wandb.init(entity = "<team_entity>", project = "<project_name>") as run:

        # アーティファクトオブジェクトを作成する
        # type パラメーターはアーティファクトオブジェクトのタイプと
        # コレクションのタイプの両方を指定する
        artifact = wandb.Artifact(name = "<name>", type = "<type>")

        # アーティファクトオブジェクトにファイルを追加する
        # ローカルマシン上のファイルへのパスを指定する
        artifact.add_file(local_path = "<local_path_to_artifact>")

        # アーティファクトをリンクするコレクションと Registry を指定する
        REGISTRY_NAME = "<registry_name>"
        COLLECTION_NAME = "<collection_name>"
        target_path=f"wandb-registry-{REGISTRY_NAME}/{COLLECTION_NAME}"

        # アーティファクトバージョンをコレクションにリンクする
        # このアーティファクトバージョンに1つ以上の별칭を追加する
        run.link_artifact(
            artifact = artifact, 
            target_path = target_path, 
            aliases = ["<alias_1>", "<alias_2>"]
            )
    ```
  </Tab>
</Tabs>

<div id="protected-aliases">
  ### 保護されたエイリアス
</div>

[保護されたエイリアス](/ja/models/registry/aliases/#protected-aliases) を使用すると、変更や削除してはならないアーティファクトバージョンにラベルを付けて識別できます。たとえば、組織の機械学習本番パイプラインで使用されているアーティファクトバージョンにラベルを付けて識別するために、`production` の保護されたエイリアスの使用を検討してください。

[Registry admin](/ja/models/registry/configure_registry/#registry-roles) ユーザーおよび **Admin** ロールを持つ [service accounts](/ja/models/support/service_account_useful/) は、保護されたエイリアスを作成し、アーティファクトバージョンに対して保護されたエイリアスを追加または削除できます。**Member**、**Viewer**、**Restricted Viewer** ロールを持つユーザーと service accounts は、保護されたバージョンのリンクを解除したり、保護されたエイリアスを含むコレクションを削除したりすることはできません。詳細については、[Configure Registry access](/ja/models/registry/configure_registry/) を参照してください。

一般的な保護されたエイリアスには次のようなものがあります。

* **Production**: アーティファクトバージョンは本番利用の準備ができています。
* **Staging**: アーティファクトバージョンはテストの準備ができています。

<div id="create-a-protected-alias">
  #### 保護された별칭を作成する
</div>

以下の手順では、W&amp;B Registry UI で保護された별칭を作成する方法を説明します。

1. W&amp;B Registry にアクセスします。
2. Registry を選択します。
3. ページ右上のギアボタンをクリックして、そのRegistryの設定を表示します。
4. **Protected Aliases** セクション内で **+** ボタンをクリックして、1 つ以上の保護された별칭を追加します。

作成後、各保護された별칭は **Protected Aliases** セクション内で、鍵アイコン付きの灰色の長方形として表示されます。

<Note>
  保護されていないカスタム별칭とは異なり、保護された별칭の作成は W&amp;B Registry UI でのみ利用可能であり、Python SDK からプログラムで行うことはできません。アーティファクトバージョンに保護された별칭を追加するには、W&amp;B Registry UI か Python SDK を使用できます。
</Note>

以下の手順では、W&amp;B Registry UI を使用してアーティファクトバージョンに保護された별칭を追加する方法を説明します。

1. W&amp;B Registry にアクセスします。
2. コレクション内の **View details** ボタンをクリックします。
3. **Versions** セクション内で、特定のアーティファクトバージョンの **View** ボタンを選択します。
4. **Aliases** フィールドの横にある **+** ボタンをクリックして、1 つ以上の保護された별칭を追加します。

保護された별칭が作成された後は、管理者が Python SDK を使ってプログラムからその별칭をアーティファクトバージョンに追加できます。アーティファクトバージョンに保護された별칭を追加する方法の例については、上記の [Create a custom alias](#custom-aliases) セクション内の W&amp;B Registry タブおよび Python SDK タブを参照してください。

<div id="find-existing-aliases">
  ## 既存の별칭を検索する
</div>

[W&amp;B Registry のグローバル検索バー](/ja/models/registry/search_registry/#search-for-registry-items)を使って、既存の별칭を検索できます。保護された별칭を見つけるには、次の手順を実行します。

1. W&amp;B Registry にアクセスします。
2. ページ上部の検索バーに検索キーワードを指定します。Enter キーを押して検索します。

指定したキーワードが既存の Registry、コレクション名、アーティファクトのバージョンタグ、コレクションタグ、または별칭と一致する場合、検索バーの下に検索結果が表示されます。

<div id="example">
  ## 例
</div>

<Note>
  次のコード例は、この [W&amp;B Registry Tutorial notebook](https://colab.research.google.com/github/wandb/examples/blob/master/colabs/wandb_registry/zoo_wandb.ipynb) の続きです。以下のコードを使用するには、まず [notebook で説明されているとおりに Zoo データセットを取得して処理する必要があります](https://colab.research.google.com/github/wandb/examples/blob/master/colabs/wandb_registry/zoo_wandb.ipynb#scrollTo=87fecd29-8146-41e2-86fb-0bb4e3e3350a)。Zoo データセットを用意できたら、アーティファクト バージョンを作成し、カスタム 별칭を追加できます。
</Note>

次のコード スニペットは、アーティファクト バージョンを作成し、カスタム 별칭を追加する方法を示しています。この例では、[UCI Machine Learning Repository](https://archive.ics.uci.edu/dataset/111/zoo) の Zoo データセットと、`Zoo_Classifier_Models` Registry 内の `Model` コレクションを使用します。

```python
import wandb

# run を初期化する
with wandb.init(entity = "smle-reg-team-2", project = "zoo_experiment") as run:

    # アーティファクト オブジェクトを作成する
    # type パラメーターはアーティファクト オブジェクトのタイプと
    # コレクションのタイプの両方を指定する
    artifact = wandb.Artifact(name = "zoo_dataset", type = "dataset")

    # アーティファクト オブジェクトにファイルを追加する
    # ローカルマシン上のファイルパスを指定する
    artifact.add_file(local_path="zoo_dataset.pt", name="zoo_dataset")
    artifact.add_file(local_path="zoo_labels.pt", name="zoo_labels")

    # アーティファクトのリンク先となるコレクションと Registry を指定する
    REGISTRY_NAME = "Model"
    COLLECTION_NAME = "Zoo_Classifier_Models"
    target_path=f"wandb-registry-{REGISTRY_NAME}/{COLLECTION_NAME}"

    # アーティファクトのバージョンをコレクションにリンクする
    # このアーティファクトのバージョンに 1 つ以上の별칭を追加する
    run.link_artifact(
        artifact = artifact,
        target_path = target_path,
        aliases = ["production-us", "production-eu"]
        )
```

1. まず、 Artifacts オブジェクト (`wandb.Artifact()`) を作成します。
2. 次に、`wandb.Artifact.add_file()` を使って、PyTorch のデータセット テンソルを 2 つ Artifacts オブジェクトに追加します。
3. 最後に、`link_artifact()` を使って、その Artifacts バージョンを `Zoo_Classifier_Models` レジストリ内の `Model` コレクションにリンクします。また、`aliases` パラメーターに引数として `production-us` と `production-eu` を渡すことで、その Artifacts バージョンに 2 つのカスタム 별칭を追加します。
