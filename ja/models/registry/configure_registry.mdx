Registry 管理者は、Registry の設定を設定することで、[Registry ロールを設定する](/ja/models/registry/configure_registry/#configure-registry-roles)、[ユーザーを追加](/ja/models/registry/configure_registry/#add-a-user-or-a-team-to-a-registry)、または [ユーザーを削除](/ja/models/registry/configure_registry/#remove-a-user-or-team-from-a-registry) を Registry から行えます。

<div id="manage-users">
  ## Users の管理
</div>

<div id="add-a-user-or-a-team">
  ### ユーザーまたはチームを追加する
</div>

Registry 管理者は、個々のユーザーまたは Teams 全体を Registry に追加できます。Registry にユーザーまたはチームを追加するには、次の手順に従います。

1. W&amp;B Registry にアクセスします。
2. ユーザーまたはチームを追加する Registry を選択します。
3. 右上隅の gear icon をクリックして、Registry 設定にアクセスします。
4. **Registry 액세스** セクションで、**Add access** をクリックします。
5. **Include users and teams** フィールドに、1 つ以上のユーザー名、メールアドレス、またはチーム名を指定します。
6. **Add access** をクリックします。

<Frame>
  <img src="/images/registry/add_team_registry.gif" alt="Registry に Teams を追加する" />
</Frame>

[Registry でのユーザーロールの設定](/ja/models/registry/configure_registry/#configure-registry-roles) または [Registry ロールの権限](/ja/models/registry/configure_registry/#registry-role-permissions) について詳しくは、こちらをご覧ください。

<div id="remove-a-user-or-team">
  ### ユーザーまたはチームを削除する
</div>

Registry 管理者は、個々のユーザーまたはチーム全体を Registry から削除できます。ユーザーまたはチームを Registry から削除するには、次の手順に従います。

1. https://wandb.ai/registry/ にある W&amp;B Registry にアクセスします。
2. ユーザーを削除したい Registry を選択します。
3. 画面右上隅の歯車アイコンをクリックして、Registry 設定に액세スします。
4. **Registry 액세ス** セクションにアクセスし、削除したい username、メールアドレス、またはチーム名を入力します。
5. **Delete** ボタンをクリックします。

<Note>
  チームからユーザーを削除すると、そのユーザーの Registry へのアクセス権も失われます。
</Note>

<div id="change-the-owner-of-a-registry">
  ### Registry のオーナーを変更する
</div>

Registry 管理者は、**Restricted Viewer** や **Viewer** を含む任意のメンバーを、その Registry のオーナーとして指定できます。Registry のオーナーシップは主に責任の所在を明確にするためのものであり、ユーザーに割り当てられているロールによって既に付与されているもの以上の権限が追加で与えられることはありません。

オーナーを変更するには:

1. https://wandb.ai/registry/ にある W&amp;B Registry にアクセスします。
2. 設定したい Registry を選択します。
3. 右上隅の歯車アイコンをクリックします。
4. **Registry members and roles** セクションまでスクロールします。
5. メンバーの行の上にカーソルを置きます。
6. 行の末尾にある **...** 작업 메뉴をクリックし、**Make owner** をクリックします。

<div id="configure-registry-roles">
  ## Registry ロールを設定する
</div>

このセクションでは、Registry メンバーのロールを設定する方法を示します。各ロールの機能、優先順位、デフォルトなど、Registry ロールの詳細については、[Registry ロールの詳細](#details-about-registry-roles)を参照してください。

1. https://wandb.ai/registry/ の W&amp;B Registry にアクセスします。
2. 設定する Registry を選択します。
3. 右上隅の歯車アイコンをクリックします。
4. **Registry members and roles** セクションまでスクロールします。
5. **Member** フィールド内で、権限を編集するユーザーまたはチームを検索します。
6. **Registry role** 列で、ユーザーのロールをクリックします。
7. ドロップダウンから、ユーザーに부여するロールを選択します。

<div id="details-about-registry-roles">
  ## Registry ロールの詳細
</div>

以下のセクションでは、Registry ロールについて詳しく説明します。

<Note>
  [チーム内でのロール](/ja/platform/app/settings-page/teams/#team-roles-and-permissions)は、どの Registry におけるあなたのロールにも影響せず、関係もありません。
</Note>

<div id="default-roles">
  ### デフォルトロール
</div>

W&amp;B では、ユーザーまたはチームが Registry に追加されると、デフォルトの **Registry ロール** が自動的に割り当てられます。このロールによって、その Registry 内で実行できる操作が決まります。

| Entity                        | デフォルトの Registry ロール<br />(専用クラウド / セルフマネージド)                                | デフォルトの Registry ロール<br />(Multi-tenant Cloud)              |
| ----------------------------- | --------------------------------------------------------------------------- | ---------------------------------------------------------- |
| チーム                           | Restricted Viewer (Server v0.75.0+) <br />Viewer (Server v0.74.x and below) | Restricted Viewer                                          |
| ユーザー or サービスアカウント (non admin) | Restricted Viewer (Server v0.75.0+) <br />Viewer (Server v0.74.x and below) | Restricted Viewer                                          |
| サービスアカウント (non admin)         | Member<sup><a href="#service_account_footnote">1</a></sup>                  | Member<sup><a href="#service_account_footnote">1</a></sup> |
| Org admin                     | Admin                                                                       | Admin                                                      |

<a id="service_account_footnote">1</a>: サービスアカウントには **Viewer** または **Restricted Viewer** ロールを割り当てることはできません。

Registry 管理者は、Registry 内のユーザーおよびチームのロールを 부여したり、変更したりできます。
詳細については、[Registry のユーザーロールを設定する](/ja/models/registry/configure_registry/#configure-registry-roles) を参照してください。

<div id="inherited-registry-role">
  ### 継承された Registry ロール
</div>

Registry のメンバーシップ一覧では、各ユーザーの継承された (有効な) Registry ロールが、その行のロールドロップダウンの横に薄いグレーで表示されます。

<Frame>
  <img src="/images/registry/role_conflict.png" alt="ユーザーの有効な Registry ロールを表示する Registry メンバーシップ一覧" />
</Frame>

特定の Registry におけるユーザーの有効なロールは、組織、Registry、およびその Registry を所有するチームでのロールのうち、継承されたものか明示的に割り当てられたものかを問わず、*最も高い* ロールになります。たとえば、次のとおりです。

* チームの **Admin** または組織の **Admin** が、そのチームが所有する特定の Registry で **Viewer** ロールを持っている場合、その Registry では実質的に **Admin** になります。
* Registry の **Viewer** がチームで **Member** ロールを持っている場合、その Registry では実質的に **Member** になります。
* チームの **Viewer** が特定の Registry で **Member** ロールを持っている場合、その Registry では実質的に **Member** になります。

<div id="restricted-viewer-role-details">
  ### Restricted Viewer ロールの詳細
</div>

**Restricted Viewer** ロールは Generally Available (GA) です。Dedicated Cloud と Self-Managed では、Server v0.75.0 以降が必要です。

このロールでは、コレクション、オートメーション、その他の Registry リソースの作成、更新、削除はできず、Registry Artifacts には読み取り専用でアクセスできます。

**Viewer** と異なり、**Restricted Viewer** では次の操作はできません:

* Artifacts のファイルをダウンロードしたり、ファイルの内容にアクセスしたりすること。
* W&amp;B SDK で `wandb.Run.use_artifact()` を使用して Artifacts を使用すること。

<div id="sdk-compatibility">
  #### SDK の互換性
</div>

<Note>
  **SDK バージョン要件**

  W&amp;B SDK を使用して **Restricted Viewer** として Artifacts にアクセスするには、W&amp;B SDK バージョン 0.19.9 以降を使用する必要があります。そうしないと、一部の SDK コマンドで権限エラーが発生します。
</Note>

**Restricted Viewer** が SDK を使用する場合、一部の function は利用できないか、動作が異なります。

次の method は利用できず、権限エラーが発生します。

* [`Run.use_artifact()`](/ja/models/ref/python/experiments/run/#method-runuse_artifact)
* [`Artifact.download()`](/ja/models/ref/python/experiments/artifact/#method-artifactdownload)
* [`Artifact.file()`](/ja/models/ref/python/experiments/artifact/#method-artifactfile)
* [`Artifact.files()`](/ja/models/ref/python/experiments/artifact/#method-artifactfiles)

次の method はArtifacts のメタデータに限定されます。

* [`Artifact.get_entry()`](/ja/models/ref/python/experiments/artifact/#method-artifactget_entry)
* [`Artifact.get_path()`](/ja/models/ref/python/experiments/artifact/#method-artifactget_path)
* [`Artifact.get()`](/ja/models/ref/python/experiments/artifact/#method-artifactget)
* [`Artifact.verify()`](/ja/models/ref/python/experiments/artifact/#method-artifactverify)

<div id="cross-registry-permissions">
  ### レジストリ間の権限
</div>

ユーザーは、異なるRegistryで異なるロールを持つことができます。たとえば、ユーザーは Registry A では **Restricted Viewer** であり、Registry B では **Viewer** である場合があります。この場合:

* 両方のRegistryにリンクされた同じアーティファクトは、異なる액세스レベルを持ちます
* Registry A では、ユーザーは **Restricted Viewer** であり、ファイルをダウンロードしたり、アーティファクトを使用したりできません
* Registry B では、ユーザーは **Viewer** であり、ファイルをダウンロードしてアーティファクトを使用できます
* つまり、액세스はアーティファクトにアクセスされるRegistryによって決まります
