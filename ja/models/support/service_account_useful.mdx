**service account** は、人ではないマシン用のアイデンティティを表し、Teams や プロジェクト をまたいだ一般的なタスクを自動化できます。サービスアカウントは、CI/CD パイプライン、自動学習ジョブ、その他のマシン同士のワークフローに最適です。

<Text />

また、サービスアカウントは、定期的な再学習やナイトリービルドなど、wandb にログされる自動ジョブをトラッキングする用途にも有用です。必要に応じて、[環境変数](/ja/models/track/environment-variables/) `WANDB_USERNAME` や `WANDB_USER_EMAIL` を使って、これらのマシンによって起動された run にユーザー名を関連付けることができます。

サービスアカウントのベストプラクティスや詳細なセットアップ手順など、包括的な情報については、[Use service accounts to automate workflows](/ja/platform/hosting/iam/service-accounts/) を参照してください。チームコンテキストにおけるサービスアカウントの動作については、[Team Service Account Behavior](/ja/platform/app/settings-page/teams/#team-service-account-behavior) を参照してください。

<Note>
  **Built-in** サービスアカウントとは別に、W&amp;B は [SDK および CLI 向けのアイデンティティフェデレーション](/ja/platform/hosting/iam/identity_federation/#external-service-accounts) を利用した **External service accounts** にも対応しています。JSON Web Token (JWT) を発行できるアイデンティティプロバイダで管理されるサービスアイデンティティを使って W&amp;B のタスクを自動化したい場合は、External service accounts を使用してください。
</Note>
