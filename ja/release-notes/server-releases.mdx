<Update label="v0.79.0" description="2026年3月24日">
  W&amp;B Server 0.79.0 では、画像のズームとパン、サービスアカウントと APIキー の管理機能の改善、パフォーマンス改善が追加されました。Inference では MiniMax M2.5 と MiniMaxAI M2.5 モデルが追加されています。Weave monitors はすべてのオペレーションでのスコアリングをサポートするようになり、Weave UI にはプロンプトのタグとエイリアスの管理強化、Claude エージェント向けのチャットビュー、絵文字のフィードバックリアクションやメモ内容によるフィルタリングなどが追加されました。

  ## サポートとサポート終了

  * W&amp;B Server v0.67 はサポート終了となりました。

  [Release policies and processes](/ja/release-notes/release-policies) を参照してください。サポートが必要な場合や質問がある場合は、[support](mailto:support@wandb.com) にお問い合わせください。

  ## セキュリティ

  * サービスアカウントの作成時に、そのアカウント用の APIキー を作成する手順へ自動的に案内されるようになりました。
  * W&amp;B App からサービスアカウントまたは APIキー を削除する際に、確認が必須になりました。
  * SQL インジェクションの防止に役立てるため、バックエンドでフィルター内の列名をエスケープするようになりました。
  * [diacritical marks](https://en.wikipedia.org/wiki/Combining_Diacritical_Marks) を含むメールアドレスは、ユーザー登録またはログインでブロックされるようになりました。

  ## 機能

  * **画像をパンおよびズーム**して、W&amp;B の全画面画像モーダル内で細かな画像の詳細を直接確認できます。任意の画像を全画面で開いたら、次の操作を行います。

    * UI コントロールまたはキーボードショートカットを使用して、ズームレベルの設定、100% へのリセット、または画面に収まるようにズームできます。
    * ズームイン時は、クリックしてドラッグするとパンできます。

    <Frame>
      ![全画面画像モーダルのパンとズームのコントロール](/images/release-notes/v0-79-0/image-zoom-pan-fullscreen-modal.png)
    </Frame>
  * ピン留めされた run とベースライン run では、[サマリーメトリクスの差分](/ja/models/runs/compare-runs#summary-metric-deltas) によって、各 run がベースラインと比べてどう違うかが表示されます。すばやく視覚的に把握できるようにセマンティックカラーリングを有効にするには、列の Metric directionality を設定します。この機能は一般提供になりました。
    <Frame>
      ![ベースラインと比較した Runs table のサマリーメトリクスの差分](/images/release-notes/v0-78-0/baseline-deltas-runs-table.png)
    </Frame>
  * run selector と Runs リストが強化されました。
    * Git commit によるフィルタリングがサポートされました。
    * 設定パラメーターを指定する run で、`CONTAINS` 演算子がサポートされました。
  * データベース負荷を軽減するため、プロジェクトフィールドに防御的キャッシュを追加しました。

  ### Registry

  * システム過負荷のリスクを軽減するため、アーティファクト作成に新しい rate limits を追加しました。

  ### Weave

  <Note>Weave を使用するデプロイでは、v0.78+ へアップグレードする前に Clickhouse v0.25.8+ が必要です。</Note>

  * Weave で Claude エージェント向けのチャットビューがサポートされるようになりました。
  * **Traces** ページと **Evaluations** ページで、メモ内容および絵文字のフィードバックリアクションによるフィルタリングがサポートされるようになりました。
  * **Assets** ページの **Prompts** タブで、プロンプトのタグとエイリアスを管理できるようになりました。
  * **Calls** ページと **Traces** ページの表で、列のドラッグ＆ドロップによる並べ替えがサポートされるようになりました。
  * Weave monitors は、すべてのオペレーションにわたるスコアリングをサポートするようになりました。

  ## 修正

  * バックエンドのスムージングにおける浮動小数点の丸め誤差を修正しました。
  * 式入力欄への入力時にカーソルが飛んでしまうバグを修正しました。
  * ギャラリーモードで実サイズの画像が重なってしまうバグを修正しました。
  * 長いキャプションがあるバウンディングボックスの位置ずれを修正しました。
  * パネルとホバーオーバーレイのドロップシャドウを削除しました。
  * パネル検索結果にカスタムチャートが含まれるようになりました。
</Update>

  * **Registry** には、強化された検索、再設計されたCard、より明確で応答性の高いリネージグラフなどが含まれます。
  * **Weave** では、**Trace Analytics** ダッシュボードにより、Project Overview ページからボトルネックや予算の主な要因を特定できます。保存済みビューで、カスタマイズしたトレース、評価、リーダーボードがサポートされるようになりました。Weave Playground で、LoRA (Low-Rank Adaptation) と新しいモデルがサポートされるようになりました。
  * **Inference** には新しいモデルが含まれます。

  <Warning>
    このレポートは、アーティファクトへの不正な액세스を可能にする重大なセキュリティ問題の影響を受けています。このバージョンをインストールまたはアップグレードしないでください。代わりに、修正が含まれている[v0.78.2](#v0-78-2)をインストールしてください。
  </Warning>

  ## サポートとサポート終了

  * v0.66.0 以前のバージョンはサポート終了しました。

  [リリースポリシーとプロセス](/ja/release-notes/release-policies)をご参照ください。サポートが必要な場合やご質問がある場合は、[サポート](mailto:support@wandb.com)までお問い合わせください。

  ## Security

  * W&amp;B の組織またはデプロイでは、個人用 APIキーを最大 100 個、ユーザーごとに組織レベルの APIキーを 100 個、サービスアカウントごとに APIキーを 100 個まで持てるようになりました。
  * Enterprise プランでは、[メンバーを招待](/ja/platform/hosting/iam/access-management/manage-organization#invite-a-user)して組織に追加すると、デフォルトで **Member** ロールが付与されます。その他のプランでは、デフォルトのロールは **Admin** です。
  * 現在組織のメンバーではない新しいユーザーをチームに招待する場合、その初期ロールは **Member** になります。
  * 組織またはチームに[招待](/ja/platform/hosting/iam/access-management/manage-organization#invite-a-user)されたユーザーの初期ロールが、招待一覧表に表示されるようになりました。

  ## 機能

  * 重要な run を追跡し、モデル実験を効率的に評価するために、[ピン留めした run とベースライン run](/ja/models/runs/compare-runs) を使用します。

    * 最大 6 件のrunをピン留めして、Workspace 内およびRuns リストの上部に表示したままにできます。baseline run がある場合、baseline は暗黙的にピン留めされるため、ピン留めできるrunは最大 5 件です。
    * 比較の参照点としてbaseline run を指定します。baseline run は常にWorkspace 内およびRuns リストの上部に表示されます。

    Runs table では、サマリーメトリクスの差分 (プレビュー) により、各 run がベースラインと比べてどう異なるかを確認できます。これは招待制でのみ利用できます。詳細については [mailto:support@wandb.com] までお問い合わせください。

    <Frame>
      ![ベースラインと比較した Runs table のサマリーメトリクスの差分](/images/release-notes/v0-78-0/baseline-deltas-runs-table.png)
    </Frame>

    折れ線グラフでは、比較しやすいように、ベースラインは視覚的に区別しやすいスタイルで表示されます。

    <Frame>
      ![折れ線グラフで異なるスタイルで表示された baseline run](/images/release-notes/v0-78-0/baseline-line-plot-styling.png)
    </Frame>

  * [複数のメトリクスを含む Panel を一括で作成](/ja/models/app/features/panels/line-plot#multi-metric-line-plot)すると、Workspace をすばやく整理し、大量のメトリクス群にまたがるパターンを見つけやすくなります。たとえば、モデルのレイヤー全体で gradient メトリクスを重ねて表示したり、特定のレイヤーを横に並べて配置したりできます。[動画デモを見る](https://www.loom.com/embed/ebbb1ab3f858473cb24420bf3abf7166)。

  * [Workspace テンプレート](/ja/models/app/features/cascade-settings#workspace-layout-options) に新しい設定が追加されました。空のセクションを非表示にする、セクション内の Panel をアルファベット順に並べ替える、またはセクション内での Panel の自動並べ替えの動作を調整するように、Workspace テンプレートを設定できるようになりました。[Workspace Layout オプション](/ja/models/app/features/cascade-settings#workspace-layout-options) を参照してください。

  * The **Sweeps** 表に、**Sweep ID** の列が追加されました。

  * 管理者は、組織の使用量と支出に関するアラートをカスタマイズできるようになりました。新しい **Usage &amp; Alerts** タブの組織設定で、使用量と支出を確認し、storage、tracked hours、Weave、Inference、トレーニングのメール通知を設定できます。詳細は、[プランの使用状況](/ja/platform/app/settings-page/billing-settings#plan-usage)を参照してください。

  ### Registry

  * Registry UI は、よりすばやく操作でき、一覧しやすくなりました。

    * 改善されたタイプアヘッド検索、再設計された Registry とコレクションの Card、新しい Registry アイコンにより、まずはスマートランキングを使用して assets を視覚的に区別しやすくなり、AI ランキングもまもなく利用可能になります。
    * Registry アイコンは初期状態では自動生成され、Registry 管理者はいつでもカスタマイズできます。

    <Frame>
      ![Registry アイコンと再設計された Card](/images/release-notes/v0-78-0/registry-icons-and-cards.gif)
    </Frame>

    * リネージ グラフでは、ノードにマウスポインタを合わせると、そのノードが強調表示され、他のノードは淡色表示されます。
    * コレクションの名をクリックすると、そのアーティファクト バージョンをすばやく表示できるようになりました。
    * 利便性向上のため、Registry の **Settings** ページからスターを付けられるようになりました。

  ### Inference

  * モデル `Moonshot AI Kimi K2.5` が利用可能になりました。[モデルカタログ](/ja/inference/models)をご覧ください。

  ### Weave

  * **Project Overview** ページに、リクエスト数、レイテンシのパーセンタイル (P50/P95/P99) 、モデル別の token 使用量、コストの内訳を表示する **Trace Analytics** ダッシュボードが追加されました。どのモデルが予算を圧迫しているか、またはどの低速なリクエストが P95 を押し下げているかを把握できます。時間バケットをクリックすると、関連するトレースに直接移動して、レイテンシ急増の原因をすばやく特定できます。

    <Frame>
      ![プロジェクトの Overview にある Trace Analytics ダッシュボード](/images/release-notes/v0-78-0/weave-trace-analytics-dashboard.png)
    </Frame>

  * **Weave Playground** は現在 **[LoRA](/ja/inference/lora)** をサポートしており、ファインチューニングした LoRA アダプターを効率的にテストできます。LoRA アーティファクト ページで **Try in Playground** ボタンをクリックします。

  * Claude Opus 4.6 が **Weave Playground** で利用可能になりました。

  * **Traces** と **Evaluations** の表で列を並べ替えられるようになりました。

    * 表内で末端列を直接ドラッグ アンド ドロップするか、列管理メニューからセクション全体を移動できます。
    * 変更内容は[保存済みビュー](/ja/weave/guides/tools/saved-views)に保存できます。動画デモをご覧ください。

    <Frame>![動画: トレース表と評価表で列を並べ替える](/images/release-notes/v0-78-0/weave-column-reorder-demo.gif)</Frame>

  * カスタマイズした[Leaderboard](/ja/weave/guides/evaluation/dynamic_leaderboards)を保存済みビューとして保存できるようになりました。

  * [monitors](/ja/weave/guides/evaluation/monitors) を使うと、トレース内のオーディオや画像コンテンツを評価できるようになりました。

  * 集計された OpenTelemetry のトークン数が、[OpenTelemetry トレース](/ja/weave/guides/tracking/otel#send-opentelemetry-traces-to-weave) のトレース詳細サイドバーに表示されるようになりました。

  * Gemini のように thinking tokens を提供するモデルでは、thinking tokens が Weave UI の **Token Usage** に表示されるようになりました。thinking tokens は、最終的な回答を生成する前に、LLM が複雑な推論 step、計算、自己反省を処理するために使用する内部 token です。

  ## 修正

  * 非表示のチームのレポートが共有できていた不具合を修正しました。
  * 改行を含むテキストをコピーすると選択範囲が切り詰められることがあるUIバグを修正しました。
  * Sweeps エージェントが再実行のためにプリエンプトされたrunを取得した場合、runが別のエージェントに送られる前に、トレーニングスクリプトが起動してrunを初期化するまでの猶予が、60秒から60分に延長されました。
  * プリエンプトされた Sweeps run が再実行のためにキューに登録されない可能性がある競合状態を修正しました。
  * その名が数字で始まる場合に sweep が失敗することがあるバグを修正しました。
</Update>

  ## Security

  * 個人アカウントとサービスアカウントの両方について、セキュリティとキー管理を改善するために、W&amp;B 全体で APIキーの更新を展開しました。これらの変更は最新のセキュリティのベスト プラクティスに沿ったものであり、同時にチームにより高い柔軟性と、より安全なオートメーションを提供します。
