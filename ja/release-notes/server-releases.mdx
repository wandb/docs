<Update label="v0.78.0" description="2026年3月3日">
  W&amp;B Server 0.78.0 では、セキュリティとアクセスのデフォルト設定が強化され、Experiments を比較してコストをより効率的に管理するためのツールが追加されるなど、さまざまな改善が行われました。以下に例を示します：

  * **Models** では、正規表現を使用して複数のメトリクスのプロットを効率的に作成し、重要な run をピン留めし、Experiments をベースラインと比較し、強化された Workspace テンプレートを使用して Workspace を効率化できます。
  * **Registry** には、強化された検索機能、再設計されたカード、よりわかりやすくレスポンシブなリネージグラフなどが備わっています。
  * **Weave** では、Project Overview ページの **Trace Analytics** ダッシュボードを使って、ボトルネックやコスト要因を特定できます。保存済みビューでは、カスタマイズした Traces、Evaluations、Leaderboards を利用できるようになりました。Weave Playground には、LoRA (Low-Rank Adaptation) と新しい モデル のサポートが追加されました。
  * **推論** には新しいモデルが含まれています。

  ## サポートとサポート終了

  * v0.66.0 およびそれ以前のバージョンはサポートを終了しています。

  [リリースポリシーとプロセス](/ja/release-notes/release-policies)を参照してください。サポートが必要な場合やご質問がある場合は、[サポート](mailto:support@wandb.com)までお問い合わせください。

  ## セキュリティ

  * W&amp;B 組織またはデプロイメントでは、Personal API Keys、ユーザーごとの組織レベルの APIキー、および Service Account ごとの APIキー が、それぞれ最大 100 個まで利用できるようになりました。
  * Enterprise プランでは、組織に [メンバーを招待](/ja/platform/hosting/iam/access-management/manage-organization#invite-a-user)すると、そのユーザーにはデフォルトで **Member** ロールが割り当てられます。その他のプランでは、デフォルトのロールは **Admin** です。
  * 組織のメンバーではない新しいユーザーをチームに招待すると、その初期ロールは **メンバー** になります。
  * 組織またはチームに[招待](/ja/platform/hosting/iam/access-management/manage-organization#invite-a-user)されたユーザーについて、その初期ロールが招待テーブルに表示されるようになりました。

  ## 機能

  * [ピン留めされた run とベースライン run](/ja/models/runs/compare-runs) を使用して、重要な run を追跡して、モデルの実験を効率的に評価します。

    * 最大 6 個の run をピン留めして、Workspace 内と runs リストの先頭に表示されるようにします。ベースライン run がある場合は、そのベースラインが暗黙的にピン留めされているため、ピン留めできる run は最大 5 個です。
    * 比較の基準となる参照ポイントとして、ベースライン run を指定します。ベースライン run は常に Workspace 内と runs リストの先頭に表示されます。

    Runs テーブルでは、サマリー メトリクスの差分が、各 run がベースラインとどう異なるかを示します。

    <Frame>
      ![ベースラインとの比較における Runs テーブル内のサマリーメトリクスの差分](/images/release-notes/v0-78-0/baseline-deltas-runs-table.png)
    </Frame>

    線グラフでは、比較しやすいように、ベースラインが視覚的に他と区別できるスタイルで表示されます。

    <Frame>
      ![折れ線グラフでスタイルを区別したベースライン run](/images/release-notes/v0-78-0/baseline-line-plot-styling.png)
    </Frame>

  * 大量のメトリクスにまたがるパターンを素早く把握し、ワークスペース を効率的に整理するために、[複数メトリクスのパネルを一括で作成](/ja/models/app/features/panels/line-plot#multi-metric-line-plot) します。たとえば、勾配メトリクスをモデルの各レイヤー上に重ねて表示したり、特定のレイヤーを横に並べて配置したりできます。[デモ動画を見る](https://www.loom.com/embed/ebbb1ab3f858473cb24420bf3abf7166)。

  * [Workspace テンプレート](/ja/models/app/features/cascade-settings#workspace-layout-options) に新しい設定が追加されました。これにより、空のセクションを非表示にしたり、セクション内のパネルをアルファベット順に並べ替えたり、セクション内でのパネルの自動並べ替えの動作を調整したりできるようになりました。詳しくは、[Workspace レイアウトオプション](/ja/models/app/features/cascade-settings#workspace-layout-options) を参照してください。

  * **Sweeps** テーブルに **Sweep ID** の列が追加されました。

  * 管理者は、組織の利用状況および支出アラートをカスタマイズできるようになりました。組織設定の新しい **Usage &amp; Alerts** タブで、ストレージ、tracked hours、Weave、Inference、トレーニング の利用状況と支出を確認し、メール通知を設定できます。詳しくは [Plan usage](/ja/platform/app/settings-page/billing-settings#plan-usage) を参照してください。

  ### Registry

  * Registry UI は、より素早く操作でき、一覧性も向上しています。

    * タイプアヘッド検索の改善、再設計された Registry と Collection カード、新しい Registry アイコンにより、アセットを視覚的に区別しやすくなりました。まずはスマートなランキングを使用し、今後 AI によるランキングが追加される予定です。
    * Registry アイコンはデフォルトで自動生成され、Registry 管理者がいつでもカスタマイズできます。

    <Frame>
      ![Registry アイコンと新しいカードデザイン](/images/release-notes/v0-78-0/registry-icons-and-cards.gif)
    </Frame>

    * lineage graph 上でノードにカーソルを合わせると、そのノードがハイライトされ、他のノードが淡色表示されます。
    * コレクション名をクリックして、そのアーティファクトのバージョンをすばやく表示できるようになりました。
    * 利便性向上のため、**Settings** ページから Registry にスターを付けられるようになりました。

  ### 推論

  * モデル `Moonshot AI Kimi K2.5` が利用可能になりました。[Model Catalog](/ja/inference/models) を参照してください。

  ### Weave

  * **プロジェクト概要** ページに、リクエスト数、レイテンシーのパーセンタイル (P50/P95/P99)、モデルごとのトークン使用量、コストの内訳を確認できる **Trace Analytics** ダッシュボードが追加されました。どのモデルが予算を食いつぶしているのか、または P95 のパフォーマンスを悪化させている遅いリクエストはどれかを把握できます。時間バケットをクリックすると該当するトレースに直接ジャンプできるため、レイテンシーのスパイクの原因をすばやく特定できます。

    <Frame>
      ![プロジェクト概要ページ上の Trace Analytics ダッシュボード](/images/release-notes/v0-78-0/weave-trace-analytics-dashboard.png)
    </Frame>

  * **Weave Playground** が **[LoRA](/ja/inference/lora)** をサポートするようになり、ファインチューニングした LoRA アダプターを効率的にテストできるようになりました。LoRA アーティファクト のページで **Try in Playground** ボタンをクリックします。

  * Claude Opus 4.6 が **Weave Playground** で利用できるようになりました。

  * **Traces** テーブルおよび **Evaluations** テーブルで列の順序を変更できるようになりました。

    * テーブル内で末端の列を直接ドラッグ＆ドロップしたり、列管理メニューからセクション全体を移動したりできます。
    * 変更内容を [保存済みビュー](/ja/weave/guides/tools/saved-views) として保存できます。ビデオ デモを参照してください。

    <Frame>![動画：Traces および Evaluations テーブルの列の並び替え](/images/release-notes/v0-78-0/weave-column-reorder-demo.gif)</Frame>

  * カスタマイズした [Leaderboard](/ja/weave/guides/evaluation/dynamic_leaderboards) を、保存済みビューとして保存できるようになりました。

  * [monitors](/ja/weave/guides/evaluation/monitors) を使用すると、トレース内の音声および画像コンテンツを評価できるようになりました。

  * 集計済みの OpenTelemetry トークン数が、[OpenTelemetry traces](/ja/weave/guides/tracking/otel#send-opentelemetry-traces-to-weave) のトレース詳細サイドバーに表示されるようになりました。

  * Gemini のように thinking トークンを提供するモデルについては、Weave UI の **Token Usage** に表示されるようになりました。thinking トークンは、LLM が最終的な回答を生成する前に、複雑な推論ステップや計算、自己省察を内部的に処理するために使用するトークンです。

  ## バグ修正

  * 非公開の team の Reports を共有できてしまう不具合を修正しました。
  * 改行を含むテキストをコピーする際に選択範囲が途中で途切れてしまう UI のバグを修正しました。
  * Sweeps エージェントが再実行のためにプリエンプトされた run を取得した場合、トレーニング スクリプトが別のエージェントに run が渡される前にその run を起動して初期化できる猶予時間は、以前の 60 秒から最大 60 分に延長されました。
  * プリエンプトされた Sweeps run が再実行用のキューに登録されない可能性のあるレースコンディションを修正しました。
  * 名前が数字で始まる Sweep が失敗するバグを修正しました。
</Update>

  ## セキュリティ

  * W&amp;B 全体で API キー の運用を更新し、個人アカウントとサービスアカウントの両方におけるセキュリティとキー管理を強化しました。これらの変更は、最新のセキュリティベストプラクティスに沿いつつ、Teams に対してより高い柔軟性と、より安全な自動化をもたらします。

    **重要なポイント**

    * API キーは作成時に一度だけ表示されるため、その場ですぐにコピーして保存する必要があります。
    * 既存のキーはもうコピーできないため、新しいキーを生成する必要があります。
    * キーはいつでも名称を変更したり削除したりできます。
    * サービスアカウントで複数の API キーを利用できるようになりました。

    [サインアップして APIキーを作成する](/ja/models/quickstart#sign-up-and-create-an-api-key) を参照してください。

  ## 機能

  * **ワークスペースのナビゲーションを更新しました。** 複数 run 用ビューと単一 run 用 Workspace ビューを統一し、単一の run を詳細表示したときにも、複数 run ビューで設定したのと同じレイアウトとパネル構成が表示されるようになりました。カスタムチャート、レイアウト、設定も含まれます。これにより、コンテキストを維持しつつ、ビューの設定をやり直す手間を避けることができます。ナビゲーションボタンの位置の変更を含めた詳細は、この [動画](https://www.loom.com/share/df232057aa2d4ee78026a4f9cf400499?sid=da3661eb-e8cd-4aa5-a666-db2c4091b02b) を参照してください。
    <Frame>
      ![更新された Workspace ナビゲーションのスクリーンショット](/images/release-notes/v0-77-0/workspace-navigation-update.png)
    </Frame>
  * セクション単位での [動画再生の同期とループ](/ja/models/app/features/panels/media#sync) 機能が一般提供されています。デフォルトでは、セクション内の動画は同期して再生されます。
  * グループ化された runs のフルフィデリティのラインプロットでは、シェーディング領域に **Min/Max**、**Standard Deviation**、**Standard Error**、または **None** を表示するように設定できるようになりました。詳しくは [Sampling](/ja/models/app/features/panels/line-plot/sampling) を参照してください。
  * 平行座標プロット上で線を **Cmd+Click** (macOS) または **Ctrl+Click** (Linux / Windows) すると、対応する run の詳細ページが新しいタブで開き、さらに詳しく確認できます。グループ化されたプロットでは、この機能はまだサポートされていません。

    </Frame>
  * オートメーションの設定をより効率的に行えるよう、アプリがその基準となるメトリクスを提案するようになりました。

  ### Registry

  * [Artifact 系譜グラフ](/ja/models/artifacts/explore-and-traverse-an-artifact-graph) は Registry で、ドラッグ＆ドロップで位置を変更したり、項目を非表示にしたりできるようになりました。
  * Artifacts バージョンの表示画面から、新しいレジストリを作成し、その Artifacts バージョンを 1 回の操作でレジストリにリンクできるようになりました。

  ### Weave

  * W&amp;B App の空の状態の Weave オンボーディングページに TypeScript のサンプルコードが追加され、Python のサンプルコードと並んで表示されるようになりました。
  * Weave の呼び出しログは、デフォルトで連続スクロールをサポートするようになりました。これまでは、最新の 10,000 行のログしか表示できませんでした。
  * [Evaluation](/ja/weave/guides/core-types/evaluations) の比較ページ内の棒グラフとレーダーチャートで、サイズ変更および全画面表示が可能になりました。また、グラフ上の領域にカーソルを合わせると、詳細を表示するツールチップが表示されるようになりました。
  * [Weave Playground](https://wandb.ai/site/playground/) には新しいモデルが含まれます：
    * GPT 5.2
    * Grok 4.1
    * Opus 4.5
  * [Weave Playground](https://wandb.ai/site/playground/) から新しいプロンプトを作成して保存できるようになりました。

  ## 修正

  * run の **Notes** テキスト フィールドが編集時に狭すぎる不具合を修正しました。これにより、閲覧時と編集時でテキストボックスの幅が同じになりました。
  * W&amp;B Report を PDF にエクスポートした際に、画像の解像度が低すぎる不具合を修正しました。
  * Report 内の折れ線グラフが run セレクターの変更と同期して更新されないことがある不具合を修正しました。
  * Report 内の折れ線プロットで、外れ値を非表示にするオプションがないにもかかわらず外れ値が表示されてしまう不具合を修正しました。
  * 複数の runset を含むラインプロットで、異なる runset 間で同じ名前を持つ run に対して誤ったデータが表示される可能性があったバグを修正しました。Runs は現在、runset と名前の組み合わせによって一意に識別されるようになりました。

  ### Weave

  * Dedicated Cloud 環境の Weave UI に、Dedicated Cloud では利用できない W&amp;B Inference へのリンクが表示されていた問題を修正しました。
  * [Evaluation](/ja/weave/guides/core-types/evaluations) の比較ページで、詳細ドロワーが開いている場合に棒グラフの幅が不自然なほど細くなってしまう不具合を修正しました。
  * [Weave Playground](https://wandb.ai/site/playground/) で入力時に、カーソルが画面の表示領域から消えてしまうことがある UI の不具合を修正しました。
  * [Weave Playground](https://wandb.ai/site/playground/) で call を読み込んだときにテンプレート変数を編集できなかった不具合を修正しました。
  * [Evaluation](/ja/weave/guides/core-types/evaluations) の概要画面から子トレースの詳細ビューに移動した際に、operation セレクターが設定されない不具合を修正しました。これにより、どのトレースが選択されているのか判別しづらくなり、一部の列が空であるように見える原因となっていました。
  * [Evaluation](/ja/weave/guides/core-types/evaluations) の比較ビュー ページで **Delete** をクリックしても比較ビューが実際には削除されないバグを修正しました。
  * カスタム型の [Evaluation](/ja/weave/guides/core-types/evaluations) 結果で、本来のカスタム型ではなく、`CellValue` フィールドが空白のまま表示されてしまう不具合を修正しました。
  * 左固定されたセルが **runs as rows** 表示モードで透明に表示されることがあった不具合を修正しました。
</Update>

  * **Registry のアクセス制御に関する更新:**
    * **プログラムからの Registry アクセス制御:** [Registry roles](/ja/models/registry/configure_registry#details-about-registry-roles) が SCIM スキーマに含まれるようになりました。[ユーザーを作成](/ja/platform/hosting/iam/scim#create-user) または更新する際に、そのユーザーの Registry ロールを設定できるほか、[ユーザーを取得](/ja/platform/hosting/iam/scim#get-user) すると Registry ロールも返されます。
    * **Registry アクセスの制限:** Registry へのフルアクセスを付与したくない場合は、新しい Registry ロールである [Restricted Viewer](/ja/models/registry/configure_registry) を付与できます。このロールは、コレクション、オートメーション、その他の Registry リソースを作成・更新・削除することなく、Registry Artifacts への読み取り専用アクセスを提供します。これは、機密性の高い、または規制対象の Registry コンテンツに対して役立ちます。
