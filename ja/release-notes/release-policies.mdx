このページでは、W&amp;B Server のリリースおよび W&amp;B のリリースポリシーについて詳しく説明します。このページは、[W&amp;B 専用クラウド](/ja/platform/hosting/hosting-options/dedicated-cloud/) および [セルフマネージド](/ja/platform/hosting/hosting-options/self-managed/) デプロイに関連しています。個々の W&amp;B Server リリースの詳細については、[W&amp;B リリースノート](/ja/release-notes/)をご参照ください。

W&amp;B は [W&amp;B Multi-tenant Cloud](/ja/platform/hosting/hosting-options/multi_tenant_cloud) を完全に管理しており、このページの内容は適用されません。

<div id="release-support-and-end-of-life-policy">
  ## リリースサポートおよびサポート終了ポリシー
</div>

W&amp;B は、W&amp;B Server のメジャーリリースを初回リリース日から12か月間サポートします。

* **専用クラウド**インスタンスは、サポートを維持するために自動的に更新されます。
* **セルフマネージド**インスタンスをご利用のお客様は、サポートを維持するために適切なタイミングでアップグレードを行う責任があります。サポートされていないバージョンを使い続けることは避けてください。

  <Note>
    W&amp;B は、**セルフマネージド**インスタンスをご利用のお客様に対し、サポートを維持し最新の機能、パフォーマンス改善、および修正を受け取るために、少なくとも四半期に1回は最新リリースでデプロイを更新することを強くお勧めします。
  </Note>

<div id="release-types-and-frequencies">
  ## リリースの種類と頻度
</div>

* **メジャーリリース** は毎月リリースされ、新機能、機能強化、パフォーマンス改善、中・低重大度のバグ修正、および非推奨の予定を含む場合があります。メジャーリリースの例は `0.68.0` です。
* **パッチリリース** は、同一メジャーバージョン内で必要に応じてリリースされ、クリティカルおよび高重大度のバグ修正を含みます。パッチリリースの例は `0.67.1` です。

<div id="release-rollout">
  ## リリースのロールアウト
</div>

1. テストと検証が完了すると、まずすべての **専用クラウド** インスタンスにリリースがロールアウトされ、常に最新の状態に保たれます。
2. その後、追加の観察期間を経てリリースが公開されると、**セルフマネージド** デプロイでは各自のスケジュールでアップグレードできます。ただし、[リリース サポートおよびサポート終了 (EOL) ポリシー](#release-support-and-end-of-life-policy) に準拠するため、期限内にアップグレードする責任があります。詳しくは、[W&amp;B Server のアップグレード](/ja/platform/hosting/server-upgrade-process/)を参照してください。

<div id="downtime-during-upgrades">
  ## アップグレード中のダウンタイム
</div>

* **専用クラウド** インスタンスをアップグレードする場合、一般的にはダウンタイムは発生しない想定ですが、次のような状況では発生する可能性があります。
  * コンピュート、ストレージ、ネットワークなどの基盤インフラストラクチャーに変更が必要となる新機能や機能強化を導入する場合。
  * セキュリティ修正などの重要なインフラストラクチャー変更をロールアウトする場合。
  * インスタンスの現在のバージョンが [サポート終了 (EOL)](/ja/platform/hosting/server-upgrade-process/) に達しており、サポートを維持するために W&amp;B によってアップグレードされる場合。
* **セルフマネージド** デプロイでは、お客様の責任で、[Kubernetes 上で W&amp;B Server を実行する](/ja/platform/hosting/self-managed/operator-airgapped/) などして、サービスレベル目標 (SLO) を満たすローリングアップデートプロセスを実装する必要があります。

<div id="feature-availability">
  ## 機能の利用可能状況
</div>

インストールまたはアップグレード後、一部の機能はすぐには利用できない場合があります。

<div id="enterprise-features">
  ### Enterprise features
</div>

Enterprise licenseには、重要なセキュリティ機能やその他のエンタープライズ向け機能のサポートが含まれています。一部の高度な機能にはEnterprise licenseが必要です。

* **専用クラウド**にはEnterprise licenseが含まれており、特別な操作は不要です。
* **セルフマネージド**のデプロイでは、Enterprise licenseが必要な機能は設定が完了するまで利用できません。詳細の確認またはEnterprise licenseの取得については、セルフマネージドの要件ページの[License](/ja/platform/hosting/self-managed/requirements#license)セクションをご参照ください。

<div id="private-preview-and-opt-in-features">
  ### プライベートプレビューとオプトイン機能
</div>

ほとんどの機能は、W&amp;B Server のインストールまたはアップグレード直後に使用できます。特定の機能をインスタンスで使用するには、事前に W&amp;B チームによる有効化が必要です。

<Warning>
  プレビュー段階の機能は、いずれも変更される可能性があります。プレビュー機能が一般提供されることは保証されません。
</Warning>

* **プライベートプレビュー**: W&amp;B は、デザインパートナーやアーリーアダプターを招待してこれらの機能をテストしてもらい、フィードバックを収集します。プライベートプレビュー機能は、本番環境での使用は推奨されません。

  使用するには、事前に W&amp;B チームがお使いのインスタンスでプライベートプレビュー機能を有効化する必要があります。公開ドキュメントは提供されず、手順は個別に案内されます。インターフェースや API は変更される可能性があり、機能が完全には実装されていない場合があります。
* **パブリックプレビュー**: 一般提供前に試すには、W&amp;B に連絡してパブリックプレビューにオプトインしてください。

  お使いのインスタンスで使用するには、事前に W&amp;B チームがパブリックプレビュー機能を有効化する必要があります。ドキュメントが完全でない場合があり、インターフェースや API は変更される可能性があり、機能が完全には実装されていない場合があります。

制限事項を含む各 W&amp;B Server リリースの詳細については、[W&amp;B Release notes](/ja/release-notes/) を参照してください。
