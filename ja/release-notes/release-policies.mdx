このページでは、W&amp;B Server のリリースおよび W&amp;B のリリースポリシーの詳細を説明します。このページは、[W&amp;B Dedicated Cloud](/ja/platform/hosting/hosting-options/dedicated-cloud/) および [Self-Managed](/ja/platform/hosting/hosting-options/self-managed/) デプロイメントに関する内容です。各 W&amp;B Server リリースの詳細については、[W&amp;B release notes](/ja/release-notes/) を参照してください。

W&amp;B は [W&amp;B Multi-tenant Cloud](/ja/platform/hosting/hosting-options/multi_tenant_cloud) を完全に管理しており、このページの内容は対象外です。

<div id="release-support-and-end-of-life-policy">
  ## リリースサポートおよびサポート終了ポリシー
</div>

W&amp;B は、W&amp;B Server のメジャーリリースを初回リリース日から 12 か月間サポートします。

* **Dedicated Cloud** インスタンスは、サポートを維持するため自動的に更新されます。
* **Self-Managed** インスタンスを利用しているお客様は、サポートを維持するためにサポート期間内にアップグレードする責任があります。サポート対象外のバージョンを使い続けないようにしてください。

  <Note>
    W&amp;B は、**Self-Managed** インスタンスを利用しているお客様に対し、サポートを維持し、最新の機能やパフォーマンス改善、修正を受け取るために、少なくとも四半期に一度は最新リリースに合わせてデプロイメントを更新することを強く推奨します。
  </Note>

<div id="release-types-and-frequencies">
  ## リリースの種類と頻度
</div>

* **メジャーリリース** は毎月リリースされ、新機能、機能強化、パフォーマンス向上、中・低重大度のバグ修正、および非推奨の予定を含む場合があります。メジャーリリースの例は `0.68.0` です。
* **パッチリリース** は、同一メジャーバージョン内で必要に応じてリリースされ、クリティカルおよび高重大度のバグ修正を含みます。パッチリリースの例は `0.67.1` です。

<div id="release-rollout">
  ## リリースのロールアウト
</div>

1. テストと検証が完了すると、まずすべての **Dedicated Cloud** インスタンスに対してリリースをロールアウトし、常に最新の状態に保ちます。
2. 一定期間の追加のモニタリングを行った後、リリースが公開され、**Self-Managed** デプロイメントは各自のスケジュールでアップグレードできますが、[リリースサポートおよびサポート終了 (EOL) ポリシー](#release-support-and-end-of-life-policy) に準拠するため、適切なタイミングでアップグレードする責任があります。詳しくは [W&amp;B Server のアップグレード](/ja/platform/hosting/server-upgrade-process/) を参照してください。

<div id="downtime-during-upgrades">
  ## アップグレード中のダウンタイム
</div>

* **Dedicated Cloud** インスタンスをアップグレードする場合、一般的にはダウンタイムは発生しない想定ですが、次のような状況では発生する可能性があります。
  * コンピュート、ストレージ、ネットワークなどの基盤インフラに変更が必要となる新機能や拡張を導入する場合。
  * セキュリティ修正などの重要なインフラ変更をロールアウトする場合。
  * インスタンスの現在のバージョンが [End of Life (EOL)](/ja/platform/hosting/server-upgrade-process/) に達しており、サポートを維持するために W&amp;B によってアップグレードされる場合。
* **Self-Managed** デプロイメントでは、お客様の責任で、[Kubernetes 上で W&amp;B Server を実行する](/ja/platform/hosting/self-managed/operator-airgapped/) などして、サービスレベル目標 (SLO) を満たすローリングアップデートプロセスを実装する必要があります。

<div id="feature-availability">
  ## 機能の提供状況
</div>

インストールまたはアップグレード後、一部の機能はすぐには利用できない場合があります。

<div id="enterprise-features">
  ### Enterprise 機能
</div>

Enterprise ライセンスには、重要なセキュリティ機能とその他のエンタープライズ向け機能のサポートが含まれます。一部の高度な機能には Enterprise ライセンスが必要です。

* **Dedicated Cloud** には Enterprise ライセンスが含まれているため、追加の対応は不要です。
* **Self-Managed** デプロイメントでは、Enterprise ライセンスが付与されるまで、Enterprise ライセンスを必要とする機能は利用できません。詳細や Enterprise ライセンスの取得方法については、Self-Managed Requirements ページの [License](/ja/platform/hosting/self-managed/requirements#license) セクションを参照してください。

<div id="private-preview-and-opt-in-features">
  ### プライベートプレビューとオプトイン機能
</div>

ほとんどの機能は、W&amp;B Server をインストールまたはアップグレードした直後から利用できます。特定の機能については、インスタンスで利用可能にする前に W&amp;B チームによる有効化が必要です。

<Warning>
  プレビュー段階にある機能は変更される可能性があります。プレビュー機能が一般提供されることは保証されません。
</Warning>

* **プライベートプレビュー**: W&amp;B はデザインパートナーおよびアーリーアダプターを招待し、これらの機能をテストしてフィードバックを提供してもらいます。プライベートプレビュー機能は本番環境での使用は推奨されません。

  プライベートプレビュー機能を使用するには、そのインスタンスに対して W&amp;B チームが機能を有効化する必要があります。公開ドキュメントは提供されず、手順は個別に提供されます。インターフェースや API は変更される可能性があり、機能が完全には実装されていない場合があります。
* **パブリックプレビュー**: 一般提供前に試用するためにパブリックプレビューへオプトインするには、W&amp;B にお問い合わせください。

  パブリックプレビュー機能を使用するには、そのインスタンスに対して W&amp;B チームが機能を有効化する必要があります。ドキュメントは未完成の場合があり、インターフェースや API は変更される可能性があり、機能が完全には実装されていない場合があります。

個々の W&amp;B Server リリースの詳細や制限事項については、[W&amp;B Release notes](/ja/release-notes/) を参照してください。
