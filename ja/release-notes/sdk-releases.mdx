このページは W&amp;B Python SDK (`wandb` パッケージ) のリリースノートです。W&amp;B Server については [W&amp;B Server リリースノート](/ja/release-notes/server-releases) を参照してください。

<Update label="0.25.0" description="February 12, 2026">
  ## 主な変更点

  * Python 3.8 のサポートを終了しました。

  ## 追加

  * `wandb beta leet` TUI が複数 run のワークスペースをサポートするようになりました。
  * `wandb beta leet config` コマンドを使って、`wandb beta leet` の永続化された設定を対話的に設定できるようになりました。
  * `wandb.apis.public.Project` の `owner` プロパティで、プロジェクト所有者のユーザー情報にアクセスできるようになりました。

  ## 修正

  * 削除された sweep に対する sweep エージェントが、404 エラーを繰り返しながら無期限に実行され続ける可能性があったバグを修正しました。
  * `wandb-core` のクラッシュが、古い Python バージョンで非常に長く反復的なトレースバックを出力することがあったバグを修正しました。
  * v0.24.0 で導入されたリグレッションを修正しました。このリグレッションにより、TensorBoard の同期が 1 MB のデータを同期した時点で停止することがありました。
</Update>

<Update label="0.24.2" description="February 4, 2026">
  ## 追加

  * `wandb.Api()` が Federated Auth (JWT ベースの認証) をサポートするようになりました。

  ## 修正

  * Artifacts ファイルのダウンロード中に有効期限が切れた場合、署名付きダウンロード URL を再取得するようになりました。
</Update>

<Update label="0.24.1" description="January 29, 2026">
  ## 主な変更点

  * `wandb==0.24.0` で作成された Runs では一部のデータのアップロードに失敗する可能性がありましたが、このリリースで修正されています。欠落したデータは run の `.wandb` ファイルに保存されており、`wandb sync` で再アップロードできます。

  ## 追加

  * Parquet ファイル形式でエクスポートされた run history をダウンロードするために、`api.Run` クラスに `download_history_exports` を追加しました。

  ## 変更

  * 設定ファイル (`./wandb/settings` や `~/.config/wandb/settings` など) に無効な設定が含まれている場合、すべての設定ファイルが無視され、エラーが出力されるようになりました。

  ## 修正

  * `wandb login --host <invalid-url>` の後に `wandb login --host <valid-url>` を使用すると、通常どおり動作するようになりました。
  * `wandb beta sync` が `Syncing...` の状態のまま止まり、認証情報の読み込みに失敗する可能性があったバグを修正しました。
  * アップロードされないデータが発生する場合があったバグを修正しました。現在は、すべてのデータが run の `.wandb` ファイルに保存され、`wandb sync` で再アップロードできます。

  これらのリグレッションは 0.24.0 で導入されました。
</Update>

<Update label="0.24.0" description="January 13, 2026">
  <Warning>
    このリリースは PyPI から削除されています。このバージョンは使用しないでください。このバージョンを使用している場合は、直ちに v0.24.1 以降のバージョンにアップグレードしてください。v0.24.1 で修正されたリグレッションにより、`wandb==0.24.0` で作成された run では一部のデータのアップロードに失敗する可能性があります。
  </Warning>

  ## 主な変更点

  このバージョンでは、レガシーかつ非推奨となっていた `wandb.beta.workflows` モジュールと、その中の `log_model()` / `use_model()` / `link_model()` 関数が削除されました。これは正式な破壊的変更です。

  ## 追加

  * `wandb agent` と `wandb.agent()` で、エージェントからその sweep の子 run へ SIGINT/SIGTERM などの捕捉可能なシグナルを転送するための `forward_signals` フラグ (CLI: `--forward-signals/-f`) を受け付けるようになり、エージェント プロセスを中断した際に、よりクリーンにシャットダウンできるようになりました。
  * `wandb beta sync` が、ログ記録中の run を同期するための `--live` オプションをサポートするようになりました。

  ## 削除

  * 非推奨だった `wandb.beta.workflows` モジュールと、その中の `log_model()`、`use_model()`、`link_model()` 関数を削除しました。これらの最新の後継は、それぞれ `Run.log_artifact`、`Run.use_artifact`、`Run.link_artifact` メソッドです。

  ## 修正

  * 例外が発生しなかった場合に渡される `None` 値を受け付けられるように、`Run.__exit__` の型アノテーションを修正しました。
  * `random.seed()` を呼び出した後に Artifacts を作成すると発生していた `Invalid Client ID digest` エラーを修正しました。ランダム状態を決定論的にシードした場合に Client ID が衝突する可能性がありました。
  * 空の Artifacts を一覧表示する際に発生していた CLI エラーを修正しました。
  * Sweeps run に対して `api.run()` を呼び出した場合のリグレッションを修正しました。
  * run の終了時に表示される「View run at」メッセージに、URL が含まれないことがあった問題を修正しました。
  * `wandb.Api()` からクエリされた Runs が、VSCode ノートブック内で壊れた HTML ウィンドウではなく、文字列表現として表示されるようになりました。
</Update>

<Update label="v0.23.1" description="2025年12月2日">
  ## 追加

  * `wandb beta leet` コマンドのメトリクスと run 概要フィルターで正規表現がサポートされました。
  * `wandb beta leet` コマンドでチャートのインスペクションが可能になりました。右クリックしてドラッグすると、最も近いデータポイントの (x, y) を表示します。Alt キーを押し続けると、表示中のすべてのチャートでインスペクションが同期されます。
  * Automations API で、run の状態をトリガーとする automation を作成・編集できるようになりました。
  * Automations API で、基本的な zscore automation イベントがサポートされました。
  * `wandb.Api` による履歴スキャンのパフォーマンスを向上させるため、`Run` オブジェクトに `beta_history_scan` メソッドが追加されました。

  ## 変更

  * `wandb.Api()` は、`WANDB_IDENTITY_TOKEN_FILE` が設定されていて明示的な API キーが指定されていない場合、`UsageError` を送出するようになりました。

  ## 非推奨

  * 匿名モードは非推奨となり、警告を出力するようになりました。この非推奨には、`anonymous` 設定、`WANDB_ANONYMOUS` 環境変数、`wandb.init(anonymous=...)`、および `wandb login --anonymously` と `wandb.login(anonymous=...)` コマンドが含まれます。

  ## 修正

  * `wandb.Image()` は非推奨警告を出力しなくなりました。
  * `Registry.description` と `ArtifactCollection.description` は空文字列を拒否しなくなりました。
  * `Artifact` オブジェクトのインスタンス化が大幅に高速になりました。
  * シンボリックリンクが無効または利用できない場合（例: ボリュームをまたぐ場合や Windows で Developer Mode が無効な場合）、`wandb.Run.save()` はハードリンク、その後必要に応じてコピーにフォールバックするようになりました（該当する場合、ファイルポリシーを「live」から「now」にダウングレードします）。
  * Artifact コレクションのエイリアスは、`ArtifactCollection` のインスタンス化時ではなく `ArtifactCollection.aliases` にアクセスしたときに遅延取得されるようになり、`Api.artifact_collections()` や `Api.registries().collections()` などのパフォーマンスが向上しました。
  * ログイン時には、`wandb.init(settings=wandb.Settings(api_key="..."))` で明示的に指定された API キーが使用されるようになりました。`run.log_artifact` で Artifacts をログする場合は、その run のキーが使用されます。
  * W&amp;B LEET TUI が、負の Y 軸目盛り値や特定のシステムメトリクスの基準単位／表示単位を正しく表示するようになりました。
  * `console_capture.py` における稀な無限ループを修正しました。
  * ファイルのアップロード／ダウンロードで、[reference artifacts](/ja/models/artifacts/track-external-files) を除き、`WANDB_X_EXTRA_HTTP_HEADERS` が尊重されるようになりました。
</Update>

<Update label="v0.23.0" description="2025年11月11日">
  ## 追加

  * 実験的な `wandb beta leet` コマンドを追加しました。Lightweight Experiment Exploration Tool であり、ローカルで W&amp;B runs を表示し、リアルタイムのメトリクス可視化とシステムモニタリングを行うためのターミナル UI です。
  * registry API で、個々の registry のユーザーおよびチームメンバーをプログラムから管理できるようになりました。
  * `Registry.id` が `Registry` オブジェクトの（読み取り専用）プロパティとして追加されました。

  ## 修正

  * `Artifact.files()` は、`names` パラメーターでフィルタリングした際に、`len()` が正しい値を返すようになりました。
  * ファイルアップロードの進捗表示において、現在値が総ファイルサイズをまれに超えてしまう問題を修正しました。
  * `Artifact.link()` は、エラーを送出する代わりに未保存の Artifacts をログするようになり、`Run.link_artifact()` の動作と一貫性が取れるようになりました。
  * VSCode の Jupyter notebook 拡張機能で ipython notebook を実行している場合でも、自動コード保存が動作するようになりました。
  * `Artifact.metadata` に無限大の浮動小数点数を含む Artifacts をログする場合、リクエスト再試行のタイムアウトを待つのではなく、早い段階で `ValueError` を送出するようになりました。
</Update>

<Update label="v0.22.3" description="2025年10月28日">

  * `wandb.Api(api_key=...)` が、スレッドローカルにキャッシュされた認証情報よりも、明示的に渡された API キーを優先するようになりました。

  * `artifact.add_reference` を介した GCS フォルダ参照のアップロードを修正しました。
  * SDK が Jupyter セッションにおけるノートブックパスを、サーバーのルートディレクトリを使用して正しく推論するようになり、その結果、サブディレクトリ（例: `code/nested/<notebook>.ipynb`）内でもコードの保存が機能するようになりました。

<Update label="v0.22.1" description="2025年9月29日">
  ## 追加

  このバージョンでは、これまで `Artifact.link()` または `Run.link_artifact()` の呼び出し中に抑制されていたエラーが送出されるようになりました。これにより、これらのメソッドにおける見落とされた失敗は防止されますが、非互換な変更でもあります。

  * run の終了時に出力される &quot;Run history&quot; と &quot;Run summary&quot; は、それぞれメトリクス 10 個までに制限されるようになりました。

  * サーバーバージョン 0.74.0 以降では、`Artifact.link()` および `Run.link_artifact()` が、不要なブロッキング GraphQL リクエストを 4〜5 回削減することで、より高速に動作するようになりました。

  GitHub 上の [v0.21.2 の変更履歴](https://github.com/wandb/wandb/releases/tag/v0.21.2)を参照してください。
</Update>

<Update label="v0.21.1" description="2025年8月7日">
  ## 主な変更点

  `Api().runs(...)` と `Api().sweeps(...)` のデフォルトの並び順は、run の `created_at` 時刻に基づく昇順になりました。

  ## 追加

  * `define_metric` において `first` サマリーオプションをサポート。
  * ページネーションされた sweep をサポート。
  * W&amp;B backend から指定されたパターンに一致するファイルのみを取得するための、`Api().run().files` への `pattern` パラメーターを追加。
  * シークレット形式の文字列を指定するために、Launch 入力 JSON Schema に任意の `format` キーを追加。

  ## 変更

  * `Sweep.name` プロパティは、利用可能な場合はユーザーが編集した表示名を返すようになりました（これがない場合は従来どおり sweep config の元の名前、さらにそれもない場合は sweep ID にフォールバックします）。
  * `Api().runs(...)` と `Api().sweeps(...)` は、run の `created_at` 時刻に従った昇順で run を返すようになりました。
  * 大きなファイル（&gt;2GB）を含む Artifacts のアップロードは、コア数の多いシステムで並列ハッシュを使用することで高速化されました。
  * Python の遅延イテレーターに合わせるため、レジストリイテレーターに対する `__bool__` の実装を削除しました。

  ## 非推奨

  * `wandb.beta.workflows` モジュールとその内容（`log_model()`, `use_model()`, `link_model()` を含む）は非推奨となり、今後のリリースで削除される予定です。

  ## 修正

  * organization registry の Artifacts に対して、 Artifacts タイプに依存しない Artifact の URL になるよう修正しました。
  * サニタイズされた `InternalArtifact` 名のサフィックスを、英数字 6 文字に短縮しました。
  * `WANDB_SILENT` / `WANDB_QUIET` 環境変数が設定されている場合、`wandb.Video` は動画エンコード中にプログレススピナーを表示しなくなりました。
  * `api.registries()` を使用して取得した registry に、name および full&#95;name フィールドに余分な `wandb-registry-` 接頭辞が付く問題を修正しました。
  * `sync_tensorboard` 使用時に発生する可能性があったクラッシュを修正しました。
  * `Api().run(...).upload_file` は、指定されたルートディレクトリからの相対パスが異なる場所にあるファイルをアップロードしてもエラーを投げなくなりました。
  * public API の run オブジェクトに対して `load()` 関数を呼び出しても、もはや `TypeError` は発生しません。
  * `wandb.agent()` API によって呼び出された Sweeps の run 関数が例外をスローした場合、その例外は run のログページに表示されるようになりました（以前は `wandb agent` CLI コマンドによって呼び出された run のみが対象でした）。

  GitHub 上の [v0.21.1 changelog](https://github.com/wandb/wandb/releases/tag/v0.21.1) を参照してください。
</Update>

<Update label="v0.21.0" description="2025年7月1日">
  ## Notable Changes

  このバージョンでは、`service` プロセスのレガシー実装を削除しました。これは互換性を破る変更（ブレイキングチェンジ）です。

  ## Added

  * `x_stats_pid` を起点として、プロセスツリー全体で使用されている RSS、CPU%、スレッド数などのプロセス固有のメトリクスを追跡するための設定 `x_stats_track_process_tree` を追加しました。これはコストが高くなる可能性があり、デフォルトでは無効になっています。
  * run を再開するときに、Notes がクライアントに返されるようになりました。
  * API を通じてカスタム Vega チャートプリセットを作成する機能を追加しました。ユーザーはカスタムチャート仕様を定義してアップロードし、その後 wandb.plot&#95;table() を使って run 間で再利用できるようになりました。

  ## Changed

  * `Artifact.link()` の呼び出し時に、一時的なプレースホルダー run を生成しないようになりました。
  * `wandb` は、localhost で待ち受ける代わりに IPC 用に Unix ソケットの利用を試みるようになり、（Databricks などの）より制限の厳しい権限環境でも動作するようになりました。
  * `Api.artifact()` は、移行済みの model registry コレクションから Artifacts を取得する際に警告を表示するようになりました。
  * `wandb.Api` からクエリされたオブジェクトの `.length` は非推奨となりました。代わりに `len(...)` を使用してください。

  ## Removed

  * `service` プロセスのレガシーな Python 実装を削除しました。`wandb.require` の `legacy-service` オプションおよび、対応する環境変数とともに `x_require_legacy_service` と `x_disable_setproctitle` 設定が削除され、使用した場合はエラーを発生させるようになりました。

  * プライベート属性 `wandb.Run._metadata` を削除しました。自動検出される CPU と GPU の数や GPU タイプを上書きするには、新しい設定 `x_stats_{cpu_count,cpu_logical_count,gpu_count,gpu_type}` を使用してください。

  ## Fixed

  * 参照 Artifacts 用に s3 形式の CoreWeave URI を許可しました。
  * 大量のデータをログした後に Ctrl+C が効かなくなるまれなバグを修正しました。
  * `Run` インスタンスに渡された `silent`、`quiet`、`show_warnings` 設定が、service プロセスが出力する警告にも適用されるようにしました。
  * `api.Runs` は、W&amp;B から読み込まれる各 run ごとに API コールを行わないようになりました。
  * 環境変数からの `x_extra_http_headers` 設定を正しくパースするようにしました。
  * データがまだ読み込まれていない場合、`.length` は `None` を返すのではなく、オブジェクトの長さを読み込むために W&amp;B バックエンドを呼び出すようになりました。

  GitHub 上の [v0.21.0 の変更履歴](https://github.com/wandb/wandb/releases/tag/v0.21.0) を参照してください。
</Update>

<Update label="v0.20.0" description="June 03, 2025">
  ## 主な変更点

  このバージョンでは、`service` プロセスを無効化する機能が削除されました。これは後方互換性のない変更です。

  ## 追加

  * 以前に追加された Artifacts ファイルを上書きできるようにするため、`Artifact.add_dir` に `merge` パラメーターを追加しました。
  * `wandb.Image` オブジェクトを作成する際の `masks` および `boxes` パラメーターで PyTorch の tensor をサポートしました。
  * `sync_tensorboard` は、S3、GCS、Azure に保存された tfevents ファイルの同期をサポートするようになりました。
    * GCS のパスは `gs://bucket/path/to/log/dir` 形式を使用し、アプリケーション デフォルト認証情報に依存します。これは `gcloud auth application-default login` を使用して構成できます。
    * S3 のパスは `s3://bucket/path/to/log/dir` 形式を使用し、`aws configure` で設定されたデフォルト認証情報に依存します。
    * Azure のパスは `az://account/container/path/to/log/dir` 形式および `az login` の認証情報を使用しますが、加えて `AZURE_STORAGE_ACCOUNT` と `AZURE_STORAGE_KEY` 環境変数を設定する必要があります。他の環境変数もサポートされています。詳細は [こちら](https://pkg.go.dev/gocloud.dev@v0.41.0/blob/azureblob#hdr-URLs) を参照してください。
  * 一部の Media オブジェクトを `pathlib.Path` で初期化するためのサポートを追加しました。
  * トランザクション ログをスキップできる新しい設定 `x_skip_transaction_log` を追加しました。注意: 復旧可能性に関する保証がなくなるため、慎重に使用してください。
  * NumPy 配列または PyTorch tensor で初期化された画像のピクセル値を正規化するために、`wandb.Image` 初期化時の `normalize` パラメーターを追加しました。

  ## 変更

  * 不正な型の引数が渡された場合に、さまざまな API が `ValueError` やその他の汎用的なエラーではなく `TypeError` を送出するようになりました。
  * さまざまな Artifacts および Automations API が、サーバー エラーに遭遇した際にサーバー側のエラーメッセージをそのまま提示できるよう、`ValueError` ではなく `CommError` を送出するようになりました。
  * `wandb.sdk.wandb_run.Run::save` メソッドは、`glob_str` 引数が必須になりました。

  ## 削除

  * `service` プロセスを無効化するサポートを削除しました。`x_disable_service`/`_disable_service` 設定および `WANDB_DISABLE_SERVICE`/`WANDB_X_DISABLE_SERVICE` 環境変数は非推奨となり、使用するとエラーを送出するようになりました。
  * `wandb` をインポートしただけの状態で `wandb.docker` を使用できる機能を削除しました。
    * `wandb.docker` は `wandb` のパブリック インターフェースの一部ではなく、後方互換性のない変更が行われる可能性があります。使用しないでください。
  * no-op の `sync` 引数を `wandb.Run::log` 関数から削除しました。
  * 非推奨だった `wandb.sdk.wandb_run.Run.mode` プロパティを削除しました。
  * 非推奨だった `wandb.sdk.wandb_run.Run::join` メソッドを削除しました。

  ## 非推奨

  * `start_method` 設定は非推奨となり、もはや効果はありません。安全に無視されます。
  * `Artifact.use_as` プロパティおよび `run.use_artifact()` の `use_as` パラメーターは、W&amp;B Launch では使用されていなかったため非推奨になりました。

  ## 修正

  * `wandb.setup()` を呼び出したプロセスの子プロセス内で `wandb.teardown()` を呼び出しても、`WandbServiceNotOwnedError` が送出されなくなりました。
    * このエラーは、W&amp;B Sweeps 使用時に発生する可能性がありました。
  * 分岐 (fork または rewind) が要求されたオフライン run が正しく同期されるようになりました。
  * Job の wait&#95;until&#95;running メソッド内で例外を送出する際、その例外を文字列としてログに記録するようになりました。
  * TensorFlow データで初期化された `wandb.Image` が、NumPy 配列で初期化された場合とは異なる方法で正規化されていた問題を修正しました。
  * `wandb login` を使用しても、`wandb.require("legacy-service")` に関する警告が表示されなくなりました。
  * `Table` (または内部 Artifacts を作成するその他のオブジェクト) を、ID に特殊文字を含む run からログしても、`ValueError` が送出されなくなりました。
  * `base_url` を指定して初期化した `wandb.Api` が、最後にログインした URL ではなく、指定した URL を正しく使用するようになりました。

  GitHub 上の [v0.20.0 changelog](https://github.com/wandb/wandb/releases/tag/v0.20.0) を参照してください。
</Update>
