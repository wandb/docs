<Update label="v0.67.0" description="2025年3月28日">
  ## 機能

  * Reports では、各パネルグリッドごとに run にカスタムの表示名を付けられるようになりました。これにより、run の (多くの場合、長くて分かりにくい) トレーニング時の名を、対象ユーザーにとってより意味のあるものに置き換えられます。レポートではすべてのパネルグリッドでその名が更新されるため、苦労して得た実験の知見を、簡潔で読みやすい形で同僚に伝えやすくなります。元の run 名はプロジェクト内でそのまま維持されるため、これを行っても共同作業者の作業を妨げることはありません。
  * Workspace でパネルを展開すると、より広いスペースを使える全画面モードで開くようになりました。この表示では、折れ線グラフが最大 10,000 bins を使用して、より細かな粒度で表示されるようになりました。run selector はパネルの横に表示されるため、コンテキスト内で run を簡単に Toggle、グループ化、またはフィルターできます。
  * 任意のパネルから、そのパネルのフルスクリーン表示に直接リンクする一意のURLをコピーできるようになりました。これにより、プロット内の興味深いパターンや異常なパターンを詳しく調べるためのリンクをこれまで以上に簡単に共有できます。
  * Run Comparer は、重要なrunの設定と主要なメトリクスを損失曲線と並べて比較するために使用できる強力なツールです。Run Comparer が更新されました:
    * **Add Panels** の展開オプションとして Run Comparer panel を追加する操作が、より速くなりました。
    * デフォルトでは、Run Comparer panel はより広いスペースを占めるため、値をすぐに確認できます。
    * Run Comparer panel の読みやすさと視認性を改善しました。新しいコントロールを使用して行と列のサイズをすばやく変更できるため、長い値やネストされた値も確認しやすくなります。
    * パネル内の任意の値を、ワンクリックでクリップボードにコピーできます。
    * 正規表現を使用してキーを検索し、比較したいメトリクスのサブセットをすばやく正確に見つけることができます。ビュー間で効率的に試行錯誤できるように、検索履歴が保存されます。
  * セグメンテーションマスクのコントロールが更新されました:
    * 各マスクタイプのオン/オフを一括で切り替えたり、すべてのマスクまたはすべての画像のオン/オフを切り替えたりできるようになりました。
    * 各クラスに割り当てられた色を変更できるようになり、複数のクラスが同じ色を使用している場合の混乱を避けるのに役立ちます。
  * メディアパネルをフルスクリーンモードで開くと、step スライダーを最初にクリックしなくても、キーボードの左右の矢印キーを使用して画像を順に切り替えられるようになりました。
  * メディアパネルでは、run selector に合わせて run 名が色分けされるようになりました。これにより、run のメディア値を関連するメトリクスやプロットと対応付けやすくなります。
  * run selector で、run が特定のメディア Key を持っているかどうかでフィルターできるようになりました。
  * W&amp;B App UI で Runs をグループ間で移動できるようになり、run のログ後に新しいグループを作成できるようになりました。
  * Automations は UI で編集できるようになりました
  * オートメーションで、アーティファクトのイベントについて Slack チャンネルに通知できるようになりました。オートメーションを作成する際は、アクション タイプとして「Slack notification」を選択します。
  * Registry はデフォルトでグローバル検索をサポートするようになり、Registry名、コレクション名、별칭、またはタグで、すべてのRegistryを横断して検索できるようになりました。
  * `runs` 式を使用する Tables と Query panel では、新しい Runs History の step slider とドロップダウン コントロールを使用して、run の各 step でのメトリクスの表を表示できます。
  * W&amp;B Weave の Playground で、新しいモデル `gpt-4.5-preview` (OpenAI) および `deepseek-chat` と `deepseek-reasoner` (Deepseek) がサポートされるようになりました。
  * Weave トレースには、2 つの新しいエージェント フレームワークのインテグレーションがあります: CrewAI と OpenAI の Agent SDK。
  * Weave UI で、トレースから Datasets を作成できるようになりました。[詳細はこちら](/ja/weave/guides/core-types/datasets#create-edit-and-delete-a-dataset-in-the-ui)
  * Weave Python SDK では、機密データがネットワーク境界の外部に出ないように、Weave データの入力と出力をフィルターする方法を利用できるようになりました。機密データをマスクするように設定できます。詳細は次をご覧ください: /weave/guides/tracking/redact-pii/
  * エクスペリエンスを簡素化するため、個々の run の Workspace ビューにある System タブは、今後のレポートで削除されます。システム メトリクスの完全な情報は、Workspace の System セクションで確認してください。ご不明な点がある場合は、[support@wandb.com](mailto:support@wandb.com) までお問い合わせください。

  ## Security

  * `golang crypto` は v0.36.0 に更新されました。
  * `golang oauth2` が v0.28.0 にアップグレードされました。
  * Weave では、`pyarrow` が v17.0.0 に固定されるようになりました。

  ## パフォーマンス

  * フロントエンドの更新により、重要なデータを訪問間でブラウザーキャッシュに保存することで、ワークスペースの再読み込み時間が大幅に短縮されます。この更新では、保存済みビュー、メトリクス名、run selector、run 数、W&amp;B の設定の詳細、およびワークスペースビューの再計算の読み込みが最適化されます。
  * Registry の概要ページの読み込みが大幅に高速化されました。
  * 数千件のRunsまたは数百個のメトリクスがあるWorkspaceで、散布プロットのX、Y、またはZの値に使用するメトリクスを選択する際のパフォーマンスを改善しました。
  * Weave 評価ログのパフォーマンス改善。

  ## 修正

  * Reports で、レポート内のセクションへのリンクをたどってもそのセクションが開かない不具合を修正しました。
  * ガウシアンスムージングにおけるインデックス反射の動作を改善し、SciPy のデフォルトの &quot;reflect&quot; モードに一致するようにしました。
  * メールで送信されたレポートのコメントリンクから、コメントを直接開けるようになりました。
  * `int32` ではなく `int64` を使用するように sweep の compute seconds の変数タイプを変更し、sweep の compute seconds が 20 億を超えると Workspace がクラッシュする可能性がある不具合を修正しました。
  * レポートに複数のrun セットが含まれている場合に発生する可能性があった表示の不具合を修正しました。
  * アルファベット順に並べ替えられたセクションで、Quick add で追加したパネルが正しく並べ替えられないバグを修正しました。
  * 不正な形式のユーザー招待リンクが生成されるバグを修正しました。
</Update>

<Update label="v0.61.0" description="2024年10月17日">
  ## 機能

  **これはミニ機能およびパッチ リリースであり、毎月の W&amp;B server major releases とは異なるスケジュールで提供されます**

  * 組織管理者は、Organization Dashboard から Models の seat と、Models および [W&amp;B Weave](/ja/weave/) の両方のアクセス制御をシームレスに設定できるようになりました。この変更により、専用クラウド または セルフマネージド インスタンスで [Weave](/ja/weave/) が有効になっている場合に、効率的なユーザー管理が可能になります。
    * [Weave](/ja/weave/) の pricing は、使用した seat 数ではなく消費量 기준으로 です。seat 管理は Models プロダクトにのみ適用されます。
  * [team と restricted スコープの project に対する project レベルの access roles](https://docs.wandb.ai/platform/hosting/iam/access-management/restricted-projects/) を設定できるようになりました。これにより、同じ チーム 内の異なる プロジェクト で、1 人の ユーザー に異なるアクセスロールを割り当てることができ、エンタープライズ ガバナンス要件に準拠するための強力な制御をさらに追加できます。

  ## 修正

  * リリースのアップグレードの一環として行われる基盤データベース スキーマの変更が、プラットフォームの起動時にタイムアウトすることがある問題を修正しました。
  * 基盤となる parquet store service にさらなる パフォーマンス改善 を追加し、ユーザーのチャート読み込み時間をさらに改善しました。Parquet store service は、専用クラウド と、[W&amp;B kubernetes operator](https://docs.wandb.ai/platform/hosting/operator) に基づく セルフマネージド インスタンスでのみ 使用可能です。
  * 基盤となる parquet store service の高い CPU 利用率の問題に対処し、効率的なチャート読み込みの信頼性を高めました。Parquet store service は、専用クラウド と、[W&amp;B kubernetes operator](https://docs.wandb.ai/platform/hosting/operator) に基づく セルフマネージド インスタンスでのみ 使用可能です。
</Update>

<Update label="v0.60.0" description="2024年9月26日">
  <Note>
    リリースのバージョン管理上の問題により、0.60.0 は 0.58.0 の次の major release です。0.59.0 は 0.58.0 向けパッチ リリースの 1 つでした。

  ## 機能

  * Web Content Accessibility Guidelines (WCAG) standards の Level AA 2.2 の 1.1.1 Compliance に関する最終更新。
  * W&amp;B は、W&amp;B kubernetes operator を使用する セルフマネージド インスタンスで auto-version-upgrade をオフにできるようになりました。これは your W&amp;B team に依頼できます。
    W&amp;B では、各バージョンの 6 か月の end-of-life 期間に準拠するため、すべてのインスタンスで定期的なアップグレードが必要です。W&amp;B は 6 か月より古いバージョンをサポートしていません。

  ## 修正

  * 専用クラウド および セルフマネージド インスタンスの instance admins が personal entities 内の Workspace にアクセスできるようにする bug を修正しました。
  * SCIM Groups および Users の GET endpoints は、応答 から サービスアカウント を除外してフィルタリングするようになりました。これらの endpoints から返されるのは、サービスアカウントではない ユーザー のみになりました。
  * チーム管理者 が、ユーザー を チーム から削除するのと同時に、インスタンス全体からも削除できる機能を削除することで、ユーザー管理の bug を修正しました。インスタンスまたは 組織管理者 が、インスタンス全体 / 組織 から ユーザー を削除する責任を負います。

  ## パフォーマンス改善

  * 多数の メトリクス がある Workspace でパネルを追加する際のレイテンシを最大 90% 削減しました。
  * run が頻繁に Resume される場合の、blob storage への parquet エクスポートの信頼性とパフォーマンスを改善しました。
    parquet 形式での Runs の blob storage へのエクスポートは、専用クラウド と、W&amp;B kubernetes operator を使用して有効化された セルフマネージド インスタンスで利用できます。
</Update>

<Update label="v0.58.1" description="2024年9月4日">
  ## 機能

  * W&amp;B は **セキュアストレージコネクタ、つまり Bring your own bucket** 機能でサブパスをサポートするようになりました。これにより、インスタンスレベルまたはチームレベルでバケットを設定する際にサブパスを指定できるようになりました。これは新しいバケット設定でのみ使用可能で、既存の設定済みバケットでは使用できません。
  * Google Cloud と Azure の新しい 専用クラウド インスタンス上の W&amp;B 管理ストレージは、デフォルトで **W&amp;B managed cloud-native keys** により暗号化されるようになりました。これは AWS インスタンスではすでに利用可能です。各インスタンスのストレージは、そのインスタンス固有のキーで暗号化されます。これまでは、Google Cloud と Azure 上のすべてのインスタンスが、デフォルトのクラウドプロバイダー管理の暗号化キーに依存していました。
  * run config と summary のフィールドが、クリックでコピー可能になりました。
  * セルフマネージドインスタンスで W&amp;B Kubernetes operator を使用している場合、controller manager にカスタム CA をオプションで使用できるようになりました。
  * W&amp;B Kubernetes operator がデフォルトで non-root コンテキストで実行されるように変更し、OpenShift の Security Context Constraints (SCCs) に準拠させました。この変更により、OpenShift のセキュリティポリシーに従うことで、OpenShift 上でのセルフマネージドインスタンスのデプロイがより円滑になります。

  ## 修正

  * Workspace からレポートにパネルをエクスポートする際に、パネル検索 regex が正しく尊重されるようになりました。
  * `GORILLA_DISABLE_PERSONAL_ENTITY` を `true` に設定しても、Users が個人 entity 内でプロジェクトを作成したり既存のプロジェクトに書き込んだりできてしまう問題を修正しました。

  ## パフォーマンス改善

  * 100k+ の logged points を持つ Experiments におけるパフォーマンスと安定性を大幅に改善しました。セルフマネージドインスタンスでは、デプロイが W&amp;B Kubernetes operator を使用して管理されている場合に利用できます。
  * 大規模な Workspace で変更の保存が非常に遅くなる、または失敗する問題を修正しました。
  * 大規模な Workspace で Workspace セクションを開く際のレイテンシを改善しました。
</Update>

<Update label="v0.57.2" description="2024年7月24日">
  ## 機能

  **アイデンティティ フェデレーション機能を使用して、wandb SDK または CLI から JWT (JSON Web Token) を使って W&amp;B インスタンスにアクセスできるようになりました。この機能はプレビュー版です。** 詳細は [Identity federation](https://docs.wandb.ai/platform/hosting/iam/identity_federation) を参照し、ご不明な点があれば your W&amp;B team までお問い合わせください。

  * artifact プロジェクトブラウザーでの Artifacts メタデータのフィルタリング。
  * webhook payload で `${artifact_metadata.KEY}` を介して artifact メタデータを渡せるようになりました。
  * RunSystemMetrics コンポーネントに GPU メモリ使用量パネルを追加し、アプリのフロントエンドでの Runs の GPU メトリクス可視化を強化しました。

  * チームレベルのサービスアカウントが、チーム設定ビューの新しいタブに分けて表示されるようになりました。サービスアカウントは、Members タブには表示されなくなりました。また、APIキー は非表示になり、チーム管理者のみがコピーできます。
  * 専用クラウド が Google Cloud の ソウル リージョンで利用可能になりました。

  * 多くのプロットでガウシアンスムージングが極端に強すぎる問題がありました。
