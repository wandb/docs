<Update label="v0.65.0" description="January 30, 2025">
  ## 機能

  * Registry の **Settings** から、Admin ロールを持つ別のユーザーに owner を変更できるようになりました。ユーザーの **Role** メニューから **Owner** を選択します。

<Update label="v0.63.0" description="2024年12月10日">
  ## 機能

  **[W&amp;B Weave](https://wandb.ai/site/weave/) が、AWS 上の Dedicated Cloud で一般提供 (GA) になりました。自信を持って Generative AI アプリケーションを構築し、本番環境に投入したい場合は、担当の W&amp;B チームまでお問い合わせください。**

  <Frame>
    ![Weave UI](https://github.com/user-attachments/assets/22786cbc-7d69-4505-b329-78cb87734d19)
  </Frame>

  このリリースには、以下の追加アップデートが含まれます。

  * W&amp;B Models が ***Azure public cloud*** とシームレスに連携できるようになりました。Azure subscription から直接、Azure リージョンに Dedicated Cloud インスタンスを作成し、Azure ISV リソースとして管理できます。[このインテグレーションは現在プライベートプレビューです](https://wandb.ai/site/partners/azure)。
  * 分散 run において、ラベル (例: node) ごとにログとメトリクスを区別できるように、run settings オブジェクトで x&#95;label (例: node-0) を指定できるようになりました。これにより、ワークスペースでの可視化のために、システムメトリクスおよびコンソールログをラベルごとにグループ化できます。
  * ***近日提供予定***：今週のパッチリリースで、インスタンス内のすべての team にまたがる W&amp;B ワークロードを自動化するために、組織レベルの service account を利用できるようになります。service account のアクセス範囲をより細かく制御したい場合には、既存の team レベルの service account を引き続き使用できます。
    * 組織レベルの service account が Registry と連携できるようになりました。これらの service account は invite モーダルから registry に招待でき、対応する組織ロールとともにメンバー一覧テーブルに表示されます。

  ## 修正

  * `Create Artifact` パーミッションを含むカスタムロールを作成したユーザーが、プロジェクトに Artifacts をログできない問題を修正しました。
  * BYOB 用に subpath サポートが構成されているインスタンスで、ファイルのメタデータログに関して発生していた問題を修正しました。
  * 組織レジストリの自動化処理で使用されている場合、webhook を削除できないようブロックするようにしました。
</Update>

<Update label="v0.61.0" description="2024年10月17日">
  ## 機能

  **これは、月次の W&amp;B server メジャーリリースとは異なるスケジュールで提供される、小規模な機能追加およびパッチリリースです。**

  * Organization admin は、organization のダッシュボードから、Models のシート数および Models と [W&amp;B Weave](/ja/weave/) のアクセス制御をシームレスに設定できるようになりました。この変更により、Dedicated Cloud または Self-Managed インスタンスで [Weave](/ja/weave/) が有効化されている場合に、ユーザー管理をより効率的に行えるようになります。
    * [Weave](/ja/weave/) の料金は、利用シート数ベースではなく利用量ベースです。シート管理は Models プロダクトにのみ適用されます。
  * [team および制限付きスコープのプロジェクト向けに、プロジェクトレベルで access role を設定](/ja/platform/hosting/iam/access-management/restricted-projects/) できるようになりました。これにより、同じ team 内の異なるプロジェクトごとに、同一ユーザーに異なる access role を割り当てられるようになり、エンタープライズのガバナンス要件に対応するための強力な制御を追加できます。

  ## 修正

  * リリースアップグレードの一部として行われる基盤となるデータベーススキーマの変更が、プラットフォーム起動時にタイムアウトする可能性があった問題を修正しました。
  * 基盤となる parquet store サービスにさらなるパフォーマンス改善を加え、ユーザー向けのチャート読み込み時間を一層短縮しました。parquet store サービスは Dedicated Cloud および [W&amp;B kubernetes operator](/ja/platform/hosting/operator) ベースの Self-Managed インスタンスでのみ利用可能です。
  * 基盤となる parquet store サービスで高い CPU 使用率が発生する問題に対処し、効率的なチャート読み込みをユーザーにとってより信頼性の高いものにしました。parquet store サービスは Dedicated Cloud および [W&amp;B kubernetes operator](/ja/platform/hosting/operator) ベースの Self-Managed インスタンスでのみ利用可能です。
</Update>

<Update label="v0.60.0" description="2024年9月26日">
  <Note>
    リリースバージョン管理上の問題により、0.60.0 が 0.58.0 に続く次のメジャーリリースとなります。0.59.0 は 0.58.0 向けのパッチリリースの 1 つです。

  ## 機能

  * Web Content Accessibility Guidelines (WCAG) 標準の Level AA 2.2 に対する 1.1.1 コンプライアンスの最終アップデートを行いました。
  * Self-Managed インスタンスについては、W&amp;B kubernetes operator 使用時に自動バージョンアップグレードをオフにできるようになりました。これについては担当の W&amp;B チームに依頼できます。
    W&amp;B は、各バージョンに対して 6 か月のサポート期間 (end-of-life まで) を順守するため、すべてのインスタンスが定期的にアップグレードされることを求めます。W&amp;B はリリースから 6 か月より古いバージョンはサポートしません。

  ## 修正

  * Dedicated Cloud および Self-Managed インスタンスで、インスタンス admin が personal Entities 内のワークスペースへアクセスできるように、不具合を修正しました。
  * SCIM Groups および Users の GET エンドポイントが、レスポンスから service account を除外するようになりました。これらのエンドポイントでは、service account ではないユーザーのみが返されます。
  * team admin が、team からユーザーを削除する際に、そのユーザーをインスタンス全体から同時に削除できてしまうユーザー管理上のバグを修正しました。インスタンスまたは Org admin が、インスタンス / organization 全体からユーザーを削除する責任を負います。

  ## パフォーマンス改善

  * 多数のメトリクスを含むワークスペースで、パネル追加時のレイテンシーを最大 90% 削減しました。
  * run が頻繁に再開される場合でも、blob storage への parquet エクスポートの信頼性とパフォーマンスを改善しました。
    parquet 形式での blob storage への run エクスポートは、Dedicated Cloud と、W&amp;B kubernetes operator を用いて有効化された Self-Managed インスタンスで利用できます。
</Update>

<Update label="v0.58.1" description="2024年9月4日">
  ## 機能

  * **Secure storage connector (Bring your own bucket)&#x20;**&#x20;機能でサブパスがサポートされました。インスタンス単位またはチーム単位でバケットを設定する際に、サブパスを指定できるようになりました。これは新しく設定するバケットにのみ利用でき、既存の設定済みバケットには適用されません。
  * Google Cloud と Azure 上の新しい Dedicated Cloud インスタンスでは、W&amp;B 管理ストレージがデフォルトで **W&amp;B managed cloud-native keys** で暗号化されるようになりました。これはすでに AWS インスタンスでは利用可能です。各インスタンス ストレージは、そのインスタンス専用のキーで暗号化されます。これまでは、Google Cloud と Azure 上のすべてのインスタンスが、クラウド プロバイダー管理のデフォルト暗号化キーに依存していました。
  * run 設定とサマリー内のフィールドを、クリックでコピーできるようにしました。
  * Self-Managed インスタンスで W&amp;B kubernetes operator を使用している場合、オプションで controller manager 用にカスタム CA を使用できるようになりました。
  * W&amp;B kubernetes operator がデフォルトで non-root コンテキストで動作するように変更し、OpenShift の Security Context Constraints (SCCs) に合わせました。この変更により、OpenShift のセキュリティ ポリシーに準拠することで、Self-Managed インスタンスをよりスムーズにデプロイできるようになります。

  ## 不具合修正

  * Workspace からレポートへパネルをエクスポートする際に、パネル検索の正規表現が正しく反映されるよう修正しました。
  * `GORILLA_DISABLE_PERSONAL_ENTITY` を `true` に設定しても、ユーザーが personal entities 内でプロジェクトを作成したり既存プロジェクトへ書き込んだりできてしまう問題を修正しました。

  ## パフォーマンスの改善

  * 100k 以上のポイントが記録された実験に対して、パフォーマンスと安定性を大幅に改善しました。Self-Managed インスタンスの場合、これは W&amp;B kubernetes operator で管理されているデプロイで利用できます。
  * 大規模なワークスペースで変更を保存する際に、非常に遅くなったり失敗したりする問題を修正しました。
  * 大規模なワークスペースで Workspace セクションを開く際のレイテンシを改善しました。
</Update>

<Update label="v0.57.2" description="2024年7月24日">
  ## 機能

  **アイデンティティ フェデレーション機能を使用して、JWTs (JSON Web Tokens) を使い、wandb SDK や CLI から自分の W&amp;B インスタンスへアクセスできるようになりました。この機能はプレビュー段階です。** 詳細は [Identity federation](/ja/platform/hosting/iam/identity_federation) を参照し、不明点があれば W&amp;B の担当チームにお問い合わせください。

  * Workspace パネルを Reports にエクスポートするための `Add to reports` ドロワーを改善しました。
  * artifact プロジェクト ブラウザでの Artifacts メタデータ フィルタリング。
  * `${artifact_metadata.KEY}` を使って、webhook ペイロードに artifact メタデータを渡せるようになりました。
  * RunSystemMetrics コンポーネントに GPU メモリ使用量パネルを追加し、アプリ フロントエンドでの run の GPU メトリクス可視化を強化しました。
  * モバイル ユーザーにとって、Workspace の操作性がよりスムーズで直感的になりました。

  * チーム単位のサービス アカウントが、チーム設定ビュー内の新しいタブで個別に表示されるようになりました。サービス アカウントは、Members タブには表示されなくなりました。また、API キー は非表示となり、チーム管理者のみがコピーできます。
  * Dedicated Cloud が Google Cloud のソウル リージョンでも利用可能になりました。

  ## 不具合修正

  * `Ignore Outliers in Chart Scaling` ボタンを押しても、UI Workspace 上で何も起きない問題を修正しました。
  * 無効化されたユーザーを組織に招待できないようにしました。
  * SCIM API を使ってインスタンスに追加されたユーザーが、正常にオンボーディングできない問題を修正しました。

  ## パフォーマンスの改善

  * パネル設定の編集と変更適用時のパフォーマンスを大幅に改善しました。
  * 大規模な Workspace での run 公開範囲トグル操作の応答性を改善しました。
  * 大規模な Workspace 内のプロットで、チャートのホバーとブラッシングのパフォーマンスを改善しました。
  * 多数のキーを含む Workspace でのメモリ使用量と読み込み時間を削減しました。
</Update>

<Update label="v0.54.0" description="2024年5月24日">
  ## 機能

  * Microsoft Azure 上の Dedicated Cloud または Self-Managed インスタンスで、チームレベルで Secure storage connector (BYOB) を構成できるようになりました。
    [この Terraform Module](https://github.com/wandb/terraform-azurerm-wandb/tree/main/modules/secure_storage_connector) を参照し、不明点があれば所属の W&amp;B チームまでお問い合わせください。
  * 組織管理者は、Organization Dashboard の `Settings` タブから組織レベルで設定することで、すべての W&amp;B Teams に対してプライバシー設定を強制できるようになりました。
    このような強制変更を行う前に、チーム管理者およびその他のユーザーに事前に通知することを W&amp;B は推奨します。
  * artifact lineage DAG で direct lineage オプションを有効化できるようになりました。
  * Organization 管理者や Instance 管理者が、自身で W&amp;B チームに参加したり自分自身を追加したりすることを制限できるようになりました。これにより、チーム内のプロジェクトへアクセスできるのが Data &amp; AI 担当者のみに制限されます。
    この設定を有効化する前に、その影響を十分に理解し、慎重に扱うことを W&amp;B は推奨します。不明点があれば所属の W&amp;B チームまでお問い合わせください。

    Reports は、ML チーム外のメンバーを含む組織全体に W&amp;B 上の成果を共有するのに役立ちます。作成と共有をできるだけ簡単かつスムーズに行えるようにするため、レポートの下書き体験を改善し、ドラフトの公開・編集・管理・共有の UX を強化するなど、さまざまな投資と改善を行いました。これにより、チームが Reports を使ってコラボレーションしやすくなります。
  * ナビゲーションの更新
    W&amp;B がカバーする ML ワークフローの範囲が拡大する中で、アプリケーション内の移動が難しいというフィードバックをいただいていました。そのため、ナビゲーションサイドバーを更新し、プロダクト エリアのラベルをより分かりやすくするとともに、特定の詳細画面へのバックリンクを追加しました。また、この機能の強力さをより正確に表すために、「Triggers」の名称を「Automations」に変更しました。

  ## 修正

  * ワークスペースまたはレポート内のプロット上でホバーしているときに、Cmd+C または Ctrl+C を使用して、hover コントロールに表示される run 名とプロット値をコピーできるようになりました。
  * デフォルト ワークスペースへの変更が自動保存されなくなりました。
  * Overview → Summary セクション内のメトリクスが、カンマ区切りで表示されるようになりました。
  * 管理者以外のユーザーがチームを作成できるようにするインストール レベルのオプションを追加しました（デフォルトはオフ。有効化するには W&amp;B サポートにお問い合わせください）。
  * Weave のプロットで対数スケールがサポートされるようになりました。
  * Launch パネルを水平方向に拡大して、パラメーターの表示領域を増やせるようになりました。
  * Launch パネルに、キューがアクティブかどうかが表示されるようになりました。
  * Launch パネルから、run を記録する先のプロジェクトを選択できるようになりました。
  * Launch キューは、チーム管理者のみが作成できるようになりました。
  * Launch パネルでの Markdown サポートを改善しました。
  * 空の Launch キュー設定に対するエラーメッセージを改善しました。
  * Sweeps の parallel coordinates プロット上のフィルターが、選択されているすべての runset に適用されるようになりました。
  * Sweeps でメトリクスが必須ではなくなりました。
  * Azure Blob Storage 上に保存された、W&amp;B の外部にある参照 Artifacts ファイルの追跡をサポートしました。
  * Reports 内での Markdown 編集に関するバグを修正しました。
  * フルスクリーンの Weave パネルから、元のパネルと設定変更を共有できるようになりました。
  * 空のテーブルの表示を改善しました。
  * ログの先頭数文字が切れてしまうバグを修正しました。
