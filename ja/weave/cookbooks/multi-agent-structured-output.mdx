<div id="structured-outputs-for-multi-agent-systems">
  # マルチエージェントシステム向け Structured Outputs
</div>

OpenAI は、強い調子のプロンプトを使わなくても、指定した JSON Schema に常に従うレスポンスをモデルが生成できるようにするため、[Structured Outputs](https://openai.com/index/introducing-structured-outputs-in-the-api/) をリリースしました。Structured Outputs を使えば、形式が正しくないレスポンスに対して検証や再試行を行う必要がありません。

新しいパラメーター `strict: true` を使用することで、レスポンスが指定したスキーマに従うことを保証できます。

マルチエージェントシステムで Structured Outputs を使用すると、エージェント間で一貫性があり処理しやすいデータをやり取りできるため、連携が向上します。また、明示的な拒否を可能にすることで安全性も高まり、再試行や検証が不要になることでパフォーマンスも向上します。これによりやり取りが簡素化され、システム全体の効率が高まります。

このチュートリアルでは、マルチエージェントシステムで Structured Outputs を活用し、それらを [Weave](/ja/weave) でトレースする方法を示します。

<Tip>
  **ソース**: この cookbook は、[OpenAI の Structured Outputs のサンプルコード](https://cookbook.openai.com/examples/structured_outputs_multi_agent) をもとに、Weave を使用した可視化を改善するために一部変更を加えています。
</Tip>

<div id="agents-set-up">
  ## エージェントの設定
</div>

今回取り組むユースケースは、データ分析タスクです。
まず、4つのエージェントから成るシステムを設定しましょう。

* Triaging エージェント: どのエージェントをcallするかを決定します
* Data pre-processing エージェント: 分析用にデータを準備します。たとえば、データをクリーンアップします
* Data Analysis エージェント: データを分析します
* Data Visualization エージェント: 分析結果を可視化し、インサイトを引き出します
  まず、これら各エージェントのsystem promptを定義します。

次に、各エージェントで使用するツールを定義します。

振り分けエージェントを除き、各エージェントには役割に応じた専用のツールを用意します。

**データ前処理エージェント** : 1. データのクリーニング, 2. データの変換, 3. データの集計

**データ分析エージェント** : 1. 統計分析, 2. 相関分析, 3. 回帰分析

**データ可視化エージェント** : 1. 棒グラフの作成, 2. 折れ線グラフの作成, 3. 円グラフの作成

<div id="enable-tracking-of-multi-agent-using-weave">
  ## Weave でマルチエージェントのトラッキングを有効にする
</div>

次の処理を行うコードロジックを記述する必要があります。

* ユーザーのクエリをマルチエージェントシステムに渡す
* マルチエージェントシステムの内部処理を扱う
* ツール呼び出し を実行する

ユーザーのクエリから、呼び出す必要があるツールは `clean_data`、`start_analysis`、`use_line_chart` だと推測できます。

まず、ツール呼び出しを実行する実行用の関数を定義します。

Python 関数に `@weave.op()` をデコレートすると、言語モデルの入力、出力、トレースをログしてデバッグできます。

マルチエージェントシステムを作成すると、多くの関数が出てきますが、それらの先頭に `@weave.op()` を追加するだけで十分です。

次に、各サブエージェント用のツールハンドラーを作成します。これらには、それぞれ固有のプロンプトとツールセットがモデルに渡されます。出力はその後、ツール呼び出しを実行する実行関数に渡されます。

最後に、ユーザーのクエリの処理を担う全体のツールを作成します。この関数はユーザーのクエリを受け取り、モデルから応答を取得し、それを他のエージェントに渡して実行します。
