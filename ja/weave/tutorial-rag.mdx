* ナレッジベースを構築する
* 関連ドキュメントを検索する取得ステップを含む RAG アプリケーションを作成する
* Weave を使って取得ステップを追跡する
* コンテキスト精度を測定するために LLM ジャッジを用いて RAG アプリケーションを評価する

<div id="create-a-rag-app">
  ## RAG アプリを作成する
</div>

次に、retrieval 関数 `get_most_relevant_document` を `weave.op()` デコレーターでラップし、`Model` クラスを作成します。`weave.init('<team-name>/rag-quickstart')` を呼び出して、後から確認できるように関数のすべての入力と出力の追跡を開始します。team 名を指定しない場合、出力はあなたの [W&amp;B default team or entity](/ja/platform/app/settings-page/user-settings#default-team) に記録されます。

<Tabs>
  <Tab title="Python">
    ```python lines
    from openai import OpenAI
    import weave
    from weave import Model
    import numpy as np
    import asyncio

    @weave.op()
    def get_most_relevant_document(query):
        openai = OpenAI()
        query_embedding = (
            openai.embeddings.create(input=query, model="text-embedding-3-small")
            .data[0]
            .embedding
        )
        similarities = [
            np.dot(query_embedding, doc_emb)
            / (np.linalg.norm(query_embedding) * np.linalg.norm(doc_emb))
            for doc_emb in article_embeddings
        ]
        # 最も類似しているドキュメントのインデックスを取得します
        most_relevant_doc_index = np.argmax(similarities)
        return articles[most_relevant_doc_index]

    class RAGModel(Model):
        system_message: str
        model_name: str = "gpt-3.5-turbo-1106"

        @weave.op()
        def predict(self, question: str) -> dict: # 注意: `question` は後で評価用の行からデータを選択するために使用されます
            from openai import OpenAI
            context = get_most_relevant_document(question)
            client = OpenAI()
            query = f"""Use the following information to answer the subsequent question. If the answer cannot be found, write "I don't know."
            Context:
            \"\"\"
            {context}
            \"\"\"
            Question: {question}"""
            response = client.chat.completions.create(
                model=self.model_name,
                messages=[
                    {"role": "system", "content": self.system_message},
                    {"role": "user", "content": query},
                ],
                temperature=0.0,
                response_format={"type": "text"},
            )
            answer = response.choices[0].message.content
            return {'answer': answer, 'context': context}

    # team 名と project 名を設定
    weave.init('<team-name>/rag-quickstart')
    model = RAGModel(
        system_message="You are an expert in finance and answer questions related to finance, financial services, and financial markets. When responding based on provided information, be sure to cite the source."
    )
    model.predict("What significant result was reported about Zealand Pharma's obesity trial?")
    ```

<div id="evaluating-with-an-llm-judge">
  ## LLM ジャッジを使った評価
</div>

アプリケーションを簡単に評価できない場合のアプローチの一つとして、LLM にアプリケーションの各側面を評価させる方法があります。次に、LLM ジャッジを使って、与えられた回答に至るうえでコンテキストが有用だったかどうかを検証するように LLM をプロンプトすることで、コンテキストの精度を測定しようとする例を示します。このプロンプトは、広く利用されている [RAGAS フレームワーク](https://docs.ragas.io/en/stable/) を基に拡張したものです。

<div id="defining-a-scoring-function">
  ### スコアリング関数の定義
</div>

[Build an Evaluation pipeline tutorial](/ja/weave/tutorial-eval) と同様に、アプリをテストするためのサンプル行の集合とスコアリング関数を定義します。スコアリング関数は 1 行を受け取り、その行を評価します。入力引数は行内の対応するキーと一致している必要があるため、ここでの `question` はその行の辞書から取得されます。`output` はモデルの出力です。モデルへの入力は、その入力引数に基づいてサンプルから取得されるため、ここでも `question` が使われます。この例では `async` 関数を使うことで、並列に高速に実行できるようにしています。async への簡単な入門が必要な場合は、[こちら](https://docs.python.org/3/library/asyncio.html) を参照してください。

<Tabs>
  <Tab title="Python">
    ```python lines
    from openai import OpenAI
    import weave
    import asyncio

    @weave.op()
    async def context_precision_score(question, output):
        context_precision_prompt = """Given question, answer and context verify if the context was useful in arriving at the given answer. Give verdict as "1" if useful and "0" if not with json output.
        Output in only valid JSON format.

        question: {question}
        context: {context}
        answer: {answer}
        verdict: """
        client = OpenAI()

        prompt = context_precision_prompt.format(
            question=question,
            context=output['context'],
            answer=output['answer'],
        )

        response = client.chat.completions.create(
            model="gpt-4-turbo-preview",
            messages=[{"role": "user", "content": prompt}],
            response_format={ "type": "json_object" }
        )
        response_message = response.choices[0].message
        response = json.loads(response_message.content)
        return {
            "verdict": int(response["verdict"]) == 1,
        }

    questions = [
        {"question": "What significant result was reported about Zealand Pharma's obesity trial?"},
        {"question": "How much did Berkshire Hathaway's cash levels increase in the fourth quarter?"},
        {"question": "What is the goal of Highmark Health's integration of Google Cloud and Epic Systems technology?"},
        {"question": "What were Rivian and Lucid's vehicle production forecasts for 2024?"},
        {"question": "Why was the Norwegian Dawn cruise ship denied access to Mauritius?"},
        {"question": "Which company achieved the first U.S. moon landing since 1972?"},
        {"question": "What issue did Intuitive Machines' lunar lander encounter upon landing on the moon?"}
    ]
    evaluation = weave.Evaluation(dataset=questions, scorers=[context_precision_score])
    asyncio.run(evaluation.evaluate(model)) # 注意: 評価に使用する model を定義する必要があります
    ```

<div id="optional-defining-a-scorer-class">
  ### オプション: `Scorer` クラスの定義
</div>

一部のアプリケーションでは、カスタムの評価クラスを作成したい場合があります。たとえば、特定のパラメーター (例: チャット モデル、プロンプト) を備えた標準化された `LLMJudge` クラス、各行に対する特定のスコアリング、および集計スコアの特定の計算方法を定義したいケースです。Weave は、すぐに使える `Scorer` クラスのリストを定義しているほか、カスタム `Scorer` を簡単に作成できるようにしています。次の例では、カスタムクラス `class CorrectnessLLMJudge(Scorer)` の作成方法を示します。

大まかには、カスタム Scorer を作成する手順は次のとおりで、とてもシンプルです。

1. `weave.flow.scorer.Scorer` を継承するカスタムクラスを定義する
2. `score` 関数をオーバーライドし、その関数の各呼び出しを追跡したい場合は `@weave.op()` を追加する
   * この関数では、モデルの予測値が渡される `output` 引数を定義する必要があります。モデルが &quot;None&quot; を返す可能性がある場合に備え、型は `Optional[dict]` として定義します。
   * 残りの引数は、一般的な `Any` や `dict` にすることもできますし、`weave.Evaluate` クラスを使って モデル を評価する際に利用される データセット から、特定の列を選択することもできます。その場合、`preprocess_model_input` を使用しているなら、その処理後の単一行のカラム名またはキーと完全に同じ名前にする必要があります。
3. *オプション:* 集計スコアの計算方法をカスタマイズするために `summarize` 関数をオーバーライドします。デフォルトでは、カスタム関数を定義しない場合、Weave は `weave.flow.scorer.auto_summarize` 関数を使用します。
   * この関数には `@weave.op()` デコレータを付ける必要があります。

<Tabs>
  <Tab title="Python">
    ```python lines
    from weave import Scorer

    class CorrectnessLLMJudge(Scorer):
        prompt: str
        model_name: str
        device: str

        @weave.op()
        async def score(self, output: Optional[dict], query: str, answer: str) -> Any:
            """pred, query, target を比較して予測の正確さをスコア付けします。
            Args:
                - output: 評価対象のモデルから提供される dict
                - query: データセット で定義された質問
                - answer: データセット で定義されたターゲット回答
            Returns:
                - 単一の dict {メトリクス名: 単一の評価値}"""

            # get_model は、指定されたパラメーターに基づいて一般的にモデルを取得する関数として定義されています (OpenAI, HF など)
            eval_model = get_model(
                model_name = self.model_name,
                prompt = self.prompt
                device = self.device,
            )
            # 評価を高速化するための非同期評価 - 非同期である必要はありません
            grade = await eval_model.async_predict(
                {
                    "query": query,
                    "answer": answer,
                    "result": output.get("result"),
                }
            )
            # 出力のパース - pydantic を使えばより堅牢に実装可能
            evaluation = "incorrect" not in grade["text"].strip().lower()

            # Weave に表示されるカラム名
            return {"correct": evaluation}

        @weave.op()
        def summarize(self, score_rows: list) -> Optional[dict]:
            """スコアリング関数で各行に対して計算されたすべてのスコアを集計します。
            Args:
                - score_rows: dict のリスト。各 dict はメトリクスとスコアを持ちます
            Returns:
                - 入力と同じ構造を持つ入れ子の dict"""

            # 何も提供されない場合は weave.flow.scorer.auto_summarize 関数が使用されます
            # return auto_summarize(score_rows)

            valid_data = [x.get("correct") for x in score_rows if x.get("correct") is not None]
            count_true = list(valid_data).count(True)
            int_data = [int(x) for x in valid_data]

            sample_mean = np.mean(int_data) if int_data else 0
            sample_variance = np.var(int_data) if int_data else 0
            sample_error = np.sqrt(sample_variance / len(int_data)) if int_data else 0

            # 追加の "correct" レイヤーは必須ではありませんが、UI 上の構造をわかりやすくします
            return {
                "correct": {
                    "true_count": count_true,
                    "true_fraction": sample_mean,
                    "stderr": sample_error,
                }
            }
    ```

<div id="pulling-it-all-together">
  ## すべてをまとめる
</div>

RAG アプリで同じことを行うには、次の手順に従います。

* LLM 呼び出しと検索ステップの関数を `weave.op()` でラップする
*  (オプション) `predict` 関数とアプリの詳細を持つ `Model` サブクラスを作成する
* 評価用の例を収集する
* 1 つの例をスコアリングするスコアリング関数を作成する
* `Evaluation` クラスを使って例に対して評価を実行する

**注意:** 場合によっては、`Evaluation` の非同期実行により OpenAI や Anthropic などのモデルでレート制限が発生することがあります。これを防ぐには、環境変数を設定して並列ワーカー数を制限します (例: `WEAVE_PARALLELISM=3`) 。

    ```python lines {34,52-77}
    @weave.op()
    def get_most_relevant_document(query):
        openai = OpenAI()
        query_embedding = (
            openai.embeddings.create(input=query, model="text-embedding-3-small")
            .data[0]
            .embedding
        )
    ```

    ```typescript lines
    function cosineSimilarity(a: number[], b: number[]): number {
        const dotProduct = a.reduce((sum, val, i) => sum + val * b[i], 0);
        const magnitudeA = Math.sqrt(a.reduce((sum, val) => sum + val * val, 0));
        const magnitudeB = Math.sqrt(b.reduce((sum, val) => sum + val * val, 0));
        return dotProduct / (magnitudeA * magnitudeB);
    }

                if (!doc.embedding) return 0;

            });
    ```
