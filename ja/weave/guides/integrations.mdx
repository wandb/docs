---
title: インテグレーションの概要
description: "一般的なエージェント SDK、ハーネス、LLM プロバイダ、オーケストレーション フレームワーク向けの Weave のインテグレーションを使って、エージェントと LLM アプリケーションをトレースします。"
---

Weave では、AI スタックと統合する方法が 2 つあります。

* **エージェントをトレース**: エージェント SDK (OpenAI Agents SDK や Google ADK など) で構築したマルチターンのエージェント アプリケーション、またはエージェント ハーネス (Claude Code、Codex、Pi.dev など) 内で実行されるアプリケーション向けです。これらのインテグレーションでは、セッション、ターン、LLM calls、tool calls を取得し、Weave の **Agents** ビューに表示します。
* **LLM アプリケーションをトレース**: LLM プロバイダ (OpenAI、Anthropic、Bedrock など) を呼び出すコード、またはオーケストレーション フレームワーク (LangChain、LlamaIndex、DSPy など) を使用するコード向けです。これらのインテグレーションでは、個々の LLM calls とパイプラインのステップを、**Traces** ビューで Weave **Calls** として取得します。

どちらを選べばよいかわからない場合は、新しいマルチターンのエージェント開発には [エージェントをトレースする](/ja/weave/guides/tracking/trace-agents) から、既存の LLM 呼び出しワークフローには [LLM アプリケーションをトレースする](/ja/weave/guides/tracking/tracing) から始めてください。

<div id="trace-agents">
  ## エージェントをトレースする
</div>

**Agents** ビューに表示されるエージェント SDK、ハーネス、カスタム インストルメンテーションについては、[エージェントのトレース向けインテグレーション](/ja/weave/guides/tracking/trace-agent-integrations)を参照してください。

<div id="trace-llm-applications">
  ## LLM アプリケーションをトレースする
</div>

アプリケーションが LLM プロバイダの API を直接呼び出す場合や、オーケストレーション フレームワークを使用する場合、Weave は多くの一般的なライブラリやフレームワークで、トレースを自動的にインターセプトできます (autopatching を使用) 。コードに Weave SDK をインポートして `weave.init` で初期化すると、各リクエストは入力、出力、レイテンシ、token 使用量、コストを含む Weave の **Call** として記録されます。

Weave が autopatch していないライブラリについては、コードに [Weave Ops](/ja/weave/guides/tracking/create-call#2-tracking-of-custom-functions) を手動で適用して、トレースを取得できます。
