title: インテグレーション概要
description: "OpenAI、Anthropic などに対応した Weave の自動パッチ適用により、30 以上のプロバイダーとフレームワークにまたがる LLM 呼び出しをトレースおよび監視できます。"
---

W&amp;B Weave は、一般的な LLM プロバイダーおよびオーケストレーションフレームワーク向けのロギングインテグレーションを提供します。これらのインテグレーションにより、さまざまなライブラリを通じて行われる Call をシームレスにトレースでき、AI アプリケーションを監視および分析する能力を高められます。

アプリケーションで LLM プロバイダーのライブラリ (OpenAI、Anthropic、Cohere、Mistral など) を使用している場合、それらの API Call を、入力、出力、レイテンシ、トークン使用量、コストを含むトレースされた Call として W&amp;B Weave に表示させたいはずです。これを使わない場合、すべての `client.chat.completions.create()` (または同等の呼び出し) を `@weave.op` または手動インストルメンテーションでラップする必要があり、これは面倒なうえ、何かを見落としやすくなります。

Weave は、サポートされている LLM クライアントライブラリを自動的にインターセプト (パッチ適用) します。アプリケーションコードはそのままで、通常どおりプロバイダーの SDK を使用するだけで、各リクエストが Weave の Call として記録されます。最小限の設定で完全なトレースが得られます。

<div id="protocols">
  ## プロトコル
</div>

Weave は、AI アプリケーションとそれを支えるサービス間の通信を可能にする標準化されたプロトコルと連携します。

* **[Model Context Protocol (MCP)](/ja/weave/guides/integrations/mcp)**

上記の一覧からインテグレーションを選択して、好みの LLM プロバイダー、フレームワーク、またはプロトコルと Weave をどのように組み合わせて使用するかを確認してください。LLM API に直接アクセスする場合でも、複雑なパイプラインを構築する場合でも、標準化されたプロトコルを使用する場合でも、Weave は AI アプリケーションを効果的にトレースして分析するためのツールを提供します。
