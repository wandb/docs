---
title: "Weave Trace view を操作する"
description: "Weave の Trace view を使って、LLM およびエージェント型アプリにおける複雑な実行パスを理解します。"
---

Weave Trace view は、LLM およびエージェント型アプリにおける複雑な実行パスを理解しやすくするために設計されています。多数のネストされた Call を持つエージェント型アプリをデバッグしている場合でも、単一のモデル予測のフローを追跡している場合でも、Trace view は何が起きたかを明確に可視化します。また、アプリケーションフローを表示・理解するための別の方法も提供します。

このガイドでは、トレーススタック内を移動する方法、コードによって呼び出された関数をフィルターおよび検索する方法、さまざまな視覚的な表示形式を切り替える方法などについて説明します。

<div id="get-started">
  ## はじめに
</div>

Trace view に入るには、次の手順を実行します。

1. [https://wandb.ai](https://wandb.ai) にアクセスし、使用するプロジェクトを選択します。
2. サイドバーメニューで **トレース** を選択し、そのプロジェクトに保存されたすべてのトレースを表示します。
3. 任意のトレースを選択して Trace Details ビューを開きます。Trace Details ビューには、トレース実行の階層的な内訳を示す追加パネルが表示されます。

<div id="traces-page-overview">
  ## トレース ページの概要
</div>

トレース ページは、3 つの主要なパネルで構成されています。

* **左パネル**: プロジェクト内のすべての トレース を、並べ替え可能でページ分割された一覧として表示します。
  * この トレース 表 には、tokens、cost、latency などの追加データが含まれます。
* **中央パネル**: 選択した トレース のインタラクティブな Trace view です。トレース ツリーは、その トレース 内で追跡されたすべての method の階層構造を表示します。
  * トレース ツリーには、トレース 中に呼び出された、`@weave.op()` でデコレートされた関数である [ops](/ja/weave/guides/tracking/ops#automatically-track-function-calls-using-ops) が表示されます。
* **右パネル**: 選択した トレース 内で選択した op の詳細です。

<div id="filter-within-a-trace">
  ### トレース内でフィルターする
</div>

* **名前による正規表現フィルター**: トレース ツリー上部のテキストフィールドを使用して、`tool` や `openai.response.create` などの名前やタイプで op をフィルタリングします。
* **メトリクス**: 利用可能な場合に、以下のデータメトリクス (cost、tokens、latency) の表示を制御します。

<div id="navigate-a-trace">
  ### トレースをナビゲートする
</div>

トレース ツリーは、トレース内で追跡されたすべての method の階層構造を表示します。ツリー内を上下に移動するには、`Cmd` (macOS) または `Alt` (Windows/Linux) を押しながら、上矢印 (↑) / 下矢印 (↓) キーを使用します。

トレース ツリーの下には、トレース内のさまざまな状態を素早くナビゲートするための複数のスクラバーがあります。スライダーを使って、トレース内を効率的に移動できます。

パネルを展開して、使用可能なすべてのスクラバーを表示します。

* **Timeline**: トレース内のイベントを時系列順に表示します。
* **Peers**: 同じタイプの op。たとえば、`predict` という関数の詳細を確認している場合、このスクラバーを使うと、トレース内で次の `predict` の実行位置にすぐにジャンプできます。
* **Siblings**: 同じ親を持つ op。親の関数呼び出しの下にネストされた op を順番にたどるために、このスクラバーを使用します。
* **Stack**: コールスタックを上下にたどります。
* **Path**: (コード構成ビューでのみ使用可能) 選択した Call と同じコードパスを持つすべての Call を順番にたどります。

<div id="alternative-trace-tree-views">
  ### 別のトレース ツリー表示
</div>

パネル上部では、ニーズに応じてトレース ツリーの表示を複数の可視化形式から切り替えることができます。デバッグの目的に応じてビューを選択してください。呼び出しロジックを理解するには **コード構成ビュー**、時間経過に伴うパフォーマンスを把握するには **フレームグラフビュー**、構造を理解するには **グラフ ビュー** を使用します。

<div id="traces-default">
  #### トレース (デフォルト)
</div>

トレース ツリーのデフォルトビューでは、スタックの階層構造、各 op のコスト (利用可能な場合) 、実行時間、およびステータスインジケーターが表示されます。

<div id="code-composition-view">
  #### コード構成ビュー
</div>

コード構成ビューでは、ボックスが op とそのネストされた Call を表します。これにより、関数 Call のフローを可視化できます。このビューでは、ボックスを選択してその op の詳細にドリルダウンし、Call パスをフィルターできます。

<Frame>
  ![トレースのコードビューを表示している Trace view](/weave/guides/tracking/imgs/trace-tree-code-view.png)
</Frame>

<div id="flame-graph">
  #### フレームグラフ
</div>

フレームグラフビューは、実行の深さと実行時間をタイムライン形式で可視化します。これは、時間の経過に伴うパフォーマンスの診断結果を把握したい場合に役立ちます。特定のフレームを選択してサブトレースを抽出できます。

<Frame>
  ![トレース内の ops のフレームグラフを表示しているTrace view](/weave/guides/tracking/imgs/trace-tree-flame-view.png)
</Frame>

<div id="graph-view">
  #### グラフ ビュー
</div>

グラフ ビューは、op 間の階層的な関係を表示します。これは、親子関係を理解するのに役立ちます。

<Frame>
  ![Trace view で Trace 内の op とその関係を示すグラフ ビュー](/weave/guides/tracking/imgs/trace-tree-graph-view.png)
</Frame>

<div id="view-details-for-a-trace-op">
  ### トレース op の詳細を表示
</div>

トレース ツリーで op を選択すると、その op の詳細が次のパネルに表示されます。これらの詳細は、次のタブにグループ化されています。

* **Call**: op 実行の入力と出力です。
* **Code**: Call が行われたときに使用されたコードです。
* **Feedback**: op で使用可能な[フィードバック](/ja/weave/guides/tracking/feedback)です。フィードバックは Weave 内で直接提供することも、API を通じて提供することもできます。
* **Scores**: op で使用可能な[スコア](/ja/weave/guides/evaluation/scorers)です。Call は評価を実行することでスコアリングされます。
* **Summary**:  op に関する一般的な情報です。
* **Use:** Call をプログラムから取得し、リアクション、メモ、またはフィードバックを追加するために使用できるコード スニペットです。
