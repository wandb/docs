---
title: "トレースからPIIをマスクする"
description: "トレースから個人を特定できる情報を自動的にマスクします"
---

<Warning>
  この機能には Python SDK からのみアクセスできます。
</Warning>

一部の組織では、大規模言語モデル (LLM) のワークフローで、名前、電話番号、メールアドレスなどの個人を特定できる情報 (PII) を処理します。このデータを Weights &amp; Biases (W&amp;B) Weave に保存すると、コンプライアンスおよびセキュリティ上のリスクが生じます。

*Sensitive Data Protection* 機能を使用すると、[トレース](/ja/weave/guides/tracking) が Weave サーバーに送信される前に、そのトレースから個人を特定できる情報 (PII) を自動的にマスクできます。この機能は [Microsoft Presidio](https://microsoft.github.io/presidio/) を Weave Python SDK に統合しているため、SDK レベルでマスキング設定を制御できます。

Sensitive Data Protection 機能では、Python SDK に次の機能が追加されます。

* PII のマスキングを有効にするために、`weave.init` 呼び出しでオンまたはオフに切り替えられる `redact_pii` 設定。
* `redact_pii = True` の場合に、[一般的なentity](#entities-redacted-by-default) を自動的にマスクします。
* 設定可能な `redact_pii_fields` 設定を使用した、カスタマイズ可能なマスキング対象フィールド。
* `redact_pii_exclude_fields` 設定を使用して、特定のentityをマスキング対象から除外します。

<div id="enable-pii-redaction">
  ## PII マスキングを有効化
</div>

Weave で Sensitive Data Protection 機能を使い始めるには、以下の手順を実行します:

1. 必要な依存関係をインストールします:

   ```bash
   pip install presidio-analyzer presidio-anonymizer
   ```

2. マスキングを有効化するために `weave.init` Call を変更します。`redact_pii=True` の場合、[一般的な Entities はデフォルトでマスクされます](#entities-redacted-by-default):

   ```python lines
   import weave

   weave.init("my-project", settings={"redact_pii": True})
   ```

3. (任意) `redact_pii_fields` パラメーターを使用してマスキング対象のフィールドをカスタマイズします:

```python lines
weave.init("my-project", settings={"redact_pii": True, "redact_pii_fields":["CREDIT_CARD", "US_SSN"]})
```

検出してマスクできるエンティティの完全な一覧については、[Presidio でサポートされる PII entity](https://microsoft.github.io/presidio/supported_entities/)を参照してください。

4. (任意) `redact_pii_exclude_fields` パラメーターを使用して、特定のエンティティをマスキング対象から除外します。これは、デフォルトのマスキングは維持しつつ、特定の entity タイプを保持したい場合に便利です。次の例は、`EMAIL_ADDRESS` と `PERSON` を除くすべての[デフォルト entity](#entities-redacted-by-default)をマスクする方法を示しています:

```python lines
weave.init("my-project", settings={"redact_pii": True, "redact_pii_exclude_fields":["EMAIL_ADDRESS", "PERSON"]})
```

<div id="entities-redacted-by-default">
  ## デフォルトでマスクされるentity
</div>

PII マスキング が有効な場合、以下のentityは自動的にマスクされます:

* `CREDIT_CARD`
* `CRYPTO`
* `EMAIL_ADDRESS`
* `ES_NIF`
* `FI_PERSONAL_IDENTITY_CODE`
* `IBAN_CODE`
* `IN_AADHAAR`
* `IN_PAN`
* `IP_ADDRESS`
* `LOCATION`
* `PERSON`
* `PHONE_NUMBER`
* `UK_NHS`
* `UK_NINO`
* `US_BANK_NUMBER`
* `US_DRIVER_LICENSE`
* `US_PASSPORT`
* `US_SSN`

<div id="redacting-sensitive-keys-with-redact_keys">
  ## `REDACT_KEYS` で機密キーをマスクする
</div>

PII マスキング に加えて、Weave SDK は `REDACT_KEYS` を使用したカスタムキーのマスキングもサポートしています。これは、PII のカテゴリには該当しない可能性があっても、非公開にしておく必要がある追加の機密データを保護したい場合に便利です。例:

* APIキー
* 認証ヘッダー
* トークン
* 内部ID
* 設定値

<div id="pre-defined-redact_keys">
  ### 事前定義済みの `REDACT_KEYS`
</div>

Weave はデフォルトで次の機密性の高いキーを自動的にマスクします。

```json
[
  "api_key",
  "auth_headers",
  "authorization"
]
```

<div id="adding-your-own-keys">
  ### 独自のキーの追加
</div>

トレースからマスクしたい独自のキーを、このリストに追加できます:

```python lines
import weave
from weave.utils import sanitize

client = weave.init("my-project", settings={"redact_pii": True})

# マスクするカスタムキーを追加する
sanitize.add_redact_key("client_id")
sanitize.add_redact_key("token")

client_id = "123"
token = "789"

@weave.op
def test(client_id, token):
    return client_id + token

test(client_id, token)
```

Weave UIで確認すると、`client_id` と `token` の値は `"REDACTED"` と表示されます:

```python lines
client_id = "REDACTED"
token = "REDACTED"
```

<div id="usage-information">
  ## 使用に関する情報
</div>

* この機能は Python SDK でのみ使用可能です。
* マスキングを有効にすると、Presidio への依存関係により処理時間が長くなります。
