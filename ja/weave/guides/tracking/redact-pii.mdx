---
title: "トレースから PII をマスキングする"
description: "トレースから個人を特定できる情報を自動的にマスキングする"
---

<Warning>
  この機能は Python SDK からのみ利用できます。
</Warning>

一部の組織では、Large Language Model (LLM) ワークフローの中で、氏名、電話番号、メールアドレスなどの個人を特定できる情報 (PII) を処理します。これらのデータを Weights &amp; Biases (W&amp;B) Weave に保存すると、コンプライアンスおよびセキュリティ上のリスクが生じます。

*Sensitive Data Protection* 機能を使用すると、[トレース](/ja/weave/guides/tracking) が Weave サーバーに送信される前に、個人を特定できる情報 (PII) を自動的にマスキングできます。この機能は [Microsoft Presidio](https://microsoft.github.io/presidio/) を Weave Python SDK に統合したものであり、SDK レベルでマスキング設定を制御できます。

Sensitive Data Protection 機能では、Python SDK に次の機能が追加されます。

* `redact_pii` 設定。`weave.init` 呼び出しでオンまたはオフを切り替えて PII のマスキングを有効化できます。
* `redact_pii = True` の場合に、[一般的な entity](#entities-redacted-by-default) が自動的にマスキングされます。
* 設定可能な `redact_pii_fields` を使用したマスキング フィールドのカスタマイズ。
* `redact_pii_exclude_fields` 設定を使用して、特定の entity をマスキング対象から除外できます。

<div id="enable-pii-redaction">
  ## PII のマスキングを有効にする
</div>

Weave の Sensitive Data Protection 機能を使い始めるには、次の手順を実行します。

1. 必要な依存関係をインストールします:

   ```bash
   pip install presidio-analyzer presidio-anonymizer
   ```

2. `weave.init` 呼び出しを変更して、マスキングを有効にします。`redact_pii=True` の場合、[一般的な entity はデフォルトでマスキングされます](#entities-redacted-by-default):

   ```python lines
   import weave

   weave.init("my-project", settings={"redact_pii": True})
   ```

3. (任意) `redact_pii_fields` パラメーターを使用して、マスキングするフィールドをカスタマイズします:

   ```python lines
   weave.init("my-project", settings={"redact_pii": True, "redact_pii_fields":["CREDIT_CARD", "US_SSN"]})
   ```

   検出およびマスキング可能な entity の一覧については、[Presidio がサポートする PII entity](https://microsoft.github.io/presidio/supported_entities/)を参照してください。

4. (任意) `redact_pii_exclude_fields` パラメーターを使用して、特定の entity をマスキング対象から除外します。これは、デフォルトのマスキングは維持したまま、特定の entity タイプを保持したい場合に便利です。次の例は、`EMAIL_ADDRESS` と `PERSON` を除くすべての[デフォルト entity](#entities-redacted-by-default)をマスキングする方法を示しています:

   ```python lines
   weave.init("my-project", settings={"redact_pii": True, "redact_pii_exclude_fields":["EMAIL_ADDRESS", "PERSON"]})
   ```

<div id="entities-redacted-by-default">
  ## デフォルトでマスクされる Entities
</div>

PII のマスキングが有効な場合、以下の Entities は自動的にマスクされます。

* `CREDIT_CARD`
* `CRYPTO`
* `EMAIL_ADDRESS`
* `ES_NIF`
* `FI_PERSONAL_IDENTITY_CODE`
* `IBAN_CODE`
* `IN_AADHAAR`
* `IN_PAN`
* `IP_ADDRESS`
* `LOCATION`
* `PERSON`
* `PHONE_NUMBER`
* `UK_NHS`
* `UK_NINO`
* `US_BANK_NUMBER`
* `US_DRIVER_LICENSE`
* `US_PASSPORT`
* `US_SSN`

<div id="redacting-sensitive-keys-with-redact_keys">
  ## `REDACT_KEYS` を使った機密キーのマスキング
</div>

PII のマスキングに加えて、Weave SDK は `REDACT_KEYS` を使用したカスタムキーのマスキングにも対応しています。これは、PII には該当しないものの秘匿する必要がある追加の機密データを保護したい場合に役立ちます。例としては次のようなものがあります。

* API キー
* 認証ヘッダー
* トークン
* 内部 ID
* 設定値

<div id="pre-defined-redact_keys">
  ### 事前定義済みの `REDACT_KEYS`
</div>

Weave ではデフォルトで、以下の機密性の高いキーが自動的にマスクされます。

```json
[
  "api_key",
  "auth_headers",
  "authorization"
]
```

<div id="adding-your-own-keys">
  ### 独自キーの追加
</div>

trace からマスクしたい独自のカスタムキーを、このリストに追加できます。

```python lines
import weave
from weave.utils import sanitize

client = weave.init("my-project", settings={"redact_pii": True})

# リダクトするカスタムキーを追加する
sanitize.add_redact_key("client_id")
sanitize.add_redact_key("token")

client_id = "123"
token = "789"

@weave.op
def test(client_id, token):
    return client_id + token

test(client_id, token)
```

Weave UI で表示すると、`client_id` と `token` の値は `"REDACTED"` と表示されます。

```python lines
client_id = "REDACTED"
token = "REDACTED"
```

<div id="usage-information">
  ## 利用に関する情報
</div>

* この機能は Python SDK でのみ利用できます。
* Presidio への依存関係により、redaction 機能を有効にすると処理時間が長くなります。
