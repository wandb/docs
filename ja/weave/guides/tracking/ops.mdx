Weave の op は、すべての Call を自動的にログするバージョン管理された関数です。

<Tabs>
  <Tab title="Python">
    op を作成するには、Python 関数に `weave.op()` デコレーターを付けます。

    ```python lines
    import weave

    @weave.op()
    def track_me(v):
        return v + 5

    weave.init('intro-example')
    track_me(15)
    ```

    op を呼び出すと、前回の呼び出しからコードが変更されている場合は新しい op バージョンが作成され、関数の入力と出力がログされます。

    `@weave.op()` でデコレートした関数は、呼び出し前に `weave.init('your-project-name')` を呼び出さない場合、通常どおり動作します (コードのバージョン管理とトラッキングは行われません) 。

    op は Weave のツールベルトを使用して [Serve](/ja/weave/guides/tools/serve) または [デプロイ](/ja/weave/guides/tools/deploy) できます。
  </Tab>

  <Tab title="TypeScript">
    op を作成するには、TypeScript 関数を `weave.op` でラップします。

    ```typescript lines
    import * as weave from 'weave'

    function trackMe(v: number) {
        return v + 5
    }

    const trackMeOp = weave.op(trackMe)
    trackMeOp(15)

    // インラインで記述することもでき、より便利な場合があります
    const trackMeInline = weave.op((v: number) => v + 5)
    trackMeInline(15)
    ```
  </Tab>
</Tabs>

<div id="apply-kinds-and-colors">
  ## カインドとカラーを適用する
</div>

Weave UI で Ops をより整理しやすくするために、コード内の `@weave.op` デコレーターに `kind` および `color` 引数を追加することで、カスタムのカインドとカラーを適用できます。たとえば、以下のコードでは、親関数に `LLM` の `kind` と `blue` の `color` を適用し、ネストされた関数に `tool` の `kind` と `red` の `color` を適用しています。

<Tabs>
  <Tab title="Python">
    ```python lines
    import weave

    weave.init("<your-team-name>/<your-project-name>")

    @weave.op(kind="LLM", color="blue")
    def llm_func():
        @weave.op(kind="tool", color="red")
        def tool_func():
            return "tool result"

        tool_result = tool_func()
        
        return f"llm result with {tool_result}"

    llm_func()
    ```
  </Tab>

  <Tab title="TypeScript">
    ```plaintext
    This feature is not available in TypeScript yet.
    ```
  </Tab>
</Tabs>

これにより、Weave UI の Ops にカラーとカインドが適用されます。表示例は以下のとおりです。

<Frame>
  ![Weave UI に、LLM カインドと blue カラーを持つ親 Call、および tool カインドと red カラーを持つネストされた Call が表示されています。](/images/weave/weave_colors_kinds.png)
</Frame>

使用可能な `kind` の値は以下のとおりです。

* `agent`
* `llm`
* `tool`
* `search`

使用可能な `color` の値は以下のとおりです。

* `red`
* `orange`
* `yellow`
* `green`
* `blue`
* `purple`

<div id="customize-logged-inputs-and-outputs">
  ## ログされる入力と出力のカスタマイズ
</div>

<Tabs>
  <Tab title="Python">
    元の関数を変更せずにWeaveがログするデータを変更したい場合 (例えば、機密データを隠すため) 、Op デコレーターに `postprocess_inputs` と `postprocess_output` を渡すことができます。

    `postprocess_inputs` は、キーが引数名で値が引数の値であるdictを受け取り、変換された入力のdictを返します。

    `postprocess_output` は、通常関数から返される任意の値を受け取り、変換された出力を返します。

    ```python lines
    from dataclasses import dataclass
    from typing import Any
    import weave

    @dataclass
    class CustomObject:
        x: int
        secret_password: str

    def postprocess_inputs(inputs: dict[str, Any]) -> dict[str, Any]:
        return {k:v for k,v in inputs.items() if k != "hide_me"}

    def postprocess_output(output: CustomObject) -> CustomObject:
        return CustomObject(x=output.x, secret_password="REDACTED")

    @weave.op(
        postprocess_inputs=postprocess_inputs,
        postprocess_output=postprocess_output,
    )
    def func(a: int, hide_me: str) -> CustomObject:
        return CustomObject(x=a, secret_password=hide_me)

    weave.init('hide-data-example') # 🐝
    func(a=1, hide_me="password123")
    ```
  </Tab>

  <Tab title="TypeScript">
    ```plaintext
    この機能はTypeScriptではまだ利用できません。
    ```
  </Tab>
</Tabs>

<div id="control-sampling-rate">
  ## サンプリングレートの制御
</div>

<Tabs>
  <Tab title="Python">
    `@weave.op` デコレーターの `tracing_sample_rate` パラメーターを設定することで、op の Call がトレースされる頻度を制御できます。これは、Call の一部のみをトレースすれば十分な高頻度 op に役立ちます。

    Weave はサンプリングレートをルート Call にのみ適用することに注意してください。ある op にサンプリングレートが設定されていても、別の op が先にそれを呼び出した場合、そのサンプリングレートは無視されます。

    ```python lines
    @weave.op(tracing_sample_rate=0.1)  # Call の約10%のみトレース
    def high_frequency_op(x: int) -> int:
        return x + 1

    @weave.op(tracing_sample_rate=1.0)  # 常にトレース（デフォルト）
    def always_traced_op(x: int) -> int:
        return x + 1
    ```

    op の Call がサンプリングされない場合:

    * 関数は通常どおり実行されます
    * Trace Data は Weave に送信されません
    * その Call に対して子 op もトレースされません

    サンプリングレートは 0.0 以上 1.0 以下である必要があります。
  </Tab>

  <Tab title="TypeScript">
    ```plaintext
    この機能はまだ TypeScript では使用できません。
    ```
  </Tab>
</Tabs>

<div id="control-call-link-output">
  ## Call linkの出力を制御する
</div>

ログ記録中にCall linkの出力を抑制したい場合は、`WEAVE_PRINT_CALL_LINK` 環境変数を `false` に設定できます。これにより、出力の冗長性を抑え、ログの煩雑さを軽減できます。

```bash lines
export WEAVE_PRINT_CALL_LINK=false
```

<div id="deleting-an-op">
  ## Op の削除
</div>

<Tabs>
  <Tab title="Python">
    Op のバージョンを削除するには、Op の ref に対して `.delete()` を呼び出します。

    ```python lines
    weave.init('intro-example')
    my_op_ref = weave.ref('track_me:v1')
    my_op_ref.delete()
    ```

    削除された Op にアクセスするとエラーが返されます。
  </Tab>

  <Tab title="TypeScript">
    ```plaintext
    この機能はまだ TypeScript では使用できません。
    ```
  </Tab>
</Tabs>
