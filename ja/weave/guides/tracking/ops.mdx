---
title: "Ops を使って関数呼び出しを自動追跡する"
description: "すべての呼び出しを自動でログするバージョン管理された関数 (Weave)"
---

Weave op は、すべての呼び出しを自動でログするバージョン管理された関数です。

<Tabs>
  <Tab title="Python">
    op を作成するには、Python 関数に `weave.op()` をデコレートします。

    ```python lines lines
    import weave

    @weave.op()
    def track_me(v):
        return v + 5

    weave.init('intro-example')
    track_me(15)
    ```

    op を呼び出すと、コードが前回の呼び出し時から変更されている場合には新しい op バージョンが作成され、関数の入力と出力がログに記録されます。

    `@weave.op()` でデコレートされた関数は、呼び出す前に `weave.init('your-project-name')` を呼び出さなければ、（コードのバージョン管理や追跡なしで）通常どおり動作します。

    Ops は、Weave toolbelt を使って [served](/ja/weave/guides/tools/serve) または [deployed](/ja/weave/guides/tools/deploy) できます。
  </Tab>

  <Tab title="TypeScript">
    op を作成するには、TypeScript 関数を `weave.op` でラップします。

    ```typescript lines
    import * as weave from 'weave'

    function trackMe(v: number) {
        return v + 5
    }

    const trackMeOp = weave.op(trackMe)
    trackMeOp(15)

    // これはインラインでも実行でき、その方が便利な場合もあります
    const trackMeInline = weave.op((v: number) => v + 5)
    trackMeInline(15)
    ```
  </Tab>
</Tabs>

<div id="apply-kinds-and-colors">
  ## 種類とカラーを適用する
</div>

Weave UI 内の ops をより整理しやすくするために、コード内の `@weave.op` デコレーターに `kind` と `color` 引数を追加して、カスタムの kind と color を指定できます。たとえば、次のコードでは親関数に `LLM` の `kind` と `blue` の `color` を、ネストした関数に `tool` の `kind` と `red` の `color` を適用しています。

<Tabs>
  <Tab title="Python">
    ```python lines
    import weave

    weave.init("<your-team-name>/<your-project-name>")

    @weave.op(kind="LLM", color="blue")
    def llm_func():
        @weave.op(kind="tool", color="red")
        def tool_func():
            return "tool result"

        tool_result = tool_func()
        
        return f"llm result with {tool_result}"

    llm_func()
    ```
  </Tab>

  <Tab title="TypeScript">
    ```plaintext
    この機能は TypeScript ではまだ利用できません。
    ```
  </Tab>
</Tabs>

これにより、Weave UI の ops に対して、次のように kind と color が適用されます。

  ![Weave UI で、親の呼び出しに LLM の kind と blue の color、ネストした呼び出しに tool の kind と red の color が設定されている例。](/images/weave/weave_colors_kinds.png)

利用可能な `kind` の値は次のとおりです:

* `agent`
* `llm`
* `tool`

利用可能な `color` の値は次のとおりです:

* `red`
* `orange`
* `yellow`
* `green`
* `blue`
* `purple`

<div id="customize-logged-inputs-and-outputs">
  ## ログされる入力と出力をカスタマイズする
</div>

<Tabs>
  <Tab title="Python">
    元の関数を変更せずに weave にログされるデータを変更したい場合（例: 機密データを隠すため）、`postprocess_inputs` と `postprocess_output` を op デコレーターに渡すことができます。

    `postprocess_inputs` はキーが引数名、値が引数の値である dict を受け取り、変換後の入力を含む dict を返します。

    `postprocess_output` は、通常その関数から返される任意の値を受け取り、変換後の出力を返します。

    ```python lines
    from dataclasses import dataclass
    from typing import Any
    import weave

    @dataclass
    class CustomObject:
        x: int
        secret_password: str

    def postprocess_inputs(inputs: dict[str, Any]) -> dict[str, Any]:
        return {k:v for k,v in inputs.items() if k != "hide_me"}

    def postprocess_output(output: CustomObject) -> CustomObject:
        return CustomObject(x=output.x, secret_password="REDACTED")

    @weave.op(
        postprocess_inputs=postprocess_inputs,
        postprocess_output=postprocess_output,
    )
    def func(a: int, hide_me: str) -> CustomObject:
        return CustomObject(x=a, secret_password=hide_me)

    weave.init('hide-data-example') # 🐝
    func(a=1, hide_me="password123")
    ```
  </Tab>

<div id="control-sampling-rate">
  ## サンプリングレートを制御する
</div>

<Tabs>
  <Tab title="Python">
    `@weave.op` デコレーター内の `tracing_sample_rate` パラメーターを設定することで、ある op の呼び出しがどの程度の頻度でトレースされるかを制御できます。これは、高頻度で呼び出される op のうち、一部の呼び出しだけトレースできれば十分な場合に有用です。

    サンプリングレートはルート呼び出しに対してのみ適用される点に注意してください。ある op にサンプリングレートが設定されていても、最初に別の op から呼び出された場合には、そのサンプリングレートは無視されます。

    ```python lines
    @weave.op(tracing_sample_rate=0.1)  # 呼び出しのおよそ 10% だけをトレース
    def high_frequency_op(x: int) -> int:
        return x + 1

    @weave.op(tracing_sample_rate=1.0)  # 常にトレース（デフォルト）
    def always_traced_op(x: int) -> int:
        return x + 1
    ```

    op の呼び出しがサンプリングされなかった場合:

    * 関数は通常どおり実行される
    * Weave にトレースデータは送信されない
    * その呼び出しにおける子 op もトレースされない

    サンプリングレートは 0.0 から 1.0 までの範囲（両端を含む）である必要があります。
  </Tab>

  <Tab title="TypeScript">
    ```plaintext
    この機能は TypeScript ではまだ利用できません。
    ```
  </Tab>
</Tabs>

<div id="control-call-link-output">
  ## コールリンク出力を制御する
</div>

ログ記録中に出力されるコールリンクを抑制したい場合は、環境変数 `WEAVE_PRINT_CALL_LINK` を `false` に設定します。これは、出力の詳細度を下げてログのノイズを減らしたい場合に有用です。

```bash
export WEAVE_PRINT_CALL_LINK=false
```

<div id="deleting-an-op">
  ## op の削除
</div>

<Tabs>
  <Tab title="Python">
    op のバージョンを削除するには、op ref に対して `.delete()` を呼び出します。

    ```python lines
    weave.init('intro-example')
    my_op_ref = weave.ref('track_me:v1')
    my_op_ref.delete()
    ```

    削除された op にアクセスしようとすると、エラーが発生します。
  </Tab>

  <Tab title="TypeScript">
    ```plaintext
    この機能はまだ TypeScript では利用できません。
    ```
  </Tab>
</Tabs>
