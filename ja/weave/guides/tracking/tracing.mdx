<div id="ops">
  ## Ops
</div>

**Op** はバージョン管理および追跡される関数です。関数に `@weave.op()`（Python）をデコレートするか、`weave.op()`（TypeScript）でラップすると、Weave がそのコード、入力、出力、および実行時メタデータを自動的に取得します。Op はトレーシング、評価スコアラー、その他あらゆる追跡対象の計算の構成要素です。

<CodeGroup>
  ```python Python
      @weave.op
      async def my_function(){
        ...  }
  ```

  ```typescript Typescript
  function myFunction() {
      ...
  }

  const myFunctionOp = weave.op(myFunction)
  ```
</CodeGroup>

<div id="calls">
  ## Calls
</div>

**Call** は、Op の実行が記録されたものです。Op が実行されるたびに、Weave は次の情報を保持する Call を作成します。

* 入力引数
* 出力値
* 実行時間とレイテンシー
* （ネストされた Call における）親子関係
* 発生したエラー

Call は Weave のトレーシングシステムの中核を成し、デバッグ、分析、評価のためのデータを提供します。

Call は [OpenTelemetry](https://opentelemetry.io) のデータモデルにおける span に似ています。Call には次のような特徴があります。

* Trace（同じ実行コンテキスト内の Call の集合）に属する
* 親と子の Call を持ち、木構造を形成する

<div id="creating-calls">
  ## Call の作成
</div>

Weave で Call を作成する方法は主に 3 つあります。

<div id="1-automatic-tracking-of-llm-libraries">
  ### 1. LLM ライブラリの自動トラッキング
</div>

<Tabs>
  <Tab title="Python">
    Weave は `openai`、`anthropic`、`cohere`、`mistral` などの [代表的な LLM ライブラリへの呼び出し](/ja/weave/guides/integrations) を自動的にトラッキングします。プログラムの先頭で [`weave.init('project_name')`](/ja/weave/reference/python-sdk#function-init) を呼び出すだけです。

    <Tip>
      Weave のデフォルトのトラッキング動作は、[`weave.init` の `autopatch_settings` 引数](#configure-autopatching) を使って制御できます。
    </Tip>

    ```python lines
    import weave

    from openai import OpenAI
    client = OpenAI()

    # Weave Tracing を初期化
    weave.init('intro-example')

    response = client.chat.completions.create(
        model="gpt-4",
        messages=[
            {
                "role": "user",
                "content": "How are you?"
            }
        ],
        temperature=0.8,
        max_tokens=64,
        top_p=1,
    )
    ```
  </Tab>

  <Tab title="TypeScript">
    Weave は `openai` などの [代表的な LLM ライブラリへの呼び出し](/ja/weave/guides/integrations) を自動的にトラッキングします。

    ```typescript lines
    import OpenAI from 'openai'
    import * as weave from 'weave'

    const client = new OpenAI()

    // Weave Tracing を初期化
    await weave.init('intro-example')

    const response = await client.chat.completions.create({
      model: 'gpt-4',
      messages: [
        {
          role: 'user',
          content: 'How are you?',
        },
      ],
      temperature: 0.8,
      max_tokens: 64,
      top_p: 1,
    });
    ```

    JS / TS プロジェクト向けのセットアップ ガイドの詳細については、[TypeScript SDK: Third-Party Integration Guide](/ja/weave/guides/integrations/js) を参照してください。
  </Tab>
</Tabs>

Call の `summary` 辞書に、メトリクスやその他の呼び出し後の値を保存できます。実行中に `call.summary` を変更すると、追加した値は呼び出し完了時に Weave が計算したサマリー データとマージされます。

<div id="2-decorating-and-wrapping-functions">
  ### 2. 関数のデコレートとラップ
</div>

一方で、LLM アプリケーションには、追跡したい追加のロジック（前処理 / 後処理、プロンプトなど）が含まれていることもよくあります。

<Tabs>
  <Tab title="Python">
    Weave では、[`@weave.op`](/ja/weave/reference/python-sdk/#function-op) デコレーターを使って、これらの呼び出しを手動で追跡できます。例えば次のようになります。

    ```python lines lines
    import weave

    # Weave Tracing を初期化
    weave.init('intro-example')

    # 関数をデコレート
    @weave.op
    def my_function(name: str):
        return f"Hello, {name}!"

    # 関数を呼び出す -- Weave が自動的に入力と出力を追跡する
    print(my_function("World"))
    ```

    [クラスのメソッド](#4-track-class-and-object-methods) も追跡できます。

    #### 同期 &amp; 非同期ジェネレーター関数のトレース

    Weave は、深くネストしたパターンも含めて、同期・非同期両方のジェネレーター関数のトレースをサポートします。

    <Warning>
      ジェネレーターは値を遅延的に `yield` するため、出力はジェネレーターが完全に消費されたときにのみログされます（たとえば `list()` に変換した場合など）。
      トレースに出力を確実に含めるには、ジェネレーターを完全に消費してください（例: `list()` を使用）。
    </Warning>

    ```python lines lines
    from typing import Generator
    import weave

    weave.init("my-project")

    # この関数はシンプルな同期ジェネレーターを使用する。
    # Weave はこの呼び出しとその入力 (`x`) をトレースするが、
    # 出力値はジェネレーターが消費されたとき（例: `list()` 経由）にのみ取得される。
    @weave.op
    def basic_gen(x: int) -> Generator[int, None, None]:
        yield from range(x)

    # ジェネレーター・パイプライン内で使用される通常の同期関数。
    # この関数の呼び出しも Weave によって個別にトレースされる。
    @weave.op
    def inner(x: int) -> int:
        return x + 1

    # 別のトレース対象関数 (`inner`) を呼び出す同期ジェネレーター。
    # `yield` される各値は、`inner` への個別のトレース付き呼び出しから得られる。
    @weave.op
    def nested_generator(x: int) -> Generator[int, None, None]:
        for i in range(x):
            yield inner(i)

    # 上記のジェネレーターを合成する、より複雑なジェネレーター。
    # ここでのトレースは階層的なコールツリーを生成する:
    # - `deeply_nested_generator` (親)
    #   - `nested_generator` (子)
    #     - `inner` (孫)
    @weave.op
    def deeply_nested_generator(x: int) -> Generator[int, None, None]:
        for i in range(x):
            for j in nested_generator(i):
                yield j

    # Weave が出力を取得するには、ジェネレーターを「消費」する必要がある。
    # これは同期・非同期の両方のジェネレーターに当てはまる。
    res = deeply_nested_generator(4)
    list(res)  # すべてのネストした呼び出しと yield のトレースをトリガーする
    ```

    ![Weave におけるジェネレーター関数のトレース。](/weave/guides/tracking/imgs/generators.png)
  </Tab>

  <Tab title="TypeScript">
    Weave では、関数を [`weave.op`](/ja/weave/reference/typescript-sdk/functions/op) でラップすることで、これらの呼び出しを手動で追跡できます。例えば次のようになります。

    ```typescript lines
    import * as weave from 'weave'

    await weave.init('intro-example')

    function myFunction(name: string) {
        return `Hello, ${name}!`
    }

    const myFunctionOp = weave.op(myFunction)
    ```

    ラップをインラインで定義することもできます。

    ```typescript
    const myFunctionOp = weave.op((name: string) => `Hello, ${name}!`)
    ```

    これは関数だけでなく、クラスのメソッドにも有効です。

    ```typescript
    class MyClass {
        constructor() {
            this.myMethod = weave.op(this.myMethod)
        }

        myMethod(name: string) {
            return `Hello, ${name}!`
        }
    }
    ```
  </Tab>
</Tabs>

<div id="getting-a-handle-to-the-call-object-during-execution">
  #### 実行中に Call オブジェクトへのハンドルを取得する
</div>

<Tabs>
  <Tab title="Python">
    場合によっては、`Call` オブジェクトそのものを取得したいことがあります。これは `op.call` メソッドを呼び出すことで実行でき、このメソッドは結果と `Call` オブジェクトの両方を返します。例えば次のようになります。

    ```python lines lines
    result, call = my_function.call("World")
    ```

    その後、`call` を使って追加のプロパティを設定・更新・取得できます（フィードバック用に利用する call の ID を取得する用途が最も一般的です）。

    <Note>
      op がクラスのメソッドである場合は、インスタンスを最初の引数として op に渡す必要があります（以下の例を参照してください）。
    </Note>

    ```python lines lines
    # 最初の引数として `instance` を渡している点に注意してください。
    print(instance.my_method.call(instance, "World"))
    ```

    ```python lines lines
    import weave

    # Weave Tracing を初期化します
    weave.init("intro-example")

    class MyClass:
        # メソッドをデコレート
        @weave.op
        def my_method(self, name: str):
            return f"Hello, {name}!"

    instance = MyClass()

    # メソッドを呼び出す -- Weave が入力と出力を自動的に追跡します
    instance.my_method.call(instance, "World")
    ```
  </Tab>

    ```plaintext
    この機能は TypeScript SDK ではまだ利用できません。
    ```

<div id="set-call-display-name-at-execution">
  #### 実行時に Call の表示名を設定する
</div>

<Tabs>
  <Tab title="Python">
    Call の表示名を上書きしたい場合があります。その場合は、次の 4 つの方法のいずれかで変更できます。

    1. Op を呼び出す際に表示名を変更する:

    ```python lines lines
    result = my_function("World", __weave={"display_name": "My Custom Display Name"})
    ```

    <Note>
      `__weave` 辞書を使うと Call の表示名が設定され、Op の表示名よりも優先されます。
    </Note>

    2. Call ごとに表示名を変更する。この方法では [`Op.call`](/ja/weave/reference/python-sdk/trace/op#function-call) メソッドを使って `Call` オブジェクトを返し、そのあとで [`Call.set_display_name`](/ja/weave/reference/python-sdk/trace/weave_client#method-set_display_name) を用いて表示名を設定できます。

    ```python lines lines
    result, call = my_function.call("World")
    call.set_display_name("My Custom Display Name")
    ```

    3. 特定の Op に対するすべての Call の表示名を変更する:

    ```python lines lines
    @weave.op(call_display_name="My Custom Display Name")
    def my_function(name: str):
        return f"Hello, {name}!"
    ```

    4. `call_display_name` には、`Call` オブジェクトを受け取って文字列を返す関数を指定することもできます。関数が呼び出されるときに `Call` オブジェクトは自動的に渡されるため、関数名、Call の入力値、フィールドなどに基づいて動的に名前を生成できます。

    5. よくあるユースケースとしては、関数名にタイムスタンプを付け加えるだけのものがあります。

       ```py
       from datetime import datetime

       @weave.op(call_display_name=lambda call: f"{call.func_name}__{datetime.now()}")
       def func():
           return ...
       ```

    6. `.attributes` を使ってカスタム メタデータをログすることもできます

       ```py
       def custom_attribute_name(call):
           model = call.attributes["model"]
           revision = call.attributes["revision"]
           now = call.attributes["date"]

           return f"{model}__{revision}__{now}"

       @weave.op(call_display_name=custom_attribute_name)
       def func():
           return ...

       with weave.attributes(
           {
               "model": "finetuned-llama-3.1-8b",
               "revision": "v0.1.2",
               "date": "2024-08-01",
           }
       ):
           func()  # 表示名は "finetuned-llama-3.1-8b__v0.1.2__2024-08-01" になります

           with weave.attributes(
               {
                   "model": "finetuned-gpt-4o",
                   "revision": "v0.1.3",
                   "date": "2024-08-02",
               }
           ):
               func()  # 表示名は "finetuned-gpt-4o__v0.1.3__2024-08-02" になります
       ```

    **技術的な補足:** 「Call」は「Op」から生成されます。Op は `@weave.op` でデコレートされた関数またはメソッドです。
    デフォルトでは、Op の名前は関数名であり、関連する Call も同じ表示名を持ちます。上記の例では、特定の Op に対するすべての Call の表示名を上書きする方法を示しています。場合によっては、ユーザーが Op 自体の名前を上書きしたいこともあります。その場合は次の 2 つの方法のいずれかで実現できます。

    1. いずれかの Call がログされる前に Op の `name` プロパティを設定する

    ```python lines lines
    my_function.name = "My Custom Op Name"
    ```

    2. Op デコレーターで `name` オプションを設定する

    ```python lines lines
    @weave.op(name="My Custom Op Name)
    ```
  </Tab>

  <Tab title="TypeScript">
    Call のデフォルト名を上書きするには、`weave.op()` を呼び出す際に `callDisplayName` オプションを使用します。

    ```typescript lines {2}
    const extractDinosOp = weave.op(extractDinos, {
    callDisplayName: (input: string) => `Your New Display Name`
    });
    ```

    実行後に [Call の表示名を更新](/ja/weave/guides/tracking/tracing#set-display-name) することもできます。
  </Tab>
</Tabs>

<div id="trace-parallel-multi-threaded-function-calls">
  #### 並列（マルチスレッド）関数呼び出しのトレース
</div>

<Tabs>
  <Tab title="Python">
    Weave では、デフォルトで並列呼び出しはすべて個別のルート call として表示されます。同じ親 `op` の下で正しくネストさせるには、[`ThreadPoolExecutor`](/ja/weave/reference/python-sdk/trace/util#class-contextawarethreadpoolexecutor) を使用します。

    次のコードサンプルは、`ThreadPoolExecutor` の使用方法を示します。
    最初の関数 `func` は、`x` を受け取り `x+1` を返す単純な `op` です。2 番目の関数 `outer` は、入力のリストを受け取る別の `op` です。
    `outer` の内部で `ThreadPoolExecutor` と `exc.map(func, inputs)` を使用することで、`func` への各呼び出しは同じ親トレースコンテキストを保持したままです。

    ```python lines
    import weave

    @weave.op
    def func(x):
        return x+1

    @weave.op
    def outer(inputs):
        with weave.ThreadPoolExecutor() as exc:
            exc.map(func, inputs)

    # 使用している Weave プロジェクト名を更新する  
    client = weave.init('my-weave-project')
    outer([1,2,3,4,5])
    ```

    Weave UI では、これは 1 つの親 call に 5 つの子 call がネストされた形で表示されるため、インクリメント処理が並列で実行されていても、完全に階層化されたトレースを取得できます。

    ![Trace UI。outer の単一の親 call と、その下にネストされた 5 つの子 call が表示されている。](/weave/guides/tracking/imgs/threadpoolexecutor.png)
  </Tab>

  <Tab title="TypeScript">
    ```plaintext
    この機能は TypeScript SDK ではまだ利用できません。
    ```
  </Tab>
</Tabs>

<div id="3-manual-call-tracking">
  ### 3. 手動による Call トラッキング
</div>

API を直接使用して、Call を手動で作成することもできます。

<Tabs>
  <Tab title="Python">
    ```python lines lines
    import weave

    # Weave Tracing を初期化
    client = weave.init('intro-example')

    def my_function(name: str):
        # Call を開始
        call = client.create_call(op="my_function", inputs={"name": name})

        # ... 関数本体のコード ...

        # Call を終了
        client.finish_call(call, output="Hello, World!")

    # 関数を呼び出す
    print(my_function("World"))
    ```
  </Tab>

  <Tab title="TypeScript">
    ```plaintext
    この機能はまだ TypeScript SDK では利用できません。
    ```
  </Tab>

  <Tab title="HTTP API">
    * Call を開始: [POST `/call/start`](https://docs.wandb.ai/weave/reference/service-api/calls/call-start)
    * Call を終了: [POST `/call/end`](https://docs.wandb.ai/weave/reference/service-api/calls/call-end)

    ```bash
    curl -L 'https://trace.wandb.ai/call/start' \
    -H 'Content-Type: application/json' \
    -H 'Accept: application/json' \

        "start": {
    ```

<div id="4-track-class-and-object-methods">
  ### 4. クラスおよびオブジェクトメソッドのトラッキング
</div>

クラスおよびオブジェクトメソッドもトラッキングできます。

<Tabs>
  <Tab title="Python">
    `weave.op` を使って、クラスの任意のメソッドをトラッキングできます。

    ```python lines lines
    import weave

    # Weave Tracing を初期化
    weave.init("intro-example")

    class MyClass:
        # メソッドをデコレート
        @weave.op
        def my_method(self, name: str):
            return f"Hello, {name}!"

    instance = MyClass()

    # メソッドを呼び出すと、Weave が自動的に入力と出力をトラッキングします
    print(instance.my_method("World"))
    ```
  </Tab>

  <Tab title="TypeScript">
    <Important>
      **TypeScript でデコレータを使用する**

      TypeScript コードで `@weave.op` デコレータを使用するには、環境が正しく設定されていることを確認してください。

      * **TypeScript v5.0 以降**: デコレータは標準でサポートされており、追加の設定は不要です。
      * **TypeScript v5.0 より前**: デコレータの試験的サポートを有効にします。詳細については、[decorators に関する公式 TypeScript ドキュメント](https://www.typescriptlang.org/docs/handbook/decorators.html)を参照してください。
    </Important>

    #### クラスメソッドをデコレートする

    インスタンスメソッドをトレースするには `@weave.op` を使用します。

    ```typescript
    class Foo {
        @weave.op
        async predict(prompt: string) {
            return "bar"
        }
    }
    ```

    #### 静的クラスメソッドをデコレートする

    クラス内のユーティリティ関数をトレースするために、静的メソッドに `@weave.op` を適用します。

    ```typescript
    class MathOps {
        @weave.op
        static square(n: number): number {
            return n * n;
        }
    }
    ```
  </Tab>
</Tabs>

<div id="viewing-calls">
  ## Calls の表示
</div>

<Tabs>
  <Tab title="Web アプリ">
    Web アプリで call を表示するには、次の手順を実行します:

    1. プロジェクトの **Traces** タブに移動します。
    2. 一覧から表示したい call を探します。
    3. call をクリックして詳細ページを開きます。

    詳細ページには、その call の入力、出力、実行時間、および追加のメタデータが表示されます。

    ```python lines
    import weave

    # クライアントを初期化します
    client = weave.init("your-project-name")

    # ID で特定の call を取得します
    call = client.get_call("call-uuid-here")

    print(call)
    ```
  </Tab>

  <Tab title="TypeScript">
    ```typescript lines
    import * as weave from 'weave'

    // Initialize the client
    const client = await weave.init('intro-example')

    console.log(call)
    ```
  </Tab>

  <Tab title="HTTP API">

    ```bash
    curl -L 'https://trace.wandb.ai/call/read' \
    -H 'Content-Type: application/json' \

        "project_id": "string",
        "id": "string",
    ```

<div id="customize-rendered-traces-with-weavemarkdown">
  ### `weave.Markdown` を使ってトレースのレンダリングをカスタマイズする
</div>

`weave.Markdown` を使うと、元のデータを失うことなくトレース情報の表示方法をカスタマイズできます。これにより、入力と出力を、基盤となるデータ構造を保持したまま、読みやすい整形済みコンテンツのブロックとしてレンダリングできます。

<Tabs>
  <Tab title="Python">
    `@weave.op` デコレータ内で `postprocess_inputs` と `postprocess_output` 関数を使用して、トレースデータを整形します。次のコードサンプルでは、ポストプロセッサーを使って、Weave 内の呼び出しを絵文字とより読みやすいフォーマットでレンダリングしています。

    ```python lines
    import weave

    def postprocess_inputs(query) -> weave.Markdown:
        search_box = f"""
    **Search Query:**
    ``+`
    {query}
    ``+`
    """
        return {"search_box": weave.Markdown(search_box),
                "query": query}

    def postprocess_output(docs) -> weave.Markdown:
        formatted_docs = f"""
    # {docs[0]["title"]}

    {docs[0]["content"]}

    [Read more]({docs[0]["url"]})

    ---

    # {docs[1]["title"]}

    {docs[1]["content"]}

    [Read more]({docs[1]["url"]})
    """
        return weave.Markdown(formatted_docs)

    @weave.op(
        postprocess_inputs=postprocess_inputs,
        postprocess_output=postprocess_output,
    )
    def rag_step(query):
        # S&P 500 に含まれる企業に関する新聞記事の例
        docs = [
            {
                "title": "OpenAI",
                "content": "OpenAI is a company that makes AI models.",
                "url": "https://www.openai.com",
            },
            {
                "title": "Google",
                "content": "Google is a company that makes search engines.",
                "url": "https://www.google.com",
            },
        ]
        return docs

    if __name__ == "__main__":
        weave.init('markdown_renderers')
        rag_step("Tell me about OpenAI")
    ```
  </Tab>

  </Tab>
</Tabs>

<div id="call-schema">
  ### Call スキーマ
</div>

すべてのフィールドの一覧は、[schema](/ja/weave/reference/python-sdk/trace_server/trace_server_interface#class-callschema) を参照してください。

| Property | Type | Description |
|----------|------|-------------|
| `id` | string (uuid) | Call の一意な識別子 |
| `project_id` | string (optional) | 関連付けられたプロジェクトの識別子 |
| `op_name` | string | オペレーション名（参照を指す場合があります） |
| `display_name` | string (optional) | Call のユーザー向け表示名 |
| `trace_id` | string (uuid) | この Call が属する trace の識別子 |
| `parent_id` | string (uuid) | 親 Call の識別子 |
| `started_at` | datetime | Call が開始した時刻のタイムスタンプ |
| `attributes` | Dict[str, Any] | Call に関するユーザー定義メタデータ（*実行中は読み取り専用*） |
| `inputs` | Dict[str, Any] | Call の入力パラメーター |
| `ended_at` | datetime (optional) | Call が終了した時刻のタイムスタンプ |
| `exception` | string (optional) | Call が失敗した場合のエラーメッセージ |
| `output` | Any (optional) | Call の結果 |
| `summary` | Optional[SummaryMap] | 実行後のサマリー情報。カスタムメトリクスを記録するために、実行中に変更できます。 |
| `wb_user_id` | Optional[str] | 関連付けられた Weights &amp; Biases ユーザー ID |
| `wb_run_id` | Optional[str] | 関連付けられた Weights &amp; Biases run ID |
| `deleted_at` | datetime (optional) |（該当する場合）Call が削除された時刻のタイムスタンプ |

上の表は、Weave における Call の主要なプロパティをまとめたものです。各プロパティは、関数呼び出しを追跡および管理する上で重要な役割を果たします。

* `id`、`trace_id`、`parent_id` フィールドは、システム内で Call を整理し、相互に関連付けるのに役立ちます。
* `started_at` と `ended_at` といったタイミング情報により、パフォーマンス分析が可能になります。
* `attributes` と `inputs` フィールドは、Call のコンテキストを提供します。`attributes` は Call が開始されると固定されるため、呼び出し前に `weave.attributes` で設定してください。`output` と `summary` は結果を保持し、実行中に `summary` を更新して追加のメトリクスを記録できます。
* Weights &amp; Biases とのインテグレーションは、`wb_user_id` と `wb_run_id` によって行われます。

この包括的なプロパティのセットにより、プロジェクト全体を通して関数呼び出しを詳細に追跡・分析できます。

Calculated Fields:

* Cost
  * Duration
  * Status

<div id="view-a-wb-run-in-the-traces-table">
  ## Traces テーブルで W&amp;B run を表示する
</div>

Weave を使うと、コード内の関数呼び出しをトレースし、それらを実行された [W&amp;B runs](https://docs.wandb.ai/models/runs/) に直接リンクできます。\
@weave.op() で関数をトレースし、wandb.init() コンテキスト内で呼び出すと、Weave はそのトレースを自動的に対応する W&amp;B run と関連付けます。\
関連付けられた run へのリンクは、Traces テーブルに表示されます。

<Tabs>
  <Tab title="Python">
    次の Python コードは、トレースされた処理が `wandb.init()` コンテキスト内で実行されたときに、どのように W&amp;B run にリンクされるかを示しています。これらのトレースは Weave UI に表示され、対応する run に関連付けられます。

    ```python lines 
    import wandb
    import weave

    def example_wandb(projname):
        # projname を entity と project に分割
        entity, project = projname.split("/", 1)

        # トレース用に Weave コンテキストを初期化
        weave.init(projname)

        # トレース可能な処理を定義
        @weave.op()
        def say(message: str) -> str:
            return f"I said: {message}"

        # 1 回目の W&B run
        with wandb.init(
            entity=entity,
            project=project,
            notes="Experiment 1",
            tags=["baseline", "paper1"],
        ) as run:
            say("Hello, world!")
            say("How are you!")
            run.log({"messages": 2})

        # 2 回目の W&B run
        with wandb.init(
            entity=entity,
            project=project,
            notes="Experiment 2",
            tags=["baseline", "paper1"],
        ) as run:
            say("Hello, world from experiment 2!")
            say("How are you!")
            run.log({"messages": 2})

    if __name__ == "__main__":
        # 実際の W&B ユーザー名 / プロジェクトに置き換えてください
        example_wandb("your-username/your-project")
    ```

    このコードサンプルを使うには、次の手順を実行します。

    1. ターミナルで依存関係をインストールします:

    ```bash
    pip install wandb weave
    ```

    2. W&amp;B にログインします:

    ```bash
    wandb login
    ```

    3. スクリプト内の `your-username/your-project` を、実際の W&amp;B entity/project に置き換えます。
    4. スクリプトを実行します:

    ```bash
    python weave_trace_with_wandb.py
    ```

    5. [https://weave.wandb.ai](https://weave.wandb.ai) にアクセスし、自分のプロジェクトを選択します。
    6. **Traces** タブでトレース出力を確認します。関連付けられた run へのリンクは Traces テーブルに表示されます。
  </Tab>

  </Tab>

<div id="configure-autopatching">
  ## 自動パッチ適用の設定
</div>

デフォルトでは、Weave は `openai`、`anthropic`、`cohere`、`mistral` など一般的な LLM ライブラリへの呼び出しを自動的にパッチして追跡します。

<Tabs>
  <Tab title="Python">
    <Warning>
      `autopatch_settings` 引数は非推奨です。暗黙的なパッチ適用を無効にするには `implicitly_patch_integrations=False` を使用するか、`patch_openai(settings={...})` のような特定のパッチ関数を呼び出して、インテグレーションごとに設定してください。
    </Warning>

    ### すべての自動パッチ適用を無効化する

    ```python lines
    weave.init(..., implicitly_patch_integrations=False)
    ```

    ### 特定のインテグレーションのみを有効化する

    ```python lines
    import weave

    weave.init(..., implicitly_patch_integrations=False)

    # その後、必要なインテグレーションだけを手動でパッチする
    weave.integrations.patch_anthropic()
    weave.integrations.patch_cohere()
    ```

    ### 入力と出力を後処理する

    パッチ関数に設定を渡すことで、入力と出力（PII データなど）の扱い方をカスタマイズできます。

    ```python lines
    import weave.integrations

    def redact_inputs(inputs: dict) -> dict:
        if "email" in inputs:
            inputs["email"] = "[REDACTED]"
        return inputs

    weave.init(...)
    weave.integrations.patch_openai(
        settings={
            "op_settings": {"postprocess_inputs": redact_inputs}
        }
    )
    ```
  </Tab>

  <Tab title="TypeScript">
    TypeScript SDK は OpenAI と Anthropic に対する自動パッチ適用のみをサポートしています。OpenAI は Weave をインポートしたときに自動的にパッチされ、追加の設定は不要です。

    さらに、TypeScript SDK は次の機能をサポートしません。

    * 自動パッチ適用の設定や無効化
    * 入力/出力の後処理

    自動パッチ適用が機能しない例外的なケース（ESM、Next.js のようなバンドラ）では、明示的なラッピングを使用してください。

    ```typescript
    import OpenAI from 'openai'
    import * as weave from 'weave'
    import { wrapOpenAI } from 'weave'

    const client = wrapOpenAI(new OpenAI())
    await weave.init('your-team/my-project')
    ```

    ESM のセットアップやトラブルシューティングの詳細については、[TypeScript SDK Integration Guide](/ja/weave/guides/integrations/js) を参照してください。
  </Tab>
</Tabs>

詳細については、[Weave を PII データと併用する方法](/ja/weave/cookbooks/pii) を参照してください。

<div id="how-do-i-disable-tracing">
  ### トレースを無効化するには？

<div id="environment-variable">
  #### 環境変数
</div>

プログラム全体で無条件に tracing を無効化したい場合は、環境変数 `WEAVE_DISABLED=true` を設定します。

`WEAVE_DISABLED` は関数定義時点で一度だけ読み込まれます。この変数を使って実行時に tracing の有効・無効を切り替えることはできません。

<div id="client-initialization">
  #### クライアント初期化
</div>

特定の初期化処理について、条件に応じてトレースを有効または無効にしたい場合があります。その場合は、init の設定で `disabled` フラグを指定してクライアントを初期化できます。

<Tabs>
  <Tab title="Python">
    ```python lines
    import weave

    # クライアントを初期化
    client = weave.init(..., settings={"disabled": True})
    ```
  </Tab>

  <Tab title="TypeScript">
    ```plaintext
    この機能は TypeScript SDK ではまだ利用できません。
    ```
  </Tab>
</Tabs>

<div id="context-manager">
  #### コンテキストマネージャー
</div>

特定のコードブロックに対して条件付きでトレースを無効化するには、トレース用のコンテキストマネージャーを使用できます。`with tracing_disabled()` を使うと、**`with` ブロック内で実行される関数呼び出しに対してのみ** トレースを抑制できます。これは、アプリケーション コード内で、ログに記録しない Call の範囲を限定するために使用することを想定しています。

```python lines
import weave
from weave.trace.context.call_context import tracing_disabled

client = weave.init('your-team/your-project-name')

@weave.op
def my_op():
    ...

with tracing_disabled():
    my_op()
```

関数を定義した時点でトレースの動作は固定されますが、アプリケーションロジックと組み合わせることで、実行時に制御できます。たとえば、条件分岐の中でコンテキストマネージャーを使い、実行時の値に基づいてトレースを動的に有効または無効にできます。

```python lines
if should_trace:
    my_op()
else:
    with tracing_disabled():
        my_op()
```

<div id="how-do-i-capture-information-about-a-call">
  ### Call の情報はどのように取得できますか？
</div>

通常は `op` を直接呼び出します。

```python lines
@weave.op
def my_op():
    ...

my_op()
```

ただし、`op` の `call` メソッドを呼び出すことで、Call オブジェクト自体に直接アクセスすることもできます。

```python lines
@weave.op
def my_op():
    ...

output, call = my_op.call()
```

ここからは、入力、出力、その他のメタデータを含む Call に関するすべての情報が `call` オブジェクトに格納されます。
