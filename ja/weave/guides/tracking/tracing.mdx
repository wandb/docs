<div id="ops">
  ## Ops
</div>

**Op** は、バージョン管理され、追跡される関数です。関数を `@weave.op()` (Python) でデコレートするか、`weave.op()` (TypeScript) でラップすると、Weave はそのコード、入力、出力、および実行メタデータを自動的に取得します。Ops は、トレース、評価 Scorer、および追跡されたあらゆる計算の構成要素です。

<CodeGroup>
  ```python Python
      @weave.op
      async def my_function(){
        ...  }
  ```

  ```typescript Typescript
  function myFunction() {
      ...
  }

  const myFunctionOp = weave.op(myFunction)
  ```
</CodeGroup>

<div id="calls">
  ## コール
</div>

**Call** は、Op のログに記録された実行です。Op が実行されるたびに、Weave は以下の内容をキャプチャした Call を作成します：

* 入力引数
* 出力値
* タイミングとレイテンシ
* 親子関係 (ネストされたコールに対して)
* 発生したエラー

コールは Weave UI で **トレース** として表示され、デバッグ、分析、評価のためのデータを提供します。Call の完全なオブジェクト構造とプロパティについては、[Call スキーマ リファレンス](/ja/weave/guides/tracking/call-schema-reference) を参照してください。

コールは [OpenTelemetry](https://opentelemetry.io) データモデル内のスパンに似ています。コールは次のことができます：

* 同じ実行コンテキスト内のコールからなるトレースに属する
* 親および子 Call を持ち、ツリー構造を形成する
