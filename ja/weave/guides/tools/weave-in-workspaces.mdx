<div id="log-traces-during-model-training-runs">
  # モデル トレーニング run 中にトレースをログする
</div>

<Frame>
  ![Weave Trace in Workspaces](/images/weave/traces-in-workspace.png)
</Frame>

モデル トレーニング run 中に Weave トレースをログし、個人用 Workspace で表示できるようになりました。

個人用 Workspace は、UI 内でカスタマイズできるダッシュボードで、モデル トレーニング run の詳細な分析やデータ可視化を確認できます。Workspace に Weave パネルを追加すると、モデル run 中にログされたトレース データを表示してアクセスできるため、トレーニング中のモデルの挙動や性能をより深く理解するのに役立ちます。

たとえば、LLM モデルをファインチューニングしていて、W&amp;B Workspace で step 500 に精度が低下していることが分かった場合、統合された Weave トレースによって、評価基準を満たさない冗長すぎる応答をモデルが生成し始めた、といったように、何が起きたのかを正確に把握できます。

<div id="use-weave-panels">
  ## Weave Panels を使用する
</div>

[ML パイプライン内の関数に `@weave.op` デコレーターを付けると](/ja/weave/quickstart)、それらの実行情報が自動的に取得され、個人の Workspace でアクセスできるようになります。

たとえば、次のスクリプトは、Weave トレースが W&amp;B のトレーニング run とどのように統合されるかを示しています。これは、各トレーニング step で W&amp;B にメトリクスをログすると同時に、詳細な Weave トレースも作成する機械学習のトレーニングループをシミュレートしたものです。関数に付与された `@weave.op` デコレーターによって、入力、出力、実行の詳細が自動的に取得されます。トレーニングループが `wandb.log()` でメトリクスをログすると、Weave のトレース情報もプロジェクトにログされます。

```python lines
import wandb
import weave
import random

# 同じプロジェクトで W&B と Weave を初期化する
project = "my-workspace-project"
weave.init(project)

@weave.op
def evaluate_model(model_state, epoch):
    # 評価メトリクスをシミュレートする
    accuracy = 0.7 + epoch * 0.02 + random.uniform(-0.05, 0.05)
    
    # 特定のモデル動作をトラッキングする
    test_responses = {
        "conciseness_check": "Model generated 500 words for simple question",
        "accuracy_check": "Model answered 8/10 questions correctly",
        "hallucination_check": "No factual errors detected"
    }
    
    return {
        "accuracy": accuracy,
        "diagnostic_results": test_responses
    }

@weave.op
def training_step(epoch, lr):
    # トレーニングロジックをシミュレートする
    loss = 1.0 / (epoch + 1) + random.uniform(-0.1, 0.1)
    
    # トレースを使った評価
    eval_results = evaluate_model(f"checkpoint_{epoch}", epoch)
    
    return {
        "loss": loss,
        "accuracy": eval_results["accuracy"],
        "diagnostics": eval_results["diagnostic_results"]
    }

# トレーニングループ
with wandb.init(project=project, config={"lr": 0.01}) as run:
    for epoch in range(5):
        # Weave トレースでトレーニングを実行する
        results = training_step(epoch, run.config.lr)
        
        # W&B にログする - インテグレーションポイントを作成する
        run.log({
            "epoch": epoch,
            "loss": results["loss"],
            "accuracy": results["accuracy"]
        })
```

このサンプルスクリプトを実行すると、トレースが他の run 情報と一緒にどのようにログされるかを確認できます。

run の実行中または完了後に Workspace 内のトレース情報を表示するには、run の開始時にターミナルに表示されるリンク (次のような形式です: `wandb: 🚀 View run at https://wandb.ai/wandb/my-project/runs/<run-ID>`) を開くか、UI から Workspace にアクセスします。

UI から Workspace にアクセスするには:

1. W&amp;B UI を開き、**Projects** タブをクリックします。プロジェクトの一覧が表示されます。
2. プロジェクト一覧から、run をログしたプロジェクトをクリックします。**Workspaces** ページが開きます。
3. Workspace を自動 Workspace として設定している場合、Workspace には Data Visualization と run に関する情報が自動的に表示されます。run の Trace Data は Workspace の **Weave** セクションにあります。Workspace が手動 Workspace の場合は、**Add panels** をクリックし、Add Panels メニューの **Weave** セクションから新しいパネルを選択して、Weave パネルを追加できます。

Workspace の詳細については、[View experiments results](https://docs.wandb.ai/models/track/workspaces) を参照してください.

<div id="reference-wb-artifacts-in-a-weave-trace">
  ## Weave トレースで W&amp;B Artifacts を参照する
</div>

Weave トレースでは、[W&amp;B Artifacts](/ja/models/artifacts) (モデル、データセット、チェックポイントなど) を参照できます。これにより、Weave UI にアーティファクトの詳細ページを直接開くクリック可能なリンクが作成され、特定の操作でどのアーティファクトのバージョンが使用されたかを追跡しやすくなります。

たとえば、2 つのバージョンのモデル (`v1` と `v2`) をトレーニングした場合、そのモデルをクエリするときに、`v1` のアーティファクト参照をトレースの属性として追加できます。これにより、どのアーティファクトのバージョンが特定の出力を生成したのかを正確に追跡できます。

アーティファクトをトレースに関連付けるには、`wandb-artifact:///` 接頭辞とアーティファクトの完全名を連結して、次の構文でアーティファクトの URL を構築します: `wandb-artifact:///<full-name-of-artifact>`。その後、次のいずれかの方法で Weave に渡します。

* 任意の属性として渡す
* `Model` の属性として渡す

アーティファクトの完全名は、W&amp;B UI の詳細ページで確認できます。形式は次のとおりです: `<entity>/<project>/<artifact_name>:<version>`。完全名は、W&amp;B UI のアーティファクト詳細ページからコピーできます。

<div id="add-it-as-an-attribute-of-a-trace">
  ### trace の属性として追加する
</div>

trace の属性としてアーティファクトを追加するには、`weave.attributes()` コンテキストマネージャーにキーと値のペアとして渡します。

<Tabs>
  <Tab title="Python">
    ```python lines {10}
    import weave

    weave.init("your-team-name/your-project-name")

    @weave.op
    def my_function(name: str):
        return f"Hello, {name}!"

    # 任意のキー名を追加し、値としてアーティファクトの URL を指定します
    with weave.attributes({'artifact_id': 'wandb-artifact:///team-name/project-name/run-38m4t5ja-history:v0'}):
        result = my_function("World")
    ```
  </Tab>

  <Tab title="TypeScript">
    ```typescript lines {12}
    import {init, op, withAttributes} from 'weave';

    async function main() {
      await init('wandb/docs');

      const myFunction = op(async function myFunction(name: string) {
        return `Hello, ${name}!`;
      });

      // 任意のキー名を追加し、値としてアーティファクトの URL を指定します
      const result = await withAttributes(
        {artifact_id: "wandb-artifact:///team-name/project-name/run-38m4t5ja-history:v0"},
        async () => myFunction('World')
      );

      console.log('Result:', result);
    }

    main().catch(console.error);
    ```
  </Tab>
</Tabs>

<div id="add-an-artifact-as-an-attribute-of-a-model">
  ### `Model` の属性としてアーティファクトを追加する
</div>

`Model` の属性としてアーティファクトを追加するには、モデルをインスタンス化する際にアーティファクトの URL を渡します。

<Note>
  `Model` クラスは現在、Weave Python SDK でのみサポートされています。
</Note>

```python lines {7,17}
import weave

weave.init("your-team-name/your-project-name")

class MyModel(weave.Model):
    # アーティファクトの参照をモデルの属性として保存する
    artifact_id: str
    temperature: float = 0.7
    
    @weave.op
    def predict(self, query: str) -> str:
        # ここにモデルの推論ロジックを記述する
        return f"Response to: {query}"

# アーティファクトの URL をモデルの属性として渡す
model = MyModel(
    artifact_id="wandb-artifact:///team-name/project-name/run-38m4t5ja-history:v0"
)

result = model.predict("Hello, World!")
```
