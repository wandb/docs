---
title: "Weave プロジェクトを管理する"
description: "Weave プロジェクトを使用して、トレース、プロンプト、評価、モデル、ダッシュボードなどの関連アセットを整理します。"
---

Weave プロジェクトを使うと、トレース、評価、モデル、データセット、そのほかの Weave アセットなど、関連するアセットを整理できます。プロジェクトを使用すると、チーム内の整理がしやすくなり、コラボレーションと共有を効率化できるほか、LLM アプリケーションの進化を継続的にトラッキングできます。

<div id="project-resources">
  ## プロジェクト リソース
</div>

プロジェクトには、次のリソースが保存されます。

| リソース             | 説明                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| ---------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Traces**       | デバッグと分析のために、入力、出力、レイテンシ、token usage を取得する、個々の LLM Call、関数実行、アプリケーション フローです。詳しくは [Traces](/ja/weave/guides/tracking/tracing) を参照してください。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| **Evaluations**  | 品質、accuracy、その他のメトリクスを測定する scoring functions を使用して、データセットに対するモデル性能を比較するテスト結果です。詳しくは [Evaluations](/ja/weave/guides/core-types/evaluations) を参照してください。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| **Monitors**     | 本番環境で、LLM-as-a-judge scorers を使用して関数呼び出しのサブセットを自動的にスコアリングし、時間の経過に伴う品質メトリクスを追跡するバックグラウンド プロセスです。詳しくは [Monitors](/ja/weave/guides/evaluation/monitors) を参照してください。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                |
| **Leaderboards** | 異なる評価 run 間でモデル性能を順位付きで比較し、最も高い性能の設定を特定するのに役立ちます。詳しくは [Leaderboards](/ja/weave/guides/core-types/leaderboards) を参照してください。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| **Threads**      | 関連する Call を共有 thread ID の下にグループ化し、完全なユーザー インタラクションを分析するための、複数ターンの会話およびセッション ベースのワークフローです。詳しくは [Threads](/ja/weave/guides/tracking/threads) を参照してください。                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| **Assets**       | **Assets** タブでは、プロジェクト内のバージョン管理された object を保存および整理します: <ul><li>**Models**: prompts、parameters、設定を取得するバージョン管理されたモデル定義で、再現性と比較のために change を自動的に追跡します。詳しくは [Models](/ja/weave/guides/core-types/models) を参照してください。</li><li>**Datasets**: ダウンロード、参照、および評価の実行に使用できる Weave object として保存されたテスト例のコレクションです。詳しくは [Datasets](/ja/weave/guides/core-types/datasets) を参照してください。</li><li>**Prompts**: モデル設定でテストおよび再利用できる、バージョン管理された system prompt です。詳しくは [Prompts](/ja/weave/guides/core-types/prompts) を参照してください。</li><li>**Scorers**: 品質、安全性、または custom metrics を測定するために、モデルの inputs と outputs を評価する関数です。詳しくは [Scorers](/ja/weave/guides/evaluation/scorers) を参照してください。</li><li>**Ops**: Call を生成し、関数の変更履歴を保持する `@weave.op()` でデコレートされた、自動的にバージョン管理および追跡される関数です。詳しくは [Ops](/ja/weave/guides/tracking/ops) を参照してください。</li><li>**Other objects and metadata**: プロジェクト内に保存される、追加の Weave object、custom types、および関連 metadata です。</li></ul> |

<div id="project-paths">
  ## プロジェクトパス
</div>

[Teams](/ja/platform/hosting/iam/org_team_struct#team) (Entities とも呼ばれます) は、W&amp;B 組織内のプロジェクトを所有および管理します。チーム名とプロジェクト名を組み合わせたものが、W&amp;B UI におけるプロジェクトへのパスになります。また、このパスはコード内でも使用して、Weave リソースをそのプロジェクトに関連付けます。

たとえば、チームの名前が `my-team` で、プロジェクトの名前が `my-project` の場合、プロジェクトへのパスは次のとおりです。

```
my-team/my-project
```

コード内で Weave を初期化する際は、このパスを使用して、アプリのデータをプロジェクトに関連付けます。

```
weave.init('my-team/my-project')
```

<div id="create-and-find-projects">
  ## プロジェクトを作成して見つける
</div>

W&amp;B UI を使用するか、コードで新しいプロジェクトパスを設定することで、Weave プロジェクトを作成できます。

<Tabs>
  <Tab title="W&B UI">
    W&amp;B UI を使用して新しいプロジェクトを作成するには、次の手順を実行します。

    1. [wandb.ai](https://wandb.ai) にアクセスします。
    2. **New project** を選択します。**Create a new project** モーダルダイアログが開きます。
    3. モーダルで、次のフィールドを設定します。
       * **Team**: プロジェクトに関連付ける既存のチーム。
       * **名**: プロジェクトの名前。
       * **Project visibility**: プロジェクトを表示および編集できるユーザーの範囲。
       * **Description**:  (任意) プロジェクトの目的を追跡して整理するのに役立つ、プロジェクトの短い説明。
    4. 必要なフィールドを設定したら、**Create project** を選択します。プロジェクトが作成されます。

    新しく作成したプロジェクトを見つけるには、左上隅の **Weights &amp; Biases logo** を選択して **Global navigation** メニューを開きます。このメニューには、最新のプロジェクトと所属しているチームが表示されるため、すばやくコンテキストを切り替えられます。プロジェクトを選択して、そのプロジェクトの Workspace を開きます。

    または、右上隅の名前の下にあるメニューから **プロフィール** を開きます。**Projects** タブには、すべてのプロジェクトが表示されます。

    新しいプロジェクトの Workspace を開くと、UI に W&amp;B Models と W&amp;B Weave の両方のクイックスタートが表示されます。プロジェクトのツールバーにも、各プロダクト用のツールが含まれています。コードを通じて Weave データをプロジェクトに関連付けると、サイドナビゲーションには Weave のツールのみが表示され、Weave ダッシュボードが利用可能になります。
  </Tab>

  <Tab title="Weave SDK">
    コードから直接 Weave プロジェクトを作成するには、`weave.init()` の引数として新しいプロジェクトパスを指定します。

    <CodeGroup>
      ```python
      import weave

      weave.init('my-team/my-llm-app')
      ```

      ```typescript
      import * as weave from 'weave';

      await weave.init('my-team/my-llm-app');
      ```
    </CodeGroup>

    コードを実行すると、Weave は W&amp;B アカウントに新しいプロジェクトを作成し、run から該当するデータをプロジェクトにアップロードします。Weave はそのプロジェクトを Weave プロジェクト (W&amp;B Models プロジェクトではなく) としても設定し、UI のサイドナビゲーションには Weave のツールのみが表示されます。
  </Tab>
</Tabs>

<div id="view-projects-in-the-ui">
  ## UIでプロジェクトを表示する
</div>

Weave UIでプロジェクトにアクセスするには、次の手順に従います。

1. [wandb.ai](https://wandb.ai) にアクセスします。
2. 左上隅にある **Weights &amp; Biases logo** をクリックして **Global navigation menu** を開き、**Projects** サイドバーからプロジェクトを選択します。これにより、プロジェクトの概要ページが開きます。すでにプロジェクトにトレースをログしている場合は、**Traces** ページが開きます。
3. 概要ページでは、次のことができます。
   * プロジェクトにログされたトレース数や、プロジェクトが使用しているストレージ量などの統計情報を確認します。
   * サイドバーを使用して、トレース、評価、その他のリソースにアクセスします。
   * 最近ログされたトレース、評価、モデル、データセットにアクセスします。

<div id="delete-a-project">
  ## プロジェクトを削除
</div>

W&amp;B アカウントからプロジェクトを削除するには、次の手順に従います。

1. [wandb.ai](https://wandb.ai) にアクセスします。
2. **Projects** サイドバーからプロジェクトを選択します。これにより、プロジェクトの概要ページ、またはすでにそのプロジェクトにトレースをログしている場合は **Traces** ページが開きます。
3. まだ開いていない場合は、サイドナビゲーションの **Project** (<Icon icon="circle-info" size={16} />) ボタンを選択して、プロジェクトの概要ページを開きます。
4. 三点メニュー (<Icon icon="ellipsis" size={16} />) を選択し、リストから **Delete Project** を選びます。これにより、**Delete project** モーダルダイアログが開きます。
5. モーダルで確認フィールドにプロジェクト名を入力し、**Delete** をクリックします。プロジェクトは W&amp;B アカウントから完全に削除されます。
