[プロンプトを作成して公開](/ja/weave/guides/core-types/prompts.mdx)した後は、コード内および本番で特定のバージョンを参照、取得、使用できます。

`weave.publish` でプロンプトを公開すると、W&amp;B Weave は不変のバージョンを作成します。各バージョンは名とバージョンで参照でき、`weave.ref(...).get()` を使ってコード内で取得でき、Weave UI で表示できます。

<div id="prompt-versions">
  # プロンプトのバージョン
</div>

Weave のプロンプトは、**バージョン化されたオブジェクト**として保存されます。これにより、プロンプトを安全に繰り返し改善しながら、評価、Experiments、本番システムが実行時に使用した正確なプロンプト バージョンを参照できるようになります。

プロンプトのバージョンには、Weave object ref を使って액세스します。このページでは、プロンプトのバージョンがどのように作成されるか、コードで特定のバージョンを取得する方法、そして Weave UI でバージョンを確認・比較する方法について説明します。

<div id="how-prompt-versions-are-created">
  ## プロンプトのバージョンが作成される仕組み
</div>

Weave は、プロンプトのすべてのバージョンを自動的にトラッキングし、プロンプトがどのように進化したかの完全な履歴を作成します。このバージョン管理システムはプロンプトエンジニアリングのworkflowにおいて重要であり、安全に実験を行い、どの変更がパフォーマンスを改善または悪化させたかをトラッキングし、必要に応じて以前のバージョンへ簡単にロールバックできます。同じ名で内容が異なるプロンプトを公開するたびに、Weave は過去のすべてのバージョンを保持したまま新しいバージョンを作成します。

各バージョンは不変です。いったん作成されると、そのバージョンの内容は変更できません。プロンプトを更新して再度公開すると、Weave は以前のバージョンを保持したまま新しいバージョンを作成します。

バージョン管理により、次のことが可能になります。

* 過去のexperimentを再現する。
* プロンプトの変更を安全に反復する。
* 管理された方法でプロンプトの更新をロールアウトする。

<div id="construct-a-fully-qualified-ref-uri">
  ## 完全修飾された ref URI を構築する
</div>

Weave object は、完全修飾された ref URI を使用して一意に識別できます。

完全修飾された Weave object ref URI は、次のようになります。

```text
weave:///<your-team-name>/<your-project-name>/object/<object_name>:<object_version>
```

* *your-team-name*: W&amp;B entity (ユーザー名またはチーム名)
* *your-project-name*: W&amp;B プロジェクト
* *object&#95;name*: オブジェクト名
* *object&#95;version*: バージョン ハッシュ、`v0` や `v1` のような文字列、または `:latest` のようなエイリアスのいずれかです。すべてのオブジェクトには `:latest` エイリアスがあります。

たとえば、完全修飾された Weave prompt URI は次のようになります。

```text
weave:///your-team-name/your-project-name/object/support_prompt:v3
```

<div id="retrieve-a-prompt-version-in-code">
  ## コードでプロンプトのバージョンを取得する
</div>

プロンプトを取得するには、その名前とバージョンへの参照を作成し、`.get()` を呼び出して読み込みます。ref は保存されたオブジェクトを指し、`.get()` はそのオブジェクトを取得して、アプリケーションで使用できるようにします。

ref はいくつかの方法で作成できます。

* `weave.ref(<name>)`: プロンプトの `:latest` バージョンを取得します。`weave.init(...)` の呼び出しが必要です。
* `weave.ref(<name>:<version>)`: 指定したバージョンのプロンプトを取得します。`weave.init(...)` の呼び出しが必要です。
* `weave.ref(<fully_qualified_ref_uri>)`: 指定した [完全修飾された ref URI](#construct-a-fully-qualified-ref-uri) にあるプロンプトを取得します。`weave.init(...)` の呼び出しは不要です。

次の例では、`support_prompt:v3` バージョンを読み込んで、アプリケーションで使用できるようにします。

```python lines
import weave

weave.init("your-team-name/your-project-name")

prompt = weave.ref("support_prompt:v3").get()
```

<div id="use-prompt-versions-in-production">
  ## 本番でプロンプトのバージョンを使用する
</div>

本番システムにプロンプトをデプロイする際は、最新バージョンを参照するのではなく、特定のプロンプト バージョンを固定してください。本番アプリケーションでは、`weave.ref(<name>:<version>).get()` を使用してそのバージョンを読み込みます。

バージョンを固定することで、後から新しいプロンプト バージョンが公開されても、本番の動作を安定した状態に保てます。

一般的なワークフローは次のとおりです。

1. 新しいプロンプト バージョンを開発してテストする。
2. データセットまたは評価スイートに対して新しいプロンプトを評価する。
3. 本番システムを更新し、新しいバージョンを参照するようにする。

この方法により、チームは本番の動作を意図せず変更することなく、安全にプロンプトを継続的に改善できます。

## <a id="view-prompts" aria-label="プロンプトのバージョンを表示して比較する" />プロンプトのバージョンを表示して比較する

UI でプロンプトのバージョンを表示するには、次の手順に従ってください。

1. [wandb.ai](https://wandb.ai/) にアクセスし、プロジェクトを選択します。
2. Weave のプロジェクトのサイドバーで **Assets** をクリックします。Assets ページが開きます。
3. Assets ページで **Prompts** をクリックします。プロジェクト内のプロンプトが一覧表示された Prompts ページが開きます。
4. **Versions** 列で、表示したいプロンプトの **(x) Versions** をクリックします。プロンプトのバージョン一覧が開きます。

![Prompt Assets の UI。選択したプロンプトのバージョンが表示されています。](/weave/guides/core-types/imgs/prompt-object.png)

5. (任意) 一覧に表示されたプロンプトの横にあるチェックボックスをオンにし、表のツールバーにある **Compare** ボタンをクリックすると、プロンプトのバージョンを比較できます。これにより、プロンプト間の差分を確認できます。

![2 つの異なるプロンプト バージョン間の差分を表示する Compare prompts UI。](/weave/guides/core-types/imgs/prompt-comparison.png)

<div id="add-prompt-tags-and-aliases">
  ## プロンプトにタグと별칭を追加する
</div>

별칭とタグを使用して、プロンプトのバージョンを整理できます。これらのラベルは、特定のバージョンを識別したり参照したりするのに役立ちます。

* 별칭: 単一のプロンプトバージョンを指す一意の名前です。별칭はいつでも別のバージョンを指すように変更できるため、本番やステージングのような固定参照に便利です。
* タグ: バージョンに付ける説明的なラベルです。1 つのバージョンに複数のタグを付けることができます。reviewed や passed-eval など、バージョンの分類やフィルタリングに使用します。

UI でプロンプトの별칭とタグを設定するには:

1. Weave のプロジェクトのサイドバーで、**Assets** をクリックします。Assets ページが開きます。
2. Assets ページで、**Prompts** をクリックします。表で、割り当てられている `Aliases` と `Tags` を確認できます。
3. Prompts 表で、変更するプロンプトのリンクをクリックします。
4. 選択したプロンプトの詳細パネルでは、タイトルバーにプロンプト名と特定のバージョンが表示されます。更新するプロンプトのバージョンを選択します。
5. パネルのツールバーで、コントロールを使用して、この特定のバージョンに割り当てられている별칭とタグを追加または削除します。

![Prompts ページのプロンプト詳細パネルで選択したプロンプトバージョン。Aliases と Tags のコントロールが表示されています。](/weave/guides/core-types/imgs/prompt-alias-tags.png)

<div id="modify-tags-and-aliases-in-code">
  ### コードでタグと별칭を変更する
</div>

タグと별칭は、次のような作業についてコードでも管理できます。

* タグと별칭をインラインで付けてプロンプトを公開する。
* 별칭を特定のバージョンに向け、それを解決してプロンプトを読み込む。
* バージョンに対してタグと별칭を追加、削除、一覧表示する。

次のコード例では、3 つのバージョンを持つプロンプト `my-prompt` を、さまざまなタグと별칭付きでプロジェクトに追加します。`'your-team-name/your-project-name'` は、プロジェクトに合わせて更新してください。

<CodeGroup>
  ```python Python lines
  import weave

  // TODO: 'your-team-name/your-project-name' を正しい値に設定します。
  client = weave.init('your-team-name/your-project-name')

  # プロンプトの 2 つのバージョンを公開します。
  v0_ref = weave.publish(
      weave.StringPrompt("Answer the user's question: {question}"),
      name="my-prompt",
  )
  v1_ref = weave.publish(
      weave.StringPrompt("Answer the user's question helpfully and concisely: {question}"),
      name="my-prompt",
  )

  # --- 별칭: 特定のバージョンを指す名前付きポインター。 ---

  # バージョンに별칭を設定します。
  client.set_aliases(v0_ref, "staging")
  client.set_aliases(v1_ref, "production")

  # バージョンに設定されている별칭を取得します。
  client.get_aliases(v1_ref)  # ["production", "latest"]

  # 別のバージョンに設定して별칭を移動します。
  client.set_aliases(v0_ref, "production")  # Automatically detaches from v1.

  # 별칭を解決してオブジェクトを読み込みます。
  prompt = weave.ref("my-prompt:production").get()

  # プロジェクト内のすべての별칭を一覧表示します。
  client.list_aliases()

  # 별칭を削除します。
  client.remove_aliases(v0_ref, "production")

  # --- タグ: 特定のバージョンに付けるラベル。 ---

  # バージョンにタグを追加します。
  client.add_tags(v0_ref, ["reviewed", "passed-eval"])
  client.add_tags(v1_ref, ["reviewed", "needs-improvement"])

  # バージョンのタグを取得します。
  client.get_tags(v0_ref)  # ["passed-eval", "reviewed"]

  # プロジェクト内の重複しないすべてのタグを一覧表示します。
  client.list_tags()

  # バージョンからタグを削除します。
  client.remove_tags(v1_ref, ["needs-improvement"])

  # --- 組み合わせ: 1 回の Call で両方を取得します。 ---

  tags, aliases = client.get_tags_and_aliases(v0_ref)

  # --- タグと별칭をインラインで付けて公開します。 ---

  ref = weave.publish(
      weave.StringPrompt("Be brief: {question}"),
      name="my-prompt",
      tags=["reviewed"],
      aliases=["production"],
  )

  ```

  ```plaintext TypeScript lines
  この機能は、TypeScript SDK ではまだ사용 가능하지 않습니다。
  ```
</CodeGroup>

例を実行すると、Weave は Weave ダッシュボード内のオブジェクトへのリンクを表示します。リンクをたどってプロンプトを確認してください。
