---
title: "モデルを使ってアプリケーションのバージョンをトラッキングする"
description: "データとコードを組み合わせた構造化されたモデルを使って、アプリケーションのバージョンをトラッキングします。"
---

`Model` は、設定、トレーニング済みモデルの重み、そのほかの情報を含むデータと、そのモデルの動作を定義するコードを組み合わせたものです。コードをこの API に対応するように構造化することで、アプリケーションを構造化された方法でバージョン管理でき、Experiments をより体系的にトラッキングできるようになります。

<Tabs>
  <Tab title="Python">
    Weave でモデルを作成するには、次のものが必要です。

    * `weave.Model` を継承するクラス
    * すべてのパラメーターに対する型定義
    * `@weave.op()` デコレーター付きの型付き `predict` 関数

    ```python lines
    from weave import Model
    import weave

    class YourModel(Model):
        attribute1: str
        attribute2: int

        @weave.op()
        def predict(self, input_data: str) -> dict:
            # モデルのロジックをここに記述します
            prediction = self.attribute1 + ' ' + input_data
            return {'pred': prediction}
    ```

    通常どおり、次のようにモデルを呼び出せます。

    ```python lines
    import weave
    weave.init('intro-example')

    model = YourModel(attribute1='hello', attribute2=5)
    model.predict('world')
    ```

    これにより、`predict` を呼び出すたびに、入力と出力に加えてモデルの設定もトラッキングされます。

    ## モデルの自動バージョン管理

    モデルを定義するパラメーターやコードを変更すると、その変更がログに記録され、バージョンが更新されます。
    これにより、異なるバージョンのモデル間で予測結果を比較できます。これを使ってプロンプトを改善したり、最新の LLM を試したり、さまざまな設定で予測を比較したりできます。

    たとえば、次のように新しいモデルを作成します。

    ```python lines
    import weave
    weave.init('intro-example')

    model = YourModel(attribute1='howdy', attribute2=10)
    model.predict('world')
    ```

    これを呼び出すと、UI にはこの Model の 2 つのバージョンが表示され、それぞれに異なるトラッキング済みの Call があることがわかります。

    ## モデルをサーブする

    モデルをサーブするには、次の呼び出しで簡単に FastAPI サーバーを立ち上げられます。

    ```bash
    weave serve <your model ref>
    ```

    追加の手順については、[serve](/ja/weave/guides/tools/serve) を参照してください。

    ## 本番環境での呼び出しをトラッキングする

    本番環境での呼び出しを分けるには、UI や API で簡単にフィルタリングできるよう、予測結果に追加の属性を付与できます。

    ```python lines
    with weave.attributes({'env': 'production'}):
        model.predict('world')
    ```
