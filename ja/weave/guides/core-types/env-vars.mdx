---
title: "Weave の環境変数を設定する"
description: "環境変数を使用して、実行時の Weave の動作を設定する"
---

Weave では、動作の設定や最適化に使用できる一連の環境変数が用意されています。これらの変数は、シェルやスクリプト内で設定し、特定の機能を制御できます。

```bash
# シェルで環境変数を設定する例
export WEAVE_PARALLELISM=10  # 並列ワーカー数を制御する
export WEAVE_PRINT_CALL_LINK=false  # call linkの出力を無効にする
```

<div id="available-environment-variables">
  ## 利用可能な環境変数
</div>

| 変数                              | タイプ      | デフォルト        | 説明                                                                                                                                                                                        |
| ------------------------------- | -------- | ------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `WANDB_API_KEY`                 | `string` | `None`       | 設定すると、APIキーの入力を求められずに W&amp;B Weave に自動でログインします。APIキーを発行するには、W&amp;B アカウントにログインし、[User Settings](https://wandb.ai/settings) に移動します。                                                       |
| `WEAVE_DISABLED`                | `bool`   | `false`      | `true` に設定すると、Weave のすべてのトレースを無効にします。Weave op は通常の関数のように動作します。                                                                                                                            |
| `WEAVE_PRINT_CALL_LINK`         | `bool`   | `true`       | Weave op を呼び出すときに Weave UI へのリンクを表示するかどうかを制御します。これは、次のように `weave.init()` の `settings` 引数を設定して、コード内で直接指定することもできます: `weave.init("your-project-name", settings={"print_call_link": False})`  |
| `WEAVE_LOG_LEVEL`               | `str`    | `INFO`       | weave ロガーのログレベルを制御します。                                                                                                                                                                    |
| `WEAVE_CAPTURE_CODE`            | `bool`   | `true`       | 後で再利用できるよう、op のコードを保存するかどうかを制御します。                                                                                                                                                        |
| `WEAVE_DEBUG_HTTP`              | `bool`   | `false`      | `true` に設定すると、デバッグ用に HTTP リクエストとレスポンスのログを有効にします。                                                                                                                                          |
| `WEAVE_PARALLELISM`             | `int`    | `20`         | 評価で、並列に評価する例の数を制御します。例を順番に実行するには `1` に設定します。                                                                                                                                              |
| `WEAVE_TRACE_LANGCHAIN`         | `bool`   | `true`       | LangChain のグローバルなトレースを制御します。LangChain のトレースを明示的に無効にするには `false` に設定します。                                                                                                                   |
| `WEAVE_USE_SERVER_CACHE`        | `bool`   | `true`       | サーバー応答のキャッシュを有効にします。有効にすると、繰り返し行うクエリのパフォーマンスを向上させるため、サーバーからのレスポンスがディスクにキャッシュされます。                                                                                                         |
| `WEAVE_SERVER_CACHE_SIZE_LIMIT` | `int`    | `1000000000` | サーバーキャッシュの最大サイズをバイト単位で設定します。キャッシュがこのサイズに達すると、新しいエントリのための空き容量を確保するために、古いエントリが自動的に削除されます。重要: 基盤となる実装では SQLite を使用しており、Write Ahead Log (WAL) はこの設定に関係なく 4MB まで増加します。この WAL はプログラム終了時に削除されます。 |
| `WEAVE_SERVER_CACHE_DIR`        | `str`    | `None`       | キャッシュファイルの保存先ディレクトリを指定します。設定されていない場合は、一時ディレクトリが使用されます。                                                                                                                                    |
| `WEAVE_MAX_CALLS_QUEUE_SIZE`    | `int`    | `100000`     | calls キューの最大サイズを設定します。デフォルトは 100&#95;000 です。値を 0 に設定すると、キューは無制限に増加できます。                                                                                                                   |
| `WEAVE_RETRY_MAX_ATTEMPTS`      | `int`    | `3`          | 失敗したリクエストに対する最大再試行回数を設定します。                                                                                                                                                               |
| `WEAVE_RETRY_MAX_INTERVAL`      | `float`  | `300.0`      | 再試行の間隔の最大値を秒単位で設定します。                                                                                                                                                                     |
| `WANDB_BASE_URL`                | `string` | `None`       | Weave のホスト URL を設定します。これは、`wandb.login()` でホスト URL の入力を求められた際に入力するのと同じです。`weave.init()` を使用する前に `WANDB_BASE_URL` と `WANDB_API_KEY` を指定すると、Weave へのログインと認証を自動で行えます。                        |

<Note>
  すべてのブール型環境変数では、次の値を使用できます (大文字・小文字は区別されません) :

  * True には `true`, `1`, `yes`, `on`
  * False には `false`, `0`, `no`, `off`
</Note>
