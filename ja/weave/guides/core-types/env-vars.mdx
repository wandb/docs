---
title: "Weave の環境変数を設定する"
description: "環境変数を使用して、実行時の Weave の動作を設定します"
---

Weave には、その動作を設定して最適化するための一連の環境変数が用意されています。これらの変数は、特定の機能を制御するために、シェルまたはスクリプト内で設定できます。

```bash
# シェルで環境変数を設定する例
export WEAVE_PARALLELISM=10  # 並列ワーカー数を制御します
export WEAVE_PRINT_CALL_LINK=false  # Call linkの出力を無効にします
```

```python lines
# Pythonで環境変数を設定する例
import os

os.environ["WEAVE_PARALLELISM"] = "10"
os.environ["WEAVE_PRINT_CALL_LINK"] = "false"
```


<div id="available-environment-variables">
  ## 利用可能な環境変数
</div>

| Variable                        | Type     | Default      | Description                                                                                                                                                                                               |
| ------------------------------- | -------- | ------------ | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `WANDB_API_KEY`                 | `string` | `None`       | 設定すると、APIキーの入力を求められることなく自動的に W&amp;B Weave にログインします。APIキーを発行するには、W&amp;B アカウントにログインし、[User Settings](https://wandb.ai/settings) に移動します。                                                                   |
| `WEAVE_DISABLED`                | `bool`   | `false`      | `true` に設定すると、すべての Weave トレースを無効にします。Weave op は通常の関数のように動作します。                                                                                                                                            |
| `WEAVE_PRINT_CALL_LINK`         | `bool`   | `true`       | Weave op を呼び出す際に Weave UI へのリンクを表示するかどうかを制御します。これはコード内で `weave.init()` の `settings` 引数を次のように設定して直接指定することもできます: `weave.init("your-project-name", settings={"print_call_link": False})`                     |
| `WEAVE_LOG_LEVEL`               | `str`    | `INFO`       | weave logger のログレベルを制御します。                                                                                                                                                                                |
| `WEAVE_CAPTURE_CODE`            | `bool`   | `true`       | 後で再読み込みして使用できるように、op のコードを保存するかどうかを制御します。                                                                                                                                                                 |
| `WEAVE_DEBUG_HTTP`              | `bool`   | `false`      | `true` に設定すると、デバッグ用に HTTP Request と応答のログ記録を有効にします。                                                                                                                                                |
| `WEAVE_PARALLELISM`             | `int`    | `20`         | 評価で、並列に評価する例の数を制御します。例を順番に実行するには `1` に設定します。                                                                                                                                                              |
| `WEAVE_TRACE_LANGCHAIN`         | `bool`   | `true`       | LangChain のグローバルなトレースを制御します。LangChain のトレースを明示的に無効にするには `false` に設定します。                                                                                                                                   |
| `WEAVE_USE_SERVER_CACHE`        | `bool`   | `true`       | サーバー応答のキャッシュを有効にします。有効にすると、繰り返しクエリのパフォーマンスを向上させるために、サーバーからの応答がディスクにキャッシュされます。                                                                                                                             |
| `WEAVE_SERVER_CACHE_SIZE_LIMIT` | `int`    | `1000000000` | サーバーキャッシュの最大サイズ制限をバイト単位で設定します。キャッシュがこのサイズに達すると、新しいエントリ用の空き容量を確保するため、古いエントリが自動的に削除されます。重要: 基盤となる実装では SQLite を使用しており、この SQLite にはこの設定にかかわらず 4MB まで増加する Write Ahead Log (WAL) があります。この WAL はプログラム終了時に削除されます。 |
| `WEAVE_SERVER_CACHE_DIR`        | `str`    | `None`       | キャッシュファイルを保存するディレクトリを指定します。設定しない場合は、一時ディレクトリが使用されます。                                                                                                                                                      |
| `WEAVE_MAX_CALLS_QUEUE_SIZE`    | `int`    | `100000`     | Call キューの最大サイズを設定します。デフォルトは 100&#95;000 です。値を 0 に設定すると、キューは無制限に増加できます。                                                                                                                                    |
| `WEAVE_RETRY_MAX_ATTEMPTS`      | `int`    | `3`          | 失敗した Request に対する再試行の最大回数を設定します。                                                                                                                                                                          |
| `WEAVE_RETRY_MAX_INTERVAL`      | `float`  | `300.0`      | 再試行の間隔の最大値を秒単位で設定します。                                                                                                                                                                                     |
| `WANDB_BASE_URL`                | `string` | `None`       | Weave ホスト URL を設定します。`wandb.login()` でホスト URL の入力を求められた際に入力するのと同等です。`weave.init()` を使用する前に `WANDB_BASE_URL` と `WANDB_API_KEY` を指定すると、自動的に Weave にログインして認証できます。                                            |

<Note>
  すべてのブール型環境変数は次の値を受け付けます (大文字と小文字は区別されません) :

  * True の場合は `true`, `1`, `yes`, `on`
  * False の場合は `false`, `0`, `no`, `off`
</Note>
