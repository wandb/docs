---
title: "OpenRouter"
description: "OpenRouter の統合インターフェースを使用して、多数の LLM を Weave と自動統合して利用する"
---

Openrouter.ai は、多数の LLM 向けの統合インターフェースです。OpenAI GPT-4、Anthropic Claude、Google Gemini などの基盤モデルに加え、Llama-3、Mixtral などのオープンソースモデルや、[そのほか多数のモデル](https://openrouter.ai/models) をサポートしています。一部のモデルは無料で提供されています。

OpenRouter は REST API と OpenAI SDK との互換性を提供しており、Weave はこれを自動的に検出して統合します (詳細については、OpenRouter の [docs](https://docs.together.ai/docs/openai-api-compatibility) および [quick start](https://openrouter.ai/docs/quick-start) を参照してください) 。

OpenAI SDK のコードを OpenRouter に切り替えるには、APIキーを [OpenRouter API](https://openrouter.ai/docs/api-keys) キーに変更し、`base_url` を `https://openrouter.ai/api/v1` に設定し、`model` には [chat models](https://openrouter.ai/docs/models) のいずれかを指定します。`weave.init()` を呼び出すときは、トレース用のプロジェクト名を指定してください。指定しない場合は、デフォルトの entity が使用されます。デフォルトの entity を確認または更新するには、W&amp;B Models のドキュメントにある [User Settings](https://docs.wandb.ai/platform/app/settings-page/user-settings/#default-team) を参照してください。

```python lines {5,10-12}
import os
import openai
import weave

weave.init('openrouter-weave')

system_content = "You are a travel agent. Be descriptive and helpful."
user_content = "Tell me about San Francisco"

client = openai.OpenAI(
    api_key=os.environ.get("OPENROUTER_API_KEY"),
    base_url="https://openrouter.ai/api/v1",
)
chat_completion = client.chat.completions.create(
    extra_headers={
    "HTTP-Referer": $YOUR_SITE_URL, # 省略可能。アプリを openrouter.ai のランキングに掲載するために使用します。
    "X-Title": $YOUR_APP_NAME, # 省略可能。openrouter.ai のランキングに表示されます。
    },
    model="meta-llama/llama-3.1-8b-instruct:free",
    messages=[
        {"role": "system", "content": system_content},
        {"role": "user", "content": user_content},
    ],
    temperature=0.7,
    max_tokens=1024,
)
response = chat_completion.choices[0].message.content
print("Model response:\n", response)
```

これは入門用の簡単な例ですが、より複雑なユースケース向けに Weave を独自の関数とインテグレーションする方法の詳細については、[OpenAI](/ja/weave/guides/integrations/openai#track-your-own-ops) ガイドを参照してください。
