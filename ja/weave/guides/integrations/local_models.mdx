<div id="local-models">
  # ローカルモデル
</div>

多くの開発者は、LLama-3、Mixtral、Gemma、Phi などのオープンソースモデルをダウンロードしてローカルで実行しています。これらのモデルをローカルで実行する方法はいろいろありますが、OpenAI SDK との互換性を備えたものについては、Weave がそのいくつかをデフォルトでサポートしています。

<div id="wrap-local-model-functions-with-weaveop">
  ## ローカルモデル関数を `@weave.op()` でラップする
</div>

`weave.init('<your-project-name>')` を使って Weave を初期化し、その後 LLM の Call を `weave.op()` でラップするだけで、任意の LLM を簡単に Weave と統合できます。詳細については、[トレース](/ja/weave/guides/tracking/tracing) ガイドを参照してください。

<div id="updating-your-openai-sdk-code-to-use-local-models">
  ## OpenAI SDK のコードをローカルモデルを使用するように更新する
</div>

OpenAI SDK との互換性をサポートするすべてのサービスやフレームワークでは、いくつかの小さな変更が必要です。

まず何よりも重要なのは、`openai.OpenAI()` の初期化時に行う `base_url` の変更です。

```python lines
client = openai.OpenAI(
    base_url="http://localhost:1234",
)
```

ローカルモデルの場合、`api_key` には任意の文字列を指定できますが、上書きしておく必要があります。そうしないと、OpenAI が環境変数からそれを使用しようとしてエラーを表示します。

<div id="openai-sdk-supported-local-model-runners">
  ## OpenAI SDK との互換性をサポートするローカルモデルランナー
</div>

以下は、Hugging Face からモデルをダウンロードしてお使いのコンピュータ上で実行でき、OpenAI SDK との互換性をサポートするアプリのリストです。

1. Nomic [GPT4All](https://www.nomic.ai/gpt4all) - Settings の Local Server 経由でサポート ([FAQ](https://docs.gpt4all.io/gpt4all_help/faq.html))
2. [LMStudio](https://lmstudio.ai/) - Local Server による OpenAI SDK サポート [docs](https://lmstudio.ai/docs/developer/core/server)
3. [Ollama](https://ollama.com/) - OpenAI SDK 向けの [OpenAI compatibility](https://ollama.readthedocs.io/en/openai/)
4. llama.cpp via [llama-cpp-python](https://llama-cpp-python.readthedocs.io/en/latest/server/) python パッケージ
5. [llamafile](https://github.com/Mozilla-Ocho/llamafile#other-example-llamafiles) - `http://localhost:8080/v1` は Llamafile run で自動的に OpenAI SDK をサポートします
