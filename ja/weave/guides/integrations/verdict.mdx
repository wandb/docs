Weave は、[Verdict Python library](https://verdict.haizelabs.com/docs/) を通じて行われる모든 Call のトラッキングとログを容易に行えるよう設計されています。

AI 評価パイプラインを扱う際は、デバッグが重要です。パイプラインの step が失敗した場合、出力が想定外である場合、またはネストされた operations によって状況が分かりにくくなる場合でも、問題箇所を特定するのは困難です。Verdict のアプリケーションは、多くの場合、複数のパイプライン step、judge、変換で構成されるため、評価 workflow の内部動作を理解することが不可欠です。

Weave は、[Verdict](https://verdict.readthedocs.io/) アプリケーションのトレースを自動的に取得することで、このプロセスを簡素化します。これにより、パイプラインのパフォーマンスを監視および分析でき、AI 評価 workflow のデバッグと最適化が容易になります。

<div id="getting-started">
  ## はじめに
</div>

開始するには、スクリプトの先頭で `weave.init(project=...)` を呼び出すだけです。`project` 인数 を使用して `team-name/project-name` で特定の W&amp;B チーム名にログするか、`project-name` を指定してデフォルトの チーム/entity にログします。

```python lines {7}
import weave
from verdict import Pipeline
from verdict.common.judge import JudgeUnit
from verdict.schema import Schema

# プロジェクト名を指定して Weave を初期化します
weave.init("verdict_demo")

# シンプルな評価パイプラインを作成します
pipeline = Pipeline()
pipeline = pipeline >> JudgeUnit().prompt("Rate the quality of this text: {source.text}")

# サンプルデータを作成します
data = Schema.of(text="This is a sample text for evaluation.")

# パイプラインを実行します。これは自動的にトレースされます
output = pipeline.run(data)

print(output)
```

<div id="tracking-call-metadata">
  ## Call メタデータのトラッキング
</div>

Verdict パイプラインの Call のメタデータをトラッキングするには、[`weave.attributes`](/ja/weave/reference/python-sdk#function-attributes) コンテキストマネージャーを使用できます。このコンテキストマネージャーを使うと、パイプラインの実行や評価バッチなど、特定のコードブロックに対して独自のメタデータを設定できます。

```python lines {7,14}
import weave
from verdict import Pipeline
from verdict.common.judge import JudgeUnit
from verdict.schema import Schema

# プロジェクト名を指定して Weave を初期化します
weave.init("verdict_demo")

pipeline = Pipeline()
pipeline = pipeline >> JudgeUnit().prompt("Evaluate sentiment: {source.text}")

data = Schema.of(text="I love this product!")

with weave.attributes({"evaluation_type": "sentiment", "batch_id": "batch_001"}):
    output = pipeline.run(data)

print(output)
```

Weave は、Verdict パイプライン Call のトレースに対するメタデータを自動的にトラッキングします。メタデータは Weave の Web インターフェースで確認できます。

<div id="traces">
  ## トレース
</div>

AI 評価パイプラインのトレースを一元的なデータベースに保存することは、開発時にも本番環境でも重要です。これらのトレースは、貴重なデータセットを提供することで、評価ワークフローのデバッグと改善に不可欠です。

Weave は、Verdict アプリケーションのトレースを自動的に取得します。Verdict ライブラリを通じて行われるすべての Call をトラッキングしてログし、これには次のものが含まれます。

* パイプライン実行の step
* Judge ユニットの評価
* レイヤー変換
* プーリング operation
* custom ユニットと変換

Weave の Web インターフェースでトレースを表示でき、パイプライン実行の階層構造を確認できます。

<div id="pipeline-tracing-example">
  ## パイプラインのトレース例
</div>

ネストされたパイプラインのオペレーションを Weave がどのようにトレースするかを示す、より複雑な例を次に示します:

```python lines {8}
import weave
from verdict import Pipeline, Layer
from verdict.common.judge import JudgeUnit
from verdict.transform import MeanPoolUnit
from verdict.schema import Schema

# プロジェクト名を指定して Weave を初期化します
weave.init("verdict_demo")

# 複数の step を含む複雑なパイプラインを作成します
pipeline = Pipeline()
pipeline = pipeline >> Layer([
    JudgeUnit().prompt("Rate coherence: {source.text}"),
    JudgeUnit().prompt("Rate relevance: {source.text}"),
    JudgeUnit().prompt("Rate accuracy: {source.text}")
], 3)
pipeline = pipeline >> MeanPoolUnit()

# サンプルデータ
data = Schema.of(text="This is a comprehensive evaluation of text quality across multiple dimensions.")

# パイプラインを実行します - すべての operation がトレースされます
result = pipeline.run(data)

print(f"Average score: {result}")
```

これにより、次を示す詳細な trace が作成されます:

* メインの Pipeline 実行
* Layer 内の各 JudgeUnit 評価
* MeanPoolUnit の集約 step
* 各操作のタイミング情報

<div id="configuration">
  ## 設定
</div>

`weave.init()` を呼び出すと、Verdict パイプラインのトレースは自動的に有効になります。このインテグレーションは、`Pipeline.__init__` method にパッチを適用して、すべての Trace Data を Weave に転送する `VerdictTracer` を挿入することで機能します。

追加の設定は不要です。Weave は自動的に次を行います。

* すべてのパイプライン操作を取得する
* 実行時間をトラッキングする
* inputs と outputs をログする
* トレース階層を維持する
* 同時実行されるパイプライン実行を処理する

<div id="custom-tracers-and-weave">
  ## カスタムトレーサーとWeave
</div>

アプリケーションでカスタムのVerdictトレーサーを使用している場合、Weaveの`VerdictTracer`はそれらと併用できます。

```python lines {8}
import weave
from verdict import Pipeline
from verdict.common.judge import JudgeUnit
from verdict.util.tracing import ConsoleTracer
from verdict.schema import Schema

# プロジェクト名を指定して Weave を初期化します
weave.init("verdict_demo")

# Verdict の組み込みトレーサーも引き続き使用できます
console_tracer = ConsoleTracer()

# Weave（自動）とコンソールトレースの両方を使用するパイプラインを作成します
pipeline = Pipeline(tracer=[console_tracer])  # Weave トレーサーは自動的に追加されます
pipeline = pipeline >> JudgeUnit().prompt("Evaluate: {source.text}")

data = Schema.of(text="Sample evaluation text")

# これにより、Weave とコンソールの両方にトレースが送信されます
result = pipeline.run(data)
```

<div id="models-and-evaluations">
  ## Models と評価
</div>

複数のパイプライン コンポーネントで構成される AI システムを整理して評価するのは難しい場合があります。[`weave.Model`](/ja/weave/guides/core-types/models) を使用すると、プロンプト、パイプラインの設定、評価パラメーターなどの実験の詳細を取得して整理できるため、異なるイテレーションを比較しやすくなります。

以下の例では、Verdict パイプラインを `WeaveModel` でラップする方法を示します。

```python lines {8,10,14}
import asyncio
import weave
from verdict import Pipeline
from verdict.common.judge import JudgeUnit
from verdict.schema import Schema

# プロジェクト名を指定して Weave を初期化します
weave.init("verdict_demo")

class TextQualityEvaluator(weave.Model):
    judge_prompt: str
    pipeline_name: str

    @weave.op()
    async def predict(self, text: str) -> dict:
        pipeline = Pipeline(name=self.pipeline_name)
        pipeline = pipeline >> JudgeUnit().prompt(self.judge_prompt)
        
        data = Schema.of(text=text)
        result = pipeline.run(data)
        
        return {
            "text": text,
            "quality_score": result.score if hasattr(result, 'score') else result,
            "evaluation_prompt": self.judge_prompt
        }

model = TextQualityEvaluator(
    judge_prompt="Rate the quality of this text on a scale of 1-10: {source.text}",
    pipeline_name="text_quality_evaluator"
)

text = "This is a well-written and informative piece of content that provides clear value to readers."

prediction = asyncio.run(model.predict(text))

# Jupyter Notebook を使用している場合は、以下を実行します:
# prediction = await model.predict(text)

print(prediction)
```

このコードは、パイプライン構造と評価結果の両方を表示する、Weave UI で可視化できるモデルを作成します。

<div id="evaluations">
  ### 評価
</div>

評価は、評価パイプライン自体のパフォーマンスを測定するのに役立ちます。[`weave.Evaluation`](/ja/weave/guides/core-types/evaluations) クラスを使用すると、特定のタスクやデータセットに対して Verdict パイプラインがどの程度適切に機能するかを取得できます。

```python lines {8}
import asyncio
import weave
from verdict import Pipeline
from verdict.common.judge import JudgeUnit
from verdict.schema import Schema

# Weave を初期化
weave.init("verdict_demo")

# 評価用モデルを作成
class SentimentEvaluator(weave.Model):
    @weave.op()
    async def predict(self, text: str) -> dict:
        pipeline = Pipeline()
        pipeline = pipeline >> JudgeUnit().prompt(
            "Classify sentiment as positive, negative, or neutral: {source.text}"
        )
        
        data = Schema.of(text=text)
        result = pipeline.run(data)
        
        return {"sentiment": result}

# テストデータ
texts = [
    "I love this product, it's amazing!",
    "This is terrible, worst purchase ever.",
    "The weather is okay today."
]
labels = ["positive", "negative", "neutral"]

examples = [
    {"id": str(i), "text": texts[i], "target": labels[i]}
    for i in range(len(texts))
]

# スコアリング関数
@weave.op()
def sentiment_accuracy(target: str, output: dict) -> dict:
    predicted = output.get("sentiment", "").lower()
    return {"correct": target.lower() in predicted}

model = SentimentEvaluator()

evaluation = weave.Evaluation(
    dataset=examples,
    scorers=[sentiment_accuracy],
)

scores = asyncio.run(evaluation.evaluate(model))
# Jupyter Notebook を使用している場合は、次を実行:
# scores = await evaluation.evaluate(model)

print(scores)
```

これにより、異なるテストケースにわたってVerdictパイプラインがどのように機能するかを示す評価トレースが作成されます。

<div id="performance-monitoring">
  ### パフォーマンス監視
</div>

Weave はすべてのパイプライン処理のタイミング情報を自動的に取得します。これを使用して、パフォーマンスのボトルネックを特定できます:

```python lines {6}
import weave
from verdict import Pipeline, Layer
from verdict.common.judge import JudgeUnit
from verdict.schema import Schema

weave.init("verdict_demo")

# パフォーマンスにばらつきが出る可能性があるパイプラインを作成します
pipeline = Pipeline()
pipeline = pipeline >> Layer([
    JudgeUnit().prompt("Quick evaluation: {source.text}"),
    JudgeUnit().prompt("Detailed analysis: {source.text}"),  # This might be slower
], 2)

data = Schema.of(text="Sample text for performance testing")

# タイミングの傾向を確認するために複数回実行します
for i in range(3):
    with weave.attributes({"run_number": i}):
        result = pipeline.run(data)
```

<div id="error-handling">
  ### エラー処理
</div>

Weave は、パイプラインの実行中に発生する例外を自動的に取得します。

```python lines {6}
import weave
from verdict import Pipeline
from verdict.common.judge import JudgeUnit
from verdict.schema import Schema

weave.init("verdict_demo")

pipeline = Pipeline()
pipeline = pipeline >> JudgeUnit().prompt("Process: {source.invalid_field}")  # ここでエラーが発生します

data = Schema.of(text="Sample text")

try:
    result = pipeline.run(data)
except Exception as e:
    print(f"Pipeline failed: {e}")
    # エラーの詳細は Weave trace に記録されます
```

Weave を Verdict と統合することで、AI 評価パイプラインを包括的に可観測化でき、評価ワークフローのデバッグ、最適化、理解が容易になります。
