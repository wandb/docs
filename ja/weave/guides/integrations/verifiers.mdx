[Verifiers](https://github.com/willccbb/verifiers) は、強化学習 (RL) 環境の構築と LLM エージェントのトレーニングに使用できるモジュール式コンポーネントのライブラリです。Verifiers で構築した環境は、LLM の評価、合成データパイプライン、OpenAI 互換の任意のエンドポイントに対応したエージェントハーネス、および RL トレーニングとして活用できます。

トレーニングのメトリクスを記録するために W&amp;B を使用するだけでなく、Weave を Verifiers の RL ワークフローと統合することで、トレーニング中のモデルのパフォーマンスを可視化できます。Weave は各 step の入力、出力、タイムスタンプを記録するため、データが各ターンでどのように変換されるかを確認したり、複雑な複数ラウンドの会話をデバッグしたり、トレーニング結果を最適化したりすることができます。

Weave と Verifiers を組み合わせて評価を実施することも可能です。

このガイドでは、Verifiers、W&amp;B、および Weave のインストール方法を説明し、Verifiers を Weave および W&amp;B と組み合わせて使用する 2 つの例を紹介します。

![verifiers wandb run page](/images/weave/verifiers.gif)

<div id="getting-started">
  ## はじめに
</div>

Verifiers を Weave で統合するには、まず `uv` を使用して Verifiers ライブラリをインストールします ([ライブラリの作者が推奨](https://github.com/willccbb/verifiers?tab=readme-ov-file#setup)) 。以下のいずれかのコマンドを使用してライブラリをインストールしてください：

```bash
# ローカル開発およびAPIベースのモデル向けにコアライブラリをインストールします
uv add verifiers

# PyTorchおよびGPUサポートを含む、すべてのオプション依存関係を備えたライブラリのフルバージョンをインストールします
uv add 'verifiers[all]' && uv pip install flash-attn --no-build-isolation

# 未リリースの最新機能や修正を含む、ライブラリの最新バージョンをGitHubから直接インストールします
uv add verifiers @ git+https://github.com/willccbb/verifiers.git
```

次に Weave と W&amp;B をインストールします：

```bash
uv pip install weave wandb
```

Weave はデフォルトでライブラリの[暗黙的なパッチ適用](/ja/weave/guides/integrations#automatic-implicit-patching)を有効にします。これにより、明示的な Call パッチ関数を必要とせずに、Verifiers で Weave を使用できます。

<div id="trace-rollouts-and-evaluate">
  ### ロールアウトのトレースと評価
</div>

必要なライブラリをインストールしたら、Weave と Verifiers を組み合わせて使用し、Call をトレースして評価を実行できます。

以下のサンプルスクリプトは、Verifiers を使用して評価を実行し、結果を Weave にログする方法を示しています。このスクリプトは、[GSM8K データセット](https://huggingface.co/datasets/openai/gsm8k)を使用して数学の問題を解く LLM の能力をテストします。GPT-4 に 2 つの数学の問題を解かせ、各レスポンスから数値を抽出し、評価フレームワークとして Verifiers を使用して解答を採点します。

サンプルを実行して、Weave で結果を確認してください。

```python lines
import os
from openai import OpenAI
import verifiers as vf
import weave

os.environ["OPENAI_API_KEY"] = "<YOUR-OPENAI-API-KEY>"

# Weave を初期化する
weave.init("verifiers_demo")

# 最小限のシングルターン環境
dataset = vf.load_example_dataset("gsm8k", split="train").select(range(2))
parser = vf.ThinkParser()

def correct_answer(parser, completion, answer):
    parsed = parser.parse_answer(completion) or ""
    return 1.0 if parsed.strip() == answer.strip() else 0.0

rubric = vf.Rubric(funcs=[correct_answer, parser.get_format_reward_func()], weights=[1.0, 0.2])

env = vf.SingleTurnEnv(
    dataset=dataset,
    system_prompt="Think step-by-step, then answer.",
    parser=parser,
    rubric=rubric,
)

client = OpenAI()
results = env.evaluate(
    client, "gpt-4.1-mini", num_examples=2, rollouts_per_example=2, max_concurrent=8
)
```

<div id="fine-tune-a-model-with-experiment-tracking-and-tracing">
  ### 実験管理とトレースを使用したモデルのファインチューニング
</div>

Weave は、トレーニング中のモデルのパフォーマンスに関する洞察を提供することで、RL ファインチューニングワークフローにおいて強力なツールとなります。W&amp;B と組み合わせて使用することで、包括的な可観測性が得られます。W&amp;B はトレーニングのメトリクスとパフォーマンスチャートをトラッキングし、Weave はトレーニングプロセス中の各インタラクションの詳細なトレースを取得します。

`verifiers` リポジトリには、すぐに実行できる[サンプル](https://github.com/PrimeIntellect-ai/verifiers)が含まれており、すぐに始めることができます。

以下のサンプル RL トレーニングパイプラインは、ローカル推論サーバーを起動し、GSM8K データセットを使用してモデルをトレーニングします。モデルは数学の問題に対する回答を返し、トレーニングループが出力をスコアリングしてモデルを更新します。W&amp;B は損失、報酬、精度などのトレーニングメトリクスをログし、Weave は入力、出力、推論、およびスコアリングを取得します。

このパイプラインを使用するには:

1. ソースからフレームワークをインストールします。以下のコマンドは、GitHub から Verifiers ライブラリと必要な依存関係をインストールします。

```bash
git clone https://github.com/willccbb/verifiers
cd verifiers
uv sync --all-extras && uv pip install flash-attn --no-build-isolation
```

2. すぐに使える環境をインストールします。次のコマンドは、事前設定済みのGSM8Kトレーニング環境をインストールします：

```bash
vf-install gsm8k --from-repo
```

3. モデルをトレーニングします。以下のコマンドはそれぞれ推論サーバーとトレーニングループを起動します。このワークフロー例ではデフォルトで `report_to=wandb` が設定されているため、`wandb.init` を個別に呼び出す必要はありません。W&amp;B にメトリクスをログするために、このマシンの認証を求めるプロンプトが表示されます。

```bash
# 推論サーバーを起動する
CUDA_VISIBLE_DEVICES=0 vf-vllm --model willcb/Qwen3-0.6B --enforce-eager --disable-log-requests

# トレーニングループを起動する
CUDA_VISIBLE_DEVICES=1 accelerate launch --num-processes 1 --config-file configs/zero3.yaml examples/grpo/train_gsm8k.py
```

<Note>
  この例は2xH100で正常にテストされており、安定性を高めるために以下の環境変数を設定しています。

  ```bash
  # 起動前に両方のシェル（サーバーとトレーナー）で設定
  export NCCL_CUMEM_ENABLE=0
  export NCCL_CUMEM_HOST_ENABLE=0
  ```

  これらの変数は、デバイスメモリ割り当てにおけるCUDA Unified Memory (CuMem) を無効にします。
</Note>

トレーニングが開始されると、run中にログされたトレースをUIで[確認できます](/ja/weave/guides/tools/weave-in-workspaces)。

トレースは`Environment.a_generate`および`Rubric.score_rollouts` methodの`logprobs`を省略します。これにより、ペイロードを小さく保ちながら、トレーニング用の元のデータはそのまま維持されます。
