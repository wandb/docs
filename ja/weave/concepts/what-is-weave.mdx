W&amp;B Weave は、信頼性の高い LLM アプリケーションを構築するための、オブザーバビリティと評価のプラットフォームです。Weave は、AI アプリケーションが何をしているのかを把握し、その性能を測定し、継続的に体系的な改善を行うのに役立ちます。

LLM アプリケーションの構築は、従来のソフトウェア開発とは本質的に異なります。LLM の出力は非決定的であるため、デバッグが困難になります。品質は主観的かつ文脈依存です。わずかなプロンプトの変更が、予期しない動作の変化を引き起こすことがあります。従来のテスト手法では対応しきれません。

<div id="the-main-threads-of-weave">
  ## Weave の主な柱
</div>

Weave は次のコア機能を提供します。

* アプリケーション内のあらゆる LLM 呼び出しと、その入力・出力の **可視化**。
* 厳選されたテストケースに対するパフォーマンスを測定するための **体系的な評価**。
* 何が変更されたかを把握できるようにする、プロンプト、モデル、データの **バージョン管理**。
* さまざまなプロンプトやモデルを比較するための **実験**。
* 人間による判断やアノテーションを取得する **フィードバック収集**。
* LLM の安全性と品質を確保するため、ガードレールや Scorer を用いた本番環境での **モニタリング**。

<div id="traces">
  ### トレース
</div>

LLM アプリケーション内でのデータフローをエンドツーエンドで追跡します。

* アプリケーションの各実行における入力と出力を確認できます。
* LLM の応答を生成するために使用されたソースドキュメントを確認できます。
* LLM Call のコスト、トークン数、レイテンシを確認できます。
* 特定のプロンプトと、その回答がどのように生成されたかを詳細に調査できます。
* ユーザーからの応答に対するフィードバックを収集できます。
* コード内では、関数が何をしているかを追跡するために Weave の [ops and calls](/ja/weave/guides/tracking/tracing) を使用できます。

[トレースを始める](/ja/weave/quickstart)

<div id="evaluations">
  ### 評価
</div>

LLM アプリケーションのパフォーマンスを体系的にベンチマークし、本番環境へのデプロイ時にも自信を持てるようにします。

* モデルやプロンプトのどのバージョンが、どの程度のパフォーマンスにつながったかを簡単に追跡できます。
* 1 つ以上のスコアリング関数を使用して応答を評価するためのメトリクスを定義できます。
* 複数のメトリクスにわたって 2 つ以上の異なる評価を比較できます。特定のサンプルを対比して、パフォーマンスの違いを確認できます。

[評価パイプラインを構築する](/ja/weave/tutorial-eval)

<div id="version-everything">
  ### すべてをバージョン管理する
</div>

Weave は、プロンプト、データセット、モデルの設定のバージョンを追跡します。問題が起きたときには、何が変わったのかを正確に確認できます。うまくいったときには、その状態を再現できます。

[バージョン管理について学ぶ](/ja/weave/guides/tracking/objects)

<div id="experiment-with-prompts-and-models">
  ### プロンプトとモデルを試す
</div>

APIキーを用意して Playground で、さまざまな商用モデルのプロンプトをすばやくテストし、応答を比較しましょう。

[Weave Playground で試す](/ja/weave/guides/tools/playground)

<div id="collect-feedback">
  ### フィードバックの収集
</div>

本番環境での利用から、人によるフィードバック、アノテーション、修正を取得します。このデータを活用して、より良いテストケースを作成し、アプリケーションを改善します。

[フィードバックを収集](/ja/weave/guides/tracking/feedback)

<div id="monitor-production">
  ### 本番環境を監視する
</div>

評価で使用しているものと同じScorerで本番トラフィックをスコアリングします。ガードレールを設定して、問題がユーザーに届く前に検知します。

[ガードレールとモニターを設定する](/ja/weave/guides/evaluation/monitors)

<div id="get-started-using-weave">
  ## Weaveの使用を開始する
</div>

WeaveはPythonとTypeScript向けのSDKを提供しています。どちらのSDKもトレース、評価、データセット、およびWeaveのコア機能をサポートしています。クラスベースのModelsやScorersなど一部の高度な機能は、現在Weave TypeScript SDKでは利用できません。

Weaveの使用を開始するには：

1. [https://wandb.ai/site](https://wandb.ai/site/?utm_source=course\&utm_medium=course\&utm_campaign=weave)でWeights &amp; Biasesアカウントを作成し、[https://wandb.ai/authorize](https://wandb.ai/authorize?utm_source=course\&utm_medium=course\&utm_campaign=weave)からAPIキーを取得してください。
2. Weaveをインストールします：

<CodeGroup>
  ```Python Python
  pip install weave
  ```

  ```Typescript Typescript
  npm install weave
  ```
</CodeGroup>

3. スクリプト内でWeaveをインポートし、プロジェクトを初期化します：

<CodeGroup>
  ```Python Python
  import weave
  client = weave.init('your-team/your-project-name')
  ```

  ```TypeScript Typescript
  import * as weave from 'weave';
  const client = await weave.init('your-team/your-project-name');
  ```
</CodeGroup>

これでWeaveを使用する準備が整いました。
Weaveは主要なLLMプロバイダーやフレームワークと統合されています。[サポートされるインテグレーション](/ja/weave/guides/integrations/)を使用すると、Weaveは追加のコード変更なしにLLM Callを自動的にトレースします。

4. サポートされるインテグレーションを利用するだけでなく、Weaveを使用してカスタム関数のトレースをログすることもできます。呼び出す関数に1行追加するだけです。

関数を`@weave.op()` (Pythonの場合) でデコレートするか、`weave.op()` (TypeScriptの場合) でラップすると、Weaveはそのコード、入力、出力、および実行メタデータを自動的に取得します。

<CodeGroup>
  ```python Python
      @weave.op
      async def my_function(){
        ...  }
  ```

  ```typescript Typescript
  function myFunction() {
      ...
  }

  const myFunctionOp = weave.op(myFunction)
  ```
</CodeGroup>

ガイド付きチュートリアルで試してみるには、[トレースを始める](/ja/weave/quickstart)を参照してください。
