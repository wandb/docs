### <kbd>method</kbd> `add_cost`

```python
add_cost(
    llm_id: 'str',
    prompt_token_cost: 'float',
    completion_token_cost: 'float',
    effective_date: 'datetime | None' = None,
    prompt_token_cost_unit: 'str | None' = 'USD',
    completion_token_cost_unit: 'str | None' = 'USD',
    provider_id: 'str | None' = 'default'
) → CostCreateRes
```

現在のプロジェクトにコストを追加します。

**例:**

```python
client.add_cost(llm_id="my_expensive_custom_model", prompt_token_cost=1, completion_token_cost=2)
client.add_cost(llm_id="my_expensive_custom_model", prompt_token_cost=500, completion_token_cost=1000, effective_date=datetime(1998, 10, 3))
```

**Args:**

* <b>`llm_id`</b>: LLM の ID。例 &quot;gpt-4o-mini-2024-07-18&quot;
* <b>`prompt_token_cost`</b>: プロンプトトークン 1 個あたりのコスト。例 .0005
* <b>`completion_token_cost`</b>: 補完トークン 1 個あたりのコスト。例 .0015
* <b>`effective_date`</b>: 省略時は現在の日付。datetime.datetime オブジェクト。
* <b>`provider_id`</b>: LLM のプロバイダー。デフォルトは &quot;default&quot;。例 &quot;openai&quot;
* <b>`prompt_token_cost_unit`</b>: プロンプトトークンのコストの単位。デフォルトは &quot;USD&quot;。（現在は未使用。将来的にコストの通貨種別を指定するために使用予定。例 &quot;tokens&quot; や &quot;time&quot;）
* <b>`completion_token_cost_unit`</b>: 補完トークンのコストの単位。デフォルトは &quot;USD&quot;。（現在は未使用。将来的にコストの通貨種別を指定するために使用予定。例 &quot;tokens&quot; や &quot;time&quot;）
  **Returns:**
  CostCreateRes オブジェクト。`ids` と呼ばれるタプルのリストからなるフィールドを 1 つ持つ。各タプルには `llm_id` と、作成されたコストオブジェクトの id が含まれる。

### <kbd>method</kbd> `get_call`

```python
get_call(
    call_id: 'str',
    include_costs: 'bool' = False,
    include_feedback: 'bool' = False,
    columns: 'list[str] | None' = None
) → WeaveObject
```

ID で 1 件の call を取得します。

* <b>`use_progress_bar`</b>: flush 時にプログレスバーを表示するかどうか。プログレスバーがうまく描画されない環境（例: CI 環境）では False に設定します。

### <kbd>method</kbd> `get_calls`

```python
get_calls(
    filter: 'CallsFilterLike | None' = None,
    limit: 'int | None' = None,
    offset: 'int | None' = None,
    sort_by: 'list[SortByLike] | None' = None,
    query: 'QueryLike | None' = None,
    include_costs: 'bool' = False,
    include_feedback: 'bool' = False,
    include_storage_size: 'bool' = False,
    include_total_storage_size: 'bool' = False,
    columns: 'list[str] | None' = None,
    expand_columns: 'list[str] | None' = None,
    return_expanded_column_values: 'bool' = True,
    scored_by: 'str | list[str] | None' = None,
    page_size: 'int' = 1000
) → CallsIter
```

このプロジェクトのトレースされた呼び出し（operation）のリストを取得します。

このメソッドは、トレースデータをクエリするための強力かつ柔軟なインターフェースを提供します。ページネーション、フィルタリング、ソート、フィールドの射影、スコアリング用メタデータをサポートし、カスタムのトレース UI や分析ツールの基盤として使用できます。

Performance Tip: 結果サイズを削減するために `columns` を指定し、`filter` または `query` を使用してください。

**Args:**

* <b>`filter`</b>:  `op_name`、`parent_ids` などのフィールドで結果を絞り込むための高レベルなフィルター。
* <b>`limit`</b>:  返す呼び出しの最大数。
* <b>`offset`</b>:  結果を返す前にスキップする呼び出し数（ページネーションに使用）。
* <b>`sort_by`</b>:  結果をソートするフィールドのリスト（例: `started_at desc`）。
* <b>`query`</b>:  高度なフィルタリングのための Mongo 風の式。すべての Mongo 演算子がサポートされているわけではありません。
* <b>`include_costs`</b>:  True の場合、`summary.weave` にトークン／コスト情報を含めます。
* <b>`include_feedback`</b>:  True の場合、`summary.weave.feedback` にフィードバックを含めます。
* <b>`include_storage_size`</b>:  True の場合、呼び出しごとのストレージサイズを含めます。
* <b>`include_total_storage_size`</b>:  True の場合、トレース全体のストレージサイズの合計を含めます。
* <b>`columns`</b>:  呼び出しごとに返すフィールドのリスト。これを減らすことでパフォーマンスが大幅に向上する可能性があります（`id`、`trace_id`、`op_name`、`started_at` など一部のフィールドは常に含まれます）。
* <b>`scored_by`</b>:  1 つ以上の scorer（名前または ref URI）でフィルタリングします。複数の scorer は AND で結合されます。
* <b>`page_size`</b>:  1 ページあたりに取得する呼び出し数。大規模なクエリではパフォーマンス向上のために調整してください。

**Returns:**

* <b>`CallsIter`</b>:  `Call` オブジェクトに対するイテレーター。スライス、反復処理、および `.to_pandas()` をサポートします。

**Example:**

```python
calls = client.get_calls(
     filter=CallsFilter(op_names=["my_op"]),
     columns=["inputs", "output", "summary"],
     limit=100,
)
for call in calls:
     print(call.inputs, call.output)
```

### <kbd>method</kbd> `get_evaluation`

```python
get_evaluation(uri: 'str') → Evaluation
```

URI で指定された特定の Evaluation オブジェクトを取得します。

Evaluation の URI は一般的に次の形式に従います: `weave:///entity/project/object/Evaluation:version`

&quot;フレンドリー&quot; な名前で Evaluation を取得することもできます: get&#95;evaluation(&quot;Evaluation:v1&quot;)

**引数:**

* <b>`uri`</b> (str):  取得する Evaluation の一意なリソース識別子。

**戻り値:**

* <b>`Evaluation`</b>:  指定された URI に対応する Evaluation オブジェクト。

**例外:**

* <b>`TypeError`</b>:  URI で参照されるオブジェクトが Evaluation インスタンスでない場合。
* <b>`ValueError`</b>:  URI が無効であるか、オブジェクトが見つからない場合。

**使用例:**

```python
client = weave.init("my-project")
evaluation = client.get_evaluation("weave:///entity/project/object/my-eval:v1")
print(evaluation.name)
```

### <kbd>method</kbd> `get_feedback`

```python
get_feedback(
    query: 'Query | str | None' = None,
    reaction: 'str | None' = None,
    offset: 'int' = 0,
    limit: 'int' = 100
) → FeedbackQuery
```

フィードバック用のプロジェクトを検索します。

**例:**

```python
# 特定のフィードバックオブジェクトを取得します。
# これはコレクションを返すことに注意してください。コレクションには
# 0 個または 1 個のアイテムが含まれることが想定されています。
client.get_feedback("1B4082A3-4EDA-4BEB-BFEB-2D16ED59AA07")

# 特定のリアクションを持つすべてのフィードバックオブジェクトを検索します。
client.get_feedback(reaction="👍", limit=10)

# mongo スタイルのクエリを使用して、特定のフィードバックタイプを持つ
# すべてのフィードバックオブジェクトを検索します。
from weave.trace_server.interface.query import Query

query = Query(
    **{
        "$expr": {
            "$eq": [
                {"$getField": "feedback_type"},
                {"$literal": "wandb.reaction.1"},
            ],
        }
    }
)
client.get_feedback(query=query)
```

**引数:**

* <b>`query`</b>: Mongo 形式のクエリ式。便宜上、feedback の UUID 文字列も指定できます。
* <b>`reaction`</b>: 便宜上、特定のリアクション絵文字でフィルターします。
* <b>`offset`</b>: feedback オブジェクトの取得を開始する位置。
* <b>`limit`</b>: 取得する feedback オブジェクトの最大数。
  **戻り値:**
  FeedbackQuery オブジェクト。

### <kbd>method</kbd> `set_wandb_run_context`

```python
set_wandb_run_context(run_id: 'str', step: 'int | None' = None) → None
```

このクライアントが作成する呼び出しに対して、wandb の run&#95;id と step を上書きします。

これにより、グローバルな wandb.run シンボルに結び付いていない特定の W&amp;B の run に Weave 呼び出しを関連付けることができます。

**引数:**

* <b>`run_id`</b>: run ID（entity/project プレフィックスを含まない）。クライアントは自動的に entity/project プレフィックスを追加します。
* <b>`step`</b>: 呼び出しで使用する step 番号。None の場合、step は設定されません。
  **例:**

```python
client = weave.init("my-project")
client.set_wandb_run_context(run_id="my-run-id", step=5)
# これ以降のすべての呼び出しは、ステップ 5 の entity/project/my-run-id に関連付けられます

# ステップを指定しない場合
client.set_wandb_run_context(run_id="my-run-id")
# 呼び出しはステップなしで entity/project/my-run-id に関連付けられます
```
