[weave](../) / EvaluationLogger

EvaluationLogger は、予測結果とスコアをインクリメンタルにログ記録できるようにします。

事前にデータセット全体を用意してバッチ処理を行う必要がある従来の Evaluation クラスとは異なり、
EvaluationLogger を使うと、予測が行われたタイミングで逐次的にログを記録でき、柔軟なスコアリングが可能です。

`Example`

```ts
const ev = new EvaluationLogger({name: 'my-eval', dataset: 'my-dataset'});

for (const example of streamingData) {
  const output = await myModel.predict(example);
  const pred = ev.logPrediction(example, output);

  if (shouldScore(output)) {
    pred.logScore("accuracy", calculateAccuracy(output));
  }
  pred.finish();
}

await ev.logSummary();
```

<div id="logprediction">
  ### logPrediction
</div>

▸ **logPrediction**(`inputs`, `output`): [`ScoreLogger`](./scorelogger)

入力および出力とともに予測をログに記録します（同期バージョン）。
predict_and_score 呼び出し（子の predict 呼び出しを伴う）を作成します。
スコアを追加するための ScoreLogger を即座に返します。

このメソッドは ScoreLogger を同期的に返します。ScoreLogger 上での操作（logScore, finish）はキューに入れられ、初期化が完了したタイミングで順次実行されます。

`使用例`

```ts
// ファイア・アンド・フォーゲット方式
const scoreLogger = evalLogger.logPrediction({input: 'test'}, 'output');
scoreLogger.logScore('accuracy', 0.95);
scoreLogger.finish();
await evalLogger.logSummary(); // すべての完了を待機
```

<div id="logpredictionasync">
  ### logPredictionAsync
</div>

▸ **logPredictionAsync**(`inputs`, `output`): `Promise`\<[`ScoreLogger`](./scorelogger)\>

入力と出力を指定して予測結果をログに記録する非同期メソッドです。
logPrediction() と同様ですが、予測呼び出しの初期化が完了した時点で resolve される Promise を返します。

処理を進める前に初期化完了を await する必要がある場合に使用してください。

<div id="logsummary">
  ### logSummary
</div>

▸ **logSummary**(`summary?`): `Promise`\<`void`\>

サマリーをログに記録し、evaluation を完了します。
summarize 呼び出しを作成し、evaluate 呼び出しを終了します。

このメソッドは await なしで（いわゆる fire-and-forget で）呼び出すこともできますが、内部的には
保留中のすべての操作が完了するまで待機します。
