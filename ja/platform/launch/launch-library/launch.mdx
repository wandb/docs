W&amp;B Launch の実験を開始します。

```python
launch(
    api: Api,
    job: Optional[str] = None,
    entry_point: Optional[List[str]] = None,
    version: Optional[str] = None,
    name: Optional[str] = None,
    resource: Optional[str] = None,
    resource_args: Optional[Dict[str, Any]] = None,
    project: Optional[str] = None,
    entity: Optional[str] = None,
    docker_image: Optional[str] = None,
    config: Optional[Dict[str, Any]] = None,
    synchronous: Optional[bool] = (True),
    run_id: Optional[str] = None,
    repository: Optional[str] = None
) -> AbstractRun
```

| Arguments       | Description                                                                                                                                                                                                                                                     |
| :-------------- | :-------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `job`           | wandb.Job への文字列参照。例: wandb/test/my-job:latest                                                                                                                                                                                                                   |
| `api`           | wandb.apis.internal からの wandb API インスタンス。                                                                                                                                                                                                                       |
| `entry_point`   | プロジェクト内で実行するエントリポイント。wandb URI の場合は元の run で使用されたエントリポイント、Git リポジトリ URI の場合は main.py がデフォルトになります。                                                                                                                                                                |
| `version`       | Git ベースのプロジェクトの場合、コミットハッシュまたはブランチ名。                                                                                                                                                                                                                             |
| `name`          | 起動する run の名前。                                                                                                                                                                                                                                                   |
| `resource`      | run の実行バックエンド。                                                                                                                                                                                                                                                  |
| `resource_args` | リモートバックエンド上で run を起動するためのリソース関連引数。生成される Launch 設定の `resource_args` に保存されます。                                                                                                                                                                                     |
| `project`       | 起動した run を送信する対象のプロジェクト                                                                                                                                                                                                                                         |
| `entity`        | 起動した run を送信する対象のエンティティ                                                                                                                                                                                                                                         |
| `config`        | run の設定を含む辞書。キー &quot;resource&#95;args&quot; の下にリソース固有の引数を含めることもできます。                                                                                                                                                                                          |
| `synchronous`   | run の完了を待つ間ブロックするかどうか。デフォルトは True です。`synchronous` が False で `backend` が &quot;local-container&quot; の場合、このメソッドは復帰しますが、ローカル run が完了するまで現在のプロセスは終了時にブロックされます。現在のプロセスが中断された場合、このメソッド経由で起動された非同期 run は終了されます。`synchronous` が True で run が失敗した場合、現在のプロセスもエラーで終了します。 |
| `run_id`        | run の ID（最終的に :name: フィールドを置き換えるためのもの）                                                                                                                                                                                                                          |
| `repository`    | リモートレジストリ用のリポジトリパスを表す文字列                                                                                                                                                                                                                                        |


<div id="example">
  #### 例：
</div>

```python
from wandb.sdk.launch import launch

job = "wandb/jobs/Hello World:latest"
params = {"epochs": 5}
# W&Bプロジェクトを実行し、ローカルホスト上に再現可能なDocker環境を作成する
api = wandb.apis.internal.Api()
launch(api, job, parameters=params)
```

| Returns                                                              | 説明 |
| :------------------------------------------------------------------- | :- |
| 起動された run に関する情報（例: run ID）を提供する `wandb.launch.SubmittedRun` インスタンス。 |    |

| Raises                                                               | 説明 |
| :------------------------------------------------------------------- | :- |
| ブロッキングモードで起動された run が失敗した場合に送出される `wandb.exceptions.ExecutionError`。 |    |
