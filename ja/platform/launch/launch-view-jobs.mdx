次のページでは、キューに追加された Launch ジョブに関する情報を表示する方法について説明します。

<div id="view-jobs">
  ## ジョブを表示する
</div>

W&amp;B App でキューに追加されたジョブを表示します。

1. https://wandb.ai/home の W&amp;B App にアクセスします。
2. 左サイドバーの **Applications** セクションで **Launch** を選択します。
3. **All entities** ドロップダウンを選択し、その Launch ジョブ が属する entity を選択します。
4. Launch Application ページで折りたたみ可能な UI を展開すると、その特定のキューに追加されたジョブの一覧を表示できます。

<Note>
  A run は、Launch エージェントが Launch ジョブ を実行すると作成されます。つまり、一覧に表示される各 run は、そのキューに追加された特定のジョブに対応しています。
</Note>

たとえば、次の画像は `job-source-launch_demo-canonical` というジョブから作成された 2 つの run を示しています。このジョブは `Start queue` というキューに追加されました。キュー内で最初に表示されている run は `resilient-snowball`、2 番目に表示されている run は `earthy-energy-165` です。

<Frame>
  <img src="/images/launch/launch_jobs_status.png" alt="Launch jobs status view" />
</Frame>

W&amp;B App UI では、Launch ジョブ から作成された run について、次のような追加情報を確認できます。

* **Run**: そのジョブに割り当てられた W&amp;B run の名前。
  * **Job ID**: ジョブの名前。
  * **Project**: run が属するプロジェクトの名前。
  * **Status**: キューに入っている run のステータス。
  * **Author**: run を作成した W&amp;B entity。
  * **Creation date**: キューが作成された日時のタイムスタンプ。
  * **Start time**: ジョブが開始された日時のタイムスタンプ。
  * **Duration**: ジョブの run が完了するまでにかかった時間 (秒) 。

<div id="list-jobs">
  ## ジョブを一覧表示する
</div>

W&amp;B CLI を使用して、プロジェクト内に存在するジョブの一覧を表示します。`wandb job list` コマンドを使用し、Launch ジョブが所属するプロジェクト名と entity 名を、それぞれ `--project` フラグと `--entity` フラグに指定します。

```bash
 wandb job list --entity your-entity --project project-name
```


<div id="check-the-status-of-a-job">
  ## ジョブのステータスを確認する
</div>

次の表は、キューに入っている run が取りうるステータスを示します。

| Status       | 説明                                                                        |
| ------------ | ------------------------------------------------------------------------- |
| **Idle**     | run はキューには入っていますが、稼働中のエージェントがいません。                                        |
| **Queued**   | run はキュー内で、エージェントによる処理待ちの状態です。                                            |
| **Pending**  | run はエージェントにピックアップされていますが、まだ開始していません。これは、クラスター上でリソースが利用できないことが原因の場合があります。 |
| **Running**  | run は現在実行中です。                                                             |
| **Killed**   | ジョブはユーザーによって強制終了されました。                                                    |
| **Crashed**  | run はデータ送信を停止したか、正常に開始できませんでした。                                           |
| **Failed**   | run はゼロ以外の終了コードで終了したか、run の開始に失敗しました。                                     |
| **Finished** | ジョブは正常に完了しました。                                                            |
