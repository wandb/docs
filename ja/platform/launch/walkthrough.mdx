<div id="what-is-launch">
  ## Launch とは何ですか？
</div>

<Card title="Colab で試す" href="https://colab.research.google.com/drive/1wX0OSVxZJDHRsZaOaOEDx-lLUrO1hHgP" icon="python" />

W&amp;B Launch を使うと、デスクトップ上の [runs](/ja/models/runs/) を、Amazon SageMaker や Kubernetes などのコンピュートリソースへ簡単にスケールさせることができます。W&amp;B Launch を一度設定しておけば、数回のクリックとコマンドだけで、トレーニングスクリプトやモデル評価スイートの実行、本番推論用のモデルの準備などをすばやく行うことができます。

<div id="how-it-works">
  ## 仕組み
</div>

Launch は、**launch ジョブ**、**キュー**、**エージェント** という 3 つの基本コンポーネントで構成されます。

[*launch job*](./launch-terminology#launch-job) は、ML ワークフローでタスクを設定して実行するための設計図です。launch ジョブを作成したら、それを [*launch queue*](./launch-terminology#launch-queue) に追加できます。launch queue は先入れ先出し (FIFO) のキューで、Amazon SageMaker や Kubernetes クラスターなどの特定のコンピュートターゲットリソース向けに、ジョブを設定して送信できます。

ジョブがキューに追加されると、[*launch agents*](./launch-terminology#launch-agent) がそのキューをポーリングし、キューのターゲットとなるシステム上でジョブを実行します。

<Frame>
  <img src="/images/launch/launch_overview.png" alt="W&B Launch の概要図" />
</Frame>

ユースケースに応じて、あなた (またはチームの誰か) が、選択した [コンピュートリソースターゲット](./launch-terminology#target-resources) (例: Amazon SageMaker) に合わせて launch queue を設定し、自身のインフラストラクチャ上に launch エージェントをデプロイします。

Launch の詳細については、[Terms and concepts](./launch-terminology) ページを参照してください。

<div id="how-to-get-started">
  ## 利用開始方法
</div>

ユースケースに応じて、以下のリソースから W&amp;B Launch の利用を開始してください。

* 初めて W&amp;B Launch を使用する場合は、[Launch walkthrough](#walkthrough) ガイドを一通り進めることをおすすめします。
* [W&amp;B Launch](/ja/platform/launch/set-up-launch/) の設定方法を学びましょう。
* [launch job](/ja/platform/launch/launch-terminology/#launch-job) を作成しましょう。
* [Triton へのデプロイ](https://github.com/wandb/launch-jobs/tree/main/jobs/deploy_to_nvidia_triton)、[LLM の評価](https://github.com/wandb/launch-jobs/tree/main/jobs/openai_evals) などの一般的なタスクのテンプレートについては、W&amp;B Launch の [public jobs GitHub リポジトリ](https://github.com/wandb/launch-jobs) を参照してください。
  * このリポジトリから作成された launch job は、公開 [`wandb/jobs` プロジェクト](https://wandb.ai/wandb/jobs/jobs) W&amp;B プロジェクトで閲覧できます。

<div id="prerequisites">
  ## 前提条件
</div>

開始する前に、次の前提条件を満たしていることを確認してください。

1. https://wandb.ai/site でアカウントを作成し、W&amp;B アカウントにログインします。
2. このチュートリアルを実行するには、Docker CLI と推論エンジンが動作するマシンへのターミナルアクセスが必要です。詳細は [Docker インストールガイド](https://docs.docker.com/engine/install/) を参照してください。
3. W&amp;B Python SDK バージョン `0.17.1` 以上をインストールします:
   ```bash
   pip install wandb>=0.17.1
   ```
4. ターミナルで `wandb login` を実行するか、`WANDB_API_KEY` 環境変数を設定して、W&amp;B に認証します。

<Tabs>
  <Tab title="W&B にログイン">
    ターミナルで次を実行します：

    ```bash
    wandb login
    ```
  </Tab>

  <Tab title="環境変数">
    ```bash
        WANDB_API_KEY=<your-api-key>
    ```

    `<your-api-key>` を自分の W&amp;B APIキーに置き換えます。
  </Tab>
</Tabs>

<div id="create-a-launch-job">
  ## launch ジョブを作成する
</div>

[launch ジョブ](./launch-terminology#launch-job) は、Docker イメージ、git リポジトリ、ローカルのソースコードのいずれかから、3 つの方法で作成できます。

<Tabs>
  <Tab title="With a Docker image">
    事前に作成されたコンテナーを実行して W&amp;B にメッセージをログするには、ターミナルを開き、次のコマンドを実行します。

    ```bash
    wandb launch --docker-image wandb/job_hello_world:main --project launch-quickstart
    ```

    上記のコマンドは、コンテナーイメージ `wandb/job_hello_world:main` をダウンロードして実行します。

    Launch は、コンテナーで `wandb` によってログされたすべての内容を `launch-quickstart` プロジェクトにレポートするようにコンテナーを設定します。コンテナーは W&amp;B にメッセージをログし、新しく作成された run へのリンクを W&amp;B 上に表示します。リンクをクリックして、W&amp;B UI で run を確認します。
  </Tab>

  <Tab title="From a git repository">
    同じ hello-world ジョブを [W&amp;B Launch jobs リポジトリ内のソースコード](https://github.com/wandb/launch-jobs) から起動するには、次のコマンドを実行します。

    ```bash
    wandb launch --uri https://github.com/wandb/launch-jobs.git \\
    --job-name hello-world-git --project launch-quickstart \\ 
    --build-context jobs/hello_world --dockerfile Dockerfile.wandb \\ 
    --entry-point "python job.py"
    ```

    このコマンドは次の処理を行います。

    1. 一時ディレクトリに [W&amp;B Launch jobs リポジトリ](https://github.com/wandb/launch-jobs) をクローンします。
    2. **hello** プロジェクトに **hello-world-git** という名前のジョブを作成します。このジョブは、コードの実行に使用された正確なソースコードと設定を追跡します。
    3. `jobs/hello_world` ディレクトリと `Dockerfile.wandb` からコンテナーイメージをビルドします。
    4. コンテナーを起動し、`job.py` Python スクリプトを実行します。

    コンソール出力には、イメージのビルドおよび実行のログが表示されます。コンテナーの出力は前の例とほぼ同一になるはずです。
  </Tab>

  <Tab title="From local source code">
    git リポジトリでバージョン管理されていないコードは、ローカル ディレクトリのパスを `--uri` 引数に指定して起動できます。

    空のディレクトリを作成し、次の内容の `train.py` という名前の Python スクリプトを追加します。

    ```python
    import wandb

    with wandb.init() as run:
        run.log({"hello": "world"})
    ```

    次の内容の `requirements.txt` ファイルを追加します。

    ```text
    wandb>=0.17.1
    ```

    ディレクトリ内で次のコマンドを実行します。

    ```bash
    wandb launch --uri . --job-name hello-world-code --project launch-quickstart --entry-point "python train.py"
    ```

    このコマンドは次の処理を行います。

    1. 現在のディレクトリの内容を、Code Artifact として W&amp;B にログします。
    2. **launch-quickstart** プロジェクトに **hello-world-code** という名前のジョブを作成します。
    3. `train.py` と `requirements.txt` をベース イメージにコピーし、`pip install` で依存パッケージをインストールしてコンテナーイメージをビルドします。
    4. コンテナーを起動し、`python train.py` を実行します。
  </Tab>
</Tabs>

<div id="create-a-queue">
  ## キューを作成する
</div>

Launch は、チームが共有コンピュートリソースを使ったワークフローを 구축하다できるように設計されています。これまでの例では、`wandb launch` コマンドはローカルマシン上でコンテナーを同期的に実行してきました。Launch のキューとエージェントを使うと、共有リソース上でジョブを非同期実行したり、優先順位付けやハイパーパラメーター最適化といった高度な機能を利用できます。基本的なキューを作成するには、次の step に従ってください。

1. [wandb.ai/launch](https://wandb.ai/launch) にアクセスし、**Create a queue** ボタンをクリックします。
2. キューに関連付ける **Entity** を選択します。
3. **Queue name** を入力します。
4. **Resource** として **Docker** を選択します。
5. **Configuration** は、ここでは空白のままにしておきます。
6. **Create queue** をクリックします :rocket:

ボタンをクリックすると、ブラウザはキューのビューの **Agents** タブにリダイレクトされます。エージェントがポーリングを開始するまで、キューは **Not active** 状態のままです。

<Frame>
  <img src="/images/launch/create_docker_queue.gif" alt="Docker queue creation" />
</Frame>

キューの高度な設定オプションについては、[高度なキュー設定ページ](/ja/platform/launch/setup-queue-advanced/) を参照してください。

<div id="connect-an-agent-to-the-queue">
  ## エージェントをキューに接続する
</div>

キューにポーリング エージェントがない場合は、キュー画面の上部にある赤いバナーに **Add an agent** ボタンが表示されます。ボタンをクリックすると、エージェントを実行するためのコマンドを表示してコピーできます。コマンドは次のようになります：

```bash
wandb launch-agent --queue <queue-name> --entity <entity-name>
```

ターミナルでコマンドを実行してエージェントを起動します。エージェントは指定されたキューをポーリングして、実行すべきジョブがないか監視します。ジョブを受信すると、エージェントはそのジョブ用のコンテナーイメージをダウンロードまたはビルドし、ローカルで `wandb launch` コマンドを実行した場合と同様にコンテナーイメージを実行します。

[Launch ページ](https://wandb.ai/launch) に戻り、キューが **Active** と表示されていることを確認します。

<div id="submit-a-job-to-the-queue">
  ## キューにジョブを送信する
</div>

W&amp;B アカウントで新しい **launch-quickstart** プロジェクトにアクセスし、画面左側のナビゲーションから **Jobs** タブを開きます。

**Jobs** ページには、以前に実行された run から作成された W&amp;B Jobs の一覧が表示されます。**launch** ジョブをクリックすると、ソースコード、依存関係、およびそのジョブから作成された run を確認できます。このチュートリアルを完了すると、一覧には 3 つのジョブが表示されているはずです。

新しいジョブの 1 つを選択し、次の手順に従ってキューに送信します。

1. **Launch** ボタンをクリックして、ジョブをキューに送信します。**Launch** ドロワーが表示されます。
2. 先ほど作成した **Queue** を選択し、**Launch** をクリックします。

これでジョブがキューに送信されます。このキューをポーリングしているエージェントがジョブを取得して実行します。ジョブの進行状況は W&amp;B UI から、またはターミナルでエージェントの出力を確認することで監視できます。

`wandb launch` コマンドでは、`--queue` 引数を指定することでジョブを直接キューに投入できます。たとえば、hello-world コンテナー ジョブをキューに送信するには、次のコマンドを実行します。

```bash
   wandb launch --docker-image wandb/job_hello_world:main --project launch-quickstart --queue <queue-name>
```
