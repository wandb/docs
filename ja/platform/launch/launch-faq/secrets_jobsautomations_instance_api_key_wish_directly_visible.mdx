---
title: Jobs や Automations に対してシークレットを指定することはできますか？例えば、Users に直接見せたくない APIキー などです。
---

はい。以下の手順に従ってください。

1. 次のコマンドを使用して、Runs 用の指定された名前空間に Kubernetes secret を作成します。
   `kubectl create secret -n <namespace> generic <secret_name> <secret_value>`

2. secret を作成した後、Runs の開始時にその secret を注入するようにキューを設定します。secret を閲覧できるのはクラスター管理者のみであり、エンド Users は閲覧できません。
