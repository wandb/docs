---
title: ジョブや自動化でsecretを指定できますか？たとえば、ユーザーに直接見せたくないAPIキーなどは可能ですか？
---

はい。次の step に従ってください。

1. 次のコマンドを使って、run 用に指定されたネームスペースに Kubernetes secret を作成します:
   `kubectl create secret -n <namespace> generic <secret_name> <secret_value>`

2. secret を作成したら、run 開始時にその secret が注入されるようにキューを設定します。secret を閲覧できるのはクラスター管理者のみであり、エンドユーザーには表示されません。
