---
title: Dockerfile を指定して、W&B に Docker イメージをビルドさせることはできますか？
---

この機能は、依存関係は安定している一方で、コードベースが頻繁に変わるプロジェクトに適しています。

<Warning>
  Dockerfile はマウントを使用する形式で記述してください。詳細は、[Docker Docs の Mounts ドキュメント](https://docs.docker.com/build/guide/mounts/)を参照してください。
</Warning>

Dockerfile を設定したら、次の 3 つのいずれかの方法で W&amp;B に指定します。

* Dockerfile.wandb を使用する
* W&amp;B CLI を使用する
* W&amp;B App を使用する

<Tabs>
  <Tab title="Dockerfile.wandb">
    W&amp;B run のエントリポイントと同じディレクトリに `Dockerfile.wandb` ファイルを配置します。W&amp;B は組み込みの Dockerfile の代わりにこのファイルを使用します。
  </Tab>

  <Tab title="W&B CLI">
    `wandb launch` コマンドで `--dockerfile` フラグを使用して、ジョブをキューに追加します。

    ```bash
    wandb launch --dockerfile path/to/Dockerfile
    ```
  </Tab>

  <Tab title="W&B App">
    W&amp;B App でキューにジョブを追加する際は、**Overrides** セクションで Dockerfile のパスを指定します。キーを `"dockerfile"`、値を Dockerfile へのパスとするキーと値のペアとして入力します。

    次の JSON は、ローカルディレクトリ内の Dockerfile を指定する方法を示しています。

    ```json title="Launch job W&B App"
    {
      "args": [],
      "run_config": {
        "lr": 0,
        "batch_size": 0,
        "epochs": 0
      },
      "entrypoint": [],
      "dockerfile": "./Dockerfile"
    }
    ```
  </Tab>
</Tabs>
