---
title: "`wandb launch -d` または `wandb job create image` では、Docker アーティファクト全体が upload され、レジストリから pull されないのですか？"
---

いいえ。`wandb launch -d` command では、イメージはレジストリに upload されません。イメージは別途レジストリに upload してください。次の step に従ってください:

1. イメージを build します。
2. イメージをレジストリに push します。

ワークフローは次のとおりです:

```bash
docker build -t <repo-url>:<tag> .
docker push <repo-url>:<tag>
wandb launch -d <repo-url>:<tag>
```

その後、Launch エージェントは指定されたコンテナーを参照するジョブを起動します。エージェントがコンテナーレジストリからイメージをプルできるようにアクセスを設定する例については、[高度なエージェント設定](/ja/platform/launch/setup-agent-advanced/#agent-configuration)を参照してください。

Kubernetes では、Kubernetes クラスターの Pod が、イメージのプッシュ先のレジストリにアクセスできることを確認してください。
