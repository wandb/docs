---
title: "`wandb launch -d` や `wandb job create image` では、Docker アーティファクト全体をアップロードし、Registry から pull していないのですか？"
---

いいえ、`wandb launch -d` コマンドはイメージを Registry にアップロードしません。イメージは別途 Registry にアップロードしてください。次の手順に従ってください。

1. イメージをビルドする。
2. イメージを Registry に push する。

ワークフローは次のとおりです。

```bash
docker build -t <repo-url>:<tag> .
docker push <repo-url>:<tag>
wandb launch -d <repo-url>:<tag>
```

Launch エージェントは、その後、指定されたコンテナーを参照するジョブを起動します。コンテナーレジストリからイメージをプルするためのエージェントのアクセス設定例については、[Advanced agent setup](/ja/platform/launch/setup-agent-advanced/#agent-configuration) を参照してください。

Kubernetes の場合、Kubernetes クラスター内の Pod が、イメージがプッシュされているレジストリにアクセスできることを確認してください。
