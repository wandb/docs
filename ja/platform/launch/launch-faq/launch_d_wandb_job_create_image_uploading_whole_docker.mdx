---
title: "`wandb launch -d` や `wandb job create image` はレジストリから pull せずに docker Artifacts 全体をアップロードしていますか？"
---

いいえ、`wandb launch -d` コマンドはイメージをレジストリにアップロードしません。イメージは別途レジストリにアップロードしてください。次の手順に従ってください。

1. イメージをビルドする。
2. イメージをレジストリに push する。

ワークフローは次のとおりです。

```bash
docker build -t <repo-url>:<tag> .
docker push <repo-url>:<tag>
wandb launch -d <repo-url>:<tag>
```

Launch エージェントは、その後、指定されたコンテナイメージを使用するジョブを起動します。コンテナレジストリからイメージをプルするためにエージェントからのアクセスを設定する例については、[Advanced agent setup](/ja/platform/launch/setup-agent-advanced/#agent-configuration) を参照してください。

Kubernetes の場合、Kubernetes クラスター内の Pod が、イメージがプッシュされているレジストリにアクセスできることを確認してください。
