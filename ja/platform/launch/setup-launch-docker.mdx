---
title: 'チュートリアル: ローカルの Docker で W&B Launch をセットアップする'
---

このガイドでは、Launch エージェントの環境とキューのターゲットリソースの両方に対して、ローカル マシン上で Docker を使うように W&amp;B Launch を設定する方法を説明します。

同じローカル マシン上で、ジョブの実行と Launch エージェントの環境の両方に Docker を使用する方法は、Kubernetes などのクラスター管理システムがインストールされていないマシンにコンピュート環境が構築されている場合に特に有用です。

また、Docker キューを使用して、高性能なワークステーション上でワークロードを実行することもできます。

<Note>
  このセットアップは、ローカル マシンで実験を行うユーザーや、SSH でログインして Launch ジョブを送信するリモートマシンを持つユーザーによく見られます。
</Note>

Docker を W&amp;B Launch と一緒に使用すると、W&amp;B はまずイメージをビルドし、そのイメージからコンテナをビルドして実行します。イメージは Docker の `docker run &lt;image-uri&gt;` コマンドでビルドされます。キューの設定は、`docker run` コマンドに渡される追加の引数として解釈されます。

<div id="configure-a-docker-queue">
  ## Docker キューを設定する
</div>

Docker ターゲットリソース向けの Launch キュー設定は、[`docker run`](/ja/models/ref/cli/wandb-docker-run) CLI コマンドで定義されているものと同じオプションを受け付けます。

エージェントは、キュー設定で定義されたオプションを受け取ります。次にエージェントは、受け取ったオプションを Launch ジョブの設定からのオーバーライドとマージし、最終的にターゲットリソース（この場合はローカルマシン）上で実行される `docker run` コマンドを生成します。

次の 2 つの構文上の変換が行われます:

1. 繰り返し指定されるオプションは、キュー設定内でリストとして定義されます。
2. フラグオプションは、値が `true` の Boolean としてキュー設定内で定義されます。

例えば、次のようなキュー設定になります:

```json
{
  "env": ["MY_ENV_VAR=value", "MY_EXISTING_ENV_VAR"],
  "volume": "/mnt/datasets:/mnt/datasets",
  "rm": true,
  "gpus": "all"
}
```

その結果、次の `docker run` コマンドが得られます：

```bash
docker run \
  --env MY_ENV_VAR=value \
  --env MY_EXISTING_ENV_VAR \
  --volume "/mnt/datasets:/mnt/datasets" \
  --rm <image-uri> \
  --gpus all
```

ボリュームは文字列のリスト、または単一の文字列として指定できます。複数のボリュームを指定する場合はリストを使用します。

Docker は、値が設定されていない環境変数を Launch エージェントの環境から自動的に引き継ぎます。つまり、Launch エージェントに環境変数 `MY_EXISTING_ENV_VAR` がある場合、その環境変数はコンテナ内でも利用可能です。これは、キューの設定内で公開することなく、他の config キーを使用したい場合に便利です。

`docker run` コマンドの `--gpus` フラグを使用すると、Docker コンテナで利用可能な GPU を指定できます。`gpus` フラグの使用方法の詳細については、[Docker ドキュメント](https://docs.docker.com/config/containers/resource_constraints/#gpu) を参照してください。

<Note>
  * Docker コンテナ内で GPU を使用するには、[NVIDIA Container Toolkit](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/install-guide.html#docker) をインストールします。
  * コードまたはアーティファクト由来のジョブからイメージをビルドする場合、NVIDIA Container Toolkit を含めるために、[agent](#configure-a-launch-agent-on-a-local-machine) によって使用されるベースイメージを上書きできます。
    たとえば、Launch キュー内でベースイメージを `tensorflow/tensorflow:latest-gpu` に上書きできます:

    ```json
    {
      "builder": {
        "accelerator": {
          "base_image": "tensorflow/tensorflow:latest-gpu"
        }
      }
    }
    ```
</Note>

<div id="create-a-queue">
  ## キューを作成する
</div>

W&amp;B CLI を使用して、Docker をコンピュートリソースとして利用するキューを作成します。

1. [Launch ページ](https://wandb.ai/launch) にアクセスします。
2. **Create Queue** ボタンをクリックします。
3. キューを作成する **Entity** を選択します。
4. **Name** フィールドにキュー名を入力します。
5. **Resource** として **Docker** を選択します。
6. **Configuration** フィールドで Docker キューの設定を指定します。
7. **Create Queue** ボタンをクリックしてキューを作成します。

<div id="configure-a-launch-agent-on-a-local-machine">
  ## ローカルマシンで launch agent を設定する
</div>

`launch-config.yaml` という名前の YAML 設定ファイルで launch agent を設定します。デフォルトでは、W&amp;B は `~/.config/wandb/launch-config.yaml` にある設定ファイルを探します。launch agent を起動するときに、別のディレクトリを指定することもできます。

<Note>
  YAML 設定ファイルの代わりに、W&amp;B CLI を使用して launch agent の主な設定項目（ジョブの最大数、W&amp;B エンティティ、launch キュー）を指定できます。詳しくは [`wandb launch-agent`](/ja/models/ref/cli/wandb-launch-agent) コマンドを参照してください。
</Note>

<div id="core-agent-config-options">
  ## エージェントのコア設定オプション
</div>

以下のタブでは、W&amp;B CLI と YAML 設定ファイルを使って、エージェントのコア設定オプションを指定する方法を示します。

<Tabs>
  <Tab title="W&B CLI">
    ```bash
    wandb launch-agent -q <queue-name> --max-jobs <n>
    ```
  </Tab>

  <Tab title="設定ファイル">
    ```yaml title="launch-config.yaml"
    max_jobs: <n concurrent jobs>
    queues:
    	- <queue-name>
    ```
  </Tab>
</Tabs>

<div id="docker-image-builders">
  ## Docker イメージビルダー
</div>

マシン上の Launch エージェントは、Docker イメージをビルドするように設定できます。デフォルトでは、これらのイメージはマシンのローカル イメージリポジトリに保存されます。Launch エージェントで Docker イメージのビルドを有効にするには、Launch エージェントの config の `builder` キーを `docker` に設定します。

```yaml title="launch-config.yaml"
builder:
	type: docker
```

エージェントに Docker イメージをビルドさせず、代わりにレジストリの事前ビルド済みイメージを使用したい場合は、Launch エージェントの設定で `builder` キーを `noop` に設定します。

```yaml title="launch-config.yaml"
builder:
  type: noop
```

<div id="container-registries">
  ## コンテナー・レジストリ
</div>

Launch は Docker Hub、Google Container Registry、Azure Container Registry、Amazon ECR などの外部コンテナー・レジストリを使用します。\
ビルドした環境とは異なる環境でジョブを実行したい場合は、エージェントがコンテナー・レジストリからイメージを pull できるように設定してください。

Launch エージェントをクラウドのコンテナー・レジストリに接続する方法の詳細については、[高度なエージェント設定](./setup-agent-advanced#agent-configuration) ページを参照してください。
