次のガイドでは、Launch エージェント環境とキューのターゲットリソースの両方で、ローカルマシン上の Docker を使用するように W&amp;B Launch を設定する方法を説明します。

同じローカルマシン上で、ジョブの実行と Launch エージェント環境の両方に Docker を使用する構成は、計算環境がクラスター管理システム (Kubernetes など) を持たないマシンにある場合に特に便利です。

Docker キューを使用して、高性能なワークステーションでワークロードを実行することもできます。

<Note>
  この設定は、ローカルマシンで実験を行うユーザーや、SSH で接続したリモートマシンから Launch ジョブを送信するユーザーによく使われます。
</Note>

W&amp;B Launch で Docker を使用すると、W&amp;B はまずイメージを 구축し、その後そのイメージからコンテナーを 구축して実行します。イメージは Docker の `docker run <image-uri>` コマンドで 구축されます。キュー設定は、`docker run` コマンドに渡される追加の引数として解釈されます。

<div id="configure-a-docker-queue">
  ## Docker キューを設定する
</div>

Launch キュー設定 (Docker ターゲットリソース向け) は、[`docker run`](/ja/models/ref/cli/wandb-docker-run) CLI コマンドで Defined in されているものと同じオプションを受け入れます。

エージェントは、キュー設定で Defined in されているオプションを受け取ります。次に、エージェントは受け取ったオプションを launch job の設定からのオーバーライドとマージし、ターゲットリソース (この場合はローカルマシン) で実行される最終的な `docker run` コマンドを生成します。

次の 2 つの構文変換が行われます。

1. 繰り返し指定するオプションは、キュー設定では list として Defined in されます。
2. フラグオプションは、キュー設定では値が `true` のブール値として Defined in されます。

たとえば、次のキュー設定です。

```json
{
  "env": ["MY_ENV_VAR=value", "MY_EXISTING_ENV_VAR"],
  "volume": "/mnt/datasets:/mnt/datasets",
  "rm": true,
  "gpus": "all"
}
```

次の`docker run`コマンドになります:

```bash
docker run \
  --env MY_ENV_VAR=value \
  --env MY_EXISTING_ENV_VAR \
  --volume "/mnt/datasets:/mnt/datasets" \
  --rm <image-uri> \
  --gpus all
```

ボリュームは、文字列のリストまたは単一の文字列として指定できます。複数のボリュームを指定する場合は、リストを使用します。

Docker は、値が割り当てられていない環境変数を Launch エージェント環境から自動的に引き継ぎます。つまり、Launch エージェントに環境変数 `MY_EXISTING_ENV_VAR` がある場合、その環境変数はコンテナー内でも使用できます。これは、キュー設定で公開せずに他の設定キーを使用したい場合に便利です。

`docker run` コマンドの `--gpus` フラグを使用すると、Docker コンテナーで使用可能な GPU を指定できます。`gpus` フラグの使い方の詳細については、[Docker documentation](https://docs.docker.com/config/containers/resource_constraints/#gpu) を参照してください。

<Note>
  * Docker コンテナー内で GPU を使用するには、[NVIDIA Container Toolkit](https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/install-guide.html#docker) をインストールしてください。
  * code またはアーティファクトをソースとするジョブからイメージを 구축する場合は、[エージェント](#configure-a-launch-agent-on-a-local-machine) が使用するベースイメージを override して、NVIDIA Container Toolkit を含めることができます。
    たとえば、Launch キュー 内でベースイメージを `tensorflow/tensorflow:latest-gpu` に override できます。

    ```json
    {
      "builder": {
        "accelerator": {
          "base_image": "tensorflow/tensorflow:latest-gpu"
        }
      }
    }
    ```
</Note>

<div id="create-a-queue">
  ## キューを作成する
</div>

W&amp;B CLI を使用して、Docker をコンピュートリソースとして使用するキューを作成します。

1. [Launch ページ](https://wandb.ai/launch) にアクセスします。
2. **Create Queue** ボタンをクリックします。
3. キューを作成する **Entity** を選択します。
4. **名** フィールドにキュー名を入力します。
5. **Resource** として **Docker** を選択します。
6. **設定** フィールドで Docker キューの設定を指定します。
7. **Create Queue** ボタンをクリックしてキューを作成します。

<div id="configure-a-launch-agent-on-a-local-machine">
  ## ローカルマシンで Launch エージェント を設定する
</div>

`launch-config.yaml` という名前の YAML 설정 파일で Launch エージェント を設定します。デフォルトでは、W&amp;B は `~/.config/wandb/launch-config.yaml` にある설정 파일を探します。Launch エージェント を起動するときに、別のディレクトリを指定することもできます。

<Note>
  YAML 설정 파일の代わりに、W&amp;B CLI を使用して Launch エージェント の主な設定項目 (ジョブの最大数、W&amp;B entity、Launch キュー) を指定できます。詳しくは [`wandb launch-agent`](/ja/models/ref/cli/wandb-launch-agent) コマンドを参照してください。
</Note>

<div id="core-agent-config-options">
  ## エージェントのコア設定オプション
</div>

以下のタブでは、W&amp;B CLI と YAML 설정 파일を使って、エージェントのコア設定オプションを指定する方法を示します。

<Tabs>
  <Tab title="W&B CLI">
    ```bash
    wandb launch-agent -q <queue-name> --max-jobs <n>
    ```
  </Tab>

  <Tab title="설정 파일">
    ```yaml title="launch-config.yaml"
    max_jobs: <n concurrent jobs>
    queues:
    	- <queue-name>
    ```
  </Tab>
</Tabs>

<div id="docker-image-builders">
  ## Docker イメージビルダー
</div>

マシン上の Launch エージェントは、Docker イメージを 구축するように設定できます。デフォルトでは、これらのイメージはマシンのローカル イメージリポジトリに保存されます。Launch エージェントで Docker イメージの 구축を有効にするには、Launch エージェントの設定の `builder` キーを `docker` に設定します。

```yaml title="launch-config.yaml"
builder:
	type: docker
```

エージェントに Docker イメージを 구축させず、代わりにレジストリ内の구축済みイメージを使用する場合は、Launch エージェントの設定で `builder` キーを `noop` に設定します

```yaml title="launch-config.yaml"
builder:
  type: noop
```

<div id="container-registries">
  ## コンテナー・レジストリ
</div>

Launch は Docker Hub、Google Container Registry、Azure Container Registry、Amazon ECR などの外部コンテナー・レジストリを使用します。
ジョブをビルドした環境とは異なる環境で実行する場合、エージェントがコンテナー・レジストリからイメージを pull できるように設定してください。

Launch エージェントをクラウドのコンテナー・レジストリに接続する方法の詳細については、[高度なエージェント設定](./setup-agent-advanced#agent-configuration) ページを参照してください。
