W&amp;B Launch を使ってハイパーパラメーター調整ジョブ ([sweeps](/ja/models/sweeps/)) を作成します。Launch 上の sweep では、指定したハイパーパラメーターを sweep するための sweep スケジューラーが Launch Queue にプッシュされます。エージェントに取得されると sweep スケジューラーが起動し、選択されたハイパーパラメーターで同じキュー上に sweep run を起動します。これは sweep が完了するか停止されるまで続きます。

標準の W&amp;B Sweep スケジューリングエンジンを使うか、独自のカスタムスケジューラーを実装できます。

1. 標準 sweep スケジューラー: [W&amp;B Sweeps](/ja/models/sweeps/) を制御する、デフォルトの W&amp;B Sweep スケジューリングエンジンを使用します。おなじみの `bayes`、`grid`、`random` メソッドが利用できます。
2. カスタム sweep スケジューラー: sweep スケジューラーをジョブとして実行するように構成します。このオプションにより、完全にカスタマイズ可能になります。標準 sweep スケジューラーを拡張してログを追加する例は、以下のセクションにあります。

<Note>
  このガイドでは、W&amp;B Launch がすでに構成済みであることを前提としています。W&amp;B Launch が未構成の場合は、Launch ドキュメントの [導入方法](./#how-to-get-started) セクションを参照してください。
</Note>

<Note>
  Launch 上で sweep を初めて利用する場合は、&#39;basic&#39; メソッドを使用して sweep を作成することをお勧めします。標準の W&amp;B スケジューリングエンジンで要件を満たせない場合は、カスタムの Launch 用 sweep スケジューラーを使用してください。
</Note>

<div id="create-a-sweep-with-a-wb-standard-scheduler">
  ## W&amp;B 標準スケジューラーで sweep を作成する
</div>

Launch を使って W&amp;B Sweeps を作成できます。W&amp;B App を使って対話的に sweep を作成することも、W&amp;B CLI を使ってプログラムから作成することもできます。スケジューラーのカスタマイズ機能を含む Launch sweeps の高度な設定を行うには、CLI を使用してください。

<Note>
  W&amp;B Launch で sweep を作成する前に、まず sweep 対象となる job を作成しておいてください。詳細は [Create a Job](/ja/platform/launch/create-launch-job/) ページを参照してください。
</Note>

<Tabs>
  <Tab title="W&B app">
    W&amp;B App を使って対話的に sweep を作成します。

    1. W&amp;B App で自分の W&amp;B プロジェクトに移動します。
    2. プロジェクトのサイドバーで Sweeps アイコン (ほうきの画像) を選択します。
    3. 次に **Create Sweep** ボタンを選択します。
    4. **Configure Launch** ボタンをクリックします。
    5. **Job** ドロップダウンメニューから、sweep を作成したい job の名前と job バージョンを選択します。
    6. **Queue** ドロップダウンメニューを使って、sweep を実行するキューを選択します。
    7. **Job Priority** ドロップダウンを使って、launch job の優先度を指定します。launch キューが優先度をサポートしていない場合、launch job の優先度は「Medium」に設定されます。
    8.  (任意) run や sweep スケジューラーに対する override 引数を設定します。たとえば、スケジューラーの override を使って、`num_workers` でスケジューラーが管理する同時実行 run 数を設定します。
    9.  (任意) **Destination Project** ドロップダウンメニューを使って、sweep を保存するプロジェクトを選択します。
    10. **Save** をクリックします。
    11. **Launch Sweep** を選択します。

    <Frame>
      <img src="/images/launch/create_sweep_with_launch.png" alt="Launch sweep configuration" />
    </Frame>
  </Tab>

  <Tab title="CLI">
    W&amp;B CLI を使って、Launch で W&amp;B Sweep をプログラム的に作成します。

    1. sweep 設定を作成します。
    2. sweep 設定内で完全な job 名を指定します。
    3. sweep エージェントを初期化します。

    <Note>
      手順 1 と 3 は、通常の W&amp;B Sweep を作成するときと同じ手順です。
    </Note>

    たとえば、次のコードスニペットでは、job の値として `'wandb/jobs/Hello World 2:latest'` を指定しています。

    ```yaml
    # launch-sweep-config.yaml

    job: 'wandb/jobs/Hello World 2:latest'
    description: launch jobs を用いた sweep の例

    method: bayes
    metric:
      goal: minimize
      name: loss_metric
    parameters:
      learning_rate:
        max: 0.02
        min: 0
        distribution: uniform
      epochs:
        max: 20
        min: 0
        distribution: int_uniform

    # 任意のスケジューラー用パラメーター:

    # scheduler:
    #   num_workers: 1  # 同時に実行する sweep run 数
    #   docker_image: <スケジューラー用のベースイメージ>
    #   resource: <例: local-container...>
    #   resource_args:  # run に渡されるリソース引数
    #     env: 
    #         - WANDB_API_KEY

    # 任意の Launch パラメーター
    # launch: 
    #    registry: <イメージ取得に使用するレジストリ>
    ```

    sweep 設定の作成方法については、[Define sweep configuration](/ja/platform/launch/sweeps-on-launch/) ページを参照してください。

    4. 次に sweep を初期化します。config ファイルへのパス、job キュー名、W&amp;B entity、プロジェクト名を指定します。

    ```bash
    wandb launch-sweep <path/to/yaml/file> --queue <queue_name> --entity <your_entity>  --project <project_name>
    ```

    W&amp;B Sweeps の詳細については、[Tune Hyperparameters](/ja/models/sweeps/) セクションを参照してください。
  </Tab>
</Tabs>

<div id="create-a-custom-sweep-scheduler">
  ## カスタム sweep スケジューラーを作成する
</div>

W&amp;B scheduler または独自のスケジューラーを使用して、カスタム sweep スケジューラーを作成します。

<Note>
  scheduler ジョブを使用するには、wandb CLI バージョンが `0.15.4` 以上である必要があります。
</Note>

<Tabs>
  <Tab title="W&B スケジューラー">
    W&amp;B の sweep スケジューリング ロジックをジョブとして使用して、Launch sweep を作成します。

    1. 公開プロジェクト wandb/sweep-jobs 内の wandb scheduler ジョブを確認するか、次のジョブ名を使用します:
       `'wandb/sweep-jobs/job-wandb-sweep-scheduler:latest'`
    2. この名前を指す `job` キーを含む `scheduler` ブロックを追加した設定 YAML ファイルを作成します。以下は例です。
    3. 新しい設定ファイルを使って `wandb launch-sweep` コマンドを実行します。

    設定例:

    ```yaml
    # launch-sweep-config.yaml  
    description: スケジューラージョブを使用した Launch sweep の設定
    scheduler:
      job: wandb/sweep-jobs/job-wandb-sweep-scheduler:latest
      num_workers: 8  # 8 つの sweep run を同時実行可能

    # sweep の run が実行するトレーニング/チューニングジョブ
    job: wandb/sweep-jobs/job-fashion-MNIST-train:latest
    method: grid
    parameters:
      learning_rate:
        min: 0.0001
        max: 0.1
    ```
  </Tab>

  <Tab title="カスタム スケジューラ">
    カスタムスケジューラーは scheduler-job を作成することで定義できます。このガイドでは、より詳細なログを取得するために `WandbScheduler` を修正します。

    1. `wandb/launch-jobs` リポジトリ (具体的には `wandb/launch-jobs/jobs/sweep_schedulers`) をクローンします。
    2. 次に、`wandb_scheduler.py` を修正して、目的どおりにログ出力を増やします。例: 関数 `_poll` にログ出力を追加します。これは、新しい sweep run を起動する前に、ポーリングサイクルごと (間隔は設定可能) に 1 回呼び出されます。
    3. 修正したファイルを次のように実行してジョブを作成します: `python wandb_scheduler.py --project <project> --entity <entity> --name CustomWandbScheduler`
    4. 作成されたジョブの名前を、UI もしくは前のコマンドの出力から確認します。このジョブは (特に指定がない限り) code-artifact ジョブになります。
    5. スケジューラーが新しいジョブを参照するように、sweep の設定を作成します。

    ```yaml
    ...
    scheduler:
      job: '<entity>/<project>/job-CustomWandbScheduler:latest'
    ...
    ```
  </Tab>

  <Tab title="Optuna スケジューラ">
    Optuna は、与えられたモデルに対して最適なハイパーパラメーターを見つけるためにさまざまなアルゴリズムを用いるハイパーパラメーター最適化フレームワークです (W&amp;B と類似) 。[sampling algorithms](https://optuna.readthedocs.io/en/stable/reference/samplers/index.html) に加えて、Optuna は性能の低い run を早期に打ち切るために利用できるさまざまな [pruning algorithms](https://optuna.readthedocs.io/en/stable/reference/pruners.html) も提供しています。これは多数の run を実行する場合に特に有用で、時間とリソースの節約につながります。これらのクラスは高い柔軟性を備えており、設定ファイルの `scheduler.settings.pruner/sampler.args` ブロックに必要なパラメーターを渡すだけで構成できます。

    Optuna のスケジューリングロジックを使い、ジョブを用いて Launch sweep を作成します。

    1. まず、自分のジョブを作成するか、あらかじめ用意された Optuna scheduler イメージを使ったジョブを使用します。
       * 独自のジョブを作成する方法の例については、[`wandb/launch-jobs`](https://github.com/wandb/launch-jobs/blob/main/jobs/sweep_schedulers) リポジトリを参照してください。
       * 事前構築済みの Optuna イメージを使用するには、`wandb/sweep-jobs` プロジェクト内の `job-optuna-sweep-scheduler` に移動するか、ジョブ名 `wandb/sweep-jobs/job-optuna-sweep-scheduler:latest` を使用します。

    2. ジョブを作成したら、次に sweep を作成できます。Optuna scheduler ジョブを指す `job` キーを含む `scheduler` ブロックを持つ sweep 設定を作成します (下記の例を参照) 。

    ```yaml
      # optuna_config_basic.yaml
      description: 基本的な Optuna スケジューラー
      job: wandb/sweep-jobs/job-fashion-MNIST-train:latest
      run_cap: 5
      metric:
        name: epoch/val_loss
        goal: minimize

      scheduler:
        job: wandb/sweep-jobs/job-optuna-sweep-scheduler:latest
        resource: local-container  # イメージから取得したスケジューラージョブに必要
        num_workers: 2

        # Optuna 固有の設定
        settings:
          pruner:
            type: PercentilePruner
            args:
              percentile: 25.0  # run の 75% を強制終了
              n_warmup_steps: 10  # 最初の x ステップはプルーニングを無効化

      parameters:
        learning_rate:
          min: 0.0001
          max: 0.1
    ```

    3. 最後に、`launch-sweep` コマンドを使用して、アクティブなキューで sweep を起動します。

    ```bash
    wandb launch-sweep <config.yaml> -q <queue> -p <project> -e <entity>
    ```

    Optuna sweep scheduler ジョブの具体的な実装については、[wandb/launch-jobs](https://github.com/wandb/launch-jobs/blob/main/jobs/sweep_schedulers/optuna_scheduler/optuna_scheduler.py) を参照してください。Optuna scheduler で可能なことのより多くの例については、[wandb/examples](https://github.com/wandb/examples/tree/master/examples/launch/launch-sweeps/optuna-scheduler) も参照してください。
  </Tab>
</Tabs>

カスタム sweep スケジューラージョブで実現できることの例は、`jobs/sweep_schedulers` 配下にある [wandb/launch-jobs](https://github.com/wandb/launch-jobs) リポジトリで確認できます。本ガイドでは、公開されている **Wandb Scheduler Job** の使い方を解説し、あわせてカスタム sweep スケジューラージョブを作成する手順も説明します。

<div id="how-to-resume-sweeps-on-launch">
  ## Launch で sweep を再開する方法
</div>

以前に起動した sweep から launch-sweep を再開することもできます。ハイパーパラメーターやトレーニングジョブは変更できませんが、スケジューラー固有のパラメーターや、送信先のキューは変更できます。

<Note>
  最初の sweep で &#39;latest&#39; のようなエイリアス付きのトレーニングジョブを使用していた場合、最後の run 以降に最新のジョブバージョンが変更されていると、再開時に結果が異なる可能性があります。
</Note>

1. 以前に実行した launch sweep の sweep 名/ID を特定します。sweep ID は 8 文字の文字列 (例: `hhd16935`) で、W&amp;B App のプロジェクト内で確認できます。
2. スケジューラー パラメーターを変更する場合は、更新した config ファイルを作成します。
3. ターミナルで次のコマンドを実行します。`<` と `>` で囲まれた部分を自分の情報に置き換えてください。

```bash
wandb launch-sweep <optional config.yaml> --resume_id <sweep id> --queue <queue_name>
```
