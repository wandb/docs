W&amp;B Launch を使用して、ハイパーパラメーターのチューニングジョブ ([sweeps](/ja/models/sweeps/)) を作成します。Launch で sweep を実行すると、探索対象のハイパーパラメーターを指定した sweep スケジューラーが Launch キュー に送信されます。sweep スケジューラーはエージェントに取得されると起動し、選択したハイパーパラメーターを使って同じキューに sweep run を起動します。これは、sweep が完了するか停止されるまで続きます。

デフォルトの W&amp;B Sweep スケジューリングエンジンを使用することも、独自のカスタムスケジューラーを実装することもできます。

1. 標準 sweep スケジューラー: [W&amp;B Sweeps](/ja/models/sweeps/) を制御するデフォルトの W&amp;B Sweep スケジューリングエンジンを使用します。使い慣れた `bayes`、`grid`、`random` メソッドを利用できます。
2. カスタム sweep スケジューラー: sweep スケジューラーをジョブとして実行するように設定します。このオプションでは、完全にカスタマイズできます。標準の sweep スケジューラーを拡張して、より詳細なログを出力する例は、以下のセクションにあります。

<Note>
  このガイドでは、W&amp;B Launch が事前に設定されていることを前提としています。W&amp;B Launch が設定されていない場合は、Launch ドキュメントの [使い始める方法](./#how-to-get-started) セクションを参照してください。
</Note>

<Note>
  Launch で sweep を初めて使用するユーザーには、&#39;basic&#39; メソッドで sweep を作成することをお勧めします。標準の W&amp;B スケジューリングエンジンで要件を満たせない場合は、カスタムの sweep on Launch スケジューラーを使用してください。
</Note>

<div id="create-a-sweep-with-a-wb-standard-scheduler">
  ## W&amp;B 標準スケジューラーで sweep を作成する
</div>

Launch を使用して W&amp;B Sweeps を作成できます。W&amp;B App で対話的に sweep を作成することも、W&amp;B CLI でプログラムから作成することもできます。スケジューラーをカスタマイズする機能を含む Launch sweeps の高度な設定を行う場合は、CLI を使用してください。

<Note>
  W&amp;B Launch で sweep を作成する前に、まず sweep の対象となるジョブを作成してください。詳細については、[Create a Job](/ja/platform/launch/create-launch-job/) ページを参照してください。
</Note>

<Tabs>
  <Tab title="W&B app">
    W&amp;B App を使って対話的に sweep を作成します。

    1. W&amp;B App で自分の W&amp;B プロジェクトにアクセスします。
    2. プロジェクトのサイドバーで Sweeps アイコン (ほうきの画像) を選択します。
    3. 次に、**Create Sweep** ボタンを選択します。
    4. **Configure Launch** ボタンをクリックします。
    5. **Job** ドロップダウンメニューから、sweep の作成元にするジョブ名とジョブ バージョンを選択します。
    6. **Queue** ドロップダウンメニューを使用して、sweep を実行するキューを選択します。
    7. **Job Priority** ドロップダウンを使用して、launch ジョブの優先度を指定します。launch キューが優先度設定をサポートしていない場合、launch ジョブの優先度は「Medium」に設定されます。
    8. (任意) run または sweep スケジューラーの override 引数を設定します。たとえば、スケジューラーの override を使用して、`num_workers` でスケジューラーが管理する同時実行 run 数を設定できます。
    9. (任意) **Destination Project** ドロップダウンメニューを使用して、sweep を保存するプロジェクトを選択します。
    10. **Save** をクリックします。
    11. **Launch Sweep** を選択します。

    <Frame>
      <img src="/images/launch/create_sweep_with_launch.png" alt="Launch sweep configuration" />
    </Frame>
  </Tab>

  <Tab title="CLI">
    W&amp;B CLI を使用して、Launch で W&amp;B Sweep をプログラムから作成します。

    1. sweep 設定を作成します。
    2. sweep 設定内で完全なジョブ名を指定します。
    3. sweep エージェントを初期化します。

    <Note>
      手順 1 と 3 は、通常 W&amp;B Sweep を作成する際と同じです。
    </Note>

    たとえば、次のコードスニペットでは、ジョブの値として `'wandb/jobs/Hello World 2:latest'` を指定しています。

    ```yaml
    # launch-sweep-config.yaml

    job: 'wandb/jobs/Hello World 2:latest'
    description: launch jobs を使用した sweep の例

    method: bayes
    metric:
      goal: minimize
      name: loss_metric
    parameters:
      learning_rate:
        max: 0.02
        min: 0
        distribution: uniform
      epochs:
        max: 20
        min: 0
        distribution: int_uniform

    # 任意のスケジューラー パラメーター:

    # scheduler:
    #   num_workers: 1  # 同時実行する sweep run 数
    #   docker_image: <scheduler 用のベースイメージ>
    #   resource: <例: local-container...>
    #   resource_args:  # run に渡される resource 引数
    #     env: 
    #         - WANDB_API_KEY

    # 任意の Launch パラメーター
    # launch: 
    #    registry: <イメージプルに使用する registry>
    ```

    sweep 設定の作成方法については、[Define sweep configuration](/ja/platform/launch/sweeps-on-launch/) ページを参照してください。

    4. 次に、sweep を初期化します。config ファイルへのパス、ジョブ キュー名、W&amp;B entity、プロジェクト名を指定します。

    ```bash
    wandb launch-sweep <path/to/yaml/file> --queue <queue_name> --entity <your_entity>  --project <project_name>
    ```

    W&amp;B Sweeps の詳細については、[Tune Hyperparameters](/ja/models/sweeps/) 章を参照してください。
  </Tab>
</Tabs>

<div id="create-a-custom-sweep-scheduler">
  ## カスタム sweep スケジューラーを作成する
</div>

W&amp;B スケジューラー または独自のスケジューラーを使用して、カスタム sweep スケジューラーを作成します。

<Note>
  スケジューラー ジョブを使用するには、wandb CLI バージョンが `0.15.4` 以上である必要があります。
</Note>

<Tabs>
  <Tab title="W&B スケジューラー">
    W&amp;B の sweep スケジューリングロジックをジョブとして使用し、Launch sweep を作成します。

    1. 公開されている wandb/sweep-jobs プロジェクトで wandb スケジューラージョブを特定するか、次のジョブ名を使用します。
       `'wandb/sweep-jobs/job-wandb-sweep-scheduler:latest'`
    2. この名前を指す `job` キーを含む `scheduler` ブロックを追加した設定 YAML を作成します。例を以下に示します。
    3. 新しい設定で `wandb launch-sweep` コマンドを実行します。

    設定例:

    ```yaml
    # launch-sweep-config.yaml  
    description: スケジューラージョブを使用した Launch sweep の設定
    scheduler:
      job: wandb/sweep-jobs/job-wandb-sweep-scheduler:latest
      num_workers: 8  # 8つの同時 sweep run を許可

    # sweep run が実行するトレーニング/チューニングジョブ
    job: wandb/sweep-jobs/job-fashion-MNIST-train:latest
    method: grid
    parameters:
      learning_rate:
        min: 0.0001
        max: 0.1
    ```
  </Tab>

  <Tab title="カスタム スケジューラー">
    カスタム スケジューラーは、scheduler ジョブを作成することで作成できます。このガイドでは、`WandbScheduler` を変更して、より多くのログを出力できるようにします。

    1. `wandb/launch-jobs` リポジトリ (具体的には `wandb/launch-jobs/jobs/sweep_schedulers`) をクローンします
    2. 次に、必要なログ出力を増やすために `wandb_scheduler.py` を変更します。例: 関数 `_poll` に logging を追加します。これは、新しい sweep run を起動する前に、各ポーリング サイクル (間隔は設定可能) ごとに 1 回呼び出されます。
    3. 次のコマンドで変更したファイルを実行し、ジョブを作成します: `python wandb_scheduler.py --project <project> --entity <entity> --name CustomWandbScheduler`
    4. 作成されたジョブ名を、UI または前の実行の出力で特定します。特に指定がない限り、これは code-artifact ジョブです。
    5. 次に、スケジューラーが新しいジョブを参照するように sweep 設定を作成します。

    ```yaml
    ...
    scheduler:
      job: '<entity>/<project>/job-CustomWandbScheduler:latest'
    ...
    ```
  </Tab>

  <Tab title="Optuna スケジューラー">
    Optuna は、さまざまなアルゴリズムを使用して、特定のモデルに最適なハイパーパラメーターを見つけるハイパーパラメーター最適化フレームワークです (W&amp;B と同様) 。[サンプリングアルゴリズム](https://optuna.readthedocs.io/en/stable/reference/samplers/index.html)に加えて、Optuna には、パフォーマンスの低い run を早期に終了するために使用できるさまざまな[プルーニングアルゴリズム](https://optuna.readthedocs.io/en/stable/reference/pruners.html)もあります。これは、多数の run を実行する場合に特に便利で、時間とリソースを節約できます。これらのクラスは高度に設定可能で、設定ファイルの `scheduler.settings.pruner/sampler.args` ブロックに必要なパラメーターを渡すだけです。

    ジョブを使用して、Optuna のスケジューリングロジックによる Launch sweep を作成します。

    1. まず、独自のジョブを作成するか、事前構築済みの Optuna スケジューラーイメージジョブを使用します。
       * 独自のジョブを作成する方法の例については、[`wandb/launch-jobs`](https://github.com/wandb/launch-jobs/blob/main/jobs/sweep_schedulers) リポジトリを参照してください。
       * 事前構築済みの Optuna イメージを使用するには、`wandb/sweep-jobs` プロジェクトの `job-optuna-sweep-scheduler` にアクセスするか、ジョブ名 `wandb/sweep-jobs/job-optuna-sweep-scheduler:latest` を使用します。

    2. ジョブを作成したら、sweep を作成できます。Optuna スケジューラージョブを指す `job` キーを含む `scheduler` ブロックを含めて、sweep 設定を作成します (以下の例を参照) 。

    ```yaml
      # optuna_config_basic.yaml
      description: A basic Optuna scheduler
      job: wandb/sweep-jobs/job-fashion-MNIST-train:latest
      run_cap: 5
      metric:
        name: epoch/val_loss
        goal: minimize

      scheduler:
        job: wandb/sweep-jobs/job-optuna-sweep-scheduler:latest
        resource: local-container  # イメージから取得されるスケジューラージョブに必須
        num_workers: 2

        # Optuna 固有の設定
        settings:
          pruner:
            type: PercentilePruner
            args:
              percentile: 25.0  # run の 75% を終了する
              n_warmup_steps: 10  # 最初の x step はプルーニングを無効にする

      parameters:
        learning_rate:
          min: 0.0001
          max: 0.1
    ```

    3. 最後に、`launch-sweep` コマンドを使用して、アクティブなキューに対して sweep を起動します。

    ```bash
    wandb launch-sweep <config.yaml> -q <queue> -p <project> -e <entity>
    ```

    Optuna の sweep スケジューラー ジョブの正確な実装については、[wandb/launch-jobs](https://github.com/wandb/launch-jobs/blob/main/jobs/sweep_schedulers/optuna_scheduler/optuna_scheduler.py) を参照してください。Optuna スケジューラーで可能なことのさらに多くの例については、[wandb/examples](https://github.com/wandb/examples/tree/master/examples/launch/launch-sweeps/optuna-scheduler) をご覧ください。
  </Tab>
</Tabs>

カスタム sweep スケジューラージョブで何ができるかの例は、[wandb/launch-jobs](https://github.com/wandb/launch-jobs) リポジトリの `jobs/sweep_schedulers` にあります。本ガイドでは、一般公開されている **Wandb Scheduler Job** の使い方を紹介するとともに、カスタム sweep スケジューラージョブを作成するプロセスも説明します。

<div id="how-to-resume-sweeps-on-launch">
  ## Launch で sweep を再開する方法
</div>

以前に実行した sweep から launch sweep を再開することもできます。ハイパーパラメーターとトレーニングジョブは変更できませんが、スケジューラー固有のパラメーターと、送信先のキューは変更できます。

<Note>
  最初の sweep で &#39;latest&#39; のようなエイリアスを持つトレーニングジョブを使用していた場合、前回の run 以降に最新のジョブバージョンが変更されていると、再開時に異なる結果になる可能性があります。
</Note>

1. 以前に実行した launch sweep の sweep 名/ID を確認します。sweep ID は 8 文字の文字列 (例: `hhd16935`) で、W&amp;B App のプロジェクト内で確認できます。
2. スケジューラーのパラメーターを変更する場合は、更新した config ファイルを作成します。
3. ターミナルで次のコマンドを実行します。`<` と `>` で囲まれた部分を自分の情報に置き換えてください。

```bash
wandb launch-sweep <optional config.yaml> --resume_id <sweep id> --queue <queue_name>
```
