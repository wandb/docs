---
title: Launch の用語と概念
description: "ジョブ、キュー、ターゲットリソース、エージェント、エージェント環境など、W&B Launch の主要な概念を学びます。"
---

W&amp;B Launch では、[ジョブ](#launch-job)を[キュー](#launch-queue)に追加して run を作成します。ジョブは、W&amp;B が組み込まれた Python スクリプトです。キューには、[ターゲットリソース](#target-resources)上で実行するジョブのリストが保持されます。[エージェント](#launch-agent)はキューからジョブを取得し、ターゲットリソース上でジョブを実行します。W&amp;B は、[Runs](/ja/models/runs/) をトラッキングするのと同様に Launch ジョブをトラッキングします。

<div id="launch-job">
  ### Launch ジョブ
</div>

Launch ジョブは、完了すべきタスクを表す特定の種類の [W&amp;B Artifact](/ja/models/artifacts/) です。たとえば、一般的な Launch ジョブには、モデルのトレーニングやモデル評価のトリガーなどがあります。ジョブ定義には次の内容が含まれます：

* 少なくとも 1 つの実行可能なエントリポイントを含む Python コードやその他のファイルアセット。
* 入力 (設定パラメーター) と出力 (記録されたメトリクス) に関する情報。
* 環境に関する情報 (たとえば、`requirements.txt`、ベースとなる `Dockerfile`) 。

ジョブ定義には主に 3 種類あります：

| Job types                           | Definition                                             | How to run this job type                                                  |
| ----------------------------------- | ------------------------------------------------------ | ------------------------------------------------------------------------- |
| Artifact-based (or code-based) jobs | コードやその他のアセットが W&amp;B アーティファクトとして保存されます。               | Artifact-based ジョブを実行するには、Launch エージェントを builder で構成する必要があります。            |
| Git-based jobs                      | コードやその他のアセットが、git リポジトリ内の特定のコミット、ブランチ、またはタグからクローンされます。 | Git-based ジョブを実行するには、Launch エージェントを builder と git リポジトリの認証情報で構成する必要があります。 |
| Image-based jobs                    | コードやその他のアセットが Docker イメージに組み込まれます。                     | Image-based ジョブを実行するには、Launch エージェントをイメージリポジトリの認証情報で構成する必要がある場合があります。     |

<Note>
  Launch ジョブは、モデル トレーニングに関連しない処理も実行できます。たとえば、モデルを Triton inference server にデプロイすることも可能です。ただし、すべてのジョブは正常に完了するために `wandb.init` を呼び出す必要があります。これにより、トラッキング用の run が W&amp;B Workspace に作成されます。
</Note>

作成したジョブは、W&amp;B App のプロジェクト ワークスペースにある `Jobs` タブで確認できます。そこからジョブを構成し、さまざまな [ターゲットリソース](#target-resources) で実行するために [Launch キュー](#launch-queue) に送信できます。

<div id="launch-queue">
  ### Launch キュー
</div>

Launch *キュー* は、特定のターゲットリソース上で実行するジョブを順序付きで保持するリストです。Launch キュー は先入れ先出し (FIFO) 方式です。作成できるキューの数に実質的な制限はありませんが、目安としてはターゲットリソースごとに 1 つのキューを用意することが推奨されます。ジョブは W&amp;B App UI、W&amp;B CLI、または Python SDK からキューに追加できます。その後、1 つ以上の Launch エージェントを設定してキューからアイテムを取得し、そのキューのターゲットリソース上で実行させることができます。

<div id="target-resources">
  ### ターゲットリソース
</div>

Launch キューがジョブを実行するように設定されている計算環境を、*ターゲットリソース* と呼びます。

W&amp;B Launch は、次のターゲットリソースをサポートしています：

* [Docker](/ja/platform/launch/setup-launch-docker/)
* [Kubernetes](/ja/platform/launch/setup-launch-kubernetes/)
* [AWS SageMaker](/ja/platform/launch/setup-launch-sagemaker/)
* [Google Cloud Vertex AI](/ja/platform/launch/setup-vertex/)

各ターゲットリソースは、*リソース設定* と呼ばれる異なる設定パラメーターのセットを受け入れます。リソース設定には各 Launch キューで定義されたデフォルト値が適用されますが、各ジョブごとに個別に上書きできます。詳細については、各ターゲットリソースのドキュメントを参照してください。

<div id="launch-agent">
  ### Launch エージェント
</div>

Launch エージェントは軽量な常駐プログラムで、定期的に Launch キューをポーリングして、実行するジョブがあるかどうかを確認します。Launch エージェントがジョブを受け取ると、まずジョブ定義に基づいてイメージをビルドまたはプルし、そのイメージをターゲットリソース上で実行します。

1 つのエージェントで複数のキューをポーリングできますが、その場合は、ポーリング対象の各キューのバックエンドとなるすべてのターゲットリソースをサポートできるように、エージェントを適切に設定する必要があります。

<div id="launch-agent-environment">
  ### Launch エージェント環境
</div>

エージェント環境とは、Launch エージェントが稼働し、ジョブをポーリングする環境のことです。

<Note>
  エージェントのランタイム環境は、キューのターゲットリソースとは独立しています。言い換えると、必須のターゲットリソースにアクセスできるよう十分に設定されていれば、エージェントはどこにでもデプロイできます。
</Note>
