title: Launch の用語と概念
description: "ジョブ、キュー、ターゲットリソース、エージェント、エージェント環境など、W&B Launch の主要な概念を学びます。"
---

W&amp;B Launch では、run を作成するために [ジョブ](#launch-job) を [キュー](#launch-queue) に追加します。ジョブは W&amp;B 用にインスツルメントされた Python スクリプトです。キューは、[ターゲットリソース](#target-resources) 上で実行するジョブの一覧を保持します。[エージェント](#launch-agent) はキューからジョブを取得し、ターゲットリソース上でジョブを実行します。W&amp;B は Launch ジョブを、W&amp;B が [runs](/ja/models/runs/) を追跡する場合と同様の方法で追跡します。

<div id="launch-job">
  ### Launch ジョブ
</div>

Launch ジョブは、完了すべきタスクを表す特定の種類の [W&amp;B Artifact](/ja/models/artifacts/) です。たとえば、一般的な Launch ジョブには、モデル のトレーニングやモデル 評価のトリガーなどがあります。ジョブ定義には次の内容が含まれます：

* 少なくとも 1 つの実行可能なエントリポイントを含む Python コードおよびその他のファイルアセット
* 入力 (設定パラメーター) と出力 (記録されるメトリクス) に関する情報
* 環境に関する情報 (例：`requirements.txt`、ベースとなる `Dockerfile`) 

ジョブ定義には主に 3 種類あります：

| Job types | Definition | How to run this job type |
| ---------- | --------- | -------------- |
|Artifact-based (or code-based) jobs| コードおよびその他のアセットが W&amp;B Artifacts として保存されます。| Artifact-based ジョブを実行するには、Launch agent を builder で構成する必要があります。 |
|Git-based jobs| コードおよびその他のアセットが、git リポジトリ内の特定のコミット、ブランチ、またはタグからクローンされます。 | Git-based ジョブを実行するには、Launch agent を builder および git リポジトリのクレデンシャルで構成する必要があります。 |
|Image-based jobs| コードおよびその他のアセットが Docker イメージに組み込まれます。 | Image-based ジョブを実行するには、Launch agent をイメージリポジトリのクレデンシャルで構成する必要がある場合があります。 |

<Note>
  Launch ジョブは、モデル トレーニングと無関係な処理も実行できます。たとえば、モデル を Triton inference server にデプロイするといったことも可能です。ただし、すべてのジョブは正常に完了するために `wandb.init` を呼び出す必要があります。これは、W&amp;B ワークスペース内でトラッキング用の run を作成します。
</Note>

作成したジョブは、W&amp;B App のプロジェクト ワークスペース内にある `Jobs` タブで確認できます。そこからジョブを構成し、さまざまな [ターゲットリソース](#target-resources) 上で実行するために [launch キュー](#launch-queue) へ送信できます。

<div id="launch-queue">
  ### Launch queue
</div>

Launch *queues* は、特定のターゲットリソース上で実行するジョブを順序付きで保持するリストです。Launch queues は先入れ先出し (FIFO) 方式です。作成できる queue の数に実質的な制限はありませんが、目安としてはターゲットリソースごとに 1 つの queue を用意することが推奨されます。ジョブは W&amp;B App UI、W&amp;B CLI、または Python SDK から queue に追加できます。その後、1 つ以上の Launch エージェントを設定して queue からアイテムを取得し、その queue のターゲットリソース上で実行させることができます。

<div id="target-resources">
  ### ターゲットリソース
</div>

Launch キューがジョブを実行するように設定されている計算環境を、*ターゲットリソース* と呼びます。

W&amp;B Launch は、次のターゲットリソースをサポートしています。

* [Docker](/ja/platform/launch/setup-launch-docker/)
* [Kubernetes](/ja/platform/launch/setup-launch-kubernetes/)
* [AWS SageMaker](/ja/platform/launch/setup-launch-sagemaker/)
* [Google Cloud Vertex AI](/ja/platform/launch/setup-vertex/)

各ターゲットリソースでは、*リソース設定* と呼ばれる、それぞれ異なる一連の設定パラメーターを使用します。リソース設定には各 Launch キューによってデフォルト値が定義されていますが、各ジョブごとに個別に上書きできます。詳細については、各ターゲットリソースのドキュメントを参照してください。

<div id="launch-agent">
  ### Launch エージェント
</div>

Launch エージェントは軽量で常駐するプログラムで、定期的に Launch キューをポーリングし、実行すべきジョブがないか確認します。Launch エージェントがジョブを受け取ると、まずジョブ定義からイメージをビルドまたはプルし、そのイメージをターゲットリソース上で実行します。

1 つのエージェントが複数のキューをポーリングすることもできますが、その場合は、ポーリング対象の各キューを支えるすべてのターゲットリソースに対応できるように、エージェントを適切に構成する必要があります。

<div id="launch-agent-environment">
  ### Launch エージェント環境
</div>

エージェント環境とは、Launch エージェントが稼働し、ジョブをポーリングしている環境のことです。

<Note>
  エージェントの実行環境は、キューのターゲットリソースとは独立しています。言い換えると、必要なターゲットリソースへアクセスできるように適切に構成されていれば、エージェントはどこにでもデプロイできます。
</Note>
