---
title: Launch の用語と概念
---

W&amp;B Launch を使用すると、[ジョブ](#launch-job)を [キュー](#launch-queue) に投入して run を作成できます。ジョブは、W&amp;B でインストルメントされた Python スクリプトです。キューは、[ターゲットリソース](#target-resources) 上で実行するジョブのリストを保持します。[エージェント](#launch-agent)はキューからジョブを取得し、ターゲットリソース上でジョブを実行します。W&amp;B は、[runs](/ja/models/runs/) を追跡するのと同様の方法で Launch ジョブを追跡します。

<div id="launch-job">
  ### Launch ジョブ
</div>

Launch ジョブは、完了すべきタスクを表す特定の種類の [W&amp;B Artifact](/ja/models/artifacts/) です。たとえば、一般的な Launch ジョブには、モデルの学習やモデル評価のトリガーなどがあります。ジョブ定義には次の情報が含まれます:

* 少なくとも 1 つの実行可能なエントリポイントを含む Python コードおよびその他のファイルアセット
* 入力（config パラメータ）と出力（記録されるメトリクス）に関する情報
* 環境に関する情報（例: `requirements.txt`、ベースとなる `Dockerfile`）

ジョブ定義には主に 3 種類があります:

| Job types | Definition | How to run this job type |
| ---------- | --------- | -------------- |
|Artifact-based (or code-based) jobs| コードおよびその他のアセットが W&amp;B アーティファクトとして保存されます。| Artifact-based ジョブを実行するには、Launch agent にビルダーを設定する必要があります。 |
|Git-based jobs| コードおよびその他のアセットが、Git リポジトリ内の特定のコミット、ブランチ、またはタグからクローンされます。 | Git-based ジョブを実行するには、Launch agent にビルダーおよび Git リポジトリの認証情報を設定する必要があります。 |
|Image-based jobs| コードおよびその他のアセットが Docker イメージに焼き込まれます。 | Image-based ジョブを実行するには、必要に応じて Launch agent にイメージリポジトリの認証情報を設定する必要があります。 |

<Note>
  Launch ジョブは、モデル学習に直接関連しない処理（たとえば、Triton inference サーバーへのモデルのデプロイ）も実行できますが、すべてのジョブは正常に完了するために `wandb.init` を呼び出す必要があります。これにより、W&amp;B Workspace 内でトラッキング用の run が作成されます。
</Note>

作成したジョブは、W&amp;B App のプロジェクト Workspace の `Jobs` タブで確認できます。そこからジョブを設定し、さまざまな [ターゲットリソース](#target-resources) 上で実行されるように [Launch キュー](#launch-queue) に送信できます。

<div id="launch-queue">
  ### Launch キュー
</div>

Launch の *キュー* は、特定のターゲットリソース上で実行するジョブを順序付きに並べたリストです。Launch キューは先入れ先出し（FIFO）で処理されます。作成できるキューの数に実質的な上限はありませんが、目安としてはターゲットリソースごとに 1 つのキューを用意することを推奨します。ジョブは W&amp;B App の UI、W&amp;B CLI、または Python SDK からキューに投入できます。その後、1 つ以上の Launch エージェントを設定して、キューからアイテムを取得し、そのキューのターゲットリソース上でジョブを実行させることができます。

<div id="target-resources">
  ### 対象リソース
</div>

Launch キューがジョブを実行するように設定されている計算環境を、*対象リソース* と呼びます。

W&amp;B Launch は、次の対象リソースをサポートします。

* [Docker](/ja/platform/launch/setup-launch-docker/)
* [Kubernetes](/ja/platform/launch/setup-launch-kubernetes/)
* [AWS SageMaker](/ja/platform/launch/setup-launch-sagemaker/)
* [Google Cloud Vertex AI](/ja/platform/launch/setup-vertex/)

各対象リソースは、*リソース設定* と呼ばれる固有の設定パラメータセットを使用します。リソース設定には各 Launch キューで定義されたデフォルト値が適用されますが、ジョブごとに個別に上書きできます。詳細については、各対象リソースのドキュメントを参照してください。

<div id="launch-agent">
  ### Launch エージェント
</div>

Launch エージェントは軽量な常駐プログラムで、定期的に Launch キューを確認し、実行すべきジョブがあるかどうかをチェックします。Launch エージェントがジョブを受け取ると、まずジョブ定義からイメージをビルドまたはプルし、その後ターゲットリソース上で実行します。

1 つのエージェントで複数のキューをポーリングすることもできますが、そのエージェントがポーリングしている各キューの裏側で利用するすべてのターゲットリソースをサポートできるよう、適切に設定されている必要があります。

<div id="launch-agent-environment">
  ### Launch エージェント環境
</div>

エージェント環境とは、Launch エージェントが動作し、ジョブをポーリングしている環境のことです。

<Note>
  エージェントの実行環境は、キューのターゲットリソースとは独立しています。言い換えると、必要なターゲットリソースにアクセスできるよう適切に構成されていれば、エージェントはどこにでもデプロイできます。
</Note>
