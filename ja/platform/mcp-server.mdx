Model Context Protocol (MCP) は、LLM エージェントがトークンのコストを最小限に抑えながら効率的にデータをクエリして解析できるようにします。このページでは、W&amp;B MCP server を使って IDE や MCP クライアントから W&amp;B データをクエリおよび解析する方法と、クライアントに W&amp;B のドキュメントへのプログラムによるアクセスを与えて、W&amp;B 関連のクエリに対してより正確な応答を生成できるようにする方法を説明します。

W&amp;B MCP server は、次のような多くの IDE、コーディングクライアント、およびチャットエージェントとネイティブに統合されています。

* Cursor
* Visual Studio Code (VS Code)
* Claude Code
* Codex
* Gemini CLI
* Mistral LeChat
* Claude Desktop

W&amp;B MCP server は、[ホスト型](#use-w\&b’s-remote-mcp-server) と [ローカル](#set-up-a-local-version-of-the-w\&b-mcp-server) の 2 つのバリエーションをサポートします。ホスト型バージョンは [W&amp;B Dedicated Cloud deployments](/ja/platform/hosting/hosting-options/dedicated_cloud) のみをサポートします。ローカルバージョンは Dedicated Cloud と [Self-Managed deployments](/ja/platform/hosting/hosting-options/self-managed) の両方をサポートします。

<div id="wb-mcp-server-capabilities">
  ## W&amp;B MCP サーバーの機能
</div>

MCP サーバーを使用して experiment を分析したり、trace をデバッグしたり、レポートを作成したり、アプリケーションを W&amp;B の機能と統合する際のサポートを受けたりできます。

次のプロンプト例は、エージェントが MCP サーバーに接続されているときに実行できるタスクの種類を示しています。

* your-team-name/your-project-name の run を、eval/accuracy が高い順に上位 5 件表示して。
* hiring agent の predict traces のレイテンシーは、ここ数か月でどのように変化しましたか？
* hiring agent が先月行った判断を比較する wandb レポートを生成して。
* Weave で leaderboard を作成するにはどうすればよいですか — SupportBot に聞いて。

<div id="available-tools">
  ### 利用可能なツール
</div>

W&amp;B MCP サーバーは、エージェントが次のツールにアクセスできるようにします。

| Tool | Description | Example Query |
|------|-------------|---------------|
| **query&#95;wandb&#95;tool** | W&amp;B の run、メトリクス、実験をクエリする | *&quot;loss が 0.1 未満の run を表示して&quot;* |
| **query&#95;weave&#95;traces&#95;tool** | LLM のトレースと評価を分析する | *&quot;平均レイテンシーはどれくらい？&quot;* |
| **count&#95;weave&#95;traces&#95;tool** | トレースを数え、ストレージメトリクスを取得する | *&quot;失敗したトレースはいくつある？&quot;* |
| **create&#95;wandb&#95;report&#95;tool** | W&amp;B Reports をプログラムから作成する | *&quot;パフォーマンスのレポートを作成して&quot;* |
| **query&#95;wandb&#95;entity&#95;projects** | Entity のプロジェクトを一覧表示する | *&quot;どんなプロジェクトが存在する？&quot;* |
| **query&#95;wandb&#95;support&#95;bot** | W&amp;B のドキュメントに基づいてヘルプを取得する | *&quot;Sweeps をどうやって使う？&quot;* |

<div id="use-wbs-remote-mcp-server">
  ## W&amp;B のリモート MCP サーバーを利用する
</div>

W&amp;B は、インストール不要で利用できるホスト提供の MCP サーバー `https://mcp.withwandb.com` を提供しています。以下の手順では、このホスト提供サーバーを各種 AI アシスタントや IDE で利用するための設定方法を説明します。

<div id="prerequisites">
  ### 前提条件
</div>

* W&amp;B Dedicated Cloud のデプロイ。
* W&amp;B API キー。新しいキーは [wandb.ai/authorize](https://wandb.ai/authorize) で作成できます。
* キーを `WANDB_API_KEY` という名前の環境変数として設定します。

<div id="configure-your-mcp-client">
  ### MCP クライアントを設定する
</div>

お使いの MCP クライアントの手順が記載されたタブを選択してください。

<Tabs>
  <Tab title="Cursor">
    [ワンクリック インストール リンク](cursor://anysphere.cursor-deeplink/mcp/install?name=wandb\&config=eyJ0cmFuc3BvcnQiOiJodHRwIiwidXJsIjoiaHR0cHM6Ly9tY3Aud2l0aHdhbmRiLmNvbS9tY3AiLCJoZWFkZXJzIjp7IkF1dGhvcml6YXRpb24iOiJCZWFyZXIgWU9VUl9XQU5EQl9BUElfS0VZIiwiQWNjZXB0IjoiYXBwbGljYXRpb24vanNvbiwgdGV4dC9ldmVudC1zdHJlYW0ifX0=)（`Authorization` フィールドに `Bearer <your-wandb-api-key>` を追加する必要があります）を使用して Cursor に W&amp;B サーバーを自動的にインストールするか、次の手順に従って手動でインストールできます。

    1. macOS の場合は **Cursor** メニューを開き、**Settings** を選択してから **Cursor Settings** を選択します。Windows または Linux の場合は **Preferences** メニューを開き、**Settings** を選択してから **Cursor Settings** を選択します。
    2. Cursor Settings メニューで **Tools and MCP** を選択します。これにより Tools メニューが開きます。
    3. Installed MCP Servers セクションで **Add Custom MCP** を選択します。これにより `mcp.json` 設定ファイルが開きます。
    4. 設定ファイル内の `mcpServers` JSON オブジェクトに、次の `wandb` オブジェクトを追加します。

    ```json
    {
      "mcpServers": {
        "wandb": {
          "transport": "http",
          "url": "https://mcp.withwandb.com/mcp",
          "headers": {
            "Authorization": "Bearer <your-wandb-api-key>",
            "Accept": "application/json, text/event-stream"
          }
        }
      }
    }
    ```

    5. Cursor を再起動して、変更を有効にします。
    6. チャット エージェントが W&amp;B MCP サーバーにアクセスできることを確認するため、プロンプトとして &quot;List the projects in my W&amp;B account.&quot; と入力します。

    詳細については、[Cursor のドキュメント](https://cursor.com/docs/context/mcp) を参照してください。
  </Tab>

  <Tab title="Claude Code">
    Claude Code に W&amp;B MCP サーバーを追加するには、次のコマンドの `Authorization` ヘッダーを自分の W&amp;B API キーに更新し、ターミナルで実行します。

    ```bash
    claude mcp add --transport http wandb https://mcp.withwandb.com/mcp \
      --header "Authorization: Bearer <your-wandb-api-key>"
    ```

    グローバルな設定にするには `--scope user` を追加し、現在のプロジェクトのみに設定する場合はこれを省略します。

    より詳細な情報については、[Claude Code のドキュメント](https://docs.anthropic.com/en/docs/claude-code/mcp)を参照してください。
  </Tab>

  <Tab title="Codex">
    Codex に W&amp;B MCP サーバーを追加するには、次のコマンドの `--bearer-token-env-var` 引数を、W&amp;B API キー を格納した環境変数の名前に更新し、ターミナルで実行します。

    ```bash
    export WANDB_API_KEY=<your-wandb-api-key>
    codex mcp add wandb --url https://mcp.withwandb.com/mcp --bearer-token-env-var <your-wandb-api-key-environment-variable>
    ```
  </Tab>

  <Tab title="OpenAI">
    OpenAI 呼び出しに W&amp;B MCP サーバーを追加するには、OpenAI のレスポンス設定の `tools` フィールドにサーバー情報を追加します。

    ```python
    from openai import OpenAI
    import os

    client = OpenAI()

    resp = client.responses.create(
        model="gpt-4o",
        tools=[{
            "type": "mcp",
            "server_label": "wandb",
            "server_description": "Query W&B data",
            "server_url": "https://mcp.withwandb.com/mcp",
            "authorization": os.getenv("<your-wandb-api-key>"),
            "require_approval": "never",
        }],
        input="List the projects in my W&B account.",
    )

    print(resp.output_text)
    ```
  </Tab>

  <Tab title="Gemini CLI">
    Gemini CLI に W&amp;B MCP サーバーを追加するには：

    1. 次の 1 行のコマンドで W&amp;B MCP 拡張機能をインストールします：

       ```bash
       # 拡張機能をインストール
       gemini extensions install https://github.com/wandb/wandb-mcp-server
       ```
    2. インストールが完了したら、Gemini CLI を再起動します。
    3. チャットエージェントが W&amp;B MCP サーバーにアクセスできることを確認するには、プロンプト「List the projects in my W&amp;B account.」を入力します。

    詳細については、[Gemini のドキュメント](https://geminicli.com/docs/tools/mcp-server/)を参照してください。
  </Tab>

  <Tab title="Mistral LeChat">
    W&amp;B MCP サーバーを Mistral LeChat に追加するには、次の手順に従います。

    1. **Intelligence** メニューから **Add Connector** を選択して、Connector ウィンドウを開きます。

    2. **Custom MCP Connector** タブを選択します。

    3. 次の値を使用して各フィールドを設定します。

       * **Connector Server**: `https://mcp.withwandb.com/mcp`
       * **Description**: （オプション）接続に関する任意の簡潔な説明。
       * **Authentication Method**: **API Token Authentication** を選択します。追加のフィールドが表示されます。
       * **Header name**: 既定値の **Authorization** のままにします。
       * **Header type**: **Bearer** を選択します。
       * **Header value**: 自分の W&amp;B API トークンを入力します。

    4. すべてのフィールドを設定したら、**Create** を選択します。LeChat は MCP サーバーを設定に追加します。

    5. チャット エージェントが W&amp;B MCP サーバーにアクセスできることを、プロンプト「List the projects in my W&amp;B account.」を入力して確認します。

    詳細については、[LeChat のドキュメント](https://mistral.ai/news/le-chat-mcp-connectors-memories)を参照してください。
  </Tab>
</Tabs>

<div id="set-up-a-local-version-of-the-wb-mcp-server">
  ## ローカル版の W&amp;B MCP サーバーをセットアップする
</div>

W&amp;B Self-Managed 環境でのデプロイ、開発、テスト、あるいはエアギャップ環境で MCP サーバーをローカルで実行する必要がある場合は、自身のマシンにインストールして実行できます。

<div id="prerequisites">
  ### 前提条件
</div>

* W&amp;B の API キー。新しいキーは [wandb.ai/authorize](https://wandb.ai/authorize) で作成できます。
* キーを `WANDB_API_KEY` という名前の環境変数として設定します。
* [W&amp;B Self-Managed](/ja/platform/hosting/hosting-options/self-managed) を使用している場合は、環境変数 `WANDB_BASE_URL` を設定します。
* Python 3.10 以上
* [uv](https://docs.astral.sh/uv/)（推奨）または pip

インストール方法については、uv のドキュメントの [installation instructions](https://docs.astral.sh/uv/getting-started/installation/) を参照してください。

<div id="install-and-configure-the-mcp-server">
  ### MCP サーバーのインストールと設定
</div>

MCP サーバーをローカルにインストールするには:

ローカルマシンに W&amp;B MCP サーバーをインストールするには、次のいずれかのインストールコマンドを使用します。

<Tabs>
  <Tab title="uv">
    ```bash
    uv install wandb-mcp-server
    ```
  </Tab>

  <Tab title="pip">
    ```bash
    pip install wandb-mcp-server
    ```
  </Tab>

  <Tab title="GitHub から直接インストール">
    ```bash
    pip install git+https://github.com/wandb/wandb-mcp-server
    ```
  </Tab>
</Tabs>

MCP サーバーをローカルにインストールしたら、それを使用するよう MCP クライアントを設定します。以下から MCP クライアントを選択して続行します。

<Tabs>
  <Tab title="Cursor">
    `mcp.json` 設定に次の内容を追加します。

    ```json
    {
      "mcpServers": {
        "wandb": {
          "command": "uvx",
          "args": [
            "--from",
            "git+https://github.com/wandb/wandb-mcp-server",
            "wandb_mcp_server"
          ],
          "env": {
            "WANDB_API_KEY": "<your-wandb-api-key>",
            "WANDB_BASE_URL": "https://your-wandb-instance.example.com"
          }
        }
      }
    }
    ```
  </Tab>

  <Tab title="VS Code">
    `.vscode/mcp.json` またはグローバル MCP 設定に次の内容を追加します。

    ```json
    {
      "servers": {
        "wandb": {
          "command": "uvx",
          "args": [
            "--from",
            "git+https://github.com/wandb/wandb-mcp-server",
            "wandb_mcp_server"
          ],
          "env": {
            "WANDB_API_KEY": "<your-wandb-api-key>",
            "WANDB_BASE_URL": "https://your-wandb-instance.example.com"
          }
        }
      }
    }
    ```
  </Tab>

  <Tab title="Claude Code">
    ターミナルで次のコマンドを実行します。グローバル設定にするには `--scope user` を追加し、現在のプロジェクトのみに設定する場合は省略します。

    ```bash
    claude mcp add wandb \
      -e WANDB_API_KEY=your-api-key \
      -e WANDB_BASE_URL=https://your-wandb-instance.example.com \
      -- uvx --from git+https://github.com/wandb/wandb-mcp-server wandb_mcp_server
    ```
  </Tab>

  <Tab title="Codex">
    ターミナルで次のコマンドを実行します。

    ```bash
    codex mcp add wandb \
      --env WANDB_API_KEY=your_api_key_here \
      --env WANDB_BASE_URL=https://your-wandb-instance.example.com \
      -- uvx --from git+https://github.com/wandb/wandb-mcp-server wandb_mcp_server
    ```
  </Tab>

  <Tab title="Claude Desktop">
    テキストエディタで Claude 設定ファイルを開きます。設定ファイルは OS ごとに次の場所にあります。

    * **macOS**: ~/Library/Application\ Support/Claude/claude&#95;desktop&#95;config.json
    * **Windows**: %APPDATA%\Claude\claude&#95;desktop&#95;config.json

    Claude 設定ファイルの JSON オブジェクトに次の内容を追加します。Claude Desktop が `uvx` のインストールを検出できない場合があるため、`uvx` のフルパスを使用してください。

    ```json
    {
      "mcpServers": {
        "wandb": {
          "command": "/full/path/to/uvx",
          "args": [
            "--from",
            "git+https://github.com/wandb/wandb-mcp-server",
            "wandb_mcp_server"
          ],
          "env": {
            "WANDB_API_KEY": "<your-wandb-api-key>",
            "WANDB_BASE_URL": "https://your-wandb-instance.example.com"
          }
        }
      }
    }
    ```

    新しい設定を有効にするには、Claude Desktop を再起動します。
  </Tab>
</Tabs>

Web ベースのクライアントやテスト用途の場合は、HTTP トランスポートを使ってサーバーを実行します。

```bash
uvx wandb_mcp_server --transport http --host 0.0.0.0 --port 8080
```

ローカルサーバーを OpenAI などの外部クライアントから利用できるように公開するには、ngrok を使用します。

```bash
uvx wandb_mcp_server --transport http --port 8080

# 別のターミナルで、ngrok を使って公開する
ngrok http 8080
```

サーバーを `ngrok` で公開する場合は、`ngrok` の URL を使用するように MCP クライアントの設定を更新してください。

<div id="usage-tips">
  ## 使用上のヒント
</div>

* **W&amp;B の project と entity 名を指定する**: 正確な結果を得るために、クエリ内で W&amp;B entity と project を指定してください。
* **漠然としすぎた質問は避ける**: 「私の一番良い評価は何ですか？」ではなく、「どの評価で F1 スコアが最も高かったですか？」のように尋ねてください。
* **データ取得を確認する**: 「最も性能の良い Runs はどれですか？」のような大まかな質問をする場合は、利用可能なすべての Runs を取得できたかどうかをアシスタントに確認するよう依頼してください。
