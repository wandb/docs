Model Context Protocol (MCP) を使用すると、LLM エージェントはトークンコストを最小限に抑えながら、効率的にデータをクエリして分析できます。このページでは、W&amp;B MCP サーバーを使って IDE や MCP クライアントから W&amp;B データをクエリして分析する方法と、クライアントに W&amp;B のドキュメントへのプログラムによるアクセスを提供し、W&amp;B 関連のクエリに対してより正確な応答を生成できるようにする方法を説明します。

これは、ほとんどの IDE、コーディングクライアント、チャットエージェントとネイティブに統合されており、次が含まれます。

* Cursor
* Visual Studio Code (VS Code)
* Claude Code
* Codex
* Gemini CLI
* Mistral LeChat
* Claude Desktop

W&amp;B MCP サーバーは、[ホスト型](#use-w\&b’s-remote-mcp-server) と [ローカル](#set-up-a-local-version-of-the-w\&b-mcp-server) のバリアントをサポートします。ホスト型バージョンは、[W&amp;B 専用クラウドのデプロイ](/ja/platform/hosting/hosting-options/dedicated-cloud) のみをサポートします。ローカルバージョンは、専用クラウドと [セルフマネージドのデプロイ](/ja/platform/hosting/hosting-options/self-managed) の両方をサポートします。

<div id="wb-mcp-server-capabilities">
  ## W&amp;B MCP Server の機能
</div>

MCP サーバーを使用して Experiments を分析したり、トレースをデバッグしたり、レポートを作成したり、アプリケーションを W&amp;B の機能と統合する際のサポートを受けたりできます。

次のプロンプト例は、エージェントが MCP サーバーに接続されているときに実行できるタスクの例を示しています。

* your-team-name/your-project-name の Runs を、eval/accuracy が高い順に上位 5 件表示して。
* 私の hiring エージェントの predict トレースのレイテンシーは、ここ数か月でどのように変化しましたか？
* hiring エージェントが先月行った判断を比較する wandb レポートを生成して。
* Weave で リーダーボード を作成するにはどうすればよいですか — SupportBot に聞いて。

<div id="available-tools">
  ### 利用可能なツール
</div>

W&amp;B MCP サーバーを使用すると、エージェントは次のツールにアクセスできます。

| ツール                                         | 説明                                    | クエリ例                                    |
| ------------------------------------------- | ------------------------------------- | --------------------------------------- |
| **query&#95;wandb&#95;tool**                | W&amp;B Runs、メトリクス、Experiments をクエリする | *&quot;loss が 0.1 未満の run を表示して&quot;*  |
| **query&#95;weave&#95;traces&#95;tool**     | LLM のトレースと評価を分析する                     | *&quot;平均レイテンシーはどれくらいですか？&quot;*        |
| **count&#95;weave&#95;traces&#95;tool**     | トレースをカウントし、ストレージ メトリクスを取得する           | *&quot;失敗したトレースはいくつありますか？&quot;*        |
| **create&#95;wandb&#95;report&#95;tool**    | W&amp;B レポートをプログラムで作成する               | *&quot;パフォーマンス レポートを作成して&quot;*         |
| **query&#95;wandb&#95;entity&#95;projects** | entity のプロジェクトを一覧表示する                 | *&quot;どのようなプロジェクトがありますか？&quot;*        |
| **query&#95;wandb&#95;support&#95;bot**     | W&amp;B のドキュメントからヘルプを取得する             | *&quot;Sweeps を使用するにはどうすればよいですか？&quot;* |

<div id="use-wbs-remote-mcp-server">
  ## W&amp;B のリモート MCP サーバーを使用する
</div>

W&amp;B は、インストール不要で使用できるホスト型の MCP サーバー `https://mcp.withwandb.com` を提供しています。以下の手順では、このホスト型サーバーを各種 AI アシスタントや IDE で使用するための設定方法を説明します。

<div id="prerequisites">
  ### 前提条件
</div>

* W&amp;B 専用クラウドのデプロイ。
* W&amp;B APIキー。[wandb.ai/authorize](https://wandb.ai/authorize) で新しいキーを生成できます。
* キーを `WANDB_API_KEY` という名の環境変数に設定してください。

<div id="configure-your-mcp-client">
  ### MCP クライアントを設定する
</div>

お使いの MCP クライアントの手順が記載されたタブを選択してください。

<Tabs>
  <Tab title="Cursor">
    [ワンクリック インストール リンク](cursor://anysphere.cursor-deeplink/mcp/install?name=wandb\&config=eyJ0cmFuc3BvcnQiOiJodHRwIiwidXJsIjoiaHR0cHM6Ly9tY3Aud2l0aHdhbmRiLmNvbS9tY3AiLCJoZWFkZXJzIjp7IkF1dGhvcml6YXRpb24iOiJCZWFyZXIgWU9VUl9XQU5EQl9BUElfS0VZIiwiQWNjZXB0IjoiYXBwbGljYXRpb24vanNvbiwgdGV4dC9ldmVudC1zdHJlYW0ifX0=)を使用すると、Cursor に W&amp;B server を自動的にインストールできます (`Authorization` フィールドに `Bearer <your-wandb-api-key>` を追加する必要があります) 。または、以下の手順で手動でインストールすることもできます。

    1. macOS では、**Cursor** メニューを開き、**Settings** を選択してから **Cursor Settings** を選択します。Windows または Linux では、**Preferences** メニューを開き、**Settings** を選択してから **Cursor Settings** を選択します。
    2. Cursor Settings メニューから **Tools and MCP** を選択します。これにより Tools メニューが開きます。
    3. Installed MCP Servers セクションで **Add Custom MCP** を選択します。これにより `mcp.json` 設定ファイルが開きます。
    4. 設定ファイルの `mcpServers` JSON オブジェクトに、次の `wandb` オブジェクトを追加します。

    ```json
    {
      "mcpServers": {
        "wandb": {
          "transport": "http",
          "url": "https://mcp.withwandb.com/mcp",
          "headers": {
            "Authorization": "Bearer <your-wandb-api-key>",
            "Accept": "application/json, text/event-stream"
          }
        }
      }
    }
    ```

    5. 変更を反映するために、Cursor を再起動します。
    6. プロンプト &quot;私の W&amp;B アカウント内のプロジェクトを一覧表示してください。&quot; を入力して、チャット エージェントが W&amp;B MCP server に 액세스できることを確認します。

    詳細については、[Cursor&#39;s documentation](https://cursor.com/docs/context/mcp)を参照してください。
  </Tab>

  <Tab title="Claude Code">
    Claude Code に W&amp;B MCP server を追加するには、次のコマンドの `Authorization` ヘッダーをお使いの W&amp;B APIキーで更新し、ターミナルで実行してください。

    ```bash
    claude mcp add --transport http wandb https://mcp.withwandb.com/mcp \
      --header "Authorization: Bearer <your-wandb-api-key>"
    ```

    グローバル設定の場合は `--scope user` を追加するか、現在のプロジェクトのみを設定するにはこれを省略します。

    詳細については、[Claude Code のドキュメント](https://docs.anthropic.com/en/docs/claude-code/mcp)を参照してください。
  </Tab>

  <Tab title="Codex">
    Codex に W&amp;B MCP server を追加するには、次のコマンドの `--bearer-token-env-var` 引数を、W&amp;B APIキー を含む環境変数に合わせて更新し、その後ターミナルで実行します:

    ```bash
    export WANDB_API_KEY=<your-wandb-api-key>
    codex mcp add wandb --url https://mcp.withwandb.com/mcp --bearer-token-env-var <your-wandb-api-key-environment-variable>
    ```
  </Tab>

  <Tab title="OpenAI">
    OpenAI CallにW&amp;B MCP serverを追加するには、サーバーの情報をOpenAIの応答設定の `tools` フィールドに追加します。

    ```python
    from openai import OpenAI
    import os

    client = OpenAI()

    resp = client.responses.create(
        model="gpt-4o",
        tools=[{
            "type": "mcp",
            "server_label": "wandb",
            "server_description": "Query W&B data",
            "server_url": "https://mcp.withwandb.com/mcp",
            "authorization": os.getenv("<your-wandb-api-key>"),
            "require_approval": "never",
        }],
        input="List the projects in my W&B account.",
    )

    print(resp.output_text)
    ```
  </Tab>

  <Tab title="Gemini CLI">
    W&amp;B MCP server を Gemini CLI に追加するには:

    1. 1 つのコマンドで W&amp;B MCP 拡張機能をインストールします:

       ```bash
       # 拡張機能をインストール
       gemini extensions install https://github.com/wandb/wandb-mcp-server
       ```
    2. インストール後、Gemini CLI を再起動します。
    3. 「List the projects in my W&amp;B account.」というプロンプトを入力し、チャット エージェントが W&amp;B MCP server にアクセスできることを確認します。

    さらに詳しい情報については、[Gemini のドキュメント](https://geminicli.com/docs/tools/mcp-server/)を参照してください。
  </Tab>

  <Tab title="Mistral LeChat">
    W&amp;B MCP server を Mistral LeChat に追加するには、次の手順に従います。

    1. **Intelligence** メニューから **Add Connector** を選択して、Connector ウィンドウを開きます。

    2. **Custom MCP Connector** タブを選択します。

    3. 次の値を使用して各フィールドを設定します。

       * **Connector Server**: `https://mcp.withwandb.com/mcp`
       * **Description**:  (任意) 接続の簡潔な任意の説明。
       * **Authentication Method**: **API Token Authentication** を選択します。追加のフィールドが開きます。
       * **Header name**: デフォルト値の **Authorization** のままにします。
       * **Header type**: **Bearer** を選択します。
       * **Header value**: W&amp;B API token を入力します。

    4. すべてのフィールドの設定が完了したら、**Create** を選択します。LeChat に MCP server が設定に追加されます。

    5. 「List the projects in my W&amp;B account.」というプロンプトを入力して、チャット エージェントが W&amp;B MCP server にアクセスできることを確認します。

    詳細については、[LeChat のドキュメント](https://mistral.ai/news/le-chat-mcp-connectors-memories) を参照してください。
  </Tab>
</Tabs>

<div id="set-up-a-local-version-of-the-wb-mcp-server">
  ## W&amp;B MCP サーバー のローカル版を設定する
</div>

W&amp;B セルフマネージド のデプロイ、開発、テスト、またはエアギャップ環境で MCP サーバーをローカルで実行する必要がある場合は、ご使用のマシンにインストールして実行できます。

<div id="prerequisites">
  ### 前提条件
</div>

* W&amp;B APIキー。新しいキーは [wandb.ai/authorize](https://wandb.ai/authorize) で作成できます。
* キーを `WANDB_API_KEY` という環境変数に設定します。
* [W&amp;B セルフマネージド](/ja/platform/hosting/hosting-options/self-managed) を使用している場合は、環境変数 `WANDB_BASE_URL` を設定します。
* Python 3.10 以上
* [uv](https://docs.astral.sh/uv/) (推奨) または pip

インストール手順については、uv のドキュメントの [インストール手順](https://docs.astral.sh/uv/getting-started/installation/) を参照してください。

<div id="install-and-configure-the-mcp-server">
  ### MCP サーバーをインストールして設定する
</div>

MCP サーバーをローカルにインストールするには:

ローカルマシンに W&amp;B MCP サーバーをインストールするには、次のいずれかのインストール コマンドを使用します。

<Tabs>
  <Tab title="uv">
    ```bash
    uv install wandb-mcp-server
    ```
  </Tab>

  <Tab title="pip">
    ```bash
    pip install wandb-mcp-server
    ```
  </Tab>

  <Tab title="Install directly from GitHub">
    ```bash
    pip install git+https://github.com/wandb/wandb-mcp-server
    ```
  </Tab>
</Tabs>

MCP サーバーをローカルにインストールしたら、それを使用するように MCP クライアントを設定します。続行するには、MCP クライアントを選択してください。

<Tabs>
  <Tab title="Cursor">
    `mcp.json` 設定に次の内容を追加します。

    ```json
    {
      "mcpServers": {
        "wandb": {
          "command": "uvx",
          "args": [
            "--from",
            "git+https://github.com/wandb/wandb-mcp-server",
            "wandb_mcp_server"
          ],
          "env": {
            "WANDB_API_KEY": "<your-wandb-api-key>",
            "WANDB_BASE_URL": "https://your-wandb-instance.example.com"
          }
        }
      }
    }
    ```
  </Tab>

  <Tab title="VS Code">
    `.vscode/mcp.json` またはグローバル MCP 設定に次の内容を追加します。

    ```json
    {
      "servers": {
        "wandb": {
          "command": "uvx",
          "args": [
            "--from",
            "git+https://github.com/wandb/wandb-mcp-server",
            "wandb_mcp_server"
          ],
          "env": {
            "WANDB_API_KEY": "<your-wandb-api-key>",
            "WANDB_BASE_URL": "https://your-wandb-instance.example.com"
          }
        }
      }
    }
    ```
  </Tab>

  <Tab title="Claude Code">
    ターミナルで次のコマンドを実行します。グローバル設定にするには `--scope user` を追加し、現在のプロジェクトのみに設定する場合は省略します。

    ```bash
    claude mcp add wandb \
      -e WANDB_API_KEY=your-api-key \
      -e WANDB_BASE_URL=https://your-wandb-instance.example.com \
      -- uvx --from git+https://github.com/wandb/wandb-mcp-server wandb_mcp_server
    ```
  </Tab>

  <Tab title="Codex">
    ターミナルで次のコマンドを実行します。

    ```bash
    codex mcp add wandb \
      --env WANDB_API_KEY=your_api_key_here \
      --env WANDB_BASE_URL=https://your-wandb-instance.example.com \
      -- uvx --from git+https://github.com/wandb/wandb-mcp-server wandb_mcp_server
    ```
  </Tab>

  <Tab title="Claude Desktop">
    Claude の設定ファイルをテキスト エディタで開きます。設定ファイルの場所は OS によって異なります。

    * **macOS**: ~/Library/Application\ Support/Claude/claude&#95;desktop&#95;config.json
    * **Windows**: %APPDATA%\Claude\claude&#95;desktop&#95;config.json

    Claude の設定ファイル内の JSON オブジェクトに次の内容を追加します。Claude Desktop で `uvx` のインストール先を見つけられない場合があるため、`uvx` にはフルパスを指定してください。

    ```json
    {
      "mcpServers": {
        "wandb": {
          "command": "/full/path/to/uvx",
          "args": [
            "--from",
            "git+https://github.com/wandb/wandb-mcp-server",
            "wandb_mcp_server"
          ],
          "env": {
            "WANDB_API_KEY": "<your-wandb-api-key>",
            "WANDB_BASE_URL": "https://your-wandb-instance.example.com"
          }
        }
      }
    }
    ```

    新しい設定を有効にするには、Claude Desktop を再起動します。
  </Tab>
</Tabs>

Web ベースのクライアントやテストでは、HTTP トランスポートでサーバーを実行します:

```bash
uvx wandb_mcp_server --transport http --host 0.0.0.0 --port 8080
```

OpenAI などの外部クライアントにローカルサーバーを公開するには、ngrok を使用します：

```bash
uvx wandb_mcp_server --transport http --port 8080

# 別のターミナルで、ngrok を使って公開する
ngrok http 8080
```

サーバーを `ngrok` で公開する場合は、`ngrok` の URL を使用するように MCP クライアントの設定を更新してください。

<div id="usage-tips">
  ## 使用上のヒント
</div>

* **W&amp;B のプロジェクト名と entity 名を指定する**: 正確な結果を得るため、クエリに W&amp;B の entity とプロジェクトを指定してください。
* **広すぎる質問は避ける**: 「what is my best evaluation?」ではなく、「what eval had the highest f1 score?」のように質問してください。
* **データ取得を確認する**: 「what are my best performing runs?」のような幅広い質問をする場合は、사용 가능한すべての Runs を取得できているか、アシスタントに確認させてください。
