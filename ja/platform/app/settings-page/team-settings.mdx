<div id="team-settings">
  # Team settings
</div>

メンバー、アバター、アラート、プライバシー、使用状況など、チームの設定を変更できます。組織管理者とチーム管理者は、チームの設定を表示および編集できます。

<Note>
  チームの設定を変更したり、チームからメンバーを削除したりできるのは、Administration アカウントタイプのみです。
</Note>

<div id="members">
  ## メンバー
</div>

Members セクションには、保留中の招待の一覧と、チームへの招待を承諾したメンバーが表示されます。一覧の各メンバーには、名前、ユーザー名、メールアドレス、チームロールに加え、組織から継承される Models と W&amp;B Weave へのアクセス権限が表示されます。標準のチームロール **Admin**、**Member**、**View-only** から選択できます。組織で [custom roles](/ja/platform/hosting/iam/access-management/manage-organization#add-and-manage-custom-roles) を作成している場合は、代わりにカスタムロールを割り当てることもできます。

チームの作成方法、Teams の管理方法、チームメンバーシップとロールの管理方法については、[Add and Manage teams](/ja/platform/hosting/iam/access-management/manage-organization#add-and-manage-teams) を参照してください。新しいメンバーを招待できるユーザーの設定や、チームのその他のプライバシー設定を行うには、[プライバシー](#privacy) を参照してください。

<div id="avatar">
  ## アバター
</div>

**Avatar** セクションに移動し、画像をアップロードしてアバターを設定します。

1. **Update Avatar** を選択すると、ファイルダイアログが表示されます。
2. ファイルダイアログから、使用する画像を選択します。

<div id="alerts">
  ## アラート
</div>

run がクラッシュしたときや完了したとき、またはカスタムアラートを設定して、チームに通知できます。チームは、メールまたは Slack のいずれかでアラートを受信できます。

アラートを受信したいイベントタイプの横にあるスイッチを切り替えます。Weights and Biases には、デフォルトで次のイベントタイプのオプションがあります。

* **Runs finished**: Weights and Biases の run が正常に完了したとき。
* **Run crashed**: run が正常に完了せず失敗したとき。

アラートの設定と管理の方法について詳しくは、[`wandb.Run.alert()` を使用したアラートの送信](/ja/models/runs/alert)を参照してください。

<div id="slack-notifications">
  ## Slack 通知
</div>

新しい artifact が作成されたときや、run のメトリクスが定義したしきい値に達したときなど、Registry またはプロジェクトでイベントが発生した際に、チームの [automations](/ja/models/automations/) が通知を送信する Slack の通知先を設定します。詳細は、[Slack automation を作成する](/ja/models/automations/create-automations/slack/)を参照してください。

  <EnterpriseOnly />
</Info>

<div id="webhooks">
  ## ウェブフック
</div>

Registry またはプロジェクトでイベントが発生した際に、チームの[automations](/ja/models/automations/)から実行されるウェブフックを設定します。たとえば、新しい artifact が作成されたときや、run のメトリクスが設定したしきい値に達したときです。詳しくは、[webhook automation を作成する](/ja/models/automations/create-automations/slack/)を参照してください。

<div id="privacy">
  ## プライバシー
</div>

プライバシー設定を変更するには、**プライバシー** セクションにアクセスします。プライバシー設定を変更できるのは組織管理者のみです。

* 今後作成するプロジェクトを公開にしたり、Reports を公開で共有したりする機能を無効にします。
* チーム管理者だけでなく、任意のチームメンバーが他のメンバーを招待できるようにします。
* code saving をデフォルトで有効にするかどうかを管理します。

<div id="usage">
  ## 使用状況
</div>

**Usage** セクションには、チームが Weights and Biases のサーバー上で使用した合計メモリ使用量が表示されます。デフォルトのストレージプランは 100GB です。ストレージと Pricing の詳細については、[Pricing](https://wandb.ai/site/pricing) ページを参照してください。

<div id="storage">
  ## ストレージ
</div>

**Storage** セクションでは、チームのデータに使用されているクラウドストレージバケットの設定について説明します。詳細は、[Secure Storage Connector](/ja/platform/app/settings-page/teams/#secure-storage-connector)を参照してください。セルフホスティングしている場合は、[W&amp;B Server](/ja/platform/hosting/data-security/secure-storage-connector)のドキュメントも参照してください。
