<div id="team-settings">
  # チーム設定
</div>

メンバー、アバター、アラート、プライバシー、利用状況など、チームの設定を変更できます。組織管理者およびチーム管理者は、チーム設定を表示および編集できます。

<Note>
  Administration アカウント タイプのみが、チーム設定を変更したり、チームからメンバーを削除したりできます。
</Note>

<div id="members">
  ## メンバー
</div>

Members セクションには、保留中の招待と、チームへの招待を承諾したメンバーの一覧が表示されます。各メンバーには、名前、ユーザー名、メールアドレス、チームロール、および組織から継承される Models と W&amp;B Weave へのアクセス権限が表示されます。標準のチームロールである **Admin**、**Member**、**View-only** から選択できます。組織で [カスタムロール](/ja/platform/hosting/iam/access-management/manage-organization#add-and-manage-custom-roles) を作成している場合は、代わりにカスタムロールを割り当てることもできます。

チームの作成方法、チームの管理、チームメンバーシップとロールの管理については、[Add and Manage teams](/ja/platform/hosting/iam/access-management/manage-organization#add-and-manage-teams) を参照してください。新しいメンバーを誰が招待できるかなど、チームのプライバシー設定を構成するには、[Privacy](#privacy) を参照してください。

<div id="avatar">
  ## アバター
</div>

**Avatar** セクションに移動し、画像をアップロードしてアバターを設定します。

1. **Update Avatar** を選択して、ファイルダイアログを表示します。
2. ファイルダイアログで、使用する画像を選択します。

<div id="alerts">
  ## アラート
</div>

run がクラッシュした場合、完了した場合、またはカスタム アラートの条件を満たしたときに、チームに通知します。チームは、メールまたは Slack を通じてアラートを受信できます。

アラートを受け取りたいイベント タイプの横にあるスイッチを切り替えます。Weights &amp; Biases は、デフォルトで以下のイベント タイプ オプションを提供しています。

* **Runs finished**: Weights &amp; Biases の run が正常に完了した場合。
* **Run crashed**: run が終了に失敗した場合。

アラートの設定および管理方法についての詳細は、[Send alerts with `wandb.Run.alert()`](/ja/models/runs/alert) を参照してください。

<div id="slack-notifications">
  ## Slack 通知
</div>

Registry やプロジェクトでイベントが発生したときに、たとえば新しいアーティファクトが作成された場合や run のメトリクスが定義済みのしきい値に達した場合に、チームの [オートメーション](/ja/models/automations/) から通知を送信する Slack の送信先を設定します。[Create a Slack automation](/ja/models/automations/create-automations/slack/) を参照してください。

  <EnterpriseOnly />
</Info>

<div id="webhooks">
  ## Webhooks
</div>

チームの[オートメーション](/ja/models/automations/)が Registry またはプロジェクトでイベント (新しいアーティファクトが作成されたときや、run のメトリクスが定義されたしきい値を満たしたときなど) が発生した際に実行できる webhook を設定します。[webhook オートメーションの作成](/ja/models/automations/create-automations/slack/)を参照してください。

<div id="privacy">
  ## プライバシー
</div>

プライバシー設定を変更するには、**Privacy** セクションにアクセスします。プライバシー設定を変更できるのは組織管理者のみです。

* 今後作成するプロジェクトを公開したり、Reports を一般公開で共有したりする機能を無効にできます。
* チーム管理者だけでなく、任意のチームメンバーが他のメンバーを招待できるようにできます。
* コードの保存をデフォルトで有効にするかどうかを管理できます。

<div id="usage">
  ## 使用状況
</div>

**Usage** セクションには、Weights &amp; Biases サーバー上でチームが利用した合計メモリ使用量が表示されます。既定のストレージプランは 100GB です。ストレージおよび料金の詳細については、[Pricing](https://wandb.ai/site/pricing) ページを参照してください。

<div id="storage">
  ## Storage
</div>

**Storage** セクションでは、チームのデータに使用されているクラウドストレージバケットの設定について説明します。詳細は、[Secure Storage Connector](/ja/platform/app/settings-page/teams/#secure-storage-connector) を参照してください。セルフホストしている場合は、[W&amp;B Server](/ja/platform/hosting/data-security/secure-storage-connector) のドキュメントもご覧ください。
