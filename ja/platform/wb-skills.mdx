W&amp;B Skills は、コーディングエージェントに W&amp;B を効果的に使う方法を教える、再利用可能な指示セットです。W&amp;B API やベストプラクティスを手作業でエージェントに教える代わりに、Skills をインストールすれば、エージェントが自力で実験管理、トレース、評価、モニタリングを行えるようになります。

Skills は、以下を含む主要なコーディングエージェントで利用できます。

* Claude Code
* Codex
* Cursor
* GitHub Copilot
* Gemini CLI

サポートされているエージェントの完全な一覧については、[W&amp;B Skills CLI ドキュメント](https://github.com/vercel-labs/skills#supported-agents) を参照してください。

<div id="wb-skills-capabilities">
  ## W&amp;B Skills の機能
</div>

Skills は、[W&amp;B Models SDK](/ja/models/ref) (トレーニング Runs、メトリクス、アーティファクト、sweeps) と [Weave SDK](/ja/weave/reference/python-sdk) (トレース、評価、スコアラー) の両方をカバーしています。ヘルパーライブラリ、リファレンスドキュメント、データ分析パターンが含まれます。

| Workflow | Capabilities |
|----------|-------------|
| **モデルのトレーニング** | <ul><li>トレーニングとファインチューニング中に、メトリクスやリッチメディアをログします。</li><li>実験を追跡して比較します。</li><li>損失曲線や精度スコアなど、Runs と結果を分析します。</li><li>ハイパーパラメータを調整します。</li></ul> |
| **エージェントの構築** | <ul><li>エージェント型 AI アプリケーションをトレースします。</li><li>トレースを分析し、障害モードを分類します。</li><li>ラベル付きデータセットを使ってモデルとエージェントを評価します。</li><li>本番環境のモニタリングのためにオンライン評価を実行します。</li></ul> |

<div id="prerequisites">
  ## 前提条件
</div>

Skills を使用するには、以下が必要です:

* [Node.js](https://nodejs.org/) (`npx` コマンドを使用するため) 。
* W&amp;B APIキー。[wandb.ai/authorize](https://wandb.ai/authorize) で作成し、次のように環境変数として設定します:
  ```shell
  export WANDB_API_KEY=<your-api-key>
  ```
* (任意) W&amp;B プロジェクト名を `WANDB_PROJECT` 環境変数として設定します。これにより、毎回指定しなくても、エージェントが正しい W&amp;B プロジェクトを対象にできます。

<div id="install-wb-skills">
  ## W&amp;B Skills をインストールする
</div>

W&amp;B Skills をグローバルにインストールするには、`--global` フラグを付けて次のコマンドを実行します。

```bash
npx skills add wandb/skills --skill '*' --yes --global
```

特定のプロジェクト用に Skills をインストールするには、プロジェクト ディレクトリで次のコマンドを実行します。

```bash
npx skills add wandb/skills --skill '*' --yes
```

特定のエージェント用に Skills をインストールするには、`--agent` フラグを使用することもできます：

```bash
npx skills add wandb/skills --agent claude-code --skill '*' --yes --global
```

`--agent` と `--skill` のオプション一覧については、[skills CLI ドキュメント](https://github.com/vercel-labs/skills#supported-agents) を参照してください。

<div id="use-wb-skills">
  ## W&amp;B Skills を使う
</div>

インストール後は、エージェントにプロジェクトに関する W&amp;B 関連のタスクを実行するよう依頼できます。以下のプロンプト例は、エージェントが W&amp;B Skills で実行できるタスクの一部を示しています。

* &quot;PyTorch モデルのトレーニング メトリクスを W&amp;B にログして。&quot;
* &quot;直近 10 件の Runs の損失曲線を分析して、最も性能の高い設定を特定して。&quot;
* &quot;LangChain エージェントをトレースして、結果を Weave にログして。&quot;
* &quot;テスト データセットを使ってエージェントの評価を実行し、結果を要約して。&quot;
* &quot;直近の評価の失敗パターンを見つけて分類して。&quot;
* &quot;run A と run B の設定を比較して、違いを表示して。&quot;

<div id="usage-tips">
  ## 使用のヒント
</div>

Skills は、広範な自由形式の質問よりも、より具体的なクエリを使用したほうがうまく機能します。次の表では、推奨されるプロンプト例と、曖昧すぎて推奨されないプロンプトを示します。

| Recommended | Not recommended |
|-------------|-----------------|
| &quot;直近 5 件の Runs の最終的な検証損失は何ですか？&quot; | &quot;私のモデルの調子はどうですか？&quot; |
| &quot;直近 10 件のトレース全体の token 使用量を要約してください。&quot; | &quot;すべてのトレースを表示してください。&quot; |
| &quot;run A と run B の設定を比較してください。&quot; | &quot;最も良い Runs はどれですか？&quot; |
| &quot;最も高い F1 スコアだった eval はどれですか？&quot; | &quot;評価の状況はどうですか？&quot; |
