title: シークレットの管理
---

W&amp;B Secret Manager は、アクセストークン、ベアラートークン、API キー、パスワードなどの機密文字列である *シークレット* を、安全かつ一元的に保存・管理し、注入できるようにします。W&amp;B Secret Manager を使用すると、コード内や webhook のヘッダー、または [ペイロード](/ja/models/automations/) を設定するときに機密文字列を直接追加する必要がなくなります。

シークレットは各チームの Secret Manager 内、[チーム設定](/ja/platform/app/settings-page/teams/) の **Team secrets** セクションで保存および管理されます。

<Note>
  * シークレットを作成、編集、または削除できるのは W&amp;B Admins のみです。
  * シークレットは、Azure、Google Cloud、AWS 上でホストする [W&amp;B Server deployments](/ja/platform/hosting/) を含む、W&amp;B の中核機能として提供されています。別のデプロイタイプを使用している場合に W&amp;B でシークレットをどのように利用できるかについては、W&amp;B のアカウントチームにお問い合わせください。
  * W&amp;B Server では、お客様のセキュリティ要件を満たすようにセキュリティ対策を構成する責任はお客様にあります。

    * W&amp;B は、クラウドプロバイダー（AWS、Google Cloud、Azure）が提供する secrets manager を利用する W&amp;B インスタンスにシークレットを保存し、それらの高度なセキュリティ機能を用いて構成することを強く推奨します。

    * AWS、Google Cloud、Azure のクラウド secrets manager を利用する W&amp;B インスタンスを使用できず、かつクラスターを使用した場合に発生しうるセキュリティ上の脆弱性を防ぐ方法を理解している場合を除き、シークレットストアのバックエンドとして Kubernetes クラスターを使用することは推奨しません。
</Note>

<div id="add-a-secret">
  ## シークレットを追加する
</div>

シークレットを追加するには、次の手順を実行します。

1. 受信サービスで、受信した webhook を認証するためにシークレットが必要な場合は、必要なトークンまたは API キーを生成します。必要に応じて、パスワードマネージャーなどを使ってこの機密文字列を安全に保存します。
2. W&amp;B にログインし、チームの **Settings** ページに移動します。
3. **Team Secrets** セクションで **New secret** をクリックします。
4. アルファベット、数字、アンダースコア（`_`）を使って、シークレット名を指定します。
5. 機密文字列を **Secret** フィールドに貼り付けます。
6. **Add secret** をクリックします。

webhook を設定する際に、その webhook オートメーションで使用するシークレットを指定します。詳細については、[Configure a webhook](#configure-a-webhook) セクションを参照してください。

<Note>
  シークレットを作成すると、そのシークレットには `${SECRET_NAME}` という形式を使って、[webhook オートメーションのペイロード](/ja/models/automations/create-automations/webhook/)内でアクセスできます。
</Note>

<div id="rotate-a-secret">
  ## シークレットをローテーションする
</div>

シークレットをローテーションして値を更新するには、次の手順を実行します。

1. シークレットの行にある鉛筆アイコンをクリックして、シークレットの詳細を開きます。
2. **Secret** を新しい値に設定します。必要に応じて **Reveal secret** をクリックして、新しい値を確認します。
3. **Add secret** をクリックします。シークレットの値が更新され、以前の値にはもう解決されなくなります。

<Note>
  シークレットを作成または更新した後は、その現在の値を表示することはできません。代わりに、シークレットをローテーションして新しい値にしてください。
</Note>

<div id="delete-a-secret">
  ## シークレットを削除する
</div>

シークレットを削除するには、次の手順を実行します。

1. シークレットの行にあるゴミ箱アイコンをクリックします。
2. 確認ダイアログの内容を確認し、**Delete** をクリックします。シークレットは直ちに完全に削除されます。

<div id="manage-access-to-secrets">
  ## シークレットへのアクセスを管理する
</div>

チームの自動化は、そのチームのシークレットを使用できます。シークレットを削除する前に、そのシークレットを使用している自動化を更新または削除し、処理が停止しないようにしてください。
