title: シークレットを管理する
---

W&amp;B Secret Manager を使用すると、アクセストークン、ベアラートークン、APIキー、パスワードなどの機密情報にあたる文字列である *secrets* を、安全かつ一元的に保存、管理、挿入できます。W&amp;B Secret Manager を使うことで、機密性の高い文字列をコードに直接追加したり、webhook のヘッダーや [payload](/ja/models/automations/) の設定時に指定したりする必要がなくなります。

シークレットは各チームの Secret Manager に保存・管理され、[Team settings](/ja/platform/app/settings-page/teams/) の **Team secrets** セクションにあります。

<Note>
  * シークレットを作成、編集、削除できるのは W&amp;B Admin のみです。
  * シークレットは W&amp;B の中核機能の一部であり、Azure、Google Cloud、または AWS でホストする [W&amp;B Server のデプロイ](/ja/platform/hosting/) にも含まれます。別のデプロイタイプを使用している場合に W&amp;B でシークレットをどのように利用できるかについては、W&amp;B の account team にご相談ください。
  * W&amp;B Server では、セキュリティ要件を満たすためのセキュリティ対策を設定する責任はお客様にあります。

    * W&amp;B では、AWS、Google Cloud、または Azure が提供する高度なセキュリティ機能を備えたクラウドプロバイダーの secrets manager 上の W&amp;B インスタンスにシークレットを保存することを強く推奨しています。

    * W&amp;B では、クラウド secrets manager (AWS、Google Cloud、または Azure) 上の W&amp;B インスタンスを使用できず、かつ cluster を使用することで発生しうるセキュリティ脆弱性を防ぐ方法を理解している場合を除き、Kubernetes cluster をシークレットストアのバックエンドとして使用しないことを推奨しています。
</Note>

<div id="add-a-secret">
  ## シークレットを追加する
</div>

シークレットを追加するには、次の手順に従います。

1. 受信側のサービスで受信 webhook の認証が必要な場合は、必要な token または APIキー を生成します。必要に応じて、その機密文字列をパスワードマネージャーなどの安全な場所に保存してください。
2. W&amp;B にログインし、Teams の **Settings** ページに移動します。
3. **Team Secrets** セクションで、**New secret** をクリックします。
4. 英字、数字、アンダースコア (`_`) を使用して、シークレット名を指定します。
5. **Secret** フィールドに機密文字列を貼り付けます。
6. **Add secret** をクリックします。

webhook を設定するときに、webhook automation で使用するシークレットを指定します。詳しくは、[webhook を設定する](#configure-a-webhook) セクションを参照してください。

<Note>
  シークレットを作成すると、[webhook automation の payload](/ja/models/automations/create-automations/webhook/) 内で `${SECRET_NAME}` の形式を使用してそのシークレットを参照できます。
</Note>

<div id="rotate-a-secret">
  ## シークレットをローテーションする
</div>

シークレットをローテーションして値を更新するには、次の手順に従います。

1. シークレットの行にある鉛筆アイコンをクリックして、シークレットの詳細を開きます。
2. **Secret** に新しい値を設定します。必要に応じて **Reveal secret** をクリックし、新しい値を確認します。
3. **Add secret** をクリックします。シークレットの値が更新され、以前の値は参照されなくなります。

<Note>
  シークレットを作成または更新すると、現在の値は表示できなくなります。代わりに、シークレットを新しい値にローテーションしてください。
</Note>

<div id="delete-a-secret">
  ## シークレットを削除する
</div>

シークレットを削除するには:

1. シークレットの行にあるごみ箱アイコンをクリックします。
2. 確認ダイアログの内容を確認し、**Delete** をクリックします。シークレットは即座に完全に削除されます。

<div id="manage-access-to-secrets">
  ## シークレットへのアクセスを管理する
</div>

チームのAutomationsは、チームのシークレットを使用できます。シークレットを削除する前に、それを使用しているAutomationsを更新または削除し、動作しなくならないようにしてください。
