---
title: SMTP を設定する
---

W&B server では、インスタンスまたはチームにユーザーを追加すると、メールによる招待が送信されます。これらの招待メールを送信するために、W&B はサードパーティのメールサーバーを使用します。場合によっては、組織に企業ネットワーク外へのトラフィックに関する厳格なポリシーがあるため、これらの招待メールがエンドユーザーに届かないことがあります。W&B server では、内部 SMTP サーバー経由でこれらの招待メールを送信するように設定できるオプションを提供しています。

設定するには、以下の手順に従ってください:

- Docker コンテナまたは Kubernetes デプロイメントで、環境変数 `GORILLA_EMAIL_SINK` を `smtp://<user:password>@smtp.host.com:<port>` に設定します
- `username` と `password` は省略可能です
- 認証不要な SMTP サーバーを使用している場合は、環境変数の値を `GORILLA_EMAIL_SINK=smtp://smtp.host.com:<port>` のように設定するだけでかまいません
- SMTP で一般的に使用されるポート番号は 587、465、25 です。使用するメールサーバーの種類によって異なる場合があります。
- SMTP 用のデフォルト送信元メールアドレスは初期状態では `noreply@wandb.com` に設定されていますが、任意のメールアドレスに変更できます。これは、サーバー上で環境変数 `GORILLA_EMAIL_FROM_ADDRESS` を希望する送信元メールアドレスに設定することで行えます。
