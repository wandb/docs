---
title: SMTP を設定する
description: "セルフマネージド W&B Server インスタンスからメール招待を送信するために、内部 SMTP サーバーを設定します。"
---

W&amp;B server では、インスタンスまたはチームにユーザー を追加すると、メール招待が送信されます。これらの招待メールを送信するために、W&amp;B はサードパーティのメールサーバーを使用します。場合によっては、組織のポリシーにより企業ネットワークから外部への通信が厳しく制限されていて、その結果、招待メールがエンドユーザー に送信されないことがあります。W&amp;B server では、内部 SMTP サーバー経由でこれらの招待メールを送信するように設定するオプションを提供しています。

設定するには、次の手順に従います。

* Docker コンテナまたは Kubernetes デプロイで `GORILLA_EMAIL_SINK` 環境変数を `smtp://<user:password>@smtp.host.com:<port>` に設定します
* `username` と `password` は任意です
* 認証なしで使用するよう設計された SMTP サーバーを使用している場合は、環境変数の値を `GORILLA_EMAIL_SINK=smtp://smtp.host.com:<port>` のように設定するだけです
* SMTP で一般的に使用されるポート番号は 587、465、25 です。使用しているメールサーバーの種類によって異なる場合がある点に注意してください。
* SMTP のデフォルト送信元メールアドレスは、初期状態では `noreply@wandb.com` に設定されていますが、任意のメールアドレスに更新できます。これは、サーバー上で `GORILLA_EMAIL_FROM_ADDRESS` 環境変数を希望する送信元メールアドレスに設定することで行えます。
