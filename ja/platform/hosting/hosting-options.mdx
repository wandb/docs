本セクションでは、W&amp;B をデプロイするさまざまな方法について説明します。

<div id="wb-multi-tenant-cloud">
  ## W&amp;B マルチテナント Cloud
</div>

[W&amp;B Multi-tenant Cloud](/ja/platform/hosting/#wb-multi-tenant-cloud) は、アップグレード、メンテナンス、プラットフォーム セキュリティ、キャパシティプランニングを含めて W&amp;B によりフルマネージドで提供されます。Multi-tenant Cloud は、[Google Cloud の北米リージョン](https://cloud.google.com/compute/docs/regions-zones) にある W&amp;B の Google Cloud アカウント上にデプロイされます。[Bring your own bucket (BYOB)](/ja/platform/hosting/data-security/secure-storage-connector) を利用すると、W&amp;B Artifacts やその他の関連する機密データを、自分のクラウドまたはオンプレミス環境に保存することもできます。

[W&amp;B Multi-tenant Cloud](/ja/platform/hosting/#wb-multi-tenant-cloud) を参照するか、[こちらから無料で始めましょう](https://app.wandb.ai/login?signup=true)。

<div id="wb-dedicated-cloud">
  ## W&amp;B Dedicated Cloud
</div>

[W&amp;B Dedicated Cloud](/ja/platform/hosting/#wb-dedicated-cloud) は、エンタープライズ企業を想定して設計された、シングルテナントかつフルマネージドなプラットフォームです。W&amp;B Dedicated Cloud は、W&amp;B の AWS、Google Cloud、または Azure アカウント上にデプロイされます。Dedicated Cloud は Multi-tenant Cloud よりも柔軟性が高く、W&amp;B Self-Managed よりも運用の複雑さが抑えられた構成を提供します。アップグレード、メンテナンス、プラットフォームのセキュリティ、キャパシティプランニングは W&amp;B が管理します。各 Dedicated Cloud インスタンスは、他の W&amp;B Dedicated Cloud インスタンスとは分離された独立のネットワーク、コンピュート、ストレージを持ちます。

お客様固有の W&amp;B メタデータおよびデータは分離されたクラウドストレージに保存され、分離されたクラウドコンピュートサービスを用いて処理されます。[Bring your own bucket (BYOB)](/ja/platform/hosting/data-security/secure-storage-connector) を利用すると、オプション機能として Artifacts やその他の機密データを、お客様自身のクラウドまたはオンプレミスインフラストラクチャに保存できます。

W&amp;B Dedicated Cloud には、重要なセキュリティ機能やその他のエンタープライズ向け機能をサポートする [エンタープライズライセンス](/ja/platform/hosting/hosting-options/self-managed/) が含まれます。

高度なセキュリティまたはコンプライアンス要件を持つ組織向けには、HIPAA 準拠、Single Sign On、Customer Managed Encryption Keys (CMEK) などの機能が、**Enterprise** サポートとともに提供されます。[詳細を問い合わせる](https://wandb.ai/site/contact)。

[W&amp;B Dedicated Cloud](/ja/platform/hosting/#wb-dedicated-cloud) を参照するか、[こちらから無料でお試しください](https://app.wandb.ai/login?signup=true)。

<div id="wb-self-managed">
  ## W&amp;B Self-Managed
</div>

[W&amp;B Self-Managed](/ja/platform/hosting/hosting-options/self-managed/) は、オンプレミス環境または自社で管理するクラウド インフラストラクチャ上に構築し、すべてを自社で管理する方式です。IT / DevOps / MLOps チームは、次の事項を担当します。

* デプロイメントのプロビジョニング
* 必要に応じて、組織のポリシーおよび [Security Technical Implementation Guidelines (STIG)](https://en.wikipedia.org/wiki/Security_Technical_Implementation_Guide) に従ってインフラストラクチャを保護すること
* アップグレードの管理とパッチの適用
* Self-Managed W&amp;B Server インスタンスの継続的な維持

W&amp;B Self-Managed 向けにエンタープライズライセンスを取得することもできます。エンタープライズライセンスには、重要なセキュリティ機能およびその他のエンタープライズ向け機能のサポートが含まれます。

[W&amp;B Self-Managed](/ja/platform/hosting/hosting-options/self-managed/) を参照するか、[リファレンス アーキテクチャ](/ja/platform/hosting/self-managed/ref-arch/) ガイドラインを確認してください。
