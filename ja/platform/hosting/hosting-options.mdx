このセクションでは、W&amp;B をデプロイするさまざまな方法について説明します。

<div id="wb-multi-tenant-cloud">
  ## W&amp;B マルチテナントクラウド
</div>

[W&amp;B マルチテナントクラウド](/ja/platform/hosting/#wb-multi-tenant-cloud) は、アップグレード、メンテナンス、プラットフォームセキュリティ、キャパシティプランニングまで含めて W&amp;B が完全に管理するサービスです。Multi-tenant Cloud は、[Google Cloud の北米リージョン](https://cloud.google.com/compute/docs/regions-zones) にある W&amp;B の Google Cloud アカウント内にデプロイされます。[Bring your own bucket (BYOB)](/ja/platform/hosting/data-security/secure-storage-connector) を利用すると、W&amp;B の Artifacts やその他の関連する機密データを、自身のクラウドまたはオンプレミス環境に保存することもできます。

[W&amp;B マルチテナントクラウド](/ja/platform/hosting/#wb-multi-tenant-cloud) を参照するか、[無料で始める](https://app.wandb.ai/login?signup=true) から利用を開始してください。

<div id="wb-dedicated-cloud">
  ## W&amp;B Dedicated Cloud
</div>

[W&amp;B Dedicated Cloud](/ja/platform/hosting/#wb-dedicated-cloud) は、エンタープライズ企業向けに設計された、シングルテナントかつフルマネージドのプラットフォームです。W&amp;B Dedicated Cloud は、W&amp;B の AWS、Google Cloud、または Azure アカウント上にデプロイされます。Dedicated Cloud は Multi-tenant Cloud よりも高い柔軟性を提供しつつ、W&amp;B Self-Managed ほどの複雑さはありません。アップグレード、メンテナンス、プラットフォームのセキュリティ、およびキャパシティプランニングは W&amp;B が管理します。各 Dedicated Cloud インスタンスは、他の W&amp;B Dedicated Cloud インスタンスとは分離された、専用のネットワーク、コンピュート、およびストレージを持ちます。

お客様固有の W&amp;B メタデータおよびデータは、専用のクラウドストレージに保存され、専用のクラウドコンピュートサービスを用いて処理されます。[Bring your own bucket (BYOB)](/ja/platform/hosting/data-security/secure-storage-connector) を使用すると、オプションでアーティファクトやその他の機密性の高い関連データを、お客様自身のクラウドまたはオンプレミスのインフラストラクチャに保存できます。

W&amp;B Dedicated Cloud には、重要なセキュリティ機能やその他のエンタープライズ向け機能をサポートする [エンタープライズライセンス](/ja/platform/hosting/hosting-options/self-managed/) が含まれます。

高度なセキュリティやコンプライアンス要件を持つ組織向けには、HIPAA コンプライアンス、シングルサインオン (SSO)、Customer Managed Encryption Keys (CMEK) などの機能が **Enterprise** サポートで利用可能です。[詳細情報をリクエスト](https://wandb.ai/site/contact) してください。

[W&amp;B Dedicated Cloud](/ja/platform/hosting/#wb-dedicated-cloud) を参照するか、[無料で利用を開始](https://app.wandb.ai/login?signup=true) してください。

<div id="wb-self-managed">
  ## W&amp;B Self-Managed
</div>

[W&amp;B Self-Managed](/ja/platform/hosting/hosting-options/self-managed/) は、オンプレミス環境または利用者が管理するクラウドインフラストラクチャ上で、すべてを自社側で管理するオプションです。IT/DevOps/MLOps チームは次の責任を負います。

* デプロイメントのプロビジョニング
* 必要に応じて、組織のポリシーおよび [Security Technical Implementation Guidelines (STIG)](https://en.wikipedia.org/wiki/Security_Technical_Implementation_Guide) に従ってインフラストラクチャを保護すること
* アップグレードの管理およびパッチの適用
* Self-Managed W&amp;B Server インスタンスの継続的な保守

必要に応じて、W&amp;B Self-Managed 用のエンタープライズライセンスを取得できます。エンタープライズライセンスには、重要なセキュリティ機能やその他のエンタープライズ向け機能に対するサポートが含まれます。

[W&amp;B Self-Managed](/ja/platform/hosting/hosting-options/self-managed/) を参照するか、[reference architecture](/ja/platform/hosting/self-managed/ref-arch/) のガイドラインを確認してください。
