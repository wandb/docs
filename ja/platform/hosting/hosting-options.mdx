本セクションでは、W&amp;B をデプロイするさまざまな方法について説明します。

<div id="wb-multi-tenant-cloud">
  ## W&amp;B multi-tenant cloud
</div>

[W&amp;B Multi-tenant Cloud](/ja/platform/hosting/#wb-multi-tenant-cloud) は、アップグレード、メンテナンス、プラットフォーム セキュリティ、キャパシティ プランニングを含め、W&amp;B によって完全マネージドで提供されます。Multi-tenant Cloud は、[Google Cloud の北米リージョン](https://cloud.google.com/compute/docs/regions-zones) にある W&amp;B の Google Cloud アカウントにデプロイされています。[Bring your own bucket (BYOB)](/ja/platform/hosting/data-security/secure-storage-connector) を使用すると、W&amp;B Artifacts やその他の関連する機密データを、自社のクラウドまたはオンプレミスのインフラストラクチャーに保存することもできます。

[W&amp;B Multi-tenant Cloud](/ja/platform/hosting/#wb-multi-tenant-cloud) を参照するか、[無料で使い始める](https://app.wandb.ai/login?signup=true) をご覧ください。

<div id="wb-dedicated-cloud">
  ## W&amp;B Dedicated Cloud
</div>

[W&amp;B Dedicated Cloud](/ja/platform/hosting/#wb-dedicated-cloud) は、エンタープライズ組織向けに設計されたシングルテナントの完全マネージドプラットフォームです。W&amp;B Dedicated Cloud は、W&amp;B の AWS、Google Cloud、または Azure アカウント上にデプロイされます。専用クラウドは Multi-tenant Cloud よりも柔軟性が高く、W&amp;B セルフマネージドよりも複雑さが低いオプションを提供します。アップグレード、メンテナンス、プラットフォームのセキュリティ、キャパシティプランニングは W&amp;B が管理します。各専用クラウド インスタンスは、他の W&amp;B Dedicated Cloud インスタンスから独立したネットワーク、コンピュート、ストレージを備えています。

お客様固有の W&amp;B メタデータおよびデータは、独立したクラウドストレージに保存され、独立したクラウドコンピュートサービスで処理されます。[Bring your own bucket (BYOB)](/ja/platform/hosting/data-security/secure-storage-connector) をオプションで利用することで、Artifacts やその他の機密データを、お客様ご自身のクラウドまたはオンプレミスインフラストラクチャーに保存できます。

W&amp;B Dedicated Cloud には、重要なセキュリティ機能やその他のエンタープライズ向け機能をサポートする [エンタープライズライセンス](/ja/platform/hosting/hosting-options/self-managed/) が含まれています。

高度なセキュリティまたはコンプライアンス要件を持つ組織向けには、HIPAA 準拠、Single Sign On、Customer Managed Encryption Keys (CMEK) などの機能が **Enterprise** サポートとともに利用可能です。[詳細情報をリクエストする](https://wandb.ai/site/contact)。

[W&amp;B Dedicated Cloud](/ja/platform/hosting/#wb-dedicated-cloud) を参照するか、[無料で始めましょう](https://app.wandb.ai/login?signup=true)。

<div id="wb-self-managed">
  ## W&amp;B セルフマネージド
</div>

[W&amp;B セルフマネージド](/ja/platform/hosting/hosting-options/self-managed/) は、オンプレミス環境またはお客様が管理するクラウド インフラストラクチャー上で、お客様が全面的に管理します。IT/DevOps/MLOps チームは、以下を担当します。

* デプロイをプロビジョニングすること。
* 該当する場合は、組織のポリシーおよび [Security Technical Implementation Guidelines (STIG)](https://en.wikipedia.org/wiki/Security_Technical_Implementation_Guide) に従って、インフラストラクチャーを保護すること。
* アップグレードを管理し、パッチを適用すること。
* セルフマネージドの W&amp;B Server インスタンスを継続的に保守すること。

必要に応じて、W&amp;B セルフマネージドのエンタープライズ ライセンスを取得できます。エンタープライズ ライセンスには、重要なセキュリティ機能やその他のエンタープライズ向け機能に対するサポートが含まれます。

[W&amp;B セルフマネージド](/ja/platform/hosting/hosting-options/self-managed/) を参照するか、[リファレンスアーキテクチャ](/ja/platform/hosting/self-managed/ref-arch/) のガイドラインを確認してください。
