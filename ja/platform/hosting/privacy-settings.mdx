---
title: プライバシー設定を構成する
description: "W&B で、チームの可視性、project へのアクセス、コードの保存のデフォルトに関するプライバシー設定を構成し、適用します。"
---

このページでは、組織管理者とチーム管理者が W&amp;B でプライバシー設定を構成して、チームの可視性、project へのアクセス、招待、report の共有、デフォルトのコードの保存を制御する方法について説明します。これらの設定を使用すると、組織全体でプライバシー設定を標準化したり、単一のチーム向けに制御を調整したりできます。

組織管理者は組織スコープでプライバシー設定を構成でき、チーム管理者はチームスコープで構成できます。組織スコープで構成した場合、その設定はその組織内のすべての Teams に適用されます。

<Note>
  チームのワークフローに予期しない変更が生じるのを避けるため、W&amp;B は、組織管理者がプライバシー設定を適用する前に、その組織内のすべてのチーム管理者と Users に事前に周知することを推奨します。
</Note>

<div id="enforce-privacy-settings-for-all-teams">
  ## すべてのチームにプライバシー設定を適用する
</div>

組織管理者は、アカウントまたは組織のダッシュボードにある **Settings** タブの **Privacy** セクションから、組織内のすべてのチームにプライバシー設定を適用できます。組織管理者が設定を適用すると、チーム管理者は各チームでその設定を行えません。

* **チームの可視性制限を適用する**: 組織全体で、メンバー以外にはすべてのチームを非表示にします。
* **今後のprojectのプライバシーを適用する**: すべてのチームで新規作成されるすべてのprojectを、非公開または[restricted](./iam/access-management/restricted-projects)にすることを必須にします。
* **招待制御を適用する**: 管理者以外のメンバーがチームにメンバーを招待できないようにします。
* **レポート共有制御を適用する**: 非公開project内のレポートの公開共有を無効にし、既存のマジックリンクも無効化します。
* **チームへの自動参加制限を適用する**: 組織のメールドメインと一致するユーザーが、自動的にチームへ参加できないように制限します。
  <Note>この設定は、[Multi-tenant Cloud](./hosting-options/multi_tenant_cloud)デプロイでのみ利用できます。</Note>
* **デフォルトのコードの保存制限を適用する**: すべてのチームで、デフォルトでコードの保存を無効にします。

組織のプライバシー設定を開くには、次の手順に従います。

1. 組織管理者としてサインインします。
2. Multi-tenant Cloud では、`https://wandb.ai/account-settings/[ORGANIZATION]/settings` にアクセスします。`[ORGANIZATION]` はご自身の組織名に置き換えてください。専用クラウドまたはセルフマネージドのデプロイでは、インスタンスの組織ダッシュボードを開きます。環境固有のURLについては、[Add and manage users](/ja/platform/hosting/iam/access-management/manage-organization#add-and-manage-users)を参照してください。
3. **Privacy** セクションを開き、**Enforce default code saving restrictions** と、必要なその他の組織全体の制御を設定します。

変更を保存すると、適用された設定が組織内のすべてのチームに反映され、チーム管理者はチームスコープでそれらを上書きできなくなります。

<div id="configure-privacy-settings-for-a-team">
  ## チームのプライバシー設定を構成する
</div>

設定が組織スコープで強制されていない場合、チーム管理者はチームの **Settings** タブ内にある **Privacy** セクションから、自分のチームの設定を管理できます。各設定は、組織スコープで強制されていない限り構成可能です。

* **Hide this team from all non-members**: メンバー以外のユーザーにはこのチームを表示しません。
* **Make all future team projects private**: 今後チーム内で作成されるすべての新しい Projects を非公開にします (公開共有はできません) 。
* **Allow any team member to invite other members**: 管理者だけでなく、すべてのチームメンバーが新しいメンバーを招待できるようにします。
* **Turn off public sharing for reports in private projects**: 非公開の Projects 内の レポート について公開共有を無効にし、既存の magic links も無効にします。
* **Allow users with matching organization email domain to join this team**: 組織のメールドメインが一致するユーザーが、このチームに自動的に参加できるようにします。
  <Note>この設定は [Multi-tenant Cloud](./hosting-options/multi_tenant_cloud) deployment でのみ使用できます。</Note>
* **Enable code saving by default**: チーム内のすべての Runs でコードの保存を自動的に行います。

チームのプライバシー設定を開くには、次の手順に従います。

1. `https://wandb.ai/[TEAM]` にアクセスします。`[TEAM]` はチーム名に置き換えてください。
2. 左側のナビゲーションで **Team settings** を選択します。
3. **Privacy** セクションを開き、**Enable code saving by default** を構成します。

変更を保存すると、選択した設定がチームに適用されます。
