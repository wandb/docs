このページでは、[SCIM API](/ja/platform/hosting/iam/scim) と [Python SDK API](/ja/models/ref/python/public-api) を使ってユーザーとチームの管理を自動化する方法について説明します。

<div id="scim-api">
  ## SCIM API
</div>

W&amp;B SCIM API を使用して、Okta や Microsoft Entra などのアイデンティティ プロバイダー (IdP) と連携し、W&amp;B 組織のユーザーとTeamsを大規模に管理できます。

<Tip>
  W&amp;B の実装には、ロールの割り当てに加えて、カスタム ロールの作成と管理、ならびに組み込み ロールおよびカスタム ロールの割り当てを行うためのエンドポイントが含まれます。ロール エンドポイントは公式の SCIM スキーマの一部ではありません。W&amp;B は、カスタム ロールの自動管理をサポートするためにロール エンドポイントを追加しています。
</Tip>

以下のセクションでは、SCIM API の各カテゴリについて説明します。

<div id="user-scim-api">
  ### User SCIM API
</div>

[User SCIM API](./scim#user-resource) を使用すると、W&amp;B 組織内でユーザーを作成、無効化、取得、および一覧表示し、あらかじめ定義されたロールやカスタムロールを割り当てることができます。リクエストおよびレスポンスの完全な例については、詳細な [SCIM reference](./scim#user-management) を参照してください。

<Note>
  `PATCH /scim/Users/{id}` を使用してユーザーを無効化し、`{"active": false}` を設定します。結果はホスティングオプションによって異なります。専用クラウドとセルフマネージド デプロイメントではユーザー レコードが保持されますが、Multi-tenant Cloud では組織からユーザーが削除されます。Multi-tenant Cloud では再有効化はできないため、代わりにユーザーを再度追加してください。[Deactivate user](./scim#deactivate-user) および [Reactivate user](./scim#reactivate-user) を参照してください。
</Note>

<div id="group-scim-api">
  ### Group SCIM API
</div>

[Group SCIM API](./scim#group-resource) は、組織内の W&amp;B Teams を管理し、Teams の作成や削除を行うためのものです。既存のチームにユーザーを追加または削除するには、`PATCH Group` を使用します。

<Note>
  W&amp;B には、`同じロールを持つユーザーのグループ` という概念は存在しません。W&amp;B チームは group に近い概念であり、異なるロールを持つ多様なペルソナが、関連する一連のプロジェクト上で協調して作業できるようにします。Teams は、異なるユーザー グループで構成できます。チーム内の各ユーザーに、team admin、member、viewer、またはカスタムロールといったロールを割り当ててください。

  groups と W&amp;B Teams が類似しているため、W&amp;B は Group SCIM API のエンドポイントを W&amp;B チームに対応付けています。
</Note>

<div id="custom-role-api">
  ### カスタム ロール API
</div>

[Custom Role SCIM API](./scim#role-resource) を使用すると、組織内のカスタム ロールの作成、一覧取得、または更新を含む管理が可能です。

<Warning>
  カスタム ロールを削除する際は注意してください。

  `DELETE Role` エンドポイントを使用して、W&amp;B 組織内のカスタム ロールを削除します。この操作では、カスタム ロールが継承している事前定義のロールが、操作前にそのカスタム ロールが割り当てられているすべてのユーザーに割り当てられます。

  `PUT Role` エンドポイントを使用して、カスタム ロールが継承するロールを更新します。この操作は、カスタム ロール内の既存の、つまり非継承のカスタム権限には影響しません。
</Warning>

<div id="wb-python-sdk-api">
  ## W&amp;B Python SDK API
</div>

[W&amp;B Python SDK API](/ja/models/ref/python/public-api/api) を使用して、組織のユーザー、Teams、およびチームメンバーシップを管理します。

* [`User`](/ja/models/ref/python/public-api/user)
* [`Team`](/ja/models/ref/python/public-api/team)
* [`Member`](/ja/models/ref/python/public-api/member)
