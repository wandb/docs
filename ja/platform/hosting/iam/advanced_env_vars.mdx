---
title: 高度な IAM 設定
description: "SSO、セッション期間、OIDC、LDAP の設定に使用する環境変数で、W&B の高度な IAM オプションを設定します。"
---

基本的な [環境変数](../env-vars) に加えて、[Dedicated Cloud](/ja/platform/hosting/hosting-options/dedicated-cloud) または [Self-Managed](/ja/platform/hosting/hosting-options/self-managed) インスタンスの IAM オプションを設定するために、環境変数を使用できます。

IAM の要件に応じて、以下の環境変数をインスタンスに設定してください。

| Environment variable                | 説明                                                                                                                                                                                                                                        |
| ----------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `DISABLE_SSO_PROVISIONING`          | W&amp;B インスタンスでユーザーの自動プロビジョニングを無効にするには、これを `true` に設定します。                                                                                                                                                                                 |
| `SESSION_LENGTH`                    | デフォルトのユーザーセッション有効期限を変更したい場合、この変数に希望する時間数を設定します。たとえば、セッションの有効期限を 24 時間に設定するには、`SESSION_LENGTH` を `24` に設定します。デフォルト値は 720 時間です。                                                                                                             |
| `GORILLA_ENABLE_SSO_GROUPS_CLAIMS`  | OIDC ベースの SSO を使用する場合、OIDC グループに基づいてインスタンス内の W&amp;B チームメンバーシップを自動化するには、この変数を `true` に設定します。また、ユーザーの OIDC トークンに `groups` クレームを追加し、ユーザーが所属するすべてのチーム名を含む文字列配列としてフォーマットする必要があります。                                                            |
| `GORILLA_LDAP_GROUP_SYNC`           | LDAP ベースの SSO を使用している場合、LDAP グループに基づいてインスタンス内の W&amp;B チームメンバーシップを自動化するには、これを `true` に設定します。                                                                                                                                              |
| `GORILLA_OIDC_CUSTOM_SCOPES`        | OIDC ベースの SSO を使用している場合、W&amp;B インスタンスがアイデンティティ プロバイダーに要求すべき追加の [スコープ](https://auth0.com/docs/get-started/apis/scopes/openid-connect-scopes) を指定できます。W&amp;B は、これらのカスタムスコープによって SSO の機能をいかなる形でも変更しません。                                    |
| `GORILLA_OIDC_SECRET`               | OIDC ベースの SSO を使用していて、IdP が OIDC Client Secret を必要とする場合、この変数にそのシークレットを設定します。                                                                                                                                                              |
| `GORILLA_USE_IDENTIFIER_CLAIMS`     | OIDC ベースの SSO を使用している場合、アイデンティティ プロバイダーからの特定の OIDC クレームを用いて、ユーザー名およびフルネームを強制するには、この変数を `true` に設定します。設定した場合、強制したいユーザー名とフルネームが、それぞれ `preferred_username` クレームと `name` クレームに設定されていることを確認してください。ユーザー名には、英数字と、特殊文字としてアンダースコアおよびハイフンのみを使用できます。 |
| `GORILLA_DISABLE_PERSONAL_ENTITY`   | `true` に設定すると、[personal entities](/ja/models/support/difference_team_entity_user_entity_mean_me/) を無効化します。personal entity 内で新しいパーソナル プロジェクトが作成されるのを防ぎ、既存のパーソナル プロジェクトへの書き込みも禁止します。                                                           |
| `GORILLA_DISABLE_ADMIN_TEAM_ACCESS` | 組織またはインスタンスの管理者が、自分自身で W&amp;B チームに参加したり、自分を追加したりすることを制限するには、これを `true` に設定します。これにより、チーム内のプロジェクトにアクセスできるのが Data &amp; AI に関わる担当者のみであることを保証します。                                                                                            |
| `WANDB_IDENTITY_TOKEN_FILE`         | [identity federation](/ja/platform/hosting/iam/identity_federation/) 用に、JSON Web Token (JWT) が保存されるローカルディレクトリへの絶対パスを指定します。                                                                                                                   |

<Warning>
  W&amp;B は、`GORILLA_DISABLE_ADMIN_TEAM_ACCESS` のような一部の設定を有効化する前に、その影響を十分に理解したうえで慎重に対応することを推奨します。質問がある場合は、担当の W&amp;B チームにお問い合わせください。
</Warning>
