---
title: 高度な IAM 設定
description: "SSO、セッション長、OIDC、LDAP の設定に使用する環境変数で、W&B の高度な IAM オプションを設定します。"
---

基本的な [環境変数](../env-vars) に加えて、[Dedicated Cloud](/ja/platform/hosting/hosting-options/dedicated-cloud) または [Self-Managed](/ja/platform/hosting/hosting-options/self-managed) インスタンスの IAM オプションを環境変数で設定できます。

IAM の要件に応じて、以下の環境変数から必要なものをインスタンスに設定してください。

| Environment variable                | 説明                                                                                                                                                                                                                                              |
| ----------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `DISABLE_SSO_PROVISIONING`          | W&amp;B インスタンスでユーザーの自動プロビジョニングを無効にするには、この変数を `true` に設定します。                                                                                                                                                                                     |
| `SESSION_LENGTH`                    | デフォルトのユーザーセッション有効期限を変更する場合は、この変数に希望する時間数を設定します。たとえば、セッションの有効期限を 24 時間にするには、`SESSION_LENGTH` を `24` に設定します。デフォルト値は 720 時間です。                                                                                                                     |
| `GORILLA_ENABLE_SSO_GROUPS_CLAIMS`  | OIDC ベースの SSO を使用している場合は、この変数を `true` に設定すると、OIDC グループに基づいてインスタンス内の W&amp;B チームメンバーシップを自動化できます。また、ユーザーの OIDC トークンに `groups` クレームを追加する必要があります。このクレームは、ユーザーが所属するすべてのチーム名を含む文字列配列として指定してください。                                                      |
| `GORILLA_LDAP_GROUP_SYNC`           | LDAP ベースの SSO を使用している場合は、これを `true` に設定すると、LDAP グループに基づいてインスタンス内の W&amp;B チームメンバーシップを自動化できます。                                                                                                                                                   |
| `GORILLA_OIDC_CUSTOM_SCOPES`        | OIDC ベースの SSO を使用している場合は、W&amp;B インスタンスが identity provider に要求する追加の [scopes](https://auth0.com/docs/get-started/apis/scopes/openid-connect-scopes) を指定できます。W&amp;B は、これらのカスタム scopes によって SSO の機能を変更することはありません。                                 |
| `GORILLA_OIDC_SECRET`               | OIDC ベースの SSO を使用していて、IdP が OIDC Client Secret を必要とする場合は、この変数にその secret を設定します。                                                                                                                                                                 |
| `GORILLA_USE_IDENTIFIER_CLAIMS`     | OIDC ベースの SSO を使用している場合は、この変数を `true` に設定すると、identity provider の特定の OIDC クレームを使ってユーザー名とフルネームを強制できます。この設定を有効にする場合は、強制するユーザー名とフルネームが、それぞれ `preferred_username` クレームと `name` クレームに設定されていることを確認してください。ユーザー名に使用できるのは、英数字に加えて、特殊文字ではアンダースコアとハイフンのみです。 |
| `GORILLA_DISABLE_PERSONAL_ENTITY`   | `true` に設定すると、[personal entities](/ja/models/support/difference_team_entity_user_entity_mean_me/) を無効にします。これにより、personal entities 内で新しい personal projects を作成できなくなり、既存の personal projects への書き込みも防止されます。                                           |
| `GORILLA_DISABLE_ADMIN_TEAM_ACCESS` | これを `true` に設定すると、Organization または Instance Admin が自分自身で W&amp;B チームに参加したり、自分を追加したりすることを制限できます。これにより、Teams 内のプロジェクトには Data &amp; AI 担当者のみが access できるようになります。                                                                                   |
| `WANDB_IDENTITY_TOKEN_FILE`         | [identity federation](/ja/platform/hosting/iam/identity_federation/) のために、Java Web Tokens (JWTs) が保存されているローカルディレクトリへの絶対パスを指定します。                                                                                                                   |

<Warning>
  W&amp;B では、`GORILLA_DISABLE_ADMIN_TEAM_ACCESS` のような一部の設定を有効にする前に、影響を十分に理解したうえで慎重に判断することを推奨しています。ご不明な点があれば、担当の W&amp;B チームまでお問い合わせください。
</Warning>
