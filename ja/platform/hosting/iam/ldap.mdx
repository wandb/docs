---
title: LDAP を使用して SSO を設定する
description: "接続パラメーターと環境変数を含む、W&B Server の LDAP ベースの SSO 認証を設定します。"
---

W&amp;B Server の LDAP サーバーを使用して認証します。次のガイドでは、W&amp;B Server の設定方法を説明します。必須および任意の設定に加え、システム設定 UI から LDAP 接続を設定する手順も紹介します。また、アドレス、ベース識別名、属性など、LDAP 設定の各入力項目に関する情報も提供します。これらの属性は、W&amp;B App UI から指定することも、環境変数を使用して指定することもできます。匿名バインドを設定することも、管理者 DN とパスワードを使用してバインドすることもできます。

{/* <Note>
  W&B Team Admin は、匿名バインド、または管理者DNとパスワードを使用するバインドのいずれかを設定できます。
  </Note> */}

<Note>
  LDAP 認証を有効にして設定できるのは、W&amp;B Admin ロールのみです。
</Note>


<div id="configure-ldap-connection">
  ## LDAP 接続を設定する
</div>

<Tabs>
  <Tab title="W&B App">
    1. W&amp;B App にアクセスします。
    2. 右上のプロフィールアイコンを選択します。ドロップダウンから **System Settings** を選択します。
    3. **Configure LDAP Client** をオンにします。
    4. フォームに必要な情報を入力します。各入力項目の詳細については、**Configuring Parameters** セクションを参照してください。
    5. **Update Settings** をクリックして設定をテストします。これにより、W&amp;B Server へのテスト用クライアント/接続が確立されます。
    6. 接続が確認できたら、**Enable LDAP Authentication** をオンにして、**Update Settings** ボタンを選択します。
  </Tab>

  <Tab title="環境変数">
    以下の環境変数を使用して LDAP 接続を設定します。

    | 環境変数                          | 必須  | 例                               |
    | ----------------------------- | --- | ------------------------------- |
    | `LOCAL_LDAP_ADDRESS`          | Yes | `ldaps://ldap.example.com:636`  |
    | `LOCAL_LDAP_BASE_DN`          | Yes | `email=mail,group=gidNumber`    |
    | `LOCAL_LDAP_BIND_DN`          | No  | `cn=admin`, `dc=example,dc=org` |
    | `LOCAL_LDAP_BIND_PW`          | No  |                                 |
    | `LOCAL_LDAP_ATTRIBUTES`       | Yes | `email=mail`, `group=gidNumber` |
    | `LOCAL_LDAP_TLS_ENABLE`       | No  |                                 |
    | `LOCAL_LDAP_GROUP_ALLOW_LIST` | No  |                                 |
    | `LOCAL_LDAP_LOGIN`            | No  |                                 |

    各環境変数の定義については、[設定パラメーター](#configuration-parameters) セクションを参照してください。わかりやすくするため、定義名では環境変数プレフィックス `LOCAL_LDAP` を省略しています。
  </Tab>
</Tabs>

<div id="configuration-parameters">
  ## 設定パラメーター
</div>

次の表は、必須および任意の LDAP 設定を示したものです。

| 環境変数 | Definition                                                                                                                                             | 必須 |
| -------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ | -------- |
| `ADDRESS`            | W&amp;B Server をホストしている VPC 内の LDAP サーバーのアドレスです。                                                                                                       | はい       |
| `BASE_DN`            | 検索の開始位置となるルートパスです。このディレクトリーに対してクエリーを実行する際に必要です。                                                                                                        | はい       |
| `BIND_DN`            | LDAP サーバーに登録されている管理ユーザーのパスです。LDAP サーバーが認証なしバインドをサポートしていない場合は必須です。指定すると、W&amp;B Server はこのユーザーとして LDAP サーバーに接続します。指定しない場合、W&amp;B Server は匿名バインドで接続します。 | いいえ      |
| `BIND_PW`            | 管理ユーザーのパスワードです。バインドの認証に使用します。空欄のままにすると、W&amp;B Server は匿名バインドで接続します。                                                                                   | いいえ      |
| `ATTRIBUTES`         | メールアドレス属性名とグループ ID 属性名を、カンマ区切りの文字列として指定します。                                                                                                            | はい       |
| `TLS_ENABLE`         | TLS を有効にします。                                                                                                                                           | いいえ      |
| `GROUP_ALLOW_LIST`   | 許可するグループのリストです。                                                                                                                                        | いいえ      |
| `LOGIN`              | W&amp;B Server に LDAP を使用して認証するよう指定します。`True` または `False` を設定してください。必要に応じて、LDAP 設定をテストするために `false` に設定できます。LDAP 認証を開始するには、`true` に設定します。              | いいえ      |
