title: LDAP で SSO を設定する
description: "接続パラメーターと環境変数を含む、W&B Server の LDAP ベースの SSO 認証を設定します。"
---

W&amp;B Server の LDAP サーバーを使用して認証情報を認証します。以下のガイドでは、W&amp;B Server の設定方法について説明します。必須および任意の設定項目に加え、System Settings UI から LDAP 接続を設定する手順も扱います。また、アドレス、ベース識別名、属性など、LDAP 設定の各入力項目に関する情報も紹介します。これらの属性は、W&amp;B App UI または環境変数を使用して指定できます。匿名バインド、または管理者 DN とパスワードを使用するバインドのいずれかを設定できます。

{/* <Note>
  W&B Team Admin であれば、匿名バインド、または管理者 DN とパスワードを使用するバインドのいずれかを設定できます。
  </Note> */}

<Note>
  LDAP 認証を有効にして設定できるのは、W&amp;B Admin ロールのみです。
</Note>


<div id="configure-ldap-connection">
  ## LDAP 接続を設定する
</div>

<Tabs>
  <Tab title="W&B App">
    1. W&amp;B App にアクセスします。
    2. 右上のプロフィールアイコンを選択し、ドロップダウンから **System Settings** を選択します。
    3. **Configure LDAP Client** をオンにします。
    4. フォームに必要な情報を入力します。各入力項目の詳細については **Configuring Parameters** セクションを参照してください。
    5. 設定をテストするには **Update Settings** をクリックします。これにより W&amp;B サーバーとのテストクライアント／接続が確立されます。
    6. 接続が検証されたら、**Enable LDAP Authentication** をオンにし、**Update Settings** ボタンをクリックします。
  </Tab>

  <Tab title="環境変数">
    次の環境変数を使用して LDAP 接続を設定します:

    | 環境変数                          | 必須  | 例                               |
    | ----------------------------- | --- | ------------------------------- |
    | `LOCAL_LDAP_ADDRESS`          | はい  | `ldaps://ldap.example.com:636`  |
    | `LOCAL_LDAP_BASE_DN`          | はい  | `email=mail,group=gidNumber`    |
    | `LOCAL_LDAP_BIND_DN`          | いいえ | `cn=admin`, `dc=example,dc=org` |
    | `LOCAL_LDAP_BIND_PW`          | いいえ |                                 |
    | `LOCAL_LDAP_ATTRIBUTES`       | はい  | `email=mail`, `group=gidNumber` |
    | `LOCAL_LDAP_TLS_ENABLE`       | いいえ |                                 |
    | `LOCAL_LDAP_GROUP_ALLOW_LIST` | いいえ |                                 |
    | `LOCAL_LDAP_LOGIN`            | いいえ |                                 |

    各環境変数の定義については、[設定パラメーター](#configuration-parameters) セクションを参照してください。わかりやすくするために、定義で使用している名称から環境変数プレフィックス `LOCAL_LDAP` は省略されています。
  </Tab>
</Tabs>

<div id="configuration-parameters">
  ## 設定パラメーター
</div>

次の表は、必須および任意の LDAP 設定を一覧し、それぞれの説明を示します。

| 環境変数               | 定義                                                                                                                                                          | 必須  |
| ------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------- | --- |
| `ADDRESS`          | W&amp;B Server をホストしている VPC 内の LDAP サーバーのアドレスです。                                                                                                            | Yes |
| `BASE_DN`          | 検索の開始元となるルートパスです。このディレクトリに対してクエリを実行するには必須です。                                                                                                                | Yes |
| `BIND_DN`          | LDAP サーバーに登録されている管理者ユーザーのパスです。LDAP サーバーが未認証バインドをサポートしていない場合に必要です。指定した場合、W&amp;B Server はこのユーザーとして LDAP サーバーに接続します。指定しない場合、W&amp;B Server は匿名バインドを使用して接続します。 | No  |
| `BIND_PW`          | 管理者ユーザーのパスワードです。バインドの認証に使用します。空のままにすると、W&amp;B Server は匿名バインドを使用して接続します。                                                                                    | No  |
| `ATTRIBUTES`       | メールアドレスおよびグループ ID の属性名を、カンマ区切りの文字列として指定します。                                                                                                                 | Yes |
| `TLS_ENABLE`       | TLS を有効にします。                                                                                                                                                | No  |
| `GROUP_ALLOW_LIST` | グループの許可リストです。                                                                                                                                               | No  |
| `LOGIN`            | W&amp;B Server で LDAP を使用して認証するかどうかを指定します。`True` または `False` を設定します。LDAP 設定をテストする場合は、必要に応じて `False` に設定できます。LDAP 認証を開始するには、`True` に設定します。                   | No  |
