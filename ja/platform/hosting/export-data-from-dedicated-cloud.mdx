description: "W&B SDK API を使用して、W&B 専用クラウドのインスタンスから run、メトリクス、Artifacts、Reports をエクスポートします。"
title: 専用クラウドからデータをエクスポートする
---

専用クラウドのインスタンスで管理されているすべてのデータをエクスポートする場合は、[Import and Export API](/ja/models/ref/python/public-api/) を使用して、W&amp;B SDK API から run、メトリクス、Artifacts などを抽出できます。次の表では、主なエクスポートのユースケースをいくつか示します。

| 目的                     | ドキュメント                                                                                                                                                                                               |
| ---------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| プロジェクトのメタデータをエクスポートする  | [Projects API](/ja/models/ref/python/public-api/projects)                                                                                                                                               |
| プロジェクト内の run をエクスポートする | [Runs API](/ja/models/ref/python/public-api/runs)                                                                                                                                                       |
| Reports をエクスポートする      | [Report and Workspace API](/ja/models/reports/clone-and-export-reports/)                                                                                                                                |
| Artifacts をエクスポートする    | [artifact グラフを探索する](/ja/models/artifacts/explore-and-traverse-an-artifact-graph/), [Artifacts をダウンロードして使用する](/ja/models/artifacts/download-and-use-an-artifact/#download-and-use-an-artifact-stored-on-wb) |

[Secure Storage Connector](/ja/platform/app/settings-page/teams/#secure-storage-connector) を使用して専用クラウドに保存されている Artifacts を管理している場合は、W&amp;B SDK API を使用して Artifacts をエクスポートする必要がないことがあります。

<Note>
  W&amp;B SDK API を使用してすべてのデータをエクスポートすると、run や Artifacts などの数が多い場合は時間がかかることがあります。W&amp;B では、専用クラウドのインスタンスに過大な負荷をかけないよう、適切なサイズのバッチに分けてエクスポート処理を実行することを推奨しています。
</Note>
