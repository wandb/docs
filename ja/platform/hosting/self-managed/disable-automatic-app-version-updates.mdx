---
title: W&B Server の自動アップデートを無効化する
description: W&B Server の自動アップデートを無効化する方法を説明します。
---

このページでは、W&amp;B Server の自動バージョンアップグレードを無効化し、バージョンを固定する方法を説明します。ここでの手順は、[W&amp;B Kubernetes Operator](/ja/platform/hosting/operator/) によって管理されているデプロイメントにのみ有効です。

<Note>
  W&amp;B は、W&amp;B Server のメジャーリリースを初回リリース日から 12 か月間サポートします。**Self-Managed** インスタンスを利用しているお客様は、サポートを維持するために適切なタイミングでアップグレードする責任があります。サポート対象外のバージョンを使い続けないでください。W&amp;B は、**Self-Managed** インスタンスを利用しているお客様に対し、サポートを維持し、最新の機能・パフォーマンス改善・修正を受け取るために、少なくとも四半期に一度はデプロイメントを最新リリースに更新することを強く推奨します。
</Note>

<div id="requirements">
  ## 必要条件
</div>

* W&amp;B Kubernetes Operator `v1.13.0` 以降
* System Console `v2.12.2` 以降

これらの要件を満たしていることを確認するには、対象インスタンスの W&amp;B Custom Resource または Helm チャートを参照し、`operator-wandb` および `system-console` コンポーネントの `version` の値を確認してください。

<div id="disable-automatic-updates">
  ## 自動アップデートを無効にする
</div>

1. `admin` ロールを持つユーザーとして W&amp;B App にログインします。
2. 上部のユーザーアイコンをクリックし、**System Console** をクリックします。
3. **Settings** &gt; **Advanced** に移動し、**Other** タブを選択します。
4. **Disable Auto Upgrades** セクションで、**Pin specific version** をオンにします。
5. **Select a version** ドロップダウンをクリックし、W&amp;B Server のバージョンを選択します。
6. **Save** をクリックします。

   <Frame>
     <img src="/images/hosting/disable_automatic_updates_saved_and_enabled.png" alt="Disable Automatic Updates Saved" />
   </Frame>

   自動アップグレードが無効になり、W&amp;B Server は選択したバージョンにピン留めされます。
7. 自動アップグレードが無効になっていることを確認します。**Operator** タブに移動し、reconciliation ログで `Version pinning is enabled` という文字列を検索します。

```
│info 2025-04-17T17:24:16Z wandb default No changes found
│info 2025-04-17T17:24:16Z wandb default Active spec found
│info 2025-04-17T17:24:16Z wandb default Desired spec
│info 2025-04-17T17:24:16Z wandb default License
│info 2025-04-17T17:24:16Z wandb default Version Pinning is enabled
│info 2025-04-17T17:24:16Z wandb default Found Weights & Biases instance, processing the spec...
│info 2025-04-17T17:24:16Z wandb default === Reconciling Weights & Biases instance...
```
