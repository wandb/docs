---
title: W&B Server の自動更新を無効化する
description: W&B Server の自動更新を無効にする方法を説明します。
---

このページでは、W&amp;B Server の自動バージョンアップグレードを無効化し、バージョンを固定する方法を説明します。この手順は、[W&amp;B Kubernetes Operator](/ja/platform/hosting/operator/) によって管理されているデプロイにのみ適用されます。

<Note>
  W&amp;B は、W&amp;B Server の各メジャーリリースを初回リリース日から 12 か月間サポートします。**セルフマネージド** インスタンスをお使いのお客様は、サポートを維持するために期限内にアップグレードする責任があります。サポート対象外のバージョンを使い続けないでください。W&amp;B は、**セルフマネージド** インスタンスをお使いのお客様に対し、サポートを維持し、最新の機能、パフォーマンス改善、および修正を受け取るために、少なくとも四半期に一度はデプロイを最新リリースに更新することを強く推奨します。
</Note>

<div id="requirements">
  ## 要件
</div>

* W&amp;B Kubernetes Operator `v1.13.0` 以上
* System Console `v2.12.2` 以上

これらの要件を満たしているか確認するには、対象インスタンスの W&amp;B Custom Resource または Helm チャートを参照してください。`operator-wandb` と `system-console` コンポーネントの `version` 値を確認してください。

<div id="disable-automatic-updates">
  ## 自動更新を無効にする
</div>

1. `admin` ロールを持つユーザーとして W&amp;B App にログインします。
2. 画面上部のユーザーアイコンをクリックし、**System Console** をクリックします。
3. **Settings** &gt; **Advanced** にアクセスし、**Other** タブを選択します。
4. **Disable Auto Upgrades** セクションで **Pin specific version** をオンにします。
5. **Select a version** ドロップダウンをクリックし、W&amp;B Server のバージョンを選択します。
6. **Save** をクリックします。

   <Frame>
     <img src="/images/hosting/disable_automatic_updates_saved_and_enabled.png" alt="自動更新の無効化が保存され、有効になっています" />
   </Frame>

   自動更新はオフになり、W&amp;B Server は選択したバージョンに固定されます。
7. 自動更新がオフになっていることを確認します。**Operator** タブにアクセスし、reconciliation ログ内で `Version pinning is enabled` という文字列を検索します。

```
│info 2025-04-17T17:24:16Z wandb default No changes found
│info 2025-04-17T17:24:16Z wandb default Active spec found
│info 2025-04-17T17:24:16Z wandb default Desired spec
│info 2025-04-17T17:24:16Z wandb default License
│info 2025-04-17T17:24:16Z wandb default Version Pinning is enabled
│info 2025-04-17T17:24:16Z wandb default Found Weights & Biases instance, processing the spec...
│info 2025-04-17T17:24:16Z wandb default === Reconciling Weights & Biases instance...
```
