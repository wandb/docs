W&amp;B Multi-tenant Cloud は、[Google Cloud の北米リージョン](https://cloud.google.com/compute/docs/regions-zones) にある W&amp;B の Google Cloud アカウント上にデプロイされる、フルマネージド プラットフォームです。W&amp;B Multi-tenant Cloud は Google Cloud のオートスケーリング機能を利用しており、トラフィックの増減に応じてプラットフォームが適切にスケールするようになっています。

<Frame>
  <img src="/images/hosting/multi_tenant_cloud_arch.png" alt="Multi-tenant Cloud のアーキテクチャ図" />
</Frame>

W&amp;B Multi-tenant Cloud は、組織のニーズに合わせてスケールし、1 つのプロジェクトあたり最大 250,000 のメトリクスを、各メトリクスあたり最大 100 万データポイントまでのログの記録をサポートします。より大規模なデプロイが必要な場合は、[サポート](mailto:support@wandb.com) までお問い合わせください。

<div id="compliance">
  ## コンプライアンス
</div>

W&amp;B Multi-tenant Cloud のホスティングプラットフォームは、[Service and Organization Controls (SOC) 2 Type 2](https://www.aicpa-cima.com/topic/audit-assurance/audit-and-assurance-greater-than-soc-2)、[Auditing Standards Board of the American Institute of Certified Public Accountants (AICPA)](https://www.aicpa-cima.com/home) の発行する要件を満たしています。SOC 2 レポートでは、サービス組織のセキュリティ、可用性、処理の完全性、機密性、およびプライバシーに関する統制が評価されます。W&amp;B Multi-tenant Cloud は、準拠が継続していることを検証するため、定期的な内部および外部監査の対象となります。SOC 2 レポートやその他のセキュリティおよびコンプライアンス関連文書をリクエストするには、[W&amp;B Security Portal](https://security.wandb.ai/) からリクエストしてください。

<Tip>W&amp;B Multi-tenant Cloud は、[Health Insurance Portability and Accountability Act of 1996 (HIPAA)](https://www.hhs.gov/hipaa/for-professionals/index.html) の要件を満たしていません。組織が HIPAA の適用対象である場合は、代わりに [W&amp;B Dedicated Cloud](/ja/platform/hosting/hosting-options/dedicated-cloud) の利用を検討してください。</Tip>

<div id="data-security">
  ## データセキュリティ
</div>

Free または Pro プランのユーザーの場合、すべてのデータは共有クラウドストレージにのみ保存され、共有クラウドコンピュートサービスで処理されます。ご利用の料金プランによっては、ストレージ容量の制限が適用される場合があります。

Enterprise プランのユーザーは、[secure storage connector を使用して独自のバケット (BYOB) を持ち込む](/ja/platform/hosting/data-security/secure-storage-connector) ことで、モデル、データセットなどのファイルを [Teams レベル](/ja/platform/hosting/data-security/secure-storage-connector#configuration-options) で保存できます。1 つのバケットを複数の Teams で共有することも、異なる W&amp;B Teams ごとに個別のバケットを使用することもできます。特定の Teams に BYOB を設定しない場合、その Teams のデータは共有クラウドストレージに保存されます。

お使いのデプロイメントが、組織のポリシーおよび (該当する場合は) [Security Technical Implementation Guidelines (STIG)](https://en.wikipedia.org/wiki/Security_Technical_Implementation_Guide) に準拠していることを確認する責任は、お客様にあります。

<div id="data-retention-policy">
  ## データ保持ポリシー
</div>

マルチテナント Cloud では、削除後も 7 日間、次の項目を保持します。

* Runs と履歴
* メディア、設定ファイル、ログファイルなどの非 Artifacts run ファイル
* Artifacts および Artifacts 参照

この期間が経過するまでは、これらの項目を復元できます。サポートが必要な場合は、[support](mailto:support@wandb.ai) もしくは担当の AISE に連絡してください。

<div id="identity-and-access-management-iam">
  ## アイデンティティおよびアクセス管理 (IAM)
</div>

Enterprise プランをご利用の場合、強化されたアイデンティティおよびアクセス管理機能により、W&amp;B デプロイメントに対して安全な認証と適切な認可を実現できます。

* OIDC または SAML を用いた SSO 認証。組織向けに SSO を構成したい場合は、W&amp;B チームまたはサポートにお問い合わせください。
* 組織およびチームのレベルで[適切なユーザー ロールを構成](/ja/platform/hosting/iam/access-management/manage-organization#assign-or-update-a-users-role)します。
* [Restricted projects](/ja/platform/hosting/iam/access-management/restricted-projects) を使用して W&amp;B プロジェクトのスコープを定義し、そのプロジェクトを閲覧・編集したり W&amp;B run を送信したりできるユーザーを制限します。

<div id="monitor">
  ## 監視
</div>

Organization の管理者は、自身のアカウント画面の `Billing` タブから、アカウントの利用状況および請求を管理できます。Multi-tenant Cloud 上の共有クラウド ストレージを使用している場合、管理者は組織内の異なる Teams 間でストレージ使用量を最適化できます。

<div id="maintenance">
  ## メンテナンス
</div>

W&amp;B Multi-tenant Cloud は、マルチテナントかつフルマネージドなプラットフォームです。W&amp;B Multi-tenant Cloud は W&amp;B によって管理されるため、W&amp;B プラットフォームのプロビジョニングや運用・保守にかかるオーバーヘッドやコストを負担する必要はありません。
