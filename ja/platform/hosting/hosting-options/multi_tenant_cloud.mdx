W&amp;B Multi-tenant Cloud は、[Google Cloud の北米リージョン](https://cloud.google.com/compute/docs/regions-zones)にある W&amp;B の Google Cloud Account にデプロイされた完全マネージドなプラットフォームです。W&amp;B Multi-tenant Cloud は Google Cloud のオートスケーリングを利用して、トラフィックの増減に基づいてプラットフォームが適切にスケールするようにします。

<Frame>
  <img src="/images/hosting/multi_tenant_cloud_arch.png" alt="Multi-tenant Cloud のアーキテクチャ図" />
</Frame>

W&amp;B Multi-tenant Cloud は組織のニーズに合わせてスケールし、1 つのプロジェクトあたり最大 250,000 件のメトリクスをサポートし、各メトリクスには最大 100 万件のデータポイントを記録できます。より大規模なデプロイについては、[サポート](mailto:support@wandb.com) までお問い合わせください。

<div id="compliance">
  ## コンプライアンス
</div>

W&amp;B Multi-tenant Cloud のホスティングプラットフォームは、[米国公認会計士協会 (AICPA) の Auditing Standards Board](https://www.aicpa-cima.com/home) が公表する [Service and Organization Controls (SOC) 2 Type 2](https://www.aicpa-cima.com/topic/audit-assurance/audit-and-assurance-greater-than-soc-2) の要件を満たしています。SOC 2 レポートでは、サービス組織におけるセキュリティ、可用性、処理の完全性、機密保持、およびプライバシーに関する統制を評価します。W&amp;B Multi-tenant Cloud は、継続的なコンプライアンスを確認するため、定期的に内部監査および外部監査を受けています。SOC 2 レポートやその他のセキュリティおよびコンプライアンス関連文書をリクエストするには、[W&amp;B Security Portal](https://security.wandb.ai/) を参照してください。

<Tip>W&amp;B Multi-tenant Cloud は、[Health Insurance Portability and Accountability Act of 1996 (HIPAA)](https://www.hhs.gov/hipaa/for-professionals/index.html) の要件を満たしていません。組織が HIPAA の対象となる場合は、代わりに [W&amp;B Dedicated Cloud](/ja/platform/hosting/hosting-options/dedicated-cloud) の利用を検討してください。</Tip>

<div id="data-security">
  ## データセキュリティ
</div>

Free または Pro プランのユーザーの場合、すべてのデータは共有クラウドストレージにのみ保存され、共有クラウドコンピュートサービスで処理されます。ご利用の料金プランによっては、ストレージ制限が適用される場合があります。

Enterprise プランのユーザーは、[セキュアストレージコネクターを使用して独自の버킷 (BYOB) を使用する](/ja/platform/hosting/data-security/secure-storage-connector) ことで、[チームレベル](/ja/platform/hosting/data-security/secure-storage-connector#configuration-options) でモデル、データセットなどのファイルを保存できます。複数のチームに対して 1 つの버킷を設定することも、異なる W&amp;B Teams ごとに個別の버킷を使用することもできます。チームに対して BYOB を設定しない場合、そのチームのデータは共有クラウドストレージに保存されます。

お使いのデプロイが組織のポリシーおよび (該当する場合は) [Security Technical Implementation Guidelines (STIG)](https://en.wikipedia.org/wiki/Security_Technical_Implementation_Guide) に準拠していることを確認する責任は、お客様にあります。

<div id="data-retention-policy">
  ## データ保持ポリシー
</div>

Multi-tenant Cloud では、削除後も 7 日間、次の項目を保持します。

* Runs と履歴
* メディア、設定ファイル、ログファイルなどのアーティファクト以外の run ファイル
* Artifacts およびアーティファクトの参照

この期間が経過するまでは、これらの項目を復元できます。支援が必要な場合は、[サポート](mailto:support@wandb.ai) または担当の AISE にお問い合わせください。

<div id="identity-and-access-management-iam">
  ## アイデンティティおよび액세스管理 (IAM)
</div>

Enterprise プランをお使いの場合、強化されたアイデンティティおよび액세스管理機能により、W&amp;B デプロイに対して安全な認証と効果的な認可を実現できます。

* OIDC または SAML を用いた SSO 認証。組織向けに SSO を構成したい場合は、W&amp;B チームまたはサポートにお問い合わせください。
* 組織のスコープおよびチーム内で[適切なユーザー ロールを構成する](/ja/platform/hosting/iam/access-management/manage-organization#assign-or-update-a-users-role)。
* [restricted projects](/ja/platform/hosting/iam/access-management/restricted-projects) を使用して W&amp;B プロジェクトのスコープを定義し、そのプロジェクトの閲覧、編集、または W&amp;B Runs の送信が可能なユーザーを制限する。

<div id="monitor">
  ## Monitor
</div>

組織管理者は、accountビューの`Billing`タブから、自身のaccountのusageと결제を管理できます。Multi-tenant Cloud の共有クラウドストレージを使用している場合、管理者は組織内の異なるチーム全体でstorage usageを最適化できます。

<div id="maintenance">
  ## メンテナンス
</div>

W&amp;B Multi-tenant Cloud は、マルチテナントで完全マネージドのプラットフォームです。W&amp;B Multi-tenant Cloud は W&amp;B によって管理されるため、W&amp;B プラットフォームのプロビジョニングや運用・保守にかかるオーバーヘッドやコストを負担する必要はありません。
