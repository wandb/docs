W&amp;B Multi-tenant Cloud は、[Google Cloud の北米リージョン](https://cloud.google.com/compute/docs/regions-zones)にある W&amp;B の Google Cloud アカウントにデプロイされる完全マネージドのプラットフォームです。W&amp;B Multi-tenant Cloud は Google Cloud のオートスケーリングを利用しており、トラフィックの増減に応じてプラットフォームを適切にスケールさせます。

<Frame>
  <img src="/images/hosting/multi_tenant_cloud_arch.png" alt="Multi-tenant Cloud アーキテクチャ図" />
</Frame>

W&amp;B Multi-tenant Cloud は組織のニーズに合わせてスケールし、各プロジェクトで最大 250,000 件のメトリクスをログでき、各メトリクスあたり最大 100 万件のデータポイントをサポートします。さらに大規模なデプロイについては、[サポート](mailto:support@wandb.com) までお問い合わせください。

<div id="compliance">
  ## コンプライアンス
</div>

W&amp;B Multi-tenant Cloud のホスティングプラットフォームは、[米国公認会計士協会 (AICPA) の監査基準審議会](https://www.aicpa-cima.com/home)が公表する [Service and Organization Controls (SOC) 2 Type 2](https://www.aicpa-cima.com/topic/audit-assurance/audit-and-assurance-greater-than-soc-2) の要件を満たしています。SOC 2 レポートでは、セキュリティ、可用性、処理の完全性、機密性、プライバシーに関するサービス組織の統制を評価します。W&amp;B Multi-tenant Cloud は、継続的なコンプライアンスを検証するため、定期的に内部監査および外部監査を受けています。SOC 2 レポートやその他のセキュリティ／コンプライアンス関連文書をリクエストするには、[W&amp;B Security Portal](https://security.wandb.ai/) を参照してください。

<Tip>W&amp;B Multi-tenant Cloud は、[1996年医療保険の相互運用性と説明責任に関する法律 (HIPAA) ](https://www.hhs.gov/hipaa/for-professionals/index.html) の要件を満たしていません。お使いの組織が HIPAA の適用対象である場合は、代わりに [W&amp;B Dedicated Cloud](/ja/platform/hosting/hosting-options/dedicated-cloud) をご検討ください。</Tip>

<div id="data-security">
  ## データセキュリティ
</div>

Free または Pro プランのユーザーのデータは、すべて共有クラウドストレージにのみ保存され、共有クラウドコンピューティングサービスで処理されます。料金プランによっては、ストレージ制限が適用される場合があります。

Enterprise プランのユーザーは、ファイル (モデル、データセットなど) を保存するために、[セキュアストレージコネクタを使用して独自の bucket (BYOB) を持ち込む](/ja/platform/hosting/data-security/secure-storage-connector)ことを[チームレベル](/ja/platform/hosting/data-security/secure-storage-connector#configuration-options)で行えます。複数の Teams に対して 1 つの bucket を設定することも、異なる W&amp;B Teams ごとに別々の bucket を使用することもできます。チームに BYOB を設定しない場合、そのチームのデータは共有クラウドストレージに保存されます。

該当する場合は、ご自身のデプロイが所属組織のポリシーおよび [Security Technical Implementation Guidelines (STIG)](https://en.wikipedia.org/wiki/Security_Technical_Implementation_Guide) に準拠していることを確認する責任があります。

<div id="data-retention-policy">
  ## データ保持ポリシー
</div>

Multi-tenant Cloud では、削除後 7 日間、次の項目が保持されます。

* Runs と履歴
* メディア、設定ファイル、ログファイルなど、artifact 以外の run ファイル
* Artifacts と artifact 参照

この期間が経過するまでは、これらの項目を復元できます。[サポート](mailto:support@wandb.ai) または担当の AISE にお問い合わせください。

<div id="identity-and-access-management-iam">
  ## アイデンティティおよびアクセス管理 (IAM)
</div>

Enterprise プランをご利用の場合、強化されたアイデンティティおよびアクセス管理機能により、W&amp;B のデプロイで安全な認証と効果的な認可を実現できます。

* OIDC または SAML による SSO 認証。組織向けに SSO を設定する場合は、担当の W&amp;B チームまたはサポートにお問い合わせください。
* 組織レベルおよびチーム内で、[適切なユーザーロールを設定する](/ja/platform/hosting/iam/access-management/manage-organization#assign-or-update-a-users-role)。
* [制限付きプロジェクト](/ja/platform/hosting/iam/access-management/restricted-projects)を使用して W&amp;B プロジェクトのアクセス範囲を定義し、そのプロジェクトを閲覧、編集、W&amp;B run の送信を行えるユーザーを制限します。

<div id="monitor">
  ## モニタリング
</div>

組織管理者は、アカウントビューの`Billing`タブから、自身のアカウントの使用状況と請求を管理できます。Multi-tenant Cloud の共有クラウドストレージを使用している場合、管理者は組織内の複数のTeamsにまたがるストレージ使用量を最適化できます。

<div id="maintenance">
  ## メンテナンス
</div>

W&amp;B Multi-tenant Cloud は、マルチテナント型の完全マネージドプラットフォームです。W&amp;B Multi-tenant Cloud は W&amp;B によって管理されているため、W&amp;B プラットフォームのプロビジョニングや保守にかかる運用負荷やコストを負う必要はありません。
