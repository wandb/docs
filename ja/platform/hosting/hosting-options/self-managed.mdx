<Tip>
  W&amp;B では、[W&amp;B Multi-tenant Cloud](/ja/platform/hosting/hosting-options/multi_tenant_cloud) や [W&amp;B Dedicated Cloud](/ja/platform/hosting/hosting-options/dedicated-cloud) などの完全マネージド型デプロイメントオプションを推奨しています。W&amp;B の完全マネージドサービスは、必要な設定がほとんど、あるいはまったくなく、シンプルかつ安全に使用できます。
</Tip>

W&amp;B Server は、お客様の [AWS、Google Cloud、または Azure のクラウドアカウント](#deploy-wb-server-within-Self-Managed-cloud-accounts) または [オンプレミスのインフラストラクチャー](#deploy-wb-server-in-on-prem-infrastructure) 内にデプロイできます。

お客様の IT/DevOps/MLOps チームは、以下について責任を負います。

* デプロイメントのプロビジョニング。
* 組織のポリシー、および該当する場合は [Security Technical Implementation Guidelines (STIG)](https://en.wikipedia.org/wiki/Security_Technical_Implementation_Guide) に従ったインフラストラクチャーの保護。
* [Service and Organization Controls (SOC) 2 Type 2](https://www.aicpa-cima.com/topic/audit-assurance/audit-and-assurance-greater-than-soc-2) や [Health Insurance Portability and Accountability Act of 1996 (HIPAA)](https://www.hhs.gov/hipaa/for-professionals/index.html) など、組織の規制要件への準拠の維持。
* アップグレードの管理とパッチの適用。
* W&amp;B Self-Managed デプロイメントの継続的な保守。

<Info>
  組織が規制要件の対象となる場合は、W&amp;B が保守を行う [W&amp;B Dedicated Cloud](/ja/platform/hosting/hosting-options/dedicated-cloud) へのデプロイを検討してください。

  * W&amp;B Dedicated Cloud のホスティングプラットフォームは、SOC 2 Type 2 の要件を満たしています。
  * 適切に設定されている場合、W&amp;B Dedicated Cloud のデプロイメントは HIPAA に準拠します。

  詳細情報をリクエストするには、[W&amp;B Security Portal](https://security.wandb.ai/) を参照してください。
</Info>

<div id="about-the-wb-kubernetes-operator">
  ## W&amp;B Kubernetes Operator について
</div>

**W&amp;B Kubernetes Operator** を使用して W&amp;B Self-Managed をデプロイします。この Operator により、W&amp;B のデプロイ、管理、トラブルシューティング、スケーリングが簡素化されます。

この Operator は中央の [deploy.wandb.ai](https://deploy.wandb.ai) サーバーに接続し、指定したリリースチャネルに対応する最新の仕様変更を取得して適用します。ライセンスが有効である限り、更新を受け取れます。[Helm](https://helm.sh/) は、Operator のインストールと、Operator による W&amp;B Kubernetes スタックの管理の両方に使用されます。デプロイメントは複数の pod で構成され、各サービスに 1 つの pod が対応します。各 pod 名には `wandb-` というプレフィックスが付きます。

設定は次の階層に従います: **Release Channel Values** (W&amp;B によるデフォルト値) 、**User Input Values** (System Console からの上書き値) 、**Custom Resource Values** (ユーザーの spec が前 2 つを上書き) 。詳細については、[Deploy W&amp;B with Kubernetes Operator](/ja/platform/hosting/self-managed/operator) を参照してください。

* パブリッククラウドまたはオンプレミスインフラストラクチャーに W&amp;B Self-Managed をデプロイするには、[Deploy W&amp;B with Kubernetes Operator](/ja/platform/hosting/self-managed/operator) を参照してください。
* AWS 以外のカスタムクラウドプラットフォームに W&amp;B Self-Managed をデプロイする場合、要件は [オンプレミスインフラストラクチャー](#deploy-wb-server-in-on-prem-infrastructure) へのデプロイ要件と同様です。
* エアギャップ環境に W&amp;B Self-Managed をデプロイするには、[Deploy on Air-Gapped Kubernetes](/ja/platform/hosting/self-managed/on-premises-deployments/kubernetes-airgapped) を参照してください。

必要に応じて、安定性を維持するためにインスタンスに [レート制限を設定](/ja/platform/hosting/self-managed/rate-limits) できます。デフォルト値と通知の動作については、[レート制限](/ja/platform/hosting/self-managed/rate-limits) を参照してください。

<div id="required-infrastructure">
  ## 必須インフラストラクチャー
</div>

すべてのデプロイオプションは、以下のインフラストラクチャーに依存しています。

* Kubernetes
* MySQL 8 データベース
* Amazon S3 互換オブジェクトストレージ
* Redis キャッシュ

詳細については、[Requirements](/ja/platform/hosting/self-managed/requirements) を参照してください。W&amp;B は各コンポーネントに関する推奨事項を提供し、インストールプロセスを通じてガイダンスを提供することができます。

<div id="obtain-your-wb-server-license">
  ## W&amp;B Server ライセンスを取得する
</div>

デプロイ前に、W&amp;B Server ライセンスが必要です。手順については、要件ページの [License](/ja/platform/hosting/self-managed/requirements#license) を参照してください。
