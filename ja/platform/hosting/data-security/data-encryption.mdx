---
title: 専用クラウドにおけるデータ暗号化
description: "クラウドネイティブキーと顧客管理暗号鍵ポリシーを使用して、W&B が Dedicated Cloud のデータをどのように暗号化するかを説明します。"
---

W&amp;B は、各クラウドの顧客管理暗号鍵 (CMEK) 機能を利用して、すべての[専用クラウド](/ja/platform/hosting/hosting-options/dedicated-cloud)において、W&amp;B が管理するデータベースとオブジェクトストレージの暗号化に W&amp;B 管理のクラウドネイティブキーを使用しています。この場合、W&amp;B はクラウドプロバイダーの`顧客`として行動しつつ、W&amp;B プラットフォームをサービスとしてお客様に提供しています。W&amp;B 管理キーを使用することで、各クラウドでデータの暗号化に使用するキーを W&amp;B 自身が管理できるため、すべてのお客様に対して高い安全性とセキュリティを備えたプラットフォームを提供するという約束を、より強固なものにしています。

W&amp;B は、各顧客インスタンスのデータを暗号化するために`一意のキー`を使用しており、Dedicated Cloud テナント間の分離をさらに強化しています。この機能は AWS、Azure、Google Cloud で利用できます。

<Note>
  2024 年 8 月より前に W&amp;B がプロビジョニングした Google Cloud および Azure 上の Dedicated Cloud インスタンスでは、W&amp;B 管理データベースとオブジェクトストレージの暗号化に、クラウドプロバイダー管理のデフォルトキーを使用しています。2024 年 8 月以降に W&amp;B が新たに作成したインスタンスのみ、該当する暗号化に W&amp;B 管理のクラウドネイティブキーを使用しています。

  AWS 上の Dedicated Cloud インスタンスでは、2024 年 8 月以前から W&amp;B 管理のクラウドネイティブキーを暗号化に使用しています。
</Note>

W&amp;B は通常、お客様が自身のクラウドネイティブキーを持ち込んで、Dedicated Cloud インスタンス内の W&amp;B 管理データベースとオブジェクトストレージを暗号化することを認めていません。これは、組織内の複数のチームや関係者が、さまざまな理由でそのクラウドインフラストラクチャーにアクセスできる可能性があるためです。それらのチームや関係者の中には、組織のテクノロジースタックにおける重要な構成要素としての W&amp;B を十分に理解していない人がいる可能性があり、その結果、クラウドネイティブキーを完全に削除したり、W&amp;B のアクセスを取り消したりするおそれがあります。そのような行為により、組織の W&amp;B インスタンス内のすべてのデータが破損し、復旧不能な状態になる可能性があります。

お客様の組織で、AI ワークフローに Dedicated Cloud を使用する承認を得るために、W&amp;B 管理データベースとオブジェクトストレージの暗号化に独自のクラウドネイティブキーを使用する必要がある場合は、W&amp;B が例外的に審査することがあります。承認された場合、暗号化におけるお客様のクラウドネイティブキーの使用は、W&amp;B Dedicated Cloud の`責任共有モデル`に従うことになります。お客様の組織内のいずれかのユーザーが、Dedicated Cloud インスタンスの稼働中のいずれかの時点でお客様のキーを削除したり、W&amp;B のアクセスを取り消したりした場合、W&amp;B はその結果生じるデータ損失または破損について責任を負わず、そのようなデータの復旧についても責任を負いません。
