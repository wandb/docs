---
title: Dedicated Cloud へのプライベート接続を設定する
description: "AWS PrivateLink、GCP、または Azure Private Link を使用して、プライベートネットワーク経由で W&B Dedicated Cloud インスタンスに接続します。"
---

クラウドプロバイダーのセキュアなプライベートネットワーク経由で、[Dedicated Cloud](/ja/platform/hosting/hosting-options/dedicated-cloud) インスタンスに接続できます。これは、AI ワークロードから W&amp;B API へのアクセスに適用され、必要に応じてユーザーのブラウザから W&amp;B App UI へのアクセスにも適用されます。プライベート接続を使用すると、該当するリクエストとレスポンスはパブリックネットワークやインターネットを経由しません。

<Note>
  Secure private connectivity は、Dedicated Cloud の高度なセキュリティオプションとして近日提供予定です。
</Note>

Secure private connectivity は、AWS、Google Cloud、Azure 上の Dedicated Cloud インスタンスで使用できます。

* AWS では [AWS Privatelink](https://aws.amazon.com/privatelink/) を使用
* Google Cloud では [Google Cloud Private Service Connect](https://cloud.google.com/vpc/docs/private-service-connect) を使用
* Azure では [Azure Private Link](https://azure.microsoft.com/products/private-link) を使用

有効にすると、W&amp;B はお使いのインスタンス用のプライベートエンドポイントサービスを作成し、接続先となる DNS URI を提供します。これにより、クラウドアカウント内にプライベートエンドポイントを作成し、該当するトラフィックをプライベートエンドポイントサービスにルーティングできます。プライベートエンドポイントは、クラウドの VPC または VNet 内で実行される AI トレーニングワークロードでは比較的簡単に設定できます。ユーザーのブラウザから W&amp;B App UI へのトラフィックにも同じ仕組みを使用するには、社内ネットワークからクラウドアカウント内のプライベートエンドポイントへの適切な DNS ベースのルーティングを設定する必要があります。

<Note>
  この機能を使用する場合は、W&amp;B チームにお問い合わせください。
</Note>

Secure private connectivity は [IP allowlisting](./ip-allowlisting) と併用できます。IP allowlisting に Secure private connectivity を使用する場合、W&amp;B は、AI ワークロードからのすべてのトラフィックと、可能であればユーザーのブラウザからのトラフィックの大部分を Secure private connectivity で保護し、特権アクセス可能な場所からのインスタンス管理には IP allowlisting を使用することを推奨しています。
