---
title: Enterprise ライセンス
description: W&B Enterprise ライセンスに含まれる機能、取得方法、設定方法について説明します。
---

Enterprise ライセンスを使用すると、セキュリティ、コンプライアンス、管理の各領域で高度な機能を必要とする組織向けに設計された W&amp;B の機能を利用できます。[Enterprise 機能](#enterprise-features)の一覧を参照してください。

* **セキュリティ**: 強化された認証、暗号化、アクセス制御。
* **コンプライアンス**: 監査ログ、HIPAA コンプライアンス オプション、データ ガバナンス制御。
* **管理**: SCIM プロビジョニング、カスタム ロール、オートメーション機能などのユーザー管理機能。

このページでは、Enterprise ライセンスの包括的な概要として、有効になる機能、取得方法、デプロイ向けの設定方法を説明します。

<div id="deployment-options">
  ## デプロイ オプション
</div>

Enterprise ライセンスは、W&amp;B 専用クラウド および W&amp;B セルフマネージド のデプロイで利用できます。

* [W&amp;B 専用クラウド](/ja/platform/hosting/hosting-options/dedicated-cloud) は、W&amp;B が管理するシングルテナントのインフラストラクチャーを提供し、Google Cloud にデプロイされます。専用クラウド のデプロイには Enterprise ライセンスが自動的に含まれます。追加の設定は不要で、すべての Enterprise 機能をプロビジョニング後すぐに利用できます。
* [W&amp;B セルフマネージド](/ja/platform/hosting/hosting-options/self-managed) は、お客様自身のインフラストラクチャーにデプロイされるため、デプロイとデータを完全に管理できます。エアギャップ環境でのデプロイもサポートされています。Enterprise ライセンスがない場合、[Enterprise 機能](#enterprise-features) は利用できません。

<div id="obtain-an-enterprise-license">
  ## Enterprise ライセンスを取得する
</div>

このセクションでは、Enterprise ライセンスを取得する方法について説明します。ユースケースに応じて、該当するタブを選択して進んでください。

<Tabs>
  <Tab title="Dedicated Cloud">
    W&amp;B Dedicated Cloud には Enterprise ライセンスが含まれているため、追加の対応は不要です。[Enterprise トライアルを開始](https://wandb.ai/site/for-enterprise/dedicated-saas-trial/)するか、要件について相談する場合は [Sales](https://wandb.ai/site/contact-sales) にお問い合わせください。
  </Tab>

  <Tab title="Self-Managed">
    Enterprise ライセンスをリクエストするには:

    * **評価**: [試用ライセンスのリクエスト](https://wandb.ai/site/for-enterprise/self-hosted-trial) を送信するか、要件について相談する場合は [Sales](mailto:sales@wandb.ai) にお問い合わせください。
    * **本番**: デプロイ要件やサイジングについて相談し、ライセンスの詳細を受け取るには [Sales](https://wandb.ai/site/contact-sales) にお問い合わせください。

    <Warning>ライセンスキーは安全に保管し、機密性の高い認証情報として扱ってください。</Warning>
  </Tab>
</Tabs>

<div id="configure-an-enterprise-license">
  ## Enterprise ライセンスを設定する
</div>

<Info>このセクションは W&amp;B Self-Managed にのみ該当します。</Info>

[Enterprise ライセンスを取得](#obtain-an-enterprise-license)したら、プログラムで設定することも、デプロイ後に W&amp;B App で設定することもできます。手順については、タブを選択してください。

<Tabs>
  <Tab title="Kubernetes Operator">
    1. ライセンスを設定するには、`operator.yaml` (`WeightsAndBiases` Custom Resource) を更新します。
       ```yaml
       spec:
         values:
           global:
             license: "YOUR_LICENSE_KEY_HERE"
       ```

    2. 更新した設定を適用します。
       ```bash
       kubectl apply -f operator.yaml
       ```
  </Tab>

  <Tab title="Terraform">
    1. Terraform 変数を更新します。
       ```hcl
       variable "license" {
         default = "YOUR_LICENSE_KEY_HERE"
       }
       ```

    2. Terraform の設定を適用します。
       ```bash
       terraform apply -var="license=$LICENSE_KEY"
       ```
  </Tab>

  <Tab title="W&B App">
    環境変数が設定されていない既存のデプロイの場合:

    1. 管理者ユーザーとしてログインします
    2. **System Settings** にアクセスします
    3. **License** セクションを見つけます
    4. 新しいライセンスキーを入力します
    5. 変更内容を Save します
  </Tab>

  <Tab title="Docker">
    <Warning>Docker での W&amp;B のデプロイは非推奨です。W&amp;B では、Helm または Terraform で管理する [Kubernetes Operator]() デプロイへの移行を強く推奨しています。詳細は [リファレンスアーキテクチャ](/ja/platform/hosting/self-managed/ref-arch.mdx) を参照するか、[サポート](mailto:support@wandb.ai) までお問い合わせください。</Warning>
    ライセンスキーを環境変数として設定します。

    ```bash
    docker run -d \
      -e LICENSE="YOUR_LICENSE_KEY_HERE" \
      -e OTHER_ENV_VARS \
      wandb/local
    ```
  </Tab>
</Tabs>

次に、[Enterprise ライセンスを確認または更新します](#verify-or-update-an-enterprise-license)。

<div id="verify-and-manage-an-enterprise-license">
  ## Enterprise license を確認して管理する
</div>

インスタンスまたは組織管理者として、Enterprise license を確認して管理するには、次の step に従います:

1. W&amp;B App で、右上のユーザープロフィールアイコンをクリックし、次に **System Console** をクリックします。
2. **License** タブに移動します。ダッシュボードには、ライセンスの使用権、有効期限、および経時的な使用状況の概要が表示されます。エラーが表示された場合は、[一般的な問題](#common-issues)を参照してください。
3. ライセンスとその使用権の詳細情報を確認するには、**View license online** をクリックします。

   このページから、新しい trial または本番ライセンスを注文することもできます。ページ上部で **New license order** をクリックし、フォームを送信します。
4. 期限切れのライセンスを更新するか、ライセンスキーを更新するには、**Update License** をクリックします。

   <Tip>
     サービスが中断しないように、W&amp;B は次のタイミングで連絡します:

     * ライセンスの有効期限が切れる 30 日前
     * ライセンスの有効期限が切れたとき
   </Tip>

<div id="common-issues">
  ## よくある問題
</div>

<div id="license-not-recognized">
  ### ライセンスが認識されない
</div>

* ライセンスキーの形式が正しいことを確認する (余分なスペースがないこと)
* ライセンスの有効期限が切れていないことを確認する
* ライセンスが正しい設定箇所に指定されていることを確認する

<div id="features-not-available-after-setting-license">
  ### ライセンス設定後も機能を利用できない
</div>

* ライセンスを設定した後、W&amp;B サービスを再起動する
* 利用しようとしている機能がライセンスに含まれていることを確認する
* ライセンスの検証エラーがないかシステムログを確認する

<div id="license-expiration-warnings">
  ### ライセンスの有効期限切れに関する警告
</div>

* 有効期限切れの通知がないか System Settings ページを確認する
* モニタリング システムでライセンスの有効期限切れに関するアラートを設定する
* アカウント担当者の連絡先情報を最新の状態に保つ
