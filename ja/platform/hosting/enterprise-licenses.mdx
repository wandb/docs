Enterprise ライセンスは、セキュリティ、コンプライアンス、または管理の面で強化された機能を必要とする組織向けに設計された W&amp;B 機能をアンロックします。[Enterprise 機能](#enterprise-機能) のリストをご覧ください。

* **セキュリティ**: 強化された認証、暗号化、アクセス制御。
* **コンプライアンス**: 操作ログ (監査ログ) 、HIPAA コンプライアンスオプション、データガバナンス管理。
* **管理**: SCIM プロビジョニング、カスタムロール、自動化機能などのユーザー管理機能。

このページでは、Enterprise ライセンスの包括的な概要を説明します。これには、有効になる機能、取得方法、デプロイメントでの設定方法などが含まれます。

<div id="deployment-options">
  ## デプロイメントオプション
</div>

Enterprise ライセンスは、W&amp;B 専用クラウドおよびセルフマネージドデプロイメントで利用できます。

* [W&amp;B Dedicated Cloud](/ja/platform/hosting/hosting-options/dedicated-cloud) は、Google Cloud にデプロイされる、W&amp;B が管理するシングルテナントのインフラストラクチャーを提供します。専用クラウドのデプロイメントには、自動的に Enterprise ライセンスが含まれます。追加の設定は不要で、プロビジョニング後すぐにすべての Enterprise 機能を利用できます。

* [W&amp;B Self-Managed](/ja/platform/hosting/hosting-options/self-managed) は、お客様自身のインフラストラクチャーにデプロイされるため、デプロイメントとデータを完全に制御できます。エアギャップ環境でのデプロイメントもサポートされています。Enterprise ライセンスがない場合、[Enterprise 機能](#enterprise-features) は利用できません。

* [W&amp;B 専用クラウド](/ja/platform/hosting/hosting-options/dedicated-cloud) は、W&amp;B が管理し Google Cloud にデプロイされるシングルテナントのインフラストラクチャーを提供します。専用クラウドのデプロイメントには Enterprise ライセンスが自動的に含まれています。追加の設定は不要で、プロビジョニング後すぐにすべての Enterprise 機能を利用できます。

* [W&amp;B セルフマネージド](/ja/platform/hosting/hosting-options/self-managed) は、お客様自身のインフラストラクチャーにデプロイされ、デプロイメントとデータを完全に制御できます。エアギャップ環境でのデプロイもサポートされています。Enterprise ライセンスがない場合、[Enterprise 機能](#enterprise-機能) は利用できません。

<div id="obtain-an-enterprise-license">
  ## Enterprise ライセンスの取得
</div>

このセクションでは、Enterprise ライセンスの取得方法について説明します。ユースケースに合わせてタブを選択してください。

<Tabs>
  <Tab title="専用クラウド">
    W&amp;B 専用クラウドには Enterprise ライセンスが含まれているため、アクションは不要です。[Enterprise トライアルを開始する](https://wandb.ai/site/for-enterprise/dedicated-saas-trial/) か、[営業担当 (Sales)](https://wandb.ai/site/contact-sales) に連絡して要件をご相談ください。
  </Tab>

  <Tab title="セルフマネージド">
    Enterprise ライセンスをリクエストする：

    * **評価**: [トライアルライセンスのリクエスト](https://wandb.ai/site/for-enterprise/self-hosted-trial) を送信するか、[営業担当 (Sales)](mailto:sales@wandb.ai) に連絡して要件をご相談ください。
    * **プロダクション**: [営業担当 (Sales)](https://wandb.ai/site/contact-sales) に連絡して、デプロイメントの要件とサイジングを相談し、ライセンスの詳細を受け取ってください。

    <Warning>ライセンスキーは安全に保管し、機密性の高い認証情報として扱ってください。</Warning>
  </Tab>
</Tabs>

<div id="configure-an-enterprise-license">
  ## Enterprise ライセンスの設定
</div>

<Info>このセクションは W&amp;B セルフマネージドにのみ関連します。</Info>

[Enterprise ライセンスを取得](#obtain-an-enterprise-license) した後、プログラムで設定するか、デプロイ後に W&amp;B アプリ内で設定できます。手順についてはタブを選択してください。

<Tabs>
  <Tab title="Kubernetes Operator">
    1. ライセンスを設定するために `operator.yaml` (`WeightsAndBiases` カスタムリソース) を更新します：
       ```yaml
       spec:
         values:
           global:
             license: "YOUR_LICENSE_KEY_HERE"
       ```

    2. 更新した設定を適用します：
       ```bash
       kubectl apply -f operator.yaml
       ```
  </Tab>

  <Tab title="Terraform">
    1. Terraform 変数を更新します：
       ```hcl
       variable "license" {
         default = "YOUR_LICENSE_KEY_HERE"
       }
       ```

    2. Terraform 設定を適用します：
       ```bash
       terraform apply -var="license=$LICENSE_KEY"
       ```
  </Tab>

  <Tab title="W&B App">
    環境変数の設定がない既存のデプロイメントの場合：

    1. 管理者ユーザーとしてログインします
    2. **System Settings** に移動します
    3. **License** セクションを探します
    4. 新しいライセンスキーを入力します
    5. 変更を保存します
  </Tab>

  <Tab title="Docker">
    <Warning>Docker への W&amp;B のデプロイは非推奨となりました。W&amp;B では、Helm または Terraform で管理される [Kubernetes Operator]() デプロイメントへの移行を強く推奨しています。詳細は [リファレンスアーキテクチャー](/ja/platform/hosting/self-managed/ref-arch.mdx) を参照するか、[サポート (Support)](mailto:support@wandb.ai) までお問い合わせください。</Warning>
    ライセンスキーを環境変数として設定します：

    ```bash
    docker run -d \
      -e LICENSE="YOUR_LICENSE_KEY_HERE" \
      -e OTHER_ENV_VARS \
      wandb/local
    ```
  </Tab>
</Tabs>

次に、[Enterprise ライセンスの確認](#verify-or-update-an-enterprise-license) を行います。

<div id="verify-and-manage-an-enterprise-license">
  ## Enterprise ライセンスの確認と管理
</div>

インスタンスまたは組織の管理者として、以下の手順で Enterprise ライセンスを確認・管理してください。

1. W&amp;B アプリで、右上のユーザープロフィールアイコンをクリックし、**System Console** をクリックします。
2. **License** タブに移動します。ダッシュボードに、ライセンスの権限、有効期限、および期間ごとの使用状況が表示されます。エラーが表示される場合は、[よくある問題](#よくある問題) を参照してください。
3. ライセンスとその権限に関する詳細情報を確認するには、**View license online** をクリックします。

   このページから、新しいトライアルまたはプロダクションライセンスを注文することもできます。ページ上部の **New license order** をクリックしてフォームを送信してください。
4. 期限切れのライセンスを更新したり、ライセンスキーをアップデートしたりするには、**Update License** をクリックします。
   <Tip>
     サービスを中断なく継続できるよう、W&amp;B は以下のタイミングでご連絡いたします：

     * ライセンス期限が切れる 30 日前
     * ライセンス期限が切れた時
   </Tip>

<div id="common-issues">
  ## よくある問題
</div>

<div id="license-not-recognized">
  ### ライセンスが認識されない
</div>

* ライセンスキーの形式が正しいか確認してください (余分なスペースがないかなど) 
* ライセンスの有効期限が切れていないか確認してください
* ライセンスが正しい設定場所に設定されているか確認してください

<div id="features-not-available-after-setting-license">
  ### ライセンス設定後に機能が利用できない
</div>

* ライセンスを設定した後、W&amp;B サービスを再起動してください
* アクセスしようとしている特定の機能がライセンスに含まれているか確認してください
* ライセンスの検証エラーがないか、システムログを確認してください

<div id="license-expiration-warnings">
  ### ライセンス期限切れの警告
</div>

* System Settings ページで有効期限の通知を確認してください
* モニタリングシステムでライセンス期限切れのアラートを設定してください
* アカウント担当者の連絡先情報を最新の状態に保ってください
