サーバー エラーは、W&amp;B Inference サービスで一時的な問題が発生していることを示しています。

<div id="error-types">
  ## エラーのタイプ
</div>

<div id="500-internal-server-error">
  ### 500 - 内部サーバーエラー
</div>

**メッセージ:** &quot;サーバーでリクエストの処理中にエラーが発生しました&quot;

これはサーバー側で一時的に発生している内部エラーです。

<div id="503-service-overloaded">
  ### 503 - サービスの過負荷
</div>

**Message:** &quot;推論エンジンは現在過負荷状態です。時間をおいて再度お試しください&quot;

サービスへのトラフィックが増加しています。

<div id="how-to-handle-server-errors">
  ## サーバー エラーへの対処方法
</div>

1. **再試行する前に待機する**
   * 500 エラー: 30〜60 秒待機する
   * 503 エラー: 60〜120 秒待機する

2. **エクスポネンシャルバックオフを使用する**
   ```python
   import time
   import openai

   def call_with_retry(client, messages, model, max_retries=5):
       for attempt in range(max_retries):
           try:
               return client.chat.completions.create(
                   model=model,
                   messages=messages
               )
           except Exception as e:
               if "500" in str(e) or "503" in str(e):
                   if attempt < max_retries - 1:
                       wait_time = min(60, (2 ** attempt))
                       time.sleep(wait_time)
                   else:
                       raise
               else:
                   raise
   ```

3. **適切なタイムアウトを設定する**
   * HTTP クライアントのタイムアウト値を長めに設定する
   * より適切に処理するため、非同期操作も検討する

<div id="when-to-contact-support">
  ## サポートに連絡するタイミング
</div>

次の場合はサポートに連絡してください。

* エラーが10分以上続く
* 特定の時間帯に失敗が発生する傾向が見られる
* エラーメッセージに追加の詳細情報が含まれている

以下を提供してください。

* エラーメッセージとコード
* エラーが発生した時刻
* コードスニペット (APIキーは削除してください)
* W&amp;B の entity 名とプロジェクト名
