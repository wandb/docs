このページでは、Hydra と W&amp;B を併用してプロセスを起動したときに発生する、プロセスの応答停止を解決する方法について説明します。Hydra で起動したときにプロセスが応答しなくなる場合、これは Hydra と W&amp;B の間のマルチプロセッシングの競合が原因である可能性が高いです。

これを解決するには、W&amp;B のマルチプロセッシングプロトコルを `"thread"` に変更してください。これにより競合が回避され、プロセスを正常に起動できます。これを行う方法は 2 つあります。

<div id="pass-a-settings-parameter-to-wandbinit">
  ## Settings パラメーターを wandb.init に渡す
</div>

```python
wandb.init(settings=wandb.Settings(start_method="thread"))
```

<div id="set-a-global-environment-variable">
  ## グローバル環境変数を設定する
</div>

```bash
export WANDB_START_METHOD=thread
```

詳細は、[Hydraインテグレーションガイド](/ja/models/integrations/hydra/)および[分散トレーニングのトラブルシューティング](/ja/models/track/log/distributed-training/)をご覧ください。
