---
title: "コードやデータセットの例を含めず、メトリクスだけをログできますか？"
keywords: ["管理者", "チーム管理", "メトリクス"]
---

デフォルトでは、W&amp;B はデータセットの例をログしません。デフォルトでは、W&amp;B はコードとシステムメトリクスをログします。

環境変数を使ってコードのログを無効にする方法は 2 つあります。

1. `WANDB_DISABLE_CODE` を `true` に設定すると、コードのトラッキングがすべて無効になります。これにより、git SHA と diff patch を取得できなくなります。
2. `WANDB_IGNORE_GLOBS` を `*.patch` に設定すると、diff patch のサーバーへの Sync を停止しつつ、`wandb restore` で適用できるよう、ローカルでは引き続き利用可能なままにできます。

組織管理者は、アカウントまたは組織のダッシュボードの **Settings** タブ内にある **Privacy** セクションから、組織内のすべてのチームに対してプライバシー設定を適用できます。組織管理者が設定を適用した場合、チーム管理者は各チーム内でその設定を構成できません。

チーム管理者は、チーム内の Runs に対するデフォルトのコードの保存を無効にできます。

1. `https://wandb.ai/<team>` にアクセスします。`<team>` はチーム名に置き換えてください。
2. 左側のナビゲーションで **Team settings** を選択します。
3. **Privacy** セクションを開き、**Enable code saving by default** をオフにします。
