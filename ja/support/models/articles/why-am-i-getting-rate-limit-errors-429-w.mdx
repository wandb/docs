title: "W&B Inference で rate limit エラー（429）が発生するのはなぜですか？"
keywords: ["Inference"]
---

同時実行制限を超えると、rate limit エラー (429) が発生します。

**エラー:** &quot;リクエストの同時実行制限に達しました&quot;

**解決策:**

* 並列リクエスト数を減らします
* リクエスト間に遅延を追加します
* 指数バックオフを実装します
* 注: レート制限 は W&amp;B プロジェクトごとに適用されます

<div id="best-practices-to-avoid-rate-limits">
  ## レート制限を回避するためのベストプラクティス
</div>

1. **指数バックオフを使用した再試行ロジックを実装する:**
   ```python
   import time

   def retry_with_backoff(func, max_retries=3):
       for i in range(max_retries):
           try:
               return func()
           except Exception as e:
               if "429" in str(e) and i < max_retries - 1:
                   time.sleep(2 ** i)
               else:
                   raise
   ```
