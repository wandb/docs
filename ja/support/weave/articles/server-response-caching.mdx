Weave は、繰り返しクエリを実行する場合や、ネットワーク帯域幅が限られた環境で作業する場合のパフォーマンスを向上させるために、サーバー応答のキャッシュ機能を提供しています。現在、この機能はデフォルトで無効ですが、今後のリリースではデフォルトの動作になる予定です。

<div id="when-to-use-caching">
  ### キャッシュを使用する場面
</div>

サーバー応答のキャッシュは、特に次のような場合に有効です。

* 同じクエリを頻繁に実行する場合
* ネットワーク帯域幅が限られている場合
* 高レイテンシーの環境で作業している場合
* オフラインで開発していて、後で使用するために応答をキャッシュしておきたい場合

この機能は、データセットに対して評価を繰り返し実行する場合に特に便利です。run 間でデータセットをキャッシュできるためです。

<div id="how-to-enable-caching">
  ### キャッシュを有効にする方法
</div>

キャッシュを有効にするには、次の環境変数を設定できます。

```bash
# サーバー応答のキャッシュを有効にする
export WEAVE_USE_SERVER_CACHE=true

# キャッシュサイズの上限を設定する（デフォルトは 1GB）
export WEAVE_SERVER_CACHE_SIZE_LIMIT=1000000000

# キャッシュディレクトリを設定する（省略可能。デフォルトは一時ディレクトリ）
export WEAVE_SERVER_CACHE_DIR=/path/to/cache
```

<div id="caching-behavior">
  ### キャッシュの挙動
</div>

厳密には、この機能はサーバーに対する冪等なリクエストをキャッシュします。具体的には、次をキャッシュします：

* `obj_read`
* `table_query`
* `table_query_stats`
* `refs_read_batch`
* `file_content_read`

<div id="cache-size-and-storage-details">
  ### キャッシュサイズとストレージの詳細
</div>

キャッシュサイズは `WEAVE_SERVER_CACHE_SIZE_LIMIT` (バイト単位) で制御されます。実際に使用されるディスク容量は、次の 3 つのコンポーネントで構成されます。

1. 固定の 32KB のチェックサムファイル
2. 実行中のクライアントごとに最大約 4MB の Write-Ahead Log (WAL) ファイル (プログラム終了時に自動的に削除されます)
3. 最小 32KB、最大 `WEAVE_SERVER_CACHE_SIZE_LIMIT` のメインデータベースファイル

合計のディスク使用量:

* 実行中 &gt;= 32KB + 約 4MB + キャッシュサイズ
* 終了後 &gt;= 32KB + キャッシュサイズ

たとえば、キャッシュ上限が 5MB の場合:

* 実行中: 最大約 9MB
* 終了後: 最大約 5MB
