트레이닝 설정이 명령줄 인수를 전달하도록 되어 있다면 W&amp;B Sweeps를 맞춤형 CLI 명령과 함께 사용할 수 있습니다.

아래 예시의 코드 스니펫은 사용자가 `train.py`라는 Python 스크립트로 트레이닝을 실행하면서 스크립트가 파싱할 값을 전달하는 bash 터미널을 보여줍니다:

```bash
/usr/bin/env python train.py -b \
    your-training-config \
    --batchsize 8 \
    --lr 0.00001
```

맞춤형 명령어를 구현하려면 YAML 파일에서 `command` 키를 수정하세요. 이전 예시를 바탕으로 하면 설정은 다음과 같습니다:

```yaml
program:
  train.py
method: grid
parameters:
  batch_size:
    value: 8
  lr:
    value: 0.0001
command:
  - ${env}
  - python
  - ${program}
  - "-b"
  - your-training-config
  - ${args}
```

The `${args}` 키는 sweep 설정의 모든 매개변수로 확장되고, `argparse`에서 사용할 수 있도록 `--param1 value1 --param2 value2` 형식으로 지정됩니다.

`argparse` 외부에서 사용할 추가 인수가 있다면 다음과 같이 구현하세요:

```python
parser = argparse.ArgumentParser()
args, unknown = parser.parse_known_args()
```

<Note>
  환경에 따라 `python`이 Python 2를 가리킬 수 있습니다. Python 3를 실행하려면 command 설정에서 `python3`를 사용하세요:

  ```yaml
  program:
    script.py
  command:
    - ${env}
    - python3
    - ${program}
    - ${args}
  ```
</Note>
