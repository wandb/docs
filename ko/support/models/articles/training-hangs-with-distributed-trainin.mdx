title: "분산 트레이닝에서 트레이닝이 멈추는 이유는 무엇인가요?"
keywords: ["Experiments", "Run Crashes"]
---

W&amp;B를 분산 트레이닝과 함께 사용할 때 트레이닝이 멈추는 일반적인 이유는 두 가지입니다:

1. **트레이닝 시작 시 멈춤**: W&amp;B의 멀티프로세싱이 분산 트레이닝 프레임워크의 멀티프로세싱과 간섭할 수 있습니다.
2. **트레이닝 종료 시 멈춤**: W&amp;B 프로세스가 언제 종료해야 하는지 알지 못합니다.

<div id="fix-hanging-at-the-start">
  ## 시작 시 멈춤 문제 해결
</div>

W&amp;B SDK `0.13.0` 이상에서는 기본으로 활성화되는 W&amp;B Service를 사용하세요. 이전 버전을 사용 중이라면 SDK를 업그레이드하세요:

```shell
pip install --upgrade wandb
```

W&amp;B SDK `0.12.5`부터 `0.12.x`까지는 W&amp;B Service를 명시적으로 활성화하세요:

```python
def main():
    wandb.require("service")
    # 나머지 스크립트
```

W&amp;B SDK `0.12.4` 이하 버전에서는 `WANDB_START_METHOD` 환경 변수를 설정하세요:

```bash
export WANDB_START_METHOD=thread
```

<div id="fix-hanging-at-the-end">
  ## 마지막에 멈추는 문제 해결
</div>

트레이닝 스크립트 마지막에 `wandb.finish()`를 호출하여 W&amp;B에 run이 완료되었음을 알리세요:

```python
wandb.finish()
```

이렇게 하면 모든 데이터가 업로드되고 W&amp;B 프로세스가 정상적으로 종료됩니다.

자세한 내용은 [분산 트레이닝](/ko/models/track/log/distributed-training/)을 참조하세요.
