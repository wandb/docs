---
title: "로깅한 것보다 데이터 포인트가 적게 표시되는 이유는 무엇인가요?"
keywords: ["Experiments", "메트릭"]
---

`Step`가 아닌 다른 X축으로 메트릭을 시각화할 때 데이터 포인트가 더 적게 표시되는 것을 예상하세요. 메트릭은 동기화 상태를 유지하기 위해 동일한 `Step`에서 로깅되어야 합니다. 샘플 간 보간 시 동일한 `Step`에서 로깅된 메트릭만 샘플링됩니다.

**지침**

메트릭을 하나의 `log()` 호출로 묶으세요. 예를 들어, 다음과 같이 하는 대신:

```python
import wandb
with wandb.init() as run:
    run.log({"Precision": precision})
    ...
    run.log({"Recall": recall})
```

다음을 사용하세요:

```python
import wandb
with wandb.init() as run:
    run.log({"Precision": precision, "Recall": recall})
```

step 파라미터를 수동으로 제어하려면 다음과 같이 코드에서 메트릭을 동기화하세요:

```python
with wandb.init() as run:
    step = 100  # 예시 step 값
    # Precision과 Recall을 동일한 step에 로깅
    run.log({"Precision": precision, "Recall": recall}, step=step)
```

메트릭이 동일한 `step`에 기록되어 함께 샘플링되도록 두 `log()` 호출 모두에서 `step` 값이 같아야 합니다. 각 호출에서 `step` 값은 단조롭게 증가해야 합니다. 그렇지 않으면 `step` 값은 무시됩니다.
