---
title: "W&B Multi-tenant에서 SSO를 어떻게 설정하나요?"
keywords: ["Security"]
---

W&amp;B는 Auth0를 통해 Multi-tenant 환경에서 Single Sign-On(SSO)을 지원하므로, 별도의 W&amp;B 자격 증명을 관리하지 않고도 기존 ID 공급자를 사용해 사용자를 인증할 수 있습니다. SSO 인테그레이션은 Okta 또는 Azure AD와 같은 OpenID Connect(OIDC) 규격을 준수하는 모든 ID 공급자를 지원합니다.

OIDC 공급자를 구성하려면 ID 공급자에서 다음 단계를 완료하세요.

1. ID 공급자에서 single-page application을 생성합니다.
2. `grant_type`을 `implicit` flow로 설정합니다.
3. callback URI를 `https://wandb.auth0.com/login/callback`로 설정합니다.

<div id="requirements-for-wb">
  ## W&amp;B 요구 사항
</div>

설정을 완료한 후 애플리케이션의 `Client ID`와 `Issuer URL`을 고객 성공 관리자(CSM)에게 전달하세요. W&amp;B는 이 정보를 사용해 Auth0 연결을 설정하고 조직에 SSO를 활성화합니다.
