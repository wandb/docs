---
title: "메트릭만 기록하고 코드나 데이터셋 예시는 기록하지 않을 수 있나요?"
keywords: ["Administrator", "Team Management", "Metrics"]
---

기본적으로 W&amp;B는 데이터셋 예시를 기록하지 않습니다. 기본적으로 W&amp;B는 코드와 시스템 메트릭을 기록합니다.

환경 변수를 사용하여 코드 기록을 끄는 방법은 두 가지가 있습니다:

1. `WANDB_DISABLE_CODE`를 `true`로 설정하면 모든 코드 추적이 비활성화됩니다. 이 액션은 git SHA 및 diff 패치 검색을 차단합니다.
2. `WANDB_IGNORE_GLOBS`를 `*.patch`로 설정하면 diff 패치가 서버에 동기화되지 않으며, `wandb restore`로 적용할 수 있도록 로컬에는 계속 보관됩니다.

관리자는 팀 설정에서 팀의 코드 저장을 끌 수도 있습니다:

1. `https://wandb.ai/<team>/settings`에서 팀 설정으로 이동하세요. `<team>`은 팀의 이름입니다.
2. Privacy 섹션으로 스크롤하세요.
3. **Enable code saving by default**를 표시/숨기기하세요.
