---
title: "워커 프로세스에서의 트레이스 데이터 손실"
keywords: ["트레이스 데이터", "성능"]
---

Weave는 애플리케이션 성능에 미치는 영향을 최소화하기 위해 백그라운드 스레드에서 트레이스 데이터를 업로드합니다. 하지만 멀티프로세싱 또는 작업 큐 시스템, 혹은 Celery와 같은 워커 프로세스를 사용하는 경우 백그라운드 스레드가 트레이스 업로드를 마치기 전에 워커 프로세스가 종료되면 트레이스 데이터가 손실됩니다.

워커 프로세스에서 데이터 손실을 방지하려면 워커 작업이 완료되기 전에 `client.flush()` 또는 `client.finish()`를 호출해 백그라운드 업로드가 완료되도록 해야 합니다.

이 방법들은 각각 용도가 다릅니다:

* `weave.flush()`: 단순하게 조용히 플러시합니다. Weave를 워커 프로세스나 CI 환경에 통합해 사용할 때 권장됩니다.
* `weave.finish()`: 진행률 표시줄이나 상태 콜백을 통해 진행 상황 피드백을 제공합니다. 대화형 스크립트나 노트북에서 권장됩니다.

두 방법 모두 모든 백그라운드 업로드가 완료될 때까지 실행을 차단하므로, 워커 프로세스가 종료될 때 트레이스 데이터가 손실되지 않도록 보장합니다.

다음 예제는 `client.finish()`를 보여줍니다:

```python
from celery import Celery
import weave

app = Celery('tasks')

@app.task
def process_task(input_data):
    weave.init("your-team-name/your-project-name")

    try:
        # 트레이스를 생성하는 태스크 로직
        result = your_processing_function(input_data)

        # 태스크가 완료되기 전에 모든 트레이스가 업로드되었는지 확인
        weave.finish()

        return result
    finally:
        pass
```

`with` 컨텍스트 관리자를 사용해 블록을 벗어날 때 `weave.finish()`가 자동으로 호출되게 할 수도 있습니다:

```python
with weave.init("your-team-name/your-project-name") as client:
    result = your_processing_function(input_data)

    return result
```

`weave.flush()`를 사용하면 애플리케이션의 성능을 더 높일 수 있습니다. 자세한 내용은 [Flushing](/ko/support/weave/articles/long-eval-clean-up-times)을 참조하세요.
