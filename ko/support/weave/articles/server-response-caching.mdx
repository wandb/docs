Weave는 반복적인 쿼리를 실행하거나 네트워크 대역폭이 제한된 환경에서 작업할 때 성능을 높일 수 있도록 서버 응답 캐싱을 제공합니다. 현재는 기본적으로 비활성화되어 있지만, 향후 릴리스에서는 이 기능이 기본 동작이 될 예정입니다.

<div id="when-to-use-caching">
  ### 캐싱을 사용하는 경우
</div>

다음과 같은 경우 서버 응답 캐싱이 특히 유용합니다.

* 동일한 쿼리를 자주 실행하는 경우
* 네트워크 대역폭이 제한적인 경우
* 지연 시간이 높은 환경에서 작업하는 경우
* 오프라인으로 개발하면서 나중에 사용하기 위해 응답을 캐시하려는 경우

이 기능은 데이터셋에 대해 반복적으로 evaluations를 실행할 때 특히 유용하며, run 사이에서 데이터셋을 캐시할 수 있습니다.

<div id="how-to-enable-caching">
  ### 캐싱 활성화 방법
</div>

캐싱을 활성화하려면 다음 환경 변수를 설정합니다:

```bash
# 서버 응답 캐싱 활성화
export WEAVE_USE_SERVER_CACHE=true

# 캐시 크기 제한 설정 (기본값은 1GB)
export WEAVE_SERVER_CACHE_SIZE_LIMIT=1000000000

# 캐시 디렉터리 설정 (선택 사항, 기본값은 임시 디렉터리)
export WEAVE_SERVER_CACHE_DIR=/path/to/cache
```

<div id="caching-behavior">
  ### 캐시 동작
</div>

기술적으로 이 기능은 서버에 대한 멱등 요청을 캐시합니다. 구체적으로는 다음 요청을 캐시합니다.

* `obj_read`
* `table_query`
* `table_query_stats`
* `refs_read_batch`
* `file_content_read`

<div id="cache-size-and-storage-details">
  ### 캐시 크기 및 저장소 세부 정보
</div>

캐시 크기는 `WEAVE_SERVER_CACHE_SIZE_LIMIT`(바이트 단위)로 제어됩니다. 실제로 사용되는 디스크 공간은 세 가지 컴포넌트로 구성됩니다.

1. 고정 32KB 체크섬 파일
2. 실행 중인 클라이언트당 최대 약 4MB의 Write-Ahead Log(WAL) 파일(프로그램 종료 시 자동으로 제거됨)
3. 최소 32KB, 최대 `WEAVE_SERVER_CACHE_SIZE_LIMIT` 크기의 기본 데이터베이스 파일

총 디스크 사용 공간:

* 실행 중: &gt;= 32KB + ~4MB + 캐시 크기
* 종료 후: &gt;= 32KB + 캐시 크기

예를 들어, 캐시 제한이 5MB인 경우:

* 실행 중: 최대 약 9MB
* 종료 후: 최대 약 5MB
