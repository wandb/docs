W&amp;B에서 만든 샌드박스에서 셸 명령, 스크립트, 기타 프로세스를 실행하세요. 이 페이지에서는 명령을 실행하고, 출력과 종료 코드를 확인하고, 동일한 샌드박스를 여러 step에서 재사용하는 방법을 설명합니다.

<div id="run-commands-in-a-sandbox">
  ## 샌드박스에서 명령 실행하기
</div>

샌드박스에서 명령을 실행하는 일반적인 방법은 두 가지입니다.

1. [단일 명령 실행](#run-a-single-command): 명령이 완료되면 샌드박스가 중지되도록 합니다.
2. 동일한 샌드박스에서 [여러 명령 실행](#run-multiple-commands-sequentially-in-the-same-sandbox)합니다.

<div id="run-a-single-command">
  ### 단일 명령 실행
</div>

단일 명령을 실행하려면 명령을 `Sandbox.run()`에 직접 전달하세요. 명령이 끝나면 샌드박스는 자동으로 중지되며, 추가 작업에는 더 이상 사용할 수 없습니다.

```python
from wandb.sandbox import Sandbox

sandbox = Sandbox.run("python", "train.py")
result = sandbox.wait_until_complete(timeout=3600.0).result()
print(f"Exit code: {result.returncode}")
print(f"Standard output: {result.stdout}")
print(f"Standard error: {result.stderr}")
```

<div id="run-multiple-commands-sequentially-in-the-same-sandbox">
  ### 동일한 샌드박스에서 여러 명령를 순차적으로 실행하기
</div>

의존성을 설치하고, 파일을 작성하고, 스크립트를 실행하고, 결과를 조회하는 등 여러 step이 필요한 워크플로에는 동일한 샌드박스를 사용하세요.

동일한 샌드박스에서 여러 명령를 실행하려면 명령를 지정하지 않고 샌드박스를 시작하세요. 그런 다음 `Sandbox.exec()`를 사용해 실행 중인 샌드박스에서 명령를 실행하세요.

다음 코드 스니펫은 메인 명령 없이 샌드박스를 시작한 뒤, 동일한 샌드박스에서 두 개의 명령를 순차적으로 실행합니다. 먼저 PyTorch를 설치한 다음 트레이닝 스크립트를 실행합니다:

```python
from wandb.sandbox import Sandbox

with Sandbox.run() as sandbox:
    result = sandbox.exec(["pip", "install", "torch"]).result()
    print(result.stdout)

    result = sandbox.exec(["python", "train.py"]).result()
    print(result.stdout)
```

`Sandbox.exec()` 메서드는 실행 중인 샌드박스 내부에서 명령어를 실행하고, 완료를 기다리거나, 출력을 조회하거나, 종료 코드를 확인하는 데 사용할 수 있는 `Process` 객체를 반환합니다.

`Sandbox.exec()` 및 해당 매개변수에 대한 자세한 내용은 [`Sandbox.exec()`](https://docs.coreweave.com/products/coreweave-sandbox/client/ref/core/sandbox#exec) 레퍼런스 문서를 참조하세요.

명령어가 실행되는 동안 샌드박스를 계속 실행 상태로 유지하기 위해 오래 실행되는 메인 프로세스를 명시적으로 정의할 수도 있습니다. 이는 머신 러닝 모델을 트레이닝하는 경우처럼 실행에 오랜 시간이 걸리는 명령어를 실행하려는 경우에 유용합니다.

다음 코드 스니펫은 `sleep infinity` 명령어로 샌드박스를 시작하며, 이 명령어는 샌드박스를 무기한 실행 상태로 유지합니다. 그런 다음 `Sandbox.exec()`를 사용하여 샌드박스 내부에서 Python 트레이닝 스크립트를 실행합니다. 트레이닝 스크립트가 실행되는 동안 샌드박스는 활성 상태를 유지하며, 필요하면 동일한 샌드박스에서 추가 명령어를 실행할 수 있습니다.

```python
from wandb.sandbox import Sandbox

with Sandbox.run("sleep", "infinity") as sandbox:
    result = sandbox.exec(["python", "train.py"]).result()
    print(result.stdout)
```

<div id="read-output-and-exit-codes">
  ## 출력 및 종료 코드 확인
</div>

`Sandbox.exec()`는 [`Process`](https://docs.coreweave.com/products/coreweave-sandbox/client/ref/types/process) 객체를 반환합니다. `Process.result()`를 호출하면 명령이 완료될 때까지 기다린 후 결과를 가져올 수 있습니다. `Process.result()`에는 표준 출력(`stdout`), 표준 오류(`stderr`), 종료 코드(`returncode`)가 포함됩니다.

종료 코드를 사용해 명령이 성공했는지 확인하세요. 관례적으로 종료 코드가 0이면 성공을 의미합니다.

예를 들어, 다음 코드 스니펫은 표준 출력(`stdout`)과 종료 코드(`returncode`)를 출력하는 Python 명령을 실행합니다. 명령이 완료되면 표준 출력, 표준 오류, 종료 코드를 콘솔에 출력합니다.

```python
from wandb.sandbox import Sandbox

with Sandbox.run() as sandbox:
    result = sandbox.exec(["python", "-c", "print('hello')"]).result()
    print(result.stdout)      # "hello\\n"
    print(result.returncode)  # 0
```

<div id="check-for-sandbox-execution-errors">
  ## 샌드박스 실행 오류 확인
</div>

명령이 0이 아닌 종료 코드로 종료될 때 오류를 발생시키려면 `check=True`(`Sandbox.exec(check=True)`)를 지정하세요. 명령이 0이 아닌 종료 코드로 종료되면 [`SandboxExecutionError`](https://docs.coreweave.com/products/coreweave-sandbox/client/ref/exceptions/exceptions#sandboxexecutionerror)가 발생합니다:

```python
from wandb.sandbox import Sandbox
from cwsandbox import SandboxExecutionError

with Sandbox.run() as sandbox:
    try:
        sandbox.exec(["python", "might_fail.py"], check=True).result()
    except SandboxExecutionError as e:
        print(f"명령이 종료 코드 {e.exec_result.returncode}(으)로 실패했습니다")
        print(e.exec_result.stderr)
```
