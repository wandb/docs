<div id="pricing">
  ## 가격
</div>

가격은 Inference, 트레이닝, 저장소의 세 가지 컴포넌트로 구성됩니다. 구체적인 결제 요율은 [가격 페이지](https://site.wandb.ai/pricing/training)에서 확인하세요.

<div id="inference">
  ### Inference
</div>

Serverless RL Inference Request의 가격은 W&amp;B Inference 가격과 같습니다. 자세한 내용은 [모델별 비용](https://site.wandb.ai/pricing/reinforcement-learning)을 참조하세요. 크레딧 구매, 계정 등급, 사용 한도에 대해 더 알아보려면 [W&amp;B Inference 문서](/ko/inference/usage-limits#purchase-more-credits)를 확인하세요.

<div id="training">
  ### 트레이닝
</div>

각 트레이닝 step마다 Serverless RL은 에이전트의 출력과 리워드 함수로 계산된 대응 리워드를 포함하는 trajectory 배치를 수집합니다. 이렇게 배치된 trajectory는 이후 LoRA 어댑터의 가중치를 업데이트하는 데 사용되며, 이 LoRA 어댑터는 기본 모델을 사용자의 작업에 특화하도록 만듭니다. 이러한 LoRA를 업데이트하는 트레이닝 작업은 Serverless RL이 관리하는 전용 GPU 클러스터에서 실행됩니다.

공개 프리뷰 기간 동안 트레이닝은 무료입니다.

<div id="model-storage">
  ### 모델 저장소
</div>

Serverless RL은 트레이닝된 LoRA의 체크포인트를 저장하므로 언제든지 이를 평가하고, 서빙하거나, 트레이닝을 계속할 수 있습니다. 저장소 요금은 총 체크포인트 크기와 [가격 플랜](https://wandb.ai/site/pricing)을 기준으로 매월 청구됩니다. 모든 플랜에는 최소 5GB의 무료 저장소가 포함되며, 이는 대략 30개의 LoRA에 충분한 용량입니다. 공간을 절약하려면 성능이 낮은 LoRA를 삭제하는 것을 권장합니다. 방법은 [ART SDK](https://art.openpipe.ai/features/checkpoint-deletion)를 참조하세요.

<div id="limits">
  ## 한도
</div>

* **Inference 동시성 한도**: 기본적으로 Serverless RL은 현재 사용자당 최대 2000개의 동시 Request와 프로젝트당 6000개의 동시 Request를 지원합니다. 요청 속도 제한을 초과하면 Inference API가 `429 Concurrency limit reached for requests` Response를 반환합니다. 이 오류를 방지하려면 트레이닝 작업 또는 프로덕션 워크로드가 한 번에 보내는 동시 Request 수를 줄이세요. 더 높은 요청 속도 제한이 필요하면 support@wandb.com으로 요청할 수 있습니다.

* **지리적 제한**: Serverless RL은 지원되는 지리적 위치에서만 사용 가능합니다. 자세한 내용은 [서비스 약관](https://site.wandb.ai/terms/)을 참조하세요.
