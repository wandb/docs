W&amp;B Weave는 신뢰할 수 있는 LLM 애플리케이션을 구축하기 위한 관찰성 및 평가 플랫폼입니다. Weave는 AI 애플리케이션이 무엇을 하고 있는지 이해하고, 얼마나 잘 작동하는지 측정하며, 시간에 따라 체계적으로 개선할 수 있도록 도와줍니다.

LLM 애플리케이션을 구축하는 일은 전통적인 소프트웨어 개발과 근본적으로 다릅니다. LLM 출력은 비결정적이기 때문에 디버깅이 더 어렵습니다. 품질은 주관적이며 맥락에 따라 달라집니다. 작은 프롬프트 변경만으로도 예기치 않은 동작 변화를 유발할 수 있습니다. 전통적인 테스트 접근 방식만으로는 충분하지 않습니다.

<div id="the-main-threads-of-weave">
  ## Weave의 주요 스레드
</div>

Weave는 다음과 같은 핵심 기능을 제공합니다:

* 애플리케이션 내 모든 LLM 호출, 입력, 출력에 대한 **가시성**
* 큐레이션된 테스트 케이스를 통해 성능을 측정하는 **체계적인 평가**
* 프롬프트, 모델, 데이터의 **버전 추적**으로 변경 사항을 파악할 수 있도록
* 다양한 프롬프트와 모델 비교를 위한 **실험**
* 인간의 판단과 어노테이션을 캡처하는 **피드백 수집**
* LLM 안전성과 품질을 위한 가드레일과 스코어러를 활용한 프로덕션 **모니터링**

<div id="traces">
  ### 트레이스
</div>

LLM 애플리케이션에서 데이터의 엔드투엔드 흐름을 추적하세요.

* 각 애플리케이션 사용 시 입력과 출력을 확인하세요.
* LLM 응답을 생성하는 데 사용된 소스 문서를 확인하세요.
* LLM 호출의 비용, 토큰 수, 레이턴시를 확인하세요.
* 특정 프롬프트와 답변이 어떻게 생성되는지 자세히 분석하세요.
* 사용자로부터 Response에 대한 피드백을 수집하세요.
* 코드에서 Weave [ops and calls](/ko/weave/guides/tracking/tracing)를 사용하여 함수에서 어떤 일이 일어나는지 추적할 수 있습니다.

[트레이싱 시작하기](/ko/weave/quickstart)

<div id="evaluations">
  ### 평가
</div>

LLM 애플리케이션의 성능을 체계적으로 벤치마크하여 프로덕션 배포 시 신뢰도를 높이세요.

* 어떤 모델/프롬프트 버전이 어떤 성능을 냈는지 쉽게 추적할 수 있습니다.
* 하나 이상의 스코어링 함수를 사용해 Response를 평가할 메트릭을 정의할 수 있습니다.
* 여러 메트릭에 걸쳐 두 개 이상의 서로 다른 평가를 비교할 수 있습니다. 개별 샘플의 성능을 서로 대비해 볼 수 있습니다.

[평가 파이프라인 구축하기](/ko/weave/tutorial-eval)

<div id="get-started-using-weave">
  ## Weave 사용 시작하기
</div>

Weave는 Python 및 TypeScript용 SDK를 제공합니다. 두 SDK 모두 트레이싱, 평가, 데이터셋, 그리고 Weave의 핵심 기능을 지원합니다. 클래스 기반 Models 및 스코어러와 같은 일부 고급 기능은 현재 Weave TypeScript SDK에서 사용할 수 없습니다.

Weave를 시작하려면:

1. [https://wandb.ai/site](https://wandb.ai/site/?utm_source=course\&utm_medium=course\&utm_campaign=weave)에서 Weights &amp; Biases 계정을 만들고, [https://wandb.ai/authorize](https://wandb.ai/authorize?utm_source=course\&utm_medium=course\&utm_campaign=weave)에서 API 키를 가져오세요.
2. Weave를 설치하세요:

<CodeGroup>
  ```Python Python
  pip install weave
  ```

  ```Typescript Typescript
  npm install weave
  ```
</CodeGroup>

3. 스크립트에서 Weave를 임포트하고 프로젝트를 초기화하세요:

<CodeGroup>
  ```Python Python
  import weave
  client = weave.init('your-team/your-project-name')
  ```

  ```TypeScript Typescript
  import * as weave from 'weave';
  const client = await weave.init('your-team/your-project-name');
  ```
</CodeGroup>

이제 Weave를 사용할 준비가 되었습니다.
Weave는 널리 사용되는 LLM 제공자 및 프레임워크와 통합됩니다. [지원되는 인테그레이션](/ko/weave/guides/integrations/)을 사용하면 Weave가 추가 코드 변경 없이 LLM 호출을 자동으로 트레이싱합니다.

4. 지원되는 인테그레이션 외에도, 호출 함수에 한 줄만 추가해 맞춤형 함수의 트레이스를 Weave에 로깅할 수 있습니다.

함수를 `@weave.op()`(Python)으로 데코레이팅하거나 `weave.op()`(TypeScript)으로 래핑하면 Weave가 해당 함수의 코드, 입력, 출력, 그리고 실행 메타데이터를 자동으로 캡처합니다.

<CodeGroup>
  ```python Python
      @weave.op
      async def my_function(){
        ...  }
  ```

  ```typescript Typescript
  function myFunction() {
      ...
  }

  const myFunctionOp = weave.op(myFunction)
  ```
</CodeGroup>

가이드형 튜토리얼로 직접 사용해 보려면 [트레이싱 시작하기](/ko/weave/quickstart)를 참조하세요.
