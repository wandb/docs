평가는 변경한 뒤 예시 집합을 대상으로 애플리케이션을 테스트하여 반복적으로 개선하는 데 도움이 됩니다. Weave는 [`Model`](/ko/weave/guides/core-types/models) 및 [`Evaluation`](/ko/weave/guides/core-types/evaluations) class를 사용한 평가 tracking을 기본적으로 지원합니다. API는 최소한의 가정을 바탕으로 설계되어 있어, 매우 다양한 사용 사례에 유연하게 대응할 수 있습니다.

이 가이드는 다음 방법을 보여줍니다:

* 스코어링 함수와 추가 기본 제공 스코어러를 사용해 데이터셋에 대해 모델을 테스트하는 평가를 실행하는 방법
* Weave UI에서 평가 결과를 확인하는 방법

<div id="build-a-model">
  ## `Model` 구축하기
</div>

Weave에서 [`Models`는 객체](/ko/weave/guides/core-types/models)이며, 모델/에이전트의 동작(로직, 프롬프트, 파라미터)과 버전 관리되는 메타데이터(파라미터, 코드, 세부 설정)를 함께 캡처하므로 안정적으로 추적, 비교, 평가, 개선할 수 있습니다.

`Model`을 인스턴스화하면 Weave가 해당 `Model`의 설정과 동작을 자동으로 캡처하고, 변경 사항이 있을 때마다 버전을 업데이트합니다. 이를 통해 반복적으로 개선해 나가면서 시간에 따른 성능을 추적할 수 있습니다.

`Model`은 `Model`을 상속한 서브클래스를 만들고, 하나의 예제를 입력받아 응답을 반환하는 `predict` 함수를 구현해 선언합니다.

다음 예시 모델은 OpenAI를 사용해 전달된 문장에서 외계 과일의 이름, 색상, 맛을 추출합니다.

<Tabs>
  <Tab title="Python">
    ```python lines {1,5}
    class ExtractFruitsModel(weave.Model):
        model_name: str
        prompt_template: str

        @weave.op()
        async def predict(self, sentence: str) -> dict:
            client = openai.AsyncClient()

            response = await client.chat.completions.create(
                model=self.model_name,
                messages=[
                    {"role": "user", "content": self.prompt_template.format(sentence=sentence)}
                ],
            )
            result = response.choices[0].message.content
            if result is None:
                raise ValueError("No response from model")
            parsed = json.loads(result)
            return parsed
    ```
  </Tab>

  <Tab title="TypeScript">
    ```typescript lines {9}
    // 참고: weave.Model은 아직 TypeScript에서 지원되지 않습니다.
    // 대신, 모델과 유사한 함수를 weave.op으로 래핑하세요.

    import * as weave from 'weave';
    import OpenAI from 'openai';

    const openaiClient = new OpenAI();

    const model = weave.op(async function myModel({datasetRow}) {
      const prompt = `Extract fields ("fruit": <str>, "color": <str>, "flavor") from the following text, as json: ${datasetRow.sentence}`;
      const response = await openaiClient.chat.completions.create({
        model: 'gpt-3.5-turbo',
        messages: [{ role: 'user', content: prompt }],
        response_format: { type: 'json_object' }
      });
      return JSON.parse(response.choices[0].message.content);
    });
    ```
  </Tab>
</Tabs>

`ExtractFruitsModel` 클래스는 Weave가 인스턴스화된 객체를 추적할 수 있도록 `weave.Model`을 상속합니다. `@weave.op`은 predict 함수에 데코레이터로 적용되어 입력과 출력을 추적합니다.

다음과 같이 `Model` 객체를 인스턴스화할 수 있습니다.

다음 예시 데이터셋은 세 개의 입력 문장과 각 문장에 대한 정답(`labels`)을 정의한 다음, 스코어링 함수가 읽을 수 있는 JSON 테이블 형식으로 구성합니다.

<div id="define-custom-scoring-functions">
  ## 맞춤형 스코어링 함수 정의하기
</div>

Weave 평가를 사용할 때 Weave는 `output`과 비교할 `target`이 필요합니다. 아래 스코어링 함수는 두 개의 사전(`target`과 `output`)을 받아서, output이 target과 일치하는지를 나타내는 불리언 값의 사전을 반환합니다. `@weave.op()` 데코레이터는 Weave가 스코어링 함수의 실행을 추적할 수 있도록 해 줍니다.

<Tabs>
  <Tab title="Python">
    ```python lines
    @weave.op()
    def fruit_name_score(target: dict, output: dict) -> dict:
        return {'correct': target['fruit'] == output['fruit']}
    ```
  </Tab>

  <Tab title="TypeScript">
    ```typescript
    import * as weave from 'weave';

    const fruitNameScorer = weave.op(
      function fruitNameScore({target, output}) {
        return { correct: target.fruit === output.fruit };
      }
    );
    ```
  </Tab>
</Tabs>

직접 스코어링 함수를 작성하려면 [Scorers](/ko/weave/guides/evaluation/scorers) 가이드를 참고하세요.

일부 애플리케이션에서는 맞춤형 `Scorer` 클래스를 만들고 싶을 수 있습니다. 예를 들어, 특정 파라미터(예: 채팅 모델 또는 프롬프트), 개별 행 스코어링 방식, 그리고 집계 점수 계산을 포함하는 표준화된 `LLMJudge` 클래스를 만들 수 있습니다. RAG 애플리케이션의 모델 기반 평가에 대한 다음 장의 [튜토리얼](/ko/weave/tutorial-rag#optional-defining-a-scorer-class)에서 `Scorer` 클래스를 정의하는 방법을 확인하세요.

<div id="use-a-built-in-scorer-and-run-the-evaluation">
  ## 기본 제공 스코어러를 사용해 평가 실행하기
</div>

맞춤형 스코어링 함수와 함께 [Weave의 기본 제공 스코어러](/ko/weave/guides/evaluation/builtin_scorers)도 사용할 수 있습니다. 다음 평가에서 `weave.Evaluation()`은 이전 섹션에서 정의한 `fruit_name_score` 함수와 [F1 점수](https://en.wikipedia.org/wiki/F-score)를 계산하는 기본 제공 스코어러 `MultiTaskBinaryClassificationF1`을 사용합니다.

다음 예시에서는 두 개의 스코어링 함수를 사용해 `fruits` 데이터셋에서 `ExtractFruitsModel`을 평가하고, 결과를 Weave에 로깅합니다.

<Tabs>
  <Tab title="Python">
    ```python lines {3-10}
    weave.init('eval_pipeline_quickstart')

    evaluation = weave.Evaluation(
        name='fruit_eval',
        dataset=dataset, 
        scorers=[
            MultiTaskBinaryClassificationF1(class_names=["fruit", "color", "flavor"]), 
            fruit_name_score
        ],
    )
    print(asyncio.run(evaluation.evaluate(model)))
    # Jupyter Notebook 환경이라면, 다음을 실행하세요:
    # await evaluation.evaluate(model)
    ```
  </Tab>

  <Tab title="TypeScript">
    ```typescript lines {5-9}
    import * as weave from 'weave';

    await weave.init('eval_pipeline_quickstart');

    const evaluation = new weave.Evaluation({
      name: 'fruit_eval',
      dataset: dataset,
      scorers: [fruitNameScorer],
    });
    const results = await evaluation.evaluate(model);
    console.log(results);
    ```
  </Tab>
</Tabs>

<Note>
  Python 스크립트에서 실행하는 경우에는 `asyncio.run`을 사용해야 합니다. 반면 Jupyter Notebook에서 실행하는 경우에는 `await`를 바로 사용할 수 있습니다.
</Note>
