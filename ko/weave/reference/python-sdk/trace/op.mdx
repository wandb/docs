### <kbd>함수</kbd> `as_op`

```python
as_op(fn: 'Callable[P, R]') → Op[P, R]
```

@weave.op 데코레이터가 적용된 함수를 받아 해당 Op를 반환합니다.

@weave.op 데코레이터가 적용된 함수는 이미 Op의 인스턴스이므로, 이 함수는 런타임에서는 실질적으로 아무 동작도 하지 않습니다(no-op). 하지만 타입 안전한 방식으로 OpDef 속성에 액세스해야 할 때, 타입 체커를 통과시키는 용도로 사용할 수 있습니다.

**매개변수:**

* <b>`fn`</b>: `weave.op` 데코레이터가 적용된 함수.
  **반환값:**
  함수의 Op.

### <kbd>함수</kbd> `call`

```python
call(
    op: 'Op',
    *args: 'Any',
    __weave: 'WeaveKwargs | None' = None,
    __should_raise: 'bool' = False,
    __require_explicit_finish: 'bool' = False,
    **kwargs: 'Any'
) → tuple[Any, Call] | Coroutine[Any, Any, tuple[Any, Call]]
```

op를 실행하고 실행을 나타내는 결과와 Call을 모두 반환합니다.

이 함수는 예외를 발생시키지 않습니다. 오류는 모두 Call 객체에 캡처됩니다.

이 방법은 `@weave.op`로 데코레이팅된 모든 함수에 자동으로 바인딩되므로, 다음과 같이 사용할 수 있습니다:

```python
@weave.op
def add(a: int, b: int) -> int:
     return a + b

result, call = add.call(1, 2)
```

### <kbd>함수</kbd> `Call`

```python
calls(op: 'Op') → CallsIter
```

이 op에 대한 모든 Call의 이터레이터를 조회합니다.

이 방법은 `@weave.op`로 데코레이팅된 모든 함수에 자동으로 바인딩되므로, 다음과 같이 사용할 수 있습니다:

```python
@weave.op
def add(a: int, b: int) -> int:
     return a + b

calls = add.calls()
for call in calls:
     print(call)
```

### <kbd>함수</kbd> `op`

```python
op(
    func: 'Callable[P, R] | None' = None,
    name: 'str | None' = None,
    call_display_name: 'str | CallDisplayNameFunc | None' = None,
    postprocess_inputs: 'PostprocessInputsFunc | None' = None,
    postprocess_output: 'PostprocessOutputFunc | None' = None,
    tracing_sample_rate: 'float' = 1.0,
    enable_code_capture: 'bool' = True,
    accumulator: 'Callable[[Any | None, Any], Any] | None' = None,
    kind: 'OpKind | None' = None,
    color: 'OpColor | None' = None
) → Callable[[Callable[P, R]], Op[P, R]] | Op[P, R]
```

함수나 방법을 Weave op로 변환(op-ify)하는 데코레이터입니다. 동기 및 비동기 함수 모두에서 동작합니다. 이터레이터(반복자) 함수인지 자동으로 감지하고, 이에 맞는 동작을 적용합니다.
