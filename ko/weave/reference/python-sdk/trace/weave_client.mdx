### <kbd>방법</kbd> `add_cost`

```python
add_cost(
    llm_id: 'str',
    prompt_token_cost: 'float',
    completion_token_cost: 'float',
    effective_date: 'datetime | None' = None,
    prompt_token_cost_unit: 'str | None' = 'USD',
    completion_token_cost_unit: 'str | None' = 'USD',
    provider_id: 'str | None' = 'default'
) → CostCreateRes
```

현재 프로젝트에 비용을 추가하세요.

**예시:**

```python
client.add_cost(llm_id="my_expensive_custom_model", prompt_token_cost=1, completion_token_cost=2)
client.add_cost(llm_id="my_expensive_custom_model", prompt_token_cost=500, completion_token_cost=1000, effective_date=datetime(1998, 10, 3))
```

**매개변수:**

* <b>`llm_id`</b>: LLM의 ID입니다. 예: &quot;gpt-4o-mini-2024-07-18&quot;
* <b>`prompt_token_cost`</b>: 프롬프트 token당 비용입니다. 예: .0005
* <b>`completion_token_cost`</b>: completion token당 비용입니다. 예: .0015
* <b>`effective_date`</b>: 기본값은 현재 날짜입니다. `datetime.datetime` 객체입니다.
* <b>`provider_id`</b>: LLM의 제공자입니다. 기본값은 &quot;default&quot;입니다. 예: &quot;openai&quot;
* <b>`prompt_token_cost_unit`</b>: 프롬프트 token 비용의 단위입니다. 기본값은 &quot;USD&quot;입니다. (현재는 사용되지 않으며, 향후에는 비용의 통화 유형(예: &quot;tokens&quot; 또는 &quot;time&quot;)을 지정하는 데 사용됩니다.)
* <b>`completion_token_cost_unit`</b>: completion token 비용의 단위입니다. 기본값은 &quot;USD&quot;입니다. (현재는 사용되지 않으며, 향후에는 비용의 통화 유형(예: &quot;tokens&quot; 또는 &quot;time&quot;)을 지정하는 데 사용됩니다.)
  **반환값:**
  CostCreateRes 객체입니다. 여기에는 ids라는 튜플 목록 필드가 하나 있습니다. 각 튜플에는 llm&#95;id와 생성된 비용 객체의 id가 포함됩니다.

### <kbd>method</kbd> `get_call`

```python
get_call(
    call_id: 'str',
    include_costs: 'bool' = False,
    include_feedback: 'bool' = False,
    columns: 'list[str] | None' = None
) → WeaveObject
```

ID로 단일 Call을 조회합니다.

* <b>`use_progress_bar`</b>: flush 중에 진행 표시줄을 표시할지 여부입니다. 진행 표시줄이 잘 렌더링되지 않는 환경(예: CI 환경)에서는 False로 설정하세요.

* <b>`callback`</b>: 상태 업데이트를 받는 선택적 콜백 함수입니다. use&#95;progress&#95;bar보다 우선합니다.
  **매개변수:**

* <b>`call_id`</b>: 조회할 Call의 ID입니다.

* <b>`include_costs`</b>: true이면 summary.weave에 비용 정보가 포함됩니다.

* <b>`include_feedback`</b>: true이면 summary.weave.feedback에 피드백 정보가 포함됩니다.

* <b>`columns`</b>: 응답에 포함할 열 목록입니다. None이면 모든 열이 포함됩니다. 더 적은 열을 지정하면 성능이 더 좋아질 수 있습니다. 일부 열(id, project&#95;id, trace&#95;id, op&#95;name, started&#95;at)은 항상 포함됩니다.
  **반환값:**
  Call 객체입니다.

### <kbd>메서드</kbd> `get_calls`

```python
get_calls(
    filter: 'CallsFilterLike | None' = None,
    limit: 'int | None' = None,
    offset: 'int | None' = None,
    sort_by: 'list[SortByLike] | None' = None,
    query: 'QueryLike | None' = None,
    include_costs: 'bool' = False,
    include_feedback: 'bool' = False,
    include_storage_size: 'bool' = False,
    include_total_storage_size: 'bool' = False,
    columns: 'list[str] | None' = None,
    expand_columns: 'list[str] | None' = None,
    return_expanded_column_values: 'bool' = True,
    scored_by: 'str | list[str] | None' = None,
    page_size: 'int' = 1000
) → CallsIter
```

이 프로젝트에 대한 추적 호출(오퍼레이션) 목록을 조회합니다.

이 메서드는 추적 데이터 조회를 위한 강력하고 유연한 인터페이스를 제공합니다. 페이지네이션, 필터링, 정렬, 필드 프로젝션, 스코어 관련 메타데이터를 지원하며, 커스텀 trace UI나 분석 도구를 구현하는 데 사용할 수 있습니다.

성능 팁: `columns`를 지정하고 `filter` 또는 `query`를 사용해 결과 크기를 줄이세요.

**매개변수:**

* <b>`filter`</b>:  `op_name`, `parent_ids` 등과 같은 필드를 기준으로 결과를 좁히기 위한 상위 수준 필터.
* <b>`limit`</b>:  반환할 호출의 최대 개수.
* <b>`offset`</b>:  결과를 반환하기 전에 건너뛸 호출 수(페이지네이션에 사용).
* <b>`sort_by`</b>:  결과를 정렬할 필드 목록(예: `started_at desc`).
* <b>`query`</b>:  고급 필터링을 위한 MongoDB 유사 표현식. 모든 Mongo 연산자가 지원되지는 않습니다.
* <b>`include_costs`</b>:  `True`인 경우 `summary.weave`에 토큰/비용 정보를 포함합니다.
* <b>`include_feedback`</b>:  `True`인 경우 `summary.weave.feedback`에 피드백을 포함합니다.
* <b>`include_storage_size`</b>:  `True`인 경우 호출의 스토리지 크기를 포함합니다.
* <b>`include_total_storage_size`</b>:  `True`인 경우 trace 전체의 스토리지 총량을 포함합니다.
* <b>`columns`</b>:  호출마다 반환할 필드 목록. 이 값을 줄이면 성능을 크게 향상할 수 있습니다.  (`id`, `trace_id`, `op_name`, `started_at` 같은 일부 필드는 항상 포함됩니다.)
* <b>`scored_by`</b>:  하나 이상의 스코어러(이름 또는 ref URI)를 기준으로 필터링합니다. 여러 스코어러는 AND 조건으로 결합됩니다.
* <b>`page_size`</b>:  페이지마다 가져올 호출 수. 대규모 쿼리에서 성능을 위해 이 값을 조정합니다.

**반환값:**

* <b>`CallsIter`</b>:  `Call` 객체에 대한 이터레이터. 슬라이싱, 반복, `.to_pandas()`를 지원합니다.

**예시:**

```python
calls = client.get_calls(
     filter=CallsFilter(op_names=["my_op"]),
     columns=["inputs", "output", "summary"],
     limit=100,
)
for call in calls:
     print(call.inputs, call.output)
```

### <kbd>메서드</kbd> `get_evaluation`

```python
get_evaluation(uri: 'str') → Evaluation
```

URI로 특정 Evaluation 객체를 조회합니다.

Evaluation URI는 일반적으로 다음 형식을 따릅니다: `weave:///entity/project/object/Evaluation:version`

&quot;friendly&quot; 이름으로도 Evaluation을 조회할 수 있습니다: get&#95;evaluation(&quot;Evaluation:v1&quot;)

**매개변수:**

* <b>`uri`</b> (str):  조회할 Evaluation의 고유 리소스 식별자입니다.

**반환값:**

* <b>`Evaluation`</b>:  제공된 URI에 해당하는 Evaluation 객체입니다.

**예외:**

* <b>`TypeError`</b>:  URI가 가리키는 객체가 Evaluation 인스턴스가 아닌 경우입니다.
* <b>`ValueError`</b>:  URI가 올바르지 않거나 객체를 찾을 수 없는 경우입니다.

**예시:**

```python
client = weave.init("my-project")
evaluation = client.get_evaluation("weave:///entity/project/object/my-eval:v1")
print(evaluation.name)
```

### <kbd>방법</kbd> `get_feedback`

```python
get_feedback(
    query: 'Query | str | None' = None,
    reaction: 'str | None' = None,
    offset: 'int' = 0,
    limit: 'int' = 100
) → FeedbackQuery
```

프로젝트의 피드백을 조회합니다.

**예시:**

```python
# 특정 피드백 객체를 가져옵니다.
# 이 함수는 여전히 컬렉션을 반환하며, 해당 컬렉션에는
# 0개 또는 1개의 항목이 포함될 수 있습니다.
client.get_feedback("1B4082A3-4EDA-4BEB-BFEB-2D16ED59AA07")

# 특정 reaction을 가진 모든 피드백 객체를 찾습니다.
client.get_feedback(reaction="👍", limit=10)

# mongo 스타일 쿼리를 사용하여 특정 피드백 유형을 가진
# 모든 피드백 객체를 찾습니다.
from weave.trace_server.interface.query import Query

query = Query(
    **{
        "$expr": {
            "$eq": [
                {"$getField": "feedback_type"},
                {"$literal": "wandb.reaction.1"},
            ],
        }
    }
)
client.get_feedback(query=query)
```

**매개변수:**

* <b>`query`</b>: MongoDB 스타일의 쿼리 표현식입니다. 편의를 위해 피드백 UUID string도 허용합니다.
* <b>`reaction`</b>: 편의를 위해 특정 리액션 이모지로 필터링할 수 있습니다.
* <b>`offset`</b>: 피드백 객체를 가져오기 시작할 오프셋입니다.
* <b>`limit`</b>: 가져올 피드백 객체의 최대 개수입니다.
  **반환값:**
  FeedbackQuery 객체입니다.

### <kbd>method</kbd> `set_wandb_run_context`

```python
set_wandb_run_context(run_id: 'str', step: 'int | None' = None) → None
```

이 클라이언트에서 생성된 Call에 대해 wandb run&#95;id와 step을 재정의합니다.

이를 통해 전역 wandb.run 심볼에 바인딩되지 않은 특정 W&amp;B run과 Weave Call을 연결할 수 있습니다.

**매개변수:**

* <b>`run_id`</b>: run ID(entity/프로젝트 접두사 제외)입니다. 클라이언트가 entity/프로젝트 접두사를 자동으로 추가합니다.
* <b>`step`</b>: Call에 사용할 step number입니다. None이면 step이 설정되지 않습니다.
  **예시:**

```python
client = weave.init("my-project")
client.set_wandb_run_context(run_id="my-run-id", step=5)
# 이제 모든 Call이 step 5에서 entity/project/my-run-id와 연결됩니다

# Or without a step
client.set_wandb_run_context(run_id="my-run-id")
# Call이 step 없이 entity/project/my-run-id와 연결됩니다
```
