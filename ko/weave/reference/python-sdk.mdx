## <kbd>클래스</kbd> `Audio`

지원되는 형식(wav 또는 mp3)의 오디오 데이터를 나타내는 클래스입니다.

이 클래스는 오디오 데이터 저장을 처리하고, 다양한 소스에서 로드하고 파일로 내보내는 메서드를 제공합니다.

**속성:**

* <b>`format`</b>:  오디오 형식(현재 &#39;wav&#39; 또는 &#39;mp3&#39;를 지원)
* <b>`data`</b>:  바이트 단위의 원시 오디오 데이터

**매개변수:**

* <b>`data`</b>: 오디오 데이터(바이트 또는 base64로 인코딩된 문자열)

* <b>`format`</b>: 오디오 형식(&#39;wav&#39; 또는 &#39;mp3&#39;)

* <b>`validate_base64`</b>: 입력 데이터의 base64 디코딩을 시도할지 여부
  **예외:**

* <b>`ValueError`</b>:  오디오 데이터가 비어 있거나 형식이 지원되지 않는 경우

## <kbd>클래스</kbd> `Content`

여러 소스의 콘텐츠를 표현하고, 이를 관련 메타데이터와 함께 통합된 바이트 기반 표현으로 변환하는 클래스입니다.

이 클래스의 인스턴스는 다음 클래스 메서드 중 하나를 사용해 생성해야 합니다:

* from&#95;path()
* from&#95;bytes()
* from&#95;text()
* from&#95;url()
* from&#95;base64()
* from&#95;data&#95;url()

<SourceLink url="https://github.com/wandb/weave/blob/v0.52.24/weave/type_wrappers/Content/content.py#L87" />

### <kbd>method</kbd> `__init__`

```python
__init__(*args: 'Any', **kwargs: 'Any') → None
```

직접 초기화는 사용할 수 없습니다. 인스턴스를 생성하려면 `Content.from_path()`와 같은 클래스 메서드를 사용하세요.

**Pydantic 필드:**

* `data`: `<class 'bytes'>`
* `size`: `<class 'int'>`
* `mimetype`: `<class 'str'>`
* `digest`: `<class 'str'>`
* `filename`: `<class 'str'>`
* `content_type`: `typing.Literal['bytes', 'text', 'base64', 'file', 'url', 'data_url', 'data_url:base64', 'data_url:encoding', 'data_url:encoding:base64']`
* `input_type`: `<class 'str'>`
* `encoding`: `<class 'str'>`
* `metadata`: `dict[str, typing.Any] | None`
* `extension`: `str | None`

## <kbd>클래스</kbd> `Dataset`

쉽게 저장할 수 있고 자동 버전 관리가 가능한 데이터셋 객체입니다.

**예시:**

```python
# 데이터셋 생성
dataset = Dataset(name='grammar', rows=[
     {'id': '0', 'sentence': "He no likes ice cream.", 'correction': "He doesn't like ice cream."},
     {'id': '1', 'sentence': "She goed to the store.", 'correction': "She went to the store."},
     {'id': '2', 'sentence': "They plays video games all day.", 'correction': "They play video games all day."}
])

# 데이터셋 게시
weave.publish(dataset)

# 데이터셋 조회
dataset_ref = weave.ref('grammar').get()

# 특정 예시 액세스
example_label = dataset_ref.rows[2]['sentence']
```

**Pydantic 필드:**

* `name`: `str | None`
* `description`: `str | None`
* `ref`: `trace.refs.ObjectRef | None`
* `rows`: `trace.table.Table | trace.vals.WeaveTable`

## <kbd>class</kbd> `Evaluation`

scorer 집합과 데이터셋을 포함하는 Evaluation을 설정합니다.

`evaluation.evaluate(model)`를 호출하면 데이터셋의 각 행이 모델에 전달되며, 이때 데이터셋의 열 이름이 `model.predict`의 인수 이름과 일치하도록 매핑됩니다.

그런 다음 모든 scorer를 호출하고 결과를 Weave에 저장합니다.

데이터셋의 행을 전처리하려면 `preprocess&#95;model&#95;input`에 함수를 전달할 수 있습니다.

**예시:**

```python
# 예시 수집
examples = [
     {"question": "What is the capital of France?", "expected": "Paris"},
     {"question": "Who wrote 'To Kill a Mockingbird'?", "expected": "Harper Lee"},
     {"question": "What is the square root of 64?", "expected": "8"},
]

# 맞춤형 채점 함수 정의
@weave.op
def match_score1(expected: str, model_output: dict) -> dict:
     # 모델 출력을 채점하는 로직을 여기에 정의하세요
     return {'match': expected == model_output['generated_text']}

@weave.op
def function_to_evaluate(question: str):
     # LLM 호출을 추가하고 출력을 반환하는 부분입니다
     return  {'generated_text': 'Paris'}

# 채점 함수를 사용하여 예시 채점
evaluation = Evaluation(
     dataset=examples, scorers=[match_score1]
)

# 평가 추적 시작
weave.init('intro-example')
# 평가 실행
asyncio.run(evaluation.evaluate(function_to_evaluate))
```

## <kbd>class</kbd> `EvaluationLogger`

이 클래스는 평가를 로깅하기 위한 명령형 인터페이스를 제공합니다.

평가는 `log_prediction` 방법을 사용해 첫 번째 예측을 로깅할 때 자동으로 시작되며, `log_summary` 방법이 호출되면 완료됩니다.

예측을 로깅할 때마다 `ScoreLogger` 객체가 반환됩니다. 이 객체를 사용해 해당 예측에 대한 점수와 메타데이터를 로깅할 수 있습니다. 자세한 내용은 `ScoreLogger` 클래스를 참조하세요.

기본 사용법 - 입력과 출력을 직접 사용해 예측을 로깅합니다:

```python
ev = EvaluationLogger()

# 알려진 inputs/outputs으로 예측 로깅
pred = ev.log_prediction(inputs={'q': 'Hello'}, outputs={'a': 'Hi there!'})
pred.log_score("correctness", 0.9)

# 평가 완료
ev.log_summary({"avg_score": 0.9})
```

고급 활용 - 동적 출력과 중첩된 오퍼레이션에 컨텍스트 관리자를 사용하세요:

```python
ev = EvaluationLogger()

# 중첩된 오퍼레이션을 캡처해야 할 때 컨텍스트 매니저를 사용하세요
with ev.log_prediction(inputs={'q': 'Hello'}) as pred:
     # 여기서 수행되는 모든 오퍼레이션(예: LLM Call)은 자동으로
     # predict Call의 하위 항목이 됩니다
     response = your_llm_call(...)
     pred.output = response.content
     pred.log_score("correctness", 0.9)

# Evaluation 완료
ev.log_summary({"avg_score": 0.9})
```

## <kbd>class</kbd> `ContextAwareThread`

호출자의 컨텍스트로 함수를 실행하는 스레드입니다.

이 클래스는 `threading.Thread`를 대체해서 바로 사용할 수 있는 구현으로, 스레드 내부에서 Call이 예상대로 동작하도록 보장합니다. Weave에서는 특정 contextvars가 설정되어 있어야 하지만(call&#95;context.py 참조), 새 스레드는 부모로부터 컨텍스트를 자동으로 복사하지 않으므로 Call 컨텍스트가 손실될 수 있습니다 -- 좋지 않습니다! 이 클래스는 contextvar 복사를 자동화하므로, 이 스레드를 사용하면 사용자가 기대하는 대로 &quot;그냥 동작합니다&quot;.

이 클래스 없이도 대신 다음과 같이 작성하면 같은 효과를 얻을 수 있습니다:

```python
def run_with_context(func, *args, **kwargs):
     context = copy_context()
     def wrapper():
         context.run(func, *args, **kwargs)
     return wrapper

thread = threading.Thread(target=run_with_context(your_func, *args, **kwargs))
thread.start()
```

## <kbd>class</kbd> `ContextAwareThreadPoolExecutor`

호출자의 컨텍스트로 함수를 실행하는 ThreadPoolExecutor입니다.

이 클래스는 executor 내부에서 Weave Call이 예상대로 동작하도록 보장하는 concurrent.futures.ThreadPoolExecutor의 대체 구현입니다. Weave는 특정 contextvars가 설정되어 있어야 하지만(`call&#95;context.py` 참조), 새 스레드는 부모의 컨텍스트를 자동으로 복사하지 않으므로 Call 컨텍스트가 손실될 수 있습니다 -- 좋지 않습니다! 이 클래스는 contextvar 복사를 자동화하므로, 사용자는 이 executor를 사용하기만 하면 &quot;그냥 동작한다&quot;고 기대할 수 있습니다.

이 클래스를 사용하지 않고도 대신 다음과 같이 작성하면 같은 효과를 얻을 수 있습니다:

```python
with concurrent.futures.ThreadPoolExecutor() as executor:
     contexts = [copy_context() for _ in range(len(vals))]

     def _wrapped_fn(*args):
         return contexts.pop().run(fn, *args)

     executor.map(_wrapped_fn, vals)
```

### <kbd>함수</kbd> `log_call`

```python
log_call(
    op: 'str',
    inputs: 'dict[str, Any]',
    output: 'Any',
    parent: 'Call | None' = None,
    attributes: 'dict[str, Any] | None' = None,
    display_name: 'str | Callable[[Call], str] | None' = None,
    use_stack: 'bool' = True,
    exception: 'BaseException | None' = None
) → Call
```

decorator 패턴을 사용하지 않고 Call을 Weave에 직접 로깅합니다.

이 함수는 오퍼레이션을 Weave에 로깅하기 위한 명령형 API를 제공합니다. 이미 실행된 후의 call을 로깅하려는 경우나 decorator 패턴이 사용 사례에 적합하지 않은 경우에 유용합니다.

**매개변수:**

* <b>`op`</b> (str):  로깅할 오퍼레이션 이름입니다. 이는 call의 `op_name`으로 사용됩니다. 익명 오퍼레이션(게시된 ops를 가리키지 않는 문자열)도 지원됩니다.
* <b>`inputs`</b> (dict[str, Any]):  오퍼레이션의 입력 파라미터를 담은 사전입니다.
* <b>`output`</b> (Any):  오퍼레이션의 출력/결과입니다.
* <b>`parent`</b> (Call | None):  이 call을 그 아래에 중첩할 선택적 부모 call입니다.  제공하지 않으면 이 call은 루트 수준 call이 되거나(또는 현재 call 컨텍스트가 있으면 그 아래에 중첩됩니다). 기본값은 None입니다.
* <b>`attributes`</b> (dict[str, Any] | None):  call에 연결할 선택적 메타데이터입니다.  이는 call이 생성되면 고정됩니다. 기본값은 None입니다.
* <b>`display_name`</b> (str | Callable[[Call], str] | None):  UI에서 call에 표시할 선택적 표시 이름입니다. 문자열이거나 call을 받아 문자열을 반환하는 callable일 수 있습니다. 기본값은 None입니다.
* <b>`use_stack`</b> (bool):  call을 런타임 스택에 푸시할지 여부입니다. True이면  call 컨텍스트에서 call을 사용할 수 있으며 `weave.require_current_call()`을 통해 액세스할 수 있습니다. False이면 call은 로깅되지만  call 스택에는 추가되지 않습니다. 기본값은 True입니다.
* <b>`exception`</b> (BaseException | None):  오퍼레이션이 실패한 경우 로깅할 선택적 예외입니다. 기본값은 None입니다.

**반환값:**

* <b>`Call`</b>:  전체 트레이스 정보가 포함된, 생성되고 완료된 Call 객체입니다.

**예시:**
기본 사용법:


````python
import weave
    >>> weave.init('my-project')
    >>> call = weave.log_call(
    ...     op="my_function",
    ...     inputs={"x": 5, "y": 10},
    ...     output=15
    ... )

    Logging with attributes and display name:
    >>> call = weave.log_call(
    ...     op="process_data",
    ...     inputs={"data": [1, 2, 3]},
    ...     output={"mean": 2.0},
    ...     attributes={"version": "1.0", "env": "prod"},
    ...     display_name="Data Processing"
    ... )

    Logging a failed operation:
    >>> try:
    ...     result = risky_operation()
    ... except Exception as e:
    ...     call = weave.log_call(
    ...         op="risky_operation",
    ...         inputs={},
    ...         output=None,
    ...         exception=e
    ...     )

    Nesting calls:
    >>> parent_call = weave.log_call("parent", {"input": 1}, 2)
    >>> child_call = weave.log_call(
    ...     "child",
    ...     {"input": 2},
    ...     4,
    ...     parent=parent_call
    ... )

    Logging without adding to call stack:
    >>> call = weave.log_call(
    ...     op="background_task",
    ...     inputs={"task_id": 123},
    ...     output="completed",
    ...     use_stack=False  # Call 스택에 추가하지 않음
    ... )
````

### <kbd>함수</kbd> `require_current_call`

```python
require_current_call() → Call
```

이렇게 하면 실행되는 동안 ID나 retour와 같은 Call의 속성에 액세스할 수 있습니다.

```python
@weave.op
def hello(name: str) -> None:
     print(f"Hello {name}!")
     current_call = weave.require_current_call()
     print(current_call.id)
```

Op가 반환된 후에도 Call에 액세스할 수 있습니다.

UI 등에서 얻은 Call의 id가 있다면, `weave.init`에서 반환된 `WeaveClient`의 `get_call` 방법을 사용하여 Call 객체를 조회할 수 있습니다.

```python
client = weave.init("<project>")
mycall = client.get_call("<call_id>")
```

또는 Op를 정의한 후 `call` 방법을 사용할 수 있습니다. 예를 들면:

```python
@weave.op
def add(a: int, b: int) -> int:
     return a + b

result, call = add.call(1, 2)
print(call.id)
```

**반환값:**
현재 실행 중인 Op의 Call 객체

**예외:**

* <b>`NoCurrentCallError`</b>:  tracking이 초기화되지 않았거나 이 방법이 Op 외부에서 호출된 경우입니다.
