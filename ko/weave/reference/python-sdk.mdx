## <kbd>class</kbd> `Audio`

지원되는 형식(wav 또는 mp3)의 오디오 데이터를 나타내는 클래스입니다. 

이 클래스는 오디오 데이터 저장을 처리하고, 다양한 소스에서 로드하거나 파일로 내보내기 위한 메서드를 제공합니다. 

**Attributes:**

- <b>`format`</b>:  오디오 형식(현재 'wav' 또는 'mp3' 지원) 
 - <b>`data`</b>:  바이트 형태의 원시 오디오 데이터 

**Args:**

- <b>`data`</b>: 오디오 데이터(바이트 또는 base64 인코딩 문자열)
 - <b>`format`</b>: 오디오 형식('wav' 또는 'mp3')
 - <b>`validate_base64`</b>: 입력 데이터를 base64 디코딩할지 여부

**Raises:**
 
 - <b>`ValueError`</b>:  오디오 데이터가 비어 있거나 형식이 지원되지 않는 경우

## <kbd>class</kbd> `Content`

여러 소스에서 온 콘텐츠를 표현하는 클래스로, 이를 관련 메타데이터와 함께 통합된 바이트 기반 표현으로 해석합니다. 

이 클래스는 다음 클래스 메서드 중 하나를 사용해 인스턴스를 생성해야 합니다: 

- from_path() 
- from_bytes() 
- from_text() 
- from_url() 
- from_base64() 
- from_data_url()

## <kbd>class</kbd> `Dataset`

간편한 저장과 자동 버전 관리를 제공하는 `Dataset` 객체입니다.

**예시:**

```python
# 데이터셋 생성
dataset = Dataset(name='grammar', rows=[
     {'id': '0', 'sentence': "He no likes ice cream.", 'correction': "He doesn't like ice cream."},
     {'id': '1', 'sentence': "She goed to the store.", 'correction': "She went to the store."},
     {'id': '2', 'sentence': "They plays video games all day.", 'correction': "They play video games all day."}
])

# 데이터셋 게시
weave.publish(dataset)

# 데이터셋 가져오기
dataset_ref = weave.ref('grammar').get()

# 특정 예제 접근
example_label = dataset_ref.rows[2]['sentence']
```

**Pydantic 필드:**

* `name`: `str | None`
* `description`: `str | None`
* `ref`: `trace.refs.ObjectRef | None`
* `rows`: `trace.table.Table | trace.vals.WeaveTable`

## <kbd>class</kbd> `Evaluation`

scorer 집합과 dataset으로 구성된 Evaluation을 초기화합니다.

evaluation.evaluate(model)을 호출하면 dataset의 행들을 model에 전달하면서, dataset의 열 이름을 model.predict의 인자 이름에 맞춰 매핑합니다.

그런 다음 모든 scorer를 호출하고 결과를 Weave에 저장합니다.

dataset의 행을 전처리하고 싶다면 preprocess&#95;model&#95;input에 함수를 전달할 수 있습니다.

**예시:**

```python
# 예시를 수집합니다
examples = [
     {"question": "What is the capital of France?", "expected": "Paris"},
     {"question": "Who wrote 'To Kill a Mockingbird'?", "expected": "Harper Lee"},
     {"question": "What is the square root of 64?", "expected": "8"},
]

# 사용자 정의 채점 함수를 정의합니다
@weave.op
def match_score1(expected: str, model_output: dict) -> dict:
     # 모델 출력을 채점하는 로직을 여기에 정의합니다
     return {'match': expected == model_output['generated_text']}

@weave.op
def function_to_evaluate(question: str):
     # LLM 호출을 추가하고 출력을 반환하는 부분입니다
     return  {'generated_text': 'Paris'}

# 채점 함수를 사용하여 예시를 채점합니다
evaluation = Evaluation(
     dataset=examples, scorers=[match_score1]
)

# 평가 추적을 시작합니다
weave.init('intro-example')
# 평가를 실행합니다
asyncio.run(evaluation.evaluate(function_to_evaluate))
```

**Pydantic 필드:**

* `name`: `str | None`
* `description`: `str | None`
* `ref`: `trace.refs.ObjectRef | None`
* `dataset`: `<class 'dataset.dataset.Dataset'>`

* `trials`: `<class 'int'>`
* `metadata`: `dict[str, typing.Any] | None`
* `evaluation_name`: `str | collections.abc.Callable[trace.call.Call, str] | None`

### <kbd>메서드</kbd> `get_evaluate_calls`

```python
get_evaluate_calls() → PaginatedIterator[CallSchema, WeaveObject]
```

이 Evaluation 객체를 사용한 모든 평가 호출을 가져옵니다.

단일 호출이 아닌 `CallsIter`를 반환합니다. 하나의 Evaluation에 대해 여러 번 평가 호출이 발생할 수 있기 때문입니다(예: 동일한 Evaluation을 여러 번 실행한 경우).

**Examples:**

```python
evaluation = Evaluation(dataset=examples, scorers=[scorer])
await evaluation.evaluate(model)  # 먼저 평가를 실행
calls = evaluation.get_evaluate_calls()
for call in calls:
     print(f"Evaluation run: {call.id} at {call.started_at}")
```

### <kbd>method</kbd> `get_score_calls`

```python
     print(f"Trace {trace_id}: {len(calls)} scorer calls")

         print(f"  Scorer: {scorer_name}, Output: {call.output}")
```

## <kbd>class</kbd> `EvaluationLogger`

이 클래스는 Evaluation을 로깅하기 위한 명령형 인터페이스를 제공합니다.

평가는 `log_prediction` 메서드를 사용해 첫 예측을 로깅하면 자동으로 시작되며, `log_summary` 메서드를 호출하면 종료됩니다.

예측을 로깅할 때마다 `ScoreLogger` 객체를 반환합니다. 이 객체를 사용해 해당 예측에 대한 점수와 메타데이터를 로깅할 수 있습니다. 더 자세한 내용은 `ScoreLogger` 클래스를 참조하세요.

기본 사용 예 — 입력과 출력을 직접 지정해 예측을 로깅하기:

```python
ev = EvaluationLogger()

# 알려진 입력/출력으로 예측 로깅
pred = ev.log_prediction(inputs={'q': 'Hello'}, outputs={'a': 'Hi there!'})
pred.log_score("correctness", 0.9)

# 평가 완료
ev.log_summary({"avg_score": 0.9})
```

고급 사용법 - 동적 출력 및 중첩 연산을 위한 컨텍스트 매니저 활용:

```python
ev = EvaluationLogger()

# 중첩 작업을 캡처해야 할 때 컨텍스트 매니저 사용
with ev.log_prediction(inputs={'q': 'Hello'}) as pred:
     # 여기서 수행되는 모든 작업(예: LLM 호출)은 자동으로
     # predict 호출의 하위 항목이 됩니다
     response = your_llm_call(...)
     pred.output = response.content
     pred.log_score("correctness", 0.9)

# 평가 완료
ev.log_summary({"avg_score": 0.9})
```

### <kbd>메서드</kbd> `set_view`

```python
set_view(
    name: 'str',
```

`weave.views` 아래 evaluation의 메인 호출 요약에 뷰를 연결합니다.

제공된 콘텐츠를 프로젝트 내 객체로 저장하고, 해당 evaluation의 `evaluate` 호출에 대해 그 참조 URI를 `summary.weave.views.<name>` 아래에 기록합니다. 문자열 입력은 제공된 확장자 또는 mimetype을 사용해 `Content.from_text`로 텍스트 콘텐츠로 감싸서 저장합니다.

**인자(Args):**

```python
         return {'pred': prediction}
```

## <kbd>class</kbd> `ContextAwareThread`

호출자의 컨텍스트를 유지한 채 함수를 실행하는 Thread입니다.

이는 threading.Thread를 그대로 대체해 사용할 수 있는 클래스로, 스레드 내부에서의 호출이 예상대로 동작하도록 보장합니다. Weave는 특정 contextvars가 설정되어 있어야 하는데(call&#95;context.py 참조), 새 스레드는 부모로부터 컨텍스트를 자동으로 복사하지 않으므로 호출 컨텍스트가 사라질 수 있습니다. 이는 좋지 않습니다! 이 클래스는 contextvar 복사를 자동화하여, 사용자가 기대하는 대로 이 스레드를 사용하기만 하면 &quot;그냥 잘 동작&quot;하도록 해 줍니다.

이 클래스를 사용하지 않고도, 대신 다음과 같이 작성하면 동일한 효과를 얻을 수 있습니다:

```python
def run_with_context(func, *args, **kwargs):
     context = copy_context()
     def wrapper():
         context.run(func, *args, **kwargs)
     return wrapper

thread = threading.Thread(target=run_with_context(your_func, *args, **kwargs))
thread.start()
```

## <kbd>class</kbd> `ContextAwareThreadPoolExecutor`

호출자(caller)의 컨텍스트에서 함수를 실행하는 ThreadPoolExecutor입니다.

이는 concurrent.futures.ThreadPoolExecutor를 그대로 교체해 사용할 수 있는 클래스로, 실행자 내부에서 Weave 호출이 예상대로 동작하도록 보장합니다. Weave는 특정 contextvars가 설정되어 있어야 하지만(call&#95;context.py 참조), 새 스레드는 부모로부터 컨텍스트를 자동으로 복사하지 않기 때문에 호출 컨텍스트가 유실될 수 있습니다. 이는 바람직하지 않습니다! 이 클래스는 contextvars 복사를 자동화하여, 사용자가 기대하는 것처럼 이 실행자를 사용하면 &quot;그냥 잘 동작&quot;하도록 해 줍니다.

이 클래스를 사용하지 않고도, 대신 다음과 같이 작성해서 동일한 효과를 얻을 수 있습니다:

```python
with concurrent.futures.ThreadPoolExecutor() as executor:
     contexts = [copy_context() for _ in range(len(vals))]

     def _wrapped_fn(*args):
         return contexts.pop().run(fn, *args)

     executor.map(_wrapped_fn, vals)
```

### <kbd>함수</kbd> `log_call`

```python
log_call(
    op: 'str',
    inputs: 'dict[str, Any]',
    output: 'Any',
    parent: 'Call | None' = None,
    attributes: 'dict[str, Any] | None' = None,
    display_name: 'str | Callable[[Call], str] | None' = None,
    use_stack: 'bool' = True,
    exception: 'BaseException | None' = None
) → Call
```

데코레이터 패턴을 사용하지 않고 Weave에 호출을 직접 로그합니다.

이 함수는 Weave에 작업을 로그하기 위한 명령형 API를 제공합니다. 이미 실행이 끝난 호출을 나중에 로그하고 싶거나, 데코레이터 패턴이 사용 사례에 적합하지 않을 때 유용합니다.

**Args:**

* <b>`op`</b> (str):  로그할 연산 이름입니다. 이 값은 호출의 op&#95;name으로 사용됩니다. 게시된 op를 참조하지 않는 문자열(익명 연산)도 지원됩니다.
* <b>`inputs`</b> (dict[str, Any]):  이 연산에 대한 입력 매개변수 딕셔너리입니다.
* <b>`output`</b> (Any):  이 연산의 출력/결과입니다.
* <b>`parent`</b> (Call | None):  이 호출을 중첩시킬 선택적 상위 호출입니다. 제공되지 않으면, 호출은 루트 수준 호출이 되거나(또는 현재 호출 컨텍스트가 있다면 그 아래에 중첩됩니다). 기본값은 None입니다.
* <b>`attributes`</b> (dict[str, Any] | None):  호출에 첨부할 선택적 메타데이터입니다. 호출이 생성되면 이 값들은 고정(frozen)됩니다. 기본값은 None입니다.
* <b>`display_name`</b> (str | Callable[[Call], str] | None):  UI에서 호출에 사용할 선택적 표시 이름입니다. 문자열이거나, Call을 받아 문자열을 반환하는 callable일 수 있습니다. 기본값은 None입니다.
* <b>`use_stack`</b> (bool):  이 호출을 런타임 스택에 푸시할지 여부입니다. True인 경우 호출이 호출 컨텍스트에서 사용 가능하며 weave.require&#95;current&#95;call()을 통해 접근할 수 있습니다. False인 경우 호출은 로그되지만 호출 스택에는 추가되지 않습니다. 기본값은 True입니다.
* <b>`exception`</b> (BaseException | None):  연산이 실패했을 때 로그할 선택적 예외입니다. 기본값은 None입니다.

**Returns:**

* <b>`Call`</b>:  전체 트레이스 정보를 포함하는, 생성 및 종료된 Call 객체입니다.

**Examples:**
기본 사용:


````python
import weave
    >>> weave.init('my-project')
    >>> call = weave.log_call(
    ...     op="my_function",
    ...     inputs={"x": 5, "y": 10},
    ...     output=15
    ... )

    Logging with attributes and display name:
    >>> call = weave.log_call(
    ...     op="process_data",
    ...     inputs={"data": [1, 2, 3]},
    ...     output={"mean": 2.0},
    ...     attributes={"version": "1.0", "env": "prod"},
    ...     display_name="Data Processing"
    ... )

    Logging a failed operation:
    >>> try:
    ...     result = risky_operation()
    ... except Exception as e:
    ...     call = weave.log_call(
    ...         op="risky_operation",
    ...         inputs={},
    ...         output=None,
    ...         exception=e
    ...     )

    Nesting calls:
    >>> parent_call = weave.log_call("parent", {"input": 1}, 2)
    >>> child_call = weave.log_call(
    ...     "child",
    ...     {"input": 2},
    ...     4,
    ...     parent=parent_call
    ... )

    Logging without adding to call stack:
    >>> call = weave.log_call(
    ...     op="background_task",
    ...     inputs={"task_id": 123},
    ...     output="completed",
    ...     use_stack=False  # 호출 스택에 추가하지 않음
    ... )
````

### <kbd>함수</kbd> `require_current_call`

```python
require_current_call() → Call
```

현재 실행 중인 Op 내부에서 해당 Op의 Call 객체를 가져옵니다.

이를 통해 Op가 실행되는 동안 Call의 id나 feedback과 같은 속성에 접근할 수 있습니다.

```python
@weave.op
def hello(name: str) -> None:
     print(f"Hello {name}!")
     current_call = weave.require_current_call()
     print(current_call.id)
```

Op이 반환된 후에도 해당 Call에 접근할 수 있습니다.

UI에서 얻은 Call의 ID가 있다면, `weave.init`에서 반환된 `WeaveClient`의 `get_call` 메서드를 사용해 Call 객체를 가져올 수 있습니다.

```python
client = weave.init("<project>")
mycall = client.get_call("<call_id>")
```

또는 Op를 정의한 다음에는 해당 `call` 메서드를 사용할 수도 있습니다. 예를 들어:

```python
@weave.op
def add(a: int, b: int) -> int:
     return a + b

result, call = add.call(1, 2)
print(call.id)
```

**반환값:**
현재 실행 중인 Op의 Call 객체

**예외:**

* <b>`NoCurrentCallError`</b>:  추적이 초기화되지 않았거나 이 메서드가 Op 외부에서 호출된 경우 발생

### <kbd>function</kbd> `thread`

```python
thread(
    thread_id: 'str | None | object' = <object object at 0x1105c49a0>

```

컨텍스트 내에서 이루어지는 호출에 대해 thread&#95;id를 설정하는 컨텍스트 매니저입니다.

**예시:**

```python
# thread_id 자동 생성

     print(f"Thread ID: {t.thread_id}")

     result = my_function("input")  # 이 호출에는 thread_id="custom_thread"가 적용됩니다

# 스레딩 비활성화
with weave.thread(None) as t:
     result = my_function("input")  # 이 호출에는 thread_id=None이 적용됩니다
```

* <b>`thread_id`</b>: 이 컨텍스트 내의 호출에 연관 지을 스레드 식별자입니다. 제공하지 않으면 UUID v7이 자동 생성됩니다. None인 경우 스레드 추적이 비활성화됩니다.
  **반환(Yields):**

* <b>`ThreadContext`</b>:  `thread_id` 및 현재 `turn_id`에 접근할 수 있는 객체입니다.
