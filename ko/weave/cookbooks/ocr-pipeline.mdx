  ## 사전 요구 사항

시작하기 전에 필요한 라이브러리를 설치 및 임포트하고, W&amp;B API 키를 준비한 뒤 Weave 프로젝트를 초기화하세요.

<div id="1-create-and-iterate-on-prompts-with-weave">
  ## 1. Weave로 프롬프트 만들고 반복적으로 개선하기
</div>

좋은 프롬프트 엔지니어링은 모델이 entity를 정확하게 추출하도록 유도하는 데 매우 중요합니다. 먼저 이미지 데이터에서 무엇을 추출할지, 그리고 어떤 형식으로 출력할지에 대한 지침을 모델에 제공하는 기본 프롬프트를 만듭니다. 그런 다음 추적하고 반복적으로 개선할 수 있도록 이 프롬프트를 Weave에 저장합니다.

다음으로, 출력 오류를 줄일 수 있도록 프롬프트에 더 많은 지침과 검증 규칙을 추가해 개선합니다.

<div id="2-get-the-dataset">
  ## 2. 데이터셋 조회
</div>

다음으로, OCR 파이프라인의 입력으로 사용할 손글씨 메모 데이터셋을 조회합니다.

데이터셋의 이미지는 이미 `base64`로 인코딩되어 있으므로, LLM이 별도의 전처리 없이 바로 사용할 수 있습니다.

<div id="3-build-the-ner-pipeline">
  ## 3. NER 파이프라인 구축
</div>

다음으로, NER 파이프라인을 구축합니다. 이 파이프라인은 두 개의 함수로 구성됩니다.

1. 데이터셋의 PIL 이미지를 받아 VLM에 전달할 수 있도록 이미지의 `base64` 인코딩 문자열 표현을 반환하는 `encode_image` 함수
2. 이미지와 시스템 프롬프트를 받아, 시스템 프롬프트에 설명된 대로 해당 이미지에서 추출한 entity를 반환하는 `extract_named_entities_from_image` 함수

이제 다음을 수행하는 `named_entity_recognation` 함수를 만드세요.

* 이미지 데이터를 NER 파이프라인에 전달합니다
* 결과를 올바른 JSON 형식으로 반환합니다

함수 실행을 W&amp;B UI에서 자동으로 추적하고 트레이스하려면 [`@weave.op()` decorator](/ko/weave/reference/python-sdk/trace/op) 데코레이터를 사용하세요.

`named_entity_recognation`이 실행될 때마다 전체 트레이스 결과를 Weave UI에서 확인할 수 있습니다. 트레이스를 보려면 Weave 프로젝트의 **트레이스** 탭으로 이동하세요.

마지막으로, 데이터셋에 파이프라인을 실행하고 결과를 확인합니다.

다음 코드는 데이터셋을 순회하면서 결과를 로컬 파일 `processing_results.json`에 저장합니다. 결과는 Weave UI에서도 확인할 수 있습니다.

<div id="4-evaluate-the-pipeline-using-weave">
  ## 4. Weave를 사용해 파이프라인 평가하기
</div>

이제 VLM을 사용해 NER를 수행하는 파이프라인을 만들었으므로, Weave를 사용해 이를 체계적으로 평가하고 성능이 얼마나 잘 나오는지 확인할 수 있습니다. [Evaluations Overview](/ko/weave/guides/core-types/evaluations)에서 Weave의 Evaluations에 관해 자세히 알아볼 수 있습니다.

Weave 평가의 기본 구성 요소 중 하나는 [Scorers](/ko/weave/guides/evaluation/scorers)입니다. Scorers는 AI의 출력 결과를 평가하고 평가 메트릭을 반환하는 데 사용됩니다. AI의 출력 결과를 받아 이를 분석한 뒤, 결과를 딕셔너리 형태로 반환합니다. 필요하면 입력 데이터를 기준 참조로 사용할 수 있으며, 평가에 대한 설명이나 추론 같은 추가 정보도 함께 출력할 수 있습니다.

이 섹션에서는 파이프라인을 평가하기 위해 두 개의 Scorer를 만듭니다:

1. 프로그래매틱 Scorer
2. LLM-as-a-judge Scorer

<div id="programatic-scorer">
  ### 프로그래매틱 scorer
</div>

프로그래매틱 scorer인 `check_for_missing_fields_programatically`는 모델 출력(`named_entity_recognition` 함수의 출력)을 받아 결과에서 누락되었거나 비어 있는 `keys`를 파악합니다.

이 검사는 모델이 어떤 필드도 추출하지 못한 샘플을 파악하는 데 매우 유용합니다.

<div id="llm-as-a-judge-scorer">
  ### LLM-as-a-judge scorer
</div>

평가의 다음 step에서는 평가가 실제 NER 성능을 반영하도록 이미지 데이터와 모델 출력이 모두 제공됩니다. 모델 출력만 참조하는 것이 아니라 이미지 콘텐츠도 명시적으로 참조합니다.

이 step에 사용되는 scorer인 `check_for_missing_fields_with_llm`은 LLM(구체적으로는 OpenAI의 `gpt-4o`)을 사용해 점수를 매깁니다. `eval_prompt`의 내용에 지정된 대로 `check_for_missing_fields_with_llm`은 `Boolean` 값을 출력합니다. 모든 필드가 이미지의 정보와 일치하고 형식도 올바르면 scorer는 `true`를 반환합니다. 필드가 하나라도 누락되거나 비어 있거나 올바르지 않거나 서로 일치하지 않으면 결과는 `false`이며, scorer는 문제를 설명하는 메시지도 함께 반환합니다.

<div id="5-run-the-evaluation">
  ## 5. 평가 실행
</div>

마지막으로, 전달된 `dataset`을 자동으로 순회하며 결과를 Weave UI에 함께 기록하는 평가 call을 정의합니다.

다음 코드는 평가를 시작하고 NER 파이프라인의 모든 출력에 두 개의 Scorer를 적용합니다. 결과는 Weave UI의 **Evals** 탭에서 확인할 수 있습니다.

```python lines
evaluation = weave.Evaluation(
    dataset=dataset,
    scorers=[
        check_for_missing_fields_with_llm,
        check_for_missing_fields_programatically,
    ],
    name="Evaluate_4.1_NER",
)

print(await evaluation.evaluate(named_entity_recognation))
```

위 코드를 실행하면 Weave UI의 평가 테이블로 연결되는 링크가 생성됩니다. 이 링크를 따라가 결과를 확인하고, 원하는 모델, 프롬프트, Datasets 전반에 걸쳐 파이프라인의 여러 반복을 비교하세요. Weave UI는 팀을 위해 아래와 같은 시각화를 자동으로 생성합니다.
