title: 인테그레이션 Overview
description: "OpenAI, Anthropic 등을 위한 Weave의 automatic patching으로 30개 이상의 제공자와 프레임워크 전반에서 LLM calls를 트레이스하고 모니터링하세요."
---

W&amp;B Weave는 널리 사용되는 LLM 제공자와 오케스트레이션 프레임워크를 위한 logging 인테그레이션을 제공합니다. 이러한 인테그레이션을 사용하면 다양한 라이브러리를 통해 이루어지는 Call을 원활하게 트레이스할 수 있어, AI 애플리케이션을 모니터링하고 분석하는 능력을 향상할 수 있습니다.

애플리케이션에서 LLM 제공자 라이브러리(OpenAI, Anthropic, Cohere, Mistral 등)를 사용하는 경우, 해당 API Call이 W&amp;B Weave에 트레이스된 Calls로 표시되기를 원할 것입니다. 즉 inputs, outputs, 지연 시간, token 사용량, 비용이 기록됩니다. 도움이 없다면 모든 `client.chat.completions.create()`(또는 이에 상응하는 호출)를 `@weave.op` 또는 manual instrumentation으로 감싸야 하며, 이는 번거롭고 무언가를 놓치기 쉽습니다.

Weave는 지원되는 LLM 클라이언트 라이브러리를 자동으로 가로채어(patch) 처리합니다. 애플리케이션 코드는 변경되지 않습니다. 평소처럼 제공자 SDK를 사용하면 각 Request가 Weave Call로 기록됩니다. 최소한의 설정으로 전체 Tracing을 이용할 수 있습니다.

<div id="protocols">
  ## 프로토콜
</div>

Weave는 AI 애플리케이션과 이를 지원하는 서비스 간의 통신을 가능하게 하는 표준화된 프로토콜과 통합됩니다.

* **[Model Context Protocol (MCP)](/ko/weave/guides/integrations/mcp)**

위 목록에서 인테그레이션을 선택하여 선호하는 LLM 제공자, 프레임워크 또는 프로토콜과 함께 Weave를 사용하는 방법을 자세히 알아보세요. LLM API에 직접 액세스하든, 복잡한 파이프라인을 구축하든, 표준화된 프로토콜을 사용하든, Weave는 AI 애플리케이션을 효과적으로 트레이스하고 분석하는 데 필요한 도구를 제공합니다.
