description: "추적된 함수 호출과 평가와 상호작용하는 방식을 사용자 지정합니다"
---

Weave에서 _저장된 뷰_를 사용하면 추적된 함수 호출과 평가와 상호작용하는 방식을 사용자 지정할 수 있습니다. 저장된 뷰를 정의하면 필터, 정렬, 열 표시 여부를 구성해 관련 데이터를 빠르게 확인할 수 있습니다.

저장된 뷰는 Weave Python SDK나 UI에서 직접 생성, 수정, 저장할 수 있습니다. Python SDK는 필터링과 쿼리를 프로그램적으로 세밀하게 제어할 수 있게 해주고, UI는 **Traces** 및 **Evals** 탭에서 다양한 테이블 구성을 쉽게 탐색하고 저장할 수 있도록 도와줍니다.

이 가이드에서는 다음 내용을 다룹니다:

- [Python SDK에서 저장된 뷰를 생성하고 수정하는 방법](#saved-views-in-the-python-sdk).
- [Weave UI에서 저장된 뷰를 생성하고 사용하는 방법](#saved-views-in-the-ui).

<div id="saved-views-in-the-python-sdk">
  ## Python SDK의 저장된 뷰
</div>

Weave의 `SavedView` 클래스는 trace 및 evals 데이터 뷰를 저장, 필터링, 정렬하고 사용자 지정할 수 있는 방법을 제공합니다. 

<div id="initialize-a-savedview">
  ### `SavedView` 초기화
</div>

Weave 프로젝트에서 `SavedView` 인스턴스를 초기화하세요:

```python lines
import weave
client = weave.init(<my_project>)

view = weave.SavedView()
```


<div id="visualize-the-savedview-as-a-grid">
  ### `SavedView`를 그리드로 시각화하기
</div>

`.to_grid()`를 사용해 `SavedView`를 그리드 형태로 표시합니다. `limit`으로 표시할 최대 행 수를 지정합니다.

```python lines
view.to_grid(limit=5)
```

`.show()`를 사용해 그리드를 표시합니다:

```python lines
view.to_grid().show()
```


<div id="set-displayed-columns">
  ### 표시할 열 설정
</div>

`.set_columns()`를 사용하여 뷰에 표시할 열을 설정합니다. 표시할 열을 하나 이상 지정하세요.

```python lines
view.set_columns("id", "op_name")
```


<div id="add-columns">
  ### 열 추가
</div>

`.add_column()`을 사용해서 뷰에 하나 이상의 새 열을 추가하세요. 추가할 열을 하나 이상 지정하세요.

```python lines
# 필드 지정자와 레이블 "Created"로 열 추가
view.add_column("Created")
# 선택 사항으로, 두 번째 인수를 추가하여 새 열에 다른 레이블 이름을 지정할 수 있습니다. 기본적으로 필드 지정자가 레이블로 사용됩니다.
```


<div id="sort-columns">
  ### 열 정렬
</div>

`.sort_by()`를 사용해 특정 열을 기준으로 결과를 정렬하세요. 정렬할 열 이름과 정렬 순서(`asc` 또는 `desc`)를 지정하세요.

```python lines
view.sort_by("started_at", "desc")
```


<div id="filter-by-operation-name">
  ### operation 이름으로 필터링
</div>

Weave에서 모든 trace나 eval에는 operation 이름이 연결됩니다.
`.filter_op()`을 사용하여 해당 operation이 실행된 호출만 포함하도록 `SavedView`를 필터링하세요.

```python lines
view.filter_op("Evaluation.predict_and_score")
```


<div id="filter-by-operator-and-condition">
  ### 연산자와 조건으로 필터링
</div>

`.add_filter()`를 사용하여 뷰에 커스텀 필터를 적용합니다. [지원되는 필터 연산자](#filter-operators)와 조건을 조합해 필터를 정의합니다.

```python lines
view.add_filter("output.model_latency", ">=", 5)
```


<div id="filter-operators">
  #### 필터 연산자
</div>

| Operator | Description | Example |
|----------|-------------|---------|
| `"contains"` | 문자열에 특정 부분 문자열이 포함되어 있는지 확인합니다. | `view.add_filter("output.status", "contains", "error")` |
| `"equals"` | 문자열이 지정된 값과 정확히 같은지 확인합니다. | `view.add_filter("input.category", "equals", "Alice")` |
| `"in"` | 문자열이 값 목록에 포함되어 있는지 확인합니다. | `view.add_filter("category", "in", ["A", "B", "C"])` |
| `"="` | 숫자가 지정된 값과 같은지 확인합니다. | `view.add_filter("output.score", "=", 80)` |
| `"≠", "!="` | 숫자가 지정된 값과 같지 않은지 확인합니다. | `view.add_filter("metrics.loss", "!=", 0.5)` |
| `"<"` | 숫자가 지정된 값보다 작은지 확인합니다. | `view.add_filter("age", "<", 30)` |
| `"≤", "<="` | 숫자가 지정된 값보다 작거나 같은지 확인합니다. | `view.add_filter("metric.value", "<=", 100)` |
| `">"` | 숫자가 지정된 값보다 큰지 확인합니다. | `view.add_filter("output.score", ">", 90)` |
| `"≥", ">="` | 숫자가 지정된 값보다 크거나 같은지 확인합니다. | `view.add_filter("output.model_latency", ">=", 5)` |
| `"is"` | 불리언 필드가 `True`인지 `False`인지 확인합니다. | `view.add_filter("is_active", "is", True)` |
| `"after"` | 날짜가 지정된 타임스탬프보다 이후인지 확인합니다. | `view.add_filter("started_at", "after", "2024-01-01")` |
| `"before"` | 날짜가 지정된 타임스탬프보다 이전인지 확인합니다. | `view.add_filter("ended_at", "before", "2024-12-31")` |
| `"is empty"` | 필드가 비어 있는지(`None` 또는 `""`) 확인합니다. | `view.add_filter("comments", "is empty", None)` |
| `"is not empty"` | 필드가 비어 있지 않은지 확인합니다. | `view.add_filter("attachments", "is not empty", None)` |

<div id="remove-filters">
  ### 필터 제거
</div>

`.remove_filter()`를 사용하여 인덱스 또는 필드 이름을 기준으로 뷰에서 특정 필터를 제거합니다.

```python lines
view.remove_filter("output.model_latency")
```

모든 필터를 제거하려면 `.remove_filters()` 메서드를 사용하세요.

```python lines
view.remove_filters()
```


<div id="save-the-savedview">
  ### `SavedView` 저장하기
</div>

`.save()`를 사용해 저장한 뷰를 Weave에 게시합니다.

```python lines
view.save()
```


<div id="retrieve-function-calls">
  ### 함수 호출 가져오기
</div>

`.get_calls()`를 사용하여 저장된 뷰의 필터와 일치하는 함수 호출을 가져옵니다. `limit` 및 `offset`과 같은 선택적 매개변수를 지정할 수 있습니다.

```python lines
calls = view.get_calls(limit=10)
```


<div id="saved-views-in-the-ui">
  ## UI에서 저장된 뷰
</div>

Weave UI에서 저장된 뷰를 생성하고, 불러오고, 이름을 바꾸고, 편집할 수 있습니다. 보다 세밀하게 제어하려면 [Python SDK](#saved-views-in-the-python-sdk)를 사용하세요.

<div id="create-a-saved-view">
  ### 저장된 뷰 생성
</div>

1. **Traces** 또는 **Evals** 탭으로 이동합니다.
2. 테이블 설정에서 다음 항목을 조정합니다.
   - Filters
   - Sort order
   - Page size
   - Column visibility
   - Column pinning
3. 다음 두 가지 방법 중 하나로 뷰를 저장합니다.
   - 오른쪽 상단에서 **Save view**를 클릭합니다. 
   - **Save view** 왼쪽에 있는 햄버거 메뉴 아이콘을 클릭합니다. 드롭다운 메뉴에서 **+ Save as new view**를 클릭합니다.

<div id="load-a-saved-view">
  ### 저장된 뷰 불러오기
</div>

1. **Traces** 또는 **Evals** 탭으로 이동합니다.
2. 탭 제목 왼쪽에 있는 햄버거 메뉴를 클릭합니다. 저장된 모든 뷰가 표시된 드롭다운 메뉴가 나타납니다. 
3. 열고 싶은 뷰를 클릭합니다. 선택한 저장된 뷰가 **Traces** 또는 **Evals** 탭에 표시됩니다. 

<div id="rename-a-saved-view">
  ### 저장된 뷰 이름 변경
</div>

1. [저장된 뷰 불러오기](#load-a-saved-view)에 설명된 단계를 따릅니다.
2. **Traces** 또는 **Evals** 탭의 왼쪽 상단에 있는 뷰 이름을 클릭합니다.
3. 뷰의 새 이름을 입력합니다.
4. 새 뷰 이름을 저장하려면 **Enter** 키를 누릅니다.

<div id="edit-a-saved-view">
  ### 저장된 뷰 편집
</div>

1. [저장된 뷰 불러오기](#load-a-saved-view)에 설명된 단계를 따릅니다.
2. 테이블 구성을 원하는 대로 조정합니다.
3. 오른쪽 상단에서 **Save view**를 클릭합니다. 

<div id="delete-a-saved-view">
  ### 저장된 뷰 삭제
</div>

<Warning>
더 이상 본인이나 팀에 도움이 되지 않는다고 판단되는 뷰는 삭제할 수 있습니다. 이 작업은 되돌릴 수 없습니다.
</Warning>

1. **Traces** 또는 **Evals** 탭으로 이동합니다.
2. 삭제하려는 [뷰를 불러옵니다](#load-a-saved-view).
3. **Save view** 왼쪽에 있는 햄버거 메뉴를 클릭합니다. 
4. 드롭다운 메뉴에서 **Delete view**를 클릭합니다.
5. 팝업 창에서 **Delete view**를 클릭해 삭제를 확인합니다. 또는 **Cancel**을 클릭하여 삭제를 중단합니다.

<div id="return-to-the-default-view">
  ### 기본 뷰로 돌아가기
</div>

1. **Traces** 또는 **Evals** 탭으로 이동합니다.
2. **Traces** 또는 **Evals** 탭 오른쪽의 햄버거 메뉴를 클릭합니다. 저장된 모든 뷰가 표시되는 드롭다운 메뉴가 나타납니다. 
3. 메뉴 하단에서 **Traces** 또는 **Evals**를 클릭합니다. 기본 뷰가 표시됩니다.
