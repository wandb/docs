[프롬프트 생성 및 게시](/ko/weave/guides/core-types/prompts.mdx) 후에는 코드와 프로덕션에서 특정 버전을 레퍼런스로 사용하고 조회하며 사용할 수 있습니다.

`weave.publish`로 프롬프트를 게시하면 Weave가 변경 불가능한 버전을 생성합니다. 각 버전은 이름과 버전으로 레퍼런스로 사용할 수 있고, `weave.ref(...).get()`으로 코드에서 조회할 수 있으며, Weave UI에서 확인할 수 있습니다.

<div id="prompt-versions">
  # Prompt versions
</div>

Weave의 프롬프트는 **버전 관리되는 객체**로 저장됩니다. 이를 통해 프롬프트를 안전하게 반복적으로 개선하면서도, 평가, 실험, 프로덕션 시스템이 실행에 사용된 정확한 프롬프트 버전을 레퍼런스로 사용할 수 있습니다.

프롬프트 버전은 Weave 객체 레퍼런스를 사용해 액세스합니다. 이 페이지에서는 프롬프트 버전이 생성되는 방식, 코드에서 특정 버전을 조회하는 방법, 그리고 Weave UI에서 버전을 확인하고 비교하는 방법을 설명합니다.

<div id="how-prompt-versions-are-created">
  ## 프롬프트 버전이 생성되는 방식
</div>

Weave는 프롬프트의 모든 버전을 자동으로 추적하여 프롬프트가 어떻게 발전해 왔는지에 대한 전체 이력을 생성합니다. 이 버전 관리 시스템은 프롬프트 엔지니어링 워크플로에서 매우 중요합니다. 이를 통해 안전하게 실험하고, 어떤 변경이 성능을 개선했거나 저하시켰는지 추적하고, 필요할 경우 이전 버전으로 쉽게 롤백할 수 있습니다. 같은 이름으로 내용만 다르게 프롬프트를 게시할 때마다 Weave는 이전 버전을 모두 보존하면서 새 버전을 생성합니다.

각 버전은 변경 불가능합니다. 한 번 생성되면 해당 버전의 내용은 바꿀 수 없습니다. 프롬프트를 업데이트한 후 다시 게시하면 Weave는 이전 버전을 보존하면서 새 버전을 생성합니다.

버전 관리를 사용하면 다음을 수행할 수 있습니다:

* 과거 실험을 재현합니다.
* 프롬프트 변경을 안전하게 반복 적용합니다.
* 프롬프트 업데이트를 통제된 방식으로 배포합니다.

<div id="construct-a-fully-qualified-ref-uri">
  ## 정규화된 ref URI 구성
</div>

Weave 객체는 정규화된 ref URI를 사용해 고유하게 식별할 수 있습니다.

정규화된 Weave 객체 ref URI는 다음과 같습니다:

```text
weave:///<your-team-name>/<your-project-name>/object/<object_name>:<object_version>
```

* *your-team-name*: W&amp;B entity(사용자 이름 또는 팀 이름)
* *your-project-name*: W&amp;B 프로젝트
* *object&#95;name*: 객체 이름
* *object&#95;version*: 버전 해시, `v0` 또는 `v1` 같은 문자열, 또는 `:latest` 같은 별칭입니다. 모든 객체에는 `:latest` 별칭이 있습니다.

예를 들어, 정규화된 Weave 프롬프트 URI는 다음과 같습니다:

```text
weave:///your-team-name/your-project-name/object/support_prompt:v3
```

<div id="retrieve-a-prompt-version-in-code">
  ## 코드에서 프롬프트 버전 조회
</div>

프롬프트를 조회하려면 이름과 버전에 대한 레퍼런스를 생성한 다음 `.get()`을 호출해 로드하세요. ref는 저장된 객체를 가리키며, `.get()`은 해당 객체를 가져와 애플리케이션에서 사용할 수 있게 합니다.

ref는 몇 가지 방식으로 생성할 수 있습니다:

* `weave.ref(<name>)`: 프롬프트의 `:latest` 버전을 조회합니다. `weave.init(...)` 호출이 필요합니다.
* `weave.ref(<name>:<version>)`:  지정한 버전의 프롬프트를 조회합니다. `weave.init(...)` 호출이 필요합니다.
* `weave.ref(<fully_qualified_ref_uri>)`: 지정된 [정규화된 ref URI](#construct-a-fully-qualified-ref-uri)에 있는 프롬프트를 조회합니다.  `weave.init(...)` 호출은 필요하지 않습니다.

다음 예시는 애플리케이션에서 사용할 수 있도록 `support_prompt:v3` 버전을 로드합니다:

```python lines
import weave

weave.init("your-team-name/your-project-name")

prompt = weave.ref("support_prompt:v3").get()
```

<div id="use-prompt-versions-in-production">
  ## 프로덕션에서 프롬프트 버전 사용하기
</div>

프로덕션 시스템에 프롬프트를 배포할 때는 최신 버전을 레퍼런스로 사용하는 대신 특정 프롬프트 버전을 고정하세요. 프로덕션 애플리케이션에서는 `weave.ref(<name>:<version>).get()`으로 해당 버전을 로드하세요.

버전을 고정하면 나중에 새 프롬프트 버전이 게시되더라도 프로덕션 동작이 안정적으로 유지됩니다.

일반적인 워크플로는 다음과 같습니다:

1. 새 프롬프트 버전을 개발하고 테스트합니다.
2. 데이터셋 또는 평가 스위트를 기준으로 새 프롬프트를 평가합니다.
3. 프로덕션 시스템이 새 버전을 레퍼런스로 사용하도록 업데이트합니다.

이 접근 방식을 사용하면 팀이 프로덕션 동작을 예기치 않게 변경하지 않고도 프롬프트를 안전하게 반복 개선할 수 있습니다.

## <a id="view-prompts" aria-label="프롬프트 버전 보기 및 비교" />프롬프트 버전 보기 및 비교

UI에서 프롬프트 버전을 보려면 다음 단계를 따르세요.

1. [wandb.ai](https://wandb.ai/)로 이동한 다음 프로젝트를 선택합니다.
2. Weave 프로젝트 사이드바에서 **Assets**를 클릭합니다. Assets 페이지가 열립니다.
3. Assets 페이지에서 **Prompts**를 클릭합니다. 그러면 프로젝트의 프롬프트가 나열된 Prompts 페이지가 열립니다.
4. **Versions** 열에서 보려는 프롬프트의 **(x) Versions**를 클릭합니다. 프롬프트 버전 목록이 열립니다.

![선택한 프롬프트의 버전을 보여주는 Prompt Assets UI.](/weave/guides/core-types/imgs/prompt-object.png)

5. (선택) 목록에 있는 프롬프트 옆의 체크박스를 클릭한 다음 테이블 툴바에서 **Compare** 버튼을 클릭하면 프롬프트 버전을 비교할 수 있습니다. 그러면 프롬프트 간 diff를 확인할 수 있습니다.

![서로 다른 두 프롬프트 버전 간의 diff를 보여주는 Compare prompts UI.](/weave/guides/core-types/imgs/prompt-comparison.png)
