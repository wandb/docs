&#95;평가 기반 LLM 애플리케이션 개발&#95;은 일관되고 엄선된 예시를 사용해 LLM 애플리케이션의 동작을 일관되게 측정함으로써 애플리케이션을 체계적으로 개선하는 데 도움이 됩니다.

<Tabs>
  <Tab title="Python">
    Weave에서 워크플로의 핵심은 &#95;`Evaluation` 객체&#95;이며, 다음을 정의합니다:

    * 테스트 예제용 [`데이터셋`](../core-types/datasets) 또는 딕셔너리 목록
    * 하나 이상의 [스코어링 함수](../evaluation/scorers)
    * [입력 전처리](#format-dataset-rows-before-evaluating)와 같은 선택적 설정

    `Evaluation`을 정의한 후에는 [`Model`](../core-types/models) 객체나 LLM 애플리케이션 로직을 포함하는 맞춤형 함수에 대해 실행할 수 있습니다. `.evaluate()`를 호출할 때마다 &#95;평가 런&#95;이 트리거됩니다. `Evaluation` 객체를 설계도로, 각 평가 런을 해당 설정에서 애플리케이션 성능을 측정한 결과로 생각하면 됩니다.
  </Tab>

  <Tab title="TypeScript">
    Weave에서 워크플로의 핵심은 &#95;`Evaluation` 객체&#95;이며, 다음을 정의합니다:

    * 테스트 예제용 [`데이터셋`](../core-types/datasets) 또는 객체 배열
    * 하나 이상의 [스코어링 함수](../evaluation/scorers)

    `Evaluation`을 정의한 후에는 `weave.op`으로 래핑된 함수에 대해 실행할 수 있습니다. `.evaluate()`를 호출할 때마다 &#95;평가 런&#95;이 트리거됩니다. `Evaluation` 객체를 설계도로, 각 평가 런을 해당 설정에서 애플리케이션 성능을 측정한 결과로 생각하면 됩니다.

    <Note>
      TypeScript SDK는 함수 기반 모델과 스코어러를 사용합니다. 클래스 기반 `Model` 및 `Scorer` 유형은 TypeScript에서 아직 사용할 수 없습니다.
    </Note>
  </Tab>
</Tabs>

<div id="1-create-an-evaluation-object">
  ## 1. `Evaluation` 객체 생성
</div>

`Evaluation` 객체를 생성하는 것은 평가 설정을 구성하는 첫 번째 step입니다. `Evaluation`은 예시 데이터, 스코어링 로직, 그리고 선택적 전처리로 구성됩니다. 이후 이를 사용해 하나 이상의 평가를 실행합니다.

Weave는 각 예시를 가져와 사용자의 애플리케이션에 통과시킨 다음, 여러 맞춤형 스코어링 함수로 output에 점수를 매깁니다. 이렇게 하면 애플리케이션의 성능을 확인할 수 있는 뷰와, 개별 output과 점수를 자세히 살펴볼 수 있는 풍부한 UI를 사용할 수 있습니다.

<div id="optional-custom-naming">
  ### (선택 사항) 맞춤형 이름 지정
</div>

<Tabs>
  <Tab title="Python">
    평가 흐름에서 사용자 지정 가능한 이름의 두 가지 유형이 있습니다:

    * [*Evaluation object name* (`evaluation_name`)](#name-the-evaluation-object): 설정된 `Evaluation` 객체에 대한 지속적인 레이블입니다.
    * [*Evaluation run display name* (`__weave["display_name"]`)](#name-individual-evaluation-runs): UI에 표시되는 특정 평가 런에 대한 레이블입니다.

    #### `Evaluation` 객체 이름 지정

    `Evaluation` 객체 자체에 이름을 지정하려면 `Evaluation` class에 `evaluation_name` 매개변수를 전달하세요. 이 이름은 코드와 UI 목록에서 `Evaluation`을 파악하는 데 도움이 됩니다.

    ```python lines
    evaluation = Evaluation(
        dataset=examples, scorers=[match_score1], evaluation_name="My Evaluation"
    )
    ```

    #### 개별 평가 런 이름 지정

    특정 평가 런(`evaluate()` 호출)에 이름을 지정하려면 `__weave` 딕셔너리에 `display_name`을 사용하세요. 이는 해당 런이 UI에 표시되는 방식에 영향을 줍니다.

    ```python lines
    evaluation = Evaluation(
        dataset=examples, scorers=[match_score1]
    )
    evaluation.evaluate(model, __weave={"display_name": "My Evaluation Run"})
    ```
  </Tab>

  <Tab title="TypeScript">
    `Evaluation` 객체에 이름을 지정하려면 `Evaluation` 생성자에 `id` 매개변수를 전달하세요. 이 이름은 코드와 UI 목록에서 `Evaluation`을 파악하는 데 도움이 됩니다.

    ```typescript lines
    const evaluation = new weave.Evaluation({
      id: 'my-evaluation',
      dataset: dataset,
      scorers: [matchScore],
    });
    ```
  </Tab>
</Tabs>

<div id="3-define-scoring-functions">
  ## 3. 스코어링 함수 정의
</div>

그런 다음 하나 이상의 [스코어링 함수](../evaluation/scorers)를 생성합니다. 이 함수는 `데이터셋`의 각 예시에 점수를 매기는 데 사용됩니다.

<Tabs>
  <Tab title="Python">
    각 스코어링 함수는 `output` 파라미터를 반드시 포함해야 하며, 점수가 담긴 딕셔너리를 반환해야 합니다. 필요에 따라 예시에 포함된 다른 입력도 추가할 수 있습니다.

    스코어링 함수에는 `output` 키워드 인수가 있어야 하지만, 다른 인수는 사용자가 정의하며 데이터셋 예시에서 가져옵니다. 인수 이름에 해당하는 딕셔너리 키를 기준으로 필요한 값만 가져옵니다.

    <Tip>
      스코어러가 `output` 인수를 기대하는데 전달받지 못한다면, 레거시 `model_output` 키를 사용하고 있는지 확인하세요. 해결하려면 스코어러 함수가 키워드 인수로 `output`을 사용하도록 업데이트하세요.
    </Tip>

    다음 예시 스코어러 함수 `match_score1`은 점수 계산에 `examples` 딕셔너리의 `expected` 값을 사용합니다.

    ```python lines
    import weave

    # 예시를 수집합니다
    examples = [
        {"question": "What is the capital of France?", "expected": "Paris"},
        {"question": "Who wrote 'To Kill a Mockingbird'?", "expected": "Harper Lee"},
        {"question": "What is the square root of 64?", "expected": "8"},
    ]

    # 맞춤형 스코어링 함수를 정의합니다
    @weave.op()
    def match_score1(expected: str, output: dict) -> dict:
        # 여기에서 모델 출력에 점수를 매기는 로직을 정의합니다
        return {'match': expected == output['generated_text']}
    ```

    ### (선택 사항) 맞춤형 `Scorer` class 정의하기

    일부 애플리케이션에서는 맞춤형 `Scorer` class를 만들고자 할 수 있습니다. 예를 들어, 특정 파라미터(예: 채팅 모델, 프롬프트)를 사용하고, 각 행에 대한 점수 계산 방식과 집계 점수 계산 방식을 지정한 표준화된 `LLMJudge` class를 만들 수 있습니다.

    자세한 내용은 [RAG 애플리케이션의 모델 기반 평가](/ko/weave/tutorial-rag#optional-defining-a-scorer-class) 튜토리얼에서 `Scorer` class를 정의하는 섹션을 참조하세요.
  </Tab>

  <Tab title="TypeScript">
    각 스코어링 함수는 `weave.op`으로 래핑되며, `modelOutput` 및 `datasetRow` 속성이 있는 객체를 전달받습니다.

    다음 예시 스코어러 함수 `matchScore`는 모델 출력을 데이터셋 행의 `expected` 값과 비교합니다.

    ```typescript lines
    import * as weave from 'weave';

    // 예시들을 하나의 데이터셋으로 수집합니다
    const dataset = new weave.Dataset({
      id: 'my-dataset',
      rows: [
        {question: 'What is the capital of France?', expected: 'Paris'},
        {question: 'Who wrote "To Kill a Mockingbird"?', expected: 'Harper Lee'},
        {question: 'What is the square root of 64?', expected: '8'},
      ],
    });

    // 맞춤형 스코어링 함수를 정의합니다
    const matchScore = weave.op(
      ({modelOutput, datasetRow}) => {
        return {match: modelOutput === datasetRow.expected};
      },
      {name: 'matchScore'}
    );
    ```

    <Note>
      class 기반 `Scorer` 유형은 아직 TypeScript에서 사용할 수 없습니다. `weave.op`으로 래핑한 함수 기반 스코어러를 사용하세요.
    </Note>
  </Tab>
</Tabs>

<div id="4-define-a-model-or-function-to-evaluate">
  ## 4. 평가할 모델 또는 함수 정의하기
</div>

<Tabs>
  <Tab title="Python">
    `Model`을 평가하려면 `Evaluation`을 사용해 해당 모델 인스턴스에서 `evaluate`를 호출하세요. `Models`는 실험해 보고 Weave에 캡처하고 싶은 파라미터가 있을 때 사용합니다.

    ```python lines
    from weave import Model, Evaluation
    import asyncio

    class MyModel(Model):
        prompt: str

        @weave.op()
        def predict(self, question: str):
            # 여기에서 LLM 호출을 추가하고 결과를 반환합니다
            return {'generated_text': 'Hello, ' + self.prompt}

    model = MyModel(prompt='World')

    evaluation = Evaluation(
        dataset=examples, scorers=[match_score1]
    )
    weave.init('intro-example') # weave로 결과 추적 시작
    asyncio.run(evaluation.evaluate(model))
    ```

    이렇게 하면 각 예시에 대해 `predict`를 실행하고, 각 스코어링 함수로 출력값의 점수를 계산합니다.

    ### (선택) 평가할 함수 정의하기

    또는 `@weave.op()`으로 추적되는 맞춤형 함수를 평가할 수도 있습니다.

    ```python lines
    @weave.op
    def function_to_evaluate(question: str):
        # 여기에서 LLM 호출을 추가하고 결과를 반환합니다
        return  {'generated_text': 'some response'}

    asyncio.run(evaluation.evaluate(function_to_evaluate))
    ```
  </Tab>

  <Tab title="TypeScript">
    TypeScript에서는 `weave.op`으로 래핑된 함수를 평가합니다. 이 함수는 데이터셋의 행을 입력으로 받고 모델 출력을 반환합니다.

    ```typescript lines
    import * as weave from 'weave';

    // Weave 초기화
    await weave.init('intro-example');

    // 평가할 함수 정의
    const myModel = weave.op(
      async ({question}) => {
        // 여기에서 LLM 호출을 추가하고 결과를 반환합니다
        return 'Paris';
      },
      {name: 'myModel'}
    );

    // Evaluation 생성
    const evaluation = new weave.Evaluation({
      id: 'my-evaluation',
      dataset: dataset,
      scorers: [matchScore],
    });

    // Evaluation 실행
    const results = await evaluation.evaluate({model: myModel});
    ```

    이렇게 하면 각 예시에 대해 `myModel`을 실행하고, 각 스코어링 함수로 출력값의 점수를 계산합니다.
  </Tab>
</Tabs>

<div id="5-run-the-evaluation">
  ## 5. 평가 실행하기
</div>

평가를 실행하려면 `Evaluation` 객체에서 `.evaluate()`를 호출합니다.

<Tabs>
  <Tab title="Python">
    `evaluation`이라는 `Evaluation` 객체와 평가할 `model`이라는 `Model` 객체가 있다고 가정하면, 아래 코드는 평가 런을 생성합니다.

    ```python lines
    asyncio.run(evaluation.evaluate(model))
    ```

    ### (선택 사항) 여러 시도 실행

    각 예시를 여러 번 실행하려면 `Evaluation` 객체의 `trials` 파라미터를 설정하면 됩니다.

    ```python lines
    evaluation = Evaluation(
        dataset=examples,
        scorers=[match_score],
        trials=3
    )
    ```
  </Tab>

  <Tab title="TypeScript">
    `evaluation`이라는 `Evaluation` 객체와 `myModel`이라는 모델 함수가 있다고 가정하면, 아래 코드는 평가를 실행합니다.

    ```typescript lines
    const results = await evaluation.evaluate({model: myModel});
    ```

    ### (선택 사항) 여러 시도 실행

    각 예시를 여러 번 실행하려면 `evaluate()`를 호출할 때 `nTrials` 파라미터를 설정하면 됩니다.

    ```typescript lines
    const results = await evaluation.evaluate({
      model: myModel,
      nTrials: 3,
    });
    ```
  </Tab>
</Tabs>

이 평가 런은 각 예시를 모델에 세 번씩 전달하며, 각 런은 Weave에서 독립적으로 점수가 매겨지고 표시됩니다.

<div id="full-evaluation-code-example">
  ## 전체 평가 코드 예시
</div>

<Tabs>
  <Tab title="Python">
    다음 코드 예시는 전체 평가 런을 처음부터 끝까지 수행하는 방법을 보여줍니다. `examples`는 `prompt` 값이 주어졌을 때 `MyModel`을 평가하기 위해 `match_score1` 및 `match_score2` 스코어링 함수에서 사용되며, 맞춤형 함수 `function_to_evaluate`를 평가하는 데에도 사용됩니다. `Model`과 함수에 대한 평가는 모두 `asyncio.run(evaluation.evaluate())`로 호출합니다.

    ```python lines
    from weave import Evaluation, Model
    import weave
    import asyncio
    weave.init('intro-example')
    examples = [
        {"question": "What is the capital of France?", "expected": "Paris"},
        {"question": "Who wrote 'To Kill a Mockingbird'?", "expected": "Harper Lee"},
        {"question": "What is the square root of 64?", "expected": "8"},
    ]

    @weave.op()
    def match_score1(expected: str, output: dict) -> dict:
        return {'match': expected == output['generated_text']}

    @weave.op()
    def match_score2(expected: dict, output: dict) -> dict:
        return {'match': expected == output['generated_text']}

    class MyModel(Model):
        prompt: str

        @weave.op()
        def predict(self, question: str):
            # 여기에서 LLM 호출을 추가하고 출력을 반환합니다
            return {'generated_text': 'Hello, ' + question + self.prompt}

    model = MyModel(prompt='World')
    evaluation = Evaluation(dataset=examples, scorers=[match_score1, match_score2])

    asyncio.run(evaluation.evaluate(model))

    @weave.op()
    def function_to_evaluate(question: str):
        # 여기에서 LLM 호출을 추가하고 출력을 반환합니다
        return  {'generated_text': 'some response' + question}

    asyncio.run(evaluation.evaluate(function_to_evaluate("What is the capitol of France?")))
    ```
  </Tab>

  <Tab title="TypeScript">
    다음 코드 예시는 전체 평가 런을 처음부터 끝까지 수행하는 방법을 보여줍니다. 데이터셋 행은 `matchScore` 스코어링 함수에서 `myModel`을 평가하는 데 사용됩니다.

    ```typescript lines
    import * as weave from 'weave';

    // Weave 초기화
    await weave.init('intro-example');

    // 예시를 수집해 데이터셋으로 구성
    const dataset = new weave.Dataset({
      id: 'my-dataset',
      rows: [
        {question: 'What is the capital of France?', expected: 'Paris'},
        {question: 'Who wrote "To Kill a Mockingbird"?', expected: 'Harper Lee'},
        {question: 'What is the square root of 64?', expected: '8'},
      ],
    });

    // 스코어링 함수 정의
    const matchScore = weave.op(
      ({modelOutput, datasetRow}) => {
        return {match: modelOutput === datasetRow.expected};
      },
      {name: 'matchScore'}
    );

    // 평가할 함수 정의
    const myModel = weave.op(
      async ({question}) => {
        // 여기에서 LLM 호출을 추가하고 출력을 반환합니다
        return 'Paris';
      },
      {name: 'myModel'}
    );

    // 평가 생성 및 실행
    const evaluation = new weave.Evaluation({
      id: 'my-evaluation',
      dataset: dataset,
      scorers: [matchScore],
    });

    const results = await evaluation.evaluate({model: myModel});
    console.log('Evaluation results:', results);
    ```

<div id="format-dataset-rows-before-evaluating">
  ### 평가 전에 데이터셋 행 형식 지정하기
</div>

<Tabs>
  <Tab title="Python">
    <Warning>
      `preprocess_model_input` 함수는 모델의 예측 함수에 전달되기 전 입력에만 적용됩니다. 스코어러 함수는 항상 어떤 전처리도 적용되지 않은 원본 데이터셋 예제를 받습니다.
    </Warning>

    `preprocess_model_input` 파라미터를 사용하면 평가 함수에 전달되기 전에 데이터셋 예제를 변환할 수 있습니다. 다음과 같은 경우에 유용합니다.

    * 필드 이름을 모델이 기대하는 입력 이름에 맞게 변경해야 할 때
    * 데이터를 올바른 형식으로 변환해야 할 때
    * 필드를 추가하거나 제거해야 할 때
    * 각 예제에 대해 추가 데이터를 로드해야 할 때

    다음은 `preprocess_model_input`을 사용해 필드 이름을 변경하는 방법을 보여 주는 간단한 예시입니다.

    ```python lines
    import weave
    from weave import Evaluation
    import asyncio

    # 데이터셋에는 "input_text"가 있지만, 모델은 "question"을 기대합니다
    examples = [
        {"input_text": "What is the capital of France?", "expected": "Paris"},
        {"input_text": "Who wrote 'To Kill a Mockingbird'?", "expected": "Harper Lee"},
        {"input_text": "What is the square root of 64?", "expected": "8"},
    ]

    @weave.op()
    def preprocess_example(example):
        # input_text를 question으로 이름 변경
        return {
            "question": example["input_text"]
        }

    @weave.op()
    def match_score(expected: str, output: dict) -> dict:
        return {'match': expected == output['generated_text']}

    @weave.op()
    def function_to_evaluate(question: str):
        return {'generated_text': f'Answer to: {question}'}

    # 전처리를 포함해 Evaluation 생성
    evaluation = Evaluation(
        dataset=examples,
        scorers=[match_score],
        preprocess_model_input=preprocess_example
    )

    # 평가 실행
    weave.init('preprocessing-example')
    asyncio.run(evaluation.evaluate(function_to_evaluate))
    ```

    이 예시에서 데이터셋에는 `input_text` 필드가 있는 예제들이 있지만, 평가 함수는 `question` 파라미터를 기대합니다. `preprocess_example` 함수는 필드 이름을 변경해 각 예제를 변환하고, 이를 통해 평가가 올바르게 동작하도록 합니다.

    전처리 함수는 다음을 수행합니다.

    1. 데이터셋의 원본 예제를 입력으로 받습니다.
    2. 모델이 기대하는 필드를 가진 딕셔너리를 반환합니다.
    3. 예제가 평가 함수에 전달되기 전에 각 예제에 적용됩니다.

    이는 외부 데이터셋을 사용할 때 특히 유용합니다. 외부 데이터셋은 모델이 기대하는 것과 다른 필드 이름이나 구조를 가질 수 있습니다.
  </Tab>

  <Tab title="TypeScript">
    TypeScript에서는 `Evaluation` 객체의 `columnMapping`을 사용해 데이터셋 컬럼 이름을 스코어러가 기대하는 이름에 매핑할 수 있습니다. 데이터셋의 필드 이름이 스코어러 함수가 기대하는 이름과 다를 때 유용합니다.

    다음 예시는 `expectedOutputTimesTwo` 컬럼을 `expected` 컬럼에 매핑합니다.

    ```typescript lines
    const myScorer = weave.op(
      ({modelOutput, datasetRow}) => {
        return modelOutput * 2 === datasetRow.expectedOutputTimesTwo;
      },
      {name: 'myScorer'}
    );

    const evaluation = new weave.Evaluation({
      id: 'my-evaluation',
      dataset: [{expected: 2}],
      scorers: [myScorer],
      columnMapping: {expectedOutputTimesTwo: 'expected'},
    });
    ```

    <Note>
      `preprocess_model_input` 파라미터는 아직 TypeScript에서 사용할 수 없습니다. 데이터셋 필드를 스코어러가 기대하는 필드에 매핑하려면 `columnMapping`을 사용하세요.
    </Note>
  </Tab>
</Tabs>
