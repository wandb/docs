Weave 프로젝트는 traces, evaluations, models, datasets 및 기타 Weave 자산과 같은 관련 자산을 체계적으로 조직합니다. 프로젝트는 팀을 더 체계적으로 운영하고, 협업과 공유를 간소화하며, 시간에 따라 LLM 애플리케이션이 어떻게 발전하는지 추적하는 데 도움을 줍니다.

<div id="project-resources">
  ## 프로젝트 리소스
</div>

프로젝트는 다음 리소스를 저장합니다:

| Resource | Description |
| --- | --- |
| **Traces** | 디버깅 및 분석을 위해 입력, 출력, 지연 시간, 토큰 사용량을 캡처하는 개별 LLM 호출, 함수 실행, 애플리케이션 흐름입니다. [Traces](/ko/weave/guides/tracking/tracing)를 참조하세요.|
| **Evaluations** | 품질, 정확도 및 기타 메트릭을 측정하기 위해 스코어링 함수를 사용하여 모델 성능을 데이터셋과 비교한 테스트 결과입니다. [Evaluations](/ko/weave/guides/core-types/evaluations)를 참조하세요. |
| **Monitors** | 운영 환경에서 시간에 따라 품질 메트릭을 추적하기 위해 LLM-as-a-judge 스코어러로 함수 호출의 일부를 자동으로 스코어링하는 백그라운드 프로세스입니다. [Monitors](/ko/weave/guides/evaluation/monitors)를 참조하세요. |
| **Leaderboards** | 서로 다른 Evaluation 실행 간 모델 성능을 순위로 비교하여 가장 성능이 좋은 설정을 식별하는 데 도움이 되는 기능입니다. [Leaderboards](/ko/weave/guides/core-types/leaderboards)를 참조하세요. |
| **Threads** | 전체 사용자 상호작용을 분석할 수 있도록 관련 호출을 공통 스레드 ID 아래로 묶는 다중 턴 대화 및 세션 기반 워크플로우입니다. [Threads](/ko/weave/guides/tracking/threads)를 참조하세요. |
| **Assets** | **Assets** 탭은 프로젝트 내 버전 관리된 객체를 저장하고 구성합니다: <ul><li>**Models**: 프롬프트, 파라미터, 구성을 캡처하여 재현성과 비교를 위해 변경 사항을 자동으로 추적하는 버전 관리된 모델 정의입니다. [Models](/ko/weave/guides/core-types/models)를 참조하세요.</li><li>**Datasets**: 다운로드, 조회, Evaluation 실행에 사용할 수 있도록 Weave 객체로 저장된 테스트 예제 모음입니다. [Datasets](/ko/weave/guides/core-types/datasets)를 참조하세요.</li><li>**Prompts**: 모델 구성에서 테스트하고 재사용할 수 있는 버전 관리된 시스템 프롬프트입니다. [Prompts](/ko/weave/guides/core-types/prompts)를 참조하세요.</li><li>**Scorers**: 품질, 안전성 또는 사용자 정의 메트릭을 측정하기 위해 모델 입력과 출력을 평가하는 함수입니다. [Scorers](/ko/weave/guides/evaluation/scorers)를 참조하세요.</li><li>**Ops**: 호출을 생성하고 함수 변경 이력을 유지하기 위해 `@weave.op()` 데코레이터가 적용된, 자동으로 버전 관리 및 추적되는 함수입니다. [Ops](/ko/weave/guides/tracking/ops)를 참조하세요.</li><li>**기타 객체 및 메타데이터**: 프로젝트에 저장된 추가 Weave 객체, 커스텀 타입 및 관련 메타데이터입니다.</li></ul> |

<div id="project-paths">
  ## 프로젝트 경로
</div>

[Teams](/ko/platform/hosting/iam/org_team_struct#team)(엔터티라고도 함)는 W&amp;B 조직에서 프로젝트를 소유하고 관리합니다. 팀 이름과 프로젝트 이름이 W&amp;B UI에서 해당 프로젝트 경로를 구성합니다. 이 경로는 코드에서도 사용하여 Weave 리소스를 해당 프로젝트와 연결합니다.

예를 들어 팀 이름이 `my-team`이고 프로젝트 이름이 `my-project`인 경우, 프로젝트 경로는 다음과 같습니다:

```
my-team/my-project
```

코드에서 Weave를 초기화할 때 이 경로를 사용해 앱 데이터를 프로젝트와 연결하세요:

```
weave.init('my-team/my-project')
```

<div id="create-and-find-projects">
  ## 프로젝트 생성 및 찾기
</div>

W&amp;B UI를 사용하거나 코드에서 새로운 프로젝트 경로를 설정하여 Weave 프로젝트를 생성할 수 있습니다.

<Tabs>
  <Tab title="W&B UI">
    W&amp;B UI를 사용해 새 프로젝트를 생성하려면:

    1. [wandb.ai](https://wandb.ai)로 이동합니다.
    2. **New project**를 선택합니다. 그러면 **Create a new project** 모달 대화 상자가 열립니다.
    3. 모달에서 다음 필드를 설정합니다:
       * **Team**: 프로젝트와 연결할 기존 팀.
       * **Name**: 프로젝트 이름.
       * **Project visibility**: 프로젝트를 보고 편집할 수 있는 대상 범위.
       * **Description**: (선택 사항) 프로젝트의 목적을 추적하고 정리하는 데 도움이 되는 짧은 설명.
    4. 필요한 필드를 설정한 후 **Create project**를 선택합니다. 그러면 프로젝트가 생성됩니다.

    방금 생성한 프로젝트를 찾으려면 왼쪽 상단의 **Weights &amp; Biases 로고**를 선택하여 **Global navigation** 메뉴를 엽니다. 이 메뉴에는 최근 프로젝트와 소속 팀이 표시되어 컨텍스트를 빠르게 전환할 수 있습니다. 프로젝트를 선택해 해당 프로젝트의 워크스페이스를 엽니다.

    또는 오른쪽 상단 이름 아래 메뉴에서 **Profile**을 열 수 있습니다. **Projects** 탭에는 사용자의 모든 프로젝트가 나열됩니다.

    새 프로젝트의 워크스페이스를 열면 UI에서 W&amp;B Models와 W&amp;B Weave 둘 다에 대한 퀵스타트를 제공합니다. 프로젝트 툴바에는 각 제품을 위한 도구도 포함되어 있습니다. 코드에서 프로젝트에 Weave 데이터를 연결한 후에는 왼쪽 사이드 내비게이션에 Weave 도구만 표시되고 Weave 대시보드를 사용할 수 있게 됩니다.
  </Tab>

  <Tab title="Weave SDK">
    코드에서 직접 Weave 프로젝트를 생성하려면 `weave.init()`에 인수로 새로운 프로젝트 경로를 전달합니다:

    <CodeGroup>
      ```python
      import weave

      weave.init('my-team/my-llm-app')
      ```

      ```typescript
      import * as weave from 'weave';

      await weave.init('my-team/my-llm-app');
      ```
    </CodeGroup>

    코드를 실행하면 Weave가 W&amp;B 계정에 새 프로젝트를 생성하고 해당 실행에서 생성된 관련 데이터를 프로젝트로 업로드합니다. 또한 Weave는 이 프로젝트를 W&amp;B Models 프로젝트가 아닌 Weave 프로젝트로 설정하며, UI 왼쪽 사이드 내비게이션에는 Weave 도구만 표시됩니다.
  </Tab>
</Tabs>

<div id="view-projects-in-the-ui">
  ## UI에서 프로젝트 보기
</div>

Weave UI에서 프로젝트에 접근하려면:

1. [wandb.ai](https://wandb.ai)로 이동합니다.
2. 왼쪽 상단의 **Weights &amp; Biases 로고**를 클릭해 **Global navigation menu**를 연 다음, **Projects** 사이드바에서 프로젝트를 선택합니다. 그러면 프로젝트 개요 페이지가 열리며, 해당 프로젝트에 이미 트레이스를 기록한 경우 **Traces** 페이지가 열립니다.
3. 개요 페이지에서 다음을 수행할 수 있습니다:
   * 프로젝트에 기록된 트레이스 수와 프로젝트가 사용 중인 스토리지 용량 등 프로젝트 통계를 검토합니다.
   * 사이드바를 사용해 트레이스, 평가 및 기타 리소스로 이동합니다.
   * 최근에 기록된 트레이스, 평가, 모델, 데이터셋으로 이동합니다.

<div id="delete-a-project">
  ## 프로젝트 삭제
</div>

W&amp;B 계정에서 프로젝트를 삭제하려면:

1. [wandb.ai](https://wandb.ai)로 이동합니다.
2. **Projects** 사이드바에서 프로젝트를 선택합니다. 프로젝트 개요 페이지가 열리거나, 해당 프로젝트에 이미 trace를 기록한 경우 **Traces** 페이지가 열립니다.
3. 사이드 내비게이션에서 **Project** (<Icon icon="circle-info" size={16} />) 버튼을 선택하여, 아직 열려 있지 않다면 프로젝트 개요 페이지를 엽니다.
4. 세 점 메뉴(<Icon icon="ellipsis" size={16} />)를 선택하고 목록에서 **Delete Project**를 선택합니다. 그러면 **Delete project** 모달 창이 열립니다.
5. 모달에서 확인 필드에 프로젝트 이름을 입력하고 **Delete**를 클릭합니다. 그러면 해당 프로젝트가 W&amp;B 계정에서 영구적으로 삭제됩니다.
