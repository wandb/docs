<div id="ops">
  ## Ops
</div>

**Op**는 버전 관리되고 추적되는 함수입니다. 함수에 `@weave.op()`(Python) 데코레이터를 적용하거나 `weave.op()`(TypeScript)으로 감싸면 Weave가 해당 함수의 코드, inputs, 출력, 실행 메타데이터를 자동으로 캡처합니다. Ops는 Tracing, Evaluation scorer, 그리고 추적되는 모든 계산의 기본 구성 요소입니다.

<CodeGroup>
  ```python Python
      @weave.op
      async def my_function(){
        ...  }
  ```

  ```typescript Typescript
  function myFunction() {
      ...
  }

  const myFunctionOp = weave.op(myFunction)
  ```
</CodeGroup>

<div id="calls">
  ## Calls
</div>

**Call**은 Op 실행이 로깅된 기록입니다. Op가 실행될 때마다 Weave는 다음 정보를 캡처하는 Call을 생성합니다:

* 입력 인수
* 출력 값
* 실행 시간 및 지연 시간
* 부모-자식 관계(중첩된 Call의 경우)
* 발생한 모든 오류

Call은 Weave UI에서 **트레이스**로 표시되며, 디버깅, 분석, 평가에 필요한 데이터를 제공합니다. 전체 Call 객체의 구조와 속성은 [Call 스키마 레퍼런스](/ko/weave/guides/tracking/call-schema-reference)를 참조하세요.

Call은 [OpenTelemetry](https://opentelemetry.io) 데이터 모델의 span과 유사합니다. Call은 다음과 같은 특성을 가질 수 있습니다:

* 동일한 실행 컨텍스트에 있는 Call들의 컬렉션인 트레이스에 속할 수 있습니다
* 부모 및 자식 Call을 가질 수 있으며, 트리 구조를 형성합니다
