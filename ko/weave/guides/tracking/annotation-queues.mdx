<div id="overview">
  ## Overview
</div>

어노테이션 큐를 사용하면 선택한 트레이스를 도메인 전문가에게 전달하여, 전체 Weave UI를 탐색할 필요 없이 구조화된 검토를 진행할 수 있습니다. 수집할 피드백을 정의하고, 어떤 트레이스가 검토가 필요한지 선택한 뒤, 나중에 완료된 어노테이션을 내보내어 분석이나 데이터셋 생성을 수행할 수 있습니다. 주요 사용 사례는 다음과 같습니다:

* **수동 트레이스 스코어링**: SME(Subject Matter Expert)가 모델 출력의 정확성, 품질, 스타일을 평가하도록 합니다.
* **실패 분석**: hallucination, 거절, 루프와 같은 실패 모드를 어노테이션하여 모델에 어떤 문제가 발생하는지 파악합니다.
* **도메인 전문가 검토**: 의료, 법률, 안전 전문가가 작업 중심 인터페이스로 콘텐츠를 검토할 수 있게 합니다.
* **데이터셋 생성**: 어노테이션된 트레이스를 평가 또는 트레이닝 데이터셋으로 변환합니다.

<div id="end-to-end-workflow">
  ## 엔드 투 엔드 워크플로
</div>

다음 워크플로는 어노테이션 큐를 사용해 검토를 수행하는 전체 과정을 요약합니다.

1. 어노테이션 필드를 정의합니다.
2. 어노테이션 큐를 생성합니다.
3. 검토할 트레이스를 큐에 로드합니다.
4. 도메인 전문가가 검토를 완료하는 동안 진행 상황을 모니터링합니다.
5. 완료된 어노테이션을 필터링하고 내보냅니다.

<div id="define-annotation-fields">
  ## 어노테이션 필드 정의
</div>

어노테이션 큐 생성을 시작하려면 먼저 어노테이션 필드를 정의해야 합니다. 그래야 큐 설정 중에 선택할 수 있습니다. 어노테이션 필드는 어노테이터가 각 트레이스 항목에 대해 제공하는 피드백을 정의합니다. 필드는 큐와 프로젝트 전반에서 재사용할 수 있습니다.

필드 유형에는 다음이 포함됩니다.

* 정확성 또는 수용 가능성과 같은 불리언 판단.
* 품질 또는 신뢰도와 같은 숫자 또는 정수 값.
* 실패 모드 또는 의도와 같은 범주형 레이블.
* 정성적 피드백을 위한 자유 형식 텍스트.

어노테이션 필드를 생성하려면 다음 단계를 따르세요.

1. [wandb.ai](https://wandb.ai)로 이동하여 프로젝트를 선택합니다.
2. Weave project sidebar에서 **Annotate**를 클릭합니다. **Annotate**가 보이지 않으면 **More** 아래 메뉴에 중첩되어 있을 수 있습니다.
3. 탭 바에서 **Fields** 탭을 클릭합니다.
4. Fields table 툴바에서 **New Field**를 클릭합니다.
5. **Create annotation field** modal dialog에서 다음을 설정합니다.
   * **Type**: Boolean, Integer, Number, String 또는 categorical 옵션.
   * **Name**: 어노테이터에게 표시될 필드 이름.
   * **Description** (선택): 어노테이터에게 표시될, 이 필드를 평가하기 위한 세부 정보.
6. **Create annotation field**를 클릭하여 필드를 저장합니다.

<Frame>
  ![트레이스 항목을 검토할 때 어노테이터가 작성하는 스키마를 정의하는 데 사용되는 이름, 설명 및 필드 유형 입력이 포함된 어노테이션 필드 생성 대화 상자.](/weave/guides/tracking/imgs/AnnotationCreateField.png)
</Frame>

어노테이션 일관성을 보장하기 위해 필드는 생성 후 편집할 수 없습니다.

<div id="create-an-annotation-queue">
  ## 어노테이션 큐 생성
</div>

어노테이션 큐는 다음으로 구성됩니다.

* 어노테이션 필드 집합
* 어노테이터에게 작업 지침을 제공하는 가이드라인
* 검토를 기다리는 트레이스 항목 모음

어노테이션 큐를 생성하려면 다음 단계를 따르세요.

1. Weave 프로젝트 사이드바에서 **Annotate**를 클릭합니다.
2. 탭 바에서 **Queues** 탭을 클릭합니다.
3. Queues 테이블 툴바에서 **Create Queue**를 클릭합니다.
4. **Create Annotation Queue** 모달 대화 상자에서 다음을 설정합니다.
   * **Queue name**: 어노테이터가 작업을 완료할 때 선택하는 큐 이름입니다.
   * **Guidelines** (**선택**): 어노테이터를 위한 추가 지침입니다.
5. **Next**를 클릭합니다.
6. **Manage fields**를 클릭한 다음, 이번 검토 작업에 포함할 어노테이션 필드를 선택합니다. 해당 프로젝트의 기존 어노테이션 필드는 모두 선택할 수 있습니다.
7. 큐에 포함할 모든 필드를 선택했으면 **Create Queue**를 클릭하여 큐를 저장합니다.

프로젝트의 모든 어노테이션 큐는 Annotation Queues 페이지에 나열됩니다.

<Frame>
  ![프로젝트의 큐 이름, 설명, 그리고 어노테이션 워크플로를 추적하기 위한 검토 진행 상황이 포함된 큐 테이블을 표시하는 Annotation Queues 페이지.](/weave/guides/tracking/imgs/AnnotationQueues.png)
</Frame>

어노테이션 큐를 생성하면 평가에 사용할 필드와 가이드라인이 정의되지만, 어떤 데이터를 평가할지 파악하려면 여전히 트레이스를 큐에 추가해야 합니다.

<div id="add-traces-to-a-queue">
  ### 큐에 트레이스 추가
</div>

**Traces** 페이지에서 직접 어노테이션 큐에 트레이스를 추가하세요.

어노테이션 큐에 트레이스를 추가하려면 다음과 같이 하세요.

1. Weave 프로젝트 사이드바에서 **Traces**를 클릭합니다.
2. Traces 테이블 툴바에서 필요에 따라 트레이스를 필터링합니다(예: hallucination 점수, failure mode 또는 특정 ops 기준).
3. 테이블에서 어노테이션할 트레이스를 선택합니다.
4. 테이블의 액션 바에서 **Add to queue**를 클릭하여 선택한 행을 어노테이션 큐에 추가합니다.

<Frame>
  ![여러 행이 선택된 Traces 테이블과 Add to queue 및 Add to dataset처럼 선택한 행에 적용되는 액션을 표시하는 일괄 작업 바](/weave/guides/tracking/imgs/AnnotationAddTraces.png)
</Frame>

5. **Add to annotation queue** 모달 대화상자에서 다음을 설정합니다:
   * **Queue Name**: 목록에서 이 트레이스를 추가할 기존 큐의 이름을 선택합니다.
   * **Select trace data to display**: 어노테이터가 결과를 평가할 때 표시할 Weave 트레이스 데이터 요소를 선택합니다.
     * **Inputs**: 어노테이션 중에 표시할 트레이스 입력 필드를 선택합니다.
     * **Outputs**: 어노테이션 중에 표시할 트레이스 출력 필드를 선택합니다.
6. **Add ***[Count]*** traces to annotation queue**를 클릭하여 이 트레이스를 어노테이션 큐 검토의 일부로 부여합니다.

트레이스를 추가할 때는 어떤 트레이스 입력과 어떤 출력 또는 모델 Response를 검토할지 제어할 수 있습니다. 이렇게 하면 어노테이터에게 정확한 판단을 내리는 데 필요한 컨텍스트만 제시할 수 있습니다.

<div id="monitor-review-progress">
  ## 검토 진행 상황 모니터링
</div>

어노테이션 큐를 생성하고 트레이스를 추가한 다음, 어노테이터가 검토를 시작할 수 있도록 큐 이름을 공유하세요. 검토 프로세스에 대한 자세한 내용은 [어노테이션 큐에서 항목 검토하기](/ko/weave/guides/tracking/annotation-review)를 참조하세요.

어노테이션 큐에 대한 직접 링크를 어노테이터와 공유하려면:

1. Weave 프로젝트 사이드바에서 **Annotate**를 클릭합니다.
2. 탭 바에서 **Queues** 탭을 클릭합니다.
3. Annotation Queues 테이블에서 큐 이름을 클릭하여 큐 항목을 엽니다.
4. Queue 헤더 바에서 링크 버튼을 클릭해 이 큐에 대한 직접 링크를 복사합니다. 브라우저 주소 표시줄에서 URL을 복사할 수도 있습니다.

![Annotation Queue 헤더 바에서 어노테이션 큐에 대한 링크 복사](/weave/guides/tracking/imgs/AnnotationLinkToQueue.png)

Annotation Queues 테이블에서 **State** 열은 검토 진행 상황을 나타냅니다:

* **Not started**: 큐에 항목은 있지만 제출된 어노테이션이 없습니다.
* **In progress**: 최소 하나의 항목이 검토되었습니다.
* **Completed**: 모든 항목이 검토되었습니다.

Annotation Queues 테이블에서 **Calls with responses** 열은 (총 **Calls** 수 대비) 최소 하나의 검토가 제출된 항목의 비율을 나타냅니다.

<div id="filter-and-export-annotations">
  ## 어노테이션 필터링 및 내보내기
</div>

Weave는 완료된 어노테이션을 트레이스의 구조화된 메타데이터로 저장합니다.

다음을 수행할 수 있습니다:

* 큐 할당 상태와 어노테이션 완료 여부로 트레이스를 필터링합니다.
* 필터링된 뷰를 저장해 다시 사용할 수 있습니다.
* 평가 또는 트레이닝 워크플로를 위해 어노테이션된 트레이스를 데이터셋으로 내보냅니다.

이를 통해 전문가의 피드백을 모델 평가 및 반복 과정과 직접 연결할 수 있습니다.

<div id="filter-annotated-traces">
  ### 어노테이션된 트레이스 필터링
</div>

**Traces** 페이지의 필터 컨트롤을 사용해 어노테이션이 있는 트레이스만 표시할 수 있습니다.

어노테이션이 있는 트레이스만 보려면 다음과 같이 하세요.

1. Weave 프로젝트 사이드바에서 **Traces**를 클릭합니다.
2. Traces 테이블 툴바에서 **Filter**를 클릭합니다.
3. 필터 행에 다음 세 값을 추가합니다:
   * **Column**에 &quot;Queue&quot;를 입력한 다음 **Enter**를 누릅니다.
   * 두 번째 목록에서 **Text: &quot;is&quot;**를 선택합니다.
   * **Select a queue**에서 어노테이션 큐 이름을 선택합니다.
4. &#39;completed&#39; 상태의 큐 항목만 추가로 필터링하려면 **+ Add Filter**를 클릭합니다:
   * **Column**에 &quot;feedback&quot;을 입력합니다. 그러면 Annotations가 표시된 대화 상자가 열리고, 어노테이션 필드 이름도 함께 표시됩니다. 큐에서 필요한 Field를 선택합니다.
   * 두 번째 목록에서 **Other: &quot;is not empty&quot;**를 선택합니다.

<Frame>
  ![특정 어노테이션 큐에 속한 트레이스만 선택하도록 필터를 구성한 Traces 테이블.](/weave/guides/tracking/imgs/AnnotationTracesFilter.png)
</Frame>

4. 필터 행은 자동으로 적용됩니다. 필터 입력 창을 닫으려면 페이지의 다른 곳을 클릭합니다.
5. (선택 사항) 빠르게 액세스할 수 있도록 뷰로 저장합니다. Traces 테이블 헤더에서 **Save View**를 클릭합니다.

<div id="export-annotated-traces-to-datasets">
  ### 어노테이션된 트레이스를 데이터셋으로 내보내기
</div>

데이터 활용 방식에 따라 UI 또는 프로그래밍 방식으로 어노테이션된 트레이스를 내보낼 수 있습니다.

<div id="add-annotated-traces-to-a-dataset">
  #### 어노테이션된 트레이스를 데이터셋에 추가하기
</div>

어노테이션된 트레이스를 선택하고 **Add to Dataset**을 클릭하여 평가(eval) 또는 트레이닝 데이터에 전문가 레이블을 포함합니다.

어노테이션된 트레이스를 데이터셋에 추가하려면:

1. Weave 프로젝트 사이드바에서 **Traces**를 클릭합니다.
2. Traces 표에서 내보낼 트레이스를 선택합니다.
3. 표 툴바에서 **Add to dataset**을 클릭합니다. 화면의 안내에 따라 추가를 완료합니다.

데이터셋 사용 방법에 대한 자세한 내용은 [데이터셋 수집 및 추적](/ko/weave/guides/core-types/datasets)을 참조하세요.

<div id="access-annotations-programmatically">
  ## 프로그래밍 방식으로 어노테이션에 액세스
</div>

어노테이션을 프로그래밍 방식으로 통합하려면 프로젝트 이름과 큐 ID를 알아야 합니다:

* **프로젝트**: W&amp;B 프로젝트 이름입니다(`project` 또는 `team/project` 형식일 수 있음). W&amp;B 팀(예: &quot;team/project&quot;)을 지정하지 않으면 기본 팀이 사용됩니다.
* **큐 ID**: 어노테이션 큐의 고유 식별자입니다.

큐 ID를 찾으려면:

1. **Annotation Queues** table에서 큐 이름을 선택하여 해당 항목을 여세요.
2. 페이지 URL 끝부분에서 ID를 복사하세요.

**예시**

```
https://wandb.ai/.../annotation-queues/019c0f63-7acb-7497-8f87-08873368fcd4
```

이 예제에서 큐 ID는 다음과 같습니다:

019c0f63-7acb-7497-8f87-08873368fcd4

다음 코드를 사용하여 큐의 트레이스(Call)를 순회할 수 있습니다.

```python lines
import weave
from weave.trace_server.trace_server_interface import AnnotationQueueItemsQueryReq

# 프로젝트 및 큐 식별자를 본인의 값으로 업데이트하세요.
PROJECT = "your-team-name/your-project-name"
QUEUE_ID = "019c0f63-7acb-7497-8f87-08873368fcd4"

# Weave를 초기화합니다.
client = weave.init(PROJECT)

# 큐에 있는 Call에서 Call ID를 조회합니다.
calls = client.server.annotation_queue_items_query(
    AnnotationQueueItemsQueryReq(
        project_id=PROJECT,
        queue_id=QUEUE_ID,
    )
)

# Call을 순회하며 피드백을 조회합니다.
for i, item in enumerate(calls.items):
    call = client.get_call(call_id=item.call_id, include_feedback=True)
    feedback = call.feedback or {}

    # Call에 대한 총 피드백 항목 수를 계산합니다.
    total_feedback_items = len(feedback)

    print(f"\nItem {i} — call_id: {item.call_id} — total feedback items: {total_feedback_items}")

    if not feedback:
        print("항목에 대한 피드백 없음")
        continue

    # 첫 번째 어노테이션 값을 조회합니다.
    # 필드 어노테이션은 반복 가능한 순서로 추가되지 않으므로 첫 번째 필드는 달라질 수 있습니다.
    field_name = next(iter(feedback))
    field_value = feedback[0]

    print(f"  {field_name}: {field_value}")
```
