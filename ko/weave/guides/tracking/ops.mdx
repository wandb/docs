Weave Op는 모든 Call을 자동으로 로깅하는 버전 관리 함수입니다.

<Tabs>
  <Tab title="Python">
    Op를 만들려면 Python 함수에 `weave.op()` 데코레이터를 추가하면 됩니다.

    ```python lines
    import weave

    @weave.op()
    def track_me(v):
        return v + 5

    weave.init('intro-example')
    track_me(15)
    ```

    Op를 Call하면 코드가 마지막 Call 이후 변경된 경우 새 Op 버전이 생성되고, 함수의 입력과 출력이 로깅됩니다.

    `@weave.op()`으로 데코레이션한 함수는 Call하기 전에 `weave.init('your-project-name')`를 실행하지 않으면 (코드 버전 관리와 추적 없이) 평소처럼 동작합니다.

    Op는 Weave toolbelt를 사용해 [서빙](/ko/weave/guides/tools/serve)하거나 [배포](/ko/weave/guides/tools/deploy)할 수 있습니다.
  </Tab>

  <Tab title="TypeScript">
    Op를 만들려면 TypeScript 함수를 `weave.op`으로 감싸면 됩니다.

    ```typescript lines
    import * as weave from 'weave'

    function trackMe(v: number) {
        return v + 5
    }

    const trackMeOp = weave.op(trackMe)
    trackMeOp(15)

    // 인라인으로도 작성할 수 있으며, 이 방법이 더 편리할 수 있습니다.
    const trackMeInline = weave.op((v: number) => v + 5)
    trackMeInline(15)
    ```
  </Tab>
</Tabs>

<div id="apply-kinds-and-colors">
  ## 종류와 색상 적용하기
</div>

Weave UI에서 Ops를 더 효과적으로 정리하려면 코드의 `@weave.op` 데코레이터에 `kind`와 `color` 인수를 추가하여 맞춤형 종류와 색상을 적용할 수 있습니다. 예를 들어, 다음 코드는 상위 함수에 `LLM` `kind`와 `blue` `color`를, 중첩 함수에 `tool` `kind`와 `red` `color`를 적용합니다:

<Tabs>
  <Tab title="Python">
    ```python lines
    import weave

    weave.init("<your-team-name>/<your-project-name>")

    @weave.op(kind="LLM", color="blue")
    def llm_func():
        @weave.op(kind="tool", color="red")
        def tool_func():
            return "tool result"

        tool_result = tool_func()
        
        return f"llm result with {tool_result}"

    llm_func()
    ```
  </Tab>

  <Tab title="TypeScript">
    ```plaintext
    This feature is not available in TypeScript yet.
    ```
  </Tab>
</Tabs>

이렇게 하면 Weave UI의 Ops에 종류와 색상이 다음과 같이 적용됩니다:

<Frame>
  ![상위 호출에 LLM 종류와 blue 색상이, 중첩 호출에 tool 종류와 red 색상이 적용된 Weave UI.](/images/weave/weave_colors_kinds.png)
</Frame>

사용 가능한 `kind` 값은 다음과 같습니다:

* `agent`
* `llm`
* `tool`
* `search`

사용 가능한 `color` 값은 다음과 같습니다:

* `red`
* `orange`
* `yellow`
* `green`
* `blue`
* `purple`

<div id="customize-logged-inputs-and-outputs">
  ## 로깅되는 입력값과 출력값 사용자 정의하기
</div>

<Tabs>
  <Tab title="Python">
    원본 함수를 수정하지 않고(예: 민감한 데이터를 숨기기 위해) Weave에 로깅되는 데이터를 변경하려면, Op 데코레이터에 `postprocess_inputs`와 `postprocess_output`을 인수로 전달하면 됩니다.

    `postprocess_inputs`는 키가 인수 이름이고 값이 인수 값인 dict를 입력으로 받아, 변환된 입력값이 들어 있는 dict를 반환합니다.

    `postprocess_output`은 함수가 일반적으로 반환하는 어떤 값이든 입력으로 받아, 변환된 출력값을 반환합니다.

    ```python lines
    from dataclasses import dataclass
    from typing import Any
    import weave

    @dataclass
    class CustomObject:
        x: int
        secret_password: str

    def postprocess_inputs(inputs: dict[str, Any]) -> dict[str, Any]:
        return {k:v for k,v in inputs.items() if k != "hide_me"}

    def postprocess_output(output: CustomObject) -> CustomObject:
        return CustomObject(x=output.x, secret_password="REDACTED")

    @weave.op(
        postprocess_inputs=postprocess_inputs,
        postprocess_output=postprocess_output,
    )
    def func(a: int, hide_me: str) -> CustomObject:
        return CustomObject(x=a, secret_password=hide_me)

    weave.init('hide-data-example') # 🐝
    func(a=1, hide_me="password123")
    ```
  </Tab>

  <Tab title="TypeScript">
    ```plaintext
    이 기능은 아직 TypeScript에서는 사용할 수 없습니다.
    ```
  </Tab>
</Tabs>

<div id="control-sampling-rate">
  ## 샘플링 비율 제어
</div>

<Tabs>
  <Tab title="Python">
    `@weave.op` 데코레이터의 `tracing_sample_rate` 파라미터를 설정하면 Op의 Call이 트레이스되는 빈도를 제어할 수 있습니다. 이는 호출 빈도가 높은 Op에서 일부 Call만 트레이스하면 되는 경우에 유용합니다.

    Weave는 샘플링 비율을 루트 Call에만 적용한다는 점에 유의하세요. 어떤 Op에 샘플링 비율이 설정되어 있더라도, 먼저 다른 Op가 이를 호출하면 해당 샘플링 비율은 무시됩니다.

    ```python lines
    @weave.op(tracing_sample_rate=0.1)  # Call의 약 10%만 트레이스
    def high_frequency_op(x: int) -> int:
        return x + 1

    @weave.op(tracing_sample_rate=1.0)  # 항상 트레이스 (기본값)
    def always_traced_op(x: int) -> int:
        return x + 1
    ```

    Op의 Call이 샘플링되지 않으면:

    * 함수는 정상적으로 실행됩니다
    * Trace Data가 Weave로 전송되지 않습니다
    * 해당 Call에 대해서는 자식 Op도 트레이스되지 않습니다

    샘플링 비율은 0.0 이상 1.0 이하의 값이어야 합니다.
  </Tab>

  <Tab title="TypeScript">
    ```plaintext
    이 기능은 아직 TypeScript에서 사용 가능하지 않습니다.
    ```
  </Tab>
</Tabs>

<div id="control-call-link-output">
  ## Call 링크 출력 제어
</div>

로깅 중에 Call 링크 출력 인쇄를 억제하려면 `WEAVE_PRINT_CALL_LINK` 환경 변수를 `false`로 설정하세요. 이는 출력의 상세성을 줄이고 로그의 혼잡을 줄이고 싶을 때 유용합니다.

```bash lines
export WEAVE_PRINT_CALL_LINK=false
```

<div id="deleting-an-op">
  ## Op 삭제하기
</div>

<Tabs>
  <Tab title="Python">
    Op 버전을 삭제하려면 Op ref에서 `.delete()` 메서드를 호출하세요.

    ```python lines
    weave.init('intro-example')
    my_op_ref = weave.ref('track_me:v1')
    my_op_ref.delete()
    ```

    삭제된 Op에 액세스하면 오류가 반환됩니다.
  </Tab>

  <Tab title="TypeScript">
    ```plaintext
    이 기능은 아직 TypeScript에서 사용할 수 없습니다.
    ```
  </Tab>
</Tabs>
