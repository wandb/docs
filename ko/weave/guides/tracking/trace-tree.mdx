title: "Weave Trace 뷰 탐색하기"
description: "Weave의 Trace 뷰를 사용해 LLM 및 에이전트 기반 앱의 복잡한 실행 경로를 이해하세요."
---

Weave Trace 뷰는 LLM 및 에이전트 기반 앱의 복잡한 실행 경로를 이해하는 데 도움이 되도록 설계되었습니다. 수십 개의 중첩 호출이 있는 에이전트 기반 앱을 디버깅하든, 단일 모델 예측의 흐름을 추적하든, Trace 뷰는 발생한 일을 명확하게 시각화해 줍니다. 또한 애플리케이션 흐름을 다양한 방식으로 살펴보고 이해할 수 있는 여러 대안적 뷰를 제공합니다.

이 가이드에서는 Trace 스택을 탐색하고, 코드에서 호출된 함수를 필터링하고 검색하며, 시각적 표현을 전환하는 방법 등을 설명합니다.

<div id="get-started">
  ## 시작하기
</div>

Trace 보기로 들어가려면:

1. [https://wandb.ai](https://wandb.ai)로 이동한 다음 프로젝트를 선택합니다.
2. 사이드바 메뉴에서 **Traces**를 선택해 프로젝트에 저장된 모든 트레이스를 확인합니다.
3. 트레이스를 하나 선택해 Trace Details 보기를 엽니다. Trace Details 보기에는 트레이스 실행을 계층 구조로 분해해 보여 주는 추가 패널들이 표시됩니다.

<div id="traces-page-overview">
  ## Traces 페이지 개요
</div>

Traces 페이지는 세 개의 주요 패널로 구성됩니다:

* **왼쪽 패널**: 해당 프로젝트의 모든 trace 실행을 정렬·페이지네이션해서 보여주는 목록입니다.
  * 이 traces 테이블에는 tokens, cost, latency 등의 추가 데이터가 포함됩니다.
* **가운데 패널**: 선택한 trace에 대한 인터랙티브 trace 뷰입니다. trace 트리는 trace 내에서 추적된 모든 메서드의 계층 구조를 보여줍니다.
  * trace 트리는 trace 동안 호출된, `@weave.op()` 데코레이터가 적용된 함수인 [ops](/ko/weave/guides/tracking/ops#automatically-track-function-calls-using-ops)를 표시합니다.
* **오른쪽 패널**: 선택한 trace 내에서 선택한 op에 대한 상세 정보입니다.

<div id="filter-within-a-trace">
  ### 트레이스 내에서 필터링
</div>

* **이름 정규식 필터**: 트레이스 트리 상단의 텍스트 필드를 사용해 `tool`, `openai.response.create`와 같이 op의 이름이나 타입을 기준으로 필터링합니다.
* **메트릭**: 사용 가능한 경우 비용, 토큰 수, 지연 시간 등의 메트릭 데이터를 표시할지 여부를 설정합니다.

<div id="navigate-a-trace">
  ### 트레이스 탐색
</div>

트레이스 트리는 트레이스 내에서 추적된 모든 메서드의 계층 구조를 보여줍니다. 트리에서 위/아래로 이동하려면 `Cmd`(macOS) 또는 `Alt`(Windows/Linux) + 위쪽 화살표(↑) / 아래쪽 화살표(↓) 키를 사용하세요.

트레이스 트리 아래에는 트레이스 내 다양한 상태를 빠르게 이동할 수 있는 여러 스크러버가 있습니다. 슬라이더를 사용해 트레이스를 효율적으로 탐색할 수 있습니다.

패널을 확장해서 사용 가능한 모든 스크러버를 확인하세요:

* **Timeline**: 트레이스 내 이벤트의 시간순 정렬입니다.
* **Peers**: 동일한 타입을 공유하는 op입니다. 예를 들어 `predict`라는 함수의 세부 정보를 살펴보고 있다면, 이 스크러버를 사용해 트레이스 내에서 `predict`의 다음 실행으로 바로 이동할 수 있습니다.
* **Siblings**: 동일한 부모를 가진 op입니다. 이 스크러버를 사용해 부모 함수 호출 아래에 중첩된 op들을 순회합니다.
* **Stack**: 호출 스택을 위/아래로 탐색합니다.
* **Path**: (코드 컴포지션 뷰에서만 사용 가능) 선택된 호출과 동일한 코드 경로를 가진 모든 호출을 순회합니다.

<div id="alternative-trace-tree-views">
  ### 다른 trace 트리 보기
</div>

패널 상단에서 필요에 따라 trace 트리를 여러 가지 시각화 방식으로 전환할 수 있습니다. 디버깅 목적에 맞게 보기를 전환하세요. 호출 로직을 파악하려면 **code composition view**를, 시간에 따른 성능을 분석하려면 **flame graph view**를, 구조를 이해하려면 **graph view**를 사용하세요.

<div id="traces-default">
  #### Traces (기본값)
</div>

트레이스 트리의 기본 보기는 스택 계층 구조, 연산(op)당 비용(가능한 경우), 실행 시간, 상태 표시기를 표시합니다.

<div id="code-composition-view">
  #### 코드 구성 보기
</div>

코드 구성 보기에서는 박스가 op와 그 안에 중첩된 호출을 나타냅니다. 이 보기는 함수 호출 흐름을 시각화하는 데 유용합니다. 이 보기에서 박스를 선택해 해당 op를 자세히 살펴보고 호출 경로를 필터링할 수 있습니다.

<Frame>
  ![트레이스의 코드 보기를 보여주는 트레이스 뷰](/weave/guides/tracking/imgs/trace-tree-code-view.png)
</Frame>

<div id="flame-graph">
  #### 플레임 그래프
</div>

플레임 그래프 보기는 실행의 깊이와 지속 시간을 타임라인 기반으로 시각화합니다. 시간 경과에 따른 성능 특성을 분석할 때 유용합니다. 프레임을 선택해 하위 트레이스를 분리해 볼 수 있습니다.

<Frame>
  ![Trace view showing a flame graph of ops within a trace](/weave/guides/tracking/imgs/trace-tree-flame-view.png)
</Frame>

<div id="graph-view">
  #### 그래프 뷰
</div>

그래프 뷰는 op 간의 계층 구조를 보여 줍니다. 이는 부모/자식 관계를 파악하는 데 유용합니다.

  ![Trace view showing a functional graph view of ops and relationships within a trace](/weave/guides/tracking/imgs/trace-tree-graph-view.png)

<div id="view-details-for-a-trace-op">
  ### 트레이스 op의 세부 정보 보기
</div>

트레이스 트리에서 op를 선택하면 해당 op의 세부 정보가 오른쪽 패널에 표시됩니다. 이 세부 정보는 다음 탭으로 구성됩니다:

* **Call**: op 실행에 사용된 입력과 생성된 출력입니다.
* **Code**: 호출이 수행될 때 사용된 코드입니다.
* **Feedback**: op에 대해 사용 가능한 [feedback](/ko/weave/guides/tracking/feedback)입니다. Weave 내에서 직접 또는 API를 통해 feedback을 제공할 수 있습니다.
* **Scores**: op에 대해 사용 가능한 [scores](/ko/weave/guides/evaluation/scorers)입니다. Call은 Evaluation을 실행하여 점수가 매겨집니다.
* **Summary**: op에 대한 일반 정보입니다.
* **Use:** 호출을 프로그래밍 방식으로 조회하고 반응, 메모 또는 feedback을 추가하는 데 사용할 수 있는 코드 스니펫입니다.
