<div id="local-models">
  # 로컬 모델
</div>

많은 개발자가 LLama-3, Mixtral, Gemma, Phi 등과 같은 오픈 소스 모델을 로컬에서 다운로드하여 실행합니다. 이러한 모델을 로컬에서 실행하는 방법은 꽤 많으며, OpenAI SDK 호환을 지원하면 Weave는 그중 일부를 기본적으로 지원합니다.

<div id="wrap-local-model-functions-with-weaveop">
  ## 로컬 모델 함수를 `@weave.op()`로 감싸기
</div>

`weave.init('<your-project-name>')`로 Weave를 초기화한 다음 LLM 호출을 `weave.op()`로 감싸면 어떤 LLM이든 직접 쉽게 Weave와 통합할 수 있습니다. 자세한 내용은 [tracing](/ko/weave/guides/tracking/tracing) 가이드를 참조하세요.

<div id="updating-your-openai-sdk-code-to-use-local-models">
  ## 로컬 모델을 사용하도록 OpenAI SDK 코드를 업데이트하기
</div>

OpenAI SDK 호환을 지원하는 모든 서비스 프레임워크에서는 몇 가지 사소한 변경이 필요합니다.

가장 먼저, 그리고 가장 중요한 변경 사항은 `openai.OpenAI()` 초기화 중 `base_url` 변경입니다.

```python lines
client = openai.OpenAI(
    base_url="http://localhost:1234",
)
```

로컬 모델의 경우 `api_key`는 어떤 문자열이어도 되지만, 재정의해야 합니다. 그렇지 않으면 OpenAI가 환경 변수에서 해당 값을 사용하려고 하다가 오류를 표시합니다.

<div id="openai-sdk-supported-local-model-runners">
  ## OpenAI SDK를 지원하는 로컬 모델 러너
</div>

다음은 컴퓨터에서 Hugging Face의 모델을 다운로드하고 실행할 수 있으며, OpenAI SDK 호환을 지원하는 앱 목록입니다.

1. Nomic [GPT4All](https://www.nomic.ai/gpt4all) - Settings의 Local Server를 통해 지원([FAQ](https://docs.gpt4all.io/gpt4all_help/faq.html))
2. [LMStudio](https://lmstudio.ai/) - Local Server OpenAI SDK 지원 [문서](https://lmstudio.ai/docs/developer/core/server)
3. [Ollama](https://ollama.com/) - OpenAI SDK용 [OpenAI 호환](https://ollama.readthedocs.io/en/openai/)
4. [llama-cpp-python](https://llama-cpp-python.readthedocs.io/en/latest/server/) Python 패키지를 통한 llama.cpp
5. [llamafile](https://github.com/Mozilla-Ocho/llamafile#other-example-llamafiles) - Llamafile run에서 `http://localhost:8080/v1`은 OpenAI SDK를 자동으로 지원합니다
