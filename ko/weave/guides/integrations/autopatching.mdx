---
title: "LLM Call 자동 추적 제어"
description: "W&B Weave가 OpenAI, Anthropic 및 기타 LLM 라이브러리에 대한 Call을 자동으로 기록하는 방식을 제어하세요"
---

애플리케이션에서 LLM 공급자 라이브러리(예: OpenAI, Anthropic, Cohere, Mistral)를 사용하는 경우, 자동 패칭은 Weave가 모든 LLM Call의 추적을 자동으로 처리하는 방식입니다. `weave.init()`을 호출하면 Weave가 지원되는 LLM 클라이언트 라이브러리를 자동으로 가로채(패치)합니다. 애플리케이션 코드는 변경 없이 유지됩니다. 공급자 SDK를 평소와 같이 사용하면 각 Request가 Weave Call로 기록됩니다. 최소한의 설정으로 전체 추적 기능을 사용할 수 있습니다.

이 페이지에서는 자동 추적을 끄거나, 특정 공급자로 제한하거나, 입력 및 출력을 후처리(예: PII 삭제)하는 등 해당 동작을 변경하는 시기와 방법을 설명합니다.

<div id="default-behavior">
  ## 기본 동작
</div>

기본적으로 Weave는 `openai` 및 `anthropic`과 같은 일반적인 LLM 라이브러리에 대한 Call을 자동으로 패치하고 추적합니다. 프로그램 시작 시 `weave.init(...)`을 호출하고 해당 라이브러리를 정상적으로 사용하세요. 해당 라이브러리의 Call이 프로젝트의 트레이스에 표시됩니다.

<div id="configure-autopatching">
  ## 자동 패칭 설정
</div>

<Tabs>
  <Tab title="Python">
    <Warning>
      `autopatch_settings` 인수는 사용 중단되었습니다. 암묵적 패칭을 비활성화하려면 `implicitly_patch_integrations=False`를 사용하거나, 인테그레이션별 설정을 구성하려면 `patch_openai(settings={...})`와 같은 특정 패치 함수를 호출하십시오.
    </Warning>

    Weave는 기본적으로 지원되는 모든 인테그레이션에 대해 **자동 암묵적 패칭**을 제공합니다.

    **암묵적 패칭(자동):** 라이브러리는 임포트 시점에 관계없이 자동으로 패칭됩니다.

    ```python lines
    # 옵션 1: weave.init() 이전에 임포트
    import openai
    import weave
    weave.init('your-team-name/your-project-name')  # OpenAI가 자동으로 패칭됩니다!

    # 옵션 2: weave.init() 이후에 임포트
    import weave
    weave.init('your-team-name/your-project-name')
    import anthropic  # 임포트 훅을 통해 자동으로 패칭됩니다!
    ```

    **암묵적 패칭 비활성화:** 명시적으로 제어하려는 경우 자동 패칭을 비활성화할 수 있습니다.

    ```python lines
    import weave

    # 옵션 1: settings 파라미터를 통해 설정
    weave.init('your-team-name/your-project-name', settings={'implicitly_patch_integrations': False})

    # 옵션 2: 환경 변수를 통해 설정
    # 스크립트 실행 전에 WEAVE_IMPLICITLY_PATCH_INTEGRATIONS=false로 설정

    # 암묵적 패칭이 비활성화된 경우, 인테그레이션을 명시적으로 패칭해야 합니다
    import openai
    weave.patch_openai()  # OpenAI 추적을 위해 필요합니다
    ```

    **명시적 패칭(수동):** 세밀한 제어를 위해 인테그레이션을 명시적으로 패칭할 수 있습니다.

    ```python lines
    import weave
    weave.init('your-team-name/your-project-name')
    weave.integrations.patch_openai()  # OpenAI 추적 활성화
    weave.integrations.patch_anthropic()  # Anthropic 추적 활성화
    ```

    ### 입력 및 출력 후처리

    패치 함수에 설정을 전달하여 입력 및 출력이 기록되는 방식을 사용자 지정할 수 있습니다(예: PII 또는 시크릿 삭제).

    ```python lines
    import weave.integrations

    def redact_inputs(inputs: dict) -> dict:
        if "email" in inputs:
            inputs["email"] = "[REDACTED]"
        return inputs

    weave.init(...)
    weave.integrations.patch_openai(
        settings={
            "op_settings": {"postprocess_inputs": redact_inputs}
        }
    )
    ```

    민감한 데이터 처리에 대한 자세한 내용은 [PII 데이터와 함께 Weave 사용하는 방법](/ko/weave/cookbooks/pii)을 참조하십시오.
  </Tab>

  <Tab title="TypeScript">
    TypeScript SDK는 OpenAI와 Anthropic에 대해서만 자동 패칭을 지원합니다. OpenAI는 Weave를 임포트할 때 자동으로 패칭되며 추가 설정이 필요하지 않습니다.

    또한 TypeScript SDK는 다음을 지원하지 않습니다.

    * 자동 패칭 설정 또는 비활성화
    * 입력/출력 후처리

    자동 패칭이 작동하지 않는 엣지 케이스(ESM, Next.js와 같은 번들러)에는 명시적 래핑을 사용하십시오.

    ```typescript
    import OpenAI from 'openai'
    import * as weave from 'weave'
    import { wrapOpenAI } from 'weave'

    const client = wrapOpenAI(new OpenAI())
    await weave.init('your-team-name/your-project-name')
    ```

    ESM 설정 및 문제 해결에 대한 자세한 내용은 [TypeScript SDK 인테그레이션 가이드](/ko/weave/guides/integrations/js)를 참조하십시오.
  </Tab>
</Tabs>
