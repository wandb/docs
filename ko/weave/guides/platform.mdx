Weave는 다음 배포 옵션으로 제공됩니다:

* **[W&amp;B Multi-tenant Cloud](https://docs.wandb.ai/platform/hosting/hosting-options/multi_tenant_cloud):** 북미 리전의 W&amp;B Google Cloud Platform(Google Cloud) 계정에 배포되는 멀티 테넌트 완전 관리형 플랫폼입니다.
* **[W&amp;B Dedicated Cloud](https://docs.wandb.ai/platform/hosting/hosting-options/dedicated-cloud):** AWS, Google Cloud, Azure에서 정식 제공됩니다.
* **[Self-Managed instances](/ko/weave/guides/platform/weave-self-managed):** Weave를 독립적으로 호스팅하려는 팀을 위해, 배포 옵션을 평가할 수 있도록 담당 W&amp;B 팀에서 가이드를 제공합니다.

<div id="identity-and-access-management">
  ## ID 및 액세스 관리
</div>

[W&amp;B 조직](/ko/platform/hosting/iam/org_team_struct#organization)에서 안전한 인증과 효과적인 권한 부여를 위해 ID 및 액세스 관리 기능을 활용하세요. 다음 기능은 배포 옵션과 [가격 플랜](https://wandb.ai/site/pricing/)에 따라 Weave 사용자에게 제공됩니다.

* **Single Sign-On(SSO)으로 인증:** Google, GitHub 같은 공개 ID 공급자와 Okta, Azure Active Directory 등의 엔터프라이즈 ID 공급자를 지원하며, [OIDC를 통해](/ko/platform/hosting/iam/sso) 사용할 수 있습니다.
* **[팀 기반 논리적 분리](/ko/platform/app/settings-page/teams):** 각 팀은 조직 내 사업부, 부서 또는 프로젝트 팀에 대응할 수 있습니다.
* **W&amp;B Projects를 사용해 이니셔티브 구성:** 팀 내에서 이니셔티브를 구성하고, 민감한 협업을 위한 `restricted` 범위를 포함해 필요한 [visibility scope](/ko/platform/hosting/iam/access-management/restricted-projects)를 설정하세요.
* **역할 기반 액세스 제어:** 사용자가 필요한 데이터에만 액세스할 수 있도록 [팀](/ko/platform/hosting/iam/access-management/restricted-projects#assign-project-level-role-to-a-user) 또는 [프로젝트](/ko/platform/hosting/iam/access-management/restricted-projects#assign-project-level-role-to-a-user) 수준에서 액세스를 구성하세요.
* **범위가 지정된 서비스 계정:** 조직 또는 팀 범위로 지정된 서비스 계정을 사용해 Gen AI 워크플로를 자동화하세요.
* **[SCIM API 및 Python SDK](/ko/platform/hosting/iam/automate_iam):** SCIM API와 Python SDK를 사용해 사용자와 팀을 효율적으로 관리하세요.

<div id="data-security">
  ## 데이터 보안
</div>

* **Multi-tenant Cloud:** 모든 Weave 사용자의 데이터는 클라우드 기본 제공 암호화로 암호화된 공유 Clickhouse Cloud 클러스터에 저장됩니다. 공유 컴퓨트 서비스가 데이터를 처리하며, 고객의 W&amp;B 조직, 팀, 프로젝트로 구성된 보안 컨텍스트를 통해 격리를 보장합니다.

* **Dedicated Cloud:** 데이터는 선택한 클라우드와 리전에 있는 전용 Clickhouse Cloud 클러스터에 저장됩니다. 전용 컴퓨트 환경이 데이터를 처리하며, 다음과 같은 추가 보호 기능이 제공됩니다:
  * **[IP allowlisting](/ko/platform/hosting/data-security/ip-allowlisting):** 특정 IP 주소에서만 인스턴스에 접근할 수 있도록 허용합니다. 이는 선택 기능입니다.
  * **[Private connectivity](/ko/platform/hosting/data-security/private-connectivity):** 클라우드 제공업체의 프라이빗 네트워크를 통해 데이터를 안전하게 전송합니다. 이는 선택 기능입니다.
  * **[Data encryption](/ko/platform/hosting/data-security/data-encryption):** W&amp;B는 고유한 W&amp;B 관리형 암호화 키를 사용해 저장 데이터를 암호화합니다.
  * **Clickhouse cluster security:** W&amp;B는 클라우드 제공업체의 프라이빗 네트워크를 통해 Dedicated Cloud 인스턴스용 전용 Clickhouse Cloud 클러스터에 연결합니다. 또한 Clickhouse의 파일 수준 암호화를 활용하면서, 고유한 W&amp;B 관리형 암호화 키로 클러스터를 암호화합니다.

<Warning>
  [W&amp;B Platform secure storage connector 또는 BYOB](/ko/platform/hosting/data-security/secure-storage-connector)는 Weave에서 사용할 수 없습니다.
</Warning>

<div id="maintenance">
  ## 유지 관리
</div>

Multi-tenant Cloud 또는 Dedicated Cloud에서 Weave를 사용하는 경우 W&amp;B 플랫폼이 완전 관리형으로 제공되므로, W&amp;B 플랫폼을 프로비저닝, 운영, 유지 관리하는 데 드는 오버헤드와 비용을 줄일 수 있습니다.

<div id="compliance">
  ## 규정 준수
</div>

<Tip>
  SOC 2 리포트 및 기타 보안/규정 준수 문서를 요청하려면 [W&amp;B Security Portal](https://security.wandb.ai/)을 참고하거나, 자세한 내용은 담당 W&amp;B 팀에 문의하세요.
</Tip>

Multi-tenant Cloud와 Dedicated Cloud의 보안 통제는 내부 및 외부 감사를 정기적으로 받습니다. 두 플랫폼 모두 SOC 2 Type II를 준수합니다. 또한 Dedicated Cloud는 생성형 AI 애플리케이션을 구축하는 과정에서 PHI 데이터를 관리하는 조직을 위한 HIPAA 규정을 준수합니다.
