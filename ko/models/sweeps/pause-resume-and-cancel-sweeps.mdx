title: 스위프 관리
---

[W&amp;B CLI](/ko/models/ref/cli/wandb-sweep)를 사용하여 스위프를 일시 중지, 재개 및 취소할 수 있습니다. CLI의 `sweep` 명령은 `--pause`, `--resume`와 같은 플래그를 사용하여 스위프가 새 W&amp;B 실행을 생성할 수 있는지를 제어하며, 기존 실행에는 서로 다른 영향을 줍니다:

* `--pause`: 스위프를 일시 중지하면 스위프를 다시 재개할 때까지 에이전트는 새 실행을 생성하지 않습니다. 기존 실행은 계속 정상적으로 수행됩니다.
* `--resume`: 스위프를 재개하면 에이전트는 검색 전략에 따라 새 실행을 계속 생성합니다.
* `--stop`: 스위프를 중지하면 에이전트는 새 실행 생성을 중단합니다. 기존 실행은 완료될 때까지 계속 실행됩니다.
* `--cancel`: 스위프를 취소하면 에이전트는 현재 실행 중인 모든 실행을 즉시 종료하고 새 실행 생성을 중단합니다.

다음을 참고하여 스위프를 일시 중지, 재개 및 취소하십시오. 각 경우 스위프를 초기화할 때 생성된 스위프 ID를 제공해야 합니다.

<div id="pause-a-sweep">
  ### 스윕 일시 중지
</div>

스윕을 일시 중지하면 새로운 실행이 일시적으로 더 이상 생성되지 않습니다. 이미 진행 중인 실행은 완료될 때까지 계속됩니다. [`wandb sweep --pause`](/ko/models/ref/cli/wandb-sweep) 명령을 사용하여 스윕을 일시 중지합니다. 일시 중지하려는 스윕 ID를 지정하세요.

```bash
wandb sweep --pause entity/project/sweep_ID
```

<div id="resume-a-sweep">
  ### 일시 중지된 스윕 재개하기
</div>

[`wandb sweep --resume`](/ko/models/ref/cli/wandb-sweep) 명령으로 일시 중지된 스윕을 재개합니다. 스윕이 검색 전략에 따라 다시 새로운 실행을 생성하기 시작합니다. 재개할 스윕 ID를 지정합니다:

```bash
wandb sweep --resume entity/project/sweep_ID
```

<div id="stop-a-sweep">
  ### 스윕 중지
</div>

스윕을 종료해 새 실행 생성은 중단하되, 현재 진행 중인 실행은 정상적으로 완료되도록 둡니다. [`wandb sweep --stop`](/ko/models/ref/cli/wandb-sweep) 명령을 사용하세요:

```bash
wandb sweep --stop entity/project/sweep_ID
```

<Warning>
  프로젝트를 삭제해도 W&amp;B에서 실행 중인 [sweeps](/ko/models/sweeps) 또는 에이전트는 종료되지 않습니다.
</Warning>

<div id="cancel-a-sweep">
  ### 스윕 취소하기
</div>

스윕을 취소하면 현재 실행 중인 모든 실행을 즉시 중단하고, 새로운 실행이 생성되지 않도록 막습니다. 이것은 기존 실행을 강제로 종료하는 유일한 스윕 명령입니다. 스윕을 취소하려면 [`wandb sweep --cancel`](/ko/models/ref/cli/wandb-sweep) 명령을 사용하십시오. 이때 취소할 스윕 ID를 함께 지정해야 합니다.

```bash
wandb sweep --cancel entity/project/sweep_ID
```

CLI 명령 옵션의 전체 목록은 [wandb sweep](/ko/models/ref/cli/wandb-sweep) CLI 참조 가이드를 확인하세요.

<Warning>
  프로젝트를 삭제해도 W&amp;B는 활성 [스윕](/ko/models/sweeps) 또는 에이전트를 종료하지 않습니다.
</Warning>

<div id="understanding-sweep-and-run-statuses">
  ## 스윕과 실행 상태 이해하기
</div>

스윕은 여러 실행을 오케스트레이션하여 하이퍼파라미터 조합을 탐색합니다. 스윕 상태와 실행 상태가 어떻게 상호 작용하는지 이해하는 것은 하이퍼파라미터 최적화를 효과적으로 관리하는 데 중요합니다.

<div id="key-differences">
  ### 주요 차이점
</div>

* **Sweep 상태**는 새 실행을 생성할지 여부를 제어합니다 (Running, Paused, Stopped, Cancelled, Finished, Failed, Crashed)
* **실행 상태**는 개별 실행의 상태를 나타냅니다 (Pending, Running, Finished, Failed, Crashed, Killed)

<div id="stop-an-individual-run">
  ### 개별 실행 중지하기
</div>

스윕에서 [실행을 중지](/ko/models/runs/stop-runs)하면 스윕 에이전트가 자동으로 스윕의 다음 실행을 시작합니다. 이렇게 하면 스윕 전체 진행을 중단하지 않고도 성능이 좋지 않은 구성은 건너뛸 수 있습니다.

<div id="best-practices">
  ### 모범 사례
</div>

* 실행 중인 실험을 잃지 않고 탐색을 일시적으로 중단하려면 취소 대신 `--pause`를 사용하세요
* 반복적으로 발생하는 실패를 식별하기 위해 개별 실행 상태를 모니터링하세요
* 만족스러운 하이퍼파라미터를 찾았을 때는 정상적으로 종료하기 위해 `--stop`을 사용하세요
* 실행이 과도한 리소스를 소비하거나 오류를 발생시키는 긴급 상황에서는 `--cancel`을 사용하세요
