description: CLI를 사용해 W&B 스윕을 일시 중지, 재개 및 취소합니다.
title: 스윕 관리
---

[W&amp;B CLI](/ko/models/ref/cli/wandb-sweep)를 사용해 스윕을 일시 중지, 재개 및 취소하세요. CLI의 `sweep` 명령은 `--pause`, `--resume`와 같은 플래그를 사용해 새 W&amp;B Runs 생성 여부를 제어하며, 기존 Runs에는 각각 다른 영향을 줍니다:

* `--pause`: 스윕을 일시 중지하면 스윕을 재개할 때까지 에이전트가 새 Runs를 생성하지 않습니다. 기존 Runs은 정상적으로 계속 실행됩니다.
* `--resume`: 스윕을 재개하면 에이전트가 검색 전략에 따라 새 Runs 생성을 다시 시작합니다.
* `--stop`: 스윕을 중지하면 에이전트가 새 Runs 생성을 중단합니다. 기존 Runs은 완료될 때까지 계속 실행됩니다.
* `--cancel`: 스윕을 취소하면 에이전트가 현재 실행 중인 모든 Runs을 즉시 종료하고 새 Runs 생성을 중단합니다.

다음 지침에 따라 스윕을 일시 중지, 재개 또는 취소하세요. 각 경우 모두 스윕을 초기화할 때 생성된 스윕 ID를 제공해야 합니다.

<div id="pause-a-sweep">
  ### 스윕 일시 중지
</div>

스윕을 일시 중지하면 새로운 Runs 생성이 일시적으로 중단됩니다. 이미 실행 중인 Runs은 완료될 때까지 계속 실행됩니다. [`wandb sweep --pause`](/ko/models/ref/cli/wandb-sweep) 명령을 사용해 스윕을 일시 중지하세요. 일시 중지할 스윕 ID를 지정하세요.

```bash
wandb sweep --pause entity/project/sweep_ID
```

<div id="resume-a-sweep">
  ### 스윕 재개
</div>

일시 중지된 스윕은 [`wandb sweep --resume`](/ko/models/ref/cli/wandb-sweep) 명령어를 사용하여 재개하세요. 스윕은 검색 전략에 따라 다시 새로운 Runs를 생성하기 시작합니다. 재개하려는 스윕 ID를 제공하세요:

```bash
wandb sweep --resume entity/project/sweep_ID
```

<div id="stop-a-sweep">
  ### 스윕 중지
</div>

스윕을 종료하면 현재 실행 중인 Runs는 정상적으로 마무리되도록 두면서 새 Runs 생성은 중지할 수 있습니다. [`wandb sweep --stop`](/ko/models/ref/cli/wandb-sweep) 명령어를 사용하세요:

```bash
wandb sweep --stop entity/project/sweep_ID
```

<Warning>
  W&amp;B에서는 프로젝트를 삭제해도 활성 [스윕](/ko/models/sweeps) 또는 에이전트를 종료하지 않습니다.
</Warning>

<div id="cancel-a-sweep">
  ### 스윕 취소
</div>

스윕을 취소하면 현재 실행 중인 모든 Runs를 즉시 종료하고 새 Runs 생성을 중지합니다. 기존 Runs를 강제로 종료하는 유일한 스윕 명령어입니다. Runs는 즉시 종료되므로, 실행 중인 프로세스는 사용자 정의 시그널 핸들러를 실행할 기회가 없습니다. 스윕을 취소하려면 [`wandb sweep --cancel`](/ko/models/ref/cli/wandb-sweep) 명령어를 사용하세요. 취소하려는 스윕 ID를 제공하세요. 시그널과 스윕 Runs에 대한 자세한 내용은 [Signal handling and sweep runs](/ko/models/sweeps/signal-handling-sweep-runs)을 참조하세요.

```bash
wandb sweep --cancel entity/project/sweep_ID
```

CLI 명령 옵션의 전체 목록은 [wandb sweep](/ko/models/ref/cli/wandb-sweep) CLI 레퍼런스 가이드를 참조하세요.

<Warning>
  프로젝트를 삭제해도 W&amp;B는 활성 [스윕](/ko/models/sweeps) 또는 에이전트를 종료하지 않습니다.
</Warning>

<div id="understanding-sweep-and-run-statuses">
  ## 스윕 및 Run 상태 이해하기
</div>

스윕은 하이퍼파라미터 조합을 탐색하기 위해 여러 Runs를 오케스트레이션합니다. 하이퍼파라미터 최적화를 효과적으로 관리하려면 스윕 상태와 Run 상태가 어떻게 상호작용하는지 이해하는 것이 중요합니다.

<div id="key-differences">
  ### 키 차이
</div>

* **스윕 상태**는 새 Runs가 생성되는지 여부를 제어합니다 (Running, Paused, Stopped, Cancelled, Finished, Failed, Crashed)
* **Run 상태**는 개별 Runs의 실행 상태를 반영합니다 (Pending, Running, Finished, Failed, Crashed, Killed)

<div id="stop-an-individual-run">
  ### 개별 run 중지하기
</div>

스윕에서 [run을 중지](/ko/models/runs/stop-runs)하면 스윕 에이전트는 자동으로 스윕의 다음 run을 시작합니다. 이렇게 하면 스윕의 전반적인 진행을 방해하지 않고 성능이 좋지 않은 설정을 건너뛸 수 있습니다.

<div id="best-practices">
  ### 모범 사례
</div>

* 실행 중인 실험을 잃지 않고 탐색을 일시적으로 중단하려면 cancel 대신 `--pause`를 사용하세요
* 개별 Run 상태를 모니터링하여 반복적으로 발생하는 실패를 파악하세요
* 만족스러운 하이퍼파라미터를 찾았을 때는 정상적으로 종료하기 위해 `--stop`을 사용하세요
* Runs이 과도한 리소스를 사용하거나 오류를 발생시키는 등의 비상 상황에서는 `--cancel`을 사용하세요
