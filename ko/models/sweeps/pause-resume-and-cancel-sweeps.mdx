title: 스윕 관리
---

이 페이지에서는 Command Line에서 활성 스윕의 라이프사이클을 제어하는 방법을 설명합니다. 불필요한 컴퓨팅 리소스 낭비를 막고, 진행 중인 실험을 보존하거나, 상태를 잃지 않고 탐색을 일시적으로 중단할 수 있습니다.

[W&amp;B CLI](/ko/models/ref/cli/wandb-sweep)를 사용해 스윕을 일시 중지, 재개, 중지 및 취소하세요. 각 명령어는 스윕이 새 Runs를 생성할지 여부와 이미 실행 중인 Runs가 어떻게 처리되는지에 영향을 줍니다. 각 경우에 스윕을 초기화할 때 W&amp;B가 생성한 sweep ID를 지정하세요.

<div id="pause-a-sweep">
  ## 스윕 일시 중지
</div>

스윕를 일시 중지하려면 [`wandb sweep --pause`](/ko/models/ref/cli/wandb-sweep) 명령어를 사용하세요. 그러면 새 run 생성이 일시적으로 중단됩니다. 이미 실행 중인 run은 완료될 때까지 계속 실행됩니다. 스윕 ID를 지정합니다.

```bash
wandb sweep --pause entity/project/sweep_ID
```

<div id="resume-a-sweep">
  ## 스윕 재개
</div>

일시 중지된 스윕은 [`wandb sweep --resume`](/ko/models/ref/cli/wandb-sweep) 명령어로 재개할 수 있으며, 스윕은 검색 전략에 따라 새 run을 다시 생성하기 시작합니다. 스윕 ID를 지정합니다.

```bash
wandb sweep --resume entity/project/sweep_ID
```

<div id="stop-a-sweep">
  ## 스윕 중지
</div>

스윕을 중지하려면 [`wandb sweep --stop`](/ko/models/ref/cli/wandb-sweep) 명령어를 사용하세요. 중지된 스윕은 새 run을 생성하지 않습니다. 이미 실행 중인 Runs는 완료될 때까지 계속 실행됩니다. 스윕 ID를 지정하세요.

```bash
wandb sweep --stop entity/project/sweep_ID
```

<div id="cancel-a-sweep">
  ## 스윕 취소
</div>

스윕을 취소하려면 [`wandb sweep --cancel`](/ko/models/ref/cli/wandb-sweep) 명령어를 사용하세요. 그러면 활성 상태인 모든 Runs가 강제로 종료되고 새 Runs가 생성되지 않습니다. Runs는 즉시 종료되므로, 실행 중인 프로세스는 사용자 정의 시그널 핸들러를 실행할 기회가 없습니다. sweep ID를 지정하세요.

```bash
wandb sweep --cancel entity/project/sweep_ID
```

시그널 및 스윕 run에 대한 자세한 내용은 [Signal handling and sweep runs](/ko/models/sweeps/signal-handling-sweep-runs)을 참조하세요. CLI 명령어 옵션의 전체 목록은 [wandb sweep](/ko/models/ref/cli/wandb-sweep) CLI 레퍼런스 가이드를 참조하세요.

<Warning>
  프로젝트를 삭제해도 활성 [sweeps](/ko/models/sweeps)나 해당 에이전트는 중지되지 않습니다. 중지하려면 `wandb sweep --cancel` 또는 `wandb sweep --stop`을 사용해 스윕을 취소하거나 중지한 다음, 각 에이전트 프로세스를 중지하세요.
</Warning>

<div id="sweep-and-run-statuses">
  ## Sweep와 run 상태
</div>

스윕는 하이퍼파라미터 조합을 탐색하기 위해 여러 run을 실행하고 관리합니다. 다음 섹션에서는 스윕 상태와 run 상태가 어떻게 상호작용하는지 설명합니다. 여기에는 두 상태의 차이점, 개별 run을 중지하면 어떤 일이 발생하는지, 그리고 어떤 라이프사이클 명령어를 선택해야 하는지가 포함되며, 이를 통해 하이퍼파라미터 최적화를 효과적으로 관리할 수 있습니다.

<div id="key-differences">
  ### 주요 차이점
</div>

* **Sweep status**는 에이전트가 새 Runs를 생성할지 여부를 제어합니다. Sweep status는 `Running`, `Paused`, `Stopped`, `Cancelled`, `Finished`, `Failed`, 또는 `Crashed`일 수 있습니다.
* **Run status**는 개별 Runs의 실행 상태를 나타냅니다. Run status는 `Pending`, `Running`, `Finished`, `Failed`, `Crashed`, 또는 `Killed`일 수 있습니다.

<div id="stop-an-individual-run">
  ### 개별 run 중지
</div>

스윕에서 [run을 중지](/ko/models/runs/stop-runs)하면 스윕 에이전트가 스윕의 다음 run을 자동으로 시작합니다. 이 메커니즘을 사용하여 스윕의 전체 진행을 중단하지 않고도 성능이 좋지 않은 설정을 건너뛸 수 있습니다.

<div id="recommendations">
  ### 권장 사항
</div>

다음 권장 사항은 상황에 맞는 적절한 라이프사이클 명령어를 선택하여 유용한 작업 손실을 방지하고, 원치 않는 컴퓨팅 리소스를 계속 붙잡아 두는 일을 피하는 데 도움이 됩니다.

* 실행 중인 실험를 잃지 않고 탐색을 일시 중지하려면 cancel 대신 `--pause`를 사용하세요.
* 개별 run 상태를 모니터링하여 반복적으로 발생하는 실패를 파악하세요.
* 만족스러운 하이퍼파라미터를 찾았다면 정상 종료를 위해 `--stop`을 사용하세요.
* run이 과도한 리소스를 소비하거나 오류를 발생시키는 긴급 상황에서만 `--cancel`을 사용하세요.
