---
title: Sweep 구성 옵션
---

Sweep 구성은 중첩된 키-값 쌍으로 이루어져 있습니다. Sweep 구성의 최상위 키를 사용하여 탐색할 파라미터([`parameters`](./sweep-config-keys#parameters) 키), 파라미터 공간을 탐색하는 방법론([`method`](./sweep-config-keys#method) 키) 등과 같이 Sweep 탐색의 특성을 정의합니다.

다음 표에는 Sweep 구성에서 사용할 수 있는 최상위 키와 간단한 설명이 나와 있습니다. 각 키에 대한 자세한 내용은 이후 섹션을 참고하세요.

| Top-level keys                        | Description                           |
| ------------------------------------- | ------------------------------------- |
| `program`                             | (필수) 실행할 학습 스크립트                      |
| `entity`                              | 이 Sweep에 대한 엔터티                       |
| `project`                             | 이 Sweep에 대한 프로젝트                      |
| `description`                         | Sweep에 대한 텍스트 설명                      |
| `name`                                | W&amp;B UI에 표시되는 Sweep 이름             |
| [`method`](#method)                   | (필수) 탐색 전략                            |
| [`metric`](#metric)                   | 최적화할 메트릭 (일부 탐색 전략 및 조기 중지 기준에서만 사용됨) |
| [`parameters`](#parameters)           | (필수) 탐색할 파라미터 범위                      |
| [`early_terminate`](#early_terminate) | 조기 중지 기준                              |
| [`command`](#command)                 | 학습 스크립트를 호출하고 인자를 전달하기 위한 명령어 구조      |
| `run_cap`                             | 이 Sweep에서 허용되는 최대 실행 수                |

Sweep 구성의 구조에 대한 더 자세한 내용은 [Sweep configuration](./sweep-config-keys)을 참고하세요.

<div id="metric">
  ## `metric`
</div>

`metric` 최상위 sweep 구성 키를 사용해서 최적화할 메트릭의 이름, 목표, 최적화 대상 메트릭을 지정합니다.

|Key | Description |
| -------- | --------------------------------------------------------- |
| `name`   | 최적화할 메트릭의 이름.                           |
| `goal`   | `minimize` 또는 `maximize` 중 하나 (기본값은 `minimize`).  |
| `target` | 최적화하려는 메트릭의 목표 값. 실행이 지정한 목표 값에 도달하면 sweep은 새로운 실행을 생성하지 않습니다. 실행을 수행 중인 활성 에이전트는(해당 실행이 목표에 도달한 경우에도) 그 실행이 완료될 때까지 기다린 후에야 새로운 실행 생성을 중단합니다. |

<div id="parameters">
  ## `parameters`
</div>

YAML 파일 또는 Python 스크립트에서 `parameters` 를 최상위 키로 지정하세요. `parameters` 키 안에는 최적화하려는 하이퍼파라미터의 이름을 지정합니다. 일반적인 하이퍼파라미터로는 learning rate, batch size, epochs, optimizers 등이 있습니다. 스윕 구성에서 정의하는 각 하이퍼파라미터에 대해 하나 이상의 탐색 제약 조건을 지정해야 합니다.

다음 표는 지원되는 하이퍼파라미터 탐색 제약 조건을 보여줍니다. 사용하는 하이퍼파라미터와 유즈 케이스에 따라 아래 탐색 제약 조건 중 하나를 사용하여 스윕 에이전트가 어디(분포의 경우) 또는 무엇을(`value`, `values` 등) 탐색하거나 사용할지 지정하세요.

| Search constraint | Description   |
| --------------- | ------------------------------------------------------------------------------ |
| `values`        | 이 하이퍼파라미터에 대해 유효한 모든 값을 지정합니다. `grid` 와 함께 사용할 수 있습니다.    |
| `value`         | 이 하이퍼파라미터에 대해 유효한 단일 값을 지정합니다. `grid` 와 함께 사용할 수 있습니다.  |
| `distribution`  | 확률 [distribution](#distribution-options-for-random-and-bayesian-search)을 지정합니다. 기본값에 대한 정보는 이 표 다음에 나오는 노트를 참조하세요. |
| `probabilities` | `random` 을 사용할 때 `values` 의 각 요소가 선택될 확률을 지정합니다.  |
| `min`, `max`    | (`int` 또는 `float`) 최솟값과 최댓값입니다. `int` 인 경우 `int_uniform` 분포의 하이퍼파라미터에 사용합니다. `float` 인 경우 `uniform` 분포의 하이퍼파라미터에 사용합니다. |
| `mu`            | (`float`) `normal` 또는 `lognormal` 분포 하이퍼파라미터의 평균 파라미터입니다. |
| `sigma`         | (`float`) `normal` 또는 `lognormal` 분포 하이퍼파라미터의 표준 편차 파라미터입니다. |
| `q`             | (`float`) 양자화된 하이퍼파라미터에 대한 양자화 스텝 크기입니다.     |
| `parameters`    | 루트 레벨 파라미터 내부에 다른 파라미터를 중첩합니다.    |

<Note>
  [distribution](#distribution-options-for-random-and-bayesian-search)이 지정되지 않은 경우, W&amp;B는 다음 조건에 따라 다음 분포를 설정합니다:

  * `values` 를 지정하면 `categorical`
  * `max` 와 `min` 을 정수로 지정하면 `int_uniform`
  * `max` 와 `min` 을 float으로 지정하면 `uniform`
  * `value` 에 단일 값을 지정하면 `constant`
</Note>

<div id="method">
  ## `method`
</div>

`method` 키를 사용해 하이퍼파라미터 탐색 전략을 지정합니다. 선택할 수 있는 하이퍼파라미터 탐색 전략은 grid, random, Bayesian search의 세 가지입니다.

<div id="grid-search">
  #### 그리드 검색
</div>

하이퍼파라미터 값의 모든 조합을 순회합니다. 그리드 검색은 각 반복에서 사용할 하이퍼파라미터 값 집합을 선택할 때 이전 반복에서 얻은 정보를 활용하지 않습니다. 그리드 검색은 계산 비용이 많이 들 수 있습니다.

그리드 검색은 연속적인 탐색 공간을 대상으로 하면 종료되지 않고 계속 실행됩니다.

<div id="random-search">
  #### Random search
</div>

분포를 기반으로 각 반복에서 사전 정보 없이 무작위로 하이퍼파라미터 값 집합을 선택합니다. Random search는 명령줄, Python 스크립트 내, 또는 [W&amp;B App](/ko/models/sweeps/sweeps-ui/)에서 프로세스를 중지하지 않는 한 계속 실행됩니다.

Random (`method: random`) search를 선택하는 경우, metric 키와 함께 탐색할 분포 공간을 지정합니다.

<div id="bayesian-search">
  #### 베이즈 검색
</div>

[랜덤](#random-search) 및 [그리드](#grid-search) 검색과 달리, 베이즈 모델은 정보에 기반해 결정을 내립니다. 베이즈 최적화는 확률론적 모델을 사용하여, 목적 함수를 직접 평가하기 전에 대리 함수에서 다양한 값을 시험하는 반복적인 과정을 통해 어떤 값을 사용할지 결정합니다. 베이즈 검색은 연속형 매개변수의 개수가 적을 때는 잘 동작하지만, 매개변수 수가 많아질수록 확장성이 떨어집니다. 베이즈 검색에 대한 자세한 내용은 [Bayesian Optimization Primer 논문](https://web.archive.org/web/20240209053347/https://static.sigopt.com/b/20a144d208ef255d3b981ce419667ec25d8412e2/static/pdf/SigOpt_Bayesian_Optimization_Primer.pdf)을 참고하세요.

{/* 베이지안 최적화에는 여러 가지 기법이 있습니다. W&B는 가우시안 프로세스를 사용해 하이퍼파라미터와 모델 지표 간의 관계를 모델링합니다. 자세한 내용은 관련 논문을 참고하세요. [LINK] */}

베이지안 검색은 명령줄, Python 스크립트 내, 또는 [W&amp;B 앱](/ko/models/sweeps/sweeps-ui/)에서 프로세스를 중지하지 않는 한 계속 실행됩니다.

<div id="distribution-options-for-random-and-bayesian-search">
  ### 랜덤 및 베이즈 검색을 위한 분포 옵션
</div>

`parameter` 키 안에 하이퍼파라미터 이름을 중첩해서 지정합니다. 그다음 `distribution` 키를 지정하고 값에 사용할 분포를 설정합니다.

아래 표는 W&amp;B가 지원하는 분포를 나열합니다.

| `distribution` 키의 값  | 설명            |
| ------------------------ | ------------------------------------ |
| `constant`               | 상수 분포. 사용할 상수 값(`value`)을 지정해야 합니다.                    |
| `categorical`            | 범주형 분포. 이 하이퍼파라미터에 대한 모든 유효한 값(`values`)을 지정해야 합니다. |
| `int_uniform`            | 정수에 대한 이산 균등 분포. `max`와 `min`을 정수로 지정해야 합니다.     |
| `uniform`                | 연속 균등 분포. `max`와 `min`을 부동소수점 값으로 지정해야 합니다.      |
| `q_uniform`              | 양자화된 균등 분포. X가 균등 분포일 때 `round(X / q) * q`를 반환합니다. `q`의 기본값은 `1`입니다.|
| `log_uniform`            | 로그-균등 분포. 자연로그가 `min`과 `max` 사이에서 균등 분포가 되도록 `exp(min)`과 `exp(max)` 사이의 값 `X`를 반환합니다.   |
| `log_uniform_values`     | 로그-균등 분포. `log(min)`과 `log(max)` 사이에서 균등 분포가 되도록 `log(X)`가 `log(min)`과 `log(max)` 사이에서 균등 분포를 따르는 `min`과 `max` 사이의 값 `X`를 반환합니다.     |
| `q_log_uniform`          | 양자화된 로그-균등 분포. `X`가 `log_uniform`일 때 `round(X / q) * q`를 반환합니다. `q`의 기본값은 `1`입니다. |
| `q_log_uniform_values`   | 양자화된 로그-균등 분포. `X`가 `log_uniform_values`일 때 `round(X / q) * q`를 반환합니다. `q`의 기본값은 `1`입니다.  |
| `inv_log_uniform`        | 역 로그-균등 분포. `log(1/X)`가 `min`과 `max` 사이에서 균등 분포가 되도록 하는 `X`를 반환합니다. |
| `inv_log_uniform_values` | 역 로그-균등 분포. `log(1/X)`가 `log(1/max)`와 `log(1/min)` 사이에서 균등 분포가 되도록 하는 `X`를 반환합니다.    |
| `normal`                 | 정규 분포. 반환 값은 평균 `mu`(기본값 `0`), 표준편차 `sigma`(기본값 `1`)를 갖는 정규 분포를 따릅니다.|
| `q_normal`               | 양자화된 정규 분포. `X`가 `normal`일 때 `round(X / q) * q`를 반환합니다. `q`의 기본값은 `1`입니다.  |
| `log_normal`             | 로그 정규 분포. 자연로그 `log(X)`가 평균 `mu`(기본값 `0`), 표준편차 `sigma`(기본값 `1`)를 갖는 정규 분포를 따르도록 하는 값 `X`를 반환합니다. |
| `q_log_normal`  | 양자화된 로그 정규 분포. `X`가 `log_normal`일 때 `round(X / q) * q`를 반환합니다. `q`의 기본값은 `1`입니다. |

<div id="early_terminate">
  ## `early_terminate`
</div>

성능이 낮은 실행을 중지하려면 조기 종료(`early_terminate`)를 사용하세요. 조기 종료가 발생하면 W&amp;B는 새로운 하이퍼파라미터 값 집합으로 새로운 실행을 생성하기 전에 현재 실행을 중지합니다.

<Note>
  `early_terminate`를 사용하는 경우 중단 알고리즘을 반드시 지정해야 합니다. 스윕 구성에서 `early_terminate` 안에 `type` 키를 중첩해서 설정하세요.
</Note>

<div id="stopping-algorithm">
  ### 중지 알고리즘
</div>

<Note>
  W&amp;B는 현재 [Hyperband](https://arxiv.org/abs/1603.06560) 중지 알고리즘을 지원합니다.
</Note>

[Hyperband](https://arxiv.org/abs/1603.06560) 하이퍼파라미터 최적화는 프로그램을 중지해야 할지, 아니면 하나 이상의 사전에 설정된 반복 횟수 지점(이를 *브래킷*이라고 부름)에서 계속 진행해야 할지를 평가합니다.

W&amp;B 실행이 브래킷에 도달하면, 스윕은 해당 실행의 메트릭을 이전에 보고된 모든 메트릭 값과 비교합니다. 목표가 최소화일 때 실행의 메트릭 값이 너무 높거나, 목표가 최대화일 때 실행의 메트릭 값이 너무 낮으면 스윕은 해당 실행을 종료합니다.

브래킷은 기록된 반복 횟수를 기반으로 합니다. 브래킷 수는 최적화 중인 메트릭을 로그하는 횟수에 해당합니다. 반복은 step, epoch 또는 그 사이의 어떤 개념에 해당할 수 있습니다. step 카운터의 수치 값 자체는 브래킷 계산에 사용되지 않습니다.

<Note>
  브래킷 스케줄을 생성하려면 `min_iter` 또는 `max_iter` 중 하나를 지정합니다.
</Note>

| Key        | Description                                                    |
| ---------- | -------------------------------------------------------------- |
| `min_iter` | 첫 번째 브래킷에 대한 반복 횟수를 지정합니다.                  |
| `max_iter` | 최대 반복 횟수를 지정합니다.                                   |
| `s`        | 총 브래킷 수를 지정합니다 (`max_iter`에 필요).                 |
| `eta`      | 브래킷 배수 스케줄을 지정합니다 (기본값: `3`).                 |
| `strict`   | 실행을 더 공격적으로 가지치기하여 원래 Hyperband 논문을 더 밀접하게 따르는 &#39;strict&#39; 모드를 활성화합니다. 기본값은 false입니다. |

<Note>
  Hyperband는 몇 분마다 한 번씩 어떤 [실행](/ko/models/ref/python/experiments/run)을 종료할지 확인합니다. 실행 또는 반복 시간이 짧은 경우, 실행 종료 타임스탬프는 지정한 브래킷과 다를 수 있습니다.
</Note>

<div id="command">
  ## `command`
</div>

{/* [`wandb agent`](/models/ref/cli/wandb-agent)로 생성된 에이전트는 기본적으로 다음 형식의 명령을 받습니다: */}

`command` 키 내의 중첩 값을 사용해 형식과 내용을 수정할 수 있습니다. 파일 이름과 같은 고정된 요소는 그대로 포함할 수 있습니다.

<Note>
  Unix 시스템에서 `/usr/bin/env`는 환경에 따라 OS가 올바른 Python 인터프리터를 선택하도록 보장합니다.
</Note>

W&amp;B는 command 내 가변 요소에 대해 다음 매크로를 지원합니다:

| Command macro              | Description                                                                                              |
| -------------------------- | -------------------------------------------------------------------------------------------------------- |
| `${env}`                   | Unix 시스템에서는 `/usr/bin/env`, Windows에서는 생략됩니다.                                                            |
| `${interpreter}`           | `python`으로 확장됩니다.                                                                                        |
| `${program}`               | sweep 설정의 `program` 키에 지정된 학습 스크립트 파일 이름입니다.                                                             |
| `${args}`                  | `--param1=value1 --param2=value2` 형식의 하이퍼파라미터와 그 값입니다.                                                   |
| `${args_no_boolean_flags}` | 하이퍼파라미터와 그 값이 `--param1=value1` 형식이며, 불리언 파라미터는 `True`일 때 `--boolean_flag_param` 형태로, `False`일 때는 생략됩니다. |
| `${args_no_hyphens}`       | `param1=value1 param2=value2` 형식의 하이퍼파라미터와 그 값입니다.                                                       |
| `${args_json}`             | JSON으로 인코딩된 하이퍼파라미터와 그 값입니다.                                                                             |
| `${args_json_file}`        | JSON으로 인코딩된 하이퍼파라미터와 그 값을 포함하는 파일의 경로입니다.                                                                |
| `${envvar}`                | 환경 변수를 전달하는 방법입니다. `${envvar:MYENVVAR}` &#95;&#95;는 MYENVVAR 환경 변수의 값으로 확장됩니다. &#95;&#95;                |
