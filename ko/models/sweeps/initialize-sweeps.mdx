title: sweep 초기화
---

W&amp;B는 클라우드(standard) 또는 하나 이상의 머신에 걸친 로컬(local) 환경에서 sweep을 관리하기 위해 &#95;Sweep Controller&#95;를 사용합니다. run이 완료되면 sweep controller는 다음에 실행할 새 run에 대한 지침을 내립니다. 이 지침은 실제로 run을 수행하는 &#95;agents&#95;가 받아 실행합니다. 일반적인 W&amp;B Sweep에서는 controller가 W&amp;B server에서 실행됩니다. Agents는 &#95;사용자&#95;의 머신에서 실행됩니다.

다음 코드 스니펫은 CLI와 Jupyter Notebook 또는 Python script에서 sweep을 초기화하는 방법을 보여줍니다.

<Warning>
  1. sweep을 초기화하기 전에 YAML 파일 또는 스크립트의 중첩된 Python dictionary 객체에 sweep 설정이 정의되어 있는지 확인하세요. 자세한 내용은 [sweep 설정 정의
     ](/ko/models/sweeps/define-sweep-configuration/)를 참조하세요. 2. W&amp;B Sweep과 W&amp;B Run은 반드시 동일한 프로젝트에 있어야 합니다. 따라서 W&amp;B를 초기화할 때 제공하는 이름
     ([`wandb.init()`](/ko/models/ref/python/functions/init))은 W&amp;B Sweep을 초기화할 때 제공하는 프로젝트 이름([`wandb.sweep()`](/ko/models/ref/python/functions/sweep))과 일치해야 합니다.
</Warning>

<Tabs>
  <Tab title="Python 스크립트 또는 노트북">
    W&amp;B SDK를 사용해 sweep을 초기화합니다. sweep 설정 dictionary를 `sweep` parameter에 전달하세요. 선택적으로 W&amp;B Run의 출력이 저장될 프로젝트 이름을 프로젝트 parameter(`project`)로 지정할 수 있습니다. 프로젝트를 지정하지 않으면 run은 &quot;Uncategorized&quot; 프로젝트에 저장됩니다.

    [`wandb.sweep()`](/ko/models/ref/python/functions/sweep) 함수는 sweep ID를 반환합니다. sweep ID에는 entity 이름과 프로젝트 이름이 포함됩니다. sweep ID를 기록해 두세요.
  </Tab>

  <Tab title="CLI">
    W&amp;B CLI를 사용해 sweep을 초기화합니다. 설정 파일 이름을 지정하세요. 선택적으로 `project` 플래그로 프로젝트 이름을 지정할 수 있습니다. 프로젝트를 지정하지 않으면 W&amp;B Run은 &quot;Uncategorized&quot; 프로젝트에 저장됩니다.

    [`wandb sweep`](/ko/models/ref/cli/wandb-sweep) 명령어를 사용해 sweep을 초기화합니다. 다음 코드 예제는 `sweeps_demo` 프로젝트용 sweep을 초기화하고, 설정 파일로 `config.yaml`을 사용합니다.

    ```bash
    wandb sweep --project sweeps_demo config.yaml
    ```

    이 명령어를 실행하면 sweep ID가 출력됩니다. sweep ID에는 entity 이름과 프로젝트 이름이 포함됩니다. sweep ID를 기록해 두세요.
  </Tab>
</Tabs>
