title: 스윕 초기화
---

스윕 설정을 정의한 후 에이전트를 시작하기 전에 스윕을 초기화하여 해당 설정을 W&amp;B에 등록하고 스윕 ID를 받으세요. 에이전트는 스윕 ID를 사용해 run 지침을 가져옵니다. 스윕이 초기화되면 하나 이상의 머신에서 하이퍼파라미터 검색을 조정할 준비가 됩니다.

W&amp;B는 하나 이상의 머신에서 스윕을 관리하기 위해 sweep controller를 사용합니다. run이 완료되면 controller는 다음에 실행할 run을 설명하는 새 지침 세트를 발행하고, 에이전트는 이 지침을 받아 실행합니다. 기본적으로 W&amp;B가 controller를 호스팅하며, 에이전트는 사용자의 머신에서 실행됩니다.

스윕을 초기화하기 전에 YAML 파일 또는 스크립트의 중첩된 Python 딕셔너리로 스윕 설정이 정의되어 있어야 합니다. 자세한 내용은 [스윕 설정 정의](/ko/models/sweeps/define-sweep-configuration)를 참조하세요.

다음 코드 스니펫은 CLI와 Jupyter 노트북 또는 Python 스크립트에서 스윕을 초기화하는 방법을 보여줍니다. 워크플로에 맞는 방법을 선택하세요.

<Warning>
  스윕과 run은 동일한 프로젝트에 있어야 합니다. W&amp;B를 초기화할 때 제공하는 프로젝트 이름([`wandb.init()`](/ko/models/ref/python/functions/init))은 스윕을 초기화할 때 제공하는 프로젝트 이름([`wandb.sweep()`](/ko/models/ref/python/functions/sweep))과 일치해야 합니다.
</Warning>

<Tabs>
  <Tab title="Python 스크립트 또는 노트북">
    W&amp;B SDK를 사용해 스윕을 초기화하세요. 스윕 설정 딕셔너리를 `sweep` 파라미터에 전달합니다. 필요에 따라 `project` 파라미터를 설정해 W&amp;B가 run의 출력을 저장할 프로젝트 이름을 지정할 수 있습니다. 프로젝트를 지정하지 않으면 W&amp;B는 run을 &quot;Uncategorized&quot; 프로젝트에 넣습니다.

    ```python
    import wandb

    # 예시 스윕 설정
    sweep_configuration = {
        "method": "random",
        "name": "sweep",
        "metric": {"goal": "maximize", "name": "val_acc"},
        "parameters": {
            "batch_size": {"values": [16, 32, 64]},
            "epochs": {"values": [5, 10, 15]},
            "lr": {"max": 0.1, "min": 0.0001},
        },
    }

    sweep_id = wandb.sweep(sweep=sweep_configuration, project="project-name")
    ```

    [`wandb.sweep()`](/ko/models/ref/python/functions/sweep) 함수는 스윕 ID를 반환합니다. 스윕 ID에는 entity 이름과 프로젝트 이름이 포함됩니다. 에이전트를 시작할 때 전달할 이 스윕 ID를 기록해 두세요. 스윕 ID가 있으면 하나 이상의 에이전트를 시작해 Runs를 실행할 준비가 된 것입니다.
  </Tab>

  <Tab title="CLI">
    W&amp;B CLI를 사용해 스윕을 초기화하세요. 설정 파일 이름을 지정하세요. 필요에 따라 `--project` 플래그를 설정해 프로젝트 이름을 지정할 수 있습니다. 프로젝트를 지정하지 않으면 W&amp;B는 run을 &quot;Uncategorized&quot; 프로젝트에 넣습니다.

    [`wandb sweep`](/ko/models/ref/cli/wandb-sweep) 명령을 사용해 스윕을 초기화하세요. 다음 코드 예시는 `sweeps_demo` 프로젝트의 스윕을 초기화하고, 설정에 `config.yaml` 파일을 사용합니다.

    ```bash
    wandb sweep --project sweeps_demo config.yaml
    ```

    이 명령은 스윕 ID를 출력합니다. 스윕 ID에는 entity 이름과 프로젝트 이름이 포함됩니다. 에이전트를 시작할 때 전달할 이 스윕 ID를 기록해 두세요. 스윕 ID가 있으면 하나 이상의 에이전트를 시작해 Runs를 실행할 준비가 된 것입니다.
  </Tab>
</Tabs>
