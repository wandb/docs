title: Sweep 결과 시각화
---

W&amp;B App에서 W&amp;B Sweeps의 결과를 시각화합니다. [W&amp;B App](https://wandb.ai/home)으로 이동합니다. sweep를 초기화할 때 지정한 프로젝트를 선택합니다. 그러면 프로젝트 [workspace](/ko/models/track/workspaces/)로 리디렉션됩니다. 프로젝트 사이드바에서 **Sweep 아이콘**(빗자루 아이콘)을 선택합니다. Sweep UI에서 목록에 있는 Sweep 이름을 선택합니다.

sweep 목록에는 각 sweep의 상태(**State**), 생성 시간(**Created**), 시작한 사용자(**Creator**), 완료된 run 수(**Run count**), 총 **Compute time**이 표시됩니다. 이산 검색 공간에 대한 grid search의 경우 W&amp;B는 **Est. Runs**(예상 run 수)도 표시합니다. 목록에서 sweep를 열면 앱에서 일시 중지, 재개, 중지 또는 강제 종료할 수 있습니다. CLI에서 동일한 제어를 하려면 [Manage sweeps](/ko/models/sweeps/pause-resume-and-cancel-sweeps/)를 참조하세요.

기본적으로 W&amp;B는 W&amp;B Sweep 작업을 시작하면 병렬 좌표 플롯, 파라미터 중요도 플롯, 산점도를 자동으로 생성합니다.

<Frame>
  <img src="/images/sweeps/navigation_sweeps_ui.gif" alt="프로젝트 사이드바의 Sweep UI" />

병렬 좌표 차트는 많은 수의 하이퍼파라미터와 모델 메트릭 간의 관계를 한눈에 요약해 보여줍니다. 병렬 좌표 플롯에 대한 자세한 내용은 [Parallel coordinates](/ko/models/app/features/panels/parallel-coordinates/)를 참조하세요.

<Frame>
  <img src="/images/sweeps/example_parallel_coordiantes_plot.png" alt="병렬 좌표 플롯 예시." />
</Frame>

산점도(왼쪽)는 Sweep 중에 생성된 W&amp;B Runs를 비교합니다. 산점도에 대한 자세한 내용은 [Scatter Plots](/ko/models/app/features/panels/scatter-plot/)를 참조하세요.

파라미터 중요도 플롯(오른쪽)에는 메트릭의 바람직한 값을 가장 잘 예측하고 높은 상관관계를 보이는 하이퍼파라미터가 나열됩니다. 파라미터 중요도 플롯에 대한 자세한 내용은 [Parameter Importance](/ko/models/app/features/panels/parameter-importance/)를 참조하세요.

<Frame>
  <img src="/images/sweeps/scatter_and_parameter_importance.png" alt="산점도와 파라미터 중요도" />
</Frame>

자동으로 사용되는 종속값과 독립값(x축과 y축)을 변경할 수 있습니다. 각 패널에는 **Edit panel**이라는 연필 아이콘이 있습니다. **Edit panel**을 선택합니다. 모달이 나타납니다. 모달 내에서 그래프의 동작을 변경할 수 있습니다.

모든 기본 W&amp;B 시각화 옵션에 대한 자세한 내용은 [Panels](/ko/models/app/features/panels/)를 참조하세요. W&amp;B Sweep에 속하지 않는 W&amp;B Runs에서 플롯을 만드는 방법은 [Data Visualization docs](/ko/models/tables/)를 참조하세요.
