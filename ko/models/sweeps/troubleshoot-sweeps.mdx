제안된 가이드를 참고하여 일반적인 오류 메시지를 해결하세요.

<div id="commerror-run-does-not-exist-and-error-error-uploading">
  ### `CommError, Run does not exist` and `ERROR Error uploading`
</div>

이 두 오류 메시지가 모두 반환된다면 W&amp;B run ID가 어딘가에서 정의되고 있을 수 있습니다. 예를 들어, Jupyter Notebook이나 Python 스크립트 어딘가에 다음과 비슷한 코드 스니펫이 정의돼 있을 수 있습니다:

```python
wandb.init(id="some-string")
```

W&amp;B Sweeps에 대해서는 Run ID를 설정할 수 없습니다. 이는 W&amp;B Sweeps로 생성되는 Runs에 대해 W&amp;B가 임의의 고유 ID를 자동으로 생성하기 때문입니다.

W&amp;B Run ID는 하나의 프로젝트 내에서 고유해야 합니다.

테이블과 그래프에 표시될 사용자 지정 이름을 설정하려면, W&amp;B를 초기화할 때 `name` 매개변수에 이름을 전달하는 것을 권장합니다. 예를 들면 다음과 같습니다:

```python
wandb.init(name="a helpful readable run name")
```

<div id="cuda-out-of-memory">
  ### `Cuda out of memory`
</div>

이 오류 메시지가 표시되면 코드가 프로세스 기반으로 실행되도록 리팩터링하십시오. 보다 구체적으로는 코드를 Python 스크립트로 다시 작성하십시오. 또한 W&amp;B Python SDK 대신 CLI에서 W&amp;B Sweep Agent를 호출하십시오.

예를 들어, 코드를 `train.py`라는 Python 스크립트로 다시 작성했다고 가정해 보겠습니다. YAML Sweep 설정 파일(이 예에서는 `config.yaml`)에 트레이닝 스크립트 이름(`train.py`)을 추가하십시오:

```yaml
program: train.py
method: bayes
metric:
  name: validation_loss
  goal: maximize
parameters:
  learning_rate:
    min: 0.0001
    max: 0.1
  optimizer:
    values: ["adam", "sgd"]
```

다음으로 `train.py` Python 스크립트에 다음 코드를 추가하세요:

```python
if _name_ == "_main_":
    train()
```

CLI에서 wandb sweep 명령으로 W&amp;B Sweep을 초기화하세요.

```shell
wandb sweep config.yaml
```

반환된 W&amp;B Sweep ID를 기록해 두십시오. 다음으로, Python SDK([`wandb.agent`](/ko/models/ref/python/functions/agent)) 대신 CLI에서 [`wandb agent`](/ko/models/ref/cli/wandb-agent)를 사용하여 Sweep 작업을 시작합니다. 아래 코드 스니펫의 `sweep_ID`를 이전 단계에서 반환된 Sweep ID로 바꾸십시오:

```shell
wandb agent sweep_ID
```

<div id="anaconda-400-error">
  ### `anaconda 400 error`
</div>

다음 오류는 일반적으로 최적화 대상 메트릭을 로깅하지 않았을 때 발생합니다:

```shell
wandb: ERROR Error while calling W&B API: anaconda 400 error: 
{"code": 400, "message": "TypeError: bad operand type for unary -: 'NoneType'"}
```

YAML 파일 또는 중첩된 사전에서 최적화할 &quot;metric&quot; 키를 지정합니다. 이 메트릭은 반드시 `wandb.log`를 사용해 로깅하세요. 추가로, Python 스크립트나 Jupyter Notebook에서 스윕이 최적화하도록 정의한 메트릭 이름과 *완전히 동일한* 이름을 사용해야 합니다. 설정 파일에 대한 자세한 내용은 [스윕 설정 정의](/ko/models/sweeps/define-sweep-configuration/)를 참조하세요.
