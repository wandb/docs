이 가이드는 하이퍼파라미터 검색 최적화를 위해 Python 트레이닝 스크립트나 노트북에 W&amp;B를 통합하는 방법에 대한 권장 사항을 제공합니다.

<div id="original-training-script">
  ## 원본 트레이닝 스크립트
</div>

모델을 트레이닝하는 Python 스크립트가 있다고 가정해 보세요 (아래 참조). 목표는 검증 정확도(`val_acc`)를 최대화하는 하이퍼파라미터를 찾는 것입니다.

Python 스크립트에서 `train_one_epoch`와 `evaluate_one_epoch`라는 두 함수를 정의합니다. `train_one_epoch` 함수는 한 에포크 동안의 트레이닝을 시뮬레이션하고 트레이닝 정확도와 손실을 반환합니다. `evaluate_one_epoch` 함수는 검증 데이터 세트에서 모델을 평가하는 것을 시뮬레이션하고 검증 정확도와 손실을 반환합니다.

하이퍼파라미터 값들(예: 학습률(`lr`), 배치 크기(`batch_size`), 에포크 수(`epochs`))을 포함하는 설정 딕셔너리(`config`)를 정의합니다. 설정 딕셔너리의 값들은 트레이닝 프로세스를 제어합니다.

다음으로 전형적인 트레이닝 루프를 모방하는 `main`이라는 함수를 정의합니다. 각 에포크마다 트레이닝 및 검증 데이터 세트에서 정확도와 손실이 계산됩니다.

<Note>
  이 코드는 모의 트레이닝 스크립트입니다. 모델을 트레이닝하지 않지만, 무작위 정확도와 손실 값을 생성하여 트레이닝 프로세스를 시뮬레이션합니다. 이 코드의 목적은 트레이닝 스크립트에 W&amp;B를 통합하는 방법을 보여주는 것입니다.
</Note>

```python
import random
import numpy as np

def train_one_epoch(epoch, lr, batch_size):
    acc = 0.25 + ((epoch / 30) + (random.random() / 10))
    loss = 0.2 + (1 - ((epoch - 1) / 10 + random.random() / 5))
    return acc, loss

def evaluate_one_epoch(epoch):
    acc = 0.1 + ((epoch / 20) + (random.random() / 10))
    loss = 0.25 + (1 - ((epoch - 1) / 10 + random.random() / 6))
    return acc, loss

# 하이퍼파라미터 값이 포함된 설정 변수
config = {"lr": 0.0001, "batch_size": 16, "epochs": 5}

def main():
    lr = config["lr"]
    batch_size = config["batch_size"]
    epochs = config["epochs"]

    for epoch in np.arange(1, epochs):
        train_acc, train_loss = train_one_epoch(epoch, lr, batch_size)
        val_acc, val_loss = evaluate_one_epoch(epoch)

        print("epoch: ", epoch)
        print("training accuracy:", train_acc, "training loss:", train_loss)
        print("validation accuracy:", val_acc, "validation loss:", val_loss)

if __name__ == "__main__":
    main()
```

다음 섹션에서는 Python 스크립트에 W&amp;B를 추가하여 트레이닝 중 하이퍼파라미터와 메트릭을 추적합니다. W&amp;B를 사용해 검증 정확도(`val_acc`)를 최대화하는 최적의 하이퍼파라미터를 찾습니다.

<div id="add-wb-to-your-training-script">
  ## 트레이닝 스크립트에 W&amp;B 추가하기
</div>

트레이닝 스크립트를 업데이트하여 W&amp;B를 포함시키세요. Python 스크립트나 노트북에 W&amp;B를 통합하는 방법은 스윕을 관리하는 방식에 따라 다릅니다.

W&amp;B Python SDK를 사용하여 스윕을 시작, 중지 및 관리하려면 **Python script or notebook** 탭의 지침을 따르세요. 대신 W&amp;B CLI를 사용하려면 **CLI** 탭의 지침을 따르세요.

<Tabs>
  <Tab title="CLI">
    스윕 설정을 포함한 YAML 설정 파일을 생성하세요. 이
    설정 파일에는 스윕이 탐색할 하이퍼파라미터가 포함되어 있습니다. 다음
    예시에서는 배치 크기 (`batch_size`), 에포크 (`epochs`), 그리고
    학습률 (`lr`) 하이퍼파라미터가 각 스윕 동안 다양하게 변합니다.

    ```yaml
    # config.yaml
    program: train.py
    method: random
    name: sweep
    metric:
      goal: maximize
      name: val_acc
    parameters:
      batch_size:
        values: [16, 32, 64]
      lr:
        min: 0.0001
        max: 0.1
      epochs:
        values: [5, 10, 15]
    ```

    W&amp;B 스윕 구성을 생성하는 방법에 대한 자세한 내용은 [스윕 구성 정의](/ko/models/sweeps/define-sweep-configuration/)를 참조하세요.

    YAML 파일의 `program` 키에 Python 스크립트 이름을 제공해야 합니다.

    다음으로, 코드 예제에 다음을 추가하세요:

    1. W&amp;B Python SDK(`wandb`)와 PyYAML(`yaml`)을 임포트하세요. PyYAML은 YAML 설정 파일을 읽어들이는 데 사용됩니다.
    2. 설정 파일을 읽어들이세요.
    3. [`wandb.init()`](/ko/models/ref/python/functions/init)을 사용하여 [W&amp;B Run](/ko/models/ref/python/experiments/run)으로 데이터를 동기화하고 로그하는 백그라운드 프로세스를 시작하세요. config 매개변수에 config 객체를 전달하세요.
    4. 하드코딩된 값 대신 `wandb.Run.config`에서 하이퍼파라미터 값을 정의하세요.
    5. [`wandb.Run.log()`](/ko/models/ref/python/experiments/run.md/#method-runlog)를 사용하여 최적화하고자 하는 메트릭을 로그하세요. 구성에서 정의된 메트릭을 로그해야 합니다. 이 예제의 설정 딕셔너리(`sweep_configuration`) 내에서 `val_acc` 값을 최대화하도록 스윕을 정의합니다.

    ```python
    import wandb
    import yaml
    import random
    import numpy as np


    def train_one_epoch(epoch, lr, batch_size):
        acc = 0.25 + ((epoch / 30) + (random.random() / 10))
        loss = 0.2 + (1 - ((epoch - 1) / 10 + random.random() / 5))
        return acc, loss


    def evaluate_one_epoch(epoch):
        acc = 0.1 + ((epoch / 20) + (random.random() / 10))
        loss = 0.25 + (1 - ((epoch - 1) / 10 + random.random() / 6))
        return acc, loss


    def main():
        # 기본 하이퍼파라미터를 설정하세요
        with open("./config.yaml") as file:
            config = yaml.load(file, Loader=yaml.FullLoader)

        with wandb.init(config=config) as run:
            for epoch in np.arange(1, run.config['epochs']):
                train_acc, train_loss = train_one_epoch(epoch, run.config['lr'], run.config['batch_size'])
                val_acc, val_loss = evaluate_one_epoch(epoch)
                run.log(
                    {
                        "epoch": epoch,
                        "train_acc": train_acc,
                        "train_loss": train_loss,
                        "val_acc": val_acc,
                        "val_loss": val_loss,
                    }
                )

    # main 함수를 호출하세요.
    main()
    ```

    CLI에서 스윕
    에이전트가 시도할 Runs의 최대 개수를 설정하세요. 이는 선택 사항입니다. 이 예제에서는 최대 개수를
    5로 설정합니다.

    ```bash
    NUM=5
    ```

    다음으로, [`wandb sweep`](/ko/models/ref/cli/wandb-sweep) 명령어를 사용해 스윕을 초기화하세요. YAML 파일의 이름을 제공하세요. 선택적으로 --project 플래그에 프로젝트 이름을 제공하세요:

    ```bash
    wandb sweep --project sweep-demo-cli config.yaml
    ```

    스윕 ID를 반환합니다. 스윕 초기화 방법에 대한 자세한 내용은
    [스윕 초기화](./initialize-sweeps)를
    참조하세요.

    스윕 ID를 복사하고 다음 코드 스니펫에서 `sweepID`를 교체하여
    [`wandb agent`](/ko/models/ref/cli/wandb-agent)
    명령어로 스윕 작업을 시작하세요:

    ```bash
    wandb agent --count $NUM your-entity/sweep-demo-cli/sweepID
    ```

    자세한 내용은 [스윕 작업 시작](./start-sweep-agents)을 참조하세요.

  <Tab title="Python 스크립트 또는 노트북">
    Python 스크립트에 W&amp;B를 추가하려면 다음 단계를 따르세요:

    1. 키-값 쌍으로 [스윕 설정](/ko/models/sweeps/define-sweep-configuration/)을 정의하는 딕셔너리 객체를 만드세요. 스윕 설정은 W&amp;B가 사용자 대신 탐색하도록 하려는 하이퍼파라미터와 최적화하려는 메트릭을 정의합니다. 이전 예제를 이어서, 배치 크기(`batch_size`), 에포크(`epochs`), 학습률(`lr`)은 각 스윕 동안 변경할 하이퍼파라미터입니다. 검증 정확도를 최대화하고 싶으므로 `"goal": "maximize"`를 설정하고, 이 경우 최적화 대상 변수의 이름으로 `val_acc` (`"name": "val_acc"`)를 지정합니다.
    2. 스윕 설정 딕셔너리를 [`wandb.sweep()`](/ko/models/ref/python/functions/sweep)에 전달하세요. 이는 스윕을 초기화하고 스윕 ID(`sweep_id`)를 반환합니다. 자세한 내용은 [Initialize sweeps](./initialize-sweeps)를 참조하세요.
    3. 스크립트의 맨 위에서 W&amp;B Python SDK (`wandb`)를 임포트하세요.
    4. `main` 함수 내에서 [`wandb.init()`](/ko/models/ref/python/functions/init)를 사용하여 [W&amp;B Run](/ko/models/ref/python/experiments/run)으로 데이터를 동기화하고 로그를 기록하는 백그라운드 프로세스를 생성하세요. `wandb.init()` 메서드에 프로젝트 이름을 매개변수로 전달하세요. 프로젝트 이름을 전달하지 않으면 W&amp;B는 기본 프로젝트 이름을 사용합니다.
    5. `wandb.Run.config` 객체에서 하이퍼파라미터 값을 가져옵니다. 이렇게 하면 하드 코딩된 값 대신 스윕 설정 딕셔너리에 정의된 하이퍼파라미터 값을 사용할 수 있습니다.
    6. 최적화하려는 메트릭을 [`wandb.Run.log()`](/ko/models/ref/python/experiments/run.md/#method-runlog)을 사용하여 W&amp;B에 로깅하세요. 설정에서 정의된 메트릭을 로깅해야 합니다. 예를 들어, 최적화할 메트릭을 `val_acc`로 정의했다면 `val_acc`를 로깅해야 합니다. 메트릭을 로깅하지 않으면 W&amp;B는 무엇을 최적화해야 할지 알 수 없습니다. 이 예제에서 `sweep_configuration` 설정 딕셔너리 내에서 `val_acc` 값을 최대화하도록 스윕을 정의합니다.
    7. [`wandb.agent()`](/ko/models/ref/python/functions/agent)를 사용하여 스윕을 시작하세요. 스윕 ID와 스윕이 실행할 함수의 이름(`function=main`)을 제공하고, 최대 4개의 런을 시도할 수 있도록 `count=4`를 지정하세요.

    이 모든 것을 종합하면, 스크립트는 다음과 비슷할 수 있습니다:

    ```python
    import wandb # W&B Python SDK를 임포트합니다
    import numpy as np
    import random
    import argparse

    def train_one_epoch(epoch, lr, batch_size):
        acc = 0.25 + ((epoch / 30) + (random.random() / 10))
        loss = 0.2 + (1 - ((epoch - 1) / 10 + random.random() / 5))
        return acc, loss

    def evaluate_one_epoch(epoch):
        acc = 0.1 + ((epoch / 20) + (random.random() / 10))
        loss = 0.25 + (1 - ((epoch - 1) / 10 + random.random() / 6))
        return acc, loss

    def main(args=None):
        # 스윕 에이전트에 의해 호출될 때 args는 None이 됩니다.
        # 따라서 스윕 설정의 프로젝트를 사용합니다
        project = args.project if args else None
        
        with wandb.init(project=project) as run:
            # `wandb.Run.config` 객체에서 하이퍼파라미터 값을 가져옵니다
            lr = run.config["lr"]
            batch_size = run.config["batch_size"]
            epochs = run.config["epochs"]

            # 트레이닝 루프를 실행하고 성능 값을 W&B에 로깅합니다
            for epoch in np.arange(1, epochs):
                train_acc, train_loss = train_one_epoch(epoch, lr, batch_size)
                val_acc, val_loss = evaluate_one_epoch(epoch)
                run.log(
                    {
                        "epoch": epoch,
                        "train_acc": train_acc,
                        "train_loss": train_loss,
                        "val_acc": val_acc, # 최적화 대상 메트릭
                        "val_loss": val_loss,
                    }
                )

    if __name__ == "__main__":
        parser = argparse.ArgumentParser()
        parser.add_argument("--project", type=str, default="sweep-example", help="W&B 프로젝트 이름")
        args = parser.parse_args()

        # 스윕 설정 딕셔너리를 정의합니다
        sweep_configuration = {
            "method": "random",
            "name": "sweep",
            # 최적화하려는 메트릭
            # 예를 들어, 검증 정확도를 최대화하려면 "goal": "maximize"로 설정하고
            # 최적화 대상 변수의 이름을 지정하십시오. 이 경우 "val_acc"입니다
            "metric": {
                "goal": "maximize",
                "name": "val_acc"
                },
            "parameters": {
                "batch_size": {"values": [16, 32, 64]},
                "epochs": {"values": [5, 10, 15]},
                "lr": {"max": 0.1, "min": 0.0001},
            },
        }

        # 설정 딕셔너리를 전달하여 스윕을 초기화합니다
        sweep_id = wandb.sweep(sweep=sweep_configuration, project=args.project)

        # 스윕 작업을 시작합니다
        wandb.agent(sweep_id, function=main, count=4)
    ```

<Note>
  **스윕에서 W&amp;B에 메트릭 로깅하기**

  스윕 설정에서 정의하고 최적화 대상으로 지정한 메트릭을 `wandb.Run.log()`를 사용해 W&amp;B에 로깅해야 합니다. 예를 들어, 스윕 설정 내에서 최적화할 메트릭으로 `val_acc`를 정의한 경우, `val_acc`를 W&amp;B에 로깅해야 합니다. 메트릭을 로깅하지 않으면 W&amp;B는 무엇을 최적화해야 할지 알 수 없습니다.

  ```python
  with wandb.init() as run:
      val_loss, val_acc = train()
      run.log(
          {
              "val_loss": val_loss,
              "val_acc": val_acc
              }
          )
  ```

  다음은 W&amp;B에 메트릭을 로깅하는 올바르지 않은 예입니다. 스윕 설정에서 최적화 대상 메트릭은 `val_acc`이지만, 코드에서는 `validation` 키 아래 중첩된 딕셔너리 안에 `val_acc`를 로깅합니다. 메트릭은 중첩된 딕셔너리 안에 넣지 말고 직접 로깅해야 합니다.

  ```python
  with wandb.init() as run:
      val_loss, val_acc = train()
      run.log(
          {
              "validation": {
                  "val_loss": val_loss, 
                  "val_acc": val_acc
                  }
              }
          )
  ```
</Note>
