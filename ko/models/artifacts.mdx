W&amp;B Artifacts를 사용하여 [W&amp;B 실행](/ko/models/runs)의 입력 및 출력 데이터로 사용되는 정보를 추적하고 버전 관리를 수행하세요. 예를 들어, 모델 학습 실행은 입력으로 데이터셋을 받아들이고 출력으로 학습된 모델을 생성할 수 있습니다. 하이퍼파라미터, 메타데이터, 지표는 실행에 로깅할 수 있으며, 모델 학습에 사용된 데이터셋을 입력으로 로깅·추적·버전 관리하기 위해 하나의 아티팩트를 사용하고, 결과로 생성된 모델 체크포인트를 출력으로 관리하기 위해 또 다른 아티팩트를 사용할 수 있습니다.

<div id="use-cases">
  ## 사용 사례
</div>

아티팩트는 전체 ML 워크플로우에서 [실행](/ko/models/runs)의 입력 및 출력으로 사용할 수 있습니다. 데이터셋, 모델, 또는 다른 아티팩트를 처리를 위한 입력으로 사용할 수 있습니다.

| 사용 사례              | 입력                        | 출력                         |
|------------------------|-----------------------------|------------------------------|
| 모델 학습              | 데이터셋(학습 및 검증 데이터) | 학습된 모델                  |
| 데이터셋 전처리        | 데이터셋(원시 데이터)       | 데이터셋(전처리된 데이터)    |
| 모델 평가              | 모델 + 데이터셋(테스트 데이터) | [W&amp;B Table](/ko/models/tables)  |
| 모델 최적화            | 모델                        | 최적화된 모델                |

<div id="create-an-artifact">
  ## 아티팩트 생성
</div>

네 줄 코드만으로 아티팩트를 생성합니다:

1. [W&amp;B 실행](/ko/models/runs)을 생성합니다.
2. [`wandb.Artifact`](/ko/models/ref/python/experiments/artifact)로 아티팩트 객체를 생성합니다.
3. `wandb.Artifact.add_file()`로 모델 파일이나 데이터셋과 같은 하나 이상의 파일을 아티팩트 객체에 추가합니다.
4. `wandb.Run.log_artifact()`로 아티팩트를 W&amp;B에 로그합니다.

예를 들어, 다음 코드 예제는 `dataset.h5`라는 파일을 `example_artifact`라는 아티팩트로 로그하는 방법을 보여줍니다:

```python
import wandb

with wandb.init(project="artifacts-example", job_type="add-dataset") as run:
    artifact = wandb.Artifact(name="example_artifact", type="dataset")
    artifact.add_file(local_path="./dataset.h5", name="training_dataset")
    run.log_artifact(artifact)
```

* Artifact의 `type`은 W&amp;B 플랫폼에서 어떻게 표시되는지에 영향을 줍니다. `type`을 지정하지 않으면 기본값은 `unspecified`입니다.
* 드롭다운의 각 레이블은 서로 다른 `type` 매개변수 값을 나타냅니다. 위 코드 예시에서 아티팩트의 `type`은 `dataset`입니다.

<Note>
  Amazon S3 버킷과 같은 외부 오브젝트 스토리지에 저장된 파일이나 디렉터리에 대한 참조를 추가하는 방법은 [track external files](/ko/models/artifacts/track-external-files) 페이지를 참조하세요.
</Note>

<div id="download-an-artifact">
  ## 아티팩트 다운로드
</div>

[`wandb.Run.use_artifact()`](/ko/models/ref/python/experiments/run#use_artifact) 메서드를 사용해 실행의 입력으로 표시하려는 아티팩트를 지정합니다.

이전 코드 스니펫에 이어, 다음 코드 예시는 앞에서 생성한 `example_artifact` 아티팩트를 사용하는 방법을 보여줍니다:

```python
with wandb.init(project="artifacts-example", job_type="add-dataset") as run:
    artifact = run.use_artifact("training_dataset:latest")  # "my_data" 아티팩트를 사용하는 실행 객체를 반환합니다
```

이것은 Artifact 객체를 반환합니다.

이제 반환된 객체를 사용해 아티팩트의 모든 내용을 다운로드합니다:

```python
datadir = artifact.download()  # 전체 `my_data` 아티팩트를 기본 디렉토리에 다운로드합니다.
```

<Note>
  `root` [파라미터](/ko/models/ref/python/experiments/artifact)에 사용자 지정 경로를 전달하여 아티팩트를 특정 디렉터리에 다운로드할 수 있습니다. 아티팩트를 다운로드하는 다른 방법과 사용할 수 있는 추가 파라미터는 [아티팩트 다운로드 및 사용](/ko/models/artifacts/download-and-use-an-artifact) 가이드를 참조하세요.
</Note>
