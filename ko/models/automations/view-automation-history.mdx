<Note>
  자동화 실행 이력은 W&amp;B Multi-tenant Cloud, W&amp;B Dedicated Cloud, W&amp;B Self-Managed v0.75.0 이상에서 사용 가능합니다.
</Note>

이 페이지에서는 W&amp;B [자동화](/ko/models/automations)의 실행 이력을 확인하고 이해하는 방법을 설명합니다. 실행 이력에는 무엇이 자동화를 트리거했는지, 어떤 액션이 실행되었는지, 그리고 실행이 성공했는지 실패했는지가 표시됩니다. 자동화 이력을 검토하면 자동화가 예상대로 실행되는지 확인하고, 실패를 해결하며, 어떤 이벤트가 다운스트림 액션을 트리거했는지 감사할 수 있습니다.

실행된 각 자동화는 다음 정보를 포함하는 레코드를 생성합니다.

* **실행 타임스탬프**: 자동화가 트리거된 시점입니다.
* **트리거 이벤트**: 자동화를 트리거한 특정 이벤트입니다.
* **상태**: 실행 상태입니다. [실행 상태](#execution-status)를 참조하세요.
* **액션 세부 정보**: Slack 채널에 알림을 보내거나 웹훅을 실행하는 등 어떤 액션이 실행되었는지에 대한 정보입니다.
* **결과 세부 정보**: 있는 경우, 실패한 실행의 오류를 포함한 자동화의 최종 결과에 대한 추가 정보입니다.

자동화 이력은 레지스트리 범위 자동화와 프로젝트 범위 자동화 모두에서 사용할 수 있습니다. 자세한 지침은 **레지스트리** 또는 **Project** 탭을 선택하세요.

<Tabs>
  <Tab title="레지스트리">
    1. 프로젝트 사이드바에서 **레지스트리**를 클릭합니다.
    2. 목록에서 레지스트리를 선택합니다.
    3. 레지스트리의 자동화가 나열된 **자동화** 탭으로 이동합니다. 실행 이력 세부 정보를 보려면 **Last execution** 타임스탬프를 클릭합니다. 검색창을 사용해 자동화 이름으로 필터링하고, 마지막 트리거 날짜순으로 정렬해 최근에 실행된 자동화를 찾습니다.
    4. **자동화 history** 탭에서 레지스트리의 자동화 실행을 최신순으로 확인합니다. 여기에는 이벤트, 액션, 상태가 포함됩니다. 특정 실행의 세부 정보를 보려면 실행 타임스탬프를 클릭합니다.

    <Tip>컬렉션에 연결된 자동화 실행이 있으면 아이콘 ![컬렉션 자동화 실행 기호, 오른쪽을 가리키는 화살표가 있는 원](/images/automations/collection-automation-execution-icon.png)과 함께 연결된 실행 수가 표시됩니다.</Tip>
  </Tab>

  <Tab title="Project">
    1. 프로젝트로 이동합니다.
    2. 프로젝트 사이드바에서 프로젝트의 자동화가 나열된 **자동화** 탭을 클릭합니다. 실행 이력 세부 정보를 보려면 **Last execution** 타임스탬프를 클릭합니다. 검색창을 사용해 자동화 이름으로 필터링하고, 마지막 트리거 날짜순으로 정렬해 최근에 실행된 자동화를 찾습니다.
    3. **History** 탭에서 프로젝트의 자동화에 대한 모든 실행을 최신순으로 확인합니다. 각 실행의 메타데이터가 표시되며, 여기에는 이벤트, 액션, 상태가 포함됩니다. 특정 실행의 세부 정보를 보려면 실행 타임스탬프를 클릭합니다.
  </Tab>
</Tabs>

<div id="execution-details">
  ## 실행 세부 정보
</div>

이력에서 실행을 찾은 후에는 다음 정보를 바탕으로 해당 실행의 상태와 결과를 해석하세요.

<div id="execution-status">
  ### 실행 상태
</div>

각 자동화 실행은 다음 상태 중 하나를 가집니다:

* **완료됨**: 자동화가 모든 작업을 성공적으로 완료했습니다.
* **실패함**: 자동화 실행 중 오류가 발생하여 완료되지 않았습니다.
* **대기 중**: 자동화가 실행을 위해 큐에서 대기 중입니다.

<div id="execution-metadata">
  ### 실행 메타데이터
</div>

이력에서 실행 항목을 클릭하면 다음 세부 정보를 확인할 수 있습니다:

* **이벤트 세부 정보**: 자동화를 트리거한 특정 이벤트로, 다음이 포함됩니다:
  * 이벤트 유형(예: &quot;새 아티팩트 버전&quot;, &quot;run 완료&quot;)
  * run ID 또는 아티팩트 이름 등의 엔터티 정보
  * 이벤트를 트리거한 사용자(해당하는 경우)
* **액션 세부 정보**: 자동화가 수행하려고 시도한 작업에 대한 정보:
  * 액션 유형(Slack 알림 또는 웹훅)
  * 대상(Slack 채널 또는 웹훅 URL)
  * 전송된 페이로드(웹훅의 경우)
* **결과 정보**:
  * 응답 상태(웹훅의 경우)
  * 오류 메시지 또는 스택 트레이스(실행이 실패한 경우)
