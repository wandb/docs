title: Automation 실행 이력 보기
---

<Note>
  이 기능을 사용하려면 Enterprise 라이선스가 필요합니다. 다음 환경에서만 사용할 수 있습니다:

  * W&amp;B Multi-tenant Cloud
  * W&amp;B Dedicated Cloud
  * W&amp;B Self-Managed v0.75.0+
</Note>

이 페이지에서는 W&amp;B [automations](/ko/models/automations)의 실행 이력을 조회하고 이해하는 방법을 설명합니다. 여기에는 automation을 트리거한 요소, 어떤 작업이 수행되었는지, 그리고 작업이 성공했는지 실패했는지가 포함됩니다.

실행된 각 automation은 다음을 포함하는 기록을 생성합니다:

* **Execution timestamp**: automation이 실행(트리거)된 시각.
* **Triggering event**: automation을 트리거한 구체적인 이벤트.
* **Status**: 실행 상태. [Execution status](#execution-status)를 참조하세요.
* **Action details**: Slack 채널 알림 전송, webhook 실행 등 어떤 작업이 수행되었는지에 대한 정보.
* **Result details**: 실패한 실행의 오류를 포함해, automation 실행의 최종 결과에 대한 추가 정보(있는 경우).

사용 사례에 따라 automation 이력을 조회하는 방법에 대한 자세한 안내를 보려면 **Registry** 또는 **Project** 탭을 선택하세요.

<Tabs>
  <Tab title="레지스트리">
    1. 왼쪽 사이드바에서 **Registry**를 클릭해 레지스트리로 이동합니다.
    2. 목록에서 레지스트리를 선택합니다.
    3. **Automations** 탭에서 레지스트리의 automation을 확인합니다. **Last execution** 타임스탬프를 클릭해 실행 이력 세부 정보를 확인합니다. 검색 창을 사용해 automation 이름으로 필터링할 수 있으며, 마지막으로 트리거된 날짜로 정렬해 최근에 실행된 automation을 찾을 수 있습니다.
    4. **Automations history** 탭에서 레지스트리의 automation 실행을 최신순(역순)으로 확인합니다. 여기에는 이벤트, 작업, 상태가 포함됩니다. 특정 실행에 대한 세부 정보를 보려면 실행 타임스탬프를 클릭합니다.

    <Tip>컬렉션에 연결된 automation 실행이 있는 경우, ![컬렉션 automation 실행 아이콘, 오른쪽을 가리키는 화살표가 있는 원](/images/automations/collection-automation-execution-icon.png) 아이콘과 함께 관련 실행 횟수가 표시됩니다.</Tip>
  </Tab>

  <Tab title="프로젝트">
    1. 왼쪽 내비게이션에서 **Automations** 탭을 클릭합니다. 프로젝트의 automation이 표시됩니다. **Last execution** 타임스탬프를 클릭해 실행 이력 세부 정보를 확인합니다. 검색 창을 사용해 automation 이름으로 필터링할 수 있으며, 마지막으로 트리거된 날짜로 정렬해 최근에 실행된 automation을 찾을 수 있습니다.
    2. **History** 탭에서 가장 최근 실행부터 시작해 프로젝트 automation의 모든 실행을 최신순(역순)으로 확인합니다. 각 실행에 대해 이벤트, 작업, 상태를 포함한 메타데이터가 표시됩니다. 특정 실행에 대한 세부 정보를 보려면 실행 타임스탬프를 클릭합니다.
  </Tab>
</Tabs>

<div id="understanding-execution-details">
  ## 실행 세부 정보 이해하기
</div>

각 자동화 실행에는 다음 상태 중 하나가 있습니다:

* **Finished**: 자동화가 모든 작업을 성공적으로 완료했습니다.
* **Failed**: 자동화 도중 오류가 발생해 정상적으로 종료되지 않았습니다.
* **Pending**: 자동화가 실행 대기 상태입니다.

히스토리에서 실행을 클릭하면 다음과 같은 세부 정보를 확인할 수 있습니다:

* **이벤트 세부 정보**: 자동화를 트리거한 구체적인 이벤트로, 다음을 포함합니다:
  * 이벤트 유형 (예: &quot;새 아티팩트 버전&quot;, &quot;실행 완료&quot;)
  * 엔터티 정보 (실행 ID, 아티팩트 이름 등)
  * 이벤트를 트리거한 사용자 (해당되는 경우)

* **동작 세부 정보**: 자동화가 수행하려고 시도한 작업에 대한 정보:
  * 동작 유형 (Slack 알림 또는 웹훅)
  * 대상 (Slack 채널 또는 웹훅 URL)
  * 전송된 페이로드 (웹훅의 경우)

* **결과 정보**:
  * 응답 상태 (웹훅의 경우)
  * 오류 메시지 또는 스택 트레이스 (실패한 실행의 경우)
