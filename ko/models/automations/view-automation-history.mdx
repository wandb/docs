<Note>
  이 기능을 사용하려면 엔터프라이즈 라이선스가 필요합니다. 다음 환경에서만 사용 가능합니다:

  * W&amp;B Multi-tenant Cloud
  * W&amp;B Dedicated Cloud
  * W&amp;B Self-Managed v0.75.0+
</Note>

이 페이지에서는 W&amp;B [automations](/ko/models/automations)의 실행 이력을 확인하고 이해하는 방법을 설명합니다. 자동화를 트리거한 항목, 수행된 액션, 성공 또는 실패 여부를 확인할 수 있습니다.

실행된 자동화마다 다음 정보를 포함하는 레코드가 생성됩니다:

* **실행 타임스탬프**: 자동화가 트리거된 시각.
* **트리거 이벤트**: 자동화를 트리거한 특정 이벤트.
* **상태**: 실행 상태. [실행 상태](#execution-status)를 참조하세요.
* **액션 세부 정보**: Slack 채널 알림 또는 웹훅 실행 등 수행된 액션에 대한 정보.
* **결과 세부 정보**: 자동화 실행의 최종 결과에 대한 추가 정보(있는 경우). 실패한 실행의 경우 오류 정보가 포함됩니다.

사용 사례에 따라 **Registry** 또는 **Project** 탭을 선택하여 자동화 이력 보기에 대한 자세한 지침을 확인하세요.

<Tabs>
  <Tab title="Registry">
    1. 프로젝트 사이드바에서 **Registry**를 클릭하여 레지스트리로 이동합니다.
    2. 목록에서 레지스트리를 선택합니다.
    3. **Automations** 탭에서 레지스트리의 자동화를 확인합니다. **Last execution** 타임스탬프를 클릭하면 실행 이력 세부 정보를 볼 수 있습니다. 검색창을 사용하여 자동화 이름으로 필터링하거나, 마지막 트리거 날짜를 기준으로 정렬하여 최근 실행된 자동화를 찾을 수 있습니다.
    4. **Automations history** 탭에서 레지스트리의 자동화 실행 내역을 역순으로 확인합니다. 이벤트, 액션, 상태가 포함됩니다. 실행 타임스탬프를 클릭하면 특정 실행에 대한 세부 정보를 볼 수 있습니다.

    <Tip>컬렉션에 연결된 자동화 실행이 있는 경우, 아이콘 ![컬렉션 자동화 실행 기호, 오른쪽을 가리키는 화살표가 있는 원](/images/automations/collection-automation-execution-icon.png)과 함께 연결된 실행 수가 표시됩니다.</Tip>
  </Tab>

  <Tab title="Project">
    1. 프로젝트 사이드바에서 **Automations** 탭을 클릭합니다. 프로젝트의 자동화가 표시됩니다. **Last execution** 타임스탬프를 클릭하면 실행 이력 세부 정보를 볼 수 있습니다. 검색창을 사용하여 자동화 이름으로 필터링하거나, 마지막 트리거 날짜를 기준으로 정렬하여 최근 실행된 자동화를 찾을 수 있습니다.
    2. **History** 탭에서 프로젝트 자동화의 모든 실행 내역을 가장 최근 실행부터 역순으로 확인합니다. 이벤트, 액션, 상태를 포함한 각 실행의 메타데이터가 표시됩니다. 실행 타임스탬프를 클릭하면 특정 실행에 대한 세부 정보를 볼 수 있습니다.
  </Tab>
</Tabs>

<div id="understanding-execution-details">
  ## 실행 세부 정보 이해하기
</div>

각 자동화 실행에는 다음 상태 중 하나가 있습니다:

* **Finished**: 자동화가 모든 액션을 성공적으로 완료했습니다.
* **Failed**: 자동화에서 오류가 발생하여 실패로 종료되었습니다.
* **Pending**: 자동화가 실행을 위해 큐에 대기 중입니다.

이력에서 실행을 클릭하여 세부 정보를 확인하세요:

* **Event details**: 자동화를 트리거한 특정 이벤트의 세부 정보로, 다음을 포함합니다:
  * 이벤트 유형(예: &quot;New artifact version&quot;, &quot;Run completed&quot;)
  * entity 정보(run ID, 아티팩트 이름 등)
  * 이벤트를 트리거한 사용자(해당되는 경우)

* **Action details**: 자동화가 수행하려고 시도한 액션에 대한 정보:
  * 액션 유형(Slack 알림 또는 웹훅)
  * 대상(Slack 채널 또는 웹훅 URL)
  * 전송된 페이로드(웹훅의 경우)

* **Result information**:
  * Response 상태(웹훅의 경우)
  * (실패한 실행의 경우) 오류 메시지나 스택 트레이스
