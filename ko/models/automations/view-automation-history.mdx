<Note>
  이 기능을 사용하려면 엔터프라이즈 라이선스가 필요합니다. 다음 환경에서만 사용할 수 있습니다.

  * W&amp;B Multi-tenant Cloud
  * W&amp;B Dedicated Cloud
  * W&amp;B Self-Managed v0.75.0+
</Note>

이 페이지에서는 무엇이 자동화를 트리거했는지, 어떤 작업이 수행되었는지, 그리고 성공 또는 실패 여부를 포함해 W&amp;B [automations](/ko/models/automations)의 실행 이력을 보고 이해하는 방법을 설명합니다.

실행된 각 자동화은 다음 정보가 포함된 기록을 생성합니다.

* **실행 Timestamp**: 자동화이 트리거된 시점입니다.
* **트리거 이벤트**: 자동화를 트리거한 특정 이벤트입니다.
* **상태**: 실행 상태입니다. [실행 상태](#execution-status)를 참조하세요.
* **작업 세부 정보**: Slack 채널에 알림을 보내거나 웹훅을 실행하는 등 어떤 작업이 수행되었는지에 대한 정보입니다.
* **결과 세부 정보**: 있는 경우 자동화 실행의 최종 결과에 대한 추가 정보이며, 실패한 실행의 오류도 포함됩니다.

사용 사례에 따라 자동화 이력을 보는 자세한 지침은 **레지스트리** 또는 **프로젝트** 탭에서 확인하세요.

<Tabs>
  <Tab title="레지스트리">
    1. 프로젝트 사이드바에서 **레지스트리**를 클릭해 레지스트리로 이동합니다.
    2. 목록에서 레지스트리를 선택합니다.
    3. **Automations** 탭에서 레지스트리의 자동화를 확인합니다. 실행 이력 세부 정보를 보려면 **Last execution** Timestamp를 클릭합니다. 검색창을 사용해 자동화 name으로 필터링할 수 있으며, 마지막 트리거 날짜순으로 정렬해 최근 실행된 자동화를 찾을 수 있습니다.
    4. **Automations 이력** 탭에서 이벤트, 작업, 상태를 포함한 레지스트리의 자동화 실행을 최신순으로 확인합니다. 특정 실행의 세부 정보를 보려면 실행 Timestamp를 클릭합니다.

    <Tip>collection에 연결된 자동화 실행이 있으면 아이콘 ![Collection automation execution symbol, a circle with a right-pointing arrow](/images/automations/collection-automation-execution-icon.png)과 함께 연결된 실행 수가 표시됩니다.</Tip>
  </Tab>

  <Tab title="프로젝트">
    1. 프로젝트 사이드바에서 **Automations** 탭을 클릭합니다. 프로젝트의 자동화가 표시됩니다. 실행 이력 세부 정보를 보려면 **Last execution** Timestamp를 클릭합니다. 검색창을 사용해 자동화 name으로 필터링할 수 있으며, 마지막 트리거 날짜순으로 정렬해 최근 실행된 자동화를 찾을 수 있습니다.
    2. **History** 탭에서 가장 최근 실행부터 프로젝트 자동화의 모든 실행을 최신순으로 확인합니다. 각 실행의 메타데이터가 표시되며, 여기에는 이벤트, 작업, 상태가 포함됩니다. 특정 실행의 세부 정보를 보려면 실행 Timestamp를 클릭합니다.
  </Tab>
</Tabs>

<div id="understanding-execution-details">
  ## 실행 세부 정보 이해하기
</div>

각 자동화 실행은 다음 상태 중 하나를 가집니다:

* **완료됨**: 자동화가 모든 작업을 성공적으로 완료했습니다.
* **실패함**: 자동화 실행 중 오류가 발생하여 실패로 종료되었습니다.
* **대기 중**: 자동화가 실행을 위해 큐에서 대기 중입니다.

이력에서 실행 항목을 클릭하면 세부 정보를 확인할 수 있습니다:

* **이벤트 세부 정보**: 자동화를 트리거한 특정 이벤트에 대한 정보로, 다음이 포함됩니다:
  * 이벤트 유형(예: &quot;새 artifact 버전&quot;, &quot;run 완료&quot;)
  * entity 정보(run ID, artifact 이름 등)
  * 이벤트를 트리거한 사용자(해당하는 경우)

* **작업 세부 정보**: 자동화가 수행하려고 한 작업에 대한 정보:
  * 작업 유형(Slack 알림 또는 웹훅)
  * 대상(Slack 채널 또는 웹훅 URL)
  * 전송된 페이로드(웹훅의 경우)

* **결과 정보**:
  * 응답 상태(웹훅의 경우)
  * 오류 메시지 또는 스택 트레이스(실패한 실행의 경우)
