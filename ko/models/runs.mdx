title: 개요
---

*실행(run)*은 W&amp;B가 로깅하는 단일 연산 단위입니다. W&amp;B 실행은 전체 프로젝트를 구성하는 가장 작은 원자적 요소라고 생각할 수 있습니다. 다시 말해, 각 실행은 모델 학습과 결과 로깅, 하이퍼파라미터 스윕 등과 같은 특정 연산을 기록한 것입니다.

실행을 초기화하고 여기에 로깅하는 일반적인 사용 사례는 다음과 같습니다.

* 모델을 학습하고 정확도와 손실과 같은 [지표를 기록](/ko/models/ref/python/experiments/run#method-run-log)할 때
* [하이퍼파라미터 튜닝](/ko/models/sweeps/)을 수행하고 새로운 실험을 실행할 때
* 다른 모델로 새로운 머신 러닝 실험을 수행할 때
* 데이터셋과 모델을 [W&amp;B Artifacts](/ko/models/artifacts/)로 추적하고 저장할 때
* 팀의 다른 구성원이 W&amp;B Artifacts로 사용한 데이터셋이나 모델을 [다운로드하여 사용](/ko/models/artifacts/download-and-use-an-artifact/)할 때

W&amp;B 실행을 초기화하려면 W&amp;B Python SDK에서 [`wandb.init()`](/ko/models/ref/python/functions/init) 메서드를 호출하세요. 이 메서드는 새 실행을 시작하고, 해당 실행에 지표, 아티팩트 및 기타 정보를 로깅하는 데 사용할 수 있는 `wandb.Run` 객체를 반환합니다. 실행 초기화에 대한 자세한 내용은 [Initialize runs](/ko/models/runs/initialize-run)를 참조하세요.

각 실행 객체에는 [*실행 ID(run ID)*로 알려진 고유 식별자](/ko/models/runs/run-identifiers#unique-run-identifiers)가 있습니다. [고유 ID를 직접 지정](/ko/models/runs/run-identifiers#unique-run-identifiers)하거나 [W&amp;B가 무작위로 생성하도록](/ko/models/runs/run-identifiers#autogenerated-run-ids) 할 수 있습니다. 각 실행 객체에는 사람이 읽을 수 있는, 고유하지 않은 [실행 이름(run name)](/ko/models/runs/run-identifiers#run-name)도 있습니다. 실행 이름을 직접 지정할 수도 있고, W&amp;B가 무작위로 생성하도록 할 수도 있습니다. 실행을 초기화한 후에 이름을 변경할 수도 있습니다.

W&amp;B는 실행을 [*프로젝트*](/ko/models/track/project-page/)에 로깅합니다. `wandb.init(project="")`로 실행을 초기화할 때 프로젝트를 지정합니다. 해당 프로젝트가 존재하지 않으면 W&amp;B가 새 프로젝트를 생성합니다. 프로젝트가 이미 존재하면, W&amp;B는 지정한 프로젝트에 실행을 로깅합니다.

<Note>
  프로젝트 이름을 지정하지 않으면 W&amp;B는 실행을 `Uncategorized`라는 프로젝트에 저장합니다.
</Note>

`wandb.init()`는 실행의 ID, 이름, 구성, 상태와 같은 속성을 포함하는 `wandb.Run` 객체를 반환합니다. 이 실행 객체를 사용하여 `wandb.Run.log()`, `wandb.Run.log_code()`, `wandb.Run.use_artifact()`와 같은 메서드로 실행에 지표, 아티팩트, 기타 정보를 로깅할 수 있습니다.

각 실행에는 현재 실행 상태를 나타내는 상태 값이 있습니다. 가능한 실행 상태 전체 목록은 [Run states](/ko/models/runs/run-states)를 참조하세요.

W&amp;B App에서 실행이 속한 프로젝트 워크스페이스 내에서 [실행과 해당 속성 보기](/ko/models/runs/view-logged-runs)가 가능합니다. 또한 [`wandb.Api.Run`](/ko/models/ref/python/experiments/run) 객체를 사용하여 프로그래밍 방식으로 실행 속성에 접근할 수도 있습니다.

다음은 W&amp;B 실행을 초기화하고 여기에 일부 지표를 로깅하는 코드 예시입니다.

<Note>
  함께 따라 하고 싶다면 아래 코드 스니펫에서 `entity` 변수에 본인의 W&amp;B 엔터티를 전달하세요. 엔터티는 W&amp;B 사용자 이름 또는 팀 이름입니다. W&amp;B App 워크스페이스의 URL에서 확인할 수 있습니다. 예를 들어 워크스페이스 URL이 `https://wandb.ai/nico/awesome-project`라면, 엔터티는 `nico`입니다.
</Note>

```python
import wandb

entity = "nico"  # W&B 엔터티로 교체하세요
project = "awesome-project"

with wandb.init(entity=entity, project=project) as run:
    run.log({"accuracy": 0.9, "loss": 0.1})
```

첫 번째 줄은 W&amp;B Python SDK를 import합니다. 두 번째 줄은 엔터티 `nico`의 프로젝트 `awesome-project`에서 실행을 초기화합니다. 세 번째 줄은 해당 실행에 모델의 accuracy와 loss를 로깅합니다.

한 시점에서 메트릭을 한 번만 로깅하는 것은 그다지 유용하지 않을 수 있습니다. 판별 모델을 학습하는 보다 현실적인 예시는 일정한 간격으로 메트릭을 로깅하는 것입니다. 예를 들어, 다음 코드 스니펫을 살펴보십시오:

```python
import wandb
import random

config = {
    "epochs": 10,
    "learning_rate": 0.01,
}

with wandb.init(project="awesome-project", config=config) as run:
    print(f"lr: {config['learning_rate']}")
      
    # 학습 실행을 시뮬레이션합니다.
    for epoch in range(config['epochs']):
      offset = random.random() / 5
      acc = 1 - 2**-epoch - random.random() / (epoch + 1) - offset
      loss = 2**-epoch + random.random() / (epoch + 1) + offset
      print(f"epoch={epoch}, accuracy={acc}, loss={loss}")
      run.log({"accuracy": acc, "loss": loss})
```

학습 스크립트는 `wandb.Run.log()`를 10번 호출합니다. 스크립트가 `wandb.Run.log()`를 호출할 때마다 W&amp;B는 해당 에포크의 accuracy와 loss 값을 기록합니다.
