### <kbd>function</kbd> `login`

```python
login(
    key: 'str | None' = None,
    relogin: 'bool | None' = None,
    host: 'str | None' = None,
    force: 'bool | None' = None,
    timeout: 'int | None' = None,
    verify: 'bool' = False,
    referrer: 'str | None' = None,
    anonymous: 'DoNotSet' = <object object at 0x102561240>
) → bool
```

W&amp;B에 로그인합니다.

일반적으로는 이 함수를 직접 사용할 필요가 없습니다. 인증이 필요한 대부분의 W&amp;B 메서드는 암시적으로 로그인을 수행합니다. 이 함수는 `wandb login` CLI의 프로그래매틱한 대응 함수입니다.

이 함수는 세션의 전역 자격 증명을 업데이트하며(이 호출 이후 현재 Python 프로세스에서의 모든 wandb 사용에 영향을 줍니다) .netrc 파일을 업데이트할 수도 있습니다.

WANDB&#95;IDENTITY&#95;TOKEN&#95;FILE 환경 변수와 같이 identity&#95;token&#95;file 설정이 지정된 경우, 이 함수는 아무 작업도 수행하지 않습니다(no-op).

그 외의 경우, 명시적인 API 키가 제공되면 해당 키를 사용하고 시스템 .netrc 파일에 기록합니다. 키가 제공되지 않았지만 세션이 이미 인증된 상태라면, 세션 키를 사용해 검증을 수행합니다(`verify`가 True인 경우) 그리고 .netrc 파일은 업데이트하지 않습니다.

위 조건이 모두 해당되지 않으면, 다음 순서로 API 키를 가져옵니다:

* WANDB&#95;API&#95;KEY 환경 변수
* 시스템 또는 워크스페이스 설정 파일의 api&#95;key 설정
* .netrc 파일(~/.netrc, ~/&#95;netrc 또는 NETRC 환경 변수로 지정된 경로)
* 대화형 프롬프트(가능한 경우)

**Args:**

* `key`:  사용할 API 키.
* `relogin`:  true인 경우, .netrc, 환경 변수 등의 읽기를 건너뛰고 대화형 프롬프트에서 API 키를 가져옵니다.
* `host`:  연결할 W&amp;B 서버 URL.
* `force`:  true인 경우, 대화형 프롬프트에서 오프라인 모드 선택을 허용하지 않습니다.
* `timeout`:  대화형 프롬프트에서 사용자 입력을 기다리는 시간(초 단위). 이 값은 대화형이 아닌 환경에서 잘못 대화형 프롬프트가 표시되는 상황에 대한 안전장치로 사용할 수 있습니다.
* `verify`:  W&amp;B 서버에 자격 증명을 검증하고, 실패 시 AuthenticationError를 발생시킵니다.
* `referrer`:  분석용으로 URL 로그인 요청에 사용할 referrer.

**Returns:**

* `bool`:  `key`가 설정되었는지 여부.

**Raises:**

* `AuthenticationError`:  `api_key`가 서버 검증에 실패한 경우.
* `UsageError`:  `api_key`를 설정할 수 없고 tty가 없는 경우.
