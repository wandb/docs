## <kbd>class</kbd> `Runs`

프로젝트 및 선택적 필터와 연관된 `Run` 객체들의 지연 이터레이터입니다. 

필요할 때마다 W&B 서버에서 실행을 페이지 단위로 조회합니다. 

일반적으로 `Api.runs` 네임스페이스를 통해 간접적으로 사용됩니다.

### <kbd>메서드</kbd> `Runs.__init__`

```python
__init__(
    client: 'RetryingClient',
    entity: 'str',
    project: 'str',
    filters: 'dict[str, Any] | None' = None,
    order: 'str' = '+created_at',
    per_page: 'int' = 50,
    include_sweeps: 'bool' = True,
    lazy: 'bool' = True,
    api: 'public.Api | None' = None
)
```

**인수:**

* `client`:  (`wandb.apis.public.RetryingClient`) 요청에 사용할 API 클라이언트입니다.
* `entity`:  (str) 프로젝트를 소유한 엔터티(사용자 이름 또는 팀)입니다.
* `project`:  (str) 실행을 가져올 프로젝트의 이름입니다.
* `filters`:  (Optional[Dict[str, Any]]) 실행 쿼리에 적용할 필터를 담은 딕셔너리입니다.
* `order`:  (str) 정렬 기준은 `created_at`, `heartbeat_at`, `config.*.value`, `summary_metrics.*` 중 하나입니다. `order` 앞에 +를 붙이면 오름차순(기본값), -를 붙이면 내림차순입니다. 기본 정렬 기준은 가장 오래된 항목부터 최신 항목까지의 run.created&#95;at입니다.
* `per_page`:  (int) 요청당 가져올 실행 개수입니다(기본값 50).
* `include_sweeps`:  (bool) 실행에 sweep 정보를 포함할지 여부입니다. 기본값은 True입니다.

### <kbd>메서드</kbd> `Runs.histories`

```python
histories(
    samples: 'int' = 500,
    keys: 'list[str] | None' = None,
    x_axis: 'str' = '_step',
    format: "Literal['default', 'pandas', 'polars']" = 'default',
    stream: "Literal['default', 'system']" = 'default'
) → list[dict[str, Any]] | pd.DataFrame | pl.DataFrame
```

필터 조건에 맞는 모든 실행에 대해 샘플링된 히스토리 메트릭을 반환합니다.

**Args:**

* `samples`:  실행당 반환할 샘플 수
* `keys`:  특정 키에 대한 메트릭만 반환
* `x_axis`:  이 메트릭을 x축으로 사용하며, 기본값은 &#95;step입니다.
* `format`:  데이터를 반환할 형식, 옵션은 &quot;default&quot;, &quot;pandas&quot;, &quot;polars&quot;
* `stream`:  일반 메트릭에는 &quot;default&quot;, 머신 메트릭에는 &quot;system&quot;

**Returns:**

* `pandas.DataFrame`:  `format="pandas"`인 경우 히스토리 메트릭의 `pandas.DataFrame`을 반환합니다.
* `polars.DataFrame`:  `format="polars"`인 경우 히스토리 메트릭의 `polars.DataFrame`을 반환합니다.
* `list of dicts`:  `format="default"`인 경우 `run_id` 키를 포함한 히스토리 메트릭 딕셔너리의 리스트를 반환합니다.

### <kbd>메서드</kbd> `Runs.upgrade_to_full`

```python
upgrade_to_full() → None
```

이 Runs 컬렉션을 지연 모드에서 전체 모드로 업그레이드합니다.

전체 실행 데이터를 가져오도록 전환하고, 이미 로드된 Run 객체들도 전체 데이터를 포함하도록 업그레이드합니다. 여러 실행을 한꺼번에 업그레이드할 때 더 나은 성능을 위해 병렬 로딩을 사용합니다.
