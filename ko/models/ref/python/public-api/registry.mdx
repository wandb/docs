### <kbd>속성</kbd> Registry.allow_all_artifact_types

Registry에서 모든 아티팩트 유형이 허용되는지 여부를 반환합니다.

`True`이면 모든 유형의 아티팩트를 추가할 수 있습니다. `False`이면 아티팩트는 `artifact_types`에 나열된 유형으로 제한됩니다.

**반환값:**

* `bool`: `allow_all_artifact_types` 속성 값입니다.

***

### <kbd>속성</kbd> Registry.artifact_types

레지스트리에서 허용되는 아티팩트 유형을 반환합니다.

`allow_all_artifact_types`가 `True`이면 `artifact_types`는 이전에 저장되었거나 현재 레지스트리에서 사용 중인 유형을 반영합니다. `allow_all_artifact_types`가 `False`이면 아티팩트는 `artifact_types`에 있는 유형으로 제한됩니다.

**참고:**

> 이전에 저장된 아티팩트 유형은 제거할 수 없습니다.

**예시:**

```python
import wandb

registry = wandb.Api().create_registry()
registry.artifact_types.append("model")
registry.save()  # 저장 후에는 아티팩트 유형 `model`을 제거할 수 없습니다
registry.artifact_types.append("accidentally_added")
registry.artifact_types.remove(
     "accidentally_added"
)  # 저장되기 전에만 유형을 제거할 수 있습니다
```

**반환값:**

* `AddOnlyArtifactTypesList`: artifact&#95;types 속성 값입니다.

### <kbd>속성</kbd> Registry.visibility

레지스트리의 공개 범위입니다.

**반환값:**

* `Literal["organization", "restricted"]`:  공개 범위 수준입니다.
  * &quot;organization&quot;: 조직 내 모든 사용자가 이 레지스트리를 볼 수 있습니다. 이후 UI의 설정에서 이들의 역할을 수정할 수 있습니다.
  * &quot;restricted&quot;: UI를 통해 초대된 구성원만 이 레지스트리에 액세스할 수 있습니다. 공개 공유는 비활성화됩니다.

**반환값:**

* `Literal['organization', 'restricted']`: 공개 범위 속성 값입니다.

### <kbd>방법</kbd> `Registry.add_members`

```python
add_members(*members: 'User | UserMember | Team | TeamMember | str') → Self
```

이 레지스트리에 Users 또는 Teams를 추가합니다.

**매개변수:**

* `members`:  레지스트리에 추가할 Users 또는 Teams입니다. `User` 객체, `Team` 객체 또는 해당 string ID를 받을 수 있습니다.

**반환값:**
필요한 경우 추가적인 메서드 체이닝을 위해 이 레지스트리를 반환합니다.

**예외:**

* `TypeError`:  인수로 전달된 멤버가 없는 경우
* `ValueError`:  사용자 또는 팀 ID를 추론하거나 파싱할 수 없는 경우

**예시:**

```python
import wandb

api = wandb.Api()

<div id="fetch-an-existing-registry">
  # 기존 레지스트리 가져오기
</div>

registry = api.registry(name="my-registry", organization="my-org")

user1 = api.user(username="some-user")
user2 = api.user(username="other-user")
registry.add_members(user1, user2)

my_team = api.team(name="my-team")
registry.add_members(my_team)
```

### <kbd>クラスメソッド</kbd> `Registry.create`

```python
create(
    client: 'RetryingClient',
    organization: 'str',
    name: 'str',
    visibility: "Literal['organization', 'restricted']",
    description: 'str | None' = None,
    artifact_types: 'list[str] | None' = None
) → Self
```

새 레지스트리를 생성합니다.

레지스트리 이름은 조직 내에서 고유해야 합니다. 이 함수는 `api.create_registry()`를 사용해 호출해야 합니다.

**매개변수:**

* `client`:  GraphQL 클라이언트.
* `organization`:  조직 이름.
* `name`:  레지스트리 이름 (`wandb-registry-` 접두사 제외).
* `visibility`:  공개 범위 수준 (`organization` 또는 `restricted`).
* `description`:  레지스트리에 대한 선택적 설명.
* `artifact_types`:  허용되는 아티팩트 유형의 선택적 목록.

**반환값:**

* `Registry`:  새로 생성된 레지스트리 객체.

**예외:**

* `ValueError`:  동일한 이름의 레지스트리가 조직 내에 이미 존재하거나 생성에 실패한 경우.

```python
remove_members(*members: 'User | UserMember | Team | TeamMember | str') → Self

### <kbd>방법</kbd> `Registry.update_member`

```python
update_member(
    member: 'User | UserMember | Team | TeamMember | str',
    role: 'MemberRole | str'
) → Self
```

이 레지스트리 내 구성원(사용자 또는 팀)의 역할을 업데이트합니다.

**매개변수:**

* `member`:  역할을 업데이트할 사용자 또는 팀입니다. `User` 객체, `Team` 객체 또는 해당 string ID를 받을 수 있습니다.
* `role`:  구성원에게 부여할 새 역할입니다. 다음 중 하나일 수 있습니다:
  * &quot;admin&quot;
  * &quot;member&quot;
  * &quot;viewer&quot;
  * &quot;restricted&#95;viewer&quot; (W&amp;B 서버에서 지원되는 경우)

**반환값:**
필요한 경우 이후 방법 체이닝을 위해 이 레지스트리를 반환합니다.

**예외:**

* `ValueError`:  사용자 또는 팀 ID를 추론할 수 없는 경우 발생합니다.

**예시:**
레지스트리의 모든 Users를 admin으로 설정합니다. ```python
import wandb

api = wandb.Api()


<div id="fetch-an-existing-registry">
  # 기존 레지스트리 가져오기
</div>

registry = api.registry(name=&quot;my-registry&quot;, organization=&quot;my-org&quot;)

for member in registry.user&#95;members():
registry.update&#95;member(member.user, role=&quot;admin&quot;)

````
