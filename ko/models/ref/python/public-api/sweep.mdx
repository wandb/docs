## <kbd>class</kbd> `Sweep`

sweep에 연결된 run 집합입니다.

**속성:**

* `runs` (Runs):  run 목록
* `id` (str):  Sweep ID
* `project` (str):  sweep가 속한 프로젝트 이름
* `config` (dict):  sweep 설정이 포함된 사전
* `state` (str):  sweep의 상태입니다. &quot;Finished&quot;, &quot;Failed&quot;, &quot;Crashed&quot; 또는 &quot;Running&quot;일 수 있습니다.
* `expected_run_count` (int):  sweep의 예상 run 수

### <kbd>속성</kbd> Sweep.entity

sweep에 연결된 entity입니다.

**반환값:**

* `str`: entity 속성 값.

***

### <kbd>속성</kbd> Sweep.expected_run_count

Sweep의 예상 run 수를 반환합니다. run 수가 무한한 경우 `None`을 반환합니다.

**반환값:**

* `int | None`: expected&#95;run&#95;count 속성 값입니다.

***

### <kbd>속성</kbd> Sweep.name

sweep의 이름입니다.

다음 우선순위에 따라 가장 먼저 존재하는 이름을 반환합니다:

1. 사용자가 수정한 표시 이름 2. 생성 시 설정한 이름 3. Sweep ID

***

### <kbd>속성</kbd> Sweep.order

Sweep의 순서 키를 반환합니다.

***

### <kbd>속성</kbd> Sweep.path

프로젝트 경로를 반환합니다.

이 경로는 entity, 프로젝트 이름, sweep ID로 구성된 목록입니다.

***

### <kbd>속성</kbd> Sweep.url

sweep의 URL입니다.

sweep URL은 entity, 프로젝트, &quot;sweeps&quot;라는 문자열, 그리고 sweep ID.run&#95;id를 조합해 생성됩니다. SaaS 사용자의 경우 `https://wandb.ai/entity/project/sweeps/sweeps_ID` 형식을 사용합니다.

***

### <kbd>클래스 메서드</kbd> `Sweep.get`

```python
get(
    client: 'RetryingClient',
    entity: 'str | None' = None,
    project: 'str | None' = None,
    sid: 'str | None' = None,
    order: 'str | None' = None,
    query: 'Document | None' = None,
    **kwargs
)
```

클라우드 백엔드에 쿼리를 실행합니다.

**매개변수:**

* `client`:  쿼리 실행에 사용할 클라이언트입니다.
* `entity`:  프로젝트를 소유한 entity(사용자 이름 또는 팀)입니다.
* `project`:  sweep를 가져올 프로젝트 이름입니다.
* `sid`:  쿼리할 sweep ID입니다.
* `order`:  sweep의 run이 반환되는 순서입니다.
* `query`:  쿼리 실행에 사용할 쿼리입니다.
* `**kwargs`:  쿼리에 전달할 추가 키워드 매개변수입니다.
