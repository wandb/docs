## <kbd>class</kbd> `Api`

W&amp;B 서버에 질의를 보낼 때 사용하는 클래스입니다.

**예제:**

```python
import wandb

wandb.Api()
```


### <kbd>method</kbd> `Api.__init__`

```python
__init__(
    overrides: 'dict[str, Any] | None' = None,
    timeout: 'int | None' = None,
    api_key: 'str | None' = None
) → None
```

API를 초기화합니다.

**인수:**

* `overrides`: `https://api.wandb.ai`가 아닌 다른 W&amp;B 서버를 사용하는 경우 `base_url`을 설정할 수 있습니다. 또한 `entity`, `project`, `run`의 기본값을 설정할 수 있습니다.
* `using a W&B server other than `https`: //api.wandb.ai`. You can also set defaults for `entity`, `project`, and `run`.
* `timeout`: API 요청에 대한 HTTP 타임아웃(초 단위)입니다. 지정하지 않으면 기본 타임아웃이 사용됩니다.
* `api_key`: 인증에 사용할 API 키입니다. 제공하지 않으면 현재 환경 또는 설정에 있는 API 키가 사용됩니다. 환경에 제공되거나 설정된 값이 전혀 없으면 API 키 입력을 요청합니다.

### <kbd>method</kbd> `Api.artifact`

```python
artifact(name: 'str', type: 'str | None' = None)
```

단일 아티팩트를 반환합니다.

**인수:**

* `name`:  아티팩트의 이름입니다. 아티팩트 이름은 파일 경로와 유사한 형태로, 최소한 아티팩트가 로깅된 프로젝트 이름, 아티팩트 이름, 아티팩트의 버전 또는 별칭으로 구성됩니다. 선택적으로, 아티팩트를 로깅한 엔터티를 접두사로 추가하고 그 뒤에 슬래시(`/`)를 붙일 수 있습니다. 이름에 엔터티가 지정되지 않은 경우, 실행 또는 API 설정의 엔터티가 사용됩니다.
* `type`:  가져올 아티팩트의 유형입니다.

**반환값:**
`Artifact` 객체.

**예외:**

* `ValueError`:  아티팩트 이름이 지정되지 않은 경우.
* `ValueError`:  아티팩트 유형이 지정되었지만 가져온 아티팩트의 유형과 일치하지 않는 경우.

**예시:**
다음 코드 스니펫에서 &quot;entity&quot;, &quot;project&quot;, &quot;artifact&quot;, &quot;version&quot;, &quot;alias&quot;는 각각 W&amp;B 엔터티, 아티팩트가 포함된 프로젝트 이름, 아티팩트 이름, 아티팩트의 버전과 별칭에 대한 플레이스홀더입니다.

```python
import wandb

# 프로젝트, 아티팩트 이름, 아티팩트 별칭 지정
wandb.Api().artifact(name="project/artifact:alias")

# 프로젝트, 아티팩트 이름, 특정 아티팩트 버전 지정
wandb.Api().artifact(name="project/artifact:version")

# 엔터티, 프로젝트, 아티팩트 이름, 아티팩트 별칭 지정
wandb.Api().artifact(name="entity/project/artifact:alias")

# 엔터티, 프로젝트, 아티팩트 이름, 특정 아티팩트 버전 지정
wandb.Api().artifact(name="entity/project/artifact:version")
```

**참고:**

> 이 메서드는 외부에서만 사용하도록 설계되었습니다. wandb 리포지토리 코드 내에서는 `api.artifact()`를 호출하지 마십시오.

### <kbd>메서드</kbd> `Api.artifact_collection`

```python
artifact_collection(type_name: 'str', name: 'str') → ArtifactCollection
```

지정된 타입의 단일 아티팩트 컬렉션을 반환합니다.

반환된 `ArtifactCollection` 객체를 사용해 해당 컬렉션에 포함된 특정 아티팩트에 대한 정보 등을 조회할 수 있습니다.

**Args:**

* `type_name`:  가져올 아티팩트 컬렉션의 타입.
* `name`:  아티팩트 컬렉션 이름. 선택적으로, 아티팩트를 로깅한 엔터티를 접두사로 붙이고 그 뒤에 슬래시를 추가할 수 있습니다.

**Returns:**
`ArtifactCollection` 객체.

**Examples:**
아래 코드 스니펫에서 &quot;type&quot;, &quot;entity&quot;, &quot;project&quot;, &quot;artifact&#95;name&quot;은 각각 컬렉션 타입, 사용자의 W&amp;B 엔터티, 아티팩트가 포함된 프로젝트 이름, 아티팩트 이름을 나타내는 플레이스홀더입니다.

```python
import wandb

collections = wandb.Api().artifact_collection(
    type_name="type", name="entity/project/artifact_name"
)

# 컬렉션에서 첫 번째 아티팩트를 가져옵니다.
artifact_example = collections.artifacts()[0]

# 아티팩트의 내용을 지정된 루트 디렉토리에 다운로드합니다.
artifact_example.download()
```

### <kbd>메서드</kbd> `Api.artifact_collection_exists`

```python
artifact_collection_exists(name: 'str', type: 'str') → bool
```

지정된 프로젝트와 엔터티 내에 아티팩트 컬렉션이 존재하는지 여부.

**Args:**

* `name`:  아티팩트 컬렉션 이름. 필요에 따라 아티팩트를 로깅한 entity를 접두사로 추가하고, 그 뒤에 슬래시(`/`)를 붙일 수 있음. entity 또는 project가 지정되지 않은 경우, 존재한다면 override 파라미터에서 컬렉션을 추론함. 그렇지 않으면 entity는 사용자 설정에서 가져오고 project는 기본적으로 &quot;uncategorized&quot;로 설정됨.
* `type`:  아티팩트 컬렉션의 유형.

**Returns:**
아티팩트 컬렉션이 존재하면 True, 그렇지 않으면 False를 반환.

**Examples:**
다음 코드 스니펫에서 &quot;type&quot;과 &quot;collection&#95;name&quot;은 각각 아티팩트 컬렉션의 유형과 컬렉션 이름을 나타냄.

```python
import wandb

wandb.Api.artifact_collection_exists(type="type", name="collection_name")
```

### <kbd>메서드</kbd> `Api.create_automation`

```python
create_automation(
    obj: 'NewAutomation',
    fetch_existing: 'bool' = False,
    **kwargs: 'Unpack[WriteAutomationsKwargs]'
) → Automation
```

새 Automation을 생성합니다.

**Args:**
obj:  생성할 Automation입니다.  fetch&#95;existing:  True이고 충돌하는 Automation이 이미 존재하는 경우, 에러를 발생시키는 대신 기존 Automation을 가져오려고 시도합니다.  **kwargs:  Automation을 생성하기 전에 할당할 추가 값들입니다.  지정된 경우, 이미 Automation에 설정되어 있을 수 있는 값들을 다음과 같이 덮어씁니다:

* `name`: Automation의 이름.
  * `description`: Automation의 설명.
  * `enabled`: Automation이 활성화되었는지 여부.
  * `scope`: Automation의 범위.
  * `event`: Automation을 트리거하는 이벤트.
  * `action`: Automation에 의해 트리거되는 액션.

**Returns:**
저장된 Automation.

**Examples:**
특정 프로젝트 내의 실행이 사용자 정의 임계값을 초과하는 메트릭을 기록할 때 Slack 알림을 보내는, 이름이 &quot;my-automation&quot;인 새 Automation을 생성합니다:

```python
import wandb
from wandb.automations import OnRunMetric, RunEvent, SendNotification

api = wandb.Api()

project = api.project("my-project", entity="my-team")

# 팀의 첫 번째 Slack 통합 사용
slack_hook = next(api.slack_integrations(entity="my-team"))

event = OnRunMetric(
     scope=project,
     filter=RunEvent.metric("custom-metric") > 10,
)
action = SendNotification.from_integration(slack_hook)

automation = api.create_automation(
     event >> action,
     name="my-automation",
     description="Send a Slack message whenever 'custom-metric' exceeds 10.",
)
```

### <kbd>메서드</kbd> `Api.create_custom_chart`

```python
create_custom_chart(
    entity: 'str',
    name: 'str',
    display_name: 'str',
    spec_type: "Literal['vega2']",
    access: "Literal['private', 'public']",
    spec: 'str | dict'
) → str
```

사용자 지정 차트 프리셋을 생성하고 해당 ID를 반환합니다.

**Args:**

* `entity`:  차트를 소유하는 엔터티(사용자 또는 팀)
* `name`:  차트 프리셋의 고유 식별자
* `display_name`:  UI에 표시되는 사람이 이해하기 쉬운 이름
* `spec_type`:  명세 유형. Vega-Lite v2 명세의 경우 반드시 &quot;vega2&quot;여야 합니다.
* `access`:  차트에 대한 액세스 수준:
  * &quot;private&quot;: 차트를 생성한 엔터티만 액세스 가능
  * &quot;public&quot;: 차트를 공개적으로 액세스 가능
* `spec`:  딕셔너리 또는 JSON 문자열 형태의 Vega/Vega-Lite 명세

**Returns:**
&quot;entity/name&quot; 형식의 생성된 차트 프리셋 ID

**Raises:**

* `wandb.Error`:  차트 생성에 실패한 경우
* `UnsupportedError`:  서버가 사용자 지정 차트를 지원하지 않는 경우

**Example:**

```python
    import wandb

    api = wandb.Api()

    # 간단한 막대 차트 사양 정의
    vega_spec = {
         "$schema": "https://vega.github.io/schema/vega-lite/v6.json",
         "mark": "bar",
```

### <kbd>메서드</kbd> `Api.registries`

```python
registries(
    organization: 'str | None' = None,
    filter: 'dict[str, Any] | None' = None,
    per_page: 'int' = 100
) → Registries
```

`Registry` 객체의 지연(lazy) 이터레이터를 반환합니다.

이 이터레이터를 사용해 조직의 registry 전체에서 registry, collection 또는 아티팩트 버전을 검색하고 필터링할 수 있습니다.

**Args:**

* `organization`:  (str, optional) 가져올 registry가 속한 organization입니다. 지정하지 않으면 사용자 설정에 지정된 organization을 사용합니다.
* `filter`:  (dict, optional) 지연 registry 이터레이터의 각 객체에 적용할 MongoDB 스타일 필터입니다. registry에 대해 필터링할 수 있는 필드는 `name`, `description`, `created_at`, `updated_at`입니다. collection에 대해 필터링할 수 있는 필드는 `name`, `tag`, `description`, `created_at`, `updated_at`입니다. version에 대해 필터링할 수 있는 필드는 `tag`, `alias`, `created_at`, `updated_at`, `metadata`입니다.
* `per_page`:  쿼리 페이지네이션을 위한 페이지 크기를 설정합니다.

**Returns:**
`Registry` 객체의 지연(lazy) 이터레이터.

**Examples:**
이름에 &quot;model&quot;이 포함된 모든 registry 찾기

```python
import wandb

api = wandb.Api()  # 엔터티가 여러 조직에 속한 경우 조직을 지정하세요
api.registries(filter={"name": {"$regex": "model"}})
```

레지스트리에서 이름이 &quot;my&#95;collection&quot;이고 태그가 &quot;my&#95;tag&quot;인 모든 컬렉션을 찾습니다.

```python
api.registries().collections(filter={"name": "my_collection", "tag": "my_tag"})
```

레지스트리에서 컬렉션 이름에 &quot;my&#95;collection&quot;이 포함되고 해당 버전에 별칭 &quot;best&quot;가 지정된 모든 아티팩트 버전을 찾습니다

```python
api.registries().collections(
    filter={"name": {"$regex": "my_collection"}}
).versions(filter={"alias": "best"})
```

레지스트리에서 &quot;model&quot;을 포함하며 태그가 &quot;prod&quot;이거나 별칭이 &quot;best&quot;인 모든 아티팩트 버전을 찾습니다

```python
api.registries(filter={"name": {"$regex": "model"}}).versions(
    filter={"$or": [{"tag": "prod"}, {"alias": "best"}]}
)
```

* `tags`: 실행에 연결된 태그입니다.
* `username`: 실행을 시작한 사용자의 사용자 이름입니다.

또한, 실행 config 항목이나 summary metric으로도 필터링할 수 있습니다. 예를 들어 `config.experiment_name`, `summary_metrics.loss` 등입니다.

더 복잡한 필터링을 위해서는 MongoDB 쿼리 연산자를 사용할 수 있습니다. 자세한 내용은 https://docs.mongodb.com/manual/reference/operator/query 를 참조하세요. 다음 연산들이 지원됩니다:

* `$and`
* `$or`
* `$nor`
* `$eq`
* `$ne`
* `$gt`
* `$gte`
* `$lt`
* `$lte`
* `$in`
* `$nin`
* `$exists`
* `$regex`

**인자(Args):**

* `path`:  (str) 프로젝트에 대한 path로, &quot;entity/project&quot; 형식이어야 합니다.
* `filters`:  (dict) MongoDB 쿼리 언어를 사용해 특정 실행을 조회하기 위한 쿼리입니다. config.key, summary&#95;metrics.key, state, entity, createdAt 등의 실행 속성으로 필터링할 수 있습니다.
* `For example`:  `{"config.experiment_name": "foo"}` 는 experiment name config 항목이 &quot;foo&quot;로 설정된 실행을 찾습니다.
* `order`:  (str) 정렬 기준으로, `created_at`, `heartbeat_at`, `config.*.value`, 또는 `summary_metrics.*` 를 사용할 수 있습니다. order 앞에 +를 붙이면 오름차순(기본값)이고, -를 붙이면 내림차순입니다. 기본 정렬은 가장 오래된 것부터 최신 순의 run.created&#95;at 입니다.
* `per_page`:  (int) 쿼리 페이지네이션을 위한 페이지 크기를 설정합니다.
* `include_sweeps`:  (bool) 결과에 sweep 실행을 포함할지 여부입니다.
* `lazy`:  (bool) 더 빠른 성능을 위해 지연 로딩을 사용할지 여부입니다. True(기본값)인 경우, 초기에는 필수 실행 메타데이터만 로드합니다. config, summaryMetrics, systemMetrics 같은 무거운 필드는 접근 시점에 온디맨드로 로드됩니다. 전체 데이터를 미리 로드하려면 False로 설정하세요.

**반환값(Returns):**
`Run` 객체들의 반복 가능한 컬렉션인 `Runs` 객체를 반환합니다.

**예시(Examples):**

```python
import wandb
from wandb.apis.public import Api

# config.experiment_name이 "foo"로 설정된 프로젝트의 실행 찾기
Api.runs(path="my_entity/project", filters={"config.experiment_name": "foo"})
```

```python
# config.experiment_name이 "foo" 또는 "bar"로 설정된 프로젝트의 실행 찾기
Api.runs(
    path="my_entity/project",
    filters={
         "$or": [
             {"config.experiment_name": "foo"},
             {"config.experiment_name": "bar"},
         ]
    },
)
```

```python
# config.experiment_name이 정규식과 일치하는 프로젝트의 실행 찾기
# (앵커는 지원되지 않음)
Api.runs(
    path="my_entity/project",
    filters={"config.experiment_name": {"$regex": "b.*"}},
)
```

```python
# 실행 이름이 정규식과 일치하는 프로젝트의 실행 찾기
# (앵커는 지원되지 않음)
Api.runs(
    path="my_entity/project", filters={"display_name": {"$regex": "^foo.*"}}
)
```

```python
# 손실값 오름차순으로 정렬된 프로젝트의 실행 찾기
Api.runs(path="my_entity/project", order="+summary_metrics.loss")
```
