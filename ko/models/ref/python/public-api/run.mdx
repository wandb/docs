## <kbd>class</kbd> `Run`

특정 엔터티와 프로젝트에 연결된 단일 실행입니다.

### <kbd>method</kbd> `Run.__init__`

```python
__init__(
    client: 'RetryingClient',
    entity: 'str',
    project: 'str',
    run_id: 'str',
    attrs: 'Mapping | None' = None,
    include_sweeps: 'bool' = True,
    lazy: 'bool' = True,
    api: 'public.Api | None' = None
)
```

**인자:**

* `client`:  W&amp;B API 클라이언트.
* `entity`:  실행과 연결된 엔터티.
* `project`:  실행과 연결된 프로젝트.
* `run_id`:  실행의 고유 식별자.
* `attrs`:  실행의 속성.
* `include_sweeps`:  실행에 스윕을 포함할지 여부.

**속성:**

* `tags` ([str]):  실행과 연결된 태그 목록
* `url` (str):  이 실행의 URL
* `id` (str):  실행의 고유 식별자(기본값은 8자)
* `name` (str):  실행의 이름
* `state` (str):  다음 중 하나: running, finished, crashed, killed, preempting, preempted
* `config` (dict):  실행과 연결된 하이퍼파라미터 딕셔너리
* `created_at` (str):  실행이 시작된 시점의 ISO 타임스탬프
* `system_metrics` (dict):  실행에 대해 기록된 최신 시스템 메트릭
* `summary` (dict):  현재 요약 정보를 보관하는 변경 가능한 딕셔너리 유사 속성. update를 호출하면 변경 사항이 영구 저장됩니다.
* `project` (str):  실행과 연결된 프로젝트
* `entity` (str):  실행과 연결된 엔터티 이름
* `project_internal_id` (int):  프로젝트의 내부 ID
* `user` (str):  실행을 생성한 사용자 이름
* `path` (str):  고유 식별자 [entity]/[project]/[run&#95;id]
* `notes` (str):  실행에 대한 노트
* `read_only` (boolean):  실행이 편집 가능한지 여부
* `history_keys` (str):  `wandb.Run.log({"key": "value"})`로 로깅된 히스토리 메트릭 키
* `metadata` (str):  wandb-metadata.json에 포함된 실행 메타데이터

Run 객체를 초기화합니다.

Run은 항상 wandb.Api의 인스턴스인 api에서 api.runs()를 호출하여 초기화됩니다.

### <kbd>method</kbd> `Run.beta_scan_history`

```python
beta_scan_history(
    keys: 'list[str] | None' = None,
    page_size: 'int' = 1000,
    min_step: 'int' = 0,
    max_step: 'int | None' = None,
    use_cache: 'bool' = True
) → public.BetaHistoryScan
```

실행에 대한 모든 history 레코드의 iterable 컬렉션을 반환합니다.

이 함수는 아직 개발 중인 기능이며, 예상대로 동작하지 않을 수 있습니다. 이 함수는 wandb-core를 사용해 실행에서 내보낸 Parquet 형식의 history를 로컬에서 읽어옵니다.

**Args:**

* `keys`:  실행 history에서 읽을 메트릭 목록입니다.  `keys`를 제공하지 않으면 모든 메트릭이 반환됩니다.
* `page_size`:  한 번에 읽을 history 레코드 개수입니다.
* `min_step`:  history를 읽기 시작할 최소 step입니다(포함).
* `max_step`:  history를 읽을 최대 step입니다(미포함).
* `use_cache`:  True로 설정하면 WANDB&#95;CACHE&#95;DIR에서 실행 history가 있는지 확인합니다.  캐시에 실행 history가 없으면 서버에서 다운로드합니다.  False로 설정하면 실행 history를 매번 서버에서 다운로드합니다.

**Returns:**
history 레코드를 얻기 위해 iterator로 사용할 수 있는 BetaHistoryScan 객체를 반환합니다.

### <kbd>classmethod</kbd> `Run.create`

```python
create(
    api: 'public.Api',
    run_id: 'str | None' = None,
    project: 'str | None' = None,
    entity: 'str | None' = None,
    state: "Literal['running', 'pending']" = 'running'
) → Self
```

지정된 프로젝트에 대한 실행을 생성합니다.

대부분의 경우에는 `wandb.init()`을 사용하십시오. `wandb.init()`은 실행을 생성하고 업데이트하기 위한 더 견고한 로직을 제공합니다. `wandb.apis.public.Run.create`는 (예: GPU가 부족하거나 경합이 심한 Kubernetes 클러스터에서) 스케줄링되지 못할 수 있는 작업을 위해 실행을 &quot;pending&quot; 상태로 생성하는 것과 같은 특정 시나리오를 위한 것입니다. 이렇게 보류 중인 실행은 나중에 재개하여 W&amp;B에서 추적할 수 있습니다.

이 메서드로 생성된 실행은 기능이 제한적입니다. 이 방식으로 생성된 실행에서 `update()`를 호출하면 예상과 다르게 동작할 수 있습니다.

**Args:**

* `api`:  W&amp;B API 인스턴스.
* `run_id`:  선택적 실행 ID. 제공하지 않으면 임의의 ID가 생성됩니다.
* `project`:  선택적 프로젝트 이름. API 설정의 프로젝트 또는 &quot;uncategorized&quot;가 기본값입니다.
* `entity`:  선택적 엔터티(사용자 또는 팀) 이름.
* `state`:  실행의 초기 상태입니다. 나중에 재개할 실행에는 &quot;pending&quot;을, 즉시 실행할 경우에는 &quot;running&quot;을 사용하십시오.

**Returns:**
생성된 실행을 나타내는 Run 객체.

**Example:**
나중에 실행하기 위해 보류 중인 실행 생성

```python
import wandb

api = wandb.Api()

run_name = "my-pending-run"

run = Run.create(
    api=api,
    project="project",
    entity="entity",
    state="pending",
    run_id=run_name,
)
```

### <kbd>메서드</kbd> `Run.logged_artifacts`

```python
logged_artifacts(per_page: 'int' = 100) → public.RunArtifacts
```

이 실행에서 기록된 모든 아티팩트를 가져옵니다.

실행 동안 기록된 모든 출력 아티팩트를 가져옵니다. 페이지네이션된 결과를 반환하며, 순회(iteration)하거나 하나의 리스트로 모을 수 있습니다.

**인자:**

* `per_page`:  API 요청당 가져올 아티팩트 개수입니다.

**반환값:**
이 실행 동안 출력으로 기록된 모든 Artifact 객체의 이터러블 컬렉션입니다.

**예시:**

```python
import wandb
import tempfile

with tempfile.NamedTemporaryFile(mode="w", delete=False, suffix=".txt") as tmp:
    tmp.write("This is a test artifact")
    tmp_path = tmp.name
run = wandb.init(project="artifact-example")
artifact = wandb.Artifact("test_artifact", type="dataset")
artifact.add_file(tmp_path)
run.log_artifact(artifact)
run.finish()

api = wandb.Api()

finished_run = api.run(f"{run.entity}/{run.project}/{run.id}")

for logged_artifact in finished_run.logged_artifacts():
    print(logged_artifact.name)
```

* `keys` ([str], optional):  이 키들만 조회하며, 모든 키가 정의된 행만 가져옵니다.
* `page_size` (int, optional):  API에서 가져올 페이지 크기입니다.
* `min_step` (int, optional):  한 번에 스캔할 최소 페이지 수입니다.
* `max_step` (int, optional):  한 번에 스캔할 최대 페이지 수입니다.

**Returns:**
히스토리 레코드(딕셔너리)에 대한 반복 가능한 컬렉션입니다.

**Example:**
예제 실행의 모든 loss 값을 내보냅니다.

```python
run = api.run("entity/project-name/run-id")
history = run.scan_history(keys=["Loss"])
losses = [row["Loss"] for row in history]
```

### <kbd>메서드</kbd> `Run.used_artifacts`

```python
used_artifacts(per_page: 'int' = 100) → public.RunArtifacts
```

이 실행에서 명시적으로 사용된 아티팩트를 가져옵니다.

실행 중에 일반적으로 `run.use_artifact()`를 통해 명시적으로 사용되었다고 선언된 입력 아티팩트만 가져옵니다. 페이지네이션된 결과를 반환하며, 이는 반복(iteration)에 사용하거나 하나의 리스트로 모을 수 있습니다.

**Args:**

* `per_page`:  API 요청당 가져올 아티팩트 개수.

**Returns:**
이 실행에서 입력으로 명시적으로 사용된 Artifact 객체의 반복(iteration)에 사용할 수 있는 컬렉션.

**Example:**

```python
import wandb

run = wandb.init(project="artifact-example")
run.use_artifact("test_artifact:latest")
run.finish()

api = wandb.Api()
finished_run = api.run(f"{run.entity}/{run.project}/{run.id}")
for used_artifact in finished_run.used_artifacts():
    print(used_artifact.name)
test_artifact
```
