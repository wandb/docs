W&amp;B의 글로벌 함수는 `wandb.init()` 또는 `wandb.login()`처럼 직접 호출하는 최상위(top-level) 함수입니다. 특정 클래스에 속하는 메서드와 달리, 이 함수들은 객체를 먼저 인스턴스화할 필요 없이 W&amp;B의 핵심 기능에 직접 접근할 수 있도록 해 줍니다.

<div id="available-functions">
  ## 사용 가능한 함수
</div>

| Function | Description |
|----------|-------------|
| [`init()`](/ko/models/ref/python/functions/init) | 새 실행을 시작하고 W&amp;B에 추적 및 로깅합니다. 일반적으로 ML 학습 파이프라인에서 가장 먼저 호출하는 함수입니다. |
| [`login()`](/ko/models/ref/python/functions/login) | 플랫폼에서 머신을 인증하기 위한 W&amp;B 로그인 자격 증명을 설정합니다. |
| [`setup()`](/ko/models/ref/python/functions/setup) | 현재 프로세스와 자식 프로세스에서 W&amp;B를 사용할 수 있도록 준비합니다. 멀티 프로세스 애플리케이션에 유용합니다. |
| [`teardown()`](/ko/models/ref/python/functions/teardown) | W&amp;B 리소스를 정리하고 백엔드 프로세스를 종료합니다. |
| [`sweep()`](/ko/models/ref/python/functions/sweep) | 최적의 모델 구성을 찾기 위한 하이퍼파라미터 스윕을 초기화합니다. |
| [`agent()`](/ko/models/ref/python/functions/agent) | 하이퍼파라미터 최적화 실험을 실행할 스윕 에이전트를 생성합니다. |
| [`controller()`](/ko/models/ref/python/functions/controller) | 스윕 에이전트와 그 실행을 관리하고 제어합니다. |
| [`restore()`](/ko/models/ref/python/functions/restore) | 이전 실행 또는 실험 상태를 복원하여 작업을 재개합니다. |
| [`finish()`](/ko/models/ref/python/functions/finish) | 실행을 종료하고 리소스를 정리합니다. |

<div id="example">
  ## 예시
</div>

가장 일반적인 워크플로는 W&amp;B에 인증한 다음 실행을 초기화하고, 학습 루프에서 정확도와 손실 같은 값을 로깅하는 것부터 시작합니다. 첫 단계는 `wandb`를 임포트하고 전역 함수 `login()`과 `init()`을 사용하는 것입니다:

```python
import wandb

# W&B 인증
wandb.login()

# 하이퍼파라미터 및 메타데이터
config = {
   "learning_rate": 0.01,
   "epochs": 10,
}

# 실행이 기록될 프로젝트
project = "my-awesome-project"

# 새 실행 초기화
with wandb.init(project=project, config=config) as run:
   # 여기에 학습 코드를 작성하세요...
   
   # W&B에 값 기록
   run.log({"accuracy": 0.9, "loss": 0.1})
```
