이 페이지에서는 W&amp;B SDK가 추적하는 시스템 메트릭에 대해 자세히 설명합니다.

<Note>
  `wandb`는 15초마다 시스템 메트릭을 자동으로 로깅합니다.
</Note>

<div id="view-system-metrics">
  ## 시스템 메트릭 보기
</div>

W&amp;B App 또는 `wandb beta leet` 터미널 UI를 사용하여 시스템 메트릭을 보고 모니터링할 수 있습니다.

<Tabs>
  <Tab title="App">
    W&amp;B App에서 시스템 메트릭을 보려면 다음 단계를 따르세요.

    1. W&amp;B App에서 프로젝트로 이동합니다.
    2. **Runs** 테이블에서 run을 선택합니다.
    3. Workspace에서 다음 차트를 표시하는 **System** 섹션을 찾습니다.
       * GPU 사용량 및 메모리
       * CPU 사용량
       * 메모리 사용량
       * 디스크 I/O
       * 네트워크 트래픽

    Workspace에 패널을 추가하여 표시할 시스템 메트릭을 사용자 지정할 수 있습니다. 시각화를 만들고 사용자 지정하는 방법에 대한 자세한 내용은 [Panels](/ko/models/app/features/panels/)를 참조하세요.
  </Tab>

  <Tab title="LEET">
    `wandb beta leet` 터미널 UI를 사용하여 터미널에서 run의 시스템 메트릭을 보려면 다음 단계를 따르세요.

    1. 스크립트에서 run을 로컬로 시작했다면, 코드를 실행한 디렉터리로 이동합니다. 이 디렉터리에는 run별 하위 디렉터리가 있는 `wandb/` 디렉터리와 `latest-run/` 심볼릭 링크가 있습니다. 각 run 디렉터리에는 `run-<run-ID>.wandb` 형식의 이름을 가진 트랜잭션 로그가 들어 있습니다.

       run을 로컬에서 시작하지 않고 대신 `.wandb` 트랜잭션 로그 파일을 다운로드했다면, 해당 파일의 위치를 메모해 두세요.
    2. 다음 명령 중 하나로 `wandb beta leet`를 시작합니다.

       ```bash
       # ./wandb/latest-run/에 저장된 최신 run 보기
       wandb beta leet

       # run 디렉터리 지정
       wandb beta leet ./wandb/run-20250813_124246-n67z9ude

       # .wandb 파일 지정
       wandb beta leet ./wandb/run-20250813_124246-n67z9ude/run-n67z9ude.wandb
       ```

    ![LEET 터미널 UI에서 run을 보는 방법을 보여주는 스크린샷](/images/leet/leet-tui.png)

    LEET는 **오른쪽 사이드바**에 시스템 메트릭을 표시하며, 다음 정보를 보여줍니다.

    * GPU 사용량(%) 및 메모리 사용량(GB)
    * CPU 사용량
    * RAM 사용량(GB)
    * 디스크 I/O
    * 네트워크 활동

    다음 키보드 단축키로 시작해 보세요.

    * `h` 또는 `?` - 모든 키보드 단축키 보기
    * `/` - 패턴으로 메트릭 필터링
    * `[` / `]` - 왼쪽/오른쪽 사이드바 표시/숨기기
    * `n` / `N` - 메트릭 페이지 간 이동
    * `q` / `CMD+C` - 종료

    자세한 내용은 [`wandb beta leet`](/ko/models/ref/cli/wandb-beta/wandb-beta-leet)를 참조하세요.
  </Tab>
</Tabs>

<div id="cpu">
  ## CPU
</div>

<div id="process-cpu-percent-cpu">
  ### 프로세스 CPU 사용률 (CPU)
</div>

프로세스의 CPU 사용 비율로, 사용 가능한 CPU 개수로 정규화한 값입니다.

W&amp;B는 이 메트릭에 `cpu` 태그를 부여합니다.

<div id="process-cpu-threads">
  ### 프로세스 CPU 스레드
</div>

프로세스가 사용하는 스레드 수입니다.

W&amp;B는 이 메트릭에 `proc.cpu.threads` 태그를 부여합니다.

<div id="disk">
  ## 디스크
</div>

기본적으로 `/` 경로의 사용 메트릭이 수집됩니다. 모니터링할 경로를 설정하려면 다음 설정을 사용하십시오:

```python
run = wandb.init(
    settings=wandb.Settings(
        x_stats_disk_paths=("/System/Volumes/Data", "/home", "/mnt/data"),
    ),
)
```

<div id="disk-usage-percent">
  ### 디스크 사용량 백분율
</div>

지정된 경로의 전체 시스템 디스크 사용량을 백분율로 나타냅니다.

W&amp;B는 이 메트릭에 `disk.{path}.usagePercent` 태그를 부여합니다.

<div id="disk-usage">
  ### 디스크 사용량
</div>

지정된 경로에 대한 전체 시스템 디스크 사용량을 기가바이트(GB) 단위로 나타냅니다.

액세스 가능한 경로를 샘플링하고, 각 경로의 디스크 사용량(GB)을 샘플에 추가합니다.

W&amp;B는 이 메트릭에 `disk.{path}.usageGB` 태그를 부여합니다.

<div id="disk-in">
  ### 디스크 In
</div>

메가바이트(MB) 단위의 총 시스템 디스크 조회량을 나타냅니다.
초기 디스크 조회 바이트는 첫 번째 샘플을 수집할 때 기록됩니다. 이후 샘플에서는 현재 조회 바이트와 초기값의 차이를 계산합니다.

W&amp;B는 이 메트릭에 `disk.in` 태그를 부여합니다.

<div id="disk-out">
  ### 디스크 Out
</div>

시스템 전체 디스크 쓰기량을 메가바이트(MB) 단위로 나타냅니다.
[디스크 In]()과 마찬가지로, 초기 디스크 쓰기 바이트 값은 첫 샘플이 수집될 때 기록됩니다. 이후 샘플에서는 현재 쓰기 바이트 값과 초기 값의 차이를 계산합니다.

W&amp;B는 이 메트릭에 `disk.out` 태그를 부여합니다.

<div id="memory">
  ## 메모리
</div>

<div id="process-memory-rss">
  ### 프로세스 메모리 RSS
</div>

프로세스의 상주 집합 크기(Resident Set Size, RSS)를 메가바이트(MB) 단위로 나타냅니다. RSS는 프로세스가 사용하는 메모리 중 주 메모리(RAM)에 상주하는 부분을 의미합니다.

W&amp;B는 이 메트릭에 `proc.memory.rssMB` 태그를 부여합니다.

<div id="process-memory-percent">
  ### 프로세스 메모리 백분율
</div>

프로세스의 메모리 사용량을 전체 사용 가능한 메모리 대비 백분율로 나타냅니다.

W&amp;B는 이 메트릭에 `proc.memory.percent` 태그를 부여합니다.

<div id="memory-percent">
  ### 메모리 비율
</div>

시스템 전체 메모리 사용량을 사용 가능한 총 메모리 대비 백분율로 나타냅니다.

W&amp;B는 이 메트릭에 `memory_percent` 태그를 부여합니다.

<div id="memory-available">
  ### 사용 가능한 메모리
</div>

메가바이트(MB) 단위로 사용 가능한 시스템 메모리의 총량을 나타냅니다.

W&amp;B는 이 메트릭에 `proc.memory.availableMB` 태그를 부여합니다.

{/* 새 섹션 */}

<div id="network">
  ## 네트워크
</div>

<div id="network-sent">
  ### 네트워크 Sent
</div>

네트워크를 통해 전송된 총 바이트 수를 나타냅니다.
메트릭이 처음 초기화될 때 초기 전송 바이트 수가 기록됩니다. 이후 샘플에서는 현재 전송된 바이트 수와 초기값의 차이를 계산합니다.

W&amp;B는 이 메트릭에 `network.sent` 태그를 부여합니다.

<div id="network-received">
  ### 네트워크 수신
</div>

네트워크를 통해 수신된 총 바이트 수를 나타냅니다.
[Network Sent]()와 비슷하게, 메트릭이 처음 초기화될 때 초기 수신 바이트 수가 기록됩니다. 이후 샘플에서는 현재 수신 바이트 수와 초기 값의 차이를 계산합니다.

W&amp;B는 이 메트릭에 `network.recv` 태그를 부여합니다.

{/* 새 섹션 */}

<div id="nvidia-gpu">
  ## NVIDIA GPU
</div>

아래에 설명된 메트릭 외에도 프로세스 및/또는 그 자식 프로세스가 특정 GPU를 사용하는 경우, W&amp;B는 해당 메트릭을 `gpu.process.{gpu_index}.{metric_name}` 형식으로 캡처합니다.

<div id="gpu-memory-utilization">
  ### GPU 메모리 사용량
</div>

각 GPU의 GPU 메모리 사용량을 백분율로 나타냅니다.

W&amp;B는 이 메트릭에 `gpu.{gpu_index}.memory` 태그를 부여합니다.

<div id="gpu-memory-allocated">
  ### GPU 메모리 할당
</div>

각 GPU에서 전체 사용 가능한 메모리 대비 할당된 GPU 메모리의 비율을 나타냅니다.

W&amp;B는 이 메트릭에 `gpu.{gpu_index}.memoryAllocated` 태그를 부여합니다.

<div id="gpu-memory-allocated-bytes">
  ### GPU 메모리 할당 바이트 수
</div>

각 GPU에 대해 할당된 GPU 메모리 양을 바이트 단위로 나타냅니다.

W&amp;B는 이 메트릭에 `gpu.{gpu_index}.memoryAllocatedBytes` 태그를 부여합니다.

<div id="gpu-utilization">
  ### GPU 사용량
</div>

각 GPU의 GPU 사용량을 %로 나타냅니다.

W&amp;B는 이 메트릭에 `gpu.{gpu_index}.gpu` 태그를 부여합니다.

<div id="gpu-temperature">
  ### GPU 온도
</div>

각 GPU의 섭씨 온도입니다.

W&amp;B는 이 메트릭에 `gpu.{gpu_index}.temp` 태그를 부여합니다.

<div id="gpu-power-usage-watts">
  ### GPU 전력 사용량(와트)
</div>

각 GPU의 전력 사용량(와트 단위)을 나타냅니다.

W&amp;B는 이 메트릭에 `gpu.{gpu_index}.powerWatts` 태그를 부여합니다.

<div id="gpu-power-usage-percent">
  ### GPU 백분율
</div>

각 GPU의 전력 용량 대비 전력 사용량을 퍼센트로 나타냅니다.

W&amp;B는 이 메트릭에 `gpu.{gpu_index}.powerPercent` 태그를 부여합니다.

<div id="gpu-sm-clock-speed">
  ### GPU SM 클럭 속도
</div>

GPU의 Streaming Multiprocessor(SM) 클럭 속도를 MHz 단위로 나타냅니다. 이 메트릭은 연산 작업을 담당하는 GPU 코어의 처리 속도를 보여줍니다.

W&amp;B는 이 메트릭에 `gpu.{gpu_index}.smClock` 태그를 부여합니다.

<div id="gpu-memory-clock-speed">
  ### GPU 메모리 클럭 속도
</div>

GPU 메모리의 클럭 속도를 MHz로 나타내며, 이는 GPU 메모리와 처리 코어 간의 데이터 전송 속도에 영향을 미칩니다.

W&amp;B는 이 메트릭에 `gpu.{gpu_index}.memoryClock` 태그를 부여합니다.

<div id="gpu-graphics-clock-speed">
  ### GPU 그래픽 클럭 속도
</div>

GPU 상의 그래픽 렌더링 오퍼레이션을 위한 기본 클럭 속도를 나타내며, MHz 단위로 표현됩니다. 이 메트릭은 시각화 또는 렌더링 작업 중 성능을 종종 반영합니다.

W&amp;B는 이 메트릭에 `gpu.{gpu_index}.graphicsClock` 태그를 부여합니다.

<div id="gpu-corrected-memory-errors">
  ### GPU 정정된 메모리 오류
</div>

W&amp;B가 오류 검사 프로토콜을 통해 자동으로 정정한 GPU 메모리 오류 수를 추적하며, 복구 가능한 하드웨어 문제를 나타냅니다.

W&amp;B는 이 메트릭에 `gpu.{gpu_index}.correctedMemoryErrors` 태그를 부여합니다.

<div id="gpu-uncorrected-memory-errors">
  ### GPU 정정되지 않은 메모리 오류
</div>

W&amp;B가 정정하지 못한 GPU 메모리 오류 수를 추적합니다. 이는 처리 신뢰성에 영향을 줄 수 있는 복구 불가능한 오류를 나타냅니다.

W&amp;B는 이 메트릭에 `gpu.{gpu_index}.unCorrectedMemoryErrors` 태그를 부여합니다.

<div id="gpu-encoder-utilization">
  ### GPU 인코더 사용량
</div>

GPU의 비디오 인코더 사용량 비율을 나타내며, 인코딩 작업(예: 비디오 렌더링)이 실행 중일 때의 부하를 보여줍니다.

W&amp;B는 이 메트릭에 `gpu.{gpu_index}.encoderUtilization` 태그를 부여합니다.

<div id="amd-gpu">
  ## AMD GPU
</div>

W&amp;B는 AMD에서 제공하는 `rocm-smi` 도구의 출력(`rocm-smi -a --json`)에서 메트릭을 추출합니다.

ROCm [6.x (latest)](https://rocm.docs.amd.com/en/latest/) 및 [5.x](https://rocm.docs.amd.com/en/docs-5.6.0/) 형식이 지원됩니다. ROCm 형식에 대한 자세한 내용은 [AMD ROCm documentation](https://rocm.docs.amd.com/en/latest/compatibility/compatibility-matrix.html)을 참조하세요. 최신 형식에는 더 많은 세부 정보가 포함됩니다.

<div id="amd-gpu-utilization">
  ### AMD GPU 사용량
</div>

각 AMD GPU 디바이스의 GPU 사용량을 백분율로 나타냅니다.

W&amp;B는 이 메트릭에 `gpu.{gpu_index}.gpu` 태그를 부여합니다.

<div id="amd-gpu-memory-allocated">
  ### AMD GPU 메모리 할당
</div>

각 AMD GPU 장치에서 전체 사용 가능한 메모리 대비 할당된 GPU 메모리의 백분율을 나타냅니다.

W&amp;B는 이 메트릭에 `gpu.{gpu_index}.memoryAllocated` 태그를 부여합니다.

<div id="amd-gpu-temperature">
  ### AMD GPU 온도
</div>

각 AMD GPU 디바이스의 섭씨 GPU 온도입니다.

W&amp;B는 이 메트릭에 `gpu.{gpu_index}.temp` 태그를 부여합니다.

<div id="amd-gpu-power-usage-watts">
  ### AMD GPU 전력 사용량(와트)
</div>

각 AMD GPU 디바이스의 와트 단위 GPU 전력 사용량입니다.

W&amp;B는 이 메트릭에 `gpu.{gpu_index}.powerWatts` 태그를 부여합니다.

<div id="amd-gpu-power-usage-percent">
  ### AMD GPU 전력 사용량 백분율
</div>

각 AMD GPU 장치에 대해 GPU 전력 사용량을 전력 용량 대비 백분율로 나타냅니다.

W&amp;B는 이 메트릭에 `gpu.{gpu_index}.powerPercent`를 부여합니다.

ARM Mac의 Apple GPU 장치에서 GPU의 전력 용량 대비 전력 사용량을 백분율로 나타낸 값입니다.

W&amp;B는 이 메트릭에 `gpu.0.powerPercent` 태그를 부여합니다.

<div id="graphcore-ipu">
  ## Graphcore IPU
</div>

Graphcore IPU(Intelligence Processing Unit)는 머신 인텔리전스 작업을 위해 특별히 설계된 전용 하드웨어 가속기입니다.

<div id="ipu-device-metrics">
  ### IPU 디바이스 메트릭
</div>

이 메트릭은 특정 IPU 디바이스의 다양한 통계를 나타냅니다. 각 메트릭에는 이를 파악하기 위한 디바이스 ID(`device_id`)와 메트릭 키(`metric_key`)가 있습니다. W&amp;B는 이 메트릭에 `ipu.{device_id}.{metric_key}` 태그를 부여합니다.

메트릭은 Graphcore의 `gcipuinfo` 바이너리와 상호작용하는 독점 `gcipuinfo` 라이브러리를 사용해 추출됩니다. `sample` 방법은 프로세스 ID(`pid`)와 연결된 각 IPU 디바이스에 대해 이러한 메트릭을 가져옵니다. 중복 데이터를 로깅하지 않도록, 시간에 따라 변경되는 메트릭이나 디바이스의 메트릭을 처음 가져오는 경우에만 로깅됩니다.

각 메트릭에 대해 `parse_metric` 방법을 사용하여 원시 문자열 표현에서 메트릭 값을 추출합니다. 그런 다음 메트릭은 `aggregate` 방법을 사용해 여러 샘플 전반에서 집계됩니다.

다음은 사용 가능한 메트릭과 해당 단위 목록입니다.

* **평균 보드 온도** (`average board temp (C)`): 섭씨 단위의 IPU 보드 온도입니다.
* **평균 다이 온도** (`average die temp (C)`): 섭씨 단위의 IPU 다이 온도입니다.
* **클럭 속도** (`clock (MHz)`): MHz 단위의 IPU 클럭 속도입니다.
* **IPU 전력** (`ipu power (W)`): 와트 단위의 IPU 전력 소비량입니다.
* **IPU 사용량** (`ipu utilisation (%)`): IPU 사용량의 백분율입니다.
* **IPU 세션 사용량** (`ipu utilisation (session) (%)`): 현재 세션에 해당하는 IPU 사용량 백분율입니다.
* **데이터 링크 속도** (`speed (GT/s)`): 초당 기가 전송 수 단위의 데이터 전송 속도입니다.

<div id="google-cloud-tpu">
  ## Google Cloud TPU
</div>

Tensor Processing Units(TPUs)는 머신 러닝 워크로드를 가속화하는 데 사용되는 Google의 맞춤형 ASIC(Application Specific Integrated Circuits)입니다.

<div id="tpu-memory-usage">
  ### TPU 메모리 사용
</div>

현재 TPU 코어별 High Bandwidth Memory 사용량(바이트)입니다.

W&amp;B는 이 메트릭에 `tpu.{tpu_index}.memoryUsageBytes` 태그를 부여합니다.

<div id="tpu-memory-usage-percentage">
  ### TPU 메모리 사용률(%)
</div>

현재 TPU 코어별 High Bandwidth Memory 사용률(%)입니다.

W&amp;B는 이 메트릭에 `tpu.{tpu_index}.memoryUsageBytes` 태그를 부여합니다.

<div id="tpu-duty-cycle">
  ### TPU 듀티 사이클
</div>

TPU 디바이스별 TensorCore 듀티 사이클 백분율입니다. 샘플 기간 동안 가속기 TensorCore가 실제로 처리 작업을 수행한 시간의 비율을 추적합니다. 값이 클수록 TensorCore 사용량이 더 높음을 의미합니다.

W&amp;B는 이 메트릭에 `tpu.{tpu_index}.dutyCycle` 태그를 부여합니다.

<div id="aws-trainium">
  ## AWS Trainium
</div>

[AWS Trainium](https://aws.amazon.com/machine-learning/trainium/)은 AWS에서 제공하는 특수 하드웨어 플랫폼으로, 머신 러닝 워크로드 가속화에 중점을 둡니다. AWS의 `neuron-monitor` 도구는 AWS Trainium 메트릭을 캡처하는 데 사용됩니다.

<div id="trainium-neuron-core-utilization">
  ### Trainium Neuron Core 사용량
</div>

각 NeuronCore의 사용량 백분율로, 코어별 기준으로 보고됩니다.

W&amp;B는 이 메트릭에 `trn.{core_index}.neuroncore_utilization` 태그를 부여합니다.

<div id="trainium-host-memory-usage-total">
  ### Trainium 호스트 총 메모리 사용량
</div>

호스트의 총 메모리 사용량(바이트)입니다.

W&amp;B는 이 메트릭에 `trn.host_total_memory_usage` 태그를 부여합니다.

<div id="trainium-neuron-device-total-memory-usage">
  ### Trainium Neuron 디바이스 총 메모리 사용량
</div>

Neuron 디바이스의 총 메모리 사용량을 바이트 단위로 나타냅니다.

W&amp;B는 이 메트릭에  `trn.neuron_device_total_memory_usage)` 태그를 부여합니다.

<div id="trainium-host-memory-usage-breakdown">
  ### Trainium 호스트 메모리 사용 분석:
</div>

다음은 호스트의 메모리 사용 분석입니다:

* **애플리케이션 메모리** (`trn.host_total_memory_usage.application_memory`): 애플리케이션에서 사용하는 메모리입니다.
* **상수** (`trn.host_total_memory_usage.constants`): 상수에 사용하는 메모리입니다.
* **DMA 버퍼** (`trn.host_total_memory_usage.dma_buffers`): Direct Memory Access 버퍼에 사용하는 메모리입니다.
* **텐서** (`trn.host_total_memory_usage.tensors`): 텐서에 사용하는 메모리입니다.

<div id="trainium-neuron-core-memory-usage-breakdown">
  ### Trainium Neuron Core 메모리 사용 세부 내역
</div>

각 NeuronCore의 자세한 메모리 사용 정보:

* **상수** (`trn.{core_index}.neuroncore_memory_usage.constants`)
* **모델 코드** (`trn.{core_index}.neuroncore_memory_usage.model_code`)
* **모델 공유 스크래치패드** (`trn.{core_index}.neuroncore_memory_usage.model_shared_scratchpad`)
* **런타임 메모리** (`trn.{core_index}.neuroncore_memory_usage.runtime_memory`)
* **텐서** (`trn.{core_index}.neuroncore_memory_usage.tensors`)
