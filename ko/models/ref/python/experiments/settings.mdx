## <kbd>class</kbd> `Settings`

W&amp;B SDK용 Settings입니다.

이 클래스는 W&amp;B SDK의 설정 값을 관리하며,
모든 설정에 대해 타입 안전성과 검증을 보장합니다. Settings는
속성으로 액세스할 수 있고, 코드에서 직접 초기화하거나, 환경 변수(`WANDB_` 접두사),
그리고 설정 파일을 통해 초기화할 수 있습니다.

Settings는 다음 세 가지 범주로 구성됩니다:

1. Public settings: 사용자가 자신의 필요에 맞게 W&amp;B의 동작을 안전하게 사용자 지정할 수 있는
   핵심 설정 옵션입니다.
2. Internal settings: &#39;x&#95;&#39; 접두사가 붙은 설정으로, 저수준 SDK 동작을 처리합니다.
   이러한 설정은 주로 내부 사용 및 디버깅용입니다. 수정은 가능하지만,
   Public API의 일부로 간주되지 않으며 향후 버전에서 예고 없이 변경될 수 있습니다.
3. Computed settings: 다른 설정이나 환경으로부터 자동으로 계산되는 조회 전용 설정입니다.

**매개변수:**

* `allow_offline_artifacts` (bool): 오프라인 모드에서 테이블 아티팩트를 동기화할 수 있도록 허용하는 플래그입니다.
  이전 동작으로 되돌리려면 이를 False로 설정하세요.

* `allow_val_change` (bool): 설정된 이후에 `Config` 값을 수정할 수 있도록 허용하는 플래그입니다.

* `anonymous` (객체): 사용 중단되었으며 제거될 예정입니다.

* `api_key` (Optional[str]): W&amp;B API 키.

* `azure_account_url_to_access_key` (Optional[Dict[str, str]]): Azure 인테그레이션을 위한 Azure 계정 URL과 해당 액세스 키의 매핑입니다.

* `app_url_override` (선택 사항인 `str`): W&amp;B UI의 `app` URL을 재정의합니다.
  `app_url`은 일반적으로 `base_url`을 기준으로 계산되지만, 이 값을 사용해
  명시적으로 설정할 수 있습니다.
  WANDB&#95;APP&#95;URL은 해당 환경 변수입니다.

* `base_url` (str): 데이터 동기화용 W&amp;B 백엔드 URL입니다.

* `code_dir` (선택[str]): W&amp;B에서 추적할 코드를 포함하는 디렉터리입니다.

* `config_paths` (Optional[Sequence]): 파일에서 `Config` 객체로 설정을 로드할 경로입니다.

* `console` (Literal[&#39;auto&#39;, &#39;off&#39;, &#39;wrap&#39;, &#39;redirect&#39;, &#39;wrap&#95;raw&#39;, &#39;wrap&#95;emu&#39;]): 적용할 콘솔 캡처의 유형입니다.
  가능한 값은 다음과 같습니다:
  * &quot;auto&quot; - 시스템 환경과 설정을 기준으로 콘솔 캡처 방법을 자동으로 선택합니다.
  * &quot;off&quot; - 콘솔 캡처를 비활성화합니다.
  * &quot;redirect&quot; - 출력을 캡처하기 위해 저수준 파일 디스크립터를 리디렉션합니다.
  * &quot;wrap&quot; - sys.stdout/sys.stderr의 write 메서드를 재정의합니다. 시스템 상태를 기준으로
    &quot;wrap&#95;raw&quot; 또는 &quot;wrap&#95;emu&quot;로 매핑됩니다.
  * &quot;wrap&#95;raw&quot; - &quot;wrap&quot;과 동일하지만 에뮬레이터를 통하지 않고 원시 출력을 직접 캡처합니다.
    `wrap` 설정에서 파생되며 수동으로 설정해서는 안 됩니다.
  * &quot;wrap&#95;emu&quot; - &quot;wrap&quot;과 동일하지만 에뮬레이터를 통해 출력을 캡처합니다.
    `wrap` 설정에서 파생되며 수동으로 설정해서는 안 됩니다.

* `console_multipart` (bool): 멀티파트 콘솔 로깅을 활성화합니다.
  `True`이면 SDK는 콘솔 출력을 단일 `output.log` 대신
  `logs/` 디렉터리 아래의 타임스탬프가 포함된 파일에 씁니다.
  각 파트는 닫히는 즉시 업로드되므로 사용자가
  run이 활성 상태인 동안 로그에 실시간으로 액세스할 수 있습니다. 롤오버 주기는
  `console_chunk_max_bytes` 및/또는 `console_chunk_max_seconds`로
  제어됩니다.
  두 limits가 모두 `0`이면 모든 로그는 run 종료 시 한 번에 업로드됩니다.
  참고: 업로드된 청크는 변경 불가능하므로 이전 줄을 수정하는
  터미널 제어 시퀀스(예: 캐리지 리턴을 사용하는 진행률 표시줄)는
  현재 청크에만 영향을 줍니다.

* `console_chunk_max_bytes` (int): 멀티파트 console 로그의 크기 기반 롤오버 임곗값(바이트)입니다.
  현재 파트가 이 크기에 도달하면 새 console 로그 파일을
  시작합니다. `console_multipart`가 `True`일 때만
  영향이 있습니다.
  `console_chunk_max_seconds`와 함께 사용할 수 있으며, 어느 한도에
  먼저 도달하든 롤오버를 트리거합니다. 값이 `0`이면
  크기 기반 한도가 비활성화됩니다.

* `console_chunk_max_seconds` (int): 멀티파트 console logs에 대한 시간 기반 롤오버 임곗값(초)입니다.
  현재 파트가 시작된 이후 이만큼의 시간이 지나면 새 console log 파일을
  시작합니다. `console_multipart`가
  `True`여야 합니다. `console_chunk_max_bytes`와 함께 사용할 수 있으며, 먼저 도달한 한도가
  파트를 닫습니다. 값이 `0`이면 시간 기반
  한도가 비활성화됩니다.

* `credentials_file` (str): 임시 액세스 토큰을 기록할 파일 경로.

* `disable_code` (bool): 코드 캡처를 비활성화할지 여부입니다.

* `disable_git` (bool): Git 상태 캡처를 비활성화할지 여부입니다.

* `disable_job_creation` (bool): W&amp;B Launch용 작업 아티팩트 생성을 비활성화할지 여부입니다.

* `docker` (선택[str]): 스크립트를 실행하는 데 사용되는 도커 이미지입니다.

* `email` (Optional[str]): 사용자의 이메일 주소입니다.

* `entity` (Optional[str]): 사용자나 팀과 같은 W&amp;B entity입니다.

* `organization` (Optional[str]): W&amp;B 조직입니다.

* `force` (bool): `wandb.login()`에 `force` 플래그를 전달할지 여부.

* `fork_from` (Optional[RunMoment]): 이전 run 실행에서 포크할 지점을 지정합니다.
  해당 지점은 run ID, metric 및 해당 값으로 정의됩니다.
  현재는 metric &#39;&#95;step&#39;만 지원됩니다.

* `git_commit` (Optional[str]): run에 연결할 git 커밋 해시입니다.

* `git_remote` (str): run에 연결할 git remote입니다.

* `git_remote_url` (선택[str]): git 원격 저장소의 URL입니다.

* `git_root` (Optional[str]): git 저장소의 루트 디렉터리.

* `host` (선택 사항[str]): 스크립트를 실행 중인 머신의 호스트 이름.

* `http_proxy` (Optional[str]): W&amp;B에 보내는 HTTP 요청에 사용할 맞춤형 프록시 서버입니다.

* `https_proxy` (Optional[str]): W&amp;B로 보내는 HTTPS 요청에 사용할 맞춤형 프록시 서버입니다.

* `identity_token_file` (선택[str]): 인증을 위한 ID 토큰(JWT)이 포함된 파일의 경로입니다.

* `ignore_globs` (Sequence): 업로드에서 제외할 파일을 지정하는 `files_dir` 기준의 Unix glob 패턴입니다.

* `init_timeout` (float): 시간 초과되기 전까지 `wandb.init` 호출이 완료되기를 기다리는 시간(초)입니다.

* `insecure_disable_ssl` (bool): SSL 검증을 안전하지 않게 비활성화할지 여부입니다.

* `job_name` (선택[str]): 스크립트를 실행 중인 Launch job의 이름입니다.

* `job_source` (Optional[Literal[&#39;repo&#39;, &#39;artifact&#39;, &#39;image&#39;]]): Launch용 소스 유형입니다.

* `label_disable` (bool): 자동 라벨링 특성을 비활성화할지 여부.

* `launch_config_path` (Optional[str]): Launch 설정 파일 경로입니다.

* `login_timeout` (Optional[float]): 로그인 작업이 시간 초과되기 전에 기다리는 시간(초)입니다.

* `mode` (Literal[&#39;online&#39;, &#39;offline&#39;, &#39;shared&#39;, &#39;disabled&#39;, &#39;dryrun&#39;, &#39;run&#39;]): W&amp;B 로깅 및 동기화를 위한 운영 모드입니다.

* `notebook_name` (Optional[str]): Jupyter와 유사한 환경에서 실행하는 경우 노트북 이름입니다.

* `program` (Optional[str]): 사용 가능한 경우, run을 생성한 스크립트의 경로입니다.

* `program_abspath` (Optional[str]): run을
  생성한 스크립트까지의 루트 저장소 디렉터리 기준 절대 경로입니다.
  루트 저장소 디렉터리는 .git 디렉터리가 존재하는 경우 해당 디렉터리를 포함하는
  디렉터리로 정의됩니다. 그렇지 않으면 현재 작업 디렉터리입니다.

* `program_relpath` (Optional[str]): run을 생성한 스크립트의 상대 경로입니다.

* `project` (Optional[str]): W&amp;B 프로젝트 ID입니다.

* `quiet` (bool): 불필요한 출력을 억제하는 플래그입니다.

* `reinit` (Union[Literal[&#39;default&#39;, &#39;return&#95;previous&#39;, &#39;finish&#95;previous&#39;, &#39;create&#95;new&#39;], bool]): run이 활성 상태인 동안 `wandb.init()`가 호출되면 어떻게 처리할지 지정합니다.
  옵션:
  * &quot;default&quot;: 노트북에서는 &quot;finish&#95;previous&quot;를 사용하고, 그 외의 경우에는
    &quot;return&#95;previous&quot;를 사용합니다.
  * &quot;return&#95;previous&quot;: 아직 종료되지 않은 가장 최근에 생성된 run을 반환합니다.
    이는 `wandb.run`을 업데이트하지 않습니다. &quot;create&#95;new&quot; 옵션을
    참조하세요.
  * &quot;finish&#95;previous&quot;: 활성 상태인 모든 run을 종료한 다음, 새 run을 반환합니다.
  * &quot;create&#95;new&quot;: 다른 활성 run을 수정하지 않고 새 run을 생성합니다.
    `wandb.run` 및 `wandb.log`와 같은 최상위 함수를 업데이트하지 않습니다.
    이 때문에 전역 run에 의존하는 일부 오래된 인테그레이션은
    작동하지 않습니다.
    불리언 값일 수도 있지만, 이는 사용 중단되었습니다. False는
    &quot;return&#95;previous&quot;와 같고, True는 &quot;finish&#95;previous&quot;와 같습니다.

* `relogin` (bool): 새로 로그인하도록 강제하는 플래그입니다.

* `resume` (Optional[Literal[&#39;allow&#39;, &#39;must&#39;, &#39;never&#39;, &#39;auto&#39;]]): `run`의 재개 동작을 지정합니다.
  옵션:
  * &quot;must&quot;: 동일한 ID를 가진 기존 run에서 재개합니다. 해당 run이 없으면
    실패합니다.
  * &quot;allow&quot;: 동일한 ID를 가진 기존 run에서 재개를 시도합니다. 찾지 못하면
    새 run이 생성됩니다.
  * &quot;never&quot;: 항상 새 run을 시작합니다. 동일한 ID를 가진 run이 이미 있으면
    실패합니다.
  * &quot;auto&quot;: 동일한 머신에서 가장 최근에 실패한 run을 자동으로 재개합니다.

* `resume_from` (Optional[RunMoment]): 이전 run 실행의 특정 지점부터 재개할 위치를 지정합니다.
  이 지점은 run ID, metric, 그리고 그 값으로 정의됩니다.
  현재는 metric &#39;&#95;step&#39;만 지원됩니다.

* `root_dir` (str): 모든 run 관련 경로의 기준으로 사용할 루트 디렉터리입니다.
  특히 이는 wandb 디렉터리와 run 디렉터리를 파생하는 데 사용됩니다.

* `run_group` (선택[str]): 관련 run의 그룹 식별자입니다.
  UI에서 run을 그룹화하는 데 사용됩니다.

* `run_id` (Optional[str]): run의 ID입니다.

* `run_job_type` (Optional[str]): 실행 중인 작업 유형입니다(예: 트레이닝, 평가).

* `run_name` (선택[str]): run의 사람이 읽을 수 있는 이름입니다.

* `run_notes` (Optional[str]): run에 대한 추가 메모나 설명입니다.

* `run_tags` (Optional[Tuple[str, Ellipsis]]): run의 조직 및 필터링에 사용할 태그입니다.

* `sagemaker_disable` (bool): SageMaker 관련 기능을 비활성화하는 플래그입니다.

* `save_code` (Optional[bool]): run과 연결된 코드를 저장할지 여부입니다.

* `settings_system` (Optional[str]): 시스템 전체 설정 파일의 경로입니다.

* `max_end_of_run_history_metrics` (int): run 종료 시 표시할 이력 스파크라인의 최대 개수입니다.

* `max_end_of_run_summary_metrics` (int): run 종료 시점에 표시할 요약 메트릭의 최대 개수입니다.

* `show_errors` (bool): 오류 메시지를 표시할지 여부입니다.

* `show_info` (bool): 정보 메시지를 표시할지 여부입니다.

* `show_warnings` (bool): 경고 메시지를 표시할지 여부입니다.

* `silent` (bool): 모든 출력을 숨기기 위한 플래그.

* `strict` (Optional[bool]): 검증 및 오류 검사를 위해 엄격 모드를 활성화할지 여부입니다.

* `summary_timeout` (int): 시간 초과가 발생하기 전에 summary 오퍼레이션을 기다리는 시간(초)입니다.

* `sweep_id` (Optional[str]): 이 run이 속해 있는 스윕의 식별자입니다.

* `sweep_param_path` (Optional[str]): 스윕 파라미터 설정의 경로입니다.

* `symlink` (bool): 심볼릭 링크 사용 여부입니다(Windows를 제외한 환경에서는 기본값이 True입니다).

* `sync_tensorboard` (Optional[bool]): TensorBoard 로그를 W&amp;B와 동기화할지 여부입니다.

* `table_raise_on_max_row_limit_exceeded` (bool): 테이블 행 제한을 초과할 때 예외를 발생시킬지 여부입니다.

* `use_dot_wandb` (Optional[bool]): run 데이터에 숨겨진 `.wandb` 디렉터리를 사용할지, 또는 표시되는 `wandb` 디렉터리를 사용할지 여부입니다.
  True이면 SDK는 `.wandb`를 사용합니다. False이면 `wandb`를 사용합니다.
  설정하지 않으면, 이미 존재하는 경우 `.wandb`를 기본값으로 사용하고, 그렇지 않으면 `wandb`를 사용합니다.

* `username` (선택[str]): 사용자 이름.

* `x_disable_meta` (bool): 시스템 메타데이터 수집을 비활성화하는 플래그입니다.

* `x_disable_stats` (bool): 시스템 메트릭 수집을 비활성화하는 플래그입니다.

* `x_extra_http_headers` (Optional[Dict[str, str]]): 모든 나가는 HTTP Request에 추가할 헤더입니다.

* `x_label` (Optional[str]): run에 대해 수집된 시스템 메트릭과 콘솔 로그에 부여할 레이블입니다.
  이 값은 프런트엔드에서 데이터를 그룹별로 묶는 데 사용되며, 분산 트레이닝 작업의
  서로 다른 프로세스에서 수집된 데이터를 구분하는 데 사용할 수 있습니다.

  분산 설정에서는 파일 덮어쓰기를 방지하는 데 유용합니다
  기본 프로세스가 주된 로깅을 처리하므로, 시스템 메트릭과 로그만 필요한 경우
  보조 프로세스에서 발생하는 덮어쓰기를 피할 수 있습니다.

* `x_save_requirements` (bool): requirements 파일을 저장할지 여부를 나타내는 플래그입니다.

* `x_server_side_derived_summary` (bool): 서버가 `history`에서 `summary`를 자동으로 계산하도록 위임하는 플래그입니다.
  이 설정은 사용자가 제공한 `summary` 업데이트를 비활성화하지 않습니다.

* `x_service_wait` (float): wandb-core 내부 서비스가 시작되기까지 대기하는 시간(초)입니다.

* `x_skip_transaction_log` (bool): run 이벤트를 트랜잭션 로그에 저장하지 않을지 여부입니다.
  이는 온라인 run에만 해당합니다. 디스크에 기록되는
  데이터 양을 줄이는 데 사용할 수 있습니다.
  복구 가능성에 대한 보장이 사라지므로
  주의해서 사용해야 합니다.

* `x_stats_sampling_interval` (float): 시스템 모니터 샘플링 간격(초)입니다.

* `x_stats_dcgm_exporter` (선택[str]): Nvidia DCGM 메트릭을 추출할 엔드포인트입니다.
  옵션:
  * Prometheus `/api/v1/query` 엔드포인트에 대한 쿼리에서 DCGM 관련 메트릭을 추출합니다.
    서로 다른 노드에서 실행되는 DCGM 익스포터 인스턴스가 보고하는 메트릭을 Prometheus를 사용해 클러스터에서 집계하는 것은 일반적인 관행입니다.
  * TODO: DCGM 익스포터의 `/metrics` 엔드포인트에서 직접 메트릭을 파싱합니다.

  * `http://localhost:9400/api/v1/query?query=DCGM_FI_DEV_GPU_TEMP{node="l1337", cluster="globular"}`.

* `x_stats_open_metrics_endpoints` (Optional[Dict[str, str]]): 시스템 메트릭을 모니터링하기 위한 OpenMetrics `/metrics` 엔드포인트입니다.

* `x_stats_open_metrics_filters` (Union[Dict[str, Dict[str, str]], Sequence, None]): OpenMetrics `/metrics` 엔드포인트에서 수집하는 메트릭에 적용할 필터입니다.
  두 가지 형식을 지원합니다:
  * `{"Endpoint name을 접두사로 포함하는 메트릭 정규식 패턴": {"label": "label 값 정규식 패턴"}}`
  * `("메트릭 정규식 패턴 1", "메트릭 정규식 패턴 2", ...)`

* `x_stats_open_metrics_http_headers` (Optional[Dict[str, str]]): OpenMetrics 요청에 추가할 HTTP 헤더입니다.

* `x_stats_disk_paths` (선택[Sequence]): 디스크 사용량을 모니터링할 시스템 경로입니다.

* `x_stats_cpu_count` (Optional[int]): 시스템 CPU 개수입니다.
  설정하면 run 메타데이터에서 자동 감지된 값을 재정의합니다.

* `x_stats_cpu_logical_count` (선택[int]): 논리 CPU 개수입니다.
  설정하면 run 메타데이터의 자동 감지된 값을 재정의합니다.

* `x_stats_gpu_count` (선택[int]): GPU 장치 수입니다.
  설정하면 run 메타데이터에서 자동 감지된 값을 재정의합니다.

* `x_stats_gpu_type` (Optional[str]): GPU 장치 유형입니다.
  설정하면 run 메타데이터에서 자동 감지된 값을 덮어씁니다.

* `x_stats_gpu_device_ids` (Optional[Sequence]): 모니터링할 GPU 디바이스 인덱스입니다.
  설정하지 않으면 시스템 모니터가 모든 GPU의 메트릭을 캡처합니다.
  CUDA/ROCm 디바이스 열거와 일치하는 0부터 시작하는 인덱싱을 가정합니다.

* `x_stats_track_process_tree` (bool): `x_stats_pid`부터 시작하여 전체 프로세스 트리의 리소스 사용을 모니터링합니다.
  `True`이면 시스템 모니터는 PID가 `x_stats_pid`인 프로세스와 그 모든 하위 프로세스의 RSS, CPU%, 스레드 수를
  집계합니다.
  이 설정은 성능 오버헤드를 유발할 수 있으며 기본적으로 비활성화되어 있습니다.

* `x_sync_dir_suffix` (str): run의 디렉터리 이름(`sync_dir`)에 추가할 접미사입니다.
  이 값은 이름 충돌을 방지하기 위해 wandb.init()에서 설정됩니다.
  설정하면 기본 이름 뒤에 대시(-)를 붙여 연결됩니다.

* `x_update_finish_state` (bool): 이 프로세스가 서버에서 run의 최종 상태를 업데이트할 수 있는지 나타내는 플래그입니다.
