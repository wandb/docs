## <kbd>class</kbd> `Settings`

W&B SDK를 위한 설정 클래스입니다.

이 클래스는 W&B SDK의 구성 설정을 관리하며,
모든 설정에 대해 타입 안정성과 검증을 보장합니다. 설정은 속성으로 접근할 수 있고,
프로그램 코드, 환경 변수(`WANDB_` 접두사), 그리고 설정 파일을 통해 초기화할 수 있습니다.

설정은 다음 세 가지 범주로 나뉩니다:

1. Public settings: 사용자가 자신의 필요에 맞게 W&B의 동작을 안전하게 커스터마이즈할 수 있는
   핵심 구성 옵션입니다.
2. Internal settings: 'x_' 접두사가 붙은 설정으로, 저수준 SDK 동작을 처리합니다.
   이 설정들은 주로 내부 사용 및 디버깅을 위한 것입니다. 수정할 수는 있지만,
   공개 API의 일부로 간주되지 않으며, 향후 버전에서 예고 없이 변경될 수 있습니다.
3. Computed settings: 다른 설정 또는 실행 환경으로부터 자동으로 유도되는 읽기 전용 설정입니다.

**매개변수:**

* `allow_offline_artifacts` (bool): 오프라인 모드에서도 테이블 아티팩트를 동기화할 수 있게 허용하는 플래그입니다.
  이전 동작 방식으로 되돌리려면 이 값을 False로 설정하세요.

* `allow_val_change` (bool): `Config` 값이 설정된 후에도 수정할 수 있는지 여부를 나타내는 플래그입니다.

* `anonymous` (object): 더 이상 사용되지 않으며 제거될 예정입니다.

* `api_key` (Optional[str]): W&amp;B API 키.

* `azure_account_url_to_access_key` (Optional[Dict[str, str]]): Azure 통합을 위해 Azure 계정 URL을 해당 액세스 키에 매핑하는 딕셔너리입니다.

* `app_url_override` (Optional[str]): W&amp;B UI의 &#39;app&#39; URL을 재정의하는 값입니다.
  `app_url`은 보통 `base_url`을 기준으로 계산되지만, 이 값을 사용하면
  명시적으로 설정할 수 있습니다.
  WANDB&#95;APP&#95;URL은 이에 대응하는 환경 변수입니다.

* `base_url` (str): 데이터 동기화를 위한 W&amp;B 백엔드의 URL입니다.

* `code_dir` (Optional[str]): W&amp;B에서 추적할 코드가 들어 있는 디렉터리입니다.

* `config_paths` (Optional[Sequence]): 설정을 로드해 `Config` 객체에 넣을 파일들의 경로.

* `console` (Literal[&#39;auto&#39;, &#39;off&#39;, &#39;wrap&#39;, &#39;redirect&#39;, &#39;wrap&#95;raw&#39;, &#39;wrap&#95;emu&#39;]): 적용할 콘솔 캡처 방식입니다.
  가능한 값은 다음과 같습니다:
  * &quot;auto&quot; - 시스템 환경과 설정을 기준으로 콘솔 캡처 방식을 자동으로 선택합니다.
  * &quot;off&quot; - 콘솔 캡처를 비활성화합니다.
  * &quot;redirect&quot; - 출력을 캡처하기 위해 저수준 파일 디스크립터를 리디렉션합니다.
  * &quot;wrap&quot; - sys.stdout/sys.stderr의 write 메서드를 오버라이드합니다. 시스템 상태에 따라 &quot;wrap&#95;raw&quot; 또는 &quot;wrap&#95;emu&quot;로 매핑됩니다.
  * &quot;wrap&#95;raw&quot; - &quot;wrap&quot;과 동일하지만, 에뮬레이터를 거치지 않고 원시 출력을 직접 캡처합니다. `wrap` 설정에서 파생되며 수동으로 설정해서는 안 됩니다.
  * &quot;wrap&#95;emu&quot; - &quot;wrap&quot;과 동일하지만, 에뮬레이터를 통해 출력을 캡처합니다.
    `wrap` 설정에서 파생되며 수동으로 설정해서는 안 됩니다.

* `console_multipart` (bool): multipart 콘솔 로깅을 활성화합니다.
  `True`이면 SDK는 콘솔 출력을 단일 `output.log` 대신
  `logs/` 디렉터리 아래의 타임스탬프가 포함된 여러 파일에 기록합니다.
  각 파트는 닫히는 즉시 업로드되며, 실행이 활성 상태인 동안
  사용자는 로그에 실시간으로 접근할 수 있습니다. 롤오버 간격은
  `console_chunk_max_bytes` 및/또는 `console_chunk_max_seconds`로 제어됩니다.
  두 값이 모두 `0`이면, 모든 로그는 실행이 종료될 때 한 번에 업로드됩니다.
  참고: 업로드된 청크는 변경할 수 없습니다. 이전 줄을 수정하는
  터미널 제어 시퀀스(예: 캐리지 리턴을 사용하는 진행률 표시줄)는
  현재 청크에만 영향을 미칩니다.

* `console_chunk_max_bytes` (int): 멀티파트 콘솔 로그의 크기 기준 롤오버 임계값(바이트 단위)입니다.
  현재 파트가 이 크기에 도달하면 새 콘솔 로그 파일을 생성합니다.
  `console_multipart`가 `True`일 때만 적용됩니다.
  `console_chunk_max_seconds`와 함께 사용할 수 있으며, 먼저 도달한 제한이
  롤오버를 트리거합니다. 값이 `0`이면 크기 기반 제한이 비활성화됩니다.

* `console_chunk_max_seconds` (int): 멀티파트 콘솔 로그에 대한 시간 기반 롤오버 기준(초 단위)입니다.
  현재 파트가 시작된 후 지정한 초 수가 경과하면 새로운 콘솔 로그 파일을 생성합니다.
  `console_multipart`가 `True`여야 합니다. `console_chunk_max_bytes`와 함께
  사용할 수 있으며, 둘 중 먼저 도달한 제한값에 따라 현재 파트가 종료됩니다. 값이 `0`이면
  시간 기반 제한이 비활성화됩니다.

* `credentials_file` (str): 임시 액세스 토큰을 쓰기 위한 파일 경로입니다.

* `disable_code` (bool): 코드 캡처를 비활성화할지 여부입니다.

* `disable_git` (bool): Git 상태를 캡처하는 기능을 비활성화할지 여부입니다.

* `disable_job_creation` (bool): W&amp;B Launch용 Job 아티팩트 생성을 비활성화할지 여부.

* `docker` (Optional[str]): 스크립트를 실행하는 데 사용할 Docker 이미지입니다.

* `email` (Optional[str]): 사용자의 이메일 주소.

* `entity` (Optional[str]): 사용자 또는 팀과 같은 W&amp;B 엔터티.

* `organization` (Optional[str]): W&amp;B 조직 이름입니다.

* `force` (bool): `wandb.login()`에 `force` 플래그를 전달할지 여부입니다.

* `fork_from` (Optional[RunMoment]): 이전 실행의 특정 시점에서 포크할 지점을 지정합니다.
  해당 시점은 실행 ID, 메트릭, 그리고 그 값으로 정의됩니다.
  현재는 메트릭 &#39;&#95;step&#39;만 지원됩니다.

* `git_commit` (Optional[str]): 실행에 연결할 git 커밋 해시입니다.

* `git_remote` (str): 실행과 연결할 Git 원격 저장소입니다.

* `git_remote_url` (Optional[str]): Git 원격 저장소의 URL입니다.

* `git_root` (Optional[str]): Git 리포지토리의 루트 디렉터리입니다.

* `host` (Optional[str]): 스크립트를 실행하는 머신의 호스트 이름입니다.

* `http_proxy` (Optional[str]): W&amp;B로 보내는 HTTP 요청에 사용할 사용자 정의 프록시 서버.

* `https_proxy` (Optional[str]): W&amp;B로 보내는 HTTPS 요청에 사용할 사용자 지정 프록시 서버.

* `identity_token_file` (Optional[str]): 인증에 사용할 ID 토큰(JWT)이 들어 있는 파일의 경로입니다.

* `ignore_globs` (Sequence): `files_dir`를 기준으로, 업로드에서 제외할 파일을 지정하는 상대 경로 Unix glob 패턴.`

* `init_timeout` (float): `wandb.init` 호출이 완료될 때까지, 타임아웃되기 전까지 기다릴 시간(초)입니다.

* `insecure_disable_ssl` (bool): 보안상 안전하지 않은 방식으로 SSL 검증을 비활성화할지 여부입니다.

* `job_name` (Optional[str]): 스크립트를 실행하는 Launch 잡의 이름입니다.

* `job_source` (Optional[Literal[&#39;repo&#39;, &#39;artifact&#39;, &#39;image&#39;]]): Launch의 소스 유형입니다.

* `label_disable` (bool): 자동 레이블링 기능을 비활성화할지 여부.

* `launch_config_path` (Optional[str]): Launch 구성 파일 경로입니다.

* `login_timeout` (Optional[float]): 로그인 작업이 시간 초과되기 전까지 대기하는 시간(초)입니다.

* `mode` (Literal[&#39;online&#39;, &#39;offline&#39;, &#39;shared&#39;, &#39;disabled&#39;, &#39;dryrun&#39;, &#39;run&#39;]): W&amp;B 로깅 및 동기화를 위한 동작 모드입니다.

* `notebook_name` (Optional[str]): Jupyter와 유사한 환경에서 실행 중인 경우의 노트북 이름입니다.

* `program` (Optional[str]): 실행을 생성한 스크립트의 경로(가능한 경우)입니다.

* `program_abspath` (Optional[str]): 실행을 생성한 스크립트까지의 절대 경로로, 루트 리포지토리 디렉터리로부터의 경로입니다.
  루트 리포지토리 디렉터리는 .git 디렉터리를 포함하는 디렉터리(존재하는 경우)로 정의됩니다. .git 디렉터리가 없으면 현재 작업 디렉터리가 루트 리포지토리 디렉터리가 됩니다.

* `program_relpath` (Optional[str]): 실행을 생성한 스크립트의 상대 경로입니다.

* `project` (Optional[str]): W&amp;B 프로젝트의 ID입니다.

* `quiet` (bool): 부가적인 출력 내용을 표시하지 않도록 하는 플래그입니다.

* `reinit` (Union[Literal[&#39;default&#39;, &#39;return&#95;previous&#39;, &#39;finish&#95;previous&#39;, &#39;create&#95;new&#39;], bool]): 실행이 활성화된 상태에서 `wandb.init()`이 호출될 때 수행할 동작.
  Options:
  * &quot;default&quot;: 노트북에서는 &quot;finish&#95;previous&quot;를, 그 외의 환경에서는
    &quot;return&#95;previous&quot;를 사용합니다.
  * &quot;return&#95;previous&quot;: 아직 종료되지 않은 실행 중 가장 최근에 생성된 실행을
    반환합니다. `wandb.run`은 업데이트하지 않습니다. &quot;create&#95;new&quot;
    옵션을 참조하세요.
  * &quot;finish&#95;previous&quot;: 활성 상태인 모든 실행을 종료한 다음, 새 실행을 반환합니다.
  * &quot;create&#95;new&quot;: 다른 활성 실행에 영향을 주지 않고 새 실행을 생성합니다.
    `wandb.run` 및 `wandb.log`와 같은 최상위 함수는 업데이트하지 않습니다.
    이로 인해 전역 실행 객체에 의존하는 일부 구버전 통합 기능은 동작하지 않을 수
    있습니다.
    불리언 값으로 지정할 수도 있으나, 이는 더 이상 권장되지 않습니다. False는
    &quot;return&#95;previous&quot;와 같고, True는 &quot;finish&#95;previous&quot;와 같습니다.

* `relogin` (bool): 새로운 로그인 시도를 강제로 수행하도록 하는 플래그입니다.

* `resume` (Optional[Literal[&#39;allow&#39;, &#39;must&#39;, &#39;never&#39;, &#39;auto&#39;]]): 실행을 재개하는 동작 방식을 지정합니다.
  Options:
  * &quot;must&quot;: 동일한 ID의 기존 실행에서 반드시 재개합니다. 해당 실행이 없으면
    실패합니다.
  * &quot;allow&quot;: 동일한 ID의 기존 실행에서 재개를 시도합니다. 실행이 없으면
    새 실행이 생성됩니다.
  * &quot;never&quot;: 항상 새 실행을 시작합니다. 동일한 ID의 실행이 이미 있으면
    실패합니다.
  * &quot;auto&quot;: 동일한 머신에서 가장 최근에 실패한 실행에서 자동으로 재개합니다.

* `resume_from` (Optional[RunMoment]): 이전 실행의 특정 지점부터 실행을 재개할 위치를 지정합니다.
  이 지점은 실행 ID, 메트릭 이름, 그리고 해당 메트릭 값으로 정의됩니다.
  현재는 메트릭 &#39;&#95;step&#39;만 지원됩니다.

* `root_dir` (str): 모든 실행 관련 경로의 기준으로 사용할 루트 디렉터리입니다.
  특히, 이 값은 wandb 디렉터리와 실행 디렉터리의 경로를 결정하는 데 사용됩니다.

* `run_group` (Optional[str]): 관련 실행을 묶기 위한 그룹 식별자입니다.
  UI에서 실행을 그룹화하는 데 사용됩니다.

* `run_job_type` (Optional[str]): 실행 중인 작업 유형 (예: training, evaluation).

* `run_name` (Optional[str]): 사람이 읽을 수 있는 실행 이름입니다.

* `run_notes` (Optional[str]): 실행에 대한 추가 메모 또는 설명입니다.

* `run_tags` (Optional[Tuple[str, Ellipsis]]): 실행을 구성하고 필터링할 때 사용할 태그입니다.

* `sagemaker_disable` (bool): SageMaker 전용 기능을 비활성화하는 플래그입니다.

* `save_code` (Optional[bool]): 실행과 관련된 코드를 저장할지 여부입니다.

* `settings_system` (Optional[str]): 시스템 전역 설정 파일의 경로입니다.

* `max_end_of_run_history_metrics` (int): 실행 종료 시 표시할 history 스파크라인의 최대 개수입니다.

* `max_end_of_run_summary_metrics` (int): 실행 종료 시 표시할 요약 지표의 최대 개수.

* `show_errors` (bool): 오류 메시지를 표시할지 여부.

* `show_info` (bool): 정보 메시지를 표시할지 여부를 지정합니다.

* `show_warnings` (bool): 경고 메시지를 표시할지 여부입니다.

* `silent` (bool): 모든 출력을 억제하는 플래그입니다.

* `strict` (Optional[bool]): 유효성 검사 및 오류 검사를 위한 strict 모드를 활성화할지 여부입니다.

* `summary_timeout` (int): 요약 작업에서 타임아웃되기 전까지 대기할 시간(초)입니다.

* `sweep_id` (Optional[str]): 이 실행이 속한 스윕의 식별자입니다.

* `sweep_param_path` (Optional[str]): 스윕 파라미터 설정이 정의된 경로입니다.

* `symlink` (bool): 심볼릭 링크를 사용할지 여부를 지정합니다(기본값은 Windows를 제외한 환경에서 True).

* `sync_tensorboard` (Optional[bool]): TensorBoard 로그를 W&amp;B와 동기화할지 여부를 지정합니다.

* `table_raise_on_max_row_limit_exceeded` (bool): 테이블 행 수가 최대 제한을 초과했을 때 예외를 발생시킬지 여부입니다.

* `use_dot_wandb` (Optional[bool]): 실행 데이터용으로 숨김 디렉터리인 `.wandb`를 사용할지, 아니면 표시되는 `wandb` 디렉터리를 사용할지 여부입니다.
  True이면 SDK는 `.wandb`를 사용합니다. False이면 `wandb`를 사용합니다.
  설정하지 않으면, `.wandb` 디렉터리가 이미 존재하는 경우 기본값은 `.wandb`이고, 그렇지 않으면 `wandb`입니다.

* `x_disable_meta` (bool): 시스템 메타데이터 수집을 비활성화할지 여부를 지정하는 플래그입니다.

  보조 프로세스에서 파일이 덮어써지는 것을 방지하는 데 유용합니다.

* `x_save_requirements` (bool): requirements 파일 저장 여부를 나타내는 플래그입니다.

* `x_server_side_derived_summary` (bool): history에서 summary를 자동으로 계산하는 작업을 서버에서 수행하도록 위임할지 여부를 나타내는 플래그입니다.

    Prometheus를 사용해 클러스터의 서로 다른 노드에서 실행 중인 DCGM Exporter 인스턴스가 보고하는 메트릭을 집계하는 것이 일반적입니다.
