## <kbd>class</kbd> `Run`

W&amp;B에 로깅되는 계산 단위입니다. 일반적으로 ML 실험을 의미합니다.

새 run을 생성하려면 [`wandb.init()`](https://docs.wandb.ai/ref/python/init/)을 호출하세요. `wandb.init()`은 새 run을 시작하고 `wandb.Run` 객체를 반환합니다. 각 run은 고유한 ID(run ID)에 연결됩니다. W&amp;B는 컨텍스트 관리자(`with` 문)를 사용해 run을 자동으로 종료할 것을 권장합니다.

분산 트레이닝 실험에서는 프로세스당 하나의 run을 사용해 각 프로세스를 개별적으로 추적하거나, 모든 프로세스를 단일 run으로 추적할 수 있습니다. 자세한 내용은 [분산 트레이닝 실험 로깅](https://docs.wandb.ai/guides/track/log/distributed-training)을 참조하세요.

`wandb.Run.log()`를 사용해 run에 데이터를 로깅할 수 있습니다. `wandb.Run.log()`로 로깅한 모든 항목은 해당 run으로 전송됩니다. 자세한 내용은 [실험 생성](https://docs.wandb.ai/guides/track/create-an-experiment/) 또는 [`wandb.init`](https://docs.wandb.ai/ref/python/init/) API 레퍼런스 페이지를 참조하세요.

[`wandb.apis.public`](https://docs.wandb.ai/ref/python/public-api/api/) 네임스페이스에는 또 다른 `Run` 객체가 있습니다. 이 객체는 이미 생성된 runs와 상호작용하는 데 사용합니다.

**속성:**

* `summary`:  (Summary) run의 요약 정보이며, 딕셔너리와 유사한 객체입니다. 자세한 내용은 다음을 참조하세요.
* `[요약 메트릭 로깅](https`: //docs.wandb.ai/guides/track/log/log-summary/).

**예시:**
`wandb.init()`으로 run 생성하기:

```python
import wandb

# 새 run을 시작하고 데이터를 로깅합니다
# 컨텍스트 매니저(`with` 문)를 사용하여 run을 자동으로 종료합니다
with wandb.init(entity="entity", project="project") as run:
    run.log({"accuracy": acc, "loss": loss})
```

### <kbd>속성</kbd> Run.group

이 run과 연결된 그룹의 이름을 반환합니다.

run을 함께 그룹화하면 관련된 실험을 W&amp;B UI에서 한데 모아 체계적으로 구성하고 시각화할 수 있습니다. 이는 특히 분산 트레이닝이나 교차 검증처럼 여러 run을 하나의 통합된 실험으로 보고 관리해야 하는 시나리오에서 유용합니다.

모든 프로세스가 동일한 run 객체를 공유하는 shared 모드에서는 run이 하나뿐이므로 일반적으로 그룹을 설정할 필요가 없습니다.

**반환값:**

* `str`: 그룹 속성 값.

### <kbd>방법</kbd> `Run.define_metric`

```python
define_metric(
    name: 'str',
    step_metric: 'str | wandb_metric.Metric | None' = None,
    step_sync: 'bool | None' = None,
    hidden: 'bool | None' = None,
    summary: 'str | None' = None,
    goal: 'str | None' = None,
    overwrite: 'bool | None' = None
) → wandb_metric.Metric
```

`wandb.Run.log()`로 로깅되는 메트릭을 사용자 지정합니다.

**매개변수:**

* `name`:  사용자 지정할 메트릭의 이름입니다.
* `step_metric`:  자동 생성 차트에서 이 메트릭의 X축으로 사용할 다른 메트릭의 이름입니다.
* `step_sync`:  명시적으로 제공되지 않은 경우 마지막 step&#95;metric 값을 `wandb.Run.log()`에 자동으로 삽입합니다. step&#95;metric이 지정된 경우 기본값은 True입니다.
* `hidden`:  이 메트릭을 자동 플롯에서 숨깁니다.
* `summary`:  summary에 추가할 집계 메트릭을 지정합니다. 지원되는 집계에는 &quot;min&quot;, &quot;max&quot;, &quot;mean&quot;, &quot;last&quot;, &quot;first&quot;, &quot;best&quot;, &quot;copy&quot;, &quot;none&quot;이 포함됩니다. &quot;none&quot;은 summary가 생성되지 않도록 합니다. &quot;best&quot;는 goal 파라미터와 함께 사용되며, &quot;best&quot;는 사용 중단되었으므로 사용하지 마시고 대신 &quot;min&quot; 또는 &quot;max&quot;를 사용하세요. &quot;copy&quot;도 사용 중단되었으므로 사용하지 마세요.
* `goal`:  &quot;best&quot; summary 유형을 해석하는 방법을 지정합니다. 지원되는 옵션은 &quot;minimize&quot;와 &quot;maximize&quot;입니다. &quot;goal&quot;은 사용 중단되었으므로 사용하지 마시고 대신 &quot;min&quot; 또는 &quot;max&quot;를 사용하세요.
* `overwrite`:  false이면 이 Call은 지정되지 않은 파라미터에 대해 이전 `define_metric` Call의 값을 사용하여 동일한 메트릭에 대한 이전 `define_metric` Call과 병합됩니다. true이면 지정되지 않은 파라미터가 이전 Call에서 지정된 값을 덮어씁니다.

**반환값:**
이 Call을 나타내는 객체이며, 그 외에는 버려도 됩니다.

### <kbd>방법</kbd> `Run.finish`

```python
finish(exit_code: 'int | None' = None, quiet: 'bool | None' = None) → None
```

Run을 종료하고 남아 있는 데이터를 모두 업로드합니다.

W&amp;B Run의 완료를 표시하고 모든 데이터가 서버에 Sync되도록 합니다. Run의 최종 상태는 종료 조건과 Sync 상태에 따라 확인됩니다.

Run 상태:

* Running: 데이터를 로깅하고 및/또는 heartbeat를 전송하는 활성 Run입니다.
* Crashed: 예기치 않게 heartbeat 전송을 중단한 Run입니다.
* Finished: 모든 데이터가 Sync된 상태에서 성공적으로 완료된 Run입니다 (`exit_code=0`).
* Failed: 오류와 함께 완료된 Run입니다 (`exit_code!=0`).
* Killed: 완료되기 전에 강제로 중지된 Run입니다.

**매개변수:**

* `exit_code`:  Run의 종료 상태를 나타내는 정수입니다. 성공에는 0을 사용하고, 그 외의 값은 Run을 failed로 표시합니다.
* `quiet`:  사용 중단되었습니다. `wandb.Settings(quiet=...)`를 사용해 로깅 상세 수준을 설정하세요.

### <kbd>방법</kbd> `Run.finish_artifact`

```python
finish_artifact(
    artifact_or_path: 'Artifact | str',
    name: 'str | None' = None,
    type: 'str | None' = None,
    aliases: 'list[str] | None' = None,
    distributed_id: 'str | None' = None
) → Artifact
```

최종 확정되지 않은 아티팩트를 run의 출력으로 완료합니다.

이후 동일한 distributed ID로 수행한 &quot;upsert&quot;는 새 버전이 생성됩니다.

**매개변수:**

* `artifact_or_path`:  이 아티팩트의 콘텐츠에 대한 경로입니다. 다음 형식을 사용할 수 있습니다:
  * `/local/directory`
  * `/local/directory/file.txt`
  * `s3://bucket/path`  또는 `wandb.Artifact`를 호출하여 생성한 Artifact 객체를 전달할 수도 있습니다.
* `name`:  아티팩트 이름입니다. entity/프로젝트 접두사를 붙일 수 있습니다. 유효한 이름은 다음 형식을 사용할 수 있습니다:
  * name:version
  * name:alias
  * digest  지정하지 않으면 현재 run id를 앞에 붙인 경로의 basename이 기본값으로 사용됩니다.
* `type`:  기록할 아티팩트의 유형입니다. 예시로는 `dataset`, `model`이 있습니다.
* `aliases`:  이 아티팩트에 적용할 별칭입니다. 기본값은 `["latest"]`입니다.
* `distributed_id`:  모든 분산 작업이 공유하는 고유 string입니다. None이면 run의 그룹 이름이 기본값으로 사용됩니다.

**반환값:**
`Artifact` 객체입니다.

### <kbd>방법</kbd> `Run.link_artifact`

```python
link_artifact(
    artifact: 'Artifact',
    target_path: 'str',
    aliases: 'list[str] | None' = None
) → Artifact
```

아티팩트를 collection에 연결합니다.

&quot;link&quot;라는 용어는 W&amp;B가 아티팩트를 저장하는 위치와 레지스트리에서 아티팩트에 액세스할 수 있는 위치를 연결하는 포인터를 의미합니다. 아티팩트를 collection에 연결해도 W&amp;B는 아티팩트를 복제하지 않습니다.

지정된 collection의 Registry UI에서 연결된 아티팩트를 확인하세요.

**매개변수:**

* `artifact`:  collection에 연결할 아티팩트 객체입니다.
* `target_path`:  collection의 경로입니다. 경로는 &quot;wandb-registry-&quot; 접두사와 레지스트리 이름 및 collection 이름으로 구성됩니다: `wandb-registry-{REGISTRY_NAME}/{COLLECTION_NAME}`.
* `aliases`:  연결된 아티팩트에 하나 이상의 별칭을 추가합니다. &quot;latest&quot; 별칭은 가장 최근에 연결한 아티팩트에 자동으로 적용됩니다.

**반환값:**
연결된 아티팩트입니다.

***


### <kbd>방법</kbd> `Run.link_model`

```python
link_model(
    path: 'StrPath',
    registered_model_name: 'str',
    name: 'str | None' = None,
    aliases: 'list[str] | None' = None
) → Artifact | None
```

모델 아티팩트 버전을 로깅하고 모델 레지스트리에 있는 등록된 모델에 연결합니다.

연결된 모델 버전은 지정된 등록된 모델의 UI에서 확인할 수 있습니다.

이 방법은 다음을 수행합니다:

* &#39;name&#39; 모델 아티팩트가 이미 로깅되었는지 확인합니다. 이미 로깅되었다면 &#39;path&#39;에 있는 파일과 일치하는 아티팩트 버전을 사용하거나 새 버전을 로깅합니다. 그렇지 않으면 &#39;path&#39; 아래의 파일을 유형이 &#39;model&#39;인 새 모델 아티팩트 &#39;name&#39;으로 로깅합니다.
* &#39;model-registry&#39; 프로젝트에 &#39;registered&#95;model&#95;name&#39;이라는 이름의 등록된 모델이 있는지 확인합니다. 없으면 &#39;registered&#95;model&#95;name&#39;이라는 이름의 새 등록된 모델을 생성합니다.
* 모델 아티팩트 &#39;name&#39;의 버전을 등록된 모델 &#39;registered&#95;model&#95;name&#39;에 연결합니다.
* &#39;aliases&#39; 목록의 별칭을 새로 연결된 모델 아티팩트 버전에 추가합니다.

**매개변수:**

* `path`:  (str) 이 모델의 콘텐츠 경로로, 다음 형식 중 하나일 수 있습니다:
  * `/local/directory`
  * `/local/directory/file.txt`
  * `s3://bucket/path`
* `registered_model_name`:  모델이 연결될 등록된 모델의 이름입니다. 등록된 모델은 모델 레지스트리에 연결된 모델 버전의 모음이며, 일반적으로 팀의 특정 ML 작업을 나타냅니다. 이 등록된 모델이 속한 entity는 run에서 파생됩니다.
* `name`:  &#39;path&#39;의 파일이 로깅될 모델 아티팩트의 이름입니다. 지정하지 않으면 경로의 basename 앞에 current run id를 붙인 값이 기본값으로 사용됩니다.
* `aliases`:  등록된 모델 내부에서 이 연결된 아티팩트에만 적용되는 별칭입니다. 연결된 아티팩트의 최신 버전에는 항상 &quot;latest&quot; 별칭이 적용됩니다.

**예외:**

* `AssertionError`:  registered&#95;model&#95;name이 경로이거나 모델 아티팩트 &#39;name&#39;의 유형에 &#39;model&#39; 부분 문자열이 포함되지 않은 경우입니다.
* `ValueError`:  name에 유효하지 않은 특수 문자가 포함된 경우입니다.

**반환값:**
연결에 성공하면 연결된 아티팩트를 반환하고, 그렇지 않으면 `None`을 반환합니다.

### <kbd>방법</kbd> `Run.log`

```python
log(
    data: 'dict[str, Any]',
    step: 'int | None' = None,
    commit: 'bool | None' = None
) → None
```

run 데이터를 업로드하세요.

`log`를 사용하여 스칼라, 이미지, 비디오, 히스토그램, 플롯, 테이블과 같은 Runs의 데이터를 로깅하세요. 코드 스니펫, 모범 사례 등의 자세한 내용은 [객체 및 미디어 로깅](https://docs.wandb.ai/guides/track/log)을 참조하세요.

기본 사용법:

```python
import wandb

with wandb.init() as run:
     run.log({"train-loss": 0.5, "accuracy": 0.9})
```

이전 코드 스니펫은 loss와 정확도를 run의 이력에 저장하고 이러한 메트릭의 summary 값을 업데이트합니다.

[wandb.ai](https://wandb.ai)의 Workspace에서 로깅한 데이터를 시각화하거나, W&amp;B 앱의 [자체 호스팅 인스턴스](https://docs.wandb.ai/guides/hosting)에서 시각화할 수 있습니다. 또는 [Public API](https://docs.wandb.ai/guides/track/public-api-guide)를 사용해 데이터를 내보내고 Jupyter notebook 같은 로컬 환경에서 시각화하고 탐색할 수도 있습니다.

로깅한 값은 스칼라일 필요가 없습니다. 이미지, 오디오, 비디오 등 [W&amp;B가 지원하는 데이터 유형](https://docs.wandb.ai/ref/python/data-types/)은 무엇이든 로깅할 수 있습니다. 예를 들어 `wandb.Table`을 사용해 구조화된 데이터를 로깅할 수 있습니다. 자세한 내용은 [Log tables, visualize and query data](https://docs.wandb.ai/guides/models/tables/tables-walkthrough) 튜토리얼을 참조하세요.

W&amp;B는 이름에 슬래시(`/`)가 포함된 메트릭을 마지막 슬래시 앞의 텍스트를 사용해 이름 붙인 섹션으로 구성합니다. 예를 들어 다음은 &quot;train&quot;과 &quot;validate&quot;라는 두 개의 섹션을 만듭니다.

```python
with wandb.init() as run:
     # "train" 섹션에 메트릭을 기록합니다.
     run.log(
         {
             "train/accuracy": 0.9,
             "train/loss": 30,
             "validate/accuracy": 0.8,
             "validate/loss": 20,
         }
     )
```

중첩은 한 단계만 지원되며, `run.log({"a/b/c": 1})`는 &quot;a&quot;라는 이름의 section을 생성합니다.

`run.log()`는 초당 몇 번 이상 호출하도록 설계되지 않았습니다. 최적의 성능을 위해 N회 반복마다 한 번만 logging하거나, 여러 반복에 걸쳐 데이터를 수집한 뒤 이를 단일 step에서 log하세요.

기본적으로 `log`를 호출할 때마다 새로운 &quot;step&quot;이 생성됩니다. step은 항상 증가해야 하며, 이전 step에 log하는 것은 불가능합니다. chart에서는 어떤 metric이든 X축으로 사용할 수 있습니다. 자세한 내용은 [사용자 지정 log 축](https://docs.wandb.ai/guides/track/log/customize-logging-axes/)을 참조하세요.

많은 경우 W&amp;B step은 트레이닝 step이라기보다 timestamp처럼 다루는 것이 더 좋습니다.

```python
with wandb.init() as run:
     # 예시: X축으로 사용할 "epoch" 메트릭을 로깅합니다.
     run.log({"epoch": 40, "train-loss": 0.5})
```

`step` 및 `commit` 파라미터를 사용하면 여러 `wandb.Run.log()` 호출을 사용해 동일한 step에 로그를 기록할 수 있습니다. 다음은 모두 동일합니다:

```python
with wandb.init() as run:
     # 일반 사용:
     run.log({"train-loss": 0.5, "accuracy": 0.8})
     run.log({"train-loss": 0.4, "accuracy": 0.9})

     # 자동 증가 없는 암묵적 step:
     run.log({"train-loss": 0.5}, commit=False)
     run.log({"accuracy": 0.8})
     run.log({"train-loss": 0.4}, commit=False)
     run.log({"accuracy": 0.9})

     # 명시적 step:
     run.log({"train-loss": 0.5}, step=current_step)
     run.log({"accuracy": 0.8}, step=current_step)
     current_step += 1
     run.log({"train-loss": 0.4}, step=current_step)
     run.log({"accuracy": 0.9}, step=current_step, commit=True)
```

**매개변수:**


* `data`:  `str` 키와 직렬화 가능한 값을 가지는 `dict`
* `Python objects including`:  `int`, `float`, `string`, `wandb.data_types`의 모든 항목, 직렬화 가능한 Python 객체의 리스트, 튜플, NumPy 배열, 그리고 이러한 구조의 다른 `dict`
* `step`:  로깅할 step 번호입니다. `None`이면 암시적으로 자동 증가하는 step이 사용됩니다. 설명의 참고 사항을 참조하세요.
* `commit`:  true이면 해당 step을 완료하고 업로드합니다. false이면 해당 step의 데이터를 누적합니다. 설명의 참고 사항을 참조하세요. `step`이 `None`이면 기본값은 `commit=True`이고, 그렇지 않으면 기본값은 `commit=False`입니다.

**예시:**

### <kbd>방법</kbd> `Run.log_code`

```python
log_code(
    root: 'str | None' = '.',
    name: 'str | None' = None,
    include_fn: 'Callable[[str, str], bool] | Callable[[str], bool]' = <function _is_py_requirements_or_dockerfile at 0x10998dda0>,
    exclude_fn: 'Callable[[str, str], bool] | Callable[[str], bool]' = <function exclude_wandb_fn at 0x10b11bb00>
) → Artifact | None
```

코드의 현재 상태를 W&amp;B 아티팩트에 저장합니다.

기본적으로 현재 디렉터리를 순회하며 `.py`로 끝나는 모든 파일을 로그합니다.

**매개변수:**

* `root`:  코드를 재귀적으로 찾을 상대 경로(`os.getcwd()` 기준) 또는 절대 경로입니다.
* `name`:  (str, 선택) 코드 아티팩트의 이름입니다. 기본적으로 아티팩트 이름은 `source-$PROJECT_ID-$ENTRYPOINT_RELPATH`로 지정됩니다. 여러 run이 동일한 아티팩트를 공유해야 하는 경우가 있을 수 있습니다. `name`을 지정하면 이를 수행할 수 있습니다.
* `include_fn`:  파일 경로와 (선택적으로) 루트 경로를 받아, 포함해야 하면 True를 반환하고 그렇지 않으면 False를 반환하는 호출 가능한 객체입니다. 이것은
* `defaults to `lambda path, root`:  path.endswith(".py")`가 기본값입니다.
* `exclude_fn`:  파일 경로와 (선택적으로) 루트 경로를 받아, 제외해야 하면 `True`를 반환하고 그렇지 않으면 `False`를 반환하는 호출 가능한 객체입니다. 기본적으로 `<root>/.wandb/` 및 `<root>/wandb/` 디렉터리 내의 모든 파일을 제외하는 함수가 사용됩니다.

**예시:**
기본 사용법

```python
import wandb

with wandb.init() as run:
    run.log_code()
```

고급 활용

```python
import wandb

with wandb.init() as run:
    run.log_code(
         root="../",
         include_fn=lambda path: path.endswith(".py") or path.endswith(".ipynb"),
         exclude_fn=lambda path, root: os.path.relpath(path, root).startswith(
             "cache/"
         ),
    )
```

**반환값:**
코드가 로깅된 경우 `Artifact` 객체

### <kbd>방법</kbd> `Run.restore`

```python
restore(
    name: 'str',
    run_path: 'str | None' = None,
    replace: 'bool' = False,
    root: 'str | None' = None
) → None | TextIO
```

클라우드 저장소에서 지정된 파일을 다운로드합니다.

파일은 현재 디렉터리 또는 run 디렉터리에 저장됩니다. 기본적으로는 파일이 아직 존재하지 않는 경우에만 다운로드합니다.

* `replace`:  로컬에 이미 파일이 있더라도 파일을 다운로드할지 여부입니다.
* `root`:  파일을 다운로드할 디렉터리입니다.  기본값은 현재 디렉터리이며, wandb.init가 호출된 경우 run 디렉터리입니다.

**반환값:**
파일을 찾을 수 없으면 None을 반환하고, 그렇지 않으면 읽기용으로 열린 파일 객체를 반환합니다.

**예외:**

* `CommError`:  W&amp;B가 W&amp;B 백엔드에 연결할 수 없는 경우입니다.
* `ValueError`:  파일을 찾을 수 없거나 run&#95;path를 찾을 수 없는 경우입니다.

### <kbd>방법</kbd> `Run.save`

```python
save(
    glob_str: 'str | os.PathLike',
    base_path: 'str | os.PathLike | None' = None,
    policy: 'PolicyName' = 'live'
) → bool | list[str]
```

하나 이상의 파일을 W&amp;B에 동기화합니다.

상대 경로는 현재 작업 디렉터리를 기준으로 합니다.

&quot;myfiles/*&quot;와 같은 Unix glob은 `policy`와 관계없이 `save`가 호출될 때 확장됩니다. 특히 새 파일은 자동으로 선택되지 않습니다.

업로드된 파일의 디렉터리 구조를 제어하기 위해 `base_path`를 제공할 수 있습니다. 이는 `glob_str`의 접두사여야 하며, 그 아래의 디렉터리 구조는 유지됩니다.

절대 경로나 glob이 주어지고 `base_path`가 없으면, 위 예시와 같이 한 단계의 디렉터리 수준이 유지됩니다.

파일은 자동으로 중복 제거됩니다. 수정 없이 같은 파일에 대해 `save()`를 여러 번 호출해도 다시 업로드되지 않습니다.

**매개변수:**

* `glob_str`:  상대 경로, 절대 경로 또는 Unix glob입니다.
* `base_path`:  디렉터리 구조를 추론하는 데 사용할 경로입니다. 예시를 참조하세요.
* `policy`:  `live`, `now`, `end` 중 하나입니다.
  * live: 파일이 변경될 때 파일을 업로드하며 이전 버전을 덮어씁니다
  * now: 지금 파일을 한 번 업로드합니다
  * end: run이 종료될 때 파일을 업로드합니다

**반환값:**
일치하는 파일에 대해 생성된 심볼릭 링크의 경로입니다.

이전 버전 코드와의 호환성 때문에, 이 값은 불리언을 반환할 수도 있습니다.

```python
import wandb

run = wandb.init()

run.save("these/are/myfiles/*")
# => run의 "these/are/myfiles/" 폴더에 파일을 저장합니다.

run.save("these/are/myfiles/*", base_path="these")
# => run의 "are/myfiles/" 폴더에 파일을 저장합니다.

run.save("/Users/username/Documents/run123/*.txt")
# => run의 "run123/" 폴더에 파일을 저장합니다. 아래 참고 사항을 참조하세요.

run.save("/Users/username/Documents/run123/*.txt", base_path="/Users")
# => run의 "username/Documents/run123/" 폴더에 파일을 저장합니다.

run.save("files/*/saveme.txt")
# => "files/"의 적절한 하위 디렉터리에 각 "saveme.txt" 파일을 저장합니다.

# 컨텍스트 매니저를 사용하지 않으므로 run을 명시적으로 종료합니다.
run.finish()
```

### <kbd>방법</kbd> `Run.upsert_artifact`

```python
upsert_artifact(
    artifact_or_path: 'Artifact | str',
    name: 'str | None' = None,
    type: 'str | None' = None,
    aliases: 'list[str] | None' = None,
    distributed_id: 'str | None' = None
) → Artifact
```

최종 확정되지 않은 아티팩트를 run의 출력으로 선언하거나(또는 여기에 추가하거나) 합니다.

아티팩트를 최종 확정하려면 반드시 run.finish&#95;artifact()를 호출해야 합니다. 이는 분산 작업이 모두 동일한 아티팩트에 기여해야 할 때 유용합니다.

**매개변수:**

* `artifact_or_path`:  이 아티팩트의 콘텐츠 경로입니다. 다음 형식 중 하나일 수 있습니다:
  * `/local/directory`
  * `/local/directory/file.txt`
  * `s3://bucket/path`
* `name`:  아티팩트 이름입니다. &quot;entity/프로젝트&quot; 접두사를 붙일 수 있습니다. 지정하지 않으면 경로의 basename 앞에 current run ID를 붙인 값이 기본값으로 사용됩니다. 유효한 이름은 다음 형식 중 하나일 수 있습니다:
  * name:version
  * name:alias
  * digest
* `type`:  로깅할 아티팩트의 유형입니다. 일반적인 예시로는 `dataset`, `model`이 있습니다.
* `aliases`:  이 아티팩트에 적용할 별칭이며, 기본값은 `["latest"]`입니다.
* `distributed_id`:  모든 분산 작업이 공유하는 고유한 string입니다. None이면 run의 그룹 이름이 기본값으로 사용됩니다.

**반환값:**
`Artifact` 객체입니다.

### <kbd>방법</kbd> `Run.use_artifact`

```python
use_artifact(
    artifact_or_name: 'str | Artifact',
    type: 'str | None' = None,
    aliases: 'list[str] | None' = None,
    use_as: 'str | None' = None
) → Artifact
```

run의 입력으로 아티팩트를 선언합니다.

반환된 객체에서 `download` 또는 `file`을 호출하여 콘텐츠를 로컬로 가져오세요.

**매개변수:**

* `artifact_or_name`:  사용할 아티팩트의 이름입니다. 아티팩트가 로깅된 프로젝트의 이름(`entity` 또는 `entity/project`)을 앞에 붙일 수 있습니다. 이름에 entity가 지정되지 않으면 Run 또는 API 설정의 entity가 사용됩니다. 유효한 이름은 다음 형식일 수 있습니다.
  * name:version
  * name:alias
* `type`:  사용할 아티팩트의 유형입니다.
* `aliases`:  이 아티팩트에 적용할 별칭입니다.
* `use_as`:  이 인수는 사용 중단되었으며 아무 작업도 수행하지 않습니다.

**반환값:**
`Artifact` 객체입니다.

**예시:**

```python
import wandb

run = wandb.init(project="<example>")

# 이름과 별칭으로 아티팩트 사용
artifact_a = run.use_artifact(artifact_or_name="<name>:<alias>")

# 이름과 버전으로 아티팩트 사용
artifact_b = run.use_artifact(artifact_or_name="<name>:v<version>")

# entity/프로젝트/이름:별칭으로 아티팩트 사용
artifact_c = run.use_artifact(
    artifact_or_name="<entity>/<project>/<name>:<alias>"
)

# entity/프로젝트/이름:버전으로 아티팩트 사용
artifact_d = run.use_artifact(
    artifact_or_name="<entity>/<project>/<name>:v<version>"
)

# 컨텍스트 매니저를 사용하지 않으므로 run을 명시적으로 종료합니다.
run.finish()
```

### <kbd>방법</kbd> `Run.use_model`

```python
use_model(name: 'str') → FilePathStr
```

모델 아티팩트 &#39;name&#39;에 로깅된 파일을 다운로드합니다.

**매개변수:**

* `name`:  모델 아티팩트 이름입니다. &#39;name&#39;은 기존에 로깅된 모델 아티팩트의 이름과 일치해야 합니다. `entity/project/` 접두사를 붙일 수 있습니다. 유효한 이름은 다음 형식일 수 있습니다
  * model&#95;artifact&#95;name:version
  * model&#95;artifact&#95;name:alias

**반환값:**

* `path` (str):  다운로드한 모델 아티팩트 파일 경로입니다.

**예외:**

* `AssertionError`:  모델 아티팩트 &#39;name&#39;의 유형에 부분 문자열 &#39;model&#39;이 포함되어 있지 않은 경우입니다.

### <kbd>방법</kbd> `Run.watch`

```python
watch(
    models: 'torch.nn.Module | Sequence[torch.nn.Module]',
    criterion: 'torch.F | None' = None,
    log: "Literal['gradients', 'parameters', 'all'] | None" = 'gradients',
    log_freq: 'int' = 1000,
    idx: 'int | None' = None,
    log_graph: 'bool' = False
) → None
```

주어진 PyTorch 모델에 훅을 연결하여 그라디언트와 모델의 계산 그래프를 모니터링합니다.

이 함수는 트레이닝 중 파라미터, 그라디언트 또는 둘 다 추적할 수 있습니다.

**매개변수:**

* `models`:  모니터링할 단일 모델 또는 모델 시퀀스입니다.
* `criterion`:  최적화되는 loss 함수입니다(선택).
* `log`:  `"gradients"`, `"parameters"`, 또는 `"all"`을 log할지 지정합니다. logging을 비활성화하려면 None으로 설정하세요. (default=`"gradients"`).
* `log_freq`:  그라디언트와 파라미터를 log하는 빈도(배치 기준)입니다. (default=1000)
* `idx`:  `wandb.watch`로 여러 모델을 추적할 때 사용하는 인덱스입니다. (default=None)
* `log_graph`:  모델의 계산 그래프를 log할지 여부입니다. (default=False)

**예외:**
`wandb.init()`이 호출되지 않았거나 모델 중 하나라도 `torch.nn.Module`의 인스턴스가 아닌 경우: ValueError입니다.
