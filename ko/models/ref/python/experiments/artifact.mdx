## <kbd>class</kbd> `Artifact`

데이터셋과 모델 버전 관리를 위한 유연하고 경량의 빌딩 블록입니다. 

비어 있는 W&B Artifact를 생성합니다. `add` 로 시작하는 메서드들을 사용해 아티팩트의 내용을 채웁니다. 아티팩트에 필요한 파일을 모두 추가한 뒤에는 `run.log_artifact()`를 호출해 로깅할 수 있습니다.

### <kbd>method</kbd> `Artifact.__init__`

```python
__init__(
    name: 'str',
    type: 'str',
    description: 'str | None' = None,
    metadata: 'dict[str, Any] | None' = None,
    incremental: 'bool' = False,
    use_as: 'str | None' = None,
    storage_region: 'str | None' = None
) → None
```

**인자:**

* `name` (str):  아티팩트의 사람이 이해하기 쉬운 이름입니다. 이 이름을 사용해 W&amp;B App UI 또는 코드에서 특정 아티팩트를 식별합니다. `use_artifact` Public API로 아티팩트를 인터랙티브하게 참조할 수 있습니다. 이름에는 문자, 숫자, 밑줄, 하이픈, 마침표를 포함할 수 있습니다. 이름은 프로젝트 전체에서 고유해야 합니다.
* `type` (str):  아티팩트의 타입입니다. 아티팩트의 타입을 사용해 아티팩트를 구성하고 구분합니다. 문자, 숫자, 밑줄, 하이픈, 마침표를 포함하는 임의의 문자열을 사용할 수 있습니다. 일반적인 타입에는 `dataset` 또는 `model`이 있습니다. 아티팩트를 W&amp;B Model Registry에 연결하려면 타입 문자열에 `model`을 포함하세요. 일부 타입은 내부 용도로 예약되어 있으며 사용자가 설정할 수 없습니다. 이러한 타입에는 `job` 및 `wandb-`로 시작하는 타입이 포함됩니다.
* `description (str | None) = None`:  아티팩트에 대한 설명입니다. Model 또는 Dataset Artifact의 경우, 팀에서 표준화한 모델 또는 데이터셋 카드에 대한 문서를 추가합니다. `Artifact.description` 속성을 사용하거나 W&amp;B App UI에서 아티팩트의 설명을 확인할 수 있습니다. W&amp;B는 W&amp;B App에서 설명을 마크다운으로 렌더링합니다.
* `metadata (dict[str, Any] | None) = None`:  아티팩트에 대한 추가 정보입니다. 메타데이터는 키-값 쌍의 딕셔너리로 지정합니다. 키는 총 100개를 초과할 수 없습니다.
* `incremental`:  기존 아티팩트를 수정하려면 대신 `Artifact.new_draft()` 메서드를 사용하세요.
* `use_as`:  더 이상 사용되지 않습니다(Deprecated).

**반환값:**
`Artifact` 객체를 반환합니다.

### <kbd>property</kbd> Artifact.aliases

하나 이상의 의미를 이해하기 쉬운 참조값 또는 

아티팩트 버전에 할당된 식별용 "별칭" 목록입니다. 

별칭은 코드에서 참조할 수 있는 변경 가능한 참조입니다. W&B App UI 또는 프로그래밍 방식으로 아티팩트의 별칭을 변경할 수 있습니다. 자세한 내용은 [Create new artifact versions](https://docs.wandb.ai/guides/artifacts/create-a-new-artifact-version)를 참조하세요. 

**반환값:**

- `list[str]`: aliases 프로퍼티 값입니다.

### <kbd>property</kbd> Artifact.collection

이 아티팩트를 가져온 컬렉션입니다. 

컬렉션은 아티팩트 버전들로 이루어진 순서 있는 그룹입니다. 이 아티팩트가 연결된 컬렉션에서 조회된 경우에는 해당 컬렉션을 반환합니다. 그렇지 않으면, 해당 아티팩트 버전이 처음 생성된 컬렉션을 반환합니다. 

아티팩트가 처음 생성된 컬렉션을 소스 시퀀스(source sequence)라고 합니다. 

**반환값:**

- `ArtifactCollection`: collection 프로퍼티 값입니다.

### <kbd>method</kbd> `Artifact.add`

```python
add(
    obj: 'WBValue',
    name: 'StrPath',
    overwrite: 'bool' = False
) → ArtifactManifestEntry
```

wandb.WBValue `obj`를 아티팩트에 추가합니다.

**Args:**

* `obj`:  추가할 객체입니다. 현재는 Bokeh, JoinedTable, PartitionedTable, Table, Classes, ImageMask, BoundingBoxes2D, Audio, Image, Video, Html, Object3D 중 하나를 지원합니다.
* `name`:  객체를 추가할 아티팩트 내부 경로입니다.
* `overwrite`:  True인 경우, 해당하는 경우 동일한 파일 경로를 가진 기존 객체를 덮어씁니다.

**Returns:**
추가된 매니페스트 엔트리

**Raises:**

* `ArtifactFinalizedError`:  현재 아티팩트 버전이 finalized 상태이므로 변경할 수 없습니다. 대신 새로운 아티팩트 버전을 로그로 남기세요.

### <kbd>메서드</kbd> `Artifact.add_dir`

```python
add_dir(
    local_path: 'str',
    name: 'str | None' = None,
    skip_cache: 'bool | None' = False,
    policy: "Literal['mutable', 'immutable'] | None" = 'mutable',
    merge: 'bool' = False
) → None
```

아티팩트에 로컬 디렉터리를 추가합니다.

**인자:**

* `local_path`:  로컬 디렉터리의 경로.
* `name`:  아티팩트 내의 하위 디렉터리 이름. 지정한 이름은 아티팩트의 `type`에 따라 계층 구조로 W&amp;B 앱 UI에 표시됩니다. 기본값은 아티팩트의 루트입니다.
* `skip_cache`:  `True`로 설정하면 업로드 중에 W&amp;B가 파일을 캐시로 복사하거나 이동하지 않습니다.
* `policy`:  기본값은 &quot;mutable&quot;입니다.
  * mutable: 업로드 중 손상을 방지하기 위해 파일의 임시 복사본을 생성합니다.
  * immutable: 보호를 비활성화하고, 사용자가 파일을 삭제하거나 변경하지 않는다는 점에 의존합니다.
* `merge`:  `False`(기본값)인 경우, 이전 `add_dir` 호출에서 이미 추가된 파일의 내용이 변경되었을 때 `ValueError`를 발생시킵니다. `True`인 경우, 변경된 내용을 가진 기존 파일을 덮어씁니다. 항상 새로운 파일은 추가하며, 파일은 제거하지 않습니다. 전체 디렉터리를 교체하려면, `add_dir(local_path, name=my_prefix)`와 같이 디렉터리를 추가할 때 `name`을 전달하고, `remove(my_prefix)`를 호출하여 디렉터리를 제거한 뒤 다시 추가하십시오.

**예외:**

* `ArtifactFinalizedError`:  현재 아티팩트 버전이 이미 최종 확정 상태이므로 더 이상 변경할 수 없습니다. 대신 새 아티팩트 버전을 기록하십시오.
* `ValueError`:  `policy`는 &quot;mutable&quot; 또는 &quot;immutable&quot;이어야 합니다.

### <kbd>메서드</kbd> `Artifact.add_file`

```python
add_file(
    local_path: 'str',
    name: 'str | None' = None,
    is_tmp: 'bool | None' = False,
    skip_cache: 'bool | None' = False,
    policy: "Literal['mutable', 'immutable'] | None" = 'mutable',
    overwrite: 'bool' = False
) → ArtifactManifestEntry
```

Artifact에 로컬 파일을 추가합니다.

**Args:**

* `local_path`:  추가할 파일의 경로입니다.
* `name`:  추가할 파일에 대해 Artifact 내에서 사용할 경로입니다. 기본값은 파일의 basename입니다.
* `is_tmp`:  `True`이면 충돌을 피하기 위해 파일 이름을 결정론적으로 변경합니다.
* `skip_cache`:  `True`이면 업로드 후 파일을 캐시에 복사하지 않습니다.
* `policy`:  기본값은 &quot;mutable&quot;입니다. &quot;mutable&quot;로 설정하면 업로드 중 손상을 방지하기 위해 파일의 임시 복사본을 만듭니다. &quot;immutable&quot;로 설정하면 이 보호를 비활성화하고, 사용자가 파일을 삭제하거나 변경하지 않는다는 가정에 의존합니다.
* `overwrite`:  `True`이면 동일한 파일이 이미 존재하더라도 덮어씁니다.

**Returns:**
추가된 manifest 엔트리입니다.

**Raises:**

* `ArtifactFinalizedError`:  현재 Artifact 버전이 최종 확정되었으므로 변경할 수 없습니다. 대신 새로운 Artifact 버전을 기록하세요.
* `ValueError`:  Policy는 &quot;mutable&quot; 또는 &quot;immutable&quot;이어야 합니다.

### <kbd>메서드</kbd> `Artifact.add_reference`

```python
add_reference(
    uri: 'ArtifactManifestEntry | str',
    name: 'StrPath | None' = None,
    checksum: 'bool' = True,
    max_objects: 'int | None' = None
) → Sequence[ArtifactManifestEntry]
```

URI로 지정된 참조를 아티팩트에 추가합니다.

아티팩트에 추가하는 파일이나 디렉터리와 달리, 참조는 W&amp;B에 업로드되지 않습니다. 자세한 내용은 [외부 파일 추적](https://docs.wandb.ai/guides/artifacts/track-external-files)을 참고하세요.

기본적으로 다음 스킴이 지원됩니다:

* http(s): 파일의 크기와 다이제스트는 서버에서 반환하는 `Content-Length` 및 `ETag` 응답 헤더로부터 추론됩니다.
* s3: 체크섬과 크기는 객체 메타데이터에서 가져옵니다. 버킷 버저닝이 활성화된 경우 버전 ID도 함께 추적됩니다.
* gs: 체크섬과 크기는 객체 메타데이터에서 가져옵니다. 버킷 버저닝이 활성화된 경우 버전 ID도 함께 추적됩니다.
* https, 도메인이 `*.blob.core.windows.net`와 일치하는 경우
* Azure: 체크섬과 크기는 blob 메타데이터에서 가져옵니다. 스토리지 계정 버저닝이 활성화된 경우 버전 ID도 함께 추적됩니다.
* file: 체크섬과 크기는 파일 시스템에서 가져옵니다. 이 스킴은 추적은 하되 반드시 업로드하지는 않을 파일을 포함한 NFS 공유나 기타 외부 마운트 볼륨이 있을 때 유용합니다.

그 외 다른 스킴의 경우, 다이제스트는 URI의 해시값일 뿐이며 크기는 비워 둡니다.

**Args:**

* `uri`:  추가할 참조의 URI 경로입니다. URI 경로는 다른 아티팩트의 엔트리에 대한 참조를 저장하기 위해 `Artifact.get_entry`가 반환한 객체가 될 수 있습니다.
* `name`:  이 참조의 내용을 아티팩트 내에서 배치할 경로입니다.
* `checksum`:  참조 URI에 위치한 리소스에 대해 체크섬을 수행할지 여부입니다. 체크섬을 수행하면 자동 무결성 검증이 가능하므로 강력히 권장됩니다. 체크섬을 비활성화하면 아티팩트 생성 속도는 빨라지지만, 참조 디렉터리를 순회하지 않으므로 디렉터리 내 객체는 아티팩트에 저장되지 않습니다. 참조 객체를 추가할 때는 `checksum=False`로 설정할 것을 권장하며, 이 경우 참조 URI가 변경될 때에만 새 버전이 생성됩니다.
* `max_objects`:  디렉터리 또는 버킷 스토어 프리픽스를 가리키는 참조를 추가할 때 고려할 최대 객체 수입니다. 기본적으로 Amazon S3, GCS, Azure, 로컬 파일에 대해 허용되는 최대 객체 수는 10,000,000입니다. 다른 URI 스킴에는 최대값이 없습니다.

**Returns:**
추가된 매니페스트 엔트리입니다.

**Raises:**

* `ArtifactFinalizedError`:  현재 아티팩트 버전이 이미 finalized 상태이므로 수정할 수 없습니다. 대신 새 아티팩트 버전을 로깅하세요.

현재 실행 중인 실행이 있으면, 해당 실행이 이 Artifact를 로그합니다. 현재 어떤 실행도 없는 경우, 이 Artifact를 추적하기 위해 타입이 &quot;auto&quot;인 실행이 자동으로 생성됩니다.

**인자:**

* `project`:  현재 컨텍스트에 실행이 없는 경우 Artifact에 사용할 프로젝트입니다.
* `settings`:  자동 실행을 초기화할 때 사용할 settings 객체입니다. 주로 테스트 하네스에서 사용됩니다.

### <kbd>메서드</kbd> `Artifact.verify`

```python
verify(root: 'str | None' = None) → None
```

Artifact의 내용이 manifest와 일치하는지 검증합니다.

디렉터리 내의 모든 파일에 대해 체크섬을 계산한 다음, 해당 체크섬을 Artifact의 manifest와 대조합니다. 참조 항목은 검증하지 않습니다.

**Args:**

* `root`:  검증할 디렉터리입니다. None이면 Artifact가 &#39;./artifacts/self.name/&#39; 경로로 다운로드됩니다.

**Raises:**

* `ArtifactNotLoggedError`:  Artifact가 로깅되지 않은 경우.
* `ValueError`:  검증에 실패한 경우.
