W&amp;B Public API는 W&amp;B에 저장된 데이터를 쿼리하고, 내보내고, 업데이트하기 위해 프로그래밍 방식으로 액세스할 수 있게 해줍니다. 이 API를 사후 분석, 데이터 내보내기, 실행·아티팩트·스윕의 프로그래밍 방식 관리를 위해 사용하세요. 기본 SDK가 학습 중 실시간 로깅을 처리하는 반면, Public API를 사용하면 과거 데이터를 조회하고 메타데이터를 업데이트하며 아티팩트를 관리하고 완료된 실험을 분석할 수 있습니다. 기본 `Api` 클래스는 대부분의 기능에 대한 진입점 역할을 합니다.

<div id="authentication">
  ## 인증
</div>

Public API는 Python SDK와 동일한 인증 메커니즘을 사용합니다. 여러 방법으로 인증할 수 있습니다:

`WANDB_API_KEY` 환경 변수를 사용해 API 키를 설정하세요:

```bash
export WANDB_API_KEY=your_api_key
```

`Api` 클래스를 초기화할 때 API 키를 직접 전달하세요:

```python
api = Api(api_key="your_api_key")
```

또는 현재 세션을 인증하려면 `wandb.login()`을 호출하세요:

```python
import wandb

wandb.login()
api = Api()
```

  ## 사용 예
</div>

<div id="download-an-artifact-by-name-and-alias">
  ### 이름과 별칭으로 Artifact 다운로드
</div>

다음 예제는 이름과 별칭을 사용해 W&amp;B에 로깅된 아티팩트를 가져와 그 내용을 다운로드하는 방법을 보여줍니다.

```python
import wandb

api = wandb.Api()
artifact = api.artifact("entity/project/artifact:alias")
artifact.download()
```

<div id="download-an-artifact-from-a-registry">
  ### 레지스트리에서 Artifact 다운로드
</div>

다음 예제는 W&amp;B Registry에서 링크된 아티팩트를 가져오는 방법을 보여줍니다.

```python
import wandb

REGISTRY = "<registry_name>"
COLLECTION = "<collection_name>"
VERSION = "<version>"

api = wandb.Api()
artifact_name = f"wandb-registry-{REGISTRY}/{COLLECTION}:{VERSION}"

# 아티팩트 가져오기
fetched_artifact = api.artifact(name = artifact_name)

# 아티팩트 다운로드. 다운로드된 콘텐츠의 경로를 반환
downloaded_path = fetched_artifact.download()
```

<div id="query-wb-registry">
  ### W&amp;B Registry 조회
</div>

MongoDB와 유사한 필터를 사용해서 W&amp;B Registry, Collection, 그리고 아티팩트를 조회할 수 있습니다. 다음 예시는 정규식을 사용해 이름으로 컬렉션을 필터링하는 방법을 보여줍니다.

```python
import wandb

# wandb API 초기화
api = wandb.Api()

# 레지스트리에 관계없이 컬렉션 이름에 `yolo` 문자열이
# 포함된 모든 컬렉션 필터링
collection_filters = {
    "name": {"$regex": "yolo"}
}

# 필터 조건에 일치하는 모든 컬렉션의 이터러블 반환
collections = api.registries().collections(filter=collection_filters)
```

레지스트리, 컬렉션 또는 아티팩트를 쿼리하는 방법에 대한 자세한 내용은 [레지스트리 항목 찾기](/ko/models/registry/search_registry) 문서를 참조하세요.
