W&amp;B Automations API를 사용하면 ML 파이프라인의 이벤트에 대응하는 자동화된 워크플로를 프로그래밍 방식으로 생성하고 관리할 수 있습니다. 모델 성능 임계값이나 아티팩트 생성과 같은 특정 조건이 충족될 때 트리거될 작업을 구성할 수 있습니다.

<div id="core-classes">
  ### 핵심 클래스
</div>

| Class | Description |
|-------|-------------|
| [`Automation`](/ko/models/ref/python/automations/automation/) | 구성과 함께 저장된 자동화 인스턴스를 나타냅니다. |
| [`NewAutomation`](/ko/models/ref/python/automations/newautomation/) | 새 자동화를 생성하기 위한 빌더 클래스입니다. |

<div id="events-triggers">
  ### 이벤트(트리거)
</div>

| Event | Description |
|-------|-------------|
| [`OnRunMetric`](/ko/models/ref/python/automations/onrunmetric/) | 실행 메트릭이 정의된 조건(임계값, 변화 등)을 만족할 때 트리거됩니다. |
| [`OnCreateArtifact`](/ko/models/ref/python/automations/oncreateartifact/) | 컬렉션에 새 아티팩트가 생성될 때 트리거됩니다. |
| [`OnLinkArtifact`](/ko/models/ref/python/automations/onlinkartifact/) | 아티팩트가 레지스트리에 연결될 때 트리거됩니다. |
| [`OnAddArtifactAlias`](/ko/models/ref/python/automations/onaddartifactalias/) | 아티팩트에 별칭이 추가될 때 트리거됩니다. |

<div id="actions">
  ### 작업
</div>

| 작업 | 설명 |
|--------|-------------|
| [`SendNotification`](/ko/models/ref/python/automations/sendnotification/) | Slack 또는 다른 통합 채널을 통해 알림을 전송합니다. |
| [`SendWebhook`](/ko/models/ref/python/automations/sendwebhook/) | 외부 서비스로 HTTP 웹훅 요청을 전송합니다. |
| [`DoNothing`](/ko/models/ref/python/automations/donothing/) | 자동화 구성을 테스트하기 위한 플레이스홀더 작업입니다. |

<div id="filters">
  ### 필터
</div>

| 필터 | 설명 |
|--------|-------------|
| [`MetricThresholdFilter`](/ko/models/ref/python/automations/metricthresholdfilter/) | 메트릭 값을 임계값과 비교한 결과를 기준으로 실행을 필터링합니다. |
| [`MetricChangeFilter`](/ko/models/ref/python/automations/metricchangefilter/) | 시간에 따른 메트릭 값의 변화를 기준으로 실행을 필터링합니다. |

<div id="common-use-cases">
  ## 주요 사용 사례
</div>

<div id="model-performance-monitoring">
  ### 모델 성능 모니터링
</div>

* 모델 정확도가 임계값 아래로 떨어지면 알림 전송
* 학습 손실이 더 이상 감소하지 않고 정체되면 팀에 알림
* 성능 지표를 기준으로 재학습 파이프라인 실행

<div id="artifact-management">
  ### 아티팩트 관리
</div>

* 새로운 모델 버전이 생성될 때 알림을 보내도록 설정
* 아티팩트에 태그가 달리면 배포 워크플로우를 시작
* 데이터셋이 업데이트되면 후속(다운스트림) 처리를 자동화

<div id="experiment-tracking">
  ### 실험 추적
</div>

* 실패하거나 비정상 종료된 실행에 대한 알림 받기
* 장시간 실행되는 실험이 완료되면 알림 받기
* 실험 지표의 일일 요약 수신하기

<div id="integration-workflows">
  ### 통합 워크플로우
</div>

* 웹훅으로 외부 추적 시스템을 업데이트합니다
* 배포 플랫폼과 모델 레지스트리를 동기화합니다
* W&amp;B 이벤트를 기준으로 CI/CD 파이프라인을 트리거합니다

<div id="example-usage">
  ## 사용 예시
</div>

다음 예시는 `custom-metric`이라는 메트릭이 10을 초과할 때마다 Slack 알림을 보내는 자동화를 설정합니다. `custom-metric`은 학습 중에 `wandb.Run.log({"custom-metric": value })`를 사용해 로그된다고 가정합니다.

```python
import wandb
from wandb.automations import OnRunMetric, RunEvent, SendNotification

api = wandb.Api()

project = api.project("<my-project>", entity="<my-team>")

# 팀의 첫 번째 Slack 통합 사용
slack_hook = next(api.slack_integrations(entity="<my-team>"))

# 트리거 이벤트 생성
event = OnRunMetric(
     scope=project,
     filter=RunEvent.metric("custom-metric") > 10,
)

# 이벤트에 응답하는 액션 생성
action = SendNotification.from_integration(slack_hook)

# 자동화 생성
automation = api.create_automation(
     event >> action,
     name="my-automation",
     description="'custom-metric'이 10을 초과할 때마다 Slack 메시지 전송.",
)
```

Automations API 사용에 대한 자세한 내용은 [Automations 가이드](/ko/models/automations/)를 참조하세요.
