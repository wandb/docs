프로그래밍 방식으로 W&amp;B Reports API를 사용할 수 있는 Python 라이브러리입니다.

```python
import wandb_workspaces.reports.v2 as wr

report = wr.Report(
     entity="entity",
     project="project",
     title="An amazing title",
     description="A descriptive description.",
)

blocks = [
     wr.PanelGrid(
         panels=[
             wr.LinePlot(x="time", y="velocity"),
             wr.ScatterPlot(x="time", y="acceleration"),
         ]
     )
]

report.blocks = blocks
report.save()
```


## <kbd>class</kbd> `BarPlot`

2D 막대 플롯을 표시하는 패널 객체입니다.

**속성:**

* `title` (Optional[str]):  플롯 상단에 표시되는 텍스트입니다.
* `metrics` (LList[MetricType]):  orientation Literal[&quot;v&quot;, &quot;h&quot;]: 막대 플롯의 방향입니다.  세로 (&quot;v&quot;) 또는 가로 (&quot;h&quot;)로 설정하세요. 기본값은 가로 (&quot;h&quot;)입니다.
* `range_x` (Tuple[float | None, float | None]):  x축 범위를 지정하는 Tuple입니다.
* `title_x` (Optional[str]):  x축의 레이블입니다.
* `title_y` (Optional[str]):  y축의 레이블입니다.
* `groupby` (Optional[str]):  리포트가 정보를 가져오는 W&amp;B 프로젝트에 로깅된 메트릭을 기준으로 Runs를 그룹화합니다.
* `groupby_aggfunc` (Optional[GroupAgg]):  지정된 함수로 Runs를 집계합니다. 옵션에는 &quot;mean&quot;, &quot;min&quot;, &quot;max&quot;, &quot;median&quot;, &quot;sum&quot;, &quot;samples&quot; 또는 `None`이 포함됩니다.
* `groupby_rangefunc` (Optional[GroupArea]):   범위를 기준으로 Runs를 그룹화합니다. 옵션에는 &quot;minmax&quot;, &quot;stddev&quot;, &quot;stderr&quot;, &quot;none&quot;, &quot;samples&quot; 또는 `None`이 포함됩니다.
* `max_runs_to_show` (Optional[int]):  플롯에 표시할 Runs의 최대 개수입니다.
* `max_bars_to_show` (Optional[int]):  막대 플롯에 표시할 막대의 최대 개수입니다.
* `custom_expressions` (Optional[LList[str]]):  막대 플롯에서 사용할 맞춤형 표현식 목록입니다.
* `legend_template` (Optional[str]):  범례용 템플릿입니다.
* `font_size` ( Optional[FontSize]):  선형 플롯 글꼴의 크기입니다.  옵션에는 &quot;small&quot;, &quot;medium&quot;, &quot;large&quot;, &quot;auto&quot; 또는 `None`이 포함됩니다.
* `line_titles` (Optional[dict]):  선의 제목입니다. 키는 선 이름이고 값은 제목입니다.
* `line_colors` (Optional[dict]):  선의 색상입니다. 키는 선 이름이고 값은 색상입니다.
* `aggregate` (Optional[bool]):  `True`로 설정하면 데이터를 집계합니다.

## <kbd>class</kbd> `LinePlot`

2D 선형 플롯이 있는 패널 객체입니다.

**속성:**

* `title` (Optional[str]):  플롯 상단에 표시되는 텍스트입니다.
* `x` (Optional[MetricType]):  리포트가 정보를 가져오는 W&amp;B 프로젝트에 로깅된 메트릭의 이름입니다. 지정한 메트릭은 x축에 사용됩니다.
* `y` (LList[MetricType]):  리포트가 정보를 가져오는 W&amp;B 프로젝트에 로깅된 하나 이상의 메트릭입니다. 지정한 메트릭은 Y축에 사용됩니다.
* `range_x` (Tuple[float | `None`, float | `None`]):  x축 범위를 지정하는 튜플입니다.
* `range_y` (Tuple[float | `None`, float | `None`]):  Y축 범위를 지정하는 튜플입니다.
* `log_x` (Optional[bool]):  x 좌표를 밑이 10인 로그 스케일로 플롯합니다.
* `log_y` (Optional[bool]):  y 좌표를 밑이 10인 로그 스케일로 플롯합니다.
* `title_x` (Optional[str]):  x축의 레이블입니다.
* `title_y` (Optional[str]):  Y축의 레이블입니다.
* `ignore_outliers` (Optional[bool]):  `True`로 설정하면 이상값을 플롯하지 않습니다.
* `groupby` (Optional[str]):  리포트가 정보를 가져오는 W&amp;B 프로젝트에 로깅된 메트릭을 기준으로 Runs를 그룹화합니다.
* `groupby_aggfunc` (Optional[GroupAgg]):  지정된 함수로 Runs를 집계합니다. 옵션에는 &quot;mean&quot;, &quot;min&quot;, &quot;max&quot;, &quot;median&quot;, &quot;sum&quot;, &quot;samples&quot; 또는 `None`이 포함됩니다.
* `groupby_rangefunc` (Optional[GroupArea]):   범위를 기준으로 Runs를 그룹화합니다. 옵션에는 &quot;minmax&quot;, &quot;stddev&quot;, &quot;stderr&quot;, &quot;none&quot;, &quot;samples&quot; 또는 `None`이 포함됩니다.
* `smoothing_factor` (Optional[float]):  smoothing 유형에 적용할 smoothing 계수입니다. 허용되는 값의 범위는 0에서 1 사이입니다.
* `smoothing_type Optional[SmoothingType]`:  지정된 분포를 기준으로 필터를 적용합니다. 옵션에는 &quot;exponentialTimeWeighted&quot;, &quot;exponential&quot;,  &quot;gaussian&quot;, &quot;average&quot; 또는 &quot;none&quot;이 포함됩니다.
* `smoothing_show_original` (Optional[bool]):    `True`로 설정하면 원본 데이터를 표시합니다.
* `max_runs_to_show` (Optional[int]):  선형 플롯에 표시할 최대 Runs 수입니다.
* `custom_expressions` (Optional[LList[str]]):  데이터에 적용할 맞춤형 표현식입니다.
* `plot_type Optional[LinePlotStyle]`:  생성할 선형 플롯의 유형입니다. 옵션에는 &quot;line&quot;, &quot;stacked-area&quot; 또는 &quot;pct-area&quot;가 포함됩니다.
* `font_size Optional[FontSize]`:  선형 플롯 글꼴의 크기입니다. 옵션에는 &quot;small&quot;, &quot;medium&quot;, &quot;large&quot;, &quot;auto&quot; 또는 `None`이 포함됩니다.
* `legend_position Optional[LegendPosition]`:  범례를 배치할 위치입니다. 옵션에는 &quot;north&quot;, &quot;south&quot;, &quot;east&quot;, &quot;west&quot; 또는 `None`이 포함됩니다.
* `legend_template` (Optional[str]):  범례용 템플릿입니다.
* `aggregate` (Optional[bool]):  `True`로 설정하면 데이터를 집계합니다.
* `xaxis_expression` (Optional[str]):  x축용 표현식입니다.
* `xaxis_format` (Optional[str]):  x축의 형식입니다. 이 옵션은 맞춤형 메트릭을 정의하면 표시됩니다. 예를 들어, x축를 날짜 및 시간으로 형식화하려면 &#39;datetime&#39;을 지정할 수 있습니다.
* `legend_fields` (Optional[LList[str]]):  범례에 포함할 필드입니다.
* `metric_regex` (Optional[str]):  Y축 메트릭과 일치하는 정규식 패턴입니다. 백엔드는 이 패턴을 사용해 일치하는 메트릭을 선택합니다.
* `point_visualization_method` (Optional[PointVizMethod]):  표시하기에 포인트가 너무 많을 때 포인트를 집계하는 데 사용하는 방법입니다. 옵션에는 &quot;bucketing-gorilla&quot;(이상값과 스파이크를 유지하기 위해 데이터 포인트를 버킷으로 나누고 버킷별 최소, 최대, 평균을 표시) 또는 &quot;sampling&quot;(더 빠른 render를 위해 포인트를 무작위로 샘플링하지만 이상값을 놓칠 수 있음)이 포함됩니다.

## <kbd>class</kbd> `Runset`

패널 그리드에 표시할 run 집합입니다.

**속성:**

* `entity` (str):  runs가 저장된 프로젝트를 소유하거나 해당 프로젝트에 대한 적절한 권한이 있는 entity입니다.
* `project` (str):  runs가 저장된 프로젝트의 이름입니다.
* `name` (str):  run 집합의 이름입니다. 기본값은 `Run set`입니다.
* `query` (str):  runs를 필터링하기 위한 쿼리 문자열입니다.
* `filters` (Union[str, LList[expr.FilterExpr]]):  runs에 적용할 필터입니다. 다음 중 하나일 수 있습니다:
  * 문자열 표현식: 예: &quot;Config(&#39;lr&#39;) = 0.001 and State = &#39;finished&#39;&quot;
* `Supports operators`: `=`, `==`, `!=`, `<`, `>`, `<=`, `>=`, `in`, `not in`
  * FilterExpr 객체 목록: 예: [expr.Config(&#39;lr&#39;) == 0.001]
* `groupby` (LList[str]):  그룹화 기준으로 사용할 metric 이름 목록입니다. 지원되는 형식은 다음과 같습니다:
  * run 속성을 기준으로 그룹화하려면 &quot;group&quot; 또는 &quot;run.group&quot;
  * 설정 매개변수를 기준으로 그룹화하려면 &quot;config.param&quot;
  * summary metric을 기준으로 그룹화하려면 &quot;summary.metric&quot;
* `order` (LList[OrderBy]):  정렬 기준으로 사용할 `OrderBy` 객체 목록입니다.
* `custom_run_colors` (dict):  run ID(또는 그룹화된 runs의 경우 `RunsetGroup` 키)를 색상에 매핑하는 사전입니다. run별로 단순하게 색상을 지정할 때는 `run_settings`를 사용하는 것이 좋습니다.
* `run_settings` (Dict[str, RunSettings]):  run ID를 키로 하는 run별 표시 설정입니다. 개별 runs의 색상을 설정하거나 숨기려면 이것을 사용하세요. `RunSettings`를 참조하세요.

**예시:**

```python
    # 문자열 필터 사용
    wr.Runset(
         entity="my-entity",
         project="my-project",
         filters="Config('learning_rate') = 0.001 and State = 'finished'"
    )

    # FilterExpr 목록 사용
    wr.Runset(
         entity="my-entity",
         project="my-project",
         filters=[expr.Config("learning_rate") == 0.001]
    )
```

## <kbd>class</kbd> `WeavePanelSummaryTable`

W&amp;B에 로깅된 W&amp;B Table, pandas 데이터프레임, 플롯 또는 기타 값을 표시하는 패널입니다. 쿼리는 다음 형식을 취합니다.

```python
runs.summary['value']
```

API 이름의 &quot;Weave&quot;라는 용어는 LLM을 추적하고 평가하는 데 사용하는 W&amp;B Weave 툴킷을 가리키는 것이 아닙니다.

**속성:**

* `table_name` (str):  테이블, 데이터프레임, 플롯 또는 값의 이름입니다.
* `layout` (Layout):  위치와 크기를 포함한 패널의 레이아웃 설정입니다. `Panel` 기본 클래스에서 상속됩니다. 너비(`w`)와 높이(`h`)를 조정할 때 사용하세요.

**예시:**
맞춤형 크기의 테이블을 표시하는 summary table 패널이 포함된 리포트를 만드세요:

```python
    import wandb_workspaces.reports as wr

    report = wr.Report(
         project="my-project",
         entity="my-entity",
         title="Summary Table Report"
    )

    report.blocks = [
         wr.PanelGrid(
             runsets=[wr.Runset(project="my-project")],
             panels=[
                 wr.WeavePanelSummaryTable(
                     table_name="my-table-name",
                     layout=wr.Layout(w=24, h=20)
                 )
             ]
         )
    ]

    report.save()
```

레이아웃 파라미터는 패널의 크기와 위치를 제어합니다:

* w (width): 그리드 단위의 너비(기본값: 8, 최대값: 24)
  * h (height): 그리드 단위의 높이(기본값: 6)
  * x (x-position): 그리드에서의 가로 위치(기본값: 0)
  * y (y-position): 그리드에서의 세로 위치(기본값: 0)
