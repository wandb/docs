<div id="usage">
  ## 사용
</div>

```bash
login [OPTIONS] [KEY]...
```

<div id="description">
  ## 설명
</div>

사용 중인 머신을 W&amp;B에 인증합니다.

W&amp;B 서비스 인증에 사용할 API 키를 로컬에 저장합니다.
기본적으로 자격 증명은 서버 측 검증 없이 저장됩니다.

인수로 API 키를 제공하지 않으면, 이 명령은 다음 순서로
자격 증명을 찾습니다:

1. `WANDB_API_KEY` 환경 변수

   2. 시스템 또는 Workspace 설정 파일의 `api_key` 설정(사용 중인 설정 파일을 확인하려면
      `wandb status`를 실행하세요)

   3. `.netrc` 파일(`~/.netrc`, `~/_netrc` 또는 `NETRC` 환경 변수 경로)

   4. 대화형 프롬프트(TTY를 사용할 수 있는 경우)

자체 호스팅 또는 Dedicated Cloud 배포의 경우, `--host`로 서버
URL을 지정하거나 `WANDB_BASE_URL` 환경 변수를 설정하세요.

예를 들어, 대화형으로 로그인하려면(API 키 입력 프롬프트가 표시됨):

```bash
wandb login
```

명시적으로 지정한 API 키(WANDB&#95;API&#95;KEY&#95;EXAMPLE)로 로그인하려면 다음을 실행하세요:

```bash
wandb login WANDB_API_KEY_EXAMPLE
```

로그인하고 API 키가 유효한지 확인하려면 다음을 수행하세요:

```bash
wandb login --verify
```

현재 설정된 자체 호스팅 인스턴스 대신 W&amp;B 퍼블릭 클라우드에 로그인하려면:

```bash
wandb login --cloud
```

자체 호스팅 W&amp;B 인스턴스에 로그인하려면:

```bash
wandb login --host https://my-wandb-server.example.com
```

이미 인증된 상태여도 새 로그인 창을 강제로 표시하려면:

```bash
wandb login --relogin
```

<div id="arguments">
  ## 인수
</div>

| 이름    | 기본값 | 유형    |
| ----- | --- | ----- |
| `key` | STR | False |

<div id="options">
  ## 옵션
</div>

| Flag                 | Type      | Description                                                                                                                                    |
| -------------------- | --------- | ---------------------------------------------------------------------------------------------------------------------------------------------- |
| `--cloud`            | BOOL Flag | W&amp;B 퍼블릭 클라우드([https://api.wandb.ai](https://api.wandb.ai))에 로그인합니다. `--host`와는 함께 사용할 수 없습니다. **기본값**: False                               |
| `--host, --base-url` | STR       | URL을 사용해 특정 W&amp;B 서버 인스턴스에 로그인합니다(예: [https://my-wandb.example.com](https://my-wandb.example.com)). `--cloud`와는 함께 사용할 수 없습니다. **기본값**: None |
| `--relogin`          | BOOL Flag | 기존 자격 증명을 무시하고 새 로그인 프롬프트를 강제로 표시합니다. **기본값**: None                                                                                            |
| `--verify`           | BOOL Flag | 저장한 후 W&amp;B에서 API 키를 확인합니다. 확인에 성공하면 자격 증명의 출처와 기본 팀을 표시합니다. **기본값**: False                                                                  |
