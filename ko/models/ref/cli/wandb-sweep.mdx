하이퍼파라미터 sweep을 초기화합니다. 다양한 조합을 테스트해 머신 러닝 모델의 비용 함수를 최적화하는 하이퍼파라미터를 검색합니다.


<div id="usage">
  ## 사용
</div>

```bash
wandb sweep CONFIG_YAML_OR_SWEEP_ID [OPTIONS]
```


<div id="arguments">
  ## 인수
</div>

| 인수                        | 설명    | 필수 |
| :------------------------ | :---- | :- |
| `CONFIG_YAML_OR_SWEEP_ID` | 설명 없음 | 예  |

<div id="options">
  ## 옵션
</div>

| 옵션                  | 설명                                                                                                                            |
| :------------------ | :---------------------------------------------------------------------------------------------------------------------------- |
| `--project`, `-p`   | sweep에서 생성된 W&amp;B run을 보낼 프로젝트 이름입니다. 프로젝트를 지정하지 않으면 run은 Uncategorized로 표시된 프로젝트로 전송됩니다.                                   |
| `--entity`, `-e`    | sweep에서 생성된 W&amp;B run을 보낼 사용자 이름 또는 팀 이름입니다. 지정한 entity가 이미 존재하는지 확인하세요. entity를 지정하지 않으면 run은 기본 entity(보통 사용자 이름)로 전송됩니다. |
| `--controller`      | 로컬 controller 실행 (기본값: False)                                                                                                 |
| `--verbose`         | 자세한 출력 표시 (기본값: False)                                                                                                        |
| `--name`            | sweep 이름입니다. 이름을 지정하지 않으면 sweep ID가 사용됩니다.                                                                                    |
| `--program`         | sweep program 설정                                                                                                              |
| `--update`          | 대기 중인 sweep 업데이트                                                                                                              |
| `--stop`            | sweep을 종료하여 새 run 실행을 중지하고 현재 실행 중인 run은 완료되도록 합니다. (기본값: False)                                                              |
| `--cancel`          | sweep을 취소하여 실행 중인 모든 run을 종료하고 새 run 실행을 중지합니다. (기본값: False)                                                                  |
| `--pause`           | sweep을 일시 중지하여 새 run 실행을 일시적으로 멈춥니다. (기본값: False)                                                                             |
| `--resume`          | sweep을 재개하여 새 run 실행을 계속합니다. (기본값: False)                                                                                     |
| `--prior_run`, `-R` | 이 sweep에 추가할 기존 run의 ID                                                                                                       |
