하이퍼파라미터 스윕을 초기화합니다. 머신러닝 모델의 비용 함수를 최적화하는 하이퍼파라미터를 찾기 위해 다양한 조합을 테스트합니다.


<div id="usage">
  ## 사용법
</div>

```bash
wandb sweep CONFIG_YAML_OR_SWEEP_ID [OPTIONS]
```


<div id="arguments">
  ## 인수
</div>

| Argument | 설명 | 필수 여부 |
| :--- | :--- | :--- |
| `CONFIG_YAML_OR_SWEEP_ID` | 설명이 제공되지 않습니다 | 예 |

<div id="options">
  ## Options
</div>

| Option | Description |
| :--- | :--- |
| `--project`, `-p` | 스윕에서 생성된 W&B 실행이 전송될 프로젝트 이름입니다. 프로젝트를 지정하지 않으면 실행은 Uncategorized라는 레이블이 붙은 프로젝트로 전송됩니다. |
| `--entity`, `-e` | 스윕에서 생성된 W&B 실행을 전송하려는 사용자 이름 또는 팀 이름입니다. 지정한 엔터티가 이미 존재하는지 확인하세요. 엔터티를 지정하지 않으면 실행은 기본 엔터티(보통은 본인 사용자 이름)로 전송됩니다. |
| `--controller` | 로컬 컨트롤러를 실행합니다 (기본값: False). |
| `--verbose` | 자세한 출력을 표시합니다 (기본값: False). |
| `--name` | 스윕의 이름입니다. 이름을 지정하지 않으면 스윕 ID가 사용됩니다. |
| `--program` | 스윕 프로그램을 설정합니다. |
| `--update` | 대기 중인 스윕을 업데이트합니다. |
| `--stop` | 새 실행 시작을 중지하고, 현재 실행 중인 실행이 완료되도록 하여 스윕을 종료합니다 (기본값: False). |
| `--cancel` | 실행 중인 모든 실행을 중단하고 새 실행 시작을 중지하여 스윕을 취소합니다 (기본값: False). |
| `--pause` | 새 실행 시작을 일시적으로 중지하여 스윕을 일시 중지합니다 (기본값: False). |
| `--resume` | 새 실행을 다시 시작하여 스윕을 재개합니다 (기본값: False). |
| `--prior_run`, `-R` | 이 스윕에 추가할 기존 실행의 ID입니다. |
