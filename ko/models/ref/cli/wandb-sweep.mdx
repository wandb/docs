하이퍼파라미터 스윕을 초기화합니다. 다양한 조합을 테스트하여 머신러닝 모델의 비용 함수를 최적화하는 하이퍼파라미터를 탐색합니다.


<div id="usage">
  ## 사용 방법
</div>

```bash
wandb sweep CONFIG_YAML_OR_SWEEP_ID [OPTIONS]
```


<div id="arguments">
  ## 인수
</div>

| 인수 | 설명 | 필수 |
| :--- | :--- | :--- |
| `CONFIG_YAML_OR_SWEEP_ID` | 설명이 없습니다 | 예 |

<div id="options">
  ## 옵션
</div>

| Option | Description |
| :--- | :--- |
| `--project`, `-p` | sweep에서 생성된 W&B run이 보내질 프로젝트 이름입니다. 프로젝트를 지정하지 않으면 run은 Uncategorized라는 라벨이 붙은 프로젝트로 보내집니다. |
| `--entity`, `-e` | sweep에서 생성된 W&B run을 보낼 사용자 이름 또는 팀 이름입니다. 지정한 entity가 이미 존재하는지 확인하십시오. entity를 지정하지 않으면 run은 기본 entity(일반적으로 본인의 사용자 이름)로 보내집니다. |
| `--controller` | 로컬 컨트롤러 실행 (기본값: False) |
| `--verbose` | 자세한 출력 표시 (기본값: False) |
| `--name` | sweep의 이름입니다. 이름을 지정하지 않으면 sweep ID가 사용됩니다. |
| `--program` | sweep 프로그램 설정 |
| `--update` | 대기 중인 sweep 업데이트 |
| `--stop` | 새로운 run 실행을 중지하고 현재 실행 중인 run이 완료되도록 하여 sweep을 종료합니다. (기본값: False) |
| `--cancel` | 실행 중인 모든 run을 종료하고 새로운 run 실행을 중지하여 sweep을 취소합니다. (기본값: False) |
| `--pause` | 새로운 run 실행을 일시적으로 중지하여 sweep을 일시 중지합니다. (기본값: False) |
| `--resume` | 새로운 run 실행을 다시 시작하도록 sweep을 재개합니다. (기본값: False) |
| `--prior_run`, `-R` | 이 sweep에 추가할 기존 run의 ID |
