<div id="usage">
  ## 사용
</div>

```bash
launch-agent [OPTIONS]
```

<div id="description">
  ## 설명
</div>

W&amp;B Launch 에이전트를 실행합니다.

<div id="options">
  ## 옵션
</div>

| Flag             | Type | 설명                                                                                                                                                       |
| ---------------- | ---- | -------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--queue, -q`    | STR  | 에이전트가 감시할 큐의 이름입니다. `-q` 플래그는 여러 번 지정할 수 있습니다. **기본값**: 없음                                                                                               |
| `--entity, -e`   | STR  | 사용할 entity입니다. 기본값은 현재 로그인한 사용자입니다. **기본값**: 없음                                                                                                          |
| `--log-file, -l` | STR  | 내부 에이전트 로그의 저장 위치입니다. stdout에는 `-`를 사용하세요. 기본적으로 모든 에이전트 로그는 `wandb/` 하위 디렉터리의 `debug.log` 또는 `WANDB_DIR`가 설정된 경우 해당 경로의 `debug.log`에 기록됩니다. **기본값**: 없음 |
| `--max-jobs, -j` | STR  | 이 에이전트가 병렬로 실행할 수 있는 Launch 작업의 최대 개수입니다. 기본값은 1입니다. 상한을 두지 않으려면 `-1`로 설정하세요. **기본값**: 없음                                                                |
| `--config, -c`   | STR  | 사용할 에이전트 설정 YAML 파일의 경로입니다. **기본값**: 없음                                                                                                                  |
| `--verbose, -v`  | INT  | 자세한 출력을 표시합니다. **기본값**: 0                                                                                                                                |
