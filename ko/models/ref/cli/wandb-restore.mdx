<div id="usage">
  ## 사용
</div>

```bash
restore [OPTIONS] RUN
```

<div id="description">
  ## 설명
</div>

이전 W&amp;B run의 코드, 설정 또는 Docker 환경을 복원합니다.

이전 run의 환경을 다시 만들어 재현할 수 있습니다.
W&amp;B 인증이 필요합니다.

원래 run이 기록한 내용에 따라 최대 세 가지 상태를 복원합니다:

1. 설정(항상): run 설정을 wandb/config.yaml에 기록합니다.

   2. Git(사용 가능한 경우): 새 `wandb/run_id` 브랜치에서 원래
      커밋을 체크아웃합니다. 저장된 diff 패치를 가져와 적용합니다.
      원래 커밋을 찾을 수 없으면
      업스트림 커밋으로 대체합니다.
      이 명령은 원래 run과 동일한 git 저장소에서 실행하세요. 이 단계를
      건너뛰려면 `--no-git`을 사용하세요.

   3. Docker(사용 가능한 경우): run이 Docker
      container 내부에서 실행된 경우, 원래 명령으로 동일한 이미지를 시작합니다.

run에 git 이력도 없고 Docker 이미지도 없으면,
설정만 복원합니다.

다음 형식 중 아무 형식으로나 run 식별자를 받을 수 있습니다:
run&#95;id, project:run&#95;id, entity/project:run&#95;id, 또는 entity/project/run&#95;id .

예를 들어, 기본 프로젝트에 있는 run ID로 run을 복원하려면
(WANDB&#95;PROJECT 환경 변수에 저장됨) 및 entity
(WANDB&#95;ENTITY 또는 인증된 사용자의 기본 entity로 설정됨):

```bash
wandb restore abcd1234
```

&quot;foobar&quot; 프로젝트와 &quot;team-awesome&quot; entity의
run ID가 abcd1234인 run을 복원하려면 다음과 같이 하세요:

```bash
wandb restore team-awesome/foobar/abcd1234
```

git 상태를 복원하지 않고 run abcd1234를 복원하려면 설정과
Docker 상태만 복원하세요:

```bash
wandb restore --no-git abcd1234
```

브랜치를 생성하지 않고 detached HEAD 모드에서 run abcd1234를 복원하려면:

```bash
wandb restore --no-branch abcd1234
```

다른 팀의 프로젝트에서 run abcd1234를 복원하려면 다음과 같이 하세요.

```bash
wandb restore other-team/their-project:abcd1234
```

<div id="arguments">
  ## 인수
</div>

| 이름    | 기본값 | 유형   |
| ----- | --- | ---- |
| `run` | STR | True |

<div id="options">
  ## 옵션
</div>

| Flag            | 유형       | 설명                                                                     |
| --------------- | -------- | ---------------------------------------------------------------------- |
| `--no-git`      | BOOL 플래그 | Git 복원을 건너뜁니다. 설정과 Docker 상태만 복원합니다. **기본값**: False                    |
| `--branch`      | BOOL 플래그 | `wandb/run_id` 브랜치를 생성하거나 detached HEAD 모드로 커밋을 체크아웃합니다. **기본값**: True |
| `--project, -p` | STR      | run을 조회할 프로젝트를 지정합니다. **기본값**: None                                    |
| `--entity, -e`  | STR      | run 조회 범위를 제한할 entity를 지정합니다. **기본값**: None                            |
