```bash

```

<div id="description">
  ## 설명
</div>

로컬 W&amp;B Server 인스턴스를 도커 컨테이너에서 시작합니다.

wandb/local 도커 이미지를 pull하여 run합니다. 지정한 호스트
포트를 컨테이너의 8080 포트에 매핑하고, 데이터를 저장할 수 있도록
&quot;wandb&quot;라는 이름의 영구 도커 볼륨을 마운트합니다.

이후 W&amp;B 클라이언트 Call이 로컬 서버를 사용하도록 &quot;base&#95;url&quot;
설정을 로컬 서버로 지정합니다. API 키를 찾을 수 없으면 로그인하라는
메시지가 표시됩니다.

호스트 머신에 Docker가 설치되어 있고 실행 중이어야 합니다.

이 command는 W&amp;B Models UI만 시작합니다. 로컬 테스트 용도로만
제공됩니다. 프로덕션 환경에서는 W&amp;B Kubernetes Operator가 권장되는
W&amp;B Serv[https://docs.wandb.ai/platform/hosting/self-managed/operator](https://docs.wandb.ai/platform/hosting/self-managed/operator)ged/operator 배포
방법입니다.

예를 들어, 기본 포트(8080)에서 로컬 W&amp;B 서버를 시작하려면 다음과 같이 하세요:

```bash
wandb server start
```

포트 9090으로 서버를 시작하려면:

```bash
wandb server start -p 9090
```

서버를 포그라운드에서 시작하려면:

```bash
wandb server start --no-daemon
```

<div id="options">
  ## 옵션
</div>

| 플래그          | 유형       | 설명                                                                   |
| ------------ | -------- | -------------------------------------------------------------------- |
| `--port, -p` | STR      | W&amp;B server를 바인딩할 호스트 포트입니다. **기본값**: 8080                        |
| `--env, -e`  | STR      | wandb/local 도커 이미지에 전달할 환경 변수입니다. **기본값**: []                        |
| `--daemon`   | BOOL 플래그 | 서버를 백그라운드에서 실행합니다. 포그라운드에서 실행하려면 `--no-daemon`을 사용하세요. **기본값**: True |
