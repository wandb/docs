아티팩트를 wandb에 업로드하기


<div id="usage">
  ## 사용 방법
</div>

```bash
wandb artifact put PATH [OPTIONS]
```


<div id="arguments">
  ## 인수
</div>

| 인수 | 설명 | 필수 여부 |
| :--- | :--- | :--- |
| `PATH` | 제공된 설명이 없습니다 | 예 |

<div id="options">
  ## 옵션
</div>

| 옵션 | 설명 |
| :--- | :--- |
| `--name`, `-n` | 푸시할 아티팩트 이름: project/artifact_name |
| `--description`, `-d` | 이 아티팩트에 대한 설명 |
| `--type`, `-t` | 아티팩트의 유형 (기본값: dataset) |
| `--alias`, `-a` | 이 아티팩트에 적용할 별칭(alias) (기본값: ['latest']) |
| `--id` | 업로드할 실행의 ID |
| `--resume` | 현재 디렉터리에서 마지막 실행을 재개합니다. |
| `--skip_cache` | 아티팩트 파일을 업로드할 때 캐싱을 사용하지 않습니다. (기본값: False) |
| `--policy` | 아티팩트 파일을 업로드할 때 저장 정책(storage policy)을 설정합니다. (기본값: mutable) |
