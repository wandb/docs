PATHS에 지정된 .wandb 파일을 업로드합니다.

이 명령은 `wandb sync`를 베타 버전으로 다시 구현한 것입니다. 아직 모든 기능을 지원하지 않으며, 정상 동작이 보장되지 않고, 향후 wandb의 어떤 릴리스에서도 이전 버전과 호환되지 않게 변경될 수 있습니다.

PATHS에는 .wandb 파일, .wandb 파일을 포함하는 실행 디렉터리, 그리고 실행 디렉터리를 포함하는 &quot;wandb&quot; 디렉터리를 지정할 수 있습니다.

예를 들어, 디렉터리 내의 모든 실행을 동기화하려면 다음과 같이 합니다:

```bash
wandb beta sync ./wandb
```

특정 실행을 동기화하려면:

```bash
wandb beta sync ./wandb/run-20250813_124246-n67z9ude
```

또는 다음과 같이:

```bash
wandb beta sync ./wandb/run-20250813_124246-n67z9ude/run-n67z9ude.wandb
```


<div id="usage">
  ## 사용법
</div>

```bash
wandb beta sync [PATHS] [OPTIONS]
```


<div id="arguments">
  ## 인수
</div>

| 인수 | 설명 | 필수 여부 |
| :--- | :--- | :--- |
| `PATHS` | 설명이 제공되지 않습니다 | 아니요 |

<div id="options">
  ## Options
</div>

| Option | Description |
| :--- | :--- |
| `--live` | 실행이 아직 로깅 중일 때 실행을 동기화합니다. 실행을 생성하는 프로세스가 비정상적으로 크래시하면 이 명령이 멈춘 것처럼 보일 수 있습니다. (default: False) |
| `-e`, `--entity` | 동기화되는 모든 실행에 사용할 엔터티 재정의 값입니다. (default: ) |
| `-p`, `--project` | 동기화되는 모든 실행에 사용할 프로젝트 재정의 값입니다. (default: ) |
| `--id` | 동기화되는 모든 실행에 사용할 실행 ID 재정의 값입니다. 이것을 설정하고 여러 파일(동일한 엔터티 및 프로젝트)을 동기화하는 경우, 파일은 시작 시간 순서대로 동기화됩니다. 이는 동일한 실행의 여러 재개된 조각(fragment)을 동기화할 때 사용하도록 설계되었습니다. (default: ) |
| `--job-type` | 동기화되는 모든 실행에 사용할 작업 유형 재정의 값입니다. (default: ) |
| `--replace-tags` | 'old1=new1,old2=new2' 형식을 사용하여 태그 이름을 변경합니다. (default: ) |
| `--skip-synced` | 이 명령으로 이미 동기화된 실행은 건너뜁니다. (default: True) |
| `--dry-run` | 실제로는 아무것도 업로드하지 않고, 어떤 작업이 수행될지 출력만 합니다. (default: False) |
| `-v`, `--verbose` | 더 자세한 정보를 출력합니다. (default: False) |
| `-n` | 한 번에 동기화할 최대 실행 수입니다. 동일한 실행의 일부인 여러 파일을 동기화할 때는, 이 설정과 관계없이 파일이 시작 시간 순서대로 순차적으로 동기화됩니다. 이는 재개된 실행이거나 --id 매개변수를 사용할 때 발생합니다. (default: 5) |
