PATHS에 지정된 .wandb 파일을 업로드합니다.

이 기능은 `wandb sync`의 베타 단계 재구현입니다. 아직 모든 기능을 제공하지 않으며, 동작이 보장되지 않고, wandb의 어떤 릴리스에서든 하위 호환성을 깨는 방식으로 변경될 수 있습니다.

PATHS에는 .wandb 파일, .wandb 파일을 포함한 run 디렉터리, 그리고 run 디렉터리를 포함한 &quot;wandb&quot; 디렉터리를 포함할 수 있습니다.

예를 들어, 디렉터리 내의 모든 run을 동기화하려면 다음과 같이 합니다:

```bash
wandb beta sync ./wandb
```

특정 run을 동기화하려면:

```bash
wandb beta sync ./wandb/run-20250813_124246-n67z9ude
```

또는 다음과 같이 표현할 수 있습니다:

```bash
wandb beta sync ./wandb/run-20250813_124246-n67z9ude/run-n67z9ude.wandb
```


<div id="usage">
  ## 사용법
</div>

```bash
wandb beta sync [PATHS] [OPTIONS]
```


<div id="arguments">
  ## 인자
</div>

| 인자 | 설명 | 필수 여부 |
| :--- | :--- | :--- |
| `PATHS` | 설명이 제공되지 않습니다 | 아니요 |

<div id="options">
  ## 옵션
</div>

| 옵션 | 설명 |
| :--- | :--- |
| `--live` | 아직 로깅 중인 run을 동기화합니다. run을 생성하는 프로세스가 비정상적으로 크래시되면 이 옵션은 명령이 멈춘 것처럼 보일 수 있습니다. (기본값: False) |
| `-e`, `--entity` | 동기화할 모든 run에 사용할 entity 재정의 값입니다. (기본값: ) |
| `-p`, `--project` | 동기화할 모든 run에 사용할 프로젝트 재정의 값입니다. (기본값: ) |
| `--id` | 동기화할 모든 run에 사용할 run ID 재정의 값입니다. 이 값을 설정하고 여러 파일(동일한 entity와 프로젝트를 사용하는)을 동기화하는 경우, 파일은 시작 시간 순서대로 동기화됩니다. 이는 동일한 run의 여러 재개(resume) 조각을 동기화할 때 사용하도록 설계되었습니다. (기본값: ) |
| `--job-type` | 동기화할 모든 run에 사용할 job 유형 재정의 값입니다. (기본값: ) |
| `--replace-tags` | 'old1=new1,old2=new2' 형식으로 태그 이름을 변경합니다. (기본값: ) |
| `--skip-synced` | 이 명령으로 이미 동기화된 run을 건너뜁니다. (기본값: True) |
| `--dry-run` | 아무것도 업로드하지 않고 수행될 작업만 출력합니다. (기본값: False) |
| `-v`, `--verbose` | 더 자세한 정보를 출력합니다. (기본값: False) |
| `-n` | 한 번에 동기화할 run의 최대 개수입니다. 동일한 run의 일부인 여러 파일을 동기화할 때는, 이 설정과 관계없이 파일이 시작 시간 순서대로 순차적으로 동기화됩니다. 이는 재개된 run이거나 --id 매개변수를 사용할 때 발생합니다. (기본값: 5) |
