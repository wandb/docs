<div id="usage">
  ## 사용 방법
</div>

```bash
wandb sync [PATH] [OPTIONS]
```


<div id="arguments">
  ## 인수
</div>

| 인수 | 설명 | 필수 여부 |
| :--- | :--- | :--- |
| `PATH` | 설명이 제공되지 않습니다 | 아니요 |

<div id="options">
  ## 옵션
</div>

| 옵션 | 설명 |
| :--- | :--- |
| `--id` | 업로드할 run. |
| `--project`, `-p` | 업로드할 프로젝트. |
| `--entity`, `-e` | 범위로 지정할 엔티티. |
| `--job_type` | 관련된 runs를 함께 그룹화하기 위한 run 유형. |
| `--sync-tensorboard` | tfevent 파일을 wandb로 스트리밍. |
| `--include-globs` | 포함할 glob 패턴의 쉼표로 구분된 목록. |
| `--exclude-globs` | 제외할 glob 패턴의 쉼표로 구분된 목록. |
| `--include-online` | 온라인 runs 포함. |
| `--include-offline` | 오프라인 runs 포함. |
| `--include-synced` | 동기화된 runs 포함. |
| `--mark-synced` | runs를 동기화된 것으로 표시 (기본값: True). |
| `--sync-all` | 모든 runs 동기화 (기본값: False). |
| `--clean` | 동기화된 runs 삭제 (기본값: False). |
| `--clean-old-hours` | 지정된 시간(시간 단위) 이전에 생성된 runs 삭제. --clean 플래그와 함께 사용. (기본값: 24) |
| `--clean-force` | 확인 프롬프트 없이 정리 실행. (기본값: False) |
| `--show` | 표시할 runs의 개수 (기본값: 5) |
| `--append` | run 추가 (기본값: False) |
| `--skip-console` | 콘솔 로그 건너뜀 (기본값: False) |
| `--replace-tags` | 'old_tag1=new_tag1,old_tag2=new_tag2' 형식으로 태그를 교체. |
