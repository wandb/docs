wandb 실행 없이 소스에서 Job을 생성합니다.

Job은 git, code, image 세 가지 유형일 수 있습니다.

git: 엔트리포인트가 경로에 있거나 명시적으로 지정되어 메인 Python 실행 스크립트를 가리키는 git 소스입니다. code: requirements.txt 파일을 포함하는 코드 경로입니다. image: Docker 이미지입니다.


<div id="usage">
  ## 사용 방법
</div>

```bash
wandb job create JOB_TYPE PATH [OPTIONS]
```


<div id="arguments">
  ## 인수
</div>

| 인수 | 설명 | 필수 여부 |
| :--- | :--- | :--- |
| `JOB_TYPE` | 설명이 제공되지 않습니다 | 예 |
| `PATH` | 설명이 제공되지 않습니다 | 예 |

<div id="options">
  ## 옵션
</div>

| 옵션 | 설명 |
| :--- | :--- |
| `--project`, `-p` | 작업을 나열할 프로젝트입니다. |
| `--entity`, `-e` | 작업이 속한 엔터티입니다. |
| `--name`, `-n` | 작업의 이름입니다. |
| `--description`, `-d` | 작업의 설명입니다. |
| `--alias`, `-a` | 작업의 별칭입니다. |
| `--entry-point`, `-E` | 실행 파일과 엔트리포인트 파일을 포함하는 스크립트의 엔트리포인트입니다. 코드 또는 리포지토리 작업에는 필수입니다. `--build-context`가 제공되면 엔트리포인트 명령의 경로는 빌드 컨텍스트를 기준으로 합니다. |
| `--git-hash`, `-g` | Git 작업의 소스로 사용할 커밋 참조입니다. |
| `--runtime`, `-r` | 작업을 실행할 Python 런타임입니다. |
| `--build-context`, `-b` | 작업 소스 코드 루트에서의 빌드 컨텍스트 경로입니다. 제공된 경우 Dockerfile과 엔트리포인트의 기준 경로로 사용됩니다. |
| `--base-image`, `-B` | 작업에 사용할 베이스 이미지입니다. 이미지 작업과는 호환되지 않습니다. |
| `--dockerfile`, `-D` | 작업에 사용할 Dockerfile 경로입니다. `--build-context`가 제공되면 Dockerfile 경로는 빌드 컨텍스트를 기준으로 합니다. |
