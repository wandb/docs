소스에서 wandb run 없이 작업을 생성합니다.

작업은 git, 코드 또는 이미지의 세 가지 유형이 될 수 있습니다.

git: git 소스로, 경로에 엔트리포인트가 있거나 기본 Python 실행 파일을 가리키도록 명시적으로 제공됩니다. 코드: requirements.txt 파일을 포함하는 코드 경로입니다. 이미지: 도커 이미지입니다.


<div id="usage">
  ## 사용
</div>

```bash
wandb job create JOB_TYPE PATH [OPTIONS]
```


<div id="arguments">
  ## 인수
</div>

| 인수         | 설명       | 필수 |
| :--------- | :------- | :- |
| `JOB_TYPE` | 설명이 없습니다 | 예  |
| `PATH`     | 설명이 없습니다 | 예  |

<div id="options">
  ## 옵션
</div>

| Option                  | 설명                                                                                                                                            |
| :---------------------- | :-------------------------------------------------------------------------------------------------------------------------------------------- |
| `--project`, `-p`       | 작업을 나열하려는 프로젝트입니다.                                                                                                                            |
| `--entity`, `-e`        | 작업이 속한 entity입니다.                                                                                                                             |
| `--name`, `-n`          | 작업의 이름입니다.                                                                                                                                    |
| `--description`, `-d`   | 작업에 대한 설명입니다.                                                                                                                                 |
| `--alias`, `-a`         | 작업의 별칭입니다.                                                                                                                                    |
| `--entry-point`, `-E`   | 실행 파일과 엔트리포인트 파일을 포함한 스크립트의 엔트리포인트입니다. 코드 또는 리포지토리 작업에는 필수입니다. `--build-context`가 제공되면 엔트리포인트 command의 경로는 build context를 기준으로 하는 상대 경로가 됩니다. |
| `--git-hash`, `-g`      | git 작업의 source로 사용할 커밋 레퍼런스입니다.                                                                                                               |
| `--runtime`, `-r`       | 작업을 실행할 Python runtime입니다.                                                                                                                    |
| `--build-context`, `-b` | 작업 source code 루트로부터의 build context 경로입니다. 제공되면 Dockerfile과 엔트리포인트의 기본 경로로 사용됩니다.                                                             |
| `--base-image`, `-B`    | 작업에 사용할 base 이미지입니다. 이미지 jobs와는 호환되지 않습니다.                                                                                                |
| `--dockerfile`, `-D`    | 작업의 Dockerfile 경로입니다. `--build-context`가 제공되면 Dockerfile 경로는 build context를 기준으로 하는 상대 경로가 됩니다.                                               |
