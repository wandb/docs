wandb run 없이 소스를 기반으로 job을 생성합니다.

job은 git, code, image의 세 가지 유형일 수 있습니다.

git: 메인 Python 실행 파일을 가리키는 엔트리 포인트가 경로에 포함되어 있거나 명시적으로 제공되는 git 소스입니다. code: `requirements.txt` 파일을 포함하는 코드 경로입니다. image: Docker 이미지입니다.


<div id="usage">
  ## 사용법
</div>

```bash
wandb job create JOB_TYPE PATH [OPTIONS]
```


<div id="arguments">
  ## 인수
</div>

| Argument | Description | Required |
| :--- | :--- | :--- |
| `JOB_TYPE` | 설명이 제공되지 않습니다 | Yes |
| `PATH` | 설명이 제공되지 않습니다 | Yes |

<div id="options">
  ## 옵션
</div>

| Option | Description |
| :--- | :--- |
| `--project`, `-p` | 작업을 나열할 프로젝트 |
| `--entity`, `-e` | 작업이 속한 entity |
| `--name`, `-n` | job의 이름 |
| `--description`, `-d` | job의 설명 |
| `--alias`, `-a` | job의 별칭 |
| `--entry-point`, `-E` | 실행 파일과 엔트리포인트 파일을 포함하는 스크립트의 엔트리포인트. 코드 또는 레포지토리 job에 필수입니다. `--build-context`가 제공되면 엔트리포인트 명령의 경로는 build context를 기준으로 합니다. |
| `--git-hash`, `-g` | git job의 소스로 사용할 커밋 참조 |
| `--runtime`, `-r` | job을 실행할 Python 런타임 |
| `--build-context`, `-b` | job 소스 코드 루트로부터의 build context 경로. 제공된 경우 이 경로가 Dockerfile과 엔트리포인트의 기준 경로로 사용됩니다. |
| `--base-image`, `-B` | job에 사용할 기본 이미지. image job과는 호환되지 않습니다. |
| `--dockerfile`, `-D` | job에 사용할 Dockerfile 경로. `--build-context`가 제공되면 Dockerfile 경로는 build context를 기준으로 합니다. |
