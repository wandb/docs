W&amp;B 작업을 Launch하거나 큐에 추가합니다. https://wandb.me/launch 참조


<div id="usage">
  ## 사용
</div>

```bash
wandb launch [OPTIONS]
```


<div id="options">
  ## 옵션
</div>

| 옵션                      | 설명                                                                                                                                                                                                                       |
| :---------------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--uri`, `-u`           | Launch할 로컬 경로 또는 git repo URI입니다. 지정하면 이 command는 해당 URI에서 작업을 생성합니다.                                                                                                                                                    |
| `--job`, `-j`           | Launch할 작업의 이름입니다. 지정하면 Launch에 URI가 필요하지 않습니다.                                                                                                                                                                          |
| `--entry-point`, `-E`   | 프로젝트 내 entry point입니다. [default: main]. entry point를 찾지 못하면 지정한 이름의 프로젝트 파일을 스크립트로 실행하려고 시도합니다. .py 파일은 `python`으로, .sh 파일은 기본 셸(환경 변수 $SHELL로 지정)로 실행합니다. 지정하면 설정 파일로 전달된 entrypoint 값을 재정의합니다.                         |
| `--build-context`       | 소스 코드 내 build context 경로입니다. 기본값은 소스 코드의 루트입니다. `-u`와만 호환됩니다.                                                                                                                                                            |
| `--name`                | run을 Launch할 때 사용할 run 이름입니다. 지정하지 않으면 임의의 run 이름이 사용됩니다. 지정하면 설정 파일로 전달된 name 값을 재정의합니다.                                                                                                                                |
| `--entity`, `-e`        | 새 run이 전송될 대상 entity의 이름입니다. 기본값은 로컬 `wandb/settings` 폴더에 설정된 entity입니다. 지정하면 설정 파일로 전달된 entity 값을 재정의합니다.                                                                                                               |
| `--project`, `-p`       | 새 run이 전송될 대상 프로젝트의 이름입니다. 기본값은 소스 URI에서 지정된 프로젝트 이름이며, GitHub run의 경우 git repo 이름입니다. 지정하면 설정 파일로 전달된 프로젝트 값을 재정의합니다.                                                                                                   |
| `--resource`, `-r`      | run에 사용할 실행 리소스입니다. 지원되는 값: &#39;local-process&#39;, &#39;local-container&#39;, &#39;kubernetes&#39;, &#39;sagemaker&#39;, &#39;gcp-vertex&#39;. 이제 리소스 설정이 없는 큐에 푸시할 경우 필수 매개변수입니다. 지정하면 설정 파일로 전달된 resource 값을 재정의합니다. |
| `--docker-image`, `-d`  | 사용할 특정 도커 이미지입니다. 형식은 `name:tag`입니다. 지정하면 설정 파일로 전달된 docker image 값을 재정의합니다.                                                                                                                                             |
| `--base-image`, `-B`    | 작업 코드를 실행할 도커 이미지입니다. `--docker-image`와 함께 사용할 수 없습니다.                                                                                                                                                                   |
| `--config`, `-c`        | Launch 설정으로 전달할 JSON 파일 경로(`.json`으로 끝나야 함) 또는 JSON 문자열입니다. Launch된 run이 어떻게 설정될지 지정합니다.                                                                                                                                 |
| `--set-var`, `-v`       | allow listing이 활성화된 큐의 템플릿 변수 값을 키-값 쌍으로 설정합니다. 예: `--set-var key1=value1 --set-var key2=value2`                                                                                                                         |
| `--queue`, `-q`         | 푸시할 run 큐의 이름입니다. 없으면 단일 run을 직접 Launch합니다. 인자 없이 지정하면(`--queue`) 기본값은 `default` 큐입니다. 이름을 지정하면, 해당 run 큐는 지정된 프로젝트와 entity 아래에 반드시 존재해야 합니다.                                                                            |
| `--async`               | 작업을 비동기적으로 실행하는 플래그입니다. 기본값은 false이며, 즉 `--async`를 설정하지 않으면 `wandb launch`는 작업이 완료될 때까지 기다립니다. 이 옵션은 `--queue`와 함께 사용할 수 없습니다. agent와 함께 실행할 때의 비동기 옵션은 `wandb launch-agent`에서 설정해야 합니다. (default: False)                |
| `--resource-args`, `-R` | 컴퓨트 리소스에 resource args로 전달할 JSON 파일 경로(`.json`으로 끝나야 함) 또는 JSON 문자열입니다. 제공해야 하는 정확한 내용은 실행 백엔드마다 다릅니다. 이 파일의 구조는 문서를 참조하세요.                                                                                              |
| `--dockerfile`, `-D`    | 작업을 빌드하는 데 사용하는 Dockerfile 경로이며, 작업 루트를 기준으로 한 상대 경로입니다.                                                                                                                                                                 |
| `--priority`, `-P`      | `--queue`를 지정하면 작업의 우선순위를 설정합니다. 우선순위가 높은 Launch 작업이 먼저 처리됩니다. 우선순위는 높은 순서부터 `critical`, `high`, `medium`, `low`입니다.                                                                                                     |
