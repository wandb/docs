<div id="usage">
  ## 사용
</div>

```bash
launch [OPTIONS]
```

<div id="description">
  ## 설명
</div>

W&amp;B 작업을 Launch하거나 큐에 추가합니다. [https://wandb.me/launch](https://wandb.me/launch) 참조

<div id="options">
  ## 옵션
</div>

| 플래그                   | 유형        | 설명                                                                                                                                                                                                                                                  |
| --------------------- | --------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--uri, -u`           | STR       | launch할 로컬 경로 또는 git repo URI입니다. 이 값이 제공되면 이 명령은 지정된 URI로부터 작업을 생성합니다. **기본값**: None                                                                                                                                                               |
| `--job, -j`           | STR       | launch할 작업 이름입니다. 이 값을 지정하면 launch에 URI가 필요하지 않습니다. **기본값**: None                                                                                                                                                                                   |
| `--entry-point, -E`   | STR       | 프로젝트 내 진입점입니다. [default: main]. 진입점을 찾지 못하면 지정한 이름의 프로젝트 파일을 스크립트로 실행하려고 시도합니다. 이때 .py 파일은 &#39;python&#39;으로, .sh 파일은 기본 셸(환경 변수 $SHELL로 지정됨)로 실행합니다. 이 값을 지정하면 설정 파일에서 전달된 entrypoint 값을 재정의합니다. **기본값**: None                                    |
| `--build-context`     | STR       | 소스 코드 내 build context 경로입니다. 기본값은 소스 코드의 루트입니다. -u에서만 사용할 수 있습니다. **기본값**: None                                                                                                                                                                     |
| `--name`              | STR       | run을 launch할 때 사용할 run 이름입니다. 지정하지 않으면 임의의 run 이름이 사용됩니다. 이 값을 지정하면 설정 파일에서 전달된 name 값을 재정의합니다. **기본값**: None                                                                                                                                       |
| `--entity, -e`        | STR       | 새 run이 전송될 대상 entity 이름입니다. 기본적으로 로컬 wandb/settings 폴더에 설정된 entity를 사용합니다. 이 값을 지정하면 설정 파일에서 전달된 entity 값을 재정의합니다. **기본값**: None                                                                                                                    |
| `--project, -p`       | STR       | 새 run이 전송될 대상 프로젝트 이름입니다. 기본적으로 소스 URI에 지정된 프로젝트 이름을 사용하며, GitHub run의 경우 git repo 이름을 사용합니다. 이 값을 지정하면 설정 파일에서 전달된 프로젝트 값을 재정의합니다. **기본값**: None                                                                                                   |
| `--resource, -r`      | STR       | run에 사용할 실행 리소스입니다. 지원되는 값은 &#39;local-process&#39;, &#39;local-container&#39;, &#39;kubernetes&#39;, &#39;sagemaker&#39;, &#39;gcp-vertex&#39;입니다. 이제 리소스 설정이 없는 큐로 푸시하는 경우 이 매개변수는 필수입니다. 이 값을 지정하면 설정 파일에서 전달된 resource 값을 재정의합니다. **기본값**: None |
| `--docker-image, -d`  | STR       | 사용할 특정 도커 이미지입니다. 형식은 name:tag입니다. 이 값을 지정하면 설정 파일에서 전달된 도커 이미지 값을 재정의합니다. **기본값**: None                                                                                                                                                            |
| `--base-image, -B`    | STR       | 작업 코드를 실행할 도커 이미지입니다. --docker-image와는 함께 사용할 수 없습니다. **기본값**: None                                                                                                                                                                                 |
| `--config, -c`        | STR       | JSON 파일 경로(&#39;.json&#39;으로 끝나야 함) 또는 launch 설정으로 전달할 JSON 문자열입니다. launch된 run이 어떻게 설정될지 지정합니다. **기본값**: None                                                                                                                                      |
| `--set-var, -v`       | STR       | allow listing이 활성화된 큐에 대해 템플릿 변수 값을 설정합니다. 키-값 쌍 형식의 예: `--set-var key1=value1 --set-var key2=value2` **기본값**: None                                                                                                                                 |
| `--queue, -q`         | STR       | 푸시할 run 큐 이름입니다. 지정하지 않으면 single run을 직접 launch합니다. 인수 없이 지정하면(`--queue`) 기본적으로 &#39;default&#39; 큐를 사용합니다. 이름을 지정한 경우에는 해당 run 큐가 지정된 프로젝트와 entity 아래에 있어야 합니다. **기본값**: None                                                                      |
| `--async`             | BOOL Flag | 작업을 비동기적으로 실행하는 플래그입니다. 기본값은 false이므로, 즉 --async를 설정하지 않으면 wandb launch는 작업이 완료될 때까지 기다립니다. 이 옵션은 --queue와 함께 사용할 수 없습니다. 에이전트와 함께 실행할 때의 비동기 옵션은 wandb launch-agent에서 설정해야 합니다. **기본값**: False                                                     |
| `--resource-args, -R` | STR       | JSON 파일 경로(&#39;.json&#39;으로 끝나야 함) 또는 컴퓨팅 리소스에 resource args로 전달할 JSON 문자열입니다. 제공해야 하는 정확한 내용은 실행 백엔드마다 다릅니다. 이 파일의 레이아웃은 문서를 참조하세요. **기본값**: None                                                                                                 |
| `--dockerfile, -D`    | STR       | 작업을 구축하는 데 사용하는 Dockerfile 경로이며, 작업 루트를 기준으로 한 상대 경로입니다. **기본값**: None                                                                                                                                                                              |
| `--priority, -P`      | Choice    | --queue가 지정되면 작업의 우선순위를 설정합니다. 우선순위가 높은 Launch 작업이 먼저 처리됩니다. 우선순위는 높은 순서부터 critical, high, medium, low입니다. **기본값**: None                                                                                                                            |
