라인리지 그래프를 사용하여 연결된 Artifact의 이력을 시각화하세요. 컬렉션의 감사 이력을 통해 해당 컬렉션의 아티팩트에 이루어진 변경 사항을 추적하세요.

<div id="lineage-graphs">
  ## 계보 그래프
</div>

W&amp;B Registry의 컬렉션 안에서 ML 실험에 사용되는 아티팩트의 이력을 확인할 수 있습니다. 이 이력을 &#95;lineage 그래프&#95;라고 합니다.

lineage 그래프는 다음을 보여줍니다:

* [실행의 입력으로 사용된](/ko/models/artifacts/explore-and-traverse-an-artifact-graph#track-the-input-of-a-run) 아티팩트
* [실행의 출력으로 생성된](/ko/models/artifacts/explore-and-traverse-an-artifact-graph#track-the-output-of-a-run) 아티팩트

즉, lineage 그래프는 실행의 입력과 출력을 보여줍니다.

예를 들어, 다음 이미지는 ML 실험 전반에서 생성·사용된 아티팩트에 대한 전형적인 lineage 그래프를 보여줍니다:

<Frame>
  <img src="/images/registry/registry_lineage.png" alt="Registry lineage" />
</Frame>

왼쪽에서 오른쪽 방향으로 이미지는 다음을 보여줍니다:

1. 여러 실행이 `split_zoo_dataset:v4` 아티팩트를 로그합니다.
2. &quot;rural-feather-20&quot; 실행은 학습을 위해 `split_zoo_dataset:v4` 아티팩트를 사용합니다.
3. &quot;rural-feather-20&quot; 실행의 출력은 `zoo-ylbchv20:v0`라는 모델 아티팩트입니다.
4. &quot;northern-lake-21&quot;이라는 실행이 모델 평가를 위해 모델 아티팩트 `zoo-ylbchv20:v0`를 사용합니다.

컬렉션 내 아티팩트에 대한 lineage 그래프를 보려면:

1. W&amp;B Registry로 이동합니다.
2. 해당 아티팩트를 포함하는 컬렉션을 선택합니다.
3. 드롭다운에서 lineage 그래프를 확인할 아티팩트 버전을 선택합니다.
4. **Lineage** 탭을 선택합니다.
5. 노드를 선택해 실행 또는 아티팩트에 대한 자세한 정보를 확인합니다.

<Info>
  W&amp;B Python SDK를 사용해 실행의 입력과 출력을 추적하는 방법은 [Enable lineage graph tracking](/ko/models/artifacts/explore-and-traverse-an-artifact-graph#enable-lineage-graph-tracking)을 참고하세요.
</Info>

다음 이미지는 lineage 그래프에서 노드를 선택했을 때 실행(`rural-feather-20`)에 대한 확장된 상세 보기를 보여줍니다:

</Frame>

다음 이미지는 lineage 그래프에서 아티팩트 노드를 선택했을 때 아티팩트(`zoo-ylbchv20:v0`)에 대한 확장된 상세 보기를 보여줍니다:

<Info>
  컬렉션에 속하지 않은, W&amp;B에 로그한 아티팩트에 대해서도 lineage 그래프를 볼 수 있습니다. 자세한 내용은 E[xplore artifact graphs](/ko/models/artifacts/explore-and-traverse-an-artifact-graph)를 참고하세요.
</Info>

<div id="audit-a-collections-history">
  ## 컬렉션 이력 감사
</div>

조직의 구성원이 해당 컬렉션에서 수행한 작업을 확인할 수 있습니다. 다음을 확인할 수 있습니다:

* 별칭이 아티팩트 버전에 추가되었는지 또는 제거되었는지.
* 아티팩트 버전이 컬렉션에 추가되었는지 또는 제거되었는지.

이 두 작업 모두에 대해, 작업을 수행한 사용자와 작업이 발생한 날짜를 확인할 수 있습니다.

컬렉션의 작업 이력을 보려면:

1. W&amp;B Registry로 이동합니다.
2. 작업 이력을 확인하려는 컬렉션을 선택합니다.
3. 컬렉션 이름 옆의 드롭다운 메뉴를 선택합니다.
4. **Action History** 옵션을 선택합니다.

{/* 실행 노드를 선택하면 해당 실행의 ID, 이름, 상태 등 실행에 대한 세부 정보를 확인할 수 있습니다. 예를 들어, 다음 이미지는 `rural-feather-20` 실행에 대한 정보를 예시로 보여줍니다:



  Artifact 노드를 선택하면 전체 이름, 유형, 생성 시각, 연결된 별칭 등 해당 아티팩트의 세부 정보를 확인할 수 있습니다. */}
