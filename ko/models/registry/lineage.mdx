리니지 그래프를 사용하여 연결된 아티팩트의 이력을 시각화하세요. 컬렉션의 이력을 감사하여 해당 컬렉션의 아티팩트에 적용된 변경 사항을 추적하세요.

<div id="lineage-graphs">
  ## 리니지 그래프
</div>

W&amp;B Registry의 컬렉션에서는 ML 실험에서 사용하는 아티팩트의 이력을 볼 수 있습니다. 이 이력을 &#95;리니지 그래프&#95;라고 합니다.

리니지 그래프에는 다음이 표시됩니다:

* [run의 입력으로 사용된](/ko/models/artifacts/explore-and-traverse-an-artifact-graph#track-the-input-of-a-run) 아티팩트.
* [run의 출력으로 생성된](/ko/models/artifacts/explore-and-traverse-an-artifact-graph#track-the-output-of-a-run) 아티팩트.

즉, 리니지 그래프는 run의 입력과 출력을 보여줍니다.

예를 들어, 다음 이미지는 ML 실험 전반에 걸쳐 생성되고 사용되는 아티팩트의 일반적인 리니지 그래프를 보여줍니다:

<Frame>
  <img src="/images/registry/registry_lineage.png" alt="Registry 리니지" />
</Frame>

이미지는 왼쪽에서 오른쪽으로 다음을 보여줍니다:

1. 여러 run이 `split_zoo_dataset:v4` 아티팩트를 로깅합니다.
2. &quot;rural-feather-20&quot; run은 트레이닝에 `split_zoo_dataset:v4` 아티팩트를 사용합니다.
3. &quot;rural-feather-20&quot; run의 출력은 `zoo-ylbchv20:v0`라는 모델 아티팩트입니다.
4. &quot;northern-lake-21&quot;이라는 run은 모델을 평가하기 위해 모델 아티팩트 `zoo-ylbchv20:v0`를 사용합니다.

컬렉션 내 아티팩트의 리니지 그래프를 보려면 다음 단계를 따르세요:

1. W&amp;B Registry로 이동합니다.
2. 아티팩트가 포함된 컬렉션을 선택합니다.
3. 드롭다운에서 리니지 그래프를 보려는 아티팩트 버전을 선택합니다.
4. **Lineage** 탭을 선택합니다.
5. 노드를 선택하여 run 또는 아티팩트의 세부 정보를 확인합니다.

<Info>
  W&amp;B Python SDK를 사용해 run의 입력과 출력을 추적하는 방법은 [리니지 그래프 추적 활성화](/ko/models/artifacts/explore-and-traverse-an-artifact-graph#enable-lineage-graph-tracking)를 참조하세요.
</Info>

</Info>

<div id="audit-a-collections-history">
  ## 컬렉션의 이력 감사하기
</div>

조직의 구성원이 해당 컬렉션에서 수행한 액션을 확인할 수 있습니다. 다음 항목을 볼 수 있습니다.

* 아티팩트 버전에 별칭이 추가되었는지 또는 제거되었는지 여부
* 컬렉션에 아티팩트 버전이 추가되었는지 또는 제거되었는지 여부

두 액션 모두에 대해, 해당 액션을 수행한 사용자와 액션이 발생한 날짜를 확인할 수 있습니다.

컬렉션의 액션 이력을 보려면 다음과 같이 하세요.

1. W&amp;B Registry로 이동합니다.
2. 액션 이력을 보려는 컬렉션을 선택합니다.
3. 컬렉션 이름 옆의 드롭다운 메뉴를 선택합니다.
4. **액션 이력** 옵션을 선택합니다.

  아티팩트 노드를 선택하여 전체 이름, 유형, 생성 시간, 연결된 별칭 등 해당 아티팩트의 세부 정보를 확인하세요. */}
