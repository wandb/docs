계보 그래프를 사용해 연결된 아티팩트의 이력을 시각화합니다. 컬렉션의 감사 이력을 통해 해당 컬렉션의 아티팩트에 가해진 변경 사항을 추적합니다.

<div id="lineage-graphs">
  ## Lineage 그래프
</div>

W&amp;B Registry의 컬렉션 내에서 ML 실험이 사용하는 아티팩트의 이력을 볼 수 있습니다. 이 이력을 *lineage 그래프* 라고 합니다.

lineage 그래프에는 다음 내용이 표시됩니다:

* [run의 입력으로 사용된](/ko/models/artifacts/explore-and-traverse-an-artifact-graph#track-the-input-of-a-run) 아티팩트
* [run의 출력으로 생성된](/ko/models/artifacts/explore-and-traverse-an-artifact-graph#track-the-output-of-a-run) 아티팩트

즉, lineage 그래프는 run의 입력과 출력을 보여줍니다.

예를 들어, 다음 이미지는 ML 실험 전반에서 생성 및 사용된 아티팩트에 대한 전형적인 lineage 그래프를 보여줍니다:

<Frame>
  <img src="/images/registry/registry_lineage.png" alt="Registry lineage" />
</Frame>

이미지는 왼쪽에서 오른쪽으로 다음을 보여줍니다:

1. 여러 run이 `split_zoo_dataset:v4` 아티팩트를 로깅합니다.
2. &quot;rural-feather-20&quot; run은 트레이닝을 위해 `split_zoo_dataset:v4` 아티팩트를 사용합니다.
3. &quot;rural-feather-20&quot; run의 출력은 `zoo-ylbchv20:v0` 라는 이름의 모델 아티팩트입니다.
4. &quot;northern-lake-21&quot; 이라는 run은 모델을 평가하기 위해 모델 아티팩트 `zoo-ylbchv20:v0` 를 사용합니다.

컬렉션 내의 아티팩트에 대한 lineage 그래프를 보려면:

1. W&amp;B Registry로 이동합니다.
2. 아티팩트를 포함하는 컬렉션을 선택합니다.
3. 드롭다운에서 lineage 그래프를 보고 싶은 아티팩트 버전을 선택합니다.
4. **Lineage** 탭을 선택합니다.
5. 노드를 선택하여 run 또는 아티팩트에 대한 세부 정보를 확인합니다.

<Info>
  W&amp;B Python SDK를 사용해 run의 입력과 출력을 추적하는 방법은 [lineage 그래프 추적 활성화](/ko/models/artifacts/explore-and-traverse-an-artifact-graph#enable-lineage-graph-tracking)를 참고하세요.
</Info>

</Info>

<div id="audit-a-collections-history">
  ## 컬렉션 이력 감사
</div>

해당 컬렉션에 대해 조직 구성원이 수행한 작업을 확인할 수 있습니다. 다음을 확인할 수 있습니다.

* 별칭이 아티팩트 버전에 추가되었는지 또는 제거되었는지
* 아티팩트 버전이 컬렉션에 추가되었는지 또는 컬렉션에서 제거되었는지

두 작업 모두에 대해, 작업을 수행한 사용자와 작업이 발생한 날짜를 확인할 수 있습니다.

컬렉션의 작업 이력을 보려면:

1. W&amp;B Registry로 이동합니다.
2. 작업 이력을 확인하려는 컬렉션을 선택합니다.
3. 컬렉션 이름 옆의 드롭다운 메뉴를 선택합니다.
4. **Action History** 옵션을 선택합니다.

{/* run 노드를 선택하면 run ID, run 이름, run 상태 등 해당 run의 세부 정보를 볼 수 있습니다. 예를 들어, 다음 이미지는 `rural-feather-20` run에 대한 정보를 보여줍니다.



  아티팩트 노드를 선택하면 전체 이름, 타입, 생성 시간, 연결된 별칭 등 해당 아티팩트의 세부 정보를 볼 수 있습니다. */}
