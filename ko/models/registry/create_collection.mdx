---

*컬렉션*은 레지스트리 내에서 연결된 아티팩트 버전의 집합입니다. 각 컬렉션은 고유한 작업 또는 사용 사례를 나타냅니다.

예를 들어 레지스트리 내에 여러 컬렉션이 있을 수 있습니다. 각 컬렉션에는 MNIST, CIFAR-10, ImageNet과 같은 서로 다른 데이터셋이 포함됩니다.

또 다른 예로, &quot;chatbot&quot;이라는 레지스트리가 있고 그 안에 모델 아티팩트용 컬렉션, 데이터셋 아티팩트용 또 다른 컬렉션, 파인튜닝된 모델 아티팩트용 또 다른 컬렉션이 있을 수 있습니다.

레지스트리와 그 컬렉션을 어떻게 구성할지는 사용자에게 달려 있습니다.

<Note>
  W&amp;B Model Registry에 익숙하다면 Registered Models를 알고 계실 수 있습니다. Model Registry의 Registered Models는 이제 W&amp;B Registry에서 컬렉션이라고 합니다.
</Note>"

<div id="collection-types">
  ## 컬렉션 유형
</div>

각 컬렉션은 정확히 하나의 아티팩트 *유형*만 허용합니다. 지정한 유형은 사용자와 조직의 다른 멤버가 해당 컬렉션에 링크할 수 있는 아티팩트의 종류를 제한합니다.

<Note>
  아티팩트 유형은 Python과 같은 프로그래밍 언어의 데이터 유형과 비슷하다고 생각할 수 있습니다. 이 비유에서 컬렉션은 문자열, 정수 또는 부동소수점을 저장할 수 있지만, 이러한 데이터 유형을 섞어서 저장할 수는 없습니다.
</Note>

예를 들어, &quot;dataset&quot; 아티팩트 유형을 허용하는 컬렉션을 만든다고 가정해 보겠습니다. 이는 앞으로 유형이 &quot;dataset&quot;인 아티팩트 버전만 이 컬렉션에 링크할 수 있음을 의미합니다. 마찬가지로, 모델 아티팩트 유형만 허용하는 컬렉션에는 유형이 &quot;model&quot;인 아티팩트만 링크할 수 있습니다.

<Note>
  아티팩트 객체를 만들 때 아티팩트의 유형을 지정합니다. `wandb.Artifact()`의 `type` 필드를 확인하세요:

  ```python
  import wandb

  # run 초기화
  with wandb.init(
    entity = "<team_entity>",
    project = "<project>"
    ) as run:

    # 아티팩트 객체 생성
    artifact = wandb.Artifact(
        name="<artifact_name>", 
        type="<artifact_type>"
        )
  ```
</Note>

컬렉션을 만들 때는 미리 정의된 아티팩트 유형 목록에서 선택할 수 있습니다. 사용 가능한 아티팩트 유형은 해당 컬렉션이 속한 레지스트리에 따라 달라집니다.

아티팩트를 컬렉션에 링크하거나 새 컬렉션을 만들기 전에 [해당 컬렉션이 허용하는 아티팩트 유형을 확인하세요](#check-the-types-of-artifact-that-a-collection-accepts).

<div id="check-the-types-of-artifact-that-a-collection-accepts">
  ### 컬렉션이 허용하는 아티팩트 유형 확인하기
</div>

컬렉션에 링크하기 전에 컬렉션이 허용하는 아티팩트 유형을 확인하세요. 컬렉션이 허용하는 아티팩트 유형은 W&amp;B Python SDK로 프로그래밍 방식으로 확인하거나 W&amp;B App에서 대화형으로 확인할 수 있습니다.

<Note>
  해당 아티팩트 유형을 허용하지 않는 컬렉션에 아티팩트를 링크하여 생성하려고 하면 오류 메시지가 표시됩니다.
</Note>

  <Tab title="W&B App">
    허용되는 아티팩트 유형은 홈페이지의 레지스트리 카드 또는 레지스트리의 Settings 페이지에서 확인할 수 있습니다.

    두 방법 모두 먼저 W&amp;B Registry로 이동하세요.

    W&amp;B Registry 홈페이지에서는 해당 레지스트리의 레지스트리 카드로 스크롤하여 허용되는 아티팩트 유형을 확인할 수 있습니다. 레지스트리 카드 안의 회색 가로 타원에는 해당 레지스트리가 허용하는 아티팩트 유형이 나열됩니다.

    레지스트리의 Settings 페이지에서 허용되는 아티팩트 유형을 보려면 다음과 같이 하세요.

    1. 설정을 보려는 레지스트리 카드를 클릭합니다.
    2. 오른쪽 위 모서리에 있는 톱니바퀴 아이콘을 클릭합니다.
    3. **Accepted artifact types** 필드로 스크롤합니다.

<div id="create-a-collection">
  ## 컬렉션 생성
</div>

레지스트리 내에서 컬렉션을 대화형 또는 프로그래밍 방식으로 생성합니다. 컬렉션을 생성한 이후에는 컬렉션에서 허용하는 아티팩트 유형을 변경할 수 없습니다.

<div id="programmatically-create-a-collection">
  ### 프로그래밍 방식으로 컬렉션 만들기
</div>

컬렉션에 아티팩트를 링크하려면 `wandb.Run.link_artifact()` 방법을 사용하세요. `target_path` 필드에는 다음 형식의 경로로 컬렉션과 레지스트리를 모두 지정하세요.

```python
f"wandb-registry-{registry_name}/{collection_name}"
```

여기서 `registry_name`은 레지스트리 이름이고 `collection_name`은 컬렉션 이름입니다. 레지스트리 이름에 `wandb-registry-` 접두사를 반드시 추가하세요.

<Note>
  존재하지 않는 컬렉션에 아티팩트를 링크하려고 하면 W&amp;B가 자동으로 컬렉션을 생성합니다. 이미 존재하는 컬렉션을 지정하면 W&amp;B는 아티팩트를 해당 기존 컬렉션에 링크합니다.
</Note>

다음 코드 스니펫은 프로그래밍 방식으로 컬렉션을 생성하는 방법을 보여줍니다. `<>`로 묶인 값은 사용자 환경에 맞게 바꾸세요:

```python
import wandb

# run을 초기화합니다
with wandb.init(entity = "<team_entity>", project = "<project>") as run:

  # 아티팩트 객체를 생성합니다
  artifact = wandb.Artifact(
    name = "<artifact_name>",
    type = "<artifact_type>"
    )

  registry_name = "<registry_name>"
  collection_name = "<collection_name>"
  target_path = f"wandb-registry-{registry_name}/{collection_name}"

  # 아티팩트를 collection에 링크합니다
  run.link_artifact(artifact = artifact, target_path = target_path)
```
"

<div id="interactively-create-a-collection">
  ### 컬렉션을 대화형으로 생성하기
</div>

다음 단계는 W&amp;B Registry를 사용해 컬렉션을 대화형으로 생성하는 방법을 설명합니다:

1. https://wandb.ai/registry/로 W&amp;B Registry에 이동하세요.
2. 레지스트리를 선택하세요.
3. 오른쪽 상단 모서리에 있는 **Create collection** 버튼을 클릭하세요.
4. **Name** 필드에 컬렉션 이름을 입력하세요.
5. **Type** 드롭다운에서 유형을 선택하세요. 또는, 레지스트리가 맞춤형 아티팩트 유형을 지원하는 경우, 이 컬렉션이 허용하는 하나 이상의 아티팩트 유형을 입력하세요.
6. 선택적으로 **Description** 필드에 컬렉션에 대한 설명을 입력하세요.
7. 선택적으로 **Tags** 필드에 하나 이상의 태그를 추가하세요.
8. **Link version**을 클릭하세요.
9. **Project** 드롭다운에서 아티팩트가 저장된 프로젝트를 선택하세요.
10. **Artifact** 컬렉션 드롭다운에서 아티팩트를 선택하세요.
11. **Version** 드롭다운에서 컬렉션에 링크하려는 아티팩트 버전을 선택하세요.
12. **Create collection** 버튼을 클릭하세요.
