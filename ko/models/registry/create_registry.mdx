---
title: 레지스트리 생성하기
---

레지스트리를 사용하면 사용할 아티팩트 유형을 유연하게 관리하고, 레지스트리의 공개 범위를 제한하는 등 더 많은 제어를 할 수 있습니다.

<div id="create-a-registry">
  ## 레지스트리 생성
</div>

W&amp;B Registry UI 또는 W&amp;B Python SDK를 사용해 레지스트리를 생성할 수 있습니다.

<Tabs>
  <Tab title=" W&B Registry UI">
    1. https://wandb.ai/registry/ 에서 W&amp;B Registry로 이동합니다.
    2. **Create registry** 버튼을 클릭합니다.
    3. **Name** 필드에 레지스트리 이름을 입력합니다.
    4. 필요하다면 레지스트리에 대한 설명을 입력합니다.
    5. **Registry visibility** 드롭다운에서 레지스트리를 볼 수 있는 대상을 선택합니다. 레지스트리 가시성 옵션에 대한 자세한 내용은 [Registry visibility types](./configure_registry#registry-visibility-types)를 참조하세요.
    6. **Accepted artifacts type** 드롭다운에서 **All types** 또는 **Specify types** 중 하나를 선택합니다.
    7. (**Specify types**를 선택한 경우) 레지스트리가 허용하는 하나 이상의 아티팩트 유형을 추가합니다.
    8. **Create registry** 버튼을 클릭합니다.
  </Tab>

  <Tab title="Python SDK">
    [`wandb.Api().create_registry()`](/ko/models/ref/python/#method-apicreate_registry) 메서드를 사용해 프로그래밍 방식으로 레지스트리를 생성합니다. `name`과 `visibility` 파라미터에 각각 레지스트리의 이름과 [visibility](#visibility-types)를 지정합니다.

    아래 코드 블록을 복사해 붙여넣고, `<>`로 둘러싸인 값을 사용자 환경에 맞게 변경하세요:

    ```python
    import wandb

    registry = wandb.Api().create_registry(
        name="<registry_name>",
        visibility="< 'restricted' | 'organization' >",
    )
    ```

    레지스트리를 생성할 때 지정할 수 있는 전체 파라미터 목록은 [`wandb.Api().create_registry()`](/ko/models/ref/python/#method-apicreate_registry) 메서드 레퍼런스를 참조하세요.
  </Tab>
</Tabs>

<Note>
  레지스트리 설정에 한 번 저장된 아티팩트 유형은 이후 레지스트리에서 제거할 수 없습니다.
</Note>

예를 들어, 다음 이미지는 사용자가 생성하려는 `Fine_Tuned_Models`라는 레지스트리를 보여줍니다. 이 레지스트리는 레지스트리에 수동으로 추가된 멤버에게만 보이도록 **Restricted**로 설정되어 있습니다.

<div id="visibility-types">
  ## 레지스트리 공개 범위 유형
</div>

레지스트리의 *공개 범위(visibility)* 는 누가 해당 레지스트리에 접근할 수 있는지를 결정합니다. 레지스트리의 공개 범위를 제한하면 지정된 멤버만 해당 레지스트리에 접근할 수 있도록 보장할 수 있습니다.

레지스트리에는 두 가지 공개 범위 옵션이 있습니다:

| 공개 범위 | 설명 |
| --- | --- |
| Restricted   | 초대된 조직 구성원만 레지스트리에 접근할 수 있습니다. |
| Organization | 조직의 모든 구성원이 레지스트리에 접근할 수 있습니다. |

팀 관리자 또는 레지스트리 관리자는 레지스트리의 공개 범위를 설정할 수 있습니다.

Restricted 공개 범위로 레지스트리를 생성한 사용자는 자동으로 해당 레지스트리의 레지스트리 관리자로 추가됩니다.

<div id="configure-the-visibility-of-a-registry">
  ## 레지스트리 가시성 구성하기
</div>

팀 관리자 또는 레지스트리 관리자는 레지스트리를 생성할 때나 생성한 이후에 레지스트리의 가시성을 설정할 수 있습니다.

기존 레지스트리의 가시성을 제한하려면 다음 단계를 따르세요:

1. https://wandb.ai/registry/ 에서 W&amp;B Registry로 이동합니다.
2. 레지스트리를 선택합니다.
3. 오른쪽 상단의 톱니바퀴 아이콘을 클릭합니다.
4. **Registry visibility** 드롭다운에서 원하는 레지스트리 가시성 옵션을 선택합니다.
5. **Restricted visibility**를 선택한 경우:
   1. 이 레지스트리에 접근 권한을 부여할 조직 구성원을 추가합니다. **Registry members and roles** 섹션으로 스크롤한 다음 **Add member** 버튼을 클릭합니다.
   2. **Member** 필드에 추가하려는 구성원의 이메일 또는 사용자 이름을 입력합니다.
   3. **Add new member**를 클릭합니다.
