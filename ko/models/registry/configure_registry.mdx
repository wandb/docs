---
title: 레지스트리 액세스 구성
---

레지스트리 관리자는 레지스트리 설정에서 [레지스트리 역할을 구성](/ko/models/registry/configure_registry/#configure-registry-roles)하고, [사용자를 추가](/ko/models/registry/configure_registry/#add-a-user-or-a-team-to-a-registry)하거나, 레지스트리에서 [사용자를 제거](/ko/models/registry/configure_registry/#remove-a-user-or-team-from-a-registry)할 수 있습니다.

<div id="manage-users">
  ## 사용자 관리
</div>

<div id="add-a-user-or-a-team">
  ### 사용자 또는 팀 추가
</div>

Registry 관리자는 개별 사용자 또는 전체 팀을 레지스트리에 추가할 수 있습니다. 사용자나 팀을 레지스트리에 추가하려면:

1. W&amp;B Registry로 이동합니다.
2. 사용자나 팀을 추가하려는 레지스트리를 선택합니다.
3. 오른쪽 상단 모서리에 있는 톱니바퀴 아이콘을 클릭하여 레지스트리 설정에 들어갑니다.
4. **Registry access** 섹션에서 **Add access**를 클릭합니다.
5. **Include users and teams** 필드에 하나 이상의 사용자 이름, 이메일 또는 팀 이름을 입력합니다.
6. **Add access**를 클릭합니다.

<Frame>
  <img src="/images/registry/add_team_registry.gif" alt="레지스트리에 팀을 추가하는 모습" />
</Frame>

[레지스트리에서 사용자 역할 구성하기](/ko/models/registry/configure_registry/#configure-registry-roles) 또는 [레지스트리 역할 권한](/ko/models/registry/configure_registry/#registry-role-permissions)에 대해 더 알아보세요.

<div id="remove-a-user-or-team">
  ### 사용자 또는 팀 제거
</div>

레지스트리 관리자라면 레지스트리에서 개별 사용자나 전체 팀을 제거할 수 있습니다. 레지스트리에서 사용자 또는 팀을 제거하려면 다음을 따르세요:

1. https://wandb.ai/registry/ 에서 W&amp;B 레지스트리 페이지로 이동합니다.
2. 사용자를 제거하려는 레지스트리를 선택합니다.
3. 오른쪽 상단에 있는 톱니바퀴 아이콘을 클릭하여 레지스트리 설정에 들어갑니다.
4. **Registry access** 섹션으로 이동한 후 제거하려는 사용자 이름, 이메일 또는 팀을 입력합니다.
5. **Delete** 버튼을 클릭합니다.

<Note>
  팀에서 사용자를 제거하면 해당 사용자의 레지스트리에 대한 액세스 권한도 함께 제거됩니다.
</Note>

<div id="change-the-owner-of-a-registry">
  ### 레지스트리의 소유자 변경
</div>

레지스트리 관리자는 **Restricted Viewer** 또는 **Viewer**를 포함해 어떤 멤버든 레지스트리의 소유자로 지정할 수 있습니다. 레지스트리 소유자는 주로 책임 소재를 명확히 하기 위한 개념이며, 사용자에게 할당된 역할을 통해 부여된 권한을 넘어서는 추가 권한을 제공하지 않습니다.

소유자를 변경하려면:

1. https://wandb.ai/registry/ 에서 W&amp;B Registry로 이동합니다.
2. 구성하려는 레지스트리를 선택합니다.
3. 오른쪽 상단의 톱니바퀴 아이콘을 클릭합니다.
4. **Registry members and roles** 섹션으로 스크롤합니다.
5. 해당 멤버의 행 위에 마우스를 올립니다.
6. 행 끝에 있는 **...** 동작 메뉴를 클릭한 다음 **Make owner**를 클릭합니다.

<div id="configure-registry-roles">
  ## Registry 역할 설정
</div>

이 섹션에서는 Registry 구성원의 역할을 설정하는 방법을 설명합니다. 각 역할의 기능, 우선순위, 기본값 등에 대한 더 자세한 내용은 [Registry 역할에 대한 자세한 내용](#details-about-registry-roles)을 참조하세요.

1. W&amp;B Registry 페이지(https://wandb.ai/registry/)로 이동합니다.
2. 설정하려는 Registry를 선택합니다.
3. 오른쪽 상단 모서리에 있는 톱니바퀴 아이콘을 클릭합니다.
4. **Registry members and roles** 섹션까지 스크롤합니다.
5. **Member** 필드에서 권한을 수정하려는 사용자 또는 팀을 검색합니다.
6. **Registry role** 열에서 해당 사용자의 역할을 클릭합니다.
7. 드롭다운 메뉴에서 사용자에게 할당할 역할을 선택합니다.

</div>

다음 섹션에서는 Registry 역할에 대해 더 자세히 설명합니다.

<Note>

<div id="default-roles">
  ### 기본 역할
</div>

사용자 또는 팀이 레지스트리에 추가되면, W&amp;B는 해당 주체에 기본 **레지스트리 역할**을 자동으로 할당합니다. 이 역할에 따라 해당 레지스트리에서 수행할 수 있는 작업이 결정됩니다.

| 엔터티                                 | 기본 레지스트리 역할<br />(Dedicated Cloud / Self-Managed)               | 기본 레지스트리 역할<br />(Multi-tenant Cloud)            |
|----------------------------------------|---------------------------------------------------------------------------|------------------------------------------------------------|
| Team                                   | Restricted Viewer (Server v0.75.0+) <br />Viewer (Server v0.74.x and below) | Restricted Viewer                               |
| User or service account (non admin)    | Restricted Viewer (Server v0.75.0+) <br />Viewer (Server v0.74.x and below) | Restricted Viewer                               |
| Service account (non admin)            | Member<sup><a href="#service_account_footnote">1</a></sup>                | Member<sup><a href="#service_account_footnote">1</a></sup> |
| Org admin                              | Admin                                                       | Admin                                           |

<a id="service_account_footnote">1</a>: Service account에는 **Viewer** 또는 **Restricted Viewer** 역할을 부여할 수 없습니다.

레지스트리 관리자는 레지스트리 내 사용자와 팀의 역할을 할당하거나 수정할 수 있습니다.
자세한 내용은 [레지스트리에서 사용자 역할 구성](/ko/models/registry/configure_registry/#configure-registry-roles)을 참고하세요.

<div id="inherited-registry-role">
  ### 상속된 Registry 역할
</div>

Registry의 멤버십 목록에는 각 사용자의 상속된(실제 적용되는) Registry 역할이 해당 행의 역할 드롭다운 옆에 옅은 회색으로 표시됩니다.

<Frame>
  <img src="/images/registry/role_conflict.png" alt="사용자의 실제 Registry 역할을 보여주는 Registry 멤버십 목록" />
</Frame>

특정 Registry에서 사용자의 실제 역할은, 상속되었는지 명시적으로 할당되었는지와 관계없이 조직, Registry, 그리고 해당 Registry를 소유한 팀에서 가진 역할 중 *가장 높은* 역할과 동일합니다. 예를 들어:

* 팀 **Admin** 또는 조직 **Admin**이, 해당 팀이 소유한 특정 Registry에서 **Viewer** 역할을 가진 경우, 해당 Registry에서의 실제 역할은 **Admin**입니다.
* Registry **Viewer**가 팀에서 **Member** 역할을 가진 경우, 해당 Registry에서의 실제 역할은 **Member**입니다.
* 팀 **Viewer**가 특정 Registry에서 **Member** 역할을 가진 경우, 해당 Registry에서의 실제 역할은 **Member**입니다.

<div id="restricted-viewer-role-details">
  ### Restricted Viewer 역할 상세
</div>

**Restricted Viewer** 역할은 GA(Generally Available, 정식 제공) 상태입니다. Dedicated Cloud 및 Self-Managed 환경에서는 서버 v0.75.0 이상이 필요합니다.

이 역할은 Registry 아티팩트에 대한 읽기 전용 접근 권한을 제공하지만, 컬렉션, 자동화 또는 기타 Registry 리소스를 생성, 업데이트, 삭제할 수 있는 권한은 없습니다.

**Viewer**와는 달리 **Restricted Viewer**는 다음을 할 수 없습니다.

* 아티팩트 파일을 다운로드하거나 파일 내용에 접근할 수 없습니다.
* W&amp;B SDK에서 `wandb.Run.use_artifact()`를 사용하여 아티팩트를 사용할 수 없습니다.

<div id="sdk-compatibility">
  #### SDK 호환성
</div>

<Note>
  **SDK 버전 요구 사항**

  **Restricted Viewer** 권한으로 W&amp;B SDK를 사용하여 아티팩트에 액세스하려면 W&amp;B SDK 버전 0.19.9 이상을 사용해야 합니다. 그렇지 않으면 일부 SDK 명령에서 권한 오류가 발생할 수 있습니다.
</Note>

**Restricted Viewer**가 SDK를 사용할 때는 일부 메서드를 사용할 수 없거나 다른 방식으로 동작합니다.

다음 메서드는 사용할 수 없으며, 권한 오류가 발생합니다:

* [`Run.use_artifact()`](/ko/models/ref/python/experiments/run/#method-runuse_artifact)
* [`Artifact.download()`](/ko/models/ref/python/experiments/artifact/#method-artifactdownload)
* [`Artifact.file()`](/ko/models/ref/python/experiments/artifact/#method-artifactfile)
* [`Artifact.files()`](/ko/models/ref/python/experiments/artifact/#method-artifactfiles)

다음 메서드는 아티팩트 메타데이터에만 사용할 수 있습니다:

* [`Artifact.get_entry()`](/ko/models/ref/python/experiments/artifact/#method-artifactget_entry)
* [`Artifact.get_path()`](/ko/models/ref/python/experiments/artifact/#method-artifactget_path)
* [`Artifact.get()`](/ko/models/ref/python/experiments/artifact/#method-artifactget)
* [`Artifact.verify()`](/ko/models/ref/python/experiments/artifact/#method-artifactverify)

<div id="cross-registry-permissions">
  ### 레지스트리 간 권한
</div>

사용자는 레지스트리마다 서로 다른 역할을 가질 수 있습니다. 예를 들어, 사용자가 Registry A에서는 **Restricted Viewer**이고 Registry B에서는 **Viewer**일 수 있습니다. 이 경우:

* 두 레지스트리에 모두 연결된 동일한 아티팩트라도 레지스트리별로 서로 다른 권한 수준을 갖습니다.
* Registry A에서 사용자는 **Restricted Viewer**이며, 파일을 다운로드하거나 아티팩트를 사용할 수 없습니다.
* Registry B에서 사용자는 **Viewer**이며, 파일을 다운로드하고 아티팩트를 사용할 수 있습니다.
* 즉, 아티팩트에 대한 권한은 해당 아티팩트에 액세스하는 레지스트리에 따라 결정됩니다.
