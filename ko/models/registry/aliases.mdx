---
title: 별칭으로 아티팩트 버전 참조하기
description: "기본, 맞춤형 및 보호된 별칭을 사용해 W&B Registry에서 특정 아티팩트 버전을 참조합니다."
---

하나 이상의 별칭을 사용해 특정 [아티팩트 버전](/ko/models/artifacts/create-a-new-artifact-version/)을 참조할 수 있습니다. [W&amp;B는 동일한 이름으로 연결된 각 아티팩트에](/ko/models/registry/aliases/#default-aliases) 자동으로 별칭을 부여합니다. 또한 [하나 이상의 맞춤형 별칭을 만들어](/ko/models/registry/aliases/#custom-aliases) 특정 아티팩트 버전을 참조할 수도 있습니다.

별칭은 레지스트리 UI에서 해당 별칭 이름이 표시된 직사각형으로 나타납니다. [별칭이 보호된 경우](/ko/models/registry/aliases/#protected-aliases) 잠금 아이콘이 있는 회색 직사각형으로 표시됩니다. 그렇지 않으면 주황색 직사각형으로 표시됩니다. 별칭은 레지스트리 간에 공유되지 않습니다.

<Note>
  **별칭을 사용할 때와 태그를 사용할 때**

  별칭은 특정 아티팩트 버전을 참조하는 데 사용합니다. 하나의 컬렉션 내에서 각 별칭은 고유합니다. 동시에 특정 별칭을 가질 수 있는 아티팩트 버전은 하나뿐입니다.

  태그는 공통된 주제를 기준으로 아티팩트 버전 또는 컬렉션을 구성하고 그룹화하는 데 사용합니다. 여러 아티팩트 버전과 컬렉션이 동일한 태그를 공유할 수 있습니다.
</Note>

아티팩트 버전에 별칭을 추가할 때 선택적으로 [레지스트리 자동화](/ko/models/automations/automation-events/#registry)을 시작하여 Slack 채널에 알림을 보내거나 웹훅을 트리거할 수 있습니다.

<div id="default-aliases">
  ## 기본 별칭
</div>

W&amp;B는 동일한 name으로 link하는 각 아티팩트 버전에 다음 별칭을 자동으로 부여합니다:

* 컬렉션에 link하는 가장 최근 아티팩트 버전에 `latest` 별칭을 부여합니다.
* 고유한 버전 number. W&amp;B는 link하는 각 아티팩트 버전(0부터 시작하는 인덱스)을 계산합니다. W&amp;B는 계산된 number를 사용하여 해당 아티팩트에 고유한 버전 number를 부여합니다.

예를 들어, `zoo_model`이라는 이름의 아티팩트를 세 번 link하면 W&amp;B는 각각 `v0`, `v1`, `v2`라는 세 개의 별칭을 생성합니다. `v2`에는 `latest` 별칭도 있습니다.

<div id="custom-aliases">
  ## 맞춤형 별칭
</div>

특정 아티팩트 버전에 대해 고유한 사용 사례에 맞춰 맞춤형 별칭을 하나 이상 생성하세요. 예를 들어:

* `dataset_version_v0`, `dataset_version_v1`, `dataset_version_v2`와 같은 별칭을 사용하여 모델이 어떤 데이터셋으로 트레이닝되었는지 파악할 수 있습니다.
* `best_model` 별칭을 사용하여 가장 성능이 좋은 아티팩트 모델 버전을 추적할 수 있습니다.

레지스트리에서 [**Member** 또는 **Admin** 레지스트리 역할](/ko/models/registry/configure_registry/#registry-roles)을 가진 모든 사용자는 해당 레지스트리의 연결된 아티팩트에 맞춤형 별칭을 추가하거나 제거할 수 있습니다. [**Restricted Viewer** 또는 **Viewer** 역할](/ko/models/registry/configure_registry/#registry-roles)을 가진 Users는 별칭을 추가하거나 제거할 수 없습니다.

<Note>
  [보호된 별칭](/ko/models/registry/aliases/#protected-aliases)은 수정 또는 삭제로부터 보호할 아티팩트 버전을 레이블링하고 파악하는 방법을 제공합니다.
</Note>

W&amp;B Registry 또는 Python SDK를 사용하여 맞춤형 별칭을 생성할 수 있습니다. 사용 사례를 기준으로 아래에서 가장 적합한 탭을 클릭하세요.

<Tabs>
  <Tab title="W&B Registry">
    1. W&amp;B Registry로 이동합니다.
    2. 컬렉션에서 **View details** 버튼을 클릭합니다.
    3. **Versions** 섹션에서 특정 아티팩트 버전의 **View** 버튼을 클릭합니다.
    4. **Aliases** 필드 옆의 **+** 버튼을 클릭하여 별칭을 하나 이상 추가합니다.
  </Tab>

  <Tab title="Python SDK">
    Python SDK를 사용하여 아티팩트 버전을 컬렉션에 연결할 때, [`link_artifact()`](/ko/models/ref/python/experiments/run.md/#link_artifact)의 `alias` 파라미터에 하나 이상의 별칭 목록을 인수로 선택적으로 전달할 수 있습니다. 제공한 별칭이 아직 존재하지 않는 경우 W&amp;B가 자동으로 별칭([보호되지 않은 별칭](#custom-aliases))을 생성합니다.

    다음 코드 스니펫은 Python SDK를 사용하여 아티팩트 버전을 컬렉션에 연결하고 해당 아티팩트 버전에 별칭을 추가하는 방법을 보여줍니다. `<>` 안의 값을 실제 값으로 교체하세요:

    ```python
    import wandb

    # run 초기화
    with wandb.init(entity = "<team_entity>", project = "<project_name>") as run:

        # 아티팩트 객체 생성
        # type 파라미터는 아티팩트 객체의 타입과 
        # 컬렉션 타입을 모두 지정합니다
        artifact = wandb.Artifact(name = "<name>", type = "<type>")

        # 아티팩트 객체에 파일을 추가합니다. 
        # 로컬 머신의 파일 경로를 지정하세요.
        artifact.add_file(local_path = "<local_path_to_artifact>")

        # 아티팩트를 연결할 컬렉션과 레지스트리를 지정합니다
        REGISTRY_NAME = "<registry_name>"
        COLLECTION_NAME = "<collection_name>"
        target_path=f"wandb-registry-{REGISTRY_NAME}/{COLLECTION_NAME}"

        # 아티팩트 버전을 컬렉션에 연결합니다
        # 이 아티팩트 버전에 하나 이상의 별칭을 추가합니다
        run.link_artifact(
            artifact = artifact, 
            target_path = target_path, 
            aliases = ["<alias_1>", "<alias_2>"]
            )
    ```
  </Tab>
</Tabs>

<div id="protected-aliases">
  ### 보호된 별칭
</div>

수정하거나 삭제해서는 안 되는 아티팩트 버전에 레이블을 지정하고 식별하려면 [보호된 별칭](/ko/models/registry/aliases/#protected-aliases)을 사용하세요. 예를 들어 조직의 머신 러닝 프로덕션 파이프라인에서 사용 중인 아티팩트 버전에 레이블을 지정하고 식별하기 위해 `production` 보호된 별칭을 사용할 수 있습니다.

[Registry admin](/ko/models/registry/configure_registry/#registry-roles) 사용자와 **Admin** 역할을 가진 [service accounts](/ko/models/support/service_account_useful/)는 보호된 별칭을 생성하고 아티팩트 버전에 보호된 별칭을 추가하거나 제거할 수 있습니다. **Member**, **Viewer**, **Restricted Viewer** 역할을 가진 Users와 service accounts는 보호된 버전의 연결을 해제하거나 보호된 별칭이 포함된 컬렉션을 삭제할 수 없습니다. 자세한 내용은 [레지스트리 액세스 설정](/ko/models/registry/configure_registry/)을 참조하세요.

일반적인 보호된 별칭은 다음과 같습니다.

* **Production**: 아티팩트 버전이 프로덕션에서 사용할 준비가 되었습니다.
* **Staging**: 아티팩트 버전이 테스트할 준비가 되었습니다.

<div id="create-a-protected-alias">
  #### 보호된 별칭 생성
</div>

다음 step에서는 W&amp;B Registry UI에서 보호된 별칭을 생성하는 방법을 설명합니다.

1. W&amp;B Registry로 이동합니다.
2. 레지스트리를 선택합니다.
3. 페이지 오른쪽 상단의 톱니바퀴 버튼을 클릭하여 레지스트리 설정을 확인합니다.
4. **Protected Aliases** 섹션에서 **+** 버튼을 클릭하여 하나 이상의 보호된 별칭을 추가합니다.

생성 후 각 보호된 별칭은 **Protected Aliases** 섹션에 자물쇠 아이콘이 있는 회색 직사각형으로 표시됩니다.

<Note>
  보호되지 않은 맞춤형 별칭과 달리, 보호된 별칭은 W&amp;B Registry UI에서만 생성할 수 있으며 Python SDK를 사용해 프로그래밍 방식으로 생성할 수는 없습니다. 아티팩트 버전에 보호된 별칭을 추가하려면 W&amp;B Registry UI 또는 Python SDK를 사용할 수 있습니다.
</Note>

다음 step에서는 W&amp;B Registry UI를 사용하여 아티팩트 버전에 보호된 별칭을 추가하는 방법을 설명합니다.

1. W&amp;B Registry로 이동합니다.
2. 컬렉션에서 **View details** 버튼을 클릭합니다.
3. **Versions** 섹션에서 특정 아티팩트 버전에 대한 **View** 버튼을 클릭합니다.
4. **Aliases** 필드 옆의 **+** 버튼을 클릭하여 하나 이상의 보호된 별칭을 추가합니다.

보호된 별칭이 생성된 후에는 관리자가 Python SDK를 사용해 프로그래밍 방식으로 해당 별칭을 아티팩트 버전에 추가할 수 있습니다. 아티팩트 버전에 보호된 별칭을 추가하는 방법은 위의 [Create a custom alias](#custom-aliases) 섹션에 있는 W&amp;B Registry 및 Python SDK 탭의 예시를 참조하세요.

<div id="find-existing-aliases">
  ## 기존 별칭 찾기
</div>

[W&amp;B Registry의 전역 검색 바](/ko/models/registry/search_registry/#search-for-registry-items)를 사용해 기존 별칭을 찾을 수 있습니다. 보호된 별칭을 찾으려면:

1. W&amp;B Registry로 이동하세요.
2. 페이지 상단의 검색 바에 검색어를 입력하세요. Enter 키를 눌러 검색하세요.

검색어가 기존 레지스트리, 컬렉션 이름, 아티팩트 버전 태그, 컬렉션 태그 또는 별칭과 일치하면 검색 바 아래에 검색 결과가 표시됩니다.

<div id="example">
  ## 예시
</div>

<Note>
  다음 코드 예시는 이 [W&amp;B 레지스트리 튜토리얼 노트북](https://colab.research.google.com/github/wandb/examples/blob/master/colabs/wandb_registry/zoo_wandb.ipynb)의 연속입니다. 다음 코드를 사용하려면 먼저 [노트북에 설명된 대로 Zoo 데이터셋을 조회하고 처리해야 합니다](https://colab.research.google.com/github/wandb/examples/blob/master/colabs/wandb_registry/zoo_wandb.ipynb#scrollTo=87fecd29-8146-41e2-86fb-0bb4e3e3350a). Zoo 데이터셋을 준비한 후에는 아티팩트 버전을 생성하고 여기에 맞춤형 별칭을 추가할 수 있습니다.
</Note>

다음 코드 스니펫은 아티팩트 버전을 생성하고 여기에 맞춤형 별칭을 추가하는 방법을 보여줍니다. 이 예시에서는 [UCI Machine Learning 저장소](https://archive.ics.uci.edu/dataset/111/zoo)의 Zoo 데이터셋과 `Zoo_Classifier_Models` 레지스트리의 `Model` 컬렉션을 사용합니다.

```python
import wandb

# run 초기화
with wandb.init(entity = "smle-reg-team-2", project = "zoo_experiment") as run:

    # 아티팩트 객체 생성
    # type 파라미터는 아티팩트 객체의 유형과
    # collection 유형을 모두 지정합니다
    artifact = wandb.Artifact(name = "zoo_dataset", type = "dataset")

    # 아티팩트 객체에 파일을 추가합니다.
    # 로컬 머신의 파일 경로를 지정하세요.
    artifact.add_file(local_path="zoo_dataset.pt", name="zoo_dataset")
    artifact.add_file(local_path="zoo_labels.pt", name="zoo_labels")

    # 아티팩트를 연결할 collection과 레지스트리를 지정합니다
    REGISTRY_NAME = "Model"
    COLLECTION_NAME = "Zoo_Classifier_Models"
    target_path=f"wandb-registry-{REGISTRY_NAME}/{COLLECTION_NAME}"

    # 아티팩트 버전을 collection에 연결합니다
    # 이 아티팩트 버전에 하나 이상의 별칭을 추가합니다
    run.link_artifact(
        artifact = artifact,
        target_path = target_path,
        aliases = ["production-us", "production-eu"]
        )
```

1. 먼저 아티팩트 객체를 생성합니다(`wandb.Artifact()`).
2. 다음으로, `wandb.Artifact.add_file()`을 사용해 두 개의 데이터셋 PyTorch 텐서를 아티팩트 객체에 추가합니다.
3. 마지막으로, `link_artifact()`를 사용해 아티팩트 버전을 `Zoo_Classifier_Models` 레지스트리의 `Model` 컬렉션에 연결합니다. 또한 `aliases` 매개변수의 인수로 `production-us`와 `production-eu`를 전달해 아티팩트 버전에 맞춤형 별칭 두 개를 추가합니다.
