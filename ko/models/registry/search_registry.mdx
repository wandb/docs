[W&amp;B Registry의 전역 검색창](./search_registry#search-for-registry-items)을 사용하여 레지스트리, 컬렉션, 아티팩트 버전 태그, 컬렉션 태그 또는 별칭을 찾을 수 있습니다. W&amp;B Python SDK를 사용하면 쿼리를 통해 특정 기준에 따라 [레지스트리, 컬렉션, 아티팩트 버전을 필터링](/ko/models/registry/search_registry#query-registry-items)할 수 있습니다.

<Info>
  W&amp;B Registry를 쿼리할 때 사용할 수 있는 구문과 연산자는 MongoDB 쿼리와 유사하지만, 완전히 동일하지는 않습니다.
</Info>

검색 결과에는 사용 권한이 있는 항목만 표시됩니다.

<div id="search-for-registry-items">
  ## 레지스트리 아이템 검색
</div>

W&amp;B App에서 레지스트리 아이템을 검색하려면:

1. W&amp;B Registry로 이동합니다.
2. 페이지 상단의 검색창에 검색어를 입력합니다. Enter 키를 눌러 검색을 실행합니다.

입력한 검색어가 기존 레지스트리, 컬렉션 이름, 아티팩트 버전 태그, 컬렉션 태그 또는 별칭과 일치하면, 검색창 아래에 검색 결과가 표시됩니다.

<div id="query-registry-items">
  ## 레지스트리 항목 쿼리하기
</div>

[`wandb.Api().registries()`](/ko/models/ref/python/public-api/api#registries)와 *쿼리 프레디킷(query predicate)*을 사용해 레지스트리, 컬렉션, 아티팩트 버전을 필터링할 수 있습니다. 쿼리 프레디킷은 반환되는 항목이 만족해야 하는 기준을 지정하는 조건입니다.

쿼리 프레디킷을 생성하려면 [쿼리 이름](/ko/models/registry/search_registry#filterable-fields), 하나 이상의 [연산자](/ko/models/registry/search_registry#supported-operators), 그리고 값으로 구성된 JSON 유사 딕셔너리를 사용합니다. 다음 코드 스니펫은 쿼리 프레디킷의 일반적인 구조를 보여줍니다:

```python
{
    "query_name": {
        "operator": value
    }
}
```

다음 섹션에서는 레지스트리에서 사용할 수 있는 [쿼리 이름](/ko/models/registry/search_registry#filterable-fields), [지원되는 연산자](/ko/models/registry/search_registry#supported-operators), 그리고 [예시 쿼리](/ko/models/registry/search_registry#example-queries)를 설명합니다.

<div id="filterable-fields">
  ### 필터링할 수 있는 필드
</div>

다음 표는 필터링하려는 항목 유형에 따라 사용할 수 있는 쿼리 이름을 보여줍니다:

| | 쿼리 이름 |
| ----- | ----- |
| registries | `name`, `description`, `created_at`, `updated_at` |
| collections | `name`, `tag`, `description`, `created_at`, `updated_at` |
| versions | `tag`, `alias`, `created_at`, `updated_at`, `metadata` |

<div id="supported-operators">
  ### 지원되는 연산자
</div>

W&amp;B는 레지스트리 항목을 필터링할 때 사용할 수 있는 다음 비교 및 논리 연산자를 지원합니다.

<div id="comparison-operators">
  #### 비교 연산자
</div>

| Operator | Description |
| ----- | ----- |
| `$eq` | 같다 |
| `$ne` | 같지 않다 |
| `$gt` | 크다 |
| `$gte` | 크거나 같다 |
| `$lt` | 작다 |
| `$lte` | 작거나 같다 |

<div id="logical-operators">
  #### 논리 연산자
</div>

| Operator | Description |
| ----- | ----- |
| `$and` | 하나 이상의 조건에 대해 [AND](https://en.wikipedia.org/wiki/Logical_conjunction) 논리 연산을 수행합니다 |
| `$or` | 하나 이상의 조건에 대해 [OR](https://en.wikipedia.org/wiki/Logical_disjunction) 논리 연산을 수행합니다 |
| `$nor` | 하나 이상의 조건에 대해 [NOR](https://en.wikipedia.org/wiki/Logical_NOR) 논리 연산을 수행합니다 |
| `$not` | 하나의 조건에 대해 [NOT](https://en.wikipedia.org/wiki/Negation) 논리 연산을 수행합니다 |

<div id="other-operators">
  #### 기타 연산자
</div>

| 연산자 | 설명 |
| ----- | ----- |
| `$regex` | 정규식 패턴 매칭 |
| `$exists` | 필드 존재 여부 |
| `$contains` | 문자열에 값이 포함되는지 |

<div id="example-queries">
  ### 예시 쿼리
</div>

다음 코드 예제는 몇 가지 일반적인 검색 시나리오를 보여 줍니다.

`wandb.Api().registries()` 메서드를 사용하려면 먼저 W&amp;B Python SDK([`wandb`](/ko/models/ref/python/)) 라이브러리를 임포트합니다:

```python
import wandb

# (선택 사항) 가독성을 위해 wandb.Api() 클래스의 인스턴스를 생성합니다
api = wandb.Api()
```

문자열 `model`이 포함된 모든 레지스트리를 필터링합니다:

```python
# 문자열 `model`을 포함하는 모든 레지스트리 필터링
registry_filters = {
    "name": {"$regex": "model"}
}

# 필터와 일치하는 모든 레지스트리의 이터러블 반환
registries = api.registries(filter=registry_filters)
```

레지스트리와 관계없이 이름에 `yolo` 문자열이 포함된 모든 컬렉션을 필터링합니다:

```python
# 레지스트리에 관계없이 컬렉션 이름에 문자열 `yolo`가 포함된
# 모든 컬렉션 필터링
collection_filters = {
    "name": {"$regex": "yolo"}
}

# 필터와 일치하는 모든 컬렉션의 이터러블 반환
collections = api.registries().collections(filter=collection_filters)
```

레지스트리와 무관하게, 컬렉션 이름에 문자열 `yolo`가 포함되어 있고 `cnn` 태그를 가진 모든 컬렉션을 필터링합니다:

```python
# 레지스트리에 관계없이 컬렉션 이름에 `yolo` 문자열을 포함하고
# `cnn` 태그를 가진 모든 컬렉션 필터링
collection_filters = {
    "name": {"$regex": "yolo"},
    "tag": "cnn"
}

# 필터와 일치하는 모든 컬렉션의 이터러블 반환
collections = api.registries().collections(filter=collection_filters)
```

문자열 `model`을 포함하고 태그 `image-classification` 또는 `latest` 별칭이 있는 모든 아티팩트 버전을 조회합니다:

```python
# 문자열 `model`을 포함하고 태그 `image-classification` 또는 별칭 `latest`를 가진
# 모든 아티팩트 버전을 찾습니다
registry_filters = {
    "name": {"$regex": "model"}
}

# 논리 연산자 $or를 사용하여 아티팩트 버전을 필터링합니다
version_filters = {
    "$or": [
        {"tag": "image-classification"},
        {"alias": "production"}
    ]
}

# 필터와 일치하는 모든 아티팩트 버전의 이터러블을 반환합니다
artifacts = api.registries(filter=registry_filters).collections().versions(filter=version_filters)
```

앞의 코드 스니펫에서 `artifacts` 이터러블의 각 항목은 `Artifact` 클래스의 인스턴스입니다. 따라서 각 아티팩트의 `name`, `collection`, `aliases`, `tags`, `created_at` 등의 속성에 접근할 수 있습니다.

```python
for art in artifacts:
    print(f"artifact name: {art.name}")
    print(f"collection artifact belongs to: { art.collection.name}")
    print(f"artifact aliases: {art.aliases}")
    print(f"tags attached to artifact: {art.tags}")
    print(f"artifact created at: {art.created_at}\n")
```

Artifact 객체의 전체 속성 목록은 API Reference 문서의 [Artifacts Class](/ko/models/ref/python/experiments/artifact/)를 참조하세요.

레지스트리나 컬렉션과 상관없이, 2024-01-08부터 2025-03-04 13:10 UTC 사이에 생성된 모든 아티팩트 버전을 필터링합니다:

```python
# 2024-01-08부터 2025-03-04 13:10 UTC 사이에 생성된 모든 아티팩트 버전을 찾습니다. 

artifact_filters = {
    "alias": "latest",
    "created_at" : {"$gte": "2024-01-08", "$lte": "2025-03-04 13:10:00"},
}

# 필터와 일치하는 모든 아티팩트 버전의 이터러블을 반환합니다
artifacts = api.registries().collections().versions(filter=artifact_filters)
```

<Note>
  `created_at` 및 `updated_at` 쿼리에서는 날짜와 시간을 `YYYY-MM-DD HH:MM:SS` 형식으로 지정하세요. 날짜만 기준으로 필터링하려면 시, 분, 초는 생략할 수 있습니다.
</Note>
