이 페이지에서는 [LLM Evaluation Jobs](/ko/models/launch)를 사용해 CoreWeave가 관리하는 인프라를 통해 공개적으로 액세스 가능한 URL에서 호스팅되는 API 모델에 대해 일련의 평가 벤치마크를 실행하는 방법을 설명합니다. W&amp;B Models에 아티팩트로 저장된 모델 체크포인트를 평가하려면 대신 [모델 체크포인트 평가하기](/ko/models/launch/evaluate-model-checkpoint)를 참조하세요.

<div id="prerequisites">
  ## 사전 요구 사항
</div>

1. LLM 평가 작업의 [요구 사항 및 제한 사항](/ko/models/launch#more-details)을 검토하세요.
2. 특정 벤치마크를 실행하려면 팀 관리자가 필요한 API 키를 팀 범위 secret으로 추가해야 합니다. 평가 작업을 구성할 때는 모든 팀 멤버가 secret을 지정할 수 있습니다.
   * **OpenAPI API 키**: 점수 산정을 위해 OpenAI 모델을 사용하는 벤치마크에서 사용됩니다. 벤치마크를 선택한 후 **Scorer API key** 필드가 나타나면 필요합니다. secret의 이름은 `OPENAI_API_KEY`여야 합니다.
   * **Hugging Face 사용자 액세스 토큰**: 하나 이상의 제한된 HF 데이터셋에 대한 액세스가 필요한 `lingoly`, `lingoly2` 같은 특정 벤치마크에 필요합니다. 벤치마크를 선택한 후 **Hugging Face Token** 필드가 나타나면 필요합니다. API 키에는 관련 데이터셋에 대한 액세스 권한이 있어야 합니다. [사용자 액세스 토큰](https://huggingface.co/docs/hub/en/security-tokens) 및 [제한된 HF 데이터셋에 액세스하기](https://huggingface.co/docs/hub/en/datasets-gated#access-gated-datasets-as-a-user)에 대해서는 Hugging Face 문서를 참조하세요.
   * [W&amp;B Inference](/ko/inference)에서 제공하는 모델을 평가하려면 조직 또는 팀 관리자가 아무 값으로든 `WANDB_API_KEY`를 생성해야 합니다. 이 secret은 실제로 인증에 사용되지 않습니다.
3. 평가할 모델은 공개적으로 액세스 가능한 URL에서 사용 가능해야 합니다. 인증을 위한 API 키로 조직 또는 팀 관리자가 팀 범위 secret을 생성해야 합니다.
4. 평가 결과를 위한 새 [W&amp;B 프로젝트](/ko/models/track/project-page)를 생성하세요. 프로젝트 사이드바에서 **새 프로젝트 생성**을 클릭하세요.
5. 지정된 벤치마크가 어떻게 작동하는지 이해하고 구체적인 요구 사항을 알아보려면 해당 벤치마크의 문서를 검토하세요. 편의를 위해 [사용 가능한 평가 벤치마크](/ko/models/launch/evaluations) 레퍼런스에 관련 링크가 포함되어 있습니다.

<div id="evaluate-your-model">
  ## 모델 평가하기
</div>

다음 단계에 따라 평가 작업을 설정하고 시작하세요:

1. W&amp;B에 로그인한 다음 프로젝트 사이드바에서 **Launch**를 클릭합니다. **LLM Evaluation Jobs** 페이지가 표시됩니다.
2. 호스팅된 API 모델 평가를 설정하려면 **Evaluate hosted API model**을 클릭합니다.
3. 평가 결과를 저장할 대상 프로젝트를 선택합니다.
4. **Model** 섹션에서 평가할 기본 URL과 모델 이름을 지정하고, 인증에 사용할 API 키를 선택합니다. 모델 이름은 [AI Security Institute](https://inspect.aisi.org.uk/providers.html#openai-api)에서 정의한 OpenAI 호환 형식으로 입력합니다. 예를 들어 OpenAI 모델은 다음 구문으로 지정합니다: `openai/<model-name>`. 호스팅된 모델 제공자와 모델의 전체 목록은 [AI Security Institute의 모델 제공자 레퍼런스](https://inspect.aisi.org.uk/providers.html)를 참조하세요.
   * [W&amp;B Inference](/ko/inference)에서 제공하는 모델을 평가하려면 기본 URL을 `https://api.inference.wandb.ai/v1`로 설정하고, 모델 이름을 다음 구문으로 지정합니다: `openai-api/wandb/<model_id>`. 자세한 내용은 [Inference 모델 카탈로그](/ko/inference/models)를 참고하세요.
   * [OpenRouter](https://inspect.aisi.org.uk/providers.html#openrouter) 제공자를 사용하려면 모델 이름 앞에 `openrouter` 접두사를 붙여 다음 구문으로 지정합니다: `openrouter/<model-name>`.
   * OpenAPI 규격을 준수하는 맞춤형 모델을 평가하려면 모델 이름을 다음 구문으로 지정합니다: `openai-api/wandb/<model-name>`.
5. **Select evaluations**를 클릭한 다음 실행할 벤치마크를 최대 4개까지 선택합니다.
6. 점수 산정에 OpenAI 모델을 사용하는 벤치마크를 선택하면 **Scorer API key** 필드가 표시됩니다. 이 필드를 클릭한 후 `OPENAI_API_KEY` secret을 선택합니다. 편의를 위해 팀 관리자는 이 drawer에서 **Create secret**을 클릭해 secret을 생성할 수 있습니다.
7. Hugging Face의 제한된 HF 데이터셋에 대한 액세스가 필요한 벤치마크를 선택하면 **Hugging Face token** 필드가 표시됩니다. [해당 데이터셋에 대한 액세스를 요청](https://huggingface.co/docs/hub/en/datasets-gated#access-gated-datasets-as-a-user)한 후 Hugging Face 사용자 액세스 토큰이 포함된 secret을 선택합니다.
8. 필요에 따라 **Sample limit**을 양의 정수로 설정해 평가할 벤치마크 샘플의 최대 개수를 제한할 수 있습니다. 설정하지 않으면 작업의 모든 샘플이 포함됩니다.
9. Leaderboard를 자동으로 생성하려면 **Publish results to leaderboard**를 클릭합니다. Leaderboard에는 모든 평가가 하나의 Workspace 패널에 함께 표시되며, 리포트에서 공유할 수도 있습니다.
10. **Launch**를 클릭해 평가 작업을 시작합니다.
11. 페이지 상단의 원형 화살표 아이콘을 클릭해 최근 run 모달을 엽니다. 평가 작업은 다른 최근 run과 함께 표시됩니다. 완료된 run의 이름을 클릭하면 single-run view에서 열리고, **Leaderboard** 링크를 클릭하면 Leaderboard를 바로 열 수 있습니다. 자세한 내용은 [결과 보기](#view-the-results)를 참조하세요.
