W&amp;B Python SDK를 사용해 메트릭, 미디어 또는 사용자 정의 객체로 이루어진 딕셔너리를 step 단위로 로그합니다. W&amp;B는 각 step 동안 키-값 쌍을 수집하고, `wandb.Run.log()`으로 데이터를 로그할 때마다 이를 하나의 통합 딕셔너리에 저장합니다. 스크립트에서 로그된 데이터는 먼저 로컬 머신의 `wandb` 디렉터리에 저장된 후, W&amp;B 클라우드 또는 [프라이빗 서버](/ko/platform/hosting/)로 동기화됩니다.

<Note>
  동일한 값을 각 step에 전달하는 경우에만 키-값 쌍이 하나의 통합 딕셔너리에 저장됩니다. `step`에 서로 다른 값을 로그하면, W&amp;B는 수집된 모든 키와 값을 메모리에 기록합니다.
</Note>

`wandb.Run.log()`을 호출할 때마다 기본적으로 새로운 `step`이 생성됩니다. W&amp;B는 차트와 패널을 생성할 때 기본 x축으로 step을 사용합니다. 필요에 따라 사용자 정의 x축을 생성해 사용할 수 있고, 사용자 정의 요약 메트릭을 수집할 수도 있습니다. 자세한 내용은 [로그 축 사용자 지정](/ko/models/track/log/customize-logging-axes/)을 참고하세요.

<div id="automatically-logged-data">
  ## 자동으로 로깅되는 데이터
</div>

W&amp;B는 W&amp;B Experiment 동안 다음 정보를 자동으로 로깅합니다.

* **시스템 메트릭**: CPU 및 GPU 사용률, 네트워크 등. GPU 메트릭은 [`nvidia-smi`](https://developer.nvidia.com/nvidia-system-management-interface)로 가져옵니다.
* **명령줄**: stdout과 stderr가 수집되어 [실행 페이지](/ko/models/runs/)의 Logs 탭에 표시됩니다.

계정의 [Settings page](https://wandb.ai/settings)에서 [Code Saving](https://wandb.me/code-save-colab)을 켜서 다음을 로깅하도록 설정합니다.

* **Git 커밋**: 최신 git 커밋을 수집하여 실행 페이지의 Overview 탭에 표시하며, 커밋되지 않은 변경 사항이 있을 경우 `diff.patch` 파일도 함께 표시합니다.
* **Dependencies**: `requirements.txt` 파일이 업로드되어 실행 페이지의 Files 탭에 표시되며, 해당 실행을 위해 `wandb` 디렉터리에 저장하는 모든 파일도 함께 업로드됩니다.

<div id="what-data-is-logged-with-specific-wb-api-calls">
  ## 특정 W&amp;B API 호출로 어떤 데이터가 로깅되나요?
</div>

W&amp;B를 사용하면 어떤 데이터를 로깅할지 정확히 결정할 수 있습니다. 아래는 자주 로깅되는 몇 가지 객체입니다:

* **Datasets**: 이미지나 기타 데이터셋 샘플이 W&amp;B로 스트리밍되려면 명시적으로 로깅해야 합니다.
* **Plots**: `wandb.plot()`과 `wandb.Run.log()`을 함께 사용하여 차트를 추적하세요. 자세한 내용은 [Log Plots](/ko/models/track/log/plots/)를 참조하세요.
* **Tables**: `wandb.Table`을 사용해 데이터를 로깅하면 W&amp;B에서 시각화하고 쿼리할 수 있습니다. 자세한 내용은 [Log Tables](/ko/models/track/log/log-tables/)를 참조하세요.
* **PyTorch gradients**: UI에서 가중치의 그래디언트를 히스토그램으로 확인하려면 `wandb.Run.watch(model)`을 추가하세요.
* **Configuration information**: 하이퍼파라미터, 데이터셋 링크, 사용 중인 아키텍처 이름 등을 config 파라미터로 다음과 같이 전달해 로깅할 수 있습니다: `wandb.init(config=your_config_dictionary)`.
* **Metrics**: `wandb.Run.log()`을 사용해 모델의 메트릭을 로깅하세요. 학습 루프 내부에서 accuracy, loss 같은 메트릭을 로깅하면 UI에서 실시간으로 업데이트되는 그래프를 볼 수 있습니다.

<div id="common-workflows">
  ## 일반적인 워크플로우
</div>

1. **최고 정확도 비교하기**: 여러 실행 간 특정 지표의 최고값을 비교하려면 해당 지표에 대한 summary 값을 설정한다. 기본적으로 summary는 각 키에 대해 마지막으로 로깅된 값으로 설정된다. 이는 UI 테이블에서 유용하며, summary 지표를 기준으로 실행을 정렬하고 필터링해서 최종 정확도가 아니라 *최고* 정확도를 기준으로 테이블이나 막대 차트에서 실행을 비교할 수 있다. 예: `wandb.run.summary["best_accuracy"] = best_accuracy`

2. **여러 지표를 하나의 차트에 보기**: 한 번의 호출에서 여러 지표를 로깅한다. 예를 들어:
   ```python
   with wandb.init() as run:
     run.log({"acc": 0.9, "loss": 0.1})
   ```
   그러면 UI에서 두 지표를 모두 하나의 차트에 플로팅할 수 있다.

3. **x축 사용자 지정하기**: 동일한 log 호출에 사용자 지정 x축을 추가해서, W&amp;B 대시보드에서 다른 축을 기준으로 지표를 시각화할 수 있다. 예를 들어:
   ```python
   with wandb.init() as run:
     run.log({'acc': 0.9, 'epoch': 3, 'batch': 117})
   ```
   특정 지표에 대한 기본 x축을 설정하려면 [Run.define&#95;metric()](/ko/models/ref/python/experiments/run#define_metric)을 사용한다.

4. **리치 미디어와 차트 로깅하기**: `wandb.Run.log()`는 [이미지와 비디오 같은 미디어](/ko/models/track/log/media/)부터 [테이블](/ko/models/track/log/log-tables/) 및 [차트](/ko/models/app/features/custom-charts/)까지 다양한 데이터 타입 로깅을 지원한다.
