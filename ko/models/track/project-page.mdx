*프로젝트*는 결과를 시각화하고, 실험을 비교하고, 아티팩트를 보고 다운로드하고, 자동화를 생성하는 등 여러 작업을 수행하는 중심 허브입니다.

<Note>
  각 프로젝트에는 누가 액세스할 수 있는지를 결정하는 가시성 설정이 있습니다. 프로젝트에 누가 접근할 수 있는지에 대한 자세한 내용은 [Project visibility](/ko/platform/hosting/iam/access-management/restricted-projects)를 참고하세요.
</Note>

각 프로젝트에는 다음과 같은 탭이 있습니다:

* [Overview](/ko/models/track/project-page/#overview-tab): 프로젝트의 스냅샷
* [Workspace](/ko/models/track/project-page/#workspace-tab): 개인용 시각화 샌드박스
* [Runs](#runs-tab): 프로젝트의 모든 실행이 나열된 테이블
* [Automations](#automations-tab): 프로젝트에서 구성한 자동화
* [Sweeps](/ko/models/sweeps): 자동화된 탐색 및 최적화
* [Reports](/ko/models/track/project-page/#reports-tab): 노트, 실행, 그래프의 저장된 스냅샷
* [Artifacts](#artifacts-tab): 모든 실행과 각 실행과 연관된 아티팩트를 포함

<div id="overview-tab">
  ## 개요 탭
</div>

W&amp;B는 프로젝트 필드에 입력한 이름으로 실행을 초기화하면 자동으로 프로젝트를 생성합니다.

**Overview** 탭에서 프로젝트 이름을 확인하고 프로젝트를 관리할 수 있습니다.

* 프로젝트의 이름, 설명 또는 팀을 변경하려면 오른쪽 상단의 **Edit**을 클릭합니다.
* 최근에 삭제한 실행을 복구하려면 오른쪽 상단의 **...**을 클릭한 다음 **Undelete recently deleted runs**를 클릭합니다.
* 프로젝트를 삭제하려면 오른쪽 상단의 **...**을 클릭한 다음 **Delete project**를 클릭합니다. 확인 대화 상자의 내용을 읽고 안내에 따르십시오. 프로젝트 삭제는 되돌릴 수 없습니다.

**Overview** 페이지의 나머지 부분은 **Details** 탭과 **Project Roles** 탭으로 나누어져 있습니다.

<div id="details">
  ### 세부 정보
</div>

프로젝트에 대한 세부 정보에는 다음이 포함됩니다:

* **Project visibility**: 프로젝트의 공개 범위입니다. 누가 프로젝트에 액세스할 수 있는지를 결정하는 가시성 설정입니다. 자세한 내용은 [Project visibility](/ko/platform/hosting/iam/access-management/restricted-projects)를 참고하세요.
* **Last active**: 이 프로젝트에 마지막으로 데이터가 로깅된 시각의 타임스탬프입니다.
* **Contributors**: 이 프로젝트에 기여한 사용자 수입니다.
* **Total runs**: 이 프로젝트에 포함된 실행의 총 개수입니다.
* **Total compute**: 이 프로젝트의 모든 실행 시간을 합산한 값입니다.

**Details** 탭에는 프로젝트의 데이터와 메트릭에 액세스하고 내보내는 방법, 스윕에서 최적 모델을 다운로드하는 방법 등 추가 안내도 포함되어 있습니다.

<div id="workspace-tab">
  ## Workspace 탭
</div>

프로젝트의 *워크스페이스(workspace)*는 실험을 비교할 수 있는 개인용 샌드박스를 제공합니다. 프로젝트에서는 동일한 문제를 서로 다른 아키텍처, 하이퍼파라미터, 데이터셋, 전처리 방식 등으로 실험하면서, 비교 가능한 모델을 체계적으로 구성할 수 있습니다.

**Runs 사이드바**: 프로젝트의 모든 실행 목록입니다.

* **점 메뉴**: 사이드바의 행 위에 마우스를 올리면 왼쪽에 메뉴가 표시됩니다. 이 메뉴를 사용해 실행 이름을 바꾸거나, 실행을 삭제하거나, 활성 실행을 중지할 수 있습니다.
* **가시성 아이콘**: 그래프에 표시할 실행을 켜거나 끄려면 눈 모양 아이콘을 클릭합니다.
* **색상**: 실행 색상을 기본 제공 프리셋 중 하나 또는 사용자 지정 색상으로 변경합니다.
* **검색**: 실행 이름으로 검색합니다. 검색 결과에 따라 플롯에 표시되는 실행도 함께 필터링됩니다.
* **필터**: 사이드바의 필터를 사용해 표시할 실행 범위를 좁힙니다.
* **그룹**: config 열을 선택해 실행을 동적으로 그룹화합니다. 예를 들어 아키텍처별로 그룹화할 수 있습니다. 그룹화를 하면 플롯에 평균 값을 따라가는 선과, 그래프에서 값의 분산을 나타내는 음영 영역이 표시됩니다.
* **정렬**: 실행을 정렬할 기준 값을 선택합니다. 예를 들어 손실이 가장 낮은 실행이나 정확도가 가장 높은 실행 기준으로 정렬할 수 있습니다. 정렬은 그래프에 어떤 실행이 표시되는지에 영향을 줍니다.
* **확장 버튼**: 사이드바를 전체 테이블로 확장합니다.
* **실행 수**: 상단 괄호 안 숫자는 프로젝트의 전체 실행 수입니다. `(N visualized)` 숫자는 눈 아이콘이 켜져 있어 각 플롯에서 시각화 가능한 실행 수입니다. 아래 예시에서 그래프는 183개 실행 중 처음 10개만 표시합니다. 그래프를 편집해 표시 가능한 최대 실행 수를 늘릴 수 있습니다.

[Runs 탭](#runs-tab)에서 열을 고정하거나 숨기거나 순서를 변경하면, Runs 사이드바에도 이러한 사용자 정의가 반영됩니다.

**Panels 레이아웃**: 이 임시 작업 공간을 사용해 결과를 탐색하고, 차트를 추가/제거하고, 다양한 지표를 기준으로 모델 버전을 비교하세요.

<div id="add-a-section-of-panels">
  ### 패널 섹션 추가
</div>

섹션 드롭다운 메뉴를 열고 &quot;Add section&quot;을 클릭하여 패널용 새 섹션을 만듭니다. 섹션 이름을 바꾸고, 드래그해서 순서를 재정렬하며, 섹션을 펼치거나 접을 수 있습니다.

각 섹션의 오른쪽 상단에는 다음과 같은 옵션이 있습니다:

* **Add section**: 드롭다운 메뉴에서 위나 아래에 섹션을 추가하거나, 페이지 하단의 버튼을 클릭해 새 섹션을 추가합니다.
* **Rename section**: 섹션 제목을 변경합니다.
* **Export section to report**: 이 섹션의 패널을 새 리포트로 저장합니다.
* **Delete section**: 전체 섹션과 모든 차트를 제거합니다. 워크스페이스 바의 페이지 하단에 있는 실행 취소 버튼으로 되돌릴 수 있습니다.
* **Add panel**: 플러스 버튼을 클릭해 섹션에 패널을 추가합니다.

<div id="metadata-panel">
  ### 메타데이터 패널
</div>

<Frame>
  <img src="/images/app_ui/metadata_panel.png" alt="Artifact 메타데이터 패널" />
</Frame>

메타데이터 패널에서는 아티팩트의 메타데이터에 접근할 수 있습니다. 이 메타데이터에는 아티팩트를 재구성하는 데 필요한 설정 인자, 추가 정보를 확인할 수 있는 URL, 아티팩트를 기록한 실행 중에 생성된 메트릭 등이 포함될 수 있습니다.
또한 아티팩트를 생성한 실행의 설정과 기록된 메트릭 히스토리도 볼 수 있습니다.

아티팩트의 메타데이터를 보려면:

1. 프로젝트 사이드바에서 **Artifacts** 탭을 선택합니다.
2. 목록에서 아티팩트를 선택하여 해당 아티팩트의 최신 버전에 대한 세부 정보 페이지를 엽니다.
3. 해당 아티팩트와 연결된 메타데이터를 보려면 **Metadata** 탭을 선택합니다.

<div id="usage-panel">
  ### Usage 패널
</div>

<Frame>
  <img src="/images/app_ui/usage_panel.png" alt="Artifact 사용 패널" />
</Frame>

Usage 패널은 웹 앱 외부, 예를 들어 로컬 머신에서 사용할 수 있도록 아티팩트를 다운로드하는 데 쓸 수 있는 코드 스니펫을 제공합니다. 이 섹션에는 아티팩트를 출력한 실행과 해당 아티팩트를 입력으로 사용하는 실행들이 표시되며, 각 실행으로 이동할 수 있는 링크도 제공됩니다.

아티팩트를 보려면

1. 프로젝트 사이드바에서 **Artifacts** 탭을 선택합니다.
2. 목록에서 아티팩트를 선택하여 해당 아티팩트의 최신 버전에 대한 상세 페이지를 확인합니다.
3. **Usage** 탭을 선택하여 코드 스니펫과 관련 실행을 확인합니다.

<div id="files-panel">
  ### Files 패널
</div>

<Frame>
  <img src="/images/app_ui/files_panel.png" alt="Artifact files panel" />
</Frame>

Files 패널에는 해당 아티팩트와 연관된 파일과 폴더가 나열됩니다. W&amp;B는 실행과 관련된 일부 파일을 자동으로 업로드합니다. 예를 들어 `requirements.txt`는 실행에서 사용한 각 라이브러리의 버전을 보여주고, `wandb-metadata.json` 및 `wandb-summary.json`에는 실행에 대한 정보가 포함됩니다. 실행 설정에 따라 아티팩트나 미디어와 같은 다른 파일도 업로드될 수 있습니다.

아티팩트에 기록된 파일을 보려면 다음을 수행합니다:

1. 프로젝트 사이드바에서 **Artifacts** 탭을 선택합니다.
2. 목록에서 아티팩트를 선택하여 해당 아티팩트의 최신 버전에 대한 세부 정보 페이지를 엽니다.
3. **Files** 탭을 선택하여 해당 아티팩트와 연관된 모든 파일을 확인합니다.

<div id="lineage-panel">
  ### Lineage 패널
</div>

<Frame>
  <img src="/images/app_ui/lineage_panel.png" alt="Artifact 계보" />
</Frame>

Lineage 패널은 프로젝트에 연결된 모든 아티팩트와, 이들을 서로 연결하는 실행을 한눈에 보여 줍니다. 실행 유형은 블록으로, 아티팩트는 원으로 표시되며, 특정 유형의 실행이 특정 유형의 아티팩트를 사용하거나 생성할 때 화살표로 표시됩니다. 왼쪽 열에서 선택한 특정 아티팩트의 유형은 강조되어 표시됩니다.

Explode 토글을 클릭하면 개별 아티팩트 버전과 이들을 연결하는 구체적인 실행을 모두 볼 수 있습니다.

<div id="versions-tab">
  ### Versions 탭
</div>

<Frame>
  <img src="/images/app_ui/versions_tab.png" alt="Artifact versions 탭" />
</Frame>

Versions 탭에는 아티팩트의 모든 버전이 표시됩니다. 특정 버전의 세부 정보를 보려면 해당 아티팩트를 선택합니다.

1. 프로젝트 사이드바에서 **Artifacts** 탭을 선택합니다.
2. 목록에서 아티팩트를 선택해, 해당 아티팩트의 최신 버전에 대한 상세 페이지를 확인합니다.
3. **Versions** 탭을 선택해 해당 아티팩트의 모든 버전을 확인합니다.
4. 드롭다운(아티팩트 이름 옆)을 열고 **All Versions**를 선택합니다.

예를 들어, 위 이미지에서는 `"zoo-wyhak4p0"`라는 이름의 모델 아티팩트에 대해 서로 다른 버전의 모델 아티팩트들이 표시되어 있습니다.

<div id="create-a-project">
  ## 프로젝트 생성
</div>

W&amp;B 앱에서 프로젝트를 생성하거나, `wandb.init()` 호출 시 `project`를 지정해 코드로 생성할 수 있습니다.

<Tabs>
  <Tab title="W&B 앱">
    W&amp;B 앱에서 **Projects** 페이지 또는 팀 랜딩 페이지에서 프로젝트를 생성할 수 있습니다.

    **Projects** 페이지에서:

    1. 왼쪽 상단의 글로벌 내비게이션 아이콘을 클릭합니다. 내비게이션 사이드바가 열립니다.
    2. 내비게이션의 **Projects** 섹션에서 **View all**을 클릭하여 프로젝트 개요 페이지를 엽니다.
    3. **Create new project**를 클릭합니다.
    4. **Team**을 이 프로젝트를 소유할 팀 이름으로 설정합니다.
    5. **Name** 필드에 프로젝트 이름을 입력합니다.
    6. 기본값이 **Team**인 **Project visibility**를 설정합니다.
    7. 필요하다면 **Description**을 입력합니다.
    8. **Create project**를 클릭합니다.

    팀 랜딩 페이지에서:

    1. 왼쪽 상단의 글로벌 내비게이션 아이콘을 클릭합니다. 내비게이션 사이드바가 열립니다.
    2. 내비게이션의 **Teams** 섹션에서 팀 이름을 클릭하여 해당 팀의 랜딩 페이지를 엽니다.
    3. 랜딩 페이지에서 **Create new project**를 클릭합니다.
    4. **Team**은 현재 보고 있던 랜딩 페이지를 소유한 팀으로 자동 설정됩니다. 필요하면 팀을 변경합니다.
    5. **Name** 필드에 프로젝트 이름을 입력합니다.
    6. 기본값이 **Team**인 **Project visibility**를 설정합니다.
    7. 필요하다면 **Description**을 입력합니다.
    8. **Create project**를 클릭합니다.
  </Tab>

  <Tab title="Python SDK">
    코드에서 프로젝트를 생성하려면 `wandb.init()`을 호출할 때 `project`를 지정합니다. 프로젝트가 아직 존재하지 않으면 자동으로 생성되며, 지정한 엔터티가 소유하게 됩니다. 예를 들어:

    ```python
    import wandb

    with wandb.init(entity="<entity>", project="<project_name>") as run:
        run.log({"accuracy": .95})
    ```

    [`wandb.init()` API 레퍼런스](/ko/models/ref/python/functions/init/#examples)를 참조하세요.
  </Tab>
</Tabs>

<div id="star-a-project">
  ## 프로젝트에 별 표시하기
</div>

프로젝트에 별을 추가해 해당 프로젝트를 중요한 프로젝트로 표시할 수 있습니다. 여러분과 팀이 별로 중요하게 표시한 프로젝트는 조직 홈 화면 상단에 표시됩니다.

예를 들어, 아래 이미지는 `zoo_experiment`와 `registry_demo` 두 프로젝트가 중요하게 표시된 예시입니다. 두 프로젝트 모두 조직 홈 화면 상단의 **Starred projects** 섹션에 나타납니다.

<Frame>
  <img src="/images/track/star-projects.png" alt="Starred projects 섹션" />
</Frame>

프로젝트를 중요하게 표시하는 방법은 두 가지입니다. 프로젝트의 개요 탭에서 하거나, 팀 프로필 페이지에서 할 수 있습니다.

<Tabs>
  <Tab title="프로젝트 개요">
    1. `https://wandb.ai/<team>/<project-name>`에서 W&amp;B App의 W&amp;B 프로젝트로 이동합니다.
    2. 프로젝트 사이드바에서 **Overview** 탭을 선택합니다.
    3. 오른쪽 상단의 **Edit** 버튼 옆에 있는 별 아이콘을 선택합니다.

    <Frame>
      <img src="/images/track/star-project-overview-tab.png" alt="Overview 탭에서 프로젝트에 별 표시" />
    </Frame>
  </Tab>

  <Tab title="팀 프로필">
    1. `https://wandb.ai/<team>/projects`에서 팀 프로필 페이지로 이동합니다.
    2. **Projects** 탭을 선택합니다.
    3. 별을 표시하려는 프로젝트 옆으로 마우스를 가져갑니다. 나타나는 별 아이콘을 클릭합니다.

    예를 들어, 아래 이미지는 &quot;Compare&#95;Zoo&#95;Models&quot; 프로젝트 옆에 있는 별 아이콘을 보여줍니다.

    <Frame>
      <img src="/images/track/star-project-team-profile-page.png" alt="팀 페이지에서 프로젝트에 별 표시" />
    </Frame>
  </Tab>
</Tabs>

앱 왼쪽 상단에서 조직 이름을 클릭해, 조직 홈 화면에 해당 프로젝트가 표시되는지 확인합니다.

<div id="delete-a-project">
  ## 프로젝트 삭제
</div>

프로젝트 개요 탭 오른쪽에 있는 점 세 개 아이콘을 클릭해 프로젝트를 삭제할 수 있습니다.

1. W&amp;B 프로젝트로 이동합니다.
2. 프로젝트 사이드바에서 **Overview** 탭을 선택합니다.
3. 오른쪽 상단 모서리에 있는 점 세 개 메뉴 아이콘을 선택합니다.
4. 드롭다운 메뉴에서 **Delete project**를 선택합니다.

<Frame>
  <img src="/images/app_ui/delete_project.png" alt="드롭다운에서 프로젝트 삭제 옵션이 표시된 프로젝트 개요 메뉴" />
</Frame>

<Warning>
  프로젝트를 삭제해도 W&amp;B는 활성 [sweeps](/ko/models/sweeps) 또는 에이전트를 종료하지 않습니다.
</Warning>

<div id="add-notes-to-a-project">
  ## 프로젝트에 노트 추가하기
</div>

프로젝트에 대한 설명 개요로 노트를 추가하거나, 워크스페이스 내 마크다운 패널에 노트를 추가할 수 있습니다.

<div id="add-description-overview-to-a-project">
  ### 프로젝트에 개요 설명 추가하기
</div>

프로젝트 페이지에 추가한 설명은 프로필의 **Overview** 탭에 표시됩니다.

1. W&amp;B 프로젝트로 이동합니다
2. 프로젝트 사이드바에서 **Overview** 탭을 선택합니다
3. 오른쪽 상단에서 Edit를 선택합니다
4. **Description** 필드에 메모를 추가합니다
5. **Save** 버튼을 선택합니다

<Note>
  **실행을 비교하는 설명용 노트를 만들려면 Report를 생성하세요**

  W&amp;B Report를 생성해 플롯과 마크다운을 나란히 추가할 수도 있습니다. 서로 다른 섹션을 사용해 서로 다른 실행을 표시하고, 수행한 작업에 대한 스토리를 전달해 보세요.
</Note>

<div id="add-notes-to-run-workspace">
  ### 실행 워크스페이스에 메모 추가
</div>

1. W&amp;B 프로젝트로 이동합니다
2. 프로젝트 사이드바에서 **Workspace** 탭을 선택합니다
3. 오른쪽 상단의 **Add panels** 버튼을 선택합니다
4. 표시되는 모달에서 **TEXT AND CODE** 드롭다운을 선택합니다
5. **Markdown**을 선택합니다
6. 워크스페이스에 표시되는 마크다운 패널에 메모를 추가합니다
