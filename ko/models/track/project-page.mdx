A *프로젝트*는 결과를 시각화하고 실험을 비교하며 아티팩트를 확인·다운로드하고 Automation을 생성하는 등 다양한 작업을 수행할 수 있는 중앙 위치입니다.

<Note>
  각 프로젝트에는 누가 액세스할 수 있는지를 결정하는 공개 범위 설정이 있습니다. 프로젝트에 누가 액세스할 수 있는지에 대한 자세한 내용은 [Project visibility](/ko/platform/hosting/iam/access-management/restricted-projects)를 참조하세요.
</Note>

각 프로젝트에는 다음과 같은 탭이 있습니다:

* [Overview](/ko/models/track/project-page/#overview-tab): 프로젝트의 스냅샷
* [Workspace](/ko/models/track/project-page/#workspace-tab): 개인용 시각화 샌드박스
* [Runs](#runs-tab): 프로젝트의 모든 run이 나열된 표
* [Automations](#automations-tab): 프로젝트에 설정된 Automation
* [Sweeps](/ko/models/sweeps): 자동화된 탐색 및 최적화
* [Reports](/ko/models/track/project-page/#reports-tab): 노트, run, 그래프의 저장된 스냅샷
* [Artifacts](#artifacts-tab): 모든 run과 해당 run과 연관된 아티팩트를 포함합니다

<div id="overview-tab">
  ## Overview 탭
</div>

W&amp;B는 run을 초기화할 때 `project` 필드에 입력한 이름으로 프로젝트를 생성합니다.

**Overview** 탭에서 프로젝트 이름을 확인하고 프로젝트를 관리할 수 있습니다.

* 프로젝트의 이름, 설명 또는 팀을 변경하려면 오른쪽 상단의 **Edit**을 클릭합니다.
* 최근에 삭제된 run을 복구하려면 오른쪽 상단의 **...**을 클릭한 다음 **Undelete recently deleted runs**를 클릭합니다.
* 프로젝트를 삭제하려면 오른쪽 상단의 **...**을 클릭한 다음 **Delete project**를 클릭합니다. 확인 대화 상자의 내용을 읽고 안내에 따르세요. 프로젝트 삭제는 되돌릴 수 없습니다.

**Overview** 페이지의 나머지 영역은 **Details**와 **Project Roles** 탭으로 구성됩니다.

<div id="details">
  ### 세부 정보
</div>

프로젝트에 대한 세부 정보는 다음과 같습니다:

* **Project visibility**: 프로젝트의 공개 범위입니다. 누가 이 프로젝트에 액세스할 수 있는지를 결정하는 공개 설정입니다. 자세한 내용은 [Project visibility](/ko/platform/hosting/iam/access-management/restricted-projects)를 참조하세요.
* **Last active**: 이 프로젝트에 마지막으로 데이터가 로깅된 시점의 타임스탬프
* **Contributors**: 이 프로젝트에 기여한 사용자 수
* **Total runs**: 이 프로젝트에 포함된 총 run 수
* **Total compute**: 이 프로젝트에 있는 모든 run의 실행 시간을 합산해 계산한 값

**Details** 탭에는 프로젝트의 데이터와 메트릭에 액세스하고 내보내는 방법, 스윕에서 최고 성능의 모델을 다운로드하는 방법 등에 대한 안내도 포함됩니다.

<div id="workspace-tab">
  ## Workspace 탭
</div>

프로젝트의 *워크스페이스*는 실험을 비교하기 위한 개인 샌드박스를 제공합니다. 프로젝트를 사용해 같은 문제를 다른 아키텍처, 하이퍼파라미터, 데이터셋, 전처리 방식 등으로 실험한 모델들을 정리하고 비교할 수 있습니다.

**Runs Sidebar**: 프로젝트의 모든 run 목록입니다.

* **점 3개 메뉴**: 사이드바의 한 행 위에 마우스를 올리면 왼쪽에 메뉴가 나타납니다. 이 메뉴를 사용해 run 이름을 바꾸거나, run을 삭제하거나, 활성 run을 중지할 수 있습니다.
* **가시성 아이콘**: 그래프에서 run을 켜고 끄려면 눈 모양 아이콘을 클릭합니다.
* **색상**: run 색상을 미리 설정된 색 또는 커스텀 색으로 변경합니다.
* **검색**: 이름으로 run을 검색합니다. 이 검색은 플롯에 표시되는 run도 함께 필터링합니다.
* **필터**: 사이드바 필터를 사용해 표시되는 run 집합을 좁힙니다.
* **그룹**: 아키텍처와 같이 run을 동적으로 그룹화할 설정(config) 컬럼을 선택합니다. 그룹화를 사용하면 플롯에 평균값을 잇는 선과, 그래프 상 값의 분산을 나타내는 음영 영역이 표시됩니다.
* **정렬**: 예를 들어 손실이 가장 낮거나 정확도가 가장 높은 run처럼, run을 정렬할 기준 값을 선택합니다. 정렬은 그래프에 어떤 run이 표시될지에 영향을 줍니다.
* **확장 버튼**: 사이드바를 전체 테이블로 확장합니다.
* **run 개수**: 맨 위 괄호 안 숫자는 프로젝트의 전체 run 개수입니다. 뒤의 `(N visualized)` 숫자는 눈 아이콘이 켜져 있어 각 플롯에서 시각화 가능한 run 개수입니다. 아래 예시에서는 그래프가 183개의 run 중 처음 10개만 보여줍니다. 그래프를 편집해 표시 가능한 run의 최대 개수를 늘릴 수 있습니다.

[Runs 탭](#runs-tab)에서 컬럼을 고정(pinning)하거나 숨기거나 순서를 바꾸면, Runs Sidebar에도 이러한 사용자 지정 내용이 반영됩니다.

**Panels 레이아웃**: 이 임시 작업 공간을 사용해 결과를 탐색하고, 차트를 추가/삭제하며, 서로 다른 메트릭을 기준으로 모델 버전을 비교합니다.

<div id="add-a-section-of-panels">
  ### 패널 섹션 추가하기
</div>

섹션 드롭다운 메뉴를 클릭한 다음 &quot;Add section&quot;을 클릭해서 패널을 위한 새 섹션을 만듭니다. 섹션 이름을 바꾸고, 드래그해서 순서를 변경하고, 섹션을 펼치거나 접을 수 있습니다.

각 섹션의 오른쪽 상단에는 다음과 같은 옵션이 있습니다:

* **Add section**: 드롭다운 메뉴에서 위나 아래에 섹션을 추가하거나, 페이지 하단의 버튼을 클릭해서 새 섹션을 추가합니다.
* **Rename section**: 섹션의 제목을 변경합니다.
* **Export section to report**: 이 섹션의 패널을 새 리포트로 저장합니다.
* **Delete section**: 전체 섹션과 모든 차트를 제거합니다. 이 작업은 워크스페이스 바의 페이지 하단에 있는 실행 취소 버튼으로 되돌릴 수 있습니다.
* **Add panel**: 더하기(+) 버튼을 클릭해서 섹션에 패널을 추가합니다.

<Tabs>
  <Tab title="Customize columns">
    Runs 탭에서는 해당 프로젝트의 run에 대한 세부 정보를 보여 줍니다. 기본적으로 매우 많은 열이 표시됩니다.

    <Note>
      Runs 탭을 사용자 지정하면, 이 변경 사항은 [Workspace 탭](#workspace-tab)의 **Runs** 선택기에도 반영됩니다.
    </Note>

    * 보이는 모든 열을 확인하려면 페이지를 수평으로 스크롤하십시오.
    * 열의 순서를 변경하려면 열을 좌우로 드래그하십시오.
    * 열을 고정하려면 열 이름 위에 마우스를 올린 뒤 표시되는 `...` 액션 메뉴를 클릭하고 **Pin column**을 선택하십시오. 고정된 열은 페이지 왼쪽, **Name** 열 다음에 표시됩니다. 고정 해제하려면 **Unpin column**을 선택하십시오.
    * 열을 숨기려면 열 이름 위에 마우스를 올린 뒤 표시되는 `...` 액션 메뉴를 클릭하고 **Hide column**을 선택하십시오. 현재 숨겨져 있는 모든 열을 보려면 **Columns**를 클릭하십시오.
    * 여러 열을 한 번에 표시, 숨기기, 고정, 고정 해제하려면 **Columns**를 클릭하십시오.
      * 숨겨진 열의 이름을 클릭하면 다시 표시됩니다.
      * 표시 중인 열의 이름을 클릭하면 숨겨집니다.
      * 표시 중인 열 옆의 핀 아이콘을 클릭하면 해당 열이 고정됩니다.
  </Tab>

  <Tab title="Sort">
    다음 옵션 중 하나를 사용하여 특정 열의 값을 기준으로 Table의 모든 행을 정렬할 수 있습니다.

    1. 아무 열 헤더에 마우스를 올렸을 때 나타나는 케밥 메뉴(세로 점 세 개)를 클릭하여 해당 열의 값으로 테이블을 빠르게 정렬합니다.
    2. **Sort** 버튼을 클릭하고 드롭다운에서 원하는 열을 선택합니다. 이 패널을 사용하면 우선순위 순서대로 여러 열을 기준으로 정렬할 수 있습니다.

<div id="metadata-panel">
  ### 메타데이터 패널
</div>

<Frame>
  <img src="/images/app_ui/metadata_panel.png" alt="Artifact metadata panel" />
</Frame>

메타데이터 패널에서는 아티팩트의 메타데이터를 확인할 수 있습니다. 이 메타데이터에는 아티팩트를 재구성하는 데 필요한 설정 매개변수, 추가 정보를 확인할 수 있는 URL, 아티팩트를 로깅한 run 동안 생성된 메트릭 등이 포함될 수 있습니다. 또한 아티팩트를 생성한 run의 설정과 기록 메트릭을 함께 확인할 수 있습니다.

아티팩트의 메타데이터를 보려면:

1. 프로젝트 사이드바에서 **Artifacts** 탭을 선택합니다.
2. 목록에서 아티팩트를 선택해 해당 아티팩트의 최신 버전 상세 페이지를 엽니다.
3. **Metadata** 탭을 선택해 해당 아티팩트와 연관된 메타데이터를 확인합니다.

Usage 패널에서는 웹 앱 외부, 예를 들어 로컬 머신에서 아티팩트를 사용할 수 있도록 다운로드하는 코드 스니펫을 제공합니다. 이 섹션에는 아티팩트를 생성한 run과 이 아티팩트를 입력으로 사용하는 run으로 이동할 수 있는 링크도 표시됩니다.

아티팩트 사용 정보를 보려면

1. 프로젝트 사이드바에서 **Artifacts** 탭을 선택합니다.
2. 목록에서 아티팩트를 선택하여 해당 아티팩트의 최신 버전 상세 페이지를 봅니다.
3. **Usage** 탭을 선택하여 코드 스니펫과 관련 run을 확인합니다.

<div id="files-panel">
  ### Files 패널
</div>

<Frame>
  <img src="/images/app_ui/files_panel.png" alt="Artifact files panel" />
</Frame>

Files 패널에는 아티팩트와 연관된 파일과 폴더가 나열됩니다. W&amp;B는 run에 대해 일부 파일을 자동으로 업로드합니다. 예를 들어 `requirements.txt`에는 run이 사용한 각 라이브러리의 버전이 표시되며, `wandb-metadata.json`과 `wandb-summary.json`에는 run에 대한 정보가 포함됩니다. run의 설정에 따라 아티팩트나 미디어와 같은 다른 파일들도 업로드될 수 있습니다.

아티팩트에 로깅된 파일을 보려면 다음을 수행합니다:

1. 프로젝트 사이드바에서 **Artifacts** 탭을 선택합니다.
2. 목록에서 아티팩트를 선택하여 해당 아티팩트 최신 버전의 상세 페이지를 엽니다.
3. **Files** 탭을 선택하여 해당 아티팩트와 연관된 모든 파일을 확인합니다.

<div id="lineage-panel">
  ### Lineage 패널
</div>

<Frame>
  <img src="/images/app_ui/lineage_panel.png" alt="Artifact lineage" />
</Frame>

Lineage 패널은 프로젝트와 연결된 모든 아티팩트와, 이들을 서로 연결하는 run을 한눈에 보여 줍니다. run 유형은 블록으로, 아티팩트는 원으로 표시되며, 특정 유형의 run이 특정 유형의 아티팩트를 사용하거나 생성할 때는 화살표로 나타냅니다. 왼쪽 열에서 선택한 아티팩트의 유형은 강조 표시됩니다.

모든 개별 아티팩트 버전과 이들을 연결하는 구체적인 run을 보려면 Explode 토글을 클릭하세요.

<div id="versions-tab">
  ### Versions 탭
</div>

<Frame>
  <img src="/images/app_ui/versions_tab.png" alt="Artifact versions 탭" />
</Frame>

Versions 탭에는 아티팩트의 모든 버전이 표시됩니다. 특정 버전에 대한 세부 정보를 보려면 아티팩트를 선택합니다.

1. 프로젝트 사이드바에서 **Artifacts** 탭을 선택합니다.
2. 목록에서 아티팩트를 선택해 해당 아티팩트의 최신 버전에 대한 세부 정보 페이지를 엽니다.
3. **Versions** 탭을 선택해 해당 아티팩트의 모든 버전을 확인합니다.
4. 드롭다운(아티팩트 이름 옆)에서 **All Versions**를 선택합니다.

예를 들어, 위 이미지는 `"zoo-wyhak4p0"`라는 이름의 모델 아티팩트에 대해 서로 다른 버전의 모델 아티팩트를 보여줍니다.

<div id="create-a-project">
  ## 프로젝트 생성
</div>

프로젝트는 W&amp;B App에서 생성하거나, `wandb.init()` 호출 시 `project`를 지정해서 프로그래밍 방식으로 생성할 수 있습니다.

<Tabs>
  <Tab title="W&B 앱">
    W&amp;B App에서는 **Projects** 페이지나 팀 랜딩 페이지에서 프로젝트를 생성할 수 있습니다.

    **Projects** 페이지에서:

    1. 왼쪽 상단의 글로벌 내비게이션 아이콘을 클릭합니다. 프로젝트 사이드바가 열립니다.
    2. 내비게이션의 **Projects** 섹션에서 **View all**을 클릭해 프로젝트 개요 페이지를 엽니다.
    3. **Create new project**를 클릭합니다.
    4. **Team**을 이 프로젝트를 소유할 팀 이름으로 설정합니다.
    5. **Name** 필드에 프로젝트 이름을 입력합니다.
    6. 기본값이 **Team**인 **Project visibility**를 설정합니다.
    7. 필요하다면 **Description**을 입력합니다.
    8. **Create project**를 클릭합니다.

    팀 랜딩 페이지에서:

    1. 왼쪽 상단의 글로벌 내비게이션 아이콘을 클릭합니다. 프로젝트 사이드바가 열립니다.
    2. 내비게이션의 **Teams** 섹션에서 팀 이름을 클릭해 해당 팀의 랜딩 페이지를 엽니다.
    3. 랜딩 페이지에서 **Create new project**를 클릭합니다.
    4. **Team**은 현재 보고 있는 랜딩 페이지를 소유한 팀으로 자동 설정됩니다. 필요하면 팀을 변경합니다.
    5. **Name** 필드에 프로젝트 이름을 입력합니다.
    6. 기본값이 **Team**인 **Project visibility**를 설정합니다.
    7. 필요하다면 **Description**을 입력합니다.
    8. **Create project**를 클릭합니다.
  </Tab>

  <Tab title="Python SDK">
    프로그래밍 방식으로 프로젝트를 생성하려면 `wandb.init()`을 호출할 때 `project`를 지정합니다. 해당 프로젝트가 아직 존재하지 않으면 지정한 entity를 소유자로 하여 자동으로 생성됩니다. 예를 들면 다음과 같습니다:

    ```python
    import wandb with wandb.init(entity="<entity>", project="<project_name>") as run: run.log({"accuracy": .95})
    ```

    [`wandb.init()` API reference](/ko/models/ref/python/functions/init/#examples)를 참조하세요.
  </Tab>
</Tabs>

<div id="star-a-project">
  ## 프로젝트에 별표 표시하기
</div>

프로젝트에 별표를 추가하여 해당 프로젝트를 중요한 프로젝트로 표시할 수 있습니다. 여러분과 팀이 별표로 중요하다고 표시한 프로젝트는 조직 홈 페이지 맨 위에 표시됩니다.

예를 들어, 다음 이미지는 중요하다고 표시된 두 개의 프로젝트 `zoo_experiment`와 `registry_demo`를 보여 줍니다. 두 프로젝트 모두 조직 홈 페이지 상단의 **Starred projects** 섹션에 나타납니다.

<Frame>
  <img src="/images/track/star-projects.png" alt="Starred projects 섹션" />
</Frame>

프로젝트를 중요하다고 표시하는 방법은 두 가지입니다. 프로젝트의 Overview 탭에서 하거나 팀 프로필 페이지에서 할 수 있습니다.

<Tabs>
  <Tab title="Project overview">
    1. `https://wandb.ai/<team>/<project-name>`에서 W&amp;B App의 W&amp;B 프로젝트로 이동합니다.
    2. 프로젝트 사이드바에서 **Overview** 탭을 선택합니다.
    3. 오른쪽 상단에서 **Edit** 버튼 옆에 있는 별 아이콘을 선택합니다.

    <Frame>
      <img src="/images/track/star-project-overview-tab.png" alt="Overview에서 프로젝트에 별표 표시" />
    </Frame>
  </Tab>

  <Tab title="Team profile">
    1. `https://wandb.ai/<team>/projects`에서 팀의 프로필 페이지로 이동합니다.
    2. **Projects** 탭을 선택합니다.
    3. 별표를 추가하려는 프로젝트 옆에 마우스를 올립니다. 나타나는 별 아이콘을 클릭합니다.

    예를 들어, 다음 이미지는 &quot;Compare&#95;Zoo&#95;Models&quot; 프로젝트 옆의 별 아이콘을 보여 줍니다.

    <Frame>
      <img src="/images/track/star-project-team-profile-page.png" alt="팀 페이지에서 프로젝트에 별표 표시" />
    </Frame>
  </Tab>
</Tabs>

앱 왼쪽 상단에서 조직 이름을 클릭하여 프로젝트가 조직 랜딩 페이지에 표시되는지 확인합니다.

<div id="delete-a-project">
  ## 프로젝트 삭제하기
</div>

오버뷰 탭 오른쪽에 있는 점 세 개 아이콘을 클릭해서 프로젝트를 삭제할 수 있습니다.

1. W&amp;B 프로젝트로 이동합니다.
2. 프로젝트 사이드바에서 **Overview** 탭을 선택합니다.
3. 오른쪽 상단 모서리에 있는 점 세 개 메뉴를 선택합니다.
4. 드롭다운 메뉴에서 **Delete project**를 선택합니다.

<Frame>
  <img src="/images/app_ui/delete_project.png" alt="드롭다운에서 프로젝트 삭제 옵션이 표시된 프로젝트 오버뷰 메뉴" />
</Frame>

<Warning>
  프로젝트를 삭제해도 W&amp;B는 활성 상태인 [Sweeps](/ko/models/sweeps)나 에이전트를 종료하지 않습니다.
</Warning>

<div id="add-notes-to-a-project">
  ## 프로젝트에 노트 추가하기
</div>

프로젝트에 노트를 추가하려면 프로젝트에 대한 설명 개요를 작성하거나 워크스페이스 내에서 마크다운 패널로 추가할 수 있습니다.

<div id="add-description-overview-to-a-project">
  ### 프로젝트에 Overview 설명 추가하기
</div>

페이지에 추가한 설명은 프로필의 **Overview** 탭에 표시됩니다.

1. W&amp;B 프로젝트로 이동합니다.
2. 프로젝트 사이드바에서 **Overview** 탭을 선택합니다.
3. 오른쪽 상단의 **Edit**를 클릭합니다.
4. **Description** 필드에 메모를 추가합니다.
5. **Save** 버튼을 클릭합니다.

<Note>
  **run 비교용 설명 메모는 Reports에서 작성하세요**

  W&amp;B Report를 만들어 플롯과 마크다운을 나란히 추가할 수도 있습니다. 서로 다른 섹션을 사용해 서로 다른 run을 보여 주고, 작업 내용에 대한 이야기를 구성해 보세요.
</Note>

<div id="add-notes-to-run-workspace">
  ### run 워크스페이스에 메모 추가
</div>

1. W&amp;B 프로젝트로 이동합니다.
2. 프로젝트 사이드바에서 **Workspace** 탭을 선택합니다.
3. 오른쪽 상단의 **Add panels** 버튼을 선택합니다.
4. 표시되는 모달에서 **TEXT AND CODE** 드롭다운 메뉴를 선택합니다.
5. **Markdown**을 선택합니다.
6. 워크스페이스에 표시되는 마크다운 패널에 메모를 작성합니다.
