---
title: 실험 재현하기
---

팀원이 생성한 실험을 재현하여 결과를 검증하고 확인합니다.

실험을 재현하기 전에 다음 정보를 미리 확인해 두어야 합니다:

* run이 기록된 프로젝트의 이름
* 재현하려는 run의 이름

실험을 재현하려면 다음 단계를 따르십시오:

1. run이 기록된 프로젝트로 이동합니다.
2. 프로젝트 사이드바에서 **Workspace** 탭을 선택합니다.
3. 재현하려는 run을 클릭합니다. run 페이지가 열리며 기본적으로 **Overview** 탭이 표시됩니다.

이어서, 특정 해시 기준으로 실험 코드를 다운로드하거나 실험이 포함된 전체 리포지토리를 클론합니다.

<Tabs>
<Tab title="Python 스크립트 또는 노트북 다운로드">
실험의 Python 스크립트 또는 노트북을 다운로드합니다:

1. 기본으로 표시되는 **Overview** 탭의 **Command** 필드에서 실험을 생성한 스크립트의 이름을 확인해 둡니다.
2. run 페이지에서 **Code** 탭을 선택합니다.
3. 해당 스크립트 또는 노트북에 대응하는 파일 옆의 **Download**를 클릭합니다.
</Tab>
<Tab title="GitHub">
팀원이 실험을 생성할 때 사용한 GitHub 리포지토리를 클론합니다. 다음 단계를 수행합니다:

1. 필요한 경우, 팀원이 실험을 생성할 때 사용한 GitHub 리포지토리에 대한 액세스 권한을 획득합니다.
2. GitHub 리포지토리 URL이 포함된 **Git repository** 필드를 복사합니다.
3. 리포지토리를 클론합니다:
    ```bash
    git clone https://github.com/your-repo.git && cd your-repo
    ```
4. **Git state** 필드를 복사해 터미널에 붙여넣습니다. Git state는 팀원이 실험을 생성할 때 사용한 정확한 커밋을 체크아웃하는 Git 명령어 집합입니다. 다음 코드 스니펫에서 표시된 값을 자신의 값으로 바꾸십시오:
    ```bash
    git checkout -b "<run-name>" 0123456789012345678901234567890123456789
    ```
</Tab>
</Tabs>

5. run 페이지에서 **Files** 탭을 선택합니다.
6. `requirements.txt` 파일을 다운로드해 작업 디렉터리에 저장합니다. 이 디렉터리에는 클론한 GitHub 리포지토리 또는 다운로드한 Python 스크립트/노트북이 포함되어야 합니다.
7. (권장) Python 가상 환경을 생성합니다.
8. `requirements.txt` 파일에 지정된 요구 사항을 설치합니다.
    ```bash
    pip install -r requirements.txt
    ```

9. 이제 코드와 의존성이 준비되었으므로, 스크립트 또는 노트북을 실행해 실험을 재현할 수 있습니다. 리포지토리를 클론했다면, 스크립트 또는 노트북이 위치한 디렉터리로 이동해야 할 수 있습니다. 그렇지 않다면 작업 디렉터리에서 그대로 스크립트 또는 노트북을 실행하면 됩니다.

<Tabs>
<Tab title="Python 노트북">
Python 노트북을 다운로드했다면, 노트북을 다운로드한 디렉터리로 이동한 후 터미널에서 다음 명령을 실행합니다:
```bash
jupyter notebook
```
</Tab>
<Tab title="Python 스크립트">
Python 스크립트를 다운로드했다면, 스크립트를 다운로드한 디렉터리로 이동한 후 터미널에서 다음 명령을 실행합니다. `<>`로 둘러싸인 값을 자신의 값으로 바꾸십시오:

```bash
python <your-script-name>.py
```
</Tab>
</Tabs>
