title: 실험 재현하기
description: "run과 연결된 코드, 의존성, 설정을 다운로드하여 팀원의 W&B 실험을 재현하세요."
---

팀원이 만든 실험을 재현해 결과를 확인하고 검증할 수 있습니다. 팀원이 실험을 만드는 데 사용한 GitHub 저장소를 클론하거나, 실험을 생성한 Python 스크립트 또는 노트북을 다운로드하여 실험을 재현할 수 있습니다. 자신에게 가장 적합한 방법을 선택하세요.

실험을 생성한 코드와 코드를 실행하는 데 필요한 의존성을 획득하세요. 아래 단계를 따르면 됩니다.

<Tabs>
  <Tab title="GitHub 저장소로 코드 상태 복제">
    다음 단계에서는 GitHub 저장소의 코드 상태를 복제하는 방법을 설명합니다.

    1. run이 로깅된 프로젝트로 이동합니다.
    2. 프로젝트 사이드바에서 **Workspace** 탭을 선택합니다.
    3. 재현하려는 run을 클릭합니다. run 페이지가 열리며 기본적으로 **Overview** 탭이 표시됩니다.
    4. **Command** 필드 옆의 **Reproduce this run** 버튼에 마우스를 올립니다. 실험 재현 단계가 포함된 모달이 나타납니다. 이 모달에는 코드 상태를 설정하는 방법과 실험을 실행할 명령이 설명되어 있습니다.

       <img src="/images/experiments/reproduce_run_button.png" alt="run 재현 버튼" width="500" />

    또는 다음 단계에 따라 수동으로 실험을 재현할 수도 있습니다.

    1. **Git repository** 필드를 복사해 터미널에 붙여 넣고 저장소를 클론합니다. 다음 코드 스니펫은 예시입니다. `your-repo`를 저장소 이름으로 바꾸세요.
       ```bash
       git clone https://github.com/your-repo.git && cd your-repo
       ```
    2. **Git state** 필드를 복사해 터미널에 붙여 넣고, 팀원이 실험을 생성할 때 사용한 정확한 커밋을 체크아웃합니다. 다음 코드 스니펫은 이를 수행하는 예시입니다. 아래 코드 스니펫의 값을 자신의 값으로 바꾸세요.
       ```bash
       git checkout -b "<run-name>" 0123456789012345678901234567890123456789
       ```
  </Tab>

  <Tab title="소스 코드 다운로드">
    다음 단계에서는 실험을 생성한 Python 스크립트 또는 노트북을 다운로드하는 방법을 설명합니다.

    1. run이 로깅된 프로젝트로 이동합니다.
    2. 프로젝트 사이드바에서 **Workspace** 탭을 선택합니다.
    3. 재현하려는 run을 클릭합니다. run 페이지가 열리며 기본적으로 **Overview** 탭이 표시됩니다.
    4. 기본적으로 표시되는 **Overview** 탭의 **Command** 필드에서 실험을 생성한 스크립트 이름을 기록해 둡니다.
    5. run 페이지에서 **Code** 탭을 선택합니다.
    6. 스크립트 또는 노트북에 해당하는 파일 옆의 **Download**를 클릭합니다.
    7. run 페이지에서 **Files** 탭을 선택합니다.
    8. `requirements.txt` 파일을 다운로드해 작업 디렉터리에 저장합니다. 이 디렉터리에는 클론한 GitHub 저장소 또는 다운로드한 Python 스크립트나 노트북이 있어야 합니다.
    9. `requirements.txt` 파일에 지정된 requirements를 설치합니다.
       ```bash
       pip install -r requirements.txt
       ```
  </Tab>
</Tabs>
