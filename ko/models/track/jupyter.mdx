Jupyter와 함께 W&amp;B를 사용하여 노트북을 벗어나지 않고 대화형 시각화를 구현하세요. 맞춤형 분석, 실험, 프로토타입을 결합하여 모두 완전히 로깅할 수 있습니다.

<div id="use-cases-for-wb-with-jupyter-notebooks">
  ## Jupyter 노트북에서 W&amp;B 활용 사례
</div>

1. **반복적 실험**: 파라미터를 조정하면서 실험을 실행하고 재실행하면, 수동으로 메모하지 않아도 모든 Runs가 자동으로 W&amp;B에 저장됩니다.
2. **Code saving**: 모델을 재현할 때 노트북에서 어떤 셀이 실행되었는지, 어떤 순서로 실행되었는지 파악하기 어렵습니다. [설정 페이지](/ko/platform/app/settings-page/)에서 code saving을 활성화하면 각 실험의 셀 실행 기록을 저장할 수 있습니다.
3. **맞춤형 분석**: Runs가 W&amp;B에 로깅되면 API에서 데이터프레임을 쉽게 조회하여 맞춤형 분석을 수행하고, 그 결과를 W&amp;B에 로깅하여 리포트에서 저장하고 공유할 수 있습니다.

<div id="getting-started-in-a-notebook">
  ## 노트북에서 시작하기
</div>

다음 코드로 노트북을 시작하여 W&amp;B를 설치하고 계정을 연결하세요:

```notebook
!pip install wandb -qqq
import wandb
wandb.login()
```

다음으로, 실험을 설정하고 하이퍼파라미터를 저장하세요:

```python
wandb.init(
    project="jupyter-projo",
    config={
        "batch_size": 128,
        "learning_rate": 0.01,
        "dataset": "CIFAR-100",
    },
)
```

`wandb.init()`를 실행한 후, 새 셀에서 `%%wandb`를 시작하면 노트북에서 실시간 그래프를 볼 수 있습니다. 이 셀을 여러 번 실행하면 데이터가 run에 추가됩니다.

```notebook
%%wandb

# 트레이닝 루프를 여기에 작성하세요
```

이 [예시 노트북](https://wandb.me/jupyter-interact-colab)에서 직접 사용해 보세요.

<div id="rendering-live-wb-interfaces-directly-in-your-notebooks">
  ### 노트북에서 직접 W&amp;B 인터페이스 렌더링하기
</div>

`%wandb` 매직을 사용하여 기존 대시보드, 스윕 또는 리포트를 노트북에서 직접 표시할 수도 있습니다:

```notebook
# 프로젝트 Workspace 표시
%wandb USERNAME/PROJECT
# 단일 run 표시
%wandb USERNAME/PROJECT/runs/RUN_ID
# 스윕 표시
%wandb USERNAME/PROJECT/sweeps/SWEEP_ID
# 리포트 표시
%wandb USERNAME/PROJECT/reports/REPORT_ID
# 삽입된 iframe의 높이 지정
%wandb USERNAME/PROJECT -h 2048
```

`%%wandb` 또는 `%wandb` 매직 대신, `wandb.init()`을 실행한 후 셀 끝에 `wandb.Run.finish()`를 추가하여 인라인 그래프를 표시하거나, API에서 반환된 리포트, 스윕, 또는 run 객체에 대해 `ipython.display(...)`를 호출할 수 있습니다.

```python
import wandb
from IPython.display import display
# run 초기화
run = wandb.init()

# 셀이 run.finish()를 출력하면 실시간 그래프를 볼 수 있습니다
run.finish()
```

<div id="additional-jupyter-features-in-wb">
  ## W&amp;B의 추가 Jupyter 기능
</div>

1. **Colab에서 간편한 인증**: Colab에서 처음으로 `wandb.init`을 호출하면, 브라우저에서 W&amp;B에 로그인되어 있는 경우 런타임이 자동으로 인증됩니다. 실행 페이지의 Overview 탭에서 Colab으로 연결되는 링크를 확인할 수 있습니다.
2. **Jupyter Magic:** 노트북에서 대시보드, 스윕, 리포트를 직접 표시합니다. `%wandb` magic은 프로젝트, 스윕 또는 리포트 경로를 받아 노트북 내에서 W&amp;B 인터페이스를 직접 렌더링합니다.
3. **도커화된 Jupyter 실행**: `wandb docker --jupyter`를 호출하여 도커 컨테이너를 실행하고, 코드를 마운트하고, Jupyter가 설치되어 있는지 확인한 후 포트 8888에서 실행합니다.
4. **임의 순서로 셀 실행 시 걱정 없음**: 기본적으로 `wandb.init`이 다음에 호출될 때까지 기다렸다가 실행을 `finished`로 표시합니다. 이를 통해 여러 셀(예: 데이터 설정용, 트레이닝용, 테스트용)을 원하는 순서로 실행하면서 모두 동일한 실행에 로깅할 수 있습니다. [User Settings](https://wandb.ai/settings)에서 code saving을 활성화하면, 실행된 셀을 순서대로, 실행 당시의 상태로 로깅하여 가장 비선형적인 파이프라인도 재현할 수 있습니다. Jupyter 노트북에서 실행을 수동으로 완료 상태로 표시하려면 `run.finish`를 호출하십시오.

```python
import wandb

run = wandb.init()

# 트레이닝 스크립트 및 로깅 코드를 여기에 작성하세요

run.finish()
```
