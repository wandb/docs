노트북을 벗어나지 않고 Jupyter에서 W&amp;B를 사용해 대화형 시각화를 확인할 수 있습니다. 사용자 정의 분석, 실험, 프로토타입을 하나로 결합하고, 전체 과정을 W&amp;B로 완전히 기록하세요.

<div id="use-cases-for-wb-with-jupyter-notebooks">
  ## Jupyter 노트북에서 W&amp;B를 사용하는 활용 사례
</div>

1. **반복적인 실험**: 실험을 실행하고 재실행하면서 파라미터를 조정할 때, 중간에 따로 메모를 남기지 않아도 수행한 모든 실행이 자동으로 W&amp;B에 저장됩니다.
2. **코드 저장**: 모델을 재현할 때는 노트북에서 어떤 셀이 어떤 순서로 실행되었는지 파악하기 어렵습니다. 각 실험에 대한 셀 실행 기록을 남기려면 [설정 페이지](/ko/platform/app/settings-page/)에서 코드 저장 기능을 켜세요.
3. **사용자 정의 분석**: 실행이 W&amp;B에 기록되면 API에서 데이터프레임을 쉽게 가져와 사용자 정의 분석을 수행한 뒤, 그 결과를 다시 W&amp;B에 기록하여 리포트에서 저장하고 공유할 수 있습니다.

<div id="getting-started-in-a-notebook">
  ## 노트북에서 시작하기
</div>

노트북을 시작할 때 아래 코드를 실행해 W&amp;B를 설치하고 계정을 연결하세요:

```notebook
!pip install wandb -qqq
import wandb
wandb.login()
```

다음으로 실험을 설정하고 하이퍼파라미터를 저장하세요:

```python
wandb.init(
    project="jupyter-projo",
    config={
        "batch_size": 128,
        "learning_rate": 0.01,
        "dataset": "CIFAR-100",
    },
)
```

`wandb.init()`을 실행한 후, 노트북에서 실시간 그래프를 보려면 `%%wandb`로 새 셀을 시작하세요. 이 셀을 여러 번 실행하면 데이터가 같은 실행에 계속 추가됩니다.

```notebook
%%wandb

# 여기에 학습 루프를 작성하세요
```

이 [예제 노트북](https://wandb.me/jupyter-interact-colab)에서 직접 시도해 보세요.

<div id="rendering-live-wb-interfaces-directly-in-your-notebooks">
  ### 노트북에서 W&amp;B 인터페이스를 실시간으로 렌더링하기
</div>

`%wandb` 매직 명령어를 사용해 기존 대시보드, 스윕, 리포트를 노트북에서 바로 표시할 수도 있습니다:

```notebook
# 프로젝트 워크스페이스 표시
%wandb USERNAME/PROJECT
# 단일 실행 표시
%wandb USERNAME/PROJECT/runs/RUN_ID
# sweep 표시
%wandb USERNAME/PROJECT/sweeps/SWEEP_ID
# 리포트 표시
%wandb USERNAME/PROJECT/reports/REPORT_ID
# 임베드된 iframe의 높이 지정
%wandb USERNAME/PROJECT -h 2048
```

`%%wandb` 또는 `%wandb` 매직 대신, `wandb.init()`을 실행한 이후에는 셀의 마지막에서 `wandb.Run.finish()`를 호출해 인라인 그래프를 표시하거나, W&amp;B API에서 반환된 리포트, 스윕, 실행 객체에 대해 `ipython.display(...)`를 호출할 수 있습니다.

```python
import wandb
from IPython.display import display
# 실행 초기화
run = wandb.init()

# 셀이 run.finish()를 출력하면 실시간 그래프를 볼 수 있습니다
run.finish()
```

<div id="additional-jupyter-features-in-wb">
  ## W&amp;B에서 제공하는 추가 Jupyter 기능
</div>

1. **Colab에서 간편한 인증**: Colab에서 `wandb.init`을 처음 호출하면, 브라우저에서 W&amp;B에 로그인되어 있는 경우 현재 런타임을 자동으로 인증합니다. 실행 페이지의 개요 탭에서 해당 Colab으로 이동하는 링크를 확인할 수 있습니다.
2. **Jupyter 매직:** 대시보드, 스윕, 리포트를 노트북 안에 직접 표시할 수 있습니다. `%wandb` 매직 명령은 프로젝트, 스윕 또는 리포트에 대한 경로를 인자로 받아, 노트북 안에서 바로 W&amp;B 인터페이스를 렌더링합니다.
3. **도커화된 Jupyter 실행**: `wandb docker --jupyter`를 호출하면 도커 컨테이너를 실행하고, 그 안에 코드를 마운트한 뒤, Jupyter가 설치되어 있는지 확인하고, 포트 8888에서 Jupyter를 실행합니다.
4. **셀을 순서에 상관없이 안심하고 실행하기**: 기본적으로, 다음에 `wandb.init`이 호출될 때까지 기다렸다가 실행을 `finished` 상태로 표시합니다. 덕분에 여러 개의 셀(예를 들어, 하나는 데이터 설정, 하나는 학습, 하나는 테스트)을 원하는 순서로 실행하더라도 모두 같은 실행에 로그를 남길 수 있습니다. [User Settings](https://wandb.ai/settings)에서 코드 저장을 켜면, 실행된 셀을 실행 순서와 당시 상태 그대로 함께 기록하여, 가장 비선형적인 파이프라인도 재현할 수 있습니다. Jupyter 노트북에서 실행을 수동으로 완료 처리하려면 `run.finish`를 호출하세요.

```python
import wandb

run = wandb.init()

# 훈련 스크립트 및 로깅 코드를 여기에 작성하세요

run.finish()
```
