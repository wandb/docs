W&amp;B Python 라이브러리를 사용해 CSV 파일을 로깅하고 [W&amp;B Dashboard](/ko/models/track/workspaces/)에서 시각화할 수 있습니다. W&amp;B Dashboard는 머신 러닝 모델 결과를 정리하고 시각화하는 중심 허브입니다. 이는 특히 W&amp;B에 기록되지 않은 [이전 머신 러닝 실험 정보가 들어 있는 CSV 파일](#import-and-log-your-csv-of-experiments)이 있거나, [데이터셋이 포함된 CSV 파일](#import-and-log-your-dataset-csv-file)이 있을 때 특히 유용합니다.

<div id="import-and-log-your-dataset-csv-file">
  ## 데이터셋 CSV 파일 가져오기 및 로그 기록하기
</div>

{/* {% embed url="https://drive.google.com/file/d/1jBG3M4VnaMgeclRzowYZEYvFxvwb9SXF/view?usp=sharing" %} */}

CSV 파일의 내용을 더 쉽게 재사용할 수 있도록 W&amp;B Artifacts 기능을 사용하는 것을 권장합니다.

1. 시작하려면 먼저 CSV 파일을 가져오십시오. 다음 코드 스니펫에서 `iris.csv` 파일 이름을 사용 중인 CSV 파일 이름으로 바꾸십시오:

```python
import wandb
import pandas as pd

# CSV를 새 DataFrame으로 읽기
new_iris_dataframe = pd.read_csv("iris.csv")
```

2. CSV 파일을 W&amp;B Table로 변환해 [W&amp;B Dashboards](/ko/models/track/workspaces/)를 활용하세요.

```python
# DataFrame을 W&B Table로 변환
iris_table = wandb.Table(dataframe=new_iris_dataframe)
```

3. 다음으로 W&amp;B Artifact를 생성하고, 해당 Artifact에 테이블을 추가합니다.

```python
# 행 제한을 200000으로 늘리고 재사용을 용이하게 하기 위해
# 테이블을 Artifact에 추가합니다
iris_table_artifact = wandb.Artifact("iris_artifact", type="dataset")
iris_table_artifact.add(iris_table, "iris_table")

# 데이터를 보존하기 위해 아티팩트 내에 원본 csv 파일을 기록합니다
iris_table_artifact.add_file("iris.csv")
```

W&amp;B Artifacts에 대한 자세한 내용은 [Artifacts 장](/ko/models/artifacts/)을 참고하세요.

4. 마지막으로, `wandb.init`으로 새 W&amp;B 실행을 시작해 W&amp;B에 추적 및 로깅하세요:

```python
# 데이터를 기록할 W&B 실행 시작
with wandb.init(project="tables-walkthrough") as run:

    # 실행으로 시각화할 테이블 기록...
    run.log({"iris": iris_table})

    # 사용 가능한 행 제한을 늘리기 위해 Artifact로도 기록!
    run.log_artifact(iris_table_artifact)
```

`wandb.init()` API는 데이터를 실행에 로그하기 위해 새로운 백그라운드 프로세스를 생성하고, 기본적으로 데이터를 wandb.ai와 동기화합니다. W&amp;B Workspace 대시보드에서 실시간 시각화를 확인할 수 있습니다. 아래 이미지는 코드 스니펫 데모를 실행했을 때의 결과를 보여줍니다.

위에서 살펴본 코드 스니펫을 모두 포함한 전체 스크립트는 아래와 같습니다:

```python
import wandb
import pandas as pd

# CSV를 새 DataFrame으로 읽기
new_iris_dataframe = pd.read_csv("iris.csv")

# DataFrame을 W&B Table로 변환
iris_table = wandb.Table(dataframe=new_iris_dataframe)

# 행 제한을 200000으로 늘리고 재사용을 용이하게 하기 위해
# 테이블을 Artifact에 추가
iris_table_artifact = wandb.Artifact("iris_artifact", type="dataset")
iris_table_artifact.add(iris_table, "iris_table")

# 데이터 보존을 위해 아티팩트 내에 원본 csv 파일 기록
iris_table_artifact.add_file("iris.csv")

# 데이터 기록을 위해 W&B 실행 시작
with wandb.init(project="tables-walkthrough") as run:

    # 실행에서 시각화하기 위해 테이블 기록...
    run.log({"iris": iris_table})

    # 사용 가능한 행 제한을 늘리기 위해 Artifact로도 기록!
    run.log_artifact(iris_table_artifact)
```

  ## 실험 CSV 가져오기 및 기록하기
</div>

  아래 표는 변환 후 다음과 같은 W&B 대시보드로 변환됩니다
  {% endembed %} */}

어떤 경우에는 실험 세부 정보가 CSV 파일에 있을 수 있습니다. 이러한 CSV 파일에 일반적으로 포함되는 세부 정보는 다음과 같습니다:

* 실험 실행 이름
* 초기 [노트](/ko/models/runs/#add-a-note-to-a-run)
* 실험을 구분하기 위한 [태그](/ko/models/runs/tags/)
* 실험에 필요한 구성(여기에 더해 [Sweeps Hyperparameter Tuning](/ko/models/sweeps/)을 활용할 수 있는 장점이 있습니다).

| Experiment   | Model Name       | Notes                      | Tags         | Num Layers | Final Train Acc | Final Val Acc | Training Losses                      |
| ------------ | ---------------- | -------------------------- | ------------ | ---------- | --------------- | ------------- | ------------------------------------ |
| Experiment 1 | mnist-300-layers | 학습 데이터에 과적합이 지나치게 심함       | [latest]     | 300        | 0.99            | 0.90          | [0.55, 0.45, 0.44, 0.42, 0.40, 0.39] |
| Experiment 2 | mnist-250-layers | 현재까지 최상의 모델                | [prod, best] | 250        | 0.95            | 0.96          | [0.55, 0.45, 0.44, 0.42, 0.40, 0.39] |
| Experiment 3 | mnist-200-layers | 베이스라인 모델보다 성능이 떨어짐. 디버깅 필요 | [debug]      | 200        | 0.76            | 0.70          | [0.55, 0.45, 0.44, 0.42, 0.40, 0.39] |
| ...          | ...              | ...                        | ...          | ...        | ...             | ...           |                                      |
| Experiment N | mnist-X-layers   | NOTES                      | ...          | ...        | ...             | ...           | [..., ...]                           |

W&amp;B는 실험 정보를 담은 CSV 파일을 받아 이를 W&amp;B 실험 실행으로 변환할 수 있습니다. 다음 코드 스니펫과 스크립트는 실험 CSV 파일을 가져와 로깅하는 방법을 보여줍니다:

1. 시작하려면 먼저 CSV 파일을 읽어서 Pandas DataFrame으로 변환합니다. `"experiments.csv"`를 사용자의 CSV 파일 이름으로 바꾸십시오:

```python
import wandb
import pandas as pd

FILENAME = "experiments.csv"
loaded_experiment_df = pd.read_csv(FILENAME)

PROJECT_NAME = "Converted Experiments"

EXPERIMENT_NAME_COL = "Experiment"
NOTES_COL = "Notes"
TAGS_COL = "Tags"
CONFIG_COLS = ["Num Layers"]
SUMMARY_COLS = ["Final Train Acc", "Final Val Acc"]
METRIC_COLS = ["Training Losses"]

# 작업하기 편하도록 Pandas DataFrame 형식 변환
for i, row in loaded_experiment_df.iterrows():
    run_name = row[EXPERIMENT_NAME_COL]
    notes = row[NOTES_COL]
    tags = row[TAGS_COL]

    config = {}
    for config_col in CONFIG_COLS:
        config[config_col] = row[config_col]

    metrics = {}
    for metric_col in METRIC_COLS:
        metrics[metric_col] = row[metric_col]

    summaries = {}
    for summary_col in SUMMARY_COLS:
        summaries[summary_col] = row[summary_col]
```

2. 다음으로, 새로운 W&amp;B 실행을 시작해 [`wandb.init()`](/ko/models/ref/python/functions/init)으로 W&amp;B에 추적하고 로그를 기록합니다:

   ```python
   with wandb.init(
       project=PROJECT_NAME, name=run_name, tags=tags, notes=notes, config=config
   ) as run:
   ```

실험이 진행되는 동안, 지표의 각 값을 모두 로깅하여 W&amp;B에서 조회, 쿼리, 분석할 수 있도록 할 수 있습니다. 이를 위해 [`run.log()`](/ko/models/ref/python/experiments/run/#method-runlog) 명령을 사용합니다:

```python
run.log({key: val})
```

[`define_metric`](/ko/models/ref/python/experiments/run#define_metric) API를 사용하면 실행의 결과를 정의하는 최종 요약 지표를 선택적으로 기록할 수 있습니다. 이 예제에서는 `run.summary.update()`를 사용해 실행에 요약 지표를 추가합니다:

```python
run.summary.update(summaries)
```

요약 메트릭에 대한 자세한 내용은 [요약 메트릭 로깅](./log-summary)을 참조하세요.

다음은 위의 샘플 테이블을 [W&amp;B Dashboard](/ko/models/track/workspaces/)로 변환하는 전체 예시 스크립트입니다:

```python
FILENAME = "experiments.csv"
loaded_experiment_df = pd.read_csv(FILENAME)

PROJECT_NAME = "Converted Experiments"

EXPERIMENT_NAME_COL = "Experiment"
NOTES_COL = "Notes"
TAGS_COL = "Tags"
CONFIG_COLS = ["Num Layers"]
SUMMARY_COLS = ["Final Train Acc", "Final Val Acc"]
METRIC_COLS = ["Training Losses"]

for i, row in loaded_experiment_df.iterrows():
    run_name = row[EXPERIMENT_NAME_COL]
    notes = row[NOTES_COL]
    tags = row[TAGS_COL]

    config = {}
    for config_col in CONFIG_COLS:
        config[config_col] = row[config_col]

    metrics = {}
    for metric_col in METRIC_COLS:
        metrics[metric_col] = row[metric_col]

    summaries = {}
    for summary_col in SUMMARY_COLS:
        summaries[summary_col] = row[summary_col]

    with  wandb.init(
        project=PROJECT_NAME, name=run_name, tags=tags, notes=notes, config=config
    ) as run:

        for key, val in metrics.items():
            if isinstance(val, list):
                for _val in val:
                    run.log({key: _val})
            else:
                run.log({key: val})

        run.summary.update(summaries)
```
