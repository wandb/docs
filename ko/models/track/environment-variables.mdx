자동화된 환경에서 스크립트를 실행하는 경우, 스크립트 실행 전에 또는 스크립트 내에서 설정한 환경 변수로 W&amp;B를 제어할 수 있습니다.

```bash
# 이 값은 비밀이므로 버전 관리에 체크인하지 마세요
WANDB_API_KEY=$YOUR_API_KEY
# 이름과 메모는 선택 사항입니다
WANDB_NAME="My first run"
WANDB_NOTES="Smaller learning rate, more regularization."
```

```bash
# wandb/settings 파일을 버전 관리에 포함하지 않는 경우에만 필요
WANDB_ENTITY=$username
WANDB_PROJECT=$project
```

```python
# 스크립트를 클라우드에 동기화하지 않으려면
os.environ["WANDB_MODE"] = "offline"

# 실행 객체 및 관련 클래스에 sweep ID 추적 추가
os.environ["WANDB_SWEEP_ID"] = "b05fq58z"
```

<div id="optional-environment-variables">
  ## 선택적 환경 변수
</div>

원격 머신에서 인증을 설정하는 등의 작업을 할 때 사용할 수 있는 선택적 환경 변수입니다.

| 변수 이름                   | 용도                                                                                                                                                                                                            |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `WANDB_API_KEY`         | 계정과 연결된 인증 키를 설정합니다. [User Settings](https://wandb.ai/settings)에서 API 키를 생성하세요. 원격 머신에서 아직 `wandb login`을 실행하지 않았다면 이 값을 반드시 설정해야 합니다.                                                                        |
| `WANDB_BASE_URL`        | [wandb/local](/ko/platform/hosting/)을 사용 중이라면 이 환경 변수를 `http://YOUR_IP:YOUR_PORT`로 설정해야 합니다.                                                                                                                     |
| `WANDB_CACHE_DIR`       | 기본값은 ~/.cache/wandb이며, 이 환경 변수를 사용해 이 위치를 변경할 수 있습니다.                                                                                                                                                         |
| `WANDB_CONFIG_DIR`      | 기본값은 ~/.config/wandb이며, 이 환경 변수를 사용해 이 위치를 변경할 수 있습니다.                                                                                                                                                        |
| `WANDB_CONFIG_PATHS`    | wandb.config에 로드할 YAML 파일들의 목록을 쉼표로 구분해 지정합니다. [config](./config#file-based-configs)를 참조하세요.                                                                                                                  |
| `WANDB_CONSOLE`         | stdout/stderr 로깅을 비활성화하려면 이 값을 &quot;off&quot;로 설정하세요. 이를 지원하는 환경에서는 기본값이 &quot;on&quot;입니다.                                                                                                                  |
| `WANDB_DATA_DIR`        | 스테이징 아티팩트를 업로드할 위치입니다. 기본 위치는 `platformdirs` Python 패키지의 `user_data_dir` 값을 사용하므로 사용하는 플랫폼에 따라 달라집니다. 이 디렉터리가 존재하며, 현재 실행 중인 사용자에게 이 디렉터리에 대한 쓰기 권한이 있는지 확인하세요.                                               |
| `WANDB_DIR`             | 모든 생성된 파일을 저장할 위치입니다. 설정하지 않으면 학습 스크립트 기준 상대 경로에 있는 `wandb` 디렉터리가 기본값입니다. 이 디렉터리가 존재하며 실행 중인 사용자가 여기에 쓸 수 있는 권한이 있는지 확인하세요. 이 환경 변수는 다운로드된 아티팩트의 위치를 제어하지 않으며, 해당 위치는 `WANDB_ARTIFACT_DIR` 환경 변수로 설정할 수 있습니다. |
| `WANDB_ARTIFACT_DIR`    | 다운로드된 모든 아티팩트를 저장할 위치입니다. 설정하지 않으면 학습 스크립트 기준 상대 경로에 있는 `artifacts` 디렉터리가 기본값입니다. 이 디렉터리가 존재하고 현재 실행 중인 사용자에게 여기에 쓸 수 있는 권한이 있는지 확인하세요. 생성된 메타데이터 파일의 위치는 제어하지 않으며, 이는 `WANDB_DIR` 환경 변수로 설정할 수 있습니다.         |
| `WANDB_DISABLE_GIT`     | wandb가 Git 저장소를 탐색해 최신 커밋과 diff를 수집하지 않도록 합니다.                                                                                                                                                                |
| `WANDB_DISABLE_CODE`    | wandb가 노트북이나 Git diff를 저장하지 않도록 하려면 이 값을 true로 설정하세요. Git 저장소 내에 있는 경우 현재 커밋은 계속 저장됩니다.                                                                                                                       |
| `WANDB_DOCKER`          | 실행 복원을 활성화하려면 docker 이미지 다이제스트로 설정합니다. 이는 wandb docker 명령으로 자동으로 설정됩니다. `wandb docker my/image/name:tag --digest`를 실행하여 이미지 다이제스트를 얻을 수 있습니다.                                                                 |
| `WANDB_ENTITY`          | 실행과 연결된 엔터티입니다. 학습 스크립트가 있는 디렉터리에서 `wandb init`을 실행하면 *wandb*라는 디렉터리가 생성되고, 기본 엔터티가 저장되어 버전 관리 시스템에 커밋할 수 있습니다. 해당 파일을 만들고 싶지 않거나 파일의 값을 덮어쓰고 싶다면 이 환경 변수를 사용할 수 있습니다.                                        |
| `WANDB_ERROR_REPORTING` | wandb가 치명적인 오류를 자체 오류 추적 시스템에 기록하지 않도록 하려면 이 값을 false로 설정하세요.                                                                                                                                                 |
| `WANDB_HOST`            | 시스템에서 제공하는 호스트 이름을 사용하고 싶지 않은 경우, wandb 인터페이스에 표시할 호스트 이름을 여기에 설정하세요.                                                                                                                                         |
| `WANDB_IGNORE_GLOBS`    | 무시할 파일 glob 패턴을 쉼표로 구분한 목록으로 설정하세요. 이러한 파일은 클라우드와 동기화되지 않습니다.                                                                                                                                                 |
| `WANDB_JOB_NAME`        | `wandb`로 생성되는 모든 잡의 이름을 지정합니다.                                                                                                                                                                                |
| `WANDB_JOB_TYPE`        | &quot;training&quot; 또는 &quot;evaluation&quot;과 같은 작업 유형을 지정해 서로 다른 실행 유형을 구분합니다. 자세한 내용은 [그룹화](/ko/models/runs/grouping/)를 참조하세요.                                                                               |
| `WANDB_MODE`            | 이 값을 &quot;offline&quot;으로 설정하면 wandb는 실행 메타데이터를 로컬에만 저장하고 서버와는 동기화하지 않습니다. 이 값을 `disabled`로 설정하면 wandb는 완전히 비활성화됩니다.                                                                                         |
| `WANDB_NAME`            | 실행의 사람이 읽을 수 있는 이름입니다. 설정하지 않으면 임의로 생성됩니다.                                                                                                                                                                    |
| `WANDB_NOTEBOOK_NAME`   | Jupyter에서 실행 중인 경우, 이 변수를 사용해 노트북 이름을 설정할 수 있습니다. 자동으로 감지하려고 시도합니다.                                                                                                                                           |
| `WANDB_NOTES`           | 실행에 대한 긴 메모입니다. Markdown을 사용할 수 있으며, UI에서 나중에 수정할 수 있습니다.                                                                                                                                                     |
| `WANDB_PROJECT`         | 실행이 속한 프로젝트입니다. `wandb init`으로도 설정할 수 있지만, 환경 변수로 설정한 값이 이를 우선합니다.                                                                                                                                            |
| `WANDB_RESUME`          | 기본값은 *never*입니다. *auto*로 설정하면 wandb가 실패한 실행을 자동으로 재개합니다. *must*로 설정하면 시작 시 해당 실행이 이미 존재해야 합니다. 항상 고유한 실행 ID를 직접 생성하려면 *allow*로 설정하고 `WANDB_RUN_ID`를 항상 지정하세요.                                                 |
| `WANDB_RUN_GROUP`       | 실험 이름을 지정하면 실행이 자동으로 그룹화됩니다. 자세한 내용은 [grouping](/ko/models/runs/grouping/)을 참고하세요.                                                                                                                               |
| `WANDB_RUN_ID`          | 스크립트의 단일 실행에 해당하는, 프로젝트 내에서 전역적으로 고유한 문자열을 설정하세요. 길이는 64자를 넘을 수 없습니다. 단어 문자가 아닌 모든 문자는 대시로 변환됩니다. 장애 발생 시 기존 실행을 재개하는 데 사용할 수 있습니다.                                                                           |
| `WANDB_QUIET`           | 이 값을 `true`로 설정하면 표준 출력으로 로깅되는 메시지를 치명적인 수준의 메시지로만 제한합니다. 이렇게 설정하면 모든 로그는 `$WANDB_DIR/debug.log`에 기록됩니다.                                                                                                      |
| `WANDB_SILENT`          | 이 값을 `true`로 설정하면 wandb 로그 메시지가 출력되지 않습니다. 스크립트 기반 명령을 실행할 때 유용합니다. 이 값을 설정하면 모든 로그는 `$WANDB_DIR/debug.log`에 기록됩니다.                                                                                           |
| `WANDB_SHOW_RUN`        | 운영 체제가 이를 지원하는 경우 브라우저에서 실행 URL이 자동으로 열리도록 이 값을 `true`로 설정합니다.                                                                                                                                                |
| `WANDB_SWEEP_ID`        | `Run` 객체 및 관련 클래스에 스윕 ID 추적을 추가하고 UI에 표시합니다.                                                                                                                                                                  |
| `WANDB_TAGS`            | 실행에 적용할 태그의 쉼표로 구분된 목록입니다.                                                                                                                                                                                    |
| `WANDB_USERNAME`        | 실행과 연결할 팀 구성원의 사용자 이름입니다. 서비스 계정 API 키와 함께 사용해 자동 실행을 해당 팀 구성원에게 귀속시키는 데 사용할 수 있습니다.                                                                                                                          |
| `WANDB_USER_EMAIL`      | 실행과 연결할 팀 구성원의 이메일 주소입니다. 서비스 계정 API 키와 함께 사용해 자동 실행을 해당 팀 구성원에게 귀속시키는 데 사용할 수 있습니다.                                                                                                                          |

<div id="singularity-environments">
  ## Singularity 환경
</div>

[Singularity](https://singularity.lbl.gov/index.html)에서 컨테이너를 실행하는 경우, 위에서 설명한 환경 변수 앞에 `SINGULARITYENV_`를 붙여 전달할 수 있습니다. Singularity 환경 변수에 대한 자세한 내용은 [여기](https://singularity.lbl.gov/docs-environment-metadata#environment)에서 확인할 수 있습니다.

<div id="running-on-aws">
  ## AWS에서 실행하기
</div>

AWS에서 배치 작업을 실행한다면, W&amp;B 자격 증명을 사용해 인스턴스를 쉽게 인증할 수 있습니다. [User Settings](https://wandb.ai/settings)에서 API 키를 생성한 다음, `WANDB_API_KEY` 환경 변수를 [AWS Batch 작업 정의](https://docs.aws.amazon.com/batch/latest/userguide/job_definition_parameters.html#parameters)에 설정하세요.
