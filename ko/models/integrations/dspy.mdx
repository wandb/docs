W&amp;B를 DSPy와 함께 사용하여 언어 모델 프로그램을 추적하고 최적화할 수 있습니다. W&amp;B는 다음과 같은 기능을 제공하여 [Weave DSPy 통합](/ko/weave/guides/integrations/dspy)을 보완합니다:

* 시간 경과에 따른 평가 지표(Evaluation metrics) 추적
* 프로그램 시그니처 변화를 위한 W&amp;B Tables
* MIPROv2와 같은 DSPy 최적화기(optimizer)와의 연동

DSPy 모듈을 최적화할 때 포괄적인 관측/가시성을 확보하려면 W&amp;B와 Weave 모두에서 통합을 활성화하십시오.

<Info>
  **Note**

  `wandb==0.21.2` 및 `weave==0.52.5`부터는 Weave를 W&amp;B와 함께 사용할 때 자동으로 초기화됩니다:

  * `weave`를 import한 다음 `wandb.init()`을 호출하는 경우(스크립트 환경)
  * `wandb.init()`을 먼저 호출한 뒤 나중에 `weave`를 import하는 경우(노트북/Jupyter 환경)

  명시적인 `weave.init(...)` 호출은 필요하지 않습니다.
</Info>

<div id="track-program-optimization-experimental">
  ## 프로그램 최적화 추적 (실험적)
</div>

`dspy.Evaluate`를 사용하는 DSPy 옵티마이저(예: MIPROv2)의 경우, `WandbDSPyCallback`을 사용하여 시간 경과에 따른 평가 지표를 로깅하고 W&amp;B Tables에서 프로그램 시그니처의 변화를 추적합니다.

```python
import dspy
from dspy.datasets import MATH

import weave
import wandb
from wandb.integration.dspy import WandbDSPyCallback

# W&B 초기화 (weave 임포트만으로 충분하며, 명시적인 weave.init 호출 불필요)
project_name = "dspy-optimization"
with wandb.init(project=project_name) as run:
    # DSPy에 W&B 콜백 추가
    dspy.settings.callbacks.append(
        WandbDSPyCallback(run=run)
    )

    # 언어 모델 구성
    teacher_lm = dspy.LM('openai/gpt-4o', max_tokens=2000, cache=True)
    student_lm = dspy.LM('openai/gpt-4o-mini', max_tokens=2000)
    dspy.configure(lm=student_lm)

    # 데이터셋 로드 및 프로그램 정의
    dataset = MATH(subset='algebra')
    program = dspy.ChainOfThought("question -> answer")

    # 옵티마이저 구성 및 실행
    optimizer = dspy.MIPROv2(
        metric=dataset.metric,
        auto="light",
        num_threads=24,
        teacher_settings=dict(lm=teacher_lm),
        prompt_model=student_lm
    )

    optimized_program = optimizer.compile(
        program,
        trainset=dataset.train,
        max_bootstrapped_demos=2,
        max_labeled_demos=2
    )
```

이 코드를 실행하면 W&amp;B 실행 URL과 Weave URL을 모두 얻게 됩니다. W&amp;B는 시간에 따른 평가 지표와 함께 프로그램 시그니처의 변화를 보여주는 Tables를 표시합니다. 실행의 **Overview** 탭에는 자세한 분석을 위한 Weave trace 링크가 포함됩니다.

`run` 객체가 `WandbDSPyCallback`에 전달되지 않으면 전역 `run` 객체가 사용됩니다.

<div id="log-predictions-to-wb-tables">
  ## 예측을 W&amp;B Tables에 기록하기
</div>

최적화 과정에서 개별 예제를 검사할 수 있도록 세부적인 예측 로깅을 활성화하세요. 이 콜백은 각 평가 단계마다 W&amp;B Tables를 생성하며, 이는 개별 성공 및 실패 사례를 분석하는 데 도움이 될 수 있습니다.

```python
from wandb.integration.dspy import WandbDSPyCallback

# 예측 로깅 활성화 (기본적으로 활성화됨)
callback = WandbDSPyCallback(log_results=True)
dspy.settings.callbacks.append(callback)

# 최적화 실행
optimized_program = optimizer.compile(program, trainset=train_data)

# 필요한 경우 예측 로깅 비활성화
# callback = WandbDSPyCallback(log_results=False)
```

<div id="access-prediction-data">
  ### 예측 데이터 확인하기
</div>

최적화가 끝나면 W&amp;B에서 예측 데이터를 다음과 같이 확인할 수 있습니다:

1. 실행의 **Overview** 페이지로 이동합니다.
2. `predictions_0`, `predictions_1` 등과 같은 패턴으로 이름이 지정된 테이블 패널(Table panel)을 찾습니다.
3. 실패 사례를 분석하려면 `is_correct`로 필터링합니다.
4. 프로젝트 워크스페이스에서 실행 간 테이블을 비교합니다.

각 테이블에는 다음과 같은 열이 포함됩니다:

* `example`: 입력 데이터
* `prediction`: 모델 출력
* `is_correct`: 평가 결과

자세한 내용은 [W&amp;B Tables 가이드](/ko/models/tables/visualize-tables/)를 참고하세요.

<div id="save-and-version-dspy-programs">
  ## DSPy 프로그램 저장 및 버전 관리
</div>

가장 우수한 DSPy 프로그램을 재현하고 버전 관리를 하려면, W&amp;B Artifact로 저장하세요. 전체 프로그램을 저장할지, 상태만 저장할지 선택할 수 있습니다.

```python
from wandb.integration.dspy import WandbDSPyCallback

# 콜백 인스턴스 생성
callback = WandbDSPyCallback()
dspy.settings.callbacks.append(callback)

# 최적화 실행
optimized_program = optimizer.compile(program, trainset=train_data)

# 저장 옵션:

# 1. 전체 프로그램 (권장) - 아키텍처 및 상태 포함
callback.log_best_model(optimized_program, save_program=True)

# 2. JSON으로 상태만 저장 - 경량, 사람이 읽을 수 있는 형식
callback.log_best_model(optimized_program, save_program=False, filetype="json")

# 3. pickle로 상태만 저장 - Python 객체 보존
callback.log_best_model(optimized_program, save_program=False, filetype="pkl")

# 버전 관리를 위한 커스텀 별칭 추가
callback.log_best_model(
    optimized_program,
    save_program=True,
    aliases=["best", "production", "v2.0"]
)
```
