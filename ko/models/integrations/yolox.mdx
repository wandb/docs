[YOLOX](https://github.com/Megvii-BaseDetection/YOLOX)는 객체 탐지를 위한 anchor-free YOLO 버전입니다. YOLOX W&amp;B 인테그레이션을 사용하면 트레이닝, 검증, 시스템 관련 메트릭 로깅을 활성화하고, 명령줄 인수 하나만으로 예측을 대화형으로 검증할 수 있습니다.

이 가이드에서는 W&amp;B로 인증하고, 인테그레이션을 설치하고, YOLOX 객체 탐지 모델을 트레이닝할 때 W&amp;B 로깅을 활성화하여 W&amp;B UI에서 메트릭을 추적하고 예측을 확인하는 방법을 설명합니다.

  ## 가입하고 API 키 만들기
</div>

API 키는 로컬 머신을 W&amp;B에 인증하는 데 사용됩니다. 프로필에서 API 키를 생성할 수 있습니다.

1. 오른쪽 상단의 프로필 아이콘을 클릭합니다.
2. **User Settings**를 선택한 다음 **API Keys** 섹션까지 스크롤합니다.

<div id="log-metrics">
  ## 메트릭 로깅
</div>

`wandb` 라이브러리를 설치하고 머신 인증을 완료하면 YOLOX 트레이닝 스크립트에서 W&amp;B 로깅을 활성화할 수 있습니다.

`wandb`로 로깅을 사용하려면 `--logger wandb` 명령줄 인수를 사용하세요. 필요에 따라 [`wandb.init()`](/ko/models/ref/python/functions/init)에 전달할 수 있는 모든 인수도 함께 전달할 수 있습니다. 각 인수 앞에는 `wandb-`를 붙이세요.

`num_eval_imges`는 모델 평가를 위해 W&amp;B가 검증 세트 이미지와 예측을 Tables에 로깅하는 수를 제어합니다.

명령을 실행하기 전에 다음 플레이스홀더를 바꾸세요:

* `<project-name>`: W&amp;B 프로젝트의 이름입니다.
* `<entity>`: W&amp;B entity(사용자 이름 또는 팀 이름)입니다.
* `<run-name>`: 이 트레이닝 run의 이름입니다.
* `<run-id>`: 이 run의 고유 식별자입니다.
* `<save-dir>`: YOLOX가 checkpoint와 로그를 저장하는 디렉터리입니다.
* `<num-images>`: 로깅할 검증 이미지 수입니다.
* `<bool>`: checkpoint를 로깅할지 여부입니다(`true` 또는 `false`).

```shell
# W&B에 로그인
wandb login

# wandb 로거 인수를 사용하여 YOLOX 트레이닝 스크립트 호출
python tools/train.py .... --logger wandb \
                wandb-project <project-name> \
                wandb-entity <entity> \
                wandb-name <run-name> \
                wandb-id <run-id> \
                wandb-save_dir <save-dir> \
                wandb-num_eval_imges <num-images> \
                wandb-log_checkpoints <bool>
```
