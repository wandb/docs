[YOLOX](https://github.com/Megvii-BaseDetection/YOLOX)는 객체 감지에서 뛰어난 성능을 제공하는 앵커 프리 YOLO 버전입니다. YOLOX W&amp;B 인테그레이션을 사용하면 트레이닝, 검증, 시스템 관련 메트릭 로깅을 활성화할 수 있으며, 단일 command-line 인수로 예측을 대화형으로 검증할 수 있습니다.

<div id="sign-up-and-create-an-api-key">
  ## 가입 및 API 키 생성하기
</div>

API 키는 사용 중인 머신을 W&amp;B에 인증하는 데 사용됩니다. 사용자 프로필에서 API 키를 생성할 수 있습니다.

<ApiKeyCreateStreamlined />

1. 오른쪽 상단의 사용자 프로필 아이콘을 클릭하세요.
2. **User Settings**를 선택한 다음 **API Keys** 섹션으로 스크롤하세요.

<div id="install-the-wandb-library-and-log-in">
  ## `wandb` 라이브러리를 설치하고 로그인하세요
</div>

로컬에 `wandb` 라이브러리를 설치하고 로그인하려면 다음을 수행하세요.

<Tabs>
  <Tab title="명령줄">
    1. `WANDB_API_KEY` [환경 변수](/ko/models/track/environment-variables/)를 API 키로 설정하세요.

       ```bash
       export WANDB_API_KEY=<your_api_key>
       ```

    2. `wandb` 라이브러리를 설치하고 로그인하세요.

       ```shell
       pip install wandb

       wandb login
       ```
  </Tab>

  <Tab title="Python">

    ```python

    ```
  </Tab>

  <Tab title="Python notebook">

</Tabs>

<div id="log-metrics">
  ## 메트릭 로깅
</div>

`--logger wandb` 명령줄 인수를 사용하여 wandb로 로깅을 켜세요. 선택적으로 [`wandb.init()`](/ko/models/ref/python/functions/init)에서 예상하는 모든 인수도 전달할 수 있습니다. 각 인수 앞에 `wandb-`를 붙이세요.

`num_eval_imges`는 모델 Evaluation을 위해 W&amp;B Tables에 로깅되는 검증 세트 이미지와 예측의 수를 제어합니다.

```shell
# wandb에 로그인
wandb login

# `wandb` 로거 인수를 사용해 yolox 트레이닝 스크립트를 실행
python tools/train.py .... --logger wandb \
                wandb-project <project-name> \
                wandb-entity <entity>
                wandb-name <run-name> \
                wandb-id <run-id> \
                wandb-save_dir <save-dir> \
                wandb-num_eval_imges <num-images> \
                wandb-log_checkpoints <bool>
```
