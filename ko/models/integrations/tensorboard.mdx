<div id="get-started">
  ## 시작하기
</div>

```python
import wandb

# `sync_tensorboard=True`로 wandb 실행 시작
wandb.init(project="my-project", sync_tensorboard=True) as run:
  # TensorBoard를 사용하는 학습 코드
  ...

```

[TensorBoard 통합 실행 예시](https://wandb.ai/rymc/simple-tensorboard-example/runs/oab614zf/tensorboard)를 확인하세요.

실행이 완료되면 W&amp;B에서 TensorBoard 이벤트 파일에 접근하고, 시스템의 CPU 및 GPU 사용량, `git` 상태, 실행에 사용된 터미널 명령 등과 같은 추가 유용한 정보와 함께 메트릭을 W&amp;B 네이티브 차트로 시각화할 수 있습니다.

<Note>
  W&amp;B는 모든 버전의 TensorFlow에서 TensorBoard를 지원합니다. 또한 PyTorch 및 TensorBoardX와 함께 TensorBoard 1.14 이상도 지원합니다.
</Note>

  ## 자주 묻는 질문
</div>

<div id="how-can-i-log-metrics-to-wb-that-arent-logged-to-tensorboard">
  ### TensorBoard에는 기록되지 않는 지표를 W&B에 어떻게 기록할 수 있나요?
</div>

TensorBoard에 기록되지 않는 사용자 정의 지표를 추가로 기록해야 하는 경우, 코드에서 `wandb.Run.log()`을 호출하면 됩니다: `run.log({"custom": 0.8})`

TensorBoard를 동기화할 때는 `run.log()`의 step 인수를 설정할 수 없습니다. 다른 step 값을 사용하고 싶다면, 다음과 같이 step 메트릭과 함께 지표를 기록할 수 있습니다:

`run.log({"custom": 0.8, "global_step": global_step})`

<div id="how-do-i-configure-tensorboard-when-im-using-it-with-wandb">
  ### `wandb`와 함께 TensorBoard를 사용할 때는 어떻게 구성하나요?
</div>

TensorBoard가 패치되는 방식을 더 세밀하게 제어하려면 `wandb.init`에 `sync_tensorboard=True`를 전달하는 대신 `wandb.tensorboard.patch`를 호출하면 됩니다.

```python
import wandb

wandb.tensorboard.patch(root_logdir="<logging_directory>")
run = wandb.init()

# wandb 실행을 완료하여 tensorboard 로그를 W&B에 업로드합니다 (Notebook에서 실행 중인 경우)
run.finish()
```

이 메서드에 `tensorboard_x=False`를 전달하면 기본 TensorBoard가 패치되도록 할 수 있습니다. TensorBoard &gt; 1.14를 PyTorch와 함께 사용하는 경우에는 `pytorch=True`를 전달하여 PyTorch용 TensorBoard가 패치되도록 할 수 있습니다. 이 두 옵션 모두 어떤 버전의 라이브러리가 임포트되었는지에 따라 자동으로 적절한 기본값이 설정됩니다.

기본적으로 `tfevents` 파일과 모든 `.pbtxt` 파일도 동기화합니다. 이렇게 하면 사용자를 대신해 TensorBoard 인스턴스를 실행할 수 있습니다. 실행 페이지에서 [TensorBoard 탭](https://www.wandb.com/articles/hosted-tensorboard)을 확인할 수 있습니다. 이 동작은 `wandb.tensorboard.patch`에 `save=False`를 전달하여 비활성화할 수 있습니다.

```python
import wandb

run = wandb.init()
wandb.tensorboard.patch(save=False, tensorboard_x=True)

# 노트북에서 실행 중인 경우, wandb 실행을 종료하여 TensorBoard 로그를 W&B에 업로드합니다
run.finish()
```

<Warning>
  `tf.summary.create_file_writer`를 호출하거나 `torch.utils.tensorboard`를 통해 `SummaryWriter`를 생성하기 **전에**, 반드시 `wandb.init()` 또는 `wandb.tensorboard.patch`를 먼저 호출해야 합니다.
</Warning>


<div id="how-do-i-sync-historical-tensorboard-runs">
  ### 기존 TensorBoard 실행 기록을 어떻게 동기화하나요?
</div>

로컬에 저장된 기존 `tfevents` 파일을 W&B로 가져오려면 `wandb sync log_dir` 명령을 실행하세요. 여기서 `log_dir`는 해당 `tfevents` 파일이 있는 로컬 디렉터리입니다.

<div id="how-do-i-use-google-colab-or-jupyter-with-tensorboard">
  ### Google Colab 또는 Jupyter에서 TensorBoard를 사용하려면 어떻게 해야 하나요?
</div>

Jupyter 또는 Colab 노트북에서 코드를 실행하는 경우, 학습이 끝날 때 반드시 `wandb.Run.finish()`를 호출해야 합니다. 이렇게 하면 해당 wandb 실행이 종료되고 TensorBoard 로그가 W&B로 업로드되어 시각화할 수 있습니다. `.py` 스크립트를 실행하는 경우에는 스크립트가 끝날 때 wandb가 자동으로 종료되므로 이 작업이 필요하지 않습니다.

노트북 환경에서 셸 명령을 실행하려면 `!wandb sync directoryname`처럼 명령어 앞에 `!`를 붙여야 합니다.

<div id="how-do-i-use-pytorch-with-tensorboard">
  ### PyTorch를 TensorBoard와 함께 사용하려면 어떻게 하나요?
</div>

PyTorch의 TensorBoard 통합을 사용하는 경우, PyTorch Profiler JSON 파일을 수동으로 업로드해야 할 수도 있습니다.

```python
with wandb.init(project="my-project", sync_tensorboard=True) as run:
    run.save(glob.glob(f"runs/*.pt.trace.json")[0], base_path=f"runs")
```


<div id="can-i-sync-tfevents-files-stored-in-the-cloud">
  ### 클라우드에 저장된 tfevents 파일을 동기화할 수 있나요?
</div>

`wandb` 0.20.0 이상 버전에서는 S3, GCS 또는 Azure에 저장된 `tfevents` 파일 동기화를 지원합니다. `wandb`는 각 클라우드 제공자의 기본 자격 증명을 사용하며, 이는 아래 표의 명령어와 대응됩니다:

| Cloud provider | Credentials                             | Logging directory format              |
| -------------- | --------------------------------------- | ------------------------------------- |
| S3             | `aws configure`                         | `s3://bucket/path/to/logs`            |
| GCS            | `gcloud auth application-default login` | `gs://bucket/path/to/logs`            |
| Azure          | `az login`[^1]                          | `az://account/container/path/to/logs` |

[^1]: `AZURE_STORAGE_ACCOUNT`와 `AZURE_STORAGE_KEY` 환경 변수도 설정해야 합니다.
