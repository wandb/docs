W&amp;B는 [Amazon SageMaker](https://aws.amazon.com/sagemaker/)와 통합되어 하이퍼파라미터를 자동으로 읽고, 분산 실행을 그룹화하고, 체크포인트에서 실행을 재개합니다.

<div id="authentication">
  ## 인증
</div>

W&B는 학습 스크립트 기준 상대 경로에서 `secrets.env`라는 파일을 찾고, `wandb.init()`이 호출될 때 이 파일의 내용을 환경 변수로 로드합니다. 실험을 실행할 때 사용하는 스크립트에서 `wandb.sagemaker_auth(path="source_dir")`를 호출해 `secrets.env` 파일을 생성할 수 있습니다. 이 파일을 반드시 `.gitignore`에 추가하세요!

<div id="existing-estimators">
  ## 기존 Estimator
</div>

SageMaker에서 미리 구성된 Estimator 중 하나를 사용하는 경우, wandb가 포함된 `requirements.txt` 파일을 소스 디렉터리에 추가해야 합니다.

```text
wandb
```

Python 2로 실행되는 estimator를 사용 중이라면, wandb를 설치하기 전에 이 [wheel](https://pythonwheels.com)을 사용해 `psutil`을 직접 설치해야 합니다.

```text
https://wheels.galaxyproject.org/packages/psutil-5.4.8-cp27-cp27mu-manylinux1_x86_64.whl
wandb
```

[GitHub](https://github.com/wandb/examples/tree/master/examples/pytorch/pytorch-cifar10-sagemaker)의 전체 예제를 살펴보고, W&amp;B [블로그](https://wandb.ai/site/articles/running-sweeps-with-sagemaker)에서 더 자세한 내용을 확인하세요.

또한 SageMaker와 W&amp;B를 사용해 감성 분석기를 배포하는 방법을 설명한 [Deploy Sentiment Analyzer Using SageMaker and W&amp;B tutorial](https://wandb.ai/authors/sagemaker/reports/Deploy-Sentiment-Analyzer-Using-SageMaker-and-W-B--VmlldzoxODA1ODE)을 읽어볼 수 있습니다.

<Warning>
  SageMaker 통합이 비활성화되어 있는 경우에만 SageMaker 작업에서 W&amp;B sweep agent가 예상대로 동작합니다. `wandb.init` 호출을 수정하여 SageMaker 통합을 비활성화하세요:

  ```python
  wandb.init(..., settings=wandb.Settings(sagemaker_disable=True))
  ```
</Warning>
