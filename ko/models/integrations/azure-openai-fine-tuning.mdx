이 가이드에서는 W&amp;B를 Azure OpenAI와 함께 사용하여 GPT-3.5 또는 GPT-4 모델의 파인튜닝 작업을 추적하고 평가하는 방법을 설명합니다. W&amp;B를 통합하면 실험 추적을 통해 메트릭, 하이퍼파라미터, 트레이닝 아티팩트를 기록할 수 있으므로 모델 성능을 분석하고 개선할 수 있습니다. 또한 W&amp;B의 평가 도구를 사용해 모델 선택에 대해 데이터 기반 의사결정을 내릴 수 있습니다.

이 가이드는 Azure OpenAI 모델을 파인튜닝하면서 Runs를 체계적으로 추적하고 비교하려는 머신 러닝 실무자를 위한 것입니다.

<div id="prerequisites">
  ## 사전 요구 사항
</div>

시작하기 전에 다음을 완료하세요:

* [공식 Azure 문서](https://wandb.me/aoai-wb-int)에 따라 Azure OpenAI 서비스를 설정합니다.
* API 키를 사용하도록 W&amp;B 계정을 설정합니다.

<div id="workflow-overview">
  ## 워크플로 개요
</div>

다음 단계에서는 일반적인 Azure OpenAI 파인튜닝 작업이 작업 준비부터 결과 모델 평가까지 W&amp;B에서 어떻게 진행되는지 요약해 설명합니다.

<div id="fine-tuning-setup">
  ### 파인튜닝 설정
</div>

파인튜닝 설정은 다음 step으로 구성됩니다:

* Azure OpenAI 요구 사항에 따라 트레이닝 데이터를 준비합니다.
* Azure OpenAI에서 파인튜닝 작업을 설정합니다.
* W&amp;B를 사용해 파인튜닝 과정을 추적하고 메트릭과 하이퍼파라미터를 로깅합니다.

<div id="experiment-tracking">
  ### 실험 추적
</div>

파인튜닝 중 W&amp;B는 다음을 기록합니다:

* 트레이닝 및 검증 메트릭.
* 모델 하이퍼파라미터.
* 리소스 사용량.
* 트레이닝 아티팩트.

<div id="model-evaluation">
  ### 모델 평가
</div>

파인튜닝 후 [W&amp;B Weave](https://weave-docs.wandb.ai)를 사용해 다음을 수행합니다:

* 모델 출력을 레퍼런스 데이터셋과 비교하여 평가합니다.
* 서로 다른 파인튜닝 run 간의 성능 비교합니다.
* 특정 테스트 사례에서 모델의 동작 분석합니다.
* 데이터에 기반해 모델 선택 결정합니다.
