[Ultralytics&#39; YOLOv5](https://ultralytics.com/yolo) (&quot;You Only Look Once&quot;) 모델 계열은 합성곱 신경망을 사용해 실시간 객체 탐지를 구현하면서도 그 과정의 번거로움을 크게 줄여 줍니다.

[W&amp;B](https://wandb.com)는 YOLOv5에 직접 통합되어 있어, 실험 지표 추적, 모델 및 데이터셋 버전 관리, 풍부한 모델 예측 시각화 등을 제공합니다. **YOLO 실험을 실행하기 전에 `pip install` 명령어 한 줄만 실행하면 될 만큼 간단합니다.**

<Note>
  모든 W&amp;B 로깅 기능은 [PyTorch DDP](https://pytorch.org/tutorials/intermediate/ddp_tutorial.html)와 같은 데이터 병렬 멀티-GPU 학습과 호환됩니다.
</Note>

<div id="track-core-experiments">
  ## 핵심 실험 추적
</div>

`wandb`를 설치하기만 해도 시스템 메트릭, 모델 메트릭, 대화형 [Dashboards](/ko/models/track/workspaces/)에 기록되는 미디어 등을 위한 W&amp;B의 [내장 로깅 기능](/ko/models/track/log/)이 자동으로 활성화됩니다.

```python
pip install wandb
git clone https://github.com/ultralytics/yolov5.git
python yolov5/train.py  # 소규모 데이터셋으로 소규모 네트워크 학습
```

wandb가 표준 출력에 표시한 링크만 따라가면 됩니다.

<div id="customize-the-integration">
  ## 통합 사용자 지정
</div>

YOLO에 몇 가지 간단한 명령줄 인수를 전달하면 더 많은 W&amp;B 기능을 활용할 수 있습니다.

* `--save_period`에 숫자를 전달하면 W&amp;B는 매 `save_period` 에포크마다 [모델 버전](/ko/models/registry/)을 저장합니다. 이 모델 버전에는 모델 가중치가 포함되며, 검증 세트에서 가장 성능이 좋은 모델에 태그가 붙습니다.
* `--upload_dataset` 플래그를 사용하면 데이터 버전 관리를 위해 데이터셋도 함께 업로드됩니다.
* `--bbox_interval`에 숫자를 전달하면 [데이터 시각화](../)가 활성화됩니다. 매 `bbox_interval` 에포크가 끝날 때마다 검증 세트에 대한 모델의 출력이 W&amp;B로 업로드됩니다.

<Tabs>
  <Tab title="모델 버전 관리만 사용">
    ```python
    python yolov5/train.py --epochs 20 --save_period 1
    ```
  </Tab>

  <Tab title="모델 버전 관리와 데이터 시각화">
    ```python
    python yolov5/train.py --epochs 20 --save_period 1 \
      --upload_dataset --bbox_interval 1
    ```
  </Tab>
</Tabs>

<Note>
  모든 W&amp;B 계정에는 데이터셋과 모델을 위한 100 GB의 무료 스토리지가 제공됩니다.
</Note>
