[Composer](https://github.com/mosaicml/composer)는 신경망을 더 잘, 더 빠르게, 더 저렴하게 트레이닝할 수 있도록 해주는 라이브러리입니다. 트레이닝을 가속하고 일반화 성능을 향상하는 방법이 포함되어 있습니다. 또한 Composer에는 다양한 개선 기법을 조합하기 위한 선택 사항인 [Trainer](https://docs.mosaicml.com/projects/composer/en/stable/trainer/using_the_trainer.html) API도 있습니다.

Composer와 함께 W&amp;B를 사용하여 트레이닝 Runs를 추적, 시각화 및 비교하세요. Composer에는 [WandBLogger](https://docs.mosaicml.com/projects/composer/en/stable/trainer/file_uploading.html#weights-biases-artifacts)를 통한 기본 제공 W&amp;B 로깅이 포함되어 있습니다.

<div id="start-logging-to-wb">
  ## W&amp;B 로깅 시작하기
</div>

Composer 트레이닝 Runs를 W&amp;B에 로깅하기 시작하려면 `composer.loggers.WandBLogger` 인스턴스를 `composer.Trainer`에 전달하세요:

```python
from composer import Trainer
from composer.loggers import WandBLogger

trainer = Trainer(..., logger=WandBLogger())
```

<div id="use-composers-wandblogger">
  ## Composer의 `WandBLogger` 사용
</div>

다음 섹션에서는 `WandBLogger`가 Composer의 `Trainer`와 어떻게 통합되는지 설명합니다.

Composer 라이브러리는 `Trainer`에서 [WandBLogger](https://docs.mosaicml.com/projects/composer/en/stable/trainer/file_uploading.html#weights-biases-artifacts) 클래스를 사용해 메트릭을 W&amp;B에 로깅합니다. 로거를 생성해 `Trainer`에 전달하기만 하면 됩니다.

```python
wandb_logger = WandBLogger(project="gpt-5", log_artifacts=True)
trainer = Trainer(logger=wandb_logger)
```

<div id="logger-arguments">
  ## Logger 매개변수
</div>

`WandBLogger`가 Runs를 기록하는 방식을 사용자 지정할 때 사용할 수 있는 매개변수는 [Composer documentation](https://docs.mosaicml.com/projects/composer/en/stable/api_reference/generated/composer.loggers.WandBLogger.html)을 참조하세요.

다음 예시는 `init_kwargs`를 통해 run notes와 설정 딕셔너리를 전달하는 일반적인 사용 방식을 보여줍니다:

```python
init_kwargs = {"notes":"Testing higher learning rate in this experiment", 
               "config":{"arch":"Llama",
                         "use_mixed_precision":True
                         }
               }

wandb_logger = WandBLogger(log_artifacts=True, init_kwargs=init_kwargs)
```

<div id="log-prediction-samples">
  ## 예측 샘플 로깅
</div>

스칼라 메트릭 외에도 모델 예측과 같은 리치 미디어를 W&amp;B에 로깅하여 정성 평가를 위해 검토할 수 있습니다. [Composer의 callback](https://docs.mosaicml.com/projects/composer/en/stable/trainer/callbacks.html) 시스템을 사용하면 `WandBLogger`로 W&amp;B에 언제 로깅할지 제어할 수 있습니다. 다음 예시에서는 검증 이미지와 예측 샘플을 로깅합니다:

```python
import wandb
from composer import Callback, State, Logger

class LogPredictions(Callback):
    def __init__(self, num_samples=100, seed=1234):
        super().__init__()
        self.num_samples = num_samples
        self.data = []
        
    def eval_batch_end(self, state: State, logger: Logger):
        """배치별 예측을 계산하고 self.data에 저장합니다"""
        
        if state.timer.epoch == state.max_duration: #마지막 검증 에포크에서
            if len(self.data) < self.num_samples:
                n = self.num_samples
                x, y = state.batch_pair
                outputs = state.outputs.argmax(-1)
                data = [[wandb.Image(x_i), y_i, y_pred] for x_i, y_i, y_pred in list(zip(x[:n], y[:n], outputs[:n]))]
                self.data += data
            
    def eval_end(self, state: State, logger: Logger):
        with wandb.init() as run:
            "Create a wandb.Table and logs it"
            columns = ['image', 'ground truth', 'prediction']
            table = wandb.Table(columns=columns, data=self.data[:self.num_samples])
            run.log({'sample_table':table}, step=int(state.timer.batch))         
...

trainer = Trainer(
    ...
    loggers=[WandBLogger()],
    callbacks=[LogPredictions()]
)
```
