title: Artifact 스토리지 및 메모리 할당 관리
---

기본적으로 W&amp;B는 Artifact 파일을 미국에 위치한 비공개 Google Cloud Storage 버킷에 저장합니다. 모든 파일은 저장 시와 전송 중에 암호화됩니다.

민감한 파일의 경우 [Private Hosting](/ko/platform/hosting/)을 설정하거나 [reference 아티팩트](/ko/models/artifacts/track-external-files/)를 사용할 것을 권장합니다.

학습 중에는 W&amp;B가 로그, 아티팩트, 설정 파일을 다음 로컬 디렉터리에 저장합니다:

| 파일 유형 | 기본 위치 | 기본 위치를 변경하려면 다음을 설정하세요: |
| ---- | ---------------- | ------------------------------- |
| logs | `./wandb` | `wandb.init`의 `dir` 또는 `WANDB_DIR` 환경 변수 |
| artifacts | `~/.cache/wandb` | `WANDB_CACHE_DIR` 환경 변수 |
| configs | `~/.config/wandb` | `WANDB_CONFIG_DIR` 환경 변수 |
| 업로드를 위한 staging artifacts  | `~/.cache/wandb-data/` | `WANDB_DATA_DIR` 환경 변수 |
| 다운로드된 artifacts | `./artifacts` | `WANDB_ARTIFACT_DIR` 환경 변수 |

환경 변수를 사용해 W&amp;B를 구성하는 전체 가이드는 [환경 변수 레퍼런스](/ko/models/track/environment-variables/)를 참고하세요.

<Warning>
  `wandb`가 초기화되는 머신에 따라 이러한 기본 폴더가 파일 시스템의 쓰기 가능한 위치가 아닐 수 있습니다. 이 경우 오류가 발생할 수 있습니다.
</Warning>

<div id="clean-up-local-artifact-cache">
  ### 로컬 아티팩트 캐시 정리
</div>

W&amp;B는 여러 버전에서 공통으로 사용하는 파일의 다운로드를 가속하기 위해 아티팩트 파일을 캐시합니다. 시간이 지나면 이 캐시 디렉터리의 크기가 커질 수 있습니다. 캐시를 정리하고 최근에 사용되지 않은 파일을 제거하려면 [`wandb artifact cache cleanup`](/ko/models/ref/cli/wandb-artifact/wandb-artifact-cache/) 명령을 실행하세요.

다음 코드 스니펫은 캐시 크기를 1GB로 제한하는 예시입니다. 코드 스니펫을 복사하여 터미널에 붙여넣으세요.

```bash
$ wandb artifact cache cleanup 1GB
```
