title: Artifact 계보 그래프 탐색
---

W&amp;B는 &#95;계보 그래프&#95;라고 불리는 방향성 비순환 그래프(DAG)를 사용하여 실행의 입력과 출력을 추적합니다. 계보 그래프는 ML 실험에서 Artifact와 실행 사이의 관계를 시각적으로 표현한 것입니다. 이 그래프는 원시 데이터 수집부터 모델 학습과 평가에 이르기까지, 데이터와 모델이 ML 수명 주기의 서로 다른 단계를 어떻게 거쳐 흐르는지를 보여줍니다.

아티팩트 계보를 추적하면 다음과 같은 주요 이점이 있습니다:

* **재현성**: 디버깅, 실험, 검증을 위해 실험, 모델, 결과를 재현할 수 있도록 지원합니다.
* **버전 관리**: 아티팩트에 대한 시간에 따른 변경 사항을 추적하여, 필요할 때 이전 데이터나 모델 버전으로 되돌릴 수 있게 합니다.
* **감사 추적**: 규정 준수와 거버넌스를 지원하기 위해 아티팩트와 변환 내역을 상세하게 기록합니다.
* **협업**: 실험 이력을 투명하게 만들어 중복 작업을 줄이고 개발 속도를 높여 팀워크를 향상하는 데 도움이 됩니다.

<div id="view-an-artifacts-lineage-graph">
  ## 아티팩트의 계보 그래프 보기
</div>

아티팩트의 계보 그래프를 보려면 다음 단계를 따르세요:

1. W&amp;B 앱으로 이동합니다.
2. 탐색하려는 실행 또는 아티팩트를 포함한 프로젝트를 선택합니다.
3. 왼쪽 사이드바에서 **Artifacts** 탭을 클릭합니다.
4. **Lineage** 탭을 선택합니다.

<div id="enable-lineage-graph-tracking">
  ## 계보 그래프 추적 활성화하기
</div>

계보 그래프 추적을 활성화하려면 W&amp;B Python SDK를 사용해서 아티팩트를 실행의 [입력](/ko/models/artifacts/explore-and-traverse-an-artifact-graph) 또는
[출력](/ko/models/artifacts/explore-and-traverse-an-artifact-graph#track-the-output-of-a-run)으로 표시해야 합니다.

<div id="track-the-input-of-a-run">
  ### 실행의 입력 추적하기
</div>

[`wandb.Run.use_artifact()`](/ko/models/ref/python/experiments/run#method-runuse_artifact) 메서드를 사용하여 실행의 입력(또는 의존성)으로 Artifact를 표시합니다.
Artifact 이름과, 해당 Artifact의 특정 버전을 참조하기 위한 선택적 별칭(alias)을 지정합니다.
Artifact 이름 형식은 `<artifact_name>:<version>` 또는 `<artifact_name>:<alias>` 입니다.

꺾쇠 괄호(`< >`)로 둘러싸인 값을 실제 값으로 바꾸십시오:

```python
import wandb

# 실행 초기화
with wandb.init(entity="<entity>", project="<project>") as run:
  # 아티팩트 가져오기, 의존성으로 표시
  artifact = run.use_artifact(artifact_or_name="<name>", aliases="<alias>")
```

<div id="track-the-output-of-a-run">
  ### 실행 출력 추적하기
</div>

[`wandb.Run.log_artifact()`](/ko/models/ref/python/experiments/run#log_artifact)를 사용하여 실행의 출력으로 아티팩트를 지정합니다. 먼저
[`wandb.Artifact()`](/ko/models/ref/python/experiments/artifact#wandb.Artifact) 생성자를 사용하여 Artifact를 생성합니다. 그런 다음 `wandb.Run.log_artifact()`로
해당 아티팩트를 실행의 출력으로 로그로 남깁니다.

꺾쇠 괄호(`< >`)로 둘러싸인 값을 실제 환경에 맞는 값으로 바꾸십시오:

```python
import wandb

# 실행 초기화
with wandb.init(entity="<entity>", project="<project>") as run:
  
  # 아티팩트 생성
  artifact = wandb.Artifact(name = "<artifact_name>", type = "<artifact_type>")
  artifact.add_file(local_path = "<local_filepath>", name="<optional-name>")

  # 아티팩트를 실행의 출력으로 기록
  run.log_artifact(artifact_or_path = artifact)
```

<div id="artifact-clusters">
  ## Artifact 클러스터
</div>

그래프의 한 레벨에 실행이나 아티팩트가 다섯 개 이상 있으면 클러스터가 생성됩니다. 클러스터에는 특정 버전의 실행이나 아티팩트를 찾기 위한 검색 바가 있으며, 클러스터 안에서 개별 노드를 꺼내 그 노드의 lineage를 계속 추적할 수 있습니다.

노드를 클릭하면 노드 개요가 포함된 미리 보기가 열립니다. 화살표 아이콘을 클릭하면 개별 실행이나 아티팩트가 클러스터에서 분리되어, 분리된 노드의 lineage를 자세히 살펴볼 수 있습니다.

<div id="navigate-an-artifact-graph-programmatically">
  ## 프로그래밍 방식으로 아티팩트 그래프 탐색하기
</div>

W&amp;B Python SDK를 사용해 그래프를 프로그래밍 방식으로 탐색할 수 있습니다. 아티팩트 객체의 [`logged_by()`](/ko/models/ref/python/experiments/artifact#method-artifact-logged-by) 및 [`used_by()`](/ko/models/ref/python/experiments/artifact#method-artifact-used-by) 메서드를 사용해 그래프를 순회합니다:

```python
with wandb.init() as run:
    artifact = run.use_artifact("artifact_name:latest")

    # 아티팩트에서 그래프를 위아래로 탐색합니다:
    producer_run = artifact.logged_by()
    consumer_runs = artifact.used_by()
```
