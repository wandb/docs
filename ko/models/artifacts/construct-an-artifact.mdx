W&amp;B Python SDK를 사용해 [W&amp;B Runs](/ko/models/ref/python/experiments/run)에서 artifact를 생성할 수 있습니다. [파일, 디렉터리, URI, 병렬 run의 파일을 artifact에 추가](#add-files-to-an-artifact)할 수 있습니다. artifact에 파일을 추가한 후에는 해당 artifact를 W&amp;B Server 또는 [직접 관리하는 프라이빗 서버](/ko/platform/hosting/hosting-options/self-managed)에 저장하세요. 각 artifact는 하나의 run과 연결됩니다.

Amazon S3에 저장된 파일과 같은 외부 파일을 추적하는 방법에 대한 자세한 내용은 [외부 파일 추적](./track-external-files) 페이지를 참조하세요.

<div id="construct-an-artifact">
  ## 아티팩트 생성
</div>

[W&amp;B Artifact](/ko/models/ref/python/experiments/artifact)를 다음 세 단계로 생성합니다:

1. [`wandb.Artifact()`로 아티팩트 Python 객체 생성](/ko/models/artifacts/construct-an-artifact#create-an-artifact-python-object-with-wandb-artifact)
2. [아티팩트에 하나 이상의 파일 추가](/ko/models/artifacts/construct-an-artifact#add-one-or-more-files-to-the-artifact)
3. [아티팩트를 W&amp;B 서버에 저장](/ko/models/artifacts/construct-an-artifact#save-your-artifact-to-the-w\&b-server)

<div id="create-an-artifact-python-object-with-wandbartifact">
  ### `wandb.Artifact()`로 아티팩트 Python 객체 생성하기
</div>

아티팩트 객체를 만들려면 [`wandb.Artifact()`](/ko/models/ref/python/experiments/artifact) 클래스를 초기화하세요. 다음 매개변수를 지정합니다.

* **Name**: 아티팩트의 이름입니다. 이름은 고유하고, 설명적이며, 기억하기 쉬워야 합니다.
* **Type**: 아티팩트의 유형입니다. 유형은 단순하고 설명적이어야 하며, 머신 러닝 파이프라인의 단일 step에 해당해야 합니다. 일반적인 아티팩트 유형으로는 `'dataset'` 또는 `'model'`이 있습니다.

<Note>
  W&amp;B는 제공한 &quot;name&quot;과 &quot;type&quot;을 사용해 W&amp;B App에서 방향성 비순환 그래프를 생성합니다. 자세한 내용은 [아티팩트 그래프 탐색 및 순회](./explore-and-traverse-an-artifact-graph)를 참조하세요.
</Note>

<Warning>
  아티팩트는 유형과 관계없이 같은 이름을 가질 수 없습니다. 즉, `dataset` 유형의 `cats`라는 아티팩트와 같은 이름의 `model` 유형 아티팩트를 따로 만들 수는 없습니다.
</Warning>

아티팩트 객체를 초기화할 때 설명과 메타데이터를 선택적으로 제공할 수도 있습니다. 사용 가능한 속성 및 매개변수에 대한 자세한 내용은 Python SDK 레퍼런스 가이드의 [`wandb.Artifact`](/ko/models/ref/python/experiments/artifact) 클래스 정의를 참조하세요.

다음 코드 스니펫을 복사해 아티팩트 객체를 만드세요. `<name>` 및 `<type>` 플레이스홀더를 원하는 값으로 바꾸세요:

```python
import wandb

# 아티팩트 객체 생성
artifact = wandb.Artifact(name="<name>", type="<type>")
```

<div id="add-one-or-more-files-to-the-artifact">
  ### 아티팩트에 하나 이상의 파일 추가
</div>

[파일, 디렉터리, 외부 URI 레퍼런스(Amazon S3 등) 등을 추가](/ko/models/artifacts/construct-an-artifact#add-files-to-an-artifact)하여 아티팩트 객체에 넣습니다.

단일 파일을 추가하려면 아티팩트 객체의 [`Artifact.add_file()`](/ko/models/ref/python/experiments/artifact#add_file) 방법을 사용합니다:

```python
artifact.add_file(local_path="path/to/file.txt", name="<name>")
```

디렉터리를 추가하려면 [`Artifact.add_dir()`](/ko/models/ref/python/experiments/artifact#add_dir) 메서드를 사용하세요:

```python
artifact.add_dir(local_path="path/to/directory", name="<name>")
```

아티팩트에 다양한 유형의 파일을 추가하는 방법에 대한 자세한 내용은 다음 섹션 [아티팩트에 파일 추가](/ko/models/artifacts/construct-an-artifact#add-files-to-an-artifact)을 참조하세요.

<div id="save-your-artifact-to-the-wb-server">
  ### 아티팩트를 W&amp;B 서버에 저장하기
</div>

아티팩트를 W&amp;B 서버에 저장합니다. 아티팩트를 저장하려면 run 객체의 [`wandb.Run.log_artifact()`](/ko/models/ref/python/experiments/run#log_artifact) 방법을 사용하세요.

```python
with wandb.init(project="<project>", job_type="<job-type>") as run:
    run.log_artifact(artifact)
```

<Tip>
  **`wandb.Run.log_artifact()`와 `Artifact.save()`의 사용 시점**

  * `wandb.Run.log_artifact()`는 새 아티팩트를 생성하고 이를 특정 run에 연결할 때 사용합니다.
  * `Artifact.save()`는 새 run을 생성하지 않고 기존 아티팩트를 업데이트할 때 사용합니다.
</Tip>

정리하면, 다음 코드 스니펫은 데이터셋 아티팩트를 생성하고, 아티팩트에 파일을 추가한 다음, 아티팩트를 W&amp;B에 저장하는 방법을 보여줍니다:

```python
import wandb

artifact = wandb.Artifact(name="<name>", type="<type>")
artifact.add_file(local_path="path/to/file.txt", name="<name>")
artifact.add_dir(local_path="path/to/directory", name="<name>")

with wandb.init(project="<project>", job_type="<job-type>") as run:
    run.log_artifact(artifact)
```

같은 이름과 유형으로 아티팩트를 로깅할 때마다 W&amp;B는 해당 아티팩트의 새 버전을 생성합니다. 자세한 내용은 [새 아티팩트 버전 만들기](/ko/models/artifacts/create-a-new-artifact-version)를 참조하세요.

<Warning>
  W&amp;B는 업로드 성능을 높이기 위해 `wandb.Run.log_artifact()` 호출을 비동기적으로 수행합니다. 이로 인해 루프에서 아티팩트를 로깅할 때 예상치 못한 동작이 발생할 수 있습니다. 예를 들면 다음과 같습니다.

  ```python
  with wandb.init() as run:
      for i in range(10):
          a = wandb.Artifact(name = "race",
              type="dataset",
              metadata={
                  "index": i,
              },
          )
          # ... artifact a에 파일 추가 ...
          run.log_artifact(a)
  ```

  아티팩트가 임의의 순서로 로깅될 수 있으므로, 아티팩트 버전 **v0**의 metadata에 있는 index가 0이 아닐 수도 있습니다.
</Warning>

<div id="add-files-to-an-artifact">
  ## 아티팩트에 파일 추가
</div>

다음 섹션에서는 아티팩트에 여러 유형의 객체를 추가하는 방법을 설명합니다. 예제를 읽어 나가기 전에, 다음과 같은 구조의 디렉터리가 있다고 가정합니다:

```text
root-directory
| - hello.txt
| - images/
| -- | cat.png
| -- | dog.png
| - checkpoints/
| -- | model.h5
| - models/
| -- | model.h5
```

<div id="add-a-single-file">
  ### 단일 파일 추가
</div>

아티팩트에 로컬 단일 파일 하나를 추가하려면 [`wandb.Artifact.add_file()`](/ko/models/ref/python/experiments/artifact#method-artifact-add-file)을 사용합니다. 파일의 로컬 경로를 `local_path` 매개변수로 지정합니다:

```python
import wandb

# 아티팩트 객체 초기화
artifact = wandb.Artifact(name="<name>", type="<type>")

# 단일 파일 추가
artifact.add_file(local_path="path/file.format")
```

예를 들어, 로컬 작업 디렉터리에 `'hello.txt'`라는 파일이 있다고 가정해 보겠습니다.

```python
artifact.add_file("hello.txt")
```

이제 아티팩트에는 다음 내용이 포함됩니다:

```text
hello.txt
```

필요하면 `name` Parameter에 다른 이름을 전달해 아티팩트 객체 자체에서 파일 이름을 바꿀 수 있습니다. 이전 예제를 이어서 보면 다음과 같습니다:

```python
artifact.add_file(
    local_path="hello.txt", 
    name="new/path/hello_world.txt"
    )
```

아티팩트는 다음과 같이 저장됩니다:

```text
new/path/hello_world.txt
```

다음 표는 서로 다른 API 호출에 따라 생성되는 아티팩트 콘텐츠가 어떻게 달라지는지 보여줍니다:

| API 호출                                                    | 생성되는 아티팩트           |
| --------------------------------------------------------- | ------------------- |
| `artifact.new_file('hello.txt')`                          | `hello.txt`         |
| `artifact.add_file('model.h5')`                           | `model.h5`          |
| `artifact.add_file('checkpoints/model.h5')`               | `model.h5`          |
| `artifact.add_file('model.h5', name='models/mymodel.h5')` | `models/mymodel.h5` |

<div id="add-multiple-files">
  ### 여러 파일 추가
</div>

로컬 디렉터리의 여러 파일을 아티팩트에 추가하려면 [`wandb.Artifact.add_dir()`](/ko/models/ref/python/experiments/artifact#method-artifact-add-dir) 방법을 사용하세요. `local_path` 매개변수로 디렉터리의 로컬 경로를 지정하세요.

```python
import wandb

# 아티팩트 객체 초기화
artifact = wandb.Artifact(name="<name>", type="<type>")

# 아티팩트에 로컬 디렉터리 추가
artifact.add_dir(local_path="path/file.format", name="optional-prefix")
```

다음 표는 서로 다른 API 호출에 따라 아티팩트 콘텐츠가 어떻게 달라지는지 보여줍니다.

| API Call                                    | Resulting 아티팩트                                                       |
| ------------------------------------------- | -------------------------------------------------------------------- |
| `artifact.add_dir('images')`                | <p><code>cat.png</code></p><p><code>dog.png</code></p>               |
| `artifact.add_dir('images', name='images')` | <p><code>images/cat.png</code></p><p><code>images/dog.png</code></p> |

<div id="add-a-uri-reference">
  ### URI 레퍼런스 추가
</div>

URI에 W&amp;B 라이브러리가 지원하는 스킴이 있으면, 아티팩트는 재현성을 위해 체크섬과 기타 정보를 추적합니다.

[`wandb.Artifact.add_reference()`](/ko/models/ref/python/experiments/artifact#method-artifact-add-reference) 방법으로 아티팩트에 외부 URI 레퍼런스를 추가합니다. `'uri'` 문자열을 사용자의 URI로 바꾸세요. 필요에 따라 name Parameter에 아티팩트 내에서 사용할 원하는 경로를 전달할 수도 있습니다.

```python
# URI 레퍼런스 추가
artifact.add_reference(uri="uri", name="optional-name")
```

아티팩트는 다음 URI 스킴을 지원합니다:

* `http(s)://`: HTTP를 통해 액세스할 수 있는 파일 경로입니다. HTTP 서버가 `ETag` 및 `Content-Length` 응답 헤더를 지원하는 경우, 아티팩트는 ETag 형식의 체크섬과 크기 메타데이터를 추적합니다.
* `s3://`: S3의 객체 또는 객체 접두사 경로입니다. 아티팩트는 참조된 객체의 체크섬과 버전 관리 정보(버킷에서 객체 버전 관리가 활성화된 경우)를 추적합니다. 객체 접두사는 해당 접두사 아래의 객체를 포함하도록 확장되며, 최대 10,000개 객체까지 포함됩니다.
* `gs://`: GCS의 객체 또는 객체 접두사 경로입니다. 아티팩트는 참조된 객체의 체크섬과 버전 관리 정보(버킷에서 객체 버전 관리가 활성화된 경우)를 추적합니다. 객체 접두사는 해당 접두사 아래의 객체를 포함하도록 확장되며, 최대 10,000개 객체까지 포함됩니다.

다음 표는 서로 다른 API 호출에 따라 생성되는 아티팩트 콘텐츠가 어떻게 달라지는지 보여줍니다:

| API 호출                                                                        | 생성되는 아티팩트 콘텐츠                                                        |
| ----------------------------------------------------------------------------- | -------------------------------------------------------------------- |
| `artifact.add_reference('s3://my-bucket/model.h5')`                           | `model.h5`                                                           |
| `artifact.add_reference('s3://my-bucket/checkpoints/model.h5')`               | `model.h5`                                                           |
| `artifact.add_reference('s3://my-bucket/model.h5', name='models/mymodel.h5')` | `models/mymodel.h5`                                                  |
| `artifact.add_reference('s3://my-bucket/images')`                             | <p><code>cat.png</code></p><p><code>dog.png</code></p>               |
| `artifact.add_reference('s3://my-bucket/images', name='images')`              | <p><code>images/cat.png</code></p><p><code>images/dog.png</code></p> |

<div id="add-files-to-artifacts-from-parallel-runs">
  ### 병렬 run에서 아티팩트에 파일 추가하기
</div>

대규모 데이터셋을 다루거나 분산 트레이닝을 수행하는 경우, 여러 병렬 run이 하나의 아티팩트에 기여해야 할 수 있습니다.

```python
import wandb
import time

# 이 예제는 데모 목적으로 Ray를 사용하여 병렬로 실행합니다.
import ray

ray.init()

artifact_type = "dataset"
artifact_name = "parallel-artifact"
table_name = "distributed_table"
parts_path = "parts"
num_parallel = 5

# 병렬 작성자의 각 배치는 고유한
# 그룹 이름을 가져야 합니다.
group_name = "writer-group-{}".format(round(time.time()))


@ray.remote
def train(i):
    """
    Our writer job. Each writer will add one image to the artifact.
    """
    with wandb.init(group=group_name) as run:
        artifact = wandb.Artifact(name=artifact_name, type=artifact_type)

        # wandb 테이블에 데이터를 추가합니다.
        table = wandb.Table(columns=["a", "b", "c"], data=[[i, i * 2, 2**i]])

        # 아티팩트의 폴더에 테이블을 추가합니다.
        artifact.add(table, "{}/table_{}".format(parts_path, i))

        # 아티팩트를 upsert하면 아티팩트에 데이터가 생성되거나 추가됩니다.
        run.upsert_artifact(artifact)


# runs를 병렬로 실행합니다.
result_ids = [train.remote(i) for i in range(num_parallel)]

# 아티팩트를 완료하기 전에 모든 작성자의 파일이
# 추가되었는지 확인하기 위해 모든 작성자를 join합니다.
ray.get(result_ids)

# 모든 작성자가 완료되면 아티팩트를 finish하여
# 준비 완료 상태로 표시합니다.
with wandb.init(group=group_name) as run:
    artifact = wandb.Artifact(artifact_name, type=artifact_type)

    # 테이블 폴더를 가리키는 "PartitionTable"을 생성하고
    # 아티팩트에 추가합니다.
    artifact.add(wandb.data_types.PartitionedTable(parts_path), table_name)

    # finish_artifact는 아티팩트를 확정하여 이 버전에 대한
    # 이후 "upsert"를 허용하지 않습니다.
    run.finish_artifact(artifact)
```

<div id="find-path-for-logged-artifacts-and-other-metadata">
  ## 로깅된 아티팩트 및 기타 메타데이터의 경로 찾기
</div>

다음 코드 스니펫은 [W&amp;B Public API](/ko/models/ref/python/public-api/)를 사용해 run에 있는 파일의 목록(이름과 URL 포함)을 확인하는 방법을 보여줍니다. `<entity/project/run-id>` 자리표시자를 자신의 값으로 바꾸세요:

```python
from wandb.apis.public.files import Files
from wandb.apis.public.api import Api

# 예시 run 객체
run = Api().run("<entity/project/run-id>")

# run의 파일을 순회하기 위한 Files 객체 생성
files = Files(api.client, run)

# 파일 순회
for file in files:
    print(f"File Name: {file.name}")
    print(f"File URL: {file.url}")
    print(f"Path to file in the bucket: {file.direct_url}")
```

사용 가능한 속성과 방법에 대한 자세한 내용은 [File](/ko/models/ref/python/public-api/file) 클래스를 참조하세요.
