*참조 아티팩트*를 사용하여 W&amp;B 서버 외부에 저장된 파일을 추적하고 활용하세요. 일반적인 외부 저장소 솔루션으로는 CoreWeave AI Object Storage, Amazon Simple Storage Service(Amazon S3) 버킷, GCS 버킷, Azure blob, HTTP 파일 서버, NFS 공유 등이 있습니다.

참조 아티팩트는 비참조 아티팩트와 유사하게 동작합니다. 핵심적인 차이점은 참조 아티팩트가 파일 크기 및 MD5 체크섬과 같은 파일에 대한 메타데이터만으로 구성된다는 것입니다. 파일 자체는 절대 시스템 외부로 전송되지 않습니다.

참조 아티팩트는 비참조 아티팩트와 유사하게 사용할 수 있습니다. W&amp;B App에서는 파일 브라우저를 통해 참조 아티팩트의 내용을 탐색하고, 전체 의존성 그래프를 탐색하며, 아티팩트의 버전 이력을 살펴볼 수 있습니다. 단, 데이터 자체가 아티팩트 내에 포함되어 있지 않으므로 UI에서는 이미지, 오디오 등의 리치 미디어를 렌더링할 수 없습니다.

<Note>
  외부 파일을 추적하지 않는 아티팩트를 로깅하면 W&amp;B는 아티팩트의 파일을 W&amp;B 서버에 저장합니다. 이것이 W&amp;B Python SDK로 아티팩트를 로깅할 때의 기본 동작입니다.

  외부 파일을 추적하는 아티팩트를 로깅하면 W&amp;B는 객체의 ETag 및 크기와 같은 객체에 대한 메타데이터를 로깅합니다. 버킷에서 객체 버전 관리가 활성화된 경우 버전 ID도 함께 로깅됩니다.
</Note>

다음 섹션에서는 외부 참조 아티팩트를 추적하는 방법을 설명합니다.

<div id="track-an-artifact-in-an-external-bucket">
  ## 외부 버킷에서 아티팩트 추적하기
</div>

W&amp;B Python SDK를 사용하여 W&amp;B 외부에 저장된 파일에 대한 레퍼런스를 추적하세요.

1. `wandb.init()`으로 run을 초기화합니다.
2. `wandb.Artifact()`로 아티팩트 객체를 생성합니다.
3. 아티팩트 객체의 `wandb.Artifact.add_reference()` 메서드를 사용하여 버킷 경로에 대한 레퍼런스를 지정합니다.
4. `run.log_artifact()`로 아티팩트의 메타데이터를 로깅합니다.

```python
import wandb

# W&B run 초기화
with wandb.init(project="my-project") as run:

  # 아티팩트 객체 생성
  artifact = wandb.Artifact(name="name", type="type")

  # 버킷 경로에 레퍼런스 추가
  artifact.add_reference(uri = "uri/to/your/bucket/path")

  # 아티팩트 메타데이터 로깅
  run.log_artifact(artifact)
```

예를 들어, 버킷의 디렉터리 구조가 다음과 같다고 가정합니다.

```text
s3://my-bucket

|datasets/
  |-- mnist/
|models/
  |-- cnn/
```

`datasets/mnist/` 디렉토리에는 이미지 컬렉션이 포함되어 있습니다. `datasets/mnist/` 디렉토리를 데이터셋 아티팩트로 추적하려면 다음을 지정하십시오.

1. `"mnist"`와 같이 아티팩트의 이름을 지정하십시오.
2. 아티팩트 객체를 생성할 때 (`wandb.Artifact(type="dataset")`) `type` 파라미터를 `"dataset"`으로 설정하십시오.
3. `wandb.Artifact.add_reference()`를 호출할 때 `datasets/mnist/` 디렉토리의 경로를 Amazon S3 URI(`s3://my-bucket/datasets/mnist/`)로 지정하십시오.
4. `run.log_artifact()`로 아티팩트를 로깅하십시오.

다음 코드 샘플은 참조 아티팩트 `mnist:latest`를 생성합니다.

```python
import wandb

with wandb.init(project="my-project") as run:
  artifact = wandb.Artifact(name="mnist", type="dataset")
  artifact.add_reference(uri="s3://my-bucket/datasets/mnist")
  run.log_artifact(artifact)
```

W&amp;B App에서 파일 브라우저를 통해 참조 아티팩트의 내용을 살펴보고, [전체 의존성 그래프를 탐색하거나](/ko/models/artifacts/explore-and-traverse-an-artifact-graph/), 아티팩트의 버전 이력을 확인할 수 있습니다. 아티팩트 내에 데이터 자체가 포함되어 있지 않으므로 W&amp;B App은 이미지, 오디오 등의 리치 미디어를 렌더링하지 않습니다.

<Note>
  W&amp;B Artifacts는 CoreWeave Storage 및 MinIO를 포함한 모든 Amazon S3 호환 인터페이스를 지원합니다. 아래에 설명된 스크립트는 `AWS_S3_ENDPOINT_URL` 환경 변수를 CoreWeave Storage 또는 MinIO 서버를 가리키도록 설정하면 두 공급자 모두에서 그대로 작동합니다.
</Note>

<Warning>
  기본적으로 W&amp;B는 객체 접두사를 추가할 때 객체 수를 10,000개로 제한합니다. `wandb.Artifact.add_reference()`를 호출할 때 `max_objects=`를 지정하여 이 제한을 조정할 수 있습니다.
</Warning>

<div id="download-an-artifact-from-an-external-bucket">
  ## 외부 버킷에서 아티팩트 다운로드
</div>

W&amp;B는 참조 아티팩트를 다운로드할 때 아티팩트가 로깅될 당시 기록된 메타데이터를 사용하여 기본 버킷에서 파일을 조회합니다. 버킷에 객체 버전 관리가 활성화되어 있는 경우, W&amp;B는 아티팩트가 로깅된 시점의 파일 상태에 해당하는 객체 버전을 조회합니다. 버킷의 내용이 변경되더라도, 아티팩트는 트레이닝 run 중 버킷의 스냅샷 역할을 하므로 특정 모델이 트레이닝된 데이터의 정확한 버전을 항상 참조할 수 있습니다.

다음 코드 샘플은 참조 아티팩트를 다운로드하는 방법을 보여줍니다. 아티팩트 다운로드 API는 참조 아티팩트와 비참조 아티팩트 모두 동일합니다:

```python
import wandb

with wandb.init(project="my-project") as run:
  artifact = run.use_artifact("mnist:latest", type="dataset")
  artifact_dir = artifact.download()
```

<Note>
  W&amp;B는 워크플로의 일부로 파일을 덮어쓰는 경우, 저장소 버킷에서 &#39;Object Versioning&#39;을 활성화할 것을 권장합니다.

  버전 관리를 활성화하면, 아티팩트가 로깅된 이후 파일이 덮어쓰여진 경우에도 아티팩트를 다운로드할 때 W&amp;B가 항상 올바른 버전의 파일을 조회할 수 있습니다.

  사용 사례를 기준으로 객체 버전 관리 활성화 방법을 참조하십시오: [AWS](https://docs.aws.amazon.com/AmazonS3/latest/userguide/manage-versioning-examples.html), [Google Cloud](https://cloud.google.com/storage/docs/using-object-versioning#set), [Azure](https://learn.microsoft.com/azure/storage/blobs/versioning-enable).
</Note>

<div id="add-and-download-an-external-from-a-bucket">
  ## 버킷에서 외부 항목 추가 및 다운로드
</div>

다음 코드 샘플은 데이터셋을 Amazon S3 버킷에 업로드하고, 참조 아티팩트로 추적한 다음 다운로드합니다.

```python
import boto3
import wandb

with wandb.init() as run:
  # 트레이닝 중...

  s3_client = boto3.client("s3")
  s3_client.upload_file(file_name="my_model.h5", bucket="my-bucket", object_name="models/cnn/my_model.h5")

  # 모델 아티팩트 로깅
  model_artifact = wandb.Artifact("cnn", type="model")
  model_artifact.add_reference("s3://my-bucket/models/cnn/")
  run.log_artifact(model_artifact)
```

나중에 모델 아티팩트를 다운로드할 수 있습니다. 아티팩트의 이름과 유형을 지정하세요:

```python
import wandb

with wandb.init() as run:
  artifact = run.use_artifact(artifact_or_name = "cnn", type="model")
  datadir = artifact.download()
```

<Note>
  Google Cloud 또는 Azure에서 참조로 아티팩트를 추적하는 방법에 대한 엔드투엔드 워크스루는 다음 리포트를 참조하세요.

  * [Google Cloud를 사용한 참조 아티팩트 추적 가이드](https://wandb.ai/stacey/artifacts/reports/Tracking-Artifacts-by-Reference--Vmlldzo1NDMwOTE)
  * [Microsoft Azure에서 참조 아티팩트 사용하기](https://wandb.ai/andrea0/azure-2023/reports/Efficiently-Harnessing-Microsoft-Azure-Blob-Storage-with-Weights-Biases--Vmlldzo0NDA2NDgw)
</Note>

<div id="cloud-storage-credentials">
  ## 클라우드 저장소 자격 증명
</div>

W&amp;B는 사용하는 클라우드 제공업체에 따라 기본 메커니즘을 사용하여 자격 증명을 조회합니다. 사용되는 자격 증명에 대한 자세한 내용은 클라우드 제공업체의 문서를 참조하십시오.

| 클라우드 제공업체                   | 자격 증명 문서                                                                                                                                         |
| --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------ |
| CoreWeave AI Object Storage | [CoreWeave AI Object Storage 문서](https://docs.coreweave.com/docs/products/storage/object-storage/how-to/manage-access-keys/cloud-console-tokens) |
| AWS                         | [Boto3 문서](https://boto3.amazonaws.com/v1/documentation/api/latest/guide/credentials.html#configuring-credentials)                               |
| Google Cloud                | [Google Cloud 문서](https://cloud.google.com/docs/authentication/provide-credentials-adc)                                                          |
| Azure                       | [Azure 문서](https://learn.microsoft.com/python/api/azure-identity/azure.identity.defaultazurecredential?view=azure-python)                        |

AWS의 경우, 버킷이 설정된 사용자의 기본 리전에 위치하지 않는다면 `AWS_REGION` 환경 변수를 버킷 리전과 일치하도록 설정해야 합니다.

<Warning>
  이미지, 오디오, 비디오, 포인트 클라우드 등의 리치 미디어는 버킷의 CORS 설정에 따라 App UI에서 렌더링되지 않을 수 있습니다. 버킷의 CORS 설정에서 **app.wandb.ai**를 허용 목록에 추가하면 W&amp;B App이 이러한 리치 미디어를 올바르게 렌더링할 수 있습니다.

  이미지, 오디오, 비디오, 포인트 클라우드 등의 리치 미디어가 App UI에서 렌더링되지 않는 경우, 버킷의 CORS 정책에서 `app.wandb.ai`가 허용 목록에 추가되어 있는지 확인하십시오.
</Warning>

<div id="track-an-artifact-in-a-filesystem">
  ## 파일 시스템에서 아티팩트 추적하기
</div>

데이터셋에 액세스하는 일반적인 패턴 중 하나는 트레이닝 작업을 실행하는 모든 머신에서 원격 파일 시스템의 NFS 마운트 포인트를 노출하는 것입니다. 트레이닝 스크립트 관점에서 파일이 로컬 파일 시스템에 있는 것처럼 보이기 때문에, 이는 클라우드 저장소 버킷의 대안이 될 수 있습니다.

{/* W&B Artifacts를 사용하여 마운트 여부에 관계없이 파일 시스템에 대한 레퍼런스를 추적하세요. */}

파일 시스템에서 아티팩트를 추적하려면:

1. `wandb.init()`으로 run을 초기화합니다.
2. `wandb.Artifact()`로 아티팩트 객체를 생성합니다.
3. 아티팩트 객체의 `wandb.Artifact.add_reference()` 메서드로 파일 시스템 경로에 대한 레퍼런스를 지정합니다.
4. `run.log_artifact()`로 아티팩트의 메타데이터를 로깅합니다.

마운트된 파일 시스템의 파일을 추적하려면 다음 코드 스니펫을 복사하여 붙여넣으세요. 꺾쇠 괄호(`< >`)로 묶인 값을 본인의 값으로 교체하십시오.

```python
import wandb

# run 초기화
with wandb.init(entity="<entity>", project="<project>") as run:

  # 아티팩트 객체 생성
  artifact = wandb.Artifact(name="<name>", type="<type>")

  # 파일시스템 경로에 대한 레퍼런스 추가
  artifact.add_reference("file:///path/to/dataset/")

  # 아티팩트 로깅 (메타데이터만)
  run.log_artifact(artifact)
```

URL의 슬래시 세 개에 주목하세요. 첫 번째 컴포넌트는 파일 시스템 레퍼런스 사용을 나타내는 `file://` 접두사입니다. 두 번째 컴포넌트는 파일 시스템의 루트 `/`입니다. 나머지 컴포넌트는 추적하려는 디렉터리 또는 파일의 경로입니다.

예를 들어, 다음과 같은 구조로 `/mount`에 마운트된 파일 시스템이 있다고 가정합니다.

```text
mount
|datasets/
  |-- mnist/
|models/
  |-- cnn/
```

`datasets/mnist/` 디렉터리를 데이터셋 아티팩트로 추적하려고 합니다. 추적하려면 다음 코드 스니펫을 사용할 수 있습니다.

```python
import wandb

with wandb.init() as run:
  artifact = wandb.Artifact("mnist", type="dataset")
  artifact.add_reference("file:///mount/datasets/mnist/")
  run.log_artifact(artifact)
```

이렇게 하면 `/mount/datasets/mnist/` 아래에 저장된 파일을 가리키는 참조 아티팩트 `mnist:latest`가 생성됩니다.

<Warning>
  기본적으로 W&amp;B는 디렉토리에 레퍼런스를 추가할 때 파일 수를 10,000개로 제한합니다. `wandb.Artifact.add_reference()`를 호출할 때 `max_objects=`를 지정하여 이 제한을 조정할 수 있습니다.
</Warning>

마찬가지로, `models/cnn/my_model.h5`에 저장된 모델을 추적하려면 다음 코드 스니펫을 사용할 수 있습니다:

```python
import wandb

with wandb.init() as run:

  # 여기서 트레이닝...

  # 모델을 디스크에 저장

  # 아티팩트 객체 생성
  model_artifact = wandb.Artifact("cnn", type="model")

  # 모델 파일 경로에 대한 레퍼런스 추가
  model_artifact.add_reference("file:///mount/cnn/my_model.h5")

  # W&B에 아티팩트 로깅
  run.log_artifact(model_artifact)
```

<div id="download-an-artifact-from-an-external-filesystem">
  ## 외부 파일 시스템에서 아티팩트 다운로드
</div>

참조 파일 시스템에서 파일을 다운로드할 때는 참조 아티팩트가 아닌 경우와 동일한 API를 사용하십시오:

1. `wandb.init()`으로 run을 초기화합니다.
2. `wandb.Run.use_artifact()` 메서드를 사용하여 다운로드할 아티팩트를 지정합니다.
3. 아티팩트의 `wandb.Artifact.download()` 메서드를 호출하여 참조된 파일 시스템에서 파일을 다운로드합니다.

```python
with wandb.init() as run:
  artifact = run.use_artifact("entity/project/mnist:latest", type="dataset")
  artifact_dir = artifact.download()
```

W&amp;B는 `/mount/datasets/mnist`의 내용을 `artifacts/mnist:v0/` 디렉토리에 복사합니다.

<Info>
  `Artifact.download()`는 아티팩트를 복원할 수 없는 경우 오류를 발생시킵니다. 예를 들어, 아티팩트에 덮어쓴 파일에 대한 레퍼런스가 포함된 경우, 아티팩트를 더 이상 복원할 수 없으므로 `Artifact.download()`는 오류를 발생시킵니다.
</Info>
