---
title: 왜 W&B Inference에서 요청 제한 오류(429)가 발생하나요?
---

요청 제한 오류(429)는 동시 처리 제한을 초과했을 때 발생합니다.

**Error:** "Concurrency limit reached for requests"

**Solution:**

- 병렬 요청 수를 줄입니다
- 요청 사이에 지연 시간을 추가합니다
- 지수 백오프를 구현합니다
- 참고: 요청 제한은 W&B 프로젝트별로 적용됩니다

1. **지수 백오프를 적용한 재시도 로직 구현:**
   ```python
   import time
   
   def retry_with_backoff(func, max_retries=3):
       for i in range(max_retries):
           try:
               return func()
           except Exception as e:
               if "429" in str(e) and i < max_retries - 1:
                   time.sleep(2 ** i)
               else:
                   raise
   ```
