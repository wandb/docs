---
title: W&B Sweep의 일부로 모든 하이퍼파라미터에 값을 제공해야 하나요?
  기본값을 설정할 수 있나요?
---

`(run.config())`를 사용하면 Sweep 설정에서 하이퍼파라미터 이름과 값을 가져올 수 있으며, 이는 사전처럼 동작합니다.

Sweep 외부에서 실행되는 run의 경우, `wandb.init()`의 `config` 인자로 사전을 전달하여 `wandb.Run.config()` 값을 설정합니다. Sweep 내부에서는, `wandb.init()`에 전달한 설정이 기본값으로 사용되며, Sweep이 이 값을 덮어쓸 수 있습니다.

명시적인 동작을 위해 `rwandb.Run.config.setdefaults()`를 사용하십시오. 다음 코드 스니펫은 두 가지 방법을 모두 보여줍니다:

<Tabs>
<Tab title="wandb.init()">
```python
# 하이퍼파라미터의 기본값 설정
config_defaults = {"lr": 0.1, "batch_size": 256}

# run을 시작하고 Sweep이 덮어쓸 수 있는
# 기본값을 제공
with wandb.init(config=config_defaults) as run:
    # 여기에서 트레이닝 코드를 추가
    ...
```
</Tab>
<Tab title="config.setdefaults()">
```python
# 하이퍼파라미터의 기본값 설정
config_defaults = {"lr": 0.1, "batch_size": 256}

# run 시작
with wandb.init() as run:
    # Sweep에서 설정하지 않은 값들을 업데이트
    run.config.setdefaults(config_defaults)

    # 여기에서 트레이닝 코드를 추가
```
</Tab>
</Tabs>
