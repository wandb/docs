---
title: 코드나 데이터셋 예시 없이 메트릭만 로그로 기록할 수 있나요?
---

기본적으로 W&B는 Datasets 예시를 로그하지 않습니다. 대신 코드와 시스템 메트릭을 로그합니다.

환경 변수를 사용하여 코드 로깅을 끄는 두 가지 메소드가 있습니다:

1. 모든 코드 트래킹을 중지하려면 `WANDB_DISABLE_CODE`를 `true`로 설정하세요. 이 설정은 git SHA 및 diff 패치 정보의 수집을 방지합니다.
2. `WANDB_IGNORE_GLOBS`를 `*.patch`로 설정하면 diff 패치를 서버로 동기화하는 것을 중단하면서도, 로컬에서 `wandb restore`를 통해 애플리케이션에 적용할 수 있도록 유지할 수 있습니다.

관리자는 Teams 설정에서 팀 전체의 코드 저장을 비활성화할 수도 있습니다:

1. `https://wandb.ai/<team>/settings`에서 팀의 설정 페이지로 이동하세요. 여기서 `<team>`은 팀의 이름입니다.
2. Privacy 섹션으로 스크롤합니다.
3. **Enable code saving by default** 설정을 토글하여 해제합니다.
