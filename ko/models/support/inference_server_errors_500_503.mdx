---
title: W&B Inference에서 발생하는 서버 오류(500, 503)를 어떻게 해결하나요?
---

서버 오류는 W&B Inference 서비스에 일시적인 장애가 발생했음을 의미합니다.

<div id="error-types">
  ## 오류 유형
</div>

<div id="500-internal-server-error">
  ### 500 - 내부 서버 오류
</div>

**메시지:** "서버에서 요청을 처리하는 동안 오류가 발생했습니다"

이는 일시적인 서버 내부 오류입니다.

<div id="503-service-overloaded">
  ### 503 - 서비스 과부하
</div>

**Message:** "엔진이 현재 과부하 상태입니다. 잠시 후 다시 시도해 주세요"

서비스에 요청이 과도하게 몰리고 있습니다.

<div id="how-to-handle-server-errors">
  ## 서버 오류를 처리하는 방법
</div>

1. **재시도 전에 대기하기**
   - 500 오류: 30–60초 대기
   - 503 오류: 60–120초 대기

2. **지수 백오프 사용하기**
   ```python
   import time
   import openai
   
   def call_with_retry(client, messages, model, max_retries=5):
       for attempt in range(max_retries):
           try:
               return client.chat.completions.create(
                   model=model,
                   messages=messages
               )
           except Exception as e:
               if "500" in str(e) or "503" in str(e):
                   if attempt < max_retries - 1:
                       wait_time = min(60, (2 ** attempt))
                       time.sleep(wait_time)
                   else:
                       raise
               else:
                   raise
   ```

3. **적절한 타임아웃 설정하기**
   - HTTP 클라이언트의 타임아웃 값을 늘리세요.
   - 더 나은 처리를 위해 비동기 작업을 사용하는 것을 고려하세요.

<div id="when-to-contact-support">
  ## 지원팀에 문의해야 할 때
</div>

다음과 같은 경우 지원팀에 문의하세요:

- 오류가 10분 이상 계속되는 경우
- 특정 시간대에 반복적인 실패가 발생하는 패턴이 보이는 경우
- 오류 메시지에 추가적인 상세 정보가 포함되어 있는 경우

다음을 함께 제공하세요:

- 오류 메시지와 에러 코드
- 오류가 발생한 시간
- 코드 스니펫 (API 키는 제거)
- W&B 엔터티 이름과 프로젝트 이름
