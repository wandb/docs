---
title: W&B Inference 서버 오류(500, 503)는 어떻게 해결하나요?
---

서버 오류는 W&B Inference 서비스에 일시적인 장애가 발생했음을 나타냅니다.

<div id="error-types">
  ## 오류 유형
</div>

<div id="500-internal-server-error">
  ### 500 - Internal Server Error
</div>

**Message:** "요청을 처리하던 중 서버에서 오류가 발생했습니다"

이는 일시적으로 발생한 서버 내부 오류입니다.

<div id="503-service-overloaded">
  ### 503 - 서비스 과부하
</div>

**Message:** "현재 서비스가 과부하 상태입니다. 잠시 후 다시 시도해 주세요."

서비스에 요청이 몰리고 있습니다.

<div id="how-to-handle-server-errors">
  ## 서버 오류를 처리하는 방법
</div>

1. **다시 시도하기 전에 기다리기**
   - 500 오류: 30~60초 대기
   - 503 오류: 60~120초 대기

2. **지수 백오프 사용하기**
   ```python
   import time
   import openai
   
   def call_with_retry(client, messages, model, max_retries=5):
       for attempt in range(max_retries):
           try:
               return client.chat.completions.create(
                   model=model,
                   messages=messages
               )
           except Exception as e:
               if "500" in str(e) or "503" in str(e):
                   if attempt < max_retries - 1:
                       wait_time = min(60, (2 ** attempt))
                       time.sleep(wait_time)
                   else:
                       raise
               else:
                   raise
   ```

3. **적절한 타임아웃 설정하기**
   - HTTP 클라이언트의 타임아웃 값을 늘리기
   - 더 안정적인 처리를 위해 비동기 작업을 고려하기

<div id="when-to-contact-support">
  ## 지원팀에 문의해야 할 때
</div>

다음과 같은 경우에는 지원팀에 문의하세요:

- 오류가 10분 이상 지속되는 경우
- 특정 시간에 반복적으로 실패가 발생하는 패턴이 보이는 경우
- 오류 메시지에 추가 정보가 포함된 경우

다음을 제공해 주세요:

- 오류 메시지와 코드
- 오류가 발생한 시각
- 코드 스니펫 (API 키는 제거)
- W&B Entity 및 프로젝트 이름
