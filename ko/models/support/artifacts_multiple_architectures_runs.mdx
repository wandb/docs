---
title: 여러 아키텍처와 실행에서 아티팩트를 사용하는 방법
---

모델에 버전을 부여하는 방법은 여러 가지가 있습니다. 아티팩트는 특정 요구 사항에 맞게 모델 버전 관리를 할 수 있는 도구를 제공합니다. 여러 모델 아키텍처를 탐색하는 프로젝트에서 흔히 사용하는 방법은 아키텍처별로 아티팩트를 분리하는 것입니다. 다음 단계를 참고하세요:

1. 서로 다른 각 모델 아키텍처마다 새로운 아티팩트를 생성합니다. 아티팩트의 `metadata` 속성을 사용해 아키텍처에 대한 자세한 설명을 제공하는데, 이는 실행에서 `config`를 사용하는 것과 비슷합니다.
2. 각 모델에 대해 `log_artifact`로 주기적으로 체크포인트를 기록합니다. W&B는 이러한 체크포인트의 이력을 만들고, 가장 최근 체크포인트에 `latest` 별칭을 붙입니다. `architecture-name:latest`를 사용해 어떤 모델 아키텍처에 대해서도 최신 체크포인트를 참조할 수 있습니다.
