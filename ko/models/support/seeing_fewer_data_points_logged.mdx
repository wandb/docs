---
title: 내가 기록한 것보다 데이터 포인트가 적게 보이는 이유는 무엇인가요?
---

`Step`이 아닌 다른 X축에 대해 메트릭을 시각화할 때는 데이터 포인트가 더 적게 보이는 것이 정상입니다. 메트릭이 동기화되려면 동일한 `Step`에서 기록되어야 합니다. 샘플 간 보간을 수행할 때는 동일한 `Step`에서 기록된 메트릭만 샘플링됩니다.

**가이드라인**

여러 메트릭을 하나의 `log()` 호출로 묶어서 기록하세요. 예를 들어, 다음과 같이 하는 대신:

```python
import wandb
with wandb.init() as run:
    run.log({"Precision": precision})
    ...
    run.log({"Recall": recall})
```

사용 방법:

```python
import wandb
with wandb.init() as run:
    run.log({"Precision": precision, "Recall": recall})
```

`step` 매개변수를 수동으로 제어하려면 코드에서 메트릭을 다음과 같이 동기화하세요:

```python
with wandb.init() as run:
    step = 100  # 예시 step 값
    # 동일한 step에서 Precision과 Recall 기록
    run.log({"Precision": precision, "Recall": recall}, step=step)
```

두 `log()` 호출에서 `step` 값이 동일하게 유지되어야 메트릭이 같은 step과 같은 샘플로 함께 기록됩니다. `step` 값은 각 호출마다 단조 증가해야 합니다. 그렇지 않으면 `step` 값은 무시됩니다.
