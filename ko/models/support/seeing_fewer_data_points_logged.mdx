---
title: 기록한 것보다 데이터 포인트가 적게 보이는 이유는 무엇인가요?
---

`Step`이 아닌 다른 X축에 대해 메트릭을 시각화할 때는 데이터 포인트가 더 적게 보일 수 있습니다. 메트릭은 동기화를 유지하려면 동일한 `Step`에서 로깅되어야 합니다. 보간하는 동안에는 동일한 `Step`에서 로깅된 메트릭만 샘플링됩니다.

**가이드라인**

메트릭을 하나의 `log()` 호출로 묶어서 기록하세요. 예를 들어, 다음과 같이 하기보다는:

```python
import wandb
with wandb.init() as run:
    run.log({"Precision": precision})
    ...
    run.log({"Recall": recall})
```

사용 방법:

```python
import wandb
with wandb.init() as run:
    run.log({"Precision": precision, "Recall": recall})
```

`step` 파라미터를 수동으로 제어하려면 코드에서 메트릭을 다음과 같이 동기화하세요:

```python
with wandb.init() as run:
    step = 100  # 예시 step 값
    # 동일한 step에서 Precision과 Recall 기록
    run.log({"Precision": precision, "Recall": recall}, step=step)
```

두 `log()` 호출에서 `step` 값이 동일하게 유지되어야 메트릭이 같은 step과 샘플에 함께 로깅됩니다. 각 호출에서 `step` 값은 단조 증가해야 합니다. 그렇지 않으면 `step` 값은 무시됩니다.
