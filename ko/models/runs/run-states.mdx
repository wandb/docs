[실행 상태](/ko/models/runs/run-states#run-states)는 W&amp;B 실행의 현재 상태를 나타냅니다. 실행의 [상태를 확인](/ko/models/runs/run-states#view-the-state-of-a-run)하려면 W&amp;B 앱에서 보거나, W&amp;B Python SDK를 사용해 프로그램 방식으로 조회할 수 있습니다.

<div id="run-states">
  ## 실행 상태
</div>

다음 표는 실행이 가질 수 있는 상태를 설명합니다:

| State | Description |
| ----- | ----- |
| `Crashed` | 내부 프로세스에서 실행이 하트비트 신호 전송을 중단한 상태입니다. 예를 들어 머신이 다운되었을 때 발생할 수 있습니다. |
| `Failed` | 실행이 0이 아닌 종료 코드로 종료된 상태입니다. |
| `Finished`| 실행이 종료되어 데이터 동기화가 완료되었거나 `wandb.Run.finish()`를 호출한 상태입니다. |
| `Killed` | 실행이 완료되기 전에 강제로 중지된 상태입니다. |
| `Running` | 실행이 여전히 진행 중이며, 최근에 하트비트 신호를 전송한 상태입니다.  |
| `Pending` | 실행이 예약되었지만 아직 시작되지 않은 상태입니다(스윕 및 Launch 작업에서 흔합니다). |

<div id="run-states-in-sweeps">
  ### 스위프에서 실행 상태
</div>

실행이 [sweep](/ko/models/sweeps/)의 일부일 때, 실행 상태는 스위프 상태와는 독립적으로 관리됩니다:

* **개별 실행 상태**는 각 실행의 상태를 반영합니다(Running, Finished, Failed 등)
* **스위프 상태**는 새로운 실행을 생성할지 여부만 제어하며, 이미 생성된 실행이 어떻게 동작하는지는 제어하지 않습니다
* 스위프를 일시 중지하거나 중지해도 이미 실행 중인 실행에는 영향을 주지 않습니다
* 스위프를 취소하는 경우에만 실행 중인 실행을 강제로 종료합니다(실행 상태가 `Killed`로 변경됨)

스위프와 실행 상태가 어떻게 상호 작용하는지에 대한 자세한 설명은 [스위프와 실행 상태 이해하기](/ko/models/sweeps/pause-resume-and-cancel-sweeps#understanding-sweep-and-run-statuses)를 참고하세요.

<div id="view-the-state-of-a-run">
  ## 실행 상태 보기
</div>

Python SDK 또는 W&amp;B App을 사용하여 실행 상태를 코드로 또는 인터랙티브하게 확인할 수 있습니다.

<Tabs>
  <Tab title="Python SDK">
    [`wandb.Api.Run`](/ko/models/ref/python/public-api/runs) 객체의 `state` 속성을 사용하여 실행의 현재 상태를 가져올 수 있습니다.

    다음 코드 스니펫은 지정된 프로젝트의 모든 실행 상태를 가져와 출력합니다. 아래 코드 스니펫을 Python 환경에 복사해 붙여넣고, 꺾쇠 괄호(`< >`)로 둘러싸인 값을 자신의 값으로 바꾸십시오:

    ```python
    import wandb

    api = wandb.Api()

    runs = api.runs(path="<entity>/<project>")

    # 실행 객체의 속성에 접근
    for run in runs:
        print(f"Run: {run.name}")
        print(f"Run state: {run.state}")
        print()
    ```

    여러 기준에 따라 다양한 필터를 적용해 프로젝트에서 원하는 실행만 가져올 수 있습니다. 실행을 프로그래밍 방식으로 필터링하는 방법은 [Filter runs](/ko/models/runs/filter-runs)를 참조하십시오.
  </Tab>

  <Tab title="W&B App">
    W&amp;B App에서 실행 상태를 보려면 다음을 수행하십시오:

    1. W&amp;B 프로젝트로 이동합니다.
    2. 프로젝트 사이드바에서 **Workspace** 또는 **Runs** 탭을 선택합니다.
    3. 상태를 확인하려는 실행을 검색하거나 스크롤하여 찾습니다.
    4. 해당 실행을 선택해 실행 개요 페이지를 엽니다.
    5. **Overview** 탭을 선택합니다.

    **State** 필드 옆에서 실행의 현재 상태를 확인할 수 있습니다.
  </Tab>
</Tabs>
