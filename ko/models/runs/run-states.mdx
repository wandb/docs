---
title: Run 상태
description: W&B run이 가질 수 있는 다양한 상태에 대해 알아보세요.
---

[Run 상태](/ko/models/runs/run-states#run-states)는 W&amp;B run의 현재 상태를 나타냅니다. W&amp;B App에서 run의 [상태를 확인](/ko/models/runs/run-states#view-the-state-of-a-run)하거나, W&amp;B Python SDK를 사용해 프로그래밍 방식으로 확인할 수 있습니다.

<div id="run-states">
  ## Run states
</div>

다음 표는 run이 가질 수 있는 가능한 상태를 설명합니다:

| State | Description |
| ----- | ----- |
| `Crashed` | 내부 프로세스에서 run이 하트비트(heartbeat)를 보내는 것을 중단한 상태로, 머신이 크래시된 경우 발생할 수 있습니다. |
| `Failed` | run이 0이 아닌 종료 상태 코드로 종료되었습니다. |
| `Finished`| run이 종료되고 모든 데이터를 완전히 동기화했거나 `wandb.Run.finish()`를 호출했습니다. |
| `Killed` | run이 완료되기 전에 강제로 중지되었습니다. |
| `Running` | run이 아직 실행 중이며 최근에 하트비트를 전송했습니다.  |
| `Pending` | run이 예약되었지만 아직 시작되지 않은 상태입니다(Sweeps 및 Launch 잡(job)에서 일반적입니다). |

<div id="run-states-in-sweeps">
  ### 스윕에서 run 상태
</div>

run이 [스윕](/ko/models/sweeps/)의 일부인 경우, 각 run의 상태는 스윕 상태와는 독립적으로 동작합니다:

* **개별 run 상태**는 각 run의 실행 상태(Running, Finished, Failed 등)를 반영합니다.
* **스윕 상태**는 기존 run의 실행 방식이 아니라 새 run을 생성할지 여부만 제어합니다.
* 스윕을 일시 중지하거나 중지해도 이미 실행 중인 run에는 영향을 주지 않습니다.
* 스윕을 취소하는 경우에만 실행 중인 run을 강제로 종료합니다(상태가 `Killed`로 변경됩니다).

스윕과 run 상태가 어떻게 상호 작용하는지에 대한 자세한 내용은 [스윕 및 run 상태 이해하기](/ko/models/sweeps/pause-resume-and-cancel-sweeps#understanding-sweep-and-run-statuses)를 참조하세요.

<div id="view-the-state-of-a-run">
  ## run 상태 보기
</div>

Python SDK 또는 W&amp;B App을 사용해 프로그래밍 방식이나 대화형 방식으로 run 상태를 확인할 수 있습니다.

<Tabs>
  <Tab title="Python SDK">
    `wandb.Api.Run` 객체의 `state` 속성을 사용해 run의 현재 상태에 접근할 수 있습니다.

    다음 코드 스니펫은 지정한 프로젝트의 모든 run 상태를 가져와 출력합니다. 아래 코드 스니펫을 Python 환경에 복사해 붙여넣은 후, 꺾쇠 괄호(`< >`) 안의 값을 본인의 값으로 교체하세요:

    ```python
    import wandb

    api = wandb.Api()

    runs = api.runs(path="<entity>/<project>")

    # run 객체의 속성에 접근
    for run in runs:
        print(f"Run: {run.name}")
        print(f"Run state: {run.state}")
        print()
    ```

    여러 가지 기준에 따라 프로젝트에서 run을 조회하기 위해 다양한 필터를 적용할 수 있습니다. run을 프로그래밍 방식으로 필터링하는 방법은 [Filter runs](/ko/models/runs/filter-runs)를 참고하세요. `wandb.Api.Run`에 대한 자세한 내용은 [`wandb.Api.Run`](/ko/models/ref/python/public-api/runs) 문서를 참조하세요.
  </Tab>

  <Tab title="W&B App">
    W&amp;B App에서 run 상태를 확인하려면 다음 단계를 따르세요:

    1. 사용 중인 W&amp;B 프로젝트로 이동합니다.
    2. 프로젝트 사이드바에서 **Workspace** 또는 **Runs** 탭을 선택합니다.
    3. 상태를 확인하려는 run을 검색하거나 스크롤하여 찾습니다.
    4. 해당 run을 선택해 run 개요 페이지를 엽니다.
    5. **Overview** 탭을 선택합니다.

    **State** 필드 옆에서 해당 run의 현재 상태를 확인할 수 있습니다.
  </Tab>
</Tabs>
