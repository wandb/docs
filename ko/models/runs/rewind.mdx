title: run 되감기
---

run의 이력을 수정하려면 run을 되감으십시오. run을 되감기하면, W&amp;B는 동일한 run ID를 유지하면서 run의 상태를 지정한 step으로 재설정합니다.

<Warning>
  run 되감기 옵션은 현재 비공개 프리뷰 단계이며 활발히 개발 중입니다. 되감기에는 알려진 성능 제한이 있으므로, W&amp;B는 일반적으로 대안으로 [Forking](./forking)을 권장합니다.

  현재 W&amp;B는 다음을 지원하지 않습니다:

  * **로그 되감기(Log rewind)**: 새 run 구간에서는 로그가 초기화됩니다.
  * **시스템 메트릭 되감기(System metrics rewind)**: W&amp;B는 되감기 지점 이후의 새로운 시스템 메트릭만 로깅합니다.
  * **아티팩트 연결(Artifact association)**: W&amp;B는 아티팩트를 이를 생성한 소스 run에 연결합니다.

  이 기능에 대한 액세스 권한을 요청하려면 support@wandb.com으로 W&amp;B 지원팀에 문의하십시오.
</Warning>

W&amp;B는 새로 로깅된 이력을 기준으로 되감은 run의 요약 메트릭을 다시 계산합니다. 이는 다음과 같은 동작을 의미합니다:

* **이력 잘라내기(History truncation)**: W&amp;B는 되감기 지점까지의 이력을 잘라내어, 이후 새로운 데이터 로깅을 허용합니다.
* **요약 메트릭(Summary metrics)**: 새로 로깅된 이력을 기준으로 다시 계산됩니다.
* **설정 유지(Configuration preservation)**: W&amp;B는 기존 설정을 유지하며, 새로운 설정을 병합할 수 있습니다.

{/* run을 되감기하면, W&B는 원본 데이터를 보존하고 일관된 run ID를 유지하면서 run 상태를 지정된 step으로 재설정합니다. 이는 다음을 의미합니다:

  - **Run archiving**: W&B는 원본 Runs를 아카이브합니다. Runs는 [Run Overview](./#overview-tab) 탭에서 액세스할 수 있습니다.
  - **Artifact association**: 아티팩트를 이를 생성한 run에 연결합니다. */}

<Note>
  **되감기 및 포킹 호환성**

  포킹은 되감기 기능을 보완합니다.

  run에서 포크를 수행하면, W&amp;B는 특정 시점의 run에서 새로운 분기(branch)를 만들어 다른 파라미터나 모델을 사용해 볼 수 있게 합니다.

  run을 되감기(되감기)하면, W&amp;B를 통해 run 이력 자체를 수정하거나 바로잡을 수 있습니다.
</Note>

<div id="prerequisites">
  ## 사전 요구 사항
</div>

run을 되감기하기 전에 다음 사전 요구 사항을 충족하는지 확인하세요:

* run을 되감기하려면 [W&amp;B Python SDK](https://pypi.org/project/wandb/) 버전이 `0.17.1` 이상이어야 합니다.
* step은 단조 증가하도록 사용해야 합니다. run 이력과 시스템 메트릭에 필요한 시간 순서를 깨뜨리므로, [`define_metric()`](/ko/models/ref/python/experiments/run#define_metric)으로 정의한 비단조(증가하지 않는 구간이 있는) step과는 함께 사용할 수 없습니다.

<div id="rewind-a-run">
  ## run 되감기
</div>

특정 step부터 run을 되감고 그 시점부터 새 데이터를 기록할 수 있습니다. 되감기를 시작할 run ID와 step을 모두 [`wandb.init()`](/ko/models/ref/python/functions/init)의 `resume_from` 파라미터에 인수로 전달하세요. `resume_from` 파라미터는 `<run ID>?_step=<step>` 형식의 문자열을 받습니다. 여기서 `<run ID>`는 되감으려는 run의 run ID이고 `<step>`은 되감기를 시작할 step입니다.

예를 들어 300 step 동안 직선 형태의 데이터를 기록한다고 가정해 보겠습니다:

```python
import wandb

# 첫 번째 run을 초기화하고 일부 메트릭을 기록합니다
with wandb.init(project="<project>", entity="wandb") as run:
    for i in range(300):
        # 선형 그래프를 그립니다
        run.log({"metric": i, "step": i})
```

프로젝트의 워크스페이스에서는 step 0부터 step 300까지의 선형 플롯을 볼 수 있습니다:

<img src="/images/runs/rewind_run_original.png" alt="원래 run의 선형 플롯" />

나중에 step 200부터 run을 되감아, `additional_metric`이라는 새 메트릭에 대해 step 200부터 step 300까지 `i*1.1`을 로깅하려고 합니다. step 250부터는 직선형 선 대신 새로운 미세한 물결 패턴(`i**2 + 2*sin(i/3)`)을 로깅하려고 합니다:

```python
import math

run_ID = "<run_ID>" # 되감기할 run의 run ID로 교체하세요

# 첫 번째 run의 특정 스텝에서 되감기하고 스텝 200부터 메트릭을 기록합니다
with wandb.init(project="<project>", entity="wandb", resume_from=f"{run_ID}?_step=200") as run:

    # 처음 몇 스텝은 run에서 메트릭을 그대로 기록합니다
    # 스텝 250 이후부터 웨이브 패턴을 기록하기 시작합니다
    for i in range(200, 300):
        if i < 250:
            run.log({"metric": i, "step": i})  # 웨이브 없이 run에서 계속 기록합니다
        else:
            # 스텝 250부터 웨이브 동작을 적용합니다
            subtle_wave = i + (2 * math.sin(i / 3.0))  # 미묘한 웨이브 패턴을 적용합니다
            run.log({"metric": subtle_wave, "step": i})
        # 모든 스텝에서 새 메트릭을 추가로 기록합니다
        run.log({"additional_metric": i * 1.1, "step": i})
```

다음 이미지는 업데이트된 프로젝트의 워크스페이스를 보여줍니다. 되감기 후 플롯에서 다음 변경 사항에 유의하세요:

* 선형 플롯은 step 0부터 step 200까지의 원래 선형 추세를 보여주며, 새로운 미세한 물결 패턴은 step 250부터 시작합니다(왼쪽 이미지).
* W&amp;B는 step 200부터 시작하는 `additional_metric` 레이블의 새 플롯(오른쪽 플롯)을 생성했습니다.

<img src="/images/runs/rewind_run_step200_wmetric.png" alt="왼쪽에서 오른쪽으로: 원래 선형 추세와 추가 메트릭" />

<div id="view-an-archived-run">
  ## 보관된 run 보기
</div>

run을 되감은 후에는 W&amp;B App에서 원래 보관된 run을 탐색할 수 있습니다. 보관된 run을 보려면 다음 단계를 따르세요:

1. **Overview 탭 열기:** run 페이지에서 [**Overview** 탭](./#overview-tab)으로 이동합니다. 이 탭에서는 run의 세부 정보와 이력을 종합적으로 확인할 수 있습니다.
2. **Forked From 필드 찾기:** **Overview** 탭에서 `Forked From` 필드를 찾습니다. 이 필드는 재개 이력을 캡처합니다. **Forked From** 필드에는 소스 run에 대한 링크가 포함되어 있어, 원래 run으로 트레이스해 올라가 전체 되감기 이력을 이해할 수 있습니다.

`Forked From` 필드를 사용하면 보관된 재개들의 트리를 손쉽게 탐색하고, 각 되감기의 순서와 출처에 대한 인사이트를 얻을 수 있습니다.

<div id="fork-from-a-run-that-you-rewind">
  ## 되감은 run에서 포크하기
</div>

되감은 run에서 포크하려면 `wandb.init()`에서 [`fork_from`](/ko/models/runs/forking/) 인수를 사용하고, 소스 run ID와 포크를 시작할 소스 run의 step을 지정합니다:

```python
import wandb

# 특정 step에서 run을 포크합니다
forked_run = wandb.init(
    project="<project>",
    entity="<entity>",
    fork_from=f"{rewind_run.id}?_step=500",
)

# 새 run에서 계속 로깅합니다
for i in range(500, 1000):
    forked_run.log({"metric": i*3})
forked_run.finish()
```
