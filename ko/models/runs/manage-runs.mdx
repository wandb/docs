---
title: run을 다른 프로젝트 또는 팀으로 이동하기
description: W&B App을 사용하여 프로젝트 또는 팀 간에 run을 이동합니다.
---

<Note>
  시작하기 전에 프로젝트나 팀 간에 run을 이동할 수 있는 권한이 있는지 확인하세요. 현재 위치와 새 위치 모두에서 해당 run에 접근할 수 있어야 합니다.
</Note>

run을 한 프로젝트에서 다른 프로젝트로, 혹은 팀 간에 이동하려면:

1. 이동하려는 run이 포함된 프로젝트로 이동합니다.
2. 프로젝트 사이드바에서 **Runs** 탭을 선택합니다.
3. 이동하려는 run 옆의 체크박스를 선택합니다.
4. 테이블 위에 있는 **프로젝트로 이동** 버튼을 클릭합니다.
5. 드롭다운에서 대상 팀과 프로젝트를 선택합니다.

<Note>
  run을 이동해도 해당 run과 연관된 과거 아티팩트는 함께 이동되지 않습니다. 아티팩트를 수동으로 이동하려면 [`wandb artifact get`](/ko/models/ref/cli/wandb-artifact/wandb-artifact-get/) SDK 명령어나 [`Api.artifact` API](/ko/models/ref/python/public-api/api/#artifact)를 사용해 아티팩트를 다운로드한 다음, [`wandb artifact put`](/ko/models/ref/cli/wandb-artifact/wandb-artifact-put/) 또는 `Api.artifact` API를 사용해 run의 새 위치로 업로드하면 됩니다.
</Note>

{/* ## run을 그룹으로 이동하기

  기존 그룹으로 run을 이동하거나 새 그룹을 만들어 run을 그룹화할 수 있습니다.

  1. 이동하려는 run이 포함된 프로젝트로 이동합니다.
  2. 프로젝트 사이드바에서 **Runs** 탭을 선택합니다.
  3. 이동하려는 run 옆의 체크박스를 선택합니다.
  4. 테이블 상단에서 **Move to group** 버튼을 클릭합니다.
  5. 모달에서 대상 그룹을 선택하거나 새 그룹을 생성합니다.
  6. **Move**를 클릭합니다. */}

{/* #### 여러 run을 일괄 관리하기
  이 섹션에서는 하나 이상의 run 그룹에 걸쳐 여러 run을 한 번에 관리하는 방법을 설명합니다.

  1. 필요한 경우 [열 기준 그룹화 전환](#toggle-grouping-by-column-in-the-ui)을 사용해 **Group** 열을 기준으로 그룹화합니다.
  1. 그룹을 확장하여 해당 그룹의 run을 확인합니다. 하나의 일괄 작업에서 서로 다른 그룹의 run을 동일한 대상 그룹으로 이동할 수 있습니다. 관련된 모든 그룹을 확장합니다.
  1. 그룹 내의 모든 run을 선택하려면 그룹 이름 옆의 체크박스를 클릭합니다.
  1. 개별 run을 선택하려면 각 run 옆의 체크박스를 클릭합니다.
  1. 모든 그룹의 모든 run을 선택하려면 **Name** 열 머리글 옆의 체크박스를 클릭합니다.
  1. 테이블 상단에서 일괄 작업으로 **Tag**, **Move to group**, **Move to project**, 또는 **New sweep** 중 하나를 선택합니다. */}
