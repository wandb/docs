title: run을 다른 프로젝트 또는 팀으로 이동
description: W&B App을 사용해 프로젝트 또는 팀 간에 run을 이동합니다.
---

<Note>
  시작하기 전에 프로젝트 또는 팀 간에 run을 이동하는 데 필요한 권한이 있는지 확인하세요. 현재 위치와 새 위치 모두에서 해당 run에 액세스할 수 있어야 합니다.
</Note>

한 프로젝트에서 다른 프로젝트로 또는 팀 간에 run을 이동하려면 다음 단계를 따르세요:

1. 이동하려는 run이 포함된 프로젝트로 이동합니다.
2. 프로젝트 사이드바에서 **Runs** 탭을 선택합니다.
3. 이동하려는 run 옆의 체크박스를 선택합니다.
4. 테이블 위에 있는 **Move to project** 버튼을 클릭합니다.
5. 드롭다운에서 대상 팀과 프로젝트를 선택합니다.

<Note>
  run을 이동해도 해당 run과 연결된 과거 artifact는 함께 이동되지 않습니다. artifact를 수동으로 이동하려면 [`wandb artifact get`](/ko/models/ref/cli/wandb-artifact/wandb-artifact-get/) 명령어 또는 [`Api.artifact` API](/ko/models/ref/python/public-api/api/#artifact)를 사용해 artifact를 다운로드한 다음, [`wandb artifact put`](/ko/models/ref/cli/wandb-artifact/wandb-artifact-put/) 또는 `Api.artifact` API를 사용해 run의 새 위치에 업로드할 수 있습니다.
</Note>

  4. 표 위에 있는 **그룹으로 이동** 버튼을 선택합니다.
  5. 모달에서 대상 그룹을 선택하거나 새 그룹을 만듭니다.
  6. **이동**을 클릭합니다. */}

  1. 그룹의 모든 run을 선택하려면 그룹 이름 옆의 체크박스를 클릭합니다.
