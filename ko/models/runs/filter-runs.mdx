title: 실행 필터링

실행 테이블에서 식 편집기를 사용해 실행을 이름, 상태, [태그](#filter-runs-with-tags) 또는 기타 속성을 기준으로 필터링합니다.

필터를 추가할 때는 먼저 필드(예: tags, timestamp, 또는 entity)를 선택합니다. 각 필드는 텍스트, 시간, ID와 같은 기본 타입을 가지고 있습니다.

표시되는 연산자 목록(예: is, in, &ge;, within last)은 이 타입에 따라 달라집니다. 필드를 선택하면 UI에는 해당 필드 타입에 대해 유효한 연산자만 표시됩니다.

<div id="common-operators-by-type">
  ## 유형별 공통 연산자
</div>

| 필터 유형 | 예제 필드 | 공통 연산자 | 사용 예제 |
| ----------- | -------------- | ---------------- | ------------- |
| 태그        | `tags`           | is, is not, in, not in  | `tags is "baseline"` |
| 시간        | `created timestamp`      | &le;, &ge;, within last | `created timestamp` &ge; `01/16/2026` |
| 문자열      | `state`         | &equals;, &ne;, IN, NOT IN | `state = "finished"` |

<Note>
위 표는 사용 가능한 필드와 연산자 중 일부만 보여 줍니다. Expression editor에는 사용 가능한 모든 필드와 연산자가 표시됩니다.
</Note>

<div id="create-a-filter-expression">
  ## 필터 식 생성
</div>

1. 프로젝트 사이드바에서 **Runs** 탭으로 이동합니다.
2. Runs 테이블 위에 있는 깔때기 모양의 **Filter** 버튼을 선택합니다.
3. 왼쪽에서 오른쪽 순서로 열 이름, 논리 연산자, 필터 값을 선택하여 필터 식을 생성합니다.
4. 필요하면 **New Filter**를 선택해 필터를 추가로 적용합니다.

다음 이미지는 손실 값이 `1` 이하인 실행을 필터링한 예시입니다:

다음 섹션에서는 Runs 테이블에서 실행을 필터링하는 몇 가지 예시를 보여 줍니다.

<div id="example-filter-runs-with-tags">
  ### 예시: 태그로 실행 필터링
</div>

실행을 태그로 필터링하려면 다음 단계를 따르세요:

1. 프로젝트 사이드바에서 **Runs** 탭을 클릭합니다.
2. 실행 테이블 위에 있는 깔때기 모양의 **Filter** 버튼을 클릭합니다.
3. 왼쪽에서 오른쪽 순서로, 첫 번째 드롭다운 메뉴에서 `"Tags"`를 선택하고 논리 연산자를 선택합니다.
4. 두 번째 드롭다운 메뉴에서 is, is not, in, 또는 not in 중 하나를 선택합니다.
5. 세 번째 드롭다운 메뉴에 필터링할 태그 이름을 입력합니다.

<div id="default-filters">
  ## 기본 필터
</div>

기본적으로 W&B에서는 다음 필터를 제공합니다:

* **내 실행만 표시**: 현재 사용자가 생성한 실행만 표시합니다.
* **충돌한 실행 숨기기**: 상태가 `crashed`인 실행을 숨깁니다.

기본 필터는 필터 식 편집기에서 **New filter** 버튼 아래에 토글로 표시됩니다.

<div id="remove-a-filter">
  ## 필터 제거하기
</div>

실행 테이블에서 필터를 제거하려면:

1. 실행 테이블 위에 있는 깔때기 모양의 **Filter** 버튼을 클릭합니다.
2. 제거하려는 필터 옆의 `x` 아이콘을 클릭합니다.

{/*
  ## 실행 색상 편집

  새 실행이 생성될 때 기본 색상이 자동으로 지정됩니다. 색상 미리보기를 클릭하면 해당 실행의 색상을 변경할 수 있습니다.
  */}

{/*
  실행 이름 옆에 초록색 점이 있는지 확인하세요. 이 점은 해당 실행이 테이블과 그래프 범례에서 활성 상태임을 나타냅니다.
  */}

{/*
