Runs Table의 표현식 에디터를 사용하여 이름, 상태, [태그](#filter-runs-with-tags) 또는 기타 속성을 기반으로 Runs를 필터링하세요.

필터를 추가할 때 먼저 필드(예: tags, timestamp 또는 entity)를 선택합니다. 각 필드에는 텍스트, 시간 또는 ID와 같은 기본 타입이 있습니다.

표시되는 연산자 목록(예: is, in, &ge;, within last)은 이 타입에 따라 달라집니다. 필드를 선택하면 UI에는 해당 필드 타입에 유효한 연산자만 표시됩니다.

## 타입별 공통 연산자

| 필터 타입 | 예시 필드 | 공통 연산자 | 사용 예시 |
| ----------- | -------------- | ---------------- | ------------- |
| Tags        | `tags`           | is, is not, in, not in  | `tags is "baseline"` |
| Time        | `created timestamp`      | &le;, &ge;, within last | `created timestamp` &ge; `01/16/2026` |
| String      | `state`         | &equals;, &ne;, IN, NOT IN | `state = "finished"` |

<Note>
위 표는 사용 가능한 필드 및 연산자의 일부 서브셋만 보여줍니다. 표현식 에디터에서 모든 사용 가능한 필드와 연산자를 확인할 수 있습니다.
</Note>

## 필터 표현식 생성하기

1. Projects 사이드바에서 **Runs** 탭으로 이동합니다.
2. Runs Table 상단에 있는 깔대기 모양의 **Filter** 버튼을 선택합니다.
3. 왼쪽에서 오른쪽 순서로 컬럼 이름, 논리 연산자, 필터 값을 선택하여 필터 표현식을 생성합니다.
4. 추가 필터를 적용하려면 선택적으로 **New Filter**를 선택합니다.

### 예시: 태그로 Runs 필터링하기

태그를 기반으로 Runs를 필터링합니다:

1. Projects 사이드바에서 **Runs** 탭을 클릭합니다.
2. Runs Table 상단에 있는 깔대기 모양의 **Filter** 버튼을 선택합니다.
3. 첫 번째 드롭다운 메뉴에서 `"Tags"`를 선택하고 논리 연산자를 선택합니다.
4. 두 번째 드롭다운 메뉴에서 is, is not, in, 또는 not in을 선택합니다.
5. 세 번째 드롭다운 메뉴에서 필터링하려는 태그 이름을 입력합니다.

## 기본 필터

기본적으로 W&B는 다음 필터를 제공합니다:

* **Show only my works**: 현재 사용자가 생성한 Runs만 표시합니다.
* **Hide crashed runs**: `crashed` 상태인 Runs를 숨깁니다.

기본 필터는 필터 표현식 에디터의 **New filter** 버튼 아래에 토글 형태로 나타납니다.

## 필터 제거하기

Runs Table에서 필터를 제거하려면:
1. Runs Table 상단의 깔대기 모양 **Filter** 버튼을 클릭합니다.
2. 제거하려는 필터 옆의 `x` 아이콘을 선택합니다.
