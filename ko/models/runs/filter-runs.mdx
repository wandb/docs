Runs 테이블의 식 편집기를 사용하여 이름, 상태, [태그](#filter-runs-with-tags) 또는 기타 속성을 기준으로 run을 필터링할 수 있습니다.

필터를 추가할 때는 먼저 필드를 선택합니다(예: tags, timestamp, entity). 각 필드에는 text, time, ID와 같은 기본 타입이 있습니다.

표시되는 연산자 목록(is, in, &ge;, within last 등)은 이 타입에 따라 달라집니다. 필드를 선택하면 UI에는 해당 필드 타입에 유효한 연산자만 표시됩니다.

<div id="common-operators-by-type">
  ## 유형별 일반적인 연산자
</div>

| 필터 유형 | 예시 필드 | 일반적인 연산자 | 사용 예 |
| ----------- | -------------- | ---------------- | ------------- |
| 태그        | `tags`           | is, is not, in, not in  | `tags is "baseline"` |
| 시간        | `created timestamp`      | &le;, &ge;, within last | `created timestamp` &ge; `01/16/2026` |
| 문자열      | `state`         | &equals;, &ne;, IN, NOT IN | `state = "finished"` |

<Note>
위의 표는 사용 가능한 필드와 연산자의 일부만 보여 줍니다. 표현식 편집기에서는 사용 가능한 모든 필드와 연산자가 표시됩니다.
</Note>

<div id="create-a-filter-expression">
  ## 필터 식 만들기
</div>

1. 프로젝트 사이드바에서 **Runs** 탭으로 이동합니다.
2. Runs 테이블 상단에 있는 깔때기 모양의 **Filter** 버튼을 선택합니다.
3. 왼쪽에서 오른쪽 순서로 열 이름, 논리 연산자, 필터 값을 선택해 필터 식을 생성합니다.
4. 필요하면 **New Filter**를 선택해 필터를 추가로 적용합니다.

다음 이미지는 loss 값이 `1` 이하인 run만 보여주도록 필터링한 예시입니다:

다음 섹션에서는 Runs 테이블에서 run을 필터링하는 몇 가지 예시를 보여줍니다.

<div id="example-filter-runs-with-tags">
  ### 예시: 태그로 run 필터링하기
</div>

run을 태그를 기준으로 필터링하려면 다음을 수행합니다:

1. 프로젝트 사이드바에서 **Runs** 탭을 클릭합니다.
2. runs 테이블 위에 있는 깔때기 모양의 **Filter** 버튼을 선택합니다.
3. 왼쪽에서 오른쪽 순서로, 첫 번째 드롭다운 메뉴에서 `"Tags"`를 선택하고 두 번째 드롭다운에서 논리 연산자를 선택합니다.
4. 두 번째 드롭다운 메뉴에서 is, is not, in, 또는 not in을 선택합니다.
5. 세 번째 드롭다운 메뉴에 필터링하려는 태그 이름을 입력합니다.

<div id="default-filters">
  ## 기본 필터
</div>

기본적으로 W&B는 다음과 같은 필터를 제공합니다:

* **내 run만 표시**: 현재 사용자가 생성한 run만 표시합니다.
* **실패한 run 숨기기**: `crashed` 상태인 run을 숨깁니다.

기본 필터는 필터 식 편집기에서 **New filter** 버튼 아래에 토글로 표시됩니다.

<div id="remove-a-filter">
  ## 필터 제거
</div>

Runs 테이블에서 필터를 제거하려면:

1. Runs 테이블 위에 있는 깔때기 모양의 **Filter** 버튼을 클릭합니다.
2. 제거하려는 필터 옆의 `x` 아이콘을 클릭합니다.

{/*
  ## run 색상 편집

  새 run이 생성되면 기본 색상이 자동으로 지정됩니다. 색상 미리보기를 클릭하여 해당 run의 색상을 편집할 수 있습니다.
  */}

{/*
  run 이름 옆에 있는 초록색 점을 확인하세요. 이 점은 해당 run이 테이블과 그래프 범례에서 활성 상태임을 나타냅니다.
  */}

{/*
