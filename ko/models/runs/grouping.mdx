run을 *그룹*으로 구성하세요. 그룹은 특정 모델의 트레이닝 run이나 특정 데이터셋의 평가 run처럼 공통된 목적을 공유하는 run 모음입니다.

*작업 유형*과 같은 다른 속성으로도 run을 구성할 수 있습니다. [작업 유형](/ko/models/runs/grouping#organize-runs-by-job-type)은 `preprocessing`, `training`, `evaluation`처럼 run의 용도를 나타냅니다.

<div id="organize-runs-into-groups">
  ## run을 그룹으로 정리하기
</div>

W&amp;B Python SDK를 사용해 프로그래밍 방식으로 run을 그룹에 추가하거나, W&amp;B App에서 대화형으로 추가할 수 있습니다.

<Note>
  W&amp;B는 그룹 이름을 run [`wandb.Run.group`](/ko/models/ref/python/experiments/run#property-run-group) 속성으로 저장합니다.
</Note>

<Tabs>
  <Tab title="Python SDK">
    W&amp;B Python SDK를 사용해 하나 이상의 run을 그룹에 프로그래밍 방식으로 추가할 수 있습니다. `wandb.init(group="")`로 run을 초기화할 때 `group` 파라미터에 그룹 이름을 인자로 전달하세요. 그룹 이름을 사용해 W&amp;B App에서 run을 정리하고 필터링할 수 있습니다.

    다음 예제에서는 `A`, `B`, `C`라는 이름의 세 그룹을 생성합니다. 각 그룹에는 세 개의 run이 포함됩니다.

    프로젝트의 Workspace에서 그룹별로 정리된 run을 볼 수 있습니다. 다음 이미지는 그룹 이름을 기준으로 Runs 테이블을 정리한 모습을 보여줍니다. Runs 테이블에는 `A`, `B`, `C`라는 이름의 세 그룹이 표시되며, 각 그룹에는 세 개의 run이 포함됩니다.

    <Frame>
      <img src="/images/runs/group_name_3_groups.png" alt="그룹 이름을 기준으로 그룹화된 Runs 테이블" />
    </Frame>
  </Tab>

  <Tab title="W&B App">
    1. W&amp;B 프로젝트로 이동합니다.
    2. 프로젝트 사이드바에서 **Runs** 탭을 선택합니다.
    3. run 목록 위에서 **Group** 버튼을 클릭합니다.
    4. 그룹으로 묶으려는 하나 이상의 run 옆에 있는 체크박스를 클릭합니다.
    5. **Move to group**을 선택합니다.
    6. 드로어에서 기존 그룹을 선택하거나 새 그룹을 만듭니다.
  </Tab>
</Tabs>

<div id="view-groups">
  ### 그룹 보기
</div>

W&amp;B App에서 그룹별로 정리된 run을 보려면 다음과 같이 하세요:

1. 프로젝트 사이드바에서 **Runs** 탭을 선택합니다.
2. run 목록 위에서 **Group** 버튼을 클릭합니다.
3. 드롭다운에서 **Group**을 선택합니다.

<div id="move-runs-between-groups">
  ### 그룹 간 run 이동
</div>

한 그룹의 run을 다른 그룹으로 이동합니다:

1. W&amp;B 프로젝트로 이동합니다.
2. 프로젝트 사이드바에서 **Runs** 탭을 선택합니다.
3. 체크박스를 클릭해 하나 이상의 run을 선택합니다.
4. 테이블 상단에서 **Move to group**을 클릭합니다.
5. 드로어에서 대상 그룹을 선택하거나 새 그룹을 만듭니다.
6. **Move**를 클릭합니다.

<div id="remove-runs-from-a-group">
  ### 그룹에서 run 제거하기
</div>

1. W&amp;B 프로젝트로 이동합니다.
2. 프로젝트 사이드바에서 **Runs** 탭을 선택합니다.
3. run 목록 위에 있는 **Group** 버튼을 클릭합니다.
4. 드롭다운에서 제거할 그룹 이름 옆의 **X**를 선택합니다.

<div id="delete-a-group">
  ### 그룹 삭제
</div>

그룹을 삭제하려면 해당 그룹의 모든 run을 제거하세요. 그러면 그룹이 자동으로 삭제됩니다.

<div id="organize-runs-by-job-type">
  ## 작업 유형별로 run 정리
</div>

run을 &#95;작업 유형&#95;별로 정리합니다. 작업 유형은 `preprocessing`, `training`, `evaluation`처럼 run의 기능을 나타냅니다.

<Note>run의 [`wandb.Run.job_type`](/ko/models/ref/python/experiments/run#property-run-job-type) 속성에서 run의 작업 유형을 확인할 수 있습니다.</Note>

`wandb.init(job_type="")`에 `job_type` 파라미터를 전달해 run에 작업 유형을 추가합니다. 예를 들어, 다음 코드 스니펫은 `training` 또는 `evaluation` 작업 유형을 가진 run을 생성합니다.

<div id="view-runs-organized-by-job-type">
  ### 작업 유형별로 정리된 run 보기
