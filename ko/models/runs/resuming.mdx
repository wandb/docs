description: 일시 중지되었거나 종료된 W&B 실행 재개
title: 실행 재개
---

실행이 중지되거나 크래시가 발생하는 경우 어떻게 동작해야 할지 지정합니다. 실행을 다시 시작하거나 자동으로 재개되도록 하려면, 해당 실행과 연결된 고유 실행 ID를 `id` 매개변수에 지정해야 합니다:

```python
import wandb
with wandb.init(entity="<entity>", project="<project>", id="<run ID>", resume="<resume>") as run:
        # 여기에 학습 코드를 작성하세요
```

<Note>
  W&amp;B는 실행을 저장할 W&amp;B 프로젝트 이름을 지정할 것을 권장합니다.
</Note>

W&amp;B가 어떻게 동작해야 할지 결정하려면 `resume` 파라미터에 다음 인자 중 하나를 전달합니다. 각 경우에 W&amp;B는 먼저 실행 ID가 이미 존재하는지 확인합니다.

| Argument  | Description                               | Run ID exists                     | Run ID does not exist             | Use case                          |
| --------- | ----------------------------------------- | --------------------------------- | --------------------------------- | --------------------------------- |
| `"must"`  | W&amp;B는 실행 ID로 지정된 실행을 반드시 재개해야 합니다.     | W&amp;B가 동일한 실행 ID를 가진 실행을 재개합니다. | W&amp;B가 오류를 발생시킵니다.              | 반드시 동일한 실행 ID를 사용해야 하는 실행을 재개합니다. |
| `"allow"` | 실행 ID가 존재하면 W&amp;B가 실행을 재개하도록 허용합니다.     | W&amp;B가 동일한 실행 ID를 가진 실행을 재개합니다. | W&amp;B가 지정된 실행 ID로 새 실행을 초기화합니다. | 기존 실행을 덮어쓰지 않고 실행을 재개합니다.         |
| `"never"` | 실행 ID로 지정된 실행을 W&amp;B가 절대 재개하지 못하도록 합니다. | W&amp;B가 오류를 발생시킵니다.              | W&amp;B가 지정된 실행 ID로 새 실행을 초기화합니다. |                                   |

{/* - `"must"`:  실행 ID가 존재하는 경우 W&B는 해당 실행 ID의 실행을 재개합니다. 실행 ID가 존재하지 않으면 W&B는 아무 작업도 수행하지 않습니다.
  - `"allow"`:  실행 ID가 존재하는 경우 W&B는 해당 실행 ID의 실행을 재개합니다. 실행 ID가 존재하지 않으면 W&B는 지정된 실행 ID로 새로운 실행을 초기화합니다.
  - `"never"`: 실행 ID가 존재하는 경우 W&B는 아무 작업도 수행하지 않습니다. 실행 ID가 존재하지 않으면 W&B는 지정된 실행 ID로 새로운 실행을 시작합니다. */}

또한 `resume="auto"`를 지정하면 W&amp;B가 자동으로 사용자를 대신해 실행을 다시 시작하려고 시도합니다. 다만, 반드시 동일한 디렉터리에서 실행을 다시 시작해야 합니다. 자세한 내용은 [실행을 자동으로 재개하도록 설정](#enable-runs-to-automatically-resume) 섹션을 참조하세요.

아래 모든 예시에서 `<>`로 둘러싸인 값은 사용자 고유의 값으로 바꾸세요.

<div id="resume-a-run-that-must-use-the-same-run-id">
  ## 동일한 run ID를 반드시 사용해 실행 재개하기
</div>

실행이 중지되었거나 크래시가 발생했거나 실패한 경우, 동일한 run ID를 사용해 실행을 재개할 수 있습니다. 이를 위해 실행을 초기화할 때 다음을 지정합니다:

* `resume` 파라미터를 `"must"`로 설정합니다 (`resume="must"`)
* 중지되었거나 크래시가 발생한 실행의 run ID를 지정합니다

{/* 실행을 초기화할 때 `resume` 파라미터를 `"must"` (`resume="must"`)로 설정하고, 중지되었거나 크래시된 실행의 실행 ID를 지정하십시오. */}

다음 코드 스니펫은 W&amp;B Python SDK로 이를 구현하는 방법을 보여줍니다:

```python
with wandb.init(entity="<entity>", project="<project>", id="<run ID>", resume="must") as run:
        # 여기에 학습 코드를 작성하세요
```

<Warning>
  여러 프로세스가 동시에 동일한 `id`를 사용하면 예기치 않은 결과가 발생할 수 있습니다.

  여러 프로세스를 관리하는 방법에 대한 자세한 내용은 [분산 학습 실험 로깅](/ko/models/track/log/distributed-training/)을 참고하세요.
</Warning>

<div id="resume-a-run-without-overriding-the-existing-run">
  ## 기존 실행을 덮어쓰지 않고 실행 다시 시작하기
</div>

중단되었거나 충돌로 종료된 실행을 기존 실행을 덮어쓰지 않고 다시 시작할 수 있습니다. 이는 프로세스가 정상적으로 종료되지 않을 때 특히 유용합니다. 다음번에 W&amp;B를 시작하면 W&amp;B는 마지막 스텝부터 로깅을 다시 시작합니다.

W&amp;B에서 실행을 초기화할 때 `resume` 파라미터를 `"allow"` (`resume="allow"`)로 설정합니다. 중단되었거나 충돌된 실행의 실행 ID를 지정합니다. 다음 코드 스니펫은 W&amp;B Python SDK를 사용하여 이를 수행하는 방법을 보여줍니다:

```python
import wandb

with wandb.init(entity="<entity>", project="<project>", id="<run ID>", resume="allow") as run:
        # 여기에 학습 코드를 작성하세요
```

<div id="enable-runs-to-automatically-resume">
  ## 실행을 자동으로 재개하도록 설정하기
</div>

다음 코드 스니펫은 Python SDK 또는 환경 변수를 사용하여 실행을 자동으로 재개하도록 설정하는 방법을 보여줍니다.

<Tabs>
  <Tab title="W&B Python SDK">
    다음 코드 스니펫은 W&amp;B 실행 ID를 Python SDK로 지정하는 방법을 보여줍니다.

    `<>`로 둘러싸인 값을 사용자 값으로 바꾸십시오:

    ```python
    with wandb.init(entity="<entity>", project="<project>", id="<run ID>", resume="<resume>") as run:
            # 여기에 학습 코드를 작성합니다
    ```
  </Tab>

  <Tab title="Shell script">
    다음 예시는 bash 스크립트에서 W&amp;B `WANDB_RUN_ID` 변수를 지정하는 방법을 보여줍니다:

    ```bash title="run_experiment.sh"
    RUN_ID="$1"

    WANDB_RESUME=allow WANDB_RUN_ID="$RUN_ID" python eval.py
    ```

    터미널에서 W&amp;B 실행 ID와 함께 셸 스크립트를 실행할 수 있습니다. 다음 코드 스니펫은 실행 ID `akj172`를 전달합니다:

    ```bash
    sh run_experiment.sh akj172 
    ```
  </Tab>
</Tabs>

<Warning>
  자동 재개는 실패한 프로세스와 동일한 파일 시스템을 사용하여 프로세스를 다시 시작하는 경우에만 동작합니다.
</Warning>

예를 들어, `Users/AwesomeEmployee/Desktop/ImageClassify/training/` 디렉터리에서 `train.py`라는 Python 스크립트를 실행한다고 가정해 보겠습니다. `train.py` 내에서 스크립트는 자동 재개가 활성화된 실행을 생성합니다. 이후 학습 스크립트가 중지되었다고 가정하면, 이 실행을 재개하려면 `Users/AwesomeEmployee/Desktop/ImageClassify/training/` 디렉터리에서 `train.py` 스크립트를 다시 실행해야 합니다.

<Note>
  파일 시스템을 공유할 수 없다면, `WANDB_RUN_ID` 환경 변수를 지정하거나 W&amp;B Python SDK를 사용하여 실행 ID를 전달하십시오. 실행 ID에 대한 자세한 내용은 &quot;실행이란 무엇인가?&quot; 페이지의 [사용자 지정 실행 ID](./#custom-run-ids) 섹션을 참조하십시오.
</Note>

<div id="resume-preemptible-sweeps-runs">
  ## 선점 가능한 Sweep 실행 재개하기
</div>

중단된 [sweep](/ko/models/sweeps/) 실행을 자동으로 다시 대기열에 넣습니다. 이는 선점 가능 큐의 SLURM 잡, EC2 스팟 인스턴스, Google Cloud 선점형 VM처럼 선점이 발생할 수 있는 컴퓨팅 환경에서 Sweep 에이전트를 실행하는 경우 특히 유용합니다.

[`mark_preempting`](/ko/models/ref/python/experiments/run#mark_preempting) 함수를 사용해 중단된 sweep 실행을 자동으로 다시 대기열에 넣을 수 있습니다. 예를 들어:

```python
with wandb.init() as run:
    run.mark_preempting()
```

The following table outlines how W&amp;B handles 실행 based on the exit status of a sweep 실행.

| Status         | Behavior                                                                                                    |
| -------------- | ----------------------------------------------------------------------------------------------------------- |
| Status code 0  | 실행이 정상적으로 종료된 것으로 간주되며, 다시 큐에 넣지 않습니다.                                                                      |
| Nonzero status | W&amp;B는 해당 실행을 스윕과 연결된 실행 큐에 자동으로 추가합니다.                                                                   |
| No status      | 실행이 스윕 실행 큐에 추가됩니다. 스윕 에이전트는 큐가 빌 때까지 실행 큐에서 실행을 가져와 처리합니다. 큐가 비면, 스윕 큐는 스윕 검색 알고리즘을 기반으로 새로운 실행을 다시 생성합니다. |
