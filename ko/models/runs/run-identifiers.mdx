---

W&amp;B run을 초기화하면 W&amp;B는 해당 run에 [*run ID*라고 하는 고유 식별자](/ko/models/runs/run-identifiers#run-id)를 할당합니다. 각 run에는 사람이 읽기 쉬운 형태의, 고유할 필요는 없는 [*run 이름*](/ko/models/runs/run-identifiers#run-name)도 있으며, 이 이름은 사용자가 직접 지정할 수 있습니다.

<div id="run-id">
  ## Run ID
</div>

run ID는 각 run을 고유하게 식별하는 값입니다. 기본적으로 W&amp;B는 새 run을 초기화할 때 [무작위이면서 고유한 run ID](#autogenerated-run-ids)를 자동으로 생성합니다. 다만 [run을 초기화할 때](/ko/models/runs/initialize-run) [사용자 고유의 run ID](#create-a-custom-run-id)를 직접 지정하면 해당 ID가 사용됩니다.

<div id="find-a-runs-id">
  ### run ID 찾기
</div>

run의 고유 ID는 W&amp;B Python SDK를 사용해 코드에서 찾거나, W&amp;B App에서 직접 확인할 수 있습니다.

<Tabs>
  <Tab title="Python SDK">
    run을 초기화하면 W&amp;B가 터미널에 고유한 run ID를 출력합니다. 예를 들어, 다음 코드 스니펫은 W&amp;B run을 초기화합니다:

    ```python
    import wandb
    entity = "nico"  # 자신의 W&B entity로 바꾸세요
    project = "awesome-project" 
    with wandb.init(entity=entity, project=project) as run:
        # 여기에 코드를 작성하세요
    ```

    터미널에는 다음과 같이 출력됩니다:

    ```bash
    wandb: Syncing run earnest-sunset-1
    wandb: ⭐️ View project at https://wandb.ai/nico/awesome-project
    wandb: 🚀 View run at https://wandb.ai/nico/awesome-project/runs/1jx1ud12
    ```

    run URL의 마지막 부분 `1jx1ud12`가 고유한 run ID입니다.
  </Tab>

  <Tab title="W&B App">
    W&amp;B App에서도 run의 고유 ID를 찾을 수 있습니다:

    1. [W&amp;B App](https://wandb.ai/home)으로 이동합니다.
    2. run을 초기화할 때 지정한 W&amp;B 프로젝트로 이동합니다.
    3. 프로젝트의 워크스페이스에서 **Workspace** 또는 **Runs** 탭을 선택합니다.
    4. 확인하려는 run을 선택합니다.
    5. **Overview** 탭을 선택합니다.

    W&amp;B는 **Run path** 필드에 run ID를 표시합니다. run path는 팀 이름, 프로젝트 이름, 그리고 run ID로 구성됩니다. 고유 ID는 run path의 마지막 부분입니다.

    예를 들어, 다음 이미지에서 고유 run ID는 `9mxi1arc`입니다:

    <Frame>
      <img src="/images/runs/unique-run-id.png" alt="Run ID 위치" />
    </Frame>
  </Tab>
</Tabs>

<Tip>
  run의 고유 ID를 사용하면 W&amp;B App에서 해당 run의 overview 페이지로 바로 이동할 수 있습니다. 다음 코드 블록은 특정 run에 대한 URL 경로 형식을 보여줍니다:

  ```text title="특정 run에 대한 W&B App URL"
  https://wandb.ai/<entity>/<project>/<run-id>
  ```

  꺾쇠 괄호(`< >`)로 둘러싸인 값을 entity, 프로젝트, 그리고 run ID의 실제 값으로 바꾸세요.
</Tip>

<div id="create-a-custom-run-id">
  ### 사용자 지정 run ID 생성
</div>

run을 초기화할 때 원하는 run ID를 문자열 형태로 `id` 매개변수에 전달하세요:

```python
import wandb

with wandb.init(entity="<project>", project="<project>", id="<run-id>") as run:
    # 여기에 코드를 작성하세요
```

<div id="run-name">
  ## Run name
</div>

각 run에는 사람이 읽기 쉬운(하지만 고유하지 않은) run 이름이 있습니다. 기본적으로 새 run을 초기화할 때 run 이름을 지정하지 않으면 W&amp;B가 임의의 run 이름을 생성합니다. run 이름은 프로젝트의 Workspace 안과 [run의 **Overview** 페이지](#overview-tab) 상단에 표시됩니다.

이전 예시를 이어서, 이 run의 이름은 `glowing-shadows-8`입니다.

<Frame>
  <img src="/images/runs/unique-run-id.png" alt="Run ID location" />
</Frame>

[run을 초기화할 때](/ko/models/runs/run-identifiers#create-a-custom-run-name) run 이름을 지정하거나, 나중에 [이름을 변경](/ko/models/runs/run-identifiers#rename-a-run)할 수 있습니다.

<div id="create-a-custom-run-name">
  ### 사용자 지정 run 이름 만들기
</div>

[`wandb.init()`](/ko/models/ref/python/functions/init) 메서드에 `name` 파라미터를 전달하여 run 이름을 지정합니다.

```python
import wandb

with wandb.init(entity="<project>", project="<project>", name="<run-name>") as run:
    # 여기에 코드를 작성하세요
```

<div id="rename-a-run">
  ### run 이름 바꾸기
</div>

Python SDK로 프로그램에서 초기화한 run이든 W&amp;B App에서 인터랙티브하게 시작한 run이든, 이름을 변경할 수 있습니다.

<Tabs>
  <Tab title="Python SDK">
    [`wandb.Api.Run`](/ko/models/ref/python/public-api/api#method-api-run)을 사용해 W&amp;B에 기록된 run을 가져옵니다. 이 메서드는 run 이름을 업데이트할 수 있는 [run object](/ko/models/ref/python/public-api/run#property-run-name)를 반환합니다. 변경 사항을 저장하려면 `wandb.Api.Run.update()` 메서드를 호출합니다.

    꺾쇠 괄호(`< >`)로 둘러싸인 값을 실제 환경에 맞는 값으로 바꾸십시오.

    ```python
    import wandb

    api = wandb.Api()

    # Access run by its path
    run = api.run(path = "<entity>/<project>/<run-id>")

    # Specify a new run name
    run.name = "<new-run-name>"
    run.update()
    ```
  </Tab>

  <Tab title="W&B App">
    1. W&amp;B 프로젝트로 이동합니다.
    2. **Workspace** 또는 **Runs** 탭을 선택합니다.
    3. 이름을 변경하려는 run을 검색하거나 스크롤하여 찾습니다.
    4. run 이름 위에 마우스를 올린 뒤 세로 점 3개 아이콘을 클릭하고 **Rename run**을 클릭합니다.
    5. run 이름을 변경하려면 **Run name** 필드를 변경합니다.
    6. **Save**를 클릭합니다.
  </Tab>
</Tabs>

<div id="run-display-name">
  ## Run 표시 이름
</div>

각 run에는 워크스페이스별로 사용자 지정할 수 있는 *run display name*도 있습니다.

<Note>
  한 워크스페이스에서 run의 표시 이름을 변경해도, 그 표시 이름은 해당 워크스페이스에만 변경되며 다른 워크스페이스나 프로젝트에는 영향을 주지 않습니다.
</Note>

표시 이름의 기본값은 run 이름과 동일합니다. 표시 이름은 run의 워크스페이스와 Runs 테이블에 표시됩니다.

run display name을 사용하면, 프로젝트에서 run 이름을 바꾸지 않고도 해당 워크스페이스에 표시되는 run 이름만 따로 지정할 수 있습니다.

<div id="rename-a-runs-display-name">
  ### run 표시 이름 변경하기
</div>

W&amp;B App에서 run의 표시 이름을 변경하려면:

1. W&amp;B 프로젝트로 이동합니다.
2. **Workspace** 또는 **Runs** 탭을 선택합니다.
3. 이름을 변경할 run을 검색하거나 스크롤하여 찾습니다.
4. run 이름에 마우스를 올린 후 세로 점 세 개 아이콘을 클릭하고, **Rename run**을 클릭합니다.
5. **Display name** 필드에 새 표시 이름을 입력합니다.
6. **Save**를 클릭합니다.

1. Runs 목록 상단의 작업 `...` 메뉴를 클릭합니다.
2. **Run name cropping**을 끝, 가운데, 또는 시작 부분을 잘라내도록 설정합니다.
