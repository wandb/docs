description: 실행의 고유 식별자와 실행 이름을 찾는 방법, 사용자 지정 실행 ID를 생성하는 방법, 그리고 실행 이름을 사용자 지정하는 방법을 알아봅니다.
---

W&amp;B 실행을 초기화하면, W&amp;B는 해당 실행에 [*실행 ID*라고 하는 고유 식별자](/ko/models/runs/run-identifiers#run-id)를 할당합니다. 각 실행에는 사용자가 직접 변경할 수 있는, 고유하지 않은 사람이 읽기 쉬운 [*실행 이름*](/ko/models/runs/run-identifiers#run-name)도 있습니다.

<div id="run-id">
  ## 실행 ID
</div>

실행 ID는 실행을 고유하게 식별합니다. 기본적으로 새 실행을 초기화할 때 W&amp;B는 [무작위이면서 고유한 실행 ID](#autogenerated-run-ids)를 자동으로 생성합니다. 다만 [실행을 초기화](/ko/models/runs/initialize-run)할 때 [직접 고유한 실행 ID를 지정](#create-a-custom-run-id)할 수도 있습니다.

<div id="find-a-runs-id">
  ### 실행 ID 찾기
</div>

W&amp;B Python SDK를 사용해 코드에서 또는 W&amp;B App에서 실행의 고유 ID를 찾을 수 있습니다.

<Tabs>
  <Tab title="Python SDK">
    실행을 초기화하면 W&amp;B가 터미널에 고유 실행 ID를 출력합니다. 예를 들어, 다음 코드 스니펫은 W&amp;B 실행을 초기화합니다:

    ```python
    import wandb
    entity = "nico"  # 자신의 W&B 엔터티로 바꾸세요
    project = "awesome-project" 
    with wandb.init(entity=entity, project=project) as run:
        # 여기에 코드를 작성하세요
    ```

    터미널에는 다음과 같이 표시됩니다:

    ```bash
    wandb: Syncing run earnest-sunset-1
    wandb: ⭐️ View project at https://wandb.ai/nico/awesome-project
    wandb: 🚀 View run at https://wandb.ai/nico/awesome-project/runs/1jx1ud12
    ```

    실행 URL의 마지막 부분인 `1jx1ud12`가 고유 실행 ID입니다.
  </Tab>

  <Tab title="W&B App">
    W&amp;B App에서도 실행의 고유 ID를 찾을 수 있습니다:

    1. [W&amp;B App](https://wandb.ai/home)으로 이동합니다.
    2. 실행을 초기화할 때 지정한 W&amp;B 프로젝트로 이동합니다.
    3. 프로젝트 워크스페이스에서 **Workspace** 또는 **Runs** 탭을 선택합니다.
    4. 확인하려는 실행을 선택합니다.
    5. **Overview** 탭을 선택합니다.

    W&amp;B는 **Run path** 필드에 실행 ID를 표시합니다. 실행 path는 팀 이름, 프로젝트 이름, 실행 ID로 구성됩니다. 고유 ID는 실행 path의 마지막 부분입니다.

    예를 들어, 다음 이미지에서 고유 실행 ID는 `9mxi1arc`입니다:

    <Frame>
      <img src="/images/runs/unique-run-id.png" alt="Run ID 위치" />
    </Frame>
  </Tab>
</Tabs>

<Tip>
  실행의 고유 ID를 사용하면 W&amp;B App에서 해당 실행의 Overview 페이지로 바로 이동할 수 있습니다. 다음 코드 블록은 실행에 대한 URL path 형식을 보여줍니다:

  ```text title="특정 실행에 대한 W&B App URL"
  https://wandb.ai/<entity>/<project>/<run-id>
  ```

  꺾쇠괄호(`< >`)로 둘러싸인 값을 실제 엔터티, 프로젝트, 실행 ID 값으로 바꾸세요.
</Tip>

<div id="create-a-custom-run-id">
  ### 사용자 지정 실행 ID 만들기
</div>

실행을 초기화할 때 원하는 실행 ID 문자열을 `id` 매개변수에 전달하세요:

```python
import wandb

with wandb.init(entity="<project>", project="<project>", id="<run-id>") as run:
    # 여기에 코드를 작성하세요
```

<div id="run-name">
  ## 실행 이름
</div>

각 실행에는 사람이 읽기 쉬운, 하지만 고유할 필요는 없는 실행 이름이 있습니다. 기본적으로 새 실행을 초기화할 때 실행 이름을 직접 지정하지 않으면, W&amp;B가 임의의 실행 이름을 생성합니다. 실행 이름은 프로젝트의 워크스페이스와 [실행의 **Overview** 페이지](#overview-tab) 상단에 표시됩니다.

앞선 예시에서와 같이, 실행의 이름은 `glowing-shadows-8` 입니다.

<Frame>
  <img src="/images/runs/unique-run-id.png" alt="실행 ID 위치" />
</Frame>

[실행을 초기화할 때](/ko/models/runs/run-identifiers#create-a-custom-run-name) 실행 이름을 지정하거나, 나중에 [이름을 변경](/ko/models/runs/run-identifiers#rename-a-run)할 수 있습니다.

<div id="create-a-custom-run-name">
  ### 사용자 지정 실행 이름 만들기
</div>

[`wandb.init()`](/ko/models/ref/python/functions/init) 메서드에 `name` 매개변수를 전달해 실행 이름을 지정합니다.

```python
import wandb

with wandb.init(entity="<project>", project="<project>", name="<run-name>") as run:
    # 여기에 코드를 작성하세요
```

<div id="rename-a-run">
  ### 실행 이름 바꾸기
</div>

Python SDK로 프로그래밍 방식으로 실행을 초기화했거나 W&amp;B App에서 대화형으로 실행을 초기화한 후 실행 이름을 변경할 수 있습니다.

<Tabs>
  <Tab title="Python SDK">
    [`wandb.Api.Run`](/ko/models/ref/python/public-api/api#method-api-run)을 사용해 W&amp;B에 기록된 실행에 액세스합니다. 이 메서드는 실행 이름을 업데이트하는 데 사용할 수 있는 [실행 객체](/ko/models/ref/python/public-api/run#property-run-name)를 반환합니다. 변경 내용을 저장하려면 `wandb.Api.Run.update()` 메서드를 호출합니다.

    꺾쇠 괄호(`< >`)로 둘러싸인 값을 사용자 값으로 바꾸십시오.

    ```python
    import wandb

    api = wandb.Api()

    # 실행 경로로 실행에 액세스
    run = api.run(path = "<entity>/<project>/<run-id>")

    # 새 실행 이름 지정
    run.name = "<new-run-name>"
    run.update()
    ```
  </Tab>

  <Tab title="W&B App">
    1. W&amp;B 프로젝트로 이동합니다.
    2. **Workspace** 또는 **Runs** 탭을 선택합니다.
    3. 이름을 변경하려는 실행을 검색하거나 스크롤하여 찾습니다.
    4. 실행 이름 위에 커서를 올리고 세로 점 세 개 아이콘을 클릭한 다음 **Rename run**을 클릭합니다.
    5. 실행 이름을 변경하려면 **Run name** 필드를 수정합니다.
    6. **Save**를 클릭합니다.
  </Tab>
</Tabs>

<div id="run-display-name">
  ## 실행 표시 이름
</div>

각 실행에는 워크스페이스마다 개별적으로 설정할 수 있는 *실행 표시 이름*도 있습니다.

<Note>
  한 워크스페이스에서 실행 표시 이름을 변경해도, 표시 이름은 해당 워크스페이스에서만 변경되며 다른 워크스페이스나 프로젝트에는 영향을 주지 않습니다.
</Note>

표시 이름의 기본값은 실행 이름과 동일합니다. 표시 이름은 실행의 워크스페이스와 실행 테이블에 표시됩니다.

실행 표시 이름을 사용하면 프로젝트에서 실행 이름 자체를 바꾸지 않고도, 해당 워크스페이스에 표시되는 실행 이름만 따로 지정할 수 있습니다.

<div id="rename-a-runs-display-name">
  ### 실행 표시 이름 바꾸기
</div>

W&amp;B App에서 실행의 표시 이름을 변경하려면:

1. W&amp;B 프로젝트로 이동합니다.
2. **Workspace** 또는 **Runs** 탭을 선택합니다.
3. 이름을 변경할 실행을 검색하거나 스크롤하여 찾습니다.
4. 실행 이름 위에 마우스를 올린 다음 세로 점 세 개 아이콘을 클릭하고 **Rename run**을 클릭합니다.
5. **Display name** 필드에 새 값을 입력합니다.
6. **Save**를 클릭합니다.

<div id="customize-run-name-truncation">
  ## 실행 이름 잘림 방식 사용자 지정
</div>

기본적으로 긴 실행 이름은 가독성을 위해 중간 부분이 잘립니다. 실행 이름의 잘림 방식을 사용자 지정하려면:

1. 실행 목록 상단의 작업 `...` 메뉴를 클릭합니다.
2. **Run name cropping**을 설정해 잘림 위치를 끝, 중간, 또는 시작으로 지정합니다.
