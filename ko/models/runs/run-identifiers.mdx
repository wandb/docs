W&amp;B Run을 초기화하면 W&amp;B가 해당 run에 [*run ID*라는 고유 식별자](/ko/models/runs/run-identifiers#run-id)를 할당합니다. 각 run에는 사용자가 지정할 수 있는, 사람이 읽기 쉬운 [고유하지 않은 *run name*](/ko/models/runs/run-identifiers#run-name)도 있습니다.

<div id="run-id">
  ## Run ID
</div>

run의 ID는 해당 run을 고유하게 식별합니다. 기본적으로 W&amp;B는 [run을 초기화](/ko/models/runs/initialize-run)할 때 [고유한 run ID를 직접 지정](#create-a-custom-run-id)하지 않는 한 [무작위로 생성된 고유한 run ID](#autogenerated-run-ids)를 자동으로 생성합니다.

<div id="find-a-runs-id">
  ### run의 ID 찾기
</div>

W&amp;B Python SDK를 사용해 프로그래밍 방식으로, 또는 W&amp;B App에서 대화형으로 run의 고유 ID를 찾을 수 있습니다.

<Tabs>
  <Tab title="Python SDK">
    run을 초기화하면 W&amp;B는 터미널에 고유한 run ID를 반환합니다. 예를 들어, 다음 코드 스니펫은 W&amp;B run을 초기화합니다.

    ```python
    import wandb
    entity = "nico"  # 사용자의 W&B entity로 바꾸세요
    project = "awesome-project" 
    with wandb.init(entity=entity, project=project) as run:
        # 여기에 코드를 작성하세요
    ```

    터미널에는 다음과 같이 표시됩니다.

    ```bash
    wandb: Syncing run earnest-sunset-1
    wandb: ⭐️ View project at https://wandb.ai/nico/awesome-project
    wandb: 🚀 View run at https://wandb.ai/nico/awesome-project/runs/1jx1ud12
    ```

    run URL의 마지막 부분(`1jx1ud12`)이 고유한 run ID입니다.
  </Tab>

  <Tab title="W&B App">
    W&amp;B App에서도 run의 고유 ID를 찾을 수 있습니다.

    1. [W&amp;B App](https://wandb.ai/home)으로 이동합니다.
    2. run을 초기화할 때 지정한 W&amp;B 프로젝트로 이동합니다.
    3. 프로젝트의 workspace에서 **Workspace** 또는 **Runs** 탭을 선택합니다.
    4. 확인하려는 run을 선택합니다.
    5. **Overview** 탭을 선택합니다.

    W&amp;B는 **Run path** 필드에 run ID를 표시합니다. Run path는 팀 이름, 프로젝트 이름, 그리고 run ID로 구성됩니다. 고유 ID는 Run path의 마지막 부분입니다.

    예를 들어, 다음 이미지에서 고유 run ID는 `9mxi1arc`입니다.

    <Frame>
      <img src="/images/runs/unique-run-id.png" alt="Run ID 위치" />
    </Frame>
  </Tab>
</Tabs>

<Tip>
  run의 고유 ID를 사용하면 W&amp;B App에서 해당 run의 Overview 페이지로 바로 이동할 수 있습니다. 다음 코드 블록은 run의 URL 경로 형식을 보여줍니다.

  ```text title="특정 run의 W&B App URL"
  https://wandb.ai/<entity>/<project>/<run-id>
  ```

  꺾쇠 괄호(`< >`)로 묶인 값은 실제 entity, 프로젝트, run ID 값으로 바꾸세요.
</Tip>

<div id="run-name">
  ## Run 이름
</div>

각 run에는 사람이 읽을 수 있지만 고유하지는 않은 run 이름이 있습니다. 기본적으로 W&amp;B는 새 run을 초기화할 때 run 이름을 지정하지 않으면 임의의 run 이름을 생성합니다. run 이름은 프로젝트의 Workspace와 [run의 **Overview** 페이지](#overview-tab) 상단에 표시됩니다.

이전 예제에 이어서, 이 run의 이름은 `glowing-shadows-8`입니다.

<Frame>
  <img src="/images/runs/unique-run-id.png" alt="Run ID 위치" />
</Frame>

[초기화할 때](/ko/models/runs/run-identifiers#create-a-custom-run-name) run 이름을 지정하거나, 나중에 [이름을 변경](/ko/models/runs/run-identifiers#rename-a-run)할 수 있습니다.

<div id="create-a-custom-run-name">
  ### 맞춤형 run 이름 만들기
</div>

[`wandb.init()`](/ko/models/ref/python/functions/init)에 `name` 매개변수를 전달해 run 이름을 지정합니다.

<div id="rename-a-run">
  ### run 이름 변경
</div>

Python SDK로 프로그래밍 방식으로 초기화한 후 또는 W&amp;B App에서 대화형으로 run 이름을 변경합니다.

<Tabs>
  <Tab title="Python SDK">
    [`wandb.Api.Run`](/ko/models/ref/python/public-api/api#method-api-run)을 사용해 W&amp;B에 로깅된 run에 액세스합니다. 이 방법은 run 이름을 업데이트하는 데 사용할 수 있는 [run 객체](/ko/models/ref/python/public-api/run#property-run-name)를 반환합니다. 변경 사항을 저장하려면 `wandb.Api.Run.update()` 방법을 호출하세요.

    꺾쇠괄호(`< >`)로 묶인 값을 자신의 값으로 바꾸세요.
  </Tab>

  <Tab title="W&B App">
    1. W&amp;B 프로젝트로 이동합니다.
    2. **Workspace** 또는 **Runs** 탭을 선택합니다.
    3. 이름을 변경하려는 run을 검색하거나 스크롤해 찾습니다.
    4. run 이름 위에 마우스를 올리고 **액션 (<Icon icon="ellipsis-vertical" iconType="solid" />)** 메뉴를 클릭한 다음 **Rename run**을 클릭합니다.
    5. run 이름을 변경하려면 **Run name** 필드를 수정합니다.
    6. **Save**를 클릭합니다.
  </Tab>
</Tabs>

<div id="run-display-name">
  ## run 표시 이름
</div>

각 run에는 workspace별로 사용자 지정할 수 있는 *run 표시 이름*도 있습니다.

<Note>
  한 workspace에서 run의 표시 이름을 변경해도 해당 workspace에서만 변경되며, 다른 workspace나 프로젝트에는 반영되지 않습니다.
</Note>

표시 이름의 기본값은 run 이름과 같습니다. 표시 이름은 run의 workspace와 Runs table에 표시됩니다.

프로젝트에서 run 이름을 바꾸지 않고도 해당 workspace에 표시되는 run 이름을 재정의하려면 run 표시 이름을 사용하세요.

<div id="rename-a-runs-display-name">
  ### run의 표시 이름 바꾸기
</div>

W&amp;B App에서 run의 표시 이름을 변경합니다:

1. W&amp;B 프로젝트로 이동합니다.
2. **Workspace** 또는 **Runs** 탭을 선택합니다.
3. 이름을 바꾸려는 run을 검색하거나 스크롤해 찾습니다.
4. run 이름 위에 마우스를 올리고 **액션 (<Icon icon="ellipsis-vertical" iconType="solid" />)** 메뉴를 클릭한 다음 **Rename run**을 클릭합니다.
5. **Display name** 필드에 새 값을 입력합니다.
6. **Save**를 클릭합니다.

<div id="customize-run-name-truncation">
  ## run 이름 잘림 방식 사용자 지정
</div>

기본적으로 긴 run 이름은 가독성을 위해 가운데가 잘립니다. run 이름이 잘리는 방식을 사용자 지정하려면 다음과 같이 하세요.

1. run 목록 상단의 **액션 (<Icon icon="ellipsis" iconType="solid" />)** 메뉴를 클릭합니다.
2. **Run name cropping**을 끝, 가운데 또는 시작 부분이 잘리도록 설정합니다.
