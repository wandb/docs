W&B Run을 초기화하면, W&B는 해당 run에 [run ID라고 알려진 고유 식별자](/models/runs/run-identifiers#run-id)를 할당합니다. 또한 각 run은 사용자가 정의할 수 있는 읽기 쉬운 [고유하지 않은 run name](/models/runs/run-identifiers#run-name)을 가집니다.

## Run ID

run ID는 해당 run을 고유하게 식별합니다. 새 run을 초기화할 때 [사용자 정의 고유 run ID를 지정](#create-a-custom-run-id)하지 않는 한, W&B는 [자동으로 랜덤하고 고유한 run ID를 생성](#autogenerated-run-ids)합니다.

### run ID 찾기

W&B Python SDK를 사용하여 프로그래밍 방식으로 또는 W&B App에서 대화형으로 run의 고유 ID를 찾을 수 있습니다.

<Tabs>
<Tab title="Python SDK">
run을 초기화하면 W&B는 터미널에 고유한 run ID를 반환합니다. 예를 들어, W&B run을 초기화하는 다음 코드조각을 살펴보세요:

```python
import wandb
entity = "nico"  # 본인의 W&B entity로 교체하세요
project = "awesome-project" 
with wandb.init(entity=entity, project=project) as run:
    # 여기에 코드를 작성하세요
```

터미널 내에서 W&B는 다음을 반환합니다:

```bash
wandb: Syncing run earnest-sunset-1
wandb: ⭐️ View project at https://wandb.ai/nico/awesome-project
wandb: 🚀 View run at https://wandb.ai/nico/awesome-project/runs/1jx1ud12
```

run URL의 마지막 부분(`1jx1ud12`)이 고유한 run ID입니다.
</Tab>
<Tab title="W&B App">
W&B App에서도 run의 고유 ID를 찾을 수 있습니다:

1. [W&B App](https://wandb.ai/home)으로 이동합니다.
2. run을 초기화할 때 지정한 W&B 프로젝트로 이동합니다.
3. 프로젝트의 워크스페이스 내에서 **Workspace** 또는 **Runs** 탭을 선택합니다.
4. 확인하려는 run을 선택합니다.
5. **Overview** 탭을 선택합니다.

W&B는 **Run path** 필드에 run ID를 표시합니다. run path는 팀 이름, 프로젝트 이름, 그리고 run ID로 구성됩니다. 고유 ID는 run path의 마지막 부분입니다.

예를 들어, 다음 이미지에서 고유 run ID는 `9mxi1arc`입니다:

</Frame>
</Tab>
</Tabs>

<Tip>
run의 고유 ID를 사용하여 W&B App의 해당 run 개요 페이지로 직접 이동하세요. 다음 코드 블록은 run에 대한 URL 경로 형식을 보여줍니다:

```text title="특정 run에 대한 W&B App URL"
https://wandb.ai/<entity>/<project>/<run-id>
```

꺽쇠괄호(`< >`)로 묶인 값들을 실제 entity, 프로젝트, run ID 값으로 교체하세요.
</Tip>

### 사용자 정의 run ID 생성

run을 초기화할 때 `id` 파라미터에 원하는 run ID를 문자열로 전달하세요:

```python 
import wandb

with wandb.init(entity="<project>", project="<project>", id="<run-id>") as run:
    # 여기에 코드를 작성하세요
```

## Run name

각 run은 사람이 읽을 수 있는 고유하지 않은 run name을 가집니다. 기본적으로 run name을 지정하지 않고 새 run을 초기화하면 W&B가 무작위로 run name을 생성합니다. run의 이름은 프로젝트의 워크스페이스와 [run의 **Overview** 페이지](#overview-tab) 상단에 나타납니다.

이전 예시를 계속해서 보면, run의 이름은 `glowing-shadows-8`입니다.

    <img src="/images/runs/unique-run-id.png" alt="Run ID 위치"  />
</Frame>

run을 [초기화할 때](/models/runs/run-identifiers#create-a-custom-run-name) 이름을 짓거나 나중에 [이름을 변경](/models/runs/run-identifiers#rename-a-run)할 수 있습니다.

### 사용자 정의 run name 생성

[`wandb.init()`](/models/ref/python/functions/init) 메소드에 `name` 파라미터를 전달하여 run의 이름을 지정하세요.

```python 
import wandb

with wandb.init(entity="<project>", project="<project>", name="<run-name>") as run:
    # 여기에 코드를 작성하세요
```

### run 이름 변경

Python SDK를 사용하여 프로그래밍 방식으로 또는 W&B App에서 대화형으로 초기화된 run의 이름을 변경할 수 있습니다.

<Tabs>
<Tab title="Python SDK">

W&B에 로그된 run에 엑세스하려면 [`wandb.Api.Run`](/models/ref/python/public-api/api#method-api-run)을 사용하세요. 이 메소드는 run name을 업데이트하는 데 사용할 수 있는 [run 오브젝트](/models/ref/python/public-api/run#property-run-name)를 반환합니다. 변경 사항을 유지하려면 `wandb.Api.Run.update()` 메소드를 호출하세요.

꺽쇠괄호(`< >`)로 묶인 값들을 실제 값으로 교체하세요.

```python
import wandb

api = wandb.Api()

# run 경로를 통해 run에 엑세스
run = api.run(path = "<entity>/<project>/<run-id>")

# 새로운 run name 지정
run.name = "<new-run-name>"
run.update()
```


</Tab>
<Tab title="W&B App">

1. 해당 W&B 프로젝트로 이동합니다.
1. **Workspace** 또는 **Runs** 탭을 선택합니다.
1. 이름을 변경하려는 run을 검색하거나 스크롤하여 찾습니다.
1. run name 위에 마우스를 올리고 세 개의 수직 점을 클릭한 다음 **Rename run**을 클릭합니다.
1. run 이름을 변경하려면 **Run name** 필드를 업데이트합니다.
1. **Save**를 클릭합니다.
</Tab>
</Tabs>

## Run display name

각 run은 개별 워크스페이스별로 사용자가 정의할 수 있는 *run display name*도 가집니다.

<Note>
한 워크스페이스에서 run의 display name을 변경하면 해당 워크스페이스에서만 변경되며, 다른 워크스페이스나 프로젝트에는 영향을 주지 않습니다.
</Note>

display name의 기본값은 run name과 동일합니다. display name은 run의 워크스페이스와 run 테이블에 표시됩니다.

프로젝트에서 run의 이름을 실제로 바꾸지 않고 해당 워크스페이스에 표시되는 이름만 덮어쓰려면 run display name을 사용하세요.

### run의 display name 변경

W&B App에서 run의 display name을 변경합니다:

1. 해당 W&B 프로젝트로 이동합니다.
1. **Workspace** 또는 **Runs** 탭을 선택합니다.
1. 이름을 변경하려는 run을 검색하거나 스크롤하여 찾습니다.
1. run name 위에 마우스를 올리고 세 개의 수직 점을 클릭한 다음 **Rename run**을 클릭합니다.
1. **Display name** 필드에 새로운 값을 지정합니다.
1. **Save**를 클릭합니다.

## run name 잘림(truncation) 설정

기본적으로 긴 run name은 가독성을 위해 중간 부분이 생략되어 표시됩니다. run name의 생략 방식을 설정하려면 다음을 수행하세요:

1. run 리스트 상단의 `...` 액션 메뉴를 클릭합니다.
1. **Run name cropping**을 끝(end), 중간(middle) 또는 시작(beginning) 부분으로 설정합니다.
