<div id="delete-runs">
  ## run 삭제
</div>

W&amp;B App 또는 Python API를 사용해 project에서 run을 삭제할 수 있습니다.

<Tabs>
  <Tab title="W&B App" value="ui">
    1. 삭제하려는 run이 포함된 project로 이동하세요.
    2. **Runs** 탭을 선택하세요.
    3. 삭제하려는 run 옆의 체크박스를 선택하세요.
    4. table 위에 있는 **Delete** 버튼(휴지통 아이콘)을 선택하세요.
    5. 표시되는 drawer에서 **Delete**를 선택하세요.

    run이 매우 많은 project의 경우, 검색창을 사용해 Regex로 삭제할 run을 필터링하거나 필터 버튼을 사용해 상태, tags 또는 기타 속성을 기준으로 run을 필터링할 수 있습니다.
  </Tab>

  <Tab title="Python" value="python">
    [`Run.delete()`](/ko/models/ref/python/public-api/run#method-run-delete)를 사용하면 프로그래밍 방식으로 run을 삭제할 수 있습니다. run과 연결된 아티팩트도 함께 제거하려면 `delete_artifacts=True`로 설정하세요.

    ```python
    import wandb

    api = wandb.Api()
    runs = api.runs("<entity>/<project>")
    for run in runs:
        if run.state == "finished":  # 사용자 조건에 맞게 바꾸세요
            run.delete(delete_artifacts=False)
    ```

    전체 메서드 시그니처와 동작은 [`Run.delete` 레퍼런스](/ko/models/ref/python/public-api/run#method-run-delete)를 참조하세요.

    로깅된 미디어처럼 run에 첨부된 개별 파일을 제거하려면 다음 단계를 따르세요.

    1. [`Run.files()`](/ko/models/ref/python/public-api/run#method-run-files)를 사용해 관련 파일 핸들을 획득하세요.
    2. [`File.delete()`](/ko/models/ref/python/public-api/file#method-file-delete)를 사용해 개별 파일을 삭제하세요.
  </Tab>
</Tabs>

run을 삭제한 후에도 run ID는 재사용할 수 없습니다. 재사용하려고 하면 해당 run은 오류와 함께 실패합니다.

<Warning>
  run을 삭제하면서 연결된 아티팩트도 삭제하도록 선택하면 해당 아티팩트는 영구적으로 제거되며, 나중에 run을 복원하더라도 복구할 수 없습니다. 여기에는 레지스트리에 연결된 아티팩트도 포함됩니다.
</Warning>

<div id="when-deleted-run-data-is-removed-from-storage">
  ## 삭제된 run 데이터가 저장소에서 제거되는 시점
</div>

[W&amp;B Dedicated Cloud](/ko/platform/hosting/hosting-options/dedicated-cloud) 및 [W&amp;B Self-Managed](/ko/platform/hosting/hosting-options/self-managed)에서는 `GORILLA_DATA_RETENTION_PERIOD` 환경 변수를 통해 **삭제된 run 데이터**를 객체 저장소에서 영구적으로 제거할 수 있게 되기 전까지 얼마나 오래 보관할지 제어합니다. **아티팩트는 이 설정의 적용을 받지 않으며**, [Delete an artifact](/ko/models/artifacts/delete-artifacts)에 설명된 아티팩트 삭제 및 가비지 컬렉션 흐름을 따릅니다.

`GORILLA_DATA_RETENTION_PERIOD`를 설정하거나 변경하면 보존 기간이 지난 데이터에는 되돌릴 수 없게 적용됩니다. 보존 기간을 활성화하거나 기간을 더 짧게 조정하기 전에 데이터베이스와 버킷을 백업하세요. 레퍼런스 표와 경고는 [Configure environment variables](/ko/platform/hosting/env-vars)를 참조하세요.

run 또는 파일을 삭제하고 보존 기간 처리까지 완료한 후에도 백그라운드 작업이 처리되는 동안 **버킷 사용량 반영이 지연될 수 있습니다**. W&amp;B는 객체 저장소 공간이 즉시 회수된다고 보장하지 않습니다. 아티팩트와 run 데이터의 차이, 예상 소요 시점, 선택적으로 수행할 수 있는 운영자 액션의 전체 개요는 [Manage bucket storage and costs](/ko/platform/hosting/managing-bucket-storage)를 참조하세요.

<Note>
  Public API를 사용할 때 W&amp;B App에서 삭제가 예상대로 표시되지 않으면 W&amp;B Python SDK를 최신 릴리스로 업그레이드한 후 다시 시도하세요.
</Note>
