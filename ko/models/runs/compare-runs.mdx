실행이 많은 워크스페이스에서는 최고의 성능을 내는 모델, 프로덕션 모델, 실패한 실험, 중요한 기준점 등을 추적하기가 어려울 수 있습니다. W&amp;B App은 실행을 정리하고 비교할 수 있도록 다음과 같은 기능을 제공합니다:

* **고정 실행(Pinned runs)**: 워크스페이스와 실행 목록 상단에 항상 표시되도록 최대 6개의 실행을 고정할 수 있습니다. 기준 실행(baseline run)이 있는 경우, 기준 실행은 자동으로 고정되므로 최대 5개의 실행만 별도로 고정할 수 있습니다.
* **기준 실행(Baseline run)**: 비교를 위한 기준점으로 사용할 기준 실행을 지정할 수 있습니다. 기준 실행은 항상 워크스페이스와 실행 목록 상단에 표시됩니다. 선형 플롯(line plot)에서는 비교를 돕기 위해 기준 실행이 시각적으로 구분되는 스타일로 표시됩니다.

다음과 같은 경우에 특히 유용합니다:

* 새로운 실험을 프로덕션 모델과 비교할 때.
* 실험 중 여러 후보 모델을 추적할 때.
* 새로운 실행이 기존 최고 성능을 갱신했는지 평가할 때.

<div id="manage-the-baseline-run">
  ## 기준 실행 관리하기
</div>

워크스페이스에서 하나의 실행을 기준 실행으로 지정해, 워크스페이스의 다른 실행을 평가할 때 기준점으로 사용할 수 있습니다.

실행 선택기와 실행 테이블에서 기준 실행은 고정된 실행과 함께 맨 위에 표시되며, 핀 아이콘 대신 북마크 아이콘이 표시됩니다.

선형 플롯에서는 기준 실행의 선이 다른 선보다 더 두껍게 표시됩니다. 플롯이나 범례 위에 마우스를 올리면 기준 실행의 선이 점선으로 표시됩니다.

<Frame>
  ![기준 실행과 다른 실행을 비교하는 데모](/images/models/pinned-and-baseline-runs/line-plot-baseline-comparison.png)
</Frame>

<div id="set-a-baseline-run">
  ### 기준 실행 설정
</div>

기준 실행을 설정하려면:

1. 워크스페이스로 이동합니다.
2. 실행 선택기 또는 실행 테이블에서 기준 실행으로 사용할 실행을 찾습니다.
3. 작업 `...` 메뉴를 클릭한 다음 **Set as baseline**을 선택합니다.

기준 실행은 다른 실행과 시각적으로 구분되는 구분선과 함께 실행 선택기 맨 위에 표시됩니다. 기준 실행에는 원 대신 북마크 아이콘이 표시됩니다.

<Frame>
  ![기준 실행과 고정된 실행이 있는 실행 테이블](/images/models/pinned-and-baseline-runs/runs-table-with-pinned-and-baseline-runs.png)
</Frame>

<div id="change-the-baseline-run">
  ### 기준 실행 변경
</div>

기준 실행은 한 번에 하나만 지정할 수 있습니다. 기준 실행을 변경하려면:

1. 워크스페이스로 이동합니다.
2. 실행 선택기 또는 실행 테이블에서 새 기준으로 지정할 실행을 찾습니다.
3. 동작 `...` 메뉴를 클릭한 다음 **Replace baseline**을 선택합니다. <Note>메뉴 항목이 비활성화되어 있다면, 사용 가능한 고정 슬롯이 최소 한 개 이상 남아 있는지 확인하세요. 필요한 경우, 이미 고정된 실행 옆의 원형 핀 아이콘을 클릭해 해당 실행의 고정을 해제하세요.</Note>
4. 새 실행이 기준 실행이 되며, 이전 기준 실행은 나중에 쉽게 찾을 수 있도록 자동으로 고정됩니다. 필요하다면 핀 아이콘을 클릭해 고정을 해제하세요.

<div id="remove-the-baseline-designation">
  ### 기준 실행 지정 해제
</div>

기준 실행 지정을 해제하려면 다음을 수행합니다:

1. 워크스페이스로 이동합니다.
2. 실행 선택기 또는 실행 테이블에서 현재 기준 실행을 찾습니다.
3. 동작 `...` 메뉴를 클릭한 다음 **기준 실행 해제**를 선택합니다. <Note>메뉴 항목이 비활성화되어 있다면, 사용 가능한 고정 슬롯이 하나 이상 있는지 확인하세요. 필요한 경우, 고정된 실행 옆의 원형 핀 아이콘을 클릭해 해당 실행의 고정을 해제하세요.</Note>
4. 이전 기준 실행은 나중에 쉽게 찾을 수 있도록 자동으로 고정됩니다. 필요하면 해당 실행의 핀 아이콘을 클릭해 고정을 해제하세요.

<div id="compare-runs-to-the-baseline">
  ## 기준 실행과 다른 실행 비교
</div>

기준 실행은 해당 실행에서 로깅한 지표의 선 그래프에 항상 표시됩니다. 선 그래프에서 기준 실행의 선은 다른 선보다 더 두껍게 표시됩니다.

* 플롯의 일부에 마우스를 올리면 기준 실행과 고정된 실행을 포함해 현재 표시된 모든 실행의 값이 툴팁으로 표시됩니다.
  <Frame>
    ![특정 지점에서 보이는 모든 실행의 세부 정보를 보여주는 데모](/images/models/pinned-and-baseline-runs/line-plot-point-details.png)
  </Frame>
* 기준 실행의 범례 레이블 위에 마우스를 올리면 해당 선이 두드러지게 표시됩니다. 굵은 점선으로 나타납니다. 다른 표시된 실행의 선은 채도를 낮춰 표시됩니다.
  <Frame>
    ![기준 실행의 세부 정보를 보여주는 데모](/images/models/pinned-and-baseline-runs/line-plot-baseline-run-foreground.png)
  </Frame>
* 다른 실행의 범례 레이블 위에 마우스를 올리면 해당 실행의 선이 두드러지게 표시되고, 굵은 점선으로 나타나는 기준 실행과 비교할 수 있습니다. 다른 표시된 실행의 선은 채도를 낮춰 표시됩니다.

  </Frame>

<div id="use-cases">
  ## 사용 사례
</div>

이 섹션에서는 고정 실행(pinned 실행)과 기준 실행(baseline 실행)이 실험을 진행하는 데 도움이 되는 몇 가지 시나리오를 설명합니다.

* **프로덕션 모델 추적**: 새 모델이 배포 전에 품질 기준을 충족하는지 확인합니다.
  1. 프로덕션 모델을 기준 실행으로 설정합니다.
  2. 프로덕션보다 성능이 뛰어난 후보를 식별하기 위해 모든 실험을 배포된 모델과 비교합니다.
* **하이퍼파라미터 실험 비교**: 하이퍼파라미터 스윕 또는 수동 실험을 현재까지 파악된 최상의 설정과 비교해 평가합니다.
  1. 현재까지 파악된 최상의 설정을 기준 실행으로 설정합니다.
  2. 유망한 후보를 발견할 때마다 고정합니다.
  3. 선 그래프(line plot)를 사용해 실행들을 기준 실행과 시각적으로 비교합니다.
  4. 더 나은 설정을 찾을 때마다 기준 실행을 계속 업데이트합니다.

<div id="example-workflow">
  ## 예시 워크플로우
</div>

이 섹션에서는 고정 실행과 기준 실행이 다른 실행을 비교하는 데 어떻게 도움이 되는지 보여줍니다.

1. 일련의 실행으로 하이퍼파라미터 튜닝 시나리오를 시뮬레이션하는 다음 예제 코드를 실행합니다. 꺾쇠 괄호(`<>`)로 둘러싸인 플레이스홀더를 자신의 값으로 바꾸세요.

   ```python
   import wandb
   import random
   import math

   def train_model(learning_rate, batch_size, run_name, tags=None):
       """주어진 하이퍼파라미터로 모델을 학습하는 것을 시뮬레이션합니다."""
       config = {
           "learning_rate": learning_rate,
           "batch_size": batch_size,
           "optimizer": "adam",
           "architecture": "resnet50"
       }
       
       with wandb.init(
         # 여기를 팀 및 프로젝트 이름으로 바꾸세요
           project="hyperparameter-tuning",
           entity="<team>",
           name=run_name,
           config=config,
           tags=tags or []
       ) as run:
           # 학습 루프 시뮬레이션
           for epoch in range(50):
               # 시뮬레이션된 지표
               accuracy = 0.6 + 0.3 * (1 - math.exp(-learning_rate * epoch / 10))
               loss = 1.0 * math.exp(-learning_rate * epoch / 10)
               
               run.log({
                   "epoch": epoch,
                   "accuracy": accuracy,
                   "loss": loss
               })

   # 표준 설정으로 기준 실행 생성
   train_model(
       learning_rate=0.001,
       batch_size=64,
       run_name="baseline-config",
       tags=["baseline", "production"]
   )

   # 서로 다른 learning rate로 실험
   train_model(
       learning_rate=0.003,
       batch_size=64,
       run_name="lr-experiment-0.003",
       tags=["experiment"]
   )

   train_model(
       learning_rate=0.0001,
       batch_size=64,
       run_name="lr-experiment-0.0001",
       tags=["experiment"]
   )
   ```

   이 코드를 실행한 후 워크스페이스에는 세 개의 실행이 있게 됩니다.
2. `baseline-config`를 기준 실행으로 설정합니다.
3. `baseline-config`를 고정해서 항상 보이도록 합니다.
4. 워크스페이스의 선 그래프를 사용해 실험 실행을 기준 실행과 비교합니다.
5. 추가 분석이 필요한 유망한 실험을 고정합니다. 이 예제에서는 50 에포크 후에 `lr-experiment-0.003`이 가장 높은 정확도(`~0.64`)와 가장 낮은 손실(`~0.86`)을 보입니다.

<div id="limitations">
  ## 제한 사항
</div>

<CloudOnly />

다음 기능은 아직 고정 실행과 기준 실행에 대해 지원되지 않습니다:

* **그룹화**: 실행 선택기 또는 실행 테이블에서 [실행을 볼 때](/ko/models/runs#view-logged-runs) 실행이 열 기준으로 그룹화된 경우, 고정 실행과 기준 실행은 다른 실행과 시각적으로 구분되지 않습니다.
* **리포트**: [W&amp;B Report](/ko/models/reports)의 실행 집합에서 고정 실행과 기준 실행은 다른 실행과 시각적으로 구분되지 않습니다.
* **워크스페이스 보기에서만**: 단일 실행의 워크스페이스를 볼 때 기준 실행은 표시되지 않습니다.
* **선형 플롯만 해당**: 기준 실행 비교는 선형 플롯에서만 사용할 수 있으며, 아직 막대 플롯이나 미디어 패널과 같은 다른 패널에서는 사용할 수 없습니다.
