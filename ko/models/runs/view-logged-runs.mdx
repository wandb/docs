description: W&B App 또는 LEET 터미널 UI를 사용하여 로깅된 특정 run과 해당 속성을 보는 방법을 알아보세요.
---

현재 상태, 아티팩트, 메트릭 등 특정 run에 대한 정보를 확인하세요. W&amp;B App 또는 `wandb beta leet` 터미널 UI를 사용해 run을 확인하고 모니터링할 수 있습니다.

<Tabs>
  <Tab title="WA&B App">
    W&amp;B App에서 특정 run을 보려면 다음 단계를 따르세요.

    1. [W&amp;B App](https://wandb.ai/home)으로 이동합니다.
    2. run을 초기화할 때 지정한 W&amp;B 프로젝트로 이동합니다.
    3. 프로젝트 사이드바에서 **Workspace** 탭을 선택합니다.
    4. run selector에서 보려는 run을 클릭하거나, run 이름의 일부를 입력해 일치하는 run만 필터링합니다.

    또는 브라우저에 URL을 입력해 특정 run의 워크스페이스에 직접 액세스할 수도 있습니다. 특정 run의 URL 경로는 다음 형식을 따릅니다.

    ```text
    https://wandb.ai/<team-name>/<project-name>/runs/<run-id>
    ```

    꺾쇠 괄호(`< >`)로 묶인 값은 팀 이름, 프로젝트 이름, [run ID](/ko/models/runs/run-identifiers#run-id)의 실제 값으로 바꾸십시오.

    [Overview](/ko/models/runs/view-logged-runs#overview), [Logs](/ko/models/runs/view-logged-runs#logs), [Files](/ko/models/runs/view-logged-runs#files), [Code](/ko/models/runs/view-logged-runs#code), [Artifacts](/ko/models/runs/view-logged-runs#artifacts) 탭으로 이동하며 run의 속성을 탐색하세요.
  </Tab>

  <Tab title="LEET">
    `wandb beta leet` 터미널 UI를 사용해 터미널에서 로컬 run을 보려면 다음 단계를 따르세요.

    1. 스크립트에서 로컬로 run을 시작했다면, 코드를 실행한 디렉터리로 이동합니다. 이 디렉터리에는 run마다 하나의 하위 디렉터리가 있는 `wandb/` 디렉터리와 `latest-run/` 심볼릭 링크가 있습니다. 각 run 디렉터리에는 `run-<run-ID>.wandb` 형식의 이름을 가진 트랜잭션 로그가 들어 있습니다.

       run을 로컬에서 시작하지 않고 `.wandb` 트랜잭션 로그 파일만 다운로드했다면, 해당 파일의 위치를 메모해 두십시오.
    2. 다음 명령 중 하나를 사용하여 `wandb beta leet`를 시작합니다.

       ```bash
       # ./wandb/latest-run/에 저장된 최신 run 보기
       wandb beta leet

       # run 디렉터리 지정
       wandb beta leet ./wandb/run-20250813_124246-n67z9ude

       # .wandb 파일 지정
       wandb beta leet ./wandb/run-20250813_124246-n67z9ude/run-n67z9ude.wandb
       ```

    ![LEET 터미널 UI에서 run을 보는 방법을 보여주는 스크린샷](/images/leet/leet-tui.png)

    LEET는 3개 패널로 구성된 인터페이스를 표시합니다.

    * **왼쪽 패널**: 환경 변수, 설정, 요약 통계를 포함한 run 개요
    * **가운데 패널**: 로깅된 메트릭을 점자 스타일의 선형 차트로 보여주는 메트릭 그리드
    * **오른쪽 패널**: GPU/CPU/RAM 사용량을 포함한 시스템 메트릭

    다음 키보드 단축키로 시작해 보세요.

    * `h` 또는 `?` - 모든 키보드 단축키 보기
    * `/` - 패턴으로 메트릭 필터링
    * `[` / `]` - 좌/우 패널 표시/숨기기
    * `n` / `N` - 메트릭 페이지 간 이동
    * `q` / `CMD+C` - 종료

    자세한 내용은 [`wandb beta leet`](/ko/models/ref/cli/wandb-beta/wandb-beta-leet) 레퍼런스를 참조하십시오.
  </Tab>
</Tabs>

<div id="overview">
  ## Overview
</div>

프로젝트의 특정 run 정보(예: 아래 항목)를 확인하려면 **Overview** 탭을 사용하세요.

* **Author**: run을 생성한 W&amp;B entity입니다.
* **Command**: run을 초기화하는 명령입니다.
* **Description**: 사용자가 제공한 run 설명입니다. run을 생성할 때 설명을 지정하지 않으면 이 필드는 비어 있습니다. W&amp;B App 또는 Python SDK를 사용해 프로그래밍 방식으로 run에 설명을 추가할 수 있습니다.
* **Tracked Hours**: 일시 중지되거나 대기한 시간을 제외하고, run이 실제로 연산을 수행하거나 데이터를 로깅한 시간입니다. 이 메트릭을 통해 run에 실제로 소요된 연산 시간을 파악할 수 있습니다.
* **Runtime**: run 시작부터 종료까지의 총 시간을 측정합니다. run이 일시 중지되거나 리소스를 기다리는 시간까지 포함한 경과 시간(벽시계 기준)입니다. 이 메트릭은 run의 전체 경과 시간을 보여줍니다.
* **Git repository**: run과 연결된 git 저장소입니다. 이 필드를 보려면 [git 활성화](/ko/platform/app/settings-page/user-settings/#personal-github-integration)를 해야 합니다.
* **Host name**: W&amp;B가 run을 연산하는 위치입니다. 로컬 머신에서 run을 초기화하면 해당 머신 이름이 표시됩니다.
* **Name**: run의 이름입니다.
* **OS**: run을 초기화한 운영 체제입니다.
* **Python executable**: run을 시작하는 명령입니다.
* **Python version**: run을 생성한 Python 버전입니다.
* **Run path**: `entity/project/run-ID` 형식의 고유한 run 식별자입니다.
* **Start time**: run을 초기화한 시각의 타임스탬프입니다.
* **State**: [run의 상태](/ko/models/runs/run-states)입니다.
* **System hardware**: W&amp;B가 run을 연산하는 데 사용하는 하드웨어입니다.
* **Tags**: 문자열 목록입니다. 태그는 관련 run을 함께 정리하거나 `baseline`, `production`과 같은 임시 레이블을 적용할 때 유용합니다.
* **W&amp;B CLI version**: run 명령을 호스팅한 머신에 설치된 W&amp;B CLI 버전입니다.
* **Git state**: run이 초기화된 저장소 또는 작업 디렉터리의 가장 최근 git 커밋 SHA입니다. run을 생성할 때 Git을 활성화하지 않았거나 git 정보를 사용할 수 없으면 이 필드는 비어 있습니다.

W&amp;B는 Overview 섹션 아래에 다음 정보를 저장합니다.

* **Artifact Outputs**: run에서 생성된 아티팩트 출력입니다.
* **Config**: [`wandb.Run.config`](/ko/models/track/config/)로 저장된 설정 파라미터 목록입니다.
* **Summary**: [`wandb.Run.log()`](/ko/models/track/log/)로 저장된 summary 파라미터 목록입니다. 기본적으로 W&amp;B는 이 값을 마지막으로 로깅된 값으로 설정합니다.

특정 파라미터를 필터링하려면 Config 및 Summary 섹션 위의 검색 상자를 사용할 수 있습니다. 예를 들어 검색 상자에 `acc`를 입력하면 W&amp;B는 이름에 `acc`가 포함된 모든 파라미터(예: `accuracy`, `val_acc`)를 필터링합니다.

<Note>
  W&amp;B는 중첩된 config 또는 summary 파라미터는 검색하지 않습니다. 예를 들어 `wandb.Run.config.update({"model": {"learning_rate": 0.01}})`로 중첩된 config 파라미터를 로깅한 경우, 검색 상자에서 `learning_rate`를 검색해도 W&amp;B는 `learning_rate` 파라미터를 반환하지 않습니다. `learning_rate` 파라미터는 `model`을 검색해야만 찾을 수 있습니다.
</Note>

예시 프로젝트 Overview는 [여기](https://wandb.ai/stacey/deep-drive/overview)에서 볼 수 있습니다.

<div id="code">
  ## Code
</div>

**Code** 탭에는 특정 run에 **연결된** 코드 파일이 표시됩니다. 여기에는 실행된 메인 스크립트뿐 아니라 run 환경의 일부였던 추가 코드 파일도 포함됩니다.
