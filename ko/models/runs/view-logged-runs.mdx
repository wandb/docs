특정 run의 현재 상태, 아티팩트, 메트릭 등을 확인할 수 있습니다.

<div id="view-a-logged-run">
  ## 기록된 run 보기
</div>

W&amp;B App 또는 `wandb beta leet` 터미널 UI를 사용하여 run을 확인하고 모니터링할 수 있습니다.

<Tabs>
  <Tab title="WA&B App">
    W&amp;B App에서 특정 run을 보려면:

    1. [W&amp;B App](https://wandb.ai/home)으로 이동합니다.
    2. run을 초기화할 때 지정한 W&amp;B 프로젝트로 이동합니다.
    3. 프로젝트 사이드바에서 **Workspace** 탭을 선택합니다.
    4. run 선택기에서 보고 싶은 run을 클릭하거나, run 이름의 일부를 입력하여 일치하는 run만 필터링합니다.

    또는 브라우저에서 URL을 입력해 특정 run의 워크스페이스에 직접 접근할 수도 있습니다. 특정 run의 URL 경로 형식은 다음과 같습니다.

    ```text
    https://wandb.ai/<team-name>/<project-name>/runs/<run-id>
    ```

    꺾쇠 괄호(`< >`)로 둘러싸인 값을 팀 이름, 프로젝트 이름, [run ID](/ko/models/runs/run-identifiers#run-id)의 실제 값으로 바꾸십시오.

    [Overview](/ko/models/runs/view-logged-runs#overview), [Logs](/ko/models/runs/view-logged-runs#logs), [Files](/ko/models/runs/view-logged-runs#files), [Code](/ko/models/runs/view-logged-runs#code), [Artifacts](/ko/models/runs/view-logged-runs#artifacts) 탭을 이동하며 run의 속성을 탐색합니다.
  </Tab>

  <Tab title="LEET">
    `wandb beta leet` 터미널 UI를 사용하여 터미널에서 로컬 run을 보려면:

    1. 스크립트에서 로컬로 run을 시작했다면, 코드를 실행한 디렉터리로 이동합니다. 이 디렉터리에는 run마다 하나의 하위 디렉터리가 있는 `wandb/` 디렉터리와 `latest-run/` 심볼릭 링크가 있습니다. 각 run 디렉터리에는 `run-<run-ID>.wandb` 형식의 이름을 가진 트랜잭션 로그가 들어 있습니다.

       run을 로컬에서 시작하지 않고 `.wandb` 트랜잭션 로그 파일만 다운로드했다면, 해당 파일의 위치를 확인해 두십시오.
    2. 다음 명령 중 하나를 사용하여 `wandb beta leet`를 시작합니다:

       ```bash
       # ./wandb/latest-run/에 저장된 최신 run 보기
       wandb beta leet

       # run 디렉터리 지정하기
       wandb beta leet ./wandb/run-20250813_124246-n67z9ude

       # .wandb 파일 지정하기
       wandb beta leet ./wandb/run-20250813_124246-n67z9ude/run-n67z9ude.wandb
       ```

    ![LEET 터미널 UI에서 run을 보는 방법을 보여주는 스크린샷](/images/leet/leet-tui.png)

    LEET는 세 개의 패널로 구성된 인터페이스를 표시합니다:

    * **왼쪽 패널**: 환경 변수, 설정, 요약 통계를 포함한 run 개요
    * **가운데 패널**: 기록된 메트릭을 점자 스타일의 선형 차트로 보여주는 메트릭 그리드
    * **오른쪽 패널**: GPU/CPU/RAM 사용률을 포함한 시스템 메트릭

    다음 키보드 단축키로 시작해 보십시오:

    * `h` 또는 `?` - 모든 키보드 단축키 보기
    * `/` - 패턴으로 메트릭 필터링
    * `[` / `]` - 좌/우 패널 토글
    * `n` / `N` - 메트릭 페이지 사이 이동
    * `q` / `CMD+C` - 종료

    자세한 내용은 [`wandb beta leet`](/ko/models/ref/cli/wandb-beta/wandb-beta-leet) 참고 문서를 확인하십시오.
  </Tab>
</Tabs>

<div id="overview">
  ### 개요
</div>

**Overview** 탭을 사용해서 프로젝트 내 특정 run 정보(예: 아래 항목들)를 확인할 수 있습니다:

* **Author**: run을 생성한 W&amp;B Entity입니다.
* **Command**: run을 초기화한 명령입니다.
* **Description**: 사용자가 제공한 run 설명입니다. run을 생성할 때 설명을 지정하지 않으면 이 필드는 비어 있습니다. W&amp;B App 또는 Python SDK를 사용해 코드로 run에 설명을 추가할 수 있습니다.
* **Tracked Hours**: 일시 중지나 대기 시간을 제외하고, run이 실제로 연산을 수행하거나 데이터를 로깅한 시간입니다. 이 메트릭을 통해 run에 소요된 실제 연산 시간을 파악할 수 있습니다.
* **Runtime**: run 시작부터 종료까지의 전체 시간을 측정합니다. run이 일시 중지되거나 리소스를 기다리는 시간을 포함한 경과 시간(벽시계 기준)입니다. 이 메트릭은 run의 전체 경과 시간을 보여줍니다.
* **Git repository**: run과 연결된 git 리포지토리입니다. 이 필드를 보려면 [git 활성화](/ko/platform/app/settings-page/user-settings/#personal-github-integration)를 해야 합니다.
* **Host name**: W&amp;B가 run을 실행한 호스트 이름입니다. 로컬 머신에서 run을 초기화하면 해당 머신 이름이 표시됩니다.
* **Name**: run의 이름입니다.
* **OS**: run을 초기화한 운영 체제입니다.
* **Python executable**: run을 시작한 명령입니다.
* **Python version**: run을 생성한 Python 버전입니다.
* **Run path**: `entity/project/run-ID` 형식의 고유한 run 식별자입니다.
* **Start time**: run을 초기화한 시각의 타임스탬프입니다.
* **State**: [run의 상태](/ko/models/runs/run-states)입니다.
* **System hardware**: W&amp;B가 run을 실행하는 데 사용하는 하드웨어입니다.
* **Tags**: 문자열 목록입니다. 태그는 관련 run을 함께 정리하거나 `baseline`, `production`과 같은 임시 레이블을 적용할 때 유용합니다.
* **W&amp;B CLI version**: run 명령을 실행한 머신에 설치된 W&amp;B CLI 버전입니다.
* **Git state**: run이 초기화된 리포지토리 또는 작업 디렉터리의 최신 git 커밋 SHA입니다. run을 생성할 때 Git을 활성화하지 않았거나 git 정보를 사용할 수 없으면 이 필드는 비어 있습니다.

W&amp;B는 개요 섹션 아래에 다음 정보를 저장합니다:

* **Artifact Outputs**: run에서 생성된 아티팩트 출력입니다.
* **Config**: [`wandb.Run.config`](/ko/models/track/config/)로 저장된 구성 파라미터 목록입니다.
* **Summary**: [`wandb.Run.log()`](/ko/models/track/log/)로 저장된 요약 파라미터 목록입니다. 기본적으로 W&amp;B는 이 값을 마지막으로 로깅된 값으로 설정합니다.

예시 프로젝트 개요는 [여기](https://wandb.ai/stacey/deep-drive/overview)에서 볼 수 있습니다.

<div id="logs">
  ### 로그
</div>

**Log** 탭에는 표준 출력(`stdout`) 및 표준 에러(`stderr`)와 같이 명령줄에 출력된 내용이 표시됩니다.

  <img src="/images/runs/run_logs_info.png" alt="Run logs tab" />
</Frame>

오른쪽 상단에 있는 **Download** 버튼을 클릭하여 로그 파일을 다운로드합니다.

로그 탭 예시는 [여기](https://app.wandb.ai/stacey/deep-drive/runs/pr0os44x/logs)에서 확인할 수 있습니다.

<div id="files">
  ### 파일
</div>

특정 run과 연관된 파일(모델 체크포인트, 검증 세트 예제 등)을 보려면 **Files** 탭을 사용하세요.

<Frame>
  <img src="/images/runs/runs_files_info.png" alt="Run files tab" />
</Frame>

**Files** 탭 예시는 [여기](https://app.wandb.ai/stacey/deep-drive/runs/pr0os44x/files/media/images)에서 확인할 수 있습니다.

<div id="code">
  ### 코드
</div>

**코드** 탭에는 특정 run과 관련된 코드 파일이 표시됩니다. 여기에는 실행된 메인 스크립트뿐 아니라 run의 환경을 구성하는 데 사용된 추가 코드 파일들도 포함됩니다.

<div id="artifacts">
  ### Artifacts
</div>

**Artifacts** 탭에는 지정된 run의 입력 및 출력 [아티팩트](/ko/models/artifacts/)가 표시됩니다.
