<div id="how-it-works">
  ## 작동 방식
</div>

몇 줄의 코드로 머신러닝 실험을 추적하세요:

1. [W&amp;B Run](/ko/models/runs/)을 생성합니다.
2. 학습률이나 모델 유형과 같은 하이퍼파라미터 딕셔너리를 설정([`wandb.Run.config`](/ko/models/track/config/))에 저장합니다.
3. 정확도와 손실 같은 메트릭을 트레이닝 루프에서 시간에 따라 로깅합니다([`wandb.Run.log()`](/ko/models/track/log/)).
4. 모델 가중치나 예측 테이블처럼 run의 출력을 저장합니다.

다음 코드는 일반적인 W&amp;B 실험 추적 워크플로를 보여줍니다:

```python
# run을 시작합니다.
#
# 이 블록이 종료되면, 로깅된 데이터가 업로드를 완료할 때까지 기다립니다.
# 예외가 발생하면 run은 실패로 표시됩니다.
with wandb.init(entity="", project="my-project-name") as run:
  # 모드 입력값과 하이퍼파라미터를 저장합니다.
  run.config.learning_rate = 0.01

  # 실험 코드를 실행합니다.
  for epoch in range(num_epochs):
    # 트레이닝을 수행합니다...

    # 시간 경과에 따른 메트릭을 로깅하여 모델 성능을 시각화합니다.
    run.log({"loss": loss})

  # 모델 출력을 아티팩트로 업로드합니다.
  run.log_artifact(model)
```

<div id="get-started">
  ## 시작하기
</div>

사용 사례에 따라 W&amp;B Experiments를 시작할 때 도움이 되는 다음 리소스를 살펴보세요:

* 데이터셋 아티팩트를 생성, 추적, 사용하는 데 활용할 수 있는 W&amp;B Python SDK 명령어의 단계별 개요는 [W&amp;B 퀵스타트](/ko/models/quickstart/)에서 조회하세요.
* 다음 방법을 알아보려면 이 장을 살펴보세요:
  * 실험 생성
  * 실험 설정
  * 실험에서 데이터 로깅
  * 실험 결과 보기
* [W&amp;B API 레퍼런스 가이드](/ko/models/ref/python/)에서 [W&amp;B Python Library](/ko/models/ref/python/)를 살펴보세요.
