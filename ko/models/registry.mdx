W&amp;B Registry는 조직 내 [W&amp;B Artifact 버전](/ko/models/artifacts/create-a-new-artifact-version)을 선별해 모아두는 중앙 저장소입니다. 조직에서 [권한을 부여받은](/ko/models/registry/configure_registry/) 사용자는 팀 소속과 관계없이 조직의 모든 아티팩트에 대해 [아티팩트를 다운로드하고 사용](/ko/models/registry/download_use_artifact/)하고, 공유하며, 수명주기를 함께 관리할 수 있습니다.

W&amp;B Registry를 사용해 아티팩트 버전을 추적하고, 아티팩트 사용 및 변경 이력을 감사하며, 아티팩트의 거버넌스와 컴플라이언스를 보장하고, [모델 CI/CD와 같은 다운스트림 프로세스를 자동화](/ko/models/automations/)하십시오.

정리하면, W&amp;B Registry를 사용하면 다음을 수행할 수 있습니다:

* 머신 러닝 작업을 충족하는 아티팩트 버전을 조직 내 다른 사용자에게 [승격](/ko/models/registry/link_version/)할 수 있습니다.
* [태그를 사용해 아티팩트를 구성](/ko/models/registry/organize-with-tags/)하여 특정 아티팩트를 찾거나 참조할 수 있습니다.
* [아티팩트의 계보](/ko/models/registry/lineage/)를 추적하고 변경 이력을 감사할 수 있습니다.
* 모델 CI/CD와 같은 [다운스트림 프로세스를 자동화](/ko/models/automations/)할 수 있습니다.
* 각 레지스트리에서 조직 내 [어떤 사용자가 아티팩트에 접근할 수 있는지](/ko/models/registry/configure_registry/)를 관리할 수 있습니다.

<div id="learn-the-basics">
  ## 기본 사항 알아보기
</div>

각 organization에는 초기 상태에서 **Models**와 **Datasets**라는 두 개의 registry가 있으며, 이를 사용하여 각각 모델과 데이터셋 아티팩트를 정리할 수 있습니다. 조직의 요구 사항에 맞게 [다른 아티팩트 유형을 관리하기 위한 추가 registry를 생성](/ko/models/registry/create_registry)할 수 있습니다.

각 [*registry*](/ko/models/registry/configure_registry/)는 하나 이상의 [*collection*](/ko/models/registry/create_collection/)으로 구성됩니다. 각 collection은 고유한 작업 또는 사용 사례를 나타냅니다.

{/* <Frame>
    <img src="/images/registry/homepage_registry.png" alt="W&B Registry"  />
  </Frame> */}

레지스트리에 아티팩트를 추가하려면 먼저 [특정 아티팩트 버전을 W&amp;B에 로깅](/ko/models/artifacts/create-a-new-artifact-version/)해야 합니다. 아티팩트를 로깅할 때마다 W&amp;B는 해당 아티팩트에 자동으로 버전을 할당합니다. 아티팩트 버전은 0부터 시작하는 인덱스를 사용하므로 첫 번째 버전은 `v0`, 두 번째 버전은 `v1`입니다.

아티팩트를 W&amp;B에 로깅한 후, 해당 아티팩트의 특정 버전을 레지스트리의 컬렉션에 연결할 수 있습니다.

<Note>
  &quot;link&quot;라는 용어는 W&amp;B가 아티팩트를 저장하는 위치와 레지스트리에서 아티팩트에 접근할 수 있는 위치를 연결하는 포인터를 의미합니다. 컬렉션에 아티팩트를 링크해도 W&amp;B는 아티팩트를 중복 저장하지 않습니다.
</Note>

예를 들어, 아래 코드 예시는 `"my_model.txt"`라는 이름의 모델 아티팩트를 `"model"`이라는 레지스트리 내 `"first-collection"`이라는 컬렉션에 로깅하고 링크합니다:

1. `wandb.init()`으로 W&amp;B 실행을 초기화합니다.
2. `wandb.Run.log()`으로 아티팩트를 W&amp;B에 로깅합니다.
3. 아티팩트 버전을 링크할 컬렉션과 레지스트리 이름을 지정합니다.
4. `wandb.Run.link_artifact()`을 사용해 아티팩트를 컬렉션에 링크합니다.

다음 Python 코드를 스크립트로 저장한 후 실행합니다. W&amp;B Python SDK 버전 0.18.6 이상이 필요합니다.

```python title="hello_collection.py"
import wandb
import random

# 아티팩트를 추적하기 위해 W&B 실행을 초기화합니다
with wandb.init(project="registry_quickstart") as run:
    # 로깅할 시뮬레이션 모델 파일을 생성합니다
    with open("my_model.txt", "w") as f:
        f.write("Model: " + str(random.random()))

    # W&B에 아티팩트를 로깅합니다
    logged_artifact = run.log_artifact(
        artifact_or_path="./my_model.txt", 
        name="gemma-finetuned", 
        type="model" # 아티팩트 유형을 지정합니다
    )

    # 아티팩트를 게시할 컬렉션과 레지스트리
    # 이름을 지정합니다
    COLLECTION_NAME = "first-collection"
    REGISTRY_NAME = "model"

    # 아티팩트를 레지스트리에 연결합니다
    run.link_artifact(
        artifact=logged_artifact, 
        target_path=f"wandb-registry-{REGISTRY_NAME}/{COLLECTION_NAME}"
    )
```

반환된 실행 객체의 `wandb.Run.link_artifact(target_path = "")` 메서드에서 지정한 컬렉션이 지정한 Registry 내에 존재하지 않으면, W&amp;B가 해당 컬렉션을 자동으로 생성합니다.

이전 예시에서 이어서, 스크립트를 실행한 후 W&amp;B Registry로 이동하여 조직 구성원들이 게시한 아티팩트 버전을 확인합니다. 왼쪽 사이드바에서 **Applications** 아래의 **Registry**를 선택합니다. `"Model"` Registry를 선택합니다. 해당 Registry 안에서, 연결된 아티팩트 버전이 포함된 `"first-collection"` 컬렉션을 확인할 수 있습니다.

Registry 내 컬렉션에 아티팩트 버전을 연결하면, 조직 구성원들은 적절한 권한이 있는 경우 아티팩트 버전을 [보기](/ko/models/registry/lineage), [다운로드](/ko/models/registry/search_registry), [정리](/ko/models/registry/organize-with-tags) 및 관리하고, 후속(다운스트림) 자동화를 생성하는 등 다양한 작업을 수행할 수 있습니다.

<Note>
  아티팩트 버전이 (`wandb.Run.log_artifact()`를 사용하는 등으로) 메트릭을 로깅하는 경우, 해당 버전의 상세 페이지에서 메트릭을 확인할 수 있으며, 컬렉션 페이지에서 아티팩트 버전 간 메트릭을 비교할 수 있습니다. [Registry에서 링크된 아티팩트 보기](/ko/models/registry/link_version/#view-linked-artifacts-in-a-registry)를 참고하세요.
</Note>

<div id="enable-wb-registry">
  ## W&amp;B Registry 활성화
</div>

배포 유형에 따라 W&amp;B Registry를 활성화하려면 다음 사항을 확인하세요:

| Deployment type | How to enable |
| ----- | ----- |
| Multi-tenant Cloud | 별도의 작업이 필요하지 않습니다. W&amp;B Registry는 W&amp;B 앱에서 바로 사용할 수 있습니다. |
| Dedicated Cloud | 계정 담당 팀에 연락하여 사용 중인 배포 환경에서 W&amp;B Registry를 활성화하세요. |
| Self-Managed | Server v0.70.0 이상에서는 별도의 작업이 필요하지 않습니다. 지원되는 이전 Server 버전의 경우, 환경 변수 `ENABLE_REGISTRY_UI`를 `true`로 설정하세요. [환경 변수 구성](/ko/platform/hosting/env-vars)을 참조하세요. |
