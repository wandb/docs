W&amp;B Registry는 조직 내 [W&amp;B Artifact 버전](/ko/models/artifacts/create-a-new-artifact-version)을 선별해 중앙에서 관리하는 저장소입니다. 조직 내에서 [권한이 있는](/ko/models/registry/configure_registry/) Users는 [Artifacts를 다운로드하고 사용할](/ko/models/registry/download_use_artifact/) 수 있으며, 사용자가 속한 팀과 관계없이 모든 Artifacts를 공유하고 라이프사이클을 협업하여 관리할 수 있습니다.

레지스트리를 사용해 아티팩트 버전을 추적하고, 아티팩트 사용 및 변경 이력을 감사하며, Artifacts의 거버넌스와 규정 준수를 보장하고, [모델 CI/CD와 같은 다운스트림 프로세스를 자동화](/ko/models/automations/)하세요.

요약하면, W&amp;B Registry를 사용하여 다음을 수행할 수 있습니다:

* 머신 러닝 작업을 충족하는 아티팩트 버전을 조직의 다른 사용자에게 [승격](/ko/models/registry/link_version/)합니다.
* 특정 Artifacts를 찾거나 레퍼런스할 수 있도록 [태그로 Artifacts를 구성](/ko/models/registry/organize-with-tags/)합니다.
* [아티팩트의 리니지](/ko/models/registry/lineage/)를 추적하고 변경 이력을 감사합니다.
* [자동화](/ko/models/automations/)를 통해 모델 CI/CD와 같은 다운스트림 프로세스를 실행합니다.
* 각 레지스트리의 Artifacts에 조직 내 누가 [액세스](/ko/models/registry/configure_registry/)할 수 있는지 관리합니다.

<div id="learn-the-basics">
  ## 기본 개념
</div>

각 조직에는 처음에 **Models**와 **Datasets**라는 두 개의 레지스트리가 있으며, 이를 사용해 각각 모델 Artifacts와 데이터셋 Artifacts를 정리할 수 있습니다. 조직의 필요에 따라 [다른 아티팩트 유형을 정리하기 위한 추가 레지스트리](/ko/models/registry/create_registry)를 생성할 수 있습니다.

각 [*레지스트리*](/ko/models/registry/configure_registry/)는 하나 이상의 [*컬렉션*](/ko/models/registry/create_collection/)으로 구성됩니다. 각 컬렉션은 고유한 작업 또는 사용 사례를 나타냅니다.

{/* <Frame>
    <img src="/images/registry/homepage_registry.png" alt="W&B Registry"  />
  </Frame> */}

레지스트리에 아티팩트를 추가하려면 먼저 [특정 아티팩트 버전을 W&amp;B에 로깅](/ko/models/artifacts/create-a-new-artifact-version/)해야 합니다. 아티팩트를 로깅할 때마다 W&amp;B는 해당 아티팩트에 자동으로 버전을 할당합니다. 아티팩트 버전은 0부터 시작하는 인덱싱을 사용하므로 첫 번째 버전은 `v0`, 두 번째 버전은 `v1`이며, 이후에도 같은 방식으로 증가합니다.

아티팩트를 W&amp;B에 로깅한 뒤에는 해당 아티팩트 버전을 레지스트리의 컬렉션에 연결할 수 있습니다.

<Note>
  &quot;연결&quot;이라는 용어는 W&amp;B가 아티팩트를 저장하는 위치와 레지스트리 내에서 아티팩트에 액세스할 수 있는 위치를 연결하는 포인터를 의미합니다. 아티팩트를 컬렉션에 연결하더라도 W&amp;B는 아티팩트를 복제하지 않습니다.
</Note>

예를 들어, 다음 코드 예제는 `"my_model.txt"`라는 이름의 모델 아티팩트를 `"model"`이라는 레지스트리 내 `"first-collection"`이라는 컬렉션에 로깅하고 연결합니다:

1. `wandb.init()`으로 W&amp;B run을 초기화합니다.
2. `wandb.Run.log()`를 사용해 아티팩트를 W&amp;B에 로깅합니다.
3. 아티팩트 버전을 연결할 컬렉션 이름과 레지스트리 이름을 지정합니다.
4. `wandb.Run.link_artifact()`를 사용해 아티팩트를 컬렉션에 연결합니다.

이 Python 코드를 스크립트로 저장한 후 실행하세요. W&amp;B Python SDK 버전 0.18.6 이상이 필요합니다.

```python title="hello_collection.py"
import wandb
import random

# 아티팩트를 추적하기 위해 W&B Run 초기화
with wandb.init(project="registry_quickstart") as run:
    # 로깅할 수 있도록 시뮬레이션된 모델 파일 생성
    with open("my_model.txt", "w") as f:
        f.write("Model: " + str(random.random()))

    # W&B에 아티팩트 로깅
    logged_artifact = run.log_artifact(
        artifact_or_path="./my_model.txt", 
        name="gemma-finetuned", 
        type="model" # 아티팩트 유형 지정
    )

    # 아티팩트를 게시할 컬렉션과 레지스트리의
    # 이름 지정
    COLLECTION_NAME = "first-collection"
    REGISTRY_NAME = "model"

    # 레지스트리에 아티팩트 연결
    run.link_artifact(
        artifact=logged_artifact, 
        target_path=f"wandb-registry-{REGISTRY_NAME}/{COLLECTION_NAME}"
    )
```

W&amp;B는 반환된 run 객체의 `wandb.Run.link_artifact(target_path = "")` 방법에서 지정한 컬렉션이 지정한 레지스트리 내에 없으면 해당 컬렉션을 자동으로 생성합니다.

이전 예제에 이어, 스크립트를 실행한 후 W&amp;B 레지스트리로 이동하여 조직 내 다른 구성원과 내가 게시한 아티팩트 버전을 확인하세요. 프로젝트 사이드바의 **Applications** 아래에서 **레지스트리**를 선택하세요. `"Model"` 레지스트리를 선택하세요. 레지스트리 내에서 연결된 아티팩트 버전이 포함된 `"first-collection"` 컬렉션을 확인할 수 있습니다.

레지스트리 내의 컬렉션에 아티팩트 버전을 연결하면, 적절한 권한이 있는 조직 구성원은 내 아티팩트 버전을 [뷰](/ko/models/registry/lineage), [다운로드](/ko/models/registry/download_use_artifact), [태그로 구성](/ko/models/registry/organize-with-tags), 관리하고 다운스트림 자동화를 생성하는 등의 작업을 수행할 수 있습니다.

<Note>
  아티팩트 버전이 메트릭을 로깅하는 경우(예: `wandb.Run.log_artifact()` 사용), 해당 버전의 상세 페이지에서 메트릭을 확인할 수 있으며 컬렉션 페이지에서는 아티팩트 버전 간 메트릭을 비교할 수 있습니다. 자세한 내용은 [레지스트리에서 연결된 아티팩트 보기](/ko/models/registry/link_version/#view-linked-artifacts-in-a-registry)를 참고하십시오.
</Note>

<div id="enable-wb-registry">
  ## W&amp;B Registry 활성화
</div>

배포 유형에 따라 다음 안내에 따라 W&amp;B Registry를 활성화하세요:

| Deployment type    | How to enable                                                                                                                                      |
| ------------------ | -------------------------------------------------------------------------------------------------------------------------------------------------- |
| Multi-tenant Cloud | 추가 액션은 필요 없습니다. W&amp;B Registry는 W&amp;B App에서 사용 가능합니다.                                                                                          |
| Dedicated Cloud    | 배포 환경에서 W&amp;B Registry를 활성화하려면 아카운트 담당자에 문의하세요.                                                                                                  |
| Self-Managed       | Server v0.70.0 이상은 추가 액션이 필요 없습니다. 지원되는 이전 Server 버전의 경우 환경 변수 `ENABLE_REGISTRY_UI`를 `true`로 설정하세요. [환경 변수 구성](/ko/platform/hosting/env-vars)을 참고하세요. |
