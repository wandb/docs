App UI에서 인터랙티브하게 리포트를 편집하거나 W&amp;B SDK를 사용해 프로그래밍 방식으로 편집할 수 있습니다.

리포트는 &#95;블록&#95;으로 구성됩니다. 블록은 리포트의 본문을 이룹니다. 이러한 블록 안에 텍스트, 이미지, 임베디드 시각화, 실험과 실행에서 생성된 플롯, 패널 그리드를 추가할 수 있습니다.

&#95;패널 그리드&#95;는 패널과 &#95;실행 세트(run set)&#95;를 포함하는 특정 유형의 블록입니다. 실행 세트는 W&amp;B의 한 프로젝트에 기록된 여러 실행의 모음입니다. 패널은 실행 세트 데이터의 시각화입니다.

<Note>
  저장된 워크스페이스 보기를 생성하고 커스터마이즈하는 방법을 단계별로 보여주는 예제는 [Programmatic workspaces notebook](https://colab.research.google.com/github/wandb/wandb-workspaces/blob/Update-wandb-workspaces-tuturial/Workspace_tutorial.ipynb)을 참고하세요.
</Note>

<Note>
  리포트를 프로그래밍 방식으로 편집하려면 W&amp;B Python SDK에 더해 W&amp;B Report 및 Workspace API인 `wandb-workspaces`가 설치되어 있는지 확인하세요:

  ```pip
  pip install wandb wandb-workspaces
  ```
</Note>

<div id="add-plots">
  ## 플롯 추가
</div>

각 패널 그리드에는 실행 집합(run set)과 패널 집합이 있습니다. 섹션 하단의 실행 집합은 그리드의 패널에 어떤 데이터가 표시될지 제어합니다. 다른 실행 집합에서 데이터를 가져오는 차트를 추가하려면 새 패널 그리드를 생성하세요.

<Tabs>
  <Tab title="W&B App">
    리포트에서 슬래시(`/`)를 입력하여 드롭다운 메뉴를 표시합니다. 패널을 추가하려면 **Add panel**을 선택합니다. 선형 플롯(line plot), 산점도(scatter plot), 평행 좌표 차트(parallel coordinates chart)를 포함하여 W&amp;B에서 지원하는 모든 패널을 추가할 수 있습니다.

    <Frame>
      <img src="/images/reports/demo_report_add_panel_grid.gif" alt="리포트에 차트 추가" />
    </Frame>
  </Tab>

  <Tab title="Report and Workspace API">
    SDK를 사용해 코드로 리포트에 플롯을 추가할 수 있습니다. `PanelGrid` Public API Class의 `panels` 매개변수에 하나 이상의 플롯 또는 차트 객체로 구성된 리스트를 전달합니다. 각 플롯이나 차트 객체는 해당 Python 클래스에서 생성합니다.

    다음 예제는 선형 플롯과 산점도를 생성하는 방법을 보여줍니다.

    ```python
    import wandb
    import wandb_workspaces.reports.v2 as wr

    report = wr.Report(
        project="report-editing",
        title="An amazing title",
        description="A descriptive description.",
    )

    blocks = [
        wr.PanelGrid(
            panels=[
                wr.LinePlot(x="time", y="velocity"),
                wr.ScatterPlot(x="time", y="acceleration"),
            ]
        )
    ]

    report.blocks = blocks
    report.save()
    ```

    코드로 리포트에 추가할 수 있는 플롯과 차트에 대한 자세한 내용은 `wr.panels`를 참조하세요.
  </Tab>
</Tabs>

<div id="add-run-sets">
  ## 실행 세트 추가
</div>

App UI 또는 W&amp;B SDK를 사용해 프로젝트에서 실행 세트를 대화형으로 추가할 수 있습니다.

<Tabs>
  <Tab title="W&B 앱">
    리포트에서 슬래시(`/`)를 입력해 드롭다운 메뉴를 표시합니다. 드롭다운에서 **Panel Grid**를 선택합니다. 그러면 리포트가 생성된 프로젝트의 실행 세트가 자동으로 가져와집니다.

    패널을 리포트에 가져오면 실행 이름은 프로젝트의 이름을 따릅니다. 리포트에서 독자에게 더 많은 정보를 제공하기 위해 [실행 이름을 변경](/ko/models/runs/#rename-a-run)할 수 있습니다. 실행 이름은 해당 개별 패널에서만 변경됩니다. 동일한 리포트에서 패널을 복제하면, 복제된 패널에서도 실행 이름이 변경됩니다.

    1. 리포트에서 연필 아이콘을 클릭해 리포트 편집기를 엽니다.

    2. 실행 세트에서 이름을 변경할 실행을 찾습니다. 실행 이름 위에 마우스를 올리고 세로 점 세 개를 클릭합니다. 다음 선택 사항 중 하나를 선택한 뒤 양식을 제출합니다.

       * **Rename run for project**: 전체 프로젝트에서 실행 이름을 변경합니다. 새 랜덤 이름을 생성하려면 필드를 비워 둡니다.
       * **Rename run for panel grid**: 다른 컨텍스트의 기존 이름은 유지한 채 리포트에서만 실행 이름을 변경합니다. 새 랜덤 이름 생성은 지원되지 않습니다.

    3. **Publish report**를 클릭합니다.
  </Tab>

  <Tab title="Report 및 Workspace API">
    `wr.Runset()` 및 `wr.PanelGrid` 클래스로 프로젝트에서 실행 세트를 추가할 수 있습니다. 다음 절차는 실행 세트를 추가하는 방법을 설명합니다:

    1. `wr.Runset()` 객체 인스턴스를 생성합니다. `project` 파라미터에는 실행 세트를 포함하는 프로젝트 이름을, `entity` 파라미터에는 해당 프로젝트를 소유한 엔터티를 전달합니다.
    2. `wr.PanelGrid()` 객체 인스턴스를 생성합니다. 하나 이상의 runset 객체 리스트를 `run sets` 파라미터에 전달합니다.
    3. 하나 이상의 `wr.PanelGrid()` 객체 인스턴스를 리스트에 저장합니다.
    4. 리포트 인스턴스의 `blocks` 속성을 패널 그리드 인스턴스 리스트로 업데이트합니다.

    ```python
    import wandb
    import wandb_workspaces.reports.v2 as wr

    report = wr.Report(
        project="report-editing",
        title="An amazing title",
        description="A descriptive description.",
    )

    panel_grids = wr.PanelGrid(
        runsets=[wr.RunSet(project="<project-name>", entity="<entity-name>")]
    )

    report.blocks = [panel_grids]
    report.save()
    ```

    원한다면, 한 번의 SDK 호출로 runset과 패널을 함께 추가할 수 있습니다:

    ```python
    import wandb

    report = wr.Report(
        project="report-editing",
        title="An amazing title",
        description="A descriptive description.",
    )

    panel_grids = wr.PanelGrid(
        panels=[
            wr.LinePlot(
                title="line title",
                x="x",
                y=["y"],
                range_x=[0, 100],
                range_y=[0, 100],
                log_x=True,
                log_y=True,
                title_x="x axis title",
                title_y="y axis title",
                ignore_outliers=True,
                groupby="hyperparam1",
                groupby_aggfunc="mean",
                groupby_rangefunc="minmax",
                smoothing_factor=0.5,
                smoothing_type="gaussian",
                smoothing_show_original=True,
                max_runs_to_show=10,
                plot_type="stacked-area",
                font_size="large",
                legend_position="west",
            ),
            wr.ScatterPlot(
                title="scatter title",
                x="y",
                y="y",
                # z='x',
                range_x=[0, 0.0005],
                range_y=[0, 0.0005],
                # range_z=[0,1],
                log_x=False,
                log_y=False,
                # log_z=True,
                running_ymin=True,
                running_ymean=True,
                running_ymax=True,
                font_size="small",
                regression=True,
            ),
        ],
        runsets=[wr.RunSet(project="<project-name>", entity="<entity-name>")],
    )


    report.blocks = [panel_grids]
    report.save()
    ```
  </Tab>
</Tabs>

<div id="group-a-run-set-programmatically">
  ## 실행 세트를 프로그래밍 방식으로 그룹화하기
</div>

[Workspace and Reports API](/ko/models/ref/wandb_workspaces/reports)를 사용해 실행 세트에 포함된 실행을 프로그래밍 방식으로 그룹화할 수 있습니다.

실행 세트의 실행을 config 값, 실행 메타데이터 또는 summary 지표를 기준으로 그룹화할 수 있습니다. 다음 표는 사용 가능한 그룹화 방법과 각 방법에서 사용할 수 있는 키를 나열합니다.

| Grouping Method | Description              | Available keys                            |
| --------------- | ------------------------ | ----------------------------------------- |
| Config values   | config 값을 기준으로 실행을 그룹화   | `wandb.init(config=)`의 config 매개변수에 지정한 값 |
| Run metadata    | 실행 메타데이터를 기준으로 실행을 그룹화   | `State`, `Name`, `JobType`                |
| Summary metrics | summary 지표를 기준으로 실행을 그룹화 | `wandb.Run.log()`로 실행에 기록한 값              |

{/* 리포트에서 실행을 그룹화하는 것과 워크스페이스에서 실행을 그룹화하는 것의 주요 차이점:
  1. 문자열 경로 vs 객체: 리포트에서는 `ws.Config("group")`와 같은 타입 객체 대신 `"config.group"`과 같은 문자열 경로를 사용합니다.
  2. Dot 표기법: `"config.model_type"`처럼 점을 사용해 중첩된 값에 접근합니다.
  3. 미리 정의된 필드: `"Name"`, `"Tags"`, `"State"` 같은 일부 필드는 특별한 실행 메타데이터 필드입니다. */}

<div id="group-runs-by-config-values">
  ### config 값으로 실행 그룹화
</div>

유사한 설정을 가진 실행을 비교하려면 config 값으로 실행을 그룹화하세요. config 값은 실행 구성 `(wandb.init(config=))`에서 지정하는 매개변수입니다. 실행을 config 값으로 그룹화하려면 `config.<key>` 구문을 사용합니다. 여기서 `<key>`는 그룹화에 사용할 config 값의 이름입니다.

예를 들어, 다음 코드 스니펫은 먼저 `group`에 대한 config 값을 사용해 실행을 초기화한 다음, 리포트에서 `group` config 값을 기준으로 실행을 그룹화합니다. `<entity>` 및 `<project>`에는 W&amp;B 엔터티와 프로젝트 이름을 입력하세요.

```python
import wandb
import wandb_workspaces.reports.v2 as wr

entity = "<entity>"
project = "<project>"

for group in ["control", "experiment_a", "experiment_b"]:
    for i in range(3):
        with wandb.init(entity=entity, project=project, group=group, config={"group": group, "run": i}, name=f"{group}_run_{i}") as run:
            # 학습 과정 시뮬레이션
            for step in range(100):
                run.log({
                    "acc": 0.5 + (step / 100) * 0.3 + (i * 0.05),
                    "loss": 1.0 - (step / 100) * 0.5
                })
```

Python 스크립트나 노트북에서 `config.group` 값에 따라 실행을 그룹화할 수 있습니다.

```python
runset = wr.Runset(
  project=project,
  entity=entity,
  groupby=["config.group"]  # "group" config 값으로 그룹화
)
```

앞선 예시에서 이어서, 그룹화된 실행 세트로 리포트를 생성할 수 있습니다:

```python
report = wr.Report(
  entity=entity,
  project=project,
  title="Grouped Runs Example",
)

report.blocks = [
  wr.PanelGrid(
      runsets=[runset],
          )
      ]

report.save()
```

<div id="group-runs-by-run-metadata">
  ### 실행 메타데이터를 기준으로 실행 그룹화하기
</div>

실행의 이름(`Name`), 상태(`State`), 또는 작업 유형(`JobType`)에 따라 실행을 그룹화할 수 있습니다.

이전 예시에 이어, 다음 코드 스니펫을 사용해 실행을 이름별로 그룹화할 수 있습니다:

```python
runset = wr.Runset(
  project=project,
  entity=entity,
  groupby=["Name"]  # 실행 이름으로 그룹화
)
```

<Note>
  실행 이름은 `wandb.init(name=)` 파라미터에 지정한 값입니다. 이름을 지정하지 않으면 W&amp;B가 실행에 대한 무작위 이름을 생성합니다.

  실행 이름은 W&amp;B App의 해당 실행 **Overview** 페이지에서 확인하거나, 코드에서 `Api.runs().run.name`을 사용해 확인할 수 있습니다.
</Note>

<div id="group-runs-by-summary-metrics">
  ### 요약 메트릭으로 실행 그룹화하기
</div>

다음 예제에서는 요약 메트릭으로 실행을 그룹화하는 방법을 보여준다. 요약 메트릭은 `wandb.Run.log()`을 사용해 실행에 기록하는 값이다. 실행을 기록한 후에는 W&amp;B 앱에서 해당 실행의 **Overview** 페이지에 있는 **Summary** 섹션에서 요약 메트릭 이름을 확인할 수 있다.

요약 메트릭으로 실행을 그룹화하는 구문은 `summary.<key>`이며, 여기서 `<key>`는 그룹화에 사용할 요약 메트릭의 이름이다.

예를 들어, `acc`라는 이름의 요약 메트릭을 기록했다고 가정해 보자:

```python
import wandb
import wandb_workspaces.reports.v2 as wr

entity = "<entity>"
project = "<project>"

for group in ["control", "experiment_a", "experiment_b"]:
    for i in range(3):
        with wandb.init(entity=entity, project=project, group=group, config={"group": group, "run": i}, name=f"{group}_run_{i}") as run:
            # 학습 시뮬레이션
            for step in range(100):
                run.log({
                    "acc": 0.5 + (step / 100) * 0.3 + (i * 0.05),
                    "loss": 1.0 - (step / 100) * 0.5
                })

```

그런 다음 `summary.acc` 요약 지표별로 실행을 그룹화할 수 있습니다:

```python
runset = wr.Runset(
  project=project,
  entity=entity,
  groupby=["summary.acc"]  # 요약 값으로 그룹화 
)
```

<div id="filter-a-run-set-programmatically">
  ## 실행 집합을 프로그래밍 방식으로 필터링하기
</div>

실행 집합을 프로그래밍 방식으로 필터링한 다음 [Workspace and Reports API](/ko/models/ref/wandb_workspaces/reports)를 사용해 리포트에 추가할 수 있습니다.

필터 표현식의 일반적인 구문은 다음과 같습니다.

```text
Filter('key') operation <value>
```

여기서 `key`는 필터의 이름이고, `operation`은 비교 연산자(예: `>`, `<`, `==`, `in`, `not in`, `or`, `and`)이며, `<value>`는 비교할 값입니다. `Filter`는 적용하려는 필터 유형을 나타내는 플레이스홀더입니다. 다음 표는 사용 가능한 필터와 해당 설명을 보여줍니다:

| Filter                 | Description          | Available keys                               |
| ---------------------- | -------------------- | -------------------------------------------- |
| `Config('key')`        | config 값으로 필터링       | `wandb.init(config=)`의 `config` 파라미터에 지정된 값. |
| `SummaryMetric('key')` | summary metric으로 필터링 | `wandb.Run.log()`으로 실행에 기록한 값.               |
| `Tags('key')`          | 태그로 필터링              | 실행에 추가한 태그 값(코드에서 또는 W&amp;B App에서 추가한 값).   |
| `Metric('key')`        | 실행 속성으로 필터링          | `tags`, `state`, `displayName`, `jobType`    |

필터를 정의한 후, 리포트를 생성하고 필터링된 실행 집합을 `wr.PanelGrid(runsets=)`에 전달할 수 있습니다. 리포트에 다양한 요소를 코드로 추가하는 방법은 이 페이지 전반에 걸쳐 있는 **Report and Workspace API** 탭을 참고하십시오.

다음 예시는 리포트에서 실행 집합을 필터링하는 방법을 보여줍니다. `<>`로 둘러싸인 값을 사용자 값으로 바꾸십시오.

<div id="config-filters">
  ### Config 필터
</div>

하나 이상의 config 값을 기준으로 runset을 필터링합니다. config 값은 실행 설정(`wandb.init(config=)`)에서 지정하는 매개변수입니다.

예를 들어, 다음 코드 스니펫은 먼저 `learning_rate`와 `batch_size`에 대한 config 값을 사용해 실행을 초기화한 다음, 리포트에서 `learning_rate` config 값을 기준으로 실행을 필터링합니다.

```python
import wandb

config = {
    "learning_rate": 0.01,
    "batch_size": 32,
}

with wandb.init(project="<project>", entity="<entity>", config=config) as run:
    # 여기에 학습 코드를 작성하세요
    pass
```

Python 스크립트나 노트북에서 학습률이 `0.01`보다 큰 실행만 프로그래밍 방식으로 필터링할 수 있습니다.

```python
import wandb_workspaces.reports.v2 as wr

runset = wr.Runset(
  entity="<entity>",
  project="<project>",
  filters="Config('learning_rate') > 0.01"
)
```

`and` 연산자를 사용해 여러 config 값을 동시에 기준으로 필터링할 수 있습니다:

```python
runset = wr.Runset(
  entity="<entity>",
  project="<project>",
  filters="Config('learning_rate') > 0.01 and Config('batch_size') == 32"
)
```

이전 예시의 연장으로, 필터링된 실행 집합을 사용해 다음과 같이 리포트를 생성할 수 있습니다:

```python
report = wr.Report(
  entity="<entity>",
  project="<project>",
  title="My Report"
)

report.blocks = [
  wr.PanelGrid(
      runsets=[runset],
      panels=[
          wr.LinePlot(
              x="Step",
              y=["accuracy"],
          )
      ]
  )
]

report.save()
```

<div id="metric-filters">
  ### Metric 필터
</div>

실행 집합을 다음 실행 속성을 기준으로 필터링합니다: 태그(`tags`), 실행 상태(`state`), 실행 이름(`displayName`), 또는 작업 유형(`jobType`).

<Note>
  `Metric` 필터는 구문이 다릅니다. 값 목록을 리스트 형태로 전달하세요.

  ```text
  Metric('key') operation [<value>]
  ```
</Note>

예를 들어, 세 개의 실행을 생성하고 각각의 실행에 이름을 지정하는 다음 Python 코드 스니펫을 살펴보겠습니다:

```python
import wandb

with wandb.init(project="<project>", entity="<entity>") as run:
    for i in range(3):
        run.name = f"run{i+1}"
        # 여기에 학습 코드를 작성하세요
        pass
```

리포트를 생성할 때 표시 이름으로 실행을 필터링할 수 있습니다. 예를 들어 이름이 `run1`, `run2`, `run3`인 실행만 포함하도록 필터링하려면 다음 코드를 사용할 수 있습니다:

```python
runset = wr.Runset(
  entity="<entity>",
  project="<project>",
  filters="Metric('displayName') in ['run1', 'run2', 'run3']"
)
```

<Note>
  W&amp;B App에서 해당 실행의 **Overview** 페이지 또는 `Api.runs().run.name`을 사용해 프로그래밍 방식으로 실행 이름을 확인할 수 있습니다.
</Note>

다음 예제는 실행의 상태(`finished`, `crashed`, `running`)에 따라 runset을 필터링하는 방법을 보여줍니다:

```python
runset = wr.Runset(
  entity="<entity>",
  project="<project>",
  filters="Metric('state') in ['finished']"
)
```

```python
runset = wr.Runset(
  entity="<entity>",
  project="<project>",
  filters="Metric('state') not in ['crashed']"
)
```

<div id="summarymetric-filters">
  ### SummaryMetric 필터
</div>

다음 예시는 요약 메트릭으로 실행 세트를 필터링하는 방법을 보여줍니다. 요약 메트릭은 `wandb.Run.log()`를 사용해 실행에 기록하는 값입니다. 실행을 기록한 후에는 W&amp;B 앱에서 해당 실행의 **Overview** 페이지에 있는 **Summary** 섹션에서 요약 메트릭 이름을 확인할 수 있습니다.

```python
runset = wr.Runset(
  entity="<entity>",
  project="<project>",
  filters="SummaryMetric('accuracy') > 0.9"
)
```

```python
runset = wr.Runset(
  entity="<entity>",
  project="<project>",
  filters="Metric('state') in ['finished'] and SummaryMetric('train/train_loss') < 0.5"
)
```

<div id="tags-filters">
  ### 태그 필터
</div>

다음 코드 예제는 태그를 기준으로 실행 집합을 필터링하는 방법을 보여줍니다. 태그는 실행에 (프로그램 코드에서 또는 W&amp;B App에서) 추가하는 값입니다.

```python
runset = wr.Runset(
  entity="<entity>",
  project="<project>",
  filters="Tags('training') == 'training'"
)
```

<div id="add-code-blocks">

App UI 또는 W&amp;B SDK를 사용해 보고서에 코드 블록을 대화형으로 추가할 수 있습니다.

    ```python
        )
    ]

    report.save()
    ```

    ```yaml
    this:
    - is
    - a
    cool:
    - yaml
    - file
    ```

    다음 예시는 Python 코드 블록을 보여 줍니다:

    ```python
    report = wr.Report(project="report-editing")


    report.blocks = [wr.CodeBlock(code=["Hello, World!"], language="python")]

    report.save()
    ```

    이 코드는 다음과 유사한 코드 블록을 렌더링합니다:

    ```md
    Hello, World!
    ```
  </Tab>
</Tabs>

<div id="visualize-relationships-across-multiple-dimensions">
  ## 여러 차원에 걸친 관계 시각화
</div>

여러 차원에 걸친 관계를 효과적으로 시각화하려면, 변수 중 하나를 색상 그라디언트로 표현하십시오. 이렇게 하면 시각적 명확성이 높아지고 패턴을 더 쉽게 해석할 수 있습니다.

1. 색상 그라디언트로 표현할 변수를 선택하십시오(예: penalty 점수, learning rate 등). 이렇게 하면 penalty(색상)가 학습 시간(x축)에 따라 reward/부작용(y축)과 어떻게 상호작용하는지 더 명확하게 파악할 수 있습니다.
2. 주요 추세를 강조하십시오. 특정 실행 그룹에 마우스를 올리면 시각화에서 해당 실행들이 강조 표시됩니다.
