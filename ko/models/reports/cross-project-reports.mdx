<Note>
  [프로젝트 간 Runs 비교를 보여주는 비디오](https://www.youtube.com/watch?v=uD4if_nGrs4)를 시청하세요 (2분).
</Note>

크로스 프로젝트 리포트를 사용하면 서로 다른 두 프로젝트의 Runs를 나란히 비교할 수 있습니다. 따라서 데이터를 중복하지 않고도 별도의 W&amp;B 프로젝트에서 추적된 실험을 평가할 때 유용합니다. 이 페이지에서는 다른 프로젝트의 Runs를 리포트로 가져오는 방법과 W&amp;B 계정이 없는 공동 작업자와 해당 리포트를 공유하는 방법을 설명합니다.

프로젝트 간 Runs를 비교하려면, run 세트 테이블의 프로젝트 선택기를 사용해 프로젝트를 선택하세요.

<Frame>
  <img src="/images/reports/howto_pick_a_different_project_to_draw_runs_from.gif" alt="서로 다른 프로젝트 간 Runs 비교" />
</Frame>

섹션의 시각화는 첫 번째 활성 run 세트에서 열을 가져옵니다. 선형 플롯에서 찾는 메트릭이 보이지 않으면, 섹션에서 먼저 선택된 run 세트에 해당 열이 사용 가능한지 확인하세요.

이 기능은 시계열 선의 이력 데이터는 지원하지만, 서로 다른 프로젝트에서 서로 다른 summary 메트릭을 가져오는 것은 지원하지 않습니다. 즉, 다른 프로젝트에만 로깅된 열로는 산점도를 만들 수 없습니다.

두 프로젝트의 Runs를 비교해야 하는데 열이 제대로 작동하지 않는다면, 한 프로젝트의 Runs에 태그를 추가한 다음 해당 Runs를 다른 프로젝트로 옮기세요. 그러면 각 프로젝트의 Runs만 필터링할 수 있으면서도, 리포트에는 두 run 세트의 모든 열이 포함됩니다.

<div id="view-only-report-links">
  ## 뷰 전용 리포트 링크
</div>

프로젝트 외부의 사람(W&amp;B 계정이 없는 이해관계자 등)과 리포트를 공유해야 하는 경우, 뷰 전용 링크를 사용하세요. 비공개 프로젝트 또는 팀 프로젝트의 리포트에 대한 뷰 전용 링크를 공유합니다.

<Frame>
  <img src="/images/reports/magic-links.gif" alt="뷰 전용 리포트 링크" />
</Frame>

뷰 전용 리포트 링크는 URL에 시크릿 액세스 토큰을 추가하므로, 링크를 여는 사람은 누구나 해당 페이지를 볼 수 있습니다. 누구나 로그인하지 않고도 매직 링크를 사용해 리포트를 볼 수 있습니다. [W&amp;B Local](/ko/platform/hosting/) 프라이빗 클라우드 설치를 사용하는 고객의 경우, 이러한 링크는 방화벽 내부에 유지됩니다. 비공개 인스턴스에 대한 액세스 권한과 뷰 전용 링크에 대한 액세스 권한을 모두 가진 팀 구성원만 리포트를 볼 수 있습니다.

뷰 전용 모드에서는 로그인하지 않은 사용자도 차트를 보고, 마우스를 올려 값 툴팁을 확인하고, 차트를 확대 및 축소하고, 테이블의 열을 스크롤할 수 있습니다. 데이터를 탐색하기 위한 새 차트나 새 테이블 쿼리를 만들 수 없습니다. 뷰 전용 방문자는 run을 클릭해 run 페이지로 이동할 수 없고, 공유 모달도 볼 수 없습니다. 대신 마우스를 올리면 `Sharing not available for view only access`라고 표시된 툴팁이 나타납니다.

<Info>
  매직 링크는 Private 및 Team Projects에서만 사용할 수 있습니다. Public Projects(누구나 조회 가능) 또는 Open Projects(누구나 조회하고 Runs에 기여 가능)의 경우 링크를 켜거나 끌 수 없습니다. 프로젝트가 이미 공개되어 있으므로 링크가 있는 사람은 누구나 이미 리포트를 볼 수 있습니다.
</Info>

<div id="send-a-graph-to-a-report">
  ## 그래프를 리포트로 보내기
</div>

워크스페이스에서 이미 만든 차트를 다시 만들지 않고도 리포트에 추가할 수 있습니다. 워크스페이스의 차트를 관련 분석과 함께 유지하려면 해당 차트를 리포트로 직접 보내세요. 복사하려는 차트 또는 Panel의 드롭다운 메뉴를 클릭한 다음 **Add to report**를 클릭하여 대상 리포트를 선택하세요.
