프로젝트 간 Reports 를 사용하여 서로 다른 두 프로젝트의 Runs 를 비교할 수 있습니다. run set 테이블에 있는 프로젝트 선택기를 사용하여 프로젝트를 선택하세요.

<Frame>
    <img src="/images/reports/howto_pick_a_different_project_to_draw_runs_from.gif" alt="Compare runs across different projects"  />
</Frame>

해당 섹션의 시각화는 첫 번째 활성 runset에서 컬럼을 가져옵니다. 라인 플롯에서 원하는 메트릭이 보이지 않는다면, 섹션에서 체크된 첫 번째 run set에 해당 컬럼이 있는지 확인하세요.

이 기능은 시계열 라인의 history 데이터를 지원하지만, 서로 다른 프로젝트에서 서로 다른 summary 메트릭을 가져오는 것은 지원하지 않습니다. 즉, 다른 프로젝트에만 로그된 컬럼으로는 산점도(scatter plot)를 생성할 수 없습니다.

두 프로젝트의 Runs 를 비교해야 하는데 컬럼이 제대로 작동하지 않는 경우, 한 프로젝트의 Runs 에 태그를 추가한 다음 해당 Runs 를 다른 프로젝트로 이동하세요. 각 프로젝트의 Runs 만 필터링하여 볼 수 있으며, Reports 에는 두 run 세트의 모든 컬럼이 포함됩니다.

## 뷰 전용 report 링크

Private 프로젝트나 Team 프로젝트에 있는 Reports 의 뷰 전용 링크를 공유하세요.

<Frame>
    <img src="/images/reports/magic-links.gif" alt="View-only report links"  />
</Frame>

뷰 전용 report 링크는 URL에 비밀 access 토큰을 추가하므로, 링크를 여는 모든 사용자가 페이지를 볼 수 있습니다. 누구든지 매직 링크를 사용하여 로그인 없이 Reports 를 볼 수 있습니다. [W&B Local](/platform/hosting/) 프라이빗 클라우드 설치형 고객의 경우, 이 링크는 방화벽 내에 유지되므로 프라이빗 인스턴스에 대한 엑세스 권한과 뷰 전용 링크에 대한 엑세스 권한을 모두 가진 팀 멤버만 Reports 를 볼 수 있습니다.

**뷰 전용 모드**에서 로그인을 하지 않은 사용자는 차트를 보고, 마우스를 올려 값의 툴팁을 확인하고, 차트를 확대/축소하거나 테이블의 컬럼을 스크롤할 수 있습니다. 뷰 전용 모드에서는 데이터를 탐색하기 위한 새로운 차트나 새로운 테이블 쿼리를 생성할 수 없습니다. 뷰 전용 방문자는 run 을 클릭하여 run 페이지로 이동할 수 없습니다. 또한, 뷰 전용 방문자에게는 공유 모달이 보이지 않으며 대신 마우스를 올렸을 때 `Sharing not available for view only access`라는 툴팁이 표시됩니다.

<Info>
매직 링크는 "Private" 및 "Team" Projects 에서만 사용할 수 있습니다. "Public"(누구나 볼 수 있음) 또는 "Open"(누구나 보고 run 을 추가할 수 있음) Projects 의 경우, 프로젝트가 이미 공개되어 있어 링크가 있는 누구에게나 열려 있음을 의미하므로 링크 기능을 켜거나 끌 수 없습니다.
</Info>

## 그래프를 report로 보내기

작업 진행 상황을 추적하기 위해 Workspace 의 그래프를 Reports 로 보내세요. Reports 로 복사하려는 차트나 패널의 드롭다운 메뉴를 클릭하고 **Add to report**를 클릭하여 대상 Reports 를 선택합니다.
