패널에 불러올 데이터와 그 시각화 방식을 제어하려면 맞춤형 차트를 사용하세요.

<div id="1-log-data-to-wb">
  ## 1. W&amp;B에 데이터 로깅하기
</div>

먼저 스크립트에서 데이터를 로깅합니다. 하이퍼파라미터처럼 트레이닝 시작 시점에 한 번만 설정하는 값에는 [wandb.Run.config](/ko/models/track/config/)를 사용하세요. 시간 경과에 따라 여러 시점의 값을 기록하려면 [wandb.Run.log()](/ko/models/track/log/)를 사용하고, 맞춤형 2D 배열은 `wandb.Table()`로 로깅하세요. 로깅된 각 키별로 데이터 포인트는 최대 10,000개까지 로깅하는 것을 권장합니다.

```python
with wandb.init() as run: 

  # 맞춤형 데이터 테이블 로깅
  my_custom_data = [[x1, y1, z1], [x2, y2, z2]]
  run.log(
    {"custom_data_table": wandb.Table(data=my_custom_data, columns=["x", "y", "z"])}
  )
```

[간단한 예시 노트북을 실행해](https://bit.ly/custom-charts-colab) 데이터 테이블을 로깅해 보고, 다음 단계에서 맞춤형 차트를 설정해 보겠습니다. 결과 차트가 어떻게 보이는지는 [라이브 리포트](https://app.wandb.ai/demo-team/custom-charts/reports/Custom-Charts--VmlldzoyMTk5MDc)에서 확인하세요.

<div id="2-create-a-query">
  ## 2. 쿼리 만들기
</div>

시각화할 데이터를 로깅한 후 프로젝트 페이지로 이동하여 새 패널을 추가하려면 **`+`** 버튼을 클릭한 다음 **맞춤형 차트**를 선택하세요. [custom charts demo workspace](https://app.wandb.ai/demo-team/custom-charts)에서 함께 따라 할 수 있습니다.

<Frame>
  <img src="/images/app_ui/create_a_query.png" alt="비어 있는 맞춤형 차트" />
</Frame>

<div id="add-a-query">
  ### 쿼리 추가
</div>

1. `summary`를 클릭한 다음 `historyTable`을 선택하여 run 이력에서 데이터를 가져오는 새 쿼리를 설정합니다.
2. `wandb.Table()`을 로깅한 키를 입력합니다. 위 코드 스니펫에서는 `my_custom_table`이었습니다. [예시 노트북](https://bit.ly/custom-charts-colab)에서는 키가 `pr_curve`와 `roc_curve`입니다.

시각화를 사용자 지정하기 위해 Vega 사양을 다음과 같이 수정했습니다:

<div id="bonus-composite-histograms">
  ## 보너스: 합성 히스토그램
</div>

히스토그램은 수치 분포를 시각화하여 더 큰 데이터셋을 이해하는 데 도움을 줍니다. 합성 히스토그램은 동일한 bin에 여러 분포를 함께 표시하므로, 서로 다른 모델 간 또는 하나의 모델 내 서로 다른 클래스 간에서 둘 이상의 메트릭을 비교할 수 있습니다. 예를 들어, 주행 장면에서 객체를 탐지하는 시맨틱 세그멘테이션 모델의 경우 정확도 최적화와 intersection over union(IOU) 최적화 중 어느 쪽이 더 효과적인지 비교해 볼 수 있습니다. 또는 서로 다른 모델이 자동차(데이터에서 크고 흔한 영역)와 교통 표지판(훨씬 작고 덜 흔한 영역)을 얼마나 잘 탐지하는지 알고 싶을 수도 있습니다. [데모 Colab](https://bit.ly/custom-charts-colab)에서는 생물 10개 클래스 중 2개 클래스의 confidence score를 비교할 수 있습니다.

<Frame>
  <img src="/images/app_ui/composite_histograms.png" alt="합성 히스토그램" />
</Frame>

직접 맞춤형 합성 히스토그램 패널을 만들려면 다음과 같이 하세요.

1. Workspace 또는 리포트에서 새 맞춤형 차트 패널을 만드세요(“맞춤형 차트” 시각화 추가). 오른쪽 상단의 “Edit” 버튼을 클릭해 기본 제공 패널 유형 중 하나를 바탕으로 Vega 사양을 수정하세요.
2. 기본 Vega 사양을 [Vega로 작성한 합성 히스토그램 MVP 코드](https://gist.github.com/staceysv/9bed36a2c0c2a427365991403611ce21)로 바꾸세요. 이 Vega 사양에서 메인 제목, 축 제목, 입력 도메인 및 기타 세부 사항을 [Vega 문법](https://vega.github.io/)으로 직접 수정할 수 있습니다(색상을 바꾸거나 세 번째 히스토그램을 추가할 수도 있습니다 :)
3. 오른쪽의 쿼리를 수정하여 wandb 로그에서 올바른 데이터를 불러오세요. `summaryTable` 필드를 추가하고 해당 `tableKey`를 `class_scores`로 설정하면 run에서 로깅한 `wandb.Table`을 가져올 수 있습니다. 그러면 드롭다운 메뉴에서 `class_scores`로 로깅된 `wandb.Table`의 열을 선택해 두 히스토그램 bin 세트(`red_bins` 및 `blue_bins`)를 채울 수 있습니다. 이 예시에서는 빨간 bin에 `animal` 클래스 예측 점수를, 파란 bin에 `plant`를 선택했습니다.
4. 미리보기 렌더링에 표시되는 plot이 만족스러울 때까지 Vega 사양과 쿼리를 계속 수정할 수 있습니다. 완료되면 상단의 **Save as**를 클릭하고 나중에 재사용할 수 있도록 맞춤형 plot의 이름을 지정하세요. 그런 다음 **Apply from panel library**를 클릭해 plot 생성을 마무리하세요.
