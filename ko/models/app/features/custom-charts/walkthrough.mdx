커스텀 차트를 사용하면 패널에 로드되는 데이터와 해당 데이터의 시각화 방식을 제어할 수 있습니다.

<div id="1-log-data-to-wb">
  ## 1. W&amp;B에 데이터 로깅하기
</div>

먼저 스크립트에서 데이터를 로깅합니다. 학습 시작 시 한 번만 설정되는 하이퍼파라미터와 같은 단일 값에는 [wandb.Run.config](/ko/models/track/config/)를 사용하세요. 시간에 따라 여러 시점의 값을 기록하려면 [wandb.Run.log()](/ko/models/track/log/)를 사용하고, `wandb.Table()`로 사용자 정의 2D 배열을 로깅하세요. 각 로깅 키당 최대 10,000개의 데이터 포인트를 기록하는 것을 권장합니다.

```python
with wandb.init() as run: 

  # 커스텀 데이터 테이블 로깅
  my_custom_data = [[x1, y1, z1], [x2, y2, z2]]
  run.log(
    {"custom_data_table": wandb.Table(data=my_custom_data, columns=["x", "y", "z"])}
  )
```

데이터 테이블을 로깅해 보려면 [간단한 예제 노트북을 실행](https://bit.ly/custom-charts-colab)하고, 다음 단계에서 커스텀 차트를 설정해 보겠습니다. 결과 차트가 어떤 모습인지 보려면 [라이브 리포트](https://app.wandb.ai/demo-team/custom-charts/reports/Custom-Charts--VmlldzoyMTk5MDc)를 확인하세요.

<div id="2-create-a-query">
  ## 2. 쿼리 생성하기
</div>

시각화할 데이터를 로깅한 후, 프로젝트 페이지로 이동해 **`+`** 버튼을 클릭해 새 패널을 추가한 다음 **Custom Chart**를 선택하세요. [custom charts 데모 workspace](https://app.wandb.ai/demo-team/custom-charts)에서 함께 따라 해볼 수 있습니다.

<Frame>
  <img src="/images/app_ui/create_a_query.png" alt="빈 커스텀 차트" />
</Frame>

<div id="add-a-query">
  ### 쿼리 추가하기
</div>

1. `summary`를 클릭하고 `historyTable`을 선택하여 실행 기록에서 데이터를 가져오는 새 쿼리를 설정합니다.
2. `wandb.Table()`을 로깅한 키를 입력합니다. 위 코드 스니펫에서는 `my_custom_table`이었습니다. [예제 노트북](https://bit.ly/custom-charts-colab)에서는 키가 `pr_curve`와 `roc_curve`입니다.

  ## 3. 차트 사용자 지정하기
</div>

이제 꽤 괜찮아졌지만, 산점도 대신 선 그래프로 바꾸고 싶습니다. 이 기본 제공 차트의 Vega 사양을 변경하려면 **Edit**를 클릭하세요. [custom charts 데모 워크스페이스](https://app.wandb.ai/demo-team/custom-charts)를 보면서 따라 해 보세요.

시각화를 사용자 지정하기 위해 Vega 사양을 다음과 같이 수정했습니다:

* 사용하지 않는 “size” 필드 제거

<div id="bonus-composite-histograms">
  ## 보너스: 합성 히스토그램
</div>

히스토그램은 수치 분포를 시각화해 대규모 데이터셋을 이해하는 데 도움을 줍니다. 합성 히스토그램은 동일한 구간(bin)에 여러 분포를 함께 표시해, 서로 다른 모델 사이 또는 하나의 모델 안에서 서로 다른 클래스 간에 두 개 이상의 지표를 비교할 수 있게 해줍니다. 주행 장면에서 객체를 탐지하는 시맨틱 세그멘테이션 모델의 경우, 정확도(accuracy)를 최적화하는 것과 교집합 대비 합집합(IOU)을 최적화하는 것의 효과를 비교하거나, 서로 다른 모델이 자동차(데이터에서 크고 자주 등장하는 큰 영역)를 얼마나 잘 탐지하는지와 교통 표지판(훨씬 작고 덜 흔한 영역)을 얼마나 잘 탐지하는지를 알고 싶을 수 있습니다. [데모 Colab](https://bit.ly/custom-charts-colab)에서는 생물 10개 클래스 중 두 클래스에 대한 신뢰도 점수(confidence score)를 비교할 수 있습니다.

<Frame>
  <img src="/images/app_ui/composite_histograms.png" alt="Composite histogram" />
</Frame>

사용자 정의 합성 히스토그램 패널을 직접 만들려면:

1. Workspace 또는 Report에서 새 Custom Chart 패널을 생성합니다(“Custom Chart” 시각화를 추가). 오른쪽 상단의 “Edit” 버튼을 눌러, 기본 제공 패널 타입 중 하나에서 시작하는 Vega 스펙을 수정합니다.
2. 해당 기본 Vega 스펙을 [Vega에서 합성 히스토그램을 위한 MVP 코드](https://gist.github.com/staceysv/9bed36a2c0c2a427365991403611ce21)로 교체합니다. 이 Vega 스펙 안에서 [Vega 문법을 사용해](https://vega.github.io/) 메인 제목, 축 제목, 입력 도메인 및 기타 세부 내용을 직접 수정할 수 있습니다(색을 바꾸거나 심지어 세 번째 히스토그램을 추가할 수도 있습니다 :)
3. 오른쪽의 쿼리를 수정해 wandb 로그에서 올바른 데이터를 불러옵니다. `summaryTable` 필드를 추가하고, 해당 `tableKey`를 `class_scores`로 설정해 실행에서 로깅한 `wandb.Table`을 가져옵니다. 그러면 드롭다운 메뉴를 통해 `class_scores`로 로깅된 `wandb.Table`의 컬럼으로 두 개의 히스토그램 빈 집합(`red_bins`와 `blue_bins`)을 채울 수 있습니다. 제 예시에서는 빨간 빈에는 `animal` 클래스 예측 점수를, 파란 빈에는 `plant`를 선택했습니다.
4. 미리보기 렌더링에서 보이는 플롯이 만족스러울 때까지 Vega 스펙과 쿼리를 계속 수정할 수 있습니다. 완료되면 상단에서 **Save as**를 클릭해 사용자 정의 플롯에 이름을 지정해 두고 재사용할 수 있도록 합니다. 그런 다음 **Apply from panel library**를 클릭해 플롯을 완성합니다.
