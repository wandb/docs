이 페이지에서는 라인 플롯 설정을 자세히 설명합니다. 라인 플롯 사용 전반에 대한 내용은 [Line plots overview](/ko/models/app/features/panels/line-plot)를 참조하세요.

<div id="data-settings">
  ## 데이터 설정
</div>

<div id="x-axis">
  ### X-axis
</div>

<Frame>
  <img src="/images/app_ui/reference_x_axis.png" alt="X-Axis 선택" />
</Frame>

`wandb.Run.log()`으로 기록한 정수 또는 부동소수점 값으로 X축 범위를 설정합니다.

사용 가능한 시간 기준 X축 옵션은 다음과 같습니다:

* **Step**: `wandb.Run.log()`이 호출될 때마다 1씩 증가합니다. 모델에서 기록된 학습 step 수를 반영합니다. (기본값)
* **Relative Time (Wall)**: 프로세스가 시작된 이후의 실제 경과 시간입니다. 실행을 시작한 후 하루 동안 일시 중지했다가 다시 재개하여 로그를 기록하면, 해당 지점은 24시간 위치에 나타납니다.
* **Relative Time (Process)**: 실행 중인 프로세스 내부에서의 경과 시간입니다. 실행을 시작하여 10초 동안 실행한 후 하루 동안 일시 중지했다가 다시 재개하면, 해당 지점은 10초 위치에 나타납니다.
* **Wall Time**: 그래프에 있는 첫 번째 실행이 시작된 시점부터 경과한 분 수입니다.
* **X range**: 기본적으로 X축의 최소값부터 최대값까지입니다. 최소값과 최대값을 사용자 지정할 수 있습니다.

<div id="y-axis">
  ### Y축
</div>

Y축 변수에는 `wandb.Run.log()`으로 로깅한 임의의 정수 또는 실수 값을 설정할 수 있습니다. 단일 값, 값의 배열, 또는 값의 히스토그램을 지정할 수 있습니다. 어떤 변수에 대해 1500개를 초과하는 데이터 포인트를 로깅한 경우, W&amp;B는 최대 1500개가 되도록 샘플링합니다.

<Note>
  실행 테이블에서 실행 색상을 변경하여 Y축 선의 색상을 사용자 지정할 수 있습니다.
</Note>

사용 가능한 Y축 옵션:

* **Y축 범위**: 기본값은 메트릭 값 중 0을 포함한 가장 작은 양의 값부터 메트릭의 가장 큰 값까지입니다. 최소값과 최대값을 직접 설정할 수 있습니다.

<div id="point-aggregation-method">
  ### 포인트 집계 방식
</div>

데이터 포인트를 표시할 때 사용할 샘플링 모드를 선택하세요:

* **랜덤 샘플링**(기본값): [랜덤 샘플링](/ko/models/app/features/panels/line-plot/sampling/#random-sampling)을 참고하세요.
* **전체 정밀도(Full fidelity)**: [전체 정밀도(Full fidelity)](/ko/models/app/features/panels/line-plot/sampling/#full-fidelity)를 참고하세요.

<div id="smoothing">
  ### 스무딩
</div>

[스무딩 계수](/ko/models/support/formula_smoothing_algorithm/)를 0과 1 사이의 값으로 설정합니다. 0은 스무딩을 적용하지 않는 것이고, 1은 최대 스무딩을 의미합니다.

사용 가능한 스무딩 방법:

* **시간 가중 EMA**(기본값): 이전 지점의 가중치를 지수적으로 감소시키는 방식으로 시계열 데이터를 스무딩하는 기법입니다.
* **이동 평균**: 특정 x 값의 앞뒤 구간에 있는 지점들의 평균값으로 해당 지점을 대체합니다.
* **가우시안**: 스무딩 파라미터로 지정한 표준 편차를 갖는 가우시안 분포에 해당하는 가중치를 사용하여, 지점들의 가중 평균을 계산합니다.
* **스무딩 없음**

자세한 내용은 [선 그래프 스무딩](/ko/models/app/features/panels/line-plot/smoothing)을 참조하세요.

<div id="ignore-outliers">
  ### 이상치 무시
</div>

기본 플롯의 최소 및 최대 스케일에서 이상치를 제외하도록 플롯의 스케일을 재조정합니다. 이 설정의 영향은 플롯의 샘플링 모드에 따라 달라집니다:

* **Random sampling mode**: 이상치를 무시하면 플롯에서 하위 5% 미만과 상위 95% 초과 지점을 생략합니다.
* **Full fidelity mode**: 이상치를 무시하면 모든 지점을 표시하되, 각 버킷을 마지막 값으로 축약해 보여주고, 하위 5% 미만과 상위 95% 초과 영역을 음영 처리합니다.

<div id="max-runs-or-groups">
  ### 최대 실행 또는 그룹 수
</div>

기본적으로 플롯에는 실행 목록 또는 실행 세트에서 처음 10개의 실행 또는 실행 그룹만 포함됩니다. 정렬 순서를 변경하여 어떤 실행 또는 그룹을 표시할지 제어할 수 있습니다.

<Note>
  워크스페이스는 구성과 관계없이 최대 1000개의 실행만 표시할 수 있습니다.
</Note>

<div id="grouping-settings">
  ## 그룹화 설정
</div>

그룹화를 켜서 모든 실행을 집계하거나, 개별 변수를 기준으로 실행을 그룹화할 수 있습니다. 실행 테이블에서 그룹화를 켜면, 해당 그룹이 그래프에 자동으로 반영됩니다.

* **실행 그룹화**: 플롯에서 실행 그룹화를 켭니다. 아래 플롯의 음영 범위를 설정하려면 이 옵션이 필요합니다.
* **그룹 기준**: 선택적으로 열을 하나 선택합니다. 해당 열에서 값이 동일한 모든 실행이 하나의 그룹으로 묶입니다.
* **집계**: 그래프에 표시되는 선의 값입니다. 선택지는 그룹의 평균(mean), 중앙값(median), 최소값(min), 최대값(max)입니다.
* **범위**: 전체 원본 해상도의 선 그래프에서 음영 영역을 설정합니다. 선택지는 Min/Max, Std Dev, Std Err, 또는 None입니다.

<div id="chart-settings">
  ## 차트 설정
</div>

제목과 범례 표시를 설정합니다:

* **패널 제목**: 패널 상단에 표시되는 제목입니다.
* **X축 제목**: X축의 레이블입니다.
* **Y축 제목**: Y축의 레이블입니다.
* **범례**: 범례의 표시 여부와 위치를 설정합니다.

<div id="legend-settings">
  ## 범례 설정
</div>

범례를 사용자 지정하여 실행에서 로깅된 config 값과 생성 시각, 실행을 생성한 사용자 등의 메타데이터를 표시할 수 있습니다.

<div id="legend-template">
  ### 범례 템플릿
</div>

범례 이름에 사용할 템플릿을 정의합니다.

1. 톱니바퀴 아이콘을 클릭해 플롯 설정을 엽니다.
2. **Display preferences** 탭으로 이동합니다.
3. **Advanced legend**를 펼친 다음, 범례 템플릿을 지정합니다.
4. **Apply**를 클릭합니다.

예:

`${run:displayName} - ${config:dropout}`은(는) `royal-sweep - 0.5`와 같은 범례 이름을 생성합니다. 여기서 `royal-sweep`은 실행 이름이고, `0.5`는 `dropout`이라는 이름의 config 매개변수입니다.

<div id="point-specific-values">
  ### 포인트별 값
</div>

차트 위에 마우스를 올렸을 때 십자선에 포인트별 값을 표시하려면 `[[ ]]` 안에 값을 설정합니다.

1. 기어 아이콘을 클릭해 플롯 설정을 엽니다.
2. **Display preferences** 탭으로 이동합니다.
3. 탭의 아래쪽에서 하나 이상의 플롯 메트릭에 대해 포인트별 값을 설정합니다.
4. **Apply**를 클릭합니다.

예시: `[[ $x: $y ($original) ]]`는 &quot;2: 3 (2.9)&quot;와 같이 표시됩니다.

`[[ ]]` 안에서 지원되는 값은 다음과 같습니다:

| Value         | Meaning                                       |
| ------------  | --------------------------------------------- |
| `${x}`        | X값                                           |
| `${y}`        | Y값 (스무딩 조정을 포함)                      |
| `${original}` | 스무딩 조정을 포함하지 않은 Y값              |
| `${mean}`     | 그룹화된 실행의 평균                          |
| `${stddev}`   | 그룹화된 실행의 표준편차                      |
| `${min}`      | 그룹화된 실행의 최소값                        |
| `${max}`      | 그룹화된 실행의 최대값                        |
| `${percent}`  | 전체에서 차지하는 비율 (누적 영역 차트용)     |

<div id="expressions">
  ## 수식(Expressions)
</div>

파생 지표를 만들기 위해 사용자 지정 계산식을 추가하세요.

<div id="y-axis-expressions">
  ### Y-axis expressions
</div>

메트릭에서 파생된 값을 그래프로 그립니다. 예를 들어 `1-accuracy` 또는 다른 산술 표현식을 계산할 수 있습니다. 현재는 단일 메트릭을 플로팅할 때만 지원됩니다.

지원되는 연산자: `+`, `-`, `*`, `/`, `%`, 그리고 `**` (지수 연산).

로그된 메트릭과 하이퍼파라미터 같은 구성 값을 함께 사용해 사용자 지정 선을 계산할 수 있습니다.

<div id="x-axis-expressions">
  ### X축 표현식
</div>

사용자 지정 표현식을 사용하여 계산된 값으로 x축을 재조정합니다.

사용할 수 있는 변수:

* `_step`: 기본 x축 값입니다.
* `${summary:value}`: summary 값을 참조합니다.

<div id="multi-metric-panel-expressions">
  ### 다중 메트릭 패널 표현식
</div>

정규식을 사용해 여러 메트릭을 하나의 라인 플롯에 함께 표시할 수 있습니다(앞으로 해당 패턴으로 기록될 메트릭까지 포함). 자세한 내용은 [라인 플롯 추가](/ko/models/app/features/panels/line-plot#multi-metric-line-plot)를 참고하세요.

예를 들어:

* 각 레이어의 메트릭마다 별도의 패널을 만드는 대신, 하나의 패널에서 모두 함께 볼 수 있습니다. 예를 들어 `layer_0_loss`, `layer_1_loss`, `layer_2_loss`처럼 일관된 이름으로 메트릭을 로깅했다면, `layer_\d+_loss`와 같은 정규식을 사용해 모든 레이어 loss를 하나의 플롯에 표시할 수 있습니다.
* 공통 네이밍 패턴을 공유하는 모든 메트릭을 일치시킬 수 있습니다. 예:
  * `train_.*`는 `train_loss`, `train_accuracy`, `train_f1_score`와 같은 모든 학습 메트릭과 일치합니다.
  * `.*_accuracy`는 `train_accuracy`, `val_accuracy`, `test_accuracy`처럼 서로 다른 데이터셋에 대한 accuracy 메트릭과 일치합니다.
* Alternation을 사용해 원하는 메트릭만 일치시킬 수 있습니다. 예를 들어, 비캡처 그룹 `(?:layer_0|layer_10)_loss`는 중간 레이어를 제외하고 첫 번째와 열 번째 레이어의 loss만 일치시킵니다.
