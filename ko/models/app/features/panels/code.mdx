title: 코드 저장 및 비교

기본적으로 W&amp;B는 최신 Git 커밋 해시만 저장합니다. UI에서 실험 간 코드를 동적으로 비교할 수 있도록 추가 코드 기능을 활성화할 수 있습니다.

`wandb` 0.8.28 버전부터는 `wandb.init()`을 호출하는 메인 학습 스크립트의 코드를 W&amp;B가 저장할 수 있습니다.

<div id="save-library-code">
  ## 라이브러리 코드 저장
</div>

코드 저장을 활성화하면 W&amp;B는 `wandb.init()`을 호출한 파일의 코드를 저장합니다. 추가 라이브러리 코드를 저장하려면 다음 세 가지 방법을 사용할 수 있습니다:

<div id="call-wandbrunlog_code-after-calling-wandbinit">
  ### `wandb.init()`를 호출한 후 `wandb.Run.log_code(".")`를 호출하세요
</div>

```python
import wandb

with wandb.init() as run:
  run.log_code(".")
```

<div id="pass-a-settings-object-to-wandbinit-with-code_dir-set">
  ### `code_dir`를 설정한 settings 객체를 `wandb.init()`에 전달하기
</div>

```python
import wandb

wandb.init(settings=wandb.Settings(code_dir="."))
```

이 설정은 현재 디렉터리와 모든 하위 디렉터리에 있는 모든 Python 소스 코드 파일을 [아티팩트](/ko/models/ref/python/experiments/artifact)로 저장합니다. 저장할 소스 코드 파일의 유형과 위치를 더 세밀하게 제어하려면 [참고 문서](/ko/models/ref/python/experiments/run#log_code)를 확인하세요.

<div id="set-code-saving-in-the-ui">
  ### UI에서 코드 저장 설정
</div>

코드 저장을 코드에서 설정하는 것뿐 아니라, W&amp;B 계정의 Settings에서 이 기능을 토글할 수도 있습니다. 이 설정을 켜면 계정과 연결된 모든 팀에서 코드 저장이 활성화됩니다.

> 기본적으로 W&amp;B는 모든 팀에 대해 코드 저장을 비활성화합니다.

1. W&amp;B 계정에 로그인합니다.
2. **Settings** &gt; **Privacy**로 이동합니다.
3. **Project and content security** 아래에서 **Disable default code saving** 토글을 켭니다.

1. 페이지 오른쪽 상단에서 **Add panels** 버튼을 선택합니다.
2. **TEXT AND CODE** 드롭다운을 펼친 다음 **Code**를 선택합니다.

<div id="jupyter-session-history">
  ## Jupyter 세션 기록
</div>

W&amp;B는 Jupyter 노트북 세션에서 실행된 코드의 히스토리를 저장합니다. Jupyter 환경에서 **wandb.init()**을 호출하면, W&amp;B는 현재 세션에서 실행된 코드의 히스토리를 담은 Jupyter 노트북을 자동으로 저장하기 위한 후크를 추가합니다.

1. 코드를 포함하는 프로젝트 Workspace로 이동합니다.
2. 왼쪽 탐색 모음에서 **Artifacts** 탭을 선택합니다.
3. **code** 아티팩트를 확장합니다.
4. **Files** 탭을 선택합니다.

<Frame>
  <img src="/images/app_ui/jupyter_session_history.gif" alt="Jupyter 세션 기록" />
</Frame>

여기에는 세션에서 실행된 셀과 iPython의 display 메서드를 호출하여 생성된 모든 출력이 함께 표시됩니다. 이를 통해 특정 실행에서 Jupyter 내에서 어떤 코드가 정확히 실행되었는지 확인할 수 있습니다. 가능한 경우 W&amp;B는 code 디렉토리에서 찾을 수 있는 노트북의 최신 버전도 함께 저장합니다.
