Run Comparer를 사용해 프로젝트의 run 전반에서 차이점과 유사점을 확인하세요.

<div id="add-a-run-comparer-panel">
  ## Run Comparer 패널 추가
</div>

1. 페이지 오른쪽 상단의 **Add panels** 버튼을 선택합니다.
2. **Evaluation** 섹션에서 **Run comparer**를 선택합니다.

<div id="use-run-comparer">
  ## Run Comparer 사용
</div>

Run Comparer는 프로젝트에서 현재 표시되는 처음 10개의 run에 대한 설정과 로깅된 메트릭을 run별로 한 열씩 보여줍니다.

* 비교할 run을 변경하려면 왼쪽의 run 목록에서 검색, 필터링, 그룹화 또는 정렬을 수행하세요. Run Comparer는 자동으로 업데이트됩니다.
* Run Comparer 상단의 검색 필드를 사용해 Python 버전이나 run의 생성 시간과 같은 메타데이터 키 또는 설정 키를 필터링하거나 검색하세요.
* 차이를 빠르게 확인하고 동일한 값을 숨기려면 패널 상단에서 **Diff only**를 켜세요.
* 열 너비 또는 행 높이를 조정하려면 패널 상단의 서식 버튼을 사용하세요.
* 설정 또는 메트릭 값을 복사하려면 값 위에 마우스를 올린 다음 복사 버튼을 클릭하세요. 화면에 모두 표시되지 않을 정도로 길어도 전체 값이 복사됩니다.

<Note>
  기본적으로 Run Comparer는 [`job_type`](/ko/models/ref/python/functions/init)의 값이 다른 run을 구분하지 않습니다. 즉, 하나의 프로젝트 내에서 서로 비교하기 적절하지 않은 run도 비교할 수 있습니다. 예를 들어, 트레이닝 run과 모델 평가 run을 비교할 수 있습니다. 트레이닝 run에는 run 로그, 하이퍼파라미터, 트레이닝 loss 메트릭, 그리고 모델 자체가 포함될 수 있습니다. 평가 run은 모델을 사용해 새로운 트레이닝 데이터에서 모델의 성능을 확인할 수 있습니다.

  Runs Table에서 run 목록을 검색, 필터링, 그룹화 또는 정렬하면 Run Comparer는 자동으로 업데이트되어 처음 10개의 run을 비교합니다. Runs Table에서 필터링하거나 검색하고, 예를 들어 `job_type`을 기준으로 목록을 필터링하거나 정렬해 서로 유사한 run을 비교하세요. 자세한 내용은 [run 필터링](/ko/models/runs/filter-runs/)을 참조하세요.
</Note>
