Workspace 패널 시각화를 사용해 키별로 [로깅한 데이터](/ko/models/ref/python/experiments/run.md/#method-runlog)를 탐색하고, 하이퍼파라미터와 출력 메트릭 간의 관계를 시각화하는 등 다양한 작업을 수행할 수 있습니다.

<div id="workspace-modes">
  ## Workspace 모드
</div>

W&amp;B 프로젝트는 두 가지 Workspace 모드를 지원합니다. 워크스페이스 이름 옆의 아이콘이 해당 모드를 표시합니다.

| Icon                                                                                  | Workspace mode                                                                                                                                                                                                                                                         |
| ------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| <img src="/images/app_ui/automated_workspace.svg" alt="자동 워크스페이스 아이콘" width="32px" /> | **자동 워크스페이스**는 프로젝트에 로깅된 모든 키에 대해 패널을 자동으로 생성합니다. 자동 워크스페이스를 선택하세요:<ul><li>프로젝트에서 사용 가능한 모든 데이터를 빠르게 시각화하며 시작하고 싶을 때.</li><li>적은 수의 키만 로깅하는 소규모 프로젝트의 경우.</li><li>더 폭넓은 분석을 수행할 때.</li></ul>자동 워크스페이스에서 패널을 삭제한 경우, [Quick add](#quick-add)를 사용해 다시 생성할 수 있습니다.        |
| <img src="/images/app_ui/manual_workspace.svg" alt="수동 워크스페이스 아이콘" width="32px" />    | **수동 워크스페이스**는 빈 상태에서 시작하며, 사용자가 의도적으로 추가한 패널만 표시합니다. 수동 워크스페이스를 선택하세요:<ul><li>프로젝트에 로깅된 키 중 일부에만 주로 관심이 있을 때.</li><li>더 집중적인 분석을 수행할 때.</li><li>불필요한 패널 로딩을 줄여 워크스페이스 성능을 향상시키고 싶을 때.</li></ul>[Quick add](#quick-add)를 사용해 수동 워크스페이스와 해당 섹션을 유용한 시각화로 빠르게 채울 수 있습니다. |

워크스페이스가 패널을 생성하는 방식을 변경하려면 [워크스페이스를 재설정](#reset-a-workspace)하세요.

<Note>
  **워크스페이스 변경 사항 되돌리기**

  워크스페이스에서 변경 사항을 되돌리려면 왼쪽 화살표 모양의 Undo 버튼을 클릭하거나 **CMD + Z** (macOS) 또는 **CTRL + Z** (Windows / Linux)를 누르세요.
</Note>

<div id="reset-a-workspace">
  ## Workspace 재설정
</div>

Workspace를 재설정하려면:

1. Workspace 상단에서 작업 메뉴 `...`를 클릭합니다.
2. **Reset workspace**를 클릭합니다.

<div id="configure-the-workspace-layout">
  ## 워크스페이스 레이아웃 설정
</div>

워크스페이스 레이아웃을 설정하려면 워크스페이스 상단의 **Settings**를 클릭한 다음 **Workspace layout**을 클릭하세요.

* **Hide empty sections during search** (기본적으로 켜져 있음)
* **Sort panels alphabetically** (기본적으로 꺼져 있음)
* **Section organization** (기본적으로 첫 번째 접두사 기준으로 그룹화됨). 이 설정을 수정하려면:
  1. 자물쇠 아이콘을 클릭하세요.
  2. 섹션 내 패널을 어떻게 그룹화할지 선택하세요.

워크스페이스 선형 플롯의 기본 설정을 설정하려면 [선형 플롯](/ko/models/app/features/panels/line-plot/)을 참고하세요.

<div id="configure-a-sections-layout">
  ### 섹션 레이아웃 설정
</div>

섹션의 레이아웃을 설정하려면 섹션의 톱니바퀴 아이콘을 클릭한 다음 **Display preferences**를 클릭하세요.

* **툴팁에서 색상이 적용된 run 이름 표시 켜기/끄기** (기본값: 켜짐)
* **보조 차트 툴팁에서 강조된 run만 표시** (기본값: 꺼짐)
* **툴팁에 표시되는 run 개수** (하나의 run, 모든 run, 또는 **Default**)
* **주요 차트 툴팁에서 전체 run 이름 표시** (기본값: 꺼짐)

<div id="view-a-panel-in-full-screen-mode">
  ## 패널을 전체 화면 모드로 보기
</div>

전체 화면 모드에서는 run 선택기가 표시되며, 패널에서는 일반적으로 사용하는 1,000 버킷 대신 10,000 버킷을 사용하는 풀 피델리티 샘플링 모드 플롯이 적용됩니다.

패널을 전체 화면 모드로 보려면:

1. 패널 위에 마우스를 올립니다.
2. 패널의 작업 메뉴 `...`를 클릭한 다음, 뷰파인더나 정사각형의 네 모서리 윤곽처럼 보이는 전체 화면 버튼을 클릭합니다.
   <Frame>
     <img src="/images/app_ui/panel_fullscreen.png" alt="전체 화면 패널" />
   </Frame>
3. 전체 화면 모드로 보고 있는 동안 [패널을 공유](#share-a-panel)하면, 생성된 링크는 자동으로 전체 화면 모드로 열립니다.

* 전체 화면 모드에서 패널의 워크스페이스로 돌아가려면 페이지 상단의 왼쪽 화살표를 클릭합니다.
* 전체 화면 모드를 종료하지 않고 섹션의 패널을 탐색하려면, 패널 아래의 **Previous** 및 **Next** 버튼이나 키보드의 왼쪽 및 오른쪽 화살표 키를 사용합니다.
* 패널에 더 많은 공간을 확보하려면 **Cmd+.** (macOS) 또는 **Ctrl+.** (Windows/Linux)를 눌러 run 선택기를 최소화합니다.
* [미디어 패널](/ko/models/app/features/panels/media)의 이미지를 전체 화면 모드로 볼 때는 키보드 단축키로 확대 또는 축소하거나, 확대를 재설정하거나, 화면에 맞게 확대할 수 있습니다. [Keyboard shortcuts](/ko/models/app/keyboard-shortcuts#media-panels)를 참조하세요.

다른 전체 화면 및 패널 단축키는 [Keyboard shortcuts](/ko/models/app/keyboard-shortcuts)를 참조하세요.

<div id="add-panels">
  ## Add panels
</div>

이 섹션에서는 Workspace에 패널을 추가하는 여러 가지 방법을 설명합니다.

<div id="add-a-panel-manually">
  ### 패널 수동으로 추가하기
</div>

Workspace에 패널을 전역 또는 섹션 수준에서 한 번에 하나씩 추가할 수 있습니다.

1. 패널을 전역으로 추가하려면 패널 검색 필드 근처의 컨트롤 바에서 **Add panels**를 클릭합니다.
2. 패널을 특정 섹션에 바로 추가하려면 해당 섹션의 액션 `...` 메뉴를 클릭한 다음 **+ Add panels**를 클릭합니다.
3. 추가할 패널 유형(예: 차트)을 선택합니다. 기본값이 선택된 상태로 패널의 설정 세부 정보가 표시됩니다.
4. 필요에 따라 패널과 표시 기본 설정을 사용자 지정합니다. 설정 옵션은 선택한 패널 유형에 따라 달라집니다. 각 패널 유형의 옵션에 대해 자세히 알아보려면 아래의 관련 섹션(예: [Line plots](/ko/models/app/features/panels/line-plot/) 또는 [Bar plots](/ko/models/app/features/panels/bar-plot/))을 참고하세요.
5. **Apply**를 클릭합니다.

  <img src="/images/app_ui/add_single_panel.gif" alt="패널 추가 데모" />
</Frame>

<div id="quick-add-panels">
  ### Quick add panels
</div>

**Quick add**를 사용하면 선택한 각 키에 대해 패널을 자동으로 추가할 수 있으며, Workspace 전체 또는 섹션 수준에서 추가할 수 있습니다.

<Note>
  삭제된 패널이 없는 자동화된 Workspace의 경우, 이미 모든 로깅된 키에 대한 패널이 Workspace에 포함되어 있으므로 **Quick add** 옵션이 표시되지 않습니다. 삭제한 패널을 다시 추가하려면 **Quick add**를 사용할 수 있습니다.
</Note>

1. **Quick add**를 사용해 패널을 전역으로 추가하려면, 패널 검색 필드 근처의 컨트롤 바에서 **Add panels**를 클릭한 다음 **Quick add**를 클릭합니다.
2. **Quick add**를 사용해 패널을 섹션에 직접 추가하려면, 섹션의 액션 `...` 메뉴를 클릭하고 **Add panels**를 클릭한 다음 **Quick add**를 클릭합니다.
3. 패널 목록이 표시됩니다. 체크 표시된 각 패널은 이미 Workspace에 포함되어 있습니다.
   * 사용 가능한 모든 패널을 추가하려면, 목록 상단에서 **Add `<N>` panels** 버튼을 클릭합니다. 그러면 **Quick Add** 목록이 닫히고 새 패널이 Workspace에 표시됩니다.
   * 목록에서 개별 패널을 추가하려면, 패널의 행 위에 마우스를 올린 다음 **Add**를 클릭합니다. 추가하려는 각 패널에 대해 이 step을 반복한 다음, 오른쪽 상단의 **X**를 클릭해 **Quick Add** 목록을 닫습니다. 새 패널이 Workspace에 표시됩니다.
4. 선택적으로 패널 Settings을 사용자 지정합니다.

<div id="share-a-panel">
  ## 패널 공유하기
</div>

이 섹션에서는 링크로 패널을 공유하는 방법을 설명합니다.

링크로 패널을 공유하려면 다음 중 하나를 수행하세요.

* 패널을 전체 화면 모드로 보고 있을 때 브라우저에서 URL을 복사하세요.

* 작업 메뉴 `...`를 클릭하고 **Copy panel URL**을 선택하세요.

링크를 사용자나 팀과 공유하세요. 링크에 액세스하면 패널이 [전체 화면 모드](#view-a-panel-in-full-screen-mode)로 열립니다.

전체 화면 모드에서 패널의 Workspace로 돌아가려면 페이지 상단의 왼쪽 화살표를 클릭하세요.

<div id="compose-a-panels-full-screen-link-programmatically">
  ### 패널의 전체 화면 링크를 프로그래밍 방식으로 생성하기
</div>

[자동화 생성](/ko/models/automations/)과 같은 특정 상황에서는 패널의 전체 화면 URL을 포함하는 것이 유용할 수 있습니다. 이 섹션에서는 패널의 전체 화면 URL 형식을 보여줍니다. 다음 예시에서 대괄호 안의 entity, 프로젝트, 패널, 섹션 이름을 실제 값으로 바꾸십시오.

```text
https://wandb.ai/<ENTITY_NAME>/<PROJECT_NAME>?panelDisplayName=<PANEL_NAME>&panelSectionName=<SECTON_NAME>
```

같은 섹션에 동일한 이름의 패널이 여러 개 있는 경우, 이 URL은 그 이름을 가진 첫 번째 패널을 엽니다.

<div id="embed-or-share-a-panel-on-social-media">
  ### 패널을 임베드하거나 소셜 미디어에서 공유하기
</div>

웹사이트에 패널을 임베드하거나 소셜 미디어에서 공유하려면 해당 패널은 링크가 있는 모든 사용자가 볼 수 있어야 합니다. 프로젝트가 비공개인 경우 프로젝트 구성원만 패널을 볼 수 있습니다. 프로젝트가 공개인 경우 링크가 있는 모든 사용자가 패널을 볼 수 있습니다.

패널을 임베드하거나 소셜 미디어에서 공유할 코드를 가져오려면:

1. Workspace에서 패널 위에 마우스를 올린 다음 작업 메뉴 `...`를 클릭하세요.
2. **Share** 탭을 클릭하세요.
3. **Only those who are invited have access**를 **Anyone with the link can view**로 변경하세요. 그렇지 않으면 다음 단계의 옵션을 사용 가능하지 않습니다.
4. **Share on Twitter**, **Share on Reddit**, **Share on LinkedIn**, 또는 **Copy embed link**를 선택하세요.

<div id="email-a-panel-report">
  ### 패널 리포트 이메일로 보내기
</div>

단일 패널을 독립형 리포트로 이메일로 보내려면 다음과 같이 하세요.

1. 패널에 마우스를 올린 다음 패널의 작업 메뉴 `...`를 클릭하세요.
2. **Share panel in report**를 클릭하세요.
3. **Invite** 탭을 선택하세요.
4. 이메일 주소 또는 사용자 이름을 입력하세요.
5. 선택 사항으로 **can view**를 **can edit**로 변경하세요.
6. **Invite**를 클릭하세요. W&amp;B는 공유하는 패널만 포함된 리포트로 연결되는 클릭 가능한 링크가 담긴 이메일을 사용자에게 보냅니다.

[패널을 공유](#share-a-panel)할 때와 달리, 수신자는 이 리포트에서 Workspace로 이동할 수 없습니다.

<div id="manage-panels">
  ## 패널 관리
</div>

<div id="edit-a-panel">
  ### 패널 편집
</div>

패널을 편집하려면:

1. 연필 아이콘을 클릭합니다.
2. 패널의 설정을 수정합니다.
3. 패널 유형을 변경하려면 다른 유형을 선택한 후 설정을 설정합니다.
4. **Apply**를 클릭합니다.

<div id="move-a-panel">
  ### 패널 이동
</div>

패널을 다른 섹션으로 이동하려면 패널의 드래그 핸들을 사용하세요. 대신 목록에서 새 섹션을 선택해 이동하려면 다음과 같이 하세요.

1. 필요한 경우 마지막 섹션 뒤에 있는 **Add section**을 클릭하여 새 섹션을 생성하세요.
2. 패널의 액션 `...` 메뉴를 클릭하세요.
3. **Move**를 클릭한 다음 새 섹션을 선택하세요.

드래그 핸들을 사용해 섹션 내에서 패널 순서를 바꿀 수도 있습니다.

<div id="duplicate-a-panel">
  ### 패널 복제
</div>

패널을 복제하려면 다음을 수행하세요.

1. 패널 상단에서 액션 `...` 메뉴를 클릭합니다.
2. **Duplicate**를 클릭합니다.

원하면 복제된 패널을 [사용자 지정하거나](#edit-a-panel) [이동](#move-a-panel)할 수 있습니다.

<div id="remove-panels">
  ### 패널 제거
</div>

패널을 제거하려면:

1. 패널 위에 마우스를 올립니다.
2. 액션 `...` 메뉴를 선택합니다.
3. **Delete**를 클릭합니다.

수동 워크스페이스에서 모든 패널을 제거하려면 해당 워크스페이스의 액션 `...` 메뉴를 클릭한 다음 **Clear all panels**를 클릭합니다.

자동 또는 수동 워크스페이스에서 모든 패널을 제거하려면 [워크스페이스를 재설정](#reset-a-workspace)할 수 있습니다. 기본 패널 세트로 시작하려면 **Automatic**을 선택하고, 패널이 없는 빈 워크스페이스로 시작하려면 **Manual**을 선택합니다.

<div id="manage-sections">
  ## 섹션 관리
</div>

기본적으로 Workspace의 섹션은 키의 logging 계층을 반영합니다. 그러나 수동 Workspace에서는 패널을 추가하기 시작한 후에만 섹션이 표시됩니다.

<div id="add-a-section">
  ### 섹션 추가
</div>

섹션을 추가하려면 마지막 섹션 뒤에 있는 **Add section**을 클릭하세요.

기존 섹션 앞이나 뒤에 새 섹션을 추가하려면, 대신 해당 섹션의 `...` 액션 메뉴를 클릭한 다음 **New section below** 또는 **New section above**를 클릭하세요.

<Warning>
  섹션 이름을 &quot;Section&quot;으로 지정하지 마세요. 제한 사항으로 인해 섹션 이름을 변경하기 전까지 이 섹션의 패널은 표시되지 않습니다.
</Warning>

<div id="manage-a-sections-panels">
  ### 섹션의 패널 관리
</div>

패널 수가 많은 섹션은 기본적으로 여러 페이지로 나뉘어 표시됩니다. 한 페이지에 표시되는 기본 패널 수는 패널의 설정과 섹션 내 패널 크기에 따라 달라집니다.

1. 패널 크기를 조정하려면 패널 위에 마우스를 올리고 드래그 핸들을 클릭한 다음 끌어서 크기를 조정합니다. 하나의 패널 크기를 조정하면 섹션의 모든 패널 크기도 함께 조정됩니다.
2. 섹션이 페이지로 나뉘어 있는 경우 한 페이지에 표시할 패널 수를 사용자 지정할 수 있습니다:
3. 섹션 상단에서 **1 to `<X>` of `<Y>`**를 클릭합니다. 여기서 `<X>`는 현재 표시되는 패널 수이고 `<Y>`는 전체 패널 수입니다.
4. 페이지당 표시할 패널 수를 선택합니다. 최대 100개까지 선택할 수 있습니다.
5. 섹션에서 패널을 삭제하려면:
6. 패널 위에 마우스를 올린 다음 해당 패널의 액션 `...` 메뉴를 클릭합니다.
7. **Delete**를 클릭합니다.

Workspace를 자동화된 Workspace로 재설정하면 삭제한 모든 패널이 다시 나타납니다.

<div id="rename-a-section">
  ### 섹션 이름 변경
</div>

섹션 이름을 변경하려면 해당 섹션의 액션 `...` 메뉴를 클릭한 다음 **섹션 이름 변경**을 클릭하세요.

<Warning>
  섹션 이름을 &quot;Section&quot;으로 지정하지 마세요. 제한 사항으로 인해 섹션 이름을 변경하기 전까지 이 섹션의 패널이 렌더링되지 않습니다.
</Warning>

<div id="delete-a-section">
  ### 섹션 삭제
</div>

섹션을 삭제하려면 해당 `...` 메뉴를 클릭한 다음 **Delete section**을 클릭하세요. 그러면 섹션과 해당 패널이 삭제됩니다.
