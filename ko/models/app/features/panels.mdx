워크스페이스 패널을 활용해 키를 기준으로 [기록된 데이터](/ko/models/ref/python/experiments/run.md/#method-runlog)를 탐색하고, 하이퍼파라미터와 출력 지표 간의 관계를 시각화하는 등 다양한 작업을 수행할 수 있습니다.

<div id="workspace-modes">
  ## 워크스페이스 모드
</div>

W&amp;B 프로젝트는 두 가지 워크스페이스 모드를 지원합니다. 워크스페이스 이름 옆의 아이콘으로 현재 모드를 확인할 수 있습니다.

| Icon | Workspace mode |
| --- | --- |
| <img src="/images/app_ui/automated_workspace.svg" alt="automated workspace icon" width="32px" /> | **자동 워크스페이스**는 프로젝트에서 로깅된 모든 키에 대한 패널을 자동으로 생성합니다. 다음과 같은 경우 자동 워크스페이스를 선택하세요:<ul><li>프로젝트의 사용 가능한 모든 데이터를 빠르게 시각화하며 시작하고 싶을 때</li><li>로깅하는 키가 적은 작은 프로젝트인 경우</li><li>보다 폭넓게 분석하고자 할 때</li></ul>자동 워크스페이스에서 패널을 삭제한 경우 [Quick add](#quick-add)를 사용해 다시 생성할 수 있습니다. |
| <img src="/images/app_ui/manual_workspace.svg" alt="manual workspace icon" width="32px" /> | **수동 워크스페이스**는 빈 상태로 시작하며, 사용자가 명시적으로 추가한 패널만 표시합니다. 다음과 같은 경우 수동 워크스페이스를 선택하세요:<ul><li>프로젝트에서 로깅된 키 중 일부에만 주로 관심이 있을 때</li><li>보다 집중된 분석을 수행하고자 할 때</li><li>덜 유용한 패널 로딩을 피해 워크스페이스 성능을 향상시키고자 할 때</li></ul>[Quick add](#quick-add)를 사용해 수동 워크스페이스와 그 섹션을 유용한 시각화로 빠르게 채울 수 있습니다. |

워크스페이스에서 패널을 생성하는 방식을 변경하려면 [워크스페이스를 초기화](#reset-a-workspace)하세요.

<Note>
  **워크스페이스 변경 사항 되돌리기**

  워크스페이스의 변경 사항을 되돌리려면 Undo 버튼(왼쪽을 가리키는 화살표)을 클릭하거나 **CMD + Z**(macOS) 또는 **CTRL + Z**(Windows / Linux)를 누르세요.
</Note>

<div id="reset-a-workspace">
  ## 워크스페이스 재설정
</div>

워크스페이스를 재설정하려면:

1. 워크스페이스 상단의 작업 메뉴 `...`를 클릭합니다.
2. **Reset workspace**를 선택합니다.

<div id="configure-the-workspace-layout">
  ## 워크스페이스 레이아웃 구성
</div>

워크스페이스 레이아웃을 구성하려면 워크스페이스 상단 근처의 **Settings**를 클릭한 다음 **Workspace layout**을 클릭합니다.

* **Hide empty sections during search** (기본적으로 사용 설정되어 있음)
* **Sort panels alphabetically** (기본적으로 사용 안 함)
* **Section organization** (기본적으로 첫 번째 접두사 기준으로 그룹화됨). 이 설정을 변경하려면:
  1. 자물쇠 아이콘을 클릭합니다.
  2. 섹션 내에서 패널을 어떻게 그룹화할지 선택합니다.

워크스페이스의 라인 플롯 기본 설정을 구성하려면 [Line plots](/ko/models/app/features/panels/line-plot/)를 참고하세요.

<div id="configure-a-sections-layout">
  ### 섹션 레이아웃 구성하기
</div>

섹션의 레이아웃을 구성하려면 섹션의 톱니바퀴 아이콘을 클릭한 다음 **Display preferences**를 클릭합니다.

* **툴팁에서 실행 이름을 색상으로 표시할지 여부** (기본적으로 켜져 있음)
* **보조 차트 툴팁에 강조된 실행만 표시** (기본적으로 꺼져 있음)
* **툴팁에 표시할 실행 개수** (단일 실행, 모든 실행 또는 **Default**)
* **기본 차트 툴팁에 전체 실행 이름 표시** (기본적으로 꺼져 있음)

<div id="view-a-panel-in-full-screen-mode">
  ## 패널을 전체 화면 모드로 보기
</div>

전체 화면 모드에서는 실행 선택기가 표시되고, 패널은 기본 1000 버킷 대신 10,000 버킷의 고충실도 샘플링 모드 플롯을 사용합니다.

패널을 전체 화면 모드로 보려면:

1. 패널 위로 마우스를 가져갑니다.
2. 패널의 동작 메뉴 `...`를 클릭한 다음, 전체 화면 버튼을 클릭합니다. 이 버튼은 뷰파인더 또는 정사각형 네 모서리의 윤곽선처럼 보입니다.
   <Frame>
     <img src="/images/app_ui/panel_fullscreen.png" alt="전체 화면 패널" />
   </Frame>
3. 전체 화면 모드에서 [패널을 공유](#share-a-panel)하면, 생성된 링크는 자동으로 전체 화면 모드로 열립니다.

전체 화면 모드에서 패널의 워크스페이스로 돌아가려면 페이지 상단의 왼쪽 화살표를 클릭합니다. 전체 화면 모드에서 나가지 않고 섹션의 패널을 이동하려면, 패널 아래의 **Previous** 및 **Next** 버튼 또는 키보드의 왼쪽/오른쪽 화살표 키를 사용합니다.

<div id="add-panels">
  ## 패널 추가
</div>

이 섹션에서는 워크스페이스에 패널을 추가하는 여러 가지 방법을 설명합니다.

<div id="add-a-panel-manually">
  ### 패널을 수동으로 추가하기
</div>

워크스페이스에 패널을 전체 또는 섹션 단위로 한 번에 하나씩 추가할 수 있습니다.

1. 패널을 전체에 추가하려면 패널 검색 필드 근처의 컨트롤 바에서 **Add panels**를 클릭합니다.
2. 패널을 특정 섹션에만 직접 추가하려면 해당 섹션의 작업 `...` 메뉴를 클릭한 다음 **+ Add panels**를 클릭합니다.
3. 추가할 패널 유형(예: 차트)을 선택합니다. 기본값이 선택된 상태로 패널의 구성 세부 정보가 표시됩니다.
4. 필요에 따라 패널과 표시 설정을 사용자 지정합니다. 구성 옵션은 선택한 패널 유형에 따라 달라집니다. 각 패널 유형의 옵션에 대한 자세한 내용은 아래의 [Line plots](/ko/models/app/features/panels/line-plot/) 또는 [Bar plots](/ko/models/app/features/panels/bar-plot/)와 같은 관련 섹션을 참조하세요.
5. **Apply**를 클릭합니다.

  <img src="/images/app_ui/add_single_panel.gif" alt="패널 추가 데모" />
</Frame>

<div id="quick-add-panels">
  ### 패널을 빠르게 추가하기
</div>

**Quick add**를 사용하면 선택한 각 키에 대해 패널을 자동으로 추가할 수 있습니다. 전역 수준이나 섹션 수준에서 사용할 수 있습니다.

<Note>
  삭제된 패널이 없는 자동화된 워크스페이스의 경우, 이미 모든 로그된 키에 대한 패널이 포함되어 있으므로 **Quick add** 옵션이 표시되지 않습니다. 삭제한 패널을 다시 추가하려면 **Quick add**를 사용할 수 있습니다.
</Note>

1. **Quick add**를 사용해 패널을 전역으로 추가하려면, 패널 검색 필드 근처 컨트롤 바에서 **Add panels**를 클릭한 다음 **Quick add**를 클릭합니다.
2. **Quick add**를 사용해 패널을 섹션에 바로 추가하려면, 섹션의 `...` 작업 메뉴를 클릭하고 **Add panels**를 클릭한 다음 **Quick add**를 클릭합니다.
3. 패널 목록이 나타납니다. 체크 표시가 있는 패널은 이미 워크스페이스에 포함된 패널입니다.
   * 사용 가능한 모든 패널을 추가하려면, 목록 상단의 **Add `<N>` panels** 버튼을 클릭합니다. 그러면 **Quick Add** 목록이 닫히고 새 패널이 워크스페이스에 표시됩니다.
   * 목록에서 개별 패널을 추가하려면, 해당 패널의 행에 마우스를 올린 다음 **Add**를 클릭합니다. 추가하려는 각 패널에 대해 이 단계를 반복한 다음 오른쪽 상단의 **X**를 클릭해 **Quick Add** 목록을 닫습니다. 새 패널이 워크스페이스에 표시됩니다.
4. 필요에 따라 패널 설정을 사용자 지정합니다.

<div id="share-a-panel">
  ## 패널 공유하기
</div>

이 섹션에서는 링크로 패널을 공유하는 방법을 설명합니다.

링크로 패널을 공유하려면 다음 중 한 가지 방법을 사용할 수 있습니다:

* 패널을 전체 화면 모드로 보고 있을 때 브라우저에서 URL을 복사합니다.
* 작업 메뉴 `...`를 클릭한 다음 **Copy panel URL**을 선택합니다.

해당 링크를 사용자나 팀과 공유하세요. 링크에 접속하면 패널이 [전체 화면 모드](#view-a-panel-in-full-screen-mode)로 열립니다.

전체 화면 모드에서 패널의 워크스페이스로 돌아가려면 페이지 상단의 왼쪽 화살표 아이콘을 클릭합니다.

<div id="compose-a-panels-full-screen-link-programmatically">
  ### 패널의 전체 화면 링크를 프로그래밍 방식으로 구성하기
</div>

[자동화 생성](/ko/models/automations/)과 같은 특정 상황에서는 패널의 전체 화면 URL을 포함하는 것이 유용할 수 있습니다. 이 섹션에서는 패널 전체 화면 URL의 형식을 설명합니다. 다음 예시에서 대괄호 안의 엔터티, 프로젝트, 패널, 섹션 이름을 실제 값으로 바꾸십시오.

```text
https://wandb.ai/<ENTITY_NAME>/<PROJECT_NAME>?panelDisplayName=<PANEL_NAME>&panelSectionName=<SECTON_NAME>
```

같은 섹션에 동일한 이름의 패널이 여러 개 있을 경우 이 URL은 해당 이름을 가진 첫 번째 패널을 엽니다.

<div id="embed-or-share-a-panel-on-social-media">
  ### 패널을 임베드하거나 소셜 미디어에 공유하기
</div>

웹사이트에 패널을 임베드하거나 소셜 미디어에 공유하려면, 해당 패널이 링크를 가진 모든 사용자에게 공개되어 있어야 합니다. 프로젝트가 비공개인 경우에는 해당 프로젝트의 구성원만 패널을 볼 수 있습니다. 프로젝트가 공개인 경우에는 링크를 가진 누구나 패널을 볼 수 있습니다.

패널을 임베드하거나 소셜 미디어에 공유할 코드를 가져오려면:

1. 워크스페이스에서 패널 위에 마우스를 올린 다음 `...` 동작 메뉴를 클릭합니다.
2. **Share** 탭을 클릭합니다.
3. **Only those who are invited have access**를 **Anyone with the link can view**로 변경합니다. 그렇지 않으면 다음 단계의 옵션을 사용할 수 없습니다.
4. **Share on Twitter**, **Share on Reddit**, **Share on LinkedIn**, 또는 **Copy embed link**를 선택합니다.

<div id="email-a-panel-report">
  ### 패널 보고서를 이메일로 보내기
</div>

단일 패널을 별도의 보고서로 이메일로 보내려면:

1. 패널 위에 마우스를 올린 다음 패널의 작업 메뉴 `...`를 클릭합니다.
2. **Share panel in report**를 클릭합니다.
3. **Invite** 탭을 선택합니다.
4. 이메일 주소 또는 사용자 이름을 입력합니다.
5. 원하면 **can view**를 **can edit**로 변경합니다.
6. **Invite**를 클릭합니다. 그러면 W&amp;B가 공유하는 해당 패널만 포함된 보고서로 연결되는 클릭 가능한 링크가 들어 있는 이메일을 보냅니다.

[패널을 공유](#share-a-panel)할 때와는 달리, 수신자는 이 보고서에서 워크스페이스로 이동할 수 없습니다.

<div id="manage-panels">
  ## 패널 관리
</div>

<div id="edit-a-panel">
  ### 패널 편집하기
</div>

패널을 편집하려면:

1. 연필 아이콘을 클릭합니다.
2. 패널 설정을 변경합니다.
3. 패널 유형을 바꾸려면 원하는 유형을 선택한 다음 해당 설정을 구성합니다.
4. **Apply**를 클릭합니다.

<div id="move-a-panel">
  ### 패널 이동하기
</div>

패널을 다른 섹션으로 이동하려면 패널의 드래그 핸들을 사용하세요. 대신 목록에서 새 섹션을 선택하려면 다음을 수행합니다.

1. 필요한 경우 마지막 섹션 뒤에 있는 **Add section**을 클릭해 새 섹션을 만듭니다.
2. 패널의 작업 `...` 메뉴를 클릭합니다.
3. **Move**를 클릭한 다음 새 섹션을 선택합니다.

섹션 내에서 패널의 순서를 변경하려면 드래그 핸들을 사용할 수도 있습니다.

<div id="duplicate-a-panel">
  ### 패널 복제하기
</div>

패널을 복제하려면:

1. 패널 상단에서 작업 `...` 메뉴를 클릭합니다.
2. **Duplicate**를 클릭합니다.

원하는 경우 복제된 패널을 [편집](#edit-a-panel)하거나 [이동](#move-a-panel)할 수 있습니다.

<div id="remove-panels">
  ### 패널 제거
</div>

패널을 제거하려면 다음을 수행하세요.

1. 마우스를 패널 위에 올립니다.
2. `...` 동작 메뉴를 선택합니다.
3. **Delete**를 클릭합니다.

수동 워크스페이스에서 모든 패널을 제거하려면 해당 워크스페이스의 `...` 동작 메뉴를 클릭한 다음 **Clear all panels**를 클릭합니다.

자동 또는 수동 워크스페이스에서 모든 패널을 제거하려면 [워크스페이스를 초기화](#reset-a-workspace)하세요. 기본 패널 세트로 시작하려면 **Automatic**을 선택하고, 패널이 없는 빈 워크스페이스로 시작하려면 **Manual**을 선택합니다.

<div id="manage-sections">
  ## 섹션 관리
</div>

기본적으로 워크스페이스의 섹션은 키 로깅 계층 구조를 반영합니다. 하지만 수동 워크스페이스에서는 패널을 추가하기 시작한 후에야 섹션이 표시됩니다.

<div id="add-a-section">
  ### 섹션 추가
</div>

섹션을 추가하려면 마지막 섹션 뒤에 있는 **Add section**을 클릭한다.

기존 섹션 앞이나 뒤에 새 섹션을 추가하려면 해당 섹션의 `...` 액션 메뉴를 클릭한 다음 **New section below** 또는 **New section above**를 클릭한다.

<Warning>
  섹션 이름을 &quot;Section&quot;으로 지정하지 마라. 알려진 제한 사항으로 인해 이 섹션의 패널은 섹션 이름을 변경하기 전까지 렌더링되지 않는다.
</Warning>

<div id="manage-a-sections-panels">
  ### 섹션의 패널 관리하기
</div>

패널 수가 많은 섹션은 기본적으로 페이지로 나뉘어 표시됩니다. 한 페이지에 표시되는 기본 패널 수는 패널 설정과 섹션 내 패널 크기에 따라 달라집니다.

1. 패널 크기를 조정하려면 패널 위에 마우스를 올린 뒤 드래그 핸들을 클릭한 상태로 끌어 크기를 조정합니다. 한 패널의 크기를 변경하면 해당 섹션의 모든 패널 크기가 함께 변경됩니다.
2. 섹션이 페이지로 나뉘어 있는 경우, 한 페이지에 표시할 패널 수를 설정할 수 있습니다:
3. 섹션 상단에서 **1 to `<X>` of `<Y>`**를 클릭합니다. 여기서 `<X>`는 보이는 패널 수이고, `<Y>`는 전체 패널 수입니다.
4. 페이지당 표시할 패널 수를 최대 100개까지 선택합니다.
5. 섹션에서 패널을 삭제하려면:
6. 패널 위에 마우스를 올린 다음 해당 패널의 작업 `...` 메뉴를 클릭합니다.
7. **Delete**를 클릭합니다.

워크스페이스를 자동화된 워크스페이스로 재설정하면 삭제된 모든 패널이 다시 나타납니다.

<div id="rename-a-section">
  ### 섹션 이름 변경
</div>

섹션 이름을 변경하려면 해당 섹션의 `...` 작업 메뉴를 클릭한 다음 **Rename section**을 클릭합니다.

<Warning>
  섹션 이름을 &quot;Section&quot;으로 설정하지 마세요. 알려진 제약으로 인해 이 섹션의 패널은 섹션 이름을 변경하기 전까지 렌더링되지 않습니다.
</Warning>

<div id="delete-a-section">
  ### 섹션 삭제
</div>

섹션을 삭제하려면 해당 섹션의 `...` 메뉴를 클릭한 다음 **Delete section**을 선택합니다. 그러면 섹션과 그 안의 패널이 모두 삭제됩니다.
