워크스페이스 패널 시각화를 사용하면 키별로 [로깅된 데이터](/ko/models/ref/python/experiments/run.md/#method-runlog)를 탐색하고, 하이퍼파라미터와 결과 메트릭 간의 관계를 시각화하는 등 다양한 작업을 수행할 수 있습니다.

<div id="workspace-modes">
  ## 워크스페이스 모드
</div>

W&amp;B 프로젝트는 두 가지 워크스페이스 모드를 지원합니다. 워크스페이스 이름 옆의 아이콘으로 현재 모드를 확인할 수 있습니다.

| 아이콘                                                                                   | 워크스페이스 모드                                                                                                                                                                                                                                                                         |
| ------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| <img src="/images/app_ui/automated_workspace.svg" alt="자동 워크스페이스 아이콘" width="32px" /> | **자동 워크스페이스**는 프로젝트에 로깅된 모든 키에 대한 패널을 자동으로 생성합니다. 다음과 같은 경우 자동 워크스페이스를 선택하세요:<ul><li>프로젝트에서 사용할 수 있는 모든 데이터를 시각화하며 빠르게 시작하고 싶을 때</li><li>로깅하는 키 수가 적은 소규모 프로젝트인 경우</li><li>더 폭넓게 분석하고 싶을 때</li></ul>자동 워크스페이스에서 패널을 삭제한 경우 [Quick add](#quick-add)를 사용해 다시 만들 수 있습니다.             |
| <img src="/images/app_ui/manual_workspace.svg" alt="수동 워크스페이스 아이콘" width="32px" />    | **수동 워크스페이스**는 빈 상태에서 시작하며, 사용자가 직접 추가한 패널만 표시합니다. 다음과 같은 경우 수동 워크스페이스를 선택하세요:<ul><li>프로젝트에 로깅된 키 중 일부만 주로 보고 싶을 때</li><li>더 집중적으로 분석하고 싶을 때</li><li>나에게 덜 유용한 패널은 로드하지 않도록 해 워크스페이스 성능을 높이고 싶을 때</li></ul>[Quick add](#quick-add)를 사용하면 수동 워크스페이스와 그 섹션을 유용한 시각화로 빠르게 채울 수 있습니다. |

워크스페이스에서 패널이 생성되는 방식을 변경하려면 [워크스페이스를 재설정](#reset-a-workspace)하세요.

<Note>
  **워크스페이스 변경 사항 실행 취소**

  워크스페이스의 변경 사항을 실행 취소하려면 Undo 버튼(왼쪽 화살표)을 클릭하거나 **CMD + Z**(macOS) 또는 **CTRL + Z**(Windows / Linux)를 누르세요.
</Note>

<div id="reset-a-workspace">
  ## 워크스페이스 재설정
</div>

워크스페이스를 재설정하려면 다음 단계를 따르세요.

1. 워크스페이스 상단에서 **액션 (<Icon icon="ellipsis" iconType="solid" />)** 메뉴를 클릭합니다.
2. **워크스페이스 재설정**을 클릭합니다.

<div id="configure-the-workspace-layout">
  ## 워크스페이스 레이아웃 구성
</div>

워크스페이스 레이아웃을 구성하려면 워크스페이스 상단의 **Settings**를 클릭한 다음 **Workspace layout**을 클릭합니다.

* **검색 중 빈 섹션 숨기기**(기본적으로 켜짐)
* **패널을 알파벳순으로 정렬**(기본적으로 꺼짐)
* **섹션 구성**(기본적으로 첫 번째 접두사별로 그룹화됨). 이 설정을 변경하려면:
  1. 자물쇠 아이콘을 클릭합니다.
  2. 섹션 내 패널의 그룹화 방식을 선택합니다.

워크스페이스의 선형 플롯 기본값을 구성하려면 [선형 플롯](/ko/models/app/features/panels/line-plot/)을 참고하세요.

<div id="configure-a-sections-layout">
  ### 섹션의 레이아웃 구성하기
</div>

섹션의 레이아웃을 구성하려면 해당 섹션의 톱니바퀴 아이콘을 클릭한 다음 **Display preferences**를 클릭합니다.

* **툴팁에서 색상으로 구분된 run 이름 표시 켜기/끄기** (기본적으로 켜짐)
* **보조 차트 툴팁에 강조 표시된 run만 표시** (기본적으로 꺼짐)
* **툴팁에 표시할 run 수** (단일 run, 모든 run 또는 **Default**)
* **기본 차트 툴팁에 전체 run 이름 표시** (기본적으로 꺼짐)

<div id="view-a-panel-in-full-screen-mode">
  ## 패널을 전체 화면 모드로 보기
</div>

전체 화면 모드에서는 run 선택기가 표시되며, 패널은 기본 1000 bucket 대신 10,000 bucket을 사용하는 전체 충실도 샘플링 모드 플롯으로 표시됩니다.

패널을 전체 화면 모드로 보려면 다음과 같이 하세요.

1. 패널 위에 마우스를 올립니다.
2. 패널의 작업 **액션 (<Icon icon="ellipsis" iconType="solid" />)** 메뉴를 클릭한 다음, 뷰파인더 또는 정사각형 네 모서리의 윤곽선처럼 보이는 전체 화면 버튼을 클릭합니다.
   <Frame>
     <img src="/images/app_ui/panel_fullscreen.png" alt="전체 화면 패널" />
   </Frame>
3. 전체 화면 모드에서 패널을 보는 동안 [패널 공유](#share-a-panel)를 하면, 생성된 링크가 자동으로 전체 화면 모드로 열립니다.

* 전체 화면 모드에서 패널의 워크스페이스로 돌아가려면 페이지 상단의 왼쪽 화살표를 클릭합니다.
* 전체 화면 모드를 종료하지 않고 section 내 패널 사이를 이동하려면 패널 아래의 **Previous** 및 **Next** 버튼이나 키보드의 왼쪽/오른쪽 화살표 키를 사용합니다.
* 패널에 더 많은 공간을 확보하려면 **Cmd+.** (macOS) 또는 **Ctrl+.** (Windows/Linux)로 run 선택기를 최소화합니다.
* [media panel](/ko/models/app/features/panels/media)의 이미지를 전체 화면 모드로 볼 때는 키보드 단축키로 확대 또는 축소, 확대 재설정, 화면에 맞춤을 할 수 있습니다. [Keyboard shortcuts](/ko/models/app/keyboard-shortcuts#media-panels)를 참조하세요.

다른 전체 화면 및 패널 단축키는 [Keyboard shortcuts](/ko/models/app/keyboard-shortcuts)를 참조하세요.

<div id="add-panels">
  ## 패널 추가
</div>

이 섹션에서는 워크스페이스에 패널을 추가하는 여러 방법을 소개합니다.

<div id="add-a-panel-manually">
  ### 수동으로 패널 추가하기
</div>

워크스페이스에 패널을 한 번에 하나씩 전역 또는 섹션 수준으로 추가할 수 있습니다.

1. 패널을 전역으로 추가하려면 패널 검색 필드 근처의 컨트롤 바에서 **Add panels**를 클릭합니다.
2. 패널을 특정 섹션에 직접 추가하려면 섹션의 **액션 (<Icon icon="ellipsis" iconType="solid" />)** 메뉴를 클릭한 다음 **+ Add panels**를 클릭합니다.
3. 차트와 같이 추가할 패널 유형을 선택합니다. 기본값이 선택된 상태로 패널 설정 세부 정보가 표시됩니다.
4. 필요에 따라 패널과 표시 기본 설정을 사용자 지정합니다. 설정 옵션은 선택한 패널 유형에 따라 달라집니다. 각 패널 유형의 옵션에 대해 자세히 알아보려면 아래 관련 섹션(예: [선형 플롯](/ko/models/app/features/panels/line-plot/) 또는 [막대 플롯](/ko/models/app/features/panels/bar-plot/))을 참고하세요.
5. **Apply**를 클릭합니다.

  <img src="/images/app_ui/add_single_panel.gif" alt="패널 추가 데모" />
</Frame>

<div id="quick-add-panels">
  ### Quick add 패널
</div>

전역 또는 섹션 수준에서 선택한 각 키에 대해 패널을 자동으로 추가하려면 **Quick add**를 사용합니다.

<Note>
  삭제된 패널이 없는 자동화된 워크스페이스에서는 이미 로깅된 모든 키의 패널이 워크스페이스에 포함되어 있으므로 **Quick add** 옵션이 표시되지 않습니다. 삭제한 패널을 다시 추가할 때는 **Quick add**를 사용할 수 있습니다.
</Note>

1. **Quick add**를 사용해 전역으로 패널을 추가하려면 패널 검색 필드 근처의 컨트롤 바에서 **Add panels**를 클릭한 다음 **Quick add**를 클릭합니다.
2. **Quick add**를 사용해 섹션에 직접 패널을 추가하려면 섹션의 **액션 (<Icon icon="ellipsis" iconType="solid" />)** 메뉴를 클릭하고 **Add panels**를 클릭한 다음 **Quick add**를 클릭합니다.
3. 패널 목록이 표시됩니다. 체크 표시가 있는 각 패널은 이미 워크스페이스에 포함되어 있습니다.
   * 사용 가능한 모든 패널을 추가하려면 목록 상단의 **Add `<N>` panels** 버튼을 클릭합니다. **Quick Add** 목록이 닫히고 새 패널이 워크스페이스에 표시됩니다.
   * 목록에서 개별 패널을 추가하려면 패널 행 위에 마우스를 올린 다음 **Add**를 클릭합니다. 추가하려는 각 패널에 대해 이 step을 반복한 다음 오른쪽 상단의 **X**를 클릭해 **Quick Add** 목록을 닫습니다. 새 패널이 워크스페이스에 표시됩니다.
4. 필요에 따라 패널의 **Settings**를 사용자 지정합니다.

<div id="share-a-panel">
  ## 패널 공유
</div>

이 섹션에서는 링크를 사용해 패널을 공유하는 방법을 설명합니다.

링크로 패널을 공유하려면 다음 중 하나를 수행하세요.

* 전체 화면 모드에서 패널을 보는 동안 브라우저의 URL을 복사합니다.
* **액션 (<Icon icon="ellipsis" iconType="solid" />)** 메뉴를 클릭한 다음 **Copy panel URL**을 선택합니다.

링크를 사용자 또는 팀과 공유합니다. 링크에 접속하면 패널이 [전체 화면 모드](#view-a-panel-in-full-screen-mode)로 열립니다.

전체 화면 모드에서 패널의 워크스페이스로 돌아가려면 페이지 상단의 왼쪽 화살표를 클릭합니다.

<div id="compose-a-panels-full-screen-link-programmatically">
  ### 패널의 전체 화면 링크를 프로그래밍 방식으로 구성하기
</div>

[Automations 생성](/ko/models/automations/)과 같은 특정 상황에서는 패널의 전체 화면 URL을 포함하는 것이 유용할 수 있습니다. 이 섹션에서는 패널의 전체 화면 URL 형식을 보여줍니다. 다음 예시에서는 대괄호 안의 entity, 프로젝트, 패널, 섹션 이름을 바꾸세요.

```text
https://wandb.ai/<ENTITY_NAME>/<PROJECT_NAME>?panelDisplayName=<PANEL_NAME>&panelSectionName=<SECTON_NAME>
```

같은 섹션에 동일한 이름의 패널이 여러 개 있으면 이 URL은 해당 이름의 첫 번째 패널을 엽니다.

<div id="embed-or-share-a-panel-on-social-media">
  ### 웹사이트에 패널을 임베드하거나 소셜 미디어에 공유하기
</div>

웹사이트에 패널을 임베드하거나 소셜 미디어에 공유하려면 링크가 있는 누구나 해당 패널을 볼 수 있어야 합니다. 프로젝트가 비공개인 경우에는 프로젝트 멤버만 패널을 볼 수 있습니다. 프로젝트가 공개인 경우에는 링크가 있는 누구나 패널을 볼 수 있습니다.

패널을 임베드하거나 소셜 미디어에 공유하는 데 필요한 코드를 가져오려면 다음과 같이 하세요.

1. 워크스페이스에서 패널 위에 마우스를 올린 다음 **액션 (<Icon icon="ellipsis" iconType="solid" />)** 메뉴를 클릭합니다.
2. **Share** 탭을 클릭합니다.
3. **Only those who are invited have access**를 **Anyone with the link can view**로 변경합니다. 그렇지 않으면 다음 step의 옵션을 사용할 수 없습니다.
4. **Share on Twitter**, **Share on Reddit**, **Share on LinkedIn** 또는 **Copy embed link**를 선택합니다.

<div id="email-a-panel-report">
  ### 패널 리포트 이메일로 보내기
</div>

단일 패널을 독립형 리포트로 이메일로 보내려면 다음 단계를 따르세요.

1. 패널 위에 마우스를 올린 다음 패널의 **작업 (<Icon icon="ellipsis" iconType="solid" />)** 메뉴를 클릭합니다.
2. **Share panel in report**를 클릭합니다.
3. **Invite** 탭을 선택합니다.
4. 이메일 주소 또는 사용자 이름을 입력합니다.
5. 필요하면 **can view**를 **can edit**로 변경합니다.
6. **Invite**를 클릭합니다. W&amp;B는 공유하는 패널만 포함된 리포트로 연결되는 클릭 가능한 링크가 포함된 이메일을 사용자에게 보냅니다.

[패널 공유](#share-a-panel)와 달리, 수신자는 이 리포트에서 워크스페이스로 이동할 수 없습니다.

<div id="manage-panels">
  ## 패널 관리
</div>

<div id="edit-a-panel">
  ### 패널 편집
</div>

패널을 편집하려면 다음과 같이 하세요.

1. 연필 아이콘을 클릭합니다.
2. 패널 설정을 수정합니다.
3. 패널을 다른 유형으로 변경하려면 유형을 선택한 다음 설정을 구성합니다.
4. **Apply**를 클릭합니다.

<div id="move-a-panel">
  ### 패널 이동
</div>

패널을 다른 섹션으로 이동하려면 패널의 드래그 핸들을 사용할 수 있습니다. 대신 목록에서 새 섹션을 선택하려면 다음 단계를 따르세요.

1. 필요하면 마지막 섹션 뒤에 있는 **섹션 추가**를 클릭해 새 섹션을 만드세요.
2. 패널의 **액션 (<Icon icon="ellipsis" iconType="solid" />)** 메뉴를 클릭하세요.
3. **이동**을 클릭한 다음 새 섹션을 선택하세요.

드래그 핸들을 사용해 같은 섹션 내에서 패널 순서를 바꿀 수도 있습니다.

<div id="duplicate-a-panel">
  ### 패널 복제
</div>

패널을 복제하려면 다음 단계를 따르세요.

1. 패널 상단에서 **액션 (<Icon icon="ellipsis" iconType="solid" />)** 메뉴를 클릭합니다.
2. **Duplicate**를 클릭합니다.

필요한 경우 복제된 패널을 [사용자 지정](#edit-a-panel)하거나 [이동](#move-a-panel)할 수 있습니다.

<div id="remove-panels">
  ### 패널 제거
</div>

패널을 제거하려면 다음을 수행합니다.

1. 패널 위에 마우스를 올립니다.
2. **액션 (<Icon icon="ellipsis" iconType="solid" />)** 메뉴를 선택합니다.
3. **삭제**를 클릭합니다.

수동 워크스페이스에서 모든 패널을 제거하려면 해당 **액션 (<Icon icon="ellipsis" iconType="solid" />)** 메뉴를 클릭한 다음 **모든 패널 초기화**를 클릭합니다.

자동 또는 수동 워크스페이스에서 모든 패널을 제거하려면 [워크스페이스를 재설정](#reset-a-workspace)할 수 있습니다. 기본 패널 세트로 시작하려면 **Automatic**을 선택하고, 패널이 없는 빈 워크스페이스로 시작하려면 **Manual**을 선택합니다.

<div id="manage-sections">
  ## 섹션 관리
</div>

기본적으로 워크스페이스의 섹션은 키의 logging 계층 구조를 반영합니다. 하지만 수동 워크스페이스에서는 패널을 추가하기 시작해야 섹션이 표시됩니다.

<div id="add-a-section">
  ### 섹션 추가
</div>

섹션을 추가하려면 마지막 섹션 뒤에 있는 **섹션 추가**를 클릭합니다.

기존 섹션 앞이나 뒤에 새 섹션을 추가하려면 섹션의 **액션 (<Icon icon="ellipsis" iconType="solid" />)** 메뉴를 클릭한 다음 **아래에 새 섹션** 또는 **위에 새 섹션**을 클릭합니다.

<Warning>
  섹션 이름을 &quot;Section&quot;으로 지정하지 마세요. 제한 사항으로 인해 이름을 변경할 때까지 이 섹션의 패널이 렌더링되지 않습니다.
</Warning>

<div id="manage-a-sections-panels">
  ### 섹션의 패널 관리
</div>

패널 수가 많은 섹션은 기본적으로 페이지로 나뉘어 표시됩니다. 한 페이지에 표시되는 기본 패널 수는 패널의 설정과 섹션 내 패널 크기에 따라 달라집니다.

1. 패널 크기를 조정하려면 패널 위에 마우스를 올리고 드래그 핸들을 클릭한 다음 드래그하여 크기를 조정합니다. 하나의 패널 크기를 조정하면 섹션의 모든 패널 크기도 함께 조정됩니다.
2. 섹션이 페이지로 나뉘어 표시되는 경우, 페이지당 표시할 패널 수를 사용자 지정할 수 있습니다:
3. 섹션 상단에서 **1 to `<X>` of `<Y>`**를 클릭합니다. 여기서 `<X>`는 현재 표시되는 패널 수이고 `<Y>`는 전체 패널 수입니다.
4. 페이지당 표시할 패널 수를 최대 100개까지 선택합니다.
5. 섹션에서 패널을 삭제하려면:
6. 패널 위에 마우스를 올린 다음 해당 패널의 **액션 (<Icon icon="ellipsis" iconType="solid" />)** 메뉴를 클릭합니다.
7. **Delete**를 클릭합니다.

워크스페이스를 자동화된 워크스페이스로 재설정하면 삭제한 모든 패널이 다시 표시됩니다.

<div id="rename-a-section">
  ### 섹션 이름 바꾸기
</div>

섹션 이름을 바꾸려면 해당 섹션의 **액션 (<Icon icon="ellipsis" iconType="solid" />)** 메뉴를 클릭한 다음 **섹션 이름 바꾸기**를 클릭합니다.

<Warning>
  섹션 이름을 &quot;Section&quot;으로 지정하지 마세요. 제한 사항으로 인해 섹션 이름을 바꿀 때까지 이 섹션의 패널이 렌더링되지 않습니다.
</Warning>

<div id="delete-a-section">
  ### 섹션 삭제
</div>

섹션을 삭제하려면 **액션 (<Icon icon="ellipsis" iconType="solid" />)** 메뉴를 클릭한 다음 **섹션 삭제**를 클릭합니다. 그러면 섹션과 그 안의 패널이 삭제됩니다.
