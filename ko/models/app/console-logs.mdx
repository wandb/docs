실험을 실행하면 콘솔에 다양한 메시지가 출력됩니다. W&amp;B는 콘솔 로그를 캡처해 W&amp;B App에 표시합니다. 이 메시지를 활용해 실험 동작을 디버그하고 모니터링하세요.

<div id="view-console-logs">
  ## 콘솔 로그 보기
</div>

W&amp;B App에서 run의 콘솔 로그를 확인하려면 다음과 같이 하세요:

1. W&amp;B App에서 프로젝트로 이동합니다.
2. **Runs** 테이블에서 run을 선택합니다.
3. 프로젝트 사이드바에서 **Logs** 탭을 클릭합니다.

<Note>
  W&amp;B는 run당 최대 100,000줄의 로그를 저장합니다. W&amp;B App에서는 한 번에 최대 10,000줄의 로그만 표시됩니다. 저장된 모든 로그 줄을 보려면 로그를 스크롤하여 이전 줄이 표시되도록 하세요.
</Note>

<div id="types-of-console-logs">
  ## 콘솔 로그 유형
</div>

W&amp;B는 정보 메시지, 경고, 오류 등 여러 유형의 콘솔 로그를 캡처하고, 각 로그의 심각도를 나타내는 접두사를 붙여 표시합니다.

<div id="informational-messages">
  ### 정보 메시지
</div>

정보 메시지는 run의 진행 상황과 상태에 대한 정보를 알려줍니다. 일반적으로 앞에 `wandb:`가 붙습니다.

```text
wandb: Starting Run: abc123
wandb: Run data is saved locally in ./wandb/run-20240125_120000-abc123
```


<div id="warning-messages">
  ### 경고 메시지
</div>

실행을 중단시키지 않는 잠재적 문제에 대한 경고 메시지 앞에는 `WARNING:` 접두사가 붙습니다.

```text
WARNING Found .wandb file, not streaming tensorboard metrics.
WARNING These runs were logged with a previous version of wandb.
```


<div id="error-messages">
  ### 오류 메시지
</div>

심각한 문제에 대한 오류 메시지에는 `ERROR:` 접두사가 붙습니다. 이는 run이 성공적으로 완료되지 않을 수 있는 문제를 나타냅니다.

```text
ERROR Unable to save notebook session history.
ERROR Failed to save notebook.
```


<div id="console-log-settings">
  ## 콘솔 로그 설정
</div>

코드에서 `wandb.Settings` 객체를 `wandb.init()`에 전달해 W&amp;B가 콘솔 로그를 처리하는 방식을 설정합니다. `wandb.Settings`에서는 다음 매개변수를 설정해 콘솔 로그 동작을 제어할 수 있습니다.

* `show_errors`: `True`로 설정하면 오류 메시지가 W&amp;B App에 표시됩니다. `False`로 설정하면 오류 메시지가 표시되지 않습니다.
* `silent`: `True`로 설정하면 모든 W&amp;B 콘솔 출력이 억제됩니다. 콘솔 출력을 최소화하려는 프로덕션 환경에서 유용합니다.
* `show_warnings`: `True`로 설정하면 경고 메시지가 W&amp;B App에 표시됩니다. `False`로 설정하면 경고 메시지가 표시되지 않습니다.
* `show_info`: `True`로 설정하면 정보 메시지가 W&amp;B App에 표시됩니다. `False`로 설정하면 정보 메시지가 표시되지 않습니다.

<div id="custom-logging">
  ## 맞춤형 로깅
</div>

W&amp;B는 애플리케이션의 콘솔 로그를 캡처하지만, 사용자의 로깅 설정에는 영향을 주지 않습니다. Python의 기본 제공 `print()` 함수나 `logging` 모듈을 사용해 메시지를 기록할 수 있습니다.

콘솔 로그는 다음과 비슷한 형태로 표시됩니다:

```text
1 epoch:  100 loss: 1.3191105127334595
2 epoch:  200 loss: 0.8664389848709106
3 epoch:  300 loss: 0.6157898902893066
4 epoch:  400 loss: 0.4961796700954437
5 epoch:  500 loss: 0.42592573165893555
6 epoch:  600 loss: 0.3771176040172577
7 epoch:  700 loss: 0.3393910825252533
8 epoch:  800 loss: 0.3082585036754608
9 epoch:  900 loss: 0.28154927492141724
```

<div id="time-stamps">
  ## 타임스탬프
</div>

각 콘솔 로그 항목에는 타임스탬프가 자동으로 추가됩니다. 이를 통해 각 로그 메시지가 언제 생성되었는지 추적할 수 있습니다.

콘솔 로그에서 타임스탬프 표시를 켜거나 끌 수 있습니다. 콘솔 페이지의 왼쪽 상단에서 **Timestamp visible** 드롭다운을 선택하세요. 타임스탬프를 표시하거나 숨길 수 있습니다.

<div id="search-console-logs">
  ## 콘솔 로그 검색
</div>

콘솔 로그 페이지 상단의 검색창을 사용해 키워드로 로그를 필터링하세요. 특정 용어, 라벨 또는 오류 메시지를 검색할 수 있습니다.

<div id="filter-with-custom-labels">
  ## 맞춤형 라벨로 필터링
</div>

<Warning>`x_` 접두사가 붙은 매개변수(`x_label` 등)는 공개 프리뷰 상태입니다. 피드백을 보내려면 [W&amp;B 저장소에 GitHub 이슈](https://github.com/wandb/wandb)를 생성하세요.</Warning>

콘솔 로그 페이지 상단의 UI 검색창에서 `wandb.Settings`의 `x_label` 인수로 전달한 라벨을 기준으로 콘솔 로그를 필터링할 수 있습니다. 

<div id="download-console-logs">
  ## 콘솔 로그 다운로드
</div>

W&amp;B App에서 run의 콘솔 로그를 다운로드합니다.

1. W&amp;B App에서 프로젝트로 이동합니다.
2. **Runs** 테이블에서 run을 선택합니다.
3. 프로젝트 사이드바에서 **Logs** 탭을 클릭합니다.
4. 콘솔 로그 페이지 오른쪽 상단의 다운로드 버튼을 클릭합니다.

<div id="copy-console-logs">
  ## 콘솔 로그 복사
</div>

W&amp;B App에서 run의 콘솔 로그를 복사하려면 다음과 같이 하세요:

1. W&amp;B App에서 프로젝트로 이동합니다.
2. **Runs** 테이블에서 run을 선택합니다.
3. 프로젝트 사이드바에서 **Logs** 탭을 클릭합니다.
4. 콘솔 로그 페이지 오른쪽 상단에 있는 복사 버튼을 클릭합니다.
