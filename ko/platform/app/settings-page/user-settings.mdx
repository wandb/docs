사용자 프로필 페이지로 이동한 다음 오른쪽 상단에 있는 사용자 아이콘을 클릭하세요. 드롭다운 메뉴에서 **Settings**를 선택하세요.

<div id="profile">
  ## 프로필
</div>

**프로필** 섹션에서는 계정 이름과 소속 기관을 관리하고 수정할 수 있습니다. 원하는 경우 자기소개, 위치, 개인 또는 소속 기관의 웹사이트 링크를 추가하고 프로필 이미지를 업로드할 수 있습니다.

<div id="edit-your-intro">
  ## 소개 편집하기
</div>

소개를 편집하려면 프로필 상단에 있는 **Edit**를 클릭합니다. 열리는 WYSIWYG 편집기는 Markdown을 지원합니다.

1. 줄을 편집하려면 해당 줄을 클릭합니다. 시간을 절약하려면 `/`를 입력한 다음 목록에서 Markdown을 선택할 수 있습니다.
2. 항목의 드래그 핸들을 사용해 이동합니다.
3. 블록을 삭제하려면 드래그 핸들을 클릭한 다음 **Delete**를 클릭합니다.
4. 변경 사항을 저장하려면 **Save**를 클릭합니다.

<div id="add-social-badges">
  ### 소셜 배지 추가하기
</div>

X에서 `@weights_biases` 계정을 팔로우하는 배지를 추가하려면, 배지 이미지를 가리키는 HTML `<img>` 태그를 포함한 마크다운 스타일의 링크를 추가하면 됩니다:

```markdown
[![X: @weights_biases](https://img.shields.io/twitter/follow/weights_biases?style=social)](https://x.com/intent/follow?screen_name=weights_biases)
```

`&lt;img&gt;` 태그에서 `width`, `height` 또는 둘 다를 지정할 수 있습니다. 둘 중 하나만 지정하면 이미지의 가로세로 비율이 유지됩니다.

<div id="default-team">
  ## 기본 팀
</div>

둘 이상의 팀에 속해 있는 경우 **기본 팀** 섹션에서 실행이나 Weave trace에 팀이 지정되지 않았을 때 사용할 기본 팀을 설정할 수 있습니다. 하나의 팀에만 속해 있는 경우 해당 팀이 기본값이 되며, 이 섹션은 표시되지 않습니다.

계속하려면 탭을 선택하세요.

<Tabs>
  <Tab title="Multi-tenant Cloud">
    **Default location to create new projects in** 옆의 드롭다운 메뉴를 클릭한 다음, 기본 팀을 선택합니다.
  </Tab>

  <Tab title="Dedicated Cloud / Self-Managed">
    1. **Default location to create new projects in** 옆의 드롭다운 메뉴를 클릭한 다음, 기본 팀 또는 개인 엔터티를 선택합니다.
    2. (**선택 사항**) 관리자가 **Account** &gt; **Settings** &gt; **Privacy**에서 공개 프로젝트를 활성화한 경우, 새 프로젝트의 기본 공개 범위를 설정합니다. **Default project privacy in your personal account** 옆의 버튼을 클릭한 다음, **Private**(기본값) 또는 **Public**을 선택합니다.
    3. (**선택 사항**) 관리자가 **Account** &gt; **Settings** &gt; **Privacy**에서 [default saving and diffing code](/ko/models/app/features/panels/code/)를 활성화한 경우, 실행에 대해 이 기능을 켜려면 **Enable code saving in your personal account**를 클릭합니다.
  </Tab>
</Tabs>

<Note>
  자동화된 환경에서 스크립트를 실행할 때 기본 팀을 지정하려면, `WANDB_ENTITY` [환경 변수](https://docs.wandb.ai/models/track/environment-variables)를 사용해 기본 위치를 지정할 수 있습니다.
</Note>

<div id="teams">
  ## 팀
</div>

**팀** 섹션에는 내 모든 팀이 표시됩니다.

1. 팀 이름을 클릭하면 팀 페이지로 이동합니다.
2. 추가 팀에 가입할 수 있는 권한이 있는 경우 **We found teams for you to join** 옆의 **View teams**를 클릭합니다.
3. 필요한 경우 **Hide teams in public profile**를 켭니다.

<Note>
  팀을 생성하거나 관리하려면 [팀 관리](/ko/platform/app/settings-page/teams/)를 참조하세요.
</Note>

  ## API keys
</div>

**API Keys** 섹션에서는 W&amp;B 서비스에 대한 인증에 사용하는 개인용 API 키를 관리할 수 있습니다.

<div id="view-your-api-keys">
  ### API 키 확인
</div>

API 키 테이블에는 다음 정보가 표시됩니다:

* **Key ID**: 각 API 키의 첫 번째 부분으로, 식별에 사용됩니다
* **Name**: 키를 생성할 때 지정한 설명용 이름
* **Created**: 키가 생성된 시점
* **Last used**: 가장 최근 사용 시각

<Note>
  보안을 위해 테이블에는 키 ID(키의 첫 번째 부분)만 표시됩니다. 전체 비공개 API 키 값은 생성할 때 한 번만 표시됩니다.
</Note>

API 키 목록을 검색/필터링하려면 키 이름 또는 ID의 일부를 입력하세요.

<div id="delete-an-api-key">
  ### API 키 삭제
</div>

API 키를 삭제하여 액세스를 철회하려면 다음을 수행합니다:

1. API 키 테이블에서 삭제하려는 키를 찾습니다.
2. 키 옆에 있는 삭제 버튼을 클릭합니다.
3. 삭제를 확인합니다.

<Warning>
  API 키를 삭제하면 해당 키를 사용하는 모든 스크립트나 서비스의 액세스가 즉시 철회됩니다. 이전 키를 삭제하기 전에 모든 시스템이 새 키를 사용하도록 업데이트되었는지 확인하십시오.
</Warning>

<div id="beta-features">
  ## 베타 기능
</div>

**베타 기능** 섹션에서는 원하는 경우 재미있는 추가 기능과 개발 중인 신규 제품의 미리 보기를 켤 수 있습니다. 사용하려는 베타 기능 옆의 토글 스위치를 선택해 활성화하세요.

<div id="alerts">
  ## 알림
</div>

실행이 중단되거나 완료될 때, 또는 [wandb.Run.alert()](/ko/models/runs/alert/)을(를) 사용해 사용자 정의 알림을 설정해 알림을 받을 수 있습니다. 이메일 또는 Slack을 통해 알림을 수신할 수 있습니다. 알림을 받고 싶은 이벤트 유형 옆의 스위치를 토글하세요.

* **실행 완료**: Weights &amp; Biases 실행이 성공적으로 완료되었는지 여부입니다.
* **실행 중단**: 실행이 완료되지 못하고 실패했을 때의 알림입니다.

알림을 설정하고 관리하는 방법에 대한 자세한 내용은 [wandb.Run.alert()로 알림 보내기](/ko/models/runs/alert/)를 참조하세요.

<div id="personal-github-integration">
  ## 개인 GitHub 통합
</div>

개인 GitHub 계정을 연결할 수 있습니다. GitHub 계정을 연결하려면 다음 단계를 따르세요.

1. **Connect GitHub** 버튼을 선택합니다. 그러면 OAuth(오픈 인증) 페이지로 이동합니다.
2. **Organization access** 섹션에서 액세스를 허용할 조직을 선택합니다.
3. **Authorize** **wandb**를 선택합니다.

<div id="delete-your-account">
  ## 계정 삭제
</div>

계정을 삭제하려면 **Delete Account** 버튼을 클릭하세요.

<Warning>
  삭제한 계정은 복구할 수 없습니다.
</Warning>

<div id="storage">
  ## Storage
</div>

**Storage** 섹션에서는 Weights &amp; Biases 서버에서 계정이 사용한 총 스토리지 용량을 설명합니다. 기본 스토리지 한도는 100GB입니다. 스토리지와 요금제에 대한 자세한 내용은 [Pricing](https://wandb.ai/site/pricing) 페이지를 참조하세요.
