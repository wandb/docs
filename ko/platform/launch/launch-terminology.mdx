이 페이지에서는 W&amp;B Launch 문서 전반에서 사용되는 핵심 용어와 개념을 정의합니다. 설정 가이드, 튜토리얼 또는 W&amp;B App에서 익숙하지 않은 용어를 접할 때 레퍼런스로 활용하세요.

W&amp;B Launch에서는 run을 생성하기 위해 [작업](#launch-job)을 [큐](#launch-queue)에 추가합니다. 작업은 W&amp;B가 적용된 Python 스크립트입니다. 큐는 [대상 리소스](#target-resources)에서 실행할 작업 목록을 보관합니다. [에이전트](#launch-agent)는 큐에서 작업을 가져와 대상 리소스에서 실행합니다. W&amp;B는 [runs](/ko/models/runs/)를 추적하는 방식과 비슷하게 launch job을 추적합니다.

다음 섹션에서는 각 개념을 더 자세히 설명합니다.

<div id="launch-job">
  ## Launch 작업
</div>

Launch 작업은 완료할 작업을 나타내는 특정 유형의 [W&amp;B 아티팩트](/ko/models/artifacts/)입니다. 예를 들어, 일반적인 Launch 작업에는 모델을 트레이닝하거나 모델 평가를 트리거하는 작업이 포함됩니다. 작업 정의에는 다음이 포함됩니다.

* 하나 이상의 실행 가능한 엔트리포인트를 포함한 Python 코드와 기타 파일 애셋
* 입력(`config` 파라미터) 및 출력(로깅된 메트릭)에 대한 정보
* 환경에 대한 정보(예: `requirements.txt`, 기본 `Dockerfile`)

작업 정의의 세 가지 주요 유형은 다음과 같습니다.

| Job types            | Definition                                    | How to run this job type                                               |
| -------------------- | --------------------------------------------- | ---------------------------------------------------------------------- |
| 아티팩트 기반(또는 코드 기반) 작업 | 코드와 기타 애셋이 W&amp;B 아티팩트로 저장됩니다.               | 아티팩트 기반 작업을 실행하려면 builder를 사용하도록 Launch agent를 구성해야 합니다.               |
| Git 기반 작업            | 코드와 기타 애셋이 git 저장소의 특정 커밋, 브랜치 또는 태그에서 클론됩니다. | git 기반 작업을 실행하려면 builder와 git 저장소 자격 증명을 사용하도록 Launch agent를 구성해야 합니다. |
| 이미지 기반 작업            | 코드와 기타 애셋이 도커 이미지에 포함되어 있습니다.                 | 이미지 기반 작업을 실행하려면 이미지 저장소 자격 증명을 사용하도록 Launch agent를 구성해야 할 수 있습니다.     |

<Note>
  Launch 작업은 모델 트레이닝과 관련 없는 활동(예: Triton 추론 서버에 모델 배포)도 수행할 수 있지만, 모든 작업은 성공적으로 완료되려면 `wandb.init()`를 호출해야 합니다. 이렇게 하면 추적을 위해 W&amp;B 워크스페이스에 run이 생성됩니다.
</Note>

생성한 작업은 W&amp;B App의 프로젝트 워크스페이스에 있는 **Jobs** 탭에서 찾을 수 있습니다. 여기에서 작업을 구성하고 [Launch 큐](#launch-queue)로 보내 서로 다른 [대상 리소스](#target-resources)에서 실행할 수 있습니다.

<div id="launch-queue">
  ## Launch 큐
</div>

Launch *큐*는 특정 대상 리소스에서 실행할 작업의 순서가 정해진 목록입니다. Launch 큐는 선입선출(FIFO) 방식입니다. 만들 수 있는 큐 수에는 사실상 제한이 없지만, 일반적으로는 대상 리소스당 큐 하나를 두는 것이 좋습니다. 작업은 W&amp;B App UI, W&amp;B CLI 또는 Python SDK를 사용해 큐에 추가할 수 있습니다. 그런 다음 하나 이상의 Launch 에이전트를 설정해 큐에서 항목을 가져오고, 이를 큐의 대상 리소스에서 실행할 수 있습니다.

<div id="target-resources">
  ## 대상 리소스
</div>

Launch 큐가 작업을 실행하는 컴퓨팅 환경을 *대상 리소스*라고 합니다.

W&amp;B Launch는 다음 대상 리소스를 지원합니다.

* [Docker](/ko/platform/launch/setup-launch-docker/)
* [Kubernetes](/ko/platform/launch/setup-launch-kubernetes/)
* [AWS SageMaker](/ko/platform/launch/setup-launch-sagemaker/)
* [Google Cloud Vertex AI](/ko/platform/launch/setup-vertex/)

각 대상 리소스는 *리소스 설정*이라고 하는 서로 다른 설정 매개변수 집합을 사용합니다. 리소스 설정에는 각 Launch 큐에 정의된 기본값이 적용되지만, 각 작업에서 개별적으로 재정의할 수 있습니다. 자세한 내용은 각 대상 리소스의 문서를 참조하세요.

<div id="launch-agent">
  ## Launch 에이전트
</div>

Launch 에이전트는 실행할 작업이 있는지 Launch 큐를 주기적으로 확인하는 가벼운 상시 실행 프로그램입니다. Launch 에이전트가 작업을 받으면 먼저 작업 정의에 따라 이미지를 빌드하거나 가져온 다음, 대상 리소스에서 이를 실행합니다.

하나의 에이전트는 여러 큐를 폴링할 수 있습니다. 하지만 폴링하는 각 큐의 모든 기반 대상 리소스를 지원할 수 있도록 에이전트를 올바르게 설정해야 합니다.

<div id="launch-agent-environment">
  ## Launch 에이전트 환경
</div>

에이전트 환경은 Launch 에이전트가 실행되면서 작업을 폴링하는 환경을 의미합니다.

<Note>
  에이전트의 런타임 환경은 큐의 대상 리소스와는 별개입니다. 즉, 필요한 대상 리소스에 접근할 수 있도록 충분히 설정하기만 하면 에이전트를 어디에나 배포할 수 있습니다.
</Note>
