---
title: Launch 용어 및 개념
---

W&amp;B Launch를 사용하면 [작업](#launch-job)을 [대기열](#launch-queue)에 추가하여 실행을 만듭니다. 작업은 W&amp;B로 계측된 Python 스크립트입니다. 대기열에는 [대상 리소스](#target-resources)에서 실행할 작업 목록이 저장됩니다. [에이전트](#launch-agent)는 대기열에서 작업을 가져와 대상 리소스에서 실행합니다. W&amp;B는 Launch 작업을 [실행](/ko/models/runs/)을 추적하는 방식과 유사하게 추적합니다.

<div id="launch-job">
  ### Launch 작업
</div>

Launch 작업은 완료해야 할 태스크를 나타내는 특정 유형의 [W&amp;B Artifact](/ko/models/artifacts/)입니다. 예를 들어, 일반적인 Launch 작업에는 모델 학습이나 모델 평가 실행 트리거와 같은 작업이 포함됩니다. 작업 정의에는 다음이 포함됩니다:

* 하나 이상의 실행 가능한 entrypoint를 포함한 Python 코드 및 기타 파일 에셋
* 입력(config 파라미터)과 출력(로그된 지표)에 대한 정보
* 환경에 대한 정보 (예: `requirements.txt`, base `Dockerfile`)

작업 정의에는 세 가지 주요 종류가 있습니다:

| 작업 유형 | 정의 | 이 작업 유형을 실행하는 방법 |
| ---------- | --------- | -------------- |
|아티팩트 기반(또는 코드 기반) 작업| 코드와 기타 에셋이 W&amp;B 아티팩트로 저장됩니다.| 아티팩트 기반 작업을 실행하려면 Launch agent를 builder와 함께 구성해야 합니다. |
|Git 기반 작업| 코드와 기타 에셋이 git 리포지토리의 특정 커밋, 브랜치 또는 태그에서 클론됩니다. | Git 기반 작업을 실행하려면 Launch agent를 builder와 git 리포지토리 자격 증명과 함께 구성해야 합니다. |
|이미지 기반 작업|코드와 기타 에셋이 Docker 이미지에 포함(bake)됩니다. | 이미지 기반 작업을 실행하려면 Launch agent를 이미지 리포지토리 자격 증명과 함께 구성해야 할 수 있습니다. |

<Note>
  Launch 작업은 모델 학습과 직접 관련이 없는 작업(예: 모델을 Triton inference server에 배포)도 수행할 수 있지만, 모든 작업이 성공적으로 완료되려면 `wandb.init`을 호출해야 합니다. 이는 W&amp;B 워크스페이스에서 추적을 위한 실행(run)을 생성합니다.
</Note>

생성한 작업은 프로젝트 워크스페이스의 `Jobs` 탭에서 W&amp;B App 내에서 찾을 수 있습니다. 거기에서 작업을 구성한 후, 다양한 [target resources](#target-resources)에서 실행될 수 있도록 [launch queue](#launch-queue)로 보낼 수 있습니다.

<div id="launch-queue">
  ### Launch 큐
</div>

Launch *큐*는 특정 대상 리소스에서 실행할 작업(job)을 순차적으로 실행하기 위한 목록입니다. Launch 큐는 선입선출(FIFO) 방식으로 동작합니다. 생성할 수 있는 큐의 개수에는 실질적인 제한이 없지만, 권장 사항은 대상 리소스마다 하나의 큐를 두는 것입니다. 작업은 W&amp;B App UI, W&amp;B CLI 또는 Python SDK를 사용하여 큐에 추가할 수 있습니다. 그런 다음 하나 이상의 Launch 에이전트를 구성하여 큐에서 작업을 가져와 해당 큐의 대상 리소스에서 실행하도록 할 수 있습니다.

<div id="target-resources">
  ### 대상 리소스
</div>

Launch 큐가 작업을 실행하도록 구성된 컴퓨팅 환경을 *대상 리소스*라고 합니다.

W&amp;B Launch는 다음과 같은 대상 리소스를 지원합니다:

* [Docker](/ko/platform/launch/setup-launch-docker/)
* [Kubernetes](/ko/platform/launch/setup-launch-kubernetes/)
* [AWS SageMaker](/ko/platform/launch/setup-launch-sagemaker/)
* [Google Cloud Vertex AI](/ko/platform/launch/setup-vertex/)

각 대상 리소스는 *리소스 구성(resource configuration)*이라고 하는 서로 다른 구성 파라미터 집합을 사용합니다. 리소스 구성은 각 Launch 큐에서 정의한 기본값을 사용하지만, 각 작업에서 개별적으로 재정의할 수 있습니다. 자세한 내용은 각 대상 리소스에 대한 문서를 참고하세요.

<div id="launch-agent">
  ### Launch 에이전트
</div>

Launch 에이전트는 주기적으로 Launch 큐를 폴링(poll)하여 실행할 작업(Job)이 있는지 확인하는 경량의 상주 프로그램입니다. Launch 에이전트가 작업을 수신하면, 먼저 작업 정의에서 이미지를 빌드하거나 가져온 다음 대상 리소스에서 실행합니다.

하나의 에이전트가 여러 큐를 폴링할 수 있지만, 폴링하는 각 큐의 백엔드 대상 리소스를 모두 지원하도록 에이전트를 올바르게 구성해야 합니다.

<div id="launch-agent-environment">
  ### Launch 에이전트 환경
</div>

에이전트 환경은 Launch 에이전트가 실행되며 작업을 폴링하는 환경입니다.

<Note>
  에이전트의 런타임 환경은 큐의 대상 리소스와는 독립적입니다. 다시 말해, 에이전트가 필요한 대상 리소스에 접근할 수 있도록 적절히 구성되어 있기만 하면 어디에나 배포할 수 있습니다.
</Note>
