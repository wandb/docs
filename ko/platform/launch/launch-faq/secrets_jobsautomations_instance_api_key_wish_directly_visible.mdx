---
title: Jobs / Automations 를 위한 비밀번호나 보안 설정(secrets)을 지정할 수 있나요? 예를 들어, 사용자에게 직접 노출되지
  않기를 원하는 API 키와 같은 정보들 말입니다.
---

네, 가능합니다. 다음 단계를 수행하세요:

1. 다음 코맨드를 사용하여 Runs를 위해 지정된 네임스페이스에 Kubernetes secret을 생성합니다:  
   `kubectl create secret -n <namespace> generic <secret_name> <secret_value>`

2. secret을 생성한 후, Runs가 시작될 때 해당 secret을 주입하도록 큐를 구성합니다. 클러스터 관리자만 secret을 볼 수 있으며, 최종 Users는 이를 볼 수 없습니다.
