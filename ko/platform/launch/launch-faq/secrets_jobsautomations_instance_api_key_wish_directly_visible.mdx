---
title: 예를 들어, 사용자에게 직접 노출하고 싶지 않은 API 키 같은 시크릿을 작업 또는 자동화에
  지정할 수 있나요?
---

예. 다음 단계를 따르세요:

1. run용으로 지정된 네임스페이스에 Kubernetes 시크릿을 생성합니다:
   `kubectl create secret -n [NAMESPACE] generic [SECRET-NAME] [SECRET-VALUE]`

2. 시크릿을 생성한 후에는 run이 시작될 때 시크릿이 주입되도록 큐를 설정합니다. 시크릿은 클러스터 관리자만 볼 수 있으며, 사용자는 볼 수 없습니다.
