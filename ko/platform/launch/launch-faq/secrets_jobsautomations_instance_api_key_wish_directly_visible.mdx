---
title: 작업/자동화에 사용할 secret을 지정할 수 있나요? 예를 들어, 사용자에게
  직접 보이지 않게 하고 싶은 API 키가 있을 때요?
---

네. 다음 단계를 따르세요:

1. 다음 명령을 사용하여 run에 사용할 지정된 네임스페이스에 Kubernetes secret을 생성합니다:
   `kubectl create secret -n <namespace> generic <secret_name> <secret_value>`

2. secret을 생성한 후에는 run이 시작될 때 secret이 주입되도록 큐를 구성합니다. 클러스터 관리자만 secret을 볼 수 있으며, 최종 사용자는 볼 수 없습니다.
