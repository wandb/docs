---
title: '`wandb launch -d` 또는 `wandb job create image`가 레지스트리에서 pull 하지 않고 전체 Docker 아티팩트를 업로드하고 있나요?'
---

아니요, `wandb launch -d` 명령은 이미지를 레지스트리에 업로드하지 않습니다. 이미지는 레지스트리에 별도로 업로드해야 합니다. 다음 단계를 따르세요:

1. 이미지를 빌드합니다.
2. 이미지를 레지스트리에 푸시합니다.

워크플로는 다음과 같습니다:

```bash
docker build -t <repo-url>:<tag> .
docker push <repo-url>:<tag>
wandb launch -d <repo-url>:<tag>
```

Launch agent는 이후 지정된 컨테이너를 대상으로 하는 job을 생성하여 실행합니다. 컨테이너 레지스트리에서 이미지를 가져올 수 있도록 에이전트 접근 권한을 구성하는 예시는 [고급 agent 설정](/ko/platform/launch/setup-agent-advanced/#agent-configuration)을 참고하세요.

Kubernetes의 경우, 이미지가 푸시된 레지스트리에 Kubernetes 클러스터의 Pod가 접근할 수 있도록 설정했는지 확인하세요.
