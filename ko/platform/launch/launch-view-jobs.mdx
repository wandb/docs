다음 페이지에서는 큐에 추가된 Launch 작업의 정보를 확인하는 방법을 설명합니다.

<div id="view-jobs">
  ## 작업 보기
</div>

W&amp;B App에서 큐에 추가된 작업을 확인합니다.

1. [https://wandb.ai/home](https://wandb.ai/home)에서 W&amp;B App으로 이동합니다.
2. 왼쪽 사이드바의 **Applications** 섹션에서 **Launch**를 선택합니다.
3. **All entities** 드롭다운을 선택한 다음, Launch 작업이 속한 entity를 선택합니다.
4. Launch Application 페이지에서 접을 수 있는 UI를 펼쳐 해당 큐에 추가된 작업 목록을 확인합니다.

<Note>
  Launch 에이전트가 Launch 작업을 실행하면 run이 생성됩니다. 즉, 목록의 각 run은 해당 큐에 추가된 특정 작업에 해당합니다.
</Note>

예를 들어, 다음 이미지는 `job-source-launch_demo-canonical`이라는 작업에서 생성된 두 개의 run을 보여줍니다. 이 작업은 `Start queue`라는 큐에 추가되었습니다. 첫 번째 run의 이름은 `resilient-snowball`이고, 두 번째 run의 이름은 `earthy-energy-165`입니다.

W&amp;B App UI에서는 Launch 작업으로 생성된 run에 대해 다음과 같은 추가 정보를 확인할 수 있습니다.

* **Run**: 해당 작업에 할당된 W&amp;B run의 이름입니다.
  * **Job ID**: 작업의 이름입니다.
  * **Project**: run이 속한 프로젝트의 이름입니다.
  * **Status**: 큐에 추가된 run의 상태입니다.
  * **Author**: run을 생성한 W&amp;B entity입니다.
  * **Creation date**: 큐가 생성된 타임스탬프입니다.
  * **Start time**: 작업이 시작된 타임스탬프입니다.
  * **Duration**: 작업의 run이 완료되는 데 걸린 시간(초)입니다.

<div id="list-jobs">
  ## 작업 목록
</div>

W&amp;B CLI를 사용해 프로젝트 내에 있는 작업 목록을 확인합니다. W&amp;B job list 명령을 사용하고, Launch 작업이 속한 프로젝트 이름과 entity 이름을 각각 `--project` 및 `--entity` 플래그로 지정합니다.

```bash
 wandb job list --entity your-entity --project project-name
```


<div id="check-the-status-of-a-job">
  ## 작업 상태 확인
</div>

다음 표는 큐에 들어간 run이 가질 수 있는 상태를 정의합니다.

| 상태           | 설명                                                                       |
| ------------ | ------------------------------------------------------------------------ |
| **Idle**     | 활성 에이전트가 없는 큐에 run이 있습니다.                                                |
| **Queued**   | 에이전트가 처리할 때까지 큐에서 대기 중인 run입니다.                                          |
| **Pending**  | 에이전트가 run을 가져갔지만 아직 시작하지 않았습니다. 클러스터에서 사용할 수 있는 리소스가 없어 이런 상태가 될 수 있습니다. |
| **Running**  | run이 현재 실행 중입니다.                                                         |
| **Killed**   | 사용자가 작업을 중지했습니다.                                                         |
| **Crashed**  | run이 데이터 전송을 중단했거나 정상적으로 시작되지 않았습니다.                                     |
| **Failed**   | run이 0이 아닌 종료 코드와 함께 종료되었거나 시작하지 못했습니다.                                  |
| **Finished** | 작업이 성공적으로 완료되었습니다.                                                       |
