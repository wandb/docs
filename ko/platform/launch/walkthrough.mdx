<div id="what-is-launch">
  ## Launch란 무엇인가요?
</div>

<Card title="Colab에서 실행해 보기" href="https://colab.research.google.com/drive/1wX0OSVxZJDHRsZaOaOEDx-lLUrO1hHgP" icon="python" />

W&amp;B Launch를 사용하면 데스크톱에서 실행한 학습 [실행](/ko/models/runs/)을 Amazon SageMaker, Kubernetes 등의 컴퓨트 리소스로 손쉽게 확장할 수 있습니다. W&amp;B Launch를 설정해 두면, 몇 번의 클릭과 명령어만으로 학습 스크립트를 실행하고, 모델 평가 스위트를 돌리고, 프로덕션 환경 추론용 모델을 준비하는 등 다양한 작업을 빠르게 수행할 수 있습니다.

<div id="how-it-works">
  ## 작동 방식
</div>

Launch는 세 가지 기본 구성 요소인 **launch job**, **launch queue**, **launch agent**로 구성됩니다.

[*launch job*](./launch-terminology#launch-job)은 ML 워크플로우에서 작업을 구성하고 실행하기 위한 설계도입니다. launch job을 만들고 나면 이를 [*launch queue*](./launch-terminology#launch-queue)에 추가할 수 있습니다. launch queue는 선입선출(FIFO) 방식의 대기열로, Amazon SageMaker나 Kubernetes 클러스터와 같은 특정 컴퓨트 대상 리소스에 작업을 구성하고 제출할 수 있습니다.

작업이 대기열에 추가되면 [*launch agent*](./launch-terminology#launch-agent)가 해당 대기열을 폴링하여, 대기열에서 지정한 시스템에서 작업을 실행합니다.

<Frame>
  <img src="/images/launch/launch_overview.png" alt="W&B Launch 개요 다이어그램" />
</Frame>

사용 사례에 따라, 선택한 [compute resource target](./launch-terminology#target-resources)(예: Amazon SageMaker)에 맞게 launch queue를 구성하고, 자체 인프라에 launch agent를 배포합니다.

Launch에 대한 더 자세한 내용은 [Terms and concepts](./launch-terminology) 페이지를 참고하세요.

<div id="how-to-get-started">
  ## 시작하기
</div>

사용 사례에 따라 다음 리소스를 참고해 W&amp;B Launch를 시작하세요:

* W&amp;B Launch를 처음 사용하는 경우, [Launch walkthrough](#walkthrough) 가이드를 먼저 살펴볼 것을 권장합니다.
* [W&amp;B Launch](/ko/platform/launch/set-up-launch/)를 설정하는 방법을 알아보세요.
* [launch job](/ko/platform/launch/launch-terminology/#launch-job)을 생성하세요.
* [Triton에 배포](https://github.com/wandb/launch-jobs/tree/main/jobs/deploy_to_nvidia_triton), [LLM 평가](https://github.com/wandb/launch-jobs/tree/main/jobs/openai_evals) 등 일반적인 작업에 대한 템플릿은 W&amp;B Launch [public jobs GitHub repository](https://github.com/wandb/launch-jobs)에서 확인하세요.
  * 이 리포지토리에서 생성된 launch job은 공개 [`wandb/jobs` project](https://wandb.ai/wandb/jobs/jobs) W&amp;B 프로젝트에서 확인할 수 있습니다.

  ## 단계별 안내
</div>

이 페이지에서는 W&amp;B Launch 워크플로의 기본 사항을 단계별로 설명합니다.

<Note>
  W&amp;B Launch는 컨테이너에서 머신러닝 워크로드를 실행합니다. 이 단계별 안내를 따라가는 데 컨테이너에 대한 사전 지식이 필수는 아니지만, 알고 있으면 도움이 될 수 있습니다. 컨테이너의 기본 개념은 [Docker 문서](https://docs.docker.com/guides/docker-concepts/the-basics/what-is-a-container/)를 참고하세요.
</Note>

<div id="prerequisites">
  ## 사전 준비 사항
</div>

시작하기 전에 다음 사항을 준비했는지 확인하세요:

1. https://wandb.ai/site 에서 계정을 만든 후 W&amp;B 계정에 로그인합니다.
2. 이 가이드를 따라 하려면 Docker CLI와 엔진이 정상적으로 동작하는 머신에 터미널로 접근할 수 있어야 합니다. 자세한 내용은 [Docker 설치 가이드](https://docs.docker.com/engine/install/)를 참고하세요.
3. W&amp;B Python SDK `0.17.1` 이상 버전을 설치합니다:
   ```bash
   pip install wandb>=0.17.1
   ```
4. 터미널에서 `wandb login`을 실행하거나 `WANDB_API_KEY` 환경 변수를 설정하여 W&amp;B에 인증합니다.

<Tabs>
  <Tab title="W&B에 로그인">
    터미널에서 다음을 실행합니다:

    ```bash
    wandb login
    ```
  </Tab>

  <Tab title="환경 변수">
    ```bash
        WANDB_API_KEY=<your-api-key>
    ```

    `<your-api-key>`를 자신의 W&amp;B API 키로 바꿉니다.
  </Tab>
</Tabs>

<div id="create-a-launch-job">
  ## Launch 잡 생성
</div>

Docker 이미지, Git 리포지토리, 로컬 소스 코드 중 하나를 사용하여 [Launch 잡](./launch-terminology#launch-job)을 생성할 수 있습니다:

<Tabs>
  <Tab title="Docker 이미지로 실행">
    미리 만들어 둔 컨테이너를 실행하여 W&amp;B에 메시지를 로깅하려면 터미널을 열고 다음 명령을 실행합니다:

    ```bash
    wandb launch --docker-image wandb/job_hello_world:main --project launch-quickstart
    ```

    위 명령은 컨테이너 이미지 `wandb/job_hello_world:main`을 다운로드한 후 실행합니다.

    Launch는 컨테이너가 `wandb`로 로깅하는 모든 내용을 `launch-quickstart` 프로젝트에 보고하도록 컨테이너를 구성합니다. 컨테이너는 W&amp;B에 메시지를 로깅하고, W&amp;B에서 새로 생성된 실행에 대한 링크를 표시합니다. 링크를 클릭하여 W&amp;B UI에서 해당 실행을 확인합니다.
  </Tab>

  <Tab title="Git 리포지토리에서 실행">
    동일한 hello-world 잡을 [W&amp;B Launch jobs 리포지토리의 소스 코드](https://github.com/wandb/launch-jobs)에서 실행하려면 다음 명령을 실행합니다:

    ```bash
    wandb launch --uri https://github.com/wandb/launch-jobs.git \\
    --job-name hello-world-git --project launch-quickstart \\ 
    --build-context jobs/hello_world --dockerfile Dockerfile.wandb \\ 
    --entry-point "python job.py"
    ```

    이 명령은 다음 작업을 수행합니다:

    1. [W&amp;B Launch jobs 리포지토리](https://github.com/wandb/launch-jobs)를 임시 디렉터리에 클론합니다.
    2. **hello** 프로젝트에 **hello-world-git**이라는 이름의 잡을 생성합니다. 이 잡은 코드를 실행하는 데 사용된 정확한 소스 코드와 구성을 추적합니다.
    3. `jobs/hello_world` 디렉터리와 `Dockerfile.wandb`를 기반으로 컨테이너 이미지를 빌드합니다.
    4. 컨테이너를 시작하고 `job.py` Python 스크립트를 실행합니다.

    콘솔 출력에는 이미지 빌드 및 실행 로그가 표시됩니다. 컨테이너의 출력은 이전 예제와 거의 동일합니다.
  </Tab>

  <Tab title="로컬 소스 코드에서 실행">
    Git 리포지토리로 버전 관리되지 않은 코드도 `--uri` 인수에 로컬 디렉터리 경로를 지정하여 실행할 수 있습니다.

    빈 디렉터리를 만들고, 다음 내용을 가진 `train.py`라는 이름의 Python 스크립트를 추가합니다:

    ```python
    import wandb

    with wandb.init() as run:
        run.log({"hello": "world"})
    ```

    다음 내용을 가진 `requirements.txt` 파일을 추가합니다:

    ```text
    wandb>=0.17.1
    ```

    해당 디렉터리에서 다음 명령을 실행합니다:

    ```bash
    wandb launch --uri . --job-name hello-world-code --project launch-quickstart --entry-point "python train.py"
    ```

    이 명령은 다음 작업을 수행합니다:

    1. 현재 디렉터리의 내용을 Code Artifact로 W&amp;B에 로깅합니다.
    2. **launch-quickstart** 프로젝트에 **hello-world-code**라는 이름의 잡을 생성합니다.
    3. `train.py`와 `requirements.txt`를 베이스 이미지에 복사하고 `pip install`로 requirements를 설치하여 컨테이너 이미지를 빌드합니다.
    4. 컨테이너를 시작하고 `python train.py`를 실행합니다.
  </Tab>
</Tabs>

<div id="create-a-queue">
  ## 큐 생성하기
</div>

Launch는 팀이 공유 컴퓨트 리소스에서 워크플로우를 구성할 수 있도록 설계되었습니다. 지금까지의 예시에서는 `wandb launch` 명령이 로컬 머신에서 컨테이너를 동기적으로 실행했습니다. Launch 큐와 에이전트는 공유 리소스에서 작업을 비동기적으로 실행할 수 있게 해주며, 우선순위 지정 및 하이퍼파라미터 최적화와 같은 고급 기능도 제공합니다. 기본 큐를 생성하려면 다음 단계를 따르십시오:

1. [wandb.ai/launch](https://wandb.ai/launch)로 이동한 다음 **Create a queue** 버튼을 클릭합니다.
2. 큐를 연결할 **Entity**를 선택합니다.
3. **Queue name**을 입력합니다.
4. **Resource**로 **Docker**를 선택합니다.
5. 지금은 **Configuration**은 비워 둡니다.
6. **Create queue**를 클릭합니다 :rocket:

버튼을 클릭하면 브라우저가 큐 뷰의 **Agents** 탭으로 리디렉션됩니다. 에이전트가 폴링을 시작할 때까지 큐는 **Not active** 상태로 유지됩니다.

<Frame>
  <img src="/images/launch/create_docker_queue.gif" alt="Docker 큐 생성" />
</Frame>

고급 큐 구성 옵션은 [고급 큐 설정 페이지](/ko/platform/launch/setup-queue-advanced/)를 참고하십시오.

<div id="connect-an-agent-to-the-queue">
  ## 에이전트를 Queue에 연결하기
</div>

Queue에 폴링 에이전트가 없으면 화면 상단의 빨간 배너에 **Add an agent** 버튼이 표시됩니다. 이 버튼을 클릭하면 에이전트를 실행하는 명령을 확인하고 복사할 수 있습니다. 명령은 다음과 유사한 형태입니다:

```bash
wandb launch-agent --queue <queue-name> --entity <entity-name>
```

터미널에서 명령을 실행해 agent를 시작합니다. agent는 실행할 작업을 위해 지정된 queue를 폴링합니다. 작업을 수신하면, agent는 해당 작업에 대한 컨테이너 이미지를 다운로드하거나 빌드한 다음, 로컬에서 `wandb launch` 명령을 실행한 것처럼 이미지를 실행합니다.

[Launch 페이지](https://wandb.ai/launch)로 돌아가서 queue 상태가 이제 **Active**로 표시되는지 확인합니다.

<div id="submit-a-job-to-the-queue">
  ## 큐에 작업 제출하기
</div>

W&amp;B 계정에서 새로 만든 **launch-quickstart** 프로젝트로 이동한 다음, 화면 왼쪽 탐색 메뉴에서 **Jobs** 탭을 엽니다.

**Jobs** 페이지에는 이전에 실행된 실행에서 생성된 W&amp;B Job 목록이 표시됩니다. Launch Job을 클릭하면 소스 코드, 종속성, 그리고 해당 Job에서 생성된 실행을 확인할 수 있습니다. 이 튜토리얼을 완료하면 목록에 세 개의 Job이 있어야 합니다.

새로 생성된 Job 중 하나를 선택하고 다음 지침에 따라 큐에 제출하세요:

1. **Launch** 버튼을 클릭해 Job을 큐에 제출합니다. 그러면 **Launch** 드로어가 나타납니다.
2. 앞에서 생성한 **Queue**를 선택하고 **Launch**를 클릭합니다.

이렇게 하면 Job이 큐에 제출됩니다. 이 큐를 폴링하는 에이전트가 Job을 가져와 실행합니다. Job의 진행 상황은 W&amp;B UI에서 모니터링하거나 터미널에서 에이전트 출력 로그를 확인해 추적할 수 있습니다.

`wandb launch` 명령은 `--queue` 인수를 지정해 Job을 큐에 직접 푸시할 수 있습니다. 예를 들어 hello-world 컨테이너 Job을 큐에 제출하려면 다음 명령을 실행하세요:

```bash
   wandb launch --docker-image wandb/job_hello_world:main --project launch-quickstart --queue <queue-name>
```
