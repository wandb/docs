<div id="what-is-launch">
  ## Launch란 무엇인가요?
</div>

<Card title="Colab에서 사용해 보기" href="https://colab.research.google.com/drive/1wX0OSVxZJDHRsZaOaOEDx-lLUrO1hHgP" icon="python" />

W&amp;B Launch를 사용하면 데스크톱에서 Amazon SageMaker, Kubernetes 등의 컴퓨팅 리소스로 트레이닝 [runs](/ko/models/runs/)을 손쉽게 확장할 수 있습니다. W&amp;B Launch를 구성한 후에는 몇 번의 클릭과 명령만으로 트레이닝 스크립트를 실행하고, 모델 평가 스위트를 실행하며, 프로덕션 추론용 모델을 준비하는 등 다양한 작업을 빠르게 수행할 수 있습니다.

<div id="how-it-works">
  ## 작동 방식
</div>

Launch는 **launch job**, **queue**, **agent**라는 세 가지 기본 구성 요소로 이루어져 있습니다.

[*launch job*](./launch-terminology#launch-job)은 ML 워크플로에서 작업을 구성하고 실행하기 위한 설계도와 같은 개념입니다. launch job을 만들고 나면, 이를 [*launch queue*](./launch-terminology#launch-queue)에 추가할 수 있습니다. launch queue는 Amazon SageMaker나 Kubernetes 클러스터와 같은 특정 컴퓨트 대상 리소스에 작업을 구성하고 제출할 수 있는 선입선출(FIFO) 큐입니다.

작업이 queue에 추가되면, [*launch agent*](./launch-terminology#launch-agent)가 해당 queue를 주기적으로 조회(polling)하고, queue에서 대상으로 지정한 시스템에서 작업을 실행합니다.

<Frame>
  <img src="/images/launch/launch_overview.png" alt="W&B Launch 개요 다이어그램" />
</Frame>

사용 사례에 따라, 여러분(또는 팀의 다른 구성원)이 선택한 [compute resource target](./launch-terminology#target-resources)(예: Amazon SageMaker)에 맞게 launch queue를 구성하고, 자체 인프라에 launch agent를 배포하게 됩니다.

Launch에 대한 자세한 내용은 [용어 및 개념](./launch-terminology) 페이지를 참고하세요.

  ## 시작 방법

<div id="prerequisites">
  ## 사전 준비 사항
</div>

시작하기 전에 아래 사전 준비 사항을 모두 충족했는지 확인하세요:

1. https://wandb.ai/site 에서 계정을 생성한 후 W&amp;B 계정에 로그인합니다.
2. 이 가이드를 따라 진행하려면 Docker CLI와 엔진이 정상적으로 동작하는 머신에 대한 터미널 접근이 필요합니다. 자세한 내용은 [Docker 설치 가이드](https://docs.docker.com/engine/install/)를 참조하세요.
3. W&amp;B Python SDK `0.17.1` 이상 버전을 설치합니다:
   ```bash
   pip install wandb>=0.17.1
   ```
4. 터미널에서 `wandb login`을 실행하거나 `WANDB_API_KEY` 환경 변수를 설정하여 W&amp;B에 인증합니다.

<Tabs>
  <Tab title="W&B에 로그인">
    터미널에서 다음 명령을 실행합니다:

    ```bash
    wandb login
    ```
  </Tab>

  <Tab title="환경 변수">
    ```bash
        WANDB_API_KEY=<your-api-key>
    ```

    `<your-api-key>`를 W&amp;B API 키로 바꾸세요.
  </Tab>
</Tabs>

<div id="create-a-launch-job">
  ## Launch job 생성
</div>

[Launch job](./launch-terminology#launch-job)은 Docker 이미지, git 리포지토리, 로컬 소스 코드 중 하나를 사용해 세 가지 방법으로 생성할 수 있습니다:

<Tabs>
  <Tab title="Docker 이미지로">
    W&amp;B에 메시지를 로깅하는 미리 만들어진 컨테이너를 실행하려면 터미널을 열고 다음 명령을 실행합니다:

    ```bash
    wandb launch --docker-image wandb/job_hello_world:main --project launch-quickstart
    ```

    위 명령은 컨테이너 이미지 `wandb/job_hello_world:main`을 다운로드하고 실행합니다.

    Launch는 컨테이너가 `wandb`로 로깅하는 모든 내용을 `launch-quickstart` 프로젝트로 보고하도록 컨테이너를 설정합니다. 컨테이너는 W&amp;B에 메시지를 로깅하고 W&amp;B에서 새로 생성된 run에 대한 링크를 표시합니다. 링크를 클릭하여 W&amp;B UI에서 run을 확인합니다.
  </Tab>

  <Tab title="git 리포지토리에서">
    동일한 hello-world job을 [W&amp;B Launch jobs 리포지토리의 소스 코드](https://github.com/wandb/launch-jobs)에서 실행하려면 다음 명령을 실행합니다:

    ```bash
    wandb launch --uri https://github.com/wandb/launch-jobs.git \\
    --job-name hello-world-git --project launch-quickstart \\ 
    --build-context jobs/hello_world --dockerfile Dockerfile.wandb \\ 
    --entry-point "python job.py"
    ```

    이 명령은 다음 작업을 수행합니다:

    1. [W&amp;B Launch jobs 리포지토리](https://github.com/wandb/launch-jobs)를 임시 디렉터리에 클론합니다.
    2. **hello** 프로젝트에 **hello-world-git**이라는 이름의 job을 생성합니다. 이 job은 코드를 실행하는 데 사용된 정확한 소스 코드와 설정을 추적합니다.
    3. `jobs/hello_world` 디렉터리와 `Dockerfile.wandb`를 사용해 컨테이너 이미지를 빌드합니다.
    4. 컨테이너를 시작하고 `job.py` 파이썬 스크립트를 실행합니다.

    콘솔 출력에는 이미지 빌드와 실행 과정이 표시됩니다. 컨테이너의 출력은 이전 예제와 거의 동일해야 합니다.
  </Tab>

  <Tab title="로컬 소스 코드에서">
    git 리포지토리에서 버전 관리되지 않은 코드도 로컬 디렉터리 경로를 `--uri` 인자로 지정하여 실행할 수 있습니다.

    빈 디렉터리를 만들고, 다음 내용을 가진 `train.py`라는 이름의 Python 스크립트를 추가합니다:

    ```python
    import wandb

    with wandb.init() as run:
        run.log({"hello": "world"})
    ```

    다음 내용을 가진 `requirements.txt` 파일을 추가합니다:

    ```text
    wandb>=0.17.1
    ```

    해당 디렉터리에서 다음 명령을 실행합니다:

    ```bash
    wandb launch --uri . --job-name hello-world-code --project launch-quickstart --entry-point "python train.py"
    ```

    이 명령은 다음 작업을 수행합니다:

    1. 현재 디렉터리의 내용을 W&amp;B에 Code 아티팩트로 로깅합니다.
    2. **launch-quickstart** 프로젝트에 **hello-world-code**라는 이름의 job을 생성합니다.
    3. `train.py`와 `requirements.txt`를 베이스 이미지에 복사하고, `pip install`로 요구 사항을 설치하여 컨테이너 이미지를 빌드합니다.
    4. 컨테이너를 시작하고 `python train.py`를 실행합니다.
  </Tab>
</Tabs>

<div id="create-a-queue">
  ## 큐 생성
</div>

Launch는 팀이 공유 컴퓨트 리소스를 기반으로 워크플로를 구축할 수 있도록 설계되었습니다. 지금까지의 예시에서는 `wandb launch` 명령이 로컬 머신에서 컨테이너를 동기식으로 실행했습니다. Launch 큐와 에이전트는 공유 리소스에서 작업을 비동기식으로 실행하고, 우선순위 지정 및 하이퍼파라미터 최적화와 같은 고급 기능을 사용할 수 있도록 합니다. 기본 큐를 생성하려면 다음 단계를 따르십시오.

1. [wandb.ai/launch](https://wandb.ai/launch)로 이동한 다음 **Create a queue** 버튼을 클릭합니다.
2. 큐와 연결할 **Entity**를 선택합니다.
3. **Queue name**을 입력합니다.
4. **Resource**로 **Docker**를 선택합니다.
5. 지금은 **Configuration**을 비워 둡니다.
6. **Create queue**를 클릭합니다. :rocket:

버튼을 클릭하면 브라우저가 큐 화면의 **Agents** 탭으로 이동합니다. 에이전트가 폴링을 시작하기 전까지 큐는 **Not active** 상태로 유지됩니다.

  <img src="/images/launch/create_docker_queue.gif" alt="Docker 큐 생성" />
</Frame>

고급 큐 설정 옵션은 [고급 큐 설정 페이지](/ko/platform/launch/setup-queue-advanced/)를 참고하십시오.

<div id="connect-an-agent-to-the-queue">
  ## 에이전트를 큐에 연결하기
</div>

큐에 폴링 에이전트가 없으면 화면 상단의 빨간 배너에 **Add an agent** 버튼이 표시됩니다. 이 버튼을 클릭하면 에이전트를 실행하는 명령을 확인하고 복사할 수 있습니다. 명령은 다음과 같은 형태입니다:

```bash
wandb launch-agent --queue <queue-name> --entity <entity-name>
```

터미널에서 명령을 실행하여 에이전트를 시작한다. 에이전트는 지정된 큐를 주기적으로 폴링하여 실행할 작업이 있는지 확인한다. 작업을 받으면, 에이전트는 해당 작업을 위해 컨테이너 이미지를 다운로드하거나 빌드한 다음, 로컬에서 `wandb launch` 명령을 실행한 것처럼 해당 이미지를 실행한다.

[Launch 페이지](https://wandb.ai/launch)로 돌아가서 큐 상태가 **Active**로 표시되는지 확인한다.

<div id="submit-a-job-to-the-queue">
  ## 큐에 작업 제출하기
</div>

W&amp;B 계정에서 새 **launch-quickstart** 프로젝트로 이동한 다음, 화면 왼쪽 탐색 메뉴에서 **Jobs** 탭을 엽니다.

**Jobs** 페이지에는 이전에 실행된 run으로부터 생성된 W&amp;B Jobs 목록이 표시됩니다. launch job을 클릭하면 소스 코드, 종속성, 그리고 이 job에서 생성된 run들을 확인할 수 있습니다. 이 가이드를 마치면 목록에 세 개의 job이 있어야 합니다.

새 job 중 하나를 선택하고 다음 안내에 따라 큐에 제출하세요:

1. **Launch** 버튼을 클릭하여 job을 큐에 제출합니다. 그러면 **Launch** 드로어가 나타납니다.
2. 앞에서 생성한 **Queue**를 선택하고 **Launch**를 클릭합니다.

이렇게 하면 job이 큐에 제출됩니다. 이 큐를 폴링하는 에이전트가 job을 가져와 실행합니다. job의 진행 상황은 W&amp;B UI에서 모니터링하거나 터미널에서 에이전트 출력을 확인하여 모니터링할 수 있습니다.

`wandb launch` 명령은 `--queue` 인수를 지정하여 job을 큐로 직접 제출할 수 있습니다. 예를 들어 hello-world 컨테이너 job을 큐에 제출하려면 다음 명령을 실행하세요:

```bash
   wandb launch --docker-image wandb/job_hello_world:main --project launch-quickstart --queue <queue-name>
```
