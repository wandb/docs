<div id="set-up-queue-config-templates">
  ## 큐 설정 템플릿 구성
</div>

Queue Config Templates를 사용하여 컴퓨팅 사용량에 대한 가드레일을 설정하고 관리하세요. 메모리 사용량, GPU, 실행 시간과 같은 필드에 대해 기본값, 최소값, 최대값을 설정할 수 있습니다.

큐에 설정 템플릿을 구성하고 나면, 팀 구성원들은 여러분이 정의한 필드 값을 지정한 범위 내에서만 변경할 수 있습니다.

<div id="configure-queue-template">
  ### 큐 템플릿 구성
</div>

기존 큐에 큐 템플릿을 설정하거나 새 큐를 생성할 수 있습니다.

1. [W&amp;B Launch App](https://wandb.ai/launch)으로 이동합니다.
2. 템플릿을 추가하려는 큐 이름 옆의 **View queue**를 선택합니다.
3. **Config** 탭을 선택합니다. 이 탭에는 큐가 생성된 시점, 큐 구성(config), 기존 실행 시점(launch time) 오버라이드 등의 큐 정보가 표시됩니다.
4. **Queue config** 섹션으로 이동합니다.
5. 템플릿을 생성하려는 config 키-값 쌍을 확인합니다.
6. 해당 config 값을 템플릿 필드로 교체합니다. 템플릿 필드는 `{{variable-name}}` 형식을 사용합니다.
7. **Parse configuration** 버튼을 클릭합니다. 구성을 파싱하면, W&amp;B가 생성한 각 템플릿에 대해 큐 config 아래에 타일을 자동으로 생성합니다.
8. 생성된 각 타일에 대해, 먼저 큐 config에서 허용할 데이터 타입(string, integer, float)을 지정해야 합니다. 이를 위해 **Type** 드롭다운 메뉴에서 데이터 타입을 선택합니다.
9. 선택한 데이터 타입에 따라 각 타일에 표시되는 필드를 완료합니다.
10. **Save config**를 클릭합니다.

예를 들어, 팀에서 사용할 수 있는 AWS 인스턴스를 제한하는 템플릿을 생성하려고 한다고 가정해 봅시다. 템플릿 필드를 추가하기 전에는 큐 config가 다음과 비슷해 보일 수 있습니다:

```yaml title="launch config"
RoleArn: arn:aws:iam:region:account-id:resource-type/resource-id
ResourceConfig:
  InstanceType: ml.m4.xlarge
  InstanceCount: 1
  VolumeSizeInGB: 2
OutputDataConfig:
  S3OutputPath: s3://bucketname
StoppingCondition:
  MaxRuntimeInSeconds: 3600
```

`InstanceType`에 대한 템플릿 필드를 추가하면 config가 다음과 같이 구성됩니다:`

```yaml title="launch config"
RoleArn: arn:aws:iam:region:account-id:resource-type/resource-id
ResourceConfig:
  InstanceType: "{{aws_instance}}"
  InstanceCount: 1
  VolumeSizeInGB: 2
OutputDataConfig:
  S3OutputPath: s3://bucketname
StoppingCondition:
  MaxRuntimeInSeconds: 3600
```

다음으로 **Parse configuration**을 클릭합니다. 그러면 **Queue config** 아래에 `aws-instance`라는 이름의 새 타일이 나타납니다.

여기에서 **Type** 드롭다운에서 데이터 유형으로 String을 선택합니다. 그러면 사용자가 선택할 수 있는 값을 지정할 수 있는 필드가 생성됩니다. 예를 들어 아래 이미지에서는 팀의 관리자가 사용자가 선택할 수 있도록 두 가지 서로 다른 AWS 인스턴스 타입(`ml.m4.xlarge`와 `ml.p3.xlarge`)을 구성해 두었습니다:

<div id="dynamically-configure-launch-jobs">
  ## Launch 작업을 동적으로 구성하기
</div>

Queue 설정은 에이전트가 큐에서 작업을 디큐할 때 평가되는 매크로를 사용해 동적으로 구성할 수 있습니다. 다음 매크로를 설정할 수 있습니다:

| Macro             | Description                                             |
|-------------------|---------------------------------------------------------|
| `${project_name}` | 실행이 시작될 프로젝트의 이름.                          |
| `${entity_name}`  | 실행이 시작될 프로젝트의 소유자(엔터티).                |
| `${run_id}`       | 시작되는 실행의 ID.                                     |
| `${run_name}`     | 시작되는 실행의 이름.                                   |
| `${image_uri}`    | 이 실행에 사용되는 컨테이너 이미지의 URI.              |

<Note>
앞의 표에 나열되지 않은 사용자 정의 매크로(예: `${MY_ENV_VAR}`)는 에이전트 환경의 환경 변수 값으로 치환됩니다.
</Note>

<div id="use-the-launch-agent-to-build-images-that-execute-on-accelerators-gpus">
  ## Launch 에이전트를 사용해 가속기(GPU)에서 실행되는 이미지 빌드하기
</div>

가속기 환경에서 실행되는 이미지를 빌드하기 위해 Launch를 사용하는 경우, 가속기용 베이스 이미지를 지정해야 할 수도 있습니다.

이 가속기 베이스 이미지는 다음 요구 사항을 충족해야 합니다:

- Debian과의 호환성 (Launch Dockerfile은 `apt-get`을 사용해 Python을 가져옵니다)
- CPU 및 GPU 하드웨어 명령어 집합과의 호환성 (사용하려는 GPU가 사용하려는 CUDA 버전을 지원하는지 확인해야 합니다)
- 제공하는 가속기 버전과 ML 알고리즘에 설치된 패키지 간의 호환성
- 하드웨어와의 호환성을 위해 추가 설정 단계가 필요한 패키지 설치

<div id="how-to-use-gpus-with-tensorflow">
  ### TensorFlow로 GPU를 사용하는 방법
</div>

TensorFlow가 GPU를 제대로 활용하도록 설정합니다. 이를 위해 큐 리소스 구성에서 `builder.accelerator.base_image` 키에 사용할 Docker 이미지와 해당 이미지 태그를 지정해야 합니다.

예를 들어, `tensorflow/tensorflow:latest-gpu` 베이스 이미지는 TensorFlow가 GPU를 올바르게 사용하도록 보장합니다. 이는 큐의 리소스 구성을 통해 설정할 수 있습니다.

다음 JSON 코드 조각은 큐 구성에서 TensorFlow 베이스 이미지를 지정하는 방법을 보여 줍니다:

```json title="Queue config"
{
    "builder": {
        "accelerator": {
            "base_image": "tensorflow/tensorflow:latest-gpu"
        }
    }
}
```
