이 페이지에서는 Launch 큐 옵션을 구성하는 방법을 설명합니다.

<div id="set-up-queue-config-templates">
  ## 큐 구성 템플릿 설정
</div>

Queue Config Templates를 사용해 컴퓨트 사용량에 대한 가드레일을 설정하고 관리하세요. 메모리 사용량, GPU, 실행 시간과 같은 필드에 대해 기본값, 최소값, 최대값을 설정할 수 있습니다.

큐를 구성 템플릿과 함께 설정한 후에는 팀 구성원들이 정의한 필드 값을 지정한 범위 내에서만 변경할 수 있습니다.

<div id="configure-queue-template">
  ### 큐 템플릿 설정
</div>

기존 큐에 큐 템플릿을 설정하거나 새로운 큐를 생성할 수 있습니다.

1. [W&amp;B Launch App](https://wandb.ai/launch)으로 이동합니다.
2. 템플릿을 추가하려는 큐 이름 옆의 **View queue**를 선택합니다.
3. **Config** 탭을 선택합니다. 여기에서 큐가 생성된 시점, 큐 config, 기존 실행 시점의 설정 재정의 값 등 큐에 대한 정보를 확인할 수 있습니다.
4. **Queue config** 섹션으로 이동합니다.
5. 템플릿으로 만들 config 키-값 쌍을 선택합니다.
6. config에 있는 값을 템플릿 필드로 교체합니다. 템플릿 필드는 `{{variable-name}}` 형식을 사용합니다.
7. **Parse configuration** 버튼을 클릭합니다. 설정을 파싱하면, W&amp;B가 생성한 각 템플릿에 대해 큐 config 아래에 자동으로 타일을 생성합니다.
8. 생성된 각 타일에 대해, 먼저 큐 config에서 허용할 데이터 타입(string, integer, float)을 지정해야 합니다. 이를 위해 **Type** 드롭다운 메뉴에서 데이터 타입을 선택합니다.
9. 선택한 데이터 타입에 따라, 각 타일에 표시되는 필드 값을 채웁니다.
10. **Save config**를 클릭합니다.

예를 들어, 팀에서 사용할 수 있는 AWS 인스턴스를 제한하는 템플릿을 만들고 싶다고 가정해 보겠습니다. 템플릿 필드를 추가하기 전에는 큐 config가 다음과 비슷하게 보일 수 있습니다.

```yaml title="launch config"
RoleArn: arn:aws:iam:region:account-id:resource-type/resource-id
ResourceConfig:
  InstanceType: ml.m4.xlarge
  InstanceCount: 1
  VolumeSizeInGB: 2
OutputDataConfig:
  S3OutputPath: s3://bucketname
StoppingCondition:
  MaxRuntimeInSeconds: 3600
```

`InstanceType`에 대한 템플릿 필드를 추가하면 구성은 다음과 같습니다:

```yaml title="launch config"
RoleArn: arn:aws:iam:region:account-id:resource-type/resource-id
ResourceConfig:
  InstanceType: "{{aws_instance}}"
  InstanceCount: 1
  VolumeSizeInGB: 2
OutputDataConfig:
  S3OutputPath: s3://bucketname
StoppingCondition:
  MaxRuntimeInSeconds: 3600
```

다음으로 **Parse configuration**을 클릭합니다. 그러면 **Queue config** 아래에 `aws-instance`라는 레이블의 새 타일이 나타납니다.

이제 **Type** 드롭다운에서 데이터 타입으로 String을 선택합니다. 그러면 사용자가 선택할 수 있는 값을 지정할 수 있는 필드가 표시됩니다. 예를 들어, 다음 이미지에서는 팀의 관리자(admin)가 사용자가 선택할 수 있도록 두 가지 다른 AWS 인스턴스 타입(`ml.m4.xlarge` 및 `ml.p3.xlarge`)을 설정했습니다:

<div id="dynamically-configure-launch-jobs">
  ## Launch 잡을 동적으로 구성하기
</div>

큐 설정은 에이전트가 큐에서 잡을 디큐할 때 평가되는 매크로를 사용해 동적으로 구성할 수 있습니다. 다음과 같은 매크로를 설정할 수 있습니다:

| Macro             | Description                                             |
|-------------------|---------------------------------------------------------|
| `${project_name}` | run이 Launch될 프로젝트의 이름.                         |
| `${entity_name}`  | run이 Launch될 프로젝트의 소유자.                       |
| `${run_id}`       | Launch되는 run의 ID.                                    |
| `${run_name}`     | Launch되는 run의 이름.                                  |
| `${image_uri}`    | 이 run에 사용할 컨테이너 이미지의 URI.                 |

<Note>
앞의 표에 나열되지 않은 사용자 지정 매크로(예: `${MY_ENV_VAR}`)는 에이전트 환경의 환경 변수 값으로 치환됩니다.
</Note>

<div id="use-the-launch-agent-to-build-images-that-execute-on-accelerators-gpus">
  ## 가속기(GPU)에서 실행되는 이미지를 빌드하기 위해 Launch 에이전트 사용하기
</div>

가속기 환경에서 실행되는 이미지를 Launch로 빌드하는 경우, 가속기용 베이스 이미지를 지정해야 할 수 있습니다.

이 가속기 베이스 이미지는 다음 요구 사항을 충족해야 합니다:

- Debian 호환성 (Launch Dockerfile은 파이썬을 설치하기 위해 apt-get을 사용합니다)
- CPU 및 GPU 하드웨어 명령어 집합 호환성 (사용하려는 GPU가 해당 CUDA 버전을 지원하는지 확인하세요)
- 제공한 가속기 버전과 ML 알고리즘에 설치된 패키지 간의 호환성
- 하드웨어와의 호환성을 위해 추가 설정 단계가 필요한 패키지 설치

<div id="how-to-use-gpus-with-tensorflow">
  ### TensorFlow에서 GPU를 사용하는 방법
</div>

TensorFlow가 GPU를 제대로 활용하도록 설정해야 합니다. 이를 위해 큐 리소스 설정에서 `builder.accelerator.base_image` 키에 사용할 Docker 이미지와 해당 이미지 태그를 지정합니다.

예를 들어, `tensorflow/tensorflow:latest-gpu` 베이스 이미지를 사용하면 TensorFlow가 GPU를 올바르게 사용하도록 보장할 수 있습니다. 이는 큐 리소스 설정에서 지정할 수 있습니다.

다음 JSON 스니펫은 큐 설정에서 TensorFlow 베이스 이미지를 지정하는 방법을 보여줍니다:

```json title="Queue config"
{
    "builder": {
        "accelerator": {
            "base_image": "tensorflow/tensorflow:latest-gpu"
        }
    }
}
```
