W&amp;B Launch 실험을 시작합니다.

```python
launch(
    api: Api,
    job: Optional[str] = None,
    entry_point: Optional[List[str]] = None,
    version: Optional[str] = None,
    name: Optional[str] = None,
    resource: Optional[str] = None,
    resource_args: Optional[Dict[str, Any]] = None,
    project: Optional[str] = None,
    entity: Optional[str] = None,
    docker_image: Optional[str] = None,
    config: Optional[Dict[str, Any]] = None,
    synchronous: Optional[bool] = (True),
    run_id: Optional[str] = None,
    repository: Optional[str] = None
) -> AbstractRun
```

| Arguments       | Description                                                                                                                                                                                                                                                         |
| :-------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| `job`           | wandb.Job에 대한 문자열 참조입니다. 예: wandb/test/my-job:latest                                                                                                                                                                                                                |
| `api`           | wandb.apis.internal에서 가져온 wandb API 인스턴스입니다.                                                                                                                                                                                                                        |
| `entry_point`   | 프로젝트 내에서 실행할 엔트리 포인트입니다. wandb URI의 경우 원래 실행에서 사용된 엔트리 포인트가 기본값이며, git 리포지토리 URI의 경우 기본값은 main.py입니다.                                                                                                                                                               |
| `version`       | Git 기반 프로젝트의 경우 커밋 해시 또는 브랜치 이름입니다.                                                                                                                                                                                                                                 |
| `name`          | 실행을 시작할 때 사용할 실행 이름입니다.                                                                                                                                                                                                                                             |
| `resource`      | 실행에 사용할 실행 백엔드입니다.                                                                                                                                                                                                                                                  |
| `resource_args` | 원격 백엔드에 실행을 시작할 때 사용할 리소스 관련 인자입니다. 생성된 Launch 구성에서 `resource_args` 키 아래에 저장됩니다.                                                                                                                                                                                    |
| `project`       | 실행된 실행을 전송할 대상 프로젝트입니다.                                                                                                                                                                                                                                             |
| `entity`        | 실행된 실행을 전송할 대상 엔터티입니다.                                                                                                                                                                                                                                              |
| `config`        | 실행에 대한 설정을 담고 있는 딕셔너리입니다. &quot;resource&#95;args&quot; 키 아래에 리소스별 인자를 포함할 수도 있습니다.                                                                                                                                                                                 |
| `synchronous`   | 실행이 완료될 때까지 대기할지 여부입니다. 기본값은 True입니다. `synchronous`가 False이고 `backend`가 &quot;local-container&quot;인 경우, 이 메서드는 바로 반환되지만 현재 프로세스는 종료 시점에 로컬 실행이 완료될 때까지 대기합니다. 현재 프로세스가 중단되면, 이 메서드를 통해 시작된 비동기 실행은 모두 종료됩니다. `synchronous`가 True이고 실행이 실패하면 현재 프로세스도 에러와 함께 종료됩니다. |
| `run_id`        | 실행의 ID입니다(궁극적으로 :name: 필드를 대체하기 위한 값).                                                                                                                                                                                                                              |
| `repository`    | 원격 레지스트리용 리포지토리 경로의 문자열 이름입니다.                                                                                                                                                                                                                                      |


<div id="example">
  #### 예시:
</div>

```python
from wandb.sdk.launch import launch

job = "wandb/jobs/Hello World:latest"
params = {"epochs": 5}
# W&B 프로젝트를 실행하고 로컬 호스트에서 재현 가능한 docker 환경 생성
api = wandb.apis.internal.Api()
launch(api, job, parameters=params)
```

| 반환 값                                                              | 설명 |
| :---------------------------------------------------------------- | :- |
| `wandb.launch.SubmittedRun` 인스턴스. 시작된 실행에 대한 정보(예: 실행 ID)를 제공합니다. |    |

| 발생 예외                                                      | 설명 |
| :--------------------------------------------------------- | :- |
| `wandb.exceptions.ExecutionError`: 블로킹 모드로 시작된 실행이 실패한 경우. |    |
