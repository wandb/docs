W&amp;B Launch 실험을 대기열에 추가합니다. source URI, job 또는 docker&#95;image 중 하나를 지정합니다.

```python
launch_add(
    uri: Optional[str] = None,
    job: Optional[str] = None,
    config: Optional[Dict[str, Any]] = None,
    template_variables: Optional[Dict[str, Union[float, int, str]]] = None,
    project: Optional[str] = None,
    entity: Optional[str] = None,
    queue_name: Optional[str] = None,
    resource: Optional[str] = None,
    entry_point: Optional[List[str]] = None,
    name: Optional[str] = None,
    version: Optional[str] = None,
    docker_image: Optional[str] = None,
    project_queue: Optional[str] = None,
    resource_args: Optional[Dict[str, Any]] = None,
    run_id: Optional[str] = None,
    build: Optional[bool] = (False),
    repository: Optional[str] = None,
    sweep_id: Optional[str] = None,
    author: Optional[str] = None,
    priority: Optional[int] = None
) -> "public.QueuedRun"
```

| Arguments            | Description                                                                                                          |
| :------------------- | :------------------------------------------------------------------------------------------------------------------- |
| `uri`                | 실행할 실험의 URI입니다. wandb run URI 또는 Git 저장소 URI를 사용할 수 있습니다.                                                            |
| `job`                | wandb.Job에 대한 문자열 참조입니다. 예: wandb/test/my-job:latest                                                                 |
| `config`             | run에 대한 설정을 포함하는 사전입니다. &quot;resource&#95;args&quot; 키 아래에 리소스별 인자를 포함할 수도 있습니다.                                    |
| `template_variables` | run 큐용 템플릿 변수 값들을 포함하는 사전입니다. 예상 형식은 `{"VAR_NAME": VAR_VALUE}`입니다.                                                   |
| `project`            | 실행된 run을 전송할 대상 프로젝트입니다.                                                                                             |
| `entity`             | 실행된 run을 전송할 대상 엔티티입니다.                                                                                              |
| `queue`              | run을 대기열에 추가할 큐의 이름입니다.                                                                                              |
| `priority`           | 작업의 우선순위 수준으로, 1이 가장 높은 우선순위입니다.                                                                                     |
| `resource`           | run의 실행 백엔드입니다. W&amp;B에서는 &quot;local-container&quot; 백엔드를 기본으로 지원합니다.                                              |
| `entry_point`        | 프로젝트 내에서 실행할 entry point입니다. wandb URI의 경우 원래 run에서 사용된 entry point를, Git 저장소 URI의 경우 main.py를 기본값으로 사용합니다.          |
| `name`               | run을 실행할 때 사용할 run 이름입니다.                                                                                            |
| `version`            | Git 기반 프로젝트의 경우 커밋 해시 또는 브랜치 이름입니다.                                                                                  |
| `docker_image`       | run에 사용할 Docker 이미지의 이름입니다.                                                                                          |
| `resource_args`      | 원격 백엔드에서 run을 실행하기 위한 리소스 관련 인자입니다. 생성된 Launch 설정의 `resource_args` 아래에 저장됩니다.                                        |
| `run_id`             | 실행된 run의 ID를 나타내는 선택적 문자열입니다.                                                                                        |
| `build`              | 선택적 플래그로 기본값은 false입니다. build가 설정된 경우 queue가 설정되어 있어야 하며, 이미지가 생성되고 job 아티팩트를 만든 뒤 해당 job 아티팩트에 대한 참조를 queue에 푸시합니다. |
| `repository`         | 레지스트리에 이미지를 푸시할 때 사용되는 원격 저장소 이름을 제어하기 위한 선택적 문자열입니다.                                                                |
| `project_queue`      | 큐에 사용할 프로젝트 이름을 제어하기 위한 선택적 문자열입니다. 주로 프로젝트 범위 큐와의 하위 호환성을 위해 사용됩니다.                                                 |


<div id="example">
  #### 예시:
</div>

```python
from wandb.sdk.launch import launch_add

project_uri = "https://github.com/wandb/examples"
params = {"alpha": 0.5, "l1_ratio": 0.01}
# W&B 프로젝트를 실행하고 로컬 호스트에서
# 재현 가능한 docker 환경을 생성합니다
api = wandb.apis.internal.Api()
launch_add(uri=project_uri, parameters=params)
```

| 반환값                                                                                                                                            | 설명 |
| :--------------------------------------------------------------------------------------------------------------------------------------------- | :- |
| 대기 중인 run에 대한 정보를 제공하는 `wandb.api.public.QueuedRun` 인스턴스를 반환하며, `wait_until_started` 또는 `wait_until_finished`가 호출되면 해당 run의 상세 정보에 접근할 수 있습니다. |    |

| 발생 예외                                            | 설명 |
| :----------------------------------------------- | :- |
| 성공하지 못한 경우 `wandb.exceptions.LaunchError` 예외가 발생 |    |
