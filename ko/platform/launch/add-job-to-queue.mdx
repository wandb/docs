title: 큐에 잡 추가하기
---

이 페이지에서는 Launch 잡을 Launch 큐에 추가하는 방법을 설명합니다.

<Note>
  사용자 또는 팀원이 이미 Launch 큐를 설정했는지 확인하세요. 자세한 내용은 [Launch 설정](/ko/platform/launch/set-up-launch/) 페이지를 참조하세요.
</Note>

<div id="add-jobs-to-your-queue">
  ## 대기열에 Job 추가하기
</div>

W&amp;B App을 사용해 대화형으로, 또는 W&amp;B CLI를 사용해 프로그램 방식으로 대기열에 Job을 추가할 수 있습니다.

<Tabs>
  <Tab title="W&B app">
    W&amp;B App을 사용해 프로그램 방식으로 대기열에 Job을 추가합니다.

    1. W&amp;B 프로젝트 페이지로 이동합니다.
    2. 왼쪽 패널에서 **Jobs** 아이콘을 선택합니다:

    <Frame>
      <img src="/images/launch/project_jobs_tab_gs.png" alt="Project Jobs 탭" />
    </Frame>

    3. **Jobs** 페이지에는 이전에 실행된 W&amp;B 실행에서 생성된 W&amp;B Launch Job 목록이 표시됩니다.

    <Frame>
      <img src="/images/launch/view_jobs.png" alt="Jobs 목록" />
    </Frame>

    4. Job 이름 옆의 **Launch** 버튼을 선택합니다. 페이지 오른쪽에 모달이 나타납니다.
    5. **Job version** 드롭다운에서 사용하려는 Launch Job의 버전을 선택합니다. Launch Job은 다른 [W&amp;B Artifact](/ko/models/artifacts/create-a-new-artifact-version/)와 마찬가지로 버전이 관리됩니다. Job을 실행하는 데 사용되는 소프트웨어 의존성이나 소스 코드를 수정하면 동일한 Launch Job에 대해 서로 다른 버전이 생성됩니다.
    6. **Overrides** 섹션에서 Launch Job에 대해 구성된 입력값에 대해 새 값을 입력합니다. 일반적인 override에는 새로운 엔트리포인트 명령, 인자, 또는 새 W&amp;B 실행의 `wandb.Run.config` 내 값 등이 포함됩니다.

    <Frame>
      <img src="/images/launch/create_starter_queue_gs.png" alt="Queue 설정" />
    </Frame>

    **Paste from...** 버튼을 클릭하면, Launch Job을 사용했던 다른 W&amp;B 실행에서 값을 복사해 붙여넣을 수 있습니다.
    7. **Queue** 드롭다운에서 Launch Job을 추가할 Launch Queue의 이름을 선택합니다.
    8. **Job Priority** 드롭다운을 사용해 Launch Job의 우선순위를 지정합니다. Launch Queue가 우선순위를 지원하지 않는 경우 Launch Job의 우선순위는 &quot;Medium&quot;으로 설정됩니다.
    9. **(선택 사항) 팀 관리자에 의해 Queue 구성 템플릿이 생성된 경우에만 이 단계를 따르십시오**\
    **Queue Configurations** 필드에 팀 관리자가 생성한 구성 옵션의 값을 입력합니다.\
    예를 들어 다음 예시에서, 팀 관리자는 팀에서 사용할 수 있는 AWS 인스턴스 유형을 구성해 두었습니다. 이 경우 팀 구성원은 모델 학습에 사용할 컴퓨트 인스턴스 유형으로 `ml.m4.xlarge` 또는 `ml.p3.xlarge` 중 하나를 선택할 수 있습니다.

    <Frame>
      <img src="/images/launch/team_member_use_config_template.png" alt="Config 템플릿 선택" />
    </Frame>

    10. 결과 실행이 표시될 **Destination project**를 선택합니다. 이 프로젝트는 Queue와 동일한 엔터티에 속해야 합니다.
    11. **Launch now** 버튼을 선택합니다.
  </Tab>

  <Tab title="W&B CLI">
    `wandb launch` 명령을 사용해 대기열에 Job을 추가합니다. 하이퍼파라미터 재정의 값을 포함하는 JSON 구성을 만듭니다. 예를 들어, [Quickstart](/ko/platform/launch/walkthrough/) 가이드의 스크립트를 사용한다고 할 때, 다음과 같은 override가 포함된 JSON 파일을 생성합니다:

    ```json title="config.json"
    {
      "overrides": {
          "args": [],
          "run_config": {
              "learning_rate": 0,
              "epochs": 0
          },   
          "entry_point": []
      }
    }
    ```

    <Note>
      JSON 구성 파일을 제공하지 않으면, W&amp;B Launch는 기본 파라미터를 사용합니다.
    </Note>

    Queue 구성을 재정의하려 하거나, Launch Queue에 구성 리소스가 정의되어 있지 않은 경우, config.json 파일에서 `resource_args` 키를 지정할 수 있습니다. 예를 들어 위 예시를 이어서 사용하면, config.json 파일은 다음과 유사한 형태가 됩니다:

    ```json title="config.json"
    {
      "overrides": {
          "args": [],
          "run_config": {
              "learning_rate": 0,
              "epochs": 0
          },
          "entry_point": []
      },
      "resource_args": {
            "<resource-type>" : {
                "<key>": "<value>"
            }
      }
    }
    ```

    `<>` 안의 값을 자신의 값으로 바꾸십시오.

    `queue`(`-q`) 플래그에는 Queue 이름을, `job`(`-j`) 플래그에는 Job 이름을, `config`(`-c`) 플래그에는 구성 파일의 경로를 지정합니다.

    ```bash
    wandb launch -j <job> -q <queue-name> \ 
    -e <entity-name> -c path/to/config.json
    ```

    W&amp;B Team 내에서 작업하는 경우, Queue가 사용할 엔터티를 나타내기 위해 `entity` 플래그(`-e`)를 지정할 것을 권장합니다.
  </Tab>
</Tabs>
