---
title: 큐에 작업 추가
---

이 문서에서는 Launch 작업을 Launch 큐에 추가하는 방법을 설명합니다.

<Note>
  사용자 또는 팀원이 이미 Launch 큐를 구성해 두었는지 확인하세요. 자세한 내용은 [Launch 설정](/ko/platform/launch/set-up-launch/) 페이지를 참고하세요.
</Note>

<div id="add-jobs-to-your-queue">
  ## 큐에 잡 추가하기
</div>

W&amp;B App을 사용해 대화형으로, 또는 W&amp;B CLI를 사용해 프로그래밍 방식으로 큐에 잡을 추가할 수 있습니다.

<Tabs>
  <Tab title="W&B app">
    W&amp;B App을 사용해 프로그래밍 방식으로 큐에 잡을 추가합니다.

    1. W&amp;B Project 페이지로 이동합니다.
    2. 프로젝트 사이드바에서 **Jobs** 아이콘을 선택합니다:

    <Frame>
      <img src="/images/launch/project_jobs_tab_gs.png" alt="Project Jobs 탭" />
    </Frame>

    3. **Jobs** 페이지에는 이전에 실행된 W&amp;B run에서 생성된 W&amp;B Launch 잡 목록이 표시됩니다.

    <Frame>
      <img src="/images/launch/view_jobs.png" alt="Jobs 목록" />
    </Frame>

    4. 잡 이름 옆에 있는 **Launch** 버튼을 선택합니다. 페이지 오른쪽에 모달이 표시됩니다.
    5. **Job version** 드롭다운에서 사용하려는 Launch 잡의 버전을 선택합니다. Launch 잡은 다른 [W&amp;B Artifact](/ko/models/artifacts/create-a-new-artifact-version/)와 마찬가지로 버전 관리됩니다. 잡을 실행하는 데 사용되는 소프트웨어 의존성이나 소스 코드를 수정하면 동일한 Launch 잡의 다른 버전이 생성됩니다.
    6. **Overrides** 섹션에서 Launch 잡에 대해 설정된 입력값에 새 값을 입력합니다. 일반적인 오버라이드에는 새로운 엔트리 포인트 명령, 인자, 또는 새 W&amp;B run의 `wandb.Run.config` 내 값이 포함됩니다.

    <Frame>
      <img src="/images/launch/create_starter_queue_gs.png" alt="Queue 설정" />
    </Frame>

    **Paste from...** 버튼을 클릭해, 이전에 해당 Launch 잡을 사용했던 다른 W&amp;B run에서 값을 복사해 붙여넣을 수 있습니다.
    7. **Queue** 드롭다운에서 Launch 잡을 추가할 Launch 큐의 이름을 선택합니다.
    8. **Job Priority** 드롭다운을 사용해 Launch 잡의 우선순위를 지정합니다. Launch 큐가 우선순위를 지원하지 않는 경우 Launch 잡의 우선순위는 &quot;Medium&quot;으로 설정됩니다.
    9. **(선택 사항) 팀 관리자에 의해 큐 config 템플릿이 생성된 경우에만 이 단계를 따르십시오**\
    **Queue Configurations** 필드에서 팀 관리자가 생성한 설정 옵션에 대한 값을 입력합니다.\
    예를 들어, 다음 예시에서 팀 관리자는 팀에서 사용할 수 있는 AWS 인스턴스 타입을 설정했습니다. 이 경우 팀 구성원은 `ml.m4.xlarge` 또는 `ml.p3.xlarge` 컴퓨팅 인스턴스 타입 중 하나를 선택해 모델을 학습할 수 있습니다.

    <Frame>
      <img src="/images/launch/team_member_use_config_template.png" alt="Config 템플릿 선택" />
    </Frame>

    10. 결과 run이 표시될 **Destination project**를 선택합니다. 이 프로젝트는 큐와 동일한 엔티티에 속해야 합니다.
    11. **Launch now** 버튼을 선택합니다.
  </Tab>

  <Tab title="W&B CLI">
    `wandb launch` 명령을 사용하여 큐에 잡을 추가합니다. 하이퍼파라미터 오버라이드를 포함하는 JSON 설정 파일을 생성합니다. 예를 들어, [퀵스타트](/ko/platform/launch/walkthrough/) 가이드의 스크립트를 사용해 다음과 같은 오버라이드가 포함된 JSON 파일을 생성할 수 있습니다:

    ```json title="config.json"
    {
      "overrides": {
          "args": [],
          "run_config": {
              "learning_rate": 0,
              "epochs": 0
          },   
          "entry_point": []
      }
    }
    ```

    <Note>
      JSON 설정 파일을 제공하지 않으면 W&amp;B Launch는 기본 파라미터를 사용합니다.
    </Note>

    큐 설정을 오버라이드하려 하거나, Launch 큐에 설정 리소스가 정의되어 있지 않은 경우 config.json 파일에 `resource_args` 키를 지정할 수 있습니다. 예를 들어 위의 예시를 이어서, config.json 파일은 다음과 비슷한 형태가 될 수 있습니다:

    ```json title="config.json"
    {
      "overrides": {
          "args": [],
          "run_config": {
              "learning_rate": 0,
              "epochs": 0
          },
          "entry_point": []
      },
      "resource_args": {
            "<resource-type>" : {
                "<key>": "<value>"
            }
      }
    }
    ```

    `<>` 안의 값을 자신의 값으로 교체합니다.

    `queue`(`-q`) 플래그에는 큐 이름을, `job`(`-j`) 플래그에는 잡 이름을, `config`(`-c`) 플래그에는 설정 파일 경로를 지정합니다.

    ```bash
    wandb launch -j <job> -q <queue-name> \ 
    -e <entity-name> -c path/to/config.json
    ```

    W&amp;B Team 내에서 작업하는 경우, 큐가 사용할 엔티티를 지정하기 위해 `entity` 플래그(`-e`)를 명시할 것을 권장합니다.
  </Tab>
</Tabs>
