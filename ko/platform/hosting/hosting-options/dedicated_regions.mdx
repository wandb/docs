title: 지원되는 Dedicated Cloud 리전
description: "W&B Dedicated Cloud 배포에 사용할 수 있는 AWS, Google Cloud, Azure 지원 리전을 모두 확인하세요."
---

이 페이지에는 W&amp;B가 Dedicated Cloud 배포에 대해 지원하는 AWS, Google Cloud, Azure 리전이 나와 있습니다. 새 Dedicated Cloud 인스턴스의 리전을 선택할 때는 이 표를 참고하여 데이터 상주 및 규정 준수, 지연 시간, 비용 효율성과 같은 요구 사항을 충족하세요.

<Note>
  선호하는 AWS, Google Cloud 또는 Azure 리전이 목록에 없으면 W&amp;B 지원팀에 문의하세요. W&amp;B는 해당 리전에 Dedicated Cloud에 필요한 모든 서비스가 있는지 확인하고, 평가 결과에 따라 지원 우선순위를 정할 수 있습니다.
</Note>

<div id="supported-aws-regions">
  ## 지원되는 AWS 리전
</div>

다음 표에는 W&amp;B가 Dedicated Cloud 인스턴스용으로 지원하는 [AWS 리전](https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/Concepts.RegionsAndAvailabilityZones.html)이 나와 있습니다.

| 리전 위치           | 리전 이름            |
| --------------- | ---------------- |
| 미국 동부(오하이오)     | `us-east-2`      |
| 미국 동부(버지니아 북부)  | `us-east-1`      |
| 미국 서부(캘리포니아 북부) | `us-west-1`      |
| 미국 서부(오리건)      | `us-west-2`      |
| 캐나다(중부)         | `ca-central-1`   |
| 유럽(프랑크푸르트)      | `eu-central-1`   |
| 유럽(아일랜드)        | `eu-west-1`      |
| 유럽(런던)          | `eu-west-2`      |
| 유럽(밀라노)         | `eu-south-1`     |
| 유럽(스톡홀름)        | `eu-north-1`     |
| 아시아 태평양(뭄바이)    | `ap-south-1`     |
| 아시아 태평양(싱가포르)   | `ap-southeast-1` |
| 아시아 태평양(시드니)    | `ap-southeast-2` |
| 아시아 태평양(도쿄)     | `ap-northeast-1` |
| 아시아 태평양(서울)     | `ap-northeast-2` |

AWS 리전에 대한 자세한 내용은 AWS 문서의 [Regions, Availability Zones, and Local Zones](https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/Concepts.RegionsAndAvailabilityZones.html)를 참조하세요.

AWS 리전을 선택할 때 고려해야 할 요소에 대한 개요는 [What to Consider when Selecting a Region for your Workloads](https://aws.amazon.com/blogs/architecture/what-to-consider-when-selecting-a-region-for-your-workloads/)를 참조하세요.

<div id="supported-google-cloud-regions">
  ## 지원되는 Google Cloud 리전
</div>

다음 표에는 W&amp;B가 Dedicated Cloud 인스턴스에 대해 지원하는 [Google Cloud 리전](https://cloud.google.com/compute/docs/regions-zones)이 나와 있습니다.

| 리전 위치    | 리전 이름                     |
| -------- | ------------------------- |
| 사우스캐롤라이나 | `us-east1`                |
| 북버지니아    | `us-east4`                |
| 아이오와     | `us-central1`             |
| 오리건      | `us-west1`                |
| 로스앤젤레스   | `us-west2`                |
| 라스베이거스   | `us-west4`                |
| 토론토      | `northamerica-northeast2` |
| 벨기에      | `europe-west1`            |
| 런던       | `europe-west2`            |
| 프랑크푸르트   | `europe-west3`            |
| 네덜란드     | `europe-west4`            |
| 시드니      | `australia-southeast1`    |
| 도쿄       | `asia-northeast1`         |
| 서울       | `asia-northeast3`         |

Google Cloud 리전에 관한 자세한 내용은 Google Cloud 문서의 [Regions and zones](https://cloud.google.com/compute/docs/regions-zones)를 참조하세요.

<div id="supported-azure-regions">
  ## 지원되는 Azure 리전
</div>

다음 표에는 W&amp;B가 Dedicated Cloud 인스턴스용으로 지원하는 [Azure 리전](https://azure.microsoft.com/explore/global-infrastructure/geographies/#geographies)이 나와 있습니다.

| 리전 위치          | 리전 이름           |
| -------------- | --------------- |
| Virginia       | `eastus`        |
| Iowa           | `centralus`     |
| Washington     | `westus2`       |
| California     | `westus`        |
| Canada Central | `canadacentral` |
| France Central | `francecentral` |
| Netherlands    | `westeurope`    |
| Tokyo, Saitama | `japaneast`     |
| Seoul          | `koreacentral`  |

Azure 리전에 대한 자세한 내용은 Azure 문서의 [Azure geographies](https://azure.microsoft.com/explore/global-infrastructure/geographies/#overview)를 참조하세요.
