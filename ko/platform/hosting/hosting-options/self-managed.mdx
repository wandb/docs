<Tip>
  W&amp;B는 [W&amp;B Multi-tenant Cloud](/ko/platform/hosting/hosting-options/multi_tenant_cloud) 또는 [W&amp;B Dedicated Cloud](/ko/platform/hosting/hosting-options/dedicated-cloud)와 같은 완전 관리형 배포 옵션을 권장합니다. W&amp;B 완전 관리형 서비스는 필요한 설정이 거의 없거나 전혀 없어 사용이 간편하고 안전합니다.
</Tip>

[AWS, Google Cloud 또는 Azure 클라우드 계정](#deploy-wb-server-within-Self-Managed-cloud-accounts)이나 [온프레미스 인프라](#deploy-wb-server-in-on-prem-infrastructure)에 W&amp;B Server를 배포하세요.

귀하의 IT/DevOps/MLOps 팀은 다음 사항을 책임집니다:

* 배포 프로비저닝
* 조직의 정책과 해당되는 경우 [보안 기술 구현 가이드라인 (STIG)](https://en.wikipedia.org/wiki/Security_Technical_Implementation_Guide)에 따라 인프라 보안 유지
* [Service and Organization Controls (SOC) 2 Type 2](https://www.aicpa-cima.com/topic/audit-assurance/audit-and-assurance-greater-than-soc-2) 및 [Health Insurance Portability and Accountability Act of 1996 (HIPAA)](https://www.hhs.gov/hipaa/for-professionals/index.html)와 같은 조직의 규제 요구 사항 준수 유지
* 업그레이드 관리 및 패치 적용
* W&amp;B Self-Managed 배포의 지속적인 유지 관리

<Info>
  조직이 규제 요구 사항의 적용을 받는 경우, W&amp;B가 유지 관리하는 [W&amp;B Dedicated Cloud](/ko/platform/hosting/hosting-options/dedicated-cloud)에 배포하는 방안을 고려해 보세요.

  * W&amp;B Dedicated Cloud의 호스팅 플랫폼은 SOC 2 Type 2 요구 사항을 충족합니다.
  * 적절히 설정하면 W&amp;B Dedicated Cloud 배포는 HIPAA를 준수합니다.

  자세한 정보는 [W&amp;B Security Portal](https://security.wandb.ai/)을 참고하세요.
</Info>

<div id="about-the-wb-kubernetes-operator">
  ## W&amp;B Kubernetes Operator 소개
</div>

W&amp;B Self-Managed를 배포하려면 **W&amp;B Kubernetes Operator**를 사용하세요. 이 Operator를 사용하면 W&amp;B의 배포, 관리, 문제 해결, 확장을 간소화할 수 있습니다.

이 Operator는 중앙 [deploy.wandb.ai](https://deploy.wandb.ai) 서버에 연결하여 지정된 릴리스 채널의 최신 사양 변경 사항을 요청하고 적용합니다. 라이선스가 유효한 동안에는 업데이트를 받을 수 있습니다. [Helm](https://helm.sh/)은 Operator를 설치하는 데 사용되며, Operator가 W&amp;B Kubernetes 스택을 관리하는 데에도 사용됩니다. 배포는 서비스별로 하나씩, 여러 개의 pod로 구성됩니다. 각 pod 이름에는 `wandb-` 접두사가 붙습니다.

설정은 **Release Channel Values**(W&amp;B 기본값), **User Input Values**(System Console을 통한 재정의), **Custom Resource Values**(사용자 사양이 앞의 두 값을 모두 재정의) 순의 계층 구조를 따릅니다. 자세한 내용은 [Deploy W&amp;B with Kubernetes Operator](/ko/platform/hosting/self-managed/operator)를 참조하세요.

* 퍼블릭 클라우드 또는 온프레미스 인프라에 W&amp;B Self-Managed를 배포하려면 [Deploy W&amp;B with Kubernetes Operator](/ko/platform/hosting/self-managed/operator)를 참조하세요.
* AWS가 아닌 맞춤형 클라우드 플랫폼에 W&amp;B Self-Managed를 배포하려면, 요구 사항은 [on-prem infrastructure](#deploy-wb-server-in-on-prem-infrastructure)에 배포하는 경우와 유사합니다.
* 에어갭 환경에 W&amp;B Self-Managed를 배포하려면 [Deploy on Air-Gapped Kubernetes](/ko/platform/hosting/self-managed/on-premises-deployments/kubernetes-airgapped)를 참조하세요.

안정성을 유지하기 위해 인스턴스에 선택적으로 [요청 속도 제한을 구성](/ko/platform/hosting/self-managed/rate-limits)할 수 있습니다. 기본값과 알림 동작에 대해서는 [요청 속도 제한](/ko/platform/hosting/self-managed/rate-limits)을 참조하세요.

<div id="required-infrastructure">
  ## 필수 인프라
</div>

모든 배포 옵션에는 다음 인프라가 필요합니다:

* Kubernetes
* MySQL 8 데이터베이스
* Amazon S3 호환 객체 저장소
* Redis 캐시

자세한 내용은 [Requirements](/ko/platform/hosting/self-managed/requirements)를 참조하세요. W&amp;B는 각 컴포넌트에 대한 권장 사항을 제공하고 설치 과정 전반에 걸쳐 안내를 제공할 수 있습니다.

<div id="obtain-your-wb-server-license">
  ## W&amp;B Server 라이선스 획득
</div>

배포 전에 W&amp;B Server 라이선스가 필요합니다. 단계별 안내는 Requirements 페이지의 [라이선스](/ko/platform/hosting/self-managed/requirements#license)를 참조하세요.
