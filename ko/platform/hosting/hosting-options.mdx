이 섹션에서는 W&amp;B를 배포하는 다양한 방법을 설명합니다.

<div id="wb-multi-tenant-cloud">
  ## W&amp;B 멀티테넌트 클라우드
</div>

[W&amp;B 멀티테넌트 클라우드](/ko/platform/hosting/#wb-multi-tenant-cloud)는 업그레이드, 유지 관리, 플랫폼 보안, 용량 계획을 포함한 모든 요소를 W&amp;B에서 완전히 관리하는 서비스입니다. 멀티테넌트 클라우드는 W&amp;B의 Google Cloud 계정에서 [Google Cloud 북미 리전](https://cloud.google.com/compute/docs/regions-zones)에 배포됩니다. 선택적으로 [Bring your own bucket (BYOB)](/ko/platform/hosting/data-security/secure-storage-connector)를 사용해 W&amp;B Artifact 및 기타 관련 민감한 데이터를 자체 클라우드나 온프레미스 인프라에 저장할 수 있습니다.

자세한 내용은 [W&amp;B 멀티테넌트 클라우드](/ko/platform/hosting/#wb-multi-tenant-cloud)를 확인하거나 [무료로 시작](https://app.wandb.ai/login?signup=true)해 보세요.

<div id="wb-dedicated-cloud">
  ## W&amp;B Dedicated Cloud
</div>

[W&amp;B Dedicated Cloud](/ko/platform/hosting/#wb-dedicated-cloud)는 엔터프라이즈 조직을 위해 설계된 단일 테넌트(single-tenant) 완전 관리형 플랫폼입니다. W&amp;B Dedicated Cloud는 W&amp;B의 AWS, Google Cloud 또는 Azure 계정에 배포됩니다. Dedicated Cloud는 Multi-tenant Cloud보다 더 높은 유연성을 제공하지만 W&amp;B Self-Managed보다는 복잡성이 낮습니다. 업그레이드, 유지 관리, 플랫폼 보안, 용량 계획은 W&amp;B에서 관리합니다. 각 Dedicated Cloud 인스턴스는 다른 W&amp;B Dedicated Cloud 인스턴스와 분리된 독립적인 네트워크, 컴퓨팅, 스토리지를 가집니다.

W&amp;B 전용 메타데이터와 데이터는 격리된 클라우드 스토리지에 저장되며, 격리된 클라우드 컴퓨팅 서비스에서 처리됩니다. 선택적으로 [Bring your own bucket (BYOB)](/ko/platform/hosting/data-security/secure-storage-connector)을 사용하여 아티팩트 및 기타 관련 민감 데이터를 사용자의 클라우드 또는 온프레미스 인프라에 저장할 수 있습니다.

W&amp;B Dedicated Cloud에는 중요한 보안 기능 및 기타 엔터프라이즈 환경에 적합한 기능을 지원하는 [엔터프라이즈 라이선스](/ko/platform/hosting/hosting-options/self-managed/)가 포함됩니다.

고급 보안 또는 컴플라이언스 요건이 있는 조직의 경우, HIPAA 컴플라이언스, Single Sign-On(SSO), Customer Managed Encryption Keys (CMEK)와 같은 기능을 **Enterprise** 지원과 함께 사용할 수 있습니다. [자세한 정보를 요청하세요](https://wandb.ai/site/contact).

[W&amp;B Dedicated Cloud](/ko/platform/hosting/#wb-dedicated-cloud)를 참고하거나 [무료로 시작](https://app.wandb.ai/login?signup=true)하세요.

<div id="wb-self-managed">
  ## W&amp;B 셀프 매니지드
</div>

[W&amp;B Self-Managed](/ko/platform/hosting/hosting-options/self-managed/)는 온프레미스 또는 사용자가 관리하는 클라우드 인프라에서 전적으로 직접 운영하는 방식입니다. 조직의 IT/DevOps/MLOps 팀은 다음 업무를 담당합니다:

* 배포 환경 프로비저닝.
* (해당되는 경우) 조직의 정책과 [Security Technical Implementation Guidelines (STIG)](https://en.wikipedia.org/wiki/Security_Technical_Implementation_Guide)에 따른 인프라 보안 구성.
* 업그레이드 관리 및 패치 적용.
* Self-Managed W&amp;B Server 인스턴스의 지속적인 유지 관리.

원하는 경우 W&amp;B Self-Managed용 엔터프라이즈 라이선스를 구매할 수 있습니다. 엔터프라이즈 라이선스에는 중요한 보안 기능과 기타 엔터프라이즈 환경에 적합한 기능에 대한 지원이 포함됩니다.

자세한 내용은 [W&amp;B Self-Managed](/ko/platform/hosting/hosting-options/self-managed/) 또는 [reference architecture](/ko/platform/hosting/self-managed/ref-arch/) 가이드를 참고하세요.
