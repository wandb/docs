---

이 페이지에서는 W&amp;B Server의 자동 버전 업그레이드를 비활성화하고 버전을 고정(pinning)하는 방법을 설명합니다. 이 지침은 [W&amp;B Kubernetes Operator](/ko/platform/hosting/operator/)로 관리되는 배포에만 적용됩니다.

<Note>
  W&amp;B는 주요 W&amp;B Server 릴리스를 최초 출시일로부터 12개월 동안 지원합니다. **Self-Managed** 인스턴스를 사용하는 고객은 지원을 유지하기 위해 적시에 업그레이드할 책임이 있습니다. 지원이 종료된 버전을 계속 사용하지 마세요. W&amp;B는 **Self-Managed** 인스턴스를 사용하는 고객이 최소 분기마다 한 번씩 최신 릴리스로 배포를 업데이트하여 지원을 유지하고, 최신 기능, 성능 개선 및 수정 사항을 받도록 강력히 권장합니다.
</Note>

<div id="requirements">
  ## 요구 사항
</div>

* W&amp;B Kubernetes Operator `v1.13.0` 이상
* System Console `v2.12.2` 이상

이 요구 사항을 충족하는지 확인하려면, 인스턴스의 W&amp;B Custom Resource 또는 Helm 차트를 참조하세요. `operator-wandb` 및 `system-console` 컴포넌트의 `version` 값을 확인하세요.

<div id="disable-automatic-updates">
  ## 자동 업데이트 비활성화
</div>

1. `admin` 역할을 가진 사용자로 W&amp;B 앱에 로그인합니다.
2. 상단의 사용자 아이콘을 클릭한 다음 **System Console**을 클릭합니다.
3. **Settings** &gt; **Advanced**로 이동한 다음 **Other** 탭을 선택합니다.
4. **Disable Auto Upgrades** 섹션에서 **Pin specific version**을 켭니다.
5. **Select a version** 드롭다운을 클릭하고 W&amp;B Server 버전을 선택합니다.
6. **Save**를 클릭합니다.

   <Frame>
     <img src="/images/hosting/disable_automatic_updates_saved_and_enabled.png" alt="자동 업데이트 비활성화 설정 저장됨" />
   </Frame>

   자동 업그레이드가 비활성화되며, W&amp;B Server는 선택한 버전에 고정됩니다.
7. 자동 업그레이드가 비활성화되었는지 확인합니다. **Operator** 탭으로 이동하여 조정(Reconciliation) 로그에서 `Version pinning is enabled` 문자열을 검색합니다.

```

│info 2025-04-17T17:24:16Z wandb default Active spec found
│info 2025-04-17T17:24:16Z wandb default Desired spec
│info 2025-04-17T17:24:16Z wandb default License
│info 2025-04-17T17:24:16Z wandb default Version Pinning is enabled
│info 2025-04-17T17:24:16Z wandb default Found Weights & Biases instance, processing the spec...
│info 2025-04-17T17:24:16Z wandb default === Reconciling Weights & Biases instance...
```
