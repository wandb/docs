---

W&amp;B Server를 설치할 때 사용한 것과 동일한 방법으로 W&amp;B Server Version 및 License를 업데이트하세요. 다음 표에는 서로 다른 배포 방법을 기준으로 라이선스와 버전을 업데이트하는 방법이 나와 있습니다:

| Release Type                        | 설명                                                                                                                                                                                                                                                                                         |
| ----------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| [Terraform](#update-with-terraform) | W&amp;B는 클라우드 배포를 위해 세 가지 공개 Terraform 모듈을 지원합니다: [AWS](https://registry.terraform.io/modules/wandb/wandb/aws/latest), [Google Cloud](https://registry.terraform.io/modules/wandb/wandb/google/latest), [Azure](https://registry.terraform.io/modules/wandb/wandb/azurerm/latest). |
| [Helm](#update-with-helm)           | 기존 Kubernetes cluster에 W&amp;B를 설치하려면 [Helm Chart](https://github.com/wandb/helm-charts)를 사용할 수 있습니다.                                                                                                                                                                                      |

<div id="update-with-terraform">
  ## Terraform으로 업데이트
</div>

Terraform으로 라이선스와 버전을 업데이트하세요. 다음 표에는 클라우드 플랫폼을 기준으로 한 W&amp;B 관리 Terraform 모듈이 나와 있습니다.

| 클라우드 제공업체    | Terraform 모듈                                                                                     |
| ------------ | ------------------------------------------------------------------------------------------------ |
| AWS          | [AWS Terraform module](https://registry.terraform.io/modules/wandb/wandb/aws/latest)             |
| Google Cloud | [Google Cloud Terraform module](https://registry.terraform.io/modules/wandb/wandb/google/latest) |
| Azure        | [Azure Terraform module](https://registry.terraform.io/modules/wandb/wandb/azurerm/latest)       |

1. 먼저, 사용 중인 클라우드 제공업체에 맞는 W&amp;B 유지 관리 Terraform 모듈로 이동하세요. 클라우드 제공업체를 기준으로 적절한 Terraform 모듈을 찾으려면 앞의 표를 참조하세요.

2. Terraform 설정에서 Terraform `wandb_app` 모듈 설정의 `wandb_version` 및 `license`를 업데이트하세요:

   ```hcl
   module "wandb_app" {
       source  = "wandb/wandb/<cloud-specific-module>"
       version = "new_version"
       license       = "new_license_key" # 새 라이선스 키
       wandb_version = "new_wandb_version" # 원하는 W&B 버전
       ...
   }
   ```

3. `terraform plan` 및 `terraform apply`로 Terraform 설정을 적용하세요.
   ```bash
   terraform init
   terraform apply
   ```

4. (선택) `terraform.tfvars` 또는 다른 `.tfvars` 파일을 사용하는 경우입니다.

   새 W&amp;B 버전과 라이선스 키로 `terraform.tfvars` 파일을 업데이트하거나 생성하세요.

   ```bash
   terraform plan -var-file="terraform.tfvars"
   ```

   설정을 적용하세요. Terraform workspace 디렉터리에서 다음을 실행하세요:

   ```bash
   terraform apply -var-file="terraform.tfvars"
   ```

<div id="update-with-helm">
  ## Helm으로 업데이트
</div>

<Warning>
  `wandb` Helm chart는 사용 중단되었으며 더 이상 지원되지 않습니다. 이 chart는 단일 pod를 배포하도록 설계되었으며, 현재는 [W&amp;B Kubernetes Operator](/ko/platform/hosting/self-managed/operator)로 대체되었습니다. 아직 이 chart를 사용 중이라면 [migration guide](/ko/platform/hosting/self-managed/operator#migrate-self-managed-instances-to-wb-operator)를 따라 Operator로 전환하세요.
</Warning>

<div id="update-wb-with-spec">
  ### spec으로 W&amp;B 업데이트
</div>

1. Helm chart `*.yaml` 설정 파일에서 `image.tag` 및/또는 `license` 값을 수정하여 새 버전을 지정하세요:

   ```yaml
   license: 'new_license'
   image:
     repository: wandb/local
     tag: 'new_version'
   ```

2. 다음 명령어로 Helm 업그레이드를 실행하세요:

   ```bash
   helm repo update
   helm upgrade --namespace=wandb --create-namespace \
     --install wandb wandb/wandb --version ${chart_version} \
     -f ${wandb_install_spec.yaml}
   ```

<div id="update-license-and-version-directly">
  ### 라이선스와 버전을 직접 업데이트하기
</div>

1. 새 라이선스 키와 이미지 태그를 환경 변수로 설정하세요:

   ```bash
   export LICENSE='new_license'
   export TAG='new_version'
   ```

2. 아래 명령으로 Helm 릴리스를 업그레이드하고, 새 값을 기존 설정과 병합하세요:

   ```bash
   helm repo update
   helm upgrade --namespace=wandb --create-namespace \
     --install wandb wandb/wandb --version ${chart_version} \
     --reuse-values --set license=$LICENSE --set image.tag=$TAG
   ```

자세한 내용은 공개 저장소의 [upgrade guide](https://github.com/wandb/helm-charts/blob/main/upgrade)를 참조하세요.

<div id="update-with-admin-ui">
  ## admin UI로 업데이트
</div>

이 방법은 일반적으로 Self-Managed Docker 설치에서 W&amp;B Server container의 환경 변수로 설정되지 않은 라이선스를 업데이트할 때만 작동합니다.

1. [W&amp;B Deployment Page](https://deploy.wandb.ai/)에서 새 라이선스를 획득하고, 업그레이드하려는 배포의 올바른 조직 및 배포 ID와 일치하는지 확인하세요.
2. W&amp;B App에서 **License** 페이지에 액세스하세요. **Settings** &gt; **License**를 클릭하거나 `<host-url>/console/settings/license`로 이동하세요.
3. 라이선스 관리 섹션으로 이동하세요.
4. 새 라이선스 키를 입력하고 변경 사항을 저장하세요.
