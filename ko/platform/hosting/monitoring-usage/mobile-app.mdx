iOS용 W&amp;B 모바일 앱을 사용하면 어디서든 W&amp;B Models 프로젝트에 계속 연결된 상태를 유지할 수 있습니다. iPhone 또는 iPad에서 실시간으로 트레이닝 run을 추적하고 run 로그를 조회하고, 선형 플롯을 확인하고, 프로젝트 기록을 탐색하고, 팀의 진행 상황을 살펴볼 수 있습니다.

<Info>
  모바일 앱은 [Multi-tenant Cloud](/ko/platform/hosting/hosting-options/multi_tenant_cloud) 계정에서만 사용할 수 있습니다. [Dedicated Cloud](/ko/platform/hosting/hosting-options/dedicated-cloud) 또는 [Self-Managed](/ko/platform/hosting/hosting-options/self-managed) deployment에서는 사용할 수 없습니다.
</Info>

<div id="features">
  ## 기능
</div>

* **프로젝트 탐색**: W&amp;B 프로젝트 전반을 둘러보고 검색하세요.
  * **전체 프로젝트 이력**: 프로젝트 패널의 메트릭 차트에는 최신 run만 아니라 프로젝트의 *모든 run* 데이터가 포함됩니다.
  * **프로젝트에 별표 표시**: 가장 중요한 프로젝트를 한눈에 확인하세요. 프로젝트 옆의 별표 아이콘을 클릭해 별표를 표시하세요. 별표가 표시된 프로젝트만 보려면 목록 상단의 **Starred** 탭을 클릭하세요.
* **실험 추적**: 실험의 run 상태, 메트릭, 선형 플롯을 실시간으로 확인하세요.
  * **run 개요 탭**: 각 run에서 **Overview** 탭을 열면 메트릭 및 로그와 함께 run 세부 정보의 요약을 확인할 수 있습니다.
  * **모바일에 최적화된 패널 그룹화**: 패널은 이름별로 자동 그룹화되어 접을 수 있는 섹션으로 표시됩니다. 섹션은 단일 수준(중첩 없음)이며, 그룹화는 W&amp;B 웹 앱의 [workspace 패널](/ko/models/app/features/panels)과 동일한 규칙을 따르므로 데스크톱과 휴대폰 사이를 이동해도 레이아웃이 일관되게 유지됩니다.
  * **패널에 별표 표시**: 가장 중요한 패널을 한눈에 확인하세요. run 또는 프로젝트를 보고 있을 때 패널 상단의 별표 아이콘을 클릭해 별표를 표시하세요. 별표가 표시된 패널만 보려면 목록 상단의 **Starred** 탭을 클릭하세요.
  * **패널 검색**: run 또는 프로젝트를 보고 있을 때 화면 하단의 **panel search** 필드를 사용해 각 차트에 표시되는 run을 필터링하세요. [JavaScript 정규 표현식](https://www.w3schools.com/js/js_regexp.asp)을 사용해 run 이름의 패턴을 검색할 수 있습니다.
  * **차트 툴팁**: 선형 차트에서는 툴팁에 메트릭 값이 소수점 넷째 자리까지 표시되므로 작은 변화도 정확하게 확인할 수 있습니다.
  * **run 중지**: 진행 중인 run을 보고 있을 때 **액션 (<Icon icon="ellipsis" iconType="solid" />)** 메뉴를 클릭한 다음 **Stop run**을 클릭하세요.
  * **콘솔 로그 보기**: 활성 상태이거나 완료된 run의 [콘솔 로그](/ko/models/runs/view-logged-runs#logs)를 확인하거나, 완료된 run의 로그를 다운로드할 수 있습니다. 앱은 활성 run에서 새 로그 줄이 있는지 5초마다 확인합니다. 기본적으로 가장 최근 10,000줄이 표시되며, 위로 스크롤해 이전 로그를 볼 수 있습니다. 출력이 매우 큰 경우에도 로그 검색은 빠르게 응답합니다.
* **Teams 전환**: **팀 선택기**를 사용해 Teams를 전환하세요. 이 picker는 더 빠른 내비게이션을 위해 업데이트된 레이아웃을 사용합니다.
* **최신 정보 확인**: 알림을 사용하면 노트북을 열지 않고도 실험 업데이트를 확인할 수 있습니다. 더 이상 필요하지 않은 알림은 **Notifications** 탭에서 스와이프해 제거하세요.

<div id="set-up-notifications">
  ## 알림 설정
</div>

알림은 두 가지 방법으로 설정할 수 있습니다.

**메트릭 임계값 알림**

1. run으로 이동합니다.
2. 그래프를 탭해 전체 화면 뷰로 들어갑니다.
3. 오른쪽 상단의 종 아이콘을 탭합니다.
4. 이후 run이 지정한 메트릭 임계값을 넘을 때 트리거되는 알림을 설정합니다.

**run 실패 알림**

1. 프로젝트로 이동합니다.
2. 오른쪽 상단의 **액션 (<Icon icon="ellipsis" iconType="solid" />)** 메뉴를 탭합니다.
3. 해당 프로젝트에서 이후 run 실패에 대한 알림을 받으려면 **Run failed alert**를 선택합니다.

**알림 관리** 화면에는 두 종류의 설정이 한곳에 표시되며, 각 섹션에서 제어하는 내용을 설명하는 짧은 설명도 함께 제공됩니다.
