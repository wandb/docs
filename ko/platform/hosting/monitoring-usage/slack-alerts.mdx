<Note>
  [W&amp;B Dedicated Cloud 배포에서 Slack 알림을 설정하는 방법을 보여주는 비디오](https://www.youtube.com/watch?v=JmvKb-7u-oU) (6분)을 시청하세요.
</Note>

<div id="create-the-slack-application">
  ## Slack 애플리케이션 만들기
</div>

아래 절차에 따라 Slack 애플리케이션을 만드세요.

1. https://api.slack.com/apps에 접속한 다음 **Create an App**를 선택하세요.

   <Frame>
     <img src="/images/hosting/create_an_app.png" alt="앱 만들기 버튼" />
   </Frame>

2. **App Name** 필드에 앱 이름을 입력하세요.

3. 앱을 개발할 Slack 워크스페이스를 선택하세요. 이때 사용하는 Slack 워크스페이스가 알림에 사용할 워크스페이스와 동일한지 확인하세요.

   <Frame>
     <img src="/images/hosting/name_app_workspace.png" alt="앱 이름 및 워크스페이스 선택" />
   </Frame>

<div id="configure-the-slack-application">
  ## Slack 애플리케이션 설정
</div>

1. 왼쪽 사이드바에서 **OAth &amp; Permissions**를 선택합니다.

   <Frame>
     <img src="/images/hosting/add_an_oath.png" alt="OAuth & Permissions 메뉴" />
   </Frame>

2. 범위 섹션에서 봇에 **incoming&#95;webhook** 범위를 부여합니다. 범위는 개발 워크스페이스에서 앱이 액션을 수행할 수 있는 권한을 제공합니다.

   Bot용 OAuth 범위에 대한 자세한 내용은 Slack API 문서의 &quot;Understanding OAuth scopes for Bots&quot; 튜토리얼을 참조하세요.

   <Frame>
     <img src="/images/hosting/save_urls.png" alt="Bot 토큰 범위" />
   </Frame>

3. Redirect URL을 W&amp;B 설치 주소를 가리키도록 설정합니다. 로컬 시스템 설정에서 호스트 URL로 설정한 것과 동일한 URL을 사용합니다. 인스턴스에 서로 다른 DNS 매핑이 있는 경우 여러 개의 URL을 지정할 수 있습니다.

   <Frame>
     <img src="/images/hosting/redirect_urls.png" alt="Redirect URLs 설정" />
   </Frame>

4. **Save URLs**를 선택합니다.

5. 선택적으로 **Restrict API Token Usage**에서 IP 범위를 지정해 W&amp;B 인스턴스의 IP 또는 IP 범위를 허용 목록(allow-list)에 추가할 수 있습니다. 허용되는 IP 주소를 제한하면 Slack 애플리케이션의 보안을 한층 더 강화할 수 있습니다.

<div id="register-your-slack-application-with-wb">
  ## W&amp;B에 Slack 애플리케이션 등록하기
</div>

1. 배포 방식에 따라 W&amp;B 인스턴스의 **System Settings** 또는 **System Console** 페이지로 이동합니다.

2. 사용 중인 시스템 페이지에 따라 아래 옵션 중 하나를 따릅니다.

   * **System Console**에 있는 경우: **Settings**로 이동한 뒤 **알림**으로 이동합니다.

     <Frame>
       <img src="/images/hosting/register_slack_app_console.png" alt="System Console notifications" />
     </Frame>

   * **System Settings**에 있는 경우: 맞춤형 Slack 애플리케이션을 활성화하려면 **Enable a custom Slack application to dispatch alerts** 표시/숨기기를 켭니다.

     <Frame>
       <img src="/images/hosting/register_slack_app.png" alt="Enable Slack application toggle" />
     </Frame>

3. **Slack client ID**와 **Slack secret**을 입력한 다음 **Save**를 클릭합니다. 애플리케이션의 client ID와 secret은 **Settings**의 **Basic Information**에서 확인할 수 있습니다.

4. W&amp;B App에서 Slack 인테그레이션을 설정해 정상적으로 동작하는지 확인합니다.
