---
title: 조직 활동 보기
---

이 페이지에서는 W&amp;B 조직 내 활동을 조회하는 방법을 설명합니다. 계속하려면 배포 유형을 선택하세요.

<div id="view-user-status-and-activity">
  ## 사용자 상태 및 활동 보기
</div>

<Tabs>
  <Tab title="Dedicated Cloud / Self-Managed">
    1. **Organization Dashboard**로 이동합니다:

       * **Dedicated Cloud**: `https://<org-name>.io/org/dashboard/` — `<org-name>`을(를) 조직 이름으로 바꿉니다.
       * **Self-Managed**: `https://<YOUR-W&B-SERVER-IP>/org/dashboard` — `<YOUR-W&B-SERVER-IP>`을(를) 배포의 IP 주소로 바꿉니다.

       기본적으로 **Users** 탭이 열리며, 조직의 모든 사용자가 목록에 표시됩니다.
    2. 사용자 상태별로 목록을 정렬하려면 **Last Active** 열 헤더를 클릭합니다. 각 사용자의 상태는 다음 중 하나입니다:
       * **Invite pending**: 초대는 전송되었지만 아직 수락되지 않은 상태입니다.
       * **A timestamp**: 사용자가 초대를 수락하고 최소 한 번 이상 로그인한 상태입니다. 타임스탬프는 가장 최근 활동 시점을 나타냅니다.
       * **Deactivated**: 관리자가 사용자의 접근을 해지한 상태입니다.
       * **No status**: 하이픈(-)으로 표시됩니다. 사용자가 이전에는 활동했지만, 지난 6개월 동안은 활동이 없는 상태입니다.
    3. 사용자의 **Last Active** 필드 위에 마우스를 올리면 사용자가 추가된 날짜와 총 활동 일수를 확인할 수 있습니다.

    사용자가 *active* 상태로 간주되는 경우는 다음과 같습니다:

    * W&amp;B에 로그인한 경우
    * W&amp;B App에서 어떤 페이지라도 연 경우
    * 실행을 로깅한 경우
    * SDK를 사용해 실험을 추적한 경우
    * 어떤 방식으로든 W&amp;B 서버와 상호작용한 경우
  </Tab>

  <Tab title="Multi-tenant Cloud">
    1. [**Members** 페이지](https://wandb.ai/account-settings/wandb/members/)를 엽니다. 이 테이블에는 조직의 모든 사용자가 표시됩니다.
    2. 사용자 상태별로 정렬하려면 **Last Active** 열 헤더를 클릭합니다. 각 사용자의 상태는 다음 중 하나입니다:
       * **A timestamp**: 사용자가 최소 한 번 이상 로그인한 상태입니다. 타임스탬프는 가장 최근 활동 시점을 나타냅니다.
       * **No status**: 하이픈(-)으로 표시됩니다. 사용자가 아직 조직 내에서 활동한 적이 없는 상태입니다.

    사용자는 2025년 5월 8일 이후 조직 범위의 감사(audit) 가능한 작업을 수행하면 *active* 상태로 간주됩니다. 전체 목록은 **Audit logging**의 [Actions](/ko/platform/hosting/monitoring-usage/audit-logging#actions)를 참조하세요.
  </Tab>
</Tabs>

<div id="view-activity-over-time">
  ## 기간별 활동 보기
</div>

<Tabs>
  <Tab title="Dedicated Cloud / Self-Managed">
    지정한 기간 동안 얼마나 많은 사용자가 활동했는지 보려면 **Activity** 탭을 사용합니다.

    1. **Organization Dashboard**를 엽니다. [View user activity](#view-user-activity)를 참조하세요.
    2. **Activity**를 클릭합니다.
    3. 다음 차트를 확인합니다:
       * **Total active users**: 선택한 기간 동안의 고유 활성 사용자 수(기본값은 3개월)입니다.
       * **Users active over time**: 해당 기간 동안 활성 사용자 수의 변동(기본값은 6개월)입니다. 포인트 위에 마우스를 올리면 해당 날짜의 정확한 수를 볼 수 있습니다.

    보고 기간을 변경하려면 차트 위의 드롭다운을 사용합니다. 선택 가능한 옵션은 Last 30 days, Last 3 months, Last 6 months, Last 12 months, All time입니다.
  </Tab>

  <Tab title="Multi-tenant Cloud">
    집계된 활동을 보려면 **Activity Dashboard**를 사용합니다.

    1. W&amp;B App 오른쪽 상단에서 사용자 아이콘을 클릭합니다.
    2. **Account** 아래에서 **Users**를 클릭합니다.
    3. 사용자 테이블 상단의 Activity Panel을 확인합니다:
       * **Active user count**: 선택한 기간 동안의 고유 활성 사용자 수(기본값은 3개월)입니다.
       * **Weekly active users**: 주당 활성 사용자 수입니다.
       * **Most active user**: 활성 일수와 마지막 활동 날짜를 기준으로 한 상위 10명의 사용자입니다.
    4. 날짜 범위를 변경하려면 오른쪽 상단의 날짜 선택기를 클릭하고 다음 중 새 값을 선택합니다: 7일, 30일(기본값), 90일, 6개월, 12개월. 모든 차트는 자동으로 업데이트됩니다.
  </Tab>
</Tabs>

<div id="export-user-details">
  ## 사용자 세부 정보 내보내기
</div>

**Users** 탭에서 각 사용자의 세부 정보(사용자 이름, 이메일 주소, 마지막 활동 시간, 역할 등)가 포함된 CSV 파일을 다운로드할 수 있습니다.

<Tabs>
  <Tab title="Dedicated Cloud / Self-Managed">
    1. **Users** 탭에서 **Invite new user** 옆의 **…** 작업 메뉴를 클릭합니다.
    2. **Export as CSV**를 클릭합니다.

    ### CSV 스키마 내보내기

    CSV 내보내기는 쉼표(`,`)를 구분 기호로 사용하고, 문자열은 큰따옴표로 둘러싸며, 다음 열이 정의된 헤더 행을 포함합니다:

    * `"Name"`
    * `"Username"`
    * `"Org Role"`
    * `"Models Seat"`
    * `"Weave Access"`
    * `"Email"`
    * `"Teams"`
    * `"Last Active"`
  </Tab>

  <Tab title="Multi-tenant Cloud">
    1. **Users** 탭에서 오른쪽 상단의 **…** 작업 메뉴를 클릭합니다.
    2. **Export as CSV**를 선택하여 파일을 다운로드합니다.

    ### CSV 스키마 내보내기

    CSV 내보내기는 쉼표(`,`)를 구분 기호로 사용하고, 문자열은 큰따옴표로 둘러싸며, 다음 열이 정의된 헤더 행을 포함합니다:

    * `"Name"`
    * `"Username"`
    * `"Last Active"`
    * `"Role"`
    * `"Email"`
    * `"Teams"`
    * `"Status"`
    * `"Number of Reports"`
    * `"Number of Runs"`
    * `"Number of active days"`
    * `"Models Seat"`
    * `"Weave Access"`
  </Tab>
</Tabs>
