---

이 페이지에서는 W&amp;B 조직의 활동을 확인하는 방법을 설명합니다. 계속하려면 배포 유형을 선택하세요.

<div id="view-user-status-and-activity">
  ## 사용자 상태와 활동 보기
</div>

<Tabs>
  <Tab title="Dedicated Cloud / Self-Managed">
    1. **Organization Dashboard**로 이동합니다.

       * **Dedicated Cloud**: `https://<org-name>.io/org/dashboard/`. `<org-name>`을 조직 이름으로 바꾸세요.
       * **Self-Managed**: `https://<YOUR-W&B-SERVER-IP>/org/dashboard`. `<YOUR-W&B-SERVER-IP>`를 배포의 IP 주소로 바꾸세요.

       기본적으로 **Users** 탭이 열리며 조직의 모든 사용자가 표시됩니다.
    2. 사용자 상태를 기준으로 목록을 정렬하려면 **Last Active** 열 헤더를 클릭합니다. 각 사용자의 상태는 다음 중 하나입니다.
       * **Invite pending**: 초대가 전송되었지만 아직 수락되지 않았습니다.
       * **타임스탬프**: 사용자가 초대를 수락했고 최소 한 번 로그인했습니다. 타임스탬프는 가장 최근 활동 시점을 나타냅니다.
       * **Deactivated**: 관리자가 사용자의 접근 권한을 철회했습니다.
       * **No status**: 하이픈으로 표시됩니다. 이전에는 활성 상태였지만 지난 6개월 동안 활동이 없었던 사용자입니다.
    3. 사용자가 추가된 날짜와 총 활성 일수를 보려면 사용자의 **Last Active** 필드에 마우스를 올리세요.

    다음 중 하나에 해당하면 사용자는 *활성* 상태입니다:

    * W&amp;B에 로그인합니다.
    * **W&amp;B App**의 페이지를 하나라도 엽니다.
    * run을 로깅합니다.
    * SDK를 사용해 실험을 추적합니다.
    * 어떤 방식으로든 W&amp;B 서버와 상호작용합니다.
  </Tab>

  <Tab title="Multi-tenant Cloud">
    1. [**Members** 페이지](https://wandb.ai/account-settings/wandb/members/)를 엽니다. 테이블에는 조직의 모든 사용자가 표시됩니다.
    2. 사용자 상태를 기준으로 정렬하려면 **Last Active** 열 헤더를 클릭합니다. 각 사용자의 상태는 다음 중 하나입니다.
       * **타임스탬프**: 사용자가 최소 한 번 로그인했습니다. 타임스탬프는 가장 최근 활동 시점을 나타냅니다.
       * **No status**: 하이픈으로 표시됩니다. 사용자가 아직 조직 내에서 활동한 적이 없습니다.

    2025년 5월 8일 이후 조직 범위의 감사 대상 작업을 하나라도 수행하면 사용자는 *활성* 상태입니다. 전체 목록은 **Audit logging**의 [Actions](/ko/platform/hosting/monitoring-usage/audit-logging#actions)를 참조하세요.
  </Tab>
</Tabs>

<div id="view-activity-over-time">
  ## 시간 경과에 따른 활동 보기
</div>

<Tabs>
  <Tab title="Dedicated Cloud / Self-Managed">
    지정한 기간 동안 활성 상태였던 사용자 수를 확인하려면 **Activity** 탭을 사용하세요.

    1. **Organization Dashboard**를 엽니다. [사용자 활동 보기](#view-user-activity)를 참조하세요.
    2. **Activity**를 클릭합니다.
    3. 다음 플롯을 확인합니다.
       * **Total active users**: 선택한 기간 동안의 고유 활성 사용자 수입니다(기본값: 3개월).
       * **Users active over time**: 해당 기간 동안 활성 사용자 수의 변동 추이입니다(기본값: 6개월). 특정 날짜의 정확한 수를 보려면 점 위에 마우스를 올리세요.

    보고 기간을 변경하려면 플롯 위의 드롭다운을 사용하세요. 옵션은 Last 30 days, Last 3 months, Last 6 months, Last 12 months, All time입니다.
  </Tab>

  <Tab title="Multi-tenant Cloud">
    집계된 활동을 보려면 **Activity Dashboard**를 사용하세요.

    1. W&amp;B App의 오른쪽 상단에서 사용자 아이콘을 클릭합니다.
    2. **Account** 아래에서 **Users**를 클릭합니다.
    3. 사용자 테이블 위의 Activity Panel을 확인합니다.
       * **Active user count**: 선택한 기간 동안의 고유 활성 사용자 수입니다(기본값: 3개월).
       * **Weekly active users**: 주별 활성 사용자 수입니다.
       * **Most active user**: 활성 일수와 마지막 활성 날짜를 기준으로 순위가 매겨진 상위 10명의 사용자입니다.
    4. 날짜 범위를 변경하려면 오른쪽 상단의 날짜 선택기를 클릭하고 새 값을 선택하세요: 7일, 30일(기본값), 90일, 6개월 또는 12개월. 모든 플롯이 자동으로 업데이트됩니다.
  </Tab>
</Tabs>

<div id="export-user-details">
  ## 사용자 세부 정보 내보내기
</div>

**Users** 탭에서 각 사용자의 세부 정보(사용자 이름, 이메일 주소, 마지막 활동 시간, 역할 등)가 나열된 CSV를 다운로드할 수 있습니다.

<Tabs>
  <Tab title="Dedicated Cloud / Self-Managed">
    1. **Users** 탭에서 **Invite new user** 옆의 **…** 작업 메뉴를 클릭합니다.
    2. **Export as CSV**를 클릭합니다.

    ### CSV 내보내기 스키마

    CSV 내보내기에서는 쉼표(`,`)를 구분자로 사용하고, 문자열을 큰따옴표로 감싸며, 다음 열을 정의하는 헤더 행을 포함합니다.

    * `"Name"`
    * `"Username"`
    * `"Org Role"`
    * `"Models Seat"`
    * `"Weave Access"`
    * `"Email"`
    * `"Teams"`
    * `"Last Active"`
  </Tab>

  <Tab title="Multi-tenant Cloud">
    1. **Users** 탭에서 오른쪽 상단의 **…** 작업 메뉴를 클릭합니다.
    2. **Export as CSV**를 선택해 파일을 다운로드합니다.

    ### CSV 내보내기 스키마

    CSV 내보내기에서는 쉼표(`,`)를 구분자로 사용하고, 문자열을 큰따옴표로 감싸며, 다음 열을 정의하는 헤더 행을 포함합니다.

    * `"Name"`
    * `"Username"`
    * `"Last Active"`
    * `"Role"`
    * `"Email"`
    * `"Teams"`
    * `"Status"`
    * `"Number of Reports"`
    * `"Number of Runs"`
    * `"Number of active days"`
    * `"Models Seat"`
    * `"Weave Access"`
  </Tab>
</Tabs>
