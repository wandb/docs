title: Dedicated Cloud에서 데이터 내보내기
---

Dedicated Cloud 인스턴스에서 관리되는 모든 데이터를 내보내고 싶다면, W&amp;B SDK API를 사용하여 [Import and Export API](/ko/models/ref/python/public-api/)를 통해 실행, 메트릭, 아티팩트 등의 데이터를 추출할 수 있습니다. 아래 표에서는 주요 내보내기 사용 사례를 정리했습니다.

| 목적 | 문서 |
|---------|---------------|
| 프로젝트 메타데이터 내보내기 | [Projects API](/ko/models/ref/python/public-api/projects) |
| 프로젝트의 실행 내보내기 | [Runs API](/ko/models/ref/python/public-api/runs) |
| 리포트 내보내기 | [Report and Workspace API](/ko/models/reports/clone-and-export-reports/) |
| 아티팩트 내보내기 | [Explore artifact graphs](/ko/models/artifacts/explore-and-traverse-an-artifact-graph/), [Download and use artifacts](/ko/models/artifacts/download-and-use-an-artifact/#download-and-use-an-artifact-stored-on-wb) |

Dedicated Cloud에 저장된 아티팩트를 [Secure Storage Connector](/ko/platform/app/settings-page/teams/#secure-storage-connector)로 관리하는 경우, W&amp;B SDK API를 사용해 아티팩트를 내보낼 필요가 없을 수도 있습니다.

<Note>
  W&amp;B SDK API를 사용해 모든 데이터를 내보내는 작업은 실행, 메트릭, 아티팩트 등이 매우 많은 경우 느릴 수 있습니다. Dedicated Cloud 인스턴스에 과부하가 걸리지 않도록 내보내기 작업을 적절한 크기의 배치로 나누어 실행할 것을 W&amp;B에서 권장합니다.
</Note>
