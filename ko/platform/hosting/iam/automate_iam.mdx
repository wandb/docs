이 페이지에서는 [SCIM API](/ko/platform/hosting/iam/scim)와 [Python SDK API](/ko/models/ref/python/public-api)를 사용해 사용자 및 팀 관리를 자동화하는 방법을 설명합니다.

<div id="scim-api">
  ## SCIM API
</div>

Okta 또는 Microsoft Entra와 같은 ID 공급자(IdP)를 사용해 W&amp;B SCIM API로 W&amp;B 조직의 사용자 및 팀를 대규모로 관리할 수 있습니다.

자세한 내용과 예시는 SCIM 레퍼런스 페이지의 [인증](./scim#authentication)을 참조하세요.

<Tip>
  W&amp;B 구현에는 역할 할당을 위한 엔드포인트, 커스텀 역할을 생성하고 관리하기 위한 엔드포인트, 그리고 기본 제공 역할과 커스텀 역할을 할당하기 위한 엔드포인트가 포함되어 있습니다. 역할 엔드포인트는 공식 SCIM 스키마에 포함되지 않습니다. W&amp;B는 커스텀 역할의 자동화된 관리를 지원하기 위해 역할 엔드포인트를 추가했습니다.
</Tip>

다음 섹션에서는 SCIM API의 각 범주를 설명합니다.

<div id="user-scim-api">
  ### 사용자 SCIM API
</div>

[사용자 SCIM API](./scim#user-resource)를 사용하면 W&amp;B 조직에서 사용자를 생성, 비활성화, 조회, 목록으로 확인하고, 미리 정의된 역할 또는 커스텀 역할을 할당할 수 있습니다. 전체 요청/응답 예시는 자세한 [SCIM 레퍼런스](./scim#user-management)에서 확인하세요.

<Note>
  `PATCH /scim/Users/{id}`로 사용자를 비활성화하고 `{"active": false}`로 설정하세요. 호스팅 옵션에 따라 결과가 달라집니다. Dedicated Cloud 및 Self-Managed 배포 환경에서는 사용자 기록이 유지되지만, Multi-tenant Cloud에서는 사용자가 조직에서 제거됩니다. Multi-tenant Cloud에서는 재활성화할 수 없으므로 대신 사용자를 다시 추가하세요. [사용자 비활성화](./scim#deactivate-user) 및 [사용자 재활성화](./scim#reactivate-user)를 참조하세요.
</Note>

<div id="group-scim-api">
  ### Group SCIM API
</div>

[Group SCIM API](./scim#group-resource)를 사용하면 조직에서 W&amp;B 팀를 생성하거나 삭제하는 것을 포함해 관리할 수 있습니다. 기존 팀에 사용자를 추가하거나 제거하려면 `PATCH Group`을 사용하세요.

<Note>
  W&amp;B에는 `같은 역할을 가진 사용자 그룹`이라는 개념이 없습니다. W&amp;B 팀은 그룹과 매우 유사하며, 서로 다른 역할을 가진 다양한 사용자가 관련 프로젝트 집합에서 협업할 수 있게 해줍니다. 팀는 서로 다른 사용자 그룹으로 구성될 수 있습니다. 팀의 각 사용자에게 역할(team admin, member, viewer 또는 커스텀 역할)을 할당하세요.

  그룹과 W&amp;B 팀이 유사하기 때문에 W&amp;B는 Group SCIM API endpoints를 W&amp;B 팀에 매핑합니다.
</Note>

<div id="custom-role-api">
  ### 커스텀 역할 API
</div>

[Custom Role SCIM API](./scim#role-resource)를 사용하면 조직에서 커스텀 역할을 생성, 목록 조회, 업데이트하는 등 커스텀 역할을 관리할 수 있습니다.

<Warning>
  커스텀 역할을 삭제할 때는 주의하세요.

  W&amp;B 조직 내 커스텀 역할은 `DELETE Role` 엔드포인트를 사용해 삭제합니다. 오퍼레이션이 수행되기 전에, 해당 커스텀 역할이 상속하는 사전 정의된 역할이 그 커스텀 역할을 할당받은 모든 사용자에게 할당됩니다.

  커스텀 역할의 상속 역할은 `PUT Role` 엔드포인트를 사용해 업데이트합니다. 이 오퍼레이션은 커스텀 역할에 이미 있는 기존 권한, 즉 상속되지 않은 커스텀 권한에는 영향을 주지 않습니다.
</Warning>

<div id="wb-python-sdk-api">
  ## W&amp;B Python SDK API
</div>

조직의 사용자, 팀, 팀 멤버십을 관리하려면 [W&amp;B Python SDK API](/ko/models/ref/python/public-api/api)를 사용하세요.

* [`User`](/ko/models/ref/python/public-api/user)
* [`Team`](/ko/models/ref/python/public-api/team)
* [`Member`](/ko/models/ref/python/public-api/member)
