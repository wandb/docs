title: 프로젝트에 대한 액세스 제어 관리
---

W&B 프로젝트의 범위를 정의하여 누가 해당 프로젝트에 W&B 실행을 조회, 편집, 제출할 수 있는지 제한합니다. 

여러 가지 제어 기능을 조합하여 W&B 팀 내 모든 프로젝트의 액세스 수준을 구성할 수 있습니다. **Visibility scope**는 상위 수준 메커니즘입니다. 이를 사용하여 어떤 사용자 그룹이 프로젝트에서 실행을 조회하거나 제출할 수 있는지 제어합니다. _Team_ 또는 _Restricted_ visibility scope를 사용하는 프로젝트의 경우 **프로젝트 수준 역할**을 사용하여 각 사용자가 프로젝트 내에서 가지는 액세스 수준을 제어할 수 있습니다.

<Note>
프로젝트 소유자, 팀 관리자, 또는 조직 관리자는 프로젝트의 가시성을 설정하거나 편집할 수 있습니다.
</Note>

<div id="visibility-scopes">
  ## 가시성 범위
</div>

선택할 수 있는 프로젝트 가시성 범위는 네 가지입니다. 가장 공개적인 설정부터 가장 비공개 설정까지 순서는 다음과 같습니다. 

| Scope | Description | 
| ----- | ----- |
| Open |프로젝트를 알고 있는 사람이라면 누구나 프로젝트를 열람하고 실행이나 리포트를 제출할 수 있습니다.|
| Public |프로젝트를 알고 있는 사람이라면 누구나 프로젝트를 열람할 수 있습니다. 실행이나 리포트를 제출할 수 있는 사람은 팀 구성원으로만 제한됩니다.|
| Team | 상위 팀의 구성원만 프로젝트를 열람하고 실행이나 리포트를 제출할 수 있습니다. 팀 외부의 사용자는 프로젝트에 접근할 수 없습니다. |
| Restricted| 상위 팀에서 초대된 구성원만 프로젝트를 열람하고 실행이나 리포트를 제출할 수 있습니다.|

<Note>
민감하거나 기밀 데이터와 관련된 워크플로에서 협업하려면 프로젝트의 가시성 범위를 **Restricted**로 설정하세요. 팀 내에서 Restricted 프로젝트를 생성하면, 해당 실험, 아티팩트, 리포트 등에 대해 협업할 특정 팀 구성원을 초대하거나 추가할 수 있습니다. 

다른 프로젝트 범위와 달리, 팀의 모든 구성원이 Restricted 프로젝트에 자동으로 접근 권한을 갖지는 않습니다. 동시에, 필요하다면 팀 관리자는 Restricted 프로젝트에 참여할 수 있습니다.
</Note>

<div id="set-visibility-scope-on-a-new-or-existing-project">
  ### 새 프로젝트 또는 기존 프로젝트의 가시성 범위 설정
</div>

프로젝트를 생성할 때 또는 나중에 편집할 때 프로젝트의 가시성 범위를 설정하세요.

<Note>
* 프로젝트 소유자 또는 팀 관리자만 해당 프로젝트의 가시성 범위를 설정하거나 편집할 수 있습니다.
* 팀 관리자가 팀의 개인정보 보호 설정에서 **Make all future team projects private (public sharing not allowed)** 옵션을 활성화하면, 해당 팀에서는 **Open** 및 **Public** 프로젝트 가시성 범위를 사용할 수 없습니다. 이 경우 팀은 **Team** 및 **Restricted** 범위만 사용할 수 있습니다.
</Note>

<div id="set-visibility-scope-when-you-create-a-new-project">
  #### 새 프로젝트를 생성할 때 가시성 범위 설정하기
</div>

1. SaaS Cloud, Dedicated Cloud, 또는 Self-Managed 인스턴스에서 W&B 조직으로 이동합니다.
2. 왼쪽 사이드바의 **My projects** 섹션에서 **Create a new project** 버튼을 클릭합니다. 또는 팀의 **Projects** 탭으로 이동한 다음, 오른쪽 상단 모서리의 **Create new project** 버튼을 클릭합니다.
3. 상위 팀을 선택하고 프로젝트 이름을 입력한 후, **Project Visibility** 드롭다운에서 원하는 범위를 선택합니다.
   <Frame>
       <img src="/images/hosting/restricted_project_add_new.gif" alt="제한된 프로젝트 생성"  />
   </Frame>

   **Restricted** 가시성 범위를 선택한 경우 다음 단계를 완료합니다. 

4. **Invite team members** 필드에 한 명 이상의 W&B 팀 구성원 이름을 입력합니다. 프로젝트에 협업이 꼭 필요한 구성원만 추가합니다.
   <Frame>
       <img src="/images/hosting/restricted_project_2.png" alt="제한된 프로젝트 구성"  />
   </Frame>

   <Note>
   이 제한된 프로젝트의 구성원은 나중에 해당 프로젝트의 **Users** 탭에서 추가하거나 제거할 수 있습니다.
   </Note>

<div id="edit-visibility-scope-of-an-existing-project">
  #### 기존 프로젝트의 가시성 범위 수정
</div>

1. W&B 프로젝트로 이동합니다.
2. 왼쪽 열에서 **Overview** 탭을 선택합니다.
3. 오른쪽 상단의 **Edit Project Details** 버튼을 클릭합니다.  
4. **Project Visibility** 드롭다운 메뉴에서 원하는 범위를 선택합니다.
   <Frame>
       <img src="/images/hosting/restricted_project_edit.gif" alt="제한된 프로젝트 설정 편집"  />
   </Frame>

   **Restricted** 가시성을 선택했다면 다음 단계를 수행합니다. 

5. 프로젝트의 **Users** 탭으로 이동한 후 **Add user** 버튼을 클릭하여 제한된 프로젝트에 초대할 특정 사용자를 추가합니다.

<Warning>
* 프로젝트의 가시성 범위를 **Team**에서 **Restricted**로 변경하면, 필요한 팀원을 프로젝트에 다시 초대하지 않는 한 팀의 모든 구성원이 해당 프로젝트에 대한 접근 권한을 잃습니다.
* 프로젝트의 가시성 범위를 **Restricted**에서 **Team**으로 변경하면, 팀의 모든 구성원이 해당 프로젝트에 접근할 수 있습니다.
* 제한된 프로젝트의 사용자 목록에서 팀원을 제거하면, 해당 팀원은 그 프로젝트에 대한 접근 권한을 잃습니다.
</Warning>

<div id="other-key-things-to-note-for-restricted-scope">
  ### 제한 범위에 대해 알아둘 기타 핵심 사항
</div>

* 제한된 프로젝트에서 팀 수준 서비스 계정을 사용하려면, 해당 서비스 계정을 해당 프로젝트에 초대하거나 명시적으로 추가해야 합니다. 그렇지 않으면 팀 수준 서비스 계정은 기본적으로 제한된 프로젝트에 액세스할 수 없습니다.
* 제한된 프로젝트에서 실행을 다른 프로젝트로 이동할 수는 없지만, 제한되지 않은 프로젝트에서 제한된 프로젝트로 실행을 이동할 수는 있습니다.
* 팀 개인정보 보호 설정인 **Make all future team projects private (public sharing not allowed)** 와 관계없이, 제한된 프로젝트의 공개 범위를 **Team** 범위로만 전환할 수 있습니다.
* 제한된 프로젝트의 소유자가 더 이상 상위 팀의 구성원이 아닌 경우, 팀 관리자가 프로젝트가 원활히 운영되도록 소유자를 변경해야 합니다.

<div id="project-level-roles">
  ## 프로젝트 수준 역할
</div>

팀에서 범위가 _Team_ 또는 _Restricted_ 로 설정된 프로젝트의 경우, 특정 사용자에게 팀 수준 역할과는 다른 프로젝트 수준 역할을 부여할 수 있습니다. 예를 들어, 사용자가 팀 수준에서 _Member_ 역할인 경우, 해당 팀의 _Team_ 또는 _Restricted_ 범위 프로젝트 내에서는 그 사용자에게 _View-Only_, _Admin_, 또는 사용 가능한 임의의 커스텀 역할을 부여할 수 있습니다.

<Note>
프로젝트 수준 역할은 SaaS Cloud, Dedicated Cloud, Self-Managed 인스턴스에서 프리뷰로 제공됩니다.
</Note>

<div id="assign-project-level-role-to-a-user">
  ### 사용자에게 프로젝트 수준 역할 할당하기
</div>

1. W&B 프로젝트로 이동합니다.
2. 왼쪽 사이드바에서 **Overview** 탭을 선택합니다.
3. 프로젝트에서 **Users** 탭으로 이동합니다.
4. **Project Role** 필드에서 해당 사용자에게 현재 할당된 역할을 클릭합니다. 그러면 사용 가능한 다른 역할이 나열된 드롭다운 메뉴가 열립니다.
5. 드롭다운에서 다른 역할을 선택합니다. 변경 사항은 즉시 저장됩니다.

<Note>
사용자의 프로젝트 수준 역할을 팀 수준 역할과 다르게 변경하면, 프로젝트 수준 역할에 차이가 있음을 나타내기 위해 **\***가 포함됩니다.
</Note>

<div id="other-key-things-to-note-for-project-level-roles">
  ### 프로젝트 수준 역할과 관련해 알아두어야 할 기타 주요 사항
</div>

* 기본적으로, _team_ 또는 _restricted_ 범위의 프로젝트에 속한 모든 사용자의 프로젝트 수준 역할은 해당 팀 수준 역할을 **상속**합니다.
* 팀 수준에서 _View-only_ 역할을 가진 사용자의 프로젝트 수준 역할은 **변경할 수 없습니다**.
* 특정 프로젝트에서 사용자의 프로젝트 수준 역할이 팀 수준 역할과 **동일한 경우**, 나중에 팀 관리자(team admin)가 팀 수준 역할을 변경하면, 해당 프로젝트 역할도 팀 수준 역할을 따르도록 자동으로 변경됩니다.
* 특정 프로젝트에서 사용자의 프로젝트 수준 역할을 팀 수준 역할과 **다르게** 변경한 경우, 이후에 팀 관리자(team admin)가 팀 수준 역할을 변경하더라도, 해당 프로젝트 수준 역할은 그대로 유지됩니다.
* 프로젝트 수준 역할이 팀 수준 역할과 다를 때 해당 사용자를 _restricted_ 프로젝트에서 제거한 후, 나중에 다시 프로젝트에 추가하면, 기본 동작에 따라 팀 수준 역할을 상속하게 됩니다. 필요하다면, 다시 프로젝트 수준 역할을 팀 수준 역할과 다르게 변경해야 합니다.
