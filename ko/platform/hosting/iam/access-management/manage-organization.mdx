title: 조직 관리
---

조직 관리자라면 조직 내에서 [개별 사용자 관리](#add-and-manage-users)와 [팀 관리](#add-and-manage-teams)를 수행할 수 있습니다.

팀 관리자라면 [팀 관리](#add-and-manage-teams)를 수행할 수 있습니다.

<Note>
  다음 워크플로우는 인스턴스 관리자 역할을 가진 사용자에게 적용됩니다. 인스턴스 관리자 권한이 있어야 한다고 생각하신다면 조직의 관리자에게 문의하십시오.
</Note>

조직에서 사용자 관리를 간소화하고 싶다면 [사용자 및 팀 관리 자동화](../automate_iam)를 참고하십시오.

<div id="change-the-name-of-your-organization">
  ## 조직 이름 변경하기
</div>

<Note>
  다음 워크플로우는 W&amp;B Multi-tenant Cloud에만 적용됩니다.
</Note>

1. https://wandb.ai/home으로 이동합니다.
2. 페이지 오른쪽 상단에서 **User menu** 드롭다운을 선택합니다. 드롭다운의 **Account** 섹션에서 **Settings**를 선택합니다.
3. **Settings** 탭에서 **General**을 선택합니다.
4. **Change name** 버튼을 선택합니다.
5. 표시되는 모달 창에서 조직의 새 이름을 입력하고 **Save name** 버튼을 선택합니다.

<div id="add-and-manage-users">
  ## 사용자 추가 및 관리
</div>

관리자인 경우 조직의 대시보드를 사용하여 다음 작업을 수행할 수 있습니다:

* 사용자를 초대하거나 제거합니다.
* 사용자의 조직 역할을 지정하거나 업데이트하고, 사용자 정의 역할을 생성합니다.
* 결제 관리자(billing admin)를 지정합니다.

조직 관리자가 조직에 사용자를 추가할 수 있는 방법은 다음과 같습니다:

1. 초대 기반 멤버 추가
2. SSO를 통한 자동 프로비저닝
3. 도메인 캡처

| Product | Seats | Cost based on |

<div id="invite-a-user">
  ### 사용자 초대
</div>

관리자는 조직 및 조직 내 특정 팀에 사용자를 초대할 수 있습니다.

<Tabs>
  <Tab title="Multi-tenant Cloud">
    1. https://wandb.ai/home으로 이동합니다.
    2. 페이지 오른쪽 상단에서 **User menu** 드롭다운을 선택합니다. 드롭다운의 **Account** 섹션에서 **Users**를 선택합니다.
    3. **Invite new user**를 선택합니다.
    4. 표시되는 모달에서 **Email or username** 필드에 사용자의 이메일 또는 사용자 이름을 입력합니다.
    5. (권장) **Choose teams** 드롭다운 메뉴에서 사용자를 팀에 추가합니다.
    6. **Select role** 드롭다운에서 사용자에게 할당할 역할을 선택합니다. 나중에 사용자의 역할을 변경할 수 있습니다. 가능한 역할에 대한 자세한 내용은 [Assign a role](#assign-or-update-a-team-members-role)에 있는 표를 참조하세요.
    7. **Send invite** 버튼을 선택합니다.

    **Send invite** 버튼을 선택하면 W&amp;B는 서드 파티 이메일 서버를 사용하여 초대 링크를 사용자의 이메일로 전송합니다. 사용자가 초대를 수락하면 조직에 접근할 수 있습니다.
  </Tab>

  <Tab title="Dedicated Cloud or Self-Managed">
    1. `https://<org-name>.io/console/settings/`로 이동합니다. `<org-name>`을 조직 이름으로 바꿉니다.
    2. **Add user** 버튼을 선택합니다.
    3. 표시되는 모달에서 **Email** 필드에 새 사용자의 이메일을 입력합니다.
    4. **Role** 드롭다운에서 사용자에게 할당할 역할을 선택합니다. 나중에 사용자의 역할을 변경할 수 있습니다. 가능한 역할에 대한 자세한 내용은 [Assign a role](#assign-or-update-a-team-members-role)에 있는 표를 참조하세요.
    5. W&amp;B가 서드 파티 이메일 서버를 사용해 초대 링크를 사용자의 이메일로 보내도록 하려면 **Send invite email to user** 체크박스를 선택합니다.
    6. **Add new user** 버튼을 선택합니다.
  </Tab>
</Tabs>

<div id="auto-provision-users">
  ### 사용자 자동 프로비저닝
</div>

일치하는 이메일 도메인을 가진 W&amp;B 사용자는 SSO(Single Sign-On, 단일 로그인)를 구성하고 SSO 공급자가 이를 허용하는 경우 SSO를 통해 W&amp;B Organization에 로그인할 수 있습니다. SSO는 모든 Enterprise 라이선스에서 사용할 수 있습니다.

<Note>
  **인증을 위한 SSO 활성화**

  W&amp;B는 사용자가 SSO(Single Sign-On)를 사용해 인증할 것을 강력히 권장합니다. 조직에 대해 SSO를 활성화하려면 W&amp;B 팀에 문의하십시오.

  Dedicated Cloud 또는 Self-Managed 인스턴스에서 SSO를 설정하는 방법에 대한 자세한 내용은 [SSO with OIDC](/ko/platform/hosting/iam/sso/) 또는 [SSO with LDAP](/ko/platform/hosting/iam/ldap/)을 참조하십시오.
</Note>

W&amp;B는 자동 프로비저닝된 사용자에게 기본값으로 &quot;Member&quot; 역할을 할당합니다. 자동 프로비저닝된 사용자의 역할은 언제든지 변경할 수 있습니다.

SSO를 사용한 자동 프로비저닝은 Dedicated Cloud 인스턴스와 Self-Managed 배포에서 기본적으로 활성화되어 있습니다. 자동 프로비저닝을 비활성화할 수 있습니다. 자동 프로비저닝을 비활성화하면 특정 사용자만 선택적으로 W&amp;B 조직에 추가할 수 있습니다.

다음 탭에서는 배포 유형에 따라 SSO를 끄는 방법을 설명합니다:

<Tabs>
  <Tab title="Multi-tenant Cloud">
    Multi-tenant Cloud에서는 SSO를 사용한 자동 프로비저닝을 구성할 수 없습니다. 도움이 필요하면 W&amp;B 팀에 문의하십시오.
  </Tab>

  <Tab title="Dedicated Cloud or Self-Managed">
    Dedicated Cloud 인스턴스의 경우 SSO를 사용한 자동 프로비저닝을 끄려면 W&amp;B 팀에 문의하십시오.

    Self-Managed 배포의 경우 W&amp;B Console을 사용해 SSO를 사용한 자동 프로비저닝을 끌 수 있습니다:

    1. `https://<org-name>.io/console/settings/`로 이동합니다. `<org-name>`을 조직 이름으로 바꾸십시오.
    2. **Security**를 선택합니다.
    3. **Disable SSO Provisioning**을 선택해 SSO를 사용한 자동 프로비저닝을 끕니다.

    {/* For Self-Managed deployments, you can configure the setting `DISABLE_SSO_PROVISIONING=true` to turn off auto provisioning with SSO. */}
  </Tab>
</Tabs>

<Note>
  SSO를 사용한 자동 프로비저닝을 사용하면 조직 관리자가 개별 사용자 초대를 생성할 필요가 없으므로, 대규모로 조직에 사용자를 추가할 때 유용합니다.
</Note>

<div id="domain-capture">
  ### 도메인 캡처
</div>

도메인 캡처는 직원들이 회사 조직에 소속되도록 도와, 새로운 사용자가 회사 관리 범위 밖에서 애셋을 생성하지 못하도록 합니다.

<Note>
  **도메인은 고유해야 합니다**

  도메인은 고유 식별자입니다. 즉, 다른 조직에서 이미 사용 중인 도메인은 사용할 수 없습니다.
</Note>

<Tabs>
  <Tab title="멀티 테넌트 클라우드">
    도메인 캡처를 사용하면 `@example.com`과 같은 회사 이메일 주소를 가진 사람들을 W&amp;B 멀티 테넌트 클라우드 조직에 자동으로 추가할 수 있습니다. 이를 통해 모든 직원이 올바른 조직에 소속되도록 돕고, 새 사용자가 회사 관리 범위 밖에서 애셋을 생성하지 않도록 보장합니다.

    다음 표는 도메인 캡처 활성화 여부에 따른 신규 및 기존 사용자 동작을 요약합니다:

    | | 도메인 캡처 사용 | 도메인 캡처 미사용 |
    | ----- | ----- | ----- |
    | 신규 사용자 | 검증된 도메인에서 W&amp;B에 가입하는 사용자는 조직의 기본 팀에 구성원으로 자동 추가됩니다. 팀 참가 기능을 활성화한 경우, 가입 시 추가로 참여할 팀을 선택할 수 있습니다. 초대를 통해 다른 조직과 팀에도 계속 참여할 수 있습니다. | 중앙화된 조직이 있다는 사실을 모르는 상태로 사용자가 W&amp;B 계정을 생성할 수 있습니다. |
    | 초대된 사용자 | 초대된 사용자는 초대를 수락하면 자동으로 조직에 참여합니다. 초대된 사용자는 조직의 기본 팀에 자동으로 구성원으로 추가되지는 않습니다. 초대를 통해 다른 조직과 팀에도 계속 참여할 수 있습니다. | 초대된 사용자는 초대를 수락하면 자동으로 조직에 참여합니다. 초대를 통해 다른 조직과 팀에도 계속 참여할 수 있습니다.|
    | 기존 사용자 | 귀사 도메인으로 검증된 이메일 주소를 가진 기존 사용자는 W&amp;B App 내에서 조직의 팀에 참여할 수 있습니다. 기존 사용자가 조직에 참여하기 전에 생성한 모든 데이터는 그대로 유지됩니다. W&amp;B는 기존 사용자의 데이터를 마이그레이션하지 않습니다. | 기존 W&amp;B 사용자는 여러 조직과 팀에 분산되어 있을 수 있습니다.|

    조직에 참여하는 초대되지 않은 신규 사용자를 기본 팀에 자동 할당하려면:

    1. https://wandb.ai/home 으로 이동합니다.
    2. 페이지 오른쪽 상단에서 **User menu** 드롭다운을 선택합니다. 드롭다운에서 **Settings**를 선택합니다.
    3. **Settings** 탭에서 **General**을 선택합니다.
    4. **Domain capture** 내에서 **Claim domain** 버튼을 선택합니다.
    5. **Default team** 드롭다운에서 새 사용자가 자동으로 참여하게 할 팀을 선택합니다. 사용 가능한 팀이 없다면 팀 설정을 업데이트해야 합니다. [Add and manage teams](#add-and-manage-teams)의 안내를 참조하세요.
    6. **Claim email domain** 버튼을 클릭합니다.

    초대되지 않은 신규 사용자를 해당 팀에 자동 할당하려면 먼저 팀 설정에서 도메인 매칭을 활성화해야 합니다.

    1. `https://wandb.ai/<team-name>`의 팀 대시보드로 이동합니다. 여기서 `<team-name>`은 도메인 매칭을 활성화하려는 팀의 이름입니다.
    2. 팀 대시보드 왼쪽의 글로벌 내비게이션에서 **Team settings**를 선택합니다.
    3. **Privacy** 섹션에서 &quot;Recommend new users with matching email domains join this team upon signing up&quot; 옵션을 토글합니다.
  </Tab>

  <Tab title="Dedicated Cloud 또는 Self-Managed">
    Dedicated Cloud 또는 Self-Managed 배포 유형을 사용하는 경우, 도메인 캡처 구성을 위해 W&amp;B Account Team에 문의하세요. 한 번 구성되면, 회사 이메일 주소로 W&amp;B 계정을 생성하는 사용자에게 Dedicated Cloud 또는 Self-Managed 인스턴스에 대한 액세스를 요청하기 위해 관리자에게 연락하라는 메시지를 W&amp;B 멀티 테넌트 클라우드 인스턴스에서 자동으로 표시합니다.

    | | 도메인 캡처 사용 | 도메인 캡처 미사용 |
    | ----- | ----- | -----|
    | 신규 사용자 | 멀티 테넌트 클라우드에서 검증된 도메인으로 W&amp;B에 가입하는 사용자는, 여러분이 지정한 이메일 주소를 통해 관리자에게 연락하라는 메시지를 자동으로 받습니다. 이들은 여전히 제품을 체험하기 위해 멀티 테넌트 클라우드에서 조직을 생성할 수 있습니다. | 사용자는 회사에 중앙에서 관리되는 Dedicated 인스턴스가 있다는 사실을 알지 못한 채 W&amp;B 멀티 테넌트 클라우드 계정을 생성할 수 있습니다. |
    | 기존 사용자 | 기존 W&amp;B 사용자는 여러 조직과 팀에 분산되어 있을 수 있습니다.| 기존 W&amp;B 사용자는 여러 조직과 팀에 분산되어 있을 수 있습니다.|
  </Tab>
</Tabs>

<div id="assign-or-update-a-users-role">
  ### 사용자의 역할 할당 또는 업데이트
</div>

조직 내 모든 구성원은 W&amp;B Models와 Weave 각각에 대해 조직 역할과 시트를 보유합니다. 시트 유형에 따라 과금 상태와 각 제품 라인에서 수행할 수 있는 작업이 결정됩니다.

조직에 사용자를 초대할 때 처음으로 조직 역할을 할당합니다. 이후 언제든지 사용자의 역할을 변경할 수 있습니다.

조직 내 사용자는 다음 역할 중 하나를 가질 수 있습니다:

| Role | Descriptions |
| ----- | ----- |
| admin| 조직에 다른 사용자를 추가하거나 제거하고, 사용자 역할을 변경하고, 사용자 지정 역할을 관리하고, 팀을 추가하는 등의 작업을 수행할 수 있는 인스턴스 관리자입니다. admin이 사용 불가능한 상황에 대비해 admin을 한 명 이상 두는 것을 W&amp;B에서 권장합니다. |
| Member | 인스턴스 admin에 의해 초대된 일반 조직 사용자입니다. 조직 구성원은 다른 사용자를 초대하거나 조직의 기존 사용자를 관리할 수 없습니다. |
| Viewer (Enterprise-only feature) | 인스턴스 admin에 의해 초대된, 조직에 대해 보기 전용 권한만 가진 사용자입니다. Viewer는 자신이 속한 팀과 조직에 대해 읽기 권한만 가집니다. |
|Custom Roles (Enterprise-only feature) | Custom Roles를 사용하면 조직 admin이 보기 전용(View-Only) 또는 Member 역할을 상속한 후 세밀한 액세스 제어를 위해 추가 권한을 부여하여 새로운 역할을 구성할 수 있습니다. 팀 admin은 그런 다음 해당 Custom Roles를 자신의 팀 내 사용자에게 할당할 수 있습니다. 자세한 내용은 [사용자 지정 역할 추가 및 관리](#add-and-manage-custom-roles)를 참조하세요.|

사용자의 역할을 변경하려면:

1. https://wandb.ai/home으로 이동합니다.
2. 페이지 오른쪽 상단에서 **User menu** 드롭다운을 선택합니다. 드롭다운에서 **Users**를 선택합니다.
3. 검색창에 사용자의 이름 또는 이메일을 입력합니다.
4. 사용자 이름 옆의 **TEAM ROLE** 드롭다운에서 역할을 선택합니다.

<div id="assign-or-update-a-users-access">
  ### 사용자의 액세스 권한 할당 또는 업데이트
</div>

조직 내 사용자는 다음과 같은 Model seat 또는 Weave 액세스 유형 중 하나를 가집니다: Full, Viewer, 또는 No access.

| Seat type | Description |
| ----- | ----- |
| Full | 이 역할 유형의 사용자는 Models 또는 Weave에 대해 데이터 쓰기, 읽기, 내보내기 권한을 모두 가집니다. |
| Viewer | 조직의 보기 전용 사용자입니다. Viewer는 조직 및 자신이 속한 하위 팀에 대해 읽기 권한만 가지며, Models 또는 Weave에 대해서도 보기 전용 액세스만 가집니다. |
| No access | 이 역할의 사용자는 Models 또는 Weave 제품에 대한 액세스 권한이 없습니다. |

Model seat 유형과 Weave 액세스 유형은 조직 수준에서 정의되며 팀에 상속됩니다. 사용자의 seat 유형을 변경하려면 조직 설정으로 이동하여 다음 단계를 따르십시오:

1. Multi-tenant Cloud 사용자라면 `https://wandb.ai/account-settings/<organization>/settings`에서 조직 설정으로 이동합니다. 꺾쇠 괄호(`< >`) 안의 값을 조직 이름으로 교체했는지 확인하십시오. Dedicated Cloud 및 Self-Managed 배포의 경우 `https://<your-instance>.wandb.io/org/dashboard`로 이동합니다.
2. **Users** 탭을 선택합니다.
3. **Role** 드롭다운에서 사용자에게 할당할 seat 유형을 선택합니다.

<Note>
  조직 역할과 구독 유형에 따라 조직에서 사용할 수 있는 seat 유형이 결정됩니다.
</Note>

<div id="remove-a-user">
  ### 사용자 제거
</div>

1. https://wandb.ai/home 으로 이동합니다.
2. 페이지 오른쪽 상단에서 **User menu** 드롭다운을 선택합니다. 드롭다운에서 **Users**를 선택합니다.
3. 검색창에 사용자의 이름 또는 이메일을 입력합니다.
4. 표시되면 줄임표 아이콘(점 세 개 아이콘, **...**)을 선택합니다.
5. 드롭다운에서 **Remove member**를 선택합니다.

<div id="assign-the-billing-admin">
  ### 결제 관리자 지정하기
</div>

1. https://wandb.ai/home으로 이동합니다.
2. 페이지 오른쪽 상단에서 **User menu** 드롭다운을 선택합니다. 드롭다운에서 **Users**를 선택합니다.
3. 검색창에 사용자의 이름 또는 이메일을 입력합니다.
4. **Billing admin** 열에서 결제 관리자로 지정할 사용자를 선택합니다.

<div id="add-and-manage-teams">
  ## 팀 추가 및 관리
</div>

조직 대시보드에서 조직 내 팀을 생성하고 관리할 수 있습니다. 조직 관리자 또는 팀 관리자는 다음 작업을 수행할 수 있습니다:

* 팀에 사용자를 초대하거나 팀에서 사용자를 제거합니다.
* 팀 구성원의 역할을 관리합니다.
* 사용자가 조직에 가입할 때 해당 팀에 자동으로 추가되도록 설정합니다.
* `https://wandb.ai/<team-name>`에 있는 팀 대시보드를 사용하여 팀 스토리지를 관리합니다.

<div id="create-a-team">
  ### 팀 생성하기
</div>

조직의 대시보드를 사용해 팀을 생성하려면 다음을 수행합니다.

1. https://wandb.ai/home 으로 이동합니다.
2. 왼쪽 탐색 패널에서 **Teams** 아래에 있는 **Create a team to collaborate**를 선택합니다.

<Frame>
  <img src="/images/hosting/create_new_team.png" alt="새 팀 생성하기" />
</Frame>

3. 표시되는 모달에서 **Team name** 필드에 팀 이름을 입력합니다.
4. 스토리지 유형을 선택합니다.
5. **Create team** 버튼을 선택합니다.

**Create team** 버튼을 선택하면 W&amp;B에서 `https://wandb.ai/<team-name>`의 새 팀 페이지로 리디렉션됩니다. 여기서 `<team-name>`은 팀을 생성할 때 지정한 이름입니다.

팀을 만든 후, 해당 팀에 사용자를 추가할 수 있습니다.

<div id="invite-users-to-a-team">
  ### 팀에 사용자 초대하기
</div>

조직의 팀에 사용자를 초대합니다. 팀의 대시보드에서 사용자의 이메일 주소 또는 이미 W&amp;B 계정이 있는 경우 W&amp;B 사용자 이름을 사용해 사용자를 초대할 수 있습니다.

1. `https://wandb.ai/<team-name>`으로 이동합니다.
2. 대시보드 왼쪽의 글로벌 내비게이션에서 **Team settings**를 선택합니다.

<Frame>
  <img src="/images/hosting/team_settings.png" alt="Team settings" />
</Frame>

3. **Users** 탭을 선택합니다.
4. **Invite a new user**를 선택합니다.
5. 표시되는 모달에서 **Email or username** 필드에 사용자의 이메일을 입력하고, **Select a team** 역할 드롭다운에서 해당 사용자에게 할당할 역할을 선택합니다. 팀에서 사용자가 가질 수 있는 역할에 대한 자세한 내용은 [팀 역할](#assign-or-update-a-team-members-role)을 참조하세요.
6. **Send invite** 버튼을 클릭합니다.

기본적으로 팀 또는 인스턴스 관리자만 팀에 구성원을 초대할 수 있습니다. 이 동작을 변경하려면 [Team settings](/ko/platform/app/settings-page/teams#privacy)를 참조하세요.

이메일 초대를 사용해 사용자를 수동으로 초대하는 것 외에도, 새 사용자의 [이메일이 조직의 도메인과 일치하는 경우](#domain-capture) 새 사용자를 자동으로 팀에 추가할 수 있습니다.

<div id="match-members-to-a-team-organization-during-sign-up">
  ### 가입 시 구성원을 팀 조직에 매칭하기
</div>

조직의 새 사용자가 가입할 때 조직 내 Teams를 발견할 수 있도록 합니다. 새 사용자는 조직의 검증된 이메일 도메인과 일치하는 검증된 이메일 도메인을 사용해야 합니다. 검증된 새 사용자는 W&amp;B 계정에 가입할 때 조직에 속한 검증된 Teams 목록을 볼 수 있습니다.

조직 관리자만 도메인 클레임을 활성화할 수 있습니다. 도메인 캡처를 활성화하려면 [Domain capture](#domain-capture)에 설명된 단계를 따르세요.

<div id="assign-or-update-a-team-members-role">
  ### 팀 구성원의 역할 할당 또는 업데이트
</div>

1. 팀 구성원 이름 옆의 계정 유형 아이콘을 선택합니다.
2. 드롭다운에서 해당 팀 구성원에게 부여할 계정 유형을 선택합니다.

다음 표는 팀 구성원에게 할당할 수 있는 역할을 나열합니다:

| Role   |   Definition   |
|-----------|---------------------------|
| Admin    | 팀에서 다른 사용자를 추가 및 제거하고, 사용자 역할을 변경하며, 팀 설정을 구성할 수 있는 사용자입니다.   |
| Member    | 팀 Admin이 이메일 또는 조직 수준 사용자 이름으로 초대한 일반 사용자입니다. Member 사용자는 다른 사용자를 팀에 초대할 수 없습니다.  |
| View-Only (Enterprise 전용 기능) | 팀 Admin이 이메일 또는 조직 수준 사용자 이름으로 초대한 읽기 전용 사용자입니다. View-Only 사용자는 팀 및 그 콘텐츠에 대해 읽기 권한만 가집니다.  |
| Custom Roles (Enterprise 전용 기능)   | Custom Roles를 사용하면 조직 Admin이 앞서 언급한 View-Only 또는 Member 역할을 상속한 뒤, 세밀한 접근 제어를 위해 추가 권한을 부여하여 새로운 역할을 구성할 수 있습니다. 이후 팀 Admin은 각자의 팀에서 이러한 Custom Roles를 사용자에게 할당할 수 있습니다. 자세한 내용은 [custom roles announcement](https://wandb.ai/wandb_fc/announcements/reports/Introducing-Custom-Roles-for-W-B-Teams--Vmlldzo2MTMxMjQ3)를 참조하세요. |

<Info>
  Service accounts는 사용자가 아니라 자동화를 위해 사용되는 비인간 ID입니다. **Service accounts**는 자동화된 워크플로를 위한 API 키를 제공하며 사용자 라이선스를 소모하지 않습니다. Service accounts 생성 및 관리에 대한 자세한 내용은 [Use service accounts to automate workflows](/ko/platform/hosting/iam/service-accounts/)를 참고하세요.
</Info>

<Note>
  Dedicated Cloud 또는 Self-Managed 배포의 Enterprise 라이선스에서만 팀 구성원에게 Custom Roles를 할당할 수 있습니다.
</Note>

<div id="remove-users-from-a-team">
  ### 팀에서 사용자 제거
</div>

팀 대시보드를 사용하여 팀에서 사용자를 제거할 수 있습니다. 팀에서 더 이상 활동하지 않는 멤버가 생성한 run이라도, 해당 팀에서 생성된 run은 W&amp;B에서 그대로 보존됩니다.

1. `https://wandb.ai/<team-name>`으로 이동합니다.
2. 왼쪽 내비게이션 바에서 **Team settings**를 선택합니다.
3. **Users** 탭을 선택합니다.
4. 삭제하려는 사용자의 이름 옆에 마우스를 올립니다. 나타나는 줄임표 아이콘(**...**)을 선택합니다.
5. 드롭다운 목록에서 **Remove user**를 선택합니다.

<div id="add-and-manage-custom-roles">
  ## 사용자 지정 역할 추가 및 관리
</div>

<Note>
  Dedicated Cloud 또는 Self-Managed 배포에서 사용자 지정 역할을 생성하거나 할당하려면 Enterprise 라이선스가 필요합니다.
</Note>

조직 관리자(Organization admin)는 View-Only 역할 또는 Member 역할을 기반으로 새 역할을 구성하고, 세분화된 액세스 제어를 위해 추가 권한을 부여할 수 있습니다. 팀 관리자(Team admin)는 팀 구성원에게 사용자 지정 역할을 할당할 수 있습니다. 사용자 지정 역할은 조직 수준에서 생성되지만 팀 수준에서 할당됩니다.

사용자 지정 역할을 생성하려면:

<Tabs>
  <Tab title="Multi-tenant Cloud">
    1. https://wandb.ai/home으로 이동합니다.
    2. 페이지 오른쪽 상단에서 **User menu** 드롭다운을 선택합니다. 드롭다운의 **Account** 섹션에서 **Settings**를 선택합니다.
    3. **Roles**를 클릭합니다.
    4. **Custom roles** 섹션에서 **Create a role**을 클릭합니다.
    5. 역할 이름을 입력합니다. 선택적으로 설명을 입력할 수 있습니다.
    6. 사용자 지정 역할의 기준이 될 역할을 **Viewer** 또는 **Member** 중에서 선택합니다.
    7. 권한을 추가하려면 **Search permissions** 필드를 클릭한 다음, 추가할 권한을 하나 이상 선택합니다.
    8. 해당 역할에 부여된 권한이 요약된 **Custom role permissions** 섹션을 검토합니다.
    9. **Create Role**을 클릭합니다.
  </Tab>

  <Tab title="Dedicated Cloud or Self-Managed">
    1. `https://<org-name>.wandb.io/org/settings/`로 이동합니다. `<org-name>`을 조직 이름으로 바꿉니다.
    2. **Custom roles** 섹션에서 **Create a role**을 클릭합니다.
    3. 역할 이름을 입력합니다. 선택적으로 설명을 입력할 수 있습니다.
    4. 사용자 지정 역할의 기준이 될 역할을 **Viewer** 또는 **Member** 중에서 선택합니다.
    5. 권한을 추가하려면 **Search permissions** 필드를 클릭한 다음, 추가할 권한을 하나 이상 선택합니다.
    6. 해당 역할에 부여된 권한이 요약된 **Custom role permissions** 섹션을 검토합니다.
    7. **Create Role**을 클릭합니다.
  </Tab>
</Tabs>

이제 팀 관리자(Team admin)는 [Team settings](#invite-users-to-a-team)에서 팀 구성원에게 사용자 지정 역할을 할당할 수 있습니다.
