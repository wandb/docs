조직 관리자라면 조직 내 [개별 사용자](#add-and-manage-users)와 [팀](#add-and-manage-teams)을 관리할 수 있습니다.

팀 관리자라면 [팀](#add-and-manage-teams)을 관리할 수 있습니다.

<Note>
  다음 워크플로는 인스턴스 관리자 역할을 가진 사용자에게 적용됩니다. 인스턴스 관리자 권한이 필요하다고 생각되면 조직 내 관리자에게 문의하세요.
</Note>

조직에서 사용자 관리를 간소화하고 싶다면 [사용자 및 팀 관리 자동화](../automate_iam)를 참고하세요.

<div id="change-the-name-of-your-organization">
  ## 조직 이름 변경하기
</div>

<Note>
  다음 워크플로우는 W&amp;B 멀티 테넌트 클라우드에만 적용됩니다.
</Note>

1. https://wandb.ai/home으로 이동합니다.
2. 페이지 오른쪽 상단에서 **User menu** 드롭다운을 선택합니다. 드롭다운의 **Account** 섹션에서 **Settings**를 선택합니다.
3. **Settings** 탭에서 **General**을 선택합니다.
4. **Change name** 버튼을 선택합니다.
5. 표시되는 모달 창에서 조직의 새 이름을 입력한 다음 **Save name** 버튼을 선택합니다.

<div id="add-and-manage-users">
  ## 사용자 추가 및 관리
</div>

관리자는 조직의 대시보드를 사용하여 다음 작업을 수행할 수 있습니다:

* 사용자를 초대하거나 제거합니다.
* 사용자의 조직 역할을 할당하거나 업데이트하고, 사용자 지정 역할을 생성합니다.
* 청구 관리자(billing admin)를 지정합니다.

조직 관리자가 조직에 사용자를 추가할 수 있는 방법은 다음과 같습니다:

1. 초대 기반 멤버 추가(Member-by-invite)
2. SSO를 통한 자동 프로비저닝(Auto provisioning with SSO)
3. 도메인 캡처(Domain capture)

| Weave | 무료  | 사용량 기준 |

<div id="invite-a-user">
  ### 사용자 초대
</div>

관리자는 조직 전체와 조직 내 특정 팀에 사용자를 초대할 수 있습니다.

<Tabs>
  <Tab title="Multi-tenant Cloud">
    1. https://wandb.ai/home 으로 이동합니다.
    2. 페이지 오른쪽 상단에서 **User menu** 드롭다운을 선택합니다. 드롭다운의 **Account** 섹션에서 **Users**를 선택합니다.
    3. **Invite new user**를 선택합니다.
    4. 표시되는 모달에서 **Email or username** 필드에 사용자의 이메일 또는 사용자 이름을 입력합니다.
    5. (권장) **Choose teams** 드롭다운 메뉴에서 사용자를 추가할 팀을 선택합니다.
    6. **Select role** 드롭다운에서 사용자에게 할당할 역할을 선택합니다. 사용자의 역할은 나중에 변경할 수 있습니다. 가능한 역할에 대한 자세한 내용은 [역할 할당](#assign-or-update-a-team-members-role)에 나와 있는 표를 참조하세요.
    7. **Send invite** 버튼을 클릭합니다.

    **Send invite** 버튼을 클릭하면, W&amp;B는 서드 파티 이메일 서버를 사용하여 해당 사용자 이메일 주소로 초대 링크를 전송합니다. 사용자가 초대를 수락하면 조직에 접근할 수 있습니다.
  </Tab>

  <Tab title="Dedicated Cloud or Self-Managed">
    1. `https://<org-name>.io/console/settings/` 로 이동합니다. `<org-name>`을 조직 이름으로 바꿉니다.
    2. **Add user** 버튼을 클릭합니다.
    3. 표시되는 모달에서 **Email** 필드에 새 사용자의 이메일을 입력합니다.
    4. **Role** 드롭다운에서 사용자에게 할당할 역할을 선택합니다. 사용자의 역할은 나중에 변경할 수 있습니다. 가능한 역할에 대한 자세한 내용은 [역할 할당](#assign-or-update-a-team-members-role)에 나와 있는 표를 참조하세요.
    5. W&amp;B가 서드 파티 이메일 서버를 사용하여 사용자 이메일 주소로 초대 링크를 보내도록 하려면 **Send invite email to user** 체크박스를 선택합니다.
    6. **Add new user** 버튼을 클릭합니다.
  </Tab>
</Tabs>

<div id="auto-provision-users">
  ### 사용자 자동 프로비저닝
</div>

이메일 도메인이 일치하는 W&amp;B 사용자는 SSO(Single Sign-On)를 구성해 두었고 SSO 공급자가 이를 허용하는 경우, SSO를 통해 W&amp;B 조직에 로그인할 수 있습니다. SSO는 모든 Enterprise 라이선스에서 사용할 수 있습니다.

<Note>
  **인증을 위해 SSO 활성화**

  W&amp;B는 사용자가 인증에 Single Sign-On(SSO)을 사용하도록 강력히 권장합니다. 조직에서 SSO를 활성화하려면 W&amp;B 팀에 문의하십시오.

  Dedicated Cloud 또는 Self-Managed 인스턴스에서 SSO를 설정하는 방법은 [SSO with OIDC](/ko/platform/hosting/iam/sso/) 및 [SSO with LDAP](/ko/platform/hosting/iam/ldap/) 문서를 참고하십시오.
</Note>

W&amp;B는 자동 프로비저닝된 사용자에게 기본적으로 &quot;Member&quot; 역할을 부여합니다. 자동 프로비저닝된 사용자의 역할은 언제든지 변경할 수 있습니다.

SSO를 통한 사용자 자동 프로비저닝은 Dedicated Cloud 인스턴스와 Self-Managed 배포에서 기본적으로 활성화되어 있습니다. 필요하다면 자동 프로비저닝을 비활성화할 수 있습니다. 자동 프로비저닝을 비활성화하면 특정 사용자를 선택적으로 W&amp;B 조직에만 추가할 수 있습니다.

아래 탭에서는 배포 유형에 따라 SSO 자동 프로비저닝을 끄는 방법을 설명합니다:

<Tabs>
  <Tab title="Multi-tenant Cloud">
    Multi-tenant Cloud에서는 SSO를 사용한 자동 프로비저닝을 구성할 수 없습니다. 도움이 필요하면 W&amp;B 팀에 문의하십시오.
  </Tab>

  <Tab title="Dedicated Cloud or Self-Managed">
    Dedicated Cloud 인스턴스의 경우, SSO를 사용한 자동 프로비저닝을 비활성화하려면 W&amp;B 팀에 문의하십시오.

    Self-Managed 배포의 경우, W&amp;B Console을 사용해 SSO를 사용한 자동 프로비저닝을 비활성화할 수 있습니다:

    1. `https://<org-name>.io/console/settings/`로 이동합니다. `<org-name>`을 조직 이름으로 바꾸십시오.
    2. **Security**를 선택합니다.
    3. **Disable SSO Provisioning**을 선택하여 SSO를 사용한 자동 프로비저닝을 비활성화합니다.

    {/* For Self-Managed deployments, you can configure the setting `DISABLE_SSO_PROVISIONING=true` to turn off auto provisioning with SSO. */}
  </Tab>
</Tabs>

<Note>
  SSO를 사용한 자동 프로비저닝을 활용하면 조직 관리자가 개별 사용자 초대를 생성할 필요가 없으므로, 대규모로 사용자를 조직에 추가할 때 유용합니다.
</Note>

<div id="domain-capture">
  ### Domain capture
</div>

Domain capture는 직원들이 회사 조직에 쉽게 가입할 수 있도록 해 주어, 새로운 사용자가 회사의 관리 범위 밖에서 자산을 생성하지 않도록 합니다.

<Note>
  **도메인은 고유해야 합니다**

  도메인은 고유 식별자입니다. 따라서 이미 다른 조직에서 사용 중인 도메인은 사용할 수 없습니다.
</Note>

<Tabs>
  <Tab title="멀티 테넌트 클라우드">
    Domain capture를 사용하면 `@example.com`과 같은 회사 이메일 주소를 가진 사람들을 자동으로 W&amp;B 멀티 테넌트 클라우드 조직에 추가할 수 있습니다. 이를 통해 모든 직원이 올바른 조직에 가입하도록 돕고, 새로운 사용자가 회사 관리 범위 밖에서 자산을 생성하지 않도록 보장합니다.

    다음 표는 domain capture 사용 여부에 따른 신규 및 기존 사용자의 동작을 요약한 것입니다:

    | | Domain capture 사용 | Domain capture 미사용 |
    | ----- | ----- | ----- |
    | 신규 사용자 | 인증된 도메인에서 W&amp;B에 가입하는 사용자는 자동으로 조직의 기본 팀 멤버로 추가됩니다. 팀 가입을 활성화한 경우, 가입 시 추가로 가입할 팀을 선택할 수 있습니다. 초대가 있는 경우 여전히 다른 조직 및 팀에 가입할 수 있습니다. | 사용자는 중앙 집중식 조직이 있다는 사실을 모른 채 W&amp;B 계정을 생성할 수 있습니다. |
    | 초대된 사용자 | 초대된 사용자는 초대를 수락하면 자동으로 조직에 가입합니다. 초대된 사용자는 자동으로 조직의 기본 팀 멤버로 추가되지는 않습니다. 초대가 있는 경우 여전히 다른 조직 및 팀에 가입할 수 있습니다. | 초대된 사용자는 초대를 수락하면 자동으로 조직에 가입합니다. 초대가 있는 경우 여전히 다른 조직 및 팀에 가입할 수 있습니다.|
    | 기존 사용자 | 도메인에서 인증된 이메일 주소를 가진 기존 사용자는 W&amp;B 앱 내에서 조직의 팀에 가입할 수 있습니다. 기존 사용자가 조직에 가입하기 전에 생성한 모든 데이터는 그대로 유지됩니다. W&amp;B는 기존 사용자의 데이터를 마이그레이션하지 않습니다. | 기존 W&amp;B 사용자는 여러 조직과 팀에 분산되어 있을 수 있습니다.|

    조직에 새로 가입하는, 초대받지 않은 신규 사용자를 기본 팀에 자동으로 할당하려면:

    1. https://wandb.ai/home 으로 이동합니다.
    2. 페이지 오른쪽 상단에서 **User menu** 드롭다운을 선택합니다. 드롭다운에서 **Settings**를 선택합니다.
    3. **Settings** 탭에서 **General**을 선택합니다.
    4. **Domain capture** 내에서 **Claim domain** 버튼을 선택합니다.
    5. **Default team** 드롭다운에서 신규 사용자가 자동으로 가입하도록 할 팀을 선택합니다. 사용 가능한 팀이 없다면 팀 설정을 업데이트해야 합니다. 자세한 내용은 [팀 추가 및 관리](#add-and-manage-teams)를 참조하세요.
    6. **Claim email domain** 버튼을 클릭합니다.

    초대받지 않은 신규 사용자를 해당 팀에 자동으로 할당하려면, 먼저 팀 설정에서 도메인 매칭을 활성화해야 합니다.

    1. `https://wandb.ai/<team-name>`의 팀 대시보드로 이동합니다. 여기서 `<team-name>`은 도메인 매칭을 활성화하려는 팀의 이름입니다.
    2. 팀 대시보드 왼쪽의 전역 내비게이션에서 **Team settings**를 선택합니다.
    3. **Privacy** 섹션에서 &quot;가입 시 일치하는 이메일 도메인을 가진 신규 사용자가 이 팀에 가입하도록 추천&quot; 옵션을 토글합니다.
  </Tab>

  <Tab title="전용 클라우드 또는 셀프 매니지드">
    Dedicated Cloud 또는 Self-Managed 배포 유형을 사용하는 경우, domain capture를 구성하려면 W&amp;B Account Team에 문의하세요. 구성 후에는 회사 이메일 주소로 W&amp;B 계정을 생성하는 사용자에게 Dedicated Cloud 또는 Self-Managed 인스턴스에 대한 액세스를 요청하기 위해 관리자에게 연락하라는 메시지를 W&amp;B 멀티 테넌트 클라우드 인스턴스가 자동으로 표시합니다.

    | | Domain capture 사용 | Domain capture 미사용 |
    | ----- | ----- | -----|
    | 신규 사용자 | 멀티 테넌트 클라우드에서 인증된 도메인으로 W&amp;B에 가입하는 사용자는, 사용자가 설정한 이메일 주소를 가진 관리자에게 연락하라는 프롬프트를 자동으로 받습니다. 여전히 제품을 체험하기 위해 멀티 테넌트 클라우드에서 조직을 생성할 수 있습니다. | 사용자는 회사에 중앙 집중식 전용 인스턴스가 있다는 사실을 모른 채 W&amp;B 멀티 테넌트 클라우드 계정을 생성할 수 있습니다. |
    | 기존 사용자 | 기존 W&amp;B 사용자는 여러 조직과 팀에 분산되어 있을 수 있습니다.| 기존 W&amp;B 사용자는 여러 조직과 팀에 분산되어 있을 수 있습니다.|
  </Tab>
</Tabs>

<div id="assign-or-update-a-users-role">
  ### 사용자의 역할 할당 또는 업데이트
</div>

조직의 모든 멤버는 W&amp;B Models와 Weave 각각에 대해 조직 역할과 seat를 가집니다. 사용자가 가진 seat 유형에 따라 과금 상태와 각 제품 라인에서 수행할 수 있는 작업이 결정됩니다.

사용자를 조직에 초대할 때 처음으로 해당 사용자에게 조직 역할을 할당합니다. 이후 언제든지 사용자의 역할을 변경할 수 있습니다.

조직 내 사용자는 다음 역할 중 하나를 가질 수 있습니다:

| Role | Descriptions |
| ----- | ----- |
| admin| 다른 사용자를 조직에 추가하거나 제거하고, 사용자 역할을 변경하고, 커스텀 역할을 관리하고, 팀을 추가하는 등의 작업을 수행할 수 있는 인스턴스 관리자입니다. admin이 부재인 상황에 대비하여 두 명 이상의 admin을 두는 것을 W&amp;B에서 권장합니다. |
| Member | 인스턴스 admin이 초대한 조직의 일반 사용자입니다. 조직의 member는 다른 사용자를 초대하거나 기존 사용자를 관리할 수 없습니다. |
| Viewer (Enterprise-only feature) | 인스턴스 admin이 초대한, 조직의 보기 전용 사용자입니다. viewer는 조직 및 자신이 속한 하위 팀에 대해 읽기 권한만 가집니다. |
|Custom Roles (Enterprise-only feature) | 커스텀 역할을 사용하면 조직 admin이 기존의 보기 전용(View-Only) 또는 Member 역할을 상속하고 추가 권한을 부여하여 세분화된 접근 제어를 구현하는 새로운 역할을 구성할 수 있습니다. 이후 팀 admin은 해당 커스텀 역할들을 자신이 관리하는 팀의 사용자들에게 할당할 수 있습니다. 자세한 내용은 [커스텀 역할 추가 및 관리](#add-and-manage-custom-roles)를 참조하세요.|

사용자의 역할을 변경하려면:

1. https://wandb.ai/home 으로 이동합니다.
2. 페이지 오른쪽 상단에서 **User menu** 드롭다운을 선택합니다. 드롭다운에서 **Users**를 선택합니다.
3. 검색창에 사용자의 이름이나 이메일을 입력합니다.
4. 사용자 이름 옆의 **TEAM ROLE** 드롭다운에서 역할을 선택합니다.

<div id="assign-or-update-a-users-access">
  ### 사용자의 액세스 할당 또는 업데이트
</div>

조직 내 사용자는 다음과 같은 Model 시트 또는 Weave 액세스 유형 중 하나를 가질 수 있습니다: 전체, 뷰어, 또는 액세스 없음.

| 시트 유형 | 설명 |
| ----- | ----- |
| 전체 | 이 역할 유형을 가진 사용자는 Models 또는 Weave에 대해 데이터 쓰기, 읽기, 내보내기의 모든 권한을 가집니다. |
| 뷰어 | 조직의 보기 전용 사용자입니다. 뷰어는 자신이 속한 조직과 해당 조직의 팀에 대해 읽기 권한만 가지며, Models 또는 Weave에 대해서도 보기 전용 액세스만 가집니다. |
| 액세스 없음 | 이 역할을 가진 사용자는 Models 또는 Weave 제품에 대한 액세스 권한이 없습니다. |

Model 시트 유형과 Weave 액세스 유형은 조직 수준에서 정의되며, 팀에서도 그대로 적용됩니다. 사용자의 시트 유형을 변경하려면 조직 설정으로 이동한 다음, 다음 단계를 따르세요:

1. Multi-tenant Cloud 사용자는 `https://wandb.ai/account-settings/<organization>/settings`에서 조직 설정으로 이동합니다. 꺾쇠 괄호 (`<>`)로 둘러싸인 값을 조직 이름으로 바꾸세요. Dedicated Cloud 및 Self-Managed 배포의 경우 `https://<your-instance>.wandb.io/org/dashboard`로 이동합니다.
2. **Users** 탭을 선택합니다.
3. **Role** 드롭다운에서 사용자에게 할당하려는 시트 유형을 선택합니다.

<Note>
  조직 역할과 구독 유형에 따라 조직 내에서 사용 가능한 시트 유형이 결정됩니다.
</Note>

<div id="remove-a-user">
  ### 사용자 제거
</div>

1. https://wandb.ai/home으로 이동합니다.
2. 페이지 오른쪽 상단에서 **User menu** 드롭다운을 선택합니다. 드롭다운에서 **Users**를 선택합니다.
3. 검색창에 사용자의 이름 또는 이메일을 입력합니다.
4. 표시되는 점 세 개 아이콘(**...**)을 선택합니다.
5. 드롭다운에서 **Remove member**를 선택합니다.

<div id="assign-the-billing-admin">
  ### 결제 관리자 지정
</div>

1. https://wandb.ai/home으로 이동합니다.
2. 페이지 오른쪽 상단에서 **User menu** 드롭다운을 클릭한 다음, **Users**를 선택합니다.
3. 검색창에 사용자의 이름 또는 이메일을 입력합니다.
4. **Billing admin** 열에서 결제 관리자로 지정할 사용자를 선택합니다.

<div id="add-and-manage-teams">
  ## 팀 추가 및 관리
</div>

조직 대시보드를 사용해 조직 내 팀을 생성하고 관리할 수 있습니다. 조직 관리자 또는 팀 관리자는 다음 작업을 수행할 수 있습니다:

* 팀에 사용자를 초대하거나 팀에서 사용자를 제거합니다.
* 팀 구성원의 역할을 관리합니다.
* 사용자가 조직에 가입할 때 자동으로 팀에 추가되도록 설정합니다.
* 팀 대시보드(`https://wandb.ai/<team-name>`)에서 팀 스토리지를 관리합니다.

<div id="create-a-team">
  ### 팀 생성하기
</div>

조직의 대시보드를 사용하여 팀을 생성하려면 다음을 수행합니다:

1. https://wandb.ai/home으로 이동합니다.
2. 왼쪽 탐색 패널에서 **Teams** 아래에 있는 **Create a team to collaborate**를 선택합니다.

<Frame>
  <img src="/images/hosting/create_new_team.png" alt="새 팀 생성" />
</Frame>

3. 나타나는 모달의 **Team name** 필드에 팀 이름을 입력합니다.
4. 스토리지 유형을 선택합니다.
5. **Create team** 버튼을 선택합니다.

**Create team** 버튼을 선택하면 W&amp;B에서 `https://wandb.ai/<team-name>`의 새 팀 페이지로 리디렉션합니다. 여기서 `<team-name>`은 팀을 생성할 때 입력한 이름을 사용합니다.

팀을 생성한 후에는 해당 팀에 사용자를 초대할 수 있습니다.

<div id="invite-users-to-a-team">
  ### 팀에 사용자 초대하기
</div>

조직의 팀에 사용자를 초대합니다. 팀 대시보드에서 사용자의 이메일 주소나, 이미 W&amp;B 계정이 있는 경우 W&amp;B 사용자 이름을 사용해 사용자를 초대할 수 있습니다.

1. `https://wandb.ai/<team-name>`으로 이동합니다.
2. 대시보드 왼쪽의 전역 내비게이션에서 **Team settings**를 선택합니다.

<Frame>
  <img src="/images/hosting/team_settings.png" alt="팀 설정" />
</Frame>

3. **Users** 탭을 선택합니다.
4. **Invite a new user**를 선택합니다.
5. 표시되는 모달에서 **Email or username** 필드에 사용자의 이메일을 입력하고, **Select a team** role 드롭다운 메뉴에서 해당 사용자에게 할당할 역할을 선택합니다. 팀에서 사용자가 가질 수 있는 역할에 대한 자세한 내용은 [Team roles](#assign-or-update-a-team-members-role)를 참조하세요.
6. **Send invite** 버튼을 선택합니다.

기본적으로 팀 또는 인스턴스 관리자만 팀에 멤버를 초대할 수 있습니다. 이 동작을 변경하려면 [Team settings](/ko/platform/app/settings-page/teams#privacy)를 참조하세요.

이메일 초대를 통해 사용자를 수동으로 초대하는 것 외에도, 새 사용자의 [이메일이 조직의 도메인과 일치하는 경우](#domain-capture) 새 사용자를 팀에 자동으로 추가할 수 있습니다.

<div id="match-members-to-a-team-organization-during-sign-up">
  ### 가입 시 팀 조직에 멤버 매칭하기
</div>

조직 내 새 사용자가 가입할 때 조직 내 팀을 찾아보고 선택할 수 있도록 설정할 수 있습니다. 새 사용자는 조직의 검증된 이메일 도메인과 일치하는 검증된 이메일 도메인을 사용해야 합니다. 이렇게 검증된 새 사용자는 W&amp;B 계정을 생성할 때, 해당 조직에 속한 검증된 팀 목록을 볼 수 있습니다.

도메인 클레임은 조직 관리자만 활성화할 수 있습니다. 도메인 캡처를 활성화하려면 [Domain capture](#domain-capture)에 설명된 단계를 참고하세요.

<div id="assign-or-update-a-team-members-role">
  ### 팀 구성원의 역할 지정 또는 업데이트
</div>

1. 팀 구성원 이름 옆의 계정 유형 아이콘을 선택합니다.
2. 드롭다운 메뉴에서 해당 팀 구성원에게 부여할 계정 유형을 선택합니다.

다음 표는 팀 구성원에게 지정할 수 있는 역할을 보여줍니다:

| Role   |   Definition   |
|-----------|---------------------------|
| Admin    | 팀 내 다른 사용자를 추가·삭제하고, 사용자 역할을 변경하며, 팀 설정을 구성할 수 있는 사용자입니다.   |
| Member    | 팀 Admin이 이메일 또는 조직 수준 사용자 이름을 통해 초대한 일반 팀 사용자입니다. Member 사용자는 다른 사용자를 팀에 초대할 수 없습니다.  |
| View-Only (Enterprise 전용 기능) | 팀 Admin이 이메일 또는 조직 수준 사용자 이름을 통해 초대한 팀의 조회 전용 사용자입니다. 조회 전용 사용자는 팀과 그 콘텐츠에 대해 읽기 전용 권한만 가집니다.  |
| Custom Roles (Enterprise 전용 기능)   | Custom Roles를 사용하면 조직 Admin이 앞서 언급한 View-Only 또는 Member 역할을 상속받아 새 역할을 구성하고, 세밀한 액세스 제어를 위해 추가 권한을 부여할 수 있습니다. 이후 팀 Admin은 각자의 팀에서 이러한 Custom Roles를 사용자에게 할당할 수 있습니다. 자세한 내용은 [custom roles 발표](https://wandb.ai/wandb_fc/announcements/reports/Introducing-Custom-Roles-for-W-B-Teams--Vmlldzo2MTMxMjQ3)를 참조하세요. |

<Info>
  Service account는 사용자가 아니라, 자동화를 위해 사용되는 비인간 ID입니다. **Service accounts**는 자동화된 워크플로를 위한 API 키를 제공하며, 사용자 라이선스를 소모하지 않습니다. Service account 생성 및 관리에 대한 자세한 내용은 [Service account를 사용하여 워크플로 자동화하기](/ko/platform/hosting/iam/service-accounts/)를 참조하세요.
</Info>

<Note>
  Dedicated Cloud 또는 Self-Managed 배포에서 Enterprise 라이선스를 사용하는 경우에만 팀 구성원에게 Custom Roles를 할당할 수 있습니다.
</Note>

<div id="remove-users-from-a-team">
  ### 팀에서 사용자 제거
</div>

팀 대시보드를 사용하여 팀에서 사용자를 제거할 수 있습니다. 특정 멤버가 더 이상 팀에 속해 있지 않더라도, 해당 팀에서 생성된 실행은 W&amp;B에 계속 보존됩니다.

1. `https://wandb.ai/<team-name>`으로 이동합니다.
2. 왼쪽 탐색 표시줄에서 **Team settings**를 선택합니다.
3. **Users** 탭을 선택합니다.
4. 삭제하려는 사용자의 이름 옆에 마우스를 올려둡니다. 아이콘이 표시되면 줄임표 또는 점 세 개 아이콘(**...**)을 선택합니다.
5. 드롭다운 메뉴에서 **Remove user**를 선택합니다.

<div id="add-and-manage-custom-roles">
  ## 사용자 지정 역할 추가 및 관리
</div>

<Note>
  Dedicated Cloud 또는 Self-Managed 배포에서 사용자 지정 역할을 생성하거나 할당하려면 Enterprise 라이선스가 필요합니다.
</Note>

Organization 관리자(org admin)는 View-Only 또는 Member 역할을 기반으로 새 역할을 구성한 후, 추가 권한을 부여해 세분화된 액세스 제어를 구현할 수 있습니다. Team 관리자(team admin)는 팀 구성원에게 사용자 지정 역할을 할당할 수 있습니다. 사용자 지정 역할은 Organization 수준에서 생성되지만 Team 수준에서 할당됩니다.

사용자 지정 역할을 생성하려면:

<Tabs>
  <Tab title="Multi-tenant Cloud">
    1. https://wandb.ai/home으로 이동합니다.
    2. 페이지 오른쪽 상단에서 **User menu** 드롭다운을 선택합니다. 드롭다운의 **Account** 섹션에서 **Settings**를 선택합니다.
    3. **Roles**를 클릭합니다.
    4. **Custom roles** 섹션에서 **Create a role**을 클릭합니다.
    5. 역할 이름을 입력합니다. 필요하다면 설명을 추가로 입력합니다.
    6. 사용자 지정 역할의 기준이 될 역할을 **Viewer** 또는 **Member** 중에서 선택합니다.
    7. 권한을 추가하려면 **Search permissions** 필드를 클릭한 다음, 추가할 권한을 하나 이상 선택합니다.
    8. 역할이 갖는 권한을 요약해서 보여 주는 **Custom role permissions** 섹션을 검토합니다.
    9. **Create Role**을 클릭합니다.
  </Tab>

  <Tab title="Dedicated Cloud 또는 Self-Managed">
    1. `https://<org-name>.wandb.io/org/settings/`로 이동합니다. `<org-name>`을 조직 이름으로 바꿉니다.
    2. **Custom roles** 섹션에서 **Create a role**을 클릭합니다.
    3. 역할 이름을 입력합니다. 필요하다면 설명을 추가로 입력합니다.
    4. 사용자 지정 역할의 기준이 될 역할을 **Viewer** 또는 **Member** 중에서 선택합니다.
    5. 권한을 추가하려면 **Search permissions** 필드를 클릭한 다음, 추가할 권한을 하나 이상 선택합니다.
    6. 역할이 갖는 권한을 요약해서 보여 주는 **Custom role permissions** 섹션을 검토합니다.
    7. **Create Role**을 클릭합니다.
  </Tab>
</Tabs>

이제 Team 관리자는 [Team 설정](#invite-users-to-a-team)에서 팀 구성원에게 사용자 지정 역할을 할당할 수 있습니다.
