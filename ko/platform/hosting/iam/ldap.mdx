이 가이드는 W&amp;B Server에서 LDAP 기반 single sign-on(SSO)을 사용 설정하는 W&amp;B 관리자용입니다. 이를 통해 사용자는 별도의 W&amp;B 자격 증명을 관리하지 않고 기존 LDAP 디렉터리를 통해 인증할 수 있습니다. 이 문서에서는 W&amp;B App 시스템 설정 UI 또는 환경 변수를 사용하여 LDAP 연결을 설정하는 방법을 설명합니다. 또한 주소, 기본 DN, 속성을 포함한 필수 및 선택 설정 매개변수도 설명합니다. 익명 바인드를 설정하거나 관리자 DN 및 비밀번호로 바인드할 수 있습니다.

{/* <Note>
    W&B Team Admin은 익명 바인드를 설정하거나 관리자 DN 및 비밀번호를 사용해 바인드를 설정할 수 있습니다.
    </Note> */}

<Note>
  LDAP 인증은 W&amp;B Admin 역할만 활성화하고 설정할 수 있습니다.
</Note>

<div id="configure-ldap-connection">
  ## LDAP 연결 구성
</div>

다음 방법 중 하나를 선택하여 LDAP 연결을 구성하세요. W&amp;B App 탭을 사용하면 시스템 설정 UI를 통해 LDAP를 구성할 수 있고, 환경 변수 탭을 사용하면 배포 시 LDAP를 구성할 수 있습니다.

<Tabs>
  <Tab title="W&B App">
    W&amp;B App을 통해 LDAP를 구성하려면 다음을 수행하세요:

    1. W&amp;B App으로 이동합니다.
    2. 프로필 아이콘을 선택한 다음 **System Settings**를 선택합니다.
    3. **Configure LDAP Client**를 켭니다.
    4. 양식에 세부 정보를 입력합니다. 각 입력 항목에 대한 자세한 내용은 [설정 매개변수](#configuration-parameters)를 참조하세요.
    5. 설정을 테스트하려면 **Update Settings**를 클릭합니다. 이 단계에서는 W&amp;B Server와의 테스트 클라이언트 또는 연결을 설정합니다.
    6. 연결이 확인되면 **Enable LDAP Authentication**를 켜고 **Update Settings**를 선택합니다.

    이 단계를 완료하면 W&amp;B Server가 LDAP 디렉터리를 기준으로 사용자를 인증합니다.
  </Tab>

  <Tab title="환경 변수">
    다음 환경 변수로 LDAP 연결을 설정합니다:

    | 환경 변수                         | 필수  | 예시                              |
    | ----------------------------- | --- | ------------------------------- |
    | `LOCAL_LDAP_ADDRESS`          | 예   | `ldaps://ldap.example.com:636`  |
    | `LOCAL_LDAP_BASE_DN`          | 예   | `email=mail,group=gidNumber`    |
    | `LOCAL_LDAP_BIND_DN`          | 아니요 | `cn=admin`, `dc=example,dc=org` |
    | `LOCAL_LDAP_BIND_PW`          | 아니요 |                                 |
    | `LOCAL_LDAP_ATTRIBUTES`       | 예   | `email=mail`, `group=gidNumber` |
    | `LOCAL_LDAP_TLS_ENABLE`       | 아니요 |                                 |
    | `LOCAL_LDAP_GROUP_ALLOW_LIST` | 아니요 |                                 |
    | `LOCAL_LDAP_LOGIN`            | 아니요 |                                 |

    다음 [설정 매개변수](#configuration-parameters) 섹션에서 각 환경 변수를 정의합니다. 명확성을 위해 정의 이름에서는 환경 변수 접두사 `LOCAL_LDAP`를 생략합니다.
  </Tab>
</Tabs>

<div id="configuration-parameters">
  ## 설정 매개변수
</div>

다음 표에는 필수 및 선택 LDAP 설정 항목과 각 항목의 설명이 나와 있습니다.

| 환경 변수              | 정의                                                                                                                                                     | 필수  |
| ------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------ | --- |
| `ADDRESS`          | W&amp;B Server를 호스팅하는 VPC 내 LDAP 서버의 주소입니다.                                                                                                            | Yes |
| `BASE_DN`          | 검색이 시작되는 루트 경로로, 이 디렉터리에 대한 모든 쿼리에 필요합니다.                                                                                                              | Yes |
| `BIND_DN`          | LDAP 서버에 등록된 관리 사용자 경로입니다. LDAP 서버가 인증되지 않은 바인딩을 지원하지 않는 경우 필요합니다. 지정하면 W&amp;B Server는 이 사용자로 LDAP 서버에 연결합니다. 그렇지 않으면 W&amp;B Server는 익명 바인딩으로 연결합니다. | No  |
| `BIND_PW`          | 관리 사용자의 비밀번호이며, 바인딩 인증에 사용됩니다. 비워 두면 W&amp;B Server는 익명 바인딩으로 연결합니다.                                                                                   | No  |
| `ATTRIBUTES`       | 이메일 및 그룹 ID 속성 이름을 쉼표로 구분된 string 값으로 제공하세요.                                                                                                           | Yes |
| `TLS_ENABLE`       | TLS를 활성화합니다.                                                                                                                                           | No  |
| `GROUP_ALLOW_LIST` | 그룹 허용 목록입니다.                                                                                                                                           | No  |
| `LOGIN`            | W&amp;B Server가 LDAP를 사용해 인증하도록 설정합니다. `True` 또는 `False`로 설정하세요. 필요에 따라 LDAP 설정을 테스트하려면 `False`로 설정할 수 있습니다. LDAP 인증을 시작하려면 `True`로 설정하세요.             | No  |
