---
title: 고급 IAM 구성
---

기본 [환경 변수](../env-vars) 외에도, [Dedicated Cloud](/ko/platform/hosting/hosting-options/dedicated_cloud) 또는 [Self-Managed](/ko/platform/hosting/hosting-options/self-managed) 인스턴스의 IAM 옵션을 구성하기 위해 환경 변수를 사용할 수 있습니다.

IAM 요구 사항에 따라 인스턴스에 필요한 다음 환경 변수들을 선택해 사용하십시오.

| Environment variable | Description |
|----------------------|-------------|
| `DISABLE_SSO_PROVISIONING` | W&amp;B 인스턴스에서 사용자 자동 프로비저닝을 비활성화하려면 이 값을 `true`로 설정하십시오. |
| `SESSION_LENGTH` | 기본 사용자 세션 만료 시간을 변경하려면 이 변수를 원하는 시간(시간 단위)으로 설정하십시오. 예를 들어, 세션 만료 시간을 24시간으로 구성하려면 SESSION&#95;LENGTH를 `24`로 설정하십시오. 기본값은 720시간입니다. |
| `GORILLA_ENABLE_SSO_GROUPS_CLAIMS` | OIDC 기반 SSO를 사용하는 경우, 이 변수를 `true`로 설정하면 OIDC 그룹을 기반으로 인스턴스 내 W&amp;B 팀 멤버십을 자동화할 수 있습니다. 또한 사용자 OIDC 토큰에 `groups` 클레임을 추가해야 하며, 사용자가 속한 모든 팀 이름을 문자열 배열로 포맷해야 합니다. |
| `GORILLA_LDAP_GROUP_SYNC` | LDAP 기반 SSO를 사용하는 경우, 이 값을 `true`로 설정하면 LDAP 그룹을 기반으로 인스턴스 내 W&amp;B 팀 멤버십을 자동화할 수 있습니다. |
| `GORILLA_OIDC_CUSTOM_SCOPES` | OIDC 기반 SSO를 사용하는 경우, W&amp;B 인스턴스가 IdP(Identity Provider)에 요청해야 하는 추가 [scope](https://auth0.com/docs/get-started/apis/scopes/openid-connect-scopes)를 지정할 수 있습니다. 이러한 맞춤 scope로 인해 W&amp;B가 SSO 동작을 어떤 방식으로든 변경하지는 않습니다. |
| `GORILLA_OIDC_SECRET` | OIDC 기반 SSO를 사용하고 IdP에서 OIDC Client Secret을 요구하는 경우, 이 변수를 해당 secret 값으로 설정하십시오. |
| `GORILLA_USE_IDENTIFIER_CLAIMS` | OIDC 기반 SSO를 사용하는 경우, 이 변수를 `true`로 설정하면 IdP에서 제공하는 특정 OIDC 클레임을 사용해 사용자의 사용자 이름과 전체 이름을 강제할 수 있습니다. 설정하는 경우, 강제할 사용자 이름과 전체 이름을 각각 `preferred_username` 및 `name` OIDC 클레임에 구성해야 합니다. 사용자 이름에는 영숫자 문자와 밑줄, 하이픈만 사용할 수 있습니다. |
| `GORILLA_DISABLE_PERSONAL_ENTITY` | true로 설정하면 [개인 엔터티](/ko/models/support/difference_team_entity_user_entity_mean_me/)를 비활성화합니다. 개인 엔터티 내에서 새로운 개인 프로젝트 생성이 차단되며, 기존 개인 프로젝트에 대한 쓰기도 차단됩니다. |
| `GORILLA_DISABLE_ADMIN_TEAM_ACCESS` | Organization 또는 Instance Admin이 스스로 W&amp;B 팀에 참여하거나 자신을 팀에 추가하지 못하도록 제한하려면 이 값을 `true`로 설정하십시오. 이를 통해 팀 내 프로젝트에 Data &amp; AI 담당자만 액세스할 수 있도록 보장합니다. |
| `WANDB_IDENTITY_TOKEN_FILE`        | [Identity Federation](/ko/platform/hosting/iam/identity_federation/)의 경우, Java Web Token(JWT)이 저장되는 로컬 디렉터리의 절대 경로입니다. |

<Warning>
  W&amp;B는 `GORILLA_DISABLE_ADMIN_TEAM_ACCESS`와 같은 일부 설정을 활성화하기 전에 모든 영향을 충분히 이해하고 신중하게 적용할 것을 권장합니다. 질문이 있는 경우 W&amp;B 팀에 문의하십시오.
</Warning>
