---
title: 고급 IAM 설정
---

기본적인 [environment variables](../env-vars) 외에도, [전용 클라우드](/platform/hosting/hosting-options/dedicated_cloud) 또는 [Self-Managed](/platform/hosting/hosting-options/self-managed) 인스턴스의 IAM 옵션을 구성하기 위해 환경 변수를 사용할 수 있습니다.

사용자의 IAM 요구 사항에 따라 인스턴스에 다음 환경 변수 중 하나를 선택하여 사용하세요.

| 환경 변수 | 설명 |
|----------------------|-------------|
| `DISABLE_SSO_PROVISIONING` | W&B 인스턴스에서 사용자 자동 프로비저닝을 끄려면 이 값을 `true`로 설정하세요. |
| `SESSION_LENGTH` | 기본 사용자 세션 만료 시간을 변경하려면 이 변수를 원하는 시간(hour)으로 설정하세요. 예를 들어, 세션 만료 시간을 24시간으로 구성하려면 SESSION_LENGTH를 `24`로 설정합니다. 기본값은 720시간입니다. |
| `GORILLA_ENABLE_SSO_GROUPS_CLAIMS` | OIDC 기반 SSO를 사용하는 경우, 이 변수를 `true`로 설정하여 OIDC 그룹을 기반으로 인스턴스의 W&B 팀 멤버십을 자동화할 수 있습니다. 또한 사용자의 OIDC 토큰에 `groups` 클레임을 추가해야 하며, 이는 사용자가 속한 모든 팀 이름의 문자열 배열로 형식화되어야 합니다. |
| `GORILLA_LDAP_GROUP_SYNC` | LDAP 기반 SSO를 사용하는 경우, 이를 `true`로 설정하여 LDAP 그룹을 기반으로 인스턴스의 W&B 팀 멤버십을 자동화할 수 있습니다. |
| `GORILLA_OIDC_CUSTOM_SCOPES` | OIDC 기반 SSO를 사용하는 경우, W&B 인스턴스가 ID 제공업체(IdP)에 요청해야 하는 추가 [scopes](https://auth0.com/docs/get-started/apis/scopes/openid-connect-scopes)를 지정할 수 있습니다. W&B는 이러한 커스텀 스코프로 인해 SSO 기능을 변경하지 않습니다. |
| `GORILLA_OIDC_SECRET` | OIDC 기반 SSO를 사용하고 IdP에서 OIDC Client Secret이 필요한 경우, 이 변수를 해당 비밀 값으로 설정하세요. |
| `GORILLA_USE_IDENTIFIER_CLAIMS` | OIDC 기반 SSO를 사용하는 경우, 이 변수를 `true`로 설정하여 ID 제공업체의 특정 OIDC 클레임을 사용해 사용자의 사용자 이름과 전체 이름을 강제할 수 있습니다. 설정하는 경우, 강제할 사용자 이름과 전체 이름이 각각 `preferred_username` 및 `name` OIDC 클레임에 구성되어 있는지 확인하세요. 사용자 이름은 영숫자와 특수 문자인 밑줄(_), 하이픈(-)만 포함할 수 있습니다. |
| `GORILLA_DISABLE_PERSONAL_ENTITY` | true로 설정하면 [personal entities](/models/support/difference_team_entity_user_entity_mean_me/)를 비활성화합니다. 개인 Entities에서 새로운 개인 Projects 생성을 방지하고 기존 개인 Projects에 쓰는 것을 방지합니다. |
| `GORILLA_DISABLE_ADMIN_TEAM_ACCESS` | 조직 또는 인스턴스 관리자가 W&B 팀에 스스로 가입하거나 자신을 추가하는 것을 제한하려면 이 값을 `true`로 설정하세요. 이를 통해 데이터 및 AI 담당자만 팀 내 Projects에 엑세스할 수 있도록 보장합니다. |
| `WANDB_IDENTITY_TOKEN_FILE` | [identity federation](/platform/hosting/iam/identity_federation/)의 경우, Java Web Tokens (JWTs)가 저장된 로컬 디렉토리의 절대 경로입니다. |

<Warning>
W&B는 `GORILLA_DISABLE_ADMIN_TEAM_ACCESS`와 같은 일부 설정을 활성화하기 전에 주의를 기울이고 모든 영향을 이해할 것을 권장합니다. 궁금한 점이 있으면 W&B 팀에 문의하세요.
</Warning>
