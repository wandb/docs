title: OIDC로 SSO 구성하기
---

OIDC(OpenID Connect) 호환 ID 공급자에 대한 W&amp;B의 지원을 통해 Okta, Keycloak, Auth0, Google, Entra와 같은 외부 ID 공급자를 사용하여 사용자 ID 및 그룹 멤버십을 관리할 수 있습니다.

<div id="openid-connect-oidc">
  ## OpenID Connect (OIDC)
</div>

W&amp;B는 외부 Identity Provider(IdP)와의 연동을 위해 다음과 같은 OIDC 인증 플로우를 지원합니다.

1. Implicit Flow with Form Post
2. Authorization Code Flow with Proof Key for Code Exchange (PKCE)

이러한 플로우는 사용자를 인증하고, W&amp;B가 액세스 제어를 관리하는 데 필요한 ID 토큰 형태의 신원 정보를 제공합니다.

ID 토큰은 사용자의 이름, 사용자명, 이메일, 그룹 멤버십과 같은 사용자 신원 정보가 포함된 JWT입니다. W&amp;B는 이 토큰을 사용해 사용자를 인증하고, 시스템 내에서 적절한 역할 또는 그룹에 매핑합니다.

W&amp;B의 맥락에서 액세스 토큰은 사용자를 대신해 API 요청을 인가하지만, W&amp;B의 주요 관심사는 사용자 인증과 신원이므로 ID 토큰만 필요합니다.

환경 변수를 사용해 [IAM 옵션을 구성](/ko/platform/hosting/iam/advanced_env_vars)하여 [Dedicated Cloud](/ko/platform/hosting/hosting-options/dedicated_cloud) 또는 [Self-Managed](/ko/platform/hosting/hosting-options/self-managed) 인스턴스를 설정할 수 있습니다.

[Dedicated Cloud](/ko/platform/hosting/hosting-options/dedicated_cloud) 또는 [Self-Managed](/ko/platform/hosting/hosting-options/self-managed) W&amp;B 설치에서 Identity Provider를 구성하는 데 도움이 되도록, 다양한 IdP에 대해 아래 가이드를 따르십시오. W&amp;B의 SaaS 버전을 사용하는 경우 조직용 Auth0 테넌트 구성을 지원받으려면 [support@wandb.com](mailto:support@wandb.com)으로 문의하십시오.

<div id="configure-your-idp">
  ## IdP 구성
</div>

이 섹션에서는 OIDC에 사용할 아이덴티티 프로바이더(IdP)를 구성하는 방법을 설명합니다. 사용 중인 IdP에 해당하는 탭을 선택해 자세한 내용을 확인하세요.

<Tabs>
  <Tab title="Cognito">
    아래 절차에 따라 AWS Cognito를 인가용으로 설정합니다:

    1. 먼저 AWS 계정에 로그인하고 [AWS Cognito](https://aws.amazon.com/cognito/) 앱으로 이동합니다.

       <Frame>
         <img src="/images/hosting/setup_aws_cognito.png" alt="AWS Cognito 설정" />
       </Frame>

    2. IdP에서 애플리케이션을 구성할 때 허용된 콜백 URL을 지정합니다.
       * 콜백 URL로 `http(s)://YOUR-W&B-HOST/oidc/callback` 을(를) 추가합니다. `YOUR-W&B-HOST` 를 사용 중인 W&amp;B 호스트 경로로 바꾸십시오.

    3. IdP가 유니버설 로그아웃을 지원하는 경우, Logout URL을 `http(s)://YOUR-W&B-HOST` 로 설정합니다. `YOUR-W&B-HOST` 를 사용 중인 W&amp;B 호스트 경로로 바꾸십시오.

       예를 들어 애플리케이션이 `https://wandb.mycompany.com` 에서 실행 중이라면, `YOUR-W&B-HOST` 를 `wandb.mycompany.com` 으로 바꾸면 됩니다.

       아래 이미지는 AWS Cognito에서 허용된 콜백 URL과 로그아웃(사인아웃) URL을 설정하는 방법을 보여줍니다.

       <Frame>
         <img src="/images/hosting/setup_aws_cognito_ui_settings.png" alt="호스트 구성" />
       </Frame>

       *wandb/local* 은(는) 기본적으로 [`form_post` 응답 타입의 `implicit` grant](https://auth0.com/docs/get-started/authentication-and-authorization-flow/implicit-flow-with-form-post)를 사용합니다.

       또한 *wandb/local* 이 [PKCE Code Exchange](https://www.oauth.com/oauth2-servers/pkce/) 플로를 사용하는 `authorization_code` grant를 수행하도록 구성할 수도 있습니다.

    4. AWS Cognito가 토큰을 애플리케이션에 전달하는 방식을 정의하기 위해 하나 이상의 OAuth grant 타입을 선택합니다.

    5. W&amp;B에는 특정 OpenID Connect(OIDC) scope가 필요합니다. AWS Cognito App에서 다음 항목을 선택합니다.

       * &quot;openid&quot;
       * &quot;profile&quot;
       * &quot;email&quot;

       예를 들어, AWS Cognito App UI는 아래 이미지와 비슷한 모습이어야 합니다.

       <Frame>
         <img src="/images/hosting/setup_aws_required_fields.png" alt="필수 필드" />
       </Frame>

       설정 페이지에서 **Auth Method** 를 선택하거나 OIDC&#95;AUTH&#95;METHOD 환경 변수를 설정해 *wandb/local* 에서 사용할 grant 타입을 지정합니다.

       Auth Method는 반드시 `pkce` 로 설정해야 합니다.

    6. Client ID와 OIDC issuer의 URL이 필요합니다. OpenID discovery 문서는 `$OIDC_ISSUER/.well-known/openid-configuration` 에서 제공되어야 합니다.

       예를 들어 **User Pools** 섹션의 **App Integration** 탭에 있는 Cognito IdP URL에 User Pool ID를 이어 붙여 issuer URL을 생성할 수 있습니다.

       <Frame>
         <img src="/images/hosting/setup_aws_cognito_issuer_url.png" alt="AWS Cognito issuer URL" />
       </Frame>

       IdP URL로 &quot;Cognito domain&quot; 을 사용하지 마십시오. Cognito는 `https://cognito-idp.$REGION.amazonaws.com/$USER_POOL_ID` 에서 discovery 문서를 제공합니다.

    다음으로 [W&amp;B에서 SSO 설정](#set-up-sso-in-w%26b)을 진행합니다.

<div id="set-up-sso-in-wb">
  ## W&amp;B에서 SSO 설정하기
</div>

SSO를 설정하려면 관리자 권한과 다음 정보가 필요합니다:

* OIDC Client ID
* OIDC Auth method(`implicit` 또는 `pkce`)
* OIDC Issuer URL
* OIDC Client Secret (선택 사항; IdP 설정 방식에 따라 필요 여부가 달라짐)

IdP에서 OIDC Client Secret을 요구하는 경우, [환경 변수](/ko/platform/hosting/env-vars) `GORILLA_OIDC_SECRET`를 설정해 지정합니다.

* W&amp;B App에서 **System Console** &gt; **Settings** &gt; **Advanced** &gt; **User Spec**로 이동한 후, 아래 예시와 같이 `extraENV` 섹션에 `GORILLA_OIDC_SECRET`를 추가합니다.
* Helm에서는 아래와 같이 `values.global.extraEnv`를 설정합니다.
  ```yaml
  values:
  global:
      extraEnv:
      GORILLA_OIDC_SECRET="<your_secret>"
  ```

<Note>
  SSO를 구성한 후 인스턴스에 로그인할 수 없다면, `LOCAL_RESTORE=true` 환경 변수를 설정한 상태로 인스턴스를 다시 시작할 수 있습니다. 이렇게 하면 컨테이너 로그에 임시 비밀번호가 출력되고 SSO가 비활성화됩니다. SSO 관련 문제를 해결한 후 SSO를 다시 활성화하려면 반드시 해당 환경 변수를 제거해야 합니다.
</Note>
