---
title: SMTP 구성
description: "Self-Managed W&B Server 인스턴스에서 이메일 초대를 보내도록 내부 SMTP 서버를 구성합니다."
---

W&amp;B Server에서는 인스턴스 또는 팀에 사용자를 추가하면 이메일 초대가 전송됩니다. 기본적으로 W&amp;B는 이러한 초대를 서드파티 메일 서버를 통해 보냅니다. 조직에서 사내 네트워크의 아웃바운드 트래픽을 제한하는 경우 이러한 초대가 사용자에게 도달하지 못할 수 있습니다. 이를 해결하기 위해 Self-Managed W&amp;B Server 관리자는 내부 SMTP 서버를 통해 초대 이메일을 보내도록 설정할 수 있습니다.

W&amp;B Server가 내부 SMTP 서버를 사용하도록 구성하려면 다음 단계를 수행하세요.

1. 도커 컨테이너 또는 Kubernetes deployment에서 `GORILLA_EMAIL_SINK` 환경 변수를 `smtp://[USER:PASSWORD]@smtp.host.com:[PORT]`로 설정합니다.
   * `USER`와 `PASSWORD`는 선택 사항입니다.
   * 인증 없이 사용하도록 설계된 SMTP 서버를 사용하는 경우 환경 변수 값을 `GORILLA_EMAIL_SINK=smtp://smtp.host.com:[PORT]`와 같이 설정합니다.
   * SMTP에 일반적으로 사용되는 포트 번호는 `587`, `465`, `25`입니다. 사용하는 메일 서버의 유형에 따라 포트가 다를 수 있습니다.
2. 선택 사항: SMTP의 기본 발신자 이메일 주소(초기값은 `noreply@wandb.com`)를 구성하려면 서버에서 `GORILLA_EMAIL_FROM_ADDRESS` 환경 변수를 원하는 발신자 이메일 주소로 설정합니다.

이러한 환경 변수를 설정하면 W&amp;B Server는 지정한 발신자 주소를 사용해 신규 사용자 및 팀 초대 이메일을 내부 SMTP 서버를 통해 전송합니다.
