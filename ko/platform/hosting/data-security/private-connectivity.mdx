---
title: 전용 클라우드 에 대한 프라이빗 연결 구성
---

클라우드 제공업체의 보안 프라이빗 네트워크를 통해 [전용 클라우드](/platform/hosting/hosting-options/dedicated_cloud/) 인스턴스에 연결할 수 있습니다. 이는 AI 워크로드에서 W&B API로의 엑세스뿐만 아니라, 선택적으로 사용자 브라우저에서 W&B 앱 UI로의 엑세스에도 적용됩니다. 프라이빗 연결을 사용하면 관련 요청과 응답이 퍼블릭 네트워크나 인터넷을 거치지 않습니다.

<Note>
보안 프라이빗 연결은 전용 클라우드의 고급 보안 옵션으로 곧 제공될 예정입니다.
</Note>

보안 프라이빗 연결은 AWS, Google Cloud 및 Azure의 전용 클라우드 인스턴스에서 사용할 수 있습니다:

* AWS에서는 [AWS Privatelink](https://aws.amazon.com/privatelink/) 사용
* Google Cloud에서는 [Google Cloud Private Service Connect](https://cloud.google.com/vpc/docs/private-service-connect) 사용
* Azure에서는 [Azure Private Link](https://azure.microsoft.com/products/private-link) 사용

이 기능이 활성화되면 W&B는 해당 인스턴스에 대한 프라이빗 엔드포인트 서비스를 생성하고 연결에 필요한 DNS URI를 제공합니다. 이를 통해 사용자의 클라우드 계정에 프라이빗 엔드포인트를 생성하여 관련 트래픽을 프라이빗 엔드포인트 서비스로 라우팅할 수 있습니다. 프라이빗 엔드포인트는 클라우드 VPC 또는 VNet 내에서 실행되는 AI 트래이닝 워크로드에 대해 설정하기가 더 쉽습니다. 사용자 브라우저에서 W&B 앱 UI로의 트래픽에도 동일한 메커니즘을 사용하려면, 회사 네트워크에서 클라우드 계정의 프라이빗 엔드포인트로 적절한 DNS 기반 라우팅을 구성해야 합니다.

<Note>
이 기능을 사용하려면 W&B 팀에 문의하세요.
</Note>

보안 프라이빗 연결을 [IP 허용 목록 (allowlisting)](./ip-allowlisting)과 함께 사용할 수 있습니다. IP 허용 목록을 위해 보안 프라이빗 연결을 사용하는 경우, W&B는 AI 워크로드의 모든 트래픽과 사용자 브라우저 트래픽의 대부분에 대해 보안 프라이빗 연결을 사용하고, 권한이 있는 위치에서의 인스턴스 관리에는 IP 허용 목록을 사용할 것을 권장합니다.
