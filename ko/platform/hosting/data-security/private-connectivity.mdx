---
title: Dedicated Cloud에 대한 프라이빗 연결 구성
---

클라우드 제공업체의 보안 프라이빗 네트워크를 통해 [Dedicated Cloud](/ko/platform/hosting/hosting-options/dedicated_cloud/) 인스턴스에 연결할 수 있습니다. 이는 AI 워크로드에서 W&amp;B API로의 액세스뿐 아니라, 선택적으로 사용자 브라우저에서 W&amp;B 앱 UI로의 액세스에도 적용됩니다. 프라이빗 연결을 사용하면 관련 요청과 응답은 공용 네트워크나 인터넷을 경유하지 않습니다.

<Note>
  보안 프라이빗 연결은 Dedicated Cloud의 고급 보안 옵션으로 곧 제공될 예정입니다.
</Note>

보안 프라이빗 연결은 AWS, Google Cloud 및 Azure의 Dedicated Cloud 인스턴스에서 사용할 수 있습니다.

* AWS에서 [AWS Privatelink](https://aws.amazon.com/privatelink/) 사용
* Google Cloud에서 [Google Cloud Private Service Connect](https://cloud.google.com/vpc/docs/private-service-connect) 사용
* Azure에서 [Azure Private Link](https://azure.microsoft.com/products/private-link) 사용

이 기능을 활성화하면, W&amp;B는 해당 인스턴스를 위한 프라이빗 엔드포인트 서비스를 생성하고, 여기에 연결하는 데 사용할 관련 DNS URI를 제공합니다. 그런 다음, 클라우드 계정 내에서 프라이빗 엔드포인트를 생성하여 관련 트래픽을 프라이빗 엔드포인트 서비스로 라우팅할 수 있습니다. 프라이빗 엔드포인트는 클라우드 VPC 또는 VNet 내에서 실행 중인 AI 트레이닝 워크로드에 대해 더 쉽게 설정할 수 있습니다. 사용자 브라우저에서 W&amp;B 앱 UI로 가는 트래픽에도 동일한 메커니즘을 사용하려면, 사내 네트워크에서 클라우드 계정의 프라이빗 엔드포인트로 향하는 적절한 DNS 기반 라우팅을 구성해야 합니다.

<Note>
  이 기능을 사용하려면 W&amp;B 팀에 문의하십시오.
</Note>

[IP allowlisting](./ip-allowlisting)과 함께 보안 프라이빗 연결을 사용할 수 있습니다. IP allowlisting에 보안 프라이빗 연결을 함께 사용하는 경우, W&amp;B는 AI 워크로드에서 발생하는 모든 트래픽과, 가능하다면 사용자 브라우저에서 발생하는 대부분의 트래픽에 대해 보안 프라이빗 연결을 사용하고, 특권 위치에서의 인스턴스 관리는 IP allowlisting으로 수행할 것을 권장합니다.
