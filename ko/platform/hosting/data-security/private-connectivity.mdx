---
title: Dedicated Cloud용 프라이빗 연결 설정
description: "AWS PrivateLink, GCP 또는 Azure Private Link를 사용해 프라이빗 네트워크를 통해 W&B Dedicated Cloud 인스턴스에 연결합니다."
---

이 문서에서는 클라우드 제공업체의 보안 프라이빗 네트워크를 통해 W&amp;B Dedicated Cloud 인스턴스에 연결하는 방법을 설명합니다. 이를 통해 사용자 환경과 W&amp;B 간의 트래픽이 퍼블릭 인터넷을 거치지 않도록 할 수 있습니다. 이 문서는 Dedicated Cloud 인스턴스를 관리하며 AI 워크로드와 사용자 트래픽의 네트워크 액세스를 강화하려는 클라우드 및 보안 관리자를 대상으로 합니다.

클라우드 제공업체의 보안 프라이빗 네트워크를 통해 [Dedicated Cloud](/ko/platform/hosting/hosting-options/dedicated-cloud) 인스턴스에 연결할 수 있습니다. 이는 AI 워크로드에서 W&amp;B API로의 액세스와, 선택적으로 사용자 브라우저에서 W&amp;B App UI로의 액세스에 적용됩니다. 프라이빗 연결을 사용하면 관련 요청과 응답이 퍼블릭 네트워크나 인터넷을 통과하지 않습니다. 따라서 퍼블릭 인터넷에 대한 노출을 줄이고, 민감한 워크로드의 네트워크 격리 요구 사항을 충족하는 데 도움이 됩니다.

<Note>
  보안 프라이빗 연결은 Dedicated Cloud의 고급 보안 옵션으로 제공됩니다.
</Note>

보안 프라이빗 연결은 AWS, Google Cloud, Azure의 Dedicated Cloud 인스턴스에서 사용 가능합니다:

* AWS에서는 [AWS PrivateLink](https://aws.amazon.com/privatelink/)를 사용하세요.
* Google Cloud에서는 [Google Cloud Private Service Connect](https://cloud.google.com/vpc/docs/private-service-connect)를 사용하세요.
* Azure에서는 [Azure Private Link](https://azure.microsoft.com/products/private-link)를 사용하세요.

W&amp;B가 이 기능을 활성화하면, W&amp;B는 사용자 인스턴스용 프라이빗 엔드포인트 서비스를 생성하고 연결에 필요한 DNS URI를 제공합니다. 그러면 사용자는 자신의 클라우드 계정에서 관련 트래픽을 프라이빗 엔드포인트 서비스로 라우팅하는 프라이빗 엔드포인트를 생성할 수 있습니다. 클라우드 VPC 또는 VNet 내에서 실행되는 AI 트레이닝 워크로드에 대해 프라이빗 엔드포인트를 설정할 수 있습니다. 사용자 브라우저에서 W&amp;B App UI로 향하는 트래픽에 동일한 메커니즘을 사용하려면, 기업 네트워크에서 클라우드 계정의 프라이빗 엔드포인트로 적절한 DNS 기반 라우팅을 구성해야 합니다. 이 DNS 설정을 통해 브라우저 트래픽이 퍼블릭 W&amp;B 엔드포인트 대신 프라이빗 엔드포인트로 해석되도록 할 수 있습니다.

<Note>
  이 기능을 사용하려면 담당 W&amp;B 팀에 문의하세요.
</Note>

보안 프라이빗 연결을 [IP allowlisting](./ip-allowlisting)과 함께 사용해 네트워크 격리와 위치 기반 액세스 제어를 결합할 수 있습니다. 두 가지를 함께 사용하는 경우, W&amp;B는 AI 워크로드의 모든 트래픽에 대해, 그리고 가능한 경우 사용자 브라우저 트래픽에 대해서도 보안 프라이빗 연결을 사용하고, 권한이 있는 위치에서의 인스턴스 관리에는 IP allowlisting을 사용할 것을 권장합니다.
