---
title: Dedicated Cloud에 대한 프라이빗 연결 구성
description: "AWS PrivateLink, GCP 또는 Azure Private Link를 사용하여 프라이빗 네트워크를 통해 W&B Dedicated Cloud 인스턴스에 연결하세요."
---

클라우드 제공업체의 안전한 프라이빗 네트워크를 통해 [Dedicated Cloud](/ko/platform/hosting/hosting-options/dedicated-cloud) 인스턴스에 연결할 수 있습니다. 이는 AI 워크로드에서 W&amp;B API로의 액세스에 적용되며, 선택적으로 사용자 브라우저에서 W&amp;B App UI로의 액세스에도 적용됩니다. 프라이빗 연결을 사용하면 관련 Request 및 Response가 퍼블릭 네트워크나 인터넷을 경유하지 않습니다.

<Note>
  안전한 프라이빗 연결은 곧 Dedicated Cloud의 고급 보안 옵션으로 제공될 예정입니다.
</Note>

안전한 프라이빗 연결은 AWS, Google Cloud, Azure의 Dedicated Cloud 인스턴스에서 사용 가능합니다.

* AWS에서 [AWS Privatelink](https://aws.amazon.com/privatelink/) 사용
* Google Cloud에서 [Google Cloud Private Service Connect](https://cloud.google.com/vpc/docs/private-service-connect) 사용
* Azure에서 [Azure Private Link](https://azure.microsoft.com/products/private-link) 사용

활성화되면 W&amp;B가 인스턴스용 프라이빗 Endpoint 서비스를 생성하고, 연결에 필요한 DNS URI를 제공합니다. 이를 바탕으로 클라우드 계정에서 관련 traffic을 프라이빗 Endpoint 서비스로 route할 수 있는 프라이빗 endpoints를 생성할 수 있습니다. 프라이빗 endpoints는 클라우드 VPC 또는 VNet 내에서 실행되는 AI 트레이닝 워크로드에 더 쉽게 설정할 수 있습니다. 사용자 브라우저에서 W&amp;B App UI로의 traffic에 동일한 메커니즘을 사용하려면 회사 네트워크에서 클라우드 계정의 프라이빗 endpoints로 적절한 DNS 기반 routing을 구성해야 합니다.

<Note>
  이 기능을 사용하려면 담당 W&amp;B 팀에 문의하세요.
</Note>

안전한 프라이빗 연결은 [IP allowlisting](./ip-allowlisting)과 함께 사용할 수 있습니다. IP allowlisting에 안전한 프라이빗 연결을 사용하는 경우, W&amp;B는 권한이 있는 위치에서의 인스턴스 관리를 위해 IP allowlisting을 사용하면서, 가능하다면 AI 워크로드의 모든 traffic과 사용자 브라우저에서 발생하는 traffic의 대부분에 대해 안전한 프라이빗 연결을 적용할 것을 권장합니다.
