---
title: Dedicated Cloud의 IP 허용 목록 구성
description: "IP 허용 목록을 사용해 W&B Dedicated Cloud 인스턴스에 대한 접근을 승인된 IP 주소로 제한합니다."
---

승인된 IP 주소 목록에 포함된 주소에서만 [Dedicated Cloud](/ko/platform/hosting/hosting-options/dedicated-cloud) 인스턴스에 접근하도록 제한할 수 있습니다. 이는 AI 워크로드에서 W&amp;B API로의 접근뿐 아니라 사용자 브라우저에서 W&amp;B App UI로의 접근에도 적용됩니다. Dedicated Cloud 인스턴스에 IP 허용 목록이 설정되면 W&amp;B는 승인되지 않은 다른 위치에서 오는 모든 요청을 거부합니다. Dedicated Cloud 인스턴스의 IP 허용 목록을 구성하려면 담당 W&amp;B 팀에 문의하세요.

IP 허용 목록은 AWS, Google Cloud, Azure의 Dedicated Cloud 인스턴스에서 사용할 수 있습니다.

IP 허용 목록은 [보안 프라이빗 연결](./private-connectivity)과 함께 사용할 수 있습니다. IP 허용 목록을 보안 프라이빗 연결과 함께 사용하는 경우, 가능하다면 AI 워크로드의 모든 트래픽과 사용자 브라우저 트래픽의 대부분에는 보안 프라이빗 연결을 사용하고, 권한이 있는 위치에서 수행하는 인스턴스 관리에는 IP 허용 목록을 사용하는 것을 W&amp;B는 권장합니다.

<Warning>
  W&amp;B는 개별 `/32` IP 주소 대신 회사 또는 비즈니스의 egress 게이트웨이에 할당된 [CIDR 블록](https://en.wikipedia.org/wiki/Classless_Inter-Domain_Routing)을 사용할 것을 강력히 권장합니다. 개별 IP 주소를 사용하는 방식은 확장성이 없고 클라우드마다 엄격한 제한이 있습니다.
</Warning>
