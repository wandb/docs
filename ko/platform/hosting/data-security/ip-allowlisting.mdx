---
title: Dedicated Cloud용 IP 허용 목록 구성
---

[Dedicated Cloud](/ko/platform/hosting/hosting-options/dedicated_cloud) 인스턴스에 대한 액세스를 허용된 IP 주소 목록으로만 제한할 수 있습니다. 이는 AI 워크로드에서 W&amp;B API로의 액세스와 사용자 브라우저에서 W&amp;B 앱 UI로의 액세스 모두에 적용됩니다. Dedicated Cloud 인스턴스에 대해 IP 허용 목록이 설정되면, W&amp;B는 허용되지 않은 다른 위치에서 오는 모든 요청을 거부합니다. Dedicated Cloud 인스턴스에 대한 IP 허용 목록 구성을 위해 W&amp;B 팀에 문의하십시오.

IP 허용 목록은 AWS, Google Cloud, Azure의 Dedicated Cloud 인스턴스에서 사용할 수 있습니다.

IP 허용 목록은 [보안 프라이빗 연결](./private-connectivity)과 함께 사용할 수 있습니다. 보안 프라이빗 연결과 함께 IP 허용 목록을 사용하는 경우, W&amp;B는 가능한 경우 AI 워크로드에서 발생하는 모든 트래픽과 사용자 브라우저에서 발생하는 트래픽의 대부분에 대해 보안 프라이빗 연결을 사용하고, 권한이 있는 위치에서 인스턴스를 관리할 때 IP 허용 목록을 사용할 것을 권장합니다.

<Warning>
  W&amp;B는 개별 `/32` IP 주소보다는 기업 이그레스 게이트웨이에 할당된 [CIDR 블록](https://en.wikipedia.org/wiki/Classless_Inter-Domain_Routing)을 사용할 것을 강력히 권장합니다. 개별 IP 주소를 사용하는 것은 확장성이 없으며, 클라우드별로 엄격한 제한이 있습니다.
</Warning>
