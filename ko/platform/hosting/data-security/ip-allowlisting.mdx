---
title: Dedicated Cloud용 IP 허용 목록 구성
description: "IP 허용 목록을 사용하여 W&B Dedicated Cloud 인스턴스에 대한 액세스를 승인된 IP 주소로 제한합니다."
---

이 페이지에서는 W&amp;B [Dedicated Cloud](/ko/platform/hosting/hosting-options/dedicated-cloud)에서 IP 허용 목록이 작동하는 방식과 인스턴스에 대해 이를 요청하는 방법을 설명합니다. IP 허용 목록을 사용하면 Dedicated Cloud 인스턴스에 대한 액세스를 승인된 IP 주소 목록으로 제한할 수 있으므로, 승인된 위치의 트래픽만 W&amp;B API와 W&amp;B App UI에 액세스할 수 있습니다.

IP 허용 목록은 AI 워크로드에서 W&amp;B API로의 액세스와 사용자 브라우저에서 W&amp;B App UI로의 액세스에 적용됩니다. Dedicated Cloud 인스턴스에 대해 IP 허용 목록을 설정하면 W&amp;B는 승인되지 않은 다른 위치에서 오는 모든 요청을 거부합니다.

IP 허용 목록은 AWS, Google Cloud, Azure의 Dedicated Cloud 인스턴스에서 사용할 수 있습니다.

Dedicated Cloud 인스턴스의 IP 허용 목록을 구성하려면 담당 W&amp;B 팀에 문의하세요.

IP 허용 목록은 [보안 프라이빗 연결](./private-connectivity)과 함께 사용할 수 있습니다. 둘 다 사용하는 경우, W&amp;B는 AI 워크로드의 모든 트래픽과 사용자 브라우저 트래픽의 대부분에 보안 프라이빗 연결을 사용하고, 권한이 있는 위치에서 수행하는 인스턴스 관리에는 IP 허용 목록을 사용할 것을 권장합니다.

<Warning>
  W&amp;B는 개별 `/32` IP 주소보다 회사 또는 비즈니스 송신 게이트웨이에 할당된 [CIDR 블록](https://en.wikipedia.org/wiki/Classless_Inter-Domain_Routing)을 사용할 것을 권장합니다. 개별 IP 주소를 사용하면 확장성이 떨어지고 클라우드별로 엄격한 제한이 적용됩니다.
</Warning>
