title: 시크릿 관리
---

W&amp;B Secret Manager를 사용하면 액세스 토큰, 베어러 토큰, API 키, 비밀번호와 같은 민감한 문자열인 &#95;시크릿&#95;을 안전하게 중앙에서 저장·관리하고 주입할 수 있습니다. W&amp;B Secret Manager를 사용하면 코드에 민감한 문자열을 직접 추가하거나, 웹훅의 헤더 또는 [페이로드](/ko/models/automations/)를 구성할 때 민감한 문자열을 넣을 필요가 없습니다.

시크릿은 각 팀의 Secret Manager, 즉 [팀 설정](/ko/platform/app/settings-page/teams/)의 **Team secrets** 섹션에서 저장 및 관리됩니다.

<Note>
  * 시크릿을 생성, 편집, 삭제할 수 있는 사람은 W&amp;B Admin뿐입니다.
  * 시크릿은 Azure, Google Cloud, AWS에 호스팅하는 [W&amp;B Server 배포](/ko/platform/hosting/)를 포함해 W&amp;B의 핵심 기능으로 제공됩니다. 다른 배포 유형을 사용하는 경우, W&amp;B에서 시크릿을 어떻게 활용할 수 있는지 논의하려면 W&amp;B 계정 팀에 문의하세요.
  * W&amp;B Server에서는 보안 요구 사항을 충족하도록 보안 설정을 구성할 책임이 사용자에게 있습니다.

    * W&amp;B는 AWS, Google Cloud, Azure에서 제공하는 클라우드 제공자 시크릿 매니저에 W&amp;B 인스턴스용 시크릿을 저장하고, 이 시크릿 매니저를 고급 보안 기능으로 구성할 것을 강력히 권장합니다.

    * W&amp;B는 AWS, Google Cloud, Azure의 클라우드 시크릿 매니저에 있는 W&amp;B 인스턴스를 사용할 수 없고, 클러스터를 사용할 때 발생할 수 있는 보안 취약점을 방지하는 방법을 충분히 이해한 경우가 아니라면, Kubernetes 클러스터를 시크릿 저장소의 백엔드로 사용하는 것은 권장하지 않습니다.
</Note>

<div id="add-a-secret">
  ## 시크릿 추가하기
</div>

시크릿을 추가하려면:

1. 수신 서비스가 webhook 인증을 위해 토큰 또는 API 키를 요구하는 경우, 필요한 토큰 또는 API 키를 생성하세요. 필요하다면 비밀번호 관리자와 같이 안전한 곳에 이 민감한 문자열을 저장하세요.
2. W&amp;B에 로그인한 다음 팀의 **Settings** 페이지로 이동하세요.
3. **Team Secrets** 섹션에서 **New secret**을 클릭하세요.
4. 문자, 숫자, 밑줄(`_`)을 사용하여 시크릿 이름을 지정하세요.
5. **Secret** 필드에 민감한 문자열을 붙여넣으세요.
6. **Add secret**을 클릭하세요.

webhook을 구성할 때, 해당 webhook 자동화에 사용할 시크릿을 지정하세요. 자세한 내용은 [Configure a webhook](#configure-a-webhook) 섹션을 참조하세요.

<Note>
  시크릿을 생성하면, `${SECRET_NAME}` 형식을 사용해 [webhook 자동화의 페이로드](/ko/models/automations/create-automations/webhook/)에서 해당 시크릿을 사용할 수 있습니다.
</Note>

<div id="rotate-a-secret">
  ## 시크릿 교체하기
</div>

시크릿을 교체하고 값을 업데이트하려면 다음을 수행한다.

1. 해당 시크릿의 행에서 연필 아이콘을 클릭해 시크릿 상세 정보를 연다.
2. **Secret**에 새 값을 설정한다. 필요하면 **Reveal secret**을 클릭해 새 값을 확인한다.
3. **Add secret**을 클릭한다. 시크릿 값이 업데이트되며, 더 이상 이전 값으로는 참조되지 않는다.

<Note>
  시크릿이 생성되거나 업데이트된 이후에는 현재 값을 더 이상 확인할 수 없다. 대신 시크릿을 새 값으로 교체해야 한다.
</Note>

<div id="delete-a-secret">
  ## 시크릿 삭제하기
</div>

시크릿을 삭제하려면:

1. 시크릿 행에서 휴지통 아이콘을 클릭합니다.
2. 확인 대화 상자의 내용을 확인한 다음 **Delete**를 클릭합니다. 시크릿은 즉시 영구적으로 삭제됩니다.

<div id="manage-access-to-secrets">
  ## 시크릿 접근 관리
</div>

팀의 자동화는 팀의 시크릿을 사용할 수 있습니다. 시크릿을 제거하기 전에 해당 시크릿을 사용하는 자동화를 업데이트하거나 제거하여 자동화가 중단되지 않도록 하세요.
