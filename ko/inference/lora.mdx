  W&B Inference에서 파인튜닝된 모델을 서빙할 수 있도록 자체 커스텀 LoRA를 가져오세요.
---

LoRA(Low-Rank Adaptation)를 사용하면 완전히 새로운 모델 전체를 만드는 대신 가벼운 ‘추가 모듈’만 트레이닝하고 저장해 대규모 언어 모델을 맞춤화할 수 있습니다. 따라서 맞춤화가 더 빠르고, 비용도 적게 들며, 배포도 더 쉬워집니다.

LoRA를 트레이닝하거나 업로드해 기본 모델에 새로운 기능을 부여할 수 있습니다. 예를 들어 고객 지원, 창의적 글쓰기, 또는 특정 기술 분야에 특화되도록 만들 수 있습니다. 이렇게 하면 전체 모델을 다시 트레이닝하거나 재배포하지 않고도 모델의 동작을 조정할 수 있습니다.

<div id="why-use-wb-inference-for-loras">
  ## LoRA에 W&amp;B Inference를 사용하는 이유는 무엇인가요?
</div>

* 한 번 업로드하면 즉시 배포할 수 있으며, 관리할 서버가 없습니다.
* 아티팩트 버전 관리를 통해 현재 어떤 버전이 배포 중인지 정확히 추적하세요.
* 전체 모델 가중치 대신 작은 LoRA 파일을 교체해 몇 초 만에 모델을 업데이트하세요.

<div id="workflow">
  ## 워크플로
</div>

1. LoRA 가중치를 W&amp;B 아티팩트로 업로드하세요
2. API에서 아티팩트 URI를 모델 name으로 레퍼런스하세요
3. W&amp;B가 Inference를 위해 가중치를 동적으로 로드합니다

다음은 W&amp;B Inference를 사용하여 커스텀 LoRA 모델을 호출하는 예시입니다:

```python
from openai import OpenAI

model_name = f"wandb-artifact:///{WB_TEAM}/{WB_PROJECT}/qwen_lora:latest"

client = OpenAI(
    base_url="https://api.inference.wandb.ai/v1",
    api_key=API_KEY,
    project=f"{WB_TEAM}/{WB_PROJECT}",
)

resp = client.chat.completions.create(
    model=model_name,
    messages=[{"role": "user", "content": "Say 'Hello World!'"}],
)
print(resp.choices[0].message.content)
```

LoRA를 만들고 이를 아티팩트로 W&amp;B에 업로드하는 방법을 대화형으로 시연한 이 [시작하기 노트북](https://wandb.me/lora_nb)을 확인하세요.

<div id="how-to-add-loras-and-use-them">
  ## LoRA를 추가하고 사용하는 방법
</div>

두 가지 방법으로 W&amp;B Account에 LoRA를 추가하고 사용을 시작할 수 있습니다:

<Tabs>
  <Tab title="다른 곳에서 트레이닝한 LoRA 업로드">
    직접 만든 커스텀 LoRA 디렉터리를 W&amp;B 아티팩트로 업로드하세요. 다른 곳(로컬 환경, 클라우드 제공업체 또는 파트너 서비스)에서 LoRA를 트레이닝한 경우에 적합합니다.

    이 Python 코드는 로컬에 저장된 LoRA 가중치를 버전 관리되는 아티팩트로 W&amp;B에 업로드합니다. 필요한 메타데이터(기본 모델 및 저장 리전)가 포함된 `lora` 유형 아티팩트를 생성하고, 로컬 디렉터리의 LoRA 파일을 추가한 뒤, Inference에 사용할 수 있도록 W&amp;B 프로젝트에 기록합니다.

    ```python
    import wandb

    run = wandb.init(entity=WB_TEAM, project=WB_PROJECT)

    artifact = wandb.Artifact(
        "qwen_lora",
        type="lora",
        metadata={"wandb.base_model": "OpenPipe/Qwen3-14B-Instruct"},
        storage_region="coreweave-us",
    )

    artifact.add_dir("<path-to-lora-weights>")
    run.log_artifact(artifact)
    ```

    ### 주요 요구 사항

    Inference에서 직접 만든 LoRA를 사용하려면:

    * LoRA는 [지원되는 기본 모델 섹션](#supported-base-models)에 나열된 모델 중 하나를 사용해 트레이닝되어야 합니다.
    * LoRA는 W&amp;B Account에 `lora` 유형 아티팩트로, PEFT 형식으로 저장되어 있어야 합니다.
    * 지연 시간을 낮추려면 LoRA가 `storage_region="coreweave-us"`에 저장되어 있어야 합니다.
    * 업로드할 때 트레이닝에 사용한 기본 모델의 이름(예: `meta-llama/Llama-3.1-8B-Instruct`)을 포함하세요. 이렇게 하면 W&amp;B가 올바른 모델과 함께 이를 로드할 수 있습니다.
  </Tab>

  <Tab title="W&B로 새 LoRA 트레이닝">
    [W&amp;B Training (serverless RL)](/ko/training)으로 새 LoRA를 트레이닝하세요. 그러면 LoRA가 자동으로 W&amp;B 아티팩트가 되어 바로 사용할 수 있습니다.

    직접 LoRA를 트레이닝하는 방법에 대한 자세한 내용은 [OpenPipe의 ART 퀵스타트](https://art.openpipe.ai/getting-started/quick-start)를 참조하세요.

    트레이닝이 완료되면 LoRA를 아티팩트로 바로 사용할 수 있습니다.
  </Tab>
</Tabs>

LoRA가 아티팩트로 프로젝트에 추가되면 Inference Call에서 아티팩트의 URI를 다음과 같이 사용하세요:

```python
# 트레이닝이 완료되면 아티팩트를 직접 사용하세요
model_name = f"wandb-artifact:///{WB_TEAM}/{WB_PROJECT}/your_trained_lora:latest"
```

<div id="pricing">
  ## 가격
</div>

Serverless LoRA Inference는 단순하고 비용 효율적입니다. 항상 켜져 있는 서버나 전용 GPU 인스턴스에 비용을 지불하는 대신, 저장소와 실제로 실행한 Inference에 대해서만 비용을 지불합니다.

* [**저장소**](https://wandb.ai/site/pricing/) - LoRA 가중치를 저장하는 비용은 저렴하며, 특히 자체 GPU 인프라를 유지하는 것과 비교하면 더욱 그렇습니다.
* **Inference 사용** - LoRA 아티팩트를 사용하는 Call은 [표준 모델 Inference](/ko/inference/usage-limits#account-tiers-and-default-usage-caps)와 동일한 요율로 청구됩니다. 커스텀 LoRA를 서빙하는 데 추가 요금은 없습니다.
