이 페이지에는 지원되는 W&amp;B Server 릴리스의 릴리스 노트가 포함되어 있습니다.

* 더 이상 지원되지 않는 릴리스는 [Archived Server releases](/ko/release-notes/server-releases-archived)를 참조하세요.
* W&amp;B Server 릴리스 일정 및 지원에 대한 자세한 내용은 [W&amp;B Server release policies](/ko/release-notes/release-policies)를 참조하세요.

<Update label="v0.79.0" description="2026년 3월 24일">
  W&amp;B Server 0.79.0에는 이미지 확대/축소 및 패닝, 서비스 계정과 API 키 관리 개선, 성능 개선이 포함되었습니다. Inference에는 MiniMax M2.5 및 MiniMaxAI M2.5 모델이 추가되었습니다. 이제 Weave monitors는 모든 오퍼레이션의 스코어링을 지원하며, Weave UI에는 프롬프트 태그 및 별칭 관리 개선, Claude 에이전트용 채팅 뷰, 이모지 피드백 반응 또는 노트 내용 기준 필터링 등도 추가되었습니다.

  ## 지원 및 지원 종료

  * W&amp;B Server v0.67은 지원 종료되었습니다.

  [Release policies and processes](/ko/release-notes/release-policies)를 참조하세요. 도움이 필요하거나 질문이 있으면 [지원팀](mailto:support@wandb.com)에 문의하세요.

  ## 보안

  * 서비스 계정을 생성할 때 이제 해당 계정의 API 키를 만들도록 자동으로 안내됩니다.
  * 이제 W&amp;B App에서 서비스 계정 또는 API 키를 삭제할 때 확인이 필요합니다.
  * 이제 백엔드는 SQL 인젝션 방지에 도움이 되도록 필터의 열 이름을 이스케이프 처리합니다.
  * [발음 구별 기호](https://en.wikipedia.org/wiki/Combining_Diacritical_Marks)가 포함된 이메일 주소는 이제 사용자 등록이나 로그인 시 차단됩니다.

  ## 기능

  * **이미지 패닝 및 확대/축소**를 통해 W&amp;B의 전체 화면 이미지 모달에서 세부적인 이미지 디테일을 직접 확인할 수 있습니다. 이미지를 전체 화면으로 연 다음, 다음을 수행하세요.

    * UI 컨트롤 또는 키보드 단축키를 사용해 확대 수준을 설정하거나, 100%로 재설정하거나, 화면에 맞게 확대할 수 있습니다.
    * 확대된 상태에서는 클릭한 채 드래그하여 패닝할 수 있습니다.

    <Frame>
      ![전체 화면 이미지 모달의 패닝 및 확대/축소 컨트롤](/images/release-notes/v0-79-0/image-zoom-pan-fullscreen-modal.png)
    </Frame>
  * 고정된 run 및 baseline run의 경우 [summary metric deltas](/ko/models/runs/compare-runs#summary-metric-deltas)에서 각 run이 baseline과 어떻게 비교되는지 확인할 수 있습니다. 빠르게 시각적으로 확인할 수 있도록 시맨틱 색상을 켜려면 열의 Metric directionality를 설정하면 됩니다. 이 기능은 이제 정식 제공됩니다.
    <Frame>
      ![baseline과 비교하는 Runs 테이블의 summary metric deltas](/images/release-notes/v0-78-0/baseline-deltas-runs-table.png)
    </Frame>
  * run selector 및 Runs list가 개선되었습니다.
    * Git 커밋 기준 필터링 지원이 추가되었습니다.
    * config 파라미터가 지정된 run에 대해 `CONTAINS` Operator 지원이 추가되었습니다.
  * 데이터베이스 부하를 줄이기 위해 프로젝트 필드에 방어적 캐싱이 추가되었습니다.

  ### 레지스트리

  * 시스템 과부하 위험을 줄이기 위해 아티팩트 생성에 대한 새로운 요청 속도 제한이 추가되었습니다.

  ### Weave

  <Note>Weave를 사용하는 배포의 경우, v0.78+로 업그레이드하기 전에 Clickhouse v0.25.8+가 필요합니다.</Note>

  * 이제 Weave는 Claude 에이전트용 채팅 뷰를 지원합니다.
  * 이제 **Traces** 및 **Evaluations** 페이지에서 노트 내용과 이모지 피드백 반응을 기준으로 필터링할 수 있습니다.
  * 이제 **Assets** 페이지의 **Prompts** 탭에서 프롬프트 태그와 별칭을 관리할 수 있습니다.
  * 이제 **Calls** 및 **Traces** 페이지의 테이블에서 열을 드래그 앤 드롭으로 재정렬할 수 있습니다.
  * 이제 Weave monitors는 모든 오퍼레이션에서 스코어링을 지원합니다.

  ## 수정 사항

  * 백엔드 smoothing의 부동소수점 반올림 오류를 수정했습니다.
  * expression inputs에 입력할 때 커서가 갑자기 이동하던 버그를 수정했습니다.
  * 실제 크기 이미지가 gallery 모드에서 겹칠 수 있던 버그를 수정했습니다.
  * 긴 캡션이 있는 바운딩 박스의 정렬 문제를 수정했습니다.
  * 패널 및 hover overlay의 그림자를 제거했습니다.
  * 이제 패널 검색 결과에 Custom Charts도 포함됩니다.
</Update>

<Update label="v0.78.0" description="2026년 3월 3일">
  W&amp;B Server 0.78.0은 보안 및 액세스 기본값을 강화하고, 실험을 비교하고 비용을 보다 효율적으로 관리할 수 있는 도구를 추가하는 등 다양한 개선 사항을 제공합니다.

  <Note>
    Weave를 사용하는 배포의 경우, v0.78로 업그레이드하기 전에 Clickhouse v0.25.8 이상이 필요합니다.
  </Note>

  * In **Models**, 정규 표현식으로 여러 메트릭 플롯을 효율적으로 만들고, 중요한 Runs를 고정하고, 기준선과 실험을 비교하고, 개선된 Workspace 템플릿으로 워크스페이스를 간소화하세요.
  * **레지스트리**에는 향상된 검색, 새롭게 디자인된 Card, 더 명확하고 반응성이 뛰어난 리니지 그래프 등이 포함됩니다.
  * **Weave**에서 **Trace Analytics** 대시보드는 프로젝트 Overview 페이지에서 병목 지점과 예산을 유발하는 요인을 파악하는 데 도움이 됩니다. 저장된 뷰는 이제 맞춤형 트레이스, 평가, Leaderboard를 지원합니다. Weave 플레이그라운드는 이제 LoRA(Low-Rank Adaptation)와 새로운 모델을 지원합니다.
  * **Inference**에는 새로운 모델이 포함됩니다.

  <Warning>
    이 릴리스는 권한이 없는 사용자가 아티팩트에 액세스할 수 있게 하는 심각한 보안 문제의 영향을 받습니다. 이 버전은 설치하거나 업그레이드하지 마세요. 대신 수정 사항이 포함된 [v0.78.2](#v0-78-2)를 설치하세요.
  </Warning>

  ## 지원 및 지원 종료

  * v0.66.0 이하 버전은 지원 종료되었습니다.

  [릴리스 정책 및 프로세스](/ko/release-notes/release-policies)를 참고하세요. 도움이 필요하거나 문의 사항이 있으시면 [지원팀](mailto:support@wandb.com)에 문의하세요.

  ## Security

  * 이제 W&amp;B 조직 또는 배포는 Personal API 키를 최대 100개, 사용자당 조직 수준 API 키를 최대 100개, 그리고 Service Account당 API 키를 최대 100개까지 보유할 수 있습니다.
  * Enterprise 플랜에서는 조직에 [사용자를 초대](/ko/platform/hosting/iam/access-management/manage-organization#invite-a-user)하면 기본적으로 **Member** 역할이 부여됩니다. 다른 플랜에서는 기본 역할이 **Admin**입니다.
  * 현재 조직의 구성원이 아닌 새 사용자를 팀에 초대하면 해당 사용자의 초기 역할은 이제 **Member**로 설정됩니다.
  * 조직 또는 팀에 [초대된](/ko/platform/hosting/iam/access-management/manage-organization#invite-a-user) 사용자의 초기 역할이 이제 초대 테이블에 표시됩니다.

  ## 특성

  * 중요한 run을 추적하고 모델 실험을 효율적으로 평가하려면 [고정된 run과 baseline run](/ko/models/runs/compare-runs)을 사용하세요.

    * Workspace와 Runs 목록 상단에 계속 표시되도록 최대 6개의 run을 고정할 수 있습니다. baseline run이 있으면 baseline이 암묵적으로 고정되므로 최대 5개의 run만 고정할 수 있습니다.
    * 비교를 위한 레퍼런스로 baseline run을 지정합니다. baseline run은 항상 Workspace와 Runs 목록 상단에 표시됩니다.

    Runs table에서 summary metric deltas (미리보기)는 각 run이 baseline과 어떻게 비교되는지 보여줍니다. 이 기능은 초대를 받은 경우에만 사용 가능합니다. 자세한 내용은 [mailto:support@wandb.com] 지원팀에 문의하세요.

    <Frame>
      ![baseline과 비교한 Runs table의 summary metric 변화량](/images/release-notes/v0-78-0/baseline-deltas-runs-table.png)
    </Frame>

    선형 플롯에서는 비교를 돕기 위해 기준선이 시각적으로 구분되는 스타일로 표시됩니다.

    <Frame>
      ![선형 플롯에서 구분된 스타일이 적용된 기준 run](/images/release-notes/v0-78-0/baseline-line-plot-styling.png)
    </Frame>

  * [여러 메트릭으로 패널 생성](/ko/models/app/features/panels/line-plot#multi-metric-line-plot) 기능을 사용해 패널을 일괄로 만들어 Workspace를 빠르게 정리하고 대규모 메트릭 세트 전반의 패턴을 파악하세요. 예를 들어, 모델 레이어 전반에 그라디언트 메트릭을 겹쳐 표시하거나 특정 레이어를 나란히 배치할 수 있습니다. [비디오 데모 보기](https://www.loom.com/embed/ebbb1ab3f858473cb24420bf3abf7166).

  * 이제 **Sweeps** table에 **Sweep ID** 열이 포함됩니다.

  * 이제 관리자는 조직의 사용량 및 지출 알림을 사용자 지정할 수 있습니다. 새 **Usage &amp; Alerts** 탭의 조직 설정에서 사용량과 지출을 확인하고 저장소, 추적된 시간, Weave, Inference, 트레이닝에 대한 이메일 알림을 설정하세요. [플랜 사용](/ko/platform/app/settings-page/billing-settings#plan-usage)을 참조하세요.

  ### 레지스트리

  * 레지스트리 UI는 더 빠르게 이동하고 더 쉽게 훑어볼 수 있습니다.

    * 개선된 type-ahead 검색, 새롭게 디자인된 레지스트리 및 Collection Card, 그리고 새로운 레지스트리 아이콘을 통해 스마트 랭킹을 우선 적용해 에셋을 시각적으로 구분할 수 있으며, AI 랭킹도 곧 제공될 예정입니다.
    * 레지스트리 아이콘은 처음에는 자동 생성되며, 레지스트리 관리자가 언제든지 사용자 지정할 수 있습니다.

    * 리니지 그래프에서 노드에 마우스를 올리면 해당 노드가 강조 표시되고 다른 노드들은 흐리게 표시됩니다.

    * 편의를 위해 이제 **Settings** 페이지에서 레지스트리에 별표 표시를 할 수 있습니다.

  ### Inference

  * 모델 `Moonshot AI Kimi K2.5`은 이제 사용 가능합니다. [모델 카탈로그](/ko/inference/models)를 참조하세요.

  ### Weave

  * 이제 **프로젝트 Overview** 페이지에는 Request 수, 지연 시간 백분위수(P50/P95/P99), 모델별 token 사용량, 비용 분석을 보여주는 **Trace Analytics** 대시보드가 포함됩니다. 어떤 모델이 예산을 빠르게 소진하고 있는지, 또는 어떤 느린 Request가 P95를 떨어뜨리고 있는지 확인하세요. 시간 버킷을 클릭하면 관련 트레이스로 바로 이동하여 지연 시간 급증의 원인을 빠르게 파악할 수 있습니다.

  * Claude Opus 4.6을 이제 **Weave 플레이그라운드**에서 사용할 수 있습니다.

  * 이제 **트레이스** 및 **Evaluations** 테이블에서 열 순서를 다시 정렬할 수 있습니다.

    * 변경 사항을 [저장된 뷰](/ko/weave/guides/tools/saved-views)에 유지하세요. 비디오 데모를 확인하세요.

  ## 특성

  * **레지스트리 액세스 제어 업데이트:**
    * **프로그래밍 방식의 레지스트리 액세스 제어:** 이제 [레지스트리 역할](/ko/models/registry/configure_registry#details-about-registry-roles)이 SCIM 스키마에 포함됩니다. [사용자를 생성](/ko/platform/hosting/iam/scim#create-user)하거나 업데이트할 때 해당 사용자의 레지스트리 역할을 설정할 수 있으며, [사용자를 조회](/ko/platform/hosting/iam/scim#get-user)할 때도 레지스트리 역할이 반환됩니다.
    * **레지스트리 액세스 제한:** 전체 레지스트리 액세스를 부여하고 싶지 않은 경우, 이제 새로운 레지스트리 역할인 [Restricted Viewer](/ko/models/registry/configure_registry)를 부여할 수 있습니다. 이 역할은 컬렉션, 자동화 또는 기타 레지스트리 리소스를 생성, 업데이트 또는 삭제할 수 있는 기능 없이 레지스트리 아티팩트에 대한 읽기 전용 액세스를 제공합니다. 이는 민감하거나 규제를 받는 레지스트리 콘텐츠에 유용합니다.
    * [**확장된 레지스트리 액세스:**](https://wandb.ai/wandb_fc/product-announcements-fc/reports/Registry-for-W-B-Models-viewers-Faster-discovery-cleaner-handoffs--VmlldzoxNDc4MzM5Mg) 이제 Models **Viewer** 역할이 있는 모든 사용자는 동시에 전체 레지스트리 **Member**이므로, 기본적으로 모델과 아티팩트를 탐색하고, 비교하고, 사용할 수 있습니다. Models **Viewer**에는 **Restricted Viewer**, **Viewer**, 또는 **Admin**과 같은 다른 레지스트리 역할을 부여할 수 있습니다. 이를 통해 조직의 모든 사용자가 레지스트리의 전체 가치를 활용할 수 있습니다.

  * **별표 표시한 레지스트리:** 규모가 커져도 레지스트리를 더 쉽게 탐색할 수 있도록, 이제 레지스트리에 별표를 표시할 수 있습니다. 레지스트리 랜딩 페이지에서 레지스트리의 Card에 마우스를 올린 다음 별 모양 윤곽선을 클릭하세요. 개별 레지스트리 페이지에서는 페이지 상단의 별 모양 윤곽선을 클릭하세요. 별표 표시한 레지스트리는 레지스트리 랜딩 페이지 맨 위에 알파벳순으로 표시됩니다.

    ![별표 표시된 레지스트리](/images/release-notes/v0-75-0/starred-registries.png)

  * **자동화 실행 이력:** **Automation history** 탭에서 프로젝트 또는 레지스트리의 [Automations](/ko/models/automations/automation-events) 실행 이력을 확인하세요. 모든 자동화 실행은 최신순으로 표시되며, 상태와 주요 세부 정보를 한눈에 볼 수 있습니다. 자동화를 이해하고 디버그하는 데 도움이 되도록, 자세한 뷰에는 실행 조건, 액션, 결과가 표시됩니다.

  ## 성능

  * 포크 및 Rewind의 성능이 개선되었습니다.

    [Forking](/ko/models/runs/forking)으로 run을 생성하면 기존 run과 동일한 설정을 사용하는 새 run이 생성됩니다. 포크된 run의 변경 사항은 부모 run에 영향을 주지 않으며, 그 반대도 마찬가지입니다. 포크된 run과 부모 run 사이에는 포인터가 유지됩니다. run을 [Rewinding](/ko/models/runs/rewind)하면 기존 데이터를 잃지 않고 해당 시점부터 새 데이터를 log할 수 있습니다.

    중첩된 포크가 많은 프로젝트에서는 캐싱 개선으로 인해 새로운 run을 포크하는 작업이 이제 훨씬 더 효율적입니다.

  ## 수정 사항

  * 새 팀에 조직 서비스 계정이 추가되지 않을 수 있던 버그를 수정했습니다.
  * 그룹화된 라인에서 호버 마크가 누락될 수 있는 버그를 수정했습니다.
  * 리포트 패널의 **Import** 드롭다운에 유효하지 않은 프로젝트 이름이 포함될 수 있는 버그를 수정했습니다.
  * run selector의 필터 정렬에서 발생한 표시 버그를 수정했습니다.
  * 타임스탬프 **Within Last** 필터를 추가할 때 발생하던 페이지 충돌을 수정했습니다
  * 전역 선형 플롯 설정에서 X축을 **Wall Time**으로 설정할 수 없게 하던 버그를 수정했습니다.
  * Table에 로깅할 때 이미지 캡션이 표시되지 않을 수 있던 버그를 수정했습니다.
  * 희소 메트릭이 패널에 표시되지 않을 수 있던 버그를 수정했습니다.
  * **Run Overview** 페이지에서 **Description** 필드 이름이 이제 **Notes**로 변경되었습니다.
