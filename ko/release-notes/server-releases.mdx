<Update label="v0.80.0" description="2026년 4월 28일">
  **2026년 4월 28일**

  W&amp;B 0.80.0에는 실험을 효율적으로 관찰하고 분석하는 데 도움이 되는 새로운 기능, 버그 수정 및 보안 개선 사항이 포함되어 있습니다. 업데이트된 홈 페이지에서 조직 전반의 최근 활동을 보고, 검색하고, 필터링할 수 있습니다. run 지표의 z-점수 기반 자동화와 미디어 나란히 비교 기능이 정식 출시(generally available)되었습니다. 테이블 패널, 평행 좌표 플롯, 산점도도 개선되었습니다. 아티팩트 리니지 그래프 상호작용 및 공유가 더욱 효율적으로 개선되었습니다. 엔터프라이즈 배포의 프로비저닝 간소화를 위해 SCIM API에서 이제 Models 시트 및 Weave 역할 관리를 지원합니다.

  ## 지원 및 서비스 종료

  * W&amp;B Server v0.68의 지원이 종료되었습니다.

  [릴리스 정책 및 프로세스](/ko/release-notes/release-policies)를 참고하세요. 도움이 필요하거나 문의 사항이 있으면 [지원팀](mailto:support@wandb.com)에 문의하세요.

  ## 특성

  * 미디어 패널용 [Compare mode](/ko/models/app/features/panels/media#compare-mode)를 이제 정식으로 사용할 수 있습니다. Compare mode를 사용하면 파일을 다운로드하거나 탭을 전환하지 않고도 Runs, step 또는 인덱스를 자유롭게 조합해 이미지나 비디오 2~4개를 선택하고 하나의 그리드에서 볼 수 있습니다.

  * W&amp;B App의 홈 페이지가 개선되었습니다.

    * 본인과 동료의 최근 활동을 확인하고, 검색하거나 필터링할 수 있습니다.
    * 홈 페이지에서 최근 Reports에 직접 액세스할 수 있습니다.
    * 별표 표시한 Projects는 더 이상 홈 페이지에 표시되지 않지만, Projects 페이지 상단과 왼쪽 내비게이션에는 표시됩니다.

[W&amp;B SDK v0.26.0](/ko/release-notes/sdk-releases#0-26-0)+ 이상이 필요합니다.

  * 메트릭의 z-score를 기준으로 자동화를 트리거하는 기능이 정식으로 제공됩니다. z-score는 프로젝트에서 설정 가능한 run 범위 전반에 걸쳐 해당 메트릭의 평균으로부터 값이 몇 표준편차만큼 벗어나 있는지를 나타냅니다. 자세한 내용은 [Automation events and scopes](/ko/models/automations/automation-events#run-metrics-z-score-change)를 참조하세요.

  * Runs 목록에서 run에 마우스를 올리면 이제 병렬 좌표 플롯에서 해당 선이 강조 표시됩니다.

  * 산점도는 현재까지의 가장 높은 Y축 값과 가장 낮은 Y축 값을 잇는 선인 프런티어 라인과 관련된 새로운 표시 옵션을 지원합니다. 이러한 옵션은 플롯 설정의 **Annotations** 탭에서 구성하세요.
    * **run 라벨 표시**: 필요에 따라 run 이름 또는 그룹 이름이 포함된 라벨을 표시합니다.
    * **프런티어가 아닌 포인트 흐리게 표시**: 필요에 따라 프런티어가 아닌 포인트를 덜 두드러지게 표시합니다.

  * 이제 table 패널에서 오디오를 다운로드하는 동안에도 재생할 수 있습니다.

  * 수동으로 추가한 맞춤형 패널의 위치 재정의 설정은 이제 다시 로드해도 유지됩니다.

  * W&amp;B App에서 deployment의 현재 버전에 대한 릴리스 노트에 액세스하세요. 상단 내비게이션에서 **?**를 클릭해 **Resources &amp; Help** 메뉴를 연 다음, **Release Notes**를 클릭하세요.

  * 개선된 리니지 DAG 내비게이션: 이제 아티팩트 유형과 업스트림/다운스트림 홉을 기준으로 리니지를 필터링할 수 있습니다. 필터링되거나 숨겨진 노드는 이제 플레이스홀더로 계속 표시되므로, 필터를 반복해서 조정하는 동안에도 그래프 구조를 더 쉽게 파악할 수 있습니다.

  * URL로 아티팩트의 리니지 뷰를 공유하면 이제 받는 사람도 필터, 숨겨진 노드, 수동 위치, 뷰포트, 확대/축소 수준을 포함한 동일한 그래프 설정을 그대로 볼 수 있습니다.

    ![Scorer 대시보드 비디오 데모](/images/release-notes/v0-80-0/weave_scorer_dashboards.gif)
  * **Monitors** 페이지의 시간 구간별 **Feedback** 및 **Score** 차트에는 모니터별로 필터링된 숫자 추세와 불리언 값 개수가 표시됩니다.
  * 이제 Playground에서 Evaluation 결과 스트리밍을 지원합니다. Evaluation Playground에서 Evaluation을 실행하면 모델당 하나의 열이 있는 compare 뷰가 즉시 표시됩니다. 각 열에는 해당 모델의 Evaluation이 성공적으로 완료되면 실제 데이터가 표시되고, 아직 실행 중이면 스켈레톤이 표시되며, Evaluation이 실패한 경우에는 실패 상태가 표시됩니다. 결과는 모든 Evaluation이 끝날 때까지 기다리지 않고 각 Evaluation이 완료되는 대로 스트리밍됩니다. 진행률 표시줄은 call별 status를 폴링해 업데이트됩니다.
  * 추천 분류기 Card를 사용해 **Monitors** 페이지에서 분류기 모니터를 생성하고 관리하세요.

  * **Calls** 테이블에서 이제 사용자 ID가 사용자 이름으로 표시됩니다.

  ## 보안

  * [SCIM](/ko/platform/hosting/iam/automate_iam)을 사용해 Models 시트 또는 Weave 역할을 부여하세요.
  * [감사 로그를 가져올 때](/ko/platform/hosting/monitoring-usage/audit-logging#fetch-audit-logs) `startDate` 파라미터는 가져올 가장 최신 로그를 제어합니다.
  * 조직 관리자는 `graphql-disable-introspection` 설정 옵션이나 환경 변수 `GORILLA_GRAPHQL_DISABLE_INTROSPECTION`를 사용해 GraphQL 인트로스펙션을 비활성화할 수 있습니다. [W&amp;B SDK v0.26.0](/ko/release-notes/sdk-releases#0-26-0)+ 이상이 필요합니다.
  * SQL을 생성할 때 특정 구조화된 run 필터가 일관되게 이스케이프되지 않던 버그를 수정했습니다. 이로 인해 악의적인 필터 형태가 권한이 있는 클라이언트가 표현할 수 있어야 하는 범위를 넘어 쿼리 생성에 영향을 줄 수 있었습니다. 이제 필터 처리에는 다른 곳에서 사용하는 것과 동일한 컬럼 이스케이프가 적용되므로 필터 키로 쿼리 구조를 변경할 수 없습니다.
  * 특정한 경우, Sweeps에 연결된 run-큐 오퍼레이션과 직접적인 큐 삭제가 호출자가 의도한 범위를 벗어난 큐 또는 큐 항목에 영향을 줄 수 있던 버그를 수정했습니다. 이제 권한 확인은 Sweeps에서 큐에 추가하거나 큐 항목을 삭제하기 전에 큐 액세스 권한과 항목 소유권을 검증합니다.
  * run 메타데이터 키 제안이 검증되지 않은 사용자 입력을 데이터베이스 쿼리에 반영할 수 있던 버그를 수정했습니다. 이제 키 경로를 제한해 안전하고 예상된 컬럼 레퍼런스만 제안 처리 경로에 사용되도록 했습니다.
  * 서비스 계정 생성 API가 호출자가 관리하는 팀과 일치하지 않는 조직 식별자를 허용해, 서로 맞지 않는 조직 연결이 발생할 수 있었던 버그를 수정했습니다. 이제 서버는 일치하지 않는 조직 입력을 거부하고, entity의 실제 조직을 기준으로 조직 멤버십을 결정합니다.
  * 큐에서 항목을 꺼낼 때 이름만을 기준으로 큐를 내부 “public” Launch 큐로 처리할 수 있었던 반면, 큐에 넣을 때는 더 엄격한 검사를 사용했고, 푸시 시 호출자가 제공한 작업 작성자가 그대로 적용될 수 있었던 버그를 수정했습니다. 이제 디큐는 인큐와 동일한 public 큐 검증을 사용하며, 작업 작성자는 인증된 사용자와 일치하도록 맞춰져 스푸핑된 큐나 위조된 작성자로 인해 내부 자격 증명 주입이 트리거되지 않도록 했습니다.
  * trace server가 공급자 설정을 로드할 때 맞춤형 LLM 공급자 설정이 허용되지 않은 호스트를 대상으로 하거나 안전하지 않은 HTTP 헤더를 포함할 수 있던 버그를 수정했습니다. 이제 기본 URL과 헤더를 검증하므로 공급자가 민감한 메타데이터 유형의 엔드포인트를 가리키거나 차단된 헤더 패턴을 포함하지 못합니다.
  * 역직렬화 중 메모리 내 트레이스 아티팩트에 제공된 파일 경로가 의도된 임시 작업 공간을 벗어날 수 있던 버그를 수정했습니다. 이제 경로를 안전하게 정제하여 아티팩트 파일이 예상된 디렉터리 레이아웃 내에만 저장되도록 합니다.
  * 평가 작업 중 evaluate-model 워커가 안전하지 않은 맞춤형 직렬화 페이로드를 처리할 수 있던 버그를 수정했습니다. 이제 이러한 페이로드는 역직렬화 전에 거부되며, 평가 스택의 다른 보안 강화 경로와 일관되게 처리됩니다.

  ## Fixes

  * 이제 테이블 패널은 기본적으로 summary 테이블로 표시되며, 테이블이 여러 step에 걸쳐 로깅된 경우 stepper 뷰로 전환할 수 있는 버튼이 제공됩니다. 이를 통해 테이블 패널이 있는 Workspace의 로드 시간이 개선되고, 재개된 run의 이력을 볼 때 발생하던 비호환성도 완화됩니다.
  * 스윕 테이블에서 500개가 넘는 스윕을 로드하지 못하던 버그를 수정했습니다.
  * 이제 열 편집기에서 열에 마우스를 올리면 전용 **Show/Hide** 버튼이 표시됩니다.
  * 이제 Runs를 검색하면 퍼지 검색에 run ID(`run.name`의 값)도 포함됩니다.
  * 테이블 패널에서 오디오 로딩 성능이 개선되었습니다.

  ## 보안

  * 서비스 계정을 생성할 때 이제 해당 계정의 API 키를 만들도록 자동으로 안내됩니다.
  * 이제 W&amp;B App에서 서비스 계정 또는 API 키를 삭제하려면 확인이 필요합니다.
  * 이제 SQL 인젝션을 방지할 수 있도록 백엔드에서 필터의 열 이름을 이스케이프 처리합니다.
  * [발음 구별 기호](https://en.wikipedia.org/wiki/Combining_Diacritical_Marks)가 포함된 이메일 주소는 이제 사용자 등록이나 로그인에 사용할 수 없습니다.

  ## 기능

  * **이미지를 이동하고 확대/축소**하여 W&amp;B&#39;s 전체 화면 이미지 모달에서 세부적인 이미지 내용을 직접 확인할 수 있습니다. 이미지를 전체 화면으로 연 다음, 다음을 수행하세요.

    * UI 컨트롤 또는 키보드 단축키를 사용해 확대 수준을 설정하거나, 100%로 재설정하거나, 화면에 맞게 확대할 수 있습니다.

  ## 특성

  * **레지스트리 액세스 제어 업데이트:**
    * **프로그래밍 방식의 레지스트리 액세스 제어:** 이제 SCIM 스키마에 [레지스트리 역할](/ko/models/registry/configure_registry#details-about-registry-roles)이 포함됩니다. 사용자를 [생성](/ko/platform/hosting/iam/scim#create-user)하거나 업데이트할 때 해당 사용자의 레지스트리 역할을 구성할 수 있으며, [사용자 조회](/ko/platform/hosting/iam/scim#get-user) 시에도 레지스트리 역할이 반환됩니다.
    * **레지스트리 액세스 제한:** 전체 레지스트리 액세스 권한을 부여하지 않으려는 경우, 이제 새로운 레지스트리 역할인 [Restricted Viewer](/ko/models/registry/configure_registry)를 부여할 수 있습니다. 이 역할은 컬렉션, 오토메이션 또는 기타 레지스트리 리소스를 생성, 업데이트, 삭제할 수 없으며, 레지스트리 아티팩트에 대한 읽기 전용 액세스만 제공합니다. 이는 민감하거나 규제를 받는 레지스트리 콘텐츠에 유용합니다.
    * [**확장된 레지스트리 액세스:**](https://wandb.ai/wandb_fc/product-announcements-fc/reports/Registry-for-W-B-Models-viewers-Faster-discovery-cleaner-handoffs--VmlldzoxNDc4MzM5Mg) 이제 Models **Viewer** 역할이 있는 모든 사용자는 전체 레지스트리 **Member**이기도 하므로, 기본적으로 모델과 아티팩트를 탐색하고 비교하고 사용할 수 있습니다. Models **Viewer**에는 **Restricted Viewer**, **Viewer**, **Admin**과 같은 다른 레지스트리 역할을 할당할 수 있습니다. 이를 통해 조직의 모든 사용자가 레지스트리의 모든 가치를 활용할 수 있습니다.

  * **별표한 레지스트리:** 규모가 커져도 레지스트리를 더 쉽게 탐색할 수 있도록 이제 레지스트리에 별표를 할 수 있습니다. 레지스트리 랜딩 페이지에서 레지스트리 카드 위에 마우스를 올린 다음 윤곽선 모양의 별 아이콘을 클릭합니다. 개별 레지스트리 페이지에서는 페이지 상단의 윤곽선 모양의 별 아이콘을 클릭합니다. 별표한 레지스트리는 레지스트리 랜딩 페이지 상단에 알파벳순으로 표시됩니다.

  * SCIM API가 개선되었습니다:

  * 새로운 편의 메서드 [`delete_all_object_versions`](/ko/weave/reference/python-sdk/trace/weave_client).
  * 이제 OpenAI 스트리밍 엔드포인트 call의 출력에는 밀리초 단위의 새 필드 `time_to_first_token`이 포함됩니다.

  ## 예정된 변경 사항

  * 다가오는 릴리스에서 멀티 run과 single run Workspace 뷰를 일치시킬 예정입니다. 따라서 single run으로 들어가면 맞춤형 차트, 레이아웃, Settings를 포함해 멀티 run 뷰에서 설정한 것과 동일한 패널 레이아웃과 설정이 표시됩니다. 이는 작업 맥락을 유지하고 뷰 설정을 다시 해야 하는 번거로움을 줄이기 위한 것입니다. 하지만 이 변경으로 인해 Workspace의 다른 부분과 분리된 고유한 single run 뷰를 사용자 지정하는 기능은 없어집니다. 마이그레이션에 관해 궁금한 점이 있으면 W&amp;B 팀 또는 [지원팀](mailto:support@wandb.ai)에 문의하세요.
  * 향후 몇 차례의 릴리스에 걸쳐 기존 팀 단위 Model Registry를 W&amp;B Registry로 계속 이전할 예정입니다. 자세한 내용과 일정에 대한 안내는 아래의 [W&amp;B Registry GA announcement](#registry_ga_announcement)을 참조하세요.

  ## 특성

  * <a id="registry_ga_announcement" aria-label="W&B Registry GA 발표" /> **W&amp;B Registry를 이제 정식으로 사용할 수 있습니다**! Registry는 더 다양한 자산, 액세스 관리, 확장성을 제공하여 규모가 크고 다양한 팀이 AI 이니셔티브 전반에서 협업할 수 있는 진정한 허브 역할을 합니다.
    * Registry는 지금 바로 `ENABLE_REGISTRY_UI` 환경 변수를 사용해 활성화하거나 W&amp;B 지원팀에 문의해 켤 수 있습니다.
    * 앞으로 몇 차례의 릴리스에 걸쳐 기존 팀 수준 Model Registry 에셋을 W&amp;B Registry로 순차적으로 마이그레이션할 예정입니다. **사용자 쪽에서 별도로 해야 할 작업은 없습니다. W&amp;B가 각 팀의 Model Registry 에셋을 해당 팀에 대해 자동 생성된 팀 수준 비공개 레지스트리로 자동 복사합니다.**  기존 팀 수준 Model Registry의 모델 collection에 대한 기존 레퍼런스도 계속 작동합니다.

      1. Server v0.71부터는 모든 조직에서 Registry가 기본으로 활성화됩니다.
      2. Server v0.73(August)에는 업그레이드 중 이 마이그레이션을 자동으로 실행하는 opt-out 환경 변수가 포함됩니다. 대부분의 조직에서는 마이그레이션에 5~30분이 소요됩니다. 마이그레이션 중에는 이전 팀의 Model Registry나 새 Registry에 artifact를 연결하려고 하면 오류가 발생하며 실패합니다. ****Artifact logging 및 retrieval** **에는 영향이 없습니다**.
      3. Server v0.74(September)에서는 이 플래그가 제거되며, 업그레이드 중 마이그레이션이 진행됩니다.

      강력한 새 환경으로 최대한 원활하고 불편을 최소화하는 업그레이드가 되도록 준비하고 있습니다. 마이그레이션에 관해 궁금한 점이 있으면 W&amp;B 팀이나 [지원팀](mailto:support@wandb.ai)에 문의하세요.
  * **CoreWeave 인프라 모니터링**: CoreWeave 인프라에서 [W&amp;B Run](/ko/models/runs)을 실행하는 동안 [CoreWeave Mission Control](https://www.coreweave.com/mission-control)이 컴퓨팅 인프라를 모니터링하므로 GPU 장애, 온도 임계치 초과 등의 인프라 알림을 확인할 수 있습니다. 오류가 발생하면 CoreWeave는 해당 정보를 W&amp;B로 전송합니다. W&amp;B는 프로젝트 Workspace에서 run의 플롯에 인프라 정보를 표시합니다. CoreWeave는 일부 문제를 자동으로 해결하려고 시도하며, W&amp;B는 해당 정보를 run 페이지에 표시합니다. W&amp;B는 run의 시스템 수준 세부 정보를 확인할 수 있도록 SLURM 작업의 Grafana 대시보드 링크를 제공합니다. 자세한 내용은 [CoreWeave 인프라 알림 시각화](/ko/models/app/features/panels/line-plot)에서 확인하세요.
  * **CoreWeave 외부 파일 추적**: 이제 참조 아티팩트를 사용해 CoreWeave AI Object Storage에 저장된 외부 파일을 추적할 수 있습니다. 자세한 내용은 [외부 파일 추적](/ko/models/artifacts/track-external-files)에서 확인하세요.
  * **CoreWeave BYOB 지원**: 이제 Dedicated Cloud 및 Self-Managed에서 Instance 및 팀 수준 BYOB용 CoreWeave AI Object Storage를 지원합니다. Multi-tenant Cloud의 CoreWeave 팀 수준 BYOB도 곧 제공될 예정입니다. 자세한 내용은 [Bring your own bucket (BYOB)](/ko/platform/hosting/data-security/secure-storage-connector)에서 확인하세요.
  * **일괄 미디어 설정**: 선형 플롯과 마찬가지로, 이제 전체 워크스페이스나 특정 섹션에 있는 모든 미디어 패널 설정을 한 번에 관리할 수 있습니다. 각 패널을 개별적으로 조정하지 않고도 미디어 패널을 에포크별로 표시하거나 사용자 지정 그리드 패턴으로 손쉽게 배열할 수 있습니다. 개별 패널 설정은 전역 설정보다 우선 적용됩니다.
  * 기존 Workspace와 동일한 Settings으로 Workspace를 빠르게 생성하려면 **Workspace templates**를 사용하세요. 현재 Workspace template에서는 맞춤형 선형 플롯 Settings을 정의할 수 있습니다. 자세한 내용은 [Workspace templates](/ko/models/track/workspaces/#workspace-templates)에서 확인하세요.
  * **incremental table logging**을 사용하면 머신 러닝 실험 중에 테이블에 행 배치를 로깅할 수 있습니다. 이는 장시간 실행되는 작업을 모니터링하거나, 업데이트를 위해 run 중에 로깅하는 것이 비효율적인 큰 테이블을 다룰 때 특히 유용합니다. UI에서는 새 행이 로깅될 때마다 테이블이 업데이트되므로, 전체 run이 끝날 때까지 기다리지 않고도 최신 데이터를 확인할 수 있습니다. 증분 단위로 step을 이동하며 서로 다른 시점의 테이블을 확인할 수 있습니다. 자세한 내용은 [테이블 로깅](/ko/models/track/log/log-tables/#adding-data-incrementally)에서 확인하세요.

  * 이제 전체 화면 패널 URL 끝의 슬래시가 제거됩니다.
  * 삭제 직후 새로고침하면 삭제한 저장된 뷰가 다시 나타나던 버그를 수정했습니다.
  * 다크 모드가 켜진 상태에서 페이지를 새로고침할 때 화면이 깜빡이는 표시 버그를 수정했습니다.
  * 비단조 값을 사용할 때 슬라이더 step이 올바르지 않게 설정되던 버그를 수정했습니다.
  * run을 그룹 간에 이동할 때 사용할 수 있는 편의 체크박스와 알림을 추가했습니다.
  * `wandb.define_metric()`에서 메트릭 이름에 와일드카드 문자 `*`를 사용하면 이름에 `/`가 포함된 메트릭과 매칭되지 않던 버그를 수정했습니다.
  * W&amp;B는 이제 collection 포트폴리오가 가비지 컬렉션될 때 소스 artifact를 더 이상 삭제하지 않습니다.
  * **Allowed types** 및 **Registry visibility** 선택 상자에서 텍스트를 입력할 수 있게 잘못 허용되던 버그를 수정했습니다.

  * 레지스트리 컬렉션에서 **작업 이력**을 클릭했을 때 **버전** 뷰가 잘못 로드되던 버그를 수정했습니다.
  * 레지스트리에서 이제 `job` 유형 artifact를 추가할 수 있습니다.
  * 리니지 타일의 너비가 더 넓어져, 텍스트가 잘리기 전에 더 많은 내용을 표시합니다.

  * 이제 artifact 브라우저에서 이름으로 검색할 때, 이전처럼 처음 500개의 artifact만 검색하는 것이 아니라 모든 artifact를 검색합니다.

  ## Upgrading

  W&amp;B v0.69.x로 [업그레이드](/ko/platform/hosting/server-upgrade-process/#update-with-helm)하려면 `operator-wandb` Helm chart v0.31.4 이상을 사용해야 합니다. 그렇지 않으면 업그레이드 후 `weave-cache-clear` 컨테이너가 시작되지 않을 수 있습니다. 배포 환경에서 다음 값을 사용하고 있는지 확인하세요:

  ```yaml
  chart:
    url: https://charts.wandb.ai
    name: operator-wandb
    version: 0.31.4
  ```

  업그레이드와 관련하여 질문이 있거나 문제가 발생한 경우 [지원팀](mailto:support@wandb.com)에 문의하세요.

  ## 특성

  * 이제 workspace에서 직접 run의 맞춤형 표시 이름을 설정할 수 있습니다. 사용자 지정한 run 이름은 모든 플롯과 테이블에 표시되지만, 본인의 workspace에만 적용되며 팀원들의 뷰에는 영향을 주지 않습니다. 이렇게 하면 workspace를 더 명확하고 깔끔하게 볼 수 있어, 모든 범례와 플롯에 `*...v6-final-restart...`* 같은 라벨이 더 이상 표시되지 않습니다.
  * run을 필터링하거나 그룹화할 때 색상이 서로 겹쳐 잘 구분되지 않는 경우가 있습니다. run 선택기의 새로운 **색상 무작위 지정** 옵션은 기본 팔레트에서 현재 선택한 run 또는 그룹에 임의의 색상을 다시 할당해 색상을 더 쉽게 구분할 수 있게 해줍니다.
  * 선형 플롯에서 이제 선을 **Cmd+Click**하면 새 탭에서 단일 run 뷰를 열 수 있습니다.
  * 이제 비디오 미디어 패널에서 재생, 일시 정지, 탐색, 전체 화면 보기, 재생 속도 조절 등을 위한 재생 컨트롤을 더 많이 제공합니다.
  * 모든 미디어 패널 유형의 Settings가 재구성되고 개선되었습니다.
  * 이제 포인트 클라우드 패널의 점 색상과 배경 색상을 사용자 지정할 수 있습니다.
  * 팀 수준 및 조직 수준의 서비스 계정이 이제 레지스트리와 상호작용할 수 있습니다.
  * 개선된 지수 가중 이동 평균(EMA) 스무딩은 전체 비집계 데이터에서 작업할 때 더 신뢰할 수 있는 [스무딩 선](/ko/models/app/features/panels/line-plot/smoothing)을 제공합니다. 대부분의 경우 성능 향상을 위해 스무딩은 백엔드에서 처리됩니다. 이 기능은 v0.68.x에서 비공개 프리뷰로 제공되었습니다.

  ## Performance

  * [`wandb` SDK](/ko/models/quickstart/#install-the-wandb-library-and-log-in) v0.19.11에서는 artifact 다운로드 속도가 평균적으로 3~5배 빨라졌습니다. 예를 들어, 이전에는 약 100 MB/초로 다운로드되던 artifact를 이제는 450 MB/초 이상으로 더 빠르게 다운로드할 수 있습니다. 실제 다운로드 속도는 네트워크 및 저장소 infrastructure와 같은 요소에 따라 달라집니다.
  * [프로젝트](/ko/models/track/project-page) 및 [User Settings](/ko/platform/app/settings-page/user-settings) 페이지에서 캐싱이 개선되었습니다.

  * 콘솔 로그를 더 높은 밀도로 표시하는 옵션을 추가했습니다.
  * 이제 Workspace 로딩 화면에 더 많은 정보가 표시됩니다.
  * 워크스페이스에서 리포트에 패널을 추가할 때, 이제 대상 리포트 목록에 현재 프로젝트의 Reports가 먼저 표시됩니다.
  * 중복 값이 표시될 정도로 Y축 값이 과도하게 반올림되던 여러 문제를 수정했습니다.
  * 잘못된 스무딩 매개변수를 입력할 때 발생하던 혼란을 주는 동작을 수정했습니다.

  * [태그 기반 run 필터](/ko/models/runs/filter-runs/#filter-runs-with-tags)를 추가하면 이제 다른 필드로 필터링할 때와 마찬가지로 해당 필터가 기본 선택됩니다.
  * 활성 run의 run selector에 표시될 수 있었던 녹색 벨 아이콘을 제거했습니다.
  * 개별 run의 System 페이지를 제거했습니다.
  * 이제 프로젝트 설명 필드에서 줄바꿈이 유지됩니다.
