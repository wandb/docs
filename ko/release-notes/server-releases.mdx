이 페이지에서는 지원 중인 W&amp;B Server 릴리스의 릴리스 노트를 제공합니다.

* 더 이상 지원되지 않는 릴리스는 [Archived Server releases](/ko/release-notes/server-releases-archived)를 참고하세요.
* W&amp;B Server 릴리스 일정 및 지원에 대한 자세한 내용은 [W&amp;B Server release policies](/ko/release-notes/release-policies)를 참고하세요.

<Update label="v0.77.1" description="2026년 1월 28일">
  **2026년 1월 28일**

  ## 변경 사항

  * **Self-Managed** 환경에서 이제 W&amp;B Workspace에 Weave 패널을 추가하고 Weave trace에서 W&amp;B Artifacts를 참조할 수 있습니다. 이 기능은 이전에는 **Dedicated Cloud** 에서만 사용할 수 있었습니다. 자세한 내용은 [Use Weave with W&amp;B training runs](/ko/weave/guides/tools/weave-in-workspaces)를 참고하세요.

  ## 수정 사항

  * alias 필터가 있을 때 Registry 버전 쿼리가 중복된 결과를 반환하던 버그를 수정했습니다.
</Update>

<Update label="v0.77.0" description="2026년 1월 21일">
  **2026년 1월 21일**

  W&amp;B 0.77은 조직의 보안과 생산성을 강화하기 위한 보안 및 사용성 개선 사항을 도입합니다.

  이번 릴리스에는 API 키 보안 강화, 단일 run 및 다중 run 워크스페이스 간 공유 설정을 통한 간소화된 탐색, 새로운 선 그래프 음영 옵션, 미디어 패널 동기화 및 루핑의 정식 출시(GA), 그리고 주요 편의성 개선 및 버그 수정이 포함되어 있습니다.

  ## 지원 및 서비스 종료

  * W&amp;B Server v0.65 이하 버전은 2026년 1월 30일부로 지원이 종료됩니다.

  [릴리스 정책 및 프로세스](/ko/release-notes/release-policies)를 참조하세요. 도움이 필요하거나 문의 사항이 있으시면 [support](mailto:support@wandb.com)로 문의하세요.

  ## 예정된 변경 사항

  * 곧 Registry UI를 더 빠르게 탐색하고, 내용을 한눈에 훑어보기 쉬워집니다.

    * 향상된 자동 완성 검색, 새로 디자인된 Registry 및 컬렉션 카드, 그리고 새로운 Registry 아이콘을 통해 자산을 시각적으로 더 잘 구분할 수 있습니다. 처음에는 스마트 랭킹을 사용하고, 곧 AI 기반 랭킹이 추가될 예정입니다.
    * Registry 아이콘은 처음에는 자동으로 생성되며, Registry 관리자는 언제든지 이를 사용자 지정할 수 있습니다.

    이러한 업데이트는 다음 릴리스에 포함될 예정입니다. 자세한 내용은 [지원팀](mailto:support@wandb.com)이나 AISE 담당자에게 문의하세요.

  ## 보안

  * W&amp;B 전반에 걸쳐 API 키를 업데이트하여 개인 계정과 서비스 계정 모두의 보안과 키 관리를 개선했습니다. 이러한 변경 사항은 최신 보안 모범 사례에 부합하면서도 Teams가 더 큰 유연성과 더 안전한 자동화를 활용할 수 있도록 해 줍니다.

    **알아둘 중요한 사항**

    * API 키는 한 번만 표시되며, 생성 직후 바로 복사해 저장해야 합니다.
    * 기존 키는 더 이상 복사할 수 없으며, 새 키를 생성해야 합니다.
    * 언제든지 키 이름을 변경하거나 삭제할 수 있습니다.
    * 서비스 계정은 이제 여러 개의 API 키를 지원합니다.

    [회원가입 및 API 키 생성](/ko/models/quickstart#sign-up-and-create-an-api-key)을 참고하세요.

  ## 기능

  * **워크스페이스 내비게이션이 업데이트되었습니다.** 멀티-run 및 단일-run 워크스페이스 뷰를 서로 일치하도록 정돈했습니다. 이제 단일 run의 세부 화면으로 들어가면, 멀티-run 뷰에 대해 설정해 둔 것과 동일한 레이아웃과 패널 설정(사용자 지정 차트, 레이아웃, 설정 포함)을 그대로 볼 수 있습니다. 이를 통해 맥락을 유지하고, 뷰 설정을 다시 하는 수고를 줄일 수 있습니다. 내비게이션 버튼 위치 변경을 포함한 변경 사항은 이 [영상](https://www.loom.com/share/df232057aa2d4ee78026a4f9cf400499?sid=da3661eb-e8cd-4aa5-a666-db2c4091b02b)에서 확인할 수 있습니다.
    <Frame>
      ![업데이트된 Workspace 내비게이션을 보여 주는 스크린샷](/images/release-notes/v0-77-0/workspace-navigation-update.png)
    </Frame>
  * 섹션 수준의 [비디오 재생 동기화 및 루프](/ko/models/app/features/panels/media#sync) 기능이 정식으로 제공됩니다. 기본적으로 섹션 내 비디오는 서로 동기화된 상태로 재생됩니다.
  * 그룹화된 run에 대한 full-fidelity 선 그래프에서 이제 음영 영역에 표시할 내용을 **Min/Max**, **Standard Deviation**, **Standard Error**, **None** 중에서 설정할 수 있습니다. [Sampling](/ko/models/app/features/panels/line-plot/sampling)을 참조하세요.
  * 평행좌표 플롯에서 이제 **Cmd+Click**(macOS) 또는 **Ctrl+Click**(Linux / Windows)으로 선을 클릭하면, 해당 run의 상세 페이지를 새 탭에서 열어 자세히 확인할 수 있습니다. 그룹화된 플롯에서는 아직 지원되지 않습니다.

      ![평행 좌표 플롯에서 선을 클릭해 run 세부 정보를 여는 애니메이션](/images/release-notes/v0-77-0/pcp-click-to-run-detail.gif)
    </Frame>
  * 자동화 설정을 더 효율적으로 할 수 있도록, 이제 앱이 자동화의 기준이 될 메트릭을 제안합니다.

  ### Registry

  * Registry의 [Artifact 계통 그래프](/ko/models/artifacts/explore-and-traverse-an-artifact-graph)에서 이제 항목을 드래그 앤 드롭으로 재배치하거나 숨길 수 있습니다.
  * 아티팩트 버전을 조회할 때, 이제 한 번에 새로운 레지스트리를 생성하고 해당 아티팩트 버전을 레지스트리에 연결할 수 있습니다.

  ### Weave

  * 이제 W&amp;B App에서 빈 상태일 때 표시되는 Weave 온보딩 페이지에 Python 예제 코드와 함께 TypeScript 예제 코드도 포함됩니다.
  * Weave 호출 로그가 이제 기본 설정으로 무한 스크롤을 지원합니다. 이전에는 가장 최근 로그 10,000줄만 볼 수 있었습니다.
  * [Evaluation](/ko/weave/guides/core-types/evaluations) 비교 페이지의 막대 차트와 레이더 차트에서 이제 크기 조정 및 전체 화면 보기 기능을 지원하며, 차트 영역에 마우스를 올리면 툴팁에 자세한 정보가 표시됩니다.
  * [Weave Playground](https://wandb.ai/site/playground/)에는 새로운 모델이 포함되어 있습니다:
    * GPT 5.2
    * Grok 4.1
    * Opus 4.5
  * 이제 [Weave Playground](https://wandb.ai/site/playground/)에서 새 프롬프트를 만들고 저장할 수 있습니다.

  ## 수정 사항

  * run의 **Notes** 텍스트 필드를 편집할 때 필드 너비가 너무 좁게 표시되던 버그를 수정했습니다. 이제 텍스트 상자는 보기와 편집 시 동일한 너비로 표시됩니다.
  * W&amp;B Report의 PDF 내보내기에서 이미지의 해상도가 너무 낮게 출력되던 버그를 수정했습니다.
  * Report의 라인 차트가 run 선택기 변경에 맞춰 동기화되어 업데이트되지 못하는 버그를 수정했습니다.
  * 리포트에서 이상치를 숨길 수 있는 옵션이 없는 선형 플롯에 이상치가 표시되던 버그를 수정했습니다.
  * 여러 runset을 포함한 Report의 선 그래프에서, 서로 다른 runset에 속하지만 이름이 같은 run에 대해 잘못된 데이터가 표시되던 버그를 수정했습니다. 이제 Runs는 runset과 이름의 조합으로 고유하게 식별됩니다.

  ### Weave

  * Dedicated Cloud의 Weave UI에 Dedicated Cloud에서는 사용할 수 없는 W&amp;B Inference에 대한 링크가 표시되던 버그를 수정했습니다.
  * [Evaluation](/ko/weave/guides/core-types/evaluations) 비교 페이지에서 세부 정보 패널이 열려 있을 때 막대 차트의 크기가 너무 좁게 조정되던 문제를 수정했습니다.
  * [Weave Playground](https://wandb.ai/site/playground/)에서 텍스트를 입력할 때 커서가 화면에 보이는 영역에서 사라질 수 있던 UI 버그를 수정했습니다.
  * [Weave Playground](https://wandb.ai/site/playground/)에서 호출을 로드할 때 템플릿 변수를 편집할 수 없던 문제를 수정했습니다.
  * [Evaluation](/ko/weave/guides/core-types/evaluations) 개요 페이지에서 하위 트레이스의 상세 보기로 이동할 때 operation 선택기가 설정되지 않던 버그를 수정했습니다. 이로 인해 어떤 트레이스가 선택되었는지 파악하기 어려웠고, 일부 열이 비어 보이는 문제가 발생했습니다.
  * [Evaluation](/ko/weave/guides/core-types/evaluations) 비교 보기 페이지에서 **Delete**를 클릭해도 비교 보기가 실제로 삭제되지 않던 버그를 수정했습니다.
  * 커스텀 타입을 사용하는 [Evaluation](/ko/weave/guides/core-types/evaluations) 결과에서 커스텀 타입 대신 `CellValue` 필드가 비어 있는 상태로 표시될 수 있던 버그를 수정했습니다.
  * 좌측에 고정된 셀이 **runs as rows** 표시 모드에서 투명하게 표시될 수 있던 버그를 수정했습니다.
</Update>

<Update label="v0.76.1" description="2025년 12월 4일">
  **2025년 12월 4일**

  ## 수정사항

  * 워크스페이스에서 선형 플롯(line plot)을 리포트로 가져올 때, 플롯에서 극단적인 이상치를 숨긴 경우 가져오기가 멈출 수 있던 버그를 수정했습니다.
  * 테이블 패널의 높이를 줄였을 때, 패널이 올바르게 리사이즈되지 않을 수 있던 버그를 수정했습니다.
  * 마스크 오버레이가 캡션이 있는 이미지의 경계를 넘어가던 버그를 수정했습니다.
  * run 노트의 편집 상자 크기가 편집하기에 너무 좁던 버그를 수정했습니다. 이제 편집 상자는 표시 모드에서의 텍스트와 동일한 크기를 사용합니다.
  * Organization Dashboard에서 **Service Accounts** 탭을 클릭하면 페이지가 다운될 수 있던 버그를 수정했습니다.
  * run 로그를 보고 있을 때 한 run에서 다른 run으로 전환하면 잘못된 run의 로그가 표시될 수 있던 캐싱 버그를 수정했습니다.
  * 클라이언트가 GraphQL 연산을 취소하는 경우, 이제 이를 `5xx` HTTP 서버 오류가 아닌 `4xx` HTTP 클라이언트 오류로 올바르게 보고합니다.
</Update>

<Update label="v0.76.0" description="2025년 11월 20일">
  **2025년 11월 20일**

  W&amp;B v0.76은 더 스마트한 미디어 도구, 원활한 워크플로우, 강력한 새 인터페이스를 도입하여 실험 인사이트와 생산성을 크게 향상시킵니다. 자동 재생 및 반복 재생을 지원하는 멀티 비디오 동기화로 나란히 빠르게 비교 평가할 수 있으며, 간소화된 워크스페이스 탐색으로 뷰 간 전환이 더욱 편리해졌습니다. 또한, 새로운 LEET 터미널 UI를 통해 커맨드 라인에서 바로 실시간 run 인사이트를 빠르게 확인할 수 있습니다.

  ## 지원 및 서비스 종료

  * W&amp;B Server v0.63 이하 버전은 2025년 11월 20일부로 지원이 종료되었습니다.
  * W&amp;B Server v0.65는 2026년 1월 30일에 제품 수명 종료(EOL)에 도달할 예정입니다.

  ## 기능

  * **Multi-video Sync**: 여러 비디오 패널을 동기화해 동시에 재생·제어하여 트레이닝 run의 정성적 평가 및 비교, 다양한 실험에서의 모델 동작 분석, 멀티모달 모델 체크포인트 평가에 활용할 수 있습니다.

    동기화된 비디오 재생을 사용하면 여러 비디오를 동시에 재생·일시 정지하고, 타임라인을 이동하며, 속도를 조절할 수 있어 스타일, 타이밍, 모션 등을 빠르게 시각적으로 검수할 수 있는 실제 나란히 비교가 가능합니다.

    <Frame>
      ![비디오 재생 동기화 기능을 보여주는 애니메이션](/images/release-notes/v0-76-0/video_playback_sync_demo.gif)
    </Frame>

    사용해 보려면 **Settings** → **Media**로 이동하세요. 현재 Dedicated Cloud에서만 제공됩니다.

  * 워크스페이스 또는 섹션의 비디오 패널에서 미디어를 **자동 재생 및 반복 재생**합니다. 현재 Dedicated Cloud에서만 사용할 수 있습니다.

  ## 수정 사항

  * 이제 그룹화된 선형 차트에 최소값과 최대값이 표시되어 값을 비교하기가 더 쉬워졌습니다.
  * Reports에서는 이제 더 깔끔한 표시를 위해 긴 미디어 패널 제목을 잘라서 보여줍니다.

  * 시맨틱 색상을 이산 값에서 연속 값으로 전환할 때 발생하던 버그를 수정했습니다.
  * 전체 화면 모드에서 차트 크기를 조정할 때 화면이 깜박일 수 있었던 버그를 수정했습니다.
  * y축이 `_step`으로 설정되었을 때 렌더링 문제가 발생할 수 있던 버그를 수정했습니다.
  * 미디어 단계 슬라이더가 최대값으로 설정되어 있을 때, 들어오는 최신 데이터를 반영하여 업데이트되지 않던 버그를 수정했습니다.
  * 이미지 스무딩이 활성화된 이미지 패널에 스무딩 설정이 항상 적용되지 않던 버그를 수정했습니다.
  * 그리드 보기에서 미디어 패널이 세로 공간을 과도하게 차지해 스크롤을 지나치게 많이 해야 했던 문제를 수정했습니다.
  * 자동화를 렌더링하거나 새로 고침할 때 발생하던 사소한 표시 문제를 수정했습니다.
  * 아티팩트를 연결할 때 스크롤 및 개수 표시 문제를 포함한 사소한 UI 버그를 수정했습니다.
  * HTML 테이블 미디어 패널에서 ASCII가 아닌 문자가 표시되지 않던 버그를 수정했습니다.
  * **Edit run name** 액션을 업데이트해, 워크스페이스에서 run 표시 이름만 변경하는 것인지, 아니면 프로젝트 전체에서 사용되는 기본 run 이름을 변경하는 것인지가 더 명확하게 구분되도록 했습니다.

  ## 기능

  * **레지스트리 액세스 제어 업데이트:**
    * **프로그래밍 방식 Registry 액세스 제어:** [Registry 역할](/ko/models/registry/configure_registry#details-about-registry-roles)이 이제 SCIM 스키마에 포함되었습니다. [사용자를 생성](/ko/platform/hosting/iam/scim#create-user)하거나 업데이트할 때 해당 사용자의 Registry 역할을 설정할 수 있으며, [사용자를 조회](/ko/platform/hosting/iam/scim#get-user)하면 Registry 역할이 함께 반환됩니다.
    * **Registry 액세스 제한:** Registry에 대한 전체 액세스를 부여하고 싶지 않은 경우, 이제 새로운 Registry 역할인 [Restricted Viewer](/ko/models/registry/configure_registry)를 부여할 수 있습니다. 이 역할은 컬렉션, 자동화 또는 기타 Registry 리소스를 생성·업데이트·삭제할 수 있는 권한 없이 Registry 아티팩트에 대한 읽기 전용 액세스를 제공합니다. 이는 민감하거나 규제가 적용되는 Registry 콘텐츠에 유용합니다.
    * [**확장된 Registry 액세스:**](https://wandb.ai/wandb_fc/product-announcements-fc/reports/Registry-for-W-B-Models-viewers-Faster-discovery-cleaner-handoffs--VmlldzoxNDc4MzM5Mg) 이제 Models **Viewer** 역할을 가진 모든 사용자는 기본적으로 전체 Registry **Member**이기도 하므로, 별도 설정 없이도 모델과 아티팩트를 탐색하고, 비교하고, 사용할 수 있습니다. Models **Viewer**는 **Restricted Viewer**, **Viewer**, **Admin**과 같은 다른 Registry 역할로 지정할 수 있습니다. 이를 통해 조직 내 모든 사람이 Registry의 모든 가치를 활용할 수 있습니다.

  * **즐겨찾는 레지스트리:** Registry를 더 크게 확장해 사용할 때 쉽게 탐색할 수 있도록, 이제 레지스트리를 즐겨찾기에 추가할 수 있습니다. Registry 랜딩 페이지에서 레지스트리 카드 위에 마우스를 올린 다음 별 윤곽선 아이콘을 클릭하세요. 개별 레지스트리 페이지에서는 페이지 상단의 별 윤곽선 아이콘을 클릭하세요. 즐겨찾는 레지스트리는 Registry 랜딩 페이지 상단에 알파벳순으로 표시됩니다.

    ![즐겨찾는 레지스트리](/images/release-notes/v0-75-0/starred-registries.png)

  * **자동화 실행 기록:** **Automation history** 탭에서 프로젝트 또는 레지스트리 [Automations](/ko/models/automations/automation-events)의 실행 기록을 확인할 수 있습니다. 모든 자동화 실행은 상태와 주요 세부 정보를 한눈에 볼 수 있도록 최신순으로 표시됩니다. 자동화를 이해하고 디버깅할 수 있도록, 상세 보기에서 해당 실행의 조건, 작업, 결과를 확인할 수 있습니다.

  * **Organization 대시보드**에서 이제 쉼표로 구분된 이메일 주소 목록을 입력해 여러 사용자를 한 번에 조직에 초대할 수 있으며, 초대한 사용자들을 동시에 Teams에 추가할 수도 있습니다.

    * run에 대한 시맨틱 범례에서 이제 하이퍼파라미터와 같은 config 값에 따라 플롯 색상을 지정할 수 있습니다. 학습률, 배치 크기, 모델 패밀리와 같이 선택한 하이퍼파라미터를 기준으로 run의 색상을 설정해, config 기반 효과를 시각적으로 파악할 수 있습니다.

      ![시맨틱 범례 config 색상 지정](/images/release-notes/v0-75-0/semantic-legend-config-coloring.gif)

  * 새로운 전체 화면 이미지 브라우저를 사용하면 로깅된 이미지를 살펴보고 비교하는 작업을 더 쉽게 할 수 있습니다. 이전에는 **Dedicated Cloud**에서만 사용할 수 있었지만 이제는 **Self-Managed**에서도 사용할 수 있습니다.

    * **전체 화면 보기에서의 단계 슬라이더:** 전체 화면에 내장된 슬라이더로 단계를 효율적으로 이동하며 출력이 어떻게 변하는지 확인할 수 있습니다.

  * 이제 워크스페이스 전체 또는 섹션에 있는 모든 플롯에 대해 **Show original** 선형 플롯 설정을 켜거나 끌 수 있습니다.

  * 스무딩을 켰을 때 시스템 메트릭 차트가 로드되지 않을 수 있는 버그를 수정했습니다.

  ## 기능

  * <a id="registry_ga_announcement" aria-label="W&B Registry GA announcement" /> 우리는 향후 몇 차례의 릴리스에 걸쳐 기존 팀 수준 Model Registry를 단계적으로 W&amp;B Registry로 마이그레이션할 예정입니다. **사용자 측에서 별도로 취할 조치는 없습니다. W&amp;B가 각 팀의 Model Registry 자산을 해당 팀에 대해 자동으로 생성되는 팀 수준 비공개 Registry로 복사합니다.** 기존 팀 수준 Model Registry에서 사용되던 모델 컬렉션 참조는 계속해서 정상적으로 동작합니다.

    1. Server v0.71(이번 릴리스)에서는 이제 모든 조직에서 Registry가 기본적으로 활성화됩니다.
    2. Server v0.73(8월)에서는 업그레이드 중 이 마이그레이션을 자동으로 실행하도록 하는 옵트아웃 환경 변수가 포함됩니다. 마이그레이션은 대부분의 조직에서 5분에서 30분 정도 소요됩니다. 마이그레이션 중에는 아티팩트를 이전 팀의 Model Registry나 새로운 Registry에 연결하는 작업이 오류와 함께 실패합니다. **아티팩트 로깅과 조회에는 영향이 없습니다**.
    3. Server v0.74(9월)에서는 이 플래그를 제거하고, 마이그레이션은 업그레이드 중에 자동으로 수행됩니다.

    더 강력한 새로운 경험으로 최대한 매끄럽고 불편을 최소화하는 업그레이드가 되도록 최적화하고 있습니다. 마이그레이션에 관해 궁금한 점이 있으면 W&amp;B 담당자에게 문의하세요.
  * 이제 아티팩트에 태그를 추가하면 자동화를 트리거할 수 있습니다. [Artifact events](/ko/models/automations/automation-events/#artifact-events)를 참조하세요.
  * 패널을 전체 화면 모드에서 볼 때 섹션 내의 다른 패널로 이동하려면 **Previous** 및 **Next** 버튼이나 왼쪽과 오른쪽 화살표 키를 사용하여 이전 또는 다음 패널을 볼 수 있습니다.
  * 미디어 패널을 전체 화면으로 볼 때 단계 슬라이더를 이동하려면, macOS에서는 **CMD + 왼쪽 또는 오른쪽 화살표** 키를, Windows / Linux에서는 **Ctrl + 왼쪽 또는 오른쪽 화살표** 키를 사용하세요.
  * 이제 동일한 step 슬라이더 키를 사용하는 미디어 패널을 서로 동기화할 수 있습니다. 한 미디어 패널에서 step 슬라이더를 이동하면 다른 패널의 step 슬라이더도 함께 이동합니다. 워크스페이스 또는 섹션 설정을 열고 **Media**로 이동한 다음 **Sync** 탭을 클릭하고 **키별 슬라이더 동기화 (Step)**를 켭니다.
  * Run Comparer에 run의 명령, Python 버전, GPU 유형 등의 메타데이터를 포함하는 새로운 **Meta** 섹션이 추가되었습니다.

  ### Weave

  * **Trace plots** 도구를 사용하면 사용자 정의 대화형 차트를 통해 시간 경과에 따른 지연(latency), 비용(cost), 토큰 수와 같은 트레이스 수준 메트릭을 탐색하고 시각화하며 디버깅할 수 있습니다. [자세히 알아보기](/ko/weave/guides/tracking/trace-plots).
  * **온라인 평가(Online Evaluations)**: Dedicated Cloud에서 수집되는 trace에 Monitors를 연결해 trace를 모니터링하세요. Monitors는 LLM 심사자 역할을 하며 백그라운드에서 동작하면서, 지정한 일부 호출에 대해 점수를 매깁니다. Monitors를 사용해 운영 환경 동작을 추적하고, 성능 회귀(regression)를 감지하며, 실제 운영 데이터를 수집하는 등 다양한 작업을 수행할 수 있습니다. [자세히 보기](/ko/weave/guides/tracking/trace-plots).

  * Weave [tracing](/ko/weave/guides/tracking)은 이제 CrewAI, OpenAI의 Agent SDK, DSPy 2.x, Google의 genai Python SDK와의 새로운 프레임워크 인테그레이션을 지원합니다.
  * Playground는 새로운 [OpenAI 모델](/ko/weave/guides/tools/playground/#openai)인 GPT‑4.1, GPT‑4.1 mini 및 GPT‑4.1 nano를 지원합니다.
  * 트레이스에서 직접 레이블이 지정된 데이터셋을 생성하면, 작성한 주석이 자동으로 데이터셋 열로 변환됩니다. [트레이스에서 데이터셋 생성](/ko/weave/guides/core-types/datasets/#create-edit-and-delete-a-dataset-in-the-ui)을 참조하세요.

  ## 보안

  * 이제 Registry 관리자는 레지스트리에서 [service account](/ko/platform/hosting/iam/service-accounts)를 Registry Admin 또는 Member로 지정할 수 있습니다. 이전에는 service account의 역할이 항상 Registry Admin이었습니다. 자세한 내용은 [Registry service account 설정](/ko/models/registry/configure_registry)을 참조하세요.

  ## 성능

  * 워크스페이스 내 다양한 상호작용의 성능을 개선했습니다. 특히 대규모 워크스페이스에서 효과가 큽니다. 예를 들어 섹션을 펼치거나 run 선택기를 사용할 때 반응 속도가 눈에 띄게 빨라졌습니다.
  * Fork 및 Rewind 성능을 개선했습니다.

    [포크](/ko/models/runs/forking)하면 기존 run과 동일한 설정을 사용하는 새 run이 생성됩니다. 포크된 run에 대한 변경 사항은 상위 run에 영향을 주지 않으며, 그 반대도 마찬가지입니다. 포크된 run과 상위 run 사이에는 포인터가 유지됩니다. run을 [되감기](/ko/models/runs/rewind)하면 기존 데이터를 잃지 않고 해당 시점부터 새로운 데이터를 로깅할 수 있습니다.

    포크가 여러 단계로 중첩된 프로젝트에서 캐시가 개선되어, 이제 새로운 run을 포크하는 작업이 훨씬 더 효율적입니다.
