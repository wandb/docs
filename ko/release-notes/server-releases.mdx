<Update label="v0.76.3" description="2025년 12월 17일">
**2025년 12월 17일**

## Fixes
- macOS에서 스크롤 바가 항상 표시되도록 설정된 경우 발생할 수 있는 디스플레이 버그를 수정했습니다. Workspace에 [Query panel](/models/app/features/panels/query-panels)이 있는 경우, Workspace에서 축소(zooming out)하면 Query panel 테이블 크기가 초당 여러 번 진동하는 리사이즈 루프가 발생할 수 있던 문제를 해결했습니다.
- Dedicated Cloud UI에서 W&B Inference가 잘못 나타나던 버그를 수정했습니다. Inference는 Dedicated Cloud에서 사용할 수 없습니다.
</Update>

<Update label="v0.76.2" description="2025년 12월 10일">
**2025년 12월 10일**

## Fixes

<Update label="v0.76.1" description="2025년 12월 4일">
**2025년 12월 4일**

## Fixes
- Workspace에서 Report로 라인 플롯을 가져올 때, 플롯에 극단적인 이상치(outliers)가 숨겨져 있으면 프로세스가 멈출 수 있는 버그를 수정했습니다.
- 테이블 패널의 높이를 줄일 때 크기가 올바르게 조정되지 않던 버그를 수정했습니다.
- 캡션이 있는 이미지에서 마스크 오버레이가 이미지를 벗어날 수 있는 버그를 수정했습니다.
- Run note의 편집 상자 크기가 너무 좁아 편집이 불편했던 버그를 수정했습니다. 이제 편집 상자는 표시 모드의 텍스트와 동일한 크기를 사용합니다.
- Organization Dashboard에서 **Service Accounts** 탭을 클릭할 때 페이지가 충돌할 수 있는 버그를 수정했습니다.
- Run 로그를 보다가 다른 run으로 전환할 때 잘못된 run의 로그가 표시될 수 있는 캐싱 버그를 수정했습니다.
- 클라이언트가 GraphQL 작업을 취소할 때, 이제 `5xx` HTTP 서버 오류가 아닌 `4xx` HTTP 클라이언트 오류로 올바르게 보고됩니다.
</Update>

<Update label="v0.76.0" description="2025년 11월 20일">
**2025년 11월 20일**

W&B v0.76은 더 스마트해진 미디어 툴, 매끄러운 워크플로우, 그리고 강력한 새 인터페이스를 도입하여 실험 인사이트와 생산성을 크게 향상시킵니다. 자동 재생 및 반복 재생을 지원하는 멀티 비디오 동기화 기능은 빠른 병렬 평가를 가능하게 하며, 간소화된 Workspace 탐색을 통해 뷰 사이를 손쉽게 이동할 수 있습니다. 또한, 새로운 LEET 터미널 UI는 실시간 run 인사이트를 커맨드라인에서 직접 빠르게 제공합니다.

## 지원 및 서비스 종료 (End of Life)

- W&B Server v0.63 이하 버전은 2025년 11월 20일에 서비스가 종료되었습니다.
- W&B Server v0.65는 2026년 1월 30일에 서비스 종료가 예정되어 있습니다.

## Features

- **Multi-video Sync**: 여러 비디오 패널을 함께 재생하고 제어하여 트레이닝 Runs의 정성적 평가 및 비교, 실험 간 모델 행동 분석, 멀티모달 모델 체크포인트 평가를 수행할 수 있습니다.
    
    동기화된 비디오 재생을 통해 여러 비디오를 동시에 재생, 일시 정지, 탐색(scrub)하고 속도를 조절할 수 있어, 스타일, 타이밍, 움직임 등을 평가하기 위한 시각적 QA를 위해 진정한 병렬 비교가 가능합니다.
    
    <Frame>
    ![비디오 재생 동기화 데모 애니메이션](/images/release-notes/v0-76-0/video_playback_sync_demo.gif)
    </Frame>
    
    기능을 사용하려면 **Settings** → **Media**로 이동하세요. 현재 Dedicated Cloud에서만 사용할 수 있습니다.
    
- Workspace 또는 섹션의 비디오 패널에서 미디어를 **자동 재생 및 반복 재생**할 수 있습니다. 현재 Dedicated Cloud에서만 사용할 수 있습니다.
- 이제 Dedicated Cloud에서 Workspace 탐색 업데이트를 통해 멀티 run 및 싱글 run Workspace 간의 이동이 간소화되었으며, 두 뷰 사이에서 변경된 뷰 설정이 유지되도록 보장합니다 ([빠른 데모 비디오](https://www.loom.com/share/df232057aa2d4ee78026a4f9cf400499?sid=da3661eb-e8cd-4aa5-a666-db2c4091b02b)).
    - 싱글 run 및 멀티 run 저장된 뷰(saved views)는 이제 공유된 설정 상세 정보를 사용하여 반복적인 설정 작업을 줄여줍니다.
    - 기존의 싱글 run 저장된 뷰는 제거되며, 필요한 뷰는 다시 생성해야 합니다.

- [W&B Lightweight Experiment Exploration Tool (LEET)](/models/ref/cli/wandb-beta/wandb-beta-leet) (Beta)는 통계, 메트릭, 시스템 상태를 포함하여 ML 트레이닝 Runs를 관찰하기 위한 빠른 터미널 UI입니다. 라이브 로그 파일을 읽고 시각화하여 실시간으로 빠르고 사용자 정의 가능한 브라우저 없는 경험을 제공합니다.

## Fixes

- 비교를 돕기 위해 그룹화된 라인 차트에 최소값과 최대값이 표시됩니다.
- Reports에서 더 깔끔한 프레젠테이션을 위해 긴 미디어 패널 제목을 자릅니다(truncate).
- [Semantic legends](/models/runs/color-code-runs#set-a-configuration-key)에서 이제 버킷의 상한값을 편집할 수 있습니다.
- Semantic colors를 이산형(discrete)에서 연속형(continuous) 값으로 전환할 때 발생하던 버그를 수정했습니다.
- 전체 화면 모드에서 차트 크기를 조정할 때 깜빡임이 발생할 수 있는 버그를 수정했습니다.
- y축이 `_step`으로 설정된 경우 렌더링 문제가 발생할 수 있는 버그를 수정했습니다.
- 미디어 스텝 슬라이더가 최대값으로 설정된 경우 최신 수신 데이터를 보여주기 위해 업데이트되지 않던 버그를 수정했습니다.
- 이미지 스무딩 설정이 켜져 있는 이미지 패널에 항상 적용되지 않던 버그를 수정했습니다.
- 그리드 뷰의 미디어 패널이 세로 공간을 너무 많이 차지하여 과도한 스크롤을 유발하던 버그를 수정했습니다.
- 자동화(automations) 렌더링 및 새로 고침 시의 사소한 디스플레이 문제를 수정했습니다.
- 아티팩트를 연결할 때 스크롤 및 개수 렌더링 문제를 포함한 사소한 UI 버그를 수정했습니다.
- HTML 테이블 미디어 패널에서 비 ASCII 문자가 표시되지 않던 버그를 수정했습니다.
- Workspace용 run 표시 이름을 업데이트하는 것인지, 프로젝트 전체의 기본 run 이름을 업데이트하는 것인지 더 명확하게 알 수 있도록 **Edit run name** 작업을 업데이트했습니다.
</Update>

<Update label="v0.75.0" description="2025년 10월 28일">
W&B v0.75는 새로운 Restricted Viewer 역할, 레지스트리 즐겨찾기(star) 기능, SCIM을 통한 프로그래밍 방식의 Registry 엑세스 제어로 Registry를 강화합니다. 새로운 자동화 실행 이력 뷰는 자동화 과정을 이해하고 디버깅하는 데 도움을 줍니다. 하이퍼파라미터와 같은 설정 속성에 따른 semantic coloring으로 설정 기반 효과를 시각화하고, 시스템 메트릭에 대한 고정밀(full-fidelity) 플롯 등을 확인해 보세요.

## 지원 및 서비스 종료 (End of Life)
- W&B Server v0.61 이하 버전은 2025년 10월 28일에 서비스가 종료되었습니다.

[Release policies and processes](/release-notes/release-policies)를 참조하세요. 도움이나 질문이 필요하시면 [support팀](mailto:support@wandb.com)에 문의해 주세요.

## 예정된 변경 사항
다음 W&B Server 릴리스에서는 멀티 run 및 싱글 run Workspace 간의 이동을 더 쉽게 만드는 Workspace 탐색 업데이트가 적용될 예정입니다. [미리보기 비디오](https://www.loom.com/share/df232057aa2d4ee78026a4f9cf400499?sid=da3661eb-e8cd-4aa5-a666-db2c4091b02b)를 확인하세요.

이 변경 사항의 일부는 Server v0.75에 포함되어 있습니다. 왼쪽 사이드바에 현재 **Project**를 보고 있는지 싱글 **Run**을 보고 있는지 표시됩니다. 다가오는 이 탐색 변경 사항은 싱글 run 및 멀티 run 저장된 뷰의 통합과 함께 진행되므로, 한쪽에서의 플롯이나 레이아웃 변경이 다른 쪽에도 반영되어 반복적인 설정을 줄여줍니다. 이 변경의 일환으로 기존의 싱글 run 저장된 뷰는 사라지게 되며, 필요한 뷰는 다시 생성해야 합니다.

![예정된 탐색 사이드바 업데이트](/images/release-notes/v0-75-0/navigation-sidebar-upcoming-update.png)

## Security
- 환경 변수 `FEDERATED_AUTH_AUDIENCES`를 쉼표로 구분된 audience 목록으로 설정하여 [federated identity](/platform/hosting/iam/identity_federation) audience를 구성할 수 있습니다. audience는 ID 토큰의 허용 가능한 수신자를 지정합니다.

## Features
- **Registry 엑세스 제어 업데이트:**
    - **프로그래밍 방식의 registry 엑세스 제어:** [Registry roles](/models/registry/configure_registry#details-about-registry-roles)가 이제 SCIM 스키마에 포함되었습니다. 사용자를 [생성](/platform/hosting/iam/scim#create-user)하거나 업데이트할 때 Registry 역할을 구성할 수 있으며, 사용자를 [조회](/platform/hosting/iam/scim#get-user)할 때 Registry 역할이 반환됩니다.
    - **Registry 엑세스 제한:** 전체 Registry 엑세스 권한을 부여하고 싶지 않은 경우, 새로운 Registry 역할인 [Restricted Viewer](/models/registry/configure_registry)를 부여할 수 있습니다. 이 역할은 컬렉션, 자동화 또는 기타 레지스트리 리소스를 생성, 업데이트 또는 삭제할 권한 없이 레지스트리 아티팩트에 대한 읽기 전용 엑세스를 제공합니다. 이는 민감하거나 규제 대상인 레지스트리 콘텐츠에 유용합니다.
    - [**확장된 Registry 엑세스:**](https://wandb.ai/wandb_fc/product-announcements-fc/reports/Registry-for-W-B-Models-viewers-Faster-discovery-cleaner-handoffs--VmlldzoxNDc4MzM5Mg) 이제 Models **Viewer** 역할을 가진 모든 사용자는 기본적으로 전체 Registry **Member**가 되어 모델과 아티팩트를 탐색, 비교 및 사용할 수 있습니다. Models **Viewer**에게 **Restricted Viewer**, **Viewer**, 또는 **Admin**과 같은 다른 Registry 역할을 할당할 수도 있습니다. 이를 통해 조직의 모든 사용자가 Registry의 가치를 완전히 누릴 수 있습니다.
- **Starred registries:** 규모가 커짐에 따라 Registry를 더 쉽게 탐색할 수 있도록 레지스트리에 별표(star)를 표시할 수 있습니다. Registry 랜딩 페이지에서 레지스트리 카드 위에 마우스를 올린 다음 별표 윤곽선을 클릭하세요. 개별 레지스트리 페이지에서는 페이지 상단의 별표 윤곽선을 클릭하세요. 즐겨찾기한 레지스트리는 Registry 랜딩 페이지 상단에 알파벳 순으로 표시됩니다.
    
    ![즐겨찾기된 레지스트리](/images/release-notes/v0-75-0/starred-registries.png)
    
- **Automation execution history:** **Automation history** 탭에서 프로젝트 또는 레지스트리 [Automations](/models/automations/automation-events)의 실행 이력을 확인하세요. 모든 자동화 Runs가 역순으로 표시되며 상태와 주요 세부 정보를 한눈에 볼 수 있습니다. 자동화 과정을 이해하고 디버깅할 수 있도록 실행 조건, 액션 및 결과를 상세 뷰에서 보여줍니다.

- 새로운 편의 메소드 [`delete_all_object_versions`](/weave/reference/python-sdk/trace/weave_client)가 추가되었습니다.

UI 개선 사항으로는 색약 친화적인 run 색상 팔레트, 메트릭 기반의 run 색상 지정을 위한 연속 색상 옵션, 더 부드러운 패널 재구성이 포함됩니다. 또한 이번 릴리스는 CoreWeave AI Object Storage를 사용하는 BYOB(Bring Your Own Bucket) 설정을 위한 서비스 주체(service principal) 인증과 개선된 콘솔 로그 처리 및 추가 향상 기능을 제공합니다.

## 예정된 변경 사항
- 향후 릴리스에서는 멀티 run 및 싱글 run Workspace 뷰를 통합할 예정입니다. 따라서 싱글 run으로 드릴다운하면 멀티 run 뷰에서 구성한 것과 동일한 패널 레이아웃 및 설정(커스텀 차트, 레이아웃 등 포함)을 보게 됩니다. 이는 컨텍스트를 유지하고 뷰 구성 시 중복 작업을 피하기 위함입니다. 그러나 이로 인해 나머지 Workspace와 분리된 고유한 싱글 run 뷰를 커스터마이징하는 기능은 제거됩니다. 마이그레이션에 관한 질문은 W&B 팀이나 [support팀](mailto:support@wandb.ai)에 문의하세요.
- 향후 여러 릴리스에 걸쳐 기존 팀 레벨 Model Registry를 W&B Registry로 계속 마이그레이션할 예정입니다. 상세 내용 및 타임라인에 대한 안내는 아래의 [W&B Registry GA 공지](#registry_ga_announcement)를 참조하세요.

## 알려진 문제
v0.73.0에서 Workspace 패널의 크기를 조정하거나 순서를 변경하는 작업이 저장된 뷰에 반영되지 않을 수 있습니다. 이 버그는 v0.73.1에서 수정되었습니다. 이 문제를 겪고 계신다면 수정 사항이 포함된 v0.73.1로 업그레이드하세요. 업그레이드가 불가능한 경우 [support팀](mailto:support@wandb.ai)에 우회 방법을 문의하세요.

## Features
- <a id="registry_ga_announcement"></a>향후 여러 릴리스에 걸쳐 기존 팀 레벨 Model Registry를 W&B Registry로 단계적으로 마이그레이션합니다. **사용자가 별도로 취해야 할 조치는 없습니다. W&B는 각 팀의 Model Registry 에셋을 자동으로 생성된 해당 팀 레벨 프라이빗 레지스트리로 자동 복사합니다.** 기존의 이전 팀 레벨 Model Registry에 있는 모델 컬렉션 참조는 계속 작동합니다.
    1. Server v0.73(이번 릴리스)에는 업그레이드 중 팀의 Model Registry를 자동으로 마이그레이션할 옵트아웃 환경 변수가 포함되어 있습니다. 마이그레이션은 대부분의 조직에서 5분에서 30분 정도 소요됩니다. 마이그레이션 중에 이전 Model Registry나 새 Registry로 아티팩트를 연결하려고 하면 오류가 발생할 수 있습니다. **아티팩트 로깅 및 검색은 영향을 받지 않습니다.**
    2. Server v0.74(9월 말)에서는 옵트아웃 플래그를 제거하고 업그레이드 중에 마이그레이션이 진행됩니다.

    W&B는 강력한 새로운 경험으로의 원활한 업그레이드를 위해 최선을 다하고 있습니다. 마이그레이션에 대해 궁금한 점이 있으면 W&B 계정 담당 팀이나 [support팀](mailto:support@wandb.ai)에 문의하세요.
- [Run Logs 콘솔 페이지](/models/app/console-logs/)에 복사 버튼을 추가하고 붙여넣기 동작을 개선했습니다.
- W&B는 이제 runs를 위한 두 가지 새로운 색약 친화적인 색상 팔레트를 제공합니다. [Workspace settings](/models/app/features/cascade-settings/#workspace-settings)의 **Runs** 섹션에서 이를 설정하세요.
- [메트릭 또는 숫자형 설정으로 run 색상을 지정](/models/runs/color-code-runs/)할 때, 이제 다섯 가지 새로운 연속 색상 팔레트 중에서 선택할 수 있습니다.
- [W&B run](/models/runs/) 중에 [CoreWeave Mission Control](https://www.coreweave.com/mission-control)은 컴퓨팅 인프라를 모니터링합니다. 이 인테그레이션은 이제 Dedicated Cloud에서 사용할 수 있습니다.
- CoreWeave AI Object Storage를 사용한 BYOB는 이제 Multi-tenant Cloud, Dedicated Cloud 및 Self-Managed 배포에서 서비스 주체 인증을 지원합니다. 이를 통해 배포 시 정적 자격 증명을 사용하는 것을 피할 수 있습니다. [Bring your own bucket (BYOB)](/platform/hosting/data-security/secure-storage-connector/)에서 자세히 알아보세요.

### Weave
- 이제 [Weave Threads](/weave/guides/tracking/threads)를 사용하여 여러 traces에 걸친 세션이나 대화를 그룹화할 수 있습니다.
- **Trace view**의 새로운 [Graph view](/weave/guides/tracking/trace-tree#graph-view)는 호출 트리 구조를 그래프로 시각화합니다.
- **Trace view**는 ChromaDB로 검색된 [documents](/weave/guides/core-types/media#documents)를 위한 새로운 전용 뷰를 포함하며, Langchain은 채팅과 같은 스키마 처리를 개선합니다. **Trace view**에서 문서를 클릭하면 오른쪽 서랍(drawer)에서 상세 뷰가 열립니다. [Navigate the trace view](/weave/guides/tracking/trace-tree)에서 자세히 알아보세요.
- 이제 미디어 유형으로 [HTML](/weave/guides/core-types/media#html)을 trace에 첨부할 수 있습니다.
- **Assets** 탭에서 이제 프롬프트를 생성하고 편집할 수 있습니다. **Assets**를 클릭한 다음 탐색 창에서 **Prompts**를 클릭하세요. **New prompt**를 클릭하거나 기존 프롬프트의 이름을 클릭하여 세부 정보를 확인하세요. 거기서 연필 아이콘을 클릭하여 프롬프트를 편집하고 다시 게시할 수 있습니다.

## Performance
- 특히 대규모 Workspace에서 많은 Workspace 상호작용의 성능을 개선했습니다. 예를 들어, 섹션 확장 및 run 선택기 사용 시 반응성이 훨씬 좋아졌습니다.
- Fork 및 Rewind 성능을 개선했습니다.

    run을 [Forking](/models/runs/forking/)하면 기존 run과 동일한 설정을 사용하는 새로운 run이 생성됩니다. 포크된 run의 변경 사항은 부모 run에 영향을 주지 않으며 그 반대도 마찬가지입니다. 포크된 run과 부모 사이에는 포인터가 유지됩니다. run을 [Rewinding](/models/runs/rewind/)하면 기존 데이터를 잃지 않고 해당 시점부터 새로운 데이터를 로깅할 수 있습니다.

    중첩된 포크가 많은 프로젝트에서 캐싱 개선으로 인해 새로운 run을 포크하는 작업이 훨씬 더 효율적이 되었습니다.
