<Update label="v0.77.1" description="2026년 1월 28일">
  **2026년 1월 28일**

  ## 변경 사항

  * 이제 **Self-Managed** 환경에서도 W&amp;B 워크스페이스에 Weave 패널을 추가하고, Weave 트레이스에서 W&amp;B Artifact를 참조할 수 있습니다. 이 기능은 이전에는 **Dedicated Cloud**에서만 사용할 수 있었습니다. 자세한 내용은 [W&amp;B 학습 실행과 함께 Weave 사용하기](/ko/weave/guides/tools/weave-in-workspaces)를 참조하세요.

  ## 수정 사항

  * alias 필터가 있을 때 레지스트리 버전 쿼리가 중복된 결과를 반환할 수 있던 버그를 수정했습니다.
</Update>

<Update label="v0.77.0" description="2026년 1월 21일">
  **2026년 1월 21일**

  W&amp;B 0.77은 조직의 보안과 생산성을 강화하기 위한 보안 및 사용성 개선 사항을 도입합니다.

  이번 릴리스에는 API 키 보안 강화, 단일 실행 및 다중 실행 워크스페이스 간 공유 구성을 통한 간소화된 탐색, 새로운 선 그래프 음영 옵션, 미디어 패널 동기화 및 루핑의 정식 출시(GA), 주요 편의성 개선 및 버그 수정이 포함되어 있습니다.

  ## 지원 및 서비스 종료

  * W&amp;B Server v0.65 및 그 이전 버전은 2026년 1월 30일자로 지원이 종료(EOL)됩니다.

  [릴리스 정책 및 프로세스](/ko/release-notes/release-policies)를 참조하세요. 도움이 필요하거나 문의 사항이 있으시면 [지원팀](mailto:support@wandb.com)에 문의하세요.

  ## 예정된 변경 사항

  * 곧 Registry UI가 더 빠르게 탐색되고 한눈에 파악하기 쉬워질 예정입니다.

    * 향상된 자동 완성 검색, 새로 디자인된 Registry 및 Collection 카드, 그리고 새로운 Registry 아이콘을 통해 자산을 시각적으로 더 쉽게 구분할 수 있습니다. 초기에는 스마트 랭킹이 적용되며, 곧 AI 기반 랭킹이 추가될 예정입니다.
    * Registry 아이콘은 처음에는 자동으로 생성되며, 이후 언제든지 Registry 관리자가 사용자 지정할 수 있습니다.

    이러한 업데이트는 다음 릴리스에 포함될 예정입니다. 자세한 내용은 [지원팀](mailto:support@wandb.com) 또는 담당 AISE에게 문의하세요.

  ## 보안

  * W&amp;B 전체에서 API 키를 업데이트하여 개인 계정과 서비스 계정 모두의 보안과 키 관리를 개선했습니다. 이러한 변경 사항은 최신 보안 모범 사례를 따르면서 팀이 자동화를 더 유연하고 안전하게 구현할 수 있도록 합니다.

    **알아두어야 할 핵심 사항**

    * API 키는 한 번만 표시되며, 생성 후 즉시 복사해 저장해야 합니다.
    * 기존 키는 더 이상 복사할 수 없으며, 새 키를 생성해야 합니다.
    * 언제든지 키의 이름을 변경하거나 삭제할 수 있습니다.
    * 서비스 계정은 이제 여러 개의 API 키를 지원합니다.

    [회원가입 및 API 키 생성](/ko/models/quickstart#sign-up-and-create-an-api-key)을 참조하세요.

  ## 기능

  * **워크스페이스 내비게이션이 업데이트되었습니다.** 다중 실행 및 단일 실행 워크스페이스 뷰를 일관되게 맞췄습니다. 이제 단일 실행으로 들어가면, 다중 실행 뷰에서 구성한 것과 동일한 패널 레이아웃과 구성을 확인할 수 있으며, 여기에 모든 사용자 정의 차트, 레이아웃 및 설정이 포함됩니다. 이를 통해 현재 맥락을 유지하고, 뷰를 다시 구성하느라 생기는 불필요한 반복 작업을 피할 수 있습니다. 변경 사항(내비게이션 버튼 위치 변경 포함)을 이해하려면 이 [동영상](https://www.loom.com/share/df232057aa2d4ee78026a4f9cf400499?sid=da3661eb-e8cd-4aa5-a666-db2c4091b02b)을 참고하세요.
    <Frame>
      ![업데이트된 워크스페이스 내비게이션을 보여주는 스크린샷](/images/release-notes/v0-77-0/workspace-navigation-update.png)
    </Frame>
  * 섹션 수준 [비디오 재생 동기화 및 루프](/ko/models/app/features/panels/media#sync) 기능이 정식으로 제공됩니다. 기본적으로 섹션 내 비디오는 서로 동기화된 상태로 재생됩니다.
  * 그룹화된 실행의 고충실도 선 그래프에서 이제 음영 영역이 **최소/최대값**, **표준 편차**, **표준 오차** 또는 **없음**을 표시하도록 구성할 수 있습니다. [Sampling](/ko/models/app/features/panels/line-plot/sampling)을 참고하세요.

  * 자동화를 더 효율적으로 설정할 수 있도록 이제 앱에서 기준으로 사용할 지표를 추천합니다.

  ### 레지스트리

  * Registry의 [Artifact lineage graphs](/ko/models/artifacts/explore-and-traverse-an-artifact-graph)가 이제 항목을 드래그 앤 드롭하여 재배치하거나 숨길 수 있도록 지원합니다.
  * 아티팩트 버전을 조회할 때 이제 단일 작업으로 새 레지스트리를 생성하고 해당 아티팩트 버전을 레지스트리에 연결할 수 있습니다.

  ### Weave

  * 이제 W&amp;B App의 빈 상태 Weave 온보딩 페이지에 Python 예제 코드와 함께 TypeScript 예제 코드도 포함됩니다.
  * 이제 Weave 호출 로그는 기본적으로 무한 스크롤을 지원합니다. 이전에는 가장 최근 로그 10,000줄만 볼 수 있었습니다.
  * [Evaluation](/ko/weave/guides/core-types/evaluations) 비교 페이지의 막대 차트와 레이더 차트가 이제 크기 조정과 전체 화면 보기 기능을 지원하며, 차트 영역에 마우스를 올리면 세부 정보를 보여 주는 툴팁이 표시됩니다.
  * [Weave Playground](https://wandb.ai/site/playground/)에는 새로운 모델이 포함되어 있습니다:
    * GPT 5.2
    * Grok 4.1
    * Opus 4.5
  * 이제 [Weave Playground](https://wandb.ai/site/playground/)에서 새 프롬프트를 만들고 저장할 수 있습니다.

  ## 수정 사항

  * 실행의 **Notes** 텍스트 필드가 편집할 때 너무 좁게 표시되던 버그를 수정했습니다. 이제 이 텍스트 상자는 보기와 편집할 때 모두 동일한 너비로 표시됩니다.
  * PDF로 내보낸 W&amp;B Report에서 이미지 해상도가 너무 낮게 출력되던 버그를 수정했습니다.
  * Report의 꺾은선형 차트가 실행 선택기 변경에 따라 동기화되어 업데이트되지 않을 수 있었던 버그를 수정했습니다.
  * 라인 플롯에서 이상치를 숨길 수 있는 옵션이 없는데도 Report에 이상치가 표시되던 버그를 수정했습니다.
  * Report에서 여러 runset이 포함된 선 플롯에서, 서로 다른 runset에 속하지만 이름이 같은 실행에 대해 잘못된 데이터가 표시될 수 있던 버그를 수정했습니다. 이제 실행은 runset과 이름의 조합으로 고유하게 식별됩니다.

  ### Weave

  * Dedicated Cloud용 Weave UI에, 해당 환경에서는 사용할 수 없는 W&amp;B Inference 링크가 표시되던 버그를 수정했습니다.
  * details drawer가 열려 있을 때 [Evaluation](/ko/weave/guides/core-types/evaluations) 비교 페이지에서 막대 차트가 지나치게 좁게 조정되던 버그를 수정했습니다.
  * [Weave Playground](https://wandb.ai/site/playground/)에서 텍스트를 입력할 때 커서가 화면에 보이는 영역에서 사라질 수 있던 UI 버그를 수정했습니다.
  * [Weave Playground](https://wandb.ai/site/playground/)에서 호출을 로드할 때 템플릿 변수를 편집할 수 없던 버그를 수정했습니다.
  * [Evaluation](/ko/weave/guides/core-types/evaluations) 개요 페이지에서 하위 트레이스의 상세 보기로 이동할 때 operation 선택기가 설정되지 않던 버그를 수정했습니다. 이로 인해 어떤 트레이스가 선택되어 있는지 파악하기 어려웠고, 일부 열이 비어 있는 것처럼 표시되는 문제가 발생했습니다.
  * [Evaluation](/ko/weave/guides/core-types/evaluations) 비교 보기 페이지에서 **Delete**를 클릭해도 비교 보기가 실제로 삭제되지 않던 버그를 수정했습니다.
  * 커스텀 타입을 가진 [Evaluation](/ko/weave/guides/core-types/evaluations) 결과에서 커스텀 타입 대신 `CellValue` 필드가 빈 값으로 표시될 수 있던 버그를 수정했습니다.
  * **실행을 행으로** 표시하는 모드에서 왼쪽에 고정된 셀들이 투명하게 표시되던 버그를 수정했습니다.
</Update>

<Update label="v0.76.2" description="2025년 12월 10일">
  **2025년 12월 10일**

  ## 수정 사항

  * **Organization Dashboard** 페이지의 부정확한 사용량 카드를 **Usage** 페이지로 연결되는 링크로 교체했습니다. 이는 UI 버그일 뿐이며 과금에는 영향이 없었습니다.
  * Parquet 읽기가 비활성화되어 있지만 내보내기는 활성화되어 있는 상황으로 이어질 수 있던 버그를 수정했습니다.
  * 보류(pending) 상태에서 실행 설정(run config) 없이 실행을 수동으로 upsert할 때 발생하던 nil 포인터 역참조 문제를 수정했습니다.
</Update>

<Update label="v0.76.1" description="2025년 12월 4일">
  **2025년 12월 4일**

  ## 수정 사항

  * 워크스페이스에서 리포트로 선 그래프를 가져올 때, 그래프에서 극단적인 이상치를 숨긴 경우 가져오기가 멈출 수 있던 버그를 수정했습니다.
  * 테이블 패널의 높이를 줄였을 때 올바르게 크기 조정되지 않을 수 있던 버그를 수정했습니다.
  * 마스크 오버레이가 캡션이 있는 이미지 영역을 넘어서 표시될 수 있던 버그를 수정했습니다.
  * 실행 노트의 편집 상자 크기가 편집하기에 너무 좁았던 버그를 수정했습니다. 이제 편집 상자는 표시 모드에서의 텍스트와 동일한 크기를 사용합니다.
  * Organization Dashboard에서 **Service Accounts** 탭을 클릭하면 페이지가 크래시될 수 있던 버그를 수정했습니다.
  * 실행 로그를 보고 있는 동안 한 실행에서 다른 실행으로 전환할 때, 캐싱 버그로 인해 잘못된 실행의 로그가 표시될 수 있던 문제를 수정했습니다.
  * 클라이언트가 GraphQL 작업을 취소하는 경우, 이제 `5xx` HTTP 서버 오류가 아니라 `4xx` HTTP 클라이언트 오류로 올바르게 보고됩니다.
</Update>

<Update label="v0.76.0" description="2025년 11월 20일">
  **2025년 11월 20일**

  W&amp;B v0.76은 실험 인사이트와 생산성을 크게 향상시키며, 더 스마트한 미디어 도구, 원활한 워크플로우, 강력한 새 인터페이스를 도입합니다. 자동 재생 및 반복 기능을 갖춘 멀티 비디오 동기화로 빠른 나란히 비교 평가가 가능하며, 간소화된 워크스페이스 탐색으로 뷰 간 이동이 더욱 편리해집니다. 또한, 새로운 LEET 터미널 UI를 통해 실행 인사이트를 명령줄에서 빠르고 실시간으로 확인할 수 있습니다.

  ## 지원 및 서비스 종료

  * W&amp;B Server v0.63 이하 버전의 지원이 2025년 11월 20일부로 종료되었습니다.
  * W&amp;B Server v0.65는 2026년 1월 30일부로 지원이 종료될 예정입니다.

  ## 기능

  * **Multi-video Sync**: 학습 실행 간 정성적 평가와 비교, 실험 전반에 걸친 모델 동작 분석, 멀티모달 모델 체크포인트 평가를 위해 여러 비디오 패널을 동시에 재생하고 제어합니다.

    동기화된 비디오 재생을 사용하면 여러 비디오를 재생, 일시 정지, 탐색하고 재생 속도를 조절하는 동작을 서로 동기화된 상태로 수행할 수 있어, 스타일, 타이밍, 모션 등 다양한 요소를 빠르게 시각적으로 품질 검토(QA)하고 평가할 수 있는 진정한 나란히 비교가 가능합니다.

  ## 수정 사항

  * 이제 그룹화된 선형 차트에 최소값과 최대값이 함께 표시되어 값을 비교하기가 더 쉬워졌습니다.
  * 이제 Reports에서 긴 미디어 패널 제목을 잘라내 더 깔끔하게 표시합니다.
  * [시맨틱 범례](/ko/models/runs/color-code-runs#set-a-configuration-key)에서 이제 버킷의 상한을 수정할 수 있습니다.
  * 시맨틱 색상을 이산 값에서 연속형 값으로 전환할 때 발생하던 버그를 수정했습니다.
  * 전체 화면 모드에서 차트 크기를 조정할 때 발생할 수 있던 깜빡임 버그를 수정했습니다.
  * y축이 `_step`으로 설정된 경우 렌더링 문제가 발생할 수 있던 버그를 수정했습니다.
  * 미디어 스텝 슬라이더가 최대값으로 설정된 경우, 새로 들어오는 최신 데이터를 표시하도록 슬라이더가 업데이트되지 않을 수 있던 버그를 수정했습니다.
  * 이미지 스무딩이 켜져 있는 이미지 패널에 스무딩 설정이 항상 적용되지 않던 문제를 수정했습니다.
  * 그리드 보기에서 미디어 패널이 세로 공간을 지나치게 차지하여 불필요한 스크롤이 발생하던 버그를 수정했습니다.
  * 자동화를 렌더링하고 새로고침할 때 발생하던 사소한 표시 문제를 수정했습니다.
  * 아티팩트 연결 시 스크롤 및 개수 표시 문제를 포함한 사소한 UI 버그를 수정했습니다.
  * HTML 테이블 미디어 패널에서 비 ASCII 문자가 표시되지 않던 문제를 수정했습니다.
  * **실행 이름 편집** 작업을 업데이트하여 워크스페이스에 표시되는 실행의 이름을 수정하는 것인지, 아니면 프로젝트 전반에 적용되는 기본 실행 이름을 수정하는 것인지 더 명확하게 구분할 수 있도록 했습니다.
</Update>

  ## 기능

  * **Registry 접근 제어 업데이트:**
    * **프로그래밍 방식 Registry 액세스 제어:** [Registry roles](/ko/models/registry/configure_registry#details-about-registry-roles)가 이제 SCIM 스키마에 포함됩니다. 사용자를 [생성](/ko/platform/hosting/iam/scim#create-user)하거나 업데이트할 때 해당 사용자의 Registry 역할을 설정할 수 있으며, 사용자를 [조회](/ko/platform/hosting/iam/scim#get-user)하면 Registry 역할이 함께 반환됩니다.
    * **Registry 액세스 제한:** 전체 Registry 액세스를 부여하고 싶지 않은 경우, 새 Registry 역할인 [Restricted Viewer](/ko/models/registry/configure_registry)를 부여할 수 있습니다. 이 역할은 레지스트리 아티팩트에 대한 읽기 전용 액세스를 제공하지만, 컬렉션, 자동화 또는 기타 레지스트리 리소스를 생성, 수정 또는 삭제할 수는 없습니다. 이는 민감하거나 규제를 받는 레지스트리 콘텐츠에 유용합니다.
    * [**확장된 Registry 액세스:**](https://wandb.ai/wandb_fc/product-announcements-fc/reports/Registry-for-W-B-Models-viewers-Faster-discovery-cleaner-handoffs--VmlldzoxNDc4MzM5Mg) 이제 Models **Viewer** 역할을 가진 모든 사용자는 Registry의 전체 **Member**이므로, 기본적으로 모델과 아티팩트를 탐색하고, 비교하고, 사용할 수 있습니다. Models **Viewer**에게는 **Restricted Viewer**, **Viewer**, **Admin**과 같은 다른 Registry 역할을 할당할 수 있습니다. 이를 통해 조직 내 모든 사용자가 Registry의 가치를 온전히 활용할 수 있습니다.

  * **즐겨찾기한 Registry:** 워크로드 규모가 커질수록 Registry를 더 쉽게 탐색할 수 있도록, 이제 Registry를 즐겨찾기에 추가할 수 있습니다. Registry 랜딩 페이지에서 특정 Registry 카드에 마우스를 올린 뒤, 별 윤곽 아이콘을 클릭하세요. 개별 Registry 페이지에서는 페이지 상단의 별 윤곽 아이콘을 클릭하면 됩니다. 즐겨찾기한 Registry는 Registry 랜딩 페이지 상단에 알파벳 순으로 표시됩니다.

    ![별표 표시된 레지스트리](/images/release-notes/v0-75-0/starred-registries.png)

  * **자동화 실행 기록:** **Automation history** 탭에서 프로젝트 또는 레지스트리의 [Automations](/ko/models/automations/automation-events) 실행 기록을 확인합니다. 모든 자동화 실행은 최신 순으로 정렬되어 상태와 주요 세부 정보를 한눈에 확인할 수 있습니다. 자동화를 이해하고 디버깅하는 데 도움이 되도록, 상세 보기에서는 해당 실행의 조건, 동작, 결과를 보여줍니다.

  * 이제 **Organization 대시보드**에서 쉼표로 구분된 이메일 주소 목록을 입력하여 여러 사용자를 한 번에 조직에 초대하고, 원하면 초대한 사용자들을 동시에 팀에 추가할 수도 있습니다.

  * **시맨틱 범례 변경 사항:**
    * 실행에 대한 시맨틱 범례에서 이제 하이퍼파라미터와 같은 config 값에 따라 플롯의 색상을 지정할 수 있습니다. 학습률, 배치 크기, 모델 패밀리와 같은 선택한 하이퍼파라미터에 따라 실행에 색상을 입혀, config에 따른 효과를 시각적으로 쉽게 파악할 수 있습니다.

      ![Semantic legend config coloring](/images/release-notes/v0-75-0/semantic-legend-config-coloring.gif)

    * 이제 다른 사용자의 Workspace를 볼 때도 시맨틱 범례를 구성할 수 있습니다.

  * 새로운 전체 화면 이미지 브라우저는 로깅된 이미지를 확인하고 비교하는 작업을 더 쉽게 만들어 줍니다. 이전에는 **Dedicated Cloud**에서만 사용할 수 있었지만, 이제 **Self-Managed**에서도 사용할 수 있습니다.
    * **진정한 전체 화면 보기:** 테두리까지 꽉 차는 전체 화면 보기로 이미지 세부 정보를 더 쉽게 확인할 수 있습니다. 더 이상 비좁은 패널은 없습니다.
    * **전체 화면 보기에서의 step 슬라이더:** 내장된 슬라이더를 사용해 step을 효율적으로 이동하며 출력이 어떻게 변하는지 확인할 수 있습니다.
    * **실행 또는 step 간 빠르게 이동:** 전체 화면 보기에서 키보드의 **왼쪽 / 오른쪽 화살표** 키로 실행을 이동하고, **Cmd + 왼쪽 / 오른쪽 화살표**(macOS) 또는 **Ctrl + 왼쪽 / 오른쪽 화살표**(Windows / Linux)를 사용해 동일한 이미지의 step을 이동할 수 있습니다.

  * **시스템 메트릭의 완전 충실도 렌더링:** GPU, CPU, 메모리와 같은 시스템 메트릭 차트에 미세한 신호까지 보존해 깊이 있는 점검과 분석이 가능한 새로운 고정밀 렌더링 모드가 추가되었습니다.

  * 실행 이력을 Parquet 형식으로 내보내는 기능이 개선되었습니다. 이제 모든 배포 환경에서 실행 이력 내보내기가 기본적으로 비동기 방식으로 수행됩니다. 또한 실행 이력을 내보낼 때, 어떤 실행이 7일 이상 동안 로그되지 않았고 해당 실행에 대한 이력이 모두 내보내진 후에는 그 실행 이력이 MySQL에서 자동으로 삭제됩니다.

  * X축을 **Relative Wall Time**으로 설정한 상태에서 그룹화된 실행을 그래프로 표시할 때 잘못된 값이 나타날 수 있던 버그를 수정했습니다.
  * 그룹 페이지네이션 컨트롤이 표시되지 않던 버그를 수정했습니다.
  * 버킷 서브패스가 설정된 S3 호환 스토리지로 BYOB 스토리지를 구성한 경우 보고서를 게시할 수 없던 버그를 수정했습니다. 이 문제는 CoreWeave AI Object Storage, Amazon S3 스토리지, MinIO와 같은 S3 호환 스토리지에 영향을 줍니다.
  * 스무딩이 활성화된 경우 시스템 메트릭 차트가 로드되지 않을 수 있는 버그를 수정했습니다.
  * 스무딩이 켜져 있을 때 여러 Y축이 있는 차트가 로드되지 않을 수 있던 버그를 수정했습니다.
  * 풀 피델리티 모드에서는 이제 선 그래프 툴팁에 최소 및 최대 경계값이 표시됩니다.
  * 그룹화된 실행 플롯에서 호버 하이라이트 동작이 개선되었습니다.
  * X축 이름이 Y축 이름에 부분 문자열로 포함되어 있는 경우, 그룹화된 플롯이 렌더링되지 않을 수 있던 버그를 수정했습니다.
  * 숨겨지거나 선택 해제된 실행이 숨겨지지 않고 회색 점으로 표시될 수 있던 버그를 수정했습니다.
  * 스크롤을 해야 볼 수 있는 경우, 미디어 패널이 전체 화면으로 열리지 않던 버그를 수정했습니다.
  * 여러 비교 선이 촘촘하게 겹치는 선 플롯에서 스파이크를 더 명확하게 볼 수 있도록, 최소/최대 값 음영 표시가 개선되었습니다.
  * 심층적인 조정이나 실험 후에도 효율적으로 복구할 수 있도록, 이제 선 그래프 편집기에서 X축 방향을 반전할 수 있습니다.

    더 강력한 새로운 환경으로의 원활한 업그레이드를 위해 준비하고 있습니다. 마이그레이션에 대해 궁금한 점이 있으면 언제든지 W&amp;B 계정 팀이나 [support](mailto:support@wandb.ai)로 문의하세요.
  * [실행 로그 콘솔 페이지](/ko/models/app/console-logs)에 복사 버튼을 추가하고 붙여넣기 동작 방식을 개선했습니다.

  ## 기능

  * <a id="registry_ga_announcement" aria-label="W&B Registry GA 공지" /> 앞으로 여러 차례의 릴리스를 거쳐 기존 팀 수준 Model Registry를 단계적으로 W&amp;B Registry로 마이그레이션할 예정입니다. **사용자 측에서 별도의 조치는 필요하지 않습니다. W&amp;B가 각 팀의 Model Registry 자산을 해당 팀 수준의 자동 생성 프라이빗 레지스트리로 자동 복사합니다.** 기존 팀 수준 Model Registry의 모델 컬렉션에 대한 참조는 계속해서 정상적으로 동작합니다.

    2. Server v0.73(8월)에는 업그레이드 중 이 마이그레이션을 자동으로 실행하는 옵트아웃 환경 변수가 포함될 예정입니다. 대부분의 조직에서 마이그레이션은 5분에서 30분 정도 소요됩니다. 마이그레이션 동안에는 기존 팀의 Model Registry 또는 새로운 Registry에 아티팩트를 연결하는 작업이 오류가 발생하며 실패합니다. **아티팩트 로깅과 조회에는 영향을 주지 않습니다**.
    3. Server v0.74(9월)에서는 이 플래그를 제거하고, 업그레이드 중에 마이그레이션이 수행됩니다.

    강력한 새로운 환경으로의 업그레이드를 최대한 매끄럽게, 사용자 불편을 최소화하는 방향으로 최적화하고 있습니다. 마이그레이션과 관련해 궁금한 점이 있으면 언제든지 W&amp;B 담당자에게 문의하세요.

  ## 성능

  * 특히 대규모 워크스페이스에서 여러 워크스페이스 내 상호작용의 성능이 개선되었습니다. 예를 들어, 섹션을 펼치거나 실행 선택기를 사용할 때 반응 속도가 눈에 띄게 빨라졌습니다.
  * Fork 및 Rewind 작업 성능이 개선되었습니다.

    [Forking](/ko/models/runs/forking)을 수행하면 기존 실행과 동일한 구성을 사용하는 새 실행이 생성됩니다. 포크된 실행에 대한 변경 사항은 부모 실행에 영향을 주지 않으며, 그 반대도 마찬가지입니다. 포크된 실행과 부모 실행 사이에는 포인터가 유지됩니다. [Rewinding](/ko/models/runs/rewind)을 사용하면 기존 데이터를 잃지 않고 해당 시점부터 새 데이터를 기록할 수 있습니다.

    포크가 여러 단계로 중첩된 프로젝트에서는 캐싱이 개선되어 새 실행을 포크하는 것이 이제 훨씬 더 효율적입니다.
