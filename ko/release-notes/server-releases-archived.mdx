<Update label="v0.63.0" description="2024년 12월 10일">
## 주요 기능

**[W&B Weave](https://wandb.ai/site/weave/)를 이제 AWS의 Dedicated Cloud에서 정식 버전(GA)으로 사용할 수 있습니다. 팀에서 Generative AI 앱을 안심하고 구축하고 프로덕션에 배포하고자 한다면 W&B 팀에 문의하세요.**

<Frame>
![Weave UI](https://github.com/user-attachments/assets/22786cbc-7d69-4505-b329-78cb87734d19)
</Frame>

이번 릴리즈에는 다음과 같은 추가 업데이트가 포함되어 있습니다:

* W&B Models가 이제 **_Azure 퍼블릭 클라우드_**와 원활하게 통합됩니다. 이제 Azure 구독에서 직접 Azure 리전에 Dedicated Cloud 인스턴스를 생성하고 Azure ISV 리소스로 관리할 수 있습니다. [이 통합은 프라이빗 프리뷰 단계에 있습니다](https://wandb.ai/site/partners/azure).
* 분산 Runs에서 레이블(예: node)별로 로그와 메트릭을 구분하기 위해 run 설정 오브젝트에 x_label(예: node-0)을 지정하는 기능이 추가되었습니다. 이를 통해 Workspace에서 시각화를 위해 시스템 메트릭과 콘솔 로그를 레이블별로 그룹화할 수 있습니다.
* **_이번 주 패치 릴리즈와 함께 곧 출시 예정_**: 조직 수준의 서비스 계정을 사용하여 인스턴스의 모든 Teams에서 W&B 워크로드를 자동화할 수 있게 됩니다. 서비스 계정의 엑세스 범위를 더 세밀하게 제어하고 싶은 경우 기존의 팀 수준 서비스 계정을 계속 사용할 수도 있습니다.
    * 조직 수준 서비스 계정이 Registry와 상호 작용할 수 있도록 허용합니다. 이러한 서비스 계정은 초대 모달을 통해 레지스트리에 초대될 수 있으며, 해당 조직 역할과 함께 멤버 테이블에 표시됩니다.

## 수정 사항

* `Create Artifact` 권한을 포함한 커스텀 역할을 생성한 Users가 프로젝트에 Artifacts를 로그할 수 없던 문제를 수정했습니다.
* BYOB를 위해 하위 경로 지원이 설정된 인스턴스에서 파일의 메타데이터 로깅 문제를 수정했습니다.
* 조직 레지스트리 자동화에서 사용 중인 웹훅의 삭제를 차단했습니다.
</Update>

<Update label="v0.61.0" description="2024년 10월 17일">
## 주요 기능

**이번 릴리즈는 매월 진행되는 W&B server 메이저 릴리즈와는 별도의 일정으로 제공되는 미니 기능 및 패치 릴리즈입니다.**

* 조직 관리자는 이제 조직 대시보드에서 Models 시트와 Models 및 [W&B Weave](/weave/)에 대한 엑세스 제어를 원활하게 구성할 수 있습니다. 이 변경을 통해 Dedicated Cloud 또는 Self-Managed 인스턴스에서 [Weave](/weave/)가 활성화된 경우 효율적인 사용자 관리가 가능해집니다.
    * [Weave](/weave/) 가격은 사용된 시트 수가 아닌 소비 기반으로 책정됩니다. 시트 관리는 Models 제품에만 적용됩니다.
* 이제 [팀 및 제한된 범위의 프로젝트에 대해 프로젝트 수준에서 엑세스 역할을 구성](https://docs.wandb.ai/platform/hosting/iam/access-management/restricted-projects/)할 수 있습니다. 이를 통해 동일한 팀 내의 서로 다른 프로젝트에서 사용자에게 서로 다른 엑세스 역할을 할당할 수 있어, 기업 거버넌스 요구 사항을 준수하기 위한 강력한 제어 수단이 추가되었습니다.


## 수정 사항

* 릴리즈 업그레이드의 일부로 수행되는 기본 데이터베이스 스키마 변경이 플랫폼 시작 시 타임아웃될 수 있는 문제를 수정했습니다.
* 기본 parquet 저장소 서비스에 더 많은 성능 개선 사항을 추가하여 사용자들을 위한 차트 로딩 시간을 더욱 개선했습니다. Parquet 저장소 서비스는 Dedicated Cloud 및 [W&B kubernetes operator](https://docs.wandb.ai/platform/hosting/operator) 기반의 Self-Managed 인스턴스에서만 사용할 수 있습니다.
* 기본 parquet 저장소 서비스의 높은 CPU 점유율 문제를 해결하여 사용자들이 효율적인 차트 로딩을 더 안정적으로 이용할 수 있도록 했습니다. Parquet 저장소 서비스는 Dedicated Cloud 및 [W&B kubernetes operator](https://docs.wandb.ai/platform/hosting/operator) 기반의 Self-Managed 인스턴스에서만 사용할 수 있습니다.
</Update>

## 주요 기능
* 웹 콘텐츠 접근성 가이드라인(WCAG) 표준의 Level AA 2.2 준수를 위한 최종 업데이트가 완료되었습니다.
* W&B는 이제 W&B kubernetes operator를 사용하는 Self-Managed 인스턴스에 대해 자동 버전 업그레이드를 끌 수 있습니다. W&B 팀에 이를 요청할 수 있습니다.
  W&B는 각 버전에 대해 6개월의 지원 종료 기간을 준수하기 위해 모든 인스턴스가 주기적으로 업그레이드될 것을 요구합니다. W&B는 6개월보다 오래된 버전을 지원하지 않습니다.

## 수정 사항
* Dedicated Cloud 및 Self-Managed 인스턴스의 인스턴스 관리자가 개인 Entities의 Workspaces에 엑세스할 수 있도록 하는 버그를 수정했습니다.
* SCIM Groups 및 Users GET 엔드포인트가 이제 응답에서 서비스 계정을 필터링합니다. 이제 해당 엔드포인트에서는 서비스 계정이 아닌 사용자만 반환됩니다.
* 팀 관리자가 사용자를 팀에서 삭제하는 동시에 전체 인스턴스에서도 삭제할 수 있었던 기능을 제거하여 사용자 관리 버그를 수정했습니다. 인스턴스 또는 조직 관리자만이 전체 인스턴스 / 조직에서 사용자를 삭제할 책임을 집니다.

## 성능 개선
* 메트릭이 많은 Workspaces에서 패널을 추가할 때의 지연 시간을 최대 90%까지 줄였습니다.
* Runs가 자주 재개될 때 blob 스토리지로의 parquet 내보내기 안정성과 성능을 개선했습니다.
  Parquet 형식의 blob 스토리지로의 Runs 내보내기는 Dedicated Cloud 및 W&B kubernetes operator를 사용하여 활성화된 Self-Managed 인스턴스에서 사용할 수 있습니다.
</Update>

<Update label="v0.58.1" description="2024년 9월 4일">
## 주요 기능
* W&B는 이제 **보안 스토리지 커넥터(Bring your own bucket)** 기능에 대해 하위 경로(sub-path)를 지원합니다. 이제 인스턴스 또는 팀 수준에서 버킷을 구성할 때 하위 경로를 제공할 수 있습니다. 이는 신규 버킷 구성에만 적용되며 기존에 구성된 버킷에는 적용되지 않습니다.
* Google Cloud 및 Azure의 신규 Dedicated Cloud 인스턴스에서 W&B가 관리하는 스토리지는 기본적으로 **W&B 관리 클라우드 네이티브 키**로 암호화됩니다. 이는 AWS 인스턴스에서는 이미 제공되고 있던 기능입니다. 각 인스턴스 스토리지는 해당 인스턴스 고유의 키로 암호화됩니다. 지금까지 Google Cloud 및 Azure의 모든 인스턴스는 기본 클라우드 제공업체 관리 암호화 키에 의존했습니다.
* run 설정 및 요약의 필드를 클릭 시 복사 가능하도록 만들었습니다.
* Self-Managed 인스턴스용 W&B kubernetes operator를 사용하는 경우, 이제 선택적으로 controller manager를 위한 커스텀 CA를 사용할 수 있습니다.
* OpenShift의 보안 컨텍스트 제약 조건(SCC)에 맞추기 위해 W&B kubernetes operator가 기본적으로 non-root 컨텍스트에서 실행되도록 수정했습니다. 이 변경을 통해 보안 정책을 준수하면서 OpenShift에서 Self-Managed 인스턴스를 더 원활하게 배포할 수 있습니다.

## 수정 사항
* Workspace의 패널을 Report로 내보낼 때 패널 검색 정규식이 올바르게 반영되도록 수정했습니다.
* `GORILLA_DISABLE_PERSONAL_ENTITY`를 `true`로 설정해도 사용자가 개인 Entities에서 Projects를 생성하고 기존 Projects에 쓸 수 있었던 문제를 수정했습니다.

## 성능 개선
* 로그된 포인트가 10만 개 이상인 Experiments에 대해 성능과 안정성을 크게 개선했습니다. Self-Managed 인스턴스의 경우, W&B kubernetes operator를 사용하여 배포를 관리하는 경우 이 기능을 사용할 수 있습니다.
* 대규모 Workspaces에서 변경 사항을 저장할 때 매우 느리거나 실패하던 문제를 수정했습니다.
* 대규모 Workspaces에서 Workspace 섹션을 여는 지연 시간을 개선했습니다.
</Update>

<Update label="v0.57.2" description="2024년 7월 24일">
## 주요 기능

**이제 ID 페더레이션 기능을 사용하여 wandb SDK 또는 CLI에서 JWT(JSON Web Tokens)를 통해 W&B 인스턴스에 엑세스할 수 있습니다. 이 기능은 프리뷰 단계입니다.** [ID 페더레이션](https://docs.wandb.ai/platform/hosting/iam/identity_federation)을 참조하고 질문이 있는 경우 W&B 팀에 문의하세요.

* Workspace 패널을 Reports로 내보내기 위한 새로운 `Add to reports` 드로어(drawer) 기능이 개선되었습니다.
* Artifact 프로젝트 브라우저에 Artifacts 메타데이터 필터링 기능이 추가되었습니다.
* `${artifact_metadata.KEY}`를 통해 웹훅 페이로드에 Artifact 메타데이터를 전달할 수 있습니다.
* RunSystemMetrics 컴포넌트에 GPU 메모리 사용량 패널을 추가하여 앱 프론트엔드에서 Runs의 GPU 메트릭 시각화를 강화했습니다.
* 모바일 사용자가 훨씬 더 부드럽고 직관적인 Workspace 경험을 누릴 수 있습니다.
* Google Cloud 또는 Azure에서 W&B Dedicated Cloud를 사용하는 경우, 이제 인스턴스에 대해 프라이빗 연결을 활성화할 수 있습니다. 이를 통해 AI 워크로드 및 선택적으로 브라우저 클라이언트의 모든 트래픽이 클라우드 제공업체의 프라이빗 네트워크만 통과하도록 보장할 수 있습니다. [프라이빗 연결](https://docs.wandb.ai/platform/hosting/data-security/private-connectivity)을 참조하고 질문이 있는 경우 W&B 팀에 문의하세요.
* 팀 수준 서비스 계정이 이제 팀 설정 뷰의 새로운 탭에 별도로 표시됩니다. 서비스 계정은 더 이상 Members 탭에 나열되지 않습니다. 또한 API 키는 이제 숨겨지며 팀 관리자만 복사할 수 있습니다.
* Dedicated Cloud를 이제 Google Cloud의 서울 리전에서 사용할 수 있습니다.

## 수정 사항

* 가우시안 스무딩(Gaussian smoothing)이 여러 플롯에서 너무 과도하게 적용되던 문제를 수정했습니다.
* UI Workspace에서 `Ignore Outliers in Chart Scaling` 버튼을 눌러도 효과가 없던 문제를 수정했습니다.
* 비활성화된 사용자를 조직에 초대하는 것을 금지했습니다.
* SCIM API를 사용하여 인스턴스에 추가된 사용자가 성공적으로 온보딩되지 않던 문제를 수정했습니다.

## 성능 개선

* 패널 설정을 편집하고 변경 사항을 적용할 때의 성능을 크게 개선했습니다.
* 대규모 Workspaces에서 run 가시성 토글의 응답성을 개선했습니다.
* 대규모 Workspaces의 플롯에서 차트 호버링(hovering) 및 브러싱(brushing) 성능을 개선했습니다.
* 키가 많은 Workspaces에서 Workspace 메모리 사용량과 로딩 시간을 줄였습니다.
</Update>

## 주요 기능

* W&B Experiments의 새로운 Full Fidelity 라인 플롯은 x축을 따라 모든 데이터를 집계하여 각 버킷 내의 최소값, 최대값 및 평균값을 표시함으로써 트레이닝 메트릭의 가시성을 높였습니다. 이를 통해 사용자는 다운샘플링 손실 없이 이상치를 쉽게 발견하고 고해상도 세부 정보를 확대하여 볼 수 있습니다. [문서에서 자세히 알아보기](https://docs.wandb.ai/models/app/features/panels/line-plot/sampling).
* 이제 Dedicated Cloud 및 Self-Managed 인스턴스에서 [팀 수준 BYOB(보안 스토리지 커넥터)를 위해 교차 클라우드 스토리지 버킷을 사용](https://docs.wandb.ai/platform/hosting/data-security/secure-storage-connector#cross-cloud-or-s3-compatible-storage-for-team-level-byob)할 수 있습니다. 예를 들어, AWS의 W&B 인스턴스에서 팀 수준 BYOB를 위해 Azure Blob Storage 또는 Google Cloud Storage를 구성할 수 있으며, 다른 모든 교차 클라우드 조합에 대해서도 마찬가지입니다. 
* 같은 맥락에서, 이제 Dedicated Cloud 및 Self-Managed 인스턴스에서 [팀 수준 BYOB(보안 스토리지 커넥터)를 위해 MinIO와 같은 S3 호환 스토리지 버킷을 사용](https://docs.wandb.ai/platform/hosting/data-security/secure-storage-connector#cross-cloud-or-s3-compatible-storage-for-team-level-byob)할 수 있습니다. 예를 들어, Google Cloud의 W&B 인스턴스에서 클라우드 또는 온프레미스에 호스팅된 MinIO 버킷을 팀 수준 BYOB용으로 구성할 수 있습니다.
* 관리자는 이제 [SCIM API의 DELETE User 엔드포인트](https://docs.wandb.ai/platform/hosting/iam/scim#delete-user)를 사용하여 Dedicated Cloud 또는 Self-Managed 인스턴스에서 사용자의 전체 삭제를 자동화할 수 있습니다. 사용자 비활성화 작업은 [PATCH User 엔드포인트](https://docs.wandb.ai/platform/hosting/iam/scim#deactivate-user)를 사용하여 다시 구현되었으며, [사용자 재활성화 작업](https://docs.wandb.ai/platform/hosting/iam/scim#reactivate-user)도 도입되었습니다.
* SCIM API를 사용하는 경우 다음과 같은 몇 가지 마이너 개선 사항도 확인할 수 있습니다:

- 임베딩 프로젝터 렌더링 문제를 수정했습니다.
- 대소문자가 혼합된 사용자 이름과 관련된 일부 사례에서 사용자가 비밀번호를 재설정할 수 없던 문제를 수정했습니다.
- 특수 문자가 포함된 파일이 Azure의 미디어 패널에 표시됩니다.

- 커스텀 차트가 포함된 Reports가 로그인하지 않은 상태에서도 로드됩니다.
- 넓은 GIF가 전체 화면 뷰를 넘치지 않도록 수정했습니다.

  모델 레지스트리 관리자를 위한 새로운 역할을 통해 모델 승인 프로세스를 제어하세요. 이 사용자들은 보호된 에일리어스(예: “challenger” 또는 “prod”) 목록을 관리하고 모델 버전에 보호된 에일리어스를 적용하거나 제거할 수 있습니다.

- 개선된 공유: 선택적 매직 링크 및 뷰어를 위한 쉬운 가입
  팀 관리자는 이제 팀과 멤버를 위해 매직 링크 공유를 끌 수 있습니다. 팀 설정에서 공개 공유를 비활성화하면 팀 프라이버시 제어를 강화할 수 있습니다. 한편, 리포트 링크를 받은 사용자가 가입 후 W&B의 리포트에 더 쉽게 엑세스할 수 있게 되었습니다.
- 개선된 리포트 작성
  <Frame>
  ![향상된 초안 게시, 편집 및 협업 기능을 갖춘 개선된 리포트 작성 인터페이스](https://github.com/wandb/server/assets/97066933/c2858fd5-d2f6-47e3-9e93-f8d048df4f59)

  W&B가 다루는 ML 워크플로우 영역이 확장됨에 따라 애플리케이션 내에서의 이동이 어려울 수 있다는 피드백을 수렴했습니다. 이에 따라 제품 영역에 더 명확한 레이블을 포함하고 특정 상세 화면에 백링크를 추가하여 네비게이션 사이드바를 업데이트했습니다. 또한 기능의 강력함을 더 잘 반영하기 위해 “Triggers”의 이름을 “Automations”로 변경했습니다.

## 수정 사항
- Workspaces 또는 리포트의 플롯 위로 마우스를 가져가면 이제 Cmd+C 또는 Ctrl+C를 사용하여 호버 제어에 표시된 run 이름과 플롯 값을 복사할 수 있습니다.
- 기본 Workspaces의 변경 사항이 더 이상 자동으로 저장되지 않습니다.
- Overview → Summary 섹션의 메트릭에 이제 쉼표가 포함된 포맷이 적용됩니다.
- 관리자가 아닌 사용자도 팀을 생성할 수 있도록 허용하는 설치 수준 옵션을 추가했습니다(기본값은 off이며 활성화하려면 W&B 지원팀에 문의하세요).
- Weave 플롯이 이제 로그 스케일을 지원합니다.
- Launch 패널을 가로로 확장하여 파라미터를 볼 수 있는 더 많은 공간을 확보할 수 있습니다.
- Launch 패널에 이제 큐의 활성 상태 여부가 표시됩니다.
- Launch 패널에서 이제 run이 기록될 프로젝트를 선택할 수 있습니다.
- Launch 큐는 이제 팀 관리자만 생성할 수 있습니다.
- Launch 패널에서 마크다운 지원을 개선했습니다.
- 비어 있는 Launch 큐 구성에 대한 에러 메시지를 개선했습니다.
- 스윕 병렬 좌표 플롯의 필터가 이제 선택된 모든 runset에 적용됩니다.
- Sweeps에서 더 이상 메트릭이 필수가 아닙니다.
- Azure Blob Storage에서 W&B 외부에 저장된 참조 Artifact 파일 추적 기능을 추가했습니다.
- 리포트의 마크다운 편집 버그를 수정했습니다.
- 전체 화면 Weave 패널이 이제 원본 패널과 설정 변경 사항을 공유할 수 있습니다.
- 빈 테이블의 표시 방식을 개선했습니다.
- 로그의 처음 몇 글자가 잘리던 버그를 수정했습니다.
