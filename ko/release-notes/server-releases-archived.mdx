<Update label="v0.63.0" description="2024년 12월 10일">
  ## 기능

  **[W&amp;B Weave](https://wandb.ai/site/weave/)가 이제 AWS Dedicated Cloud에서 일반 공급(GA) 상태로 제공됩니다. 팀에서 신뢰할 수 있는 Generative AI 앱을 만들고 이를 프로덕션에 배포하려는 경우, 담당 W&amp;B 팀에 문의하세요.**

  <Frame>
    ![Weave UI](https://github.com/user-attachments/assets/22786cbc-7d69-4505-b329-78cb87734d19)
  </Frame>

  이번 릴리스에는 다음과 같은 추가 업데이트가 포함됩니다:

  * W&amp;B Models가 이제 ***Azure 퍼블릭 클라우드***&#xC640; 원활하게 연동됩니다. Azure 구독에서 직접 Azure 리전에 Dedicated Cloud 인스턴스를 생성하고, 이를 Azure ISV 리소스로 관리할 수 있게 되었습니다. [이 인테그레이션은 현재 비공개 프리뷰 단계입니다](https://wandb.ai/site/partners/azure).
  * 분산 runs에서 레이블(예: node)에 따라 로그와 메트릭을 구분할 수 있도록, run 설정 객체에서 x&#95;label(예: node-0)을 지정할 수 있는 기능을 추가했습니다. 이를 통해 워크스페이스에서 레이블별로 시스템 메트릭과 콘솔 로그를 그룹화해 시각화할 수 있습니다.
  * ***이번 주 패치 릴리스에서 곧 제공될 기능으로***, 인스턴스 내 모든 팀에 걸친 W&amp;B 워크로드 자동화를 위해 조직 수준 서비스 계정을 사용할 수 있게 됩니다. 서비스 계정의 액세스 범위를 더 세밀하게 제어하고 싶다면 기존 팀 수준 서비스 계정도 계속 사용할 수 있습니다.
    * 조직 수준 서비스 계정이 Registry와 상호작용할 수 있도록 했습니다. 이러한 서비스 계정은 초대 모달을 통해 Registry에 초대할 수 있으며, 해당 조직 역할과 함께 멤버 테이블에 표시됩니다.

  ## 수정 사항

  * `Create Artifact` 권한을 포함한 커스텀 역할을 생성한 사용자가 프로젝트에 아티팩트를 로깅할 수 없던 문제를 수정했습니다.
  * BYOB용 서브패스 지원이 구성된 인스턴스에서 파일 메타데이터 로깅에 문제가 발생하던 현상을 수정했습니다.
  * 조직 Registry 자동화에서 사용 중인 경우, 웹후크가 삭제되지 않도록 차단했습니다.
</Update>

<Update label="v0.61.0" description="2024년 10월 17일">
  ## 기능

  **이 릴리스는 월간 W&amp;B 서버 메이저 릴리스와는 다른 일정으로 제공되는 소규모 기능 추가 및 패치 릴리스입니다.**

  * 조직 관리자는 이제 조직 대시보드에서 Models 및 [W&amp;B Weave](/ko/weave/) 모두에 대해 Models 좌석 수와 액세스 제어를 통합적으로 구성할 수 있습니다. 이 변경으로 Dedicated Cloud 또는 Self-Managed 인스턴스에서 [Weave](/ko/weave/)가 활성화된 경우 사용자 관리를 더 효율적으로 수행할 수 있습니다.
    * [Weave](/ko/weave/) 요금은 사용 좌석 수 기준이 아닌 사용량 기반입니다. 좌석 관리는 Models 제품에만 적용됩니다.
  * 이제 [팀 및 제한 범위 프로젝트에 대해 프로젝트 수준 액세스 역할을 구성](https://docs.wandb.ai/platform/hosting/iam/access-management/restricted-projects/)할 수 있습니다. 이를 통해 동일한 팀 내 서로 다른 프로젝트에서 사용자에게 서로 다른 액세스 역할을 부여할 수 있으며, 엔터프라이즈 거버넌스 요구 사항을 충족하기 위한 강력한 제어 수단을 하나 더 제공할 수 있습니다.

  ## 수정 사항

  * 릴리스 업그레이드의 일부로 적용되는 기본 데이터베이스 스키마 변경으로 인해 플랫폼 시작 시 타임아웃이 발생할 수 있던 문제를 수정했습니다.
  * 기본 parquet store 서비스에 추가 성능 개선을 적용해, 사용자에게 차트 로딩 시간을 한층 더 단축했습니다. Parquet store 서비스는 Dedicated Cloud와 [W&amp;B kubernetes operator](https://docs.wandb.ai/platform/hosting/operator)를 기반으로 하는 Self-Managed 인스턴스에서만 제공됩니다.
  * 기본 parquet store 서비스에서 CPU 사용률이 높게 나오는 문제를 해결해, 효율적인 차트 로딩이 사용자에게 더 안정적으로 제공되도록 했습니다. Parquet store 서비스는 Dedicated Cloud와 [W&amp;B kubernetes operator](https://docs.wandb.ai/platform/hosting/operator)를 기반으로 하는 Self-Managed 인스턴스에서만 제공됩니다.
</Update>

<Update label="v0.60.0" description="2024년 9월 26일">
  <Note>
    릴리스 버전 관리 문제로 인해 0.60.0은 0.58.0 이후의 다음 메이저 릴리스입니다. 0.59.0은 0.58.0에 대한 패치 릴리스 중 하나였습니다.

  ## 기능

  * 웹 콘텐츠 접근성 지침(WCAG) 표준의 Level AA 2.2에 대한 1.1.1 컴플라이언스 최종 업데이트.
  * 이제 Self-Managed 인스턴스에서 W&amp;B kubernetes operator를 사용할 때 자동 버전 업그레이드를 끌 수 있습니다. 이 기능은 담당 W&amp;B 팀에 요청할 수 있습니다.
    W&amp;B는 각 버전에 대해 6개월 사용 종료(end-of-life) 기간을 준수하기 위해 모든 인스턴스가 주기적으로 업그레이드되도록 요구합니다. W&amp;B는 6개월보다 오래된 버전은 지원하지 않습니다.

  ## 버그 수정

  * Dedicated Cloud 및 Self-Managed 인스턴스에서 인스턴스 관리자(instance admin)가 개인 엔터티 내 워크스페이스에 접근할 수 없던 버그를 수정했습니다.
  * SCIM Groups 및 Users GET endpoint가 응답에서 서비스 계정을 필터링하도록 변경했습니다. 이제 이 endpoint들은 서비스 계정이 아닌 사용자만 반환합니다.
  * 팀 관리자가 팀에서 사용자를 삭제할 때, 동시에 전체 인스턴스에서도 사용자를 삭제할 수 있던 사용자 관리 버그를 수정했습니다. 전체 인스턴스/조직에서 사용자를 삭제하는 책임은 인스턴스 또는 Org 관리자에게 있습니다.

  ## 성능 개선

  * 많은 메트릭이 있는 워크스페이스에서 패널을 추가할 때 지연 시간을 최대 90%까지 줄였습니다.
  * run을 자주 재개(resume)할 때 blob 스토리지로의 parquet 내보내기의 신뢰성과 성능을 개선했습니다.
    Dedicated Cloud 및 W&amp;B kubernetes operator로 활성화된 Self-Managed 인스턴스에서는 parquet 포맷으로 blob 스토리지에 내보내기가 가능합니다.
</Update>

<Update label="v0.58.1" description="2024년 9월 4일">
  ## 기능

  * 이제 W&amp;B는 **Secure storage connector, 즉 Bring your own bucket** 기능에서 서브 경로(sub-path)를 지원합니다. 인스턴스 또는 팀 수준에서 버킷을 설정할 때 서브 경로를 지정할 수 있습니다. 이는 새로 설정하는 버킷에만 적용되며, 이미 설정된 기존 버킷에는 적용되지 않습니다.
  * Google Cloud 및 Azure의 신규 Dedicated Cloud 인스턴스에서 W&amp;B가 관리하는 스토리지는 기본적으로 **W&amp;B managed cloud-native keys**로 암호화됩니다. 이 기능은 이미 AWS 인스턴스에서 제공되고 있습니다. 각 인스턴스 스토리지는 인스턴스마다 고유한 키로 암호화됩니다. 지금까지는 Google Cloud 및 Azure의 모든 인스턴스가 클라우드 제공업체가 기본으로 제공하는 암호화 키에 의존했습니다.
  * run config와 summary 필드를 클릭해서 복사할 수 있도록 했습니다.
  * Self-Managed 인스턴스에서 W&amp;B kubernetes operator를 사용하는 경우, 이제 컨트롤러 매니저에 대해 사용자 정의 CA를 선택적으로 사용할 수 있습니다.
  * OpenShift의 Security Context Constraints(SCC)에 맞추기 위해, W&amp;B kubernetes operator가 기본적으로 non-root 컨텍스트에서 실행되도록 수정했습니다. 이 변경으로 OpenShift의 보안 정책을 준수하면서 Self-Managed 인스턴스를 보다 원활하게 배포할 수 있습니다.

  ## 버그 수정

  * 워크스페이스에서 패널을 리포트로 내보낼 때, 이제 패널 검색 정규식(regex)을 올바르게 반영하도록 수정했습니다.
  * `GORILLA_DISABLE_PERSONAL_ENTITY`를 `true`로 설정해도 개인 엔터티에서 사용자가 프로젝트를 생성하고 기존 프로젝트에 쓸 수 있었던 문제를 수정했습니다.

  ## 성능 개선

  * 100k개 이상의 포인트가 로깅된 experiment에서 성능과 안정성을 크게 개선했습니다. Self-Managed 인스턴스의 경우, W&amp;B kubernetes operator로 배포를 관리하는 경우에 제공됩니다.
  * 대형 워크스페이스에서 변경 사항을 저장할 때 매우 느리거나 실패하던 문제를 수정했습니다.
  * 대형 워크스페이스에서 워크스페이스 섹션을 여는 지연 시간을 개선했습니다.
</Update>

<Update label="v0.57.2" description="2024년 7월 24일">
  ## 기능

  **이제 identity federation 기능을 사용해 JWTs(JSON Web Tokens)를 통해 wandb SDK 또는 CLI에서 W&amp;B 인스턴스에 액세스할 수 있습니다. 이 기능은 현재 프리뷰 단계입니다.** [Identity federation](https://docs.wandb.ai/platform/hosting/iam/identity_federation)을 참고하고, 궁금한 점이 있으면 W&amp;B 팀에 문의하세요.

  * Workspace 패널을 Reports로 내보내기 위한 `Add to reports` 드로어가 개선되었습니다.
  * artifact 프로젝트 브라우저에서 Artifacts 메타데이터를 필터링할 수 있습니다.
  * `${artifact_metadata.KEY}`를 통해 웹훅 페이로드에 아티팩트 메타데이터를 전달할 수 있습니다.
  * RunSystemMetrics 컴포넌트에 GPU 메모리 사용량 패널이 추가되어, 앱 프론트엔드에서 run의 GPU 메트릭 시각화가 향상되었습니다.
  * 모바일 사용자는 이제 훨씬 더 부드럽고 직관적인 Workspace 경험을 할 수 있습니다.
  * Google Cloud 또는 Azure에서 W&amp;B Dedicated Cloud를 사용하는 경우, 이제 인스턴스에 대해 프라이빗 연결을 활성화할 수 있어 AI 워크로드와 (선택적으로) 브라우저 클라이언트에서 발생하는 모든 트래픽이 클라우드 제공자의 프라이빗 네트워크를 통해서만 전송되도록 보장할 수 있습니다. [Private connectivity](https://docs.wandb.ai/platform/hosting/data-security/private-connectivity)를 참고하고, 궁금한 점이 있으면 W&amp;B 팀에 문의하세요.
  * 팀 설정 보기에서 팀 수준 서비스 계정이 새 탭에 별도로 표시됩니다. 서비스 계정은 더 이상 Members 탭에 나열되지 않습니다. 또한 이제 API 키는 숨겨지며 팀 관리자만 복사할 수 있습니다.
  * Dedicated Cloud가 이제 Google Cloud 서울 리전에서도 제공됩니다.

  ## 수정 사항

  * 많은 플롯에서 가우시안 스무딩이 지나치게 강하게 적용되던 문제를 수정했습니다.
  * `Ignore Outliers in Chart Scaling` 버튼을 눌러도 UI 워크스페이스에서 아무런 효과가 없던 문제를 수정했습니다.
  * 비활성화된 사용자를 조직에 초대할 수 없도록 했습니다.
  * SCIM API를 사용해 인스턴스에 추가된 사용자가 정상적으로 온보딩하지 못하던 문제를 수정했습니다.

  ## 성능 개선

  * 패널 설정을 편집하고 변경 사항을 적용할 때의 성능을 크게 향상했습니다.
  * 큰 워크스페이스에서 run 가시성 토글의 반응성을 개선했습니다.
  * 큰 워크스페이스의 플롯에서 차트 호버 및 브러싱 성능을 개선했습니다.
  * 키가 많은 워크스페이스에서 메모리 사용량과 로딩 시간을 줄였습니다.
</Update>

<Update label="v0.56.0" description="2024년 6월 29일">

    릴리스 버전 관리 문제로 인해 0.56.0이 0.54.0 이후의 다음 메이저 릴리스입니다. 0.55.0은 0.54.0에 대한 패치 릴리스였습니다.

  ## Features

  * W&amp;B Experiments의 새로운 Full Fidelity line plot은 x축을 따라 모든 데이터를 집계해 각 버킷의 최소값, 최대값, 평균값을 표시함으로써 트레이닝 메트릭의 가시성을 높여 주며, 사용자가 다운샘플링에 따른 손실 없이 이상치를 쉽게 찾아내고 고충실도 세부 정보를 확대해 볼 수 있도록 합니다. [문서에서 더 알아보기](https://docs.wandb.ai/models/app/features/panels/line-plot/sampling).
  * 이제 Dedicated Cloud 및 Self-Managed 인스턴스에서 [팀 수준 BYOB(secure storage connector)를 위한 cross-cloud 스토리지 버킷](https://docs.wandb.ai/platform/hosting/data-security/secure-storage-connector#cross-cloud-or-s3-compatible-storage-for-team-level-byob)을 사용할 수 있습니다. 예를 들어 AWS 상의 W&amp;B 인스턴스에서 팀 수준 BYOB를 위해 Azure Blob Storage 또는 Google Cloud Storage를 구성할 수 있으며, 각 cross-cloud 조합에 대해서도 동일하게 설정할 수 있습니다.
  * 같은 맥락에서 이제 Dedicated Cloud 및 Self-Managed 인스턴스에서 [MinIO와 같은 S3-compatible 스토리지 버킷을 팀 수준 BYOB(secure storage connector)로](https://docs.wandb.ai/platform/hosting/data-security/secure-storage-connector#cross-cloud-or-s3-compatible-storage-for-team-level-byob) 사용할 수 있습니다. 예를 들어 Google Cloud 상의 W&amp;B 인스턴스에서 클라우드 또는 온프렘 환경에 호스팅된 MinIO 버킷을 팀 수준 BYOB로 구성할 수 있습니다.

    이제 W&amp;B Server의 Viewer 역할 도입으로 더 넓은 대상에게 W&amp;B 결과를 공유할 수 있습니다. 이 역할을 가진 사용자는 자신이 속한 팀에서 만든 모든 것을 볼 수 있지만, 생성, 편집, 삭제는 할 수 없습니다. 이 좌석은 기존 W&amp;B Server 좌석과 별도로 집계되므로, 라이선스 업데이트를 요청하려면 W&amp;B 계정 팀에 문의하세요.
  * 공유 기능 개선: 선택적 매직 링크, 뷰어를 위한 더 쉬운 가입
    팀 관리자는 이제 팀 및 팀 구성원에 대해 매직 링크 공유를 끌 수 있습니다. 팀 설정에서 공개 공유(public sharing)를 비활성화하면 팀 프라이버시 제어를 강화할 수 있습니다. 한편, Reports 링크를 받은 사용자가 가입 이후 W&amp;B에서 해당 Reports에 접근하는 것도 더 쉬워졌습니다.
  * Reports 작성 기능 개선
    <Frame>
      ![향상된 초안 발행, 편집 및 협업 기능을 갖춘 개선된 리포트 작성 인터페이스](https://github.com/wandb/server/assets/97066933/c2858fd5-d2f6-47e3-9e93-f8d048df4f59)

  ## 수정 사항

  * 워크스페이스나 리포트에서 플롯 위에 마우스를 올렸을 때, 이제 Cmd+C 또는 Ctrl+C를 사용해 hover 컨트롤에 표시된 run 이름과 플롯 값을 복사할 수 있습니다.
  * 기본 워크스페이스에 대한 변경 사항은 더 이상 자동 저장되지 않습니다.
  * Overview → Summary 섹션의 메트릭이 이제 천 단위 구분 기호(쉼표)와 함께 표시됩니다.
  * 관리자 권한이 없는 사용자도 팀을 생성할 수 있도록 하는 설치 수준 옵션을 추가했습니다(기본값 비활성화; 활성화를 원하면 W&amp;B 지원팀에 문의하세요).
  * Weave 플롯에서 이제 로그 스케일을 지원합니다.
  * Launch 패널을 가로로 확장하여 파라미터를 볼 수 있는 공간을 더 확보할 수 있습니다.
  * Launch 패널에서 이제 큐가 활성 상태인지 여부를 표시합니다.
  * Launch 패널에서 이제 run이 기록될 프로젝트를 선택할 수 있습니다.
  * Launch 큐는 이제 팀 관리자만 생성할 수 있습니다.
  * Launch 패널의 Markdown 지원을 개선했습니다.
  * 비어 있는 Launch 큐 설정에 대한 오류 메시지가 개선되었습니다.
  * Sweeps parallel coordinates 플롯의 필터가 이제 선택된 모든 runset에 적용됩니다.
  * Sweeps에서 더 이상 메트릭이 필수가 아닙니다.
  * Azure Blob Storage에 W&amp;B 외부에 저장된 참조 아티팩트 파일을 추적하는 기능을 추가했습니다.
  * Reports에서 Markdown 편집과 관련된 버그를 수정했습니다.
  * 전체 화면 Weave 패널이 이제 원래 패널과 설정 변경 사항을 공유할 수 있습니다.
  * 비어 있는 테이블의 표시를 개선했습니다.
  * 로그의 처음 몇 글자가 잘려 나가던 버그를 수정했습니다.
