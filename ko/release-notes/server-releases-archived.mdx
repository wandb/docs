<Update label="v0.63.0" description="2024년 12월 10일">
  ## 기능

  **[W&amp;B Weave](https://wandb.ai/site/weave/)가 이제 AWS Dedicated Cloud에서 일반 공급(GA)으로 제공됩니다. 팀에서 신뢰할 수 있는 방식으로 생성형 AI 앱을 만들고 프로덕션에 배포하려는 경우 W&amp;B 팀에 문의하세요.**

  <Frame>
    ![Weave UI](https://github.com/user-attachments/assets/22786cbc-7d69-4505-b329-78cb87734d19)
  </Frame>

  이번 릴리스에는 다음과 같은 추가 업데이트가 포함됩니다:

  * W&amp;B Models가 이제 ***Azure 공용 클라우드***&#xC640; 원활하게 통합됩니다. 이제 Azure 구독에서 직접 Azure 리전에 Dedicated Cloud 인스턴스를 생성하고 Azure ISV 리소스로 관리할 수 있습니다. [이 통합은 현재 비공개 프리뷰 단계입니다](https://wandb.ai/site/partners/azure).
  * 분산 실행에서 레이블(예: node)에 따라 로그와 메트릭을 구분할 수 있도록, 실행 설정 객체에서 x&#95;label(예: node-0)을 지정하는 기능이 추가되었습니다. 이를 통해 워크스페이스에서 레이블별로 시스템 메트릭과 콘솔 로그를 그룹화하여 시각화할 수 있습니다.
  * ***이번 주 패치 릴리스에서 곧 제공 예정***&#xC73C;로, 인스턴스 내 모든 팀 전반에 걸친 W&amp;B 워크로드를 자동화하기 위해 조직 단위 서비스 계정을 사용할 수 있게 됩니다. 서비스 계정의 액세스 범위를 더 세밀하게 제어하려는 경우, 기존 팀 단위 서비스 계정도 계속 사용할 수 있습니다.
    * 조직 단위 서비스 계정이 Registry와 상호작용할 수 있도록 허용합니다. 이러한 서비스 계정은 초대 모달을 사용해 Registry에 초대할 수 있으며, 각 조직 역할과 함께 멤버 테이블에 표시됩니다.

  ## 수정 사항

  * `Create Artifact` 권한을 포함하는 커스텀 역할을 생성한 사용자가 아티팩트를 프로젝트에 기록할 수 없던 문제를 수정했습니다.
  * BYOB에 대해 서브패스 지원이 구성된 인스턴스에서 파일 메타데이터 로깅과 관련된 문제를 수정했습니다.
  * 조직 Registry 자동화에서 사용 중인 경우 웹훅이 삭제되지 않도록 차단했습니다.
</Update>

<Update label="v0.61.0" description="2024년 10월 17일">
  ## 기능

  **이 릴리스는 월간 W&amp;B 서버 메이저 릴리스와는 다른 일정으로 제공되는 소규모 기능 및 패치 릴리스입니다.**

  * 조직 관리자는 이제 조직 대시보드에서 Models 및 [W&amp;B Weave](/ko/weave/) 모두에 대해 좌석 및 액세스 제어를 원활하게 구성할 수 있습니다. 이 변경으로 Dedicated Cloud 또는 Self-Managed 인스턴스에서 [Weave](/ko/weave/)가 활성화된 경우 사용자 관리를 더욱 효율적으로 수행할 수 있습니다.
    * [Weave](/ko/weave/) 요금은 좌석 수 기준이 아니라 사용량 기반입니다. 좌석 관리는 Models 제품에만 적용됩니다.
  * 이제 [팀 및 제한 범위 프로젝트에 대해 프로젝트 수준에서 액세스 역할을 구성](https://docs.wandb.ai/platform/hosting/iam/access-management/restricted-projects/)할 수 있습니다. 이를 통해 동일한 팀 내 서로 다른 프로젝트에 대해 사용자에게 서로 다른 액세스 역할을 부여할 수 있으며, 엔터프라이즈 거버넌스 요구사항을 충족하기 위한 또 하나의 강력한 제어 수단을 제공합니다.

  ## 수정 사항

  * 릴리스 업그레이드의 일부로 수행되는 기본 데이터베이스 스키마 변경이 플랫폼 시작 중 타임아웃될 수 있는 문제를 수정했습니다.
  * 사용자에게 차트 로딩 시간을 더욱 개선하기 위해 기본 parquet store 서비스의 성능을 추가로 향상시켰습니다. parquet store 서비스는 Dedicated Cloud와 [W&amp;B kubernetes operator](https://docs.wandb.ai/platform/hosting/operator)를 기반으로 하는 Self-Managed 인스턴스에서만 사용할 수 있습니다.
  * 사용자에게 보다 안정적인 효율적인 차트 로딩을 제공하기 위해 기본 parquet store 서비스에서 발생하던 높은 CPU 사용률 문제를 해결했습니다. parquet store 서비스는 Dedicated Cloud와 [W&amp;B kubernetes operator](https://docs.wandb.ai/platform/hosting/operator)를 기반으로 하는 Self-Managed 인스턴스에서만 사용할 수 있습니다.
</Update>

  ## 기능

  * 웹 콘텐츠 접근성 지침(WCAG) 2.2 레벨 AA의 성공 기준 1.1.1 준수를 위한 최종 업데이트를 적용했습니다.
  * 이제 Self-Managed 인스턴스에서 W&amp;B kubernetes operator를 사용해 자동 버전 업그레이드를 비활성화할 수 있습니다. 이 기능은 W&amp;B 팀에 요청해 활성화할 수 있습니다.
    W&amp;B는 각 버전에 대해 6개월 수명 종료(end-of-life) 정책을 준수하기 위해 모든 인스턴스가 주기적으로 업그레이드되도록 요구합니다. W&amp;B는 6개월이 지난 버전은 지원하지 않습니다.

  ## 수정 사항

  * Dedicated Cloud 및 Self-Managed 인스턴스에서 인스턴스 관리자가 개인 엔터티의 워크스페이스에 접근할 수 있도록 버그를 수정했습니다.
  * 이제 SCIM Groups 및 Users GET 엔드포인트는 응답에서 서비스 계정을 필터링합니다. 해당 엔드포인트는 이제 서비스 계정이 아닌 사용자만 반환합니다.
  * 팀 관리자가 팀에서 사용자를 삭제할 때 인스턴스 전체에서도 동시에 사용자를 삭제할 수 있던 사용자 관리 버그를 수정했습니다. 인스턴스 또는 Org 관리자만이 인스턴스/조직 전체에서 사용자를 삭제할 수 있습니다.

  ## 성능 개선

  * 많은 메트릭이 있는 워크스페이스에서 패널을 추가할 때 지연 시간을 최대 90%까지 줄였습니다.
  * 실행이 자주 재개(resume)되는 경우 blob storage로의 parquet 내보내기(export)의 안정성과 성능을 개선했습니다.
    blob storage로의 parquet 형식 실행 내보내기(export)는 Dedicated Cloud와 W&amp;B kubernetes operator로 활성화된 Self-Managed 인스턴스에서 사용할 수 있습니다.
</Update>

<Update label="v0.58.1" description="2024년 9월 4일">
  ## 기능

  * 이제 W&amp;B는 **Secure storage connector, 즉 Bring your own bucket** 기능에 대해 서브 경로(sub-path)를 지원합니다. 인스턴스 또는 팀 수준에서 버킷을 구성할 때 서브 경로를 제공할 수 있습니다. 이는 새 버킷 구성에만 사용할 수 있으며, 이미 구성된 기존 버킷에는 적용되지 않습니다.
  * Google Cloud 및 Azure의 최신 Dedicated Cloud 인스턴스에서 W&amp;B가 관리하는 스토리지는 기본적으로 **W&amp;B가 관리하는 클라우드 네이티브 키**로 암호화됩니다. 이는 이미 AWS 인스턴스에서는 제공 중입니다. 각 인스턴스 스토리지는 인스턴스 고유 키로 암호화됩니다. 지금까지 Google Cloud 및 Azure의 모든 인스턴스는 기본 클라우드 제공업체 관리 암호화 키에 의존했습니다.
  * 실행 구성(run config)과 요약(summary)의 필드를 클릭 한 번으로 복사할 수 있도록 했습니다.
  * Self-Managed 인스턴스에서 W&amp;B kubernetes operator를 사용하는 경우, 이제 선택적으로 controller manager에 대해 커스텀 CA를 사용할 수 있습니다.
  * W&amp;B kubernetes operator가 기본적으로 non-root 컨텍스트에서 실행되도록 변경하여 OpenShift의 Security Context Constraints(SCCs)에 맞췄습니다. 이 변경으로 OpenShift의 보안 정책을 준수함으로써 Self-Managed 인스턴스를 OpenShift에 더 원활하게 배포할 수 있습니다.

  ## 수정 사항

  * 워크스페이스에서 리포트로 패널을 내보낼 때 이제 패널 검색 정규식(regex)이 올바르게 반영되도록 문제를 수정했습니다.
  * `GORILLA_DISABLE_PERSONAL_ENTITY`를 `true`로 설정했을 때 사용자가 개인 엔터티에서 프로젝트를 생성하고 기존 프로젝트에 기록하는 기능이 비활성화되지 않던 문제를 수정했습니다.

  ## 성능 개선

  * 10만 개 이상의 로그 포인트가 있는 실험에 대해 성능과 안정성을 크게 개선했습니다. Self-Managed 인스턴스에서는 배포가 W&amp;B kubernetes operator로 관리되는 경우에 사용할 수 있습니다.
  * 대형 워크스페이스에서 변경 사항을 저장할 때 매우 느리거나 실패하던 문제를 수정했습니다.
  * 대형 워크스페이스에서 워크스페이스 섹션을 여는 지연 시간을 개선했습니다.
</Update>

<Update label="v0.57.2" description="2024년 7월 24일">
  ## 기능

  **이제 identity federation 기능을 사용해 JWT(JSON Web Token)로 wandb SDK 또는 CLI에서 W&amp;B 인스턴스에 접근할 수 있습니다. 이 기능은 프리뷰 단계입니다.** [Identity federation](https://docs.wandb.ai/platform/hosting/iam/identity_federation)를 참고하고, 궁금한 점이 있으면 W&amp;B 팀에 문의하세요.

  * Workspace 패널을 Reports로 내보내는 `Add to reports` 드로어가 개선되었습니다.
  * Artifact 프로젝트 브라우저에서 아티팩트 메타데이터 필터링을 지원합니다.
  * `${artifact_metadata.KEY}`를 통해 웹훅 페이로드에 아티팩트 메타데이터를 전달할 수 있습니다.
  * RunSystemMetrics 컴포넌트에 GPU 메모리 사용량 패널이 추가되어, 앱 프런트엔드에서 실행의 GPU 메트릭 시각화가 향상되었습니다.
  * 모바일 사용자는 이제 훨씬 더 부드럽고 직관적인 Workspace 경험을 누릴 수 있습니다.
  * Google Cloud 또는 Azure에서 W&amp;B Dedicated Cloud를 사용하는 경우, 이제 인스턴스에 대해 프라이빗 연결을 활성화할 수 있어, AI 워크로드와 (선택적으로) 브라우저 클라이언트의 모든 트래픽이 클라우드 제공자의 프라이빗 네트워크만 통과하도록 보장할 수 있습니다. [Private connectivity](https://docs.wandb.ai/platform/hosting/data-security/private-connectivity)를 참고하고, 궁금한 점이 있으면 W&amp;B 팀에 문의하세요.
  * 팀 수준 서비스 계정은 이제 팀 설정 뷰에서 새 탭에 별도로 표시됩니다. 서비스 계정은 더 이상 Members 탭에 나열되지 않습니다. 또한 API 키는 이제 숨겨지며 팀 관리자만 복사할 수 있습니다.
  * Dedicated Cloud가 이제 Google Cloud의 서울 리전에서 사용할 수 있습니다.

  ## 버그 수정

  * 많은 플롯에서 Gaussian smoothing이 지나치게 강하게 적용되던 문제를 수정했습니다.
  * `Ignore Outliers in Chart Scaling` 버튼을 눌러도 Workspace UI에서 아무 효과가 없던 문제를 수정했습니다.
  * 비활성화된 사용자를 조직에 초대하지 못하도록 막았습니다.
  * SCIM API를 사용해 인스턴스에 추가된 사용자가 온보딩에 실패하던 문제를 수정했습니다.

  ## 성능 개선

  * 패널 설정을 편집하고 변경 사항을 적용할 때의 성능을 크게 개선했습니다.
  * 대규모 Workspace에서 실행 가시성 토글의 응답성을 개선했습니다.
  * 대규모 Workspace의 플롯에서 차트 호버 및 브러싱 성능을 개선했습니다.
  * 키가 많은 Workspace에서 메모리 사용량과 로딩 시간을 줄였습니다.
</Update>

  ## 기능

  * W&amp;B Experiments의 새로운 Full Fidelity 선 그래프는 x축 전체의 데이터를 집계해 각 버킷의 최소·최대·평균 값을 표시함으로써 학습 지표의 가시성을 향상시킵니다. 이를 통해 사용자는 다운샘플링으로 인한 손실 없이 이상치를 쉽게 발견하고, 고충실도(high-fidelity) 세부 정보를 확대해 볼 수 있습니다.  [문서에서 더 알아보기](https://docs.wandb.ai/models/app/features/panels/line-plot/sampling).
  * 이제 Dedicated Cloud 및 Self-Managed 인스턴스에서 [팀 단위 BYOB(secure storage connector)를 위한 크로스 클라우드 스토리지 버킷](https://docs.wandb.ai/platform/hosting/data-security/secure-storage-connector#cross-cloud-or-s3-compatible-storage-for-team-level-byob)을 사용할 수 있습니다. 예를 들어 AWS 상의 W&amp;B 인스턴스에서 팀 단위 BYOB용 Azure Blob Storage 또는 Google Cloud Storage를 구성할 수 있으며, 다른 모든 크로스 클라우드 조합에도 동일하게 적용됩니다.
  * 같은 맥락에서, 이제 Dedicated Cloud 및 Self-Managed 인스턴스에서 [MinIO와 같은 S3 호환 스토리지 버킷을 팀 단위 BYOB(secure storage connector)로](https://docs.wandb.ai/platform/hosting/data-security/secure-storage-connector#cross-cloud-or-s3-compatible-storage-for-team-level-byob) 사용할 수 있습니다. 예를 들어 Google Cloud 상의 W&amp;B 인스턴스에서 클라우드 또는 온프레미스에 호스팅된 MinIO 버킷을 팀 단위 BYOB용으로 구성할 수 있습니다.

  * SCIM API를 사용하는 경우 다음과 같은 몇 가지 사소한 개선 사항도 확인할 수 있습니다:
    인증 실패 시 API가 더 적절한 오류 메시지를 반환합니다.
    관련 엔드포인트는 사용 가능한 경우 SCIM User 객체에 사용자의 전체 이름을 포함해 반환합니다.

  * 아티팩트 다운로드 속도가 대폭 개선되었습니다. 100만 파일 아티팩트 벤치마크에서 6배 이상 빨라졌습니다. SDK 버전 0.15.5 이상으로 업그레이드하세요.
  * (Launch) Optuna가 이제 [Sweeps on Launch에서 사용하는 sweeps 스케줄러](https://docs.wandb.ai/guides/launch/sweeps-on-launch#create-a-custom-sweep-scheduler)로 제공되어, 하이퍼파라미터 탐색을 더 효율적으로 수행할 수 있습니다.
  * Run 데이터 영구 삭제(permadeletion)가 이제 가능합니다(기본값: 비활성). 이는 `GORILLA_DATA_RETENTION_PERIOD` 환경 변수를 시간 단위로 설정하여 활성화할 수 있습니다. 삭제는 되돌릴 수 없으므로, 이 변수를 변경하기 전에 반드시 주의하고 필요 시 W&amp;B Support와 상의하세요. 이 설정으로 아티팩트는 삭제되지 않습니다.
  * 보고서 공유 이메일에 미리보기를 포함하도록 업데이트했습니다.
  * 프로젝트 내 보고서를 위한 HTML sanitization 규칙을 완화했습니다. 이전에는 이로 인해 드물게 보고서 렌더링 문제가 발생하기도 했습니다.
  * 차트 구성에서 정규식으로 매칭할 수 있는 메트릭의 최대 개수를 확장했습니다. 이전에는 항상 10개였으나, 이제 최대 100개까지 가능합니다.
  * 미디어 패널의 step 슬라이더가 표시되는 미디어와 동기화되지 않던 문제를 수정했습니다.
  * 플롯에서 smoothing 옵션으로 시간 가중 지수 이동 평균을 추가했습니다.
  * 워크스페이스의 &quot;Search panels&quot; 텍스트 상자가 이제 사용자의 마지막 검색어를 유지합니다.
  * Run이 그룹화된 상태에서 사용자 이름 필터를 적용해도 더 이상 오류가 발생하지 않습니다.

  * 감사 로그에 리포트 생성 및 업데이트 작업이 추가되었습니다.
  * 이제 SDK를 통해 읽은 아티팩트가 [감사 로그](https://docs.wandb.ai/platform/hosting/audit-logging/)에 기록됩니다.
  * 리포트 생성 시, 새 리포트에 추가할 모든 플롯을 선택하는 버튼이 추가되었습니다.

  * 팀 관리자는 이제 보호된 별칭을 추가할 수 있습니다.
  * 중간 단계를 표시하는 미디어 패널 처리 방식이 개선되었습니다.
  * 익명 사용자의 모델 레지스트리 홈에서 비활성화된 &#39;New Model&#39; 버튼이 제거되었습니다.

  * 모델 레지스트리 온보딩 체크리스트에서 잘못된 진행률 표시기가 수정되었습니다.

  * 환경 변수에 지정된 기간이 지나면 실행과 해당 실행의 메트릭을 영구적으로 삭제하는 옵션이 추가되었습니다.

    <Frame>

    모델 레지스트리 관리자용 신규 역할을 통해 모델 프로모션 프로세스를 제어할 수 있습니다. 이 역할의 사용자는 보호된 별칭 목록(예: &quot;challenger&quot; 또는 &quot;prod&quot;)을 관리하고, 모델 버전에 보호된 별칭을 적용하거나 제거할 수 있습니다.
  * Viewer 역할
    이제 W&amp;B Server용 Viewer 역할이 도입되어, 더 폭넓은 대상과 W&amp;B 결과를 공유할 수 있습니다. 이 역할을 가진 사용자는 팀에서 만든 모든 것을 볼 수 있지만, 생성, 편집 또는 삭제는 할 수 없습니다. 이 시트 수는 기존 W&amp;B Server 시트와는 별도로 계산되므로, 업데이트된 라이선스를 요청하려면 W&amp;B 계정 담당자에게 문의하세요.
  * 공유 기능 개선: 선택적 매직 링크, 뷰어를 위한 간편한 가입
    팀 관리자는 이제 팀 및 팀 구성원에 대해 매직 링크 공유를 끌 수 있습니다. 팀 설정에서 공개 공유를 비활성화하면 팀의 프라이버시 제어를 강화할 수 있습니다. 한편, 리포트 링크를 받은 사용자가 회원 가입 후 W&amp;B에서 해당 리포트에 액세스하는 절차도 더 간단해졌습니다.
  * 리포트 작성 기능 개선
    <Frame>
      ![향상된 초안 게시, 편집 및 협업 기능을 갖춘 개선된 리포트 작성 인터페이스](https://github.com/wandb/server/assets/97066933/c2858fd5-d2f6-47e3-9e93-f8d048df4f59)

    W&amp;B가 지원하는 ML 워크플로우 범위가 확장됨에 따라, 애플리케이션 내에서 이동하기가 어렵다는 피드백을 받았습니다. 이에 따라 제품 영역에 더 명확한 레이블을 추가하고, 일부 상세 화면에는 이전 화면으로 돌아갈 수 있는 링크를 추가하도록 내비게이션 사이드바를 업데이트했습니다. 또한 기능의 강력함을 더 잘 반영하기 위해 &quot;Triggers&quot;를 &quot;Automations&quot;로 이름을 변경했습니다.
