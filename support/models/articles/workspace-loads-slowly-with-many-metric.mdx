---
title: "Why does my workspace load slowly?"
keywords: ["Workspaces", "Metrics"]
---

Workspaces can load slowly when a project has a large number of metrics, runs, or panels. Here are common causes and fixes.

### Too many metrics

If your workspace suddenly slows down, check whether recent runs have unintentionally logged thousands of new metrics. If they have, consider deleting those runs and recreating them with the desired metrics.

For runs that log between 5,000 and 100,000 metrics, switch to a [manual workspace](/models/app/features/panels/#workspace-modes). In manual mode, you can selectively add panels for only the metrics you need to visualize. Metrics that are not plotted are still collected and stored.

### Too many panels

If an automatic workspace includes panels for many logged keys, it may be slow to load. To fix this:

1. Reset the workspace to manual mode.
2. Use [Quick add](/models/app/features/panels/#quick-add) to selectively add panels for the metrics you need.

<Note>
Deleting unused panels one at a time has little impact on performance. Instead, reset the workspace and selectively add back only those panels you need.
</Note>

### Too many sections

Having hundreds of sections hurts performance. Consider using the workspace setting to create sections by prefix rather than suffix, which can result in fewer, higher-level groupings.

### Too many runs

Keep the total number of runs in a single project under 100,000 on W&B Multi-tenant Cloud, or 10,000 on Dedicated Cloud or Self-Managed. If your project exceeds these limits, consider [moving less frequently used runs](/models/runs/manage-runs/) to an archive project.

For more information, see [Experiments limits and performance](/models/track/limits/).
