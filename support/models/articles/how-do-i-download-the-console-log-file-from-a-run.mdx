---
title: "How do I download the console log file from a run?"
keywords: ["Logs", "Runs"]
---

W&B captures your training script's stdout and stderr output and stores it as `output.log` in the run's file storage. You can access and download it from the W&B UI or programmatically via the API.

**From the UI**

1. Open the run page.
2. Click the **Files** tab.
3. Find `output.log` in the file list and click the download icon.

If the run crashed before finishing, the log file may still be available in the Files tab. If the download button is missing on a crashed run, the log was not fully synced before the crash. In this case, check your local `wandb/` directory — W&B writes a local copy of the log to `wandb/run-<timestamp>-<run-id>/logs/output.log`.

**Programmatically via the API**

```python
import wandb

api = wandb.Api()
run = api.run("entity/project/run_id")

# Download output.log to the current directory
run.file("output.log").download(replace=True)
```

**Viewing logs longer than 10,000 lines**

The W&B UI displays up to 10,000 lines of console output. To access the full log for long training jobs, download the file using one of the methods above.

**Resuming the output.log across restarts**

If you resume a run using `wandb.init(resume="allow", id="RUN_ID")`, W&B appends new output to the same `output.log` file. You can download the combined log after all restarts complete.
