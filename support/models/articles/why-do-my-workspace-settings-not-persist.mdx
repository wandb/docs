---
title: "Why do my workspace settings not persist between sessions?"
keywords: ["Workspace", "Runs", "Experiments"]
---

Workspace layout (panels, filters, grouping) persists _only_ when you save a view. Unsaved changes stay in your browser session. To create and manage views, see [Workspaces](/models/track/workspaces).

## Unsaved changes

W&B does not auto-save workspace edits. After changing filters or panels, choose **Views > Save view** or **Save changes** on an existing view. The **Views** dropdown shows an asterisk (`*`) while changes are pending.

## Browser-only vs server-side settings

| Setting | Stored |
|---------|--------|
| Panel layout, filters, grouping, run colors in a view | Server-side (saved view) |
| Browser zoom, personally hidden runs | Local browser only |

Incognito or a different browser loads saved views correctly but not local-only preferences.

## Team projects and conflicting edits

On shared projects, whoever saves last updates the default view. Use **Views > Save as new view** for a personal copy so teammates do not overwrite each other's layouts.
