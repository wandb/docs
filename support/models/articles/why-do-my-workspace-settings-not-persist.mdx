---
title: "Why do my workspace settings not persist between sessions?"
keywords: ["Workspace", "Runs", "Experiments"]
---

Workspace layout changes (panel positions, filters, grouping) are stored in your browser session by default. If you close the tab, refresh, or open the project in a different browser, the workspace reverts to its last *saved* view. The fix is to explicitly save a view before leaving.

**The most common cause: forgetting to save**

W&B does not auto-save workspace state. After configuring filters, panel layout, or grouping, you must click **Views → Save view** (or **Save changes** on an existing view) for the configuration to persist. Unsaved changes are local to your current browser session.

**How to verify your view was saved**

The view name in the **Views** dropdown will show an asterisk (`*`) or a "unsaved changes" indicator if you have pending changes. Once saved, the indicator disappears and the configuration is stored server-side.

**Settings that don't persist per-user vs. per-project**

Some settings are personal (stored in your browser's local storage) while others are project-wide (stored with the view):

| Setting | Where it's stored |
|---|---|
| Panel layout and positions | View (server-side, persists everywhere) |
| Active filters | View (server-side) |
| Grouping configuration | View (server-side) |
| Run color overrides | View (server-side) |
| Browser zoom level | Local browser only |
| Personally hidden runs | Local browser session only |

**Different browsers or incognito mode**

If you're seeing the workspace reset when switching browsers or using incognito mode, it's because browser-local settings (run color, personal overrides) don't carry over. The saved view structure itself will load correctly — only browser-local preferences are lost.

**The workspace reverts to a different configuration on team projects**

If multiple teammates edit the same default view, whoever saves last wins. To avoid conflicts, each team member should fork the shared view into a personal copy (**Views → Save as new view**) and work on their own copy, reserving the shared default view for the agreed-upon team layout.
