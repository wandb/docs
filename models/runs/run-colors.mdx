W&B automatically assigns a color to each run that you create in your project. The following sections describe how to change, randomize, or reset run colors. Custom colors help you visually distinguish runs in the table and graphs in your project workspace.

Run colors are locally scoped. On the project page, custom colors apply only to your own workspace. In reports, custom colors for runs apply only at the section level. You can visualize the same run in different sections, and each section can use different custom colors.

## Edit default run colors

To change the color assigned to an individual run so that it stands out from other runs in the table and graphs:

1. Click the **Runs** tab from the project sidebar.
2. Click the dot color next to the run name in the **Name** column.
3. Select a color from the color palette or the color picker, or enter a hex code.

## Randomize run colors

To randomize the colors of all runs in the table:

1. Click the **Runs** tab from the project sidebar.
2. Hover over the **Name** column header, click the **action (<Icon icon="ellipsis" iconType="solid"/>)** menu, and select **Randomize run colors** from the dropdown menu.

<Note>
The option to randomize run colors is available only after you modify the run table by sorting, filtering, searching, or grouping.
</Note>


## Reset run colors

{/* The option to randomize run colors is only available if the table or selector contains at least two runs and you have modified the view by sorting, filtering, searching, or grouping. */}

To discard your custom run colors and restore the default colors W&B assigned to all runs in the table:

1. Click the **Runs** tab from the project sidebar.
2. Hover over the **Name** column header, click the **action (<Icon icon="ellipsis" iconType="solid"/>)** menu, and select **Reset colors** from the dropdown menu.

<Image src="/images/runs/reset-run-colors.png" alt="Reset run colors in project workspace" />
