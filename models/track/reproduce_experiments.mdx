---
title: Reproduce experiments
description: "Reproduce a teammate's W&B experiment by downloading the associated code, dependencies, and configuration from a run."
---

Reproduce an experiment that a team member creates to verify and validate their results. You can reproduce the experiment by either cloning the GitHub repository that your teammate used to create the experiment or by downloading the Python script or notebook that created the experiment. Choose the option that works best for you.

Obtain the code that created the experiment and the dependencies required to run the code. You can do this by following the steps below.

<Tabs>
<Tab title="Replicate code state with GitHub repository">

The following steps show you how to replicate the code state of the GitHub repository that your teammate used to create the experiment:

1. Navigate to the project where the run is logged to.
1. Select the **Workspace** tab from the project sidebar.
1. Click the run that you want to reproduce. The run page opens with the **Overview** tab shown by default.
1. Hover your mouse over the **Reproduce this run** button next to the **Command** field. A modal appears with steps to reproduce the experiment. It describes how to set up the code state and the the command to run the experiment.

<Tab title="Download source code">

The following steps show you how to download the Python script or notebook that created the experiment:

1. Navigate to the project where the run is logged to.
1. Select the **Workspace** tab from the project sidebar.
1. Click the run that you want to reproduce. The run page opens with the **Overview** tab shown by default.
1. On the **Overview** tab (shown by default), in the **Command** field, make a note of the name of the script that created the experiment.
1. Select the **Code** tab on the run page.
1. Click **Download** next to the file that corresponds to the script or notebook.
1. On the run page, select the **Files** tab.
1. Download the `requirements.txt` file and store it in your working directory. This directory should contain either the cloned GitHub repository or the downloaded Python script or notebook.
1. Install the requirements specified in the `requirements.txt` file.
    ```bash
    pip install -r requirements.txt
    ```


</Tab>
</Tabs>

Once you have the code, you can run the script or notebook to reproduce the experiment. You might need to navigate to the directory where the script or notebook is located before running it.
