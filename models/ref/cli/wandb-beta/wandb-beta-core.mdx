title: "wandb beta core"
---

Manage a shared local wandb-core service for multi-process workloads.

wandb-core is the local backend process that handles run data, file uploads, and system metrics collection. By default, each process that calls `wandb.init()` starts its own backend. On a machine running many independent workers, that duplicates work and wastes CPU and memory.

Use these commands to start one detached wandb-core instance and point multiple workers on the same machine at it with the WANDB_SERVICE environment variable.

Typical workflow:

```bash
$ wandb beta core start
$ export WANDB_SERVICE=printed_value
$ python -m your_launcher
$ wandb beta core stop
```

For shell scripts, capture the raw WANDB_SERVICE value from stdout:

```bash
$ export WANDB_SERVICE="$(wandb beta core start)"
```

The shared service exits after 10 minutes of idleness by default. Override this with --idle-timeout on the start command.

## Usage

```bash
wandb beta core COMMAND [ARGS]...
```

## Commands

| Command | Description |
| :--- | :--- |
| [start](/models/ref/cli/wandb-beta/wandb-beta-core/wandb-beta-core-start) | Start a detached wandb-core service. |
| [stop](/models/ref/cli/wandb-beta/wandb-beta-core/wandb-beta-core-stop) | Stop a detached wandb-core service. |
