Create an evaluation table using the W&B Python SDK.

## Create an evaluation table

Suppose you have a pandas DataFrame with the following data:

```python
import pandas as pd
import wandb

df = pd.DataFrame(
    [
        {
            "image_id": "img_001",
            "true_label": "cat",
            "predicted_label": "cat",
            "confidence": 0.97,
            "correct": True,
        },
        {
            "image_id": "img_002",
            "true_label": "dog",
            "predicted_label": "cat",
            "confidence": 0.72,
            "correct": False,
        },
        {
            "image_id": "img_003",
            "true_label": "car",
            "predicted_label": "car",
            "confidence": 0.89,
            "correct": True,
        },
    ]
)
```

To create and log an evaluation table, pass the DataFrame to `wandb.EvalTable` and specify which columns to use for the inputs, outputs, and scores in the eval table. For example, the following code creates an eval table with the `image_id` column as the input, the `predicted_label` column as the output, and the `correct` and `confidence` columns as scores:

```python
with wandb.init(project="classifier-eval-table-demo") as run:
    eval_table = wandb.EvalTable(
        dataframe=df,
        input_columns=["image_id"],
        output_columns=["predicted_label"],
        score_columns=["correct", "confidence"],
    )
    run.log({"validation_predictions_eval": eval_table})    
```


## Convert a W&B Table to an Eval Table

Use [ARIA](/aria/) to migrate your W&B Table to an Evaluation Table:

1. Navigate to the project that contains the W&B Table you want to migrate.
2. Click on the blue circle on the upper right corner of the page to open the ARIA converstaion window.
3. Type `/convert-eval-table` in the ARIA conversation window.
4. Click the **Send** button, the upward-pointing arrow in the lower-right corner of the chat window.

<Info>
Existing W&B Tables remain unchanged if you convert them to an Eval Table. 
</Info>
