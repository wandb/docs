title: "Python vs TypeScript SDK Comparison"
description: "Feature comparison between Weave Python and TypeScript SDKs"

This page provides a side-by-side comparison of features available in the Weave Python and TypeScript SDKs to help you understand capabilities and limitations when choosing between them.

## Legend

| Symbol | Meaning |
|--------|---------|
| ✅ | Fully supported |
| ⚠️ | Partial support / workaround available |
| ❌ | Not available |

---

## Initialization & Authentication

| Feature | Python | TypeScript | Notes |
|---------|--------|------------|-------|
| Initialize client | ✅ `weave.init()` | ✅ `weave.init()` | |
| Login with API key | ✅ Environment variable | ✅ `weave.login()` | TS has explicit login function |
| Global attributes on init | ✅ `global_attributes` param | ❌ | Use `withAttributes` in TS |
| Custom settings | ✅ `settings` param | ✅ `settings` param | |
| Finish/shutdown | ✅ `weave.finish()` | ❌ | TS uses `waitForBatchProcessing()` |

## Call Management

| Feature | Python | TypeScript | Notes |
|---------|--------|------------|-------|
| Get call by ID | ✅ `client.get_call()` | ✅ `client.getCall()` | |
| Query calls | ✅ `client.get_calls()` | ✅ `client.getCalls()` | |
| Calls iterator | ✅ Pagination built-in | ✅ `client.getCallsIterator()` | |
| Filter calls | ✅ `CallsFilter` | ✅ `CallsFilter` | |
| Include costs | ✅ `include_costs` | ✅ `includeCosts` | |
| Export to pandas | ✅ `.to_pandas()` | ❌ | |
| **Get call during execution** | ✅ `weave.require_current_call()` | ⚠️ `requireCurrentCallStackEntry()` | TS returns limited info |
| **Get call + result together** | ✅ `op.call()` → `(result, call)` | ❌ | Key Python advantage |
| Set display name (after) | ✅ `call.set_display_name()` | ✅ `call.setDisplayName()` | |
| Set display name (on call) | ✅ `__weave={"display_name": ...}` | ❌ | |
| Set display name (decorator) | ✅ `call_display_name` param | ❌ | |
| Delete call | ✅ `call.delete()` | ❌ | |
| Delete multiple calls | ✅ `client.delete_calls()` | ❌ | |
