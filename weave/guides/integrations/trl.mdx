[Transformer Reinforcement Learning (TRL)](https://huggingface.co/docs/trl/en/index) is a full-stack library for training transformer language models. It supports methods such as Supervised Fine-Tuning (SFT), Group Relative Policy Optimization (GRPO), Direct Preference Optimization (DPO), Reward Modeling, and more. The library integrates with Hugging Face Transformers.

Along with using W&B to record your training metrics, you can integrate Weave with your TRL workflows to observe how your model performs during training. Weave records inputs, outputs, and timestamps at each evaluation step so you can inspect the quality of responses generated by the model and track how it changes over time.

<Note>
This integration logs data with the Weave [`EvaluationLogger`](/weave/guides/evaluation/evaluation_logger). Logged predictions appear on the **Traces** page of your project, and scored runs appear on the **Evals** page. If you're instrumenting an agent for the **Agents** view instead, see [Trace your agents](/weave/guides/tracking/trace-agents).
</Note>

## Get started

Install TRL with `uv` or `pip`. Choose one of the following options based on your setup:

```bash
# Using uv
uv pip install trl

# Using pip
pip install trl
```

Then install Weave and W&B using one of the following options based on your setup:

```bash
# Using uv
uv pip install weave wandb

# Using pip
pip install weave wandb
```

## Train models with TRL and log completions with Weave

Once you have installed the necessary libraries, use the built-in [`WeaveCallback`](https://huggingface.co/docs/trl/main/en/callbacks#trl.WeaveCallback) in TRL to log completions at each evaluation step. The callback logs data during evaluation phases, so you must pass an evaluation dataset to the trainer and configure an evaluation schedule in the training arguments.

The following example trains a model with the `GRPOTrainer` and logs completions to Weave at each evaluation step:

```python
import wandb
from datasets import load_dataset
from trl import GRPOConfig, GRPOTrainer, WeaveCallback

# Log in to W&B. Alternatively, set the WANDB_API_KEY environment variable.
wandb.login()

# Load the datasets
train_dataset, eval_dataset = load_dataset(
    "trl-lib/ultrafeedback-prompt", split=["train[:5%]", "test[:5%]"]
)

# Example reward function that rewards completions with more unique letters
def reward_num_unique_letters(completions, **kwargs):
    completion_contents = [completion[0]["content"] for completion in completions]
    return [float(len(set(content))) for content in completion_contents]

# The callback logs during evaluation, so configure an evaluation schedule
training_args = GRPOConfig(
    output_dir="Qwen2-0.5B-GRPO",
    eval_strategy="steps",
    eval_steps=100,
)
trainer = GRPOTrainer(
    model="Qwen/Qwen2-0.5B-Instruct",
    reward_funcs=reward_num_unique_letters,
    args=training_args,
    train_dataset=train_dataset,
    eval_dataset=eval_dataset,
)

# project_name is optional if a W&B run or Weave client is already active
weave_callback = WeaveCallback(trainer=trainer, project_name="trl-grpo-demo")
trainer.add_callback(weave_callback)

trainer.train()
```

## Score completions during training

`WeaveCallback` supports two modes:

- **Tracing mode**: When you don't provide scorers, as in the previous example, the callback logs model predictions for exploration and analysis.
- **Evaluation mode**: When you provide a `scorers` dictionary, the callback logs predictions along with scores and summary metrics.

To use evaluation mode, pass one or more scorer functions with the signature `scorer(prompt: str, completion: str) -> float | int`:

```python
def completion_length(prompt: str, completion: str) -> float:
    return float(len(completion))

weave_callback = WeaveCallback(
    trainer=trainer,
    project_name="trl-grpo-demo",
    scorers={"completion_length": completion_length},
)
trainer.add_callback(weave_callback)
```

## View the results

After training starts, open your Weave project to inspect the logged data:

- The **Traces** page shows the logged predictions from each evaluation step, including inputs, outputs, and timestamps.
- In evaluation mode, the **Evals** page shows the scored evaluation runs so you can compare model quality across evaluation steps.
