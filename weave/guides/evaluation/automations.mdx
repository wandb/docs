---
title: "Set up automations"
description: "Create event-driven automations that trigger actions based on monitor metrics and trace activity."
---

Automations let you define event-driven rules that trigger actions when specific conditions are met in your project. Instead of manually watching dashboards for metric changes, you configure an automation once, and Weave runs the action for you when the event fires. Use cases include:

- **Threshold alerts**: Send a Slack notification when a monitor's average score crosses a threshold.
- **Regression detection**: Alert your team when a scorer detects a drop in accuracy or an increase in toxicity.
- **Deployment gates**: Trigger a webhook when a quality metric exceeds a confidence threshold over a rolling window.
- **Operational monitoring**: Notify on-call engineers when error rates or latency metrics change.

Automations require no code changes to your application. Set them up using the W&B Weave UI.

## End-to-end workflow

The following workflow summarizes how to create and manage automations:

1. Ensure you have at least one active [monitor](/weave/guides/evaluation/monitors) generating scorer results.
2. Create an automation by selecting an event, configuring thresholds, and choosing an action.
3. Review the summary and save the automation.
4. Monitor automation activity from the Automations page or from within a monitor's detail view.
5. Inspect individual automation instances in the automation drawer.


## Create an automation

Creating an automation is a three-step process: configure the triggering event, select the action to perform, and review a summary before saving.

### Step 1: Configure the event

The event configuration defines what condition triggers the automation. You select a monitor metric, a time window, and a threshold.

To start creating an automation:
1. Navigate to [wandb.ai](https://wandb.ai) and open your Weave project.
2. In the Weave project sidebar, click **Automations**.
3. In the Automations page header, click **New automation**.
4. In the **Create automation** panel, configure the event:
    - **Event type**: Select the type of event that triggers this automation, such as a metric threshold.
    - **Event Source**: Select the automation source to be either an **Operation** or a **Monitor** from your project.
        - **Operation**: Select the Op whose traces are evaluated by the monitor. The list contains Ops that have logged at least one trace in the project.
            - **Metric**: Select the metric to measure for the Op.
        - **Monitor**: Select the monitor that produces the metric you want to track.
            - **Output field**: Select the specific numeric scorer output field to evaluate.
    - **Window**: Configure the window used to trigger the automation to be either **Time based** or **Count-based**. Select the aggregation function (such as **Average**) and the rolling period (such as **1 day**).
    - **Threshold**/**Condition**: Define the condition that triggers the automation. Select a comparator (such as "is above") and provide the threshold value (such as `0.9`).
5. Click **Next** to proceed to action configuration.

### Step 2: Select an action

The action configuration defines what happens when the event condition is met.

To configure the action:
1. In the **Step 2 of 3** panel, configure:
    - **Team**
    - **Action type**: Select the type of action to perform, such as sending a **Slack notification**.
    - **Action details**: Depending on the action type, provide the required configuration:
        - For Slack notifications, select the **Slack channel** to send the notification to.
        - For webhooks, provide the endpoint URL and any required headers.
2. Click **Next** to proceed to the summary.

### Step 3: Review and save

The summary step displays a complete overview of the automation before you save it.

To save:
1. Configure the automation metadata:
    - **Name**: Provide a name for the automation.
    - **Description** (Optional): Add a description to help identify the automation's purpose.
2. Click **Create** to create the automation.

The new automation appears in the Automations table on the Automations page.


## View and manage automations

All automations in your project are listed on the **Automations** page. The table displays the following information for each automation:

- **Name**: The automation name and description.
- **Event type**: The type of event that triggers the automation.
- **Action Type**: The action that is triggered.
- **Last updated**: When the automation was last modified.
- **Last instance**: When the automation last triggered.


To view automations:
1. In the Weave project sidebar, click **Automations**.
2. Use the search field in the table toolbar to filter automations by name.


## Create an automation from a monitor

You can also create automations directly from a monitor's detail view. This approach pre-fills the event configuration with the monitor's context, which makes the setup faster.

To create an automation from a monitor:
1. In the Weave project sidebar, click **Monitors**.
2. In the Monitors table, click the name of the monitor to open its detail panel.
3. In the monitor detail panel, select the **Automations** tab to see any existing automations for this monitor.
4. In the Automations section toolbar, click **Create automation**.
5. In the **Create automation** panel, the event configuration is pre-filled with the selected monitor's details. Adjust the configuration as needed:
    - **Output field**: Select the scorer output field to evaluate.
    - **Window**: Configure the aggregation function and rolling period.
    - **Condition**: Define the comparator and threshold value.
6. Click **Next** and complete the remaining steps (select action and save) as described in [Create an automation](#create-an-automation).


## View existing automations

Existing automations can be reviewed in the Automations page.

To inspect automation details:
1. In the Weave project sidebar, click **Automations**.
2. In the table, click the name of an automation to open the automation drawer.
3. In the automation drawer, review the metric or monitor, the threshold information, and the action.


### History
When an automation fires, Weave creates an automation instance that records what happened. You can inspect these instances to understand when and why an automation triggered.

To inspect an automation instance:
1. In the Weave project sidebar, click **Automations**.
1. In the Automations tab bar, click **History**.
1. In the table, click the name of an automation to open the automation drawer.
1. In the automation drawer, review:
    - **Instance details**: A panel showing the event data and context that caused the automation to fire.
    - **Event summary**: The trigger condition that was met, including the metric, time window, and threshold.
    - **Action details**: The action that was performed, such as the Slack channel notified or the webhook endpoint called.

The automation drawer provides a complete audit trail connecting the triggering event to the resulting action.
