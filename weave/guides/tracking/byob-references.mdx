Weave can render media that already lives in your own cloud storage bucket. Instead of uploading bytes, you log the object's `s3://` URI as a plain string. Once you register the bucket as a reference bucket for your team, Weave presigns the URI on read and renders the image or video inline in the trace, exactly like natively logged media.

The bytes never enter Weave. This is different from logging media with the [Content API](/weave/guides/core-types/media#contents-api), which ingests and stores the bytes in Weave.

## Overview

Reference media in your own bucket when you want to:

- **Keep large media in your bucket.** Only the URI is sent to Weave, so the media bytes are never uploaded or stored by Weave.
- **Log instantly.** There is no upload, so logging latency is a short string write no matter how large the asset is.
- **Reuse assets you already store.** If the media is already in your bucket, point Weave at it instead of copying it.

<Note>
This is a Weave-specific feature and is separate from [platform BYOB (secure storage connector)](/platform/hosting/data-security/secure-storage-connector), which backs W&B artifact and run storage. Reference media buckets are configured per team and only need read access.
</Note>

## How it works

1. You store a media object in your bucket, for example `s3://your-bucket/frame.png`.
2. You log that URI as a string value in a trace (an op input/output or an agent tool result).
3. When you open the trace, Weave requests a short-lived [presigned URL](/platform/hosting/data-security/presigned-urls) for the object and renders it inline.

Weave renders a URI as media only when it points at a bucket that is registered for your team, so the same string is inert until you complete registration.

## Prerequisites

- A W&B account and [API key](https://wandb.ai/authorize).
- An AWS S3 bucket you control, containing at least one object.
- Permission to edit your team settings.

## Configure your bucket

Grant W&B read access to the bucket and allow the browser to fetch objects with CORS. For provisioning details and other providers, see the [secure storage connector guide](/platform/hosting/data-security/secure-storage-connector).

1. Apply a bucket policy that lets the W&B integration role read objects. Replace `<your-bucket>` and use the principal for your environment:

   ```json
   {
     "Version": "2012-10-17",
     "Id": "WandBAccess",
     "Statement": [{
       "Sid": "WandbAccess",
       "Effect": "Allow",
       "Principal": {"AWS": "arn:aws:iam::725579432336:role/WandbIntegration"},
       "Action": "s3:GetObject",
       "Resource": "arn:aws:s3:::<your-bucket>/*"
     }]
   }
   ```

2. Apply a CORS configuration so the app can fetch objects in the browser:

   ```json
   [{
     "AllowedHeaders": ["*"],
     "AllowedMethods": ["GET", "HEAD"],
     "AllowedOrigins": ["*"],
     "ExposeHeaders": ["ETag"],
     "MaxAgeSeconds": 3000
   }]
   ```

<Note>
If an object is encrypted with a customer-managed KMS key, its key policy must also grant the same principal `kms:Decrypt`. The bucket policy alone is not sufficient.
</Note>

## Register the reference bucket

1. Go to your team settings page (`https://wandb.ai/<your-team>/settings`).
2. In the **Reference buckets** section, select **Add bucket**.
3. Enter your bucket URI, for example `s3://your-bucket`, and save.

<Warning>
Until the bucket is registered, referenced URIs do not render. Registration is what lets Weave presign and display the object.
</Warning>

## Log a media reference

Log the object's URI as a string value in a trace. No Content annotation is needed, because the bytes stay in your bucket.

<Tabs>
<Tab title="Python">

```python lines
import weave

weave.init("your-team/your-project")

@weave.op
def render_frame(source_uri: str) -> dict[str, str]:
    # Return the URI verbatim; Weave presigns and renders it inline.
    return {"image": source_uri}

render_frame("s3://your-bucket/frame.png")
render_frame("s3://your-bucket/clip.mp4")
```

</Tab>
<Tab title="TypeScript">

```typescript lines
import * as weave from 'weave';

await weave.init('your-team/your-project');

const renderFrame = weave.op(function renderFrame(sourceUri: string) {
  // Return the URI verbatim; Weave presigns and renders it inline.
  return { image: sourceUri };
});

await renderFrame('s3://your-bucket/frame.png');
await renderFrame('s3://your-bucket/clip.mp4');
```

</Tab>
</Tabs>

<Tip>
You can also carry a reference URI in an agent tool result so it renders in the Agents view. See the [custom agents quickstart](/weave/custom-agents-quickstart) for the agent span API.
</Tip>

## View the reference in Weave

Open the trace from the link that `weave.init()` prints. The referenced image or video renders inline in the call view, and referenced media in an agent tool result renders in the **Agents** view. If a URI does not resolve, Weave shows an error placeholder instead of the media, which usually means the object is missing or the bucket is not registered.
