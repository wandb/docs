# 팀

W&B 팀을 사용하여 ML 팀이 더 빠르게 더 나은 모델을 구축할 수 있는 중앙 워크스페이스로 활용하세요.

* 팀이 시도한 **모든 실험을 추적**하여 작업을 중복하지 않습니다.
* 이전에 트레이닝된 모델을 **저장하고 재현**합니다.
* 상사와 협업자와 **진행 상황 및 결과를 공유**합니다.
* 성능이 떨어질 때 즉시 알림을 받아 **회귀를 감지**합니다.
* 모델 성능을 **벤치마크하고 모델 버전을 비교**합니다.

![](/images/app_ui/teams_overview.webp)

## 협업 팀 생성

1. 무료 W&B 계정에 [**가입하거나 로그인**](https://app.wandb.ai/login?signup=true)합니다.
2. 네비게이션 바에서 **팀 초대**를 클릭합니다.
3. 팀을 생성하고 협업자를 초대합니다.

:::info
**참고**: 조직의 관리자만 새 팀을 생성할 수 있습니다.
:::

## 팀 프로필 생성

팀의 프로필 페이지를 사용자 정의하여 소개 및 공개 또는 팀 멤버에게만 보이는 리포트 및 프로젝트를 전시할 수 있습니다. 리포트, 프로젝트 및 외부 링크를 제시하세요.

* 방문자에게 최고의 공개 리포트를 전시하여 **최고의 연구를 강조**하세요.
* 팀원이 쉽게 찾을 수 있도록 **가장 활발한 프로젝트를 전시**하세요.
* 귀하의 회사나 연구실의 웹사이트 및 발표한 논문에 대한 외부 링크를 추가하여 **협력자를 찾으세요**.

## 팀 멤버 제거

팀 관리자는 팀 설정 페이지를 열고 퇴사하는 멤버의 이름 옆에 있는 삭제 버튼을 클릭할 수 있습니다. 사용자가 제거된 후에도 해당 사용자가 팀에 로그한 run은 남아 있습니다.

## 팀 역할 및 권한
팀에 동료를 초대할 때 팀 역할을 선택합니다. 다음과 같은 팀 역할 옵션이 있습니다.

- **관리자**: 팀 관리자는 다른 관리자나 팀 멤버를 추가하거나 제거할 수 있습니다. 모든 프로젝트를 수정하고 완전한 삭제 권한을 가집니다. 이에는 run, 프로젝트, 아티팩트 및 스윕 삭제 등이 포함됩니다.
- **멤버**: 팀의 일반 멤버입니다. 팀 관리자가 이메일로 팀 멤버를 초대합니다. 팀 멤버는 다른 멤버를 초대할 수 없습니다. 팀 멤버는 해당 멤버가 생성한 run 및 스윕 run만 삭제할 수 있습니다. 예를 들어 두 멤버 A와 B가 있습니다. 멤버 B가 팀 B의 프로젝트에서 멤버 A가 소유한 다른 프로젝트로 Run을 이동합니다. 멤버 A는 멤버 B가 멤버 A의 프로젝트로 이동한 Run을 삭제할 수 없습니다. Run을 생성한 멤버 또는 팀 관리자만 run을 삭제할 수 있습니다.
- **서비스 (기업 전용 기능)**: 서비스 워커, run 자동화 툴과 W&B를 사용하기 위한 유용한 API 키입니다. 팀의 서비스 계정에서 API 키를 사용하는 경우, 환경 변수 **WANDB_USERNAME**을 설정하여 run을 올바른 사용자에게 할당해야 합니다. 아래에서 관련 행동에 대해 자세히 보세요.
- **읽기 전용 (기업 전용 기능)**: 읽기 전용 멤버는 팀 내 자산을 볼 수 있습니다. 자산에는 run, 리포트, 워크스페이스가 포함됩니다. 그들은 리포트를 팔로우하고 코멘트할 수 있지만, 프로젝트 개요, 리포트 또는 run을 생성, 편집 또는 삭제할 수 없습니다. 읽기 전용 멤버는 API 키가 없습니다.
- **사용자 정의 역할 (기업 전용 기능)**: 사용자 정의 역할을 통해 조직 관리자는 위의 **읽기 전용** 또는 **멤버** 역할에서 상속하고 추가 권한을 부여하여 세밀한 엑세스 제어를 달성할 수 있는 새로운 역할을 구성할 수 있습니다. 그런 다음 팀 관리자는 해당 사용자 정의 역할을 자신의 팀에 있는 사용자에게 할당할 수 있습니다. 자세한 내용은 [이 기사](https://wandb.ai/wandb_fc/announcements/reports/Introducing-Custom-Roles-for-W-B-Teams--Vmlldzo2MTMxMjQ3)를 참조하세요.

:::note
W&B는 팀에 관리자가 한 명 이상 있을 것을 권장합니다. 주 관리자가 사용할 수 없을 때 관리 작업이 계속될 수 있도록 하는 것이 모범 사례입니다.
:::

:::note
W&B 서버(전용 클라우드 또는 자체 관리 배포)를 사용하는 경우, **사용자 정의 역할** 기능을 사용하려면 업데이트된 기업 라이센스가 필요합니다.
:::

### 팀 서비스 계정 행동

* 교육 환경에서 팀을 구성할 때, 그 팀의 서비스 계정을 사용하여 해당 팀 내의 비공개 또는 공개 프로젝트에 run을 로그할 수 있습니다. 또한, 환경에 **WANDB_USERNAME** 또는 **WANDB_USER_EMAIL** 변수가 구성되어 있고 참조된 사용자가 해당 팀의 일원인 경우, 해당 run을 사용자에게 할당할 수 있습니다.
* 교육 환경에서 팀을 구성하지 않고 서비스 계정을 사용하는 경우, run은 그 서비스 계정의 상위 팀 내 명명된 프로젝트에 로그됩니다. 이 경우에도 환경에 **WANDB_USERNAME** 또는 **WANDB_USER_EMAIL** 변수가 구성되어 있고 참조된 사용자가 서비스 계정의 상위 팀의 일원인 경우, run을 사용자에게 할당할 수 있습니다.
* 서비스 계정은 상위 팀과 다른 팀의 비공개 프로젝트에 run을 로그하는 데 사용될 수 없지만, 다른 팀의 공개 프로젝트에 run을 로그하는 데 사용될 수 있습니다.

## 팀 트라이얼

W&B는 비즈니스 팀을 위한 무료 트라이얼을 제공하며, **신용카드가 필요하지 않습니다**. 트라이얼 기간 동안 당신과 동료들은 W&B의 모든 기능에 엑세스할 수 있습니다. 트라이얼이 끝나면 플랜을 업그레이드하여 협업을 위한 W&B 팀을 계속 사용할 수 있습니다. 개인 W&B 계정은 항상 무료로 유지되며, 학생이나 교사인 경우 학술 플랜에 등록할 수 있습니다.

[가격 페이지](https://wandb.ai/site/pricing)에서 우리의 플랜에 대한 자세한 정보를 확인하세요. 대시보드 UI 또는 [Export API](../../../ref/python/public-api/README.md)를 통해 언제든지 데이터를 다운로드할 수 있습니다.

### 보안 스토리지 커넥터

:::caution
W&B는 현재 버킷을 마이그레이션하는 것을 지원하지 않습니다. 구체적으로 W&B는 다음을 지원하지 않습니다:
* 한 팀 레벨 버킷에서 다른 팀 레벨 버킷으로의 마이그레이션
* W&B SaaS 버킷에서 팀 레벨 버킷으로의 마이그레이션 및 그 반대.
:::

팀 레벨 보안 스토리지 커넥터를 사용하면 팀이 자체 클라우드 스토리지 버킷을 W&B와 함께 사용할 수 있습니다. 이를 통해 매우 민감한 데이터를 가진 팀 또는 엄격한 준수 요구 사항을 가진 팀은 데이터 엑세스 제어 및 데이터 분리를 더욱 강화할 수 있습니다.
