# 프로젝트 페이지

프로젝트 **워크스페이스**는 여러분의 개인적인 실험 비교를 위한 모래상자를 제공합니다. 프로젝트를 사용하여 비교할 수 있는 모델들을 조직화하세요. 같은 문제를 다른 아키텍처, 하이퍼파라미터, 데이터셋, 전처리 등으로 작업합니다.

프로젝트 페이지 탭:

1. [**Overview**](project-page.md#overview-tab): 프로젝트의 스냅샷
2. [**Workspace**](project-page.md#workspace-tab): 개인 시각화 모래상자
3. [**Table**](project-page.md#table-tab): 모든 run의 새로운 전망
4. [**Reports**](project-page.md#reports-tab): 노트, run, 그래프의 저장된 스냅샷
5. [**Sweeps**](project-page.md#sweeps-tab): 자동화된 탐색 및 최적화

## Overview 탭

* **프로젝트 이름**: 프로젝트 이름을 편집하려면 클릭하세요
* **프로젝트 설명**: 프로젝트 설명을 편집하고 노트를 추가하려면 클릭하세요
* **프로젝트 삭제**: 오른쪽 상단의 점 메뉴를 클릭하여 프로젝트를 삭제하세요
* **프로젝트 프라이버시**: 누가 run과 리포트를 볼 수 있는지 편집하세요 - 자물쇠 아이콘을 클릭하세요
* **최근 활동**: 이 프로젝트에 가장 최근에 데이터가 기록된 시간을 확인하세요
* **총 계산량**: 프로젝트의 모든 run 시간을 합산하여 이 총계를 얻습니다
* **run 복구**: 드롭다운 메뉴를 클릭하고 "모든 run 복구"를 클릭하여 프로젝트에서 삭제된 run을 복구하세요.

## Workspace 탭

**Runs 사이드바**: 프로젝트의 모든 run 목록

* **점 메뉴**: 사이드바에서 행 위로 마우스를 옮기면 왼쪽에 메뉴가 나타납니다. 이 메뉴를 사용하여 run의 이름을 변경하거나 run을 삭제하거나 활성 run을 중지하세요.
* **가시성 아이콘**: 눈 모양을 클릭하여 그래프에서 run을 켜고 끕니다
* **색상**: run 색상을 우리의 프리셋 중 하나 또는 사용자 정의 색상으로 변경하세요
* **검색**: 이름으로 run을 검색하세요. 이것은 또한 그래프에서 보이는 run을 필터링합니다.
* **필터**: 사이드바 필터를 사용하여 보이는 run의 집합을 좁힙니다
* **그룹**: 예를 들어 아키텍처에 의해 run을 동적으로 그룹화하기 위해 설정 열을 선택하세요. 그룹화는 그래프에 평균 값에 따른 선과 그래프 상의 점들의 분산에 대한 음영 처리된 영역이 나타나게 합니다.
* **정렬**: 예를 들어 가장 낮은 손실이나 가장 높은 정확도를 가진 run에 의해 run을 정렬할 값을 선택하세요. 정렬은 그래프에 나타나는 run에 영향을 미칩니다.
* **확장 버튼**: 사이드바를 전체 테이블로 확장합니다
* **Run 수**: 괄호 안의 숫자는 프로젝트의 총 run 수입니다. 숫자 (N 시각화됨)는 눈이 켜져 있고 각 그래프에서 시각화할 수 있는 run의 수입니다. 아래 예에서, 그래프는 183 run 중 처음 10개만 보여줍니다. 그래프에서 보이는 run의 최대 수를 늘리려면 그래프를 편집하세요.

**패널 레이아웃**: 이 스크래치 공간을 사용하여 결과를 탐색하고, 차트를 추가 및 제거하며, 다른 메트릭을 기반으로 모델의 버전을 비교하세요

[실제 예제 보기 →](https://app.wandb.ai/example-team/sweep-demo)

### 패널 섹션 추가

섹션 드롭다운 메뉴를 클릭하고 "섹션 추가"를 클릭하여 패널의 새 섹션을 생성하세요. 섹션을 이름을 변경하고, 재구성하고, 확장 및 축소할 수 있습니다.

각 섹션은 오른쪽 상단에 옵션이 있습니다:

* **커스텀 레이아웃으로 전환**: 커스텀 레이아웃을 사용하면 패널을 개별적으로 크기 조정할 수 있습니다.
* **표준 레이아웃으로 전환**: 표준 레이아웃을 사용하면 섹션의 모든 패널을 한 번에 크기 조정할 수 있으며 페이징을 제공합니다.
* **섹션 추가**: 드롭다운 메뉴에서 위나 아래에 섹션을 추가하거나 페이지 하단의 버튼을 클릭하여 새 섹션을 추가하세요.
* **섹션 이름 변경**: 섹션의 제목을 변경하세요.
* **리포트로 섹션 내보내기**: 이 패널 섹션을 새 리포트에 저장하세요.
* **섹션 삭제**: 모든 차트와 함께 전체 섹션을 제거하세요. 이는 워크스페이스 막대의 페이지 하단에 있는 실행 취소 버튼으로 실행 취소할 수 있습니다.
* **패널 추가**: 섹션에 패널을 추가하려면 플러스 버튼을 클릭하세요.

![](@site/static/images/app_ui/add-section.gif)

### 섹션 간에 패널 이동

패널을 드래그 앤 드롭하여 섹션으로 재정렬하고 구성하세요. 패널의 오른쪽 상단에 있는 "이동" 버튼을 클릭하여 패널을 이동할 섹션을 선택할 수도 있습니다.

![](@site/static/images/app_ui/move-panel.gif)

### 패널 크기 조정

* **표준 레이아웃**: 모든 패널은 동일한 크기를 유지하며 패널의 페이지가 있습니다. 패널의 오른쪽 하단을 클릭하고 드래그하여 패널 크기를 조정할 수 있습니다. 섹션의 오른쪽 하단을 클릭하고 드래그하여 섹션 크기를 조정하세요.
* **커스텀 레이아웃**: 모든 패널은 개별적으로 크기가 지정되며 페이지가 없습니다.

![](@site/static/images/app_ui/resize-panel.gif)

## Artifacts 탭

트레이닝 데이터셋과 [파인튜닝된 모델](../../model_registry/intro.md)부터 [메트릭 및 미디어의 테이블](../../tables/tables-walkthrough.md)에 이르기까지 프로젝트와 관련된 모든 [아티팩트](../../artifacts/intro.md)을 보세요.

### 개요 패널

![](/images/app_ui/overview_panel.png)

개요 패널에서는 아티팩트의 이름과 버전, 변경 사항 감지 및 중복 방지를 위해 사용되는 해시 다이제스트, 생성 날짜, 모든 에일리어스를 포함한 다양한 고급 정보를 찾을 수 있습니다. 여기에서 에일리어스를 추가하거나 제거하고, 버전뿐만 아니라 아티팩트 전체에 대해 노트를 작성할 수 있습니다.

### 메타데이터 패널

![](/images/app_ui/metadata_panel.png)

메타데이터 패널은 아티팩트가 구성될 때 제공된 아티팩트의 메타데이터에 액세스를 제공합니다. 이 메타데이터에는 아티팩트를 재구성하는 데 필요한 설정 인수, 추가 정보를 찾을 수 있는 URL 또는 아티팩트를 기록한 run 동안 생성된 메트릭이 포함될 수 있습니다. 추가로, 아티팩트를 생성한 run의 설정 및 아티팩트 기록 시점의 이력 메트릭을 볼 수 있습니다.

### 사용 패널

웹 앱 외부에서, 예를 들어 로컬 기계에서 아티팩트를 사용하기 위해 다운로드하는 코드 조각이 제공되는 사용 패널입니다. 이 섹션에서는 또한 아티팩트를 출력한 run과 아티팩트를 입력으로 사용하는 모든 run을 나타내고 링크합니다.

### 파일 패널

![](/images/app_ui/files_panel.png)

파일 패널은 아티팩트와 관련된 파일과 폴더 목록을 나열합니다. 이 파일 트리를 탐색하고 W&B 웹 앱에서 직접 내용을 볼 수 있습니다.

아티팩트와 연관된 [테이블](../../tables/tables-walkthrough.md)은 이 컨텍스트에서 특히 풍부하고 상호 작용적입니다. 아티팩트와 함께 테이블을 사용하는 방법에 대해 [여기](../../tables/visualize-tables.md)에서 자세히 알아보세요.

![](/images/app_ui/files_panel_table.png)

### 계보 패널

![](/images/app_ui/lineage_panel.png)

계보 패널은 프로젝트와 관련된 모든 아티
