<div id="evaluation-playground">
  # Playground d’évaluation
</div>

Le Playground d’évaluation vous permet d’accéder à des modèles existants et de comparer leurs performances à l’aide de jeux de données d’évaluation et de juges de notation LLM. Vous pouvez ainsi commencer à tester et à comparer vos modèles sans avoir à écrire de code. Vous pouvez également enregistrer les modèles, Scorers et jeux de données que vous développez dans le playground pour les faire évoluer et les déployer plus tard.

Par exemple, vous pouvez ouvrir le Playground d’évaluation, ajouter deux modèles que vous avez déjà enregistrés, puis évaluer leurs performances à l’aide d’un nouveau jeu de données d’évaluation de type questions-réponses ou d’un jeu de données déjà enregistré. Vous pouvez ensuite ajouter un nouveau modèle dans l’interface, lui ajouter des prompts système, puis lancer une nouvelle évaluation sur les trois modèles pour voir comment ils se comparent les uns aux autres.

  ![Interface du Playground d’évaluation](/weave/guides/evaluation/img/eval-playground-ui.png)
</Frame>

<div id="set-up-an-evaluation-in-the-playground">
  ## Configurer une évaluation dans le playground
</div>

Pour configurer une évaluation dans le playground d’évaluation :

1. Ouvrez [l’interface Weave](https://wandb.ai/wandb/quickstart_playground/weave/playground), puis le projet dans lequel vous souhaitez effectuer l’évaluation. Cela ouvre la page Traces.
2. Depuis la page Traces, cliquez sur l’icône **Playground** dans le menu de gauche, puis sélectionnez l’onglet **Evaluate** sur la page Playground. Sur la page Evaluate, vous pouvez soit :
   * **Charger un exemple de démonstration** : cela charge une configuration prédéfinie qui évalue le modèle MoonshotAI Kimi K2 par rapport à la sortie attendue et utilise un juge LLM pour en déterminer l’exactitude. Vous pouvez utiliser cette configuration pour vous familiariser avec l’interface.
   * **Partir de zéro** : cela charge une configuration vide à partir de laquelle vous pouvez construire.
3. Si vous avez sélectionné **Partir de zéro**, ajoutez un titre et une description explicites pour votre évaluation dans les champs **Title** et **Description**.

Suivez les instructions des sections suivantes pour configurer vos jeux de données, modèles et Scorers.

<div id="add-a-dataset">
  ### Ajouter un jeu de données
</div>

[Datasets](../core-types/datasets) sont des collections organisées d&#39;exemples d&#39;entrées utilisateur et de réponses attendues de vos modèles. Pendant l&#39;évaluation, le playground envoie chaque entrée de test à votre modèle, collecte sa sortie, puis calcule un score pour cette sortie en fonction de la métrique que vous avez sélectionnée, par exemple l&#39;exactitude. Vous pouvez choisir de créer un jeu de données dans l&#39;interface utilisateur, d&#39;ajouter un jeu de données existant déjà enregistré dans votre projet, ou de téléverser un nouveau jeu de données.

Vous pouvez téléverser des jeux de données dans les formats suivants :

* `.csv`
* `.tsv`
* `.json`
* `.jsonl`

Voir les pages Datasets pour plus d&#39;informations sur la façon de formater et d&#39;enregistrer des jeux de données dans Weave.

Pour ajouter un jeu de données dans la section **Dataset** :

1. Cliquez sur le menu déroulant, puis sélectionnez l&#39;une des options suivantes :
   * **partir de zéro** pour créer un nouveau jeu de données dans l&#39;interface utilisateur.
   * **Upload a file** pour téléverser un jeu de données depuis votre machine locale.
   * Un jeu de données existant déjà enregistré dans votre projet.
2. Facultatif : cliquez sur **Save** pour enregistrer le jeu de données dans votre projet afin de l&#39;utiliser ultérieurement.

Une fois que vous avez sélectionné une option, le jeu de données apparaît dans le panneau de droite de l&#39;interface utilisateur, et vous pouvez modifier chaque champ selon vos besoins en cliquant dessus. Vous pouvez également ajouter de nouvelles lignes au jeu de données en cliquant sur **Add row**.

<Note>
  Vous ne pouvez utiliser l&#39;interface utilisateur que pour modifier de nouveaux jeux de données.

  Il est également important de nommer correctement les colonnes de votre jeu de données `user_input` et `expected_output` afin que les Scorers puissent accéder aux données.
</Note>

<div id="add-a-model">
  ### Ajouter un modèle
</div>

[Models](/fr/weave/guides/core-types/models), dans le contexte de Weave, correspond à l’association d’un modèle d’IA (comme GPT) et de l’environnement (dans ce cas, le prompt système) qui définit la manière dont le modèle se comporte pendant l’évaluation. Vous pouvez sélectionner des modèles existants dans votre projet ou en créer de nouveaux à évaluer, et vous pouvez ajouter plusieurs modèles à la fois pour les évaluer simultanément avec le même jeu de données et le même scorer. **Vous pouvez uniquement utiliser des modèles créés à l’aide de la fonctionnalité playground.**

Pour ajouter un modèle dans la section **Models** du playground d’évaluation :

1. Cliquez sur **Add Model**, puis sélectionnez **New Model** ou un modèle existant dans le menu déroulant.

2. Si vous avez sélectionné **New Model**, configurez les champs suivants :

   * **Name** : ajoutez un nom descriptif à votre nouveau modèle.
   * **LLM Model** : sélectionnez un modèle de base sur lequel construire votre nouveau modèle, comme GPT-4 d’OpenAI. Vous pouvez choisir dans une liste de modèles de base pour lesquels vous avez déjà configuré l’accès, ou ajouter l’accès à un modèle de base en sélectionnant **Add AI provider**, puis un modèle. L’ajout d’un fournisseur vous invite à saisir vos identifiants d’accès pour ce fournisseur. Consultez la documentation de votre fournisseur pour savoir comment trouver votre clé API, vos endpoints et toute information de configuration supplémentaire nécessaire pour accéder au modèle avec Weave.
   * **System Prompt** : fournissez au modèle des instructions sur la manière dont il doit se comporter, par exemple `You are a helpful assistant specializing in Python programming.` Le `user_input` de votre jeu de données est envoyé dans un message ultérieur et vous n’avez pas besoin de l’inclure dans le prompt système.

   Si vous choisissez un modèle existant, un nouveau champ apparaît à côté du nom du modèle et vous permet de sélectionner une version du modèle existant, sans autre champ à configurer. Si vous souhaitez modifier votre modèle existant avant ou après l’évaluation, utilisez le [Prompt Playground](../tools/playground).

3. Facultatif : cliquez sur **Save** pour enregistrer le modèle dans votre projet afin de le réutiliser plus tard.

4. Facultatif : vous pouvez ajouter d’autres modèles à évaluer simultanément en cliquant de nouveau sur **Add Model** et en ajoutant autant d’autres modèles que nécessaire.

<div id="add-scorers">
  ### Ajouter des Scorers
</div>

Les [Scorers](../evaluation/scorers) utilisent des juges LLM pour mesurer et évaluer la qualité des sorties des modèles d’IA. Vous pouvez sélectionner des Scorers existants dans votre projet ou en créer de nouveaux pour évaluer vos modèles.

Pour ajouter un Scorer dans le playground d’évaluation :

1. Cliquez sur **Ajouter un Scorer**, puis configurez les champs suivants :
   * **Nom** : ajoutez un nom descriptif à votre Scorer.
   * **Type** : sélectionnez le format de sortie des scores : un booléen ou un nombre. Les Scorers booléens renvoient `True` ou `False` selon que la sortie du modèle respecte ou non les paramètres d’évaluation que vous avez définis. Les Scorers numériques produisent un score compris entre `0` et `1`, fournissant une note générale sur la mesure dans laquelle la sortie du modèle respecte vos paramètres d’évaluation.
   * **LLM-as-a-judge-model** : sélectionnez un modèle de base à utiliser comme juge pour le Scorer. Comme pour le champ LLM Model dans la section **Models**, vous pouvez choisir parmi les modèles de base auxquels vous avez déjà configuré l’accès, ou configurer l’accès à de nouveaux modèles de base.
   * **Scoring Prompt** : indiquez au juge LLM les critères sur lesquels il doit évaluer la sortie. Par exemple, si vous voulez qu’il vérifie la présence d’hallucinations, vous pouvez saisir un prompt d’évaluation semblable à celui-ci :

     ```
     Given the following context and answer, determine if the answer contains any information not supported by the context.

     User input: {user_input}
     Expected output: {expected_output}
     Model Output: {output}

     Is the model output correct?
     ```

     Vous pouvez utiliser les champs de vos jeux de données et de vos réponses comme variables dans le prompt d’évaluation, par exemple `{user_input}`, `{expected_output}` et `{output}`. Pour voir une liste des variables disponibles, cliquez sur **Insérer une variable** dans l’interface utilisateur.

2. Facultatif : cliquez sur **Save** pour enregistrer le Scorer dans votre projet afin de l’utiliser plus tard.

<div id="run-the-evaluation">
  ### Exécuter l’évaluation
</div>

Une fois que vous avez configuré vos jeux de données, modèles et Scorers, vous pouvez exécuter l’évaluation.

* Pour exécuter l’évaluation dans le playground d’évaluation, cliquez sur **Run eval**.

Weave exécute une évaluation distincte pour chaque modèle que vous avez ajouté et collecte des métriques pour chaque requête effectuée à partir du jeu de données. Weave enregistre chacune de ces évaluations dans la section **Evals** pour que vous puissiez les consulter ultérieurement.

<div id="review-evaluation-results">
  ### Examiner les résultats de l’évaluation
</div>

Une fois l’évaluation terminée, le playground ouvre un rapport qui affiche diverses métriques collectées pour chaque requête envoyée à vos modèles.

<Frame>
  ![Evals hero](/images/weave/evals-hero.png)
</Frame>

L’onglet **Résultats du jeu de données** affiche l’entrée, la sortie attendue, la sortie réelle du modèle, la latence, l’utilisation des jetons et les scores obtenus. Vous pouvez cliquer sur les ID dans la colonne **Ligne** pour ouvrir une vue détaillée des métriques pour un ensemble précis de requêtes. Vous pouvez également utiliser les boutons de format d’affichage situés juste sous les onglets pour modifier le format d’affichage des cellules du rapport.

L’onglet **Résumé** fournit un aperçu des performances de chaque modèle à l’aide de représentations visuelles des données.

Pour plus d’informations sur l’ouverture et la comparaison des évaluations, voir [Evaluations](../core-types/evaluations).
