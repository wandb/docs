---
title: "Définir et enregistrer des attributs"
description: "Utilisez des attributs pour ajouter des métadonnées à vos traces et évaluations."
---

Dans Weave, les attributs vous permettent d&#39;associer des métadonnées personnalisées à vos traces et évaluations. Ces métadonnées peuvent inclure des informations comme des noms d&#39;environnement, des versions de modèle, des ID d&#39;expérience, des ID d&#39;utilisateur ou d&#39;autres informations contextuelles pour vous aider à organiser, filtrer et analyser vos données Weave.

Les attributs sont particulièrement utiles lorsque vous devez regrouper ou filtrer les traces en fonction des éléments suivants :

* Déploiements
* tenants
* Experiments

Weave propose deux façons d&#39;ajouter des attributs :

* **Attributs par appel** : utilisez `weave.attributes()` pour ajouter des métadonnées à des opérations spécifiques ou à des blocs de code
* **Attributs globaux** : utilisez le champ `global_attributes` pour définir des attributs lors de l&#39;initialisation, qui s&#39;appliquent à toutes les traces et évaluations de votre projet

Vous pouvez afficher tous les attributs enregistrés au cours des traces et des évaluations dans l&#39;UI. Vous pouvez ensuite les utiliser pour filtrer et regrouper les données.

<Tip>
  `call.attributes` ne peut pas être modifié une fois l&#39;appel commencé. Utilisez ce
  gestionnaire de contexte pour définir les métadonnées avant d&#39;invoquer l&#39;op.
</Tip>

</div>

<div id="global-attributes">
  ## Attributs globaux
</div>

Lorsque vous définissez des attributs globaux lors de l’initialisation de Weave, ils s’appliquent automatiquement à **toutes** les traces et évaluations de votre projet. C’est utile pour transmettre des métadonnées à l’échelle du projet, comme l’environnement, la version du déploiement ou les informations sur l’équipe.

<div id="get-attributes-during-execution">
  ## Obtenir des attributs pendant l’exécution
</div>

Renvoie l’ensemble actuel des attributs enregistrés pour votre appel. Cela peut vous aider à déboguer vos appels ou à exploiter le contexte dans une logique conditionnelle.

<Tabs>
  <Tab title="Python">
    L’exemple suivant utilise le décorateur Weave pour enregistrer la fonction `process_data`, configure les attributs à enregistrer, puis les renvoie à l’exécution.

    Cela produit :

    ```shell
    Attributes: {'env': 'production', 'version': '1.0.0', 'region': 'us-west-2'}
    Attributes: {'env': 'production', 'version': '1.1.0', 'region': 'us-west-2', 'experiment': 'exp-456'}
    ```

    <Note>
      `weave.get_current_call()` fonctionne uniquement **à l’intérieur** d’une fonction décorée avec `@weave.op`. En dehors d’un op, il renvoie `None`.
    </Note>
  </Tab>

  <Tab title="TypeScript">
    Avec le SDK TypeScript de Weave, vous pouvez obtenir les attributs actuels avec `client.getCurrentAttributes()`. Contrairement au SDK Python de Weave, vous pouvez accéder aux attributs TypeScript dans un contexte `withAttributes()`, et pas uniquement à l’intérieur d’un op.

    Cela produit :

    ```shell
    Current attributes: { env: 'production', version: '1.0.0', region: 'us-west-2' }
    Attributes inside op: { env: 'production', version: '1.0.0', region: 'us-west-2' }
    Nested attributes: { env: 'production', version: '1.1.0', region: 'us-west-2', experiment: 'exp-456' }
    Attributes inside op: { env: 'production', version: '1.1.0', region: 'us-west-2', experiment: 'exp-456' }
    ```
  </Tab>
</Tabs>
