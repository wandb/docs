<div id="log-traces-during-model-training-runs">
  # Enregistrer des traces pendant les runs d’entraînement
</div>

<Frame>
  ![Trace Weave dans les espaces de travail](/images/weave/traces-in-workspace.png)
</Frame>

Vous pouvez désormais enregistrer des traces Weave pendant les runs d’entraînement et les afficher dans votre espace de travail personnel.

Les espaces de travail personnels sont des tableaux de bord personnalisables dans l’interface utilisateur qui vous permettent d’afficher des analyses détaillées et des visualisations de données de vos runs d’entraînement. En ajoutant des panneaux Weave à votre espace de travail, vous pouvez afficher et consulter les données de trace enregistrées pendant les runs afin de mieux comprendre les performances de vos modèles pendant l’entraînement.

Par exemple, si vous effectuez le Fine-tuning d’un modèle LLM et que votre espace de travail W&amp;B montre que la précision chute à l’étape 500, les traces Weave intégrées peuvent révéler exactement ce qui s’est passé : votre modèle a commencé à générer des réponses trop verbeuses qui ne respectaient plus les critères d’évaluation.

<div id="use-weave-panels">
  ## Utiliser les panneaux Weave
</div>

[En décorant les fonctions avec `@weave.op`](/fr/weave/quickstart) dans vos pipelines d’apprentissage automatique, vous capturez automatiquement leurs informations d’exécution et y accédez dans un Workspace personnel.

Par exemple, le script suivant montre comment les traces Weave s’intègrent aux runs d’entraînement W&amp;B. Il simule une boucle d’entraînement en apprentissage automatique dans laquelle chaque étape d’entraînement envoie des métriques à W&amp;B tout en créant simultanément des traces Weave détaillées. Les décorateurs `@weave.op` sur les fonctions capturent automatiquement les entrées, les sorties et les détails d’exécution. Lorsque la boucle d’entraînement envoie des métriques avec `wandb.log()`, elle envoie également les informations de trace Weave à votre projet.

```python lines
import wandb
import weave
import random

# Initialiser W&B et Weave avec le même projet
project = "my-workspace-project"
weave.init(project)

@weave.op
def evaluate_model(model_state, epoch):
    # Simuler les métriques d'évaluation
    accuracy = 0.7 + epoch * 0.02 + random.uniform(-0.05, 0.05)
    
    # Suivre les comportements spécifiques du modèle
    test_responses = {
        "conciseness_check": "Model generated 500 words for simple question",
        "accuracy_check": "Model answered 8/10 questions correctly",
        "hallucination_check": "No factual errors detected"
    }
    
    return {
        "accuracy": accuracy,
        "diagnostic_results": test_responses
    }

@weave.op
def training_step(epoch, lr):
    # Simuler la logique d'entraînement
    loss = 1.0 / (epoch + 1) + random.uniform(-0.1, 0.1)
    
    # Évaluation avec traces
    eval_results = evaluate_model(f"checkpoint_{epoch}", epoch)
    
    return {
        "loss": loss,
        "accuracy": eval_results["accuracy"],
        "diagnostics": eval_results["diagnostic_results"]
    }

# Boucle d'entraînement
with wandb.init(project=project, config={"lr": 0.01}) as run:
    for epoch in range(5):
        # Exécuter l'entraînement avec le Tracing Weave
        results = training_step(epoch, run.config.lr)
        
        # Journaliser vers W&B - crée le point d'intégration
        run.log({
            "epoch": epoch,
            "loss": results["loss"],
            "accuracy": results["accuracy"]
        })
```

Vous pouvez exécuter ce script d’exemple pour voir comment les traces sont enregistrées avec le reste des informations de votre run.

Pour afficher les informations de trace dans votre Workspace pendant ou après un run, vous pouvez soit ouvrir le lien fourni dans le terminal au début d’un run (cela ressemble à ceci : `wandb: 🚀 View run at https://wandb.ai/wandb/my-project/runs/<run-ID>`) soit accéder au Workspace dans l’interface utilisateur.

Pour accéder à un Workspace depuis l’interface utilisateur :

1. Ouvrez l’interface utilisateur W&amp;B et cliquez sur l’onglet **Projects**. Cela ouvre une liste de vos projets.
2. Dans la liste des projets, cliquez sur le projet dans lequel vous avez enregistré votre run. Cela ouvre la page **Workspaces**.
3. Si vous avez configuré votre Workspace comme un Workspace automatique, le Workspace se remplit automatiquement avec des visualisations de données et des informations sur votre run. Les Données de trace de votre run se trouvent dans la section **Weave** du Workspace. Si votre Workspace est un Workspace manuel, vous pouvez ajouter des panneaux Weave en cliquant sur **Ajouter des panneaux** et en sélectionnant de nouveaux panneaux dans la section **Weave** du menu Add Panels.

Pour plus d’informations sur les Workspaces, voir [Afficher les résultats des expériences](https://docs.wandb.ai/models/track/workspaces).

<div id="reference-wb-artifacts-in-a-weave-trace">
  ## Référencer des artefacts W&amp;B dans une trace Weave
</div>

Vous pouvez référencer des [artefacts W&amp;B](/fr/models/artifacts) (tels que des modèles, des jeux de données et des points de contrôle) dans vos traces Weave. Cela crée un lien cliquable dans l’interface Weave qui mène directement à la page de détails de l’artefact et vous aide à suivre les versions d’artefact utilisées au cours d’opérations spécifiques.

Par exemple, si vous avez entraîné deux versions d’un modèle (`v1` et `v2`), vous pouvez ajouter la référence de l’artefact `v1` comme attribut de la trace lorsque vous interrogez ce modèle. Cela vous aide à identifier précisément quelle version d’artefact a produit une sortie donnée.

Pour associer un artefact à une trace, construisez l’URL de l’artefact en concaténant le préfixe `wandb-artifact:///` avec le nom complet de l’artefact selon la syntaxe suivante : `wandb-artifact:///<full-name-of-artifact>`. Passez-la ensuite à Weave de l’une des deux façons suivantes :

* Comme attribut arbitraire
* Comme attribut d’un `Model`

Vous pouvez trouver le nom complet de l’artefact sur sa page de détails dans l’interface utilisateur W&amp;B. Il ressemble à ceci : `<entity>/<project>/<artifact_name>:<version>`. Vous pouvez copier le nom complet depuis la page de détails de l’artefact dans l’interface utilisateur W&amp;B.

<div id="add-it-as-an-attribute-of-a-trace">
  ### Ajoutez-le comme attribut d’une trace
</div>

Pour ajouter un artefact comme attribut d’une trace, transmettez-le sous forme de paire clé-valeur au gestionnaire de contexte `weave.attributes()` :

<Tabs>
  <Tab title="Python">
    ```python lines {10}
    import weave

    weave.init("your-team-name/your-project-name")

    @weave.op
    def my_function(name: str):
        return f"Hello, {name}!"

    # Ajoutez un nom de clé arbitraire et l’URL de l’artefact comme valeur
    with weave.attributes({'artifact_id': 'wandb-artifact:///team-name/project-name/run-38m4t5ja-history:v0'}):
        result = my_function("World")
    ```
  </Tab>

  <Tab title="TypeScript">
    ```typescript lines {12}
    import {init, op, withAttributes} from 'weave';

    async function main() {
      await init('wandb/docs');

      const myFunction = op(async function myFunction(name: string) {
        return `Hello, ${name}!`;
      });

      // Ajoutez un nom de clé arbitraire et l’URL de l’artefact comme valeur
      const result = await withAttributes(
        {artifact_id: "wandb-artifact:///team-name/project-name/run-38m4t5ja-history:v0"},
        async () => myFunction('World')
      );

      console.log('Result:', result);
    }

    main().catch(console.error);
    ```
  </Tab>
</Tabs>

<div id="add-an-artifact-as-an-attribute-of-a-model">
  ### Ajouter un artefact en tant qu’attribut d’un `Model`
</div>

Pour ajouter un artefact en tant qu’attribut d’un `Model`, passez l’URL de l’artefact lors de l’instanciation du modèle :

<Note>
  La classe `Model` n’est actuellement prise en charge que dans le SDK Python de Weave.
</Note>

```python lines {7,17}
import weave

weave.init("your-team-name/your-project-name")

class MyModel(weave.Model):
    # Stocker la référence de l'artefact en tant qu'attribut du modèle
    artifact_id: str
    temperature: float = 0.7
    
    @weave.op
    def predict(self, query: str) -> str:
        # Votre logique d'inférence du modèle ici
        return f"Response to: {query}"

# Passer l'URL de l'artefact en tant qu'attribut du modèle
model = MyModel(
    artifact_id="wandb-artifact:///team-name/project-name/run-38m4t5ja-history:v0"
)

result = model.predict("Hello, World!")
```
