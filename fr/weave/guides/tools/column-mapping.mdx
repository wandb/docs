    Dans le SDK Python de Weave, utilisez l&#39;attribut `column_map` sur un scorer pour faire correspondre les noms de paramètres attendus aux noms de colonnes de votre jeu de données. Le format de mappage est `{scorer_parameter: dataset_column}`.

    Pour en savoir plus sur le mappage de colonnes pour les scorers, voir [Mappage des noms de colonnes avec `column_map`](../evaluation/scorers#mapping-column-names-with-column_map).
  </Tab>

  <Tab title="TypeScript">
    Dans le SDK TypeScript de Weave, le mappage des colonnes se configure sur l&#39;objet `Evaluation` à l&#39;aide de l&#39;option `columnMapping`, et non sur chaque scorer individuellement. Le format de mappage est `{scorer_key: dataset_column}`.

    L&#39;exemple suivant associe `expectedOutputTimesTwo` (utilisé par le scorer) à la colonne `expected` du jeu de données :

    ```typescript
    import * as weave from 'weave';

    const myScorer = weave.op(
      ({modelOutput, datasetRow}) => {
        return modelOutput * 2 === datasetRow.expectedOutputTimesTwo;
      },
      {name: 'myScorer'}
    );

    const myEval = new weave.Evaluation({
      id: 'my-evaluation',
      dataset: [{expected: 2}],
      scorers: [myScorer],
      columnMapping: {expectedOutputTimesTwo: 'expected'},
    });
    ```

    Pour en savoir plus sur les arguments des scorers en TypeScript, voir [Arguments nommés du scorer](../evaluation/scorers#scorer-keyword-arguments).
