---
title: "Désactiver le traçage"
description: "Découvrez les options permettant de désactiver le traçage W&B Weave ou de le désactiver conditionnellement"
---

Plusieurs options sont disponibles pour contrôler le niveau de traçage Weave dans votre application, selon votre environnement et vos besoins.

<div id="environment-variable">
  ## Variable d’environnement
</div>

Si vous souhaitez désactiver systématiquement le traçage pour tout le programme, vous pouvez définir la variable d’environnement `WEAVE_DISABLED=true`.

`WEAVE_DISABLED` n’est lu qu’une seule fois, au moment de la définition de la fonction. Cette variable ne peut pas être utilisée pour activer ou désactiver le traçage à l’exécution.

<div id="client-initialization">
  ## Initialisation du client
</div>

Vous pouvez parfois vouloir activer ou désactiver le traçage pour une initialisation donnée en fonction d’une condition. Dans ce cas, initialisez le client avec l’indicateur `disabled` dans les paramètres d’initialisation.

<Tabs>
  <Tab title="Python">
    ```python lines
    import weave

    # Initialiser le client
    client = weave.init(..., settings={"disabled": True})
    ```
  </Tab>

  <Tab title="TypeScript">
    ```plaintext
    Cette fonctionnalité n’est pas encore disponible en TypeScript.
    ```
  </Tab>
</Tabs>

<div id="context-manager">
  ## Gestionnaire de contexte
</div>

Pour désactiver conditionnellement le traçage sur un bloc de code spécifique, vous pouvez utiliser un gestionnaire de contexte de traçage. Utilisez `with tracing_disabled()` pour supprimer le traçage **uniquement pour les appels de fonction exécutés à l’intérieur du bloc `with`**. Utilisez-le dans le code de l’application pour définir la portée des appels qui ne doivent pas être enregistrés.

<Tabs>
  <Tab title="Python">
    ```python lines
    import weave
    from weave.trace.context.call_context import tracing_disabled

    client = weave.init('your-team/your-project-name')

    @weave.op
    def my_op():
        ...

    with tracing_disabled():
        my_op()
    ```
  </Tab>

  <Tab title="TypeScript">
    ```plaintext
    Cette fonctionnalité n’est pas encore disponible en TypeScript.
    ```
  </Tab>
</Tabs>

Bien que le comportement du traçage soit fixé au moment où les fonctions sont définies, cette approche permet de le contrôler à l’exécution lorsqu’elle est combinée à la logique de l’application. Par exemple, vous pouvez placer le gestionnaire de contexte dans une condition pour activer ou désactiver dynamiquement le traçage en fonction d’une valeur disponible à l’exécution :

<Tabs>
  <Tab title="Python">
    ```python lines
    if should_trace:
        my_op()
    else:
        with tracing_disabled():
            my_op()
    ```
  </Tab>

  <Tab title="TypeScript">
    ```plaintext
    Cette fonctionnalité n’est pas encore disponible en TypeScript.
    ```
  </Tab>
</Tabs>
