Cette page explique comment publier, récupérer, supprimer et référencer des objets versionnés dans W&amp;B Weave. Utilisez-la lorsque vous devez suivre des données structurées telles que des jeux de données, des modèles ou des prompts d&#39;un run à l&#39;autre et au fil du temps.

<div id="objects">
  ## Objets
</div>

Un **objet** désigne des données sérialisables et versionnées. Weave versionne automatiquement les objets à chaque modification et crée un historique immuable. Les objets comprennent :

* **Datasets** : collections d&#39;exemples pour l&#39;évaluation
* **Models** : configurations et paramètres pour la logique de votre LLM
* **Prompts** : modèles de prompt versionnés

```python lines
dataset = weave.Dataset(
    name="test-cases",
    rows=[
        {"input": "What is 2+2?", "expected": "4"},
        {"input": "What is the capital of France?", "expected": "Paris"},
    ]
)
weave.publish(dataset)
```


<div id="publish-an-object">
  ## Publier un objet
</div>

La couche de sérialisation de Weave enregistre les objets et en gère les versions.

<Tabs>
  <Tab title="Python">
    ```python lines
    import weave
    weave.init("your-team-name/your-project-name")
    # Enregistrer une liste sous le nom 'cat-names'
    weave.publish(['felix', 'jimbo', 'billie'], 'cat-names')
    ```
  </Tab>

  <Tab title="TypeScript">
    La prise en charge de la publication en TypeScript est limitée. Tous les objets ne sont pas pris en charge.

    ```typescript lines
    import * as weave from 'weave'

    const client = await weave.init("your-team-name/your-project-name")

    // Enregistrer un tableau sous le nom 'cat-names'
    client.publish(['felix', 'jimbo', 'billie'], 'cat-names')
    ```
  </Tab>
</Tabs>

Lorsque vous enregistrez un objet sous un nom, Weave en crée la première version s’il n’existe pas encore.

<div id="get-an-object-back">
  ## Récupérer un objet
</div>

Après publication, vous pouvez récupérer un objet stocké à l’aide de sa référence.

<Tabs>
  <Tab title="Python">
    `weave.publish()` renvoie une réf. Vous pouvez appeler `.get()` sur n’importe quelle réf. pour récupérer l’objet.

    Vous pouvez créer une réf., puis récupérer l’objet.

    ```python lines
    weave.init("your-team-name/your-project-name")
    cat_names = weave.ref('cat-names').get()
    ```
  </Tab>

  <Tab title="TypeScript">
    ```plaintext lines
    Cette fonctionnalité n’est pas encore disponible en TypeScript.
    ```
  </Tab>
</Tabs>

<div id="delete-an-object">
  ## Supprimer un objet
</div>

Si vous n’avez plus besoin d’une version spécifique d’un objet, vous pouvez la supprimer à l’aide de sa réf.

<Tabs>
  <Tab title="Python">
    Pour supprimer une version d&#39;un objet, appelez `.delete()` sur la réf. de l&#39;objet.

    ```python lines
    weave.init("your-team-name/your-project-name")
    cat_names_ref = weave.ref('cat-names:v1')
    cat_names_ref.delete()
    ```

    L&#39;accès à un objet supprimé renvoie une erreur. La résolution d&#39;un objet qui référence un objet supprimé renvoie un `DeletedRef` à la place de l&#39;objet supprimé.
  </Tab>

  <Tab title="TypeScript">
    ```plaintext
    Cette fonctionnalité n'est pas encore disponible en TypeScript.
    ```
  </Tab>
</Tabs>

<div id="construct-object-refs">
  ## Construire des réf. d&#39;objet
</div>

Les réf. permettent d&#39;identifier de manière unique un objet stocké et sa version. Les sections suivantes décrivent la structure de l&#39;URI et les différentes façons de construire une réf.

Dans Weave, une URI de réf. d&#39;objet complète se présente comme suit :

```text
weave:///[YOUR-TEAM-NAME]/[YOUR-PROJECT-NAME]/object/[OBJECT-NAME]:[OBJECT-VERSION]
```

* `[YOUR-TEAM-NAME]` : entité W&amp;B (nom d&#39;utilisateur ou nom d&#39;équipe)
* `[YOUR-PROJECT-NAME]` : projet W&amp;B
* `[OBJECT-NAME]` : nom de l&#39;objet
* `[OBJECT-VERSION]` : soit un hachage de version, une chaîne comme `v0` ou `v1`, soit un alias comme `:latest`. Tous les objets ont l&#39;alias `:latest`.

Vous pouvez construire des réf. de plusieurs manières :

* `weave.ref([NAME])` : récupère la version `:latest` d&#39;un objet. Nécessite d&#39;appeler `weave.init(...)`.
* `weave.ref([NAME]:[VERSION])` : récupère la version spécifiée d&#39;un objet. Nécessite d&#39;appeler `weave.init(...)`.
* `weave.ref([FULLY-QUALIFIED-REF-URI])` : récupère l&#39;objet situé à l&#39;URI de réf. d&#39;objet complète indiquée. Ne nécessite pas d&#39;appeler `weave.init()`."
