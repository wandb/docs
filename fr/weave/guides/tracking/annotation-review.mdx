Les files d&#39;attente d&#39;annotation offrent une interface de révision ciblée aux experts métier. Vous examinez un élément à la fois, consultez le contexte fourni et soumettez des retours structurés à l&#39;aide de champs prédéfinis. En terminant une file d&#39;attente, vous apportez des évaluations d&#39;expert que votre équipe peut utiliser pour évaluer et améliorer les sorties de LLM.

Vous n&#39;avez pas besoin de comprendre le modèle sous-jacent ni le système de Tracing pour effectuer ces révisions. Cela permet à des experts externes d&#39;examiner les données sans avoir besoin d&#39;un contexte de projet plus large.

<div id="annotation-workflow">
  ## Flux de travail d&#39;annotation
</div>

Cette section décrit le processus complet d&#39;examen des éléments d&#39;une file d&#39;attente qui a été partagée avec vous.

En tant qu&#39;annotateur, votre tâche consiste à évaluer l&#39;exactitude des réponses du LLM. Weave enregistre vos retours sous forme d&#39;annotations dans les données.

Pour examiner une file d’annotation :

1. Ouvrez la file d’annotation partagée à l&#39;aide d&#39;un lien de file d&#39;attente fourni par votre équipe.
2. Examinez chaque élément. Vous pouvez avancer ou revenir en arrière dans la file d&#39;attente à tout moment.
3. Soumettez une annotation à l&#39;aide des champs d&#39;annotation fournis.

Weave enregistre automatiquement votre progression après l&#39;envoi d&#39;un élément. Si vous devez faire une pause et reprendre votre travail plus tard, l&#39;examen reprend automatiquement au premier élément incomplet. Une fois que tous les éléments ont été examinés, le tableau de la file d’annotation indique que le travail est terminé.

<div id="review-a-queue-item">
  ### Examiner un élément de la file d&#39;attente
</div>

Ce qui suit décrit l&#39;interface de révision et la manière de soumettre une annotation pour un seul élément de la file d&#39;attente.

Pour chaque élément, l&#39;interface de révision affiche deux volets :

* Un volet de trace affichant le contexte d&#39;entrée sélectionné (comme des prompts, des documents ou des images), ainsi que la réponse ou la décision correspondante du modèle.
* Un volet d&#39;élément contenant un formulaire de saisie d&#39;annotation avec des champs d&#39;annotation.

Remplissez chaque champ d&#39;annotation conformément aux instructions fournies pour la file d&#39;attente, puis cliquez sur **Soumettre et suivant** pour enregistrer cet élément et passer au suivant.

</Frame>

<Tip>
  Dans l&#39;en-tête du volet de trace, vous pouvez cliquer sur le suffixe de l&#39;ID de trace (par exemple, &quot;b3ea&quot;) pour ouvrir un panneau qui affiche les détails complets de la trace pour l&#39;appel, où vous pouvez inspecter la trace et afficher ou comparer l&#39;annotation fournie.
</Tip>

Si un annotateur a déjà examiné un élément de la file d&#39;attente, les en-têtes des deux volets de l&#39;interface de révision affichent les indicateurs **Annotation présente** et **Réponse soumise**.

Vous pouvez toujours fournir une annotation pour un élément déjà examiné par quelqu&#39;un d&#39;autre. Dans ce cas, le bouton d&#39;envoi de l&#39;annotation affiche toujours **Modifier l&#39;annotation**, mais l&#39;envoi crée une nouvelle entrée d&#39;annotation avec vos réponses.

<Note> Si vous soumettez des retours pour un élément déjà examiné par quelqu&#39;un d&#39;autre, vos retours sont stockés **en plus** de ceux de l&#39;annotateur d&#39;origine. </Note>
