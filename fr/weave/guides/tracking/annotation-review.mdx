---
title: "Examiner les éléments d’une file d’annotation"
description: "Évaluez les éléments de trace et soumettez des commentaires structurés à l’aide d’une interface de révision simplifiée."
---

Les files d’annotation offrent une interface d’examen ciblée pour les experts métier. Vous examinez un élément à la fois, consultez le contexte fourni et soumettez des commentaires structurés à l’aide de champs prédéfinis.

Vous n’avez pas besoin de comprendre le modèle sous-jacent ni le système de traçage pour effectuer ces examens. Cela permet à des experts externes d’examiner uniquement les données, sans avoir besoin d’une connaissance plus large du projet.

<div id="annotation-workflow">
  ## Flux de travail d’annotation
</div>

En tant qu’annotateur, votre tâche consiste à examiner la précision des réponses du LLM. Vos commentaires sont enregistrés sous forme d’annotations dans les données.

Pour examiner une file d’annotation :

1. Ouvrez la file d’annotation partagée à l’aide d’un lien vers la file d’annotation fourni par votre équipe.
2. Examinez chaque élément. Vous pouvez avancer ou revenir en arrière dans la file d’annotation à tout moment.
3. Soumettez une annotation à l’aide des champs d’annotation fournis.

Weave enregistre automatiquement votre progression après l’envoi d’un élément. Si vous devez faire une pause et reprendre votre travail plus tard, l’examen reprend automatiquement au premier élément incomplet. Une fois tous les éléments examinés, le tableau de la file d’annotation indique que le travail est terminé.

<div id="review-a-queue-item">
  ### Examiner un élément de la file d’annotation
</div>

Pour chaque élément, l’interface de révision affiche deux volets :

* Un volet de trace affichant le contexte d’entrée sélectionné (par exemple des prompts, des documents ou des images), ainsi que la réponse ou la décision correspondante du modèle.
* Un volet d’élément contenant un formulaire d’annotation avec des champs d’annotation.

Remplissez chaque champ d’annotation conformément aux instructions fournies pour la file d’annotation, puis cliquez sur **Soumettre et passer au suivant** pour enregistrer cet élément et continuer avec le suivant.

</Frame>

<Tip>
  Dans l’en-tête du volet de trace, vous pouvez cliquer sur le suffixe de l’ID de trace (par exemple, &quot;b3ea&quot;) pour ouvrir un panneau affichant les détails complets de la trace pour l’appel, où vous pouvez inspecter la trace et afficher ou comparer l’annotation fournie.
</Tip>

Si un annotateur a déjà examiné un élément de la file d’annotation, les en-têtes des deux volets de l’interface de révision affichent les indicateurs **Has annotation** et **Response submitted**.

Vous pouvez toujours fournir une annotation pour un élément déjà examiné par quelqu’un d’autre. Dans ce cas, le bouton de soumission de l’annotation affiche toujours **Edit annotation**, mais l’envoi crée une nouvelle entrée d’annotation avec vos réponses.

<Note> Si vous soumettez un retour pour un élément déjà examiné par quelqu’un d’autre, votre retour est enregistré **en plus de** celui de l’annotateur d’origine. </Note>
