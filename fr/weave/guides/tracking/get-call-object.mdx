---
title: "Obtenir une référence à l'objet Call pendant l'exécution"
description: "Accédez à l'objet Call de W&B Weave au moment de l'exécution pour les retours, les noms d'affichage et d'autres métadonnées"
---

Dans Weave, lorsque vous utilisez une op, vous pouvez appeler les fonctions directement, comme n&#39;importe quelle autre fonction :

<CodeGroup>
  ```python Python lines
  @weave.op
  def my_op():
      ...

  my_op()
  ```

  ```typescript Typescript lines
  function myFunction() {
      ...
  }

  const myFunctionOp = weave.op(myFunction)
  ```
</CodeGroup>

Cependant, vous pouvez aussi accéder directement à l&#39;objet Call en invoquant la méthode `op.call`, qui renvoie à la fois le résultat et l&#39;objet `Call`.

<Tabs>
  <Tab title="Python">
    ```python lines
    @weave.op
    def my_op():
    ...

    output, call = my_op.call()
    ```

    À partir de là, l&#39;objet `call` contient toutes les informations sur le Call, y compris les entrées, les sorties et d&#39;autres métadonnées. Vous pouvez utiliser `call` pour définir ou mettre à jour des propriétés, récupérer des propriétés supplémentaires ou ajouter des retours.

    Si votre op est une méthode d&#39;une classe, vous devez passer l&#39;instance de la classe comme premier argument à `call`. L&#39;exemple suivant montre comment obtenir une référence à un objet Call correspondant à une méthode de classe :

    ```python lines
    import weave

    # Initialiser Weave Tracing
    weave.init("intro-example")

    class MyClass:
        # Décorer votre méthode
        @weave.op
        def my_method(self, name: str):
            return f"Hello, {name}!"

    instance = MyClass()

    # Passer `instance` comme premier argument à `call`.
    result, call = instance.my_method.call(instance, "World")
    ```
  </Tab>

  <Tab title="TypeScript">
    ```plaintext
    Cette fonctionnalité n'est pas encore disponible dans le SDK TypeScript.
    ```
  </Tab>
</Tabs>
