---
title: "Contrôler le suivi automatique des appels LLM"
description: "Contrôlez la façon dont W&B Weave enregistre automatiquement les appels à OpenAI, Anthropic et à d'autres bibliothèques LLM"
---

Si vous utilisez des bibliothèques de fournisseurs de LLM (comme OpenAI, Anthropic, Cohere ou Mistral) dans votre application, l’autopatching permet à Weave de prendre en charge automatiquement le traçage de tous vos appels LLM. Lorsque vous appelez `weave.init()`, Weave intercepte automatiquement (patche) les bibliothèques clientes LLM prises en charge. Le code de votre application reste inchangé : vous utilisez le SDK du fournisseur comme d’habitude, et chaque requête est enregistrée en tant qu’appel Weave. Vous bénéficiez ainsi d’un traçage complet avec une configuration minimale.

Cette page explique quand et comment modifier ce comportement : désactiver le suivi automatique, le limiter à certains fournisseurs précis ou post-traiter les entrées et les sorties (par exemple, pour masquer des informations personnelles identifiables).

<div id="default-behavior">
  ## Comportement par défaut
</div>

Par défaut, Weave patche automatiquement et suit les appels aux bibliothèques LLM courantes comme `openai` et `anthropic`. Appelez `weave.init(...)` au début de votre programme et utilisez ces bibliothèques normalement. Leurs appels apparaîtront dans les traces de votre projet.

<div id="configure-autopatching">
  ## Configurer le patching automatique
</div>

<Tabs>
  <Tab title="Python">
    <Warning>
      L&#39;argument `autopatch_settings` est obsolète. Utilisez `implicitly_patch_integrations=False` pour désactiver le patching implicite, ou appelez des fonctions de patch spécifiques comme `patch_openai(settings={...})` pour configurer les paramètres de chaque intégration.
    </Warning>

    Weave fournit par défaut **un patching implicite automatique** pour toutes les intégrations prises en charge :

    **Patching implicite (automatique) :** les bibliothèques sont automatiquement patchées, quel que soit le moment où elles sont importées.

    ```python lines
    # Option 1 : importer avant weave.init()
    import openai
    import weave
    weave.init('your-team-name/your-project-name')  # OpenAI est automatiquement patché !

    # Option 2 : importer après weave.init()
    import weave
    weave.init('your-team-name/your-project-name')
    import anthropic  # Automatiquement patché via le hook d'importation !
    ```

    **Désactiver le patching implicite :** vous pouvez désactiver le patching automatique si vous préférez un contrôle explicite.

    ```python lines
    import weave

    # Option 1 : via le paramètre settings
    weave.init('your-team-name/your-project-name', settings={'implicitly_patch_integrations': False})

    # Option 2 : via la variable d'environnement
    # Définissez WEAVE_IMPLICITLY_PATCH_INTEGRATIONS=false avant d'exécuter votre script

    # Lorsque le patching implicite est désactivé, vous devez patcher explicitement les intégrations
    import openai
    weave.patch_openai()  # Désormais requis pour le traçage OpenAI
    ```

    **Patching explicite (manuel) :** vous pouvez patcher explicitement les intégrations pour un contrôle plus fin.

    ```python lines
    import weave
    weave.init('your-team-name/your-project-name')
    weave.integrations.patch_openai()  # Activer le traçage OpenAI
    weave.integrations.patch_anthropic()  # Activer le traçage Anthropic
    ```

    ### Post-traiter les entrées et les sorties

    Vous pouvez personnaliser la façon dont les entrées et les sorties sont enregistrées (par exemple, pour masquer des données personnelles ou des secrets) en transmettant des paramètres à la fonction de patch :

    ```python lines
    import weave.integrations

    def redact_inputs(inputs: dict) -> dict:
        if "email" in inputs:
            inputs["email"] = "[REDACTED]"
        return inputs

    weave.init(...)
    weave.integrations.patch_openai(
        settings={
            "op_settings": {"postprocess_inputs": redact_inputs}
        }
    )
    ```

    Pour en savoir plus sur la gestion des données sensibles, voir [Comment utiliser Weave avec des données personnelles identifiables](/fr/weave/cookbooks/pii).
  </Tab>

  <Tab title="TypeScript">
    Le SDK TypeScript ne prend en charge le patching automatique que pour OpenAI et Anthropic. OpenAI est automatiquement patché lorsque vous importez Weave et ne nécessite aucune configuration supplémentaire.

    De plus, le SDK TypeScript ne prend pas en charge :

    * la configuration ou la désactivation du patching automatique ;
    * le post-traitement des entrées/sorties.

    Pour les cas limites où le patching automatique ne fonctionne pas (ESM, bundlers comme Next.js), utilisez un encapsulage explicite :

    ```typescript
    import OpenAI from 'openai'
    import * as weave from 'weave'
    import { wrapOpenAI } from 'weave'

    const client = wrapOpenAI(new OpenAI())
    await weave.init('your-team-name/your-project-name')
    ```

    Pour plus de détails sur la configuration ESM et le dépannage, voir le [Guide d&#39;intégration du SDK TypeScript](/fr/weave/guides/integrations/js).
  </Tab>
</Tabs>
