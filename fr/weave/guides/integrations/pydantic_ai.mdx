Vous pouvez tracer les appels d’agent et d’outil de [PydanticAI](https://ai.pydantic.dev/) dans Weave avec [OpenTelemetry (OTEL)](https://opentelemetry.io/). PydanticAI est un framework Python pour agents, créé par l’équipe Pydantic pour faciliter la création d’applications d’IA générative prêtes pour la production, avec un typage sûr. Il utilise OTEL pour le traçage de tous les appels d’agent et d’outil.

<Tip>
  Pour plus d’informations sur le traçage OTEL dans Weave, voir [Envoyer les traces OTEL vers Weave](../tracking/otel).
</Tip>

Ce guide vous montre comment tracer les appels d’agent et d’outil de PydanticAI avec OTEL et visualiser ces traces dans Weave. Vous apprendrez à installer les dépendances requises, à configurer un traceur OTEL pour envoyer des données vers Weave, et à instrumenter vos agents et outils PydanticAI. Vous verrez également comment activer le traçage par défaut pour tous les agents de votre application.

<div id="prerequisites">
  ## Prérequis
</div>

Avant de commencer, installez les dépendances OTEL requises :

```bash
pip install opentelemetry-sdk OTELemetry-exporter-otlp-proto-http
```

Ensuite, [configurez le traçage OTEL dans Weave](#configure-otel-tracing-in-weave).

<div id="configure-otel-tracing-in-weave">
  ### Configurer le traçage OTEL dans Weave
</div>

Pour envoyer des traces de PydanticAI vers Weave, configurez OTEL avec un `TracerProvider` et un `OTLPSpanExporter`. Configurez l&#39;exportateur avec le [point de terminaison correct et les en-têtes HTTP requis pour l&#39;authentification et l&#39;identification du projet](#required-configuration).

<Warning>
  Il est recommandé de stocker les variables d&#39;environnement sensibles, comme votre clé API et les informations de votre projet, dans un fichier d&#39;environnement (par exemple, `.env`), puis de les charger à l&#39;aide de `os.environ`. Cela permet de sécuriser vos identifiants et d&#39;éviter de les inclure dans votre codebase.
</Warning>

<div id="required-configuration">
  ### Configuration requise
</div>

* **Point de terminaison :** `https://trace.wandb.ai/otel/v1/traces`
* **En-têtes :**
  * `Authorization` : authentification Basic avec votre clé API W&amp;B
  * `project_id` : le nom de votre entité/projet W&amp;B (par exemple, `myteam/myproject`)

<div id="example-set-up">
  ### Exemple de configuration
</div>

L’extrait de code suivant montre comment configurer un exportateur de spans OTLP et un fournisseur de traceurs afin d’envoyer des traces OTEL depuis une application PydanticAI vers Weave.

```python lines
import base64
import os
from opentelemetry.exporter.otlp.proto.http.trace_exporter import OTLPSpanExporter
from opentelemetry.sdk import trace as trace_sdk
from opentelemetry.sdk.trace.export import SimpleSpanProcessor

# Charger les valeurs sensibles depuis les variables d'environnement
WANDB_BASE_URL = "https://trace.wandb.ai"
PROJECT_ID = os.environ.get("WANDB_PROJECT_ID")  # Votre nom d'entité/projet W&B, par ex. "myteam/myproject"
WANDB_API_KEY = os.environ.get("WANDB_API_KEY")  # Votre clé API W&B

OTEL_EXPORTER_OTLP_ENDPOINT = f"{WANDB_BASE_URL}/otel/v1/traces"
AUTH = base64.b64encode(f"api:{WANDB_API_KEY}".encode()).decode()

OTEL_EXPORTER_OTLP_HEADERS = {
    "Authorization": f"Basic {AUTH}",
    "project_id": PROJECT_ID,
}

# Créer l'exportateur de spans OTLP avec le point de terminaison et les en-têtes
exporter = OTLPSpanExporter(
    endpoint=OTEL_EXPORTER_OTLP_ENDPOINT,
    headers=OTEL_EXPORTER_OTLP_HEADERS,
)

# Créer un fournisseur de traceur et ajouter l'exportateur
tracer_provider = trace_sdk.TracerProvider()
tracer_provider.add_span_processor(SimpleSpanProcessor(exporter))
```

<div id="trace-pydanticai-agents-with-otel">
  ## Tracer des agents PydanticAI avec OTEL
</div>

Pour tracer vos agents PydanticAI et envoyer des données de trace à Weave, passez un objet `InstrumentationSettings` configuré avec votre fournisseur de traceurs au `constructeur Agent`. Cela garantit que tous les appels d&#39;agent et d&#39;outil sont tracés conformément à votre configuration OTEL.

L&#39;exemple suivant montre comment créer un agent simple avec le traçage activé. L&#39;étape clé consiste à définir l&#39;argument instrument lors de l&#39;initialisation de l&#39;agent :

```python lines
from pydantic_ai import Agent
from pydantic_ai.models.instrumented import InstrumentationSettings

# Créer un agent PydanticAI avec le traçage OTEL
agent = Agent(
    "openai:gpt-4o",
    instrument=InstrumentationSettings(tracer_provider=tracer_provider),
)

result = agent.run_sync("What is the capital of France?")
print(result.output)
```

Tous les appels à l’agent sont tracés et envoyés à Weave.

<div id="trace-pydanticai-tools-with-otel">
  ## Tracer les outils PydanticAI avec OTEL
</div>

Weave peut tracer toutes les opérations PydanticAI instrumentées avec OTEL, y compris les appels d’agent et d’outil. Cela signifie que lorsque votre agent invoque un outil (par exemple, une fonction décorée avec `@agent.tool_plain`), l&#39;ensemble de l&#39;interaction est capturé et visualisé dans Weave, y compris les entrées et sorties de l&#39;outil, ainsi que le raisonnement du modèle.

L&#39;exemple suivant montre comment créer un agent avec un prompt système et un outil. Le traçage est automatiquement activé à la fois pour l&#39;agent et pour l&#39;outil :

```python lines
from pydantic_ai import Agent
from pydantic_ai.models.instrumented import InstrumentationSettings

# Créer un agent PydanticAI avec un prompt système et le traçage OTEL
agent = Agent(
    "openai:gpt-4o",
    system_prompt=(
        "You are a helpful assistant that can multiply numbers. "
        "When asked to multiply numbers, use the multiply tool."
    ),
    instrument=InstrumentationSettings(tracer_provider=tracer_provider),
)

# Définir un outil
@agent.tool_plain
def multiply(a: int, b: int) -> int:
    """Multiplier deux nombres."""
    return a * b

# Demander à l'agent d'utiliser l'outil
result = agent.run_sync("What is 7 multiplied by 8?")
print(result.output)
```

L’appel de l’agent comme l’appel d’outil sont tracés dans Weave, ce qui vous permet d’inspecter l’intégralité du raisonnement et du déroulement de l’exécution de votre application.

<div id="instrument-all-agents-by-default">
  ## Instrumentez tous les agents par défaut
</div>

Pour appliquer le traçage OTEL à tous les agents PydanticAI de votre application, utilisez la méthode `Agent.instrument_all()`. Cette méthode définit une instance `InstrumentationSettings` par défaut pour tout agent qui ne spécifie pas explicitement le paramètre `instrument`.

```python lines
from pydantic_ai import Agent
from pydantic_ai.models.instrumented import InstrumentationSettings

# Configurer l'instrumentation par défaut pour tous les agents
Agent.instrument_all(InstrumentationSettings(tracer_provider=tracer_provider))

# Désormais, tout nouvel agent utilisera cette instrumentation par défaut
agent1 = Agent("openai:gpt-4o")
agent2 = Agent("openai:gpt-4o", system_prompt="Be helpful.")

result = agent1.run_sync("What is the capital of France?")
print(result.output)
```

C&#39;est utile pour les applications de plus grande taille, où vous souhaitez un traçage cohérent pour tous les agents sans avoir à répéter la configuration. Pour plus de détails, voir la [documentation OTEL de PydanticAI](https://ai.pydantic.dev/logfire/#using-logfire).
