Vous pouvez tracer les appels d&#39;agents et d&#39;outils [PydanticAI](https://ai.pydantic.dev/) dans Weave en utilisant [OpenTelemetry (OTEL)](https://opentelemetry.io/). PydanticAI est un framework Python pour agents, développé par l&#39;équipe Pydantic afin de simplifier et de sécuriser la création d&#39;applications prêtes pour la production avec l&#39;IA générative. Il utilise OTEL pour tracer tous les appels d&#39;agents et d&#39;outils.

<Tip>
  Pour plus d’informations sur le tracing OTEL dans Weave, voir [Envoyer des traces OTEL vers Weave](../tracking/otel).
</Tip>

Ce guide vous montre comment tracer les appels d&#39;agents et d&#39;outils PydanticAI en utilisant OTEL et visualiser ces traces dans Weave. Vous apprendrez comment installer les dépendances requises, configurer un traceur OTEL pour envoyer des données à Weave, et instrumenter vos agents et outils PydanticAI. Vous verrez également comment activer le traçage par défaut pour tous les agents de votre application.

<div id="prerequisites">
  ## Prérequis
</div>

Avant de commencer, installez les dépendances OTEL requises :

```bash
pip install opentelemetry-sdk OTELemetry-exporter-otlp-proto-http
```

Puis, [configurez le tracing OTEL dans Weave](#configure-otel-tracing-in-weave).

<div id="configure-otel-tracing-in-weave">
  ### Configurer le tracing OTEL dans Weave
</div>

Pour envoyer des traces de PydanticAI vers Weave, configurez OTEL avec un `TracerProvider` et un `OTLPSpanExporter`. Configurez l&#39;exportateur pour le [point de terminaison correct et les en-têtes HTTP pour l&#39;authentification et l&#39;identification du projet](#required-configuration).

<Warning>
  Il est recommandé de stocker les variables d&#39;environnement sensibles, comme votre clé API et les informations du projet, dans un fichier d&#39;environnement (par exemple, `.env`), et de les charger en utilisant `os.environ`. Cela permet de garder vos identifiants sécurisés et à l&#39;abri de votre base de code.
</Warning>

<div id="required-configuration">
  ### Configuration requise
</div>

* **Point de terminaison :** `https://trace.wandb.ai/otel/v1/traces`
* **En-têtes :**
  * `Authorization` : Authentification de base avec votre clé API W&amp;B
  * `project_id` : Votre nom d&#39;entité/projet W&amp;B (par exemple, `myteam/myproject`)

<div id="example-set-up">
  ### Exemple de configuration
</div>

L&#39;extrait de code suivant montre comment configurer un exportateur de spans OTLP et un fournisseur de traceurs pour envoyer des traces OTEL d&#39;une application PydanticAI vers Weave.

```python lines
import base64
import os
from opentelemetry.exporter.otlp.proto.http.trace_exporter import OTLPSpanExporter
from opentelemetry.sdk import trace as trace_sdk
from opentelemetry.sdk.trace.export import SimpleSpanProcessor

# Charger les valeurs sensibles depuis les variables d'environnement
WANDB_BASE_URL = "https://trace.wandb.ai"
PROJECT_ID = os.environ.get("WANDB_PROJECT_ID")  # Votre nom d'entité/projet W&B, par exemple « myteam/myproject »
WANDB_API_KEY = os.environ.get("WANDB_API_KEY")  # Votre clé API W&B

OTEL_EXPORTER_OTLP_ENDPOINT = f"{WANDB_BASE_URL}/otel/v1/traces"
AUTH = base64.b64encode(f"api:{WANDB_API_KEY}".encode()).decode()

OTEL_EXPORTER_OTLP_HEADERS = {
    "Authorization": f"Basic {AUTH}",
    "project_id": PROJECT_ID,
}

# Créer l'exportateur de spans OTLP avec le point de terminaison et les en-têtes
exporter = OTLPSpanExporter(
    endpoint=OTEL_EXPORTER_OTLP_ENDPOINT,
    headers=OTEL_EXPORTER_OTLP_HEADERS,
)

# Créer un fournisseur de traceurs et y ajouter l'exportateur
tracer_provider = trace_sdk.TracerProvider()
tracer_provider.add_span_processor(SimpleSpanProcessor(exporter))
```

<div id="trace-pydanticai-agents-with-otel">
  ## Tracer les agents PydanticAI avec OTEL
</div>

Pour tracer vos agents PydanticAI et envoyer les données de trace vers Weave, passez un objet `InstrumentationSettings` configuré avec votre fournisseur de traceurs au constructeur `Agent`. Cela garantit que tous les appels d&#39;agents et d&#39;outils sont tracés selon votre configuration OTEL.

L&#39;exemple suivant montre comment créer un agent simple avec le tracing activé. L&#39;étape clé est de définir l&#39;argument instrument lors de l&#39;initialisation de l&#39;agent :

```python lines
from pydantic_ai import Agent
from pydantic_ai.models.instrumented import InstrumentationSettings

# Créer un agent PydanticAI avec traçage OTEL
agent = Agent(
    "openai:gpt-4o",
    instrument=InstrumentationSettings(tracer_provider=tracer_provider),
)

result = agent.run_sync("What is the capital of France?")
print(result.output)
```

Tous les appels à l&#39;agent sont tracés et envoyés vers Weave.

<div id="trace-pydanticai-tools-with-otel">
  ## Tracer les outils PydanticAI avec OTEL
</div>

Weave peut tracer toutes les opérations PydanticAI instrumentées avec OTEL, y compris les appels d’agent et d’outil. Cela signifie que lorsque votre agent invoque un outil (par exemple, une fonction décorée avec `@agent.tool_plain`), l’ensemble de l’interaction est capturée et visualisée dans Weave, y compris les entrées et sorties de l’outil, ainsi que le raisonnement du modèle.

L’exemple suivant montre comment créer un agent avec un prompt système et un outil. Le Tracing est activé automatiquement pour l’agent et l’outil :

```python lines
from pydantic_ai import Agent
from pydantic_ai.models.instrumented import InstrumentationSettings

# Créer un agent PydanticAI avec un prompt système et traçage OTEL
agent = Agent(
    "openai:gpt-4o",
    system_prompt=(
        "Vous êtes un assistant utile capable de multiplier des nombres. "
        "Quand on vous demande de multiplier des nombres, utilisez l'outil multiply."
    ),
    instrument=InstrumentationSettings(tracer_provider=tracer_provider),
)

# Définir un outil
@agent.tool_plain
def multiply(a: int, b: int) -> int:
    """Multiplier deux nombres."""
    return a * b

# Demander à l'agent d'utiliser l'outil
result = agent.run_sync("Quel est 7 multiplié par 8 ?")
print(result.output)
```

L&#39;appel de l&#39;agent et l&#39;appel de l&#39;outil sont tous deux tracés dans Weave, ce qui vous permet d&#39;inspecter le raisonnement complet et le chemin d&#39;exécution de votre application.

<div id="instrument-all-agents-by-default">
  ## Instrumenter tous les agents par défaut
</div>

Pour appliquer le tracing OTEL à tous les agents PydanticAI dans votre application, utilisez la méthode `Agent.instrument_all()`. Cela définit une instance par défaut de `InstrumentationSettings` pour tout agent qui ne spécifie pas explicitement le paramètre `instrument`.

```python lines
from pydantic_ai import Agent
from pydantic_ai.models.instrumented import InstrumentationSettings

# Configurer l'instrumentation par défaut pour tous les agents
Agent.instrument_all(InstrumentationSettings(tracer_provider=tracer_provider))

# Désormais, tout nouvel agent utilisera cette instrumentation par défaut
agent1 = Agent("openai:gpt-4o")
agent2 = Agent("openai:gpt-4o", system_prompt="Be helpful.")

result = agent1.run_sync("What is the capital of France?")
print(result.output)
```

Ceci est utile pour les applications plus grandes où vous souhaitez un Tracing cohérent à travers tous les agents sans répéter la configuration. Pour plus de détails, voir la [documentation PydanticAI OTEL](https://ai.pydantic.dev/logfire/#using-logfire).
