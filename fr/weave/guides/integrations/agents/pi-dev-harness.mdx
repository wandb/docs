[Pi](https://pi.dev/) est un agent de codage en ligne de commande. Weave trace automatiquement les sessions Pi, les appels LLM et les exécutions d’outils à l’aide de l’intégration `createOtelExtension`, qui est conforme aux [conventions sémantiques GenAI](https://opentelemetry.io/docs/specs/semconv/gen-ai/).

<Note>
  Pi est un framework TypeScript/Node.js sans équivalent en Python. Pi nécessite le système de modules ESM — votre projet doit utiliser `"type": "module"` dans `package.json`, ou compiler TypeScript en sortie ESM. Les projets CommonJS généreront une erreur.  Pour plus d’informations sur la configuration d’un projet ESM, voir [l’intégration du SDK TypeScript](/fr/weave/guides/integrations/js#set-up-an-esm-project).
</Note>

<div id="prerequisites">
  ## Prérequis
</div>

* [Node.js](https://nodejs.org/) (v18 ou version ultérieure)
* Un compte W&amp;B et une [clé API](https://wandb.ai/authorize) définie dans la variable d&#39;environnement `WANDB_API_KEY`

<div id="install-packages">
  ## Installer les paquets
</div>

* Installez Weave, Pi et les définitions de types pour Node comme dépendances locales du project :

```bash lines
npm install weave @earendil-works/pi-coding-agent
npm install --save-dev @types/node tsx typescript
```

<div id="trace-a-pi-prompt-and-response">
  ## Tracer un prompt et une réponse dans Pi
</div>

Appelez `weave.init()` avant de créer votre session d&#39;agent, puis transmettez `createOtelExtension()` en tant que fabrique d&#39;extensions. Weave trace l&#39;intégralité du cycle de vie de l&#39;agent : la session, chaque cycle prompt/réponse (`invoke_agent`), les appels LLM individuels (`chat`) et les exécutions d&#39;outils (`execute_tool`). L&#39;ID de session est généré automatiquement par `SessionManager.inMemory()`.

```typescript lines
import {init, createOtelExtension} from 'weave';

import {
  createAgentSession,
  DefaultResourceLoader,
  SessionManager,
  getAgentDir,
} from '@earendil-works/pi-coding-agent';

async function main() {
  // 1. Initialiser Weave — configure le TracerProvider OTEL pointant vers votre
  //    projet Weave. Tous les spans créés par createOtelExtension() sont
  //    automatiquement exportés ici.
  await init('[YOUR-TEAM]/[YOUR-PROJECT]'); 

// 2. Créer un chargeur de ressources et injecter l'extension OTEL de Weave.
//    Le chargeur de ressources fournit l'environnement d'exécution Pi et
//    le cycle de vie des extensions utilisé pour tracer l'activité de l'agent.
  const resourceLoader = new DefaultResourceLoader({
    cwd: process.cwd(),
    agentDir: getAgentDir(),
    extensionFactories: [createOtelExtension({})],
  });

  await resourceLoader.reload();

  // 3. Démarrer la session de l'agent
  const {session} = await createAgentSession({
    resourceLoader,
    sessionManager: SessionManager.inMemory(),
  });

  // 4. Lier les extensions — déclenche l'événement session_start afin que l'adaptateur OTEL
  //    crée le span de session racine et capture l'identifiant de conversation.
  await session.bindExtensions({});

  // 5. Diffuser la sortie de l'assistant vers stdout
  session.subscribe(event => {
    if (
      event.type === 'message_update' &&
      event.assistantMessageEvent.type === 'text_delta'
    ) {
      process.stdout.write(event.assistantMessageEvent.delta);
    }
  });

  // 6. Envoyer un prompt et attendre la réponse complète
  await session.prompt('What files are in the current directory?');
  console.log();
}

main();
```

Compilez et exécutez avec :

```bash
npx tsx [filename].ts 
```

Lorsque vous exécutez votre code, vos traces apparaissent dans l’onglet **Agents** de votre projet Weave à l’adresse `https://wandb.ai/[YOUR-TEAM]/[YOUR-PROJECT]/weave/agents`.

<div id="next-steps">
  ### Étapes suivantes
</div>

Vous pouvez transformer cet exemple en session sur plusieurs tours de conversation en ajoutant des prompts supplémentaires. Chaque appel à `session.prompt()` est tracé sous la forme d’un span `invoke_agent` distinct, tous imbriqués sous un seul span racine. L’agent conserve automatiquement le contexte d’un prompt à l’autre.

Après avoir exécuté le code, l’onglet **Agents** affiche la chronologie complète de la session sur plusieurs tours, avec les appels LLM imbriqués, les exécutions d’outils, l’utilisation des jetons et le coût.
