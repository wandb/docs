title: "Aperçu du scoring"
description: "Évaluez les sorties de l’IA et renvoyez des métriques d’évaluation avec les évaluateurs Weave"
---

Dans Weave, les évaluateurs servent à évaluer les sorties de l’IA et à renvoyer des métriques d’évaluation. Ils prennent la sortie de l’IA, l’analysent et renvoient un dictionnaire de résultats. Les évaluateurs peuvent utiliser vos données d’entrée comme référence si nécessaire, et aussi renvoyer des informations supplémentaires, comme des explications ou le raisonnement issu de l’évaluation.

<Tabs>
  <Tab title="Python">
    Les évaluateurs sont passés à un objet `weave.Evaluation` pendant l’évaluation. Il existe deux types d’évaluateurs dans Weave :

    1. **Évaluateurs basés sur des fonctions :** fonctions Python simples décorées avec `@weave.op`.
    2. **Évaluateurs basés sur des classes :** classes Python qui héritent de `weave.Scorer` pour des évaluations plus complexes.

    Les évaluateurs doivent renvoyer un dictionnaire et peuvent renvoyer plusieurs métriques, des métriques imbriquées et des valeurs non numériques, comme du texte renvoyé par un évaluateur LLM à propos de son raisonnement.
  </Tab>

  <Tab title="TypeScript">
    Les évaluateurs sont des ops spéciales passées à un objet `weave.Evaluation` pendant l’évaluation.
  </Tab>
</Tabs>

<div id="function-based-scorers">
  ### Évaluateurs basés sur des fonctions
</div>

<Tabs>
  <Tab title="Python">
    Il s’agit de fonctions décorées avec `@weave.op` qui renvoient un dictionnaire. Elles conviennent parfaitement à des évaluations simples comme :

    ```python lines
    import weave

    @weave.op
    def evaluate_uppercase(text: str) -> dict:
        return {"text_is_uppercase": text.isupper()}

    my_eval = weave.Evaluation(
        dataset=[{"text": "HELLO WORLD"}],
        scorers=[evaluate_uppercase]
    )
    ```

    Lorsque l’évaluation s’exécute, `evaluate_uppercase` vérifie si le texte est entièrement en majuscules.
  </Tab>

  <Tab title="TypeScript">
    Il s’agit de fonctions enveloppées avec `weave.op` qui prennent un objet contenant `modelOutput` et, facultativement, `datasetRow`. Elles conviennent parfaitement à des évaluations simples comme :

    ```typescript lines
    import * as weave from 'weave'

    const evaluateUppercase = weave.op(
        ({modelOutput}) => modelOutput.toUpperCase() === modelOutput,
        {name: 'textIsUppercase'}
    );

    const myEval = new weave.Evaluation({
        dataset: [{text: 'HELLO WORLD'}],
        scorers: [evaluateUppercase],
    })
    ```
  </Tab>
</Tabs>

<div id="class-based-scorers">
  ### Évaluateurs basés sur des classes
</div>

<Tabs>
  <Tab title="Python">
    Pour des évaluations plus avancées, en particulier lorsque vous devez suivre des métadonnées supplémentaires associées au scorer, essayer différents prompts pour vos évaluateurs LLM ou effectuer plusieurs appels de fonction, vous pouvez utiliser la classe `Scorer`.

    **Prérequis :**

    1. Héritez de `weave.Scorer`.
    2. Définissez une méthode `score` décorée avec `@weave.op`.
    3. La méthode `score` doit renvoyer un dictionnaire.

    Exemple :

    ```python lines {7}
    import weave
    from openai import OpenAI
    from weave import Scorer

    llm_client = OpenAI()

    class SummarizationScorer(Scorer):
        model_id: str = "gpt-4o"
        system_prompt: str = "Evaluate whether the summary is good."

        @weave.op
        def some_complicated_preprocessing(self, text: str) -> str:
            processed_text = "Original text: \n" + text + "\n"
            return processed_text

        @weave.op
        def call_llm(self, summary: str, processed_text: str) -> dict:
            res = llm_client.chat.completions.create(
                messages=[
                    {"role": "system", "content": self.system_prompt},
                    {"role": "user", "content": (
                        f"Analyze how good the summary is compared to the original text."
                        f"Summary: {summary}\n{processed_text}"
                    )}])
            return {"summary_quality": res}

        @weave.op
        def score(self, output: str, text: str) -> dict:
            """Évalue la qualité du résumé.

            Arguments:
                output: Le résumé généré par un système d’IA
                text: Le texte original à résumer
            """
            processed_text = self.some_complicated_preprocessing(text)
            eval_result = self.call_llm(summary=output, processed_text=processed_text)
            return {"summary_quality": eval_result}

    evaluation = weave.Evaluation(
        dataset=[{"text": "The quick brown fox jumps over the lazy dog."}],
        scorers=[summarization_scorer])
    ```

    Cette classe évalue la qualité d’un résumé en le comparant au texte original.
  </Tab>

  <Tab title="TypeScript">
    ```plaintext
    Cette fonctionnalité n’est pas encore disponible en TypeScript.
    ```
  </Tab>
</Tabs>

<div id="how-scorers-work">
  ## Fonctionnement des évaluateurs
</div>

<div id="scorer-keyword-arguments">
  ### Arguments nommés du scorer
</div>

<Tabs>
  <Tab title="Python">
    Les évaluateurs peuvent accéder à la fois à la sortie de votre système d&#39;IA et aux données d&#39;entrée de la ligne du jeu de données.

    * **Entrée :** Si vous voulez que votre scorer utilise des données de la ligne de votre jeu de données, comme une colonne « label » ou « target », vous pouvez facilement les rendre accessibles au scorer en ajoutant un argument nommé `label` ou `target` à la définition de votre scorer.

    Par exemple, si vous vouliez utiliser une colonne appelée « label » dans votre jeu de données, votre fonction de scorer (ou la méthode de classe `score`) aurait une liste de paramètres comme celle-ci :

    ```python lines
    @weave.op
    def my_custom_scorer(output: str, label: int) -> dict:
        ...
    ```

    Lorsqu&#39;une `Evaluation` Weave est exécutée, la sortie du système d&#39;IA est transmise au paramètre `output`. L&#39;`Evaluation` essaie aussi automatiquement de faire correspondre les noms des arguments supplémentaires du scorer aux colonnes de votre jeu de données. Si vous ne pouvez pas personnaliser les arguments de votre scorer ou les colonnes du jeu de données, vous pouvez utiliser le mappage de colonnes ; voir ci-dessous pour plus d&#39;informations.

    * **Sortie :** Ajoutez un paramètre `output` à la signature de votre fonction de scorer pour accéder à la sortie du système d&#39;IA.

    ### Mapper les noms de colonnes avec `column_map`

    Il arrive que les noms des arguments des méthodes `score` ne correspondent pas aux noms de colonnes de votre jeu de données. Vous pouvez corriger cela à l&#39;aide de `column_map`.

    Si vous utilisez un scorer basé sur une classe, transmettez un dictionnaire à l&#39;attribut `column_map` de `Scorer` lors de l&#39;initialisation de votre classe de scorer. Ce dictionnaire associe les noms des arguments de votre méthode `score` aux noms des colonnes du jeu de données, dans l&#39;ordre suivant : `{scorer_keyword_argument: dataset_column_name}`.

    Exemple :

    ```python lines
    import weave
    from weave import Scorer

    # Un jeu de données contenant des articles de presse à résumer
    dataset = [
        {"news_article": "The news today was great...", "date": "2030-04-20", "source": "Bright Sky Network"},
        ...
    ]

    # Classe de scorer
    class SummarizationScorer(Scorer):

        @weave.op
        def score(self, output, text) -> dict:
            """
                output: résumé généré par un système de résumé utilisant un LLM
                text: le texte à résumer
            """
            ...  # évaluer la qualité du résumé

    # créer un scorer avec un mappage de colonnes associant l'argument `text` à la colonne de données `news_article`
    scorer = SummarizationScorer(column_map={"text" : "news_article"})
    ```

    Désormais, l&#39;argument `text` de la méthode `score` recevra les données de la colonne `news_article` du jeu de données.

    **Notes :**

    * Une autre option équivalente pour mapper vos colonnes consiste à sous-classer `Scorer` et à surcharger la méthode `score` en mappant explicitement les colonnes.

    ```python lines
    import weave
    from weave import Scorer

    class MySummarizationScorer(SummarizationScorer):

        @weave.op
        def score(self, output: str, news_article: str) -> dict:  # Indications de type ajoutées
            # surcharger la méthode score et mapper manuellement les colonnes
            return super().score(output=output, text=news_article)
    ```
  </Tab>

  <Tab title="TypeScript">
    Les évaluateurs peuvent accéder à la fois à la sortie de votre système d&#39;IA et au contenu de la ligne du jeu de données.

    Vous pouvez facilement accéder aux colonnes pertinentes de la ligne du jeu de données en ajoutant un argument nommé `datasetRow` à la définition de votre scorer.

    ```typescript lines
    const myScorer = weave.op(
        ({modelOutput, datasetRow}) => {
            return modelOutput * 2 === datasetRow.expectedOutputTimesTwo;
        },
        {name: 'myScorer'}
    );
    ```

    ### Mapper les noms de colonnes avec `columnMapping`

    <Warning>
      En TypeScript, cette fonctionnalité est actuellement portée par l&#39;objet `Evaluation`, et non par les évaluateurs individuels.
    </Warning>

    Il arrive que les clés de votre `datasetRow` ne correspondent pas exactement à la convention de nommage du scorer, tout en étant sémantiquement similaires. Vous pouvez mapper les colonnes à l&#39;aide de l&#39;option `columnMapping` de l&#39;`Evaluation`.

    Le mappage se fait toujours du point de vue du scorer, c&#39;est-à-dire `{scorer_key: dataset_column_name}`.

    Exemple :

    ```typescript lines
    const myScorer = weave.op(
        ({modelOutput, datasetRow}) => {
            return modelOutput * 2 === datasetRow.expectedOutputTimesTwo;
        },
        {name: 'myScorer'}
    );

    const myEval = new weave.Evaluation({
        dataset: [{expected: 2}],
        scorers: [myScorer],
        columnMapping: {expectedOutputTimesTwo: 'expected'}
    });
    ```
  </Tab>
</Tabs>

<div id="access-variables-from-your-ops-in-scoring-prompts">
  ### Accéder aux variables de vos ops dans les prompts de scoring
</div>

Dans les prompts de scoring des évaluateurs LLM-as-a-judge, vous pouvez faire référence à des variables de votre op. Ces valeurs sont automatiquement extraites lorsque le scorer s’exécute.

Pour une fonction comme :

```python
@weave.op
def summarize_article(article: str, max_length: int) -> str:
    # Votre logique de résumé ici
    return summary
```

Les variables suivantes sont disponibles :

| Variable       | Description                                                |
| -------------- | ---------------------------------------------------------- |
| `{article}`    | La valeur de l’argument d’entrée `article`                 |
| `{max_length}` | La valeur de l’argument d’entrée `max_length`              |
| `{inputs}`     | Un dictionnaire JSON contenant tous les arguments d’entrée |
| `{output}`     | Le résultat renvoyé par votre op                           |

Exemple d’invite de notation :

```text
Évaluez la qualité de ce résumé.

Article original : {article}
Résumé : {output}
Longueur maximale demandée : {max_length}

Notez le résumé sur une échelle de 1 à 10 en vous basant sur :
- Précision : représente-t-il fidèlement l'article ?
- Exhaustivité : couvre-t-il les points clés ?
- Concision : est-il suffisamment bref ?

Renvoyez un objet JSON avec votre note et votre justification.
```

<div id="final-summarization-of-the-scorer">
  ### Synthèse finale du scorer
</div>

<Tabs>
  <Tab title="Python">
    Pendant l&#39;évaluation, le scorer est calculé pour chaque ligne de votre jeu de données. Pour fournir un score final pour l&#39;évaluation, nous utilisons `auto_summarize`, qui dépend du type renvoyé en sortie.

    * Des moyennes sont calculées pour les colonnes numériques
    * Le nombre et la proportion sont calculés pour les colonnes booléennes
    * Les autres types de colonnes sont ignorés

    Vous pouvez redéfinir la méthode `summarize` de la classe `Scorer` et fournir votre propre façon de calculer les scores finaux. La fonction `summarize` attend :

    * Un seul paramètre `score_rows` : il s&#39;agit d&#39;une liste de dictionnaires, où chaque dictionnaire contient les scores renvoyés par la méthode `score` pour une seule ligne de votre jeu de données.
    * Elle doit renvoyer un dictionnaire contenant les scores synthétisés.

    **Pourquoi est-ce utile ?**

    Lorsque vous devez scorer toutes les lignes avant de déterminer la valeur finale du score pour le jeu de données.

    ```python lines
    class MyBinaryScorer(Scorer):
        """
        Renvoie True si la sortie complète correspond à la cible, False sinon
        """

        @weave.op
        def score(self, output, target):
            return {"match": output == target}

        def summarize(self, score_rows: list) -> dict:
            full_match = all(row["match"] for row in score_rows)
            return {"full_match": full_match}
    ```

    > Dans cet exemple, `auto_summarize` aurait par défaut renvoyé le nombre et la proportion de valeurs True.

    Si vous souhaitez en savoir plus, consultez l&#39;implémentation de [CorrectnessLLMJudge](/fr/weave/tutorial-rag#optional-defining-a-scorer-class).
  </Tab>

  <Tab title="TypeScript">
    Pendant l&#39;évaluation, le scorer est calculé pour chaque ligne de votre jeu de données. Pour fournir un score final, nous utilisons une fonction interne `summarizeResults` qui agrège les résultats selon le type de sortie.

    * Des moyennes sont calculées pour les colonnes numériques
    * Le nombre et la proportion sont calculés pour les colonnes booléennes
    * Les autres types de colonnes sont ignorés

    Nous ne prenons pas encore en charge la synthèse personnalisée.
  </Tab>
</Tabs>

<div id="applying-scorers-to-a-call">
  ### Appliquer des évaluateurs à un appel
</div>

Pour appliquer des évaluateurs à vos ops Weave, vous devez utiliser la méthode `.call()`, qui donne accès à la fois au résultat de l’opération et à ses informations de traçage. Cela vous permet d’associer les résultats des évaluateurs à des appels précis dans la base de données de Weave.

Pour en savoir plus sur l’utilisation de la méthode `.call()`, voir le guide [Appeler des ops](../tracking/tracing#calling-ops#getting-a-handle-to-the-call-object-during-execution).

<Tabs>
  <Tab title="Python">
    Voici un exemple de base :

    ```python lines
    # Obtenir à la fois le résultat et l’objet Call
    result, call = generate_text.call("Say hello")

    # Appliquer un scorer
    score = await call.apply_scorer(MyScorer())
    ```

    Vous pouvez également appliquer plusieurs évaluateurs au même appel :

    ```python lines
    # Appliquer plusieurs évaluateurs en parallèle
    await asyncio.gather(
        call.apply_scorer(quality_scorer),
        call.apply_scorer(toxicity_scorer)
    )
    ```

    **Notes :**

    * Les résultats des évaluateurs sont automatiquement stockés dans la base de données de Weave
    * Les évaluateurs s’exécutent de manière asynchrone une fois l’opération principale terminée
    * Vous pouvez consulter les résultats des évaluateurs dans l’interface ou les interroger via l’API

    Pour des informations plus détaillées sur l’utilisation des évaluateurs comme guardrails ou monitors, y compris les bonnes pratiques de production et des exemples complets, consultez notre [guide Guardrails and Monitors](./monitors).
  </Tab>

<div id="use-preprocess_model_input">
  ### Utiliser `preprocess_model_input`
</div>

Vous pouvez utiliser le paramètre `preprocess_model_input` pour modifier les exemples du jeu de données avant qu’ils ne soient transmis à votre modèle pendant l’évaluation.

<Important>
  La fonction `preprocess_model_input` transforme uniquement les entrées avant leur transmission à la fonction de prédiction du modèle.

  Les fonctions scorer reçoivent toujours les exemples originaux du jeu de données, sans aucun prétraitement.
</Important>

Pour plus d’informations sur l’utilisation et pour voir un exemple, voir [Utiliser `preprocess_model_input` pour formater les lignes du jeu de données avant l’évaluation](../core-types/evaluations#using-preprocess_model_input-to-format-dataset-rows-before-evaluating).

<div id="score-analysis">
  ## Analyse des scores
</div>

Dans cette section, nous allons vous montrer comment analyser les scores d’un appel unique, de plusieurs appels et de tous les appels évalués par un scorer spécifique.

<div id="analyze-a-single-calls-scores">
  ### Analyser les scores d’un Appel unique
</div>

<div id="single-call-api">
  #### API d’un appel unique
</div>

Pour récupérer les appels associés à un appel unique, vous pouvez utiliser la méthode `get_call`.

```python lines
client = weave.init("my-project")

# Obtenir un appel unique
call = client.get_call("call-uuid-here")

# Obtenir le feedback de l'appel qui contient les scores
feedback = list(call.feedback)
```

<div id="analyze-multiple-calls-scores">
  ### Analyser les scores de plusieurs appels
</div>

<div id="multiple-calls-api">
  #### API des appels multiples
</div>

Pour récupérer les appels associés à plusieurs appels, vous pouvez utiliser la méthode `get_calls`.

```python lines
client = weave.init("my-project")

# Obtenir plusieurs appels - utilisez les filtres de votre choix et incluez le feedback
calls = client.get_calls(..., include_feedback=True)

# Parcourez les appels et accédez au feedback qui contient les scores
for call in calls:
    feedback = list(call.feedback)
```

<div id="analyze-all-calls-scored-by-a-specific-scorer">
  ### Analyser tous les appels évalués par un scorer spécifique
</div>

<div id="all-calls-by-scorer-api">
  #### API de tous les appels par scorer
</div>

Pour récupérer tous les appels évalués par un scorer spécifique, vous pouvez utiliser la méthode `get_calls`.

```python lines
client = weave.init("my-project")

# Pour obtenir tous les appels scorés par n'importe quelle version d'un scorer, utilisez le nom du scorer (généralement le nom de la classe)
calls = client.get_calls(scored_by=["MyScorer"], include_feedback=True)

# Pour obtenir tous les appels scorés par une version spécifique d'un scorer, utilisez la référence complète
# Les références peuvent être obtenues depuis l'objet scorer ou via l'interface utilisateur.
calls = client.get_calls(scored_by=[myScorer.ref.uri()], include_feedback=True)

# Parcourez les appels et accédez au feedback qui contient les scores
for call in calls:
    feedback = list(call.feedback)
```
