title: "Configurer les variables d'environnement de Weave"
description: "Configurez le comportement de Weave à l'exécution à l'aide de variables d'environnement"
---

Weave fournit un ensemble de variables d&#39;environnement permettant de configurer et d&#39;optimiser son comportement. Cette page s&#39;adresse aux développeurs qui utilisent le SDK Weave et souhaitent ajuster le comportement à l&#39;exécution, notamment le traçage, la journalisation, la mise en cache et les nouvelles tentatives, sans modifier le code de l&#39;application. Vous pouvez définir ces variables dans votre shell ou dans des scripts afin de contrôler des fonctionnalités spécifiques.

Les exemples suivants montrent deux façons courantes de définir les variables d&#39;environnement de Weave. Utilisez l&#39;approche shell pour configurer Weave pour une session entière, ou l&#39;approche Python pour définir des valeurs dans votre script.

```bash
# Exemple de définition des variables d'environnement dans le shell
export WEAVE_PARALLELISM=10  # Contrôle le nombre de workers parallèles
export WEAVE_PRINT_CALL_LINK=false  # Désactive la sortie du lien d'appel
```

```python lines
# Exemple de définition des variables d'environnement en Python
import os

os.environ["WEAVE_PARALLELISM"] = "10"
os.environ["WEAVE_PRINT_CALL_LINK"] = "false"
```

<div id="available-environment-variables">
  ## Variables d&#39;environnement disponibles
</div>

Le tableau suivant répertorie chaque variable d&#39;environnement prise en charge, son type, sa valeur par défaut et le comportement qu&#39;elle contrôle. Pour connaître les valeurs acceptées lors de la définition de variables booléennes, voir la note sur les valeurs booléennes.

| Variable                        | Type     | Par défaut      | Description                                                                                                                                                                                                                                                                                                                                                                                                                               |
| ------------------------------- | -------- | --------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `WANDB_API_KEY`                 | `string` | `None`          | Si elle est définie, vous connecte automatiquement à W&amp;B Weave sans vous demander votre clé API. Pour générer une clé API, connectez-vous à votre compte W&amp;B et accédez à [Paramètres utilisateur](https://wandb.ai/settings).                                                                                                                                                                                                    |
| `WEAVE_DISABLED`                | `bool`   | `false`         | Lorsqu&#39;elle est définie sur `true`, désactive tout le traçage Weave. Les ops Weave se comportent comme des fonctions classiques.                                                                                                                                                                                                                                                                                                      |
| `WEAVE_PRINT_CALL_LINK`         | `bool`   | `true`          | Contrôle l&#39;affichage d&#39;un lien vers l&#39;interface Weave lors de l&#39;appel d&#39;une op Weave. Vous pouvez également le définir directement dans votre code en configurant l&#39;argument `settings` de `weave.init()` comme ceci : `weave.init("your-project-name", settings={"print_call_link": False})`                                                                                                                     |
| `WEAVE_LOG_LEVEL`               | `str`    | `INFO`          | Contrôle le niveau de journalisation de Weave.                                                                                                                                                                                                                                                                                                                                                                                            |
| `WEAVE_CAPTURE_CODE`            | `bool`   | `true`          | Contrôle si le code des ops doit être enregistré afin que vous puissiez le recharger ultérieurement.                                                                                                                                                                                                                                                                                                                                      |
| `WEAVE_DEBUG_HTTP`              | `bool`   | `false`         | Lorsqu&#39;elle est définie sur `true`, active la journalisation des requêtes et des réponses HTTP à des fins de débogage.                                                                                                                                                                                                                                                                                                                |
| `WEAVE_PARALLELISM`             | `int`    | `20`            | Dans les évaluations, contrôle le nombre d&#39;exemples à évaluer en parallèle. Définissez-la sur `1` pour exécuter les exemples de façon séquentielle.                                                                                                                                                                                                                                                                                   |
| `WEAVE_TRACE_LANGCHAIN`         | `bool`   | `true`          | Contrôle le traçage global pour LangChain. Définissez-la sur `false` pour désactiver explicitement le traçage de LangChain.                                                                                                                                                                                                                                                                                                               |
| `WEAVE_USE_SERVER_CACHE`        | `bool`   | `true`          | Active la mise en cache des réponses du serveur. Lorsqu&#39;elle est activée, Weave met en cache les réponses du serveur sur disque afin que les requêtes répétées puissent être traitées depuis le cache au lieu d&#39;être récupérées à nouveau depuis le serveur.                                                                                                                                                                      |
| `WEAVE_SERVER_CACHE_SIZE_LIMIT` | `int`    | `1,000,000,000` | Définit la taille maximale du cache serveur, en octets. Lorsque le cache atteint cette taille, Weave supprime automatiquement les entrées les plus anciennes pour libérer de l&#39;espace pour les nouvelles. Important : l&#39;implémentation sous-jacente utilise SQLite, qui possède un journal à écriture anticipée (WAL) atteignant 4 Mo quelle que soit cette configuration. Weave supprime ce WAL lorsque le programme se termine. |
| `WEAVE_SERVER_CACHE_DIR`        | `str`    | `None`          | Spécifie le répertoire dans lequel Weave stocke les fichiers de cache. Si elle n&#39;est pas définie, Weave utilise un répertoire temporaire.                                                                                                                                                                                                                                                                                             |
| `WEAVE_MAX_CALLS_QUEUE_SIZE`    | `int`    | `100000`        | Définit la taille maximale de la file d&#39;attente des appels. La valeur par défaut est `100000`. Définir une valeur de `0` permet à la file d&#39;attente de croître sans limite.                                                                                                                                                                                                                                                       |
| `WEAVE_ENABLE_WAL`              | `bool`   | `false`         | Active le [journal à écriture anticipée (WAL)](/fr/weave/guides/tracking/write-ahead-log) de Weave. Lorsqu&#39;elle est définie sur `true`, Weave écrit les requêtes API sur disque avant de les envoyer au serveur, afin qu&#39;elles puissent être récupérées si le processus se termine avant leur envoi.                                                                                                                                 |
| `WEAVE_DISABLE_WAL_SENDER`      | `bool`   | `false`         | Désactive l&#39;expéditeur du journal à écriture anticipée (WAL) de Weave. Lorsqu&#39;elle est définie sur `true`, Weave écrit les requêtes sur disque mais ne les envoie pas au serveur. Utile pour le débogage.                                                                                                                                                                                                                         |
| `WEAVE_RETRY_MAX_ATTEMPTS`      | `int`    | `3`             | Définit le nombre maximal de tentatives de réessai pour les requêtes ayant échoué.                                                                                                                                                                                                                                                                                                                                                        |
| `WEAVE_RETRY_MAX_INTERVAL`      | `float`  | `300.0`         | Définit l&#39;intervalle maximal entre les tentatives de réessai, en secondes.                                                                                                                                                                                                                                                                                                                                                            |
| `WANDB_BASE_URL`                | `string` | `None`          | Définit l&#39;URL de l&#39;hôte Weave. Cela équivaut à saisir l&#39;URL de l&#39;hôte lorsque `wandb.login()` vous la demande. Vous pouvez spécifier `WANDB_BASE_URL` et `WANDB_API_KEY` avant d&#39;utiliser `weave.init()` pour vous connecter et vous authentifier automatiquement à Weave.                                                                                                                                            |

<Note>
  Toutes les variables d&#39;environnement booléennes acceptent les valeurs suivantes (sans distinction de casse) :

  * `true`, `1`, `yes`, `on` pour `True`
  * `false`, `0`, `no`, `off` pour `False`
</Note>
