---
title: "Configurer les variables d'environnement de Weave"
description: "Configurez le comportement de Weave à l'exécution à l'aide de variables d'environnement"
---

Weave fournit un ensemble de variables d&#39;environnement permettant de configurer et d&#39;optimiser son comportement. Vous pouvez définir ces variables dans votre shell ou dans des scripts afin de contrôler des fonctionnalités spécifiques.

```bash
# Exemple de définition des variables d'environnement dans le shell
export WEAVE_PARALLELISM=10  # Contrôle le nombre de workers parallèles
export WEAVE_PRINT_CALL_LINK=false  # Désactive la sortie du lien d'appel
```

<div id="available-environment-variables">
  ## Variables d&#39;environnement disponibles
</div>

| Variable                        | Type     | Valeur par défaut | Description                                                                                                                                                                                                                                                                                                                                                                                                     |
| ------------------------------- | -------- | ----------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `WANDB_API_KEY`                 | `string` | `None`            | Si elle est définie, vous connecte automatiquement à W&amp;B Weave sans vous demander votre clé API. Pour générer une clé API, connectez-vous à votre compte W&amp;B, puis accédez à [Paramètres utilisateur](https://wandb.ai/settings).                                                                                                                                                                       |
| `WEAVE_DISABLED`                | `bool`   | `false`           | Lorsqu&#39;elle est définie sur `true`, désactive tout le Tracing Weave. Les ops Weave se comportent alors comme des fonctions normales.                                                                                                                                                                                                                                                                        |
| `WEAVE_PRINT_CALL_LINK`         | `bool`   | `true`            | Indique s&#39;il faut afficher un lien d&#39;appel vers la Weave UI lors de l&#39;appel d&#39;une op Weave. Vous pouvez aussi le définir directement dans votre code en configurant l&#39;argument `settings` de `weave.init()` comme ceci : `weave.init("your-project-name", settings={"print_call_link": False})`                                                                                                         |
| `WEAVE_LOG_LEVEL`               | `str`    | `INFO`            | Contrôle le niveau de journalisation du logger weave.                                                                                                                                                                                                                                                                                                                                                           |
| `WEAVE_CAPTURE_CODE`            | `bool`   | `true`            | Indique s&#39;il faut enregistrer le code des ops afin de pouvoir les recharger ultérieurement.                                                                                                                                                                                                                                                                                                                 |
| `WEAVE_DEBUG_HTTP`              | `bool`   | `false`           | Lorsqu&#39;elle est définie sur `true`, active la journalisation des requêtes et des réponses HTTP à des fins de débogage.                                                                                                                                                                                                                                                                                      |
| `WEAVE_PARALLELISM`             | `int`    | `20`              | Dans les évaluations, contrôle le nombre d&#39;exemples à évaluer en parallèle. Définissez-la sur `1` pour exécuter les exemples de manière séquentielle.                                                                                                                                                                                                                                                       |
| `WEAVE_TRACE_LANGCHAIN`         | `bool`   | `true`            | Contrôle le Tracing global de LangChain. Définissez-la sur `false` pour désactiver explicitement le Tracing LangChain.                                                                                                                                                                                                                                                                                          |
| `WEAVE_USE_SERVER_CACHE`        | `bool`   | `true`            | Active la mise en cache des réponses du serveur. Lorsqu&#39;elle est activée, les réponses du serveur sont mises en cache sur disque afin d&#39;améliorer les performances des requêtes répétées.                                                                                                                                                                                                               |
| `WEAVE_SERVER_CACHE_SIZE_LIMIT` | `int`    | `1000000000`      | Définit la taille maximale du cache serveur, en octets. Lorsque le cache atteint cette taille, les anciennes entrées sont automatiquement supprimées pour libérer de l&#39;espace pour les nouvelles. Important : l&#39;implémentation sous-jacente utilise SQLite, qui possède un Write Ahead Log (WAL) atteignant 4 Mo quelle que soit cette configuration. Ce WAL sera supprimé à la fermeture du programme. |
| `WEAVE_SERVER_CACHE_DIR`        | `str`    | `None`            | Indique le répertoire dans lequel les fichiers de cache doivent être stockés. S&#39;il n&#39;est pas défini, un répertoire temporaire est utilisé.                                                                                                                                                                                                                                                              |
| `WEAVE_MAX_CALLS_QUEUE_SIZE`    | `int`    | `100000`          | Définit la taille maximale de la file d&#39;attente des appels. La valeur par défaut est 100&#95;000. Définir une valeur de 0 signifie que la file d&#39;attente peut croître sans limite.                                                                                                                                                                                                                      |
| `WEAVE_RETRY_MAX_ATTEMPTS`      | `int`    | `3`               | Définit le nombre maximal de tentatives pour les requêtes ayant échoué.                                                                                                                                                                                                                                                                                                                                         |
| `WEAVE_RETRY_MAX_INTERVAL`      | `float`  | `300.0`           | Définit l&#39;intervalle maximal entre les tentatives, en secondes.                                                                                                                                                                                                                                                                                                                                             |
| `WANDB_BASE_URL`                | `string` | `None`            | Définit l&#39;URL de l&#39;hôte Weave. Cela équivaut à saisir l&#39;URL de l&#39;hôte lorsque `wandb.login()` vous la demande. Vous pouvez spécifier `WANDB_BASE_URL` et `WANDB_API_KEY` avant d&#39;utiliser `weave.init()` pour vous connecter et vous authentifier automatiquement auprès de Weave.                                                                                                          |

<Note>
  Toutes les variables d&#39;environnement booléennes acceptent les valeurs suivantes (sans tenir compte de la casse) :

  * `true`, `1`, `yes`, `on` pour True
  * `false`, `0`, `no`, `off` pour False
</Note>
