<Info>
  **Prérequis** : veuillez installer Node.js (version 19 ou ultérieure) avant de continuer. <br />
  Veuillez migrer vers `docs.json` avant de continuer et supprimer l&#39;ancien fichier `mint.json`.
</Info>

Suivez ces étapes pour installer et exécuter Mintlify sur votre système d&#39;exploitation :

**Étape 1** : installez Mintlify :

<CodeGroup>
  ```bash npm
  npm i -g mintlify
  ```

  ```bash yarn
  yarn global add mintlify
  ```
</CodeGroup>

**Étape 2** : accédez au répertoire docs (où se trouve le fichier `docs.json`) et exécutez la commande suivante :

```bash
mintlify dev
```

Un aperçu local de votre documentation sera disponible à l’adresse `http://localhost:3000`.


<div id="custom-ports">
  ### Ports personnalisés
</div>

Par défaut, Mintlify utilise le port 3000. Vous pouvez personnaliser le port sur lequel Mintlify s’exécute à l’aide de l’option `--port`. Pour exécuter Mintlify sur le port 3333, par exemple, utilisez cette commande :

```bash
mintlify dev --port 3333
```

Si vous essayez d’exécuter Mintlify sur un port déjà utilisé, il utilisera le port disponible suivant :

```md
Le port 3000 est déjà utilisé. Essai du port 3001 à la place.
```


<div id="mintlify-versions">
  ## Versions de Mintlify
</div>

Notez que chaque version de la CLI est associée à une version spécifique de Mintlify. Si votre site web local ne correspond pas à la version en production, mettez à jour la CLI :

<CodeGroup>
  ```bash npm
  npm i -g mintlify@latest
  ```

  ```bash yarn
  yarn global upgrade mintlify
  ```
</CodeGroup>

<div id="validating-links">
  ## Vérification des liens
</div>

Le CLI peut vous aider à vérifier les liens de référence dans votre documentation. Pour identifier les liens rompus, utilisez la commande suivante :

```bash
mintlify broken-links
```

<div id="code-formatting">
  ## Formatage du code
</div>

Nous vous recommandons d’utiliser des extensions dans votre IDE pour reconnaître et formater le MDX. Si vous utilisez VSCode, pensez à l’[extension MDX pour VSCode](https://marketplace.visualstudio.com/items?itemName=unifiedjs.vscode-mdx) pour la coloration syntaxique, et à [Prettier](https://marketplace.visualstudio.com/items?itemName=esbenp.prettier-vscode) pour le formatage du code.

<div id="troubleshooting">
  ## Dépannage
</div>

<AccordionGroup>
  <Accordion title="Erreur : impossible de charger le module &#x22;sharp&#x22; avec le runtime darwin-arm64">
    Cela peut être dû à une version obsolète de Node. Essayez ce qui suit :

    1. Supprimez la version actuellement installée de mintlify : `npm remove -g mintlify`
    2. Passez à Node v19 ou à une version ultérieure.
    3. Réinstallez mintlify : `npm install -g mintlify`
  </Accordion>

  <Accordion title="Problème : vous rencontrez une erreur inconnue">
    Solution : accédez au répertoire racine de votre appareil et supprimez le dossier ~/.mintlify. Ensuite, exécutez à nouveau `mintlify dev`.
  </Accordion>
</AccordionGroup>

Vous vous demandez ce qui a changé dans la version CLI ? [Consultez le journal des modifications de la CLI.](https://www.npmjs.com/package/mintlify?activeTab=versions)
