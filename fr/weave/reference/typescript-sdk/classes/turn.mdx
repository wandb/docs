## Méthodes

<div id="addevent">
  ### addEvent
</div>

<Warning>
  **Obsolète.** Enregistrez plutôt ces données via [setAttributes](./turn#setattributes).
  OpenTelemetry abandonne progressivement l’API Span Event (`Span.addEvent`). Ces
  méthode fonctionne toujours et les données d’événements de span existantes restent valides.
  Voir [https://opentelemetry.io/blog/2026/deprecating-span-events/](https://opentelemetry.io/blog/2026/deprecating-span-events/)

  `Exemple`

  ```ts twoslash
  // @noErrors
  span.addEvent('context_compacted', {removedMessages: 12});
  ```
</Warning>

▸ **addEvent**(`name`, `attributes?`, `startTime?`): `this`

Ajoutez un événement nommé au span. Utile pour marquer des moments qui ne correspondent pas à des spans,
comme le compactage du contexte, la détection de boucles d&#39;outil ou le déclenchement de garde-fous.
Émet un avertissement et ne fait rien après `end()`. Reproduit le comportement de `Span.addEvent` d’OTel.

<div id="parameters">
  #### Paramètres
</div>

| Nom           | Type         |
| :------------ | :----------- |
| `name`        | `string`     |
| `attributes?` | `Attributes` |
| `startTime?`  | `TimeInput`  |

  #### Renvoie
</div>

`this`

<div id="end">
  ### end
</div>

▸ **end**(`opts?`): `void`

Lire les valeurs actuelles des champs (pour refléter les mutations effectuées via `record()`
depuis `start`) et fermer le span. Cette opération est idempotente. Passez `error` pour le marquer comme ayant échoué ; passez
`endTime` pour antidater la fermeture.

<div id="parameters">
  #### Paramètres
</div>

| Nom     | Type             |
| :------ | :--------------- |
| `opts?` | `SpanEndOptions` |

  ### record
</div>

▸ **record**(`opts`): `this`

Définit en bloc n’importe quel sous-ensemble des champs modifiables. Remplace (sans fusionner).
Utile pour tout attribuer en une seule fois après le renvoi d’un appel au provider.

<div id="setattributes">
  ### setAttributes
</div>

▸ **setAttributes**(`attributes`): `this`

Définit plusieurs attributs sur le span en une seule fois. Émet un avertissement puis n’a aucun effet après
`end()`. Équivaut à `Span.setAttributes` d’OTel (et à
`set_attributes` du SDK Python).

<div id="parameters">
  #### Paramètres
</div>

| Nom          | Type         |
| :----------- | :----------- |
| `attributes` | `Attributes` |

<div id="returns">
  #### Renvoie
</div>

`this`

`Exemple`

```ts twoslash
// @noErrors
span.setAttributes({'weave.tag': 'prod', 'gen_ai.response.id': id});
```
