## <kbd>classe</kbd> `AnnotatorQueueItemsProgressUpdateReq`

Requête permettant de mettre à jour l&#39;état d&#39;annotation d&#39;un élément de la file d&#39;attente pour l&#39;annotateur actuel.

Transitions d&#39;état valides :

* (absence) -&gt; &#39;in&#95;progress&#39; : Marquer l&#39;élément comme étant en cours (uniquement lorsqu&#39;aucun enregistrement n&#39;existe)
* (absence) -&gt; &#39;completed&#39; ou &#39;skipped&#39; : Terminer ou ignorer directement l&#39;élément
* &#39;in&#95;progress&#39; ou &#39;unstarted&#39; -&gt; &#39;completed&#39; ou &#39;skipped&#39; : Terminer ou ignorer un élément déjà démarré
* same&#95;state -&gt; same&#95;state : Aucune opération idempotente (renvoie l&#39;élément existant sans modification)

**Champs Pydantic :**

* `project_id`: `<class 'str'>`
* `queue_id`: `<class 'str'>`
* `item_id`: `<class 'str'>`
* `annotation_state`: `<class 'str'>`
* `wb_user_id`: `str | None`

## <kbd>class</kbd> `TraceUsageReq`

Requête permettant de calculer l’utilisation par appel pour une trace, avec cumul des descendants.

Ce point de terminaison renvoie des métriques d’utilisation pour chaque appel de la trace. Les métriques de chaque appel incluent la somme de sa propre utilisation et de celle de tous ses descendants. Utilisez-le dans la Vue de trace lorsque vous souhaitez afficher des métriques cumulées par appel.

Remarque : tous les appels correspondants sont chargés en mémoire pour l’agrégation. Pour les ensembles de résultats très volumineux (&gt;10k appels), envisagez d’utiliser des filtres plus spécifiques ou une pagination au niveau de l’application.

**Champs Pydantic :**

* `project_id`: `<class 'str'>`
* `filter`: `CallsFilter | None`
* `query`: `weave.trace_server.interface.query.Query | None`
* `include_costs`: `<class 'bool'>`
* `limit`: `<class 'int'>`
