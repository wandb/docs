### <kbd>méthode</kbd> `add_cost`

```python
add_cost(
    llm_id: 'str',
    prompt_token_cost: 'float',
    completion_token_cost: 'float',
    effective_date: 'datetime | None' = None,
    prompt_token_cost_unit: 'str | None' = 'USD',
    completion_token_cost_unit: 'str | None' = 'USD',
    provider_id: 'str | None' = 'default'
) → CostCreateRes
```

Ajoutez un coût au projet actuel.

**Exemples :**

```python
client.add_cost(llm_id="my_expensive_custom_model", prompt_token_cost=1, completion_token_cost=2)
client.add_cost(llm_id="my_expensive_custom_model", prompt_token_cost=500, completion_token_cost=1000, effective_date=datetime(1998, 10, 3))
```

**Arguments :**

* <b>`llm_id`</b> : l’ID du LLM. Par ex. &quot;gpt-4o-mini-2024-07-18&quot;
* <b>`prompt_token_cost`</b> : le coût par jeton de prompt. Par ex. .0005
* <b>`completion_token_cost`</b> : le coût par jeton de complétion. Par ex. .0015
* <b>`effective_date`</b> : par défaut, la date actuelle. Objet `datetime.datetime`.
* <b>`provider_id`</b> : le fournisseur du LLM. La valeur par défaut est &quot;default&quot;. Par ex. &quot;openai&quot;
* <b>`prompt_token_cost_unit`</b> : l’unité du coût des jetons de prompt. La valeur par défaut est &quot;USD&quot;. (Actuellement inutilisée ; sera utilisée à l’avenir pour spécifier le type d’unité de coût, par ex. &quot;tokens&quot; ou &quot;time&quot;)
* <b>`completion_token_cost_unit`</b> : l’unité du coût des jetons de complétion. La valeur par défaut est &quot;USD&quot;. (Actuellement inutilisée ; sera utilisée à l’avenir pour spécifier le type d’unité de coût, par ex. &quot;tokens&quot; ou &quot;time&quot;)
  **Retourne :**
  Un objet CostCreateRes. Il contient un champ `ids`, qui est une liste de tuples. Chaque tuple contient le llm&#95;id et l’ID de l’objet de coût créé.

```python
get_call(
```

### <kbd>méthode</kbd> `get_calls`

```python
get_calls(
    filter: 'CallsFilterLike | None' = None,
    limit: 'int | None' = None,
    offset: 'int | None' = None,
    sort_by: 'list[SortByLike] | None' = None,
    query: 'QueryLike | None' = None,
    include_costs: 'bool' = False,
    include_feedback: 'bool' = False,
    include_storage_size: 'bool' = False,
    include_total_storage_size: 'bool' = False,
    columns: 'list[str] | None' = None,
    expand_columns: 'list[str] | None' = None,
    return_expanded_column_values: 'bool' = True,
    scored_by: 'str | list[str] | None' = None,
    page_size: 'int' = 1000
) → CallsIter
```

Récupère une liste d’Appels tracés (opérations) pour ce projet.

Cette méthode offre une interface puissante et flexible pour interroger les données de trace. Elle prend en charge la pagination, le filtrage, le tri, la projection de champs et les métadonnées de score, et peut être utilisée pour créer des interfaces utilisateur de trace personnalisées ou des outils d’analyse.

Astuce de performance : spécifiez `columns` et utilisez `filter` ou `query` pour réduire la taille des résultats.

**Arguments :**

* <b>`filter`</b>:  Filtre de haut niveau pour restreindre les résultats selon des champs tels que `op_name`, `parent_ids`, etc.
* <b>`limit`</b>:  Nombre maximal d’Appels à renvoyer.
* <b>`offset`</b>:  Nombre d’Appels à ignorer avant de renvoyer les résultats (utilisé pour la pagination).
* <b>`sort_by`</b>:  Liste des champs selon lesquels trier les résultats (par exemple, `started_at desc`).
* <b>`query`</b>:  Une expression de type Mongo pour un filtrage avancé. Tous les opérateurs Mongo ne sont pas pris en charge.
* <b>`include_costs`</b>:  Si True, inclut les informations sur les jetons et les coûts dans `summary.weave`.
* <b>`include_feedback`</b>:  Si True, inclut les retours dans `summary.weave.feedback`.
* <b>`include_storage_size`</b>:  Si True, inclut la taille de stockage d’un Appel.
* <b>`include_total_storage_size`</b>:  Si True, inclut la taille totale de stockage d’une trace.
* <b>`columns`</b>:  Liste des champs à renvoyer par Appel. Réduire cette liste peut considérablement améliorer les performances.  (Certains champs, comme `id`, `trace_id`, `op_name` et `started_at`, sont toujours inclus.)
* <b>`scored_by`</b>:  Filtre par un ou plusieurs évaluateurs (nom ou URI de référence). Plusieurs évaluateurs sont combinés avec un ET.
* <b>`page_size`</b>:  Nombre d’Appels récupérés par page. Ajustez ce paramètre pour optimiser les performances des requêtes volumineuses.

**Retourne :**

* <b>`CallsIter`</b>:  Un itérateur sur des objets `Call`. Prend en charge le slicing, l’itération et `.to_pandas()`.

**Exemple :**

```python
calls = client.get_calls(
     filter=CallsFilter(op_names=["my_op"]),
     columns=["inputs", "output", "summary"],
     limit=100,
)
for call in calls:
     print(call.inputs, call.output)
```

### <kbd>méthode</kbd> `get_evaluation`

```python
get_evaluation(uri: 'str') → Evaluation
```

Récupérer un objet Évaluation spécifique à partir de son URI.

Les URI d’évaluation suivent généralement le format : `weave:///entity/project/object/Evaluation:version`

Vous pouvez également obtenir l’évaluation à partir de son nom « convivial » : get&#95;evaluation(&quot;Evaluation:v1&quot;)

**Arguments :**

* <b>`uri`</b> (str):  L’identifiant de ressource unique de l’évaluation à récupérer.

**Retourne :**

* <b>`Evaluation`</b>:  L’objet Évaluation correspondant à l’URI fournie.

**Exceptions levées :**

* <b>`TypeError`</b>:  Si l’objet à l’URI n’est pas une instance d’Évaluation.
* <b>`ValueError`</b>:  Si l’URI n’est pas valide ou si l’objet est introuvable.

**Exemples :**

```python
client = weave.init("my-project")
evaluation = client.get_evaluation("weave:///entity/project/object/my-eval:v1")
print(evaluation.name)
```

* <b>`limit`</b>: Le nombre maximal d&#39;objets de feedback à récupérer.

### <kbd>méthode</kbd> `set_wandb_run_context`

```python
set_wandb_run_context(run_id: 'str', step: 'int | None' = None) → None
```

Redéfinissez `run_id` et `step` du run wandb pour les Appels créés par ce client.

Cela vous permet d’associer les Appels Weave à un run W&amp;B spécifique qui n’est pas lié au symbole global `wandb.run`.

**Arguments :**

* <b>`run_id`</b>: ID du run (sans le préfixe entité/projet). Le client ajoutera automatiquement le préfixe entité/projet.
* <b>`step`</b>: Le numéro d’étape à utiliser pour les Appels. Si la valeur est `None`, l’étape ne sera pas définie.
  **Exemples :**

```python
client = weave.init("my-project")
client.set_wandb_run_context(run_id="my-run-id", step=5)
# Tous les appels seront désormais associés à entity/project/my-run-id à l'étape 5

# Ou sans étape
client.set_wandb_run_context(run_id="my-run-id")
# Les appels seront associés à entity/project/my-run-id sans étape
```
