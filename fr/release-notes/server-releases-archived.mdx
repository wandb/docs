<Update label="v0.67.0" description="28 mars 2025">
  ## Fonctionnalités

  * Dans Reports, vous pouvez désormais attribuer à un run un nom d’affichage personnalisé pour chaque grille de panneaux. Cela vous permet de remplacer le nom du run au moment de l’entraînement, souvent long et peu explicite, par un nom plus parlant pour votre public. Le rapport met à jour ce nom dans toutes les grilles de panneaux, ce qui vous aide à présenter à vos collègues vos résultats expérimentaux de manière concise et lisible. Le nom original du run reste intact dans le projet, donc cela ne perturbera pas vos collaborateurs.
  * Lorsque vous développez un panneau dans le Workspace, il s’ouvre désormais en mode plein écran avec davantage d’espace. Dans cette vue, les graphiques en courbes offrent maintenant un rendu plus fin, avec jusqu’à 10 000 bins. Le run selector apparaît à côté du panneau, ce qui vous permet de basculer, regrouper ou filtrer facilement les runs directement dans ce contexte.
  * Depuis n’importe quel panneau, vous pouvez désormais copier une URL unique qui renvoie directement à la vue plein écran de ce panneau. Cela facilite encore davantage le partage d’un lien pour explorer des motifs intéressants ou pathologiques dans vos graphiques.
  * Run Comparer est un outil puissant qui vous permet de comparer les configurations et les métriques clés des runs importants, ainsi que leurs courbes de perte. Run Comparer a été mis à jour :
    * Ajout plus rapide d’un panneau Run Comparer, désormais proposé comme option développée dans **Add Panels**.
    * Par défaut, un panneau Run Comparer occupe plus d’espace, pour que vous puissiez voir les valeurs immédiatement.
    * Lisibilité améliorée d’un panneau Run Comparer. De nouveaux contrôles permettent de modifier rapidement la taille des lignes et des colonnes pour lire plus facilement les valeurs longues ou imbriquées.
    * Vous pouvez copier n’importe quelle valeur du panneau dans votre presse-papiers en un seul clic.
    * Vous pouvez rechercher des clés à l’aide d’expressions régulières pour trouver rapidement exactement le sous-ensemble de métriques que vous souhaitez comparer. Votre historique de recherche est enregistré pour vous aider à passer efficacement d’une vue à l’autre.
  * Les commandes du masque de segmentation ont été mises à jour :
    * Vous pouvez désormais activer ou désactiver en une seule fois chaque type de masque, ou activer ou désactiver tous les masques ou toutes les images.
    * Vous pouvez désormais modifier la couleur attribuée à chaque classe, afin d&#39;éviter toute confusion si plusieurs classes utilisent la même couleur.
  * Lorsque vous ouvrez un panneau multimédia en mode plein écran, vous pouvez désormais utiliser les flèches gauche et droite de votre clavier pour parcourir les images, *sans* avoir à cliquer d’abord sur le curseur d’étape.
  * Les panneaux multimédias colorent désormais les noms des runs, comme dans le run selector. Il est ainsi plus facile d’associer les valeurs multimédias d’un run aux métriques et graphiques correspondants.
  * Dans le sélecteur de run, vous pouvez désormais filtrer selon qu’un run possède ou non une clé média donnée.
  * Vous pouvez désormais déplacer des runs d’un groupe à l’autre dans l’UI de l’application W&amp;B, et créer de nouveaux groupes après l’enregistrement du run.
  * Les Automations peuvent désormais être modifiées dans l’interface utilisateur
  * Une automatisation peut désormais notifier un canal Slack en cas d’événements liés à un artifact. Lors de la création d’une automatisation, sélectionnez « notification Slack » comme type d’action.
  * Le registre prend désormais en charge la recherche globale par défaut, ce qui vous permet d’effectuer des recherches dans tous les registres par nom de registre, nom de collection, alias ou tag.
  * Dans les panneaux Tables et Query qui utilisent l’expression `runs`, vous pouvez utiliser le nouveau curseur d’étape de l’historique des Runs ainsi que les menus déroulants pour afficher un tableau des métriques à chaque étape d’un run.
  * Le playground W&amp;B Weave prend en charge de nouveaux modèles : `gpt-4.5-preview` d&#39;OpenAI, ainsi que `deepseek-chat` et `deepseek-reasoner` de Deepseek.
  * Tracing Weave propose deux nouvelles intégrations avec des frameworks d’agents : CrewAI et l’Agent SDK d’OpenAI.
  * Dans l’interface Weave UI, vous pouvez désormais créer des Datasets à partir de traces. [En savoir plus](/fr/weave/guides/core-types/datasets#create-edit-and-delete-a-dataset-in-the-ui)
  * Le SDK Python Weave fournit désormais un moyen de filtrer les entrées et les sorties de vos données dans Weave afin de garantir que les données sensibles ne quittent pas le périmètre de votre réseau. Vous pouvez configurer le SDK pour masquer les données sensibles. En savoir plus : /weave/guides/tracking/redact-pii/
  * Pour améliorer votre expérience, l’onglet System dans la vue Workspace d’un run individuel sera supprimé dans une prochaine version. Consultez les informations complètes sur les métriques système dans la section System du Workspace. Pour toute question, contactez [support@wandb.com](mailto:support@wandb.com).

  ## Sécurité

  * `golang crypto` a été mis à jour vers la version v0.36.0.
  * `golang oauth2` a été mis à jour vers la version v0.28.0.
  * Dans Weave, `pyarrow` est désormais verrouillé sur la version v17.0.0.

  ## Performances

  * Les mises à jour du front-end réduisent considérablement les temps de rechargement du Workspace en stockant les données essentielles dans le cache du navigateur d’une visite à l’autre. Cette mise à jour optimise le chargement des vues enregistrées, des noms de métriques, du run selector, du nombre de runs, des détails de configuration de W&amp;B, ainsi que le recalcul des vues du Workspace.
  * Les pages d’aperçu du registre se chargent désormais bien plus rapidement.
  * Amélioration des performances de la sélection des métriques utilisées pour les valeurs X, Y ou Z dans un nuage de points au sein d’un workspace comportant des milliers de runs ou des centaines de métriques.
  * Améliorations des performances de la journalisation des évaluations dans Weave.

  ## Correctifs

  * Correction d’un bug dans Reports : suivre un lien vers une section d’un rapport n’ouvrait pas cette section.
  * Amélioration de la gestion de la réflexion des indices par le lissage gaussien, afin qu’elle corresponde au mode &quot;reflect&quot; par défaut de SciPy.
  * Un lien vers un commentaire de Report envoyé par e-mail mène désormais directement au commentaire.
  * Correction d’un bug susceptible de faire planter un espace de travail si un balayage dépassait 2 milliards de secondes de calcul, en remplaçant le type de variable des secondes de calcul du balayage par `int64` au lieu de `int32`.
  * Correction de bugs d’affichage pouvant survenir lorsqu’un report contenait plusieurs ensembles de runs.
  * Correction d’un bug où les panneaux ajoutés avec Quick add à une section triée par ordre alphabétique n’étaient pas triés correctement.
  * Correction d’un bug qui générait des liens d’invitation utilisateur invalides.
</Update>

<Update label="v0.58.1" description="4 septembre 2024">
  ## Fonctionnalités

  * W&amp;B prend désormais en charge les sous-chemins pour la fonctionnalité **connecteur de stockage sécurisé, c.-à-d. Bring your own bucket**. Vous pouvez maintenant indiquer un sous-chemin lors de la configuration d’un bucket au niveau de l’instance ou de l’équipe. Cette possibilité n’est disponible que pour les nouvelles configurations de bucket, pas pour les buckets déjà configurés.
  * Le stockage géré par W&amp;B sur les nouvelles instances Cloud dédié dans Google Cloud et Azure sera désormais chiffré par défaut avec des **clés cloud natives gérées par W&amp;B**. Cette fonctionnalité est déjà disponible sur les instances AWS. Le stockage de chaque instance est chiffré avec une clé propre à l’instance. Jusqu’à présent, toutes les instances sur Google Cloud et Azure utilisaient les clés de chiffrement par défaut gérées par le fournisseur cloud.
  * Les champs de la run config et du summary peuvent désormais être copiés par clic.
  * Si vous utilisez l’opérateur Kubernetes W&amp;B pour une instance Autogéré, vous pouvez désormais utiliser, de façon facultative, une autorité de certification personnalisée pour le controller manager.
  * Nous avons modifié l’opérateur Kubernetes W&amp;B pour qu’il s’exécute par défaut dans un contexte non root, en conformité avec les Security Context Constraints (SCC) d’OpenShift. Ce changement garantit un déploiement plus fluide des instances Autogéré sur OpenShift en respectant ses politiques de sécurité.

  ## Correctifs

  * Correction d’un problème où l’exportation de panneaux d’un Workspace vers un report respectait désormais correctement la regex de recherche des panneaux.
  * Correction d’un problème où la définition de `GORILLA_DISABLE_PERSONAL_ENTITY` sur `true` n’empêchait pas les Users de créer des projets et d’écrire dans des projets existants dans leurs entités personnelles.

  ## Améliorations des performances

  * Nous avons considérablement amélioré les performances et la stabilité pour les expériences avec plus de 100k points enregistrés. Sur une instance Autogéré, cette amélioration est disponible si le déploiement est géré à l’aide de l’opérateur Kubernetes W&amp;B.
  * Correction d’un problème où l’enregistrement des modifications dans de grands Workspaces pouvait être très lent ou échouer.
  * Amélioration de la latence à l’ouverture des sections de Workspace dans de grands Workspaces.
</Update>
