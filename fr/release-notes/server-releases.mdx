<Update label="v0.79.0" description="24 mars 2026">
  Le serveur W&amp;B 0.79.0 ajoute le zoom et le déplacement sur les images, une gestion améliorée des comptes de service et des clés API, ainsi que des améliorations des performances. Inference ajoute les modèles MiniMax M2.5 et MiniMaxAI M2.5. Les moniteurs Weave prennent désormais en charge le calcul du score pour toutes les opérations, et l’interface Weave ajoute une gestion améliorée des tags et des alias de prompts, une vue de chat pour les agents Claude, le filtrage par réactions emoji ou par contenu des notes, et plus encore.

  ## Support et fin de vie

  * Le serveur W&amp;B v0.67 a atteint sa fin de vie.

  Consultez [Politiques et processus de version](/fr/release-notes/release-policies). Pour toute demande d’assistance ou question, contactez le [support](mailto:support@wandb.com).

  ## Sécurité

  * Lors de la création d’un compte de service, vous êtes désormais automatiquement guidé pour créer une clé API.
  * Une confirmation est désormais requise pour supprimer un compte de service ou une clé API depuis l’application W&amp;B.
  * Le backend échappe désormais les noms de colonnes dans les filtres afin de contribuer à prévenir les injections SQL.
  * Les adresses e-mail contenant des [marques diacritiques](https://en.wikipedia.org/wiki/Combining_Diacritical_Marks) sont désormais bloquées lors de l’inscription ou de la connexion des utilisateurs.

  ## Fonctionnalités

  * **Zoomez et déplacez-vous dans une image** pour inspecter les détails fins directement dans la fenêtre modale d’image en plein écran de W&amp;B. Ouvrez n’importe quelle image en plein écran, puis :

    * Utilisez les contrôles de l’interface utilisateur ou les raccourcis clavier pour définir votre niveau de zoom, réinitialiser à 100 % ou adapter l’image à l’écran.
    * Cliquez et faites glisser pour déplacer l’image lorsque vous êtes zoomé.

    <Frame>
      ![Contrôles de déplacement et de zoom dans la fenêtre modale d’image en plein écran](/images/release-notes/v0-79-0/image-zoom-pan-fullscreen-modal.png)
    </Frame>
  * Pour les runs épinglés et les runs de référence, les [écarts de métriques de synthèse](/fr/models/runs/compare-runs#summary-metric-deltas) montrent comment chaque run se compare à la référence. Pour activer la coloration sémantique comme repère visuel rapide, vous pouvez définir le sens de la métrique pour une colonne. Cette fonctionnalité est désormais disponible de manière générale.
    <Frame>
      ![Écarts de métriques de synthèse dans le tableau des runs comparés à la référence](/images/release-notes/v0-78-0/baseline-deltas-runs-table.png)
    </Frame>
  * Le sélecteur de runs et la liste des Runs ont été améliorés :
    * Ajout de la prise en charge du filtrage par commit Git.
    * Pour les runs qui spécifient des paramètres de configuration, ajout de la prise en charge de l’opérateur `CONTAINS`.
  * Ajout d’une mise en cache défensive pour les champs de projet afin de réduire la charge sur la base de données.

  ### Registre

  * Ajout de nouvelles limites de débit pour la création d’artefacts afin de réduire le risque de surcharge du système.

  ### Weave

  <Note>Pour les déploiements utilisant Weave, ClickHouse v0.25.8+ est requis avant la mise à niveau vers v0.78+.</Note>

  * Weave prend désormais en charge la vue de chat pour les agents Claude.
  * Les pages **Traces** et **Évaluations** prennent désormais en charge le filtrage par contenu des notes et par réactions emoji.
  * Vous pouvez désormais gérer les tags et alias des prompts dans l’onglet **Prompts** de la page **Assets**.
  * Les tableaux des pages **Appels** et **Traces** prennent désormais en charge la réorganisation des colonnes par glisser-déposer.
  * Les moniteurs Weave prennent désormais en charge le calcul du score sur l’ensemble des opérations.

  ## Correctifs

  * Correction des erreurs d’arrondi en virgule flottante dans le lissage du backend.
  * Correction d’un bug où la saisie dans les champs d’expression faisait sauter le curseur.
  * Correction d’un bug où des images à taille réelle pouvaient se chevaucher en mode galerie.
  * Correction d’un problème d’alignement des boîtes englobantes avec des légendes longues.
  * Suppression des ombres portées des panneaux et des superpositions au survol.
  * Les résultats de recherche des panneaux incluent désormais les graphiques personnalisés.
</Update>

  * Une organisation W&amp;B ou un déploiement W&amp;B peut désormais avoir au maximum 100 clés API personnelles, 100 clés API au niveau de l’organisation pour chaque utilisateur et 100 clés API par compte de service.

  * Correction d’un bug : le champ de texte **Notes** d’un run était trop étroit en mode modification. Désormais, la zone de texte a la même largeur en mode affichage et en mode modification.

  * Correctif d’un bug : l’interface Weave dans le Cloud dédié affichait des liens vers W&amp;B Inférence, qui n’est pas disponible dans le Cloud dédié.
  * Correction d’un bug où les graphiques à barres de la page de comparaison [Évaluation](/fr/weave/guides/core-types/evaluations) pouvaient être redimensionnés à une largeur insuffisante lorsque le volet latéral des détails était ouvert.

  * Correction d’un bug qui empêchait de modifier les variables de template lors du chargement d’un appel dans [Weave Playground](https://wandb.ai/site/playground/).

  * Les performances de nombreuses interactions dans l’espace de travail ont été améliorées, en particulier dans les grands espaces de travail. Par exemple, le dépliage des sections et l’utilisation du sélecteur de runs sont désormais nettement plus réactifs.
  * Performances de Fork et Rewind améliorées.

    [Forking](/fr/models/runs/forking) d’un run crée un nouveau run qui utilise la même configuration qu’un run existant. Les modifications apportées au run forké n’affectent pas le run parent, et inversement. Une référence est conservée entre le run forké et le run parent. [Rembobiner](/fr/models/runs/rewind) un run vous permet de consigner de nouvelles données à partir de ce moment-là sans perdre les données existantes.

    Dans les projets comportant de nombreux forks imbriqués, la création de nouveaux runs par fork est désormais bien plus efficace grâce aux améliorations apportées à la mise en cache.

  ## Correctifs

  * Correctif d’un bug pouvant empêcher l’ajout d’un compte de service d’organisation à de nouvelles équipes.
  * Correction d’un bug pouvant entraîner l’absence de marqueurs au survol pour les lignes groupées.
  * Correctif d’un bug qui pouvait faire apparaître des noms de projet non valides dans le menu déroulant **Import** d’un panneau de rapport.
  * Correction d’un bug d’affichage affectant l’alignement des filtres dans le sélecteur de run.
  * Correction d’un plantage de la page lors de l’ajout du filtre d’horodatage **Within Last**
  * Correctif d’un bug pouvant empêcher de régler l’axe des x sur **Wall Time** dans les paramètres globaux du graphique en courbes.
  * Correction d’un bug pouvant empêcher l’affichage des légendes d’images lorsqu’elles sont enregistrées dans un tableau.
  * Correction d’un bug pouvant empêcher l’affichage des métriques éparses dans les panneaux.
  * Dans les pages **Aperçu du run**, le champ **Description** s&#39;appelle désormais **Notes**.
