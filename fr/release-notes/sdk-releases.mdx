Cette page présente les notes de version du SDK Python W&amp;B (package `wandb`). Pour W&amp;B Server, voir les [notes de version de W&amp;B Server](/fr/release-notes/server-releases).

<Update label="0.25.1" description="10 mars 2026">
  ## Ajouts

  * Nouveau volet de journaux de console du run dans la commande `wandb beta leet`. Basculez avec `l`.
  * Nouveau volet de métriques système dans le mode Workspace multi-run de la commande `wandb beta leet`. Basculez avec `s`.
  * La commande `wandb beta leet` prend désormais en charge le filtrage des métriques système. Basculez avec `\`.
  * `ArtifactType.collections()` prend désormais en charge le filtrage et le tri des collections.
  * Un avertissement est désormais enregistré si `run.log_artifact` ne crée pas de nouvelle version parce que le contenu de l’artefact est identique à celui d’une version existante.
  * La nouvelle méthode `Project.collections()` récupère les collections d’artefacts filtrées et triées d’un projet.
  * La nouvelle commande CLI `wandb purge-cache` nettoie les fichiers mis en cache.

  ## Correctifs

  * Correction d’un interblocage rare provoqué lorsque le ramasse-miettes se déclenche à un moment mal choisi et exécute une méthode `__del__` qui affiche du texte.
  * Correction d’un bug où le chargement de plusieurs runs avec `api.Runs("...")`, puis la tentative d’obtenir l’utilisateur d’un seul run avec un code comme celui-ci, pouvait échouer avec une `AttributeError` :

    ```python
    for run in runs:
      print(run.user)
    ```
</Update>

<Update label="0.25.0" description="12 février 2026">
  ## Changements notables

  * Python 3.8 n’est plus pris en charge.

  ## Ajouts

  * Le TUI `wandb beta leet` prend désormais en charge les espaces de travail multi-run.
  * Configurez de manière interactive la configuration persistante de `wandb beta leet` avec la commande `wandb beta leet config`.
  * Accédez aux informations utilisateur du propriétaire du projet via la propriété `owner` de `wandb.apis.public.Project`.

  ## Correctifs

  * Correction d’un bug où un agent de balayage pour un balayage supprimé pouvait s’exécuter indéfiniment avec des erreurs 404 répétées.
  * Correction d’un bug où un plantage de `wandb-core` pouvait produire des traces d’erreur extrêmement longues et répétitives dans les anciennes versions de Python.
  * Correction d’une régression introduite dans la v0.24.0 où une synchronisation TensorBoard pouvait s’arrêter après avoir synchronisé 1 Mo de données.
</Update>

<Update label="0.24.2" description="4 février 2026">
  ## Ajouts

  * `wandb.Api()` prend désormais en charge l’authentification fédérée (basée sur JWT).

  ## Correctifs

  * L’URL de téléchargement présignée est désormais actualisée si elle expire pendant le téléchargement de fichiers d’artefact.
</Update>

<Update label="0.24.1" description="29 janvier 2026">
  ## Changements notables

  * Les runs créés avec `wandb==0.24.0` peuvent ne pas parvenir à téléverser certaines données, ce que cette version corrige. Les données manquantes sont stockées dans le fichier `.wandb` du run et peuvent être téléversées à nouveau avec `wandb sync`.

  ## Ajouts

  * `download_history_exports` dans la classe `api.Run` pour télécharger l’historique exporté du run au format de fichier parquet.

  ## Modifications

  * Lorsqu’un fichier de paramètres (tel que `./wandb/settings` ou `~/.config/wandb/settings`) contient un paramètre non valide, tous les fichiers de paramètres sont ignorés et une erreur est affichée.

  ## Correctifs

  * Après `wandb login --host <invalid-url>`, l’utilisation de `wandb login --host <valid-url>` fonctionne normalement.
  * Correction d’un bug où `wandb beta sync` pouvait rester bloqué sur `Syncing...` et ne pas parvenir à charger les informations d’identification.
  * Correction d’un bug pouvant occasionnellement entraîner des données non téléversées. Désormais, toutes les données sont stockées dans le fichier `.wandb` du run et peuvent être téléversées à nouveau avec `wandb sync`.

  Ces régressions ont été introduites dans la version 0.24.0.
</Update>

<Update label="0.24.0" description="13 janvier 2026">
  <Warning>
    Cette version a été retirée de PyPI. N’utilisez pas cette version. Si vous utilisez cette version, passez immédiatement à la v0.24.1 ou à une version plus récente. En raison d’une régression corrigée dans la v0.24.1, les runs créés avec `wandb==0.24.0` risquent de ne pas téléverser certaines données.
  </Warning>

  ## Changements notables

  Cette version supprime le module hérité et obsolète `wandb.beta.workflows`, y compris ses fonctions `log_model()`/`use_model()`/`link_model()`. Il s’agit officiellement d’un changement avec rupture de compatibilité.

  ## Ajouts

  * `wandb agent` et `wandb.agent()` acceptent désormais l’indicateur `forward_signals` (CLI : `--forward-signals/-f`) pour relayer SIGINT/SIGTERM et d’autres signaux interceptables de l’agent vers les runs enfants de son balayage, ce qui permet des arrêts plus propres lorsque vous interrompez un processus d’agent.
  * `wandb beta sync` prend désormais en charge l’option `--live` pour synchroniser un run pendant sa journalisation.

  ## Suppression

  * Suppression du module obsolète `wandb.beta.workflows`, y compris ses fonctions `log_model()`, `use_model()` et `link_model()`, dont les successeurs modernes sont respectivement les méthodes `Run.log_artifact`, `Run.use_artifact` et `Run.link_artifact`.

  ## Correctif

  * Correction des annotations de type de `Run.__exit__` pour accepter les valeurs `None`, transmises lorsqu’aucune exception n’est levée.
  * Correction de l’erreur `Invalid Client ID digest` lors de la création d’artefacts après l’appel à `random.seed()`. Des ID client pouvaient entrer en collision lorsque l’état aléatoire était initialisé de manière déterministe.
  * Correction d’une erreur CLI lors du listage d’artefacts vides.
  * Correction d’une régression lors de l’appel à `api.run()` sur un run Sweeps.
  * Correction du message « View run at » affiché à la fin d’un run, qui ne contenait parfois pas d’URL.
  * Les Runs interrogés via `wandb.Api()` affichent désormais une représentation sous forme de chaîne dans les notebooks VSCode au lieu d’une fenêtre HTML défectueuse.
</Update>

<Update label="v0.23.1" description="2 décembre 2025">
  ## Ajouts

  * Prise en charge des regex dans les filtres de métriques et d’aperçu des runs de la commande `wandb beta leet`.
  * Inspection des graphiques dans la commande `wandb beta leet` : faites un clic droit et faites glisser pour afficher (x, y) au point de données le plus proche ; maintenez Alt enfoncé pour une inspection synchronisée sur tous les graphiques visibles.
  * L’API Automations prend désormais en charge la création et la modification d’automatisations qui se déclenchent en fonction des états des runs.
  * L’API Automations prend désormais en charge les événements d’automatisation zscore de base.
  * Ajout de la méthode `beta_history_scan` aux objets `Run` pour accélérer le parcours de l’historique avec `wandb.Api`.

  ## Modification

  * `wandb.Api()` lève désormais une `UsageError` si `WANDB_IDENTITY_TOKEN_FILE` est défini et qu’aucune clé API explicite n’est fournie.

  ## Obsolète

  * Le mode anonyme est obsolète et émettra des avertissements. Cette obsolescence inclut le paramètre `anonymous`, la variable d’environnement `WANDB_ANONYMOUS`, `wandb.init(anonymous=...)` et les commandes `wandb login --anonymously` et `wandb.login(anonymous=...)`.

  ## Correctif

  * `wandb.Image()` n’affiche plus d’avertissement d’obsolescence.
  * `Registry.description` et `ArtifactCollection.description` ne rejettent plus les chaînes vides.
  * L’instanciation des objets `Artifact` est désormais nettement plus rapide.
  * `wandb.Run.save()` revient désormais aux liens physiques et, si nécessaire, à la copie (en rétrogradant la politique de fichier &#39;live&#39; à &#39;now&#39;, le cas échéant) lorsque les liens symboliques sont désactivés ou indisponibles (par ex. entre volumes ou sans Developer Mode sous Windows).
  * Les alias de collection d’artifacts sont désormais récupérés à la demande lors de l’accès à `ArtifactCollection.aliases` plutôt que lors de l’instanciation d’`ArtifactCollection`, ce qui améliore les performances de `Api.artifact_collections()`, `Api.registries().collections()`, etc.
  * La clé API explicitement fournie dans `wandb.init(settings=wandb.Settings(api_key="..."))` est désormais utilisée pour la connexion. La clé du run est utilisée lors de la journalisation d’artefacts via `run.log_artifact`.
  * L’interface TUI W&amp;B LEET affiche correctement les valeurs négatives des graduations de l’axe Y ainsi que les unités de base et d’affichage de certaines métriques système.
  * Correction d’une rare boucle infinie dans `console_capture.py`.
  * Le téléversement/téléchargement de fichiers respecte désormais `WANDB_X_EXTRA_HTTP_HEADERS`, sauf pour les [artefacts de référence](/fr/models/artifacts/track-external-files).
</Update>

<Update label="v0.21.0" description="1er juillet 2025">
  ## Changements notables

  Cette version supprime l’implémentation héritée du processus `service`. Il s’agit d’un changement incompatible.

  ## Ajouts

  * Ajout du paramètre `x_stats_track_process_tree` pour suivre des métriques propres au processus, telles que la RSS, le pourcentage d’utilisation du CPU et le nombre de threads utilisés sur l’ensemble de l’arborescence des processus, à partir de `x_stats_pid`. Cette opération peut être coûteuse et est désactivée par défaut.
  * Les notes sont désormais renvoyées au client lors de la reprise d’un run.
  * Ajout de la prise en charge de la création de préréglages de graphiques Vega personnalisés via l’API. Les utilisateurs peuvent désormais définir et téléverser des spécifications de graphiques personnalisées, puis les réutiliser dans plusieurs runs avec wandb.plot&#95;table().

  ## Modifications

  * L’appel à `Artifact.link()` n’instancie plus de run factice temporaire.
  * `wandb` tente désormais d’utiliser des sockets Unix pour l’IPC au lieu d’écouter sur localhost, ce qui lui permet de fonctionner dans des environnements aux autorisations plus restrictives (comme Databricks).
  * `Api.artifact()` affiche désormais un avertissement lors de la récupération d’Artifacts à partir de collections migrées du registre de modèles.
  * La propriété `.length` des objets interrogés via `wandb.Api` a été déclarée obsolète. Utilisez `len(...)` à la place.

  ## Suppressions

  * Suppression de l’implémentation Python héritée du processus `service`. L’option `legacy-service` de `wandb.require`, ainsi que les paramètres `x_require_legacy_service` et `x_disable_setproctitle` et les variables d’environnement correspondantes, ont été supprimés et provoqueront désormais une erreur s’ils sont utilisés.

  * Suppression de l’attribut privé `wandb.Run._metadata`. Pour redéfinir le nombre de CPU et de GPU détectés automatiquement, ainsi que le type de GPU, utilisez les nouveaux paramètres `x_stats_{cpu_count,cpu_logical_count,gpu_count,gpu_type}`.

  ## Correctifs

  * Autorise les URI CoreWeave au format S3 pour les artefacts de référence.
  * Correction d’un bug rare qui rendait Ctrl+C inopérant après la journalisation de grandes quantités de données.
  * Respecte les paramètres `silent`, `quiet` et `show_warnings` transmis à une instance de `Run` pour les avertissements émis par le processus `service`.
  * `api.Runs` n’effectue plus d’appel d’API pour chaque run chargé depuis W&amp;B.
  * Analyse correctement le paramètre `x_extra_http_headers` à partir de la variable d’environnement.
  * `.length` appelle le backend W&amp;B pour charger la longueur des objets lorsqu’aucune donnée n’a été chargée, au lieu de renvoyer `None`.

  Consultez le [journal des modifications de la version v0.21.0](https://github.com/wandb/wandb/releases/tag/v0.21.0) sur GitHub.
</Update>

<Update label="v0.20.1" description="4 juin 2025">
  ## Correctifs

  * `wandb.Image()` ne fonctionnait plus dans la version 0.20.0 lorsqu’on lui fournissait des tableaux NumPy avec des valeurs dans l’intervalle [0, 1] ; le problème est désormais corrigé.

  * wandb.Table : ajout d’un nouveau paramètre de constructeur, `log_mode`, avec les options `"IMMUTABLE"` et `"MUTABLE"`.
