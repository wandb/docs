Cette page présente les notes de version du W&amp;B Python SDK (package `wandb`). Pour W&amp;B Server, voir les [notes de version de W&amp;B Server](/fr/release-notes/server-releases).

<Update label="0.26.1" description="23 avril 2026">
  ## Ajouts

  * Les méthodes `wandb.Api` qui renvoient des artifacts, des registres, des automatisations et les paginateurs associés acceptent un argument `start` facultatif, ce qui vous permet de reprendre l’itération à partir d’un curseur enregistré.
  * Le paramètre `stop_fn` de `wandb.Settings` permet de personnaliser la façon dont le SDK demande au processus Python de s’arrêter lorsqu’un arrêt de run est demandé (par défaut, il envoie SIGINT). Transmettez ce paramètre avec `wandb.setup(settings=...)` ou `wandb.init(settings=...)`. Comme la valeur est un callable, elle ne peut pas être définie via une variable d’environnement.

  ## Modifications

  * Dans `wandb beta leet`, les touches WASD et les touches fléchées sont interchangeables pour déplacer le focus dans chaque volet (grilles de graphiques, navigation dans les listes). Les touches **Home**, **End**, **Page Up** et **Page Down** fonctionnent de manière cohérente. Le volet multimédia utilise toujours les touches fléchées pour parcourir l’axe X et les touches WASD pour se déplacer entre les vignettes d’image.

  ## Correctifs

  * `wandb.init(id=run_id, reinit="create_new")` renvoie désormais une erreur si un autre run avec le même `run_id` est encore actif dans le même script.
  * `wandb.Api` ne renvoie plus d’erreur pour certaines opérations lorsque le mode hors ligne est activé avec la variable d’environnement `WANDB_MODE` ou le paramètre `mode`.
</Update>

<Update label="0.26.0" description="13 avril 2026">
  ## Modifications notables

  Cette version met fin à la compatibilité avec les versions de Server antérieures à 0.63.0 pour les déploiements W&amp;B en Cloud dédié et Autogéré.

  ## Ajouts

  * Les commandes `wandb beta core start|stop` lancent un service `wandb-core` détaché et le réutilisent entre plusieurs processus via la variable d&#39;environnement `WANDB_SERVICE`.
  * Le décorateur `@wandb_log` ajoute la prise en charge de Kubeflow Pipelines v2 (`kfp>=2.0.0`).
  * Définissez la propriété `wandb.Settings(allow_media_symlink=true)` pour créer un lien vers les fichiers multimédias dans le répertoire d&#39;exécution au lieu de les copier. Cela améliore les performances et économise de l&#39;espace disque. Un lien symbolique est utilisé si le système de fichiers le prend en charge. Si ce n&#39;est pas le cas, comme sous Windows, un lien physique est utilisé à la place. Le SDK ne vérifie pas si le fichier multimédia a été modifié localement avant son téléversement.
  * Utilisez la méthode `run.pin_config_keys(keys)` pour épingler par programmation certaines clés de configuration afin de les afficher dans la section **References** de la page **Run Overview**.
  * Les méthodes `Sweep.agent().Runs()` et `Sweep.agents().Runs()` renvoient les détails des exécutions d&#39;un sweep en cours sur un agent donné ou sur l&#39;ensemble des agents.
  * Sur l&#39;infrastructure Google TPU, le SDK tente de collecter de nouvelles métriques telles que `tensorcore_util` et des distributions de latence, en plus de `duty_cycle_pct` et `hbm_capacity_total`/`hbm_capacity_usage`. Voir [TPU monitoring library: Supported metrics](https://docs.cloud.google.com/tpu/docs/tpu-monitoring-library#list-all-supported-metric-names) dans la documentation Google Cloud.
  * Sur les GPU Nvidia Hopper+ (H100 et versions ultérieures), les métriques de profilage NVML GPU Performance Monitoring sont collectées, offrant une meilleure visibilité sur l&#39;utilisation des SM, l&#39;activité des pipelines tensor/FP, la bande passante DRAM et le débit PCIe/NVLink, sans exécuter le démon DCGM.
  * Mises à jour de l&#39;interface TUI `wandb beta leet` :
    * Prise en charge de l&#39;affichage des tags et des notes dans l&#39;aperçu de l&#39;exécution.
    * Prise en charge du filtrage des exécutions par métadonnées en mode espace de travail multi-exécutions à l&#39;aide du raccourci clavier `f`.
    * Prise en charge de l&#39;affichage des données `wandb.Image` sous forme de vignettes ANSI, avec disposition en grille, défilement sur l&#39;axe X, mode plein écran et navigation au clavier ou au pointeur.
    * Parcourez les modes d&#39;affichage suivants dans un graphique individuel à l&#39;aide du raccourci clavier `y` :
      * Appuyez une fois pour activer l&#39;affichage en échelle logarithmique sur l&#39;axe Y.
      * Pour les métriques système comme l&#39;utilisation du GPU exprimée en pourcentage, appuyez de nouveau pour désactiver l&#39;échelle logarithmique et activer le mode carte thermique par compartiments.
      * Appuyez encore une fois pour revenir au mode d&#39;affichage par défaut.
    * Nouveau mode autonome de surveillance du système : `wandb beta leet symon`.
    * Nouveaux jeux de couleurs adaptés au daltonisme :

      * `dusk-shore` : dégradé sienne/bleu/gris, adapté au mode mono-exécution par graphique.
      * `clear-signal` : thème sienne/bleu/gris avec couleurs entrelacées pour maximiser la différenciation entre séries adjacentes.

      Configurez-les avec `wandb beta leet config`.

  ## Modifications

  * La sérialisation et la désérialisation JSON utilisent désormais `orjson` pour améliorer les performances.
  * Observabilité améliorée des métriques système dans l&#39;interface TUI `wandb beta leet`, avec superpositions multi-séries, inspection et zoom en direct/historique.
  * Réduction du risque de collision entre les couleurs de base des exécutions dans `wandb beta leet`. Désormais, un allocateur de couleurs d&#39;exécution à l&#39;échelle de l&#39;espace de travail préserve l&#39;affinité de la palette hachée et résout les collisions avec des variantes HSL proches.

  ## Correctifs

  * Correction d&#39;un bug où, si aucune branche de suivi amont n&#39;était définie, le SDK ne créait pas de fichier `.patch` `git diff` entre la branche locale et la branche amont la plus proche. Pour activer ce correctif, transmettez la propriété `wandb.Settings(disable_git_fork_point=False)`. Voir [Settings()](/fr/models/ref/python/experiments/settings).
  * Correction d&#39;un bug où `update_automation()` pouvait supprimer silencieusement des filtres d&#39;événements, comme des conditions d&#39;alias sur `OnAddArtifactAlias`, lorsqu&#39;un nouvel événement était fourni.
  * Correction d&#39;un bug pouvant provoquer des collisions d&#39;ID client d&#39;artifact dans des processus enfants issus d&#39;un fork en réinitialisant le générateur d&#39;ID rapide après `fork()`.
  * Correction d&#39;un bug où la variable d&#39;environnement `WANDB__EXTRA_HTTP_HEADERS` n&#39;était pas appliquée aux requêtes de téléversement et de téléchargement présignées vers le stockage d&#39;objets.
  * Correction d&#39;un interblocage dans `artifact.download()` pour les artifacts contenant de nombreux fichiers volumineux.
  * Correction d&#39;un bug où `User.generate_api_key()` pouvait ne pas parvenir à récupérer la liste des noms de clés API existants.
</Update>

<Update label="0.25.1" description="10 mars 2026">

  * Correction d&#39;un bug où le chargement de plusieurs runs avec `api.Runs("...")`, puis la tentative d&#39;obtenir l&#39;utilisateur d&#39;un seul run avec un code comme le suivant, pouvait échouer avec une `AttributeError` :

    ```python
    for run in runs:
      print(run.user)
    ```

  * Correction d&#39;un bug où `wandb beta sync` pouvait rester bloqué sur `Syncing...` et ne pas parvenir à charger les identifiants.

  ## Changements notables

  Cette version supprime le module hérité et obsolète `wandb.beta.workflows`, y compris ses fonctions `log_model()`/`use_model()`/`link_model()`. Il s&#39;agit officiellement d&#39;un changement incompatible.

  ## Ajouté

  * `wandb agent` et `wandb.agent()` acceptent désormais un indicateur `forward_signals` (CLI : `--forward-signals/-f`) pour relayer SIGINT/SIGTERM et d&#39;autres signaux interceptables de l&#39;agent vers les runs enfants de son sweep, ce qui permet des arrêts plus propres lorsque vous interrompez un processus d&#39;agent.
  * `wandb beta sync` prend désormais en charge l&#39;option `--live` pour synchroniser un run pendant son enregistrement.

  ## Supprimé

  * Suppression du module obsolète `wandb.beta.workflows`, y compris ses fonctions `log_model()`, `use_model()` et `link_model()`, dont les successeurs modernes sont respectivement les méthodes `Run.log_artifact`, `Run.use_artifact` et `Run.link_artifact`.

  * Correction de l&#39;erreur `Invalid Client ID digest` lors de la création d&#39;artefacts après l&#39;appel à `random.seed()`. Les ID client pouvaient entrer en collision lorsque l&#39;état aléatoire était initialisé de façon déterministe.
  * Correction d&#39;une erreur CLI lors de l&#39;affichage d&#39;artefacts vides.
  * Correction d&#39;une régression lors de l&#39;appel à `api.run()` sur un run Sweeps.
  * Correction du message &quot;View run at&quot; affiché à la fin d&#39;un run, qui n&#39;incluait parfois pas d&#39;URL.
  * Les Runs interrogés via `wandb.Api()` affichent désormais une représentation sous forme de chaîne dans les notebooks VSCode au lieu d&#39;une fenêtre HTML défectueuse.
