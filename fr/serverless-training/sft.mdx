Utilisez Serverless SFT pour effectuer le fine-tuning de LLM avec un apprentissage supervisé sur des jeux de données sélectionnés. Serverless SFT est en préversion publique. W&amp;B fournit pour vous l&#39;infrastructure d&#39;entraînement ([sur CoreWeave](https://docs.coreweave.com/docs/platform)) et vous offre une flexibilité totale pour configurer votre environnement. Vous bénéficiez d&#39;un accès immédiat à un cluster d&#39;entraînement géré, avec mise à l&#39;échelle automatique, pour prendre en charge vos charges de travail d&#39;entraînement.

Serverless SFT est idéal pour des tâches telles que :

* **Distillation** : transférer les connaissances d&#39;un modèle plus grand et plus performant vers un modèle plus petit et plus rapide.
* **Apprentissage du style et du format de sortie** : entraîner un modèle à respecter des formats de réponse, un ton ou une structure spécifiques.
* **Amorçage avant le RL** : préentraîner un modèle avec des exemples supervisés avant d&#39;appliquer l&#39;apprentissage par renforcement pour l&#39;affiner davantage.

Serverless SFT entraîne des adaptateurs de bas rang (LoRA) pour spécialiser un modèle pour votre tâche. W&amp;B stocke automatiquement les LoRA que vous entraînez comme artefacts dans votre compte. Vous pouvez également les enregistrer localement ou sur un service tiers à des fins de sauvegarde. Serverless Inference héberge aussi automatiquement les modèles que vous entraînez via Serverless SFT.

Pour commencer à entraîner un modèle avec Serverless SFT, voir la [documentation ART sur Serverless SFT](https://art.openpipe.ai/fundamentals/sft-training).

<div id="why-serverless-sft">
  ## Pourquoi Serverless SFT
</div>

Le fine-tuning supervisé (SFT) est une technique d&#39;entraînement dans laquelle un modèle apprend à partir d&#39;exemples d&#39;entrée-sortie sélectionnés. Serverless SFT sur W&amp;B offre les avantages suivants :

* **Coûts d&#39;entraînement réduits** : Serverless SFT mutualise l&#39;infrastructure entre de nombreux utilisateurs, évite le processus de configuration pour chaque tâche et ramène vos coûts GPU à 0 lorsque vous n&#39;êtes pas activement en entraînement. Cela réduit les coûts d&#39;entraînement.
* **Entraînement plus rapide** : en provisionnant immédiatement l&#39;infrastructure d&#39;entraînement lorsque vous en avez besoin, Serverless SFT accélère vos tâches d&#39;entraînement et vous permet d&#39;itérer plus rapidement.
* **Déploiement automatique** : Serverless SFT déploie automatiquement chaque point de contrôle que vous entraînez. Vous n&#39;avez donc pas besoin de configurer manuellement une infrastructure d&#39;hébergement. Vous pouvez accéder immédiatement aux modèles entraînés et les tester dans des environnements locaux, de préproduction ou de production.

<div id="how-serverless-sft-uses-wb-services">
  ## Comment Serverless SFT utilise les services W&amp;B
</div>

Serverless SFT utilise une combinaison des composants W&amp;B suivants pour fonctionner :

* [Inference](/fr/inference) : Pour exécuter vos modèles.
* [Models](/fr/models) : Pour suivre les métriques de performances pendant l’entraînement de l’adaptateur LoRA.
* [Artifacts](/fr/models/artifacts) : Pour stocker et versionner les adaptateurs LoRA.
* [Weave](/fr/weave) (facultatif) : Pour bénéficier d’une meilleure observabilité de la façon dont le modèle répond à chaque étape de la boucle d’entraînement.

<Note>
  Serverless SFT est en préversion publique. Pendant cette période, W&amp;B vous facture uniquement l’utilisation d’Inference et le stockage des artefacts. W&amp;B ne facture pas l’entraînement des adaptateurs pendant la période de préversion.
</Note>
