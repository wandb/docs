title: Régions du Cloud dédié prises en charge
description: "Consultez toutes les régions AWS, Google Cloud et Azure prises en charge, disponibles pour les déploiements W&B Cloud dédié."
---

Cette page répertorie les régions AWS, Google Cloud et Azure que W&amp;B prend en charge pour les déploiements Cloud dédié. Consultez ces tableaux lorsque vous choisissez une région pour une nouvelle instance de Cloud dédié, afin de répondre à des exigences telles que la résidence des données et la conformité, la latence et l’optimisation des coûts.

<Note>
  Contactez l’assistance W&amp;B si la région AWS, Google Cloud ou Azure de votre choix ne figure pas dans la liste. W&amp;B peut vérifier si la région concernée dispose de tous les services requis pour le Cloud dédié. W&amp;B priorise sa prise en charge en fonction du résultat de cette évaluation.
</Note>

<div id="supported-aws-regions">
  ## Régions AWS prises en charge
</div>

Le tableau suivant répertorie les [régions AWS](https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/Concepts.RegionsAndAvailabilityZones.html) prises en charge par W&amp;B pour les instances Dedicated Cloud.

| Emplacement de la région       | Nom de la région |
| ------------------------------ | ---------------- |
| USA Est (Ohio)                 | `us-east-2`      |
| USA Est (Virginie du Nord)     | `us-east-1`      |
| USA Ouest (Californie du Nord) | `us-west-1`      |
| USA Ouest (Oregon)             | `us-west-2`      |
| Canada (Centre)                | `ca-central-1`   |
| Europe (Francfort)             | `eu-central-1`   |
| Europe (Irlande)               | `eu-west-1`      |
| Europe (Londres)               | `eu-west-2`      |
| Europe (Milan)                 | `eu-south-1`     |
| Europe (Stockholm)             | `eu-north-1`     |
| Asie-Pacifique (Mumbai)        | `ap-south-1`     |
| Asie-Pacifique (Singapour)     | `ap-southeast-1` |
| Asie-Pacifique (Sydney)        | `ap-southeast-2` |
| Asie-Pacifique (Tokyo)         | `ap-northeast-1` |
| Asie-Pacifique (Séoul)         | `ap-northeast-2` |

Pour plus d’informations sur les régions AWS, voir [Regions, Availability Zones, and Local Zones](https://docs.aws.amazon.com/AmazonRDS/latest/UserGuide/Concepts.RegionsAndAvailabilityZones.html) dans la documentation AWS.

Voir [What to Consider when Selecting a Region for your Workloads](https://aws.amazon.com/blogs/architecture/what-to-consider-when-selecting-a-region-for-your-workloads/) pour un aperçu des facteurs à prendre en compte lors du choix d’une région AWS.
