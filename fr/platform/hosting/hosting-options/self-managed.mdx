<Tip>
  W&amp;B recommande des options de déploiement entièrement gérées comme [Cloud mutualisé de W&amp;B](/fr/platform/hosting/hosting-options/multi_tenant_cloud) ou [Cloud dédié de W&amp;B](/fr/platform/hosting/hosting-options/dedicated-cloud). Les services entièrement gérés de W&amp;B sont simples et sécurisés à utiliser, avec peu, voire pas, de configuration requise.
</Tip>

Déployez le serveur W&amp;B sur votre [compte cloud AWS, Google Cloud ou Azure](#deploy-wb-server-within-Self-Managed-cloud-accounts) ou dans votre [infrastructure sur site](#deploy-wb-server-in-on-prem-infrastructure).

Votre équipe IT/DevOps/MLOps est responsable de :

* Provisionner votre déploiement.
* Sécuriser votre infrastructure conformément aux politiques de votre organisation et, le cas échéant, aux [Security Technical Implementation Guidelines (STIG)](https://en.wikipedia.org/wiki/Security_Technical_Implementation_Guide).
* Maintenir la conformité avec les exigences réglementaires de votre organisation, telles que [Service and Organization Controls (SOC) 2 Type 2](https://www.aicpa-cima.com/topic/audit-assurance/audit-and-assurance-greater-than-soc-2) et [Health Insurance Portability and Accountability Act of 1996 (HIPAA)](https://www.hhs.gov/hipaa/for-professionals/index.html).
* Gérer les mises à niveau et appliquer les correctifs.
* Assurer en continu la maintenance de votre déploiement W&amp;B Autogéré.

<Info>
  Si votre organisation est soumise à des exigences réglementaires, envisagez un déploiement sur [Cloud dédié de W&amp;B](/fr/platform/hosting/hosting-options/dedicated-cloud), qui est maintenu par W&amp;B.

  * La plateforme d&#39;hébergement de Cloud dédié de W&amp;B répond aux exigences de SOC 2 Type 2.
  * Lorsqu&#39;il est correctement configuré, un déploiement Cloud dédié de W&amp;B est conforme à HIPAA.

  Pour plus d&#39;informations, référez-vous au [W&amp;B Security Portal](https://security.wandb.ai/).
</Info>

<div id="about-the-wb-kubernetes-operator">
  ## À propos de l’opérateur Kubernetes W&amp;B
</div>

Utilisez l’**opérateur Kubernetes W&amp;B** pour déployer W&amp;B Autogéré. L’opérateur simplifie le déploiement, l’administration, le dépannage et la mise à l’échelle de W&amp;B.

L’opérateur se connecte à un serveur central [deploy.wandb.ai](https://deploy.wandb.ai) pour demander les dernières modifications de spécification pour un canal de version donné et les appliquer. Les mises à jour sont fournies tant que la licence est valide. [Helm](https://helm.sh/) est utilisé à la fois pour installer l’opérateur et par l’opérateur pour gérer la pile Kubernetes de W&amp;B. Le déploiement se compose de plusieurs pods, un par service ; chaque nom de pod est préfixé par `wandb-`.

La configuration suit une hiérarchie : **Valeurs du canal de version** (valeurs par défaut de W&amp;B), **Valeurs saisies par l’utilisateur** (surcharges via la console système) et **Valeurs de la ressource personnalisée** (votre spécification prévaut sur les deux). Pour plus de détails, voir [Déployer W&amp;B avec l’opérateur Kubernetes](/fr/platform/hosting/self-managed/operator).

* Pour déployer W&amp;B Autogéré dans une infrastructure de cloud public ou sur site, voir [Déployer W&amp;B avec l’opérateur Kubernetes](/fr/platform/hosting/self-managed/operator).
* Pour déployer W&amp;B Autogéré sur une plateforme cloud personnalisée autre qu’AWS, les exigences sont similaires à celles d’un déploiement dans une [infrastructure sur site](#deploy-wb-server-in-on-prem-infrastructure).
* Pour déployer W&amp;B Autogéré dans des environnements isolés du réseau, voir [Déployer sur Kubernetes isolé du réseau](/fr/platform/hosting/self-managed/on-premises-deployments/kubernetes-airgapped).

Vous pouvez éventuellement [configurer des limites de débit](/fr/platform/hosting/self-managed/rate-limits) sur votre instance afin d’en préserver la stabilité. Voir [Limites de débit](/fr/platform/hosting/self-managed/rate-limits) pour les valeurs par défaut et le comportement des notifications.

<div id="required-infrastructure">
  ## Infrastructure requise
</div>

Toutes les options de déploiement reposent sur l’infrastructure suivante :

* Kubernetes
* Base de données MySQL 8
* stockage d’objets compatible Amazon S3
* cache Redis

Voir [Requirements](/fr/platform/hosting/self-managed/requirements) pour plus de détails. W&amp;B peut fournir des recommandations sur les différents composants et vous guider tout au long du processus d’installation.

<div id="obtain-your-wb-server-license">
  ## Obtenir votre licence serveur W&amp;B
</div>

Vous avez besoin d’une licence serveur W&amp;B avant le déploiement. Pour obtenir des instructions étape par étape, voir [License](/fr/platform/hosting/self-managed/requirements#license) sur la page des exigences.
