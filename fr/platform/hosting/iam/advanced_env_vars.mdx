---
title: Configuration IAM avancée
description: "Configurer des options IAM avancées pour W&B à l’aide de variables d’environnement pour le SSO, la durée de session, l’OIDC et les paramètres LDAP."
---

En plus des [variables d’environnement](../env-vars) de base, vous pouvez utiliser des variables d’environnement pour configurer les options IAM de votre instance [Cloud dédié](/fr/platform/hosting/hosting-options/dedicated-cloud) ou [Autogéré](/fr/platform/hosting/hosting-options/self-managed).

Choisissez parmi les variables d’environnement suivantes pour votre instance, selon vos besoins IAM.

| Variable d’environnement            | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                             |
| ----------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `DISABLE_SSO_PROVISIONING`          | Définissez cette valeur sur `true` pour désactiver le provisionnement automatique des utilisateurs dans votre instance W&amp;B.                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| `SESSION_LENGTH`                    | Si vous souhaitez modifier la durée d’expiration par défaut de la session utilisateur, définissez cette variable sur le nombre d’heures souhaité. Par exemple, définissez `SESSION_LENGTH` sur `24` pour configurer une expiration de session après 24 heures. La valeur par défaut est de 720 heures.                                                                                                                                                                                                                                                                  |
| `GORILLA_ENABLE_SSO_GROUPS_CLAIMS`  | Si vous utilisez le SSO basé sur OIDC, définissez cette variable sur `true` pour automatiser l’appartenance aux équipes W&amp;B dans votre instance en fonction de vos groupes OIDC. Vous devez également ajouter un claim `groups` au jeton OIDC de l’utilisateur, sous la forme d’un tableau de chaînes contenant tous les noms d’équipe auxquels l’utilisateur appartient.                                                                                                                                                                                           |
| `GORILLA_LDAP_GROUP_SYNC`           | Si vous utilisez le SSO basé sur LDAP, définissez cette variable sur `true` pour automatiser l’appartenance aux équipes W&amp;B dans votre instance en fonction de vos groupes LDAP.                                                                                                                                                                                                                                                                                                                                                                                    |
| `GORILLA_OIDC_CUSTOM_SCOPES`        | Si vous utilisez le SSO basé sur OIDC, vous pouvez spécifier des [scopes](https://auth0.com/docs/get-started/apis/scopes/openid-connect-scopes) supplémentaires que l’instance W&amp;B doit demander à votre fournisseur d’identité. W&amp;B ne modifie en rien le fonctionnement du SSO en raison de ces scopes personnalisés.                                                                                                                                                                                                                                         |
| `GORILLA_OIDC_SECRET`               | Si vous utilisez le SSO basé sur OIDC et que votre fournisseur d’identité exige un secret client OIDC, définissez cette variable sur ce secret.                                                                                                                                                                                                                                                                                                                                                                                                                         |
| `GORILLA_USE_IDENTIFIER_CLAIMS`     | Si vous utilisez le SSO basé sur OIDC, définissez cette variable sur `true` pour appliquer le nom d’utilisateur et le nom complet de vos utilisateurs à partir de claims OIDC spécifiques fournis par votre fournisseur d’identité. Si cette variable est définie, veillez à configurer le nom d’utilisateur et le nom complet imposés respectivement dans les claims OIDC `preferred_username` et `name`. Les noms d’utilisateur ne peuvent contenir que des caractères alphanumériques, ainsi que des traits de soulignement et des tirets comme caractères spéciaux. |
| `GORILLA_DISABLE_PERSONAL_ENTITY`   | Lorsqu’elle est définie sur `true`, cette option désactive les [entités personnelles](/fr/support/models/articles/what-is-the-difference-between-team-and-). Elle empêche la création de nouveaux projets personnels dans ces entités et interdit l’écriture dans les projets personnels existants.                                                                                                                                                                                                                                                                              |
| `GORILLA_DISABLE_ADMIN_TEAM_ACCESS` | Définissez cette valeur sur `true` pour empêcher les administrateurs d’organisation ou d’instance de rejoindre eux-mêmes une équipe W&amp;B ou de s’y ajouter, afin de garantir que seuls les profils Data &amp; AI ont accès aux projets au sein des équipes.                                                                                                                                                                                                                                                                                                          |
| `WANDB_IDENTITY_TOKEN_FILE`         | Pour la [fédération d’identités](/fr/platform/hosting/iam/identity_federation/), le chemin absolu vers le répertoire local où les JSON Web Tokens (JWT) sont stockés.                                                                                                                                                                                                                                                                                                                                                                                                      |

<Warning>
  W&amp;B recommande de faire preuve de prudence et de bien comprendre toutes les implications avant d’activer certains de ces paramètres, comme `GORILLA_DISABLE_ADMIN_TEAM_ACCESS`. Contactez votre équipe W&amp;B pour toute question.
</Warning>
