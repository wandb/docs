Intégrez serveur W&amp;B à [Slack](https://slack.com/) afin que votre instance W&amp;B puisse envoyer des alertes et des notifications vers un espace de travail Slack que votre équipe utilise déjà. Cette page guide les administrateurs de serveur W&amp;B dans la création d’une application Slack, la configuration de ses scopes OAuth et de son URL de redirection, ainsi que l’enregistrement de l’application auprès de W&amp;B.

<Note>
  Regardez une [vidéo montrant comment configurer les alertes Slack sur un déploiement W&amp;B Dedicated Cloud](https://www.youtube.com/watch?v=JmvKb-7u-oU) (6 min).
</Note>

<div id="create-the-slack-application">
  ## Créer l&#39;application Slack
</div>

Le serveur W&amp;B utilise une application Slack personnalisée comme passerelle pour acheminer les alertes. Suivez cette procédure pour créer cette application dans l&#39;espace de travail Slack où vous souhaitez recevoir des notifications.

1. Accédez à [https://api.slack.com/apps](https://api.slack.com/apps) et sélectionnez **Créer une application**.

   <Frame>
     <img src="/images/hosting/create_an_app.png" alt="Bouton Créer une application" />
   </Frame>

2. Saisissez un nom pour votre application dans le champ **Nom de l&#39;application**.

3. Sélectionnez l&#39;espace de travail Slack dans lequel vous souhaitez développer votre application. Assurez-vous d&#39;utiliser le même espace de travail Slack pour les alertes.

   <Frame>
     <img src="/images/hosting/name_app_workspace.png" alt="Nom de l'application et sélection de l'espace de travail" />
   </Frame>

<div id="configure-the-slack-application">
  ## Configurer l’application Slack
</div>

Après avoir créé l’application Slack, accordez-lui les autorisations nécessaires pour publier des messages et accepter la redirection depuis W&amp;B pendant le processus OAuth.

1. Dans la barre latérale gauche, sélectionnez **OAuth &amp; Permissions**.

   <Frame>
     <img src="/images/hosting/add_an_oath.png" alt="Menu OAuth & Permissions" />
   </Frame>

2. Dans la section **Scopes**, accordez au bot le scope `incoming_webhook`. Les scopes donnent à votre application l’autorisation d’effectuer des actions dans votre espace de travail de développement.

   Pour plus d’informations sur les scopes OAuth pour les bots, voir [Understanding OAuth scopes for bots](https://api.slack.com/legacy/oauth-scopes) dans la documentation de l’API Slack.

   <Frame>
     <img src="/images/hosting/save_urls.png" alt="Scopes du jeton du bot" />
   </Frame>

3. Configurez l’**URL de redirection** pour qu’elle pointe vers votre installation W&amp;B. Utilisez la même URL que celle définie comme URL d’hôte dans les paramètres de votre système local. Vous pouvez spécifier plusieurs URL si votre instance dispose de plusieurs mappages DNS.

   <Frame>
     <img src="/images/hosting/redirect_urls.png" alt="Configuration des URL de redirection" />
   </Frame>

4. Sélectionnez **Save URLs**.

5. Facultatif : sous **Restrict API Token Usage**, spécifiez une adresse IP ou une plage d’adresses IP à ajouter à la liste d’autorisation pour vos instances W&amp;B. Le fait de limiter les adresses IP autorisées contribue à sécuriser votre application Slack.

<div id="register-your-slack-application-with-wb">
  ## Enregistrez votre application Slack auprès de W&amp;B
</div>

Enregistrez l’application Slack que vous avez configurée auprès de votre instance W&amp;B afin que W&amp;B puisse l’utiliser pour envoyer des alertes.

1. Accédez à la page **System Settings** ou **System Console** de votre instance W&amp;B, selon votre déploiement.

2. Selon la page système sur laquelle vous vous trouvez, suivez l’une des procédures suivantes :

   * Si vous êtes dans **System Console** : accédez à **Settings**, puis à **Notifications**.

     <Frame>
       <img src="/images/hosting/register_slack_app_console.png" alt="Notifications de la System Console" />
     </Frame>

   * Si vous êtes dans **System Settings** : activez l’option **Enable a custom Slack application to dispatch alerts** pour activer une application Slack personnalisée.

     <Frame>
       <img src="/images/hosting/register_slack_app.png" alt="Option d’activation de l’application Slack" />
     </Frame>

3. Saisissez votre **Slack client ID** et votre **Slack secret**, puis sélectionnez **Save**. Pour trouver l’ID client et le secret de votre application, accédez à **Basic Information** dans **Settings**.

4. Pour vérifier que tout fonctionne, configurez une intégration Slack dans la W&amp;B App.

Votre serveur W&amp;B est maintenant enregistré auprès de l’application Slack et peut envoyer des alertes à l’espace de travail Slack configuré.
