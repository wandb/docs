title: Configurer les paramètres de confidentialité
description: "Configurez et appliquez les paramètres de confidentialité pour la visibilité de l'équipe, l'accès au projet et les valeurs par défaut de l'enregistrement du code dans W&B."
---

Cette page décrit comment les Administrateurs de l’organisation et les Administrateurs d’équipe peuvent configurer les paramètres de confidentialité dans W&amp;B afin de contrôler la visibilité de l&#39;équipe, l&#39;accès au projet, les invitations, le partage de rapports et les paramètres par défaut de l’enregistrement du code. Utilisez ces paramètres pour harmoniser la confidentialité à l’échelle d’une organisation ou pour ajuster les contrôles pour une seule équipe.

Les Administrateurs de l’organisation peuvent configurer les paramètres de confidentialité au niveau de l’organisation, et les Administrateurs d’équipe peuvent les configurer au niveau de l’équipe. Lorsqu’ils sont configurés au niveau de l’organisation, ces paramètres sont appliqués à toutes les équipes de cette organisation.

<Note>
  Pour éviter des modifications inattendues dans les flux de travail des équipes, W&amp;B recommande aux Administrateurs de l’organisation d’appliquer un paramètre de confidentialité uniquement après en avoir informé à l’avance tous les Administrateurs d’équipe et utilisateurs de l’organisation.
</Note>

<div id="enforce-privacy-settings-for-all-teams">
  ## Appliquer les paramètres de confidentialité à tous les équipes
</div>

Les Administrateurs de l’organisation peuvent appliquer des paramètres de confidentialité à tous les équipes de leur organisation depuis la section **Confidentialité** de l’onglet **Settings** du tableau de bord du compte ou de l’organisation. Si les Administrateurs de l’organisation appliquent un paramètre, les Administrateurs d’équipe ne peuvent pas le configurer dans leurs équipes respectifs.

* **Appliquer les restrictions de visibilité de l&#39;équipe** : Masque tous les équipes pour les non-membres dans l’ensemble de l’organisation.
* **Appliquer la confidentialité aux futurs projets** : Exige que tous les nouveaux projets de tous les équipes soient privés ou [restreints](./iam/access-management/restricted-projects).
* **Appliquer le contrôle des invitations** : Empêche les non-administrateurs d’inviter des membres dans une équipe.
* **Appliquer le contrôle du partage des Reports** : Désactive le partage public des Reports dans les projets privés et désactive les liens magiques existants.
* **Appliquer les restrictions d’adhésion autonome aux équipes** : Empêche les utilisateurs dont le domaine de messagerie correspond à celui de l’organisation de rejoindre automatiquement une équipe.
  <Note>Ce paramètre est disponible uniquement dans les déploiements [Cloud mutualisé](./hosting-options/multi_tenant_cloud).</Note>
* **Appliquer les restrictions par défaut pour l’enregistrement du code** : Désactive l’enregistrement du code par défaut pour tous les équipes.

Pour ouvrir les paramètres de confidentialité de l’organisation :

1. Connectez-vous en tant qu’Administrateur de l’organisation.
2. Sur Cloud mutualisé, accédez à `https://wandb.ai/account-settings/[ORGANIZATION]/settings`. Remplacez `[ORGANIZATION]` par le nom de votre organisation. Sur les déploiements Cloud dédié ou Autogéré, ouvrez le tableau de bord de l’organisation de votre instance. Pour les URL propres à chaque environnement, voir [Add and manage users](/fr/platform/hosting/iam/access-management/manage-organization#add-and-manage-users).
3. Ouvrez la section **Confidentialité** et configurez **Appliquer les restrictions par défaut pour l’enregistrement du code** ainsi que tout autre contrôle à l’échelle de l’organisation dont vous avez besoin.

Après avoir enregistré les modifications, les paramètres appliqués s’appliquent à toutes les équipes de l’organisation, et les Administrateurs d’équipe ne peuvent plus les redéfinir au niveau de l’équipe.

<div id="configure-privacy-settings-for-a-team">
  ## Configurer les paramètres de confidentialité d’une équipe
</div>

Si un paramètre n’est pas appliqué à l’échelle de l’organisation, les administrateurs d’équipe peuvent le gérer pour leur propre équipe depuis la section **Confidentialité** de l’onglet **Paramètres** de l’équipe. Chaque paramètre peut être configuré tant qu’il n’est pas appliqué à l’échelle de l’organisation :

* **Masquer cette équipe à tous les non-membres** : masque l’équipe pour les utilisateurs qui n’en sont pas membres.
* **Rendre privés tous les futurs projets de l’équipe** : garantit que tous les nouveaux Projects créés dans l’équipe sont privés (le partage public n’est pas autorisé).
* **Autoriser tout membre de l’équipe à inviter d’autres membres** : permet à tous les membres de l’équipe d’inviter de nouveaux membres, et pas uniquement aux administrateurs.
* **Désactiver le partage public des Reports dans les projets privés** : désactive le partage public des Reports et les liens magiques existants.
* **Autoriser les utilisateurs dont le domaine d’adresse e-mail de l’organisation correspond à rejoindre cette équipe** : permet aux utilisateurs ayant le même domaine d’adresse e-mail de rejoindre l’équipe automatiquement.
  <Note>Ce paramètre est disponible uniquement dans les déploiements [Cloud mutualisé](./hosting-options/multi_tenant_cloud).</Note>
* **Activer l’enregistrement du code par défaut** : enregistre automatiquement le code pour tous les runs de l’équipe.

Pour ouvrir les paramètres de confidentialité de l’équipe :

1. Accédez à `https://wandb.ai/[TEAM]`. Remplacez `[TEAM]` par le nom de votre équipe.
2. Sélectionnez **Paramètres de l’équipe** dans la navigation de gauche.
3. Ouvrez la section **Confidentialité** et configurez **Activer l’enregistrement du code par défaut**.

Après avoir enregistré les modifications, les paramètres sélectionnés s’appliquent à votre équipe.
