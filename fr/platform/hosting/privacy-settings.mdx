---
title: Configurer les paramètres de confidentialité
description: "Configurer et appliquer les paramètres de confidentialité pour la visibilité de l'équipe, l'accès au projet et les valeurs par défaut de l'enregistrement du code dans W&B."
---

Les administrateurs de l’organisation et les administrateurs d’équipe peuvent configurer un ensemble de paramètres de confidentialité, respectivement au niveau de l’organisation et de l’équipe. Lorsqu’ils sont configurés au niveau de l’organisation, les administrateurs de l’organisation appliquent ces paramètres à toutes les équipes de cette organisation.

<Note>
  W&amp;B recommande aux administrateurs de l’organisation d’appliquer un paramètre de confidentialité uniquement après en avoir informé à l’avance tous les administrateurs d’équipe et utilisateurs de leur organisation. Cela permet d’éviter des changements inattendus dans leur flux de travail.
</Note>

<div id="configure-privacy-settings-for-a-team">
  ## Configurer les paramètres de confidentialité pour une équipe
</div>

Les Administrateurs d’équipe peuvent configurer les paramètres de confidentialité de leur équipe dans la section `Privacy` de l’onglet **Paramètres** de l’équipe. Chaque paramètre peut être configuré tant qu’il n’est pas appliqué au niveau de l’organisation :

* **Hide this team from all non-members** - Empêche les utilisateurs qui ne sont pas membres de voir l’équipe
* **Make all future team projects private** - Garantit que tous les nouveaux projets créés dans l’équipe sont privés (partage public non autorisé)
* **Allow any team member to invite other members** - Permet à tous les membres de l’équipe d’inviter de nouveaux membres, et pas seulement aux administrateurs
* **Turn off public sharing for reports in private projects** - Désactive le partage public des rapports et désactive les liens magiques existants
* **Allow users with matching organization email domain to join this team** - Permet aux utilisateurs ayant le domaine d’e-mail de l’organisation correspondant de rejoindre automatiquement cette équipe
  <Note>Ce paramètre est uniquement disponible dans les déploiements [Cloud mutualisé](./hosting-options/multi_tenant_cloud).</Note>
* **Enable code saving by default** - Active automatiquement l’enregistrement du code pour tous les runs de l’équipe

<div id="enforce-privacy-settings-for-all-teams">
  ## Appliquer les paramètres de confidentialité à toutes les équipes
</div>

Les Administrateurs de l’organisation peuvent appliquer les paramètres de confidentialité à toutes les équipes de leur organisation dans la section `Privacy` de l’onglet **Paramètres** du tableau de bord du compte ou de l’organisation. Si les Administrateurs de l’organisation appliquent un paramètre, les Administrateurs d’équipe ne peuvent pas le configurer dans leurs équipes respectives.

* **Appliquer les restrictions de visibilité des équipes** - Masque toutes les équipes aux non-membres dans toute l’organisation
* **Appliquer la confidentialité aux futurs projets** - Exige que tous les nouveaux projets de toutes les équipes soient privés ou [restreints](./iam/access-management/restricted-projects)
* **Appliquer le contrôle des invitations** - Empêche les non-administrateurs d’inviter des membres dans une équipe
* **Appliquer le contrôle du partage des rapports** - Désactive le partage public des rapports dans les projets privés et désactive les liens magiques existants
* **Appliquer les restrictions d’adhésion automatique aux équipes** - Empêche les utilisateurs dont le domaine d’e-mail correspond à celui de l’organisation de rejoindre automatiquement une équipe
  <Note>Ce paramètre est uniquement disponible dans les déploiements [Cloud mutualisé](./hosting-options/multi_tenant_cloud).</Note>
* **Appliquer les restrictions par défaut pour l’enregistrement du code** - Désactive l’enregistrement du code par défaut pour toutes les équipes
