---
title: Configurer les paramètres de confidentialité
description: "Configurez et appliquez dans W&B les paramètres de confidentialité relatifs à la visibilité de l'équipe, à l'accès au projet et aux valeurs par défaut d'enregistrement du code."
---

Les administrateurs de l’organisation et les administrateurs d’équipe peuvent configurer un ensemble de paramètres de confidentialité au niveau de l’organisation et de l’équipe, respectivement. Lorsqu’ils sont configurés au niveau de l’organisation, les administrateurs de l’organisation appliquent ces paramètres à toutes les équipes de cette organisation.

<Note>
  Pour éviter des changements inattendus dans les flux de travail des équipes, W&amp;B recommande aux administrateurs de l’organisation d’appliquer un paramètre de confidentialité uniquement après en avoir informé à l’avance tous les administrateurs d’équipe et utilisateurs de l’organisation.
</Note>

<div id="enforce-privacy-settings-for-all-teams">
  ## Appliquer les paramètres de confidentialité à tous les équipes
</div>

Les Administrateurs de l’organisation peuvent appliquer des paramètres de confidentialité à tous les équipes de leur organisation depuis la section **Privacy** de l’onglet **Settings** du tableau de bord du compte ou de l’organisation. Si les Administrateurs de l’organisation appliquent un paramètre, les Administrateurs d’équipe ne peuvent pas le configurer dans leurs équipes respectifs.

* **Appliquer les restrictions de visibilité de l&#39;équipe** - Masque tous les équipes pour les non-membres dans l’ensemble de l’organisation
* **Appliquer la confidentialité aux futurs projets** - Exige que tous les nouveaux projets de tous les équipes soient privés ou [restreints](./iam/access-management/restricted-projects)
* **Appliquer le contrôle des invitations** - Empêche les non-administrateurs d’inviter des membres dans une équipe
* **Appliquer le contrôle du partage des Reports** - Désactive le partage public des Reports dans les projets privés et désactive les liens magiques existants
* **Appliquer les restrictions d’adhésion autonome aux équipes** - Empêche les utilisateurs dont le domaine de messagerie correspond à celui de l’organisation de rejoindre automatiquement une équipe
  <Note>Ce paramètre est disponible uniquement dans les déploiements [Cloud mutualisé](./hosting-options/multi_tenant_cloud).</Note>
* **Appliquer les restrictions par défaut pour l’enregistrement du code** - Désactive l’enregistrement du code par défaut pour tous les équipes

Pour ouvrir les paramètres de confidentialité de l’organisation :

1. Connectez-vous en tant qu’Administrateur de l’organisation.
2. Sur Cloud mutualisé, accédez à `https://wandb.ai/account-settings/<organization>/settings`. Remplacez `<organization>` par le nom de votre organisation. Sur les déploiements Cloud dédié ou Autogéré, ouvrez le tableau de bord de l’organisation de votre instance. Pour les URL propres à chaque environnement, voir [Add and manage users](/fr/platform/hosting/iam/access-management/manage-organization#add-and-manage-users).
3. Ouvrez la section **Privacy** et configurez **Appliquer les restrictions par défaut pour l’enregistrement du code** ainsi que tout autre contrôle à l’échelle de l’organisation dont vous avez besoin.

<div id="configure-privacy-settings-for-a-team">
  ## Configurer les paramètres de confidentialité d’une équipe
</div>

Les administrateurs d’équipe peuvent configurer les paramètres de confidentialité de leurs équipes respectives dans la section **Privacy** de l’onglet **Settings** de l’équipe. Chaque paramètre peut être configuré tant qu’il n’est pas appliqué au niveau de l’organisation :

* **Masquer cette équipe pour tous les non-membres** - Empêche l’équipe d’être visible par les utilisateurs qui n’en sont pas membres
* **Rendre privés tous les futurs projets de l’équipe** - Garantit que tous les nouveaux projets créés dans l’équipe sont privés (partage public non autorisé)
* **Autoriser tout membre de l’équipe à inviter d’autres membres** - Permet à tous les membres de l’équipe d’inviter de nouveaux membres, et pas uniquement aux administrateurs
* **Désactiver le partage public des Reports dans les projets privés** - Désactive le partage public des Reports et désactive les liens magiques existants
* **Autoriser les utilisateurs dont le domaine de messagerie de l’organisation correspond à celui de cette équipe à la rejoindre** - Permet aux utilisateurs ayant le même domaine de messagerie de rejoindre automatiquement l’équipe
  <Note>Ce paramètre est disponible uniquement dans les déploiements [Cloud mutualisé](./hosting-options/multi_tenant_cloud).</Note>
* **Activer l’enregistrement du code par défaut** - Enregistre automatiquement le code pour tous les runs de l’équipe

Pour ouvrir les paramètres de confidentialité de l’équipe :

1. Accédez à `https://wandb.ai/<team>`. Remplacez `<team>` par le nom de votre équipe.
2. Sélectionnez **Team settings** dans la navigation de gauche.
3. Ouvrez la section **Privacy** et configurez **Activer l’enregistrement du code par défaut**.
