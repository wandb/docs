Cette page montre aux administrateurs des déploiements W&amp;B **Autogérés** comment désactiver les mises à niveau automatiques de serveur W&amp;B et fixer sa version sur une version spécifique. Le fait de fixer une version vous permet de contrôler le moment où les mises à niveau ont lieu. Ce contrôle est utile lorsque vous devez coordonner les mises à niveau avec des processus internes de gestion des modifications ou valider une version dans un environnement de préproduction avant de la déployer. Ces instructions s&#39;appliquent uniquement aux déploiements gérés par l&#39;[opérateur Kubernetes W&amp;B](/fr/platform/hosting/self-managed/operator).

<Note>
  W&amp;B prend en charge une version majeure de serveur W&amp;B pendant 12 mois à compter de sa date de sortie initiale. Les clients disposant d&#39;instances **Autogérées** sont responsables d&#39;effectuer la mise à niveau à temps pour continuer à bénéficier de l&#39;assistance. Évitez de rester sur une version non prise en charge. W&amp;B recommande aux clients disposant d&#39;instances **Autogérées** de mettre à jour leurs déploiements vers la dernière version au moins une fois par trimestre afin de continuer à bénéficier de l&#39;assistance et de recevoir les dernières fonctionnalités, améliorations des performances et correctifs.
</Note>

<div id="requirements">
  ## Prérequis
</div>

* opérateur Kubernetes W&amp;B `v1.13.0` ou version ultérieure
* System Console `v2.12.2` ou version ultérieure

Pour vérifier que vous respectez ces prérequis, référez-vous à la ressource personnalisée W&amp;B ou au chart Helm de votre instance. Vérifiez les valeurs `version` des composants `operator-wandb` et `system-console`.

<div id="disable-automatic-updates">
  ## Désactiver les mises à jour automatiques
</div>

Pour désactiver les mises à jour automatiques, utilisez la System Console pour épingler le serveur W&amp;B à une version spécifique, puis vérifiez dans les journaux de réconciliation de l’opérateur que l’épinglage de version est actif.

1. Connectez-vous à la W&amp;B App avec un utilisateur disposant du rôle `admin`.
2. Cliquez sur l’icône utilisateur en haut, puis sur **System Console**.
3. Accédez à **Settings** &gt; **Advanced**, puis sélectionnez l’onglet **Other**.
4. Dans la section **Disable Auto Upgrades**, activez **Pin specific version**.
5. Cliquez sur la liste déroulante **Select a version**, puis sélectionnez une version de serveur W&amp;B.
6. Cliquez sur **Save**.

   <Frame>
     <img src="/images/hosting/disable_automatic_updates_saved_and_enabled.png" alt="System Console affichant Pin specific version activé avec une version de serveur W&B sélectionnée" />
   </Frame>

   Les mises à jour automatiques sont désactivées et serveur W&amp;B est épinglé sur la version que vous avez sélectionnée.
7. Vérifiez que les mises à jour automatiques sont bien désactivées. Accédez à l’onglet **Operator** et recherchez dans les journaux de réconciliation la chaîne `Version pinning is enabled`.

```text

│info 2025-04-17T17:24:16Z wandb default Active spec found
│info 2025-04-17T17:24:16Z wandb default Desired spec
│info 2025-04-17T17:24:16Z wandb default License
│info 2025-04-17T17:24:16Z wandb default Version Pinning is enabled
│info 2025-04-17T17:24:16Z wandb default Found Weights & Biases instance, processing the spec...
│info 2025-04-17T17:24:16Z wandb default === Reconciling Weights & Biases instance...
```
