Une licence d’entreprise débloque des fonctionnalités W&amp;B conçues pour les organisations qui ont besoin de capacités avancées en matière de sécurité, de conformité ou d’administration. Voir la liste des [fonctionnalités Enterprise](#enterprise-features).

* **Sécurité** : authentification renforcée, chiffrement et contrôles d’accès.
* **Conformité** : options de journalisation d’audit, options de conformité HIPAA et contrôles de gouvernance des données.
* **Administration** : fonctionnalités de gestion des utilisateurs, comme le provisionnement SCIM, les rôles personnalisés et les capacités d’automatisation.

Cette page fournit un aperçu complet des licences d’entreprise, y compris les fonctionnalités qu’elles permettent d’utiliser, comment les obtenir et comment les configurer pour votre déploiement.

<div id="deployment-options">
  ## Options de déploiement
</div>

Les licences d’entreprise sont disponibles pour les déploiements Cloud dédié de W&amp;B et W&amp;B Autogéré.

* [Cloud dédié de W&amp;B](/fr/platform/hosting/hosting-options/dedicated-cloud) fournit une infrastructure à locataire unique gérée par W&amp;B, déployée sur Google Cloud. Les déploiements Cloud dédié incluent automatiquement une licence Enterprise. Aucune configuration supplémentaire n’est requise, et toutes les fonctionnalités Enterprise sont disponibles dès le provisionnement.
* [W&amp;B Autogéré](/fr/platform/hosting/hosting-options/self-managed) est déployé sur votre propre infrastructure, ce qui vous donne un contrôle complet sur votre déploiement et vos données. Les déploiements isolés du réseau sont pris en charge. Sans licence Enterprise, les [fonctionnalités Enterprise](#enterprise-features) ne sont pas disponibles.

<div id="obtain-an-enterprise-license">
  ## Obtenir une licence d’entreprise
</div>

Cette section explique comment obtenir une licence d’entreprise. Selon votre cas d’utilisation, sélectionnez un onglet pour continuer.

<Tabs>
  <Tab title="Cloud dédié">
    Le Cloud dédié de W&amp;B inclut une licence d’entreprise et aucune action n’est requise. [Démarrez un essai Enterprise](https://wandb.ai/site/for-enterprise/dedicated-saas-trial/) ou contactez [Sales](https://wandb.ai/site/contact-sales) pour discuter de vos exigences.
  </Tab>

  <Tab title="Autogéré">
    Demandez une licence d’entreprise :

    * **Évaluation** : Soumettez une [demande de licence d’essai](https://wandb.ai/site/for-enterprise/self-hosted-trial) ou contactez [Sales](mailto:sales@wandb.ai) pour discuter de vos exigences.
    * **Production** : Contactez [Sales](https://wandb.ai/site/contact-sales) pour discuter de vos exigences de déploiement et de dimensionnement, et recevoir les détails de votre licence.

    <Warning>Stockez les clés de licence en lieu sûr et traitez-les comme des identifiants sensibles.</Warning>
  </Tab>
</Tabs>

<div id="configure-an-enterprise-license">
  ## Configurer une licence d’entreprise
</div>

<Info>Cette section s’applique uniquement à W&amp;B Autogéré.</Info>

Après avoir [obtenu une licence d’entreprise](#obtain-an-enterprise-license), vous pouvez la configurer par programmation, ou dans la W&amp;B App après le déploiement. Sélectionnez un onglet pour afficher les instructions.

<Tabs>
  <Tab title="Opérateur Kubernetes">
    1. Mettez à jour votre fichier `operator.yaml` (la ressource personnalisée `WeightsAndBiases`) pour définir la licence :
       ```yaml
       spec:
         values:
           global:
             license: "YOUR_LICENSE_KEY_HERE"
       ```

    2. Appliquez la configuration mise à jour :
       ```bash
       kubectl apply -f operator.yaml
       ```
  </Tab>

  <Tab title="Terraform">
    1. Mettez à jour vos variables Terraform :
       ```hcl
       variable "license" {
         default = "YOUR_LICENSE_KEY_HERE"
       }
       ```

    2. Appliquez la configuration Terraform :
       ```bash
       terraform apply -var="license=$LICENSE_KEY"
       ```
  </Tab>

  <Tab title="W&B App">
    Pour les déploiements existants sans configuration par variable d’environnement :

    1. Connectez-vous en tant qu’administrateur
    2. Accédez à **Paramètres système**
    3. Repérez la section **License**
    4. Saisissez votre nouvelle clé de licence
    5. Enregistrez les modifications
  </Tab>

  <Tab title="Docker">
    <Warning>Le déploiement de W&amp;B sur Docker est obsolète. W&amp;B encourage vivement la migration vers un déploiement avec l’[opérateur Kubernetes]() géré par Helm ou Terraform. Référez-vous à l’[architecture de référence](/fr/platform/hosting/self-managed/ref-arch.mdx) ou contactez l’[assistance](mailto:support@wandb.ai) pour obtenir des conseils.</Warning>
    Définissez la clé de licence comme variable d’environnement :

    ```bash
    docker run -d \
      -e LICENSE="YOUR_LICENSE_KEY_HERE" \
      -e OTHER_ENV_VARS \
      wandb/local
    ```
  </Tab>
</Tabs>

Ensuite, [vérifiez votre licence d’entreprise](#verify-or-update-an-enterprise-license).

<div id="verify-and-manage-an-enterprise-license">
  ## Vérifier et gérer une licence d’entreprise
</div>

En tant qu’administrateur d’instance ou d’organisation, suivez ces étapes pour vérifier et gérer votre licence d’entreprise :

1. Dans la W&amp;B App, cliquez sur l’icône de votre profil en haut à droite, puis sur **System Console**.
2. Accédez à l’onglet **License**. Un tableau de bord récapitule les droits associés à la licence, son expiration et son utilisation dans le temps. Si une erreur s’affiche, voir [Problèmes courants](#common-issues).
3. Pour obtenir des informations détaillées sur la licence et les droits associés, cliquez sur **View license online**.

   Depuis cette page, vous pouvez également demander une nouvelle licence d’essai ou de production. En haut de la page, cliquez sur **New license order** et soumettez le formulaire.
4. Pour renouveler une licence expirée ou mettre à jour la clé de licence, cliquez sur **Update License**.

   <Tip>
     Pour garantir la continuité du service, W&amp;B vous contacte :

     * 30 jours avant l’expiration de la licence
     * Lorsque la licence a expiré
   </Tip>

<div id="common-issues">
  ## Problèmes courants
</div>

<div id="license-not-recognized">
  ### License non reconnue
</div>

* Vérifier que la clé de licence est correctement formatée (sans espaces superflus)
* S&#39;assurer que la licence n&#39;a pas expiré
* Vérifier que la licence est définie au bon emplacement de la configuration

<div id="features-not-available-after-setting-license">
  ### Fonctionnalités non disponibles après la définition de la licence
</div>

* Redémarrez vos services W&amp;B après avoir défini la licence
* Vérifiez que la licence inclut les fonctionnalités spécifiques auxquelles vous essayez d&#39;accéder
* Consultez les journaux système pour détecter toute erreur de validation de la licence

<div id="license-expiration-warnings">
  ### Avertissements avant expiration de la License
</div>

* Surveillez les notifications d’expiration sur la page Paramètre système
* Configurez des alertes d’expiration de License dans votre système de surveillance
* Gardez à jour les coordonnées de votre équipe de compte
