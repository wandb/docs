title: Mettre à jour la licence et la version W&B
---

Mettez à jour la version et la licence de votre serveur W&amp;B en utilisant la même méthode que celle employée lors de l&#39;installation du serveur W&amp;B. Le tableau suivant répertorie les procédures de mise à jour de votre licence et de votre version selon les différentes méthodes de déploiement :

| Type de version                     | Description                                                                                                                                                                                                                                                                                                            |
| ----------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [Terraform](#update-with-terraform) | W&amp;B prend en charge trois modules Terraform publics pour le déploiement cloud : [AWS](https://registry.terraform.io/modules/wandb/wandb/aws/latest), [Google Cloud](https://registry.terraform.io/modules/wandb/wandb/google/latest) et [Azure](https://registry.terraform.io/modules/wandb/wandb/azurerm/latest). |
| [Helm](#update-with-helm)           | Vous pouvez utiliser le [chart Helm](https://github.com/wandb/helm-charts) pour installer W&amp;B dans un cluster Kubernetes existant.                                                                                                                                                                                 |

<div id="update-with-terraform">
  ## Mettre à jour avec Terraform
</div>

Mettez à jour votre licence et votre version avec Terraform. Le tableau suivant répertorie les modules Terraform gérés par W&amp;B basés sur la plateforme cloud.

| Fournisseur de cloud | Module Terraform                                                                                 |
| -------------------- | ------------------------------------------------------------------------------------------------ |
| AWS                  | [Module Terraform AWS](https://registry.terraform.io/modules/wandb/wandb/aws/latest)             |
| Google Cloud         | [Module Terraform Google Cloud](https://registry.terraform.io/modules/wandb/wandb/google/latest) |
| Azure                | [Module Terraform Azure](https://registry.terraform.io/modules/wandb/wandb/azurerm/latest)       |

1. Accédez d&#39;abord au module Terraform maintenu par W&amp;B correspondant à votre fournisseur de cloud. Voir le tableau précédent pour trouver le module Terraform approprié basé sur votre fournisseur de cloud.

2. Dans votre configuration Terraform, mettez à jour `wandb_version` et `license` dans la configuration de votre module Terraform `wandb_app` :

   ```hcl
   module "wandb_app" {
       source  = "wandb/wandb/<cloud-specific-module>"
       version = "new_version"
       license       = "new_license_key" # Votre nouvelle clé de licence
       wandb_version = "new_wandb_version" # Version W&B souhaitée
       ...
   }
   ```

3. Appliquez la configuration Terraform avec `terraform plan` et `terraform apply`.
   ```bash
   terraform init
   terraform apply
   ```

4. (Facultatif) Si vous utilisez un fichier `terraform.tfvars` ou un autre fichier `.tfvars`.

   Mettez à jour ou créez un fichier `terraform.tfvars` avec la nouvelle version W&amp;B et la clé de licence.

   ```bash
   terraform plan -var-file="terraform.tfvars"
   ```

   Appliquez la configuration. Dans le répertoire de votre workspace Terraform, exécutez :

   ```bash
   terraform apply -var-file="terraform.tfvars"
   ```

<div id="update-with-helm">
  ## Mettre à jour avec Helm
</div>

<Warning>
  Le chart Helm `wandb` est obsolète et n&#39;est plus pris en charge. Il déployait un seul pod et a été remplacé par l&#39;[opérateur Kubernetes W&amp;B](/fr/platform/hosting/self-managed/operator). Si vous utilisez encore ce chart, suivez le [guide de migration](/fr/platform/hosting/self-managed/operator#migrate-self-managed-instances-to-wb-operator) pour passer à l&#39;opérateur.
</Warning>

<div id="update-wb-with-spec">
  ### Mettre à jour W&amp;B avec une spécification
</div>

1. Spécifiez une nouvelle version en modifiant les valeurs `image.tag` et/ou `license` dans le fichier de configuration `*.yaml` de votre chart Helm :

   ```yaml
   license: 'new_license'
   image:
     repository: wandb/local
     tag: 'new_version'
   ```

2. Exécutez la mise à niveau Helm avec la commande suivante :

   ```bash
   helm repo update
   helm upgrade --namespace=wandb --create-namespace \
     --install wandb wandb/wandb --version ${chart_version} \
     -f ${wandb_install_spec.yaml}
   ```

<div id="update-license-and-version-directly">
  ### Mettre à jour la licence et la version directement
</div>

1. Définissez la nouvelle clé de licence et le tag d&#39;image en tant que variables d&#39;environnement :

   ```bash
   export LICENSE='new_license'
   export TAG='new_version'
   ```

2. Mettez à niveau votre version Helm avec la commande ci-dessous, en fusionnant les nouvelles valeurs avec la configuration existante :

   ```bash
   helm repo update
   helm upgrade --namespace=wandb --create-namespace \
     --install wandb wandb/wandb --version ${chart_version} \
     --reuse-values --set license=$LICENSE --set image.tag=$TAG
   ```

Pour plus de détails, voir le [guide de mise à niveau](https://github.com/wandb/helm-charts/blob/main/upgrade) dans le dépôt public.

<div id="update-with-admin-ui">
  ## Mettre à jour avec l&#39;interface utilisateur d&#39;administration
</div>

Cette méthode fonctionne uniquement pour mettre à jour les licences qui ne sont pas définies via une variable d&#39;environnement dans le conteneur du serveur W&amp;B, généralement dans les installations Docker autogérées.

1. Obtenez une nouvelle licence depuis la [page de déploiement W&amp;B](https://deploy.wandb.ai/), en vous assurant qu&#39;elle correspond à la bonne organisation et à l&#39;ID de déploiement pour le déploiement que vous souhaitez mettre à niveau.
2. Accédez à la page **License** dans la W&amp;B App. Cliquez sur **Settings** &gt; **License** ou accédez à `<host-url>/console/settings/license`.
3. Naviguez jusqu&#39;à la section de gestion des licences.
4. Saisissez la nouvelle clé de licence et enregistrez vos modifications.
