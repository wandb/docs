---
title: Configurer la connectivité privée pour Cloud dédié
description: "Connectez-vous à une instance W&B Dedicated Cloud via un réseau privé sécurisé à l'aide d'AWS PrivateLink, de GCP ou d'Azure Private Link."
---

Vous pouvez vous connecter à votre instance [Dedicated Cloud](/fr/platform/hosting/hosting-options/dedicated-cloud) via le réseau privé sécurisé de votre fournisseur cloud. Cela s&#39;applique à l&#39;accès entre vos charges de travail d&#39;IA et les API W&amp;B, et éventuellement aussi entre les navigateurs de vos utilisateurs et l&#39;interface de l&#39;application W&amp;B. Lorsque vous utilisez une connectivité privée, les requêtes et réponses concernées ne transitent ni par le réseau public ni par Internet.

<Note>
  La connectivité privée sécurisée sera bientôt disponible comme option de sécurité avancée avec Cloud dédié.
</Note>

La connectivité privée sécurisée est disponible sur les instances Dedicated Cloud sur AWS, Google Cloud et Azure :

* En utilisant [AWS Privatelink](https://aws.amazon.com/privatelink/) sur AWS
* En utilisant [Google Cloud Private Service Connect](https://cloud.google.com/vpc/docs/private-service-connect) sur Google Cloud
* En utilisant [Azure Private Link](https://azure.microsoft.com/products/private-link) sur Azure

Une fois cette fonctionnalité activée, W&amp;B crée un service d&#39;endpoint privé pour votre instance et vous fournit l&#39;URI DNS correspondante à utiliser pour vous y connecter. Vous pouvez ensuite créer des endpoints privés dans vos comptes cloud afin d&#39;acheminer le trafic concerné vers ce service d&#39;endpoint privé. Les endpoints privés sont plus simples à configurer pour vos charges de travail d&#39;entraînement de l&#39;IA exécutées dans votre VPC ou VNet cloud. Pour utiliser ce même mécanisme pour le trafic entre les navigateurs de vos utilisateurs et l&#39;interface de l&#39;application W&amp;B, vous devez configurer un routage basé sur DNS approprié depuis votre réseau d&#39;entreprise vers les endpoints privés dans vos comptes cloud.

<Note>
  Si vous souhaitez utiliser cette fonctionnalité, contactez votre équipe W&amp;B.
</Note>

Vous pouvez utiliser la connectivité privée sécurisée avec la [liste d&#39;autorisation d&#39;IP](./ip-allowlisting). Si vous utilisez la connectivité privée sécurisée avec la liste d&#39;autorisation d&#39;IP, W&amp;B recommande de sécuriser la connectivité privée pour tout le trafic provenant de vos charges de travail d&#39;IA et, si possible, pour la majorité du trafic provenant des navigateurs de vos utilisateurs, tout en réservant la liste d&#39;autorisation d&#39;IP à l&#39;administration des instances depuis des emplacements privilégiés.
