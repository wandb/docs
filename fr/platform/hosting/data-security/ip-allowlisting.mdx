---
title: Configurer la liste d’autorisation d’adresses IP pour Cloud dédié
description: "Restreignez l’accès à une instance W&B Dedicated Cloud aux seules adresses IP autorisées à l’aide d’une liste d’autorisation d’adresses IP."
---

Vous pouvez restreindre l’accès à votre instance [Cloud dédié](/fr/platform/hosting/hosting-options/dedicated-cloud) aux seules adresses IP figurant sur une liste autorisée. Cela s’applique aussi bien à l’accès de vos charges de travail d’IA aux API W&amp;B qu’à l’accès des navigateurs de vos utilisateurs à l’interface de l’application W&amp;B. Une fois la liste d’autorisation d’adresses IP configurée pour votre instance Dedicated Cloud, W&amp;B refuse toute requête provenant d’emplacements non autorisés. Contactez votre équipe W&amp;B pour configurer la liste d’autorisation d’adresses IP pour votre instance Dedicated Cloud.

La liste d’autorisation d’adresses IP est disponible pour les instances Dedicated Cloud sur AWS, Google Cloud et Azure.

Vous pouvez utiliser la liste d’autorisation d’adresses IP avec la [connectivité privée sécurisée](./private-connectivity). Si vous utilisez la liste d’autorisation d’adresses IP avec la connectivité privée sécurisée, W&amp;B recommande d’utiliser la connectivité privée sécurisée pour tout le trafic de vos charges de travail d’IA et, si possible, pour la majeure partie du trafic provenant des navigateurs de vos utilisateurs, tout en réservant la liste d’autorisation d’adresses IP à l’administration de l’instance depuis des emplacements privilégiés.

<Warning>
  W&amp;B recommande vivement d’utiliser des [blocs CIDR](https://en.wikipedia.org/wiki/Classless_Inter-Domain_Routing) attribués à vos passerelles de sortie d’entreprise plutôt que des adresses IP individuelles en `/32`. L’utilisation d’adresses IP individuelles n’est pas évolutive et est soumise à des limites strictes selon le cloud.
</Warning>
