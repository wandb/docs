Cette page explique le fonctionnement de la liste d’autorisation d’adresses IP sur [Cloud dédié](/fr/platform/hosting/hosting-options/dedicated-cloud) de W&amp;B et comment en faire la demande pour votre instance. Utilisez la liste d’autorisation d’adresses IP pour restreindre l’accès à votre instance de Cloud dédié à une liste autorisée d’adresses IP, afin que seul le trafic provenant d’emplacements approuvés puisse atteindre les API W&amp;B et l’interface de l’application W&amp;B.

La liste d’autorisation d’adresses IP s’applique aux accès entre vos charges de travail d’IA et les API W&amp;B, ainsi qu’entre les navigateurs de vos utilisateurs et l’interface de l’application W&amp;B. Après avoir configuré la liste d’autorisation d’adresses IP pour votre instance de Cloud dédié, W&amp;B refuse toute requête provenant d’autres emplacements non autorisés.

La liste d’autorisation d’adresses IP est disponible pour les instances de Cloud dédié sur AWS, Google Cloud et Azure.

Pour configurer la liste d’autorisation d’adresses IP pour votre instance de Cloud dédié, contactez votre équipe W&amp;B.

Vous pouvez utiliser la liste d’autorisation d’adresses IP avec la [connectivité privée sécurisée](./private-connectivity). Si vous utilisez les deux, W&amp;B recommande d’utiliser la connectivité privée sécurisée pour tout le trafic provenant de vos charges de travail d’IA et pour la majeure partie du trafic provenant des navigateurs de vos utilisateurs, et d’utiliser la liste d’autorisation d’adresses IP pour l’administration de l’instance depuis des emplacements privilégiés.

<Warning>
  W&amp;B recommande d’utiliser des [blocs CIDR](https://en.wikipedia.org/wiki/Classless_Inter-Domain_Routing) attribués à vos passerelles de sortie d’entreprise plutôt que des adresses IP individuelles en `/32`. L’utilisation d’adresses IP individuelles ne passe pas à l’échelle et est soumise à des limites strictes selon le fournisseur de cloud.
</Warning>
