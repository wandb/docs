description: Aperçu des secrets W&B, de leur fonctionnement et de la manière de commencer à les utiliser.
title: Gérer les secrets
---

W&amp;B Secret Manager vous permet de stocker, gérer et injecter de manière sécurisée et centralisée des *secrets*, c’est-à-dire des chaînes de caractères sensibles telles que des jetons d’accès, des bearer tokens, des clés API ou des mots de passe. W&amp;B Secret Manager évite d’avoir à ajouter des chaînes sensibles directement dans votre code ou lors de la configuration de l’en-tête ou du [payload](/fr/models/automations/) d’un webhook.

Les secrets sont stockés et gérés dans le Secret Manager de chaque équipe, dans la section **Secrets d’équipe** des [paramètres de l’équipe](/fr/platform/app/settings-page/teams/).

<Note>
  * Seuls les administrateurs W&amp;B peuvent créer, modifier ou supprimer un secret.
  * Les secrets font partie intégrante de W&amp;B, y compris dans les [déploiements W&amp;B Server](/fr/platform/hosting/) que vous hébergez sur Azure, Google Cloud ou AWS. Contactez l’équipe W&amp;B en charge de votre compte pour discuter de l’utilisation des secrets dans W&amp;B si vous utilisez un autre type de déploiement.
  * Dans W&amp;B Server, il vous incombe de configurer des mesures de sécurité adaptées à vos besoins.

    * W&amp;B recommande vivement de stocker les secrets dans le gestionnaire de secrets cloud de votre instance W&amp;B, fourni par AWS, Google Cloud ou Azure, et configuré avec des fonctionnalités de sécurité avancées.

    * W&amp;B déconseille d’utiliser un cluster Kubernetes comme backend de votre magasin de secrets, sauf si vous ne pouvez pas utiliser une instance W&amp;B d’un gestionnaire de secrets cloud (AWS, Google Cloud ou Azure) et que vous savez comment éviter les vulnérabilités de sécurité pouvant survenir avec un cluster.
</Note>

<div id="add-a-secret">
  ## Ajouter un secret
</div>

Pour ajouter un secret :

1. Si le service destinataire l’exige pour authentifier les webhooks entrants, générez le token ou la clé API requis. Si nécessaire, conservez cette chaîne sensible en lieu sûr, par exemple dans un gestionnaire de mots de passe.
2. Connectez-vous à W&amp;B, puis accédez à la page **Settings** de l’équipe.
3. Dans la section **Secrets d’équipe**, cliquez sur **New secret**.
4. À l’aide de lettres, de chiffres et de traits de soulignement (`_`), indiquez un nom pour le secret.
5. Collez la chaîne sensible dans le champ **Secret**.
6. Cliquez sur **Add secret**.

Indiquez les secrets que vous souhaitez utiliser pour votre webhook automation lors de la configuration du webhook. Voir la section [Configurer un webhook](#configure-a-webhook) pour plus d’informations.

<Note>
  Une fois le secret créé, vous pouvez y accéder dans la [charge utile d’une webhook automation](/fr/models/automations/create-automations/webhook/) au format `${SECRET_NAME}`.
</Note>

<div id="rotate-a-secret">
  ## Faire pivoter un secret
</div>

Pour faire pivoter un secret et mettre à jour sa valeur :

1. Cliquez sur l’icône en forme de crayon sur la ligne du secret pour ouvrir ses détails.
2. Définissez **Secret** sur la nouvelle valeur. Vous pouvez aussi cliquer sur **Reveal secret** pour vérifier la nouvelle valeur.
3. Cliquez sur **Add secret**. La valeur du secret est mise à jour et ne correspond plus à la valeur précédente.

<Note>
  Après la création ou la mise à jour d’un secret, vous ne pouvez plus afficher sa valeur actuelle. À la place, faites pivoter le secret vers une nouvelle valeur.
</Note>

<div id="delete-a-secret">
  ## Supprimer un secret
</div>

Pour supprimer un secret :

1. Cliquez sur l’icône de corbeille sur la ligne du secret.
2. Lisez la boîte de dialogue de confirmation, puis cliquez sur **Delete**. Le secret est supprimé immédiatement et de façon permanente.

<div id="manage-access-to-secrets">
  ## Gérer l’accès aux secrets
</div>

Les automations d’une équipe peuvent utiliser les secrets de cette équipe. Avant de supprimer un secret, mettez à jour ou supprimez les automations qui l’utilisent afin d’éviter qu’elles ne cessent de fonctionner.
