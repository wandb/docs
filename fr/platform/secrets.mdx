description: Aperçu des secrets W&B, de leur fonctionnement et de la façon de commencer à les utiliser.
title: Gérer les secrets
---

W&amp;B Secret Manager vous permet de stocker, gérer et injecter de façon sécurisée et centralisée des *secrets*, c’est-à-dire des chaînes sensibles telles que des jetons d’accès, des bearer tokens, des clés API ou des mots de passe. W&amp;B Secret Manager évite d’avoir à ajouter des chaînes sensibles directement dans votre code ou lors de la configuration de l’en-tête d’un webhook ou de son [payload](/fr/models/automations/).

Les secrets sont stockés et gérés dans le Secret Manager de chaque équipe, dans la section **Team secrets** des [paramètres de l’équipe](/fr/platform/app/settings-page/teams/).

<Note>
  * Seuls les administrateurs W&amp;B peuvent créer, modifier ou supprimer un secret.
  * Les secrets font partie intégrante de W&amp;B, y compris dans les [déploiements W&amp;B Server](/fr/platform/hosting/) que vous hébergez sur Azure, Google Cloud ou AWS. Contactez votre équipe de compte W&amp;B pour discuter de la façon dont vous pouvez utiliser les secrets dans W&amp;B si vous utilisez un autre type de déploiement.
  * Dans W&amp;B Server, vous êtes responsable de configurer les mesures de sécurité adaptées à vos besoins.

    * W&amp;B recommande vivement de stocker les secrets dans une instance W&amp;B s’appuyant sur le gestionnaire de secrets d’un fournisseur de cloud (AWS, Google Cloud ou Azure), configuré avec des fonctionnalités de sécurité avancées.

    * W&amp;B déconseille d’utiliser un cluster Kubernetes comme backend de votre magasin de secrets, sauf si vous ne pouvez pas utiliser une instance W&amp;B d’un gestionnaire de secrets cloud (AWS, Google Cloud ou Azure) et que vous savez comment prévenir les vulnérabilités de sécurité pouvant survenir avec un cluster.
</Note>

<div id="add-a-secret">
  ## Ajouter un secret
</div>

Pour ajouter un secret :

1. Si le service de destination l&#39;exige pour authentifier les webhooks entrants, générez le jeton ou la clé API requis. Si nécessaire, enregistrez cette valeur sensible de manière sécurisée, par exemple dans un gestionnaire de mots de passe.
2. Connectez-vous à W&amp;B et accédez à la page **Settings** de l&#39;équipe.
3. Dans la section **Team Secrets**, cliquez sur **New secret**.
4. À l&#39;aide de lettres, de chiffres et de traits de soulignement (`_`), saisissez un nom pour le secret.
5. Collez la valeur sensible dans le champ **Secret**.
6. Cliquez sur **Add secret**.

Indiquez les secrets que vous souhaitez utiliser pour votre automatisation de webhook lorsque vous configurez le webhook. Voir la section [Configurer un webhook](#configure-a-webhook) pour plus d&#39;informations.

<Note>
  Une fois le secret créé, vous pouvez y accéder dans le [payload d&#39;une automatisation de webhook](/fr/models/automations/create-automations/webhook/) au format `${SECRET_NAME}`.
</Note>

<div id="rotate-a-secret">
  ## Effectuer la rotation d’un secret
</div>

Pour effectuer la rotation d’un secret et mettre à jour sa valeur :

1. Cliquez sur l’icône en forme de crayon dans la ligne du secret pour ouvrir les détails du secret.
2. Définissez **Secret** sur la nouvelle valeur. Facultativement, cliquez sur **Révéler le secret** pour vérifier la nouvelle valeur.
3. Cliquez sur **Add secret**. La valeur du secret est mise à jour et ne renvoie plus à l’ancienne valeur.

<Note>
  Après la création ou la mise à jour d’un secret, vous ne pouvez plus révéler sa valeur actuelle. Effectuez plutôt la rotation du secret vers une nouvelle valeur.
</Note>

<div id="delete-a-secret">
  ## Supprimer un secret
</div>

Pour supprimer un secret :

1. Cliquez sur l’icône de corbeille sur la ligne du secret.
2. Lisez la boîte de dialogue de confirmation, puis cliquez sur **Supprimer**. Le secret est supprimé immédiatement et de façon permanente.

<div id="manage-access-to-secrets">
  ## Gérer l’accès aux secrets
</div>

Les automatisations d’une équipe peuvent utiliser les secrets de l’équipe. Avant de supprimer un secret, mettez à jour ou supprimez les automatisations qui l’utilisent afin d’éviter qu’elles ne cessent de fonctionner.
