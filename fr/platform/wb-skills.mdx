Les W&amp;B Skills sont des ensembles d’instructions réutilisables qui apprennent aux agents de code à utiliser efficacement W&amp;B. Au lieu de guider manuellement votre agent dans les API W&amp;B et les bonnes pratiques, installez des Skills afin qu’il puisse gérer de manière autonome le suivi des expériences, Tracing, les évaluations et le monitoring.

Les Skills fonctionnent avec plusieurs agents de code majeurs, notamment :

* Claude Code
* Codex
* Cursor
* GitHub Copilot
* Gemini CLI

Pour obtenir la liste complète des agents pris en charge, consultez la [documentation CLI de W&amp;B Skills](https://github.com/vercel-labs/skills#supported-agents).

<div id="wb-skills-capabilities">
  ## Capacités de W&amp;B Skills
</div>

Skills couvre à la fois le [SDK W&amp;B Models](/fr/models/ref) (runs d&#39;entraînement, métriques, Artifacts, Sweeps) et le [SDK Weave](/fr/weave/reference/python-sdk) (traces, évaluations, évaluateurs). Inclut des bibliothèques utilitaires, de la documentation de référence et des approches d&#39;analyse de données.

| Flux de travail            | Capacités                                                                                                                                                                                                                                                                                          |
| -------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Entraînement du modèle** | <ul><li>Journalisez des métriques et des médias enrichis pendant l&#39;entraînement et le fine-tuning.</li><li>Suivez et comparez les expériences.</li><li>Analysez les Runs et les résultats, comme les courbes de perte et les scores de précision.</li><li>Ajustez les hyperparamètres.</li></ul> |
| **Création d&#39;agents**  | <ul><li>Tracez les applications d&#39;IA agentique.</li><li>Analysez les traces et classez les modes de défaillance.</li><li>Évaluez les modèles et les agents à l&#39;aide de jeux de données étiquetés.</li><li>Exécutez des évaluations en ligne pour le monitoring en production.</li></ul>    |

<div id="prerequisites">
  ## Prérequis
</div>

Skills requiert les éléments suivants :

* [Node.js](https://nodejs.org/) (pour la commande `npx`).
* Une clé API W&amp;B. Créez-en une sur [wandb.ai/authorize](https://wandb.ai/authorize), puis définissez-la comme variable d&#39;environnement :
  ```shell
  export WANDB_API_KEY=<your-api-key>
  ```
* (Facultatif) Définissez le nom de votre projet W&amp;B comme variable d&#39;environnement `WANDB_PROJECT`. Cela permet à votre agent de cibler le bon projet W&amp;B sans avoir à le préciser à chaque fois.

<div id="install-wb-skills">
  ## Installer W&amp;B Skills
</div>

Pour installer W&amp;B Skills globalement, exécutez la commande suivante avec l’indicateur `--global` :

```bash
npx skills add wandb/skills --skill '*' --yes --global
```

Pour installer Skills pour un projet spécifique, exécutez la commande suivante dans le répertoire de votre projet :

```bash
npx skills add wandb/skills --skill '*' --yes
```

Vous pouvez également installer des Skills pour des agents spécifiques à l’aide de l’option `--agent` :

```bash
npx skills add wandb/skills --agent claude-code --skill '*' --yes --global
```

Pour consulter la liste des options `--agent` et `--skill`, voir la [documentation CLI de skills](https://github.com/vercel-labs/skills#supported-agents).

<div id="use-wb-skills">
  ## Utiliser W&amp;B Skills
</div>

Une fois installé, vous pouvez demander à l&#39;agent d&#39;effectuer des tâches liées à W&amp;B pour votre projet. Les prompts d’exemple suivants illustrent certaines des tâches que votre agent peut accomplir avec W&amp;B Skills :

* &quot;Journalise les métriques d&#39;entraînement de mon modèle PyTorch dans W&amp;B.&quot;
* &quot;Analyse les courbes de perte de mes 10 derniers runs et identifie la configuration la plus performante.&quot;
* &quot;Trace mon agent LangChain et journalise les résultats dans Weave.&quot;
* &quot;Exécute une évaluation sur mon agent à l&#39;aide du jeu de données de test et résume les résultats.&quot;
* &quot;Trouve les modes de défaillance dans ma dernière évaluation et classe-les.&quot;
* &quot;Compare les configurations du run A et du run B et montre-moi les différences.&quot;

<div id="usage-tips">
  ## Conseils d’utilisation
</div>

Skills fonctionne mieux lorsque vous utilisez des requêtes plus précises plutôt que des questions larges et ouvertes. Le tableau suivant présente quelques exemples de prompts recommandés, ainsi que des prompts trop vagues.

| Recommandé                                                                    | Non recommandé                                   |
| ----------------------------------------------------------------------------- | ------------------------------------------------ |
| &quot;Quelle est la perte de validation finale de mes 5 derniers runs ?&quot; | &quot;Comment se comporte mon modèle ?&quot;     |
| &quot;Résume l’utilisation des tokens sur mes 10 dernières traces.&quot;      | &quot;Montre-moi toutes mes traces.&quot;        |
| &quot;Compare les configurations du run A et du run B.&quot;                  | &quot;Quels sont mes meilleurs runs ?&quot;      |
| &quot;Quelle évaluation avait le score F1 le plus élevé ?&quot;               | &quot;Comment se passent mes évaluations ?&quot; |
