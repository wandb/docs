Vos paramètres utilisateur vous permettent de gérer votre profil, l&#39;équipe par défaut, les clés API, les alertes, les intégrations et d&#39;autres options au niveau du compte. Cette page décrit chaque section des paramètres utilisateur et comment l&#39;utiliser.

Pour ouvrir vos paramètres utilisateur, accédez à la page de votre profil et sélectionnez votre icône utilisateur dans le coin supérieur droit. Dans le menu déroulant, choisissez **Settings**.

<div id="profile">
  ## Profil
</div>

Dans la section **Profil**, vous pouvez gérer et modifier le nom de votre compte ainsi que votre établissement. Vous pouvez également ajouter, si vous le souhaitez, une biographie, un emplacement, un lien vers votre site web personnel ou celui de votre établissement, et téléverser une image de profil.

<div id="edit-your-intro">
  ### Modifiez votre introduction
</div>

Votre introduction apparaît en haut de votre profil et constitue un bon endroit pour vous présenter, ajouter des liens vers des projets ou partager des comptes associés.

Pour modifier votre introduction, cliquez sur **Edit** en haut de votre profil. L’éditeur WYSIWYG qui s’ouvre prend en charge Markdown.

1. Pour modifier une ligne, cliquez dessus. Pour gagner du temps, tapez `/` et choisissez Markdown dans la liste.
2. Utilisez les poignées de déplacement d’un élément pour le déplacer.
3. Pour supprimer un bloc, cliquez sur la poignée de déplacement, puis sur **Delete**.
4. Pour enregistrer vos modifications, cliquez sur **Save**.

Votre introduction mise à jour est désormais visible sur votre profil.

<div id="add-social-badges">
  #### Ajouter des badges de réseaux sociaux
</div>

Pour ajouter un badge de suivi pour le compte `@weights_biases` sur X, ajoutez un lien au format Markdown avec une balise HTML `<img>` pointant vers l’image du badge :

```markdown
[![X: @weights_biases](https://img.shields.io/twitter/follow/weights_biases?style=social)](https://x.com/intent/follow?screen_name=weights_biases)
```

Dans une balise `<img>`, vous pouvez spécifier `width`, `height`, ou les deux. Si vous n’en spécifiez qu’un seul, les proportions de l’image sont conservées.

<div id="default-team">
  ## Équipe par défaut
</div>

Si vous êtes membre de plusieurs équipes, la section **Équipe par défaut** vous permet de configurer l’équipe par défaut à utiliser lorsqu’un run ou une trace Weave ne spécifie pas d’équipe. Si vous n’êtes membre que d’une seule équipe, cette équipe devient l’équipe par défaut et cette section n’apparaît pas.

Les étapes pour définir votre équipe par défaut varient selon votre déploiement. Sélectionnez l’onglet correspondant à votre environnement.

<Tabs>
  <Tab title="Cloud mutualisé">
    À côté de **Emplacement par défaut pour créer de nouveaux Projects**, cliquez sur la liste déroulante, puis sélectionnez votre équipe par défaut.
  </Tab>

  <Tab title="Cloud dédié / Autogéré">
    1. À côté de **Emplacement par défaut pour créer de nouveaux Projects**, cliquez sur la liste déroulante, puis sélectionnez votre équipe par défaut ou votre entité personnelle.
    2. **Facultatif :** si un administrateur a activé les projets publics dans **Compte** &gt; **Settings** &gt; **Confidentialité**, configurez la visibilité par défaut de vos nouveaux Projects. Cliquez sur le bouton à côté de **Confidentialité par défaut du projet dans votre compte personnel**, puis sélectionnez **Privé** (par défaut) ou **Public**.
    3. **Facultatif :** si un administrateur a activé [l’enregistrement et la comparaison du code par défaut](/fr/models/app/features/panels/code/) dans **Compte** &gt; **Settings** &gt; **Confidentialité**, cliquez sur **Activer l’enregistrement du code dans votre compte personnel** pour l’activer pour vos runs.
  </Tab>
</Tabs>

<Note>
  Pour spécifier l’équipe par défaut lorsque vous exécutez un script dans un environnement automatisé, indiquez l’emplacement par défaut à l’aide de la [variable d’environnement](https://docs.wandb.ai/models/track/environment-variables) `WANDB_ENTITY`.
</Note>

<div id="teams">
  ## Équipes
</div>

La section **Équipes** répertorie toutes vos équipes.

* Cliquez sur le nom d’une équipe pour accéder à sa page.
* Si vous êtes autorisé à rejoindre d’autres équipes, cliquez sur **Voir les équipes** à côté de **Nous avons trouvé des équipes que vous pouvez rejoindre**.
* **Facultatif :** Activez **Masquer les équipes dans le profil public**.

<Note>
  Pour créer ou gérer une équipe, voir [Gérer les équipes](/fr/platform/app/settings-page/teams/).
</Note>

<div id="api-keys">
  ## Clés API
</div>

La section **Clés API** vous permet de gérer vos clés API personnelles pour vous authentifier auprès des services W&amp;B. Dans cette section, vous pouvez consulter les clés existantes, en créer de nouvelles et révoquer celles dont vous n&#39;avez plus besoin.

<div id="view-your-api-keys">
  ### Consultez vos clés API
</div>

Le tableau des clés API affiche :

* **ID de la clé** : La première partie de chaque clé API, utilisée pour l’identifier.
* **Nom** : Un nom descriptif que vous avez donné lorsque vous avez créé la clé.
* **Créée le** : La date de création de la clé.
* **Dernière utilisation** : La date et l’heure de la dernière utilisation.

<Note>
  Pour des raisons de sécurité, le tableau n’affiche que l’ID de la clé (la première partie de la clé). La clé API secrète complète n’apparaît qu’une seule fois, au moment de sa création.
</Note>

Pour filtrer la liste des clés API, saisissez une partie du nom de la clé ou de son ID.

<div id="delete-an-api-key">
  ### Supprimer une clé API
</div>

Supprimez une clé API lorsque vous n’en avez plus besoin ou si elle a pu être compromise. Pour révoquer l’accès en supprimant une clé API :

1. Trouvez la clé que vous souhaitez supprimer dans le tableau des clés API.
2. Cliquez sur le bouton Supprimer à côté de la clé.
3. Confirmez la suppression.

W&amp;B supprime la clé du tableau et vous ne pouvez plus l’utiliser pour vous authentifier.

<Warning>
  La suppression d’une clé API révoque immédiatement l’accès de tous les scripts ou services qui utilisent cette clé. Avant de supprimer l’ancienne clé, assurez-vous d’avoir mis à jour tous les systèmes afin qu’ils utilisent une nouvelle clé.
</Warning>

<div id="beta-features">
  ## Fonctionnalités bêta
</div>

Dans la section **Fonctionnalités bêta**, vous pouvez, si vous le souhaitez, activer des modules complémentaires et des aperçus de produits en cours de développement. Sélectionnez le bouton de bascule à côté de la fonctionnalité bêta que vous souhaitez activer.

<div id="alerts">
  ## Alertes
</div>

Recevez des notifications lorsque vos runs se terminent ou plantent, ou définissez des alertes personnalisées avec [`wandb.Run.alert()`](/fr/models/runs/alert/). Vous recevez des notifications par e-mail ou dans Slack. Basculez l’interrupteur à côté du type d’événement pour lequel vous souhaitez recevoir des alertes :

* **Runs terminés** : Vous avertit lorsqu’un run W&amp;B se termine avec succès.
* **Run planté** : Vous avertit lorsqu’un run ne parvient pas à se terminer.

Pour plus d’informations sur la configuration et la gestion des alertes, voir [Envoyer des alertes avec `wandb.Run.alert()`](/fr/models/runs/alert/).

<div id="personal-github-integration">
  ## Intégration GitHub personnelle
</div>

Connectez un compte GitHub personnel afin que W&amp;B puisse accéder aux ressources GitHub en votre nom. Pour connecter un compte GitHub :

1. Sélectionnez le bouton **Connect Github**. W&amp;B vous redirige vers une page d’autorisation OAuth.
2. Dans la section **Organization access**, sélectionnez l’organisation à laquelle accorder l’accès.
3. Sélectionnez **Authorize wandb**.

Votre compte GitHub personnel est maintenant lié à votre compte W&amp;B.

<div id="delete-your-account">
  ## Supprimer votre compte
</div>

Sélectionnez le bouton **Supprimer le compte** pour supprimer votre compte.

<Warning>
  La suppression du compte est irréversible.
</Warning>

<Note>
  Vous ne pouvez pas supprimer un compte utilisateur si celui-ci est l’**administrateur de la facturation** d’une organisation qui compte plus d’un membre. Avant de supprimer un tel compte, transférez le rôle d’administrateur de la facturation à un autre utilisateur sur la page des [paramètres de facturation](/fr/platform/app/settings-page/billing-settings#billing-admin). Si vous êtes le seul membre de l’organisation, la suppression de votre compte supprime également l’organisation.
</Note>

Pour supprimer le compte d’une organisation entière plutôt qu’un compte utilisateur, voir [Comment puis-je supprimer le compte de mon organisation ?](/fr/support/models/articles/how-do-i-delete-my-organization-account).

<div id="storage">
  ## Stockage
</div>

La section **Stockage** indique l’espace de stockage total utilisé par votre compte sur les serveurs de W&amp;B. Le plan de stockage par défaut est de 100 Go. Pour plus d’informations sur le stockage et la tarification, voir la page [Tarification](https://wandb.ai/site/pricing).
