<div id="team-settings">
  # Paramètres de l’équipe
</div>

Modifiez les paramètres de votre équipe, notamment les membres, l’avatar, les alertes, la confidentialité et l’utilisation. Les Administrateurs de l’organisation et les Administrateurs d’équipe peuvent consulter et modifier les paramètres d’une équipe.

<Note>
  Seuls les comptes de type Administration peuvent modifier les paramètres de l’équipe ou supprimer un membre de l’équipe.
</Note>

<div id="members">
  ## Membres
</div>

La section Membres affiche une liste de toutes les invitations en attente, ainsi que des membres ayant accepté l’invitation à rejoindre l’équipe. Pour chaque membre répertorié, vous voyez son nom, son nom d’utilisateur, son e-mail, son rôle dans l’équipe, ainsi que ses privilèges d’accès à Models et à W&amp;B Weave, hérités de l’organisation. Vous pouvez choisir parmi les rôles d’équipe standard **Admin**, **Membre** et **Lecture seule**. Si votre organisation a créé des [rôles personnalisés](/fr/platform/hosting/iam/access-management/manage-organization#add-and-manage-custom-roles), vous pouvez attribuer un rôle personnalisé à la place.

Voir [Ajouter et gérer les équipes](/fr/platform/hosting/iam/access-management/manage-organization#add-and-manage-teams) pour savoir comment créer une équipe, gérer les équipes, ainsi que gérer les appartenances et les rôles des équipes. Pour configurer qui peut inviter de nouveaux membres et définir les autres paramètres de confidentialité de l’équipe, voir [Confidentialité](#privacy).

<div id="avatar">
  ## Avatar
</div>

Définissez votre avatar en accédant à la section **Avatar** et en important une image.

1. Sélectionnez **Update Avatar** pour ouvrir une boîte de dialogue de sélection de fichier.
2. Dans la boîte de dialogue, choisissez l’image que vous souhaitez utiliser.

<div id="alerts">
  ## Alertes
</div>

Informez votre équipe lorsqu’un run plante ou se termine, ou configurez des alertes personnalisées. Votre équipe peut recevoir ces alertes par e-mail ou via Slack.

Basculez le commutateur à côté du type d’événement pour lequel vous souhaitez recevoir des alertes. Weights and Biases fournit par défaut les types d’événement suivants :

* **Runs terminés** : un run Weights and Biases s’est terminé avec succès.
* **Run planté** : un run n’a pas pu se terminer.

Pour plus d’informations sur la configuration et la gestion des alertes, voir [Envoyer des alertes avec `wandb.Run.alert()`](/fr/models/runs/alert).

<div id="slack-notifications">
  ## Notifications Slack
</div>

Configurer les destinations Slack où les [automatisations](/fr/models/automations/) de votre équipe peuvent envoyer des notifications lorsqu’un événement se produit dans un registre ou un projet, par exemple lorsqu’un nouvel artefact est créé ou lorsqu’une métrique de run atteint un seuil défini. Se référer à [Créer une automatisation Slack](/fr/models/automations/create-automations/slack/).

<div id="webhooks">
  ## Webhooks
</div>

Configurez les webhooks que les [automatisations](/fr/models/automations/) de votre équipe peuvent déclencher lorsqu’un événement se produit dans un registre ou un projet, par exemple lorsqu’un nouvel artefact est créé ou lorsqu’une métrique de run atteint un seuil défini. Référez-vous à [Créer une automatisation de webhook](/fr/models/automations/create-automations/slack/).

<div id="privacy">
  ## Confidentialité
</div>

Accédez à la section **Confidentialité** pour modifier les paramètres de confidentialité. Seuls les administrateurs de l’organisation peuvent modifier les paramètres de confidentialité.

* Désactivez la possibilité de rendre les futurs projets publics ou de partager des rapports publiquement.
* Autorisez n’importe quel membre de l’équipe à inviter d’autres membres, plutôt que les seuls administrateurs d’équipe.
* Définissez si l’enregistrement du code est activé par défaut.

<div id="usage">
  ## Utilisation
</div>

La section **Utilisation** décrit la quantité totale de mémoire utilisée par l’équipe sur les serveurs de Weights and Biases. Le plan de stockage par défaut est de 100GB. Pour plus d’informations sur le stockage et la tarification, voir la page [Tarification](https://wandb.ai/site/pricing).

<div id="storage">
  ## Stockage
</div>

La section **Stockage** décrit la configuration du bucket de stockage cloud utilisé pour les données de l&#39;équipe. Pour plus d&#39;informations, voir [Connecteur de stockage sécurisé](/fr/platform/app/settings-page/teams/#secure-storage-connector) ou consultez notre documentation [W&amp;B Server](/fr/platform/hosting/data-security/secure-storage-connector) si vous êtes en auto-hébergement.
