La page suivante décrit comment configurer les options de la file d’attente Launch.

<div id="set-up-queue-config-templates">
  ## Configurer les modèles de configuration de file d&#39;attente
</div>

Administrez et gérez les garde-fous sur la consommation de calcul avec les modèles de configuration de file d&#39;attente. Définissez les valeurs par défaut, les minimums et les maximums pour des champs tels que la consommation de mémoire, le GPU et la durée d&#39;exécution.

Une fois que vous avez configuré une file d&#39;attente avec des modèles de configuration, les membres de votre équipe peuvent modifier les champs que vous avez définis uniquement dans la plage que vous avez définie.

<div id="configure-queue-template">
  ### Configurer le modèle de file d&#39;attente
</div>

Vous pouvez configurer un modèle de file d&#39;attente sur une file d&#39;attente existante ou créer une nouvelle file d&#39;attente.

1. Accédez à l&#39;[application W&amp;B Launch](https://wandb.ai/launch).
2. Sélectionnez **View queue** à côté du nom de la file d&#39;attente à laquelle vous souhaitez ajouter un modèle.
3. Sélectionnez l&#39;onglet **Config**. Cela affichera des informations sur votre file d&#39;attente telles que la date de création de la file d&#39;attente, la **Queue config**, et les redéfinitions existantes au moment du lancement.
4. Accédez à la section **Queue config**.
5. Identifiez les paires clé-valeur de configuration pour lesquelles vous souhaitez créer un modèle.
6. Remplacez la valeur dans la configuration par un champ de modèle. Les champs de modèle prennent la forme de `{{variable-name}}`.
7. Cliquez sur le bouton **Parse configuration**. Lorsque vous analysez votre configuration, W&amp;B créera automatiquement des tuiles sous la **Queue config** pour chaque modèle que vous avez créé.
8. Pour chaque tuile générée, vous devez d&#39;abord spécifier le type de données (chaîne, entier ou flottant) que la **Queue config** peut autoriser. Pour ce faire, sélectionnez le type de données dans le menu déroulant **Type**.
9. En fonction de votre type de données, complétez les champs qui apparaissent dans chaque tuile.
10. Cliquez sur **Save config**.

Par exemple, supposez que vous souhaitiez créer un modèle qui limite les instances AWS que votre équipe peut utiliser. Avant d&#39;ajouter un champ de modèle, votre **Queue config** pourrait ressembler à quelque chose comme :

```yaml title="launch config"
RoleArn: arn:aws:iam:region:account-id:resource-type/resource-id
ResourceConfig:
  InstanceType: ml.m4.xlarge
  InstanceCount: 1
  VolumeSizeInGB: 2
OutputDataConfig:
  S3OutputPath: s3://bucketname
StoppingCondition:
  MaxRuntimeInSeconds: 3600
```

Lorsque vous ajoutez un champ de template pour le `InstanceType`, votre configuration ressemblera à :

```yaml title="launch config"
RoleArn: arn:aws:iam:region:account-id:resource-type/resource-id
ResourceConfig:
  InstanceType: "{{aws_instance}}"
  InstanceCount: 1
  VolumeSizeInGB: 2
OutputDataConfig:
  S3OutputPath: s3://bucketname
StoppingCondition:
  MaxRuntimeInSeconds: 3600
```

Ensuite, vous cliquez sur **Parse configuration**. Une nouvelle tuile étiquetée `aws-instance` apparaîtra sous le **Queue config**.

À partir de là, vous sélectionnez String comme type de données dans la liste déroulante **Type**. Cela remplira des champs où vous pouvez spécifier les valeurs parmi lesquelles un utilisateur peut choisir. Par exemple, dans l&#39;image suivante, l&#39;administrateur de l&#39;équipe a configuré deux types d&#39;instances AWS différents que les utilisateurs peuvent choisir (`ml.m4.xlarge` et `ml.p3.xlarge`) :

<div id="dynamically-configure-launch-jobs">
  ## Configurer dynamiquement les jobs Launch
</div>

Les Queue configs peuvent être configurées dynamiquement à l&#39;aide de macros qui sont évaluées lorsque l&#39;agent extrait un job de la file d&#39;attente. Vous pouvez utiliser les macros suivantes :

| Macro             | Description                                                       |
| ----------------- | ----------------------------------------------------------------- |
| `${project_name}` | Le nom du projet dans lequel le run est lancé.                    |
| `${entity_name}`  | L&#39;entité propriétaire du projet dans lequel le run est lancé. |
| `${run_id}`       | L&#39;ID du run qui est lancé.                                    |
| `${run_name}`     | Le nom du run qui est lancé.                                      |
| `${image_uri}`    | L&#39;URI de l&#39;image de conteneur pour ce run.                |

<Note>
  Toute macro personnalisée qui ne figure pas dans le tableau précédent (par exemple `${MY_ENV_VAR}`) est substituée par une variable d&#39;environnement de l&#39;environnement de l&#39;agent.
</Note>

<div id="use-the-launch-agent-to-build-images-that-execute-on-accelerators-gpus">
  ## Utilisez l&#39;agent Launch pour construire des images qui s&#39;exécutent sur des accélérateurs (GPU)
</div>

Vous pourriez avoir besoin de spécifier une image de base pour accélérateur si vous utilisez Launch pour construire des images qui s&#39;exécutent dans un environnement d&#39;accélérateur.

Cette image de base pour accélérateur doit satisfaire les exigences suivantes :

* Compatibilité Debian (le Dockerfile de Launch utilise apt-get pour récupérer Python)
* Compatibilité de l&#39;ensemble d&#39;instructions du matériel CPU et GPU (Assurez-vous que votre version CUDA est prise en charge par le GPU que vous envisagez d&#39;utiliser)
* Compatibilité entre la version d&#39;accélérateur que vous fournissez et les paquets installés dans votre algorithme ML
* Paquets installés qui nécessitent des étapes supplémentaires pour configurer la compatibilité avec le matériel

<div id="how-to-use-gpus-with-tensorflow">
  ### Comment utiliser les GPU avec TensorFlow
</div>

Assurez-vous que TensorFlow utilise correctement votre GPU. Pour ce faire, spécifiez une image Docker et son tag d&#39;image pour la clé `builder.accelerator.base_image` dans la configuration des ressources de la file d&#39;attente.

Par exemple, l&#39;image de base `tensorflow/tensorflow:latest-gpu` garantit que TensorFlow utilise correctement votre GPU. Cela peut être configuré à l&#39;aide de la configuration des ressources dans la file d&#39;attente.

L&#39;extrait JSON suivant illustre comment spécifier l&#39;image de base TensorFlow dans votre Queue config :

```json title="Queue config"
{
    "builder": {
        "accelerator": {
            "base_image": "tensorflow/tensorflow:latest-gpu"
        }
    }
}
```
