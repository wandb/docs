<div id="what-is-launch">
  ## Qu’est-ce que Launch ?
</div>

<Card title="Essayer sur Colab" href="https://colab.research.google.com/drive/1wX0OSVxZJDHRsZaOaOEDx-lLUrO1hHgP" icon="python" />

Faites facilement passer à l’échelle les [runs](/fr/models/runs/) d’entraînement depuis votre ordinateur vers une ressource de calcul comme Amazon SageMaker, Kubernetes, etc., avec W&amp;B Launch. Une fois W&amp;B Launch configuré, vous pouvez rapidement exécuter des scripts d’entraînement, des suites d’évaluation de modèles, préparer des modèles pour l’inférence en production, et bien plus encore en quelques clics et commandes.

<div id="how-it-works">
  ## Comment cela fonctionne
</div>

Launch se compose de trois composants fondamentaux : **jobs Launch**, **files d&#39;attente** et **agents**.

Un [*job Launch*](./launch-terminology#launch-job) est un modèle permettant de configurer et d’exécuter des tâches dans votre flux de travail ML. Une fois que vous disposez d’un job Launch, vous pouvez l’ajouter à une [*file d&#39;attente Launch*](./launch-terminology#launch-queue). Une file d&#39;attente Launch est une file d’attente premier entré, premier sorti (FIFO) dans laquelle vous pouvez configurer et soumettre vos jobs à une ressource de calcul cible spécifique, comme Amazon SageMaker ou un cluster Kubernetes.

À mesure que des jobs sont ajoutés à la file d’attente, les [*agents Launch*](./launch-terminology#launch-agent) interrogent cette file d’attente et exécutent le job sur le système ciblé par celle-ci.

<Frame>
  <img src="/images/launch/launch_overview.png" alt="Schéma d’aperçu de W&B Launch" />
</Frame>

Selon votre cas d’usage, vous (ou un membre de votre équipe) configurerez la file d&#39;attente Launch en fonction de la [ressource de calcul cible](./launch-terminology#target-resources) choisie (par exemple Amazon SageMaker) et déploierez un agent Launch sur votre propre infrastructure.

Voir la page [Termes et concepts](./launch-terminology) pour plus d’informations sur Launch.

<div id="how-to-get-started">
  ## Comment démarrer
</div>

Selon votre cas d’usage, explorez les ressources suivantes pour bien démarrer avec W&amp;B Launch :

* Si c’est la première fois que vous utilisez W&amp;B Launch, nous vous recommandons de suivre le [guide pas à pas de Launch](#walkthrough).
* Découvrez comment configurer [W&amp;B Launch](/fr/platform/launch/set-up-launch/).
* Créez un [job Launch](/fr/platform/launch/launch-terminology/#launch-job).
* Consultez le [dépôt GitHub public des jobs](https://github.com/wandb/launch-jobs) de W&amp;B Launch pour trouver des modèles de tâches courantes, comme le [déploiement sur Triton](https://github.com/wandb/launch-jobs/tree/main/jobs/deploy_to_nvidia_triton), l’[évaluation d’un LLM](https://github.com/wandb/launch-jobs/tree/main/jobs/openai_evals), entre autres.
  * Affichez les jobs Launch créés à partir de ce dépôt dans ce projet W&amp;B public [`wandb/jobs` project](https://wandb.ai/wandb/jobs/jobs).

<div id="walkthrough">
  ## Guide pas à pas
</div>

Cette page présente les notions de base du flux de travail de W&amp;B Launch.

<Note>
  W&amp;B Launch exécute des charges de travail de machine learning dans des conteneurs. Il n’est pas nécessaire de bien connaître les conteneurs, mais cela peut être utile pour ce guide pas à pas. Voir la [documentation Docker](https://docs.docker.com/guides/docker-concepts/the-basics/what-is-a-container/) pour une introduction aux conteneurs.

<div id="prerequisites">
  ## Prérequis
</div>

Avant de commencer, assurez-vous de satisfaire aux prérequis suivants :

1. Créez un compte sur https://wandb.ai/site, puis connectez-vous à votre compte W&amp;B.
2. Ce guide pas à pas nécessite un accès à un terminal sur une machine avec une CLI Docker et un moteur fonctionnels. Voir le [guide d’installation de Docker](https://docs.docker.com/engine/install/) pour plus d’informations.
3. Installez la version `0.17.1` ou ultérieure du SDK Python W&amp;B :
   ```bash
   pip install wandb>=0.17.1
   ```
4. Dans votre terminal, exécutez `wandb login` ou définissez la variable d’environnement `WANDB_API_KEY` pour vous authentifier auprès de W&amp;B.

<Tabs>
  <Tab title="Se connecter à W&B">
    Dans votre terminal, exécutez :

    ```bash
    wandb login
    ```
  </Tab>

  <Tab title="Variable d’environnement">
    ```bash
        WANDB_API_KEY=<your-api-key>
    ```

    Remplacez `<your-api-key>` par votre clé API W&amp;B.
  </Tab>
</Tabs>

<div id="create-a-launch-job">
  ## Créer un job Launch
</div>

Créez un [job Launch](./launch-terminology#launch-job) de l&#39;une des trois manières suivantes : avec une image Docker, à partir d&#39;un dépôt Git ou à partir du code source local :

<Tabs>
  <Tab title="Avec une image Docker">
    Pour exécuter un conteneur préconfiguré qui enregistre un message dans W&amp;B, ouvrez un terminal et exécutez la commande suivante :

    ```bash
    wandb launch --docker-image wandb/job_hello_world:main --project launch-quickstart
    ```

    La commande précédente télécharge et exécute l&#39;image de conteneur `wandb/job_hello_world:main`.

    Launch configure le conteneur pour envoyer au projet `launch-quickstart` tout ce qui est enregistré avec `wandb`. Le conteneur enregistre un message dans W&amp;B et affiche un lien vers le run nouvellement créé dans W&amp;B. Cliquez sur le lien pour consulter le run dans l&#39;interface W&amp;B.
  </Tab>

  <Tab title="À partir d'un dépôt Git">
    Pour lancer le même job hello-world à partir de son [code source dans le dépôt de jobs W&amp;B Launch](https://github.com/wandb/launch-jobs), exécutez la commande suivante :

    ```bash
    wandb launch --uri https://github.com/wandb/launch-jobs.git \\
    --job-name hello-world-git --project launch-quickstart \\ 
    --build-context jobs/hello_world --dockerfile Dockerfile.wandb \\ 
    --entry-point "python job.py"
    ```

    La commande effectue les opérations suivantes :

    1. Clone le [dépôt de jobs W&amp;B Launch](https://github.com/wandb/launch-jobs) dans un répertoire temporaire.
    2. Crée un job nommé **hello-world-git** dans le projet **hello**. Ce job suit le code source exact et la configuration utilisés pour exécuter ce code.
    3. Construit une image de conteneur à partir du répertoire `jobs/hello_world` et du `Dockerfile.wandb`.
    4. Démarre le conteneur et exécute le script Python `job.py`.

    La sortie de la console affiche le build de l&#39;image et son exécution. La sortie du conteneur devrait être presque identique à celle de l&#39;exemple précédent.
  </Tab>

  <Tab title="À partir du code source local">
    Le code qui n&#39;est pas versionné dans un dépôt Git peut être lancé en indiquant un chemin de répertoire local à l&#39;argument `--uri`.

    Créez un répertoire vide et ajoutez un script Python nommé `train.py` avec le contenu suivant :

    ```python
    import wandb

    with wandb.init() as run:
        run.log({"hello": "world"})
    ```

    Ajoutez un fichier `requirements.txt` avec le contenu suivant :

    ```text
    wandb>=0.17.1
    ```

    Depuis ce répertoire, exécutez la commande suivante :

    ```bash
    wandb launch --uri . --job-name hello-world-code --project launch-quickstart --entry-point "python train.py"
    ```

    La commande effectue les opérations suivantes :

    1. Enregistre le contenu du répertoire actuel dans W&amp;B en tant qu&#39;artefact de code.
    2. Crée un job nommé **hello-world-code** dans le projet **launch-quickstart**.
    3. Construit une image de conteneur en copiant `train.py` et `requirements.txt` dans une image de base, puis exécute `pip install` pour installer les dépendances.
    4. Démarre le conteneur et exécute `python train.py`.
  </Tab>
</Tabs>

<div id="create-a-queue">
  ## Créer une file d&#39;attente
</div>

Launch est conçu pour aider les équipes à créer des flux de travail autour de ressources de calcul partagées. Dans les exemples précédents, la commande `wandb launch` a exécuté un conteneur de manière synchrone sur la machine locale. Les files d&#39;attente et les agents Launch permettent d&#39;exécuter des jobs de façon asynchrone sur des ressources partagées, ainsi que d&#39;utiliser des fonctionnalités avancées comme la priorisation et l&#39;optimisation des hyperparamètres. Pour créer une file d&#39;attente de base, suivez ces étapes :

1. Accédez à [wandb.ai/launch](https://wandb.ai/launch) et cliquez sur le bouton **Créer une file d&#39;attente**.
2. Sélectionnez une **Entité** à associer à la file d&#39;attente.
3. Saisissez un **Nom de la file d&#39;attente**.
4. Sélectionnez **Docker** comme **Ressource**.
5. Laissez le champ **Configuration** vide pour le moment.
6. Cliquez sur **Créer une file d&#39;attente** :rocket:

Après avoir cliqué sur le bouton, le navigateur vous redirige vers l&#39;onglet **Agents** de la vue de la file d&#39;attente. La file d&#39;attente reste dans l&#39;état **Non active** jusqu&#39;à ce qu&#39;un agent commence à l&#39;interroger.

<Frame>
  <img src="/images/launch/create_docker_queue.gif" alt="Création d'une file d'attente Docker" />
</Frame>

Pour les options avancées de configuration de la file d&#39;attente, voir la [page de configuration avancée de la file d’attente](/fr/platform/launch/setup-queue-advanced/).

<div id="connect-an-agent-to-the-queue">
  ## Connecter un agent à la file d&#39;attente
</div>

La vue de la file d&#39;attente affiche un bouton **Ajouter un agent** dans une bannière rouge en haut de l&#39;écran si la file d&#39;attente n&#39;a aucun agent de sondage. Cliquez sur ce bouton pour afficher et copier la commande permettant d&#39;exécuter un agent. La commande doit ressembler à ce qui suit :

```bash
wandb launch-agent --queue <queue-name> --entity <entity-name>
```

Exécutez la commande dans un terminal pour démarrer l&#39;agent. L&#39;agent interroge la file d&#39;attente spécifiée pour récupérer les jobs à exécuter. Lorsqu&#39;un job est reçu, l&#39;agent télécharge ou build l&#39;image de conteneur du job, puis l&#39;exécute, comme si la commande `wandb launch` avait été exécutée localement.

Revenez à [la page Launch](https://wandb.ai/launch) et vérifiez que la file d&#39;attente apparaît désormais comme **Active**.

<div id="submit-a-job-to-the-queue">
  ## Soumettre un job à la file d&#39;attente
</div>

Accédez à votre nouveau projet **launch-quickstart** dans votre compte W&amp;B, puis ouvrez l’onglet des jobs dans le menu de navigation à gauche de l’écran.

La page **Jobs** affiche une liste de jobs W&amp;B créés à partir de runs exécutés précédemment. Cliquez sur votre job Launch pour afficher le code source, les dépendances et tous les runs créés à partir de ce job. Une fois ce guide pas à pas terminé, la liste devrait contenir trois jobs.

Choisissez l’un des nouveaux jobs et suivez ces instructions pour le soumettre à la file d’attente :

1. Cliquez sur le bouton **Launch** pour soumettre le job à une file d’attente. Le panneau **Launch** s’affiche.
2. Sélectionnez la **File d’attente** que vous avez créée précédemment, puis cliquez sur **Launch**.

Le job est alors soumis à la file d’attente. L’agent qui interroge cette file d’attente le récupère et l’exécute. Vous pouvez suivre la progression du job dans l’interface W&amp;B ou en consultant la sortie de l’agent dans le terminal.

La commande `wandb launch` peut envoyer directement des jobs à la file d’attente en spécifiant l’argument `--queue`. Par exemple, pour soumettre le job de conteneur hello-world à la file d’attente, exécutez la commande suivante :

```bash
   wandb launch --docker-image wandb/job_hello_world:main --project launch-quickstart --queue <queue-name>
```
