<div id="what-is-launch">
  ## Qu’est-ce que Launch ?
</div>

<Card title="Essayer dans Colab" href="https://colab.research.google.com/drive/1wX0OSVxZJDHRsZaOaOEDx-lLUrO1hHgP" icon="python" />

Faites facilement passer à l’échelle les [Runs](/fr/models/runs/) d’entraînement depuis votre ordinateur vers une ressource de calcul comme Amazon SageMaker, Kubernetes, et plus encore avec W&amp;B Launch. Une fois W&amp;B Launch configuré, vous pouvez rapidement exécuter des scripts d’entraînement, des suites d’évaluation de modèles, préparer des modèles pour l’inférence en production, et plus encore en quelques clics et commandes.

<div id="how-it-works">
  ## Comment cela fonctionne
</div>

Launch est composé de trois composants fondamentaux : les **jobs Launch**, les **files d’attente**, et les **agents**.

Un [*job Launch*](./launch-terminology#launch-job) est un plan pour configurer et exécuter des tâches dans votre flux de travail ML. Une fois que vous avez un job Launch, vous pouvez l’ajouter à une [*file d’attente Launch*](./launch-terminology#launch-queue). Une file d’attente Launch est une file d’attente premier entré, premier sorti (FIFO) dans laquelle vous pouvez configurer et soumettre vos jobs à une ressource cible de calcul particulière, telle qu’Amazon SageMaker ou un cluster Kubernetes.

À mesure que des jobs sont ajoutés à la file d’attente, les [*agents Launch*](./launch-terminology#launch-agent) interrogent cette file d’attente et exécutent le job sur le système ciblé par la file d’attente.

<Frame>
  <img src="/images/launch/launch_overview.png" alt="Diagramme d’aperçu de W&B Launch" />
</Frame>

Selon votre cas d’usage, vous (ou une personne de votre équipe) configurerez la file d’attente Launch en fonction de la [ressource cible de calcul](./launch-terminology#target-resources) que vous avez choisie (par exemple Amazon SageMaker) et déploierez un agent Launch sur votre propre infrastructure.

Voir la page [Termes et concepts](./launch-terminology) pour plus d’informations sur Launch.

<div id="walkthrough">
  ## Guide pas à pas
</div>

Cette page présente les notions de base du flux de travail de W&amp;B Launch.

<Note>
  W&amp;B Launch exécute des charges de travail d’apprentissage automatique dans des conteneurs. Une connaissance des conteneurs n’est pas requise, mais peut être utile pour ce guide pas à pas. Voir la [documentation Docker](https://docs.docker.com/guides/docker-concepts/the-basics/what-is-a-container/) pour une introduction aux conteneurs.
</Note>

<div id="prerequisites">
  ## Prérequis
</div>

Avant de commencer, assurez-vous de remplir les prérequis suivants :

1. Inscrivez-vous pour créer un compte sur https://wandb.ai/site, puis connectez-vous à votre compte W&amp;B.
2. Ce guide pas à pas nécessite un accès au terminal sur une machine avec un CLI Docker et un moteur fonctionnels. Voir le [guide d’installation de Docker](https://docs.docker.com/engine/install/) pour plus d’informations.
3. Installez la version `0.17.1` ou ultérieure du SDK Python W&amp;B :
   ```bash
   pip install wandb>=0.17.1
   ```
4. Dans votre terminal, exécutez `wandb login` ou définissez la variable d’environnement `WANDB_API_KEY` pour vous authentifier auprès de W&amp;B.

<Tabs>
  <Tab title="Se connecter à W&B">
    Dans votre terminal, exécutez :

    ```bash
    wandb login
    ```
  </Tab>

  <Tab title="Variable d’environnement">
    ```bash
        WANDB_API_KEY=<your-api-key>
    ```

    Remplacez `<your-api-key>` par votre clé API W&amp;B.
  </Tab>
</Tabs>

<div id="create-a-launch-job">
  ## Créer un job Launch
</div>

Créez un [job Launch](./launch-terminology#launch-job) de l’une des trois façons suivantes : avec une image Docker, à partir d’un dépôt Git ou de code source local :

<Tabs>
  <Tab title="Avec une image Docker">
    Pour exécuter un conteneur prédéfini qui enregistre un message dans W&amp;B, ouvrez un terminal et exécutez la commande suivante :

    ```bash
    wandb launch --docker-image wandb/job_hello_world:main --project launch-quickstart
    ```

    La commande précédente télécharge et exécute l’image de conteneur `wandb/job_hello_world:main`.

    Launch configure le conteneur pour que tout ce qui est enregistré avec `wandb` soit envoyé au projet `launch-quickstart`. Le conteneur enregistre un message dans W&amp;B et affiche un lien vers le run nouvellement créé dans W&amp;B. Cliquez sur le lien pour afficher le run dans l’interface utilisateur W&amp;B.
  </Tab>

  <Tab title="Depuis un dépôt Git">
    Pour lancer le même job hello-world à partir de son [code source dans le dépôt de jobs W&amp;B Launch](https://github.com/wandb/launch-jobs), exécutez la commande suivante :

    ```bash
    wandb launch --uri https://github.com/wandb/launch-jobs.git \\
    --job-name hello-world-git --project launch-quickstart \\ 
    --build-context jobs/hello_world --dockerfile Dockerfile.wandb \\ 
    --entry-point "python job.py"
    ```

    La commande effectue les opérations suivantes :

    1. Clone le [dépôt de jobs W&amp;B Launch](https://github.com/wandb/launch-jobs) dans un répertoire temporaire.
    2. Crée un job nommé **hello-world-git** dans le projet **hello**. Ce job suit exactement le code source et la configuration utilisés pour exécuter le code.
    3. Build une image de conteneur à partir du répertoire `jobs/hello_world` et du fichier `Dockerfile.wandb`.
    4. Démarre le conteneur et exécute le script Python `job.py`.

    La sortie de la console montre le build et l’exécution de l’image. La sortie du conteneur devrait être presque identique à celle de l’exemple précédent.
  </Tab>

  <Tab title="À partir de code source local">
    Le code qui n’est pas versionné dans un dépôt Git peut être lancé en indiquant un chemin de répertoire local pour l’argument `--uri`.

    Créez un répertoire vide et ajoutez un script Python nommé `train.py` avec le contenu suivant :

    ```python
    import wandb

    with wandb.init() as run:
        run.log({"hello": "world"})
    ```

    Ajoutez un fichier `requirements.txt` avec le contenu suivant :

    ```text
    wandb>=0.17.1
    ```

    Depuis ce répertoire, exécutez la commande suivante :

    ```bash
    wandb launch --uri . --job-name hello-world-code --project launch-quickstart --entry-point "python train.py"
    ```

    La commande effectue les opérations suivantes :

    1. Enregistre le contenu du répertoire actuel dans W&amp;B en tant qu’artefact de code.
    2. Crée un job nommé **hello-world-code** dans le projet **launch-quickstart**.
    3. Build une image de conteneur en copiant `train.py` et `requirements.txt` dans une image de base, puis exécute `pip install` pour installer les dépendances.
    4. Démarre le conteneur et exécute `python train.py`.
  </Tab>
</Tabs>

<div id="create-a-queue">
  ## Créer la file d’attente
</div>

Launch est conçu pour aider les équipes à créer des flux de travail autour d’un calcul partagé. Dans les exemples précédents, la commande `wandb launch` a exécuté un conteneur de manière synchrone sur la machine locale. Les files d’attente Launch et les agents permettent l’exécution asynchrone des jobs sur des ressources partagées ainsi que des fonctionnalités avancées comme la priorisation et l’optimisation des hyperparamètres. Pour créer une file d’attente de base, suivez ces étapes :

1. Accédez à [wandb.ai/launch](https://wandb.ai/launch) et cliquez sur le bouton **Créer la file d’attente**.
2. Sélectionnez une **Entité** à associer à la file d’attente.
3. Saisissez un **Nom de la file d’attente**.
4. Sélectionnez **Docker** comme **Ressource**.
5. Laissez le champ **Configuration** vide pour le moment.
6. Cliquez sur **Créer la file d’attente** :rocket:

Après avoir cliqué sur le bouton, le navigateur vous redirige vers l’onglet **Agents** de la vue de la file d’attente. La file d’attente reste à l’état **Not active** jusqu’à ce qu’un agent commence à l’interroger.

<Frame>
  <img src="/images/launch/create_docker_queue.gif" alt="Création d’une file d’attente Docker" />
</Frame>

Pour les options avancées de configuration de la file d’attente, voir la [page de configuration avancée des files d’attente](/fr/platform/launch/setup-queue-advanced/).

<div id="connect-an-agent-to-the-queue">
  ## Connecter un agent à la file d&#39;attente
</div>

La vue de la file d&#39;attente affiche un bouton **Add an agent** dans une bannière rouge en haut de l&#39;écran si la file d&#39;attente n&#39;a aucun agent d&#39;interrogation. Cliquez sur le bouton pour voir et copier la commande à exécuter pour lancer un agent. La commande doit ressembler à ce qui suit :

```bash
wandb launch-agent --queue <queue-name> --entity <entity-name>
```

Exécutez la commande dans un terminal pour démarrer l&#39;agent. L&#39;agent interroge la file d&#39;attente spécifiée pour trouver les jobs à exécuter. Une fois reçu, l&#39;agent télécharge ou build, puis exécute une image de conteneur pour le job, comme si la commande `wandb launch` avait été exécutée localement.

Naviguez de nouveau vers [la page Launch](https://wandb.ai/launch) et vérifiez que la file d&#39;attente s&#39;affiche désormais comme **Active**.

<div id="submit-a-job-to-the-queue">
  ## Soumettre un job à la file d&#39;attente
</div>

Accédez à votre nouveau projet **launch-quickstart** dans votre compte W&amp;B et ouvrez l’onglet des jobs depuis la navigation sur le côté gauche de l’écran.

La page **Jobs** affiche une liste de W&amp;B Jobs créés à partir de runs exécutés précédemment. Cliquez sur votre job Launch pour voir le code source, les dépendances et tous les runs créés à partir du job. Après avoir terminé ce guide pas à pas, il devrait y avoir trois jobs dans la liste.

Choisissez l’un des nouveaux jobs et suivez ces instructions pour le soumettre à la file d&#39;attente :

1. Cliquez sur le bouton **Launch** pour soumettre le job à une file d&#39;attente. Le volet latéral **Launch** apparaît.
2. Sélectionnez la **file d&#39;attente** que vous avez créée précédemment et cliquez sur **Launch**.

Cela soumet le job à la file d&#39;attente. L’agent qui interroge cette file d&#39;attente récupère et exécute le job. Vous pouvez surveiller la progression du job depuis l’interface utilisateur W&amp;B ou en inspectant la sortie de l’agent dans le terminal.

La commande `wandb launch` peut envoyer des jobs directement à la file d&#39;attente en spécifiant l’argument `--queue`. Par exemple, pour soumettre le job de conteneur hello-world à la file d&#39;attente, exécutez la commande suivante :

```bash
   wandb launch --docker-image wandb/job_hello_world:main --project launch-quickstart --queue <queue-name>
```
