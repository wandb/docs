---
title: Lorsqu’un même artifact est téléchargé par plusieurs jobs d’une file d’attente Docker, une mise en cache est-elle utilisée, ou est-il retéléchargé à chaque run ?
keywords: ["mise en cache d’artefact", "cache Docker", "téléchargement d’artefact", "montage d’artefact"]
---

Les jobs Launch ne partagent aucun cache. Chaque launch job s’exécute indépendamment. Pour partager un cache, configurez la file d’attente ou l’agent pour monter un cache partagé à l’aide d’arguments Docker dans la configuration de la file d’attente.

Pour des cas d’usage spécifiques, vous pouvez également monter le cache Artifacts de W&amp;B comme volume persistant.
