---
title: Lorsqu’un même artifact est téléchargé par plusieurs jobs d’une file d’attente Docker, une mise en cache est-elle utilisée, ou est-il retéléchargé à chaque run ?
---

Il n’existe aucun mécanisme de mise en cache. Chaque launch job s’exécute indépendamment. Configurez la file d’attente ou l’agent pour monter un cache partagé à l’aide d’arguments Docker dans la configuration de la file d’attente.

De plus, montez le cache Artifacts de W&amp;B comme volume persistant pour des cas d’usage spécifiques.
