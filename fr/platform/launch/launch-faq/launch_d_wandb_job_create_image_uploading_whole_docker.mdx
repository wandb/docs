---
title: Est-ce que `wandb launch -d` ou `wandb job create image` téléverse un artefact Docker complet
  au lieu de le récupérer depuis un registre ?
keywords: ["wandb launch -d", "registre Docker", "téléversement d’image", "téléverser vers un registre"]
---

Non, la commande `wandb launch -d` ne téléverse pas d’images vers un registre. Téléversez les images vers un registre séparément. Suivez ces étapes :

1. Construisez une image.
2. Poussez l’image vers un registre.

Le flux de travail est le suivant. Remplacez `[REPO-URL]` par l’URL de votre registre et `[TAG]` par le tag de l’image :

```bash
docker build -t [REPO-URL]:[TAG] .
docker push [REPO-URL]:[TAG]
wandb launch -d [REPO-URL]:[TAG]
```

L’agent Launch démarre alors un job qui pointe vers le conteneur spécifié. Pour voir des exemples de configuration de l’accès de l’agent afin d’extraire des images depuis un registre de conteneurs, consultez [Configuration avancée de l’agent](/fr/platform/launch/setup-agent-advanced/#agent-configuration).

Pour Kubernetes, assurez-vous que les Pods du cluster ont accès au registre dans lequel l’image est envoyée.
