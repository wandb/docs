---
title: Launch prend-il en charge la parallélisation ? Comment puis-je limiter les ressources consommées
  par un job ?
---

Launch prend en charge la mise à l&#39;échelle des jobs sur plusieurs GPU et nœuds. Voir le [guide d’intégration Volcano](/fr/models/integrations) pour plus de détails.

Chaque agent Launch est configuré avec un paramètre `max_jobs`, qui détermine le nombre maximal de jobs simultanés qu’il peut exécuter. Plusieurs agents peuvent pointer vers une seule file d’attente, à condition qu’ils se connectent à une infrastructure de lancement appropriée.

Vous pouvez définir des limites sur le CPU, le GPU, la mémoire et d’autres ressources au niveau de la file d’attente ou de l’exécution du job dans la configuration des ressources. Pour des informations sur la configuration de files d’attente avec des limites de ressources sur Kubernetes, voir le [guide de configuration Kubernetes](/fr/platform/launch/setup-launch-kubernetes/).

Pour les balayages, incluez le bloc suivant dans la configuration de la file d’attente afin de limiter le nombre de runs simultanés :

```yaml title="queue config"
  scheduler:
    num_workers: 4
```
