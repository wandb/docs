---
title: "Launch prend-il en charge la parallélisation ? Comment puis-je limiter les ressources consommées
  par un job ?"
---

Launch permet de répartir les jobs sur plusieurs GPU et nœuds. Consultez le [guide d’intégration Volcano](/fr/models/integrations) pour plus de détails.

Chaque agent Launch est configuré avec un paramètre `max_jobs`, qui détermine le nombre maximal de jobs simultanés qu’il peut exécuter. Plusieurs agents peuvent pointer vers une même file d&#39;attente, à condition de se connecter à une infrastructure de lancement appropriée.

Vous pouvez définir des limites pour le CPU, le GPU, la mémoire et d’autres ressources au niveau de la file d&#39;attente ou de l’exécution du job dans la configuration des ressources. Pour savoir comment configurer des files d&#39;attente avec des limites de ressources sur Kubernetes, consultez le [guide de configuration Kubernetes](/fr/platform/launch/setup-launch-kubernetes/).

Pour les Sweeps, incluez le bloc suivant dans la configuration de la file d’attente afin de limiter le nombre de runs simultanés :

```yaml title="queue config"
  scheduler:
    num_workers: 4
```
