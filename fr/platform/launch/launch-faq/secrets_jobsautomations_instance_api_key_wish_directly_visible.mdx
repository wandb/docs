---
title: Peut-on spécifier des secrets pour des jobs/automations ? Par exemple, une clé API que
  vous ne souhaitez pas rendre directement visible aux utilisateurs ?
---

Oui. Suivez ces étapes :

1. Créez un secret Kubernetes dans l’espace de noms désigné pour les runs à l’aide de la commande :
   `kubectl create secret -n <namespace> generic <secret_name> <secret_value>`

2. Après avoir créé le secret, configurez la file d’attente pour l’injecter au démarrage des runs. Seuls les administrateurs du cluster peuvent le voir ; les utilisateurs finaux n’y ont pas accès.
