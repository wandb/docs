---
title: Peut-on spécifier des secrets pour des jobs ou automatisations ? Par exemple, une clé API que
  vous ne souhaitez pas rendre directement visible aux utilisateurs ?
keywords: ["secret Kubernetes", "secret kubectl", "clé API masquée", "injection de secret", "injection d’identifiants"]
---

Oui. Suivez ces étapes :

1. Créez un secret Kubernetes dans l’espace de noms désigné pour les runs :
   `kubectl create secret -n [NAMESPACE] generic [SECRET-NAME] [SECRET-VALUE]`

2. Après avoir créé le secret, configurez la file d’attente pour l’injecter au démarrage des runs. Seuls les administrateurs du cluster peuvent le voir. Les utilisateurs finaux ne peuvent pas le voir.
