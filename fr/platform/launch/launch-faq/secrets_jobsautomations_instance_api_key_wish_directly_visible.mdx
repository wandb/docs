---
title: Peut-on spécifier des secrets pour les jobs/Automations ? Par
  exemple, une clé API que vous ne souhaitez pas rendre directement visible aux utilisateurs ?
---

Oui. Suivez ces étapes :

1. Créez un secret Kubernetes dans l’espace de noms dédié aux runs à l’aide de la commande :
   `kubectl create secret -n <namespace> generic <secret_name> <secret_value>`

2. Après avoir créé le secret, configurez la file d’attente pour qu’elle l’injecte au démarrage des runs. Seuls les administrateurs du cluster peuvent voir le secret ; les utilisateurs finaux n’y ont pas accès.
