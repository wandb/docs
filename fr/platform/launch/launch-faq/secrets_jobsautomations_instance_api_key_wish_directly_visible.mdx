---
title: Peut-on spécifier des secrets pour les jobs/automatisations ? Par
  exemple, une clé API que vous ne souhaitez pas rendre directement visible aux utilisateurs ?
---

Oui. Suivez ces étapes :

1. Créez un secret Kubernetes dans l&#39;espace de noms prévu pour les runs à l&#39;aide de la commande :
   `kubectl create secret -n <namespace> generic <secret_name> <secret_value>`

2. Après avoir créé le secret, configurez la file d&#39;attente pour injecter le secret lorsque les runs démarrent. Seuls les administrateurs du cluster peuvent voir le secret ; les utilisateurs finaux n&#39;y ont pas accès.
