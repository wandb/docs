---
title: 'Tutoriel : Configurer W&B Launch sur SageMaker'
description: "Configurez W&B Launch pour soumettre des tâches d'entraînement à Amazon SageMaker, avec des instructions de configuration pour ECR, S3 et IAM."
---

Vous pouvez utiliser W&amp;B Launch pour soumettre des jobs Launch à Amazon SageMaker afin d’entraîner des modèles de machine learning à l’aide d’algorithmes fournis ou personnalisés sur la plateforme SageMaker. SageMaker se charge de provisionner et de libérer les ressources de calcul, ce qui peut en faire un bon choix pour les équipes qui ne disposent pas d’un cluster EKS.

Les jobs Launch envoyés à une file d’attente W&amp;B Launch connectée à Amazon SageMaker sont exécutés en tant que tâches d&#39;entraînement SageMaker via l’API [CreateTrainingJob](https://docs.aws.amazon.com/sagemaker/latest/APIReference/API_CreateTrainingJob.html). Utilisez la configuration de la file d’attente Launch pour contrôler les arguments envoyés à l’API `CreateTrainingJob`.

Amazon SageMaker [utilise des images Docker pour exécuter des tâches d&#39;entraînement](https://docs.aws.amazon.com/sagemaker/latest/dg/your-algorithms-training-algo-dockerfile.html). Les images récupérées par SageMaker doivent être stockées dans Amazon Elastic Container Registry (ECR). Cela signifie que l’image utilisée pour l’entraînement doit être stockée dans ECR.

<Note>
  Ce guide montre comment exécuter des tâches d&#39;entraînement SageMaker. Pour savoir comment déployer des modèles pour l’inférence sur Amazon SageMaker, voir [cet exemple de job Launch](https://github.com/wandb/launch-jobs/tree/main/jobs/deploy_to_sagemaker_endpoints).
</Note>

<div id="prerequisites">
  ## Prérequis
</div>

Avant de commencer, assurez-vous de remplir les prérequis suivants :

* [Configurez les ressources AWS et recueillez des informations sur les rôles IAM pour S3, ECR et SageMaker.](#set-up-aws-resources)
* [Créez un rôle IAM pour l’agent Launch](#create-an-iam-role-for-launch-agent).

<div id="decide-if-you-want-the-launch-agent-to-build-a-docker-images">
  ### Décidez si vous voulez que l’agent Launch build une image Docker
</div>

Décidez si vous voulez que l’agent Launch de W&amp;B build une image Docker pour vous. Vous avez le choix entre deux options :

* Autoriser l’agent Launch à build une image Docker, à pousser l’image vers Amazon ECR et à soumettre des tâches d’entraînement [SageMaker Training](https://docs.aws.amazon.com/sagemaker/latest/APIReference/API_CreateTrainingJob.html) pour vous. Cette option peut simplifier la tâche des ingénieurs ML qui itèrent rapidement sur du code d’entraînement.
* L’agent Launch utilise une image Docker existante qui contient vos scripts d’entraînement ou d’inférence. Cette option s’intègre bien aux systèmes de CI existants. Si vous choisissez cette option, vous devrez téléverser manuellement votre image Docker vers votre registre de conteneurs sur Amazon ECR.

<div id="set-up-aws-resources">
  ### Configurer les ressources AWS
</div>

Assurez-vous que les ressources AWS suivantes sont configurées dans la région AWS de votre choix :

1. Un [dépôt Amazon ECR](https://docs.aws.amazon.com/AmazonECR/latest/userguide/repository-create.html) pour stocker des images de conteneur.
2. Un ou plusieurs [buckets S3](https://docs.aws.amazon.com/AmazonS3/latest/userguide/create-bucket-overview.html) pour stocker les données d’entrée et de sortie de vos tâches d&#39;entraînement SageMaker.
3. Un rôle IAM pour Amazon SageMaker qui autorise SageMaker à exécuter des tâches d&#39;entraînement et à interagir avec Amazon ECR et Amazon S3.

Notez les ARN de ces ressources. Vous en aurez besoin lorsque vous définirez la [configuration de la file d’attente Launch](#configure-launch-queue-for-sagemaker).

<div id="create-a-iam-policy-for-launch-agent">
  ### Créer une stratégie IAM pour l’agent Launch
</div>

1. Dans l’écran IAM d’AWS, créez une stratégie.
2. Basculez vers l’éditeur JSON de stratégie, puis collez la stratégie suivante selon votre cas d’usage. Remplacez les valeurs entre `<>` par vos propres valeurs :

<Tabs>
  <Tab title="L’agent soumet une image Docker prébuildée">
    ```json
      {
        "Version": "2012-10-17",
        "Statement": [
          {
            "Effect": "Allow",
            "Action": [
              "logs:DescribeLogStreams",
              "SageMaker:AddTags",
              "SageMaker:CreateTrainingJob",
              "SageMaker:DescribeTrainingJob"
            ],
            "Resource": "arn:aws:sagemaker:<region>:<account-id>:*"
          },
          {
            "Effect": "Allow",
            "Action": "iam:PassRole",
            "Resource": "arn:aws:iam::<account-id>:role/<RoleArn-from-queue-config>"
          },
        {
            "Effect": "Allow",
            "Action": "kms:CreateGrant",
            "Resource": "<ARN-OF-KMS-KEY>",
            "Condition": {
              "StringEquals": {
                "kms:ViaService": "SageMaker.<region>.amazonaws.com",
                "kms:GrantIsForAWSResource": "true"
              }
            }
          }
        ]
      }
    ```
  </Tab>

  <Tab title="L’agent build et soumet une image Docker">
    ```json
      {
        "Version": "2012-10-17",
        "Statement": [
          {
            "Effect": "Allow",
            "Action": [
              "logs:DescribeLogStreams",
              "SageMaker:AddTags",
              "SageMaker:CreateTrainingJob",
              "SageMaker:DescribeTrainingJob"
            ],
            "Resource": "arn:aws:sagemaker:<region>:<account-id>:*"
          },
          {
            "Effect": "Allow",
            "Action": "iam:PassRole",
            "Resource": "arn:aws:iam::<account-id>:role/<RoleArn-from-queue-config>"
          },
           {
          "Effect": "Allow",
          "Action": [
            "ecr:CreateRepository",
            "ecr:UploadLayerPart",
            "ecr:PutImage",
            "ecr:CompleteLayerUpload",
            "ecr:InitiateLayerUpload",
            "ecr:DescribeRepositories",
            "ecr:DescribeImages",
            "ecr:BatchCheckLayerAvailability",
            "ecr:BatchDeleteImage"
          ],
          "Resource": "arn:aws:ecr:<region>:<account-id>:repository/<repository>"
        },
        {
          "Effect": "Allow",
          "Action": "ecr:GetAuthorizationToken",
          "Resource": "*"
        },
        {
            "Effect": "Allow",
            "Action": "kms:CreateGrant",
            "Resource": "<ARN-OF-KMS-KEY>",
            "Condition": {
              "StringEquals": {
                "kms:ViaService": "SageMaker.<region>.amazonaws.com",
                "kms:GrantIsForAWSResource": "true"
              }
            }
          }
        ]
      }
    ```
  </Tab>
</Tabs>

3. Cliquez sur **Next**.
4. Attribuez un nom et une description à la stratégie.
5. Cliquez sur **Créer une stratégie**.

<div id="create-an-iam-role-for-launch-agent">
  ### Créer un rôle IAM pour l&#39;agent Launch
</div>

L&#39;agent Launch doit disposer de l&#39;autorisation nécessaire pour créer des tâches d&#39;entraînement Amazon SageMaker. Suivez la procédure ci-dessous pour créer un rôle IAM :

1. Dans l&#39;écran IAM d&#39;AWS, créez un nouveau rôle.
2. Pour **Entité approuvée**, sélectionnez **AWS Account** (ou une autre option conforme aux politiques de votre organisation).
3. Faites défiler l&#39;écran des autorisations et sélectionnez le nom de la stratégie que vous venez de créer ci-dessus.
4. Donnez un nom et une description au rôle.
5. Sélectionnez **Créer un rôle**.
6. Notez l&#39;ARN du rôle. Vous indiquerez cet ARN lorsque vous configurerez l&#39;agent Launch.

Pour créer des rôles IAM, voir la [documentation AWS Identity and Access Management](https://docs.aws.amazon.com/IAM/latest/UserGuide/introduction.html).

<Note>
  * Si vous voulez que l&#39;agent Launch effectue des builds d&#39;images, voir [Configuration avancée de l&#39;agent](./setup-agent-advanced) pour connaître les autorisations supplémentaires requises.
  * L&#39;autorisation `kms:CreateGrant` pour les files d&#39;attente SageMaker n&#39;est requise que si le `ResourceConfig` associé spécifie un `VolumeKmsKeyId` et que le rôle associé ne dispose pas d&#39;une stratégie autorisant cette action.
</Note>

<div id="configure-launch-queue-for-sagemaker">
  ## Configurer la file d&#39;attente Launch pour SageMaker
</div>

Ensuite, créez une file d&#39;attente dans l&#39;application W&amp;B qui utilise SageMaker comme sa ressource de calcul :

1. Accédez à la [Launch App](https://wandb.ai/launch).
2. Cliquez sur le bouton **Créer une file d&#39;attente**.
3. Sélectionnez l&#39;**entité** dans laquelle vous souhaitez créer la file d&#39;attente.
4. Saisissez un nom pour votre file d&#39;attente dans le champ **Nom**.
5. Sélectionnez **SageMaker** comme **Ressource**.
6. Dans le champ **Configuration**, fournissez les informations relatives à votre job SageMaker. Par défaut, W&amp;B préremplit le corps de requête `CreateTrainingJob` au format YAML et JSON :
   ```json
   {
     "RoleArn": "<REQUIRED>", 
     "ResourceConfig": {
         "InstanceType": "ml.m4.xlarge",
         "InstanceCount": 1,
         "VolumeSizeInGB": 2
     },
     "OutputDataConfig": {
         "S3OutputPath": "<REQUIRED>"
     },
     "StoppingCondition": {
         "MaxRuntimeInSeconds": 3600
     }
   }
   ```

Vous devez au minimum spécifier :

* `RoleArn` : ARN du rôle IAM d&#39;exécution SageMaker (voir [prérequis](#prerequisites)). À ne pas confondre avec le rôle IAM de l&#39;**agent** Launch.
* `OutputDataConfig.S3OutputPath` : URI Amazon S3 indiquant où les sorties de SageMaker seront stockées.
* `ResourceConfig` : spécification requise de la configuration des ressources. Les options pour la configuration des ressources sont décrites [ici](https://docs.aws.amazon.com/sagemaker/latest/APIReference/API_ResourceConfig.html).
* `StoppingCondition` : spécification requise des conditions d&#39;arrêt pour la tâche d&#39;entraînement. Les options pour les conditions d&#39;arrêt sont décrites [ici](https://docs.aws.amazon.com/sagemaker/latest/APIReference/API_StoppingCondition.html).

7. Cliquez sur le bouton **Créer une file d&#39;attente** .

<div id="set-up-the-launch-agent">
  ## Configurer l&#39;agent Launch
</div>

La section suivante décrit où vous pouvez déployer votre agent et comment le configurer en fonction de son emplacement de déploiement.

Il existe [plusieurs options de déploiement pour l&#39;agent Launch d&#39;une file d&#39;attente](#decide-where-to-run-the-launch-agent) Amazon SageMaker : sur une machine locale, sur une instance EC2 ou dans un cluster EKS. [Configurez votre agent Launch de manière appropriée](#configure-a-launch-agent) en fonction de l&#39;endroit où vous le déployez.

<div id="decide-where-to-run-the-launch-agent">
  ### Choisissez où exécuter l&#39;agent Launch
</div>

Pour les charges de travail de Production et pour les clients qui disposent déjà d&#39;un cluster EKS, W&amp;B recommande de déployer l&#39;agent Launch sur le cluster EKS à l&#39;aide de ce chart Helm.

Pour les charges de travail de Production sans cluster EKS existant, une instance EC2 est une bonne option. Bien que l&#39;instance de l&#39;agent Launch reste en cours d&#39;exécution en permanence, l&#39;agent n&#39;a pas besoin de plus d&#39;une instance EC2 de type `t2.micro`, ce qui reste relativement abordable.

Pour des cas d&#39;usage expérimentaux ou individuels, exécuter l&#39;agent Launch sur votre machine locale peut être un moyen rapide de démarrer.

Selon votre cas d&#39;usage, suivez les instructions fournies dans les onglets suivants pour configurer correctement votre agent Launch :

<Tabs>
  <Tab title="EKS">
    W&amp;B vous encourage fortement à utiliser le [chart Helm géré par W&amp;B](https://github.com/wandb/helm-charts/tree/main/charts/launch-agent) pour installer l&#39;agent dans un cluster EKS.
  </Tab>

  <Tab title="EC2">
    Accédez au tableau de bord Amazon EC2 et suivez les étapes suivantes :

    1. Cliquez sur **Launch instance**.
    2. Indiquez un nom dans le champ **Name**. Ajoutez éventuellement un tag.
    3. Dans **Instance type**, sélectionnez un type d&#39;instance pour votre instance EC2. Vous n&#39;avez pas besoin de plus de 1vCPU et 1GiB de Mémoire (par exemple, une t2.micro).
    4. Créez une paire de clés pour votre organisation dans le champ **Key pair (login)**. Vous utiliserez cette paire de clés pour [vous connecter à votre instance EC2](https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/connect.html) avec un client SSH à une étape ultérieure.
    5. Dans **Network settings**, sélectionnez un groupe de sécurité approprié pour votre organisation.
    6. Développez **Advanced details**. Pour **IAM instance profile**, sélectionnez le rôle IAM de l&#39;agent Launch que vous avez créé ci-dessus.
    7. Vérifiez le champ **Summary**. Si tout est correct, sélectionnez **Launch instance**.

    Accédez à **Instances** dans le panneau de gauche du tableau de bord EC2 sur AWS. Assurez-vous que l&#39;instance EC2 que vous avez créée est en cours d&#39;exécution (voir la colonne **Instance state**). Une fois que vous avez confirmé que votre instance EC2 est en cours d&#39;exécution, ouvrez le terminal de votre machine locale et effectuez les opérations suivantes :

    1. Sélectionnez **Connect**.
    2. Sélectionnez l&#39;onglet **SSH client** et suivez les instructions indiquées pour vous connecter à votre instance EC2.
    3. Dans votre instance EC2, installez les packages suivants :
       ```bash
       sudo yum install python311 -y && python3 -m ensurepip --upgrade && pip3 install wandb && pip3 install wandb[launch]
       ```
    4. Ensuite, installez et démarrez Docker sur votre instance EC2 :
       ```bash
       sudo yum update -y && \
       sudo yum install -y docker python3 && \
       sudo systemctl start docker && \
       sudo systemctl enable docker && \
       sudo usermod -a -G docker ec2-user

       newgrp docker
       ```

    Vous pouvez maintenant passer à la configuration de l&#39;agent Launch.
  </Tab>

  <Tab title="Local machine">
    Utilisez les fichiers de configuration AWS situés dans `~/.aws/config` et `~/.aws/credentials` pour associer un rôle à un agent exécuté sur une machine locale. Fournissez l&#39;ARN du rôle IAM que vous avez créé pour l&#39;agent Launch à l&#39;étape précédente.

    ```yaml title="~/.aws/config"
    [profile SageMaker-agent]
    role_arn = arn:aws:iam::<account-id>:role/<agent-role-name>
    source_profile = default                                                                   
    ```

    ```yaml title="~/.aws/credentials"
    [default]
    aws_access_key_id=<access-key-id>
    aws_secret_access_key=<secret-access-key>
    aws_session_token=<session-token>
    ```

    Notez que les jetons de session ont une [durée maximale](https://docs.aws.amazon.com/cli/latest/reference/sts/get-session-token.html#description) de 1 heure ou 3 jours selon l&#39;identité à laquelle ils sont associés.
  </Tab>
</Tabs>

<div id="configure-a-launch-agent">
  ### Configurer un agent Launch
</div>

Configurez l’agent Launch à l’aide d’un fichier de configuration YAML nommé `launch-config.yaml`.

Par défaut, W&amp;B recherche le fichier de configuration dans `~/.config/wandb/launch-config.yaml`. Vous pouvez également spécifier un autre répertoire lorsque vous activez l’agent Launch avec l’option `-c`.

L’extrait YAML suivant montre comment spécifier les options principales de configuration de l’agent :

```yaml title="launch-config.yaml"
max_jobs: -1
queues:
  - <queue-name>
environment:
  type: aws
  region: <your-region>
registry:
  type: ecr
  uri: <ecr-repo-arn>
builder: 
  type: docker

```

Lancez maintenant l’agent avec `wandb launch-agent`
