Créez un job d&#39;ajustement d&#39;hyperparamètres ([sweeps](/fr/models/sweeps/)) avec W&amp;B Launch. Avec les balayages dans Launch, un planificateur de balayage est envoyé vers une file d’attente Launch avec les hyperparamètres spécifiés à explorer. Le planificateur de balayage démarre lorsqu’il est pris en charge par l’agent, puis lance des runs de balayage dans la même file d’attente avec les hyperparamètres sélectionnés. Ce processus se poursuit jusqu’à la fin du balayage ou jusqu’à son arrêt.

Vous pouvez utiliser le moteur de planification W&amp;B Sweep par défaut ou implémenter votre propre planificateur personnalisé :

1. Planificateur de balayage standard : utilisez le moteur de planification W&amp;B Sweep par défaut qui pilote [W&amp;B Sweeps](/fr/models/sweeps/). Les méthodes classiques `bayes`, `grid` et `random` sont disponibles.
2. Planificateur de balayage personnalisé : configurez le planificateur de balayage pour qu’il s’exécute en tant que job. Cette option permet une personnalisation complète. Un exemple montrant comment étendre le planificateur de balayage standard pour inclure davantage de journalisation figure dans la section ci-dessous.

<Note>
  Ce guide suppose que W&amp;B Launch a déjà été configuré. Si W&amp;B Launch n’est pas configuré, voir la section [comment démarrer](./#how-to-get-started) de la documentation Launch.
</Note>

<Note>
  Nous vous recommandons de créer un balayage dans Launch à l’aide de la méthode « de base » si vous utilisez les balayages dans Launch pour la première fois. Utilisez un planificateur personnalisé pour les balayages dans Launch lorsque le moteur de planification W&amp;B standard ne répond pas à vos besoins.
</Note>

<div id="create-a-sweep-with-a-wb-standard-scheduler">
  ## Créer un balayage avec un planificateur W&amp;B standard
</div>

Créez des Balayages W&amp;B avec Launch. Vous pouvez créer un balayage de manière interactive dans la W&amp;B App ou par programmation avec la CLI W&amp;B. Pour les configurations avancées des balayages Launch, notamment pour personnaliser le planificateur, utilisez la CLI.

<Note>
  Avant de créer un balayage avec W&amp;B Launch, assurez-vous d’abord d’avoir créé un job sur lequel effectuer le balayage. Voir la page [Create a Job](/fr/platform/launch/create-launch-job/) pour plus d’informations.
</Note>

<Tabs>
  <Tab title="W&B app">
    Créez un balayage de manière interactive dans la W&amp;B App.

    1. Accédez à votre projet W&amp;B dans la W&amp;B App.
    2. Sélectionnez l’icône des balayages dans la barre latérale du projet (icône en forme de balai).
    3. Ensuite, sélectionnez le bouton **Create Sweep**.
    4. Cliquez sur le bouton **Configure Launch**.
    5. Dans le menu déroulant **Job**, sélectionnez le nom de votre job et la version du job à partir desquels vous souhaitez créer un balayage.
    6. Sélectionnez une file d&#39;attente sur laquelle exécuter le balayage à l’aide du menu déroulant **Queue**.
    7. Utilisez le menu déroulant **Job Priority** pour définir la priorité de votre job Launch. La priorité d’un job Launch est définie sur « Medium » si la file d’attente Launch ne prend pas en charge la priorisation.
    8. (Facultatif) Configurez des arguments de substitution pour le run ou le planificateur de balayage. Par exemple, utilisez les substitutions du planificateur pour configurer le nombre de runs simultanés qu’il gère avec `num_workers`.
    9. (Facultatif) Sélectionnez un projet dans lequel enregistrer le balayage à l’aide du menu déroulant **Destination Project**.
    10. Cliquez sur **Save**
    11. Sélectionnez **Launch Sweep**.

    <Frame>
      <img src="/images/launch/create_sweep_with_launch.png" alt="Configuration d’un balayage Launch" />
    </Frame>
  </Tab>

  <Tab title="CLI">
    Créez un Balayage W&amp;B avec Launch par programmation à l’aide de la CLI W&amp;B.

    1. Créez une configuration de Balayage
    2. Spécifiez le nom complet du job dans votre configuration de Balayage
    3. Initialisez un agent de balayage.

    <Note>
      Les étapes 1 et 3 sont les mêmes que celles que vous suivez habituellement pour créer un Balayage W&amp;B.
    </Note>

    Par exemple, dans l’extrait de code suivant, nous indiquons `'wandb/jobs/Hello World 2:latest'` comme valeur pour le job :

    ```yaml
    # launch-sweep-config.yaml

    job: 'wandb/jobs/Hello World 2:latest'
    description: exemples de balayage utilisant des jobs Launch

    method: bayes
    metric:
      goal: minimize
      name: loss_metric
    parameters:
      learning_rate:
        max: 0.02
        min: 0
        distribution: uniform
      epochs:
        max: 20
        min: 0
        distribution: int_uniform

    # Paramètres facultatifs du planificateur :

    # scheduler:
    #   num_workers: 1  # runs de balayage simultanés
    #   docker_image: <image de base pour le planificateur>
    #   resource: <par ex. local-container...>
    #   resource_args:  # arguments de ressource transmis aux runs
    #     env: 
    #         - WANDB_API_KEY

    # Paramètres Launch facultatifs
    # launch: 
    #    registry: <registre pour la récupération d’image>
    ```

    Pour savoir comment créer une configuration de balayage, voir la page [Define sweep configuration](/fr/platform/launch/sweeps-on-launch/).

    4. Ensuite, initialisez un balayage. Indiquez le chemin vers votre fichier de configuration, le nom de votre file d’attente de jobs, votre entité W&amp;B et le nom du projet.

    ```bash
    wandb launch-sweep <path/to/yaml/file> --queue <queue_name> --entity <your_entity>  --project <project_name>
    ```

    Pour plus d’informations sur les Balayages W&amp;B, voir le chapitre [Tune Hyperparameters](/fr/models/sweeps/).
  </Tab>
</Tabs>

<div id="create-a-custom-sweep-scheduler">
  ## Créer un planificateur de balayage personnalisé
</div>

Créez un planificateur de balayage personnalisé, soit avec le planificateur W&amp;B, soit avec un planificateur personnalisé.

<Note>
  L’utilisation des jobs de planificateur nécessite la version &gt;= `0.15.4` de la CLI wandb
</Note>

<Tabs>
  <Tab title="planificateur W&B">
    Créez un balayage Launch à l’aide de la logique d’ordonnancement des balayages de W&amp;B sous forme de job.

    1. Identifiez le job d’ordonnancement wandb dans le projet public wandb/sweep-jobs, ou utilisez le nom de job :
       `'wandb/sweep-jobs/job-wandb-sweep-scheduler:latest'`
    2. Créez un fichier YAML de configuration avec un bloc `scheduler` supplémentaire contenant une clé `job` qui pointe vers ce nom, comme dans l’exemple ci-dessous.
    3. Utilisez la commande `wandb launch-sweep` avec la nouvelle configuration.

    Exemple de configuration :

    ```yaml
    # launch-sweep-config.yaml  
    description: Configuration de balayage Launch utilisant un job planificateur
    scheduler:
      job: wandb/sweep-jobs/job-wandb-sweep-scheduler:latest
      num_workers: 8  # autorise 8 runs de balayage simultanés

    # job d'entraînement/réglage que les runs de balayage exécuteront
    job: wandb/sweep-jobs/job-fashion-MNIST-train:latest
    method: grid
    parameters:
      learning_rate:
        min: 0.0001
        max: 0.1
    ```
  </Tab>

  <Tab title="Ordonnanceur personnalisé">
    Des ordonnanceurs personnalisés peuvent être créés en créant un job d’ordonnanceur. Dans ce guide, nous allons modifier `WandbScheduler` afin d’ajouter une journalisation plus détaillée.

    1. Clonez le dépôt `wandb/launch-jobs` (plus précisément : `wandb/launch-jobs/jobs/sweep_schedulers`)
    2. Vous pouvez maintenant modifier `wandb_scheduler.py` pour obtenir le niveau de journalisation souhaité. Exemple : ajoutez de la journalisation à la fonction `_poll`. Elle est appelée une fois par cycle d’interrogation (intervalle configurable), avant de lancer de nouveaux runs de balayage.
    3. Exécutez le fichier modifié pour créer un job, avec : `python wandb_scheduler.py --project <project> --entity <entity> --name CustomWandbScheduler`
    4. Identifiez le nom du job créé, soit dans l’interface utilisateur, soit dans la sortie de l’appel précédent ; il s’agira d’un job code-artifact (sauf indication contraire).
    5. Créez maintenant une configuration de balayage dans laquelle le scheduler pointe vers votre nouveau job.

    ```yaml
    ...
    scheduler:
      job: '<entity>/<project>/job-CustomWandbScheduler:latest'
    ...
    ```
  </Tab>

  <Tab title="Ordonnanceur Optuna">
    Optuna est un framework d&#39;optimisation d&#39;hyperparamètres qui utilise différents algorithmes pour trouver les meilleurs hyperparamètres pour un modèle donné (à l&#39;instar de W&amp;B). En plus des [algorithmes d&#39;échantillonnage](https://optuna.readthedocs.io/en/stable/reference/samplers/index.html), Optuna fournit également différents [algorithmes d&#39;élagage](https://optuna.readthedocs.io/en/stable/reference/pruners.html) qui permettent d&#39;interrompre prématurément les runs peu performants. Cela est particulièrement utile lorsque vous exécutez un grand nombre de runs, car cela peut vous faire gagner du temps et économiser des ressources. Les classes sont très configurables : il vous suffit de transmettre les paramètres attendus dans le bloc `scheduler.settings.pruner/sampler.args` du fichier de configuration.

    Créez un balayage Launch en utilisant la logique d&#39;ordonnancement d&#39;Optuna avec un job.

    1. Commencez par créer votre propre job ou utilisez un job basé sur une image de planificateur Optuna préconstruite.
       * Voir le dépôt [`wandb/launch-jobs`](https://github.com/wandb/launch-jobs/blob/main/jobs/sweep_schedulers) pour des exemples montrant comment créer votre propre job.
       * Pour utiliser une image Optuna préconstruite, vous pouvez soit accéder à `job-optuna-sweep-scheduler` dans le projet `wandb/sweep-jobs`, soit utiliser le nom de job : `wandb/sweep-jobs/job-optuna-sweep-scheduler:latest`.

    2. Après avoir créé un job, vous pouvez maintenant créer un balayage. Créez une configuration de balayage qui inclut un bloc `scheduler` avec une clé `job` pointant vers le job du planificateur Optuna (exemple ci-dessous).

    ```yaml
      # optuna_config_basic.yaml
      description: Un planificateur Optuna de base
      job: wandb/sweep-jobs/job-fashion-MNIST-train:latest
      run_cap: 5
      metric:
        name: epoch/val_loss
        goal: minimize

      scheduler:
        job: wandb/sweep-jobs/job-optuna-sweep-scheduler:latest
        resource: local-container  # requis pour les jobs de planificateur provenant d'images
        num_workers: 2

        # paramètres spécifiques à Optuna
        settings:
          pruner:
            type: PercentilePruner
            args:
              percentile: 25.0  # arrête 75 % des runs
              n_warmup_steps: 10  # élagage désactivé pour les x premières étapes

      parameters:
        learning_rate:
          min: 0.0001
          max: 0.1
    ```

    3. Enfin, lancez le balayage dans une file d’attente active avec la commande launch-sweep :

    ```bash
    wandb launch-sweep <config.yaml> -q <queue> -p <project> -e <entity>
    ```

    Pour l’implémentation exacte du job de planificateur de balayage Optuna, voir [wandb/launch-jobs](https://github.com/wandb/launch-jobs/blob/main/jobs/sweep_schedulers/optuna_scheduler/optuna_scheduler.py). Pour d’autres exemples de ce qu’il est possible de faire avec le planificateur Optuna, consultez [wandb/examples](https://github.com/wandb/examples/tree/master/examples/launch/launch-sweeps/optuna-scheduler).
  </Tab>
</Tabs>

Des exemples de ce qu&#39;il est possible de faire avec des jobs de planificateur de balayage personnalisés sont disponibles dans le dépôt [wandb/launch-jobs](https://github.com/wandb/launch-jobs), sous `jobs/sweep_schedulers`. Ce guide montre comment utiliser le **Wandb Scheduler Job** disponible publiquement et présente également un processus de création de jobs de planificateur de balayage personnalisés.

<div id="how-to-resume-sweeps-on-launch">
  ## Comment reprendre des Balayages dans Launch
</div>

Il est également possible de reprendre un launch-sweep à partir d’un balayage précédemment lancé. Bien que les hyperparamètres et la tâche d&#39;entraînement ne puissent pas être modifiés, vous pouvez modifier les paramètres propres au planificateur, ainsi que la file d&#39;attente vers laquelle le balayage est envoyé.

<Note>
  Si le balayage initial utilisait une tâche d&#39;entraînement avec un alias tel que &#39;latest&#39;, la reprise peut produire des résultats différents si la dernière version du job a été modifiée depuis le dernier run.
</Note>

1. Identifiez le nom ou l’ID d’un launch-sweep déjà exécuté. L’ID du balayage est une chaîne de huit caractères (par exemple, `hhd16935`) que vous pouvez trouver dans votre projet dans l’application W&amp;B.
2. Si vous modifiez les paramètres du planificateur, créez un fichier de configuration mis à jour.
3. Dans votre terminal, exécutez la commande suivante. Remplacez le contenu entre `<` et `>` par vos informations :

```bash
wandb launch-sweep <optional config.yaml> --resume_id <sweep id> --queue <queue_name>
```
