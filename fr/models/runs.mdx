Un *run* est une unité de calcul unique journalisée par W&amp;B. Vous pouvez voir un Run W&amp;B comme l&#39;élément atomique de l&#39;ensemble de votre projet. En d&#39;autres termes, chaque run est l&#39;enregistrement d&#39;un calcul précis, par exemple l&#39;entraînement d&#39;un modèle et la journalisation des résultats, des balayages d&#39;hyperparamètres, etc.

Les cas d&#39;utilisation courants de l&#39;initialisation d&#39;un run et de la journalisation d&#39;informations dans celui-ci incluent :

* Entraîner un modèle et [journaliser des métriques](/fr/models/ref/python/experiments/run#method-run-log) telles que la précision et la perte
* Effectuer un [réglage des hyperparamètres](/fr/models/sweeps/) et lancer de nouvelles expériences
* Mener une nouvelle expérience de machine learning avec un modèle différent
* Suivre et enregistrer des jeux de données et des modèles en tant que [W&amp;B Artifacts](/fr/models/artifacts/)
* [Télécharger et utiliser](/fr/models/artifacts/download-and-use-an-artifact/) des jeux de données ou des modèles utilisés par d&#39;autres membres de votre équipe en tant que W&amp;B Artifacts

Pour initialiser un run W&amp;B, appelez la méthode [`wandb.init()`](/fr/models/ref/python/functions/init) dans le SDK Python de W&amp;B. Cela démarre un nouveau run et renvoie un objet `wandb.Run` que vous pouvez utiliser pour journaliser des métriques, des artefacts et d&#39;autres informations dans le run. Pour plus d&#39;informations sur l&#39;initialisation d&#39;un run, voir [Initialiser les runs](/fr/models/runs/initialize-run).

Chaque objet run possède un [identifiant unique appelé *run ID*](/fr/models/runs/run-identifiers#unique-run-identifiers). [Vous pouvez spécifier un ID unique](/fr/models/runs/run-identifiers#unique-run-identifiers) ou laisser [W&amp;B en générer un aléatoirement pour vous](/fr/models/runs/run-identifiers#autogenerated-run-ids). Chaque objet run possède également un [nom de l&#39;exécution](/fr/models/runs/run-identifiers#run-name), lisible par l&#39;humain mais non unique. Vous pouvez spécifier un nom pour votre run ou laisser W&amp;B en générer un aléatoirement pour vous. Vous pouvez renommer un run après l&#39;avoir initialisé.

W&amp;B journalise votre run dans un [*projet*](/fr/models/track/project-page/). Vous spécifiez le projet lorsque vous initialisez le run avec `wandb.init(project="")`. W&amp;B crée un nouveau projet si le projet n&#39;existe pas. Si le projet existe déjà, W&amp;B journalise le run dans le projet que vous avez spécifié.

<Note>
  Si vous ne spécifiez pas de nom de projet, W&amp;B stocke le run dans un projet appelé `Uncategorized`.
</Note>

`wandb.init()` renvoie un objet `wandb.Run` qui contient les propriétés du run, telles que son ID, son nom, sa configuration et son état. Utilisez l&#39;objet run pour journaliser des métriques, des artefacts et d&#39;autres informations dans le run avec des méthodes telles que `wandb.Run.log()`, `wandb.Run.log_code()` et `wandb.Run.use_artifact()`.

Chaque run possède un état qui décrit son statut actuel. Voir [États des runs](/fr/models/runs/run-states) pour une liste complète des états possibles d&#39;un run.

[Consultez les runs et leurs propriétés](/fr/models/runs/view-logged-runs) dans l&#39;espace de travail du projet du run dans l&#39;application W&amp;B. Vous pouvez également accéder par programmation aux propriétés du run avec l&#39;objet [`wandb.Api.Run`](/fr/models/ref/python/experiments/run).

À titre d&#39;exemple, considérez l&#39;extrait de code suivant, qui initialise un run W&amp;B et y consigne quelques métriques :

<Note>
  Renseignez votre entité W&amp;B dans la variable `entity` des extraits de code ci-dessous si vous souhaitez suivre l&#39;exemple. Votre entité correspond à votre nom d&#39;utilisateur W&amp;B ou au nom de votre équipe. Vous pouvez la trouver dans l&#39;URL de votre espace de travail dans l&#39;application W&amp;B. Par exemple, si l&#39;URL de votre espace de travail est `https://wandb.ai/nico/awesome-project`, alors votre entité est `nico`.
</Note>

```python
import wandb

entity = "nico"  # Remplacez par votre entité W&B
project = "awesome-project"

with wandb.init(entity=entity, project=project) as run:
    run.log({"accuracy": 0.9, "loss": 0.1})
```

La première ligne importe le SDK Python de W&amp;B. La deuxième ligne initialise un run dans le projet `awesome-project` sous l’entité `nico`. La troisième ligne journalise la précision et la perte du modèle pour ce run.

Journaliser une métrique à un instant donné n’est pas forcément très utile. Un exemple plus réaliste, dans le cadre de l’entraînement de modèles discriminatifs, consiste à journaliser des métriques à intervalles réguliers. Par exemple, prenez l’extrait de code suivant :

```python
import wandb
import random

config = {
    "epochs": 10,
    "learning_rate": 0.01,
}

with wandb.init(project="awesome-project", config=config) as run:
    print(f"lr: {config['learning_rate']}")
      
    # Simulation d’un run d’entraînement
    for epoch in range(config['epochs']):
      offset = random.random() / 5
      acc = 1 - 2**-epoch - random.random() / (epoch + 1) - offset
      loss = 2**-epoch + random.random() / (epoch + 1) + offset
      print(f"epoch={epoch}, accuracy={acc}, loss={loss}")
      run.log({"accuracy": acc, "loss": loss})
```

Le script d’entraînement appelle `wandb.Run.log()` 10 fois. Chaque fois qu’il appelle `wandb.Run.log()`, W&amp;B journalise la précision et la perte pour cette époque.
