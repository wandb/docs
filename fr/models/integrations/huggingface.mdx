<div id="train-the-model">
  ## Entraîner le modèle
</div>

Ensuite, appelez le script d&#39;entraînement téléchargé [run&#95;glue.py](https://huggingface.co/transformers/examples.html#glue) et voyez l&#39;entraînement être automatiquement suivi dans le tableau de bord W&amp;B. Ce script affine BERT sur le Microsoft Research Paraphrase Corpus — des paires de phrases accompagnées d&#39;annotations humaines indiquant si elles sont sémantiquement équivalentes.

```python
%env WANDB_PROJECT=huggingface-demo
%env TASK_NAME=MRPC

!python run_glue.py \
  --model_name_or_path bert-base-uncased \
  --task_name $TASK_NAME \
  --do_train \
  --do_eval \
  --max_seq_length 256 \
  --per_device_train_batch_size 32 \
  --learning_rate 2e-4 \
  --num_train_epochs 3 \
  --output_dir /tmp/$TASK_NAME/ \
  --overwrite_output_dir \
  --logging_steps 50
```

<div id="track-key-information-effortlessly-by-default">
  ## Suivez automatiquement les informations clés, sans effort
</div>

W&amp;B enregistre un nouveau run pour chaque expérience. Voici les informations enregistrées par défaut :

* **Hyperparamètres** : les paramètres de votre modèle sont enregistrés dans Config
* **Métriques du modèle** : les données de séries temporelles des métriques en flux sont enregistrées dans Journal
* **Journaux du terminal** : les sorties de la ligne de commande sont enregistrées et accessibles dans un onglet
* **Métriques système** : utilisation du GPU et du CPU, mémoire, température, etc.
