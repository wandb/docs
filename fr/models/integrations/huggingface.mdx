Ce tutoriel vous montre comment utiliser l’intégration W&amp;B avec [Hugging Face Transformers](https://github.com/huggingface/transformers) pour suivre automatiquement les métriques d’entraînement et d’évaluation, les hyperparamètres et les statistiques système lors du fine-tuning d’un modèle. En suivant ce tutoriel, vous apprendrez à visualiser les performances de votre modèle dans le tableau de bord [W&amp;B](https://wandb.ai/site) afin de comparer les expériences et d’itérer sur vos modèles en toute confiance.

Vous pouvez comparer les hyperparamètres, les métriques de sortie et les statistiques système, comme l’utilisation du GPU, entre vos modèles.

</div>

* **Tableau de bord unifié** : dépôt central pour toutes les métriques et prédictions de votre modèle.

* **Accessible** : gratuit pour les particuliers et les équipes universitaires.

W&amp;B fonctionne comme GitHub pour les modèles de machine learning. Enregistrez vos expériences de machine learning dans votre tableau de bord privé hébergé. Expérimentez en toute confiance : W&amp;B enregistre toutes les versions de vos modèles, quel que soit l’endroit où vous exécutez vos scripts.

Les intégrations légères de W&amp;B fonctionnent avec n’importe quel script Python. Inscrivez-vous pour obtenir un compte W&amp;B gratuit et commencer à suivre et à visualiser vos modèles.

Dans le dépôt Hugging Face Transformers, W&amp;B a instrumenté le Trainer pour journaliser automatiquement les métriques d’entraînement et d’évaluation vers W&amp;B à chaque étape de journalisation.

Voici un aperçu détaillé du fonctionnement de l’intégration : [Hugging Face + W&amp;B Report](https://app.wandb.ai/jxmorris12/huggingface-demo/reports/Train-a-model-with-Hugging-Face-and-Weights-%26-Biases--VmlldzoxMDE2MTU).

<div id="train-the-model">
  ## Entraîner le modèle
</div>

Une fois l’environnement configuré et l’authentification terminée, vous êtes prêt à démarrer un run d’entraînement. Ensuite, exécutez le script d’entraînement téléchargé [`run_glue.py`](https://huggingface.co/transformers/examples.html#glue) et constatez que l’entraînement est automatiquement suivi dans le tableau de bord W&amp;B. Ce script effectue le fine-tuning de BERT sur le Microsoft Research Paraphrase Corpus — des paires de phrases avec des annotations humaines indiquant si elles sont sémantiquement équivalentes.

```python
%env WANDB_PROJECT=huggingface-demo
%env TASK_NAME=MRPC

!python run_glue.py \
  --model_name_or_path bert-base-uncased \
  --task_name $TASK_NAME \
  --do_train \
  --do_eval \
  --max_seq_length 256 \
  --per_device_train_batch_size 32 \
  --learning_rate 2e-4 \
  --num_train_epochs 3 \
  --output_dir /tmp/$TASK_NAME/ \
  --overwrite_output_dir \
  --logging_steps 50
```

<div id="track-key-information-by-default">
  ## Suivre automatiquement les informations clés
</div>

Cette section décrit ce que W&amp;B capture automatiquement afin que vous sachiez quelles données sont disponibles dans votre tableau de bord sans configuration supplémentaire. W&amp;B enregistre un nouveau run pour chaque expérience. Voici les informations enregistrées par défaut :

* **Hyperparamètres** : W&amp;B enregistre les paramètres de votre modèle dans Config.
* **Métriques du modèle** : W&amp;B enregistre les données de séries temporelles des métriques envoyées à Log.
* **Journaux du terminal** : W&amp;B enregistre les sorties de la ligne de commande et les affiche dans un onglet.
* **Métriques système** : utilisation du GPU et du CPU, mémoire et température.
