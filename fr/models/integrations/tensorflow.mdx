Si vous utilisez déjà TensorBoard, l’intégration avec wandb est simple.

```python
import tensorflow as tf
import wandb
```


<div id="log-custom-metrics">
  ## Journaliser des métriques personnalisées
</div>

Si vous devez journaliser des métriques personnalisées supplémentaires qui ne sont pas journalisées dans TensorBoard, vous pouvez appeler `run.log()` dans votre code `run.log({"custom": 0.8}) `

La définition de l’argument étape dans `run.log()` est désactivée lors de la synchronisation avec TensorBoard. Si vous souhaitez définir un nombre d’étapes différent, vous pouvez journaliser les métriques avec une métrique d’étape comme suit :

```python
with wandb.init(config=tf.flags.FLAGS, sync_tensorboard=True) as run:
    run.log({"custom": 0.8, "global_step":global_step}, step=global_step)
```

<div id="tensorflow-estimators-hook">
  ## Hook pour les estimateurs TensorFlow
</div>

Si vous souhaitez mieux contrôler ce qui est journalisé, wandb propose également un hook pour les estimateurs TensorFlow. Il journalise toutes les valeurs `tf.summary` du graphe.

```python
import tensorflow as tf
import wandb

run = wandb.init(config=tf.FLAGS)

estimator.train(hooks=[wandb.tensorflow.WandbHook(steps_per_log=1000)])
run.finish()
```

<div id="log-manually">
  ## Journaliser manuellement
</div>

Le moyen le plus simple de journaliser des métriques dans TensorFlow consiste à utiliser `tf.summary` avec le journaliseur TensorFlow :

```python
import wandb
run = wandb.init(config=tf.flags.FLAGS, sync_tensorboard=True)
with tf.Session() as sess:
    # ...
    wandb.tensorflow.log(tf.summary.merge_all())
```

Avec TensorFlow 2, la méthode recommandée pour entraîner un modèle avec une boucle personnalisée consiste à utiliser `tf.GradientTape`. Pour en savoir plus, consultez le [guide pas à pas de TensorFlow sur l&#39;entraînement personnalisé](https://www.tensorflow.org/tutorials/customization/custom_training_walkthrough). Si vous souhaitez intégrer `wandb` pour journaliser des métriques dans vos boucles d&#39;entraînement TensorFlow personnalisées, vous pouvez vous appuyer sur cet extrait :

```python
    with tf.GradientTape() as tape:
        # Obtenir les probabilités
        predictions = model(features)
        # Calculer la perte
        loss = loss_func(labels, predictions)

    # Enregistrer vos métriques
    run.log("loss": loss.numpy())
    # Obtenir les gradients
    gradients = tape.gradient(loss, model.trainable_variables)
    # Mettre à jour les poids
    optimizer.apply_gradients(zip(gradients, model.trainable_variables))
```

Un [exemple complet pour personnaliser les boucles d’entraînement dans TensorFlow 2](https://www.wandb.com/articles/wandb-customizing-training-loops-in-tensorflow-2) est disponible.

<div id="how-is-wb-different-from-tensorboard">
  ## En quoi W&amp;B se distingue-t-il de TensorBoard ?
</div>

Lorsque les cofondateurs ont commencé à travailler sur W&amp;B, ils ont voulu créer un outil pour les utilisateurs frustrés de TensorBoard chez OpenAI. Voici quelques aspects que nous avons particulièrement cherché à améliorer :

1. **Reproduire les modèles** : W&amp;B est particulièrement utile pour expérimenter, explorer et reproduire des modèles par la suite. Nous capturons non seulement les métriques, mais aussi les hyperparamètres et la version du code. Nous pouvons également enregistrer l’état de votre contrôle de version ainsi que les points de contrôle du modèle, afin que votre projet soit reproductible.
2. **Organisation automatique** : Que vous repreniez un projet d’un collaborateur, reveniez de vacances ou ressortiez un ancien projet, W&amp;B permet de voir facilement tous les modèles déjà testés, afin que personne ne perde des heures, des cycles GPU ou n’émette plus de carbone à relancer des expériences.
3. **Intégration rapide et flexible** : Ajoutez W&amp;B à votre projet en 5 minutes. Installez notre package Python open source gratuit, ajoutez quelques lignes à votre code, et chaque fois que vous exécutez votre modèle, vous obtenez des métriques et des enregistrements clairs et exploitables.
4. **Tableau de bord persistant et centralisé** : Peu importe où vous entraînez vos modèles, que ce soit sur votre machine locale, dans un cluster de laboratoire partagé ou sur des instances spot dans le cloud, vos résultats sont envoyés vers le même tableau de bord centralisé. Vous n’avez pas besoin de passer du temps à copier et à organiser des fichiers TensorBoard provenant de différentes machines.
5. **Tableaux puissants** : Recherchez, filtrez, triez et regroupez les résultats de différents modèles. Il est facile de parcourir des milliers de versions de modèle et de trouver les modèles les plus performants pour différentes tâches. TensorBoard n’est pas conçu pour bien fonctionner sur de grands projets.
6. **Outils de collaboration** : Utilisez W&amp;B pour organiser des projets complexes de machine learning. Il est facile de partager un lien vers W&amp;B, et vous pouvez utiliser des équipes privées pour que tout le monde envoie ses résultats vers un projet partagé. Nous prenons également en charge la collaboration via des rapports — ajoutez des visualisations interactives et décrivez votre travail en markdown. C’est un excellent moyen de tenir un journal de travail, de partager vos conclusions avec votre superviseur ou de présenter vos résultats à votre laboratoire ou à votre équipe.
