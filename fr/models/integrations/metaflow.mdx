<div id="overview">
  ## Aperçu
</div>

[Metaflow](https://docs.metaflow.org) est un framework créé par Netflix pour créer et exécuter des flux de travail de ML.

Cette intégration permet aux utilisateurs d’appliquer des décorateurs aux [étapes et flux](https://docs.metaflow.org/metaflow/basics) de Metaflow afin de journaliser automatiquement les paramètres et les artefacts dans W&amp;B.

* Décorer une étape active ou désactive la journalisation pour certains types dans cette étape.
* Décorer le flux active ou désactive la journalisation pour toutes les étapes du flux.

<div id="decorate-your-flows-and-steps">
  ### Décorez vos flux et vos étapes
</div>

<Tabs>
  <Tab title="Étape">
    Décorer une étape active ou désactive la journalisation pour certains types au sein de cette étape.

    Dans cet exemple, tous les jeux de données et modèles de `start` seront journalisés.

    ```python
    from wandb.integration.metaflow import wandb_log

    class WandbExampleFlow(FlowSpec):
        @wandb_log(datasets=True, models=True, settings=wandb.Settings(...))
        @step
        def start(self):
            self.raw_df = pd.read_csv(...).    # pd.DataFrame -> téléversé comme jeu de données
            self.model_file = torch.load(...)  # nn.Module    -> téléversé comme modèle
            self.next(self.transform)
    ```
  </Tab>

  <Tab title="Flux">
    Décorer un flux revient à décorer toutes les étapes qui le composent avec des valeurs par défaut.

    Dans ce cas, toutes les étapes de `WandbExampleFlow` journalisent par défaut les jeux de données et les modèles, comme si chaque étape était décorée avec `@wandb_log(datasets=True, models=True)`.

    ```python
    from wandb.integration.metaflow import wandb_log

    @wandb_log(datasets=True, models=True)  # décore tous les @step 
    class WandbExampleFlow(FlowSpec):
        @step
        def start(self):
            self.raw_df = pd.read_csv(...).    # pd.DataFrame -> téléversé comme jeu de données
            self.model_file = torch.load(...)  # nn.Module    -> téléversé comme modèle
            self.next(self.transform)
    ```
  </Tab>

  <Tab title="Flux et étapes">
    Décorer le flux revient à décorer toutes les étapes avec des valeurs par défaut. Cela signifie que si vous décorez ensuite une étape avec un autre `@wandb_log`, celui-ci remplace la décoration définie au niveau du flux.

    Dans cet exemple :

    * `start` et `mid` journalisent à la fois les jeux de données et les modèles.
    * `end` ne journalise ni jeux de données ni modèles.

    ```python
    from wandb.integration.metaflow import wandb_log

    @wandb_log(datasets=True, models=True)  # équivaut à décorer start et mid
    class WandbExampleFlow(FlowSpec):
      # cette étape enregistrera les jeux de données et les modèles
      @step
      def start(self):
        self.raw_df = pd.read_csv(...).    # pd.DataFrame -> téléversé comme jeu de données
        self.model_file = torch.load(...)  # nn.Module    -> téléversé comme modèle
        self.next(self.mid)

      # cette étape enregistrera également les jeux de données et les modèles
      @step
      def mid(self):
        self.raw_df = pd.read_csv(...).    # pd.DataFrame -> téléversé comme jeu de données
        self.model_file = torch.load(...)  # nn.Module    -> téléversé comme modèle
        self.next(self.end)

      # cette étape remplace ce comportement et n’enregistrera PAS de jeux de données NI de modèles
      @wandb_log(datasets=False, models=False)
      @step
      def end(self):
        self.raw_df = pd.read_csv(...).    
        self.model_file = torch.load(...)
    ```
  </Tab>
</Tabs>

<div id="access-your-data-programmatically">
  ## Accédez à vos données par programmation
</div>

Vous pouvez accéder aux informations que nous avons capturées de trois façons : dans le processus Python d’origine où elles sont journalisées à l’aide de la [bibliothèque cliente `wandb`](/fr/models/ref/python/), via l’[interface de l’application web](/fr/models/track/workspaces/), ou par programmation à l’aide de [notre API publique](/fr/models/ref/python/public-api/). Les `Parameter`s sont enregistrés dans la [`config`](/fr/models/) de W&amp;B et se trouvent dans l’[onglet Vue d’ensemble](/fr/models/runs/#overview-tab). Les `datasets`, `models` et `others` sont enregistrés dans les [Artifacts de W&amp;B](/fr/models/artifacts/) et se trouvent dans l’[onglet Artifacts](/fr/models/runs/#artifacts-tab). Les types Python de base sont enregistrés dans le dict [`summary`](/fr/models/) de W&amp;B et se trouvent dans l’onglet Vue d’ensemble. Voir notre [guide de l’API publique](/fr/models/track/public-api-guide/) pour plus de détails sur l’utilisation de l’API afin d’obtenir ces informations par programmation depuis l’extérieur.

<div id="quick-reference">
  ### Référence rapide
</div>

| Données                                            | Bibliothèque cliente                          | Interface utilisateur                |
| -------------------------------------------------- | --------------------------------------------- | ------------------------------------ |
| `Parameter(...)`                                   | `wandb.Run.config`                            | onglet Vue d’ensemble, Configuration |
| `datasets`, `models`, `others`                     | `wandb.Run.use_artifact("{var_name}:latest")` | onglet Artifacts                     |
| Types Python de base (`dict`, `list`, `str`, etc.) | `wandb.Run.summary`                           | onglet Vue d’ensemble, Résumé        |

<div id="wandb_log-kwargs">
  ### kwargs de `wandb_log`
</div>

| kwarg      | Options                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                   |
| ---------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `datasets` | <ul><li><code>True</code> : Journaliser les variables d’instance de type jeu de données</li><li><code>False</code></li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                                              |
| `models`   | <ul><li><code>True</code> : Journaliser les variables d’instance de type modèle</li><li><code>False</code></li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                                                      |
| `others`   | <ul><li><code>True</code> : Journaliser tout autre élément sérialisable avec pickle</li><li><code>False</code></li></ul>                                                                                                                                                                                                                                                                                                                                                                                                                                                  |
| `settings` | <ul><li><code>wandb.Settings(...)</code> : Spécifiez vos propres paramètres <code>wandb</code> pour cette étape ou ce flux</li><li><code>None</code> : Équivaut à passer <code>wandb.Settings()</code></li></ul><p>Par défaut, si :</p><ul><li><code>settings.run&#95;group</code> est <code>None</code>, il sera défini sur <code>&#123;flow&#95;name&#125;/&#123;run&#95;id&#125;</code></li><li><code>settings.run&#95;job&#95;type</code> est <code>None</code>, il sera défini sur <code>&#123;run&#95;job&#95;type&#125;/&#123;step&#95;name&#125;</code></li></ul> |

<div id="frequently-asked-questions">
  ## Questions fréquentes
</div>

<div id="what-exactly-do-you-log-do-you-log-all-instance-and-local-variables">
  ### Que journalisez-vous exactement ? Journalisez-vous toutes les variables d’instance et les variables locales ?
</div>

`wandb_log` ne journalise que les variables d’instance. Les variables locales ne sont JAMAIS journalisées. Cela permet d’éviter de journaliser des données inutiles.

<div id="which-data-types-get-logged">
  ### Quels types de données sont journalisés ?
</div>

Nous prenons actuellement en charge les types suivants :

| Paramètre de journalisation  | Type                                                                                                                                   |
| ---------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| par défaut (toujours activé) | <ul><li><code>dict, list, set, str, int, float, bool</code></li></ul>                                                                  |
| `datasets`                   | <ul><li><code>pd.DataFrame</code></li><li><code>pathlib.Path</code></li></ul>                                                          |
| `models`                     | <ul><li><code>nn.Module</code></li><li><code>sklearn.base.BaseEstimator</code></li></ul>                                               |
| `others`                     | <ul><li>Tout élément <a href="https://wiki.python.org/moin/UsingPickle">sérialisable avec pickle</a> et sérialisable en JSON</li></ul> |

<div id="how-can-i-configure-logging-behavior">
  ### Comment puis-je configurer le comportement de la journalisation ?
</div>

| Type de variable | comportement                       | Exemple         | Type de données |
| ---------------- | ---------------------------------- | --------------- | --------------- |
| Instance         | Journalisé automatiquement         | `self.accuracy` | `float`         |
| Instance         | Journalisé si `datasets=True`      | `self.df`       | `pd.DataFrame`  |
| Instance         | Non journalisé si `datasets=False` | `self.df`       | `pd.DataFrame`  |
| Locale           | Jamais journalisé                  | `accuracy`      | `float`         |
| Locale           | Jamais journalisé                  | `df`            | `pd.DataFrame`  |

<div id="is-artifact-lineage-tracked">
  ### La traçabilité des artefacts est-elle prise en charge ?
</div>

Oui. Si vous avez un artefact qui est une sortie de l’étape A et une entrée de l’étape B, nous construisons automatiquement pour vous le DAG de traçabilité.
