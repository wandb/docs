title: Fine-tuning Azure OpenAI
---

Ce guide vous montre comment utiliser W&amp;B avec Azure OpenAI pour suivre et évaluer les tâches de fine-tuning des modèles GPT-3.5 ou GPT-4. Lorsque vous intégrez W&amp;B, le suivi des expériences consigne les métriques, les hyperparamètres et les artefacts d&#39;entraînement afin de vous permettre d&#39;analyser et d&#39;améliorer les performances du modèle. Vous pouvez également utiliser les outils d&#39;évaluation de W&amp;B pour prendre des décisions fondées sur les données concernant la sélection du modèle.

Ce guide s&#39;adresse aux professionnels du machine learning qui effectuent le fine-tuning de modèles Azure OpenAI et souhaitent disposer d&#39;une méthode systématique pour suivre et comparer les exécutions.

<div id="prerequisites">
  ## Prérequis
</div>

Avant de commencer, effectuez les actions suivantes :

* Configurez le service Azure OpenAI en suivant la [documentation officielle d’Azure](https://wandb.me/aoai-wb-int).
* Configurez un compte W&amp;B à l’aide d’une clé API.

<div id="workflow-overview">
  ## Aperçu du flux de travail
</div>

Les étapes suivantes résument le déroulement typique d’un job de fine-tuning Azure OpenAI dans W&amp;B, depuis la préparation du job jusqu’à l’évaluation du modèle obtenu.

<div id="fine-tuning-setup">
  ### Configuration du fine-tuning
</div>

La configuration du fine-tuning comprend les étapes suivantes :

* Préparez les données d’entraînement conformément aux exigences d’Azure OpenAI.
* Configurez le job de fine-tuning dans Azure OpenAI.
* Suivez le processus de fine-tuning et consignez les métriques et les hyperparamètres avec W&amp;B.

<div id="experiment-tracking">
  ### Suivi des expériences
</div>

Lors du fine-tuning, W&amp;B capture :

* Les métriques d’entraînement et de validation.
* Les hyperparamètres du modèle.
* L’utilisation des ressources.
* Les artefacts d’entraînement.

<div id="model-evaluation">
  ### Évaluation du modèle
</div>

Après le Fine-tuning, utilisez [W&amp;B Weave](https://weave-docs.wandb.ai) pour :

* Évaluer les résultats du modèle à l’aide de jeux de données de référence.
* Comparer les performances entre différentes itérations de Fine-tuning.
* Analyser le comportement du modèle sur des cas de test spécifiques.
* Prendre des décisions fondées sur les données pour choisir le modèle.

  ## Exemple concret
