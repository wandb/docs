W&amp;B s&#39;intègre à [Amazon SageMaker](https://aws.amazon.com/sagemaker/), lit automatiquement les hyperparamètres, regroupe les runs distribués et permet de reprendre les runs à partir de points de contrôle.

<div id="authentication">
  ## Authentification
</div>

W&amp;B recherche un fichier nommé `secrets.env` par rapport au script d’entraînement et le charge dans l’environnement lors de l’appel à `wandb.init()`. Vous pouvez générer un fichier `secrets.env` en appelant `wandb.sagemaker_auth(path="source_dir")` dans le script que vous utilisez pour lancer vos expériences. Veillez à ajouter ce fichier à votre `.gitignore` !

<div id="existing-estimators">
  ## Estimateurs existants
</div>

Si vous utilisez l’un des estimateurs préconfigurés de SageMaker, vous devez ajouter un fichier `requirements.txt` à votre répertoire source qui inclut wandb

```text
wandb
```

Si vous utilisez un estimateur exécuté avec Python 2, vous devez installer `psutil` directement à partir de ce fichier wheel [wheel](https://pythonwheels.com) avant d’installer wandb :

```text
https://wheels.galaxyproject.org/packages/psutil-5.4.8-cp27-cp27mu-manylinux1_x86_64.whl
wandb
```

Consultez un exemple complet sur [GitHub](https://github.com/wandb/examples/tree/master/examples/pytorch/pytorch-cifar10-sagemaker) et découvrez-en plus sur notre [blog](https://wandb.ai/site/articles/running-sweeps-with-sagemaker).

Vous pouvez également consulter le [tutoriel « Déployer un analyseur de sentiments avec SageMaker et W&amp;B »](https://wandb.ai/authors/sagemaker/reports/Deploy-Sentiment-Analyzer-Using-SageMaker-and-W-B--VmlldzoxODA1ODE) sur le déploiement d&#39;un analyseur de sentiments avec SageMaker et W&amp;B.

<Warning>
  L&#39;agent de balayage W&amp;B se comporte comme prévu dans un job SageMaker uniquement si votre intégration SageMaker est désactivée. Désactivez l&#39;intégration SageMaker en modifiant votre appel à `wandb.init` :

  ```python
  wandb.init(..., settings=wandb.Settings(sagemaker_disable=True))
  ```
</Warning>
