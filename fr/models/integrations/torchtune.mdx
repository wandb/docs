[torchtune](https://meta-pytorch.org/torchtune/stable/index.html) est une bibliothèque PyTorch conçue pour simplifier la création, le fine-tuning et l’expérimentation autour des grands modèles de langage (LLM). De plus, torchtune inclut une prise en charge intégrée de la [journalisation avec W&amp;B](https://meta-pytorch.org/torchtune/stable/deep_dives/wandb_logging.html), ce qui améliore le suivi et la visualisation des processus d’entraînement.

Consultez l’article de blog de W&amp;B consacré au [Fine-tuning de Mistral 7B avec torchtune](https://wandb.ai/capecape/torchtune-mistral/reports/torchtune-The-new-PyTorch-LLM-fine-tuning-library---Vmlldzo3NTUwNjM0).

<div id="wb-logging-at-your-fingertips">
  ## La journalisation W&amp;B à portée de main
</div>

<Tabs>
  <Tab title="Ligne de commande">
    Redéfinissez les arguments de la ligne de commande au lancement :

    ```bash
    tune run lora_finetune_single_device --config llama3/8B_lora_single_device \
      metric_logger._component_=torchtune.utils.metric_logging.WandBLogger \
      metric_logger.project="llama3_lora" \
      log_every_n_steps=5
    ```
  </Tab>

  <Tab title="Recette">
    Activez la journalisation W&amp;B dans la configuration de la recette :

    ```yaml
    # dans llama3/8B_lora_single_device.yaml
    metric_logger:
      _component_: torchtune.utils.metric_logging.WandBLogger
      project: llama3_lora
    log_every_n_steps: 5
    ```
  </Tab>
</Tabs>

<div id="use-the-wb-metric-logger">
  ## Utilisez le logger de métriques W&amp;B
</div>

Activez la journalisation W&amp;B dans le fichier de configuration de la recette en modifiant la section `metric_logger`. Remplacez `_component_` par la classe `torchtune.utils.metric_logging.WandBLogger`. Vous pouvez également définir un nom de `project` et `log_every_n_steps` pour personnaliser le comportement de journalisation.

Vous pouvez aussi transmettre n’importe quels autres `kwargs`, comme vous le feriez avec la méthode [wandb.init()](/fr/models/ref/python/functions/init). Par exemple, si vous travaillez au sein d’une équipe, vous pouvez transmettre l’argument `entity` à la classe `WandBLogger` pour spécifier le nom de l’équipe.

<Tabs>
  <Tab title="Recette">
    ```yaml
    # dans llama3/8B_lora_single_device.yaml
    metric_logger:
      _component_: torchtune.utils.metric_logging.WandBLogger
      project: llama3_lora
      entity: my_project
      job_type: lora_finetune_single_device
      group: my_awesome_experiments
    log_every_n_steps: 5
    ```
  </Tab>

  <Tab title="Ligne de commande">
    ```shell
    tune run lora_finetune_single_device --config llama3/8B_lora_single_device \
      metric_logger._component_=torchtune.utils.metric_logging.WandBLogger \
      metric_logger.project="llama3_lora" \
      metric_logger.entity="my_project" \
      metric_logger.job_type="lora_finetune_single_device" \
      metric_logger.group="my_awesome_experiments" \
      log_every_n_steps=5
    ```
  </Tab>
</Tabs>

<div id="logged-metrics">
  ### Métriques enregistrées
</div>

Chaque recette a sa propre boucle d&#39;entraînement. Consultez chaque recette individuellement pour voir ses métriques enregistrées, qui incluent par défaut les éléments suivants :

| Métrique            | Description                                                                                                                                                                                                                                                                                                                            |
| ------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `loss`              | La perte du modèle                                                                                                                                                                                                                                                                                                                     |
| `lr`                | Le taux d&#39;apprentissage                                                                                                                                                                                                                                                                                                            |
| `tokens_per_second` | Les jetons par seconde du modèle                                                                                                                                                                                                                                                                                                       |
| `grad_norm`         | La norme du gradient du modèle                                                                                                                                                                                                                                                                                                         |
| `global_step`       | Correspond à l&#39;étape actuelle dans la boucle d&#39;entraînement. Tient compte de l&#39;accumulation de gradient ; en pratique, chaque fois qu&#39;une étape de l&#39;optimiseur est exécutée, le modèle est mis à jour, les gradients sont accumulés et le modèle est mis à jour une fois toutes les `gradient_accumulation_steps` |

<Note>
  `global_step` n&#39;est pas identique au nombre d&#39;étapes d&#39;entraînement. Il correspond à l&#39;étape actuelle dans la boucle d&#39;entraînement. Tient compte de l&#39;accumulation de gradient ; en pratique, chaque fois qu&#39;une étape de l&#39;optimiseur est exécutée, le `global_step` est incrémenté de 1. Par exemple, si le chargeur de données comporte 10 lots, que le nombre d&#39;étapes d&#39;accumulation de gradient est 2 et que l&#39;exécution dure 3 époques, l&#39;optimiseur effectuera 15 étapes ; dans ce cas, `global_step` ira de 1 à 15.
</Note>

La conception épurée de torchtune permet d&#39;ajouter facilement des métriques personnalisées ou de modifier celles qui existent déjà. Il suffit de modifier le [fichier de recette](https://github.com/meta-pytorch/torchtune/tree/main/recipes) correspondant. Par exemple, on pourrait enregistrer `current_epoch` comme pourcentage du nombre total d&#39;époques comme suit :

```python
# dans la fonction `train.py` du fichier de recette
self._metric_logger.log_dict(
    {"current_epoch": self.epochs * self.global_step / self._steps_per_epoch},
    step=self.global_step,
)
```

<Note>
  Il s’agit d’une bibliothèque qui évolue rapidement, les métriques actuelles sont susceptibles de changer. Si vous souhaitez ajouter une métrique personnalisée, vous devez modifier la recette et appeler la fonction `self._metric_logger.*` correspondante.
</Note>

<div id="save-and-load-checkpoints">
  ## Enregistrer et charger des points de contrôle
</div>

La bibliothèque torchtune prend en charge différents [formats de points de contrôle](https://meta-pytorch.org/torchtune/stable/deep_dives/checkpointer.html). Selon l’origine du modèle que vous utilisez, vous devez passer à la [classe de checkpointer](https://meta-pytorch.org/torchtune/stable/deep_dives/checkpointer.html) appropriée.

Si vous souhaitez enregistrer les points de contrôle du modèle dans les [W&amp;B Artifacts](/fr/models/artifacts/), la solution la plus simple consiste à redéfinir la fonction `save_checkpoint` dans la recette correspondante.

Voici un exemple montrant comment redéfinir la fonction `save_checkpoint` pour enregistrer les points de contrôle du modèle dans W&amp;B Artifacts.

```python
def save_checkpoint(self, epoch: int) -> None:
    ...
    ## Sauvegardons le point de contrôle dans W&B
    ## selon la classe Checkpointer, le fichier sera nommé différemment
    ## Voici un exemple pour le cas full_finetune
    checkpoint_file = Path.joinpath(
        self._checkpointer._output_dir, f"torchtune_model_{epoch}"
    ).with_suffix(".pt")
    wandb_artifact = wandb.Artifact(
        name=f"torchtune_model_{epoch}",
        type="model",
        # description du point de contrôle du modèle
        description="Point de contrôle du modèle",
        # vous pouvez ajouter toutes les métadonnées souhaitées sous forme de dict
        metadata={
            utils.SEED_KEY: self.seed,
            utils.EPOCHS_KEY: self.epochs_run,
            utils.TOTAL_EPOCHS_KEY: self.total_epochs,
            utils.MAX_STEPS_KEY: self.max_steps_per_epoch,
        },
    )
    wandb_artifact.add_file(checkpoint_file)
    wandb.log_artifact(wandb_artifact)
```
