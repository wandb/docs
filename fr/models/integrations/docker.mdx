<div id="docker-integration">
  ## Intégration Docker
</div>

W&amp;B peut stocker une référence à l’image Docker dans laquelle votre code a été exécuté, ce qui vous permet de restaurer une expérience précédente dans l’environnement exact où elle a été exécutée. La bibliothèque wandb recherche la variable d’environnement **WANDB&#95;DOCKER** afin de conserver cet état. Nous fournissons quelques utilitaires qui le définissent automatiquement.

<div id="local-development">
  ### Développement local
</div>

`wandb docker` est une commande qui démarre un conteneur Docker, transmet les variables d’environnement wandb, monte votre code et s’assure que wandb est installé. Par défaut, la commande utilise une image Docker avec TensorFlow, PyTorch, Keras et Jupyter déjà installés. Vous pouvez utiliser la même commande pour démarrer votre propre image Docker : `wandb docker my/image:latest`. La commande monte le répertoire courant dans le répertoire &quot;/app&quot; du conteneur ; vous pouvez modifier cela avec l’indicateur &quot;--dir&quot;.

<div id="production">
  ### Production
</div>

La commande `wandb docker-run` est fournie pour les charges de travail de production. Elle est conçue pour remplacer directement `nvidia-docker`. Il s&#39;agit d&#39;un simple wrapper autour de la commande `docker run` qui ajoute vos identifiants ainsi que la variable d&#39;environnement **WANDB&#95;DOCKER** à la commande. Si vous ne spécifiez pas l&#39;option &quot;--runtime&quot; et que `nvidia-docker` est disponible sur la machine, cela garantit également que le runtime est défini sur nvidia.

<div id="kubernetes">
  ### Kubernetes
</div>

Si vous exécutez vos charges de travail d’entraînement dans Kubernetes et que l’API k8s est exposée à votre pod (ce qui est le cas par défaut), wandb effectuera une requête auprès de l’API pour récupérer le digest de l’image Docker et définira automatiquement la variable d’environnement **WANDB&#95;DOCKER**.

<div id="restoring">
  ## Restauration
</div>

Si un run a été instrumenté avec la variable d&#39;environnement **WANDB&#95;DOCKER**, l&#39;appel à `wandb restore username/project:run_id` créera une nouvelle branche pour restaurer votre code, puis lancera l&#39;image Docker exacte utilisée pour l&#39;entraînement, préremplie avec la commande originale.
