[Prodigy](https://prodi.gy/) est un outil d&#39;annotation qui permet de créer des données d&#39;entraînement et d&#39;évaluation pour les modèles de machine learning, ainsi que pour l&#39;analyse des erreurs, l&#39;inspection et le nettoyage des données. Les [W&amp;B Tables](/fr/models/tables/tables-walkthrough/) vous permettent de consigner, visualiser, analyser et partager des jeux de données (et bien plus encore !) dans W&amp;B.

Ce guide vous montre comment utiliser [l&#39;intégration de W&amp;B avec Prodigy](https://github.com/wandb/wandb/blob/master/wandb/integration/prodigy/prodigy.py) pour téléverser directement votre jeu de données annoté avec Prodigy vers W&amp;B afin de pouvoir l&#39;explorer et le partager sous forme de tableau interactif. Utilisez cette intégration si vous souhaitez inspecter la qualité des annotations, comparer les versions d&#39;un jeu de données étiqueté ou partager les résultats avec des collaborateurs.

Avec quelques lignes de code, comme celles-ci :

```python
import wandb
from wandb.integration.prodigy import upload_dataset

with wandb.init(project="prodigy"):
    upload_dataset("news_headlines_ner")
```

<div id="quickstart">
  ## Démarrage rapide
</div>

Utilisez `wandb.integration.prodigy.upload_dataset` pour importer votre jeu de données Prodigy annoté directement depuis la base de données Prodigy locale vers W&amp;B au format [tableau](/fr/models/ref/python/data-types/table). Pour plus d’informations sur Prodigy, y compris l’installation et la configuration, voir la [documentation Prodigy](https://prodi.gy/docs/).

Lorsque vous téléversez un jeu de données, W&amp;B convertit automatiquement les images et les champs d’entités nommées en [`wandb.Image`](/fr/models/ref/python/data-types/image) et [`wandb.Html`](/fr/models/ref/python/data-types/html) respectivement, afin qu’ils s’affichent sous forme de visualisations interactives dans votre tableau. W&amp;B peut ajouter des colonnes supplémentaires au tableau obtenu pour inclure ces visualisations.
