Hugging Face Accelerate est une bibliothèque qui permet d&#39;exécuter le même code PyTorch avec n&#39;importe quelle configuration distribuée, afin de simplifier l&#39;entraînement du modèle et l&#39;inférence à grande échelle.

Accelerate inclut un suivi W&amp;B, dont nous expliquons l&#39;utilisation ci-dessous. Vous pouvez également en savoir plus sur les [suivis d&#39;Accelerate dans Hugging Face](https://huggingface.co/docs/accelerate/main/en/usage_guides/tracking).

<div id="start-logging-with-accelerate">
  ## Commencez à journaliser avec Accelerate
</div>

Pour bien démarrer avec Accelerate et W&amp;B, vous pouvez suivre le pseudo-code ci-dessous :

```python
from accelerate import Accelerator

# Indiquer à l'objet Accelerator de journaliser avec wandb
accelerator = Accelerator(log_with="wandb")

# Initialiser votre run wandb en transmettant les paramètres wandb et les informations de configuration
accelerator.init_trackers(
    project_name="my_project", 
    config={"dropout": 0.1, "learning_rate": 1e-2}
    init_kwargs={"wandb": {"entity": "my-wandb-team"}}
    )

...

# Journaliser dans wandb en appelant `accelerator.log`, `step` est facultatif
accelerator.log({"train_loss": 1.12, "valid_loss": 0.8}, step=global_step)


# S'assurer que le suivi wandb se termine correctement
accelerator.end_training()
```

Plus précisément, vous devez :

1. Passer `log_with="wandb"` lors de l’initialisation de la classe Accelerator
2. Appeler la méthode [`init_trackers`](https://huggingface.co/docs/accelerate/main/en/package_reference/accelerator#accelerate.Accelerator.init_trackers) et lui passer :

* un nom de projet via `project_name`
* tous les paramètres que vous souhaitez transmettre à [`wandb.init()`](/fr/models/ref/python/functions/init) via un dict imbriqué dans `init_kwargs`
* toute autre information de configuration de l’expérience que vous souhaitez journaliser dans votre run wandb, via `config`

3. Utiliser la méthode `.log` pour journaliser dans Weights &amp; Biases ; l’argument `step` est facultatif
4. Appeler `.end_training` une fois l’entraînement terminé

<div id="access-the-wb-tracker">
  ## Accéder au suivi W&amp;B
</div>

Pour accéder au suivi W&amp;B, utilisez la méthode `Accelerator.get_tracker()`. Passez la chaîne correspondant à l’attribut `.name` du suivi ; la méthode renvoie alors le suivi sur le processus `main`.

```python
wandb_tracker = accelerator.get_tracker("wandb")

```

À partir de là, vous pouvez interagir avec l’objet run de wandb normalement :

```python
wandb_tracker.log_artifact(some_artifact_to_log)
```

<Warning>
  Les suivis intégrés à Accelerate s’exécutent automatiquement sur le bon processus. Ainsi, si un suivi ne doit s’exécuter que sur le processus principal, cela se fera automatiquement.

  Si vous souhaitez vraiment supprimer complètement l’encapsulation d’Accelerate, vous pouvez obtenir le même résultat avec :

  ```python
  wandb_tracker = accelerator.get_tracker("wandb", unwrap=True)
  with accelerator.on_main_process:
      wandb_tracker.log_artifact(some_artifact_to_log)
  ```
</Warning>
