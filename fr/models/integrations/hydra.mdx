> [Hydra](https://hydra.cc) est un framework Python open source qui simplifie le développement d’applications de recherche et d’autres applications complexes. Sa fonctionnalité clé est la possibilité de créer dynamiquement une configuration hiérarchique par composition et de la redéfinir via des fichiers de configuration et la ligne de commande.

Vous pouvez continuer à utiliser Hydra pour gérer la configuration tout en tirant parti de la puissance de W&amp;B.

<div id="track-metrics">
  ## Suivre les métriques
</div>

Suivez vos métriques comme d’habitude avec `wandb.init()` et `wandb.Run.log()`. Ici, `wandb.entity` et `wandb.project` sont définis dans un fichier de configuration Hydra.

```python
import wandb


@hydra.main(config_path="configs/", config_name="defaults")
def run_experiment(cfg):

    with wandb.init(entity=cfg.wandb.entity, project=cfg.wandb.project) as run:
      run.log({"loss": loss})
```

<div id="track-hyperparameters">
  ## Suivre les hyperparamètres
</div>

Hydra utilise [omegaconf](https://omegaconf.readthedocs.io/en/2.1_branch/) comme interface par défaut avec les dictionnaires de configuration. Les dictionnaires d&#39;`OmegaConf` ne sont pas une sous-classe des dictionnaires primitifs. Par conséquent, passer directement le `Config` de Hydra à `wandb.Run.config` entraîne des résultats inattendus dans le tableau de bord. Il est nécessaire de convertir `omegaconf.DictConfig` en type primitif `dict` avant de le passer à `wandb.Run.config`.

```python
@hydra.main(config_path="configs/", config_name="defaults")
def run_experiment(cfg):
  with wandb.init(entity=cfg.wandb.entity, project=cfg.wandb.project) as run:
    run.config = omegaconf.OmegaConf.to_container(
        cfg, resolve=True, throw_on_missing=True
    )
    run = wandb.init(entity=cfg.wandb.entity, project=cfg.wandb.project)
    run.log({"loss": loss})
    model = Model(**run.config.model.configs)
```

<div id="troubleshoot-multiprocessing">
  ## Résoudre les problèmes liés au multiprocessing
</div>

Si votre processus se bloque au démarrage, cela peut être dû à [ce problème connu](/fr/models/track/log/distributed-training). Pour y remédier, essayez de modifier le protocole de multiprocessing de wandb en ajoutant un paramètre `settings` supplémentaire à `wandb.init()`, comme suit :

```python
wandb.init(settings=wandb.Settings(start_method="thread"))
```

ou en définissant une variable d’environnement globale depuis votre shell :

```bash
$ export WANDB_START_METHOD=thread
```

<div id="optimize-hyperparameters">
  ## Optimiser les hyperparamètres
</div>

[W&amp;B Sweeps](/fr/models/sweeps) est une plateforme de recherche d&#39;hyperparamètres très évolutive qui fournit, avec un minimum de code, des visualisations et des informations utiles sur les expériences W&amp;B. Sweeps s&#39;intègre de manière transparente aux projets Hydra, sans nécessiter de code. Il suffit d&#39;un fichier de configuration décrivant, de manière classique, les différents paramètres à balayer.

Un exemple simple de fichier `sweep.yaml` serait :

```yaml
program: main.py
method: bayes
metric:
  goal: maximize
  name: test/accuracy
parameters:
  dataset:
    values: [mnist, cifar10]

command:
  - ${env}
  - python
  - ${program}
  - ${args_no_hyphens}
```

Lancez le balayage :

```bash
wandb sweep sweep.yaml
```

W&amp;B crée automatiquement un balayage dans votre projet et renvoie la commande `wandb agent` à exécuter sur chaque machine sur laquelle vous souhaitez lancer votre balayage.

<div id="pass-parameters-not-present-in-hydra-defaults">
  ### Passer des paramètres absents des valeurs par défaut d’Hydra
</div>

<a id="pitfall-3-sweep-passing-parameters-not-present-in-defaults" aria-label="Passer des paramètres absents des valeurs par défaut d’Hydra" />

Hydra prend en charge le passage de paramètres supplémentaires via la ligne de commande qui ne figurent pas dans le fichier de configuration par défaut, en utilisant un `+` devant la commande. Par exemple, vous pouvez transmettre un paramètre supplémentaire avec une valeur donnée en appelant simplement :

```bash
$ python program.py +experiment=some_experiment
```

Vous ne pouvez pas effectuer un balayage sur ce type de configurations `+`, comme on peut le faire lors de la configuration de [Hydra Experiments](https://hydra.cc/docs/patterns/configuring_experiments/). Pour contourner ce problème, vous pouvez initialiser le paramètre d&#39;expérience avec un fichier vide par défaut, puis utiliser W&amp;B Sweep pour redéfinir ces configurations vides à chaque appel. Pour plus d&#39;informations, lisez [ce rapport W&amp;B](https://wandb.ai/adrishd/hydra-example/reports/Configuring-W-B-Projects-with-Hydra--VmlldzoxNTA2MzQw?galleryTag=posts\&utm_source=fully_connected\&utm_medium=blog\&utm_campaign=hydra).
