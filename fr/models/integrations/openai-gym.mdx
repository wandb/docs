Cette page explique comment utiliser W&amp;B avec [OpenAI Gym](https://github.com/openai/gym) pour capturer automatiquement des vidéos de vos environnements d&#39;apprentissage par renforcement. Vous pouvez ensuite examiner le comportement de l&#39;agent en parallèle des métriques de votre expérience dans W&amp;B.

Si vous utilisez OpenAI Gym, W&amp;B enregistre automatiquement les vidéos de votre environnement générées par `gym.wrappers.Monitor`. Pour l&#39;activer, définissez l&#39;argument mot-clé `monitor_gym` de [`wandb.init()`](/fr/models/ref/python/functions/init) sur `True`, ou appelez `wandb.gym.monitor()`.

L&#39;intégration gym est légère. Elle [regarde le nom du fichier vidéo](https://github.com/wandb/wandb/blob/master/wandb/integration/gym/__init__.py#L15) enregistré depuis `gym` et lui donne ce nom, ou utilise `"videos"` par défaut si elle ne trouve pas de correspondance. Si vous voulez plus de contrôle, vous pouvez [enregistrer une vidéo](/fr/models/track/log/media/) manuellement.

Le [OpenRL Benchmark](https://wandb.me/openrl-benchmark-report) de [CleanRL](https://github.com/vwxyzjn/cleanrl) utilise cette intégration pour ses exemples OpenAI Gym. Vous pouvez trouver le code source (y compris [le code spécifique utilisé pour certains runs](https://wandb.ai/cleanrl/cleanrl.benchmark/runs/2jrqfugg/code?workspace=user-costa-huang)) qui montre comment utiliser gym avec W&amp;B.
