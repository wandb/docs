Tenez dans W&amp;B un journal des métriques et de la configuration de fine-tuning de votre modèle OpenAI GPT-3.5 ou GPT-4. Utilisez l&#39;écosystème W&amp;B pour suivre vos expériences de fine-tuning, vos modèles et vos jeux de données, et partager vos résultats avec vos collègues.

<Note>
  Voir la [documentation OpenAI](https://platform.openai.com/docs/guides/fine-tuning/which-models-can-be-fine-tuned) pour obtenir la liste des modèles que vous pouvez affiner.
</Note>

Voir la section [W&amp;B Integration](https://developers.openai.com/cookbook/examples/third_party/gpt_finetuning_with_wandb) de la documentation OpenAI pour plus d&#39;informations sur la manière d&#39;intégrer W&amp;B à OpenAI pour le fine-tuning.

<div id="install-or-update-openai-python-api">
  ## Installer ou mettre à jour l’API Python OpenAI
</div>

L’intégration W&amp;B de fine-tuning OpenAI fonctionne avec OpenAI à partir de la version 1.0. Voir la documentation PyPI pour connaître la dernière version de la bibliothèque [API Python OpenAI](https://pypi.org/project/openai/).

Pour installer l’API Python OpenAI, exécutez :

```python
pip install openai
```

Si vous avez déjà installé l’API Python OpenAI, vous pouvez la mettre à jour avec :

```python
pip install -U openai
```

<div id="sync-your-openai-fine-tuning-results">
  ## Synchronisez les résultats de votre fine-tuning OpenAI
</div>

Intégrez W&amp;B à l&#39;API de fine-tuning d&#39;OpenAI pour tenir dans W&amp;B un journal des métriques et de la configuration de votre fine-tuning. Pour ce faire, utilisez la classe `WandbLogger` du module `wandb.integration.openai.fine_tuning`.

```python
from wandb.integration.openai.fine_tuning import WandbLogger

# Logique de fine-tuning

WandbLogger.sync(fine_tune_job_id=FINETUNE_JOB_ID)
```

<div id="sync-your-fine-tunes">
  ### Synchronisez vos tâches de fine-tuning
</div>

Synchronisez vos résultats depuis votre script

```python
from wandb.integration.openai.fine_tuning import WandbLogger

# commande en une ligne
WandbLogger.sync()

# passage de paramètres facultatifs
WandbLogger.sync(
    fine_tune_job_id=None,
    num_fine_tunes=None,
    project="OpenAI-Fine-Tune",
    entity=None,
    overwrite=False,
    model_artifact_name="model-metadata",
    model_artifact_type="model",
    **kwargs_wandb_init
)
```

<div id="the-fine-tuned-model-and-model-versioning">
  ## Le modèle affiné et la gestion des versions du modèle
</div>

OpenAI vous fournit un ID pour le modèle affiné. Comme nous n’avons pas accès aux poids du modèle, le `WandbLogger` crée un fichier `model_metadata.json` contenant tous les détails (hyperparamètres, ID des fichiers de données, etc.) du modèle ainsi que l’ID `fine_tuned_model`, et l’enregistre comme un artefact W&amp;B.

Cet artefact de modèle (métadonnées) peut ensuite être lié à un modèle dans le [W&amp;B Registry](/fr/models/registry/).

<div id="how-do-i-share-my-fine-tune-results-with-my-team-in-wb">
  ### Comment puis-je partager les résultats de mon fine-tuning avec mon équipe dans W&amp;B ?
</div>

Tenez un journal de vos jobs de fine-tuning sur le compte de votre équipe avec :

```python
WandbLogger.sync(entity="YOUR_TEAM_NAME")
```

<div id="how-can-i-organize-my-runs">
  ### Comment puis-je organiser mes runs ?
</div>

Vos W&amp;B Runs sont automatiquement organisés et peuvent être filtrés ou triés selon n&#39;importe quel paramètre de configuration, comme le type de job, le modèle de base, le taux d&#39;apprentissage, le nom du fichier d&#39;entraînement ou tout autre hyperparamètre.

Vous pouvez également renommer vos runs, ajouter des notes ou créer des tags pour les regrouper.

Une fois que cela vous convient, vous pouvez enregistrer votre Workspace et l&#39;utiliser pour créer un rapport en important des données depuis vos runs et vos artefacts enregistrés (fichiers d&#39;entraînement/de validation).

<div id="how-can-i-access-my-fine-tuned-model">
  ### Comment puis-je accéder à mon modèle affiné ?
</div>

L’ID du modèle affiné est enregistré dans W&amp;B sous forme d’artefact (`model_metadata.json`), ainsi que dans la configuration.

```python
import wandb
    
with wandb.init(project="OpenAI-Fine-Tune", entity="YOUR_TEAM_NAME") as run:
    ft_artifact = run.use_artifact("ENTITY/PROJECT/model_metadata:VERSION")
    artifact_dir = ft_artifact.download()
```

où `VERSION` correspond à l’un des éléments suivants :

* un numéro de version, tel que `v2`
* l’identifiant du fine-tuning, tel que `ft-xxxxxxxxx`
* un alias ajouté automatiquement, comme `latest`, ou ajouté manuellement

Vous pouvez ensuite récupérer l’ID du modèle `fine_tuned_model` en lisant le fichier `model_metadata.json` téléchargé.

<div id="what-if-a-fine-tune-was-not-synced-successfully">
  ### Que faire si un Fine-tuning ne s’est pas synchronisé correctement ?
</div>

Si un Fine-tuning n’a pas été enregistré dans W&amp;B correctement, vous pouvez utiliser `overwrite=True` et fournir l’ID du job de Fine-tuning :

```python
WandbLogger.sync(
    fine_tune_job_id="FINE_TUNE_JOB_ID",
    overwrite=True,
)
```

<div id="can-i-track-my-datasets-and-models-with-wb">
  ### Puis-je suivre mes jeux de données et mes modèles avec W&amp;B ?
</div>

Les données d’entraînement et de validation sont automatiquement enregistrées dans W&amp;B sous forme d’artefacts. Les métadonnées, y compris l’ID du modèle affiné, sont elles aussi enregistrées sous forme d’artefacts.

Vous pouvez toujours contrôler le pipeline à l’aide d’API wandb de bas niveau, comme `wandb.Artifact`, `wandb.Run.log`, etc. Cela permet d’assurer une traçabilité complète de vos données et de vos modèles.
