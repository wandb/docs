title: Filtrer les runs
---

Filtrez les runs en fonction de leur nom, de leur état, de leurs [tags](#filter-runs-with-tags) ou d’autres propriétés à l’aide de l’éditeur d’expressions du tableau des runs.

Lorsque vous ajoutez un filtre, vous choisissez d’abord un champ (par exemple, tags, horodatage ou entité). Chaque champ a un type sous-jacent, par exemple texte, date/heure ou ID.

La liste des opérateurs proposés (par exemple, is, in, ≥, within last) dépend de ce type. Une fois le champ choisi, l’interface utilisateur n’affiche que les opérateurs valides pour le type de ce champ.

<div id="common-operators-by-type">
  ## Opérateurs courants par type
</div>

| Type de Filtre       | Champs d&#39;exemple | Opérateurs courants                | Exemple d&#39;utilisation          |
| -------------------- | -------------------- | ---------------------------------- | ---------------------------------- |
| Tags                 | `tags`               | est, n&#39;est pas, dans, pas dans | `tags is "baseline"`               |
| Temps                | `created timestamp`  | ≤, ≥, au cours des derniers        | `created timestamp` ≥ `01/16/2026` |
| Chaîne de caractères | `state`              | =, ≠, IN, NOT IN                   | `state = "finished"`               |

<Note>
  Le tableau ci-dessus ne montre qu&#39;un sous-ensemble des champs et opérateurs disponibles. L&#39;éditeur d’expressions affiche tous les champs et opérateurs disponibles.
</Note>

<div id="create-a-filter-expression">
  ## Créer une expression de filtre
</div>

1. Accédez à l’onglet **Runs** depuis la barre latérale du projet.
2. Sélectionnez le bouton **Filtre**, en forme d’entonnoir, au-dessus du tableau des runs.
3. De gauche à droite, sélectionnez un nom de colonne, un opérateur logique et une valeur de filtre pour créer une expression de filtre.
4. Vous pouvez également sélectionner **Nouveau filtre** pour appliquer des filtres supplémentaires.

<div id="example-filter-runs-with-tags">
  ### Exemple : filtrer les runs avec des tags
</div>

Filtrez les runs selon leurs tags :

1. Cliquez sur l’onglet **Runs** dans la barre latérale du projet.
2. Sélectionnez le bouton **Filtre**, en forme d’entonnoir, au-dessus du tableau des runs.
3. De gauche à droite, sélectionnez `"Tags"` dans le premier menu déroulant, puis un opérateur logique.
4. Sélectionnez is, is not, in ou not in dans le deuxième menu déroulant.
5. Saisissez le nom du tag à utiliser pour le filtrage dans le troisième menu déroulant.

<div id="default-filters">
  ## Filtres par défaut
</div>

Par défaut, W&amp;B fournit les filtres suivants :

* **Afficher uniquement mes runs** : affiche uniquement les runs créés par l&#39;utilisateur actuel.
* **Masquer les runs plantés** : masque les runs dont l&#39;état est `crashed`.

Les filtres par défaut apparaissent sous forme de boutons de bascule sous le bouton **Nouveau filtre** dans l&#39;éditeur d&#39;expressions de filtre.

<div id="remove-a-filter">
  ## Supprimer un filtre
</div>

Pour supprimer un filtre du tableau des runs :

1. Cliquez sur le bouton **Filtre**, qui ressemble à un entonnoir, au-dessus du tableau des runs.
2. Sélectionnez l’icône `x` à côté du filtre que vous voulez supprimer.
