title: Filtrer les runs
---

Filtrez les runs selon leur nom, leur état, leurs [tags](#filter-runs-with-tags) ou d’autres propriétés à l’aide de l’éditeur d’expressions dans le tableau des Runs.

Lorsque vous ajoutez un filtre, vous choisissez d’abord un champ (par exemple, tags, timestamp ou entité). Chaque champ a un type sous-jacent, comme texte, heure ou ID.

La liste des opérateurs affichés (par exemple, is, in, ≥, within last) dépend de ce type. Après avoir choisi un champ, l’UI n’affiche que les opérateurs valides pour le type de ce champ.

<div id="common-operators-by-type">
  ## Opérateurs courants par type
</div>

| Type de filtre       | Exemples de champs  | Opérateurs courants    | Exemple d’utilisation              |
| -------------------- | ------------------- | ---------------------- | ---------------------------------- |
| Tags                 | `tags`              | is, is not, in, not in | `tags is "baseline"`               |
| Date/heure           | `created timestamp` | ≤, ≥, within last      | `created timestamp` ≥ `01/16/2026` |
| Chaîne de caractères | `state`             | =, ≠, IN, NOT IN       | `state = "finished"`               |

<Note>
  Le tableau ci-dessus ne présente qu’un sous-ensemble des champs et opérateurs disponibles. L’éditeur d’expressions affiche tous les champs et opérateurs disponibles.
</Note>

<div id="create-a-filter-expression">
  ## Créer une expression de filtre
</div>

1. Accédez à l’onglet **Runs** dans la barre latérale du projet.
2. Sélectionnez le bouton **Filtre**, en forme d’entonnoir, au-dessus du tableau des Runs.
3. De gauche à droite, sélectionnez un nom de colonne, un opérateur logique et une valeur de filtre pour créer une expression de filtre.
4. Vous pouvez aussi sélectionner **New Filter** pour appliquer des filtres supplémentaires.

<div id="example-filter-runs-with-tags">
  ### Exemple : filtrer les runs avec des tags
</div>

Filtrez les runs en fonction de leurs tags :

1. Cliquez sur l’onglet **Runs** dans la barre latérale du projet.
2. Sélectionnez le bouton **Filtre**, en forme d’entonnoir, au-dessus du tableau des runs.
3. De gauche à droite, sélectionnez `"Tags"` dans le menu déroulant, puis sélectionnez un opérateur logique.
4. Sélectionnez is, is not, in ou not in dans le deuxième menu déroulant.
5. Saisissez le nom du tag sur lequel vous voulez filtrer dans le troisième menu déroulant.

<div id="default-filters">
  ## Filtres par défaut
</div>

Par défaut, W&amp;B propose les filtres suivants :

* **Afficher uniquement mes runs** : affiche uniquement les runs créés par l’utilisateur actuel.
* **Masquer les runs plantés** : masque les runs dont l’état est `crashed`.

Les filtres par défaut apparaissent sous forme de boutons bascule sous le bouton **Nouveau filtre** dans l’éditeur d’expressions de filtre.

<div id="remove-a-filter">
  ## Supprimer un filtre
</div>

Pour supprimer un filtre du tableau des Runs :

1. Cliquez sur le bouton **Filtre**, en forme d’entonnoir, au-dessus du tableau des Runs.
2. Cliquez sur l’icône `x` à côté du filtre que vous voulez supprimer.
