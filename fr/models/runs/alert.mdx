Créez des alertes avec Slack ou par e-mail en cas de plantage de votre run ou à l’aide d’un déclencheur personnalisé. Par exemple, vous pouvez créer une alerte si le gradient de votre boucle d’entraînement commence à diverger (renvoie NaN) ou si une étape de votre pipeline de ML se termine. Les alertes s’appliquent à tous les projets dans lesquels vous initialisez des runs, y compris les projets personnels et les projets d’équipe.

Vous verrez ensuite les messages des alertes W&amp;B dans Slack (ou dans votre e-mail) :

</Frame>

<Note>
  Les alertes W&amp;B nécessitent que vous ajoutiez `run.alert()` à votre code. Sans modifier votre code, [Automations](/fr/models/automations/) offrent une autre façon de notifier Slack en fonction d’un événement dans W&amp;B, par exemple lorsqu’une version d’[artefact](/fr/models/artifacts/) est créée ou lorsqu’une [métrique de run](/fr/models/runs/) atteint un seuil ou varie au-delà d’un seuil.

  Par exemple, une automatisation peut notifier un canal Slack lorsqu’une nouvelle version est créée, exécuter un webhook de test automatisé lorsque l’alias `production` est ajouté à un artefact, ou démarrer un job de validation uniquement lorsque la `loss` d’un run se situe dans des limites acceptables.

  Lisez l’[aperçu d’Automations](/fr/models/automations/) ou [créez une automatisation](/fr/models/automations/create-automations/).
</Note>

<Note>
  Le guide suivant s’applique uniquement aux alertes dans le Cloud mutualisé.

  Si vous utilisez [W&amp;B Server](/fr/platform/hosting/) dans votre Cloud privé ou sur W&amp;B Cloud dédié, reportez-vous à [Configurer les alertes Slack dans W&amp;B Server](/fr/platform/hosting/monitoring-usage/slack-alerts) pour configurer les alertes Slack.
</Note>

Pour configurer une alerte, suivez ces étapes, détaillées dans les sections suivantes :

1. Activez Alerts dans vos [Paramètres utilisateur W&amp;B](https://wandb.ai/settings).
2. Ajoutez `run.alert()` à votre code.
3. Testez la configuration.

<div id="1-turn-on-alerts-in-your-wb-user-settings">
  ### 1. Activez les alertes dans vos Paramètres utilisateur W&amp;B
</div>

Dans vos [Paramètres utilisateur](https://wandb.ai/settings) :

* Faites défiler jusqu’à la section **Alerts**
* Activez **Scriptable run alerts** pour recevoir les alertes de `run.alert()`
* Utilisez **Connect Slack** pour choisir un canal Slack dans lequel publier les alertes. Nous recommandons le canal **Slackbot**, car il permet de garder les alertes privées.
* Les **Email** seront envoyés à l’adresse e-mail utilisée lors de votre inscription à W&amp;B. Nous vous recommandons de configurer un filtre dans votre messagerie afin que toutes ces alertes soient placées dans un dossier et n’encombrent pas votre boîte de réception.

Vous n’aurez à le faire qu’une seule fois, lors de la configuration initiale des alertes W&amp;B, ou lorsque vous souhaiterez modifier la manière dont vous les recevez.

  <img src="/images/track/demo_connect_slack.png" alt="Paramètres des alertes dans les Paramètres utilisateur W&B" />

<div id="2-add-runalert-to-your-code">
  ### 2. Ajoutez `run.alert()` à votre code
</div>

Ajoutez `run.alert()` à votre code (dans un notebook ou un script Python), à l’endroit où vous souhaitez le déclencher

```python
import wandb

with wandb.init() as run:
    run.alert(title="High Loss", text="Loss is increasing rapidly")
```

<div id="3-test-the-configuration">
  ### 3. Tester la configuration
</div>

Vérifiez dans Slack ou vos e-mails si vous avez reçu le message d’alerte. Si ce n’est pas le cas, assurez-vous d’avoir activé les e-mails ou Slack pour les **alertes scriptables** dans vos [Paramètres utilisateur](https://wandb.ai/settings)

<div id="example">
  ## Exemple
</div>

Cette alerte simple envoie un avertissement lorsque la précision passe sous un seuil. Dans cet exemple, elle n’envoie des alertes qu’à au moins 5 minutes d’intervalle.

```python
import wandb
from wandb import AlertLevel

with wandb.init() as run:

    if acc < threshold:
        run.alert(
            title="Low accuracy",
            text=f"Accuracy {acc} is below the acceptable threshold {threshold}",
            level=AlertLevel.WARN,
            wait_duration=300,
        )
```

<div id="tag-or-mention-users">
  ## Identifier ou mentionner des utilisateurs
</div>

Utilisez le signe arobase `@` suivi de l’ID utilisateur Slack pour vous mentionner ou mentionner vos collègues, dans le titre comme dans le texte de l’alerte. Vous trouverez l’ID utilisateur Slack sur la page de profil Slack de la personne concernée.

```python
run.alert(title="Loss is NaN", text=f"Hey <@U1234ABCD> loss has gone to NaN")
```

<div id="configure-team-alerts">
  ## Configurer les alertes d’équipe
</div>

Les administrateurs d’équipe peuvent configurer des alertes pour l’équipe sur la page des paramètres de l’équipe : `wandb.ai/teams/your-team`.

Les alertes d’équipe s’appliquent à tous les membres de votre équipe. W&amp;B recommande d’utiliser le canal **Slackbot**, car il permet de garder les alertes privées.

<div id="change-slack-channel-to-send-alerts-to">
  ## Modifier le canal Slack auquel envoyer les alertes
</div>

Pour modifier le canal auquel les alertes sont envoyées, cliquez sur **Disconnect Slack**, puis reconnectez Slack. Une fois la reconnexion effectuée, choisissez un autre canal Slack.
