Dans un Workspace comportant de nombreux runs, il peut être difficile de suivre vos meilleures performances, vos modèles de Production, vos expériences ayant échoué ou vos points de référence importants. L’application W&amp;B fournit des fonctionnalités pour vous aider à organiser et à comparer les runs :

* **Runs épinglés** : Épinglez jusqu’à 6 runs pour les garder visibles dans le Workspace et en haut de la liste des runs. Si vous avez un run de référence, vous pouvez épingler jusqu’à 5 runs, car le run de référence est implicitement épinglé.
* **Run de référence** : Spécifiez un run de référence comme point de comparaison pour les comparaisons. Le run de référence est toujours visible dans le Workspace et en haut de la liste des runs. Dans le tableau Runs, les écarts des métriques de synthèse montrent comment chaque run se compare au run de référence. Dans les graphiques en courbes, le run de référence apparaît avec un style visuel distinct pour faciliter la comparaison.

![Graphique en courbes avec run de référence et runs épinglés](/images/models/pinned-and-baseline-runs/baseline-run-visual-distinction.png)

Ces fonctionnalités sont particulièrement utiles pour :

* Comparer de nouvelles expériences à votre modèle de Production.
* Suivre plusieurs modèles candidats pendant l’expérimentation.
* Évaluer si de nouveaux runs améliorent vos meilleurs résultats.

Voir [Limites](#limitations).

<div id="manage-the-baseline-run">
  ## Gérer le run de référence
</div>

Vous pouvez définir un run comme run de référence pour l’espace de travail afin de l’utiliser comme point de comparaison pour évaluer les autres runs de votre espace de travail.

Dans le sélecteur de runs et le tableau Runs, le run de référence apparaît en haut, à côté des runs épinglés, et affiche une icône de signet au lieu d’une épingle.

Dans les graphiques en courbes, les lignes du run de référence apparaissent plus épaisses que les autres. Lorsque vous survolez le graphique ou la légende, la ligne du run de référence s’affiche en pointillés.

<Frame>
  ![Démonstration de la comparaison entre un autre run et le run de référence](/images/models/pinned-and-baseline-runs/line-plot-baseline-comparison.png)
</Frame>

<div id="set-a-baseline-run">
  ### Définir un run de référence
</div>

Pour définir un run de référence :

1. Accédez à votre espace de travail.
2. Dans le sélecteur de runs ou le tableau Runs, trouvez le run que vous souhaitez utiliser comme run de référence.
3. Cliquez sur le menu d’action `...`, puis sélectionnez **Définir comme référence**.

Le run de référence apparaît en haut du sélecteur de runs, séparé des autres runs par une ligne de séparation visuelle. Le run de référence affiche une icône de signet au lieu d’un cercle.

<Frame>
  ![Tableau Runs avec un run de référence et des runs épinglés](/images/models/pinned-and-baseline-runs/runs-table-with-pinned-and-baseline-runs.png)
</Frame>

<div id="change-the-baseline-run">
  ### Modifier le run de référence
</div>

Un seul run peut servir de référence à la fois. Pour modifier le run de référence :

1. Accédez à votre espace de travail.
2. Dans le sélecteur de runs ou le tableau Runs, trouvez le run que vous souhaitez utiliser comme nouveau run de référence.
3. Cliquez sur le menu d’action `...`, puis sélectionnez **Remplacer le run de référence**. <Note>Si l’élément de menu est grisé, assurez-vous d’avoir au moins un emplacement d’épinglage disponible. Si nécessaire, désépinglez un run épinglé en cliquant sur l’icône d’épingle circulaire à côté de ce run.</Note>
4. Le nouveau run devient le run de référence, et l’ancien run de référence est automatiquement épinglé pour que vous puissiez le retrouver facilement. Si vous le souhaitez, désépinglez-le en cliquant sur son icône d’épingle.

<div id="remove-the-baseline-designation">
  ### Supprimer le statut de référence
</div>

Pour supprimer le statut de référence :

1. Accédez à votre espace de travail.
2. Dans le sélecteur de runs ou le tableau Runs, trouvez le run actuellement défini comme référence.
3. Cliquez sur le menu d’action `...`, puis sélectionnez **Supprimer la référence**. <Note>Si cette option de menu est inactive, assurez-vous de disposer d’au moins un emplacement d’épinglage libre. Si nécessaire, désépinglez un run épinglé en cliquant sur l’icône d’épingle circulaire à côté de ce run.</Note>
4. L’ancien run de référence est automatiquement épinglé pour que vous puissiez le retrouver facilement. Si vous le souhaitez, désépinglez-le en cliquant sur son icône d’épingle.

* Survoler une partie du graphique pour afficher une info-bulle avec les valeurs de tous les runs visibles, y compris le run de référence et les runs épinglés.

<div id="summary-metric-deltas">
  ### Écarts des métriques de synthèse
</div>

Lorsqu’un run est défini comme run de référence, tout autre run qui journalise la même métrique de synthèse affiche, par défaut, l’écart (ampleur de la variation) de cette métrique par rapport au run de référence. Cet écart apparaît à droite de la valeur de la métrique, sur la ligne du run dans le tableau Runs.

Par défaut, l’écart s’affiche en gris foncé sur un fond gris foncé. Pour activer une coloration sémantique qui serve de repère visuel rapide, vous pouvez définir la **directionnalité de la métrique** pour une colonne. Une fois la directionnalité définie :

* Si l’autre run **surpasse** la référence (**est directionnellement meilleur**), l’écart s’affiche en rouge foncé sur un fond rouge clair.
* Si l’autre run **est moins performant** que la référence (**est directionnellement moins bon**), l’écart s’affiche en sarcelle foncé sur un fond sarcelle clair.

Pour définir la directionnalité d’une métrique :

1. Dans le tableau Runs, survoler l’en-tête de la colonne de la métrique.
2. Cliquer sur le menu d’actions `...` qui apparaît.
3. Définir **Directionnalité de la métrique** sur **Les valeurs plus élevées sont meilleures** ou **Les valeurs plus faibles sont meilleures**.

La capture d’écran suivante montre comment les runs `nanochat-train-base` et `nanochat-train-mid` se comparent au run de référence `nanochat-train`. Des écarts de métriques sont affichés pour `TOTAL_TRAINING_TIME`, `TRAIN/DT` et `TRAIN/GRAD_NORM`.
![Capture d’écran comparant les écarts des métriques de synthèse du run de référence](/images/models/pinned-and-baseline-runs/baseline_run_deltas.png)

<div id="hide-summary-metric-deltas-in-a-workspace">
  ## Masquer les écarts des métriques de synthèse dans un espace de travail
</div>

Par défaut, un espace de travail avec un run de référence affiche toujours les écarts des métriques de synthèse. Pour les masquer dans un espace de travail :

1. Dans l’espace de travail, cliquez sur **Paramètres**.
2. Dans le volet latéral qui s’ouvre, cliquez sur **Runs**.
3. Dans l’onglet **Baseline**, désactivez **Afficher les écarts de valeurs dans le tableau Runs**.
4. Fermez le volet latéral des paramètres de l’espace de travail.

<div id="use-cases">
  ## Cas d&#39;utilisation
</div>

Cette section décrit quelques scénarios dans lesquels les runs épinglés et les runs de référence peuvent vous aider à orienter vos expériences.

* **Suivre les modèles en production** : assurez-vous que les nouveaux modèles atteignent votre niveau d&#39;exigence en matière de qualité avant le déploiement.
  1. Définissez votre modèle de production comme référence.
  2. Comparez toutes les expériences à votre modèle déployé afin d&#39;identifier les candidates qui surpassent le modèle en production.
* **Comparer des expériences d&#39;hyperparamètres** : évaluez les balayages d&#39;hyperparamètres ou les expériences manuelles par rapport à votre meilleure configuration connue.
  1. Définissez votre meilleure configuration connue comme référence.
  2. Épinglez les candidates prometteuses au fur et à mesure que vous les découvrez.
  3. Utilisez les graphiques en courbes pour comparer visuellement les runs à la référence.
  4. Continuez à mettre à jour la référence à mesure que vous trouvez de meilleures configurations.

<div id="example-workflow">
  ## Exemple de flux de travail
</div>

Cette section montre comment les runs épinglés et les runs de référence peuvent vous aider à comparer des runs.

1. Exécutez cet exemple de code, qui simule un scénario d’ajustement d’hyperparamètres avec une série de runs. Remplacez les espaces réservés entourés de chevrons (`<>`) par vos propres valeurs.

   ```python
   import wandb
   import random
   import math

   def train_model(learning_rate, batch_size, run_name, tags=None):
       """Simuler l’entraînement d’un modèle avec les hyperparamètres donnés."""
       config = {
           "learning_rate": learning_rate,
           "batch_size": batch_size,
           "optimizer": "adam",
           "architecture": "resnet50"
       }
       
       with wandb.init(
         # Remplacez par le nom de votre équipe et de votre projet
           project="hyperparameter-tuning",
           entity="<team>",
           name=run_name,
           config=config,
           tags=tags or []
       ) as run:
           # Simuler la boucle d’entraînement
           for epoch in range(50):
               # Métriques simulées
               accuracy = 0.6 + 0.3 * (1 - math.exp(-learning_rate * epoch / 10))
               loss = 1.0 * math.exp(-learning_rate * epoch / 10)
               
               run.log({
                   "epoch": epoch,
                   "accuracy": accuracy,
                   "loss": loss
               })

   # Créer un run de référence avec la configuration standard
   train_model(
       learning_rate=0.001,
       batch_size=64,
       run_name="baseline-config",
       tags=["baseline", "production"]
   )

   # Expérimenter avec différents taux d’apprentissage
   train_model(
       learning_rate=0.003,
       batch_size=64,
       run_name="lr-experiment-0.003",
       tags=["experiment"]
   )

   train_model(
       learning_rate=0.0001,
       batch_size=64,
       run_name="lr-experiment-0.0001",
       tags=["experiment"]
   )
   ```

   Après avoir exécuté ce code, votre espace de travail contient trois runs.
2. Définissez `baseline-config` comme run de référence.
3. Épinglez `baseline-config` pour le garder visible.
4. Comparez les runs d’expérimentation au run de référence.

* Dans le tableau Runs, examinez les écarts des métriques de synthèse à côté des valeurs de chaque run afin de comparer le run au run de référence.
  * Dans les graphiques en courbes, comparez les performances d’un ou de plusieurs runs au run de référence, qui reste toujours visible.

5. Épinglez les expériences prometteuses pour approfondir l’analyse. Dans cet exemple, après 50 époques, `lr-experiment-0.003` présente la précision la plus élevée (`~0.64`) et la perte la plus faible (`~0.86`).

<div id="limitations">
  ## Limitations
</div>

Les fonctionnalités suivantes ne sont pas encore prises en charge pour les runs épinglés et de référence :

* **Regroupement** : Lors de l’[affichage des runs](/fr/models/runs#view-logged-runs) dans le sélecteur de runs ou le tableau Runs, si les runs sont regroupés selon une colonne, les runs épinglés et de référence ne se distinguent pas visuellement des autres runs.
* **Reports** : Dans un ensemble de runs d’un [W&amp;B Report](/fr/models/reports), les runs épinglés et de référence ne se distinguent pas visuellement des autres runs.
* **Vue Workspace uniquement** : Le run de référence n’apparaît pas lors de l’affichage de l’espace de travail d’un seul run.
* **Graphiques en courbes uniquement** : La comparaison avec le run de référence est disponible uniquement pour les graphiques en courbes et n’est pas encore proposée pour les autres panneaux, comme les graphiques à barres ou les panneaux multimédias.
