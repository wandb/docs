Dans un espace de travail comportant de nombreux runs, il peut être difficile de garder une vue d’ensemble de vos runs les plus performants, de vos modèles de production, de vos expériences ayant échoué ou de vos points de référence importants. L’application W&amp;B propose des fonctionnalités pour vous aider à organiser et comparer les runs :

* **Runs épinglés** : épinglez jusqu’à 6 runs pour les garder visibles dans l’espace de travail et en haut de la liste des runs. Si vous avez un run de référence, vous pouvez épingler jusqu’à 5 runs, car le run de référence est implicitement épinglé.
* **Run de référence** : définissez un run de référence comme point de comparaison. Le run de référence est toujours visible dans l’espace de travail et en haut de la liste des runs. Dans le tableau des runs, les écarts des métriques de synthèse montrent comment chaque run se compare au run de référence. Dans les graphiques en courbes, le run de référence apparaît avec un style visuel distinct afin de faciliter la comparaison.

![Graphique en courbes avec run de référence et runs épinglés](/images/m
odels/pinned-and-baseline-runs/baseline-run-visual-distinction.png)

Ces fonctionnalités sont particulièrement utiles pour :

* Comparer de nouvelles expériences à votre modèle de production.
* Suivre plusieurs modèles candidats pendant l’expérimentation.
* Évaluer si de nouveaux runs améliorent vos meilleurs résultats.

Voir [Limitations](#limitations).

<div id="manage-the-baseline-run">
  ## Gérer le run de référence
</div>

Vous pouvez définir un run comme référence afin que l’espace de travail l’utilise comme point de comparaison pour évaluer les autres runs de votre espace de travail.

Dans le sélecteur de runs et le tableau des runs, le run de référence apparaît en haut, à côté des runs épinglés, et affiche une icône de signet au lieu d’une épingle.

Dans les graphiques en courbes, la ligne du run de référence apparaît plus épaisse que les autres. Lorsque vous survolez le graphique ou la légende, la ligne du run de référence s’affiche en pointillés.

<Frame>
  ![Démo de comparaison d’un autre run avec le run de référence](/images/models/pinned-and-baseline-runs/line-plot-baseline-comparison.png)
</Frame>

<div id="set-a-baseline-run">
  ### Définir un run de référence
</div>

Pour définir un run de référence :

1. Accédez à votre espace de travail.
2. Dans le sélecteur de runs ou le tableau des runs, trouvez le run que vous voulez utiliser comme référence.
3. Cliquez sur le menu d’actions `...`, puis sélectionnez **Définir comme référence**.

Le run de référence apparaît en haut du sélecteur de runs, séparé des autres runs par un séparateur visuel. Le run de référence affiche une icône de signet à la place d’un cercle.

<Frame>
  ![Tableau des runs avec un run de référence et des runs épinglés](/images/models/pinned-and-baseline-runs/runs-table-with-pinned-and-baseline-runs.png)
</Frame>

<div id="change-the-baseline-run">
  ### Modifier le run de référence
</div>

Un seul run peut servir de référence à la fois. Pour changer le run de référence :

1. Accédez à votre espace de travail.
2. Dans le sélecteur de runs ou le tableau des runs, trouvez le run que vous souhaitez utiliser comme nouvelle référence.
3. Cliquez sur le menu d’actions `...`, puis sélectionnez **Replace baseline**. <Note>Si l’élément de menu est inactif, assurez-vous d’avoir au moins un emplacement d’épinglage disponible. Si nécessaire, désépinglez un run épinglé en cliquant sur l’icône d’épingle circulaire à côté de ce run.</Note>
4. Le nouveau run devient le run de référence, et le précédent run de référence est automatiquement épinglé pour que vous puissiez le retrouver facilement. Facultativement, vous pouvez le désépingler en cliquant sur son icône d’épingle.

<div id="remove-the-baseline-designation">
  ### Retirer le statut de référence
</div>

Pour retirer le statut de référence :

1. Accédez à votre espace de travail.
2. Dans le sélecteur de runs ou le tableau des runs, trouvez le run de référence actuel.
3. Cliquez sur le menu d’actions `...`, puis sélectionnez **Remove baseline**. <Note>Si l’élément du menu est grisé, assurez-vous qu’au moins un emplacement d’épinglage est disponible. Si nécessaire, désépinglez un run épinglé en cliquant sur l’icône d’épingle circulaire à côté de ce run.</Note>
4. L’ancien run de référence est automatiquement épinglé pour que vous puissiez le retrouver facilement. Si vous le souhaitez, désépinglez-le en cliquant sur son icône d’épingle.

* Survolez une partie du graphique pour afficher une info-bulle avec les valeurs de tous les runs visibles, y compris le run de référence et les runs épinglés.

<div id="summary-metric-deltas">
  ### Écart des métriques de synthèse
</div>

Lorsqu’un run est défini comme référence, chaque autre run qui journalise la même métrique récapitulative que le run de référence affiche par défaut l’écart des métriques de synthèse (la variation) de cette métrique par rapport à la référence. L’écart des métriques de synthèse apparaît à droite de la valeur de la métrique, dans la ligne du run du tableau des runs.

Par défaut, l’écart des métriques de synthèse s’affiche en gris foncé sur fond gris foncé. Pour activer une couleur sémantique afin d’obtenir un repère visuel rapide, vous pouvez définir la **directionnalité de la métrique** pour une colonne. Une fois la directionnalité définie :

* Si l’autre run **surpasse** (est meilleur selon la direction définie) la référence, l’écart des métriques de synthèse s’affiche en rouge foncé sur fond rouge clair.
* Si l’autre run **est moins performant** (est moins bon selon la direction définie) que la référence, l’écart des métriques de synthèse s’affiche en bleu sarcelle foncé sur fond bleu sarcelle clair.

Pour définir la directionnalité d’une métrique :

1. Dans le tableau des runs, survolez l’en-tête de colonne de la métrique.
2. Cliquez sur le menu d’actions `...` qui apparaît.
3. Définissez **Directionnalité de la métrique** sur **Les valeurs les plus élevées sont préférables** ou **Les valeurs les plus faibles sont préférables**.

La capture d’écran suivante montre comment les runs `nanochat-train-base` et `nanochat-train-mid` se comparent au run de référence `nanochat-train`. Des écarts des métriques de synthèse sont affichées pour `TOTAL_TRAINING_TIME`, `TRAIN/DT` et `TRAIN/GRAD_NORM`.
![Capture d’écran comparant les écarts des métriques de synthèse du run de référence](/images/models/pinned-and-baseline-runs/baseline_run_deltas.png)

<div id="hide-summary-metric-deltas-in-a-workspace">
  ## Masquer les écarts des métriques de synthèse dans un espace de travail
</div>

Par défaut, un espace de travail avec un run de référence affiche toujours les écarts des métriques de synthèse. Pour les masquer dans un espace de travail :

1. Dans l&#39;espace de travail, cliquez sur **Settings**.
2. Dans le panneau qui s&#39;affiche, cliquez sur **Runs**.
3. Dans l&#39;onglet **Baseline**, désactivez **Show value deltas in the runs table**.
4. Fermez le panneau des paramètres de l&#39;espace de travail.

<div id="use-cases">
  ## Cas d’usage
</div>

Cette section décrit quelques scénarios dans lesquels les runs épinglés et les runs de référence peuvent vous aider à orienter vos expériences.

* **Suivre les modèles de Production** : Assurez-vous que les nouveaux modèles répondent à vos exigences de qualité avant le déploiement.
  1. Définissez votre modèle de Production comme référence.
  2. Comparez toutes les expériences à votre modèle déployé afin d’identifier les candidats qui surpassent ceux de Production.
* **Comparer des expériences d’hyperparamètres** : Évaluez les Sweeps d’hyperparamètres ou les expériences manuelles par rapport à votre meilleure configuration connue.
  1. Définissez votre meilleure configuration connue comme référence.
  2. Épinglez les candidats prometteurs à mesure que vous les identifiez.
  3. Utilisez les graphiques en courbes pour comparer visuellement les runs à la référence.
  4. Continuez à mettre à jour la référence à mesure que vous trouvez de meilleures configurations.

<div id="example-workflow">
  ## Exemple de flux de travail
</div>

Cette section montre comment les runs épinglés et les runs de référence peuvent vous aider à comparer des runs.

1. Exécutez cet exemple de code, qui simule un scénario d’ajustement d’hyperparamètres avec une série de runs. Remplacez les espaces réservés entourés de chevrons (`<>`) par vos propres valeurs.

   ```python
   import wandb
   import random
   import math

   def train_model(learning_rate, batch_size, run_name, tags=None):
       """Simuler l’entraînement d’un modèle avec des hyperparamètres donnés."""
       config = {
           "learning_rate": learning_rate,
           "batch_size": batch_size,
           "optimizer": "adam",
           "architecture": "resnet50"
       }
       
       with wandb.init(
         # Remplacez par le nom de votre équipe et de votre projet
           project="hyperparameter-tuning",
           entity="<team>",
           name=run_name,
           config=config,
           tags=tags or []
       ) as run:
           # Simuler la boucle d’entraînement
           for epoch in range(50):
               # Métriques simulées
               accuracy = 0.6 + 0.3 * (1 - math.exp(-learning_rate * epoch / 10))
               loss = 1.0 * math.exp(-learning_rate * epoch / 10)
               
               run.log({
                   "epoch": epoch,
                   "accuracy": accuracy,
                   "loss": loss
               })

   # Créer un run de référence avec une configuration standard
   train_model(
       learning_rate=0.001,
       batch_size=64,
       run_name="baseline-config",
       tags=["baseline", "production"]
   )

   # Expérimenter avec différents taux d’apprentissage
   train_model(
       learning_rate=0.003,
       batch_size=64,
       run_name="lr-experiment-0.003",
       tags=["experiment"]
   )

   train_model(
       learning_rate=0.0001,
       batch_size=64,
       run_name="lr-experiment-0.0001",
       tags=["experiment"]
   )
   ```

   Après avoir exécuté ce code, votre espace de travail contient trois runs.
2. Définissez `baseline-config` comme run de référence.
3. Épinglez `baseline-config` pour qu’il reste visible.
4. Comparez les runs d’expérience au run de référence.

* Dans le tableau des runs, examinez les écarts des métriques de synthèse affichés à côté des valeurs de chaque run pour comparer le run à la référence.
  * Dans les graphiques en courbes, comparez les performances d’un ou de plusieurs runs à la référence, qui reste toujours visible.

5. Épinglez les expériences prometteuses pour les examiner plus en détail. Dans cet exemple, après 50 époques, `lr-experiment-0.003` a la précision la plus élevée (`~0.64`) et la perte la plus faible (`~0.86`).

<div id="limitations">
  ## Limitations
</div>

Les fonctionnalités suivantes ne sont pas encore prises en charge pour les runs épinglés et les runs de référence :

* **Regroupement** : lorsque vous [affichez les runs](/fr/models/runs#view-logged-runs) dans le sélecteur de runs ou dans le tableau des runs, si les runs sont regroupés par colonne, les runs épinglés et les runs de référence ne se distinguent pas visuellement des autres runs.
* **Reports** : dans un ensemble de runs d’un [W&amp;B Report](/fr/models/reports), les runs épinglés et les runs de référence ne se distinguent pas visuellement des autres runs.
* **Vue espace de travail uniquement** : le run de référence n’apparaît pas lorsque vous affichez l’espace de travail d’un seul run.
* **Graphiques en courbes uniquement** : la comparaison avec le run de référence est disponible uniquement pour les graphiques en courbes et n’est pas encore disponible pour les autres panneaux, tels que les graphiques à barres ou les panneaux multimédias.
