---
title: Trouver et personnaliser l'ID ou le nom d'un run
description: Découvrez comment trouver l'ID unique et le nom d'un run, créer un ID du run personnalisé et personnaliser le nom d'un run.
---

Lorsque vous initialisez un run W&amp;B, W&amp;B attribue à ce run un [ID unique appelé *ID du run*](/fr/models/runs/run-identifiers#run-id). Chaque run possède également un [*nom de l&#39;exécution* non unique](/fr/models/runs/run-identifiers#run-name) que vous pouvez personnaliser.

<div id="run-id">
  ## ID du run
</div>

L’ID d’un run identifie de manière unique le run. Par défaut, W&amp;B génère automatiquement un [ID du run aléatoire et unique](#autogenerated-run-ids) lorsque vous initialisez un nouveau run, sauf si vous [spécifiez votre propre ID du run unique](#create-a-custom-run-id) lorsque vous [initialisez le run](/fr/models/runs/initialize-run).

<div id="find-a-runs-id">
  ### Trouver l&#39;ID d&#39;un run
</div>

Trouvez l&#39;ID unique d&#39;un run programmatiquement avec le SDK Python W&amp;B ou de manière interactive dans la W&amp;B App.

<Tabs>
  <Tab title="SDK Python">
    Lorsque vous initialisez un run, W&amp;B renvoie l&#39;ID unique du run dans le terminal. Par exemple, considérez l&#39;extrait de code suivant, qui initialise un run W&amp;B :

    ```python
    import wandb
    entity = "nico"  # Remplacez par votre entité W&B
    project = "awesome-project" 
    with wandb.init(entity=entity, project=project) as run:
        # Votre code ici
    ```

    Dans le terminal, W&amp;B renvoie :

    ```bash
    wandb: Syncing run earnest-sunset-1
    wandb: ⭐️ View project at https://wandb.ai/nico/awesome-project
    wandb: 🚀 View run at https://wandb.ai/nico/awesome-project/runs/1jx1ud12
    ```

    La dernière partie de l&#39;URL du run (`1jx1ud12`) est l&#39;ID unique du run.
  </Tab>

  <Tab title="W&B App">
    Vous pouvez également trouver l&#39;ID unique d&#39;un run dans la W&amp;B App :

    1. Accédez à la [W&amp;B App](https://wandb.ai/home).
    2. Accédez au projet W&amp;B que vous avez spécifié lorsque vous avez initialisé le run.
    3. Dans l&#39;espace de travail de votre projet, sélectionnez l&#39;onglet  **Workspace** ou **Runs**.
    4. Sélectionnez le run que vous voulez afficher.
    5. Choisissez l&#39;onglet **Aperçu**.

    W&amp;B affiche l&#39;ID du run dans le champ **Run path**. Le chemin du run se compose du nom de votre équipe, du nom du projet et de l&#39;ID du run. L&#39;ID unique est la dernière partie du chemin du run.

    Par exemple, dans l&#39;image suivante, l&#39;ID unique du run est `9mxi1arc` :

    <Frame>
      <img src="/images/runs/unique-run-id.png" alt="Emplacement de l'ID du run" />
    </Frame>
  </Tab>
</Tabs>

<Tip>
  Utilisez l&#39;ID unique d&#39;un run pour accéder directement à sa page d&#39;aperçu dans la W&amp;B App. Le bloc de code suivant montre le format du chemin d&#39;URL d&#39;un run :

  ```text title="URL de la W&B App pour un run spécifique"
  https://wandb.ai/<entity>/<project>/<run-id>
  ```

  Remplacez les valeurs entre chevrons (`< >`) par les valeurs réelles de l&#39;entité, du projet et de l&#39;ID du run.
</Tip>

<div id="create-a-custom-run-id">
  ### Créer un ID du run personnalisé
</div>

Indiquez l’ID du run souhaité sous forme de chaîne de caractères dans le paramètre `id` lorsque vous initialisez un run :

```python
import wandb

with wandb.init(entity="<project>", project="<project>", id="<run-id>") as run:
    # Votre code ici
```

<div id="run-name">
  ## Nom de l&#39;exécution
</div>

Chaque run possède un nom lisible par l&#39;humain, qui n&#39;est pas unique. Par défaut, W&amp;B génère un nom de l&#39;exécution aléatoire lorsque vous initialisez un nouveau run si vous n&#39;en spécifiez pas. Le nom de l&#39;exécution apparaît dans le Workspace de votre projet et en haut de la page [**Aperçu** du run](#overview-tab).

En reprenant l&#39;exemple précédent, le nom de l&#39;exécution est `glowing-shadows-8`.

<Frame>
  <img src="/images/runs/unique-run-id.png" alt="Emplacement de l'ID du run" />
</Frame>

Vous pouvez nommer votre run lorsque [vous l&#39;initialisez](/fr/models/runs/run-identifiers#create-a-custom-run-name) ou le [renommer](/fr/models/runs/run-identifiers#rename-a-run) plus tard.

<div id="create-a-custom-run-name">
  ### Créer un nom de run personnalisé
</div>

Indiquez un nom pour votre run en passant le paramètre `name` à la méthode [`wandb.init()`](/fr/models/ref/python/functions/init).

```python
import wandb

with wandb.init(entity="<project>", project="<project>", name="<run-name>") as run:
    # Votre code ici
```

<div id="rename-a-run">
  ### Renommer un run
</div>

Renommez un run après l’avoir initialisé par programmation avec le SDK Python ou de manière interactive dans la W&amp;B App.

<Tabs>
  <Tab title="SDK Python">
    Utilisez [`wandb.Api.Run`](/fr/models/ref/python/public-api/api#method-api-run) pour accéder à un run consigné dans W&amp;B. Cette méthode renvoie un [objet run](/fr/models/ref/python/public-api/run#property-run-name) que vous pouvez utiliser pour mettre à jour le nom de l&#39;exécution. Appelez la méthode `wandb.Api.Run.update()` pour enregistrer les modifications.

    Remplacez les valeurs entre chevrons (`< >`) par vos propres valeurs.

    ```python
    import wandb

    api = wandb.Api()

    # Accéder au run par son chemin
    run = api.run(path = "<entity>/<project>/<run-id>")

    # Définir un nouveau nom d'exécution
    run.name = "<new-run-name>"
    run.update()
    ```
  </Tab>

  <Tab title="W&B App">
    1. Accédez à votre projet W&amp;B.
    2. Sélectionnez l’onglet **Workspace** ou **Runs**.
    3. Recherchez le run que vous souhaitez renommer ou faites défiler la page jusqu’à celui-ci.
    4. Survolez le nom du run, cliquez sur les trois points verticaux, puis sur **Rename run**.
    5. Pour modifier le nom du run, mettez à jour le champ **Run name**.
    6. Cliquez sur **Save**.
  </Tab>
</Tabs>

<div id="run-display-name">
  ## Nom d&#39;affichage du run
</div>

Chaque run possède également un *nom d&#39;affichage du run* que vous pouvez personnaliser dans chaque Workspace.

<Note>
  Si vous modifiez le nom d&#39;affichage d&#39;un run dans un Workspace, ce changement s&#39;applique uniquement à ce Workspace, et non aux autres Workspaces ni aux autres projets.
</Note>

Par défaut, le nom d&#39;affichage a la même valeur que le nom de l&#39;exécution. Le nom d&#39;affichage apparaît dans le Workspace du run et dans le tableau des runs.

Utilisez le nom d&#39;affichage du run pour remplacer le nom de l&#39;exécution affiché dans ce Workspace, sans renommer le run dans le projet.

<div id="rename-a-runs-display-name">
  ### Renommer le nom d’affichage d’un run
</div>

Modifiez le nom d’affichage d’un run depuis la W&amp;B App :

1. Accédez à votre projet W&amp;B.
2. Sélectionnez l’onglet **Workspace** ou **Runs**.
3. Recherchez le run que vous souhaitez renommer, ou faites défiler la page jusqu’à celui-ci.
4. Survolez le nom de l&#39;exécution, cliquez sur les trois points verticaux, puis sur **Rename run**.
5. Saisissez une nouvelle valeur dans le champ **Display name**.
6. Cliquez sur **Save**.

<div id="customize-run-name-truncation">
  ## Personnaliser la troncature du nom de l&#39;exécution
</div>

Par défaut, les noms de l&#39;exécution trop longs sont tronqués au milieu pour une meilleure lisibilité. Pour personnaliser la troncature des noms de l&#39;exécution :

1. Cliquez sur le menu d’action `...` en haut de la liste des runs.
2. Réglez **Run name cropping** pour tronquer à la fin, au milieu ou au début.
