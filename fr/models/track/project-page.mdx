Un *projet* est un espace central où vous pouvez visualiser les résultats, comparer les expériences, consulter et télécharger des artefacts, créer une automatisation, et bien plus encore.

<Note>
  Chaque projet dispose d&#39;un paramètre de visibilité qui détermine qui peut y accéder. Pour plus d&#39;informations sur les personnes pouvant accéder à un projet, voir [Visibilité du projet](/fr/platform/hosting/iam/access-management/restricted-projects).
</Note>

Chaque projet contient les onglets suivants :

* [Aperçu](/fr/models/track/project-page/#overview-tab) : aperçu de votre projet
* [Workspace](/fr/models/track/project-page/#workspace-tab) : espace personnel de visualisation
* [Runs](#runs-tab) : tableau répertoriant tous les runs de votre projet
* [Automations](#automations-tab) : automatisations configurées dans votre projet
* [Sweeps](/fr/models/sweeps) : exploration et optimisation automatisées
* [Reports](/fr/models/track/project-page/#reports-tab) : instantanés enregistrés des notes, des runs et des graphiques
* [Artifacts](#artifacts-tab) : contient tous les runs et les artefacts qui leur sont associés

<div id="overview-tab">
  ## onglet Aperçu
</div>

W&amp;B crée un projet pour vous lorsque vous initialisez un run avec le nom indiqué dans le champ du projet.

Depuis l’onglet **Aperçu**, vous pouvez trouver le nom du projet et le gérer.

* Pour modifier le nom, la description ou l’équipe du projet, cliquez sur **Edit** dans le coin supérieur droit.
* Pour restaurer des runs récemment supprimés, cliquez sur **...** dans le coin supérieur droit, puis sur **Undelete recently deleted runs**.
* Pour supprimer le projet, cliquez sur **...** dans le coin supérieur droit, puis sur **supprimer le projet**. Lisez la boîte de dialogue de confirmation et suivez les instructions. La suppression d’un projet est irréversible.

Le reste de la page **Aperçu** est divisé en onglets **Détails** et **Rôles du projet**.

<div id="details">
  ### Détails
</div>

Les détails du projet comprennent :

* **Visibilité du projet** : la visibilité du projet, c’est-à-dire le paramètre qui détermine qui peut y accéder. Voir [la visibilité du projet](/fr/platform/hosting/iam/access-management/restricted-projects) pour plus d’informations.
* **Dernière activité** : Horodatage de la dernière fois où des données ont été enregistrées dans ce projet
* **Contributeurs** : le nombre d’utilisateurs qui contribuent à ce projet
* **Total des runs** : le nombre total de runs dans ce projet
* **Temps de calcul total** : nous additionnons tous les temps d’exécution des runs de votre projet pour obtenir ce total

L’onglet **Détails** inclut également des instructions pour accéder aux données et aux métriques du projet, les exporter, télécharger le meilleur modèle d’un balayage, et plus encore.

<div id="workspace-tab">
  ## Onglet Workspace
</div>

L&#39;*Workspace* d&#39;un projet vous offre un espace personnel pour comparer des expériences. Utilisez les projets pour organiser des modèles comparables, qui traitent le même problème avec des architectures, hyperparamètres, jeux de données, prétraitements, etc. différents.

**Barre latérale Runs** : liste de tous les runs de votre projet.

* **Menu à points** : survolez une ligne dans la barre latérale pour faire apparaître le menu sur la gauche. Utilisez ce menu pour renommer un run, supprimer un run ou arrêter un run actif.
* **Icône de visibilité** : cliquez sur l&#39;œil pour afficher ou masquer des runs sur les graphiques
* **Couleur** : modifiez la couleur du run en choisissant l&#39;un de nos préréglages ou une couleur personnalisée
* **Recherche** : recherchez des runs par nom. Cela filtre également les runs visibles dans les graphiques.
* **Filtre** : utilisez le filtre de la barre latérale pour restreindre l&#39;ensemble des runs visibles
* **Grouper** : sélectionnez une colonne de configuration pour regrouper dynamiquement vos runs, par exemple par architecture. Le regroupement ajoute aux graphiques une ligne représentant la valeur moyenne, ainsi qu&#39;une zone ombrée correspondant à la variance des points du graphique.
* **Trier** : choisissez une valeur selon laquelle trier vos runs, par exemple les runs avec la perte la plus faible ou la précision la plus élevée. Le tri affecte les runs affichés sur les graphiques.
* **Bouton d&#39;extension** : développez la barre latérale pour afficher le tableau complet
* **Réduire** : appuyez sur **Cmd+.** (macOS) ou **Ctrl+.** (Windows/Linux) pour réduire ou restaurer le sélecteur de Runs. Voir [Raccourcis clavier](/fr/models/app/keyboard-shortcuts) pour plus de détails.
* **Nombre de runs** : le nombre entre parenthèses en haut correspond au nombre total de runs dans le projet. Le nombre (N visualisés) correspond au nombre de runs dont l&#39;œil est activé et qui peuvent être visualisés dans chaque graphique. Dans l&#39;exemple ci-dessous, les graphiques n&#39;affichent que les 10 premiers runs sur 183. Modifiez un graphique pour augmenter le nombre maximal de runs visibles.

Si vous épinglez, masquez ou modifiez l&#39;ordre des colonnes dans l&#39;[onglet Runs](#runs-tab), la barre latérale Runs reflète ces personnalisations.

**Mise en page des panneaux** : utilisez cet espace pour explorer les résultats, ajouter et supprimer des graphiques, et comparer des versions de vos modèles en fonction de différentes métriques

<div id="add-a-section-of-panels">
  ### Ajouter une section de panneaux
</div>

Cliquez sur le menu déroulant de la section, puis sur &quot;Ajouter une section&quot; pour créer une nouvelle section de panneaux. Vous pouvez renommer les sections, les faire glisser pour les réorganiser, et les déplier ou les replier.

Chaque section comporte des options dans le coin supérieur droit :

* **Ajouter une section** : ajoutez une section au-dessus ou en dessous à partir du menu déroulant, ou cliquez sur le bouton en bas de la page pour ajouter une nouvelle section.
* **Renommer la section** : modifiez le titre de votre section.
* **Exporter la section vers un rapport** : enregistrez cette section de panneaux dans un nouveau rapport.
* **Supprimer la section** : supprimez toute la section et tous les graphiques. Vous pouvez annuler cette action avec le bouton Annuler en bas de la page, dans la barre du Workspace.
* **Ajouter un panneau** : cliquez sur le bouton + pour ajouter un panneau à la section.

<div id="create-a-project">
  ## Créer un projet
</div>

Vous pouvez créer un projet dans la W&amp;B App ou par programmation en spécifiant un projet dans un appel à `wandb.init()`.

<Tabs>
  <Tab title="W&B App">
    Dans la W&amp;B App, vous pouvez créer un projet depuis la page **Projects** ou depuis la page d’accueil d’une équipe.

    Depuis la page **Projects** :

    1. Cliquez sur l’icône de navigation globale en haut à gauche. La barre latérale du projet s’ouvre.
    2. Dans la section **Projects** de la navigation, cliquez sur **View all** pour ouvrir la page d’aperçu des projets.
    3. Cliquez sur **Create new project**.
    4. Définissez **Team** sur le nom de l’équipe qui sera propriétaire du projet.
    5. Indiquez un nom pour votre projet dans le champ **Name**.
    6. Définissez **visibilité du projet**, dont la valeur par défaut est **Team**.
    7. Facultativement, ajoutez une **Description**.
    8. Cliquez sur **Create project**.

    Depuis la page d’accueil d’une équipe :

    1. Cliquez sur l’icône de navigation globale en haut à gauche. La barre latérale du projet s’ouvre.
    2. Dans la section **Teams** de la navigation, cliquez sur le nom d’une équipe pour ouvrir sa page d’accueil.
    3. Sur la page d’accueil, cliquez sur **Create new project**.
    4. **Team** est automatiquement défini sur l’équipe propriétaire de la page d’accueil que vous consultiez. Si nécessaire, changez d’équipe.
    5. Indiquez un nom pour votre projet dans le champ **Name**.
    6. Définissez **visibilité du projet**, dont la valeur par défaut est **Team**.
    7. Facultativement, ajoutez une **Description**.
    8. Cliquez sur **Create project**.
  </Tab>

  <Tab title="Python SDK">
    Pour créer un projet par programmation, spécifiez un `project` lors de l’appel à `wandb.init()`. Si le projet n’existe pas encore, il est créé automatiquement et appartient à l’entité spécifiée. Par exemple :

    ```python
    import wandb with wandb.init(entity="<entity>", project="<project_name>") as run: run.log({"accuracy": .95})
    ```

    Référez-vous à la [référence de l’API `wandb.init()`](/fr/models/ref/python/functions/init/#examples).
  </Tab>
</Tabs>
