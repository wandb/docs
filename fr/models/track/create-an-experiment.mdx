title: Créer une expérience
---

Utilisez le SDK Python de W&amp;B pour suivre vos expériences de machine learning. Vous pouvez ensuite consulter les résultats dans un tableau de bord interactif ou exporter vos données vers Python pour y accéder par programmation avec l’[API publique W&amp;B](/fr/models/ref/python/public-api/).

Ce guide explique comment utiliser les éléments de base de W&amp;B pour créer une expérience W&amp;B.

<div id="how-to-create-a-wb-experiment">
  ## Comment créer une expérience W&amp;B
</div>

Créez une expérience W&amp;B en quatre étapes :

1. [Initialiser un run W&amp;B](#initialize-a-wb-run)
2. [Capturer un dictionnaire d’hyperparamètres](#capture-a-dictionary-of-hyperparameters)
3. [Consigner des métriques dans votre boucle d’entraînement](#log-metrics-inside-your-training-loop)
4. [Journaliser un artefact dans W&amp;B](#log-an-artifact-to-wb)

<div id="initialize-a-wb-run">
  ### Initialiser un run W&amp;B
</div>

Utilisez [`wandb.init()`](/fr/models/ref/python/functions/init) pour créer un run W&amp;B.

L’extrait suivant crée un run dans un projet W&amp;B nommé `“cat-classification”`, avec la description `“My first experiment”`, pour mieux identifier ce run. Les tags `“baseline”` et `“paper1”` sont inclus pour rappeler que ce run est une expérience de référence destinée à la publication d’un futur article.

```python
import wandb

with wandb.init(
    project="cat-classification",
    notes="My first experiment",
    tags=["baseline", "paper1"],
) as run:
    ...
```

`wandb.init()` renvoie un objet [Run](/fr/models/ref/python/experiments/run).

<Note>
  Remarque : les Runs sont ajoutés à des projets existants si le projet existe déjà lorsque vous appelez `wandb.init()`. Par exemple, si vous avez déjà un projet appelé `“cat-classification”`, ce projet continue d’exister et n’est pas supprimé. À la place, un nouveau run est ajouté à ce projet.
</Note>

<div id="capture-a-dictionary-of-hyperparameters">
  ### Capturer un dictionnaire d’hyperparamètres
</div>

Enregistrez un dictionnaire d’hyperparamètres, comme le taux d’apprentissage ou le type de modèle. Les paramètres du modèle que vous capturez dans la configuration vous seront utiles par la suite pour organiser vos résultats et y effectuer des requêtes.

```python
with wandb.init(
    ...,
    config={"epochs": 100, "learning_rate": 0.001, "batch_size": 128},
) as run:
    ...
```

Pour plus d’informations sur la manière de configurer une expérience, voir [Configurer Experiments](./config).

<div id="log-metrics-inside-your-training-loop">
  ### Consigner des métriques dans votre boucle d’entraînement
</div>

Appelez [`run.log()`](/fr/models/ref/python/experiments/run/#method-runlog) pour consigner les métriques de chaque étape d’entraînement, telles que la précision et la perte.

```python
model, dataloader = get_model(), get_data()

for epoch in range(run.config.epochs):
    for batch in dataloader:
        loss, accuracy = model.training_step()
        run.log({"accuracy": accuracy, "loss": loss})
```

Pour plus d&#39;informations, voir les différents types de données que vous pouvez journaliser avec W&amp;B, voir [Journaliser des données pendant les Experiments](/fr/models/track/log/).

<div id="log-an-artifact-to-wb">
  ### Journaliser un artefact dans W&amp;B
</div>

Vous pouvez également journaliser un artefact W&amp;B. Les Artifacts facilitent le versionnement des jeux de données et des modèles.

```python
# Vous pouvez sauvegarder n'importe quel fichier ou même un répertoire. Dans cet exemple, nous supposons
# que le modèle possède une méthode save() qui génère un fichier ONNX.
model.save("path_to_model.onnx")
run.log_artifact("path_to_model.onnx", name="trained-model", type="model")
```

Pour en savoir plus sur [Artifacts](/fr/models/artifacts/) ou sur la gestion des versions de modèles dans le [registre](/fr/models/registry/).

<div id="putting-it-all-together">
  ### Récapitulatif
</div>

Vous trouverez ci-dessous le script complet avec les extraits de code précédents :

```python
import wandb

with wandb.init(
    project="cat-classification",
    notes="",
    tags=["baseline", "paper1"],
    # Enregistrer les hyperparamètres du run.
    config={"epochs": 100, "learning_rate": 0.001, "batch_size": 128},
) as run:
    # Configurer le modèle et les données.
    model, dataloader = get_model(), get_data()

    # Lancez l'entraînement en journalisant les métriques pour visualiser les performances du modèle.
    for epoch in range(run.config["epochs"]):
        for batch in dataloader:
            loss, accuracy = model.training_step()
            run.log({"accuracy": accuracy, "loss": loss})

    # Téléverser le modèle entraîné en tant qu'artefact.
    model.save("path_to_model.onnx")
    run.log_artifact("path_to_model.onnx", name="trained-model", type="model")
```

<div id="best-practices">
  ## Bonnes pratiques
</div>

Voici quelques recommandations à suivre lorsque vous créez des expériences :

1. **Terminez vos runs** : utilisez `wandb.init()` dans une instruction `with` afin que le run soit automatiquement marqué comme terminé lorsque le code s&#39;achève ou lève une exception.
   * Dans les notebooks Jupyter, il peut être plus pratique de gérer vous-même l&#39;objet Run. Dans ce cas, vous pouvez appeler explicitement `finish()` sur l&#39;objet Run pour le marquer comme terminé :

     ```python
     # Dans une cellule de notebook :
     run = wandb.init()

     # Dans une autre cellule :
     run.finish()
     ```
2. **Configuration** : suivez les hyperparamètres, l&#39;architecture, le jeu de données et tout autre élément utile pour reproduire votre modèle. Ils apparaîtront dans des colonnes ; utilisez les colonnes de configuration pour regrouper, trier et filtrer dynamiquement les runs dans l&#39;application.
3. **Projet** : un projet est un ensemble d&#39;expériences que vous pouvez comparer entre elles. Chaque projet dispose d&#39;une page de tableau de bord dédiée, et vous pouvez facilement activer ou désactiver différents groupes de runs afin de comparer différentes versions de modèle.
4. **Notes** : définissez un court message de commit directement depuis votre script. Modifiez et consultez les notes dans la section Aperçu d&#39;un run dans l&#39;application W&amp;B.
5. **Tags** : identifiez les runs de référence et vos runs favoris. Vous pouvez filtrer les runs à l&#39;aide de tags. Vous pouvez modifier les tags ultérieurement dans la section Aperçu du tableau de bord de votre projet dans l&#39;application W&amp;B.
6. **Créez plusieurs ensembles de runs pour comparer les expériences** : lorsque vous comparez des expériences, créez plusieurs ensembles de runs pour faciliter la comparaison des métriques. Vous pouvez activer ou désactiver les ensembles de runs sur un même graphique ou groupe de graphiques.

L&#39;extrait de code suivant montre comment définir une expérience W&amp;B en appliquant les bonnes pratiques ci-dessus :

```python
import wandb

config = {
    "learning_rate": 0.01,
    "momentum": 0.2,
    "architecture": "CNN",
    "dataset_id": "cats-0192",
}

with wandb.init(
    project="detect-cats",
    notes="tweak baseline",
    tags=["baseline", "paper1"],
    config=config,
) as run:
    ...
```

Pour plus d&#39;informations, voir les paramètres disponibles lors de la définition d’une expérience W&amp;B, voir la documentation de l’API [`wandb.init()`](/fr/models/ref/python/functions/init) dans le [Guide de référence de l’API](/fr/models/ref/python/).
