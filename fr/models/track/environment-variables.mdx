title: Variables d’environnement
---

Lorsque vous exécutez un script dans un environnement automatisé, vous pouvez contrôler W&amp;B à l’aide de variables d’environnement définies avant l’exécution du script ou directement dans le script.

```bash
# Ceci est secret et ne doit pas être versionné
WANDB_API_KEY=$YOUR_API_KEY
# Nom et notes facultatifs
WANDB_NAME="My first run"
WANDB_NOTES="Smaller learning rate, more regularization."
```

```bash
# Nécessaire uniquement si vous ne versionnez pas le fichier wandb/settings
WANDB_ENTITY=$username
WANDB_PROJECT=$project
```

```python
# Si vous ne souhaitez pas synchroniser votre script avec le cloud
os.environ["WANDB_MODE"] = "offline"

# Ajouter le suivi de l'ID de balayage aux objets Run et aux classes associées
os.environ["WANDB_SWEEP_ID"] = "b05fq58z"
```

<div id="optional-environment-variables">
  ## Variables d&#39;environnement facultatives
</div>

Utilisez ces variables d&#39;environnement facultatives pour, par exemple, configurer l&#39;authentification sur des machines distantes.

| Nom de variable         | Utilisation                                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| ----------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `WANDB_API_KEY`         | Définit la clé d’authentification associée à votre compte. Créez une clé API dans [Paramètres utilisateur](https://wandb.ai/settings). Cette variable doit être définie si `wandb login` n&#39;a pas été exécuté sur la machine distante.                                                                                                                                                                                                                                           |
| `WANDB_BASE_URL`        | Si vous utilisez [wandb/local](/fr/platform/hosting/), vous devez définir cette variable d&#39;environnement sur `http://YOUR_IP:YOUR_PORT`                                                                                                                                                                                                                                                                                                                                            |
| `WANDB_CACHE_DIR`       | La valeur par défaut est ~/.cache/wandb. Vous pouvez remplacer cet emplacement à l&#39;aide de cette variable d&#39;environnement.                                                                                                                                                                                                                                                                                                                                                  |
| `WANDB_CONFIG_DIR`      | La valeur par défaut est ~/.config/wandb. Vous pouvez remplacer cet emplacement à l&#39;aide de cette variable d&#39;environnement.                                                                                                                                                                                                                                                                                                                                                 |
| `WANDB_CONFIG_PATHS`    | Liste de fichiers YAML, séparés par des virgules, à charger dans wandb.config. Voir [configuration](./config#file-based-configs).                                                                                                                                                                                                                                                                                                                                                   |
| `WANDB_CONSOLE`         | Définissez cette valeur sur &quot;off&quot; pour désactiver la journalisation de stdout / stderr. La valeur par défaut est &quot;on&quot; dans les environnements qui le prennent en charge.                                                                                                                                                                                                                                                                                        |
| `WANDB_DATA_DIR`        | Emplacement où téléverser les artefacts de staging. L’emplacement par défaut dépend de votre plateforme, car il utilise la valeur de `user_data_dir` du package Python `platformdirs`. Assurez-vous que ce répertoire existe et que l’utilisateur exécutant le processus dispose des autorisations d’écriture.                                                                                                                                                                      |
| `WANDB_DIR`             | Où stocker tous les fichiers générés. Si elle n’est pas définie, la valeur par défaut est le répertoire `wandb` relatif à votre script d’entraînement. Assurez-vous que ce répertoire existe et que l’utilisateur exécutant le processus dispose des autorisations d’écriture. Cela ne contrôle pas l’emplacement des artefacts téléchargés, que vous pouvez définir à l’aide de la variable d’environnement `WANDB_ARTIFACT_DIR`.                                                  |
| `WANDB_ARTIFACT_DIR`    | Où stocker tous les artefacts téléchargés. S&#39;il n&#39;est pas défini, le répertoire `artifacts`, relatif à votre script d&#39;entraînement, est utilisé par défaut. Assurez-vous que ce répertoire existe et que l&#39;utilisateur exécutant le processus dispose des autorisations nécessaires pour y écrire. Cela ne contrôle pas l&#39;emplacement des fichiers de métadonnées générés, que vous pouvez définir à l&#39;aide de la variable d&#39;environnement `WANDB_DIR`. |
| `WANDB_DISABLE_GIT`     | Empêche wandb de rechercher un dépôt git et d&#39;enregistrer le commit / diff le plus récent.                                                                                                                                                                                                                                                                                                                                                                                      |
| `WANDB_DISABLE_CODE`    | Définissez cette valeur sur true pour empêcher wandb d’enregistrer les notebooks ou les diffs Git. Nous enregistrerons tout de même le commit actuel si nous sommes dans un dépôt Git.                                                                                                                                                                                                                                                                                              |
| `WANDB_DOCKER`          | Renseignez cette variable avec le digest d’une image Docker pour permettre la restauration des runs. Cette valeur est définie automatiquement avec la commande wandb docker. Vous pouvez obtenir un digest d’image en exécutant `wandb docker my/image/name:tag --digest`                                                                                                                                                                                                           |
| `WANDB_ENTITY`          | L’entité associée à votre run. Si vous avez exécuté `wandb init` dans le répertoire de votre script d’entraînement, cela créera un répertoire nommé *wandb* et enregistrera une entité par défaut qui peut être ajoutée au contrôle de version. Si vous ne souhaitez pas créer ce fichier ou si vous voulez le remplacer, vous pouvez utiliser la variable d’environnement.                                                                                                         |
| `WANDB_ERROR_REPORTING` | Définissez cette valeur sur false pour empêcher wandb de journaliser les erreurs fatales dans son système de suivi des erreurs.                                                                                                                                                                                                                                                                                                                                                     |
| `WANDB_HOST`            | Définissez cette valeur sur le nom d’hôte que vous souhaitez voir dans l’interface wandb si vous ne voulez pas utiliser celui fourni par le système                                                                                                                                                                                                                                                                                                                                 |
| `WANDB_IGNORE_GLOBS`    | Définissez cette valeur sur une liste, séparée par des virgules, de motifs glob de fichiers à ignorer. Ces fichiers ne seront pas synchronisés vers le cloud.                                                                                                                                                                                                                                                                                                                       |
| `WANDB_JOB_NAME`        | Spécifiez un nom pour tous les jobs créés par `wandb`.                                                                                                                                                                                                                                                                                                                                                                                                                              |
| `WANDB_JOB_TYPE`        | Spécifiez le type de job, par exemple &quot;entraînement&quot; ou &quot;évaluation&quot;, pour indiquer différents types de runs. Voir [grouping](/fr/models/runs/grouping/) pour plus d’informations.                                                                                                                                                                                                                                                                                 |
| `WANDB_MODE`            | Si vous définissez cette valeur sur &quot;offline&quot;, wandb enregistrera localement les métadonnées de votre run et ne les synchronisera pas avec le serveur. Si vous la définissez sur `disabled`, wandb sera complètement désactivé.                                                                                                                                                                                                                                           |
| `WANDB_NAME`            | Le nom explicite de votre run. S’il n’est pas défini, il sera généré aléatoirement pour vous                                                                                                                                                                                                                                                                                                                                                                                        |
| `WANDB_NOTEBOOK_NAME`   | Si vous l’exécutez dans Jupyter, vous pouvez définir le nom du notebook avec cette variable. Nous essayons de le détecter automatiquement.                                                                                                                                                                                                                                                                                                                                          |
| `WANDB_NOTES`           | Des notes plus détaillées sur votre run. Le Markdown est pris en charge et vous pourrez les modifier plus tard dans l’interface utilisateur.                                                                                                                                                                                                                                                                                                                                        |
| `WANDB_PROJECT`         | Le projet associé à votre run. Vous pouvez également le définir avec `wandb init`, mais la variable d’environnement aura priorité sur cette valeur.                                                                                                                                                                                                                                                                                                                                 |
| `WANDB_RESUME`          | Par défaut, cette valeur est définie sur *never*. Si elle est définie sur *auto*, wandb reprendra automatiquement les runs ayant échoué. Si elle est définie sur *must*, le run doit exister au démarrage. Si vous souhaitez toujours générer vos propres ID uniques, définissez cette valeur sur *allow* et définissez toujours `WANDB_RUN_ID`.                                                                                                                                    |
| `WANDB_RUN_GROUP`       | Spécifiez le nom de l’expérience pour regrouper automatiquement les runs. Voir [grouping](/fr/models/runs/grouping/) pour en savoir plus.                                                                                                                                                                                                                                                                                                                                              |
| `WANDB_RUN_ID`          | Définissez cette variable sur une chaîne globalement unique (par projet) correspondant à un seul run de votre script. Elle ne doit pas dépasser 64 caractères. Tous les caractères non alphanumériques seront convertis en tirets. Cela peut être utilisé pour reprendre un run existant en cas d’échec.                                                                                                                                                                            |
| `WANDB_QUIET`           | Définissez cette variable sur `true` pour limiter à seulement des messages critiques ceux enregistrés dans la sortie standard. Si elle est définie, tous les journaux seront écrits dans `$WANDB_DIR/debug.log`.                                                                                                                                                                                                                                                                    |
| `WANDB_SILENT`          | Définissez cette variable sur `true` pour désactiver les messages de journalisation de wandb. Cela est utile pour les commandes scriptées. Si cette option est définie, tous les journaux seront écrits dans `$WANDB_DIR/debug.log`.                                                                                                                                                                                                                                                |
| `WANDB_SHOW_RUN`        | Définissez cette variable sur `true` pour ouvrir automatiquement un navigateur à l’URL du run si votre système d’exploitation le prend en charge.                                                                                                                                                                                                                                                                                                                                   |
| `WANDB_SWEEP_ID`        | Ajouter le suivi de l’ID du balayage aux objets `Run` et aux classes associées, et l’afficher dans l’interface utilisateur.                                                                                                                                                                                                                                                                                                                                                         |
| `WANDB_TAGS`            | Une liste de balises séparées par des virgules à appliquer au run.                                                                                                                                                                                                                                                                                                                                                                                                                  |
| `WANDB_USERNAME`        | Le nom d’utilisateur d’un membre de votre équipe associé au run. Vous pouvez l’utiliser avec la clé API d’un compte de service pour permettre d’attribuer des runs automatisés à des membres de votre équipe.                                                                                                                                                                                                                                                                       |
| `WANDB_USER_EMAIL`      | L’adresse e-mail d’un membre de votre équipe associée au run. Vous pouvez l’utiliser avec la clé API d’un compte de service pour permettre d’attribuer des runs automatisés à des membres de votre équipe.                                                                                                                                                                                                                                                                          |

<div id="singularity-environments">
  ## Environnements Singularity
</div>

Si vous exécutez des conteneurs avec [Singularity](https://singularity.lbl.gov/index.html), vous pouvez transmettre des variables d&#39;environnement en ajoutant le préfixe `SINGULARITYENV_` aux variables ci-dessus. Vous trouverez plus de détails sur les variables d&#39;environnement de Singularity [ici](https://singularity.lbl.gov/docs-environment-metadata#environment).

<div id="running-on-aws">
  ## Exécution sur AWS
</div>

Si vous exécutez des jobs batch sur AWS, vous pouvez facilement authentifier vos machines avec vos identifiants W&amp;B. Créez une clé API dans les [Paramètres utilisateur](https://wandb.ai/settings), puis définissez la variable d&#39;environnement `WANDB_API_KEY` dans la [spécification du job AWS Batch](https://docs.aws.amazon.com/batch/latest/userguide/job_definition_parameters.html#parameters).
