Enregistrez un dictionnaire de métriques, de médias ou d&#39;objets personnalisés pour une étape avec le SDK Python W&amp;B. W&amp;B collecte les paires clé-valeur à chaque étape et les stocke dans un dictionnaire unifié chaque fois que vous enregistrez des données avec `wandb.Run.log()`. Les données enregistrées depuis votre script sont sauvegardées localement sur votre machine dans un répertoire appelé `wandb`, puis synchronisées vers le cloud W&amp;B ou votre [serveur privé](/fr/platform/hosting/).

<Note>
  Les paires clé-valeur ne sont stockées dans un dictionnaire unifié que si vous passez la même valeur à chaque étape. W&amp;B conserve en mémoire toutes les clés et valeurs collectées si vous journalisez une valeur différente pour `step`.
</Note>

Chaque appel à `wandb.Run.log()` correspond par défaut à une nouvelle `step`. W&amp;B utilise les étapes comme axe x par défaut lorsqu&#39;il crée des graphiques et des panneaux. Vous pouvez également créer et utiliser un axe x personnalisé ou capturer une métrique de synthèse personnalisée. Pour plus d&#39;informations, voir [Personnaliser les axes de journalisation](/fr/models/track/log/customize-logging-axes/).

<div id="automatically-logged-data">
  ## Données enregistrées automatiquement
</div>

W&amp;B enregistre automatiquement les informations suivantes pendant une expérience W&amp;B :

* **Métriques système** : utilisation du CPU et du GPU, réseau, etc. Pour le GPU, elles sont récupérées avec [`nvidia-smi`](https://developer.nvidia.com/nvidia-system-management-interface).
* **Ligne de commande** : stdout et stderr sont capturés et s’affichent dans l’onglet des journaux de la [page du run.](/fr/models/runs/)

Activez l’[enregistrement du code](https://wandb.me/code-save-colab) dans la page [Settings](https://wandb.ai/settings) de votre compte pour enregistrer :

* **Commit Git** : le dernier commit git est récupéré et visible dans l’onglet Vue d’ensemble de la page du run, ainsi qu’un fichier `diff.patch` s’il existe des modifications non validées.
* **Dépendances** : le fichier `requirements.txt` est téléversé et affiché dans l’onglet des fichiers de la page du run, avec tous les fichiers que vous enregistrez dans le répertoire `wandb` du run.

<div id="what-data-is-logged-with-specific-wb-api-calls">
  ## Quelles données sont enregistrées avec des appels d’API W&amp;B spécifiques ?
</div>

Avec W&amp;B, vous pouvez décider exactement de ce que vous souhaitez enregistrer. La liste suivante présente quelques objets couramment enregistrés :

* **Datasets** : Vous devez enregistrer explicitement des images ou d’autres échantillons de jeu de données pour qu’ils soient envoyés en streaming vers W&amp;B.
* **Plots** : Utilisez `wandb.plot()` avec `wandb.Run.log()` pour suivre des graphiques. Voir [Consigner des graphiques](/fr/models/track/log/plots/) pour plus d’informations.
* **Tables** : Utilisez `wandb.Table` pour enregistrer des données afin de les visualiser et de les interroger avec W&amp;B. Voir [Consigner des tableaux](/fr/models/track/log/log-tables/) pour plus d’informations.
* **Gradients PyTorch** : Ajoutez `wandb.Run.watch(model)` pour voir les gradients des poids sous forme d’histogrammes dans l’UI.
* **Informations de configuration** : Enregistrez les hyperparamètres, un lien vers votre jeu de données ou le nom de l’architecture que vous utilisez comme paramètres de configuration, transmis comme ceci : `wandb.init(config=your_config_dictionary)`.
* **Métriques** : Utilisez `wandb.Run.log()` pour voir les métriques de votre modèle. Si vous enregistrez des métriques comme accuracy et loss dans votre boucle d’entraînement, vous obtiendrez des graphiques mis à jour en direct dans l’UI.

<div id="common-workflows">
  ## Flux de travail courants
</div>

1. **Comparer la meilleure accuracy** : Pour comparer la meilleure valeur d&#39;une métrique entre plusieurs runs, définissez la valeur summary de cette métrique. Par défaut, summary correspond à la dernière valeur enregistrée pour chaque clé. Cela est utile dans le tableau de l&#39;UI, où vous pouvez trier et filtrer les runs selon leurs métriques summary, afin de comparer les runs dans un tableau ou un graphique en barres en fonction de leur *meilleure* accuracy, plutôt que de leur accuracy finale. Par exemple : `wandb.run.summary["best_accuracy"] = best_accuracy`

2. **Afficher plusieurs métriques sur un même graphique** : Enregistrez plusieurs métriques dans le même appel. Par exemple :
   ```python
   with wandb.init() as run:
     run.log({"acc": 0.9, "loss": 0.1})
   ```
   Vous pouvez ensuite afficher les deux métriques dans l&#39;UI.

3. **Personnaliser l&#39;axe X** : Ajoutez un axe X personnalisé au même appel `log` pour visualiser vos métriques par rapport à un autre axe dans le tableau de bord W&amp;B. Par exemple :
   ```python
   with wandb.init() as run:
     run.log({'acc': 0.9, 'epoch': 3, 'batch': 117})
   ```
   Pour définir l&#39;axe X par défaut pour une métrique donnée, utilisez [Run.define&#95;metric()](/fr/models/ref/python/experiments/run#define_metric).

4. **Enregistrer des médias enrichis et des graphiques** : `wandb.Run.log()` prend en charge l&#39;enregistrement d&#39;une grande variété de types de données, des [médias comme les images et les vidéos](/fr/models/track/log/media/) aux [tableaux](/fr/models/track/log/log-tables/) et aux [graphiques](/fr/models/app/features/custom-charts/).
