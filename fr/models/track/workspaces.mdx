Le Workspace W&amp;B est votre bac à sable personnel pour personnaliser des graphiques et explorer les résultats des modèles. Un Workspace W&amp;B se compose de *Tables* et de *sections de panneaux* :

* **Tables** : Tous les runs enregistrés dans votre projet sont répertoriés dans le tableau du projet. Activez ou désactivez les runs, modifiez les couleurs et développez le tableau pour voir les notes, la configuration et les métriques récapitulatives de chaque run.
* **Panel sections** : Une section contenant un ou plusieurs [panneaux](/fr/models/app/features/panels/). Créez de nouveaux panneaux, organisez-les et exportez-les vers Reports pour enregistrer des instantanés de votre Workspace.

<div id="workspace-types">
  ## Types de Workspace
</div>

Il existe deux grandes catégories de Workspace : **Workspaces personnels** et **vues enregistrées**.

* **Workspaces personnels :**  Un Workspace personnalisable pour analyser en profondeur les modèles et les visualisations de données. Seul le propriétaire du Workspace peut le modifier et enregistrer les modifications. Les membres de l’équipe peuvent consulter un Workspace personnel, mais ils ne peuvent pas modifier le Workspace personnel de quelqu’un d’autre.
* **vues enregistrées :** Les vues enregistrées sont des instantanés collaboratifs d’un Workspace. Toute personne de votre équipe peut consulter, modifier et enregistrer les modifications apportées aux vues enregistrées du Workspace. Utilisez les vues enregistrées du Workspace pour passer en revue et discuter des expériences, des runs, etc.

<div id="saved-workspace-views">
  ## Vues enregistrées du Workspace
</div>

Améliorez la collaboration au sein de l’équipe grâce à des vues du Workspace personnalisées. Créez des vues enregistrées pour organiser votre configuration préférée de graphiques et de données.

<div id="create-a-new-saved-workspace-view">
  ### Créer une nouvelle vue enregistrée du Workspace
</div>

1. Accédez à un Workspace personnel ou à une vue enregistrée.
2. Modifiez le Workspace.
3. Cliquez sur le menu à trois points (trois points horizontaux) dans le coin supérieur droit de votre Workspace. Cliquez sur **Enregistrer en tant que nouvelle vue**.

Les nouvelles vues enregistrées apparaissent dans le menu de navigation du Workspace.

<Frame>
  <img src="/images/app_ui/Menu_Views.jpg" alt="Menu des vues enregistrées" />
</Frame>

<div id="update-a-saved-workspace-view">
  ### Mettre à jour une vue enregistrée du Workspace
</div>

Les modifications enregistrées remplacent l’état précédent de la vue enregistrée. Les modifications non enregistrées ne sont pas conservées. Pour mettre à jour une vue enregistrée du Workspace dans W&amp;B :

1. Accédez à une vue enregistrée.
2. Apportez les modifications souhaitées à vos graphiques et à vos données dans le Workspace.
3. Cliquez sur le bouton **Save** pour confirmer vos modifications.

<Note>
  Une boîte de dialogue de confirmation s’affiche lorsque vous enregistrez les modifications apportées à une vue du Workspace. Si vous préférez ne plus voir cette invite à l’avenir, sélectionnez l’option **Do not show this modal next time** avant de confirmer l’enregistrement.
</Note>

<div id="delete-a-saved-workspace-view">
  ### Supprimer une vue enregistrée du Workspace
</div>

Supprimez les vues enregistrées dont vous n’avez plus besoin.

1. Accédez à la vue enregistrée que vous souhaitez supprimer.
2. Sélectionnez l’icône à trois points horizontaux (**...**) en haut à droite de la vue.
3. Choisissez **Supprimer la vue**.
4. Confirmez la suppression pour supprimer la vue du menu de votre Workspace.

<div id="share-a-workspace-view">
  ### Partager une vue du Workspace
</div>

Partagez votre Workspace personnalisé avec votre équipe en partageant directement son URL. Tous les utilisateurs ayant accès au projet du Workspace peuvent voir les vues enregistrées de ce dernier.

<div id="workspace-templates">
  ## Modèles de Workspace
</div>

<Note>
  Cette fonctionnalité nécessite une licence [Enterprise](https://wandb.ai/site/pricing/).
</Note>

Utilisez les *modèles de Workspace* pour créer des Workspace avec les mêmes paramètres qu’un Workspace existant, plutôt qu’avec les [paramètres par défaut des nouveaux Workspace](#default-workspace-settings).

<div id="default-workspace-settings">
  ### Paramètres par défaut du Workspace
</div>

W&amp;B utilise les paramètres par défaut suivants pour les Workspaces :

<div id="hide-empty-sections-during-search">
  #### Masquer les sections vides lors d’une recherche
</div>

Par défaut, W&amp;B n’affiche pas les sections vides dans le Workspace. Vous pouvez modifier ce paramètre pour afficher les sections vides dans le Workspace.

<div id="sort-panels-alphabetically">
  #### Trier les panneaux par ordre alphabétique
</div>

Par défaut, W&amp;B ne trie pas les panneaux par ordre alphabétique en fonction du titre du graphique. Vous pouvez modifier ce paramètre pour trier les panneaux par ordre alphabétique en fonction de leur titre de graphique.

<div id="section-organization">
  #### Organisation des sections
</div>

Par défaut, W&amp;B organise les panneaux en sections selon le premier préfixe du nom de la métrique. Par exemple, si votre Workspace inclut les métriques `a/b/c/d` et `a/e/f`, W&amp;B organise ces métriques dans une section appelée `a`. Si votre Workspace inclut les métriques `a/b/c/d` et `e/f/g`, W&amp;B organise ces métriques dans des sections appelées `a` et `e`.

Vous pouvez modifier l&#39;organisation par défaut des sections pour effectuer le regroupement selon le dernier préfixe. Par exemple, si votre Workspace inclut les métriques `a/b/c/d` et `a/e/f`, W&amp;B organise ces métriques dans des sections appelées `d` et `f`. Si votre Workspace inclut les métriques `a/b/c/d` et `e/f/g`, W&amp;B organise ces métriques dans des sections appelées `d` et `g`.

<div id="line-plot-settings">

<div id="configure-your-workspace-template">
  ### Configurez votre modèle de Workspace
</div>

1. Ouvrez n’importe quel Workspace ou créez-en un nouveau.
2. Cliquez sur le bouton **Settings** dans le coin supérieur droit du Workspace.
3. Sélectionnez **Mise en page du Workspace** dans le panneau.
4. Configurez les paramètres du Workspace selon vos préférences.

<div id="save-your-workspace-template">
  ### Enregistrer votre modèle de Workspace
</div>

1. En haut du Workspace, cliquez sur le menu d’actions `...` à côté des icônes fléchées **Annuler** et **Rétablir**.
2. Cliquez sur **Enregistrer le modèle de Workspace personnel**.
3. Vérifiez les paramètres du modèle, puis cliquez sur **Enregistrer**.

Les nouveaux Workspaces utiliseront ces paramètres au lieu des valeurs par défaut.

<div id="view-your-workspace-template">
  ### Consultez votre modèle de Workspace
</div>

Pour consulter la configuration actuelle de votre modèle de Workspace :

1. Depuis n’importe quelle page, sélectionnez votre icône utilisateur dans le coin supérieur droit. Dans le menu déroulant, choisissez **Paramètres utilisateur**.
2. Accédez à la section **Modèle de Workspace personnel**. Si vous utilisez un modèle de Workspace, sa configuration s’affiche. Sinon, la section ne contient aucune information.

<div id="update-your-workspace-template">
  ### Mettez à jour votre modèle de Workspace
</div>

Pour mettre à jour votre modèle de Workspace :

1. Ouvrez n’importe quel Workspace.
2. Modifiez les paramètres du Workspace. Par exemple, définissez le nombre de runs à inclure sur `11`.
3. Pour enregistrer les modifications dans le modèle, cliquez sur le menu d’action `...` à côté des icônes en forme de flèche **Annuler** et **Rétablir**, puis cliquez sur **Mettre à jour le modèle de Workspace personnel**.
4. Vérifiez les paramètres, puis cliquez sur **Mettre à jour**. Le modèle est mis à jour et réappliqué à tous les Workspaces qui l’utilisent.

<div id="delete-your-workspace-template">
  ### Supprimez votre modèle de Workspace
</div>

Pour supprimer votre modèle de Workspace et revenir aux paramètres par défaut :

1. Depuis n’importe quelle page, sélectionnez votre icône utilisateur dans le coin supérieur droit. Dans le menu déroulant, choisissez **Paramètres utilisateur**.
2. Accédez à la section **Modèle de Workspace personnel**. La configuration de votre modèle de Workspace s’affiche.
3. Cliquez sur l’icône de corbeille à côté de **Paramètres**.

<Note>
  Pour Cloud dédié et Autogéré, la suppression de votre modèle de Workspace est prise en charge à partir de la version 0.70. Sur les versions antérieures de Server, mettez plutôt à jour votre modèle de Workspace pour utiliser les [paramètres par défaut](#default-workspace-settings).
</Note>

<div id="install-workspace-api">
  ### Installer l’API Workspace
</div>

En plus de `wandb`, veillez à installer `wandb-workspaces` :

```bash
pip install wandb wandb-workspaces
```

<div id="define-and-save-a-workspace-view-programmatically">
  ### Définir et enregistrer une vue de Workspace par programmation
</div>

```python
import wandb_workspaces.reports.v2 as ws

workspace = ws.Workspace(entity="your-entity", project="your-project", views=[...])
workspace.save()
```

<div id="edit-an-existing-view">
  ### Modifier une vue existante
</div>

```python
existing_workspace = ws.Workspace.from_url("workspace-url")
existing_workspace.views[0] = ws.View(name="my-new-view", sections=[...])
existing_workspace.save()
```

<div id="copy-a-workspace-saved-view-to-another-workspace">
  ### Copier une `vue enregistrée` d’un Workspace vers un autre Workspace
</div>

```python
old_workspace = ws.Workspace.from_url("old-workspace-url")
old_workspace_view = old_workspace.views[0]
new_workspace = ws.Workspace(entity="new-entity", project="new-project", views=[old_workspace_view])

new_workspace.save()
```

Voir [`wandb-workspace examples`](https://github.com/wandb/wandb-workspaces/tree/main/examples/workspaces) pour des exemples détaillés de l’API Workspace.
