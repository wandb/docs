---
title: Personnaliser les axes des métriques consignées
description: "Utilisez define_metric() pour définir un axe des x personnalisé pour les métriques consignées, au lieu du compteur d'étapes W&B par défaut."
---

Définissez un axe des x personnalisé lorsque vous consignez des métriques dans W&amp;B. Par défaut, W&amp;B consigne les métriques en *étapes*. Chaque étape correspond à un appel d&#39;API `wandb.Run.log()`.

Par exemple, le script suivant contient une boucle `for` qui s&#39;exécute 10 fois. À chaque itération, le script consigne une métrique appelée `validation_loss` et incrémente le numéro d&#39;étape de 1.

```python
import wandb

with wandb.init() as run:
  # la fonction range crée une séquence de nombres de 0 à 9
  for i in range(10):
    log_dict = {
        "validation_loss": 1/(i+1)   
    }
    run.log(log_dict)
```

Dans le Workspace du projet, la métrique `validation_loss` est tracée en fonction de l&#39;axe des x `step`, qui s&#39;incrémente de 1 à chaque appel de `wandb.Run.log()`. D&#39;après le code précédent, l&#39;axe des x affiche les numéros d&#39;étape 0, 1, 2, ..., 9.

Dans certaines situations, il est plus pertinent de consigner des métriques par rapport à un autre axe des x, par exemple un axe des x logarithmique. Utilisez la méthode [`define_metric()`](/fr/models/ref/python/experiments/run/#define_metric) pour utiliser n&#39;importe quelle métrique consignée comme axe des x personnalisé.

Spécifiez la métrique que vous souhaitez afficher sur l&#39;axe Y avec le paramètre `name`. Le paramètre `step_metric` indique la métrique à utiliser comme axe des x. Lorsque vous consignez une métrique personnalisée, indiquez une valeur pour l&#39;axe des x et pour l&#39;axe Y sous forme de paires clé-valeur dans un dictionnaire.

Copiez-collez l&#39;extrait de code suivant pour définir une métrique d&#39;axe des x personnalisée. Remplacez les valeurs entre `<>` par vos propres valeurs :

```python
import wandb

custom_step = "<custom_step>"  # Nom de l'axe des x personnalisé
metric_name = "<metric>"  # Nom de la métrique de l'axe Y

with wandb.init() as run:
    # Spécifier la métrique d'étape (axe des x) et la métrique à consigner en regard (axe Y)
    run.define_metric(step_metric = custom_step, name = metric_name)

    for i in range(10):
        log_dict = {
            custom_step : int,  # Valeur de l'axe des x
            metric_name : int,  # Valeur de l'axe Y
        }
        run.log(log_dict)
```

Par exemple, l&#39;extrait de code suivant crée un axe des x personnalisé appelé `x_axis_squared`. La valeur de cet axe des x personnalisé est le carré de l&#39;indice `i` de la boucle `for` (`i**2`). L&#39;axe Y se compose de valeurs factices pour la perte de validation (`"validation_loss"`), à l&#39;aide du module intégré `random` de Python :

```python
import wandb
import random

with wandb.init() as run:
    run.define_metric(step_metric = "x_axis_squared", name = "validation_loss")

    for i in range(10):
        log_dict = {
            "x_axis_squared": i**2,
            "validation_loss": random.random(),
        }
        run.log(log_dict)
```

L&#39;image suivante montre le graphique obtenu dans l&#39;interface de W&amp;B App. La métrique `validation_loss` est tracée par rapport à l&#39;axe des x personnalisé `x_axis_squared`, qui correspond au carré de l&#39;indice `i` de la boucle `for`. Notez que les valeurs de l&#39;axe des x sont `0, 1, 4, 9, 16, 25, 36, 49, 64, 81`, ce qui correspond respectivement aux carrés de `0, 1, 2, ..., 9`.

Vous pouvez définir un axe des x personnalisé pour plusieurs métriques à l&#39;aide de `globs` avec des préfixes de chaîne. Par exemple, l&#39;extrait de code suivant trace les métriques consignées avec le préfixe `train/*` sur l&#39;axe des x `train/step` :

```python
import wandb

with wandb.init() as run:

    # définir toutes les autres métriques train/ pour utiliser cette étape
    run.define_metric("train/*", step_metric="train/step")

    for i in range(10):
        log_dict = {
            "train/step": 2**i,  # croissance exponentielle avec l'étape W&B interne
            "train/loss": 1 / (i + 1),  # l'axe des x est train/step
            "train/accuracy": 1 - (1 / (1 + i)),  # l'axe des x est train/step
            "val/loss": 1 / (1 + i),  # l'axe des x est l'étape wandb interne
        }
        run.log(log_dict)
```
