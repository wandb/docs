title: Initialiser un balayage
---

W&amp;B utilise un *contrôleur de balayage* pour gérer les balayages dans le cloud (standard) ou localement (local), sur une ou plusieurs machines. Une fois qu&#39;un run est terminé, le contrôleur de balayage émet un nouvel ensemble d&#39;instructions décrivant un nouveau run à exécuter. Ces instructions sont prises en charge par des *agents* qui exécutent effectivement les runs. Dans un balayage W&amp;B classique, le contrôleur s&#39;exécute sur le serveur W&amp;B. Les agents s&#39;exécutent sur *vos* machines.

Les extraits de code suivants montrent comment initialiser des balayages avec la CLI et dans un notebook Jupyter ou un script Python.

<Warning>
  1. Avant d&#39;initialiser un balayage, assurez-vous d&#39;avoir défini une configuration de balayage soit dans un fichier YAML, soit dans un objet dictionnaire Python imbriqué dans votre script. Pour plus d&#39;informations, voir [Définir une configuration de balayage
     ](/fr/models/sweeps/define-sweep-configuration/). 2. Le balayage W&amp;B et le run W&amp;B doivent tous deux se trouver dans le même projet. Par conséquent, le nom que vous fournissez lorsque vous initialisez W&amp;B
     ([`wandb.init()`](/fr/models/ref/python/functions/init)) doit correspondre au nom du projet que vous fournissez lorsque vous initialisez un balayage W&amp;B ([`wandb.sweep()`](/fr/models/ref/python/functions/sweep)).
</Warning>

<Tabs>
  <Tab title="Script Python ou notebook">
    Utilisez le SDK W&amp;B pour initialiser un balayage. Passez le dictionnaire de configuration du balayage au paramètre `sweep`. Vous pouvez également fournir le nom du projet via le paramètre de projet (`project`) dans lequel vous souhaitez stocker la sortie du run W&amp;B. Si le projet n&#39;est pas spécifié, le run est placé dans un projet « Non catégorisé ».

    La fonction [`wandb.sweep()`](/fr/models/ref/python/functions/sweep) renvoie l&#39;ID du balayage. L&#39;ID du balayage inclut le nom de l&#39;entité et le nom du projet. Notez l&#39;ID du balayage.
  </Tab>

  <Tab title="CLI">
    Utilisez la CLI W&amp;B pour initialiser un balayage. Indiquez le nom de votre fichier de configuration. Vous pouvez également fournir le nom du projet avec l&#39;indicateur `project`. Si le projet n&#39;est pas spécifié, le run W&amp;B est placé dans un projet « Non catégorisé ».

    Utilisez la commande [`wandb sweep`](/fr/models/ref/cli/wandb-sweep) pour initialiser un balayage. L&#39;exemple de code suivant initialise un balayage pour un projet `sweeps_demo` et utilise un fichier `config.yaml` comme configuration.

    ```bash
    wandb sweep --project sweeps_demo config.yaml
    ```

    Cette commande affiche un ID de balayage. L&#39;ID du balayage inclut le nom de l&#39;entité et le nom du projet. Notez l&#39;ID du balayage.
  </Tab>
</Tabs>
