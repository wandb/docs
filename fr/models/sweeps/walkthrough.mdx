Cette page explique comment définir, initialiser et exécuter un balayage. Les quatre étapes principales sont les suivantes :

1. [Configurer votre code d&#39;entraînement](#set-up-your-training-code)
2. [Définir l&#39;espace de recherche avec une configuration de balayage](#define-the-search-space-with-a-sweep-configuration)
3. [Initialiser le balayage](#initialize-the-sweep)
4. [Démarrer le balayage](#start-the-sweep)

Copiez-collez le code suivant dans un notebook Jupyter ou un script Python :

```python
# Importer la bibliothèque Python W&B et se connecter à W&B
import wandb

# 1 : Définir la fonction objectif/d'entraînement
def objective(config):
    score = config.x**3 + config.y
    return score

def main():
    with wandb.init(project="my-first-sweep") as run:
        score = objective(run.config)
        run.log({"score": score})

# 2 : Définir l'espace de recherche
sweep_configuration = {
    "method": "random",
    "metric": {"goal": "minimize", "name": "score"},
    "parameters": {
        "x": {"max": 0.1, "min": 0.01},
        "y": {"values": [1, 3, 7]},
    },
}

# 3 : Démarrer le balayage
sweep_id = wandb.sweep(sweep=sweep_configuration, project="my-first-sweep")

wandb.agent(sweep_id, function=main, count=10)
```

Les sections suivantes détaillent et expliquent chaque étape de l’exemple de code.

<div id="set-up-your-training-code">
  ## Configurer votre code d&#39;entraînement
</div>

Définissez une fonction d&#39;entraînement qui récupère les valeurs d&#39;hyperparamètre depuis `wandb.Run.config` et les utilise pour entraîner un modèle et renvoyer des métriques.

Facultativement, indiquez le nom du projet dans lequel vous souhaitez stocker la sortie du run W&amp;B (paramètre `project` dans [`wandb.init()`](/fr/models/ref/python/functions/init)). Si le projet n&#39;est pas spécifié, le run est placé dans un projet &quot;Uncategorized&quot;.

<Note>
  Le balayage et le run doivent tous deux se trouver dans le même projet. Par conséquent, le nom que vous fournissez lors de l&#39;initialisation de W&amp;B doit correspondre au nom du projet que vous fournissez lors de l&#39;initialisation d&#39;un balayage.
</Note>

```python
# 1 : Définir la fonction objectif/d'entraînement
def objective(config):
    score = config.x**3 + config.y
    return score


def main():
    with wandb.init(project="my-first-sweep") as run:
        score = objective(run.config)
        run.log({"score": score})
```

<div id="define-the-search-space-with-a-sweep-configuration">
  ## Définissez l&#39;espace de recherche avec une configuration de balayage
</div>

Spécifiez les hyperparamètres à balayer dans un dictionnaire. Pour les options de configuration, voir [Définir la configuration de balayage](/fr/models/sweeps/define-sweep-configuration/).

L&#39;exemple suivant présente une configuration de balayage qui utilise une recherche aléatoire (`'method':'random'`). Le balayage sélectionnera aléatoirement un ensemble de valeurs listées dans la configuration pour la taille de lot, l&#39;époque et le taux d&#39;apprentissage.

W&amp;B minimise la métrique spécifiée dans la clé `metric` lorsque `"goal": "minimize"` y est associé. Dans ce cas, W&amp;B cherchera à minimiser la métrique `score` (`"name": "score"`).

```python
# 2 : Définir l'espace de recherche
sweep_configuration = {
    "method": "random",
    "metric": {"goal": "minimize", "name": "score"},
    "parameters": {
        "x": {"max": 0.1, "min": 0.01},
        "y": {"values": [1, 3, 7]},
    },
}
```

<div id="initialize-the-sweep">
  ## Initialiser le balayage
</div>

W&amp;B utilise un *contrôleur de balayage* pour gérer les balayages dans le cloud (standard) ou en local (local), sur une ou plusieurs machines. Pour plus d&#39;informations sur les contrôleurs de balayage, voir [Algorithmes de recherche et d&#39;arrêt en local](./local-controller).

Un identifiant de balayage est renvoyé lorsque vous initialisez un balayage :

```python
sweep_id = wandb.sweep(sweep=sweep_configuration, project="my-first-sweep")
```

Pour en savoir plus sur l’initialisation des balayages, voir [Initialiser les balayages](./initialize-sweeps).

<div id="start-the-sweep">
  ## Démarrer le balayage
</div>

Utilisez l’appel d’API [`wandb.agent()`](/fr/models/ref/python/functions/agent) pour démarrer un balayage.

```python
wandb.agent(sweep_id, function=main, count=10)
```

<Warning>
  **Multitraitement**

  Vous devez entourer vos appels à `wandb.agent()` et `wandb.sweep()` avec `if __name__ == '__main__':` si vous utilisez `multiprocessing` de la bibliothèque standard Python ou le package `pytorch.multiprocessing` de PyTorch. Par exemple :

  ```python
  if __name__ == '__main__':
      wandb.agent(sweep_id="<sweep_id>", function="<function>", count="<count>")
  ```

  Le fait d&#39;entourer votre code de cette convention garantit qu&#39;il est exécuté uniquement lorsque le script est lancé directement, et non lorsqu&#39;il est importé comme module dans un processus de travail.

  Voir [Python standard library `multiprocessing`](https://docs.python.org/3/library/multiprocessing.html#the-spawn-and-forkserver-start-methods) ou [PyTorch `multiprocessing`](https://docs.pytorch.org/docs/stable/notes/multiprocessing.html#asynchronous-multiprocess-training-e-g-hogwild) pour plus d&#39;informations sur le multitraitement. Voir aussi https://realpython.com/if-name-main-python/ pour plus d&#39;informations sur la convention `if __name__ == '__main__':`.
</Warning>
