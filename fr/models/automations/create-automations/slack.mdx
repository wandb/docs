Cette page explique comment créer une [automatisation](/fr/models/automations/) Slack. Pour créer une automatisation webhook, consultez plutôt [Créer une automatisation webhook](/fr/models/automations/create-automations/webhook/).

Dans les grandes lignes, pour créer une automatisation Slack, suivez ces étapes :

1. [Ajouter une intégration Slack](#add-a-slack-integration), ce qui autorise W&amp;B à publier sur l’instance Slack et dans le canal.
2. [Créer l’automatisation](#create-an-automation), qui définit l’[événement](/fr/models/automations/automation-events/) à surveiller et le canal à notifier.

<div id="add-a-slack-integration">
  ## Ajouter une intégration Slack
</div>

Un administrateur d’équipe peut ajouter une intégration Slack à l’équipe.

1. Connectez-vous à W&amp;B, puis accédez à **Team Settings**.
2. Dans la section **Slack channel integrations**, cliquez sur **Connect Slack** pour ajouter une nouvelle instance Slack. Pour ajouter un canal à une instance Slack existante, cliquez sur **New integration**.

   ![Capture d’écran montrant deux intégrations Slack dans une Team](/images/automations/slack_integrations.png)
3. Si nécessaire, connectez-vous à Slack dans votre navigateur. Lorsque vous y êtes invité, autorisez W&amp;B à publier dans le canal Slack que vous sélectionnez. Lisez la page, puis cliquez sur **Search for a channel** et commencez à saisir le nom du canal. Sélectionnez le canal dans la liste, puis cliquez sur **Allow**.
4. Dans Slack, accédez au canal que vous avez sélectionné. Si vous voyez un message du type `[Votre identifiant Slack] added an integration to this channel: Weights & Biases`, l’intégration est correctement configurée.

Vous pouvez maintenant [créer une automatisation](#create-an-automation) qui envoie des notifications au canal Slack que vous avez configuré.

<div id="view-and-manage-slack-integrations">
  ## Afficher et gérer les intégrations Slack
</div>

Un administrateur d’équipe peut afficher et gérer les instances et les canaux Slack de l’équipe.

1. Connectez-vous à W&amp;B, puis accédez à **Team Settings**.
2. Affichez chaque destination Slack dans la section **Slack channel integrations**.
3. Supprimez une destination en cliquant sur l’icône de corbeille correspondante.

<div id="create-an-automation">
  ## Créer une automatisation
</div>

Après avoir [ajouté une intégration Slack](#add-a-slack-integreation), sélectionnez **Registre** ou **Projet**, puis suivez ces étapes pour créer une automatisation qui envoie une notification au canal Slack.

<Tabs>
  <Tab title="Registre">
    Un administrateur du registre peut créer des automatisations dans ce registre.

    1. Connectez-vous à W&amp;B.
    2. Cliquez sur le nom d&#39;un registre pour en afficher les détails.
    3. Pour créer une automatisation au niveau du registre, cliquez sur l&#39;onglet **Automations**, puis sur **Créer une automatisation**. Une automatisation définie au niveau d&#39;un registre s&#39;applique automatiquement à toutes ses collections (y compris celles qui seront créées ultérieurement).
    4. Choisissez l&#39;[événement](/fr/models/automations/automation-events/#registry-events) à surveiller.

       Renseignez tous les champs supplémentaires qui s&#39;affichent, selon l&#39;événement. Par exemple, si vous sélectionnez **Un alias d&#39;artifact est ajouté**, vous devez spécifier le **Regex d&#39;alias**.

       Cliquez sur **Étape suivante**.
    5. Sélectionnez l&#39;équipe propriétaire de l&#39;[intégration Slack](#add-a-slack-integration).
    6. Définissez **Type d&#39;action** sur **Notification Slack**. Sélectionnez le canal Slack, puis cliquez sur **Étape suivante**.
    7. Indiquez un nom pour l&#39;automatisation. Vous pouvez aussi ajouter une description.
    8. Cliquez sur **Créer une automatisation**.
  </Tab>

  <Tab title="Projet">
    Un administrateur W&amp;B peut créer des automatisations dans un projet.

    1. Connectez-vous à W&amp;B.
    2. Accédez à la page du projet et cliquez sur l&#39;onglet **Automations**, puis sur **Créer une automatisation**.

       Vous pouvez aussi créer rapidement une [automatisation de métrique de run](/fr/models/automations/automation-events/#run-events) à partir d&#39;un graphique en courbes dans le Workspace, pour la métrique qu&#39;il affiche. Survolez le panneau, puis cliquez sur l&#39;icône en forme de cloche en haut du panneau.

       <Frame>
         <img src="/images/automations/run_metric_automation_from_panel.png" alt="Emplacement de l'icône de cloche d'automatisation" />
       </Frame>
    3. Choisissez l&#39;[événement](/fr/models/automations/automation-events/#project) à surveiller.

       1. Renseignez tous les champs supplémentaires qui s&#39;affichent. Par exemple, si vous sélectionnez **Un alias d&#39;artifact est ajouté**, vous devez spécifier le **Regex d&#39;alias**.

          1. Pour les automatisations déclenchées par un run, vous pouvez facultativement spécifier un ou plusieurs filtres de run.

             * **Filtrer sur les runs d&#39;un utilisateur** : incluez uniquement les runs créés par l&#39;utilisateur spécifié. Cliquez sur le bouton bascule pour activer le filtre, puis indiquez un nom d&#39;utilisateur.
             * **Filtrer sur le nom du run** : incluez uniquement les runs dont le nom correspond à l&#39;expression régulière indiquée. Cliquez sur le bouton bascule pour activer le filtre, puis indiquez une expression régulière.
       2. Cliquez sur **Étape suivante**.
    4. Sélectionnez l&#39;équipe propriétaire de l&#39;[intégration Slack](#add-a-slack-integration).
    5. Définissez **Type d&#39;action** sur **Notification Slack**. Sélectionnez le canal Slack, puis cliquez sur **Étape suivante**.
    6. Indiquez un nom pour l&#39;automatisation. Vous pouvez aussi ajouter une description.
    7. Cliquez sur **Créer une automatisation**.
  </Tab>
</Tabs>

<div id="view-and-manage-automations">
  ## Afficher et gérer les automatisations
</div>

<Tabs>
  <Tab title="Registre">
    Gérez les automatisations du registre depuis l’onglet **Automations** du registre.

    * Pour afficher les détails d’une automatisation, cliquez sur son nom.
    * Pour modifier une automatisation, cliquez sur son menu d’action `...`, puis sur **Edit automation**.
    * Pour supprimer une automatisation, cliquez sur son menu d’action `...`, puis sur **Delete automation**. Une confirmation est requise.
  </Tab>

  <Tab title="Projet">
    Un administrateur W&amp;B peut afficher et gérer les automatisations d’un projet depuis l’onglet **Automations** du projet.

    * Pour afficher les détails d’une automatisation, cliquez sur son nom.
    * Pour modifier une automatisation, cliquez sur son menu d’action `...`, puis sur **Edit automation**.
    * Pour supprimer une automatisation, cliquez sur son menu d’action `...`, puis sur **Delete automation**. Une confirmation est requise.
  </Tab>
</Tabs>
