<Note>
  L’historique d’exécution des automatisations est disponible sur le Cloud mutualisé de W&amp;B, le Cloud dédié de W&amp;B et W&amp;B Autogéré à partir de la version v0.75.0.
</Note>

Cette page explique comment consulter et interpréter l’historique d’exécution de vos [automatisations](/fr/models/automations) W&amp;B. L’historique d’exécution indique ce qui a déclenché une automatisation, quelles actions ont été exécutées et si elles ont réussi ou échoué. L’examen de l’historique des automatisations vous aide à confirmer qu’elles s’exécutent comme prévu, à résoudre les échecs et à vérifier quels événements ont déclenché des actions en aval.

Chaque automatisation exécutée génère un enregistrement qui comprend :

* **Horodatage d’exécution** : moment où l’automatisation s’est déclenchée.
* **Événement déclencheur** : l’événement spécifique qui a déclenché l’automatisation.
* **Statut** : le statut de l’exécution. Voir [Execution status](#execution-status).
* **Détails de l’action** : informations sur l’action exécutée, par exemple la notification d’un canal Slack ou l’exécution d’un webhook.
* **Détails du résultat** : informations supplémentaires, le cas échéant, sur le résultat final de l’automatisation, y compris l’erreur en cas d’échec de l’exécution.

L’historique des automatisations est disponible à la fois pour les automatisations au niveau du Registry et au niveau du projet. Sélectionnez l’onglet **Registry** ou **Project** pour obtenir des instructions détaillées.

<Tabs>
  <Tab title="Registry">
    1. Cliquez sur **Registry** dans la barre latérale du projet.
    2. Sélectionnez votre registre dans la liste.
    3. Accédez à l’onglet **Automations**, qui répertorie les automatisations du registre. Cliquez sur l’horodatage **Last execution** pour afficher les détails de l’historique d’exécution. Utilisez la barre de recherche pour filtrer par nom d’automatisation, puis triez par date de dernier déclenchement afin de trouver les automatisations exécutées récemment.
    4. Consultez les exécutions des automatisations du registre dans l’ordre chronologique inverse dans l’onglet **Automations history**, avec l’événement, l’action et le statut. Cliquez sur l’horodatage d’une exécution pour afficher les détails d’une exécution particulière.

    <Tip>Si une collection comporte des exécutions d’automatisation associées, l’icône ![Symbole d’exécution d’automatisation de collection, un cercle avec une flèche pointant vers la droite](/images/automations/collection-automation-execution-icon.png) s’affiche, ainsi que le nombre d’exécutions associées.</Tip>
  </Tab>

  <Tab title="Project">
    1. Accédez à votre projet.
    2. Cliquez sur l’onglet **Automations** dans la barre latérale du projet, qui répertorie les automatisations du projet. Cliquez sur l’horodatage **Last execution** pour afficher les détails de l’historique d’exécution. Utilisez la barre de recherche pour filtrer par nom d’automatisation, puis triez par date de dernier déclenchement afin de trouver les automatisations exécutées récemment.
    3. Dans l’onglet **History**, consultez toutes les exécutions des automatisations du projet dans l’ordre chronologique inverse. Les métadonnées de chaque exécution s’affichent, y compris l’événement, l’action et le statut. Cliquez sur l’horodatage d’une exécution pour afficher les détails d’une exécution particulière.
  </Tab>
</Tabs>

<div id="execution-details">
  ## Détails d’exécution
</div>

Après avoir repéré une exécution dans l’historique, utilisez les informations suivantes pour interpréter son statut et son résultat.

<div id="execution-status">
  ### Statut d’exécution
</div>

Chaque exécution d&#39;automatisation a l&#39;un des statuts suivants :

* **Terminée** : l&#39;automatisation a exécuté toutes les actions avec succès.
* **Échouée** : l&#39;automatisation a rencontré une erreur et s&#39;est arrêtée sans aboutir.
* **En attente** : l&#39;automatisation est en file d&#39;attente pour être exécutée.

<div id="execution-metadata">
  ### Métadonnées d’exécution
</div>

Cliquez sur une exécution dans l’historique pour afficher les détails suivants :

* **Détails de l’événement** : l’événement précis qui a déclenché l’automatisation, notamment :
  * Le type d’événement (par exemple, &quot;Nouvelle version d’artefact&quot;, &quot;Run terminé&quot;).
  * Les informations sur l’entité, comme l’ID du run ou le nom de l’artefact.
  * L’utilisateur ayant déclenché l’événement (le cas échéant).
* **Détails de l’action** : des informations sur ce que l’automatisation a tenté de faire :
  * Le type d’action (notification Slack ou webhook).
  * La cible (canal Slack ou URL du webhook).
  * La charge utile envoyée (pour les webhooks).
* **Informations sur le résultat** :
  * Le statut de la réponse (pour les webhooks).
  * Les éventuels messages d’erreur ou traces de pile (pour les exécutions ayant échoué).
