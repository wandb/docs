description: "Créez et gérez des alias personnalisés pour faire référence à des versions spécifiques d'artifact W&B avec des noms explicites comme best ou production."
title: Créer un alias d'artifact
---

Utilisez les alias comme pointeurs vers des versions spécifiques. Par défaut, `wandb.Run.log_artifact()` ajoute l&#39;alias `latest` à la version enregistrée.

W&amp;B crée une version d&#39;artifact `v0` et l&#39;associe à votre artifact lorsque vous enregistrez cet artifact pour la première fois. W&amp;B calcule une somme de contrôle du contenu lorsque vous l&#39;enregistrez à nouveau dans le même artifact. Si l&#39;artifact a changé, W&amp;B enregistre une nouvelle version `v1`.

Par exemple, si vous voulez que votre script d&#39;entraînement récupère la version la plus récente d&#39;un jeu de données, indiquez `latest` lorsque vous utilisez cet artifact. L&#39;exemple de code suivant montre comment télécharger un artifact de jeu de données récent nommé `bike-dataset` avec l&#39;alias `latest` :

```python
import wandb

with wandb.init(project="<project>") as run:
    artifact = run.use_artifact("bike-dataset:latest")
    artifact.download()
```

Vous pouvez également attribuer un alias personnalisé à une version d&#39;artifact. Par exemple, si vous voulez indiquer que ce point de contrôle du modèle est le meilleur selon la métrique AP-50, vous pouvez ajouter la chaîne `'best-ap50'` comme alias lorsque vous journalisez l&#39;artifact du modèle.

```python
with wandb.init(project="<project>") as run:
    artifact = wandb.Artifact("run-3nq3ctyy-bike-model", type="model")
    artifact.add_file("model.h5")
    run.log_artifact(artifact, aliases=["latest", "best-ap50"])
```
