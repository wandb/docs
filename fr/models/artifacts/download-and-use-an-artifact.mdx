Téléchargez et utilisez un artefact déjà stocké sur le serveur W&amp;B, ou créez un objet artefact et transmettez-le pour la déduplication si nécessaire.

<Note>
  Les membres de l’équipe disposant d’une licence en lecture seule ne peuvent pas télécharger d’artefacts.
</Note>

<div id="download-and-use-an-artifact-stored-on-wb">
  ### Télécharger et utiliser un artefact stocké dans W&amp;B
</div>

Téléchargez et utilisez un artefact stocké dans W&amp;B, pendant ou en dehors d&#39;un run W&amp;B. Utilisez l&#39;API publique ([`wandb.Api`](/fr/models/ref/python/public-api/api)) pour exporter ou mettre à jour des données déjà enregistrées dans W&amp;B.

<Tabs>
  <Tab title="Pendant un run">
    Importez d&#39;abord le SDK Python W&amp;B. Créez ensuite un [Run](/fr/models/ref/python/experiments/run) W&amp;B :

    ```python
    import wandb

    with wandb.init(project="<example>", job_type="<job-type>") as run:
        # Voir l'étape suivante
    ```

    Indiquez l&#39;artefact que vous souhaitez utiliser avec la méthode [`wandb.Run.use_artifact()`](/fr/models/ref/python/experiments/run#use_artifact). Cette méthode renvoie un objet run. Dans l&#39;extrait de code suivant, l&#39;artefact appelé `'bike-dataset'` est spécifié avec l&#39;alias `'latest'` :

    ```python
    # Indique l'artefact à utiliser. Le format est "name:alias"
    artifact = run.use_artifact("bike-dataset:latest")
    ```

    Utilisez l&#39;objet renvoyé pour télécharger l&#39;intégralité du contenu de l&#39;artefact :

    ```python
    # Télécharge l'intégralité de l'artefact
    datadir = artifact.download()
    ```

    Vous pouvez aussi transmettre un chemin au paramètre `root` pour télécharger le contenu de l&#39;artefact dans un répertoire spécifique.

    Utilisez la méthode [`wandb.Artifact.get_entry()`](/fr/models/ref/python/experiments/artifact#get_entry) pour télécharger uniquement un sous-ensemble des fichiers :

    ```python
    # Télécharge un fichier spécifique
    entry = artifact.get_entry(name)
    ```

    En réunissant le tout, l&#39;exemple de code complet ressemble à ceci :

    ```python
    import wandb    

    with wandb.init(project="<example>", job_type="<job-type>") as run:
        # Indique l'artefact à utiliser. Le format est "name:alias"
        artifact = run.use_artifact("bike-dataset:latest")

        # Télécharge l'intégralité de l'artefact
        datadir = artifact.download()

        # Télécharge un fichier spécifique
        entry = artifact.get_entry("bike.png")
    ```

    Cela récupère uniquement le fichier situé au chemin `name`. Cela renvoie un objet `Entry` avec les méthodes suivantes :

    * `Entry.download` : télécharge le fichier depuis l&#39;artefact au chemin `name`
    * `Entry.ref` : si `add_reference` a stocké l&#39;entrée comme référence, renvoie l&#39;URI

    {/* Les références dont le schéma est pris en charge par W&B sont téléchargées comme des fichiers d'artefact. Pour plus d'informations, voir [Track external files](/models/artifacts/track-external-files/). */}
  </Tab>

  <Tab title="En dehors d'un run">
    Importez d&#39;abord le SDK W&amp;B. Créez ensuite un objet artefact à partir de la classe d&#39;API publique. Fournissez l&#39;entité, le projet, l&#39;artefact et l&#39;alias associés à cet artefact :

    ```python
    import wandb

    api = wandb.Api()
    artifact = api.artifact("entity/project/artifact:alias")
    ```

    Utilisez l&#39;objet renvoyé pour télécharger le contenu de l&#39;artefact :

    ```python
    artifact.download()
    ```

    Vous pouvez aussi transmettre un chemin au paramètre `root` pour télécharger le contenu de l&#39;artefact dans un répertoire spécifique. Pour plus d&#39;informations, voir le [Guide de référence du SDK Python](/fr/models/ref/python/experiments/artifact#download).
  </Tab>

  <Tab title="W&B CLI">
    Utilisez la commande `wandb artifact get` pour télécharger un artefact depuis le serveur W&amp;B.

    ```
    $ wandb artifact get project/artifact:alias --root mnist/
    ```
  </Tab>
</Tabs>

<div id="partially-download-an-artifact">
  ### Télécharger partiellement un artefact
</div>

Vous pouvez, si nécessaire, télécharger une partie d’un artefact en utilisant un préfixe. Utilisez le paramètre `path_prefix=` (`wandb.Artifact.download(path_prefix=)`) pour télécharger un seul fichier ou le contenu d’un sous-dossier.

```python
with wandb.init(project="<example>", job_type="<job-type>") as run:
    # Indiquer l'artefact à utiliser. Le format est "name:alias"
    artifact = run.use_artifact("bike-dataset:latest")

    # Télécharger un fichier spécifique ou un sous-dossier
    artifact.download(path_prefix="bike.png") # télécharge uniquement bike.png
```

Vous pouvez également télécharger des fichiers depuis un répertoire spécifique. Pour ce faire, indiquez le répertoire dans le paramètre `path_prefix=`. En reprenant l’extrait de code précédent :

```python
# télécharge les fichiers du répertoire images/bikes
artifact.download(path_prefix="images/bikes/") 
```

<div id="use-an-artifact-from-a-different-project">
  ### Utiliser un artefact d’un autre projet
</div>

Spécifiez le nom de l’artefact ainsi que celui du projet pour faire référence à un artefact. Vous pouvez également faire référence à des artefacts d’autres entités en indiquant le nom de l’artefact avec le nom de l’entité.

L’exemple de code suivant montre comment récupérer un artefact d’un autre projet pour l’utiliser comme entrée du run W&amp;B actuel.

```python
with wandb.init(project="<example>", job_type="<job-type>") as run:
    # Récupérer un artefact d'un autre projet depuis W&B et le marquer
    # comme entrée de ce run.
    artifact = run.use_artifact("my-project/artifact:alias")

    # Utiliser un artefact d'une autre entité et le marquer comme entrée
    # de ce run.
    artifact = run.use_artifact("my-entity/my-project/artifact:alias")

```

<div id="construct-and-use-an-artifact-simultaneously">
  ### Créer et utiliser un artefact simultanément
</div>

Créez et utilisez un artefact simultanément. Créez un objet artefact et transmettez-le à use&#95;artifact. Cela crée un artefact dans W&amp;B s’il n’existe pas encore. L’API [`wandb.Run.use_artifact()`](/fr/models/ref/python/experiments/run#use_artifact) est idempotente, vous pouvez donc l’appeler autant de fois que vous le souhaitez.

```python
import wandb

with wandb.init(project="<example>", job_type="<job-type>") as run:
    artifact = wandb.Artifact("reference model")
    artifact.add_file("model.h5")
    run.use_artifact(artifact)
```

Pour plus d&#39;informations sur la création d&#39;un artefact, voir [Construire un artefact](/fr/models/artifacts/construct-an-artifact/).
