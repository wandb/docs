title: Explorer les graphes de lignage des artefacts
---

W&amp;B suit les entrées et sorties des Runs à l&#39;aide de graphes orientés acycliques (DAG), appelés *graphes de lignage*. Les graphes de lignage sont des représentations visuelles des relations entre les artefacts et les Runs dans une expérience de ML. Ils montrent comment les données et les modèles
circulent à travers les différentes étapes du cycle de vie du ML, de l&#39;ingestion des données brutes à l&#39;entraînement et à l&#39;évaluation du modèle.

Le suivi de la traçabilité des artefacts offre plusieurs avantages clés :

* **Reproductibilité** : permet aux équipes de reproduire des expériences, des modèles et des résultats à des fins de débogage, d&#39;expérimentation et de validation.
* **Contrôle de version** : suit les modifications apportées aux artefacts au fil du temps, ce qui permet aux équipes de revenir à des versions antérieures des données ou des versions de modèle si nécessaire.
* **Audit** : conserve un historique détaillé des artefacts et des transformations pour soutenir la conformité et la gouvernance.
* **Collaboration** : aide les équipes à mieux collaborer en rendant l&#39;historique des expériences transparent, en réduisant les efforts dupliqués et en accélérant le développement.

<div id="view-an-artifacts-lineage-graph">
  ## Afficher le graphe de lignage d&#39;un artefact
</div>

Pour afficher le graphe de lignage d&#39;un artefact :

1. Accédez au Workspace de votre projet dans la W&amp;B App.
2. Cliquez sur l&#39;onglet **Artifacts** dans la barre latérale du projet.
3. Sélectionnez un artefact, puis cliquez sur l&#39;onglet **Lineage**.

<div id="navigate-lineage-graphs">
  ## Naviguer dans les graphes de lignage
</div>

Le graphe de lignage est une représentation visuelle des relations entre les artefacts et les runs dans une expérience de ML. Utilisez l’interface de l’application W&amp;B ou le SDK Python pour explorer et parcourir le graphe de lignage d’un artefact.

<Tabs>
  <Tab title="W&B App UI">
    Les nœuds avec des icônes vertes représentent des runs. Les nœuds avec des icônes bleues représentent des artefacts. Les flèches entre les nœuds indiquent les entrées et les sorties d’un run ou d’un artefact.

    Les nœuds d’artefact affichent le nom de l’artefact ainsi que sa version sous la forme `<artifact_name>:<version>`. Le type d’un artefact s’affiche au-dessus du nom de l’artefact.

    <Note>
      Vous pouvez voir le type et le nom de l’artefact à la fois dans la barre latérale gauche et dans le nœud du graphe de lignage.
    </Note>

    Les nœuds de run affichent le nom du run.

  <Tab title="SDK Python W&B">
    Naviguez dans un graphe par programmation à l’aide du SDK Python W&amp;B. Utilisez les méthodes
    [`logged_by()`](/fr/models/ref/python/experiments/artifact#method-artifact-logged-by) et [`used_by()`](/fr/models/ref/python/experiments/artifact#method-artifact-used-by) d’un objet artefact pour parcourir le graphe :

    ```python
    with wandb.init() as run:
        artifact = run.use_artifact("artifact_name:latest")

        # Parcourez le graphe vers le haut et vers le bas à partir d’un artefact :
        producer_run = artifact.logged_by()
        consumer_runs = artifact.used_by()
    ```

<div id="enable-lineage-graph-tracking">
  ## Activer le suivi du graphe de lignage
</div>

Pour activer le suivi du graphe de lignage, vous devez désigner les artefacts comme [entrées](/fr/models/artifacts/explore-and-traverse-an-artifact-graph) ou
[sorties](/fr/models/artifacts/explore-and-traverse-an-artifact-graph#track-the-output-of-a-run) d&#39;un run à l&#39;aide du SDK Python W&amp;B.

<div id="track-the-input-of-a-run">
  ### Suivre l&#39;entrée d&#39;un run
</div>

Marquez un artefact comme entrée (ou dépendance) d&#39;un run à l&#39;aide de la
méthode [`wandb.Run.use_artifact()`](/fr/models/ref/python/experiments/run#method-runuse_artifact).
Indiquez le nom de l&#39;artefact et, si nécessaire, un alias pour référencer une version spécifique de cet artefact. Le nom de l&#39;artefact
est au format `<artifact_name>:<version>` ou `<artifact_name>:<alias>`.

Remplacez les valeurs entre chevrons (`< >`) par vos propres valeurs :

```python
import wandb

# Initialiser un run
with wandb.init(entity="<entity>", project="<project>") as run:
  # Obtenir l'artefact, le marquer comme dépendance
  artifact = run.use_artifact(artifact_or_name="<name>", aliases="<alias>")
```

<div id="track-the-output-of-a-run">
  ### Suivre la sortie d’un run
</div>

Utilisez [`wandb.Run.log_artifact()`](/fr/models/ref/python/experiments/run#log_artifact) pour déclarer un artefact comme sortie d’un run. Commencez par
créer un artefact avec le constructeur [`wandb.Artifact()`](/fr/models/ref/python/experiments/artifact#wandb.Artifact). Ensuite, consignez
l’artefact comme sortie du run avec `wandb.Run.log_artifact()`.

Remplacez les valeurs entre chevrons (`< >`) par vos valeurs :

```python
import wandb

# Initialiser un run
with wandb.init(entity="<entity>", project="<project>") as run:
  
  # Créer un artefact
  artifact = wandb.Artifact(name = "<artifact_name>", type = "<artifact_type>")
  artifact.add_file(local_path = "<local_filepath>", name="<optional-name>")

  # Enregistrer l'artefact en tant que sortie du run
  run.log_artifact(artifact_or_path = artifact)
```

<div id="artifact-clusters">
  ## Clusters d’artefacts
</div>

Lorsqu’un niveau du graphe comporte cinq runs ou artefacts ou plus, il crée un cluster. Un cluster dispose d’une barre de recherche pour trouver des versions spécifiques de runs ou d’artefacts et extrait un nœud individuel d’un cluster pour continuer à examiner la traçabilité d’un nœud à l’intérieur d’un cluster.

Cliquer sur un nœud ouvre un aperçu avec une vue d’ensemble du nœud. Cliquer sur la flèche extrait le run ou l’artefact individuel afin que vous puissiez examiner la traçabilité du nœud extrait.
