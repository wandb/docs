title: Explorer les graphes de traçabilité des artefacts
---

W&amp;B suit les entrées et les sorties des runs à l’aide de graphes orientés acycliques (DAG), appelés *graphes de traçabilité*. Les graphes de traçabilité sont des représentations visuelles des relations entre les artefacts et les runs dans une expérience de ML. Ils montrent comment les données et les modèles
circulent à travers les différentes étapes du cycle de vie du ML, de l’ingestion des données brutes à l’entraînement et à l’évaluation du modèle.

Le suivi de la traçabilité des artefacts offre plusieurs avantages clés :

* **Reproductibilité** : permet aux équipes de reproduire des expériences, des modèles et des résultats à des fins de débogage, d’expérimentation et de validation.
* **Contrôle de version** : suit les modifications apportées aux artefacts au fil du temps, ce qui permet aux équipes de revenir à des versions antérieures des données ou des modèles si nécessaire.
* **Audit** : conserve un historique détaillé des artefacts et des transformations pour faciliter la conformité et la gouvernance.
* **Collaboration** : aide les équipes à mieux collaborer en rendant l’historique des expériences transparent, en réduisant les doublons et en accélérant le développement.

<div id="view-an-artifacts-lineage-graph">
  ## Afficher le graphe de traçabilité d&#39;un artefact
</div>

Pour afficher le graphe de traçabilité d&#39;un artefact :

1. Accédez à l&#39;espace de travail de votre projet dans l&#39;application W&amp;B.
2. Cliquez sur l&#39;onglet **Artifacts** dans la barre latérale du projet.
3. Sélectionnez un artefact, puis cliquez sur l&#39;onglet **Lineage**.

<div id="navigate-lineage-graphs">
  ## Naviguer dans les graphes de traçabilité
</div>

Le graphe de traçabilité est une représentation visuelle des relations entre les artefacts et les runs dans une expérience de ML. Utilisez l’interface de l’application W&amp;B ou le SDK Python pour explorer et parcourir le graphe de traçabilité d’un artefact.

<Tabs>
  <Tab title="Interface de l’application W&B">
    Les nœuds avec des icônes vertes représentent des runs. Les nœuds avec des icônes bleues représentent des artefacts. Les flèches entre les nœuds indiquent les entrées et les sorties d’un run ou d’un artefact.

    Les nœuds d’artefact affichent le nom de l’artefact ainsi que sa version sous la forme `<artifact_name>:<version>`. Le type d’un artefact s’affiche au-dessus du nom de l’artefact.

    <Note>
      Vous pouvez voir le type et le nom de l’artefact à la fois dans la barre latérale gauche et dans le nœud du graphe de traçabilité.
    </Note>

    Les nœuds de run affichent le nom du run.

    Cliquez sur un run individuel pour obtenir plus d’informations à son sujet, comme son heure de début, sa durée, son auteur, son type de job, et plus encore. Cliquez sur un artefact individuel pour obtenir plus d’informations à son sujet, comme ses alias, son heure de création, son type, sa version, sa description, le run qui a journalisé l’artefact, la taille du fichier, et plus encore.

    Cliquez sur un nœud et faites-le glisser pour réorganiser le graphe afin d’en personnaliser la mise en page. Vous pouvez également zoomer et dézoomer sur le graphe pour mieux voir les nœuds et leurs relations.

    <Frame>

  </Tab>

  <Tab title="SDK Python W&B">
    Naviguez dans un graphe par programmation à l’aide du SDK Python W&amp;B. Utilisez les méthodes
    [`logged_by()`](/fr/models/ref/python/experiments/artifact#method-artifact-logged-by) et [`used_by()`](/fr/models/ref/python/experiments/artifact#method-artifact-used-by) d’un objet artefact pour parcourir le graphe :

    ```python
    with wandb.init() as run:
        artifact = run.use_artifact("artifact_name:latest")

        # Parcourez le graphe vers le haut et vers le bas à partir d’un artefact :
        producer_run = artifact.logged_by()
        consumer_runs = artifact.used_by()
    ```
  </Tab>
</Tabs>

<div id="enable-lineage-graph-tracking">
  ## Activer le suivi du graphe de traçabilité
</div>

Pour activer le suivi du graphe de traçabilité, vous devez désigner les artefacts comme [entrées](/fr/models/artifacts/explore-and-traverse-an-artifact-graph) ou
[sorties](/fr/models/artifacts/explore-and-traverse-an-artifact-graph#track-the-output-of-a-run) d’un run à l’aide du SDK Python de W&amp;B.

<div id="track-the-input-of-a-run">
  ### Suivre l’entrée d’un run
</div>

Marquez un artefact comme entrée (ou dépendance) d’un run à l’aide de la
méthode [`wandb.Run.use_artifact()`](/fr/models/ref/python/experiments/run#method-runuse_artifact).
Indiquez le nom de l’artefact et, si nécessaire, un alias pour référencer une version spécifique de cet artefact. Le nom de l’artefact
est au format `<artifact_name>:<version>` ou `<artifact_name>:<alias>`.

Remplacez les valeurs entre chevrons (`< >`) par vos propres valeurs :

```python
import wandb

# Initialiser un run
with wandb.init(entity="<entity>", project="<project>") as run:
  # Obtenir l'artefact, le marquer comme dépendance
  artifact = run.use_artifact(artifact_or_name="<name>", aliases="<alias>")
```

<div id="track-the-output-of-a-run">
  ### Suivre la sortie d’un run
</div>

Utilisez [`wandb.Run.log_artifact()`](/fr/models/ref/python/experiments/run#log_artifact) pour déclarer un artefact comme sortie d’un run. Commencez par créer un artefact avec le constructeur [`wandb.Artifact()`](/fr/models/ref/python/experiments/artifact#wandb.Artifact). Ensuite, journalisez l’artefact comme sortie du run avec `wandb.Run.log_artifact()`.

Remplacez les valeurs entre chevrons (`< >`) par vos valeurs :

```python
import wandb

# Initialiser un run
with wandb.init(entity="<entity>", project="<project>") as run:
  
  # Créer un artefact
  artifact = wandb.Artifact(name = "<artifact_name>", type = "<artifact_type>")
  artifact.add_file(local_path = "<local_filepath>", name="<optional-name>")

  # Enregistrer l'artefact en tant que sortie du run
  run.log_artifact(artifact_or_path = artifact)
```

<div id="artifact-clusters">
  ## Clusters d’artefacts
</div>

Lorsqu’un niveau du graphe comporte au moins cinq runs ou artefacts, un cluster est créé. Un cluster comprend une barre de recherche pour trouver des versions spécifiques de runs ou d’artefacts et permet d’extraire un nœud individuel du cluster afin de poursuivre l’examen de la traçabilité d’un nœud qu’il contient.

En cliquant sur un nœud, vous ouvrez un aperçu avec une vue d’ensemble du nœud. En cliquant sur la flèche, vous extrayez le run ou l’artefact individuel afin de pouvoir examiner la traçabilité du nœud extrait.
