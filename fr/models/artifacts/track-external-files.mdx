title: Suivre les fichiers externes
---

Utilisez des *artefacts de référence* pour suivre et utiliser des fichiers enregistrés en dehors des serveurs W&amp;B. Les solutions de stockage externe courantes incluent : CoreWeave AI Object Storage, un bucket Amazon Simple Storage Service (Amazon S3), un bucket GCS, un blob Azure, un serveur de fichiers HTTP ou un partage NFS.

Les artefacts de référence se comportent comme des artefacts sans référence. La principale différence est que les artefacts de référence ne consistent qu’en métadonnées sur les fichiers, telles que leurs tailles et leurs sommes de contrôle MD5. Les fichiers eux-mêmes ne quittent jamais votre système.

Vous pouvez interagir avec un artefact de référence comme avec des artefacts sans référence. Dans la W&amp;B App, vous pouvez parcourir le contenu de l’artefact de référence à l’aide du navigateur de fichiers, explorer le graphe de dépendances complet et parcourir l’historique versionné de votre artefact. En revanche, l’interface utilisateur ne peut pas afficher de médias enrichis tels que des images ou de l’audio, car les données elles-mêmes ne sont pas contenues dans l’artefact.

<Note>
  Si vous journalisez un artefact qui ne suit pas de fichiers externes, W&amp;B enregistre les fichiers de l’artefact sur les serveurs W&amp;B. Il s’agit du comportement par défaut lorsque vous journalisez des artefacts avec le SDK Python W&amp;B.

  Si vous journalisez un artefact qui suit des fichiers externes, W&amp;B enregistre des métadonnées sur l’objet, telles que l’ETag de l’objet et sa taille. Si la gestion des versions des objets est activée sur le bucket, l’ID de version est également enregistré.
</Note>

Les sections suivantes décrivent comment suivre des artefacts de référence externes.

<div id="track-an-artifact-in-an-external-bucket">
  ## Suivre un artefact dans un bucket externe
</div>

Utilisez le SDK Python de W&amp;B pour suivre des références vers des fichiers stockés en dehors de W&amp;B.

1. Initialisez un run avec `wandb.init()`.
2. Créez un objet artefact avec `wandb.Artifact()`.
3. Spécifiez la référence au chemin du bucket à l’aide de la méthode `wandb.Artifact.add_reference()` de l’objet artefact.
4. Journalisez les métadonnées de l’artefact avec `run.log_artifact()`.

```python
import wandb

# Initialiser un run W&B
with wandb.init(project="my-project") as run:

  # Créer un objet artefact
  artifact = wandb.Artifact(name="name", type="type")

  # Ajouter une référence au chemin du bucket
  artifact.add_reference(uri = "uri/to/your/bucket/path")

  # Journaliser les métadonnées de l'artefact
  run.log_artifact(artifact)
```

Par exemple, supposons que votre bucket ait l’arborescence suivante :

```text
s3://my-bucket

|datasets/
  |-- mnist/
|models/
  |-- cnn/
```

Le répertoire `datasets/mnist/` contient une collection d’images. Pour suivre le répertoire `datasets/mnist/` en tant qu’artefact de jeu de données, spécifiez :

1. Donnez un nom à l’artefact, par exemple `"mnist"`.
2. Définissez le paramètre `type` sur `"dataset"` lorsque vous créez l’objet artefact (`wandb.Artifact(type="dataset")`).
3. Indiquez le chemin du répertoire `datasets/mnist/` sous la forme d’un URI Amazon S3 (`s3://my-bucket/datasets/mnist/`) lorsque vous appelez `wandb.Artifact.add_reference()`.
4. Journalisez l’artefact avec `run.log_artifact()`.

L’exemple de code suivant crée un artefact de référence `mnist:latest` :

```python
import wandb

with wandb.init(project="my-project") as run:
  artifact = wandb.Artifact(name="mnist", type="dataset")
  artifact.add_reference(uri="s3://my-bucket/datasets/mnist")
  run.log_artifact(artifact)
```

Dans la W&amp;B App, vous pouvez parcourir le contenu de l’artefact de référence à l’aide du navigateur de fichiers, [explorer le graphe de dépendances complet](/fr/models/artifacts/explore-and-traverse-an-artifact-graph/), et parcourir l’historique versionné de votre artefact. La W&amp;B App n’affiche pas les médias enrichis tels que les images, l’Audio, etc., car les données elles-mêmes ne sont pas contenues dans l’artefact.

<Note>
  W&amp;B Artifacts prend en charge toute interface compatible Amazon S3, y compris CoreWeave Storage et MinIO. Les scripts décrits ci-dessous fonctionnent tels quels avec les deux fournisseurs, à condition de définir la variable d’environnement `AWS_S3_ENDPOINT_URL` pour qu’elle pointe vers votre serveur CoreWeave Storage ou MinIO.
</Note>

<Warning>
  Par défaut, W&amp;B impose une limite de 10 000 objets lors de l’ajout d’un préfixe d’objet. Vous pouvez ajuster cette limite en spécifiant `max_objects=` lors de l’appel à `wandb.Artifact.add_reference()`.
</Warning>

<div id="download-an-artifact-from-an-external-bucket">
  ## Télécharger un artefact depuis un bucket externe
</div>

W&amp;B récupère les fichiers du bucket sous-jacent lorsqu&#39;il télécharge un artefact de référence, à l&#39;aide des métadonnées enregistrées au moment où l&#39;artefact est journalisé. Si la gestion des versions des objets est activée sur votre bucket, W&amp;B récupère la version de l&#39;objet correspondant à l&#39;état du fichier au moment où l&#39;artefact a été journalisé. À mesure que le contenu de votre bucket évolue, vous pouvez toujours retrouver la version exacte de vos données sur laquelle un modèle donné a été entraîné, car l&#39;artefact sert d&#39;instantané de votre bucket pendant le run d&#39;entraînement.

L&#39;exemple de code suivant montre comment télécharger un artefact de référence. Les API de téléchargement d&#39;artefacts sont les mêmes pour les artefacts de référence et les artefacts sans référence :

```python
import wandb

with wandb.init(project="my-project") as run:
  artifact = run.use_artifact("mnist:latest", type="dataset")
  artifact_dir = artifact.download()
```

<Note>
  W&amp;B vous recommande d’activer &#39;Object Versioning&#39; sur vos buckets de stockage si vous écrasez des fichiers dans le cadre de votre flux de travail.

  Si la gestion des versions est activée, W&amp;B peut toujours récupérer la bonne version du fichier lorsque vous téléchargez un artefact, même si le fichier a été écrasé depuis que l’artefact a été enregistré.

  Selon votre cas d’utilisation, consultez les instructions pour activer la gestion des versions des objets : [AWS](https://docs.aws.amazon.com/AmazonS3/latest/userguide/manage-versioning-examples.html), [Google Cloud](https://cloud.google.com/storage/docs/using-object-versioning#set), [Azure](https://learn.microsoft.com/azure/storage/blobs/versioning-enable).
</Note>

<div id="add-and-download-an-external-from-a-bucket">
  ## Ajouter et télécharger un élément externe à partir d’un bucket
</div>

L’exemple de code suivant téléverse un jeu de données dans un bucket Amazon S3, le suit à l’aide d’un artefact de référence, puis le télécharge :

```python
import boto3
import wandb

with wandb.init() as run:
  # Entraînement ici...

  s3_client = boto3.client("s3")
  s3_client.upload_file(file_name="my_model.h5", bucket="my-bucket", object_name="models/cnn/my_model.h5")

  # Journaliser l'artefact du modèle
  model_artifact = wandb.Artifact("cnn", type="model")
  model_artifact.add_reference("s3://my-bucket/models/cnn/")
  run.log_artifact(model_artifact)
```

Vous pourrez ensuite télécharger l’artefact du modèle. Indiquez le nom de l’artefact et son type :

```python
import wandb

with wandb.init() as run:
  artifact = run.use_artifact(artifact_or_name = "cnn", type="model")
  datadir = artifact.download()
```

<Note>
  Voir les rapports suivants pour un guide pas à pas complet sur le suivi des artefacts par référence avec Google Cloud ou Azure :

  * [Guide pour le suivi des artefacts par référence avec Google Cloud](https://wandb.ai/stacey/artifacts/reports/Tracking-Artifacts-by-Reference--Vmlldzo1NDMwOTE)
  * [Utiliser des artefacts de référence dans Microsoft Azure](https://wandb.ai/andrea0/azure-2023/reports/Efficiently-Harnessing-Microsoft-Azure-Blob-Storage-with-Weights-Biases--Vmlldzo0NDA2NDgw)

<div id="cloud-storage-credentials">
  ## Identifiants de stockage dans le cloud
</div>

W&amp;B utilise le mécanisme par défaut pour rechercher les identifiants en fonction du fournisseur de cloud que vous utilisez. Consultez la documentation de votre fournisseur de cloud pour en savoir plus sur les identifiants utilisés :

| Fournisseur de cloud        | Documentation sur les identifiants                                                                                                                          |
| --------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| CoreWeave AI Object Storage | [documentation CoreWeave AI Object Storage](https://docs.coreweave.com/docs/products/storage/object-storage/how-to/manage-access-keys/cloud-console-tokens) |
| AWS                         | [documentation Boto3](https://boto3.amazonaws.com/v1/documentation/api/latest/guide/credentials.html#configuring-credentials)                               |
| Google Cloud                | [documentation Google Cloud](https://cloud.google.com/docs/authentication/provide-credentials-adc)                                                          |
| Azure                       | [documentation Azure](https://learn.microsoft.com/python/api/azure-identity/azure.identity.defaultazurecredential?view=azure-python)                        |

Pour AWS, si le bucket ne se trouve pas dans la région par défaut de l&#39;utilisateur configuré, vous devez définir la variable d&#39;environnement `AWS_REGION` pour qu&#39;elle corresponde à la région du bucket.

<Warning>
  Les médias enrichis tels que les images, l&#39;audio, la vidéo et les nuages de points peuvent ne pas s&#39;afficher correctement dans l&#39;interface de l&#39;application en fonction de la configuration CORS de votre bucket. Ajouter **app.wandb.ai** à la liste d&#39;autorisation dans les paramètres CORS de votre bucket permettra à la W&amp;B App d&#39;afficher correctement ces médias enrichis.

  Si des médias enrichis tels que des images, de l&#39;audio, de la vidéo et des nuages de points ne s&#39;affichent pas dans l&#39;interface de l&#39;application, assurez-vous que `app.wandb.ai` figure dans la liste d&#39;autorisation de la politique CORS de votre bucket.
</Warning>

<div id="track-an-artifact-in-a-filesystem">
  ## Suivre un artefact dans un système de fichiers
</div>

Une approche courante pour accéder aux jeux de données consiste à exposer, sur toutes les machines exécutant des tâches d&#39;entraînement, un point de montage NFS d&#39;un système de fichiers distant. Cela peut constituer une alternative à un bucket de stockage cloud, car du point de vue du script d&#39;entraînement, les fichiers apparaissent comme des fichiers locaux.

{/* Utilisez W&B Artifacts pour suivre les références à des systèmes de fichiers, qu’ils soient montés ou non. */}

Pour suivre un artefact dans un système de fichiers :

1. Initialisez un run avec `wandb.init()`.
2. Créez un objet artefact avec `wandb.Artifact()`.
3. Spécifiez la référence vers le chemin du système de fichiers à l’aide de la méthode `wandb.Artifact.add_reference()` de l’objet artefact.
4. Journalisez les métadonnées de l’artefact avec `run.log_artifact()`.

Copiez-collez l’extrait de code suivant pour suivre des fichiers dans un système de fichiers monté. Remplacez les valeurs entre chevrons (`< >`) par vos propres valeurs.

```python
import wandb

# Initialiser un run
with wandb.init(entity="<entity>", project="<project>") as run:

  # Créer un objet artefact
  artifact = wandb.Artifact(name="<name>", type="<type>")

  # Ajouter une référence au chemin du système de fichiers
  artifact.add_reference("file:///path/to/dataset/")

  # Journaliser l'artefact (métadonnées uniquement)
  run.log_artifact(artifact)
```

Notez la triple barre oblique dans l’URL. Le premier composant est le préfixe `file://`, qui indique l’utilisation de références de système de fichiers. Le deuxième composant est la racine `/` du système de fichiers. Les composants restants correspondent au chemin vers le répertoire ou le fichier que vous souhaitez suivre.

À titre d’exemple, supposons que vous ayez un système de fichiers monté sur `/mount` avec la structure suivante :

```text
mount
|datasets/
  |-- mnist/
|models/
  |-- cnn/
```

Vous souhaitez suivre le répertoire `datasets/mnist/` en tant qu’artefact de jeu de données. Pour ce faire, vous pouvez utiliser l’extrait de code suivant.

```python
import wandb

with wandb.init() as run:
  artifact = wandb.Artifact("mnist", type="dataset")
  artifact.add_reference("file:///mount/datasets/mnist/")
  run.log_artifact(artifact)
```

Cela crée un artefact de référence `mnist:latest` qui pointe vers les fichiers stockés dans `/mount/datasets/mnist/`.

<Warning>
  Par défaut, W&amp;B impose une limite de 10 000 fichiers lors de l&#39;ajout d&#39;une référence à un répertoire. Vous pouvez ajuster cette limite en spécifiant `max_objects=` lors de l&#39;appel à `wandb.Artifact.add_reference()`.
</Warning>

De même, pour suivre un modèle stocké dans `models/cnn/my_model.h5`, vous pouvez utiliser l&#39;extrait de code suivant :

```python
import wandb

with wandb.init() as run:

  # Entraînement ici...

  # Écrire le modèle sur le disque

  # Créer un objet artefact
  model_artifact = wandb.Artifact("cnn", type="model")

  # Ajouter une référence au chemin du fichier modèle
  model_artifact.add_reference("file:///mount/cnn/my_model.h5")

  # Journaliser l'artefact dans W&B
  run.log_artifact(model_artifact)
```

<div id="download-an-artifact-from-an-external-filesystem">
  ## Télécharger un artefact depuis un système de fichiers externe
</div>

Téléchargez des fichiers à partir d’un système de fichiers référencé à l’aide des mêmes API que pour les artefacts sans référence :

1. Initialisez un run avec `wandb.init()`.
2. Utilisez la méthode `wandb.Run.use_artifact()` pour spécifier l’artefact que vous souhaitez télécharger.
3. Appelez la méthode `wandb.Artifact.download()` de l’artefact pour télécharger les fichiers à partir du système de fichiers référencé

```python
with wandb.init() as run:
  artifact = run.use_artifact("entity/project/mnist:latest", type="dataset")
  artifact_dir = artifact.download()
```

W&amp;B copie le contenu de `/mount/datasets/mnist` dans le répertoire `artifacts/mnist:v0/`.

<Info>
  `Artifact.download()` lève une erreur s&#39;il ne peut pas reconstruire l&#39;artefact. Par exemple, si un artefact contient une référence à un fichier qui a été écrasé, `Artifact.download()` lève une erreur, car l&#39;artefact ne peut plus être reconstruit.
</Info>

{/* ### Journaliser des artefacts en dehors des runs

  Utilisez la commande [`wandb artifact put`](/models/ref/cli/wandb-artifact/wandb-artifact-put) pour téléverser un artefact vers le serveur W&B en dehors d’un run W&B.  
  ```bash
  $ wandb artifact put --name project/artifact_name --type TYPE PATH
  ```

  W&B créera un nouveau projet si le projet que vous spécifiez n’existe pas. Pour des informations sur la façon de télécharger un artefact, voir [Download and use artifacts](/models/artifacts/download-and-use-an-artifact). */}
