Cette page décrit les *automatisations* dans W&amp;B. [Créez une automatisation](/fr/models/automations/create-automations/) pour déclencher des étapes de flux de travail, telles que les tests et le déploiement automatisés de modèles, basé sur un événement dans W&amp;B.

Par exemple, une automatisation peut notifier un canal Slack lorsqu&#39;une nouvelle version est créée, déclencher un webhook de test automatisé lorsque l&#39;alias `production` est ajouté à un Artifact, ou démarrer un job de validation uniquement lorsque la `perte` d&#39;un run se situe dans des limites acceptables.

<div id="automation-events">
  ## Événements d&#39;automatisation
</div>

Les automatisations peuvent être déclenchées par des événements liés aux runs ou aux Artifacts d&#39;une collection, d&#39;un projet ou d&#39;un registre. Une automatisation peut être lancée :

* Lorsqu&#39;une métrique de run atteint un seuil absolu ou relatif défini. Par exemple, déclenchez-la lorsque la `loss` d&#39;un run est inférieure à 0.01 ou lorsque l&#39;`accuracy` d&#39;un run s&#39;améliore de 5 %.
* Lorsque le score z (score standard) d&#39;un run s&#39;écarte de la moyenne d&#39;une valeur donnée. Un score z de 0 indique que le résultat correspond à la moyenne. Un score z de +2 signifie que le résultat est supérieur de deux écarts types à la moyenne, et un score z de -2 signifie que le résultat est inférieur de deux écarts types à la moyenne.
* Lorsqu&#39;un événement se produit dans un projet ou un registre. Par exemple, déclenchez-la lorsqu&#39;une nouvelle version d&#39;un artefact de modèle est créée ou lorsque l&#39;alias `production` est ajouté à un artefact de modèle.

Pour plus de détails, voir [événements et portée des automatisations](/fr/models/automations/automation-events/).

<div id="automation-actions">
  ## Actions d’automatisation
</div>

Lorsqu’un événement déclenche une automatisation, elle peut effectuer l’une de ces actions :

<div id="slack-notification">
  ### Notification Slack
</div>

Envoyez un message dans un canal Slack avec des détails sur l’événement qui a déclenché la notification. Le message résume l’événement et inclut un lien pour afficher plus de détails dans W&amp;B.

<div id="webhook">
  ### Webhook
</div>

Appelez une URL de webhook avec un payload JSON contenant des informations sur l’événement déclencheur. Cela permet l’intégration avec des systèmes externes tels que des pipelines CI/CD, des services de déploiement de modèles ou des flux de travail personnalisés. Le corps de la requête webhook peut contenir n’importe quel payload sérialisable en JSON.

Pour plus de détails sur l’implémentation, voir :

* [Créer une automatisation Slack](/fr/models/automations/create-automations/slack/)
* [Créer une automatisation webhook](/fr/models/automations/create-automations/webhook/)

<div id="how-automations-work">
  ## Fonctionnement des automatisations
</div>

Pour [créer une automatisation](/fr/models/automations/create-automations/), vous devez :

1. Si nécessaire, configurer les [secrets](/fr/platform/secrets/) pour les chaînes de caractères sensibles requises par l’automatisation, comme les jetons d’accès, les mots de passe ou les détails de configuration sensibles. Les secrets sont définis dans vos **Team settings**. Ils sont le plus souvent utilisés dans les automatisations webhook pour transmettre en toute sécurité des identifiants ou des jetons au service externe du webhook, sans les exposer en clair ni les coder en dur dans son payload.
2. Configurer des intégrations webhook ou Slack au niveau de l’équipe afin d’autoriser W&amp;B à publier sur Slack ou à exécuter le webhook en votre nom. Une même action d’automatisation (webhook ou notification Slack) peut être utilisée par plusieurs automatisations. Ces actions sont définies dans vos **Team settings**.
3. Dans le projet ou le registre, créer l’automatisation :
   1. Définir l’[événement](#automation-events) à surveiller, par exemple lorsqu’une nouvelle version d’artefact est ajoutée.
   2. Définir l’action à exécuter lorsque l’événement se produit (publication dans un canal Slack ou exécution d’un webhook). Pour un webhook, spécifier un secret à utiliser pour le jeton d’accès et/ou un secret à envoyer avec le payload, si nécessaire.

  ## Limites
