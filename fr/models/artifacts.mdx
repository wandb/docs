Utilisez W&amp;B Artifacts pour suivre et versionner les données en tant qu’entrées et sorties de vos [W&amp;B Runs](/fr/models/runs). Par exemple, un run d’entraînement du modèle peut prendre un jeu de données en entrée et produire un modèle entraîné en sortie. Vous pouvez journaliser des hyperparamètres, des métadonnées et des métriques dans un run, et utiliser un artefact pour journaliser, suivre et versionner le jeu de données utilisé pour entraîner le modèle en entrée, ainsi qu’un autre artefact pour les points de contrôle du modèle produits en sortie.

<div id="use-cases">
  ## Cas d’usage
</div>

Vous pouvez utiliser des artefacts tout au long de votre flux de travail ML comme entrées et sorties de [runs](/fr/models/runs). Vous pouvez utiliser des jeux de données, des modèles ou même d’autres artefacts comme entrées pour le traitement.

| Cas d’usage                     | Entrée                                                   | Sortie                               |
| ------------------------------- | -------------------------------------------------------- | ------------------------------------ |
| Entraînement du modèle          | Jeu de données (données d’entraînement et de validation) | Modèle entraîné                      |
| Prétraitement du jeu de données | Jeu de données (données brutes)                          | Jeu de données (données prétraitées) |
| Évaluation du modèle            | Modèle + jeu de données (données de test)                | [Table W&amp;B](/fr/models/tables)      |
| Optimisation du modèle          | Modèle                                                   | Modèle optimisé                      |

<div id="create-an-artifact">
  ## Créer un artefact
</div>

Créez un artefact en quatre lignes de code :

1. Créez un [run W&amp;B](/fr/models/runs).
2. Créez un objet d’artefact avec [`wandb.Artifact`](/fr/models/ref/python/experiments/artifact).
3. Ajoutez un ou plusieurs fichiers, tels qu’un fichier de modèle ou un jeu de données, à l’objet artefact avec `wandb.Artifact.add_file()`.
4. Journalisez votre artefact dans W&amp;B avec `wandb.Run.log_artifact()`.

Par exemple, l’extrait de code suivant montre comment journaliser un fichier appelé `dataset.h5` dans un artefact appelé `example_artifact` :

```python
import wandb

with wandb.init(project="artifacts-example", job_type="add-dataset") as run:
    artifact = wandb.Artifact(name="example_artifact", type="dataset")
    artifact.add_file(local_path="./dataset.h5", name="training_dataset")
    run.log_artifact(artifact)
```

* Le `type` de l’artefact influe sur la façon dont il apparaît dans la plateforme W&amp;B. Si vous ne spécifiez pas de `type`, il prend par défaut la valeur `unspecified`.
* Chaque libellé de la liste déroulante correspond à une valeur différente du paramètre `type`. Dans l’extrait de code ci-dessus, le `type` de l’artefact est `dataset`.

<Note>
  Voir la page [suivre des fichiers externes](/fr/models/artifacts/track-external-files) pour savoir comment ajouter des références à des fichiers ou à des répertoires stockés dans un stockage d’objets externe, comme un bucket Amazon S3.
</Note>

<div id="download-an-artifact">
  ## Télécharger un artefact
</div>

Indiquez l’artefact que vous souhaitez marquer comme entrée de votre run à l’aide de la méthode [`wandb.Run.use_artifact()`](/fr/models/ref/python/experiments/run#use_artifact).

En reprenant l’extrait de code précédent, l’exemple de code suivant montre comment utiliser l’artefact nommé `example_artifact`, créé précédemment :

```python
with wandb.init(project="artifacts-example", job_type="add-dataset") as run:
    artifact = run.use_artifact("training_dataset:latest")  # renvoie un objet run utilisant l'artefact "my_data"
```

Cela renvoie un objet d’artefact.

Ensuite, utilisez l’objet renvoyé pour télécharger l’intégralité du contenu de l’artefact :

```python
datadir = artifact.download()  # télécharge l'artefact complet `my_data` dans le répertoire par défaut.
```

<Note>
  Vous pouvez spécifier un chemin personnalisé dans le [paramètre](/fr/models/ref/python/experiments/artifact) `root` pour télécharger un artefact dans un répertoire précis. Pour découvrir d&#39;autres façons de télécharger des artefacts et voir des paramètres supplémentaires, Voir le guide sur le [téléchargement et l&#39;utilisation des artefacts](/fr/models/artifacts/download-and-use-an-artifact).
</Note>
