---
title: Référencer une version d’artifact avec des alias
description: "Utilisez des alias par défaut, personnalisés et protégés pour référencer des versions d’artifact spécifiques dans W&B Registry."
---

Référencez une [version d’artifact](/fr/models/artifacts/create-a-new-artifact-version/) spécifique avec un ou plusieurs alias. [W&amp;B attribue automatiquement des alias](/fr/models/registry/aliases/#default-aliases) à chaque artifact que vous liez sous le même nom. Vous pouvez également [créer un ou plusieurs alias personnalisés](/fr/models/registry/aliases/#custom-aliases) pour référencer une version d’artifact spécifique.

Les alias apparaissent sous forme de rectangles portant leur nom dans l’interface du registre. Si un [alias est protégé](/fr/models/registry/aliases/#protected-aliases), il apparaît sous la forme d’un rectangle gris avec une icône de cadenas. Sinon, l’alias apparaît sous la forme d’un rectangle orange. Les alias ne sont pas partagés entre les registres.

<Note>
  **Quand utiliser un alias plutôt qu’un tag**

  Utilisez un alias pour référencer une version d’artifact spécifique. Chaque alias au sein d’une collection est unique. Une seule version d’artifact peut avoir un alias donné à la fois.

  Utilisez des tags pour organiser et regrouper des versions d’artifact ou des collections autour d’un thème commun. Plusieurs versions d’artifact et collections peuvent partager le même tag.
</Note>

Lorsque vous ajoutez un alias à une version d’artifact, vous pouvez, si vous le souhaitez, démarrer une [automation du registre](/fr/models/automations/automation-events/#registry) pour notifier un canal Slack ou déclencher un webhook.

<div id="default-aliases">
  ## Alias par défaut
</div>

W&amp;B attribue automatiquement les alias suivants à chaque version d&#39;artifact que vous liez sous le même nom :

* L&#39;alias `latest` à la version d&#39;artifact la plus récente que vous liez à une collection.
* Un numéro de version unique. W&amp;B compte chaque version d&#39;artifact que vous liez (à partir de l&#39;index zéro). W&amp;B utilise ce nombre pour attribuer un numéro de version unique à cet artifact.

Par exemple, si vous liez trois fois un artifact nommé `zoo_model`, W&amp;B crée respectivement trois alias : `v0`, `v1` et `v2`. `v2` a également l&#39;alias `latest`.

<div id="custom-aliases">
  ## Alias personnalisés
</div>

Créez un ou plusieurs alias personnalisés pour des versions spécifiques d&#39;artifact, selon votre cas d&#39;utilisation. Par exemple :

* Vous pouvez utiliser des alias comme `dataset_version_v0`, `dataset_version_v1` et `dataset_version_v2` pour identifier le jeu de données sur lequel un modèle a été entraîné.
* Vous pouvez utiliser l&#39;alias `best_model` pour suivre la version de modèle artifact la plus performante.

Tout utilisateur ayant le rôle de registre [**Member** ou **Admin**](/fr/models/registry/configure_registry/#registry-roles) sur un registre peut ajouter ou supprimer un alias personnalisé d&#39;un artifact lié dans ce registre. Les Users ayant les rôles [**Restricted Viewer** ou **Viewer**](/fr/models/registry/configure_registry/#registry-roles) ne peuvent pas ajouter ni supprimer d&#39;alias.

<Note>Les [alias protégés](/fr/models/registry/aliases/#protected-aliases) permettent d&#39;étiqueter et d&#39;identifier les versions d&#39;artifact à protéger contre toute modification ou suppression.</Note>

Vous pouvez créer un alias personnalisé dans le W&amp;B Registry ou avec le SDK Python. Selon votre cas d&#39;utilisation, cliquez sur l&#39;onglet ci-dessous qui correspond le mieux à vos besoins.

    2. Cliquez sur le bouton **View details** dans une collection.
    3. Dans la section **Versions**, cliquez sur le bouton **View** pour une version d&#39;artifact spécifique.
    4. Cliquez sur le bouton **+** pour ajouter un ou plusieurs alias à côté du champ **Aliases**.
  </Tab>

<div id="protected-aliases">
  ### Alias protégés
</div>

Utilisez un [alias protégé](/fr/models/registry/aliases/#protected-aliases) pour étiqueter et identifier des versions d&#39;artifact qui ne doivent pas être modifiées ou supprimées. Par exemple, vous pouvez utiliser un alias protégé `production` pour étiqueter et identifier des versions d&#39;artifact utilisées dans le pipeline de production de machine learning de votre organisation.

Les [administrateurs du registre](/fr/models/registry/configure_registry/#registry-roles) et les [comptes de service](/fr/support/models/articles/what-is-a-service-account-and-why-is-it-) disposant du rôle **Admin** peuvent créer des alias protégés et ajouter ou supprimer des alias protégés sur une version d&#39;artifact. Les Users et les comptes de service disposant des rôles **Member**, **Viewer** et **Restricted Viewer** ne peuvent pas dissocier une version protégée ni supprimer une collection contenant un alias protégé. Voir [Configurer l’accès au registre](/fr/models/registry/configure_registry/) pour plus de détails.

Les alias protégés courants incluent :

* **Production** : La version d&#39;artifact est prête à être utilisée en production.
* **Staging** : La version d&#39;artifact est prête pour les tests.

<div id="create-a-protected-alias">
  #### Créer un alias protégé
</div>

Les étapes suivantes expliquent comment créer un alias protégé dans l’UI de W&amp;B Registry :

1. Accédez à W&amp;B Registry.
2. Sélectionnez un registre.
3. Cliquez sur le bouton en forme d’engrenage en haut à droite de la page pour afficher les paramètres du registre.
4. Dans la section **Protected Aliases**, cliquez sur le bouton **+** pour ajouter un ou plusieurs alias protégés.

Après sa création, chaque alias protégé apparaît sous la forme d’un rectangle gris avec une icône de cadenas dans la section **Protected Aliases**.

<Note>
  Contrairement aux alias personnalisés non protégés, la création d’alias protégés est disponible exclusivement dans l’UI de W&amp;B Registry, et non par programmation avec le SDK Python. Pour ajouter un alias protégé à une version d’artifact, vous pouvez utiliser l’UI de W&amp;B Registry ou le SDK Python.
</Note>

Les étapes suivantes expliquent comment ajouter un alias protégé à une version d’artifact dans l’UI de W&amp;B Registry :

1. Accédez à W&amp;B Registry.
2. Cliquez sur le bouton **View details** dans une collection.
3. Dans la section **Versions**, sélectionnez le bouton **View** pour une version d’artifact spécifique.
4. Cliquez sur le bouton **+** pour ajouter un ou plusieurs alias protégés à côté du champ **Aliases**.

Une fois un alias protégé créé, un administrateur peut l’ajouter par programmation à une version d’artifact avec le SDK Python. Voir les onglets W&amp;B Registry et SDK Python dans la section [Create a custom alias](#custom-aliases) ci-dessus pour un exemple montrant comment ajouter un alias protégé à une version d’artifact.

<div id="find-existing-aliases">
  ## Trouver des alias existants
</div>

Vous pouvez trouver des alias existants à l’aide de la [barre de recherche globale du W&amp;B Registry](/fr/models/registry/search_registry/#search-for-registry-items). Pour trouver un alias protégé :

1. Accédez au W&amp;B Registry.
2. Saisissez le terme recherché dans la barre de recherche en haut de la page. Appuyez sur Entrée pour lancer la recherche.

Les résultats de recherche s’affichent sous la barre de recherche si le terme saisi correspond à un registre existant, à un nom de collection, à un tag de version d’artifact, à un tag de collection ou à un alias.
