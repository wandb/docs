---
title: Lier une version d’artefact à une collection
description: "Liez une version d’artefact à une collection dans W&B Registry pour la partager avec votre organisation."
---

Pour rendre une version d’artefact disponible pour votre organisation, *liez-la* à une [collection](/fr/models/registry/create_collection) dans
le W&amp;B Registry. Cette liaison fait passer la version d’une [portée privée au niveau du projet à une portée partagée au niveau de l’organisation](/fr/models/registry/create_registry#visibility-types). Vous
pouvez lier une version d’artefact [par programmation avec le SDK Python W&amp;B ou de manière interactive dans la W&amp;B App](/fr/models/registry/link_version#link-an-artifact-to-a-collection).

Lorsque vous liez un artefact, W&amp;B crée une référence entre l’artefact source et l’entrée de collection. La version liée renvoie vers la version de l’artefact source qui a été enregistrée dans un run d’un projet. Vous pouvez consulter à la fois la version liée dans la collection et la version source dans le projet où elle a été enregistrée.

<div id="link-an-artifact-to-a-collection">
  ## Lier un artefact à une collection
</div>

Selon votre cas d’usage, suivez les instructions indiquées dans les onglets ci-dessous pour lier une version d’artefact.

<Note>
  Avant de commencer, vérifiez les points suivants :

  * Les types d’artefacts autorisés par la collection. Pour plus d’informations sur les types de collection, voir &quot;Types de collection&quot; dans [Créer une collection](./create_collection).
  * Le registre auquel appartient la collection existe déjà. Pour vérifier que le registre existe, accédez à [l’application du registre et recherchez](/fr/models/registry/search_registry) le nom du registre.
</Note>

    Le chemin cible est composé du préfixe `"wandb-registry"`, du nom du registre et du nom de la collection, séparés par des barres obliques :

    {/* Utilisez le paramètre `target_path` pour indiquer la collection et le registre auxquels vous souhaitez lier la version de l’artefact. Le chemin cible est composé du préfixe "wandb-registry", du nom du registre et du nom de la collection, séparés par des barres obliques : */}

    ```text
    wandb-registry-{REGISTRY_NAME}/{COLLECTION_NAME}
    ```

    ### Lier une version d&#39;artefact dans le contexte d&#39;un run

    Utilisez `wandb.Run.link_artifact()` pour lier une version d&#39;artefact dans le contexte d&#39;un run. Pour ce faire,
    initialisez d&#39;abord un run avec `wandb.init()`. Ensuite, créez un objet artefact et ajoutez-y des fichiers. Enfin, utilisez la méthode `wandb.Run.link_artifact()` pour lier la version d&#39;artefact à la collection.

    Lorsque vous utilisez cette approche, un run est créé dans votre projet W&amp;B. La version de l’artefact est liée à la collection et associée à ce run.

    Copiez-collez l’extrait de code ci-dessous. Remplacez les valeurs entre `<>` par vos propres valeurs :

    ```python
    import wandb

    entity = "<team_entity>"          # Votre entité d'équipe
    project = "<project_name>"        # Le nom du projet contenant votre artefact

    # Initialiser un run
    with wandb.init(entity = entity, project = project) as run:

      # Créer un objet artefact
      # Le paramètre type spécifie à la fois le type de l'
      # objet artefact et le type de la collection
      artifact = wandb.Artifact(name = "<name>", type = "<type>")

      # Ajouter le fichier à l'objet artefact.
      # Indiquez le chemin vers le fichier sur votre machine locale.
      artifact.add_file(local_path = "<local_path_to_artifact>")

      # Spécifier la collection et le registre auxquels lier l'artefact
      REGISTRY_NAME = "<registry_name>"  
      COLLECTION_NAME = "<collection_name>"
      target_path=f"wandb-registry-{REGISTRY_NAME}/{COLLECTION_NAME}"

      # Lier l'artefact à la collection
      run.link_artifact(artifact = artifact, target_path = target_path)
    ```

    ### Lier une version d&#39;artefact en dehors du contexte d&#39;un run

    Utilisez `wandb.Artifact.link()` pour lier une version d&#39;artefact existante en dehors du contexte d&#39;un run. Avec cette approche,
    vous n&#39;avez pas besoin d&#39;initialiser un run avec `wandb.init()`. Aucun run n&#39;est donc créé dans votre projet W&amp;B. En d&#39;autres
    termes, la version de l&#39;artefact est liée à la collection sans être associée à un run.

    Commencez par créer un objet artefact et ajoutez-y des fichiers. Utilisez ensuite la méthode `wandb.Artifact.link()` pour lier la version de l&#39;artefact à la collection.

    Copiez-collez l&#39;extrait de code ci-dessous. Remplacez les valeurs entre `<>` par les vôtres :

    ```python
    import wandb

    # Créer un objet artefact
    # Le paramètre type spécifie à la fois le type de
    # l'objet artefact et le type de la collection
    artifact = wandb.Artifact(name = "<name>", type = "<type>")

    # Ajouter le fichier à l'objet artefact. 
    # Indiquez le chemin vers le fichier sur votre machine locale.
    artifact.add_file(local_path = "<local_path_to_artifact>")

    # Spécifier la collection et le registre auxquels lier l'artefact
    REGISTRY_NAME = "<registry_name>"  
    COLLECTION_NAME = "<collection_name>"
    target_path=f"wandb-registry-{REGISTRY_NAME}/{COLLECTION_NAME}"

    # Lier l'artefact à la collection
    artifact.link(target_path = target_path)
    ```

<div id="view-linked-artifacts-in-a-registry">
  ## Consulter les artefacts liés dans un registre
</div>

Consultez les informations sur les artefacts liés, comme les métadonnées, la traçabilité et les informations d’utilisation, dans le W&amp;B Registry.

1. Accédez au W&amp;B Registry.
2. Sélectionnez le nom du registre auquel vous avez lié l’artefact.
3. Sélectionnez le nom de la collection.
4. Si les artefacts de la collection consignent des métriques, comparez les métriques entre les versions en cliquant sur **Afficher les métriques**.
5. Dans la liste des versions d’artefact, sélectionnez la version que vous souhaitez consulter. Les numéros de version sont attribués de façon incrémentielle à chaque version d’artefact liée, à partir de `v0`.
6. Pour afficher les détails d’une version d’artefact, cliquez sur la version. Dans les onglets de cette page, vous pouvez consulter les métadonnées de cette version (y compris les métriques enregistrées), sa traçabilité et ses informations d’utilisation.

Notez le champ **Nom complet** dans l’onglet **Version**. Le nom complet d’un artefact lié se compose du registre, du nom de la collection et de l’alias ou de l’index de la version d’artefact.

```text title="Full name of a linked artifact"
wandb-registry-{REGISTRY_NAME}/{COLLECTION_NAME}:v{INTEGER}
```

Vous avez besoin du nom complet d’un artefact lié pour accéder par programmation à sa version.

<div id="troubleshooting">
  ## Dépannage
</div>

Voici quelques points courants à vérifier si vous ne parvenez pas à lier un artefact.

<div id="logging-artifacts-from-a-personal-account">
  ### Journalisation d’artefacts depuis un compte personnel
</div>

Les artefacts consignés dans W&amp;B avec une entité personnelle ne peuvent pas être liés au registre. Veillez à consigner vos artefacts à l’aide d’une entité d’équipe de votre organisation. Seuls les artefacts enregistrés au sein d’une équipe de l’organisation peuvent être liés au registre de l’organisation.

<Note>
  Veillez à consigner un artefact avec une entité d’équipe si vous souhaitez le lier à un registre.
</Note>

<div id="find-your-team-entity">
  #### Trouver l’entité de votre équipe
</div>

W&amp;B utilise le nom de votre équipe comme entité d’équipe. Par exemple, si votre équipe s’appelle **team-awesome**, son entité d’équipe est `team-awesome`.

Vous pouvez vérifier le nom de votre équipe comme suit :

1. Accédez à la page de profil W&amp;B de votre équipe.
2. Copiez l’URL du site. Elle se présente sous la forme `https://wandb.ai/<team>`, où `<team>` correspond à la fois au nom de votre équipe et à son entité.

<div id="log-from-a-team-entity">
  #### Enregistrer depuis une entité d’équipe
</div>

1. Spécifiez l’équipe comme `entity` lorsque vous initialisez un run avec [`wandb.init()`](/fr/models/ref/python/functions/init). Si vous ne spécifiez pas `entity` lors de l’initialisation du run, celui-ci utilise votre entité par défaut, qui peut être ou non votre entité d’équipe.

   ```python
   import wandb   

   with wandb.init(entity='<team_entity>', project='<project_name>') as run:
       # Enregistrez votre artefact ici
   ```

2. Enregistrez l’artefact dans le run, soit avec `wandb.Run.log_artifact()`, soit en créant un objet Artifact puis en y ajoutant des fichiers avec :

   ```python
   artifact = wandb.Artifact(name="<artifact_name>", type="<type>")
   ```

   Pour enregistrer des artefacts, voir [Construire des artefacts](/fr/models/artifacts/construct-an-artifact/).

3. Si un artefact est enregistré dans votre entité personnelle, vous devrez le réenregistrer dans une entité de votre organisation.

<div id="confirm-the-path-of-a-registry-in-the-wb-app-ui">
  ### Confirmer le chemin d’un registre dans l’interface de l’application W&amp;B
</div>

Il existe deux façons de confirmer le chemin d’un registre dans l’interface utilisateur : créer une collection vide et consulter les détails de la collection, ou copier-coller le code généré automatiquement sur la page d’accueil de la collection.

<div id="copy-and-paste-autogenerated-code">
  #### Copier-coller le code généré automatiquement
</div>

1. Accédez au W&amp;B Registry à l’adresse https://wandb.ai/registry/.
2. Cliquez sur le registre auquel vous souhaitez lier un artefact.
3. En haut de la page, vous verrez un bloc de code généré automatiquement.
4. Copiez-collez ce code dans votre code et veillez à remplacer la dernière partie du chemin par le nom de votre collection.

<Frame>
  <img src="/images/registry/get_autogenerated_code.gif" alt="Extrait de code généré automatiquement" />
</Frame>

<div id="create-an-empty-collection">
  #### Créer une collection vide
</div>

1. Accédez au W&amp;B Registry à l’adresse https://wandb.ai/registry/.
2. Cliquez sur le registre auquel vous souhaitez lier un artefact.
3. Cliquez sur la collection vide. Si aucune collection vide n’existe, créez-en une.
4. Dans l’extrait de code qui s’affiche, identifiez le champ `target_path` dans `.link_artifact()`.
5. (Facultatif) Supprimez la collection.

<Frame>
  <img src="/images/registry/check_empty_collection.gif" alt="Créer une collection vide" />
</Frame>

Par exemple, après avoir effectué les étapes ci-dessus, vous trouverez le bloc de code contenant le paramètre `target_path` :

```python
target_path = 
      "smle-registries-bug-bash/wandb-registry-Golden Datasets/raw_images"
```

En le décomposant en ses composants, vous pouvez voir ce que vous devrez utiliser pour créer le chemin permettant de lier votre artefact par programmation :

```python
ORG_ENTITY_NAME = "smle-registries-bug-bash"
REGISTRY_NAME = "Golden Datasets"
COLLECTION_NAME = "raw_images"
```

<Note>
  Veillez à remplacer le nom de la collection temporaire par celui de la collection à laquelle vous souhaitez lier votre artefact.
</Note>
