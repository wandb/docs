Utilisez le SDK Python de W&amp;B pour télécharger un artefact lié à un registre. Pour télécharger et utiliser un artefact, vous devez connaître le nom du registre, le nom de la collection, ainsi que l’alias ou l’index de la version de l’artefact que vous souhaitez télécharger.

Une fois que vous connaissez les propriétés de l’artefact, vous pouvez [construire le chemin vers l’artefact lié](#construct-path-to-linked-artifact) et télécharger l’artefact. Vous pouvez également [copier-coller un extrait de code pré-généré](#copy-and-paste-pre-generated-code-snippet) depuis l’interface de l’application W&amp;B pour télécharger un artefact lié à un registre.

<div id="construct-path-to-linked-artifact">
  ## Construire le chemin vers l&#39;artefact lié
</div>

Pour télécharger un artefact lié à un registre, vous devez connaître le chemin de cet artefact. Ce chemin se compose du nom du registre, du nom de la collection et de l&#39;alias ou de l&#39;index de la version de l&#39;artefact à laquelle vous voulez accéder.

Une fois que vous disposez du registre, de la collection et de l&#39;alias ou de l&#39;index de la version de l&#39;artefact, vous pouvez construire le chemin vers l&#39;artefact lié à l&#39;aide du modèle de chaîne de caractères suivant :

```python
# Nom de l'artefact avec l'index de version spécifié
f"wandb-registry-{REGISTRY}/{COLLECTION}:v{INDEX}"

# Nom de l'artefact avec l'alias spécifié
f"wandb-registry-{REGISTRY}/{COLLECTION}:{ALIAS}"
```

Remplacez les valeurs entre accolades `{}` par le nom du registre, de la collection, ainsi que l&#39;alias ou l&#39;index de la version de l&#39;artefact à laquelle vous souhaitez accéder.

Utilisez la méthode `wandb.Run.use_artifact()` pour accéder à l&#39;artefact et télécharger son contenu une fois que vous disposez du chemin de l&#39;artefact lié. L&#39;extrait de code suivant montre comment utiliser et télécharger un artefact lié au registre W&amp;B. Veillez à remplacer les valeurs entre `<>` par les vôtres :

```python
import wandb

REGISTRY = '<registry_name>'
COLLECTION = '<collection_name>'
ALIAS = '<artifact_alias>'

with wandb.init(entity = '<team_name>', project = '<project_name>')  as run:
    artifact_name = f"wandb-registry-{REGISTRY}/{COLLECTION}:{ALIAS}"
    # artifact_name = '<artifact_name>' # Copier-coller le nom complet indiqué dans l'interface W&B Registry
    fetched_artifact = run.use_artifact(artifact_or_name = artifact_name)  
    download_path = fetched_artifact.download()  
```

La méthode `wandb.Run.use_artifact()` crée à la fois un [run](/fr/models/runs) et définit l&#39;artefact que vous téléchargez comme entrée de ce run.
Le fait de définir un artefact comme entrée d&#39;un run permet à W&amp;B d&#39;en suivre la traçabilité.

Si vous ne souhaitez pas créer de run, vous pouvez utiliser l&#39;objet `wandb.Api()` pour accéder à l&#39;artefact :

```python
import wandb

REGISTRY = "<registry_name>"
COLLECTION = "<collection_name>"
VERSION = "<version>"

api = wandb.Api()
artifact_name = f"wandb-registry-{REGISTRY}/{COLLECTION}:{VERSION}"
artifact = api.artifact(name = artifact_name)
```

<details>
  <summary>Exemple : utiliser et télécharger un artefact lié au registre W&amp;B</summary>

  L’exemple de code suivant montre comment un utilisateur peut télécharger un artefact lié à une collection nommée `phi3-finetuned` dans le registre **Fine-tuned Models**. L’alias de la version de l’artefact est défini sur `production`.

  ```python
  import wandb

  TEAM_ENTITY = "product-team-applications"
  PROJECT_NAME = "user-stories"

  REGISTRY = "Fine-tuned Models"
  COLLECTION = "phi3-finetuned"
  ALIAS = 'production'

  # Initialisez un run dans l’équipe et le projet spécifiés
  with wandb.init(entity=TEAM_ENTITY, project = PROJECT_NAME) as run:

      artifact_name = f"wandb-registry-{REGISTRY}/{COLLECTION}:{ALIAS}"

      # Accédez à un artefact et marquez-le comme entrée de votre run pour assurer le suivi de la traçabilité
      fetched_artifact = run.use_artifact(artifact_or_name = artifact_name)  

      # Téléchargez l’artefact. Renvoie le chemin vers le contenu téléchargé
      downloaded_path = fetched_artifact.download()  
  ```
</details>

Voir [`wandb.Run.use_artifact()`](/fr/models/ref/python/experiments/run#use_artifact) et [`Artifact.download()`](/fr/models/ref/python/experiments/artifact#download) dans la Référence de l’API pour connaître les paramètres et le type de retour.

<Note>
  **Utilisateurs disposant d’une entité personnelle et appartenant à plusieurs organisations**

  Les utilisateurs disposant d’une entité personnelle et appartenant à plusieurs organisations doivent également spécifier soit le nom de leur organisation, soit utiliser une entité d’équipe lorsqu’ils accèdent à des artefacts liés à un registre.

  ```python
  import wandb

  REGISTRY = "<registry_name>"
  COLLECTION = "<collection_name>"
  VERSION = "<version>"

  # Assurez-vous d’utiliser votre entité d’équipe pour instancier l’API
  api = wandb.Api(overrides={"entity": "<team-entity>"})
  artifact_name = f"wandb-registry-{REGISTRY}/{COLLECTION}:{VERSION}"
  artifact = api.artifact(name = artifact_name)

  # Construisez le chemin
  api = wandb.Api()
  artifact_name = f"wandb-registry-{REGISTRY}/{COLLECTION}:{VERSION}"
  artifact = api.artifact(name = artifact_name)
  ```
</Note>

<div id="copy-and-paste-pre-generated-code-snippet">
  ## Copier-coller l’extrait de code prégénéré
</div>

W&amp;B génère un extrait de code que vous pouvez copier-coller dans votre script Python, notebook ou terminal pour télécharger un artefact lié à un registre.

1. Accédez à W&amp;B Registry.
2. Sélectionnez le nom du registre qui contient votre artefact.
3. Sélectionnez le nom de la collection.
4. Dans la liste des versions d’artefact, sélectionnez la version à laquelle vous souhaitez accéder.
5. Sélectionnez l’onglet **Usage**.
6. Copiez l’extrait de code affiché dans la section **Usage API**.
7. Collez l’extrait de code dans votre script Python, notebook ou terminal.
