title: Organiser les versions avec des tags
---

Créez et ajoutez des tags pour organiser vos collections ou les versions d&#39;artifact dans votre registre. Ajoutez, modifiez, affichez ou supprimez des tags sur une collection ou une version d&#39;artifact avec l&#39;interface utilisateur de W&amp;B App ou le SDK Python de W&amp;B.

<Note>
  **Quand utiliser un tag plutôt qu&#39;un alias**

  Utilisez des alias lorsque vous devez référencer de façon unique une version d&#39;artifact précise. Par exemple, utilisez un alias tel que &#39;production&#39; ou &#39;latest&#39; pour garantir que `artifact_name:alias` pointe toujours vers une seule version précise.

  Utilisez des tags lorsque vous avez besoin de plus de souplesse pour le regroupement ou la recherche. Les tags sont idéaux lorsque plusieurs versions ou collections peuvent partager le même libellé et que vous n&#39;avez pas besoin de la garantie qu&#39;une seule version soit associée à un identifiant donné.
</Note>

<div id="add-a-tag-to-a-collection">
  ## Ajouter un tag à une collection
</div>

Utilisez l’interface W&amp;B App ou le SDK Python pour ajouter un tag à une collection :

<Tabs>
  <Tab title="W&B App">
    Utilisez l’interface W&amp;B App pour ajouter un tag à une collection :

    1. Accédez à [W&amp;B Registry](https://wandb.ai/registry).
    2. Cliquez sur une carte du registre.
    3. Cliquez sur **View details** à côté du nom d’une collection.
    4. Dans la carte de la collection, cliquez sur l’icône plus (**+**) à côté du champ **Tags**, puis saisissez le nom du tag.
    5. Appuyez sur **Entrée** sur votre clavier.

    <Frame>
      <img src="/images/registry/add_tag_collection.gif" alt="Ajout de tags à une collection du registre" />
    </Frame>
  </Tab>

  <Tab title="Python SDK">
    ```python
    import wandb

    COLLECTION_TYPE = "<collection_type>"
    REGISTRY_NAME = "<registry_name>"
    COLLECTION_NAME = "<collection_name>"

    full_name = f"wandb-registry-{REGISTRY_NAME}/{COLLECTION_NAME}"

    collection = wandb.Api().artifact_collection(
      type_name = COLLECTION_TYPE, 
      name = full_name
      )

    collection.tags = ["your-tag"]
    collection.save()
    ```
  </Tab>
</Tabs>

<div id="update-tags-that-belong-to-a-collection">
  ## Mettre à jour les tags appartenant à une collection
</div>

Mettez à jour un tag par programmation en réattribuant l’attribut `tags` ou en le modifiant. W&amp;B recommande, et c’est une bonne pratique en Python, de réattribuer l’attribut `tags` plutôt que de le modifier sur place.

Par exemple, l’extrait de code suivant montre des façons courantes de mettre à jour une liste par réattribution. Par souci de brièveté, nous reprenons l’exemple de code de la [section Ajouter un tag à une collection](#add-a-tag-to-a-collection) :

L’extrait de code suivant montre comment utiliser une mutation en place pour mettre à jour les tags d’une version d’artifact :

```python
collection.tags += ["new-tag", "other-tag"]
collection.tags.append("new-tag")

collection.tags.extend(["new-tag", "other-tag"])
collection.tags[:] = ["new-tag", "other-tag"]
collection.tags.remove("existing-tag")
collection.tags.pop()
collection.tags.clear()
```

<div id="remove-a-tag-from-a-collection">
  ## Supprimer un tag d’une collection
</div>

Utilisez l’interface de l’application W&amp;B pour supprimer un tag d’une collection :

1. Accédez au [W&amp;B Registry](https://wandb.ai/registry).
2. Cliquez sur une carte du registre.
3. Cliquez sur **Voir les détails** à côté du nom d’une collection.
4. Dans la carte de la collection, survolez le nom du tag que vous souhaitez supprimer.
5. Cliquez sur le bouton de fermeture (icône **X**).

    ```python title="Ajouter un tag à une version d'artifact sans créer de nouveau run"
    import wandb

    ARTIFACT_TYPE = "<TYPE>"
    REGISTRY_NAME = "<registry_name>"
    COLLECTION_NAME = "<collection_name>"
    VERSION = "<artifact_version>"

    artifact_name = f"wandb-registry-{REGISTRY_NAME}/{COLLECTION_NAME}:v{VERSION}"

    artifact = wandb.Api().artifact(name = artifact_name, type = artifact_type)
    print(artifact.tags)
    ```

    L&#39;extrait de code suivant montre comment récupérer et afficher les tags d&#39;une version d&#39;artifact en créant un nouveau run :

    ```python title="Ajouter un tag à une version d'artifact pendant un run"
    import wandb

    REGISTRY_NAME = "<registry_name>"
    COLLECTION_NAME = "<collection_name>"
    VERSION = "<artifact_version>"

    with wandb.init(entity = "<entity>", project="<project>") as run:

        artifact_name = f"wandb-registry-{REGISTRY_NAME}/{COLLECTION_NAME}:v{VERSION}"

        artifact = run.use_artifact(artifact_or_name = artifact_name)
        print(artifact.tags)
    ```

<div id="find-artifact-versions-with-a-specific-tag">
  ## Trouver des versions d&#39;artifact avec un tag spécifique
</div>

Utilisez le SDK Python de W&amp;B pour trouver des versions d&#39;artifact associées à un ensemble de tags :

```python
import wandb

api = wandb.Api()
tagged_artifact_versions = api.artifacts(
    type_name = "<artifact_type>",
    name = "<artifact_name>",
    tags = ["<tag_1>", "<tag_2>"]
)

for artifact_version in tagged_artifact_versions:
    print(artifact_version.tags)
```
