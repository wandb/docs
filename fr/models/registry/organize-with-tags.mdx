title: Organiser les versions avec des tags
---

Créez et ajoutez des tags pour organiser vos collections ou les versions d&#39;artefact dans votre registre. Ajoutez, modifiez, consultez ou supprimez des tags sur une collection  ou une version d&#39;artefact avec l&#39;interface de l&#39;application W&amp;B ou le SDK Python W&amp;B.

<Note>
  **Quand utiliser un tag plutôt qu&#39;un alias**

  Utilisez des alias lorsque vous devez référencer de façon unique une version d&#39;artefact précise. Par exemple, utilisez un alias comme &#39;production&#39; ou &#39;latest&#39; pour garantir que `artifact_name:alias` pointe toujours vers une seule version précise.

  Utilisez des tags lorsque vous avez besoin de plus de flexibilité pour le regroupement ou la recherche. Les tags sont idéaux lorsque plusieurs versions ou collections peuvent partager le même libellé, et que vous n&#39;avez pas besoin de garantir qu&#39;une seule version soit associée à un identifiant précis.
</Note>

<div id="add-a-tag-to-a-collection">
  ## Ajouter un tag à une collection
</div>

Utilisez l’interface de l’application W&amp;B ou le SDK Python pour ajouter un tag à une collection :

<Tabs>
  <Tab title="W&B App">
    Utilisez l’interface de l’application W&amp;B pour ajouter un tag à une collection :

    1. Accédez à [W&amp;B Registry](https://wandb.ai/registry).
    2. Cliquez sur une carte du registre.
    3. Cliquez sur **Voir les détails** à côté du nom d’une collection.
    4. Dans la carte de la collection, cliquez sur l’icône plus (**+**) à côté du champ **Tags** et saisissez le nom du tag.
    5. Appuyez sur **Entrée** sur votre clavier.

    <Frame>
      <img src="/images/registry/add_tag_collection.gif" alt="Ajout de tags à une collection du registre" />
    </Frame>
  </Tab>

  <Tab title="SDK Python">
    ```python
    import wandb

    COLLECTION_TYPE = "<collection_type>"
    REGISTRY_NAME = "<registry_name>"
    COLLECTION_NAME = "<collection_name>"

    full_name = f"wandb-registry-{REGISTRY_NAME}/{COLLECTION_NAME}"

    collection = wandb.Api().artifact_collection(
      type_name = COLLECTION_TYPE, 
      name = full_name
      )

    collection.tags = ["your-tag"]
    collection.save()
    ```
  </Tab>
</Tabs>

<div id="update-tags-that-belong-to-a-collection">
  ## Mettre à jour les tags d’une collection
</div>

Mettez à jour un tag par programmation en réattribuant l’attribut `tags` ou en modifiant cet attribut. W&amp;B recommande — et c’est une bonne pratique en Python — de réattribuer l’attribut `tags` plutôt que d’effectuer une mutation en place.

Par exemple, l’extrait de code suivant montre des méthodes courantes pour mettre à jour une liste par réattribution. Par souci de concision, nous reprenons l’exemple de code de la [section Ajouter un tag à une collection](#add-a-tag-to-a-collection) :

```python
collection.tags = [*collection.tags, "new-tag", "other-tag"]
collection.tags = collection.tags + ["new-tag", "other-tag"]

collection.tags = set(collection.tags) - set(tags_to_delete)
collection.tags = []  # supprime tous les tags
```

L&#39;extrait de code suivant montre comment vous pouvez utiliser une mutation en place pour mettre à jour les tags associés à une version d&#39;artefact :

```python
collection.tags += ["new-tag", "other-tag"]
collection.tags.append("new-tag")

collection.tags.extend(["new-tag", "other-tag"])
collection.tags[:] = ["new-tag", "other-tag"]
collection.tags.remove("existing-tag")
collection.tags.pop()
collection.tags.clear()
```

</div>

<div id="add-a-tag-to-an-artifact-version">
  ## Ajouter un tag à une version d’artefact
</div>

Ajoutez un tag à une version d’artefact liée à une collection à l’aide de l’interface de l’application W&amp;B ou du SDK Python.

<Tabs>
  <Tab title="W&B App">
    1. Accédez au W&amp;B Registry à l’adresse https://wandb.ai/registry
    2. Cliquez sur une carte du registre.
    3. Cliquez sur **Voir les détails** à côté du nom de la collection à laquelle vous souhaitez ajouter un tag.
    4. Faites défiler jusqu’à **Versions**.
    5. Cliquez sur **vue** à côté d’une version d’artefact.
    6. Dans l’onglet **Version**, cliquez sur l’icône plus (**+**) à côté du champ **Tags**, puis saisissez le nom du tag.
    7. Appuyez sur **Entrée** sur votre clavier.

    <Frame>
      <img src="/images/registry/add_tag_linked_artifact_version.gif" alt="Ajout de tags à des versions d’artefact" />
    </Frame>
  </Tab>

  <Tab title="Python SDK">
    Récupérez la version d’artefact à laquelle vous souhaitez ajouter ou mettre à jour un tag. Une fois que vous avez récupéré la version d’artefact, vous pouvez accéder à l’attribut `tags` de l’objet artefact pour ajouter ou modifier les tags de cet artefact. Passez un ou plusieurs tags sous forme de liste à l’attribut `tags` de l’artefact.

    Comme pour les autres artefacts, vous pouvez récupérer un artefact depuis W&amp;B sans créer de run, ou créer un run puis récupérer l’artefact dans ce run. Dans les deux cas, veillez à appeler la méthode `save` de l’objet artefact pour mettre à jour l’artefact sur les serveurs W&amp;B.

    Copiez-collez l’une des cellules de code ci-dessous pour ajouter ou modifier le tag d’une version d’artefact. Remplacez les valeurs dans `<>` par les vôtres.

    L’extrait de code suivant montre comment récupérer un artefact et ajouter un tag sans créer de nouveau run :

    ```python title="Ajouter un tag à une version d’artefact sans créer de nouveau run"
    import wandb

    ARTIFACT_TYPE = "<TYPE>"
    REGISTRY_NAME = "<registry_name>"
    COLLECTION_NAME = "<collection_name>"
    VERSION = "<artifact_version>"

    artifact_name = f"wandb-registry-{REGISTRY_NAME}/{COLLECTION_NAME}:v{VERSION}"

    artifact = wandb.Api().artifact(name = artifact_name, type = ARTIFACT_TYPE)
    artifact.tags = ["tag2"] # Fournissez un ou plusieurs tags dans une liste
    artifact.save()
    ```

    L’extrait de code suivant montre comment récupérer un artefact et ajouter un tag en créant un nouveau run :

    ```python title="Ajouter un tag à une version d’artefact pendant un run"
    import wandb

    REGISTRY_NAME = "<registry_name>"
    COLLECTION_NAME = "<collection_name>"
    VERSION = "<artifact_version>"

    with wandb.init(entity = "<entity>", project="<project>") as run:

        artifact_name = f"wandb-registry-{REGISTRY_NAME}/{COLLECTION_NAME}:v{VERSION}"

        artifact = run.use_artifact(artifact_or_name = artifact_name)
        artifact.tags = ["tag2"] # Fournissez un ou plusieurs tags dans une liste
        artifact.save()
    ```
  </Tab>
</Tabs>

<div id="find-artifact-versions-with-a-specific-tag">
  ## Trouver des versions d&#39;artefact avec un tag spécifique
</div>

Utilisez le SDK Python W&amp;B pour trouver les versions d&#39;artefact qui ont un ensemble de tags :

```python
import wandb

api = wandb.Api()
tagged_artifact_versions = api.artifacts(
    type_name = "<artifact_type>",
    name = "<artifact_name>",
    tags = ["<tag_1>", "<tag_2>"]
)

for artifact_version in tagged_artifact_versions:
    print(artifact_version.tags)
```
