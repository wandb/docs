Par défaut, W&amp;B n’enregistre que le hachage du dernier commit Git. Vous pouvez activer d’autres fonctionnalités liées au code pour comparer dynamiquement le code entre vos expériences dans l’interface utilisateur.

À partir de la version 0.8.28 de `wandb`, W&amp;B peut enregistrer le code de votre fichier principal d’entraînement, dans lequel vous appelez `wandb.init()`.

<div id="save-library-code">
  ## Enregistrer du code de bibliothèque
</div>

Lorsque vous activez l’enregistrement du code, W&amp;B enregistre le code du fichier qui a appelé `wandb.init()`. Pour enregistrer du code de bibliothèque supplémentaire, trois options s’offrent à vous :

<div id="call-wandbrunlog_code-after-calling-wandbinit">
  ### Appel `wandb.Run.log_code(".")` après l’appel à `wandb.init()`
</div>

```python
import wandb

with wandb.init() as run:
  run.log_code(".")
```

<div id="pass-a-settings-object-to-wandbinit-with-code_dir-set">
  ### Passez un objet de paramètres à `wandb.init()` en définissant `code_dir`
</div>

```python
import wandb

wandb.init(settings=wandb.Settings(code_dir="."))
```

Cette opération capture tous les fichiers de code source Python du répertoire actuel et de tous ses sous-répertoires sous forme d’[artefact](/fr/models/ref/python/experiments/artifact). Pour mieux contrôler les types de fichiers de code source enregistrés et leurs emplacements, voir la [documentation de référence](/fr/models/ref/python/experiments/run#log_code).

<div id="set-code-saving-in-the-ui">
  ### Configurer l’enregistrement du code dans l’interface utilisateur
</div>

En plus de configurer l’enregistrement du code par programmation, vous pouvez également basculer cette fonctionnalité dans les paramètres de votre compte W&amp;B. Notez que cela activera l’enregistrement du code pour tous les Teams associés à votre compte.

> Par défaut, W&amp;B désactive l’enregistrement du code pour tous les Teams.

1. Connectez-vous à votre compte W&amp;B.
2. Accédez à **Paramètres** &gt; **Confidentialité**.
3. Sous **Sécurité du projet et du contenu**, basculez **Désactiver l’enregistrement du code par défaut** sur activé.

<div id="code-comparer">
  ## Comparateur de code
</div>

Comparez le code utilisé dans différents W&amp;B Runs :

1. Cliquez sur le bouton **Ajouter des panneaux** dans le coin supérieur droit de la page.
2. Ouvrez le menu déroulant **TEXT AND CODE**, puis sélectionnez **Code**.

<div id="jupyter-session-history">
  ## Historique de session Jupyter
</div>

W&amp;B enregistre l’historique du code exécuté dans votre session de notebook Jupyter. Lorsque vous appelez **wandb.init()** dans Jupyter, W&amp;B ajoute un hook qui enregistre automatiquement un notebook Jupyter contenant l’historique du code exécuté dans votre session en cours.

1. Accédez aux espaces de travail du projet contenant votre code.
2. Sélectionnez l’onglet **Artifacts** dans la barre latérale du projet.
3. Développez l’artefact **code**.
4. Sélectionnez l’onglet **Files**.

<Frame>
  <img src="/images/app_ui/jupyter_session_history.gif" alt="Historique de session Jupyter" />
</Frame>

Cela affiche les cellules qui ont été exécutées dans votre session, ainsi que toutes les sorties créées en appelant la méthode display d’IPython. Cela vous permet de voir exactement quel code a été exécuté dans Jupyter dans une run donnée. Lorsque c’est possible, W&amp;B enregistre également la version la plus récente du notebook, que vous trouveriez aussi dans le répertoire code.
