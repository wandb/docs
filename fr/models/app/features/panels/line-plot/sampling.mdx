---
title: Agrégation de points
description: "Comprenez les deux modes d’agrégation de points pour les graphiques en courbes W&B : l’échantillonnage par bucket en mode de fidélité totale et l’échantillonnage aléatoire."
---

Utilisez les méthodes d’agrégation de points dans vos graphiques en courbes pour améliorer la précision de la visualisation des données et les performances. Il existe deux modes d’agrégation de points : [fidélité totale](#full-fidelity) et [échantillonnage aléatoire](#random-sampling). W&amp;B utilise le mode de fidélité totale par défaut.

<div id="full-fidelity">
  ## Fidélité totale
</div>

Lorsque vous utilisez le mode de fidélité totale, W&amp;B divise l’axe des x en buckets dynamiques. Le nombre de points par ligne s’adapte à la taille du graphique et au nombre de runs. Il calcule les valeurs minimale et maximale dans chaque bucket (utilisées pour l’ombrage facultatif) et utilise la dernière valeur de chaque bucket (et non la moyenne) pour tracer la ligne principale.

Le mode de fidélité totale présente trois principaux avantages pour l’agrégation de points :

* Préserver les valeurs extrêmes et les pics : conservez les valeurs extrêmes et les pics dans vos données.
* Configurer le rendu des points minimum et maximum : utilisez W&amp;B App pour choisir de manière interactive si vous souhaitez afficher les valeurs extrêmes (min/max) sous forme de zone ombrée.
* Explorer vos données sans perte de fidélité : W&amp;B recalcule la taille des buckets de l’axe des x lorsque vous zoomez sur des points de données spécifiques. Cela vous permet d’explorer vos données sans perdre en précision. La mise en cache est utilisée pour stocker les agrégations déjà calculées afin de réduire les temps de chargement, ce qui est particulièrement utile si vous parcourez de grands jeux de données.

<div id="turn-on-full-fidelity">
  ### Activer la fidélité totale
</div>

W&amp;B utilise le mode de fidélité totale par défaut. Pour le configurer manuellement, suivez ces étapes :

<Tabs>
  <Tab title="Tous les graphiques d’un Workspace">
    1. Accédez à votre Workspace.
    2. Sélectionnez l’icône d’engrenage dans le coin supérieur droit de l’écran, à gauche du bouton **ajouter des panneaux**.
    3. Dans le volet de l’interface qui s’affiche, sélectionnez **Line plots**
    4. Choisissez **Full fidelity** dans la section **agrégation de points**.
    5. Configurez l’algorithme et les paramètres de **Lissage**.
    6. Définissez **Agrégation** sur **Mean**, **Min** ou **Max**.
    7. Cliquez sur **appliquer**.
  </Tab>

  <Tab title="Graphique individuel dans un Workspace">
    1. Accédez à votre Workspace.
    2. Sélectionnez l’icône **Workspace** dans l’onglet de gauche
    3. Survolez le panneau de graphique linéaire que vous souhaitez configurer, puis cliquez sur l’icône d’engrenage.
    4. Dans la fenêtre modale qui s’affiche, définissez **Méthode d’agrégation des points** sur **Full fidelity**.
    5. Configurez l’algorithme et les paramètres de **Lissage**.
    6. Cliquez sur **appliquer**.
  </Tab>
</Tabs>

<div id="configure-shading">
  ### Configurer l’ombrage
</div>

Les zones ombrées d’un graphique en courbes à fidélité totale peuvent afficher :

* **Min/Max** : pour chaque point de l’axe des x, ombrer la zone comprise entre les valeurs minimale et maximale. La zone ombrée montre tous les points, de la valeur la plus faible à la plus élevée, dans chaque bucket :
  ```math
  \text{Min/Max Range} = [\min(x_1, x_2, \ldots, x_n),\ \max(x_1, x_2, \ldots, x_n)]
  ```
  où $x&#95;1, x&#95;2, \ldots, x&#95;n$ sont les valeurs d’un bucket donné.

* **Écart type** : pour chaque point de l’axe des x, calculer la variabilité des valeurs à l’aide de l’écart type, puis ombrer la zone résultante.
  ```math
  SD = \sqrt{\frac{1}{n}\sum_{i=1}^{n}(x_i - \overline{x})^2}
  ```

* **Erreur standard** : pour chaque point de l’axe des x, calculer la probabilité d’une erreur d’échantillonnage en divisant la valeur par la racine carrée de la taille de l’échantillon :
  ```math
  SE = \frac{SD}{\sqrt{n}}
  ```

* **Aucun** : aucun ombrage (par défaut).

L’image suivante montre un graphique en courbes bleu. La zone ombrée en bleu clair représente les valeurs minimale et maximale pour chaque bucket.

<Frame>
  <img src="/images/app_ui/shaded-areas.png" alt="Zones de confiance ombrées" />
</Frame>

Pour configurer l’ombrage :

1. Accédez à votre Workspace.
2. Survolez un graphique en courbes, puis cliquez sur l’icône d’engrenage.
3. Dans l’onglet **Data**, définissez **agrégation de points** sur **Full fidelity** si nécessaire, puis configurez l’algorithme de lissage.
4. Dans l’onglet **Grouping**, activez **Group runs**. Facultatif : définissez **Group by** sur un attribut de run.
5. Définissez **Agg** sur **Mean** (par défaut), **Min** ou **Max**.
6. Définissez **Range** sur **Min/Max**, **Std Dev**, **Std Err** ou **None**.
7. Cliquez sur **Appliquer**.

<div id="explore-your-data-without-losing-data-fidelity">
  ### Explorez vos données sans perte de fidélité
</div>

Analysez des régions spécifiques du jeu de données sans passer à côté de points critiques comme les valeurs extrêmes ou les pics. Lorsque vous zoomez sur un graphique en courbes, W&amp;B ajuste la taille des buckets utilisés pour calculer la valeur minimale, la valeur maximale et la dernière valeur dans chaque bucket.

<Frame>
  <img src="/images/app_ui/zoom_in.gif" alt="Fonctionnalité de zoom du graphique" />
</Frame>

W&amp;B utilise un binning dynamique pour diviser l’axe des x en buckets. Le nombre de points affichés par ligne s’adapte à la taille du graphique et au nombre de runs : des graphiques plus petits ou un plus grand nombre de runs peuvent réduire le nombre de points par ligne afin que le graphique reste réactif et puisse afficher davantage de lignes. Pour chaque bucket, W&amp;B calcule les valeurs suivantes :

* **Minimum** : La valeur la plus basse dans ce bucket (utilisée pour l’ombrage).
* **Maximum** : La valeur la plus élevée dans ce bucket (utilisée pour l’ombrage).
* **Valeur de la ligne** : La dernière valeur dans ce bucket, utilisée pour tracer la ligne.

W&amp;B trace les valeurs dans les buckets de manière à préserver une représentation complète des données et à inclure les valeurs extrêmes dans chaque graphique. La ligne est tracée à partir de la dernière valeur de chaque bucket. Lorsque vous zoomez suffisamment, le mode de fidélité totale peut afficher chaque point de données sans agrégation supplémentaire ; le seuil exact dépend des dimensions actuelles du graphique et du nombre de runs.

Pour zoomer sur un graphique en courbes, suivez ces étapes :

1. Accédez à votre projet W&amp;B
2. Sélectionnez l’icône **Workspace** dans l’onglet de gauche
3. Ajoutez facultativement un panneau de graphique linéaire à votre espace de travail ou accédez à un panneau de graphique linéaire existant.
4. Cliquez et faites glisser pour sélectionner une région spécifique à agrandir.

<Note>
  **Regroupement des graphiques en courbes et expressions**

  Lorsque vous utilisez le regroupement de graphiques en courbes, W&amp;B applique le comportement suivant selon le mode sélectionné :

  * **Échantillonnage non fenêtré (regroupement)** : Aligne les points entre les runs sur l’axe des x. Une moyenne est calculée si plusieurs points partagent la même valeur x ; sinon, ils apparaissent comme des points distincts.
  * **Échantillonnage fenêtré (regroupement et expressions)** : Divise l’axe des x soit en 250 buckets, soit selon le nombre de points de la ligne la plus longue (la plus petite des deux valeurs étant retenue). W&amp;B calcule une moyenne des points dans chaque bucket.
  * **Fidélité totale (regroupement et expressions)** : Semblable à l’échantillonnage non fenêtré, mais récupère jusqu’à 500 points par run afin d’équilibrer les performances et le niveau de détail.
</Note>

<div id="random-sampling">
  ## Échantillonnage aléatoire
</div>

L’échantillonnage aléatoire utilise 1 500 points sélectionnés aléatoirement pour le rendu des graphiques en courbes. Comme cet échantillonnage élimine certains points, il devient plus difficile de repérer les valeurs aberrantes ou les pics.

<Warning>
  L’échantillonnage aléatoire n’est pas déterministe. Cela signifie qu’il exclut parfois des valeurs aberrantes ou des pics importants dans les données, ce qui réduit la précision des données.
</Warning>

<div id="enable-random-sampling">
  ### Activer l’échantillonnage aléatoire
</div>

Par défaut, W&amp;B utilise le mode de fidélité totale. Pour activer l’échantillonnage aléatoire, suivez ces étapes :

<Tabs>
  <Tab title="Tous les graphiques d’un Workspace">
    1. Accédez à votre projet W&amp;B
    2. Sélectionnez l’icône **Workspace** dans l’onglet de gauche
    3. Sélectionnez l’icône d’engrenage dans le coin supérieur droit de l’écran, à gauche du bouton **Add panels**.
    4. Dans le volet de l’interface utilisateur qui s’affiche, sélectionnez **Line plots**
    5. Choisissez **Random sampling** dans la section **agrégation de points**
  </Tab>

  <Tab title="Graphique individuel dans un Workspace">
    1. Accédez à votre projet W&amp;B
    2. Sélectionnez l’icône **Workspace** dans l’onglet de gauche
    3. Sélectionnez le panneau de graphique linéaire pour lequel vous souhaitez activer l’échantillonnage aléatoire
    4. Dans la fenêtre modale qui s’affiche, sélectionnez **Random sampling** dans la section **méthode d’agrégation des points**
  </Tab>
</Tabs>

<div id="access-non-sampled-data">
  ### Accéder aux données non échantillonnées
</div>

Vous pouvez accéder à l’historique complet des métriques enregistrées au cours d’un run à l’aide de l’[API Run de W&amp;B](/fr/models/ref/python/public-api/runs). L’exemple suivant montre comment récupérer et traiter les valeurs de perte d’un run spécifique :

```python
# Initialiser l'API W&B
run = api.run("l2k2/examples-numpy-boston/i0wt6xua")

# Récupérer l'historique de la métrique 'Loss'
history = run.scan_history(keys=["Loss"])

# Extraire les valeurs de perte de l'historique
losses = [row["Loss"] for row in history]
```
