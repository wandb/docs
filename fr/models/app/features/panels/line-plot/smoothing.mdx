<div id="time-weighted-exponential-moving-average-twema-smoothing-default">
  ## Lissage par moyenne mobile exponentielle pondérée par le temps (TWEMA) (par défaut)
</div>

L’algorithme de lissage Time Weighted Exponential Moving Average (TWEMA) est une technique de lissage des données de séries temporelles qui applique une décroissance exponentielle au poids des points précédents. Pour plus de détails sur cette technique, voir [Exponential Smoothing](https://www.wikiwand.com/en/Exponential_smoothing). La plage de valeurs s’étend de 0 à 1. Un terme de correction du biais est ajouté afin que les premières valeurs de la série temporelle ne soient pas biaisées vers zéro.

L’algorithme TWEMA prend en compte la densité des points sur la ligne (le nombre de valeurs `y` par unité de plage sur l’axe des x). Cela permet un lissage cohérent lors de l’affichage simultané de lignes multiples aux caractéristiques différentes.

Voici un exemple de code montrant comment cela fonctionne en interne :

```javascript
const smoothingWeight = Math.min(Math.sqrt(smoothingParam || 0), 0.999);
let lastY = yValues.length > 0 ? 0 : NaN;
let debiasWeight = 0;

return yValues.map((yPoint, index) => {
  const prevX = index > 0 ? index - 1 : 0;
  // VIEWPORT_SCALE met le résultat à l'échelle de la plage de l'axe des x du graphique
  const changeInX =
    ((xValues[index] - xValues[prevX]) / rangeOfX) * VIEWPORT_SCALE;
  const smoothingWeightAdj = Math.pow(smoothingWeight, changeInX);

  lastY = lastY * smoothingWeightAdj + yPoint;
  debiasWeight = debiasWeight * smoothingWeightAdj + 1;
  return lastY / debiasWeight;
});
```

<div id="gaussian-smoothing">
  ## Lissage gaussien
</div>

Le lissage gaussien (ou lissage par noyau gaussien) calcule une moyenne pondérée des points, où les poids correspondent à une distribution gaussienne dont l’écart-type est défini par le paramètre de lissage. La valeur lissée est calculée pour chaque valeur x en entrée, basée sur les points situés avant et après celle-ci.

<div id="running-average-smoothing">
  ## Lissage par moyenne glissante
</div>

La moyenne glissante est un algorithme de lissage qui remplace un point par la moyenne des points situés dans une fenêtre avant et après la valeur x considérée. Voir [&quot;Boxcar Filter&quot; sur Wikipedia](https://en.wikipedia.org/wiki/Moving_average). Le paramètre sélectionné pour la moyenne glissante indique à Weights and Biases le nombre de points à prendre en compte dans cette moyenne glissante.

Utilisez plutôt le lissage gaussien si vos points sont espacés de façon irrégulière sur l&#39;axe des x.

<div id="exponential-moving-average-ema-smoothing">
  ## Lissage par moyenne mobile exponentielle (EMA)
</div>

L’algorithme de lissage par moyenne mobile exponentielle (EMA) est une technique empirique permettant de lisser des données de séries temporelles à l’aide d’une fonction de fenêtre exponentielle. Pour plus de détails sur cette technique, voir [Exponential Smoothing](https://www.wikiwand.com/en/Exponential_smoothing). La plage de valeurs va de 0 à 1. Un terme de correction du biais est ajouté afin que les premières valeurs de la série temporelle ne soient pas biaisées vers zéro.

Dans de nombreuses situations, le lissage EMA est appliqué sur un parcours complet de l’historique, plutôt qu’après un regroupement en buckets avant le lissage. Cela produit souvent un lissage plus précis.

Dans les situations suivantes, le lissage EMA est appliqué après le regroupement en buckets à la place :

* Échantillonnage
* Regroupement
* Expressions
* Axes x non monotones
* Axes x basés sur le temps

Voici un exemple de code montrant comment cela fonctionne en interne :

```javascript
  data.forEach(d => {
    const nextVal = d;
    last = last * smoothingWeight + (1 - smoothingWeight) * nextVal;
    numAccum++;
    debiasWeight = 1.0 - Math.pow(smoothingWeight, numAccum);
    smoothedData.push(last / debiasWeight);
```
