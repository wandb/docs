Les [embeddings](https://developers.google.com/machine-learning/crash-course/embeddings/video-lecture) servent à représenter des objets (personnes, images, publications, mots, etc.) à l&#39;aide d&#39;une liste de nombres, parfois appelée *vecteur*. Dans les cas d&#39;usage en machine learning et en science des données, les embeddings peuvent être générés de différentes façons pour un large éventail d&#39;applications. Cette page suppose que le lecteur connaît déjà les embeddings et souhaite les analyser visuellement dans W&amp;B.

<div id="hello-world">
  ### Hello World
</div>

W&amp;B vous permet d’effectuer la journalisation des embeddings à l’aide de la classe `wandb.Table`. Prenons l’exemple suivant de 3 embeddings, chacun comportant 5 dimensions :

```python
import wandb

with wandb.init(project="embedding_tutorial") as run:
  embeddings = [
      # D1   D2   D3   D4   D5
      [0.2, 0.4, 0.1, 0.7, 0.5],  # embedding 1
      [0.3, 0.1, 0.9, 0.2, 0.7],  # embedding 2
      [0.4, 0.5, 0.2, 0.2, 0.1],  # embedding 3
  ]
  run.log(
      {"embeddings": wandb.Table(columns=["D1", "D2", "D3", "D4", "D5"], data=embeddings)}
  )
  run.finish()
```

Après avoir exécuté le code ci-dessus, le tableau de bord W&amp;B affichera un nouveau tableau contenant vos données. Vous pouvez sélectionner `2D Projection` dans le sélecteur de panneau en haut à droite pour faire un graphique des embeddings en 2 dimensions. L’option valeur par défaut intelligente sera automatiquement sélectionnée, mais vous pouvez facilement la remplacer dans le menu de configuration accessible en cliquant sur l’icône d’engrenage. Dans cet exemple, nous utilisons automatiquement les 5 dimensions numériques disponibles.

<div id="digits-mnist">
  ### Chiffres MNIST
</div>

Bien que l’exemple ci-dessus montre les mécanismes de base de la journalisation des embeddings, en pratique, vous travaillez généralement avec bien plus de dimensions et d’échantillons. Prenons le jeu de données MNIST Digits ([jeu de données UCI ML de chiffres manuscrits](https://archive.ics.uci.edu/ml/datasets/Optical+Recognition+of+Handwritten+Digits)[s](https://archive.ics.uci.edu/ml/datasets/Optical+Recognition+of+Handwritten+Digits)), mis à disposition via [SciKit-Learn](https://scikit-learn.org/stable/modules/generated/sklearn.datasets.load_digits.html). Ce jeu de données contient 1797 enregistrements, chacun comportant 64 dimensions. Il s’agit d’un cas d’utilisation de classification à 10 classes. Vous pouvez également convertir les données d’entrée en image pour la visualisation.

```python
import wandb
from sklearn.datasets import load_digits

with wandb.init(project="embedding_tutorial") as run:

  # Charger le jeu de données
  ds = load_digits(as_frame=True)
  df = ds.data

  # Créer une colonne "target"
  df["target"] = ds.target.astype(str)
  cols = df.columns.tolist()
  df = df[cols[-1:] + cols[:-1]]

  # Créer une colonne "image"
  df["image"] = df.apply(
      lambda row: wandb.Image(row[1:].values.reshape(8, 8) / 16.0), axis=1
  )
  cols = df.columns.tolist()
  df = df[cols[-1:] + cols[:-1]]

  run.log({"digits": df})
```

Après avoir exécuté le code ci-dessus, un tableau s’affiche de nouveau dans l’interface. En sélectionnant `2D Projection`, vous pouvez configurer la définition de l’embedding, la couleur, l’algorithme (PCA, UMAP, t-SNE), les paramètres de l’algorithme, et même la superposition (dans ce cas, l’image s’affiche au survol d’un point). Dans ce cas précis, il s’agit uniquement de &quot;valeurs par défaut intelligentes&quot;, et vous devriez voir quelque chose de très similaire d’un simple clic sur `2D Projection`. ([Interagissez avec cet exemple interactif du tutoriel sur les embeddings](https://wandb.ai/timssweeney/embedding_tutorial/runs/k6guxhum?workspace=user-timssweeney)).

<div id="logging-options">
  ## Options de journalisation
</div>

Vous pouvez effectuer la journalisation des embeddings dans différents formats :

1. **Colonne d&#39;embedding unique :** Souvent, vos données sont déjà dans un format de type « matrice ». Dans ce cas, vous pouvez créer une seule colonne d&#39;embedding, dans laquelle le type de données des valeurs de cellule peut être `list[int]`, `list[float]` ou `np.ndarray`.
2. **Plusieurs colonnes numériques :** Dans les deux exemples ci-dessus, nous utilisons cette approche et créons une colonne pour chaque dimension. Actuellement, les cellules acceptent des valeurs Python de type `int` ou `float`.

<Frame>
  <img src="/images/weave/logging_options.png" alt="Colonne d'embedding unique" />
</Frame>

<Frame>
  <img src="/images/weave/logging_option_image_right.png" alt="Plusieurs colonnes numériques" />
</Frame>

En outre, comme pour tous les tableaux, vous disposez de nombreuses options pour construire le tableau :

1. Directement à partir d&#39;un **dataframe** avec `wandb.Table(dataframe=df)`
2. Directement à partir d&#39;une **liste de données** avec `wandb.Table(data=[...], columns=[...])`
3. Construisez le tableau **de manière incrémentielle, ligne par ligne** (idéal si vous avez une boucle dans votre code). Ajoutez des lignes à votre tableau avec `table.add_data(...)`
4. Ajoutez une **colonne d&#39;embedding** à votre tableau (idéal si vous avez une liste de prédictions sous forme d&#39;embeddings) : `table.add_col("col_name", ...)`
5. Ajoutez une **colonne calculée** (idéal si vous avez une fonction ou un modèle à appliquer à votre tableau) : `table.add_computed_columns(lambda row, ndx: {"embedding": model.predict(row)})`
