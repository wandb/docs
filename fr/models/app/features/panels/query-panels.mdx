<div id="create-a-query-panel">
  ## Créer un panneau de requête
</div>

Ajoutez une requête à votre espace de travail ou à un rapport.

<Tabs>
  <Tab title="Espace de travail du projet">
    1. Accédez à l’espace de travail de votre projet.
    2. Dans le coin supérieur droit, cliquez sur `Add panel`.
    3. Dans le menu déroulant, sélectionnez `Query panel`.
  </Tab>

  <Tab title="Rapport W&B">
    Saisissez, puis sélectionnez `/Query panel`.

    <Frame>
      <img src="/images/weave/add_weave_panel_report_1.png" alt="Option « panneau de requête »" />
    </Frame>

    Vous pouvez également associer une requête à un ensemble de runs :

    1. Dans votre rapport, saisissez, puis sélectionnez `/Panel grid`.
    2. Cliquez sur le bouton `Add panel`.
    3. Dans le menu déroulant, sélectionnez `Query panel`.
  </Tab>
</Tabs>

  ## Composants de requête
</div>

<div id="expressions">
  ### Expressions
</div>

Utilisez des expressions de requête pour interroger les données stockées dans W&amp;B, telles que les runs, les artefacts, les modèles, les tableaux, etc.

<div id="example-query-a-table">
  #### Exemple : interroger un tableau
</div>

Supposons que vous souhaitiez effectuer une requête sur un tableau W&amp;B. Dans votre code d’entraînement, vous ajoutez au journal un tableau nommé `"cifar10_sample_table"` :

```python
import wandb
with wandb.init() as run:
  run.log({"cifar10_sample_table":<MY_TABLE>})
```

Dans le panneau de requête, vous pouvez effectuer une requête sur votre tableau avec :

```python
runs.summary["cifar10_sample_table"]
```

<Frame>
  <img src="/images/weave/basic_weave_expression.png" alt="Expression de requête de tableau" />
</Frame>

Décomposons cela :

* `runs` est une variable automatiquement injectée dans les expressions du panneau de requête lorsque celui-ci se trouve dans un Workspace. Sa « valeur » est la liste des runs visibles dans ce Workspace précis. [Pour en savoir plus sur les différents attributs disponibles dans un run, voir ici](/fr/models/track/public-api-guide/#understanding-the-different-attributes).
* `summary` est un op qui renvoie l&#39;objet Summary d&#39;un Run. Les ops sont *mappées*, ce qui signifie que cet op est appliqué à chaque Run de la liste et produit ainsi une liste d&#39;objets Summary.
* `["cifar10_sample_table"]` est un op Pick (indiqué par des crochets), avec `predictions` comme paramètre. Comme les objets Summary se comportent comme des dictionnaires ou des maps, cette opération sélectionne le champ `predictions` dans chaque objet Summary.

Pour apprendre à écrire vos propres requêtes de manière interactive, consultez la [démo du panneau de requête](https://wandb.ai/luis_team_test/weave_example_queries/reports/Weave-queries---Vmlldzo1NzIxOTY2?accessToken=bvzq5hwooare9zy790yfl3oitutbvno2i6c2s81gk91750m53m2hdclj0jvryhcr).

<div id="concat">
  ### Concat
</div>

L’opération concat permet de concaténer 2 tableaux et d’effectuer une concaténation ou une jointure depuis les Paramètres du panneau.

<div id="join">
  ### Jointure
</div>

Il est également possible d’effectuer une jointure entre des tableaux directement dans la requête. Considérez l’expression de requête suivante :

```python
project("luis_team_test", "weave_example_queries").runs.summary["short_table_0"].table.rows.concat.join(\
project("luis_team_test", "weave_example_queries").runs.summary["short_table_1"].table.rows.concat,\
(row) => row["Label"],(row) => row["Label"], "Table1", "Table2",\
"false", "false")
```

<Frame>
  <img src="/images/weave/weave_join.png" alt="Opération de jointure de tableaux" />
</Frame>

Le tableau de gauche est généré à partir de :

```python
project("luis_team_test", "weave_example_queries").\
runs.summary["short_table_0"].table.rows.concat.join
```

Le tableau de droite est généré à partir de :

```python
project("luis_team_test", "weave_example_queries").\
runs.summary["short_table_1"].table.rows.concat
```

Où :

* `(row) => row["Label"]` sont les sélecteurs de chaque tableau et déterminent la colonne sur laquelle effectuer la jointure
* `"Table1"` et `"Table2"` sont les noms de chaque tableau une fois la jointure effectuée
* `true` et `false` correspondent aux paramètres de jointure interne/externe gauche et droite

<div id="runs-object">
  ## Objet Runs
</div>

Utilisez les panneaux de requête pour accéder à l&#39;objet `runs`. Les objets run stockent les enregistrements de vos expériences. Vous trouverez plus de détails dans [Accéder à l&#39;objet runs](https://wandb.ai/luis_team_test/weave_example_queries/reports/Weave-queries---Vmlldzo1NzIxOTY2?accessToken=bvzq5hwooare9zy790yfl3oitutbvno2i6c2s81gk91750m53m2hdclj0jvryhcr#3.-accessing-runs-object), mais voici un aperçu rapide de ce que contient l&#39;objet `runs` :

* `summary` : Un dictionnaire d&#39;informations qui résume les résultats du run. Il peut s&#39;agir de scalaires comme la précision et la perte, ou de fichiers volumineux. Par défaut, `wandb.Run.log()` définit `summary` sur la valeur finale d&#39;une série temporelle enregistrée. Vous pouvez également définir directement le contenu de `summary`. Considérez `summary` comme les sorties du run.
* `history` : Une liste de dictionnaires destinée à stocker les valeurs qui changent pendant l&#39;entraînement du modèle, comme la perte. La commande `wandb.Run.log()` ajoute des éléments à cet objet.
* `config` : Un dictionnaire contenant les informations de configuration du run, comme les hyperparamètres d&#39;un run d&#39;entraînement ou les méthodes de prétraitement d&#39;un run qui crée un artefact de jeu de données. Considérez cela comme les « entrées » du run

<div id="access-artifacts">
  ## Accéder aux Artifacts
</div>

Les Artifacts sont un concept fondamental de W&amp;B. Il s&#39;agit d&#39;une collection versionnée et nommée de fichiers et de répertoires. Utilisez les Artifacts pour suivre les poids de modèles, les jeux de données et tout autre fichier ou répertoire. Les Artifacts sont stockés dans W&amp;B et peuvent être téléchargés ou utilisés dans d&#39;autres runs. Vous trouverez plus de détails et des exemples dans [Accéder aux artefacts](https://wandb.ai/luis_team_test/weave_example_queries/reports/Weave-queries---Vmlldzo1NzIxOTY2?accessToken=bvzq5hwooare9zy790yfl3oitutbvno2i6c2s81gk91750m53m2hdclj0jvryhcr#4.-accessing-artifacts). Les Artifacts sont généralement accessibles à partir de l&#39;objet `project` :

* `project.artifactVersion()` : renvoie la version spécifique de l&#39;artefact pour un nom et une version donnés dans un projet
* `project.artifact("")` : renvoie l&#39;artefact correspondant à un nom donné dans un projet. Vous pouvez ensuite utiliser `.versions` pour obtenir la liste de toutes les versions de cet artefact
* `project.artifactType()` : renvoie l&#39;`artifactType` correspondant à un nom donné dans un projet. Vous pouvez ensuite utiliser `.artifacts` pour obtenir la liste de tous les artefacts de ce type
* `project.artifactTypes` : renvoie la liste de tous les types d&#39;artefact du projet
