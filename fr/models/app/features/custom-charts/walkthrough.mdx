Utilisez des graphiques personnalisés pour contrôler les données que vous chargez dans un panneau ainsi que leur visualisation.

<div id="1-log-data-to-wb">
  ## 1. Journaliser des données dans W&amp;B
</div>

Tout d&#39;abord, journalisez des données dans votre script. Utilisez [wandb.Run.config](/fr/models/track/config/) pour des points uniques définis au début de l&#39;entraînement, comme des hyperparamètres. Utilisez [wandb.Run.log()](/fr/models/track/log/) pour plusieurs points au fil du temps, et journalisez des tableaux 2D personnalisés avec `wandb.Table()`. Nous recommandons de journaliser jusqu&#39;à 10 000 points de données par clé enregistrée.

```python
with wandb.init() as run: 

  # Journalisation d'un tableau de données personnalisé
  my_custom_data = [[x1, y1, z1], [x2, y2, z2]]
  run.log(
    {"custom_data_table": wandb.Table(data=my_custom_data, columns=["x", "y", "z"])}
  )
```

[Essayez un notebook d’exemple rapide](https://bit.ly/custom-charts-colab) pour enregistrer les tableaux de données, et dans l’étape suivante, configurez des graphiques personnalisés. Voyez à quoi ressemblent les graphiques résultants dans le [rapport en direct](https://app.wandb.ai/demo-team/custom-charts/reports/Custom-Charts--VmlldzoyMTk5MDc).

<div id="2-create-a-query">
  ## 2. Créer une requête
</div>

Une fois que vous avez enregistré les données à visualiser, accédez à la page de votre projet et cliquez sur le bouton **`+`** pour ajouter un nouveau panneau, puis sélectionnez **Graphique personnalisé**. Vous pouvez suivre l’exemple dans l’[espace de travail de démonstration des graphiques personnalisés](https://app.wandb.ai/demo-team/custom-charts).

  <img src="/images/app_ui/create_a_query.png" alt="Graphique personnalisé vide" />
</Frame>

<div id="add-a-query">
  ### Ajouter une requête
</div>

1. Cliquez sur `summary`, puis sélectionnez `historyTable` pour configurer une nouvelle requête qui récupère les données de l’historique du run.
2. Saisissez la clé sous laquelle vous avez enregistré le `wandb.Table()`. Dans l’extrait de code ci-dessus, il s’agissait de `my_custom_table` . Dans le [notebook d’exemple](https://bit.ly/custom-charts-colab), les clés sont `pr_curve` et `roc_curve`.

<div id="set-vega-fields">
  ### Définir les champs Vega
</div>

Maintenant que les résultats de la requête sont chargés dans ces colonnes, elles sont disponibles comme options à sélectionner dans les menus déroulants des champs Vega :

<Frame>
  <img src="/images/app_ui/set_vega_fields.png" alt="Utilisation des colonnes des résultats de la requête pour définir les champs Vega" />
</Frame>

* **axe des x :** runSets&#95;historyTable&#95;r (rappel)
* **axe Y :** runSets&#95;historyTable&#95;p (précision)
* **couleur :** runSets&#95;historyTable&#95;c (étiquette de classe)

<div id="bonus-composite-histograms">
  ## Bonus : histogrammes composites
</div>

Les histogrammes permettent de visualiser des distributions numériques pour mieux comprendre des jeux de données volumineux. Les histogrammes composites affichent plusieurs distributions dans les mêmes intervalles, ce qui permet de comparer deux métriques ou plus entre différents modèles ou entre différentes classes au sein d’un même modèle. Pour un modèle de segmentation sémantique qui détecte des objets dans des scènes de conduite, vous pouvez par exemple comparer l’efficacité d’une optimisation pour la précision par rapport à l’intersection sur union (IoU), ou chercher à savoir dans quelle mesure différents modèles détectent les voitures (zones larges et fréquentes dans les données) par rapport aux panneaux de signalisation (zones beaucoup plus petites et moins fréquentes). Dans le [Colab de démonstration](https://bit.ly/custom-charts-colab), vous pouvez comparer les scores de confiance de deux des dix classes d’êtres vivants.

<Frame>
  <img src="/images/app_ui/composite_histograms.png" alt="Histogramme composite" />
</Frame>

Pour créer votre propre version du panneau d’histogramme composite personnalisé :

1. Créez un nouveau panneau de graphique personnalisé dans votre Workspace ou votre rapport (en ajoutant une visualisation « Graphique personnalisé »). Cliquez sur le bouton « Edit » en haut à droite pour modifier la spécification Vega à partir de n’importe quel type de panneau intégré.
2. Remplacez cette spécification Vega intégrée par mon [code MVP pour un histogramme composite en Vega](https://gist.github.com/staceysv/9bed36a2c0c2a427365991403611ce21). Vous pouvez modifier directement dans cette spécification Vega [à l’aide de la syntaxe Vega](https://vega.github.io/) le titre principal, les titres des axes, le domaine d’entrée et tout autre détail (vous pouvez changer les couleurs ou même ajouter un troisième histogramme :)
3. Modifiez la requête dans le côté droit pour charger les bonnes données à partir de vos journaux wandb. Ajoutez le champ `summaryTable` et définissez la valeur de `tableKey` sur `class_scores` pour récupérer le `wandb.Table` enregistré par votre run. Cela vous permettra de remplir les deux ensembles d’intervalles de l’histogramme (`red_bins` et `blue_bins`) via les menus déroulants avec les colonnes du `wandb.Table` enregistré sous `class_scores`. Dans mon exemple, j’ai choisi les scores de prédiction de la classe `animal` pour les intervalles rouges et `plant` pour les intervalles bleus.
4. Vous pouvez continuer à modifier la spécification Vega et la requête jusqu’à ce que le graphique affiché dans l’aperçu de rendu vous convienne. Une fois terminé, cliquez sur **Save as** en haut et donnez un nom à votre graphique personnalisé pour pouvoir le réutiliser. Cliquez ensuite sur **Apply from panel library** pour finaliser votre graphique.

Voici à quoi ressemblent mes résultats après une expérience très brève : un entraînement sur seulement 1000 exemples pendant une époque génère un modèle très confiant que la plupart des images ne sont pas des plantes, et très incertain sur les images qui pourraient être des animaux.
