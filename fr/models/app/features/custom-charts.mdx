<div id="how-it-works">
  ### Fonctionnement
</div>

1. **Journaliser les données** : Depuis votre script, journalisez les données de [configuration](/fr/models/track/config/) et de résumé.
2. **Personnaliser le graphique** : Récupérez les données journalisées avec une requête [GraphQL](https://graphql.org). Visualisez les résultats de votre requête avec [Vega](https://vega.github.io/vega/), une grammaire de visualisation puissante.
3. **Journaliser le graphique** : Appelez votre préréglage personnalisé depuis votre script avec `wandb.plot_table()`.

Si vous ne voyez pas les données attendues, il se peut que la colonne recherchée ne soit pas journalisée dans les runs sélectionnés. Enregistrez votre graphique, revenez au tableau des runs, puis vérifiez les runs sélectionnés à l’aide de l’icône **œil**.

<div id="builtin-presets">
  ### Préréglages intégrés
</div>

W&amp;B propose plusieurs préréglages de graphiques intégrés que vous pouvez journaliser directement depuis votre script. Il s&#39;agit notamment de graphiques en courbes, de nuages de points, de graphiques à barres, d&#39;histogrammes, de courbes PR et de courbes ROC.

<Tabs>
  <Tab title="Graphique en courbes">
    `wandb.plot.line()`

    Ajoutez au journal un graphique en courbes personnalisé — une liste ordonnée de points connectés (x,y) sur des axes x et y arbitraires.

    ```python
    with wandb.init() as run:
      data = [[x, y] for (x, y) in zip(x_values, y_values)]
      table = wandb.Table(data=data, columns=["x", "y"])
      run.log(
          {
              "my_custom_plot_id": wandb.plot.line(
                  table, "x", "y", title="Custom Y vs X Line Plot"
              )
          }
      )
    ```

    Un graphique en courbes permet de journaliser des courbes sur deux dimensions quelconques. Si vous tracez deux listes de valeurs l’une par rapport à l’autre, le nombre de valeurs dans les listes doit correspondre exactement (par exemple, chaque point doit avoir un x et un y).

  <Tab title="Nuage de points">
    `wandb.plot.scatter()`

    Journalisez un graphique de dispersion personnalisé — une liste de points (x, y) sur une paire arbitraire d’axes x et y.

    ```python
    with wandb.init() as run:
      data = [[x, y] for (x, y) in zip(class_x_prediction_scores, class_y_prediction_scores)]
      table = wandb.Table(data=data, columns=["class_x", "class_y"])
      run.log({"my_custom_id": wandb.plot.scatter(table, "class_x", "class_y")})
    ```

    Vous pouvez utiliser ceci pour consigner des points sur n’importe quelles deux dimensions. Notez que si vous tracez deux listes de valeurs l’une par rapport à l’autre, le nombre de valeurs dans les listes doit correspondre exactement (par exemple, chaque point doit avoir un x et un y).

  <Tab title="Graphique en barres">
    `wandb.plot.bar()`

    Enregistrez un graphique à barres personnalisé — une liste de valeurs libellées représentées sous forme de barres — nativement en quelques lignes :

    ```python
    with wandb.init() as run:
      data = [[label, val] for (label, val) in zip(labels, values)]
      table = wandb.Table(data=data, columns=["label", "value"])
      run.log(
          {
              "my_bar_chart_id": wandb.plot.bar(
                  table, "label", "value", title="Custom Bar Chart"
              )
          }
      )
    ```

    Vous pouvez utiliser ceci pour journaliser des graphiques à barres arbitraires. Notez que le nombre de libellés et de valeurs dans les listes doit correspondre exactement (par exemple, chaque point de données doit comporter les deux).

  <Tab title="Histogramme">
    `wandb.plot.histogram()`

    Enregistrez un histogramme personnalisé — en répartissant une liste de valeurs en classes selon leur nombre/leur fréquence d’occurrence — de manière native en quelques lignes. Supposons que j’aie une liste de scores de confiance de prédiction (`scores`) et que je veuille visualiser leur distribution :

    ```python
    with wandb.init() as run:
      data = [[s] for s in scores]
      table = wandb.Table(data=data, columns=["scores"])
      run.log({"my_histogram": wandb.plot.histogram(table, "scores", title=None)})
    ```

    Vous pouvez utiliser ceci pour journaliser des histogrammes arbitraires. Notez que `data` est une liste de listes, afin de prendre en charge un tableau 2D organisé en lignes et en colonnes.

  <Tab title="Courbe précision-rappel">
    `wandb.plot.pr_curve()`

    Créez une [courbe de précision-rappel](https://scikit-learn.org/stable/modules/generated/sklearn.metrics.precision_recall_curve.html#sklearn.metrics.precision_recall_curve) en une ligne :

    ```python
    with wandb.init() as run:
      plot = wandb.plot.pr_curve(ground_truth, predictions, labels=None, classes_to_plot=None)

      run.log({"pr": plot})
    ```

    Vous pouvez journaliser ces données chaque fois que votre code a accès à :

    * les scores prédits d’un modèle (`predictions`) sur un ensemble d’exemples
    * les libellés de vérité terrain correspondants (`ground_truth`) pour ces exemples
    * (facultatif) une liste des libellés/noms de classes (`labels=["cat", "dog", "bird"...]` si l’indice de libellé 0 signifie chat, 1 = chien, 2 = oiseau, etc.)
    * (facultatif) un sous-ensemble des libellés à visualiser dans le graphique (toujours au format liste)

  <Tab title="Courbe ROC">
    `wandb.plot.roc_curve()`

    Créez une [courbe ROC](https://scikit-learn.org/stable/modules/generated/sklearn.metrics.roc_curve.html#sklearn.metrics.roc_curve) en une seule ligne :

    ```python
    with wandb.init() as run:
      # ground_truth est une liste de vraies étiquettes, predictions est une liste de scores prédits
      ground_truth = [0, 1, 0, 1, 0, 1]
      predictions = [0.1, 0.4, 0.35, 0.8, 0.7, 0.9]

      # Créer le graphique de la courbe ROC
      # labels est une liste facultative de noms de classes, classes_to_plot est un sous-ensemble facultatif de ces étiquettes à visualiser
      plot = wandb.plot.roc_curve(
          ground_truth, predictions, labels=None, classes_to_plot=None
      )

      run.log({"roc": plot})
    ```

    Vous pouvez journaliser ceci chaque fois que votre code a accès à :

    * des scores prédits par un modèle (`predictions`) sur un ensemble d’exemples
    * les étiquettes de référence correspondantes (`ground_truth`) pour ces exemples
    * (facultatif) une liste des étiquettes/noms de classes (`labels=["cat", "dog", "bird"...]` si l’index d’étiquette 0 signifie cat, 1 = dog, 2 = bird, etc.)
    * (facultatif) un sous-ensemble (toujours au format liste) de ces étiquettes à afficher sur le graphique

<div id="custom-presets">
  ### Préréglages personnalisés
</div>

Modifiez un préréglage intégré ou créez-en un nouveau, puis enregistrez le graphique. Utilisez l’ID du graphique pour journaliser des données dans ce préréglage personnalisé directement depuis votre script. [Essayez un notebook d’exemple Google Colab](https://tiny.cc/custom-charts).

```python
# Créer un tableau avec les colonnes à représenter graphiquement
table = wandb.Table(data=data, columns=["step", "height"])

# Mapper les colonnes du tableau aux champs du graphique
fields = {"x": "step", "value": "height"}

# Utiliser le tableau pour remplir le nouveau préréglage de graphique personnalisé
# Pour utiliser votre propre préréglage de graphique enregistré, modifiez vega_spec_name
my_custom_chart = wandb.plot_table(
    vega_spec_name="carey/new_chart",
    data_table=table,
    fields=fields,
)
```

<div id="log-data">
  ## Journaliser des données
</div>

Vous pouvez journaliser les types de données suivants depuis votre script et les utiliser dans un graphique personnalisé :

* **Configuration** : paramètres initiaux de votre expérience (vos variables indépendantes). Cela inclut tous les champs nommés que vous avez journalisés comme clés dans `wandb.Run.config` au début de votre entraînement. Par exemple : `wandb.Run.config.learning_rate = 0.0001`
* **Résumé** : valeurs uniques journalisées pendant l&#39;entraînement (vos résultats ou variables dépendantes). Par exemple, `wandb.Run.log({"val_acc" : 0.8})`. Si vous écrivez dans cette clé plusieurs fois pendant l&#39;entraînement via `wandb.Run.log()`, le résumé prend la valeur finale de cette clé.
* **Historique** : la série temporelle complète du scalaire journalisé est disponible dans la requête via le champ `history`
* **summaryTable** : si vous devez journaliser une liste de plusieurs valeurs, utilisez `wandb.Table()` pour journaliser ces données, puis interrogez-les dans votre panneau personnalisé.
* **historyTable** : si vous devez consulter les données d&#39;historique, interrogez `historyTable` dans votre panneau de graphique personnalisé. Chaque fois que vous appelez `wandb.Table()` ou journalisez un graphique personnalisé, vous créez un nouveau tableau dans l&#39;historique pour cette étape.

<div id="how-to-log-a-custom-table">
  ### Comment journaliser un tableau personnalisé
</div>

Utilisez `wandb.Table()` pour journaliser vos données sous forme de tableau 2D. En général, chaque ligne de ce tableau représente un point de données, et chaque colonne correspond aux champs/dimensions pertinents de chaque point de données que vous souhaitez représenter dans un graphique. Lorsque vous configurez un panneau personnalisé, l’ensemble du tableau est accessible via la clé spécifiée transmise à `wandb.Run.log()` (`custom_data_table` ci-dessous), et les champs individuels sont accessibles via les noms des colonnes (`x`, `y` et `z`). Vous pouvez journaliser des tableaux à plusieurs étapes au cours de votre expérience. La taille maximale de chaque tableau est de 10 000 lignes. [Essayez un exemple dans Google Colab](https://tiny.cc/custom-charts).

```python
with wandb.init() as run:
  # Journalisation d'un tableau personnalisé de données
  my_custom_data = [[x1, y1, z1], [x2, y2, z2]]
  run.log(
      {"custom_data_table": wandb.Table(data=my_custom_data, columns=["x", "y", "z"])}
  )
```

<div id="how-to-edit-vega">
  ### Comment modifier Vega
</div>

Cliquez sur **Edit** en haut du panneau pour passer en mode d’édition [Vega](https://vega.github.io/vega/). Vous pouvez y définir une [spécification Vega](https://vega.github.io/vega/docs/specification/) qui crée un graphique interactif dans l’interface utilisateur. Vous pouvez modifier n’importe quel aspect du graphique. Par exemple, vous pouvez changer le titre, choisir un autre jeu de couleurs, afficher les courbes sous forme d’une série de points plutôt que de lignes reliées. Vous pouvez également modifier les données elles-mêmes, par exemple en utilisant une transformation Vega pour regrouper un tableau de valeurs dans un histogramme. L’aperçu du panneau se met à jour de manière interactive, afin que vous puissiez voir l’effet de vos modifications à mesure que vous modifiez la spécification Vega ou la requête. Référez-vous à la [documentation et aux tutoriels Vega ](https://vega.github.io/vega/).

**Références de champ**

Pour importer des données de W&amp;B dans votre graphique, ajoutez des chaînes de modèle de la forme `"${field:<field-name>}"` n’importe où dans votre spécification Vega. Cela crée un menu déroulant dans la zone **Chart Fields** à droite, que les utilisateurs peuvent utiliser pour sélectionner une colonne de résultats de requête à faire correspondre dans Vega.

Pour définir une valeur par défaut pour un champ, utilisez cette syntaxe : `"${field:<field-name>:<placeholder text>}"`

<div id="saving-chart-presets">
  ### Enregistrer des préréglages de graphiques
</div>

Appliquez les modifications à un panneau de visualisation spécifique à l’aide du bouton situé en bas de la fenêtre modale. Vous pouvez également enregistrer la spécification Vega pour l’utiliser ailleurs dans votre projet. Pour enregistrer cette définition de graphique réutilisable, cliquez sur **Enregistrer sous** en haut de l’éditeur Vega, puis donnez un nom à votre préréglage.
