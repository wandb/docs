Comparez des runs issus de deux projets différents à l’aide de rapports interprojets. Utilisez le sélecteur de projet dans le tableau des ensembles de runs pour choisir un projet.

<Frame>
  <img src="/images/reports/howto_pick_a_different_project_to_draw_runs_from.gif" alt="Comparer des runs entre différents projets" />
</Frame>

Les visualisations de la section utilisent les colonnes du premier ensemble de runs actif. Assurez-vous que le premier ensemble de runs coché dans la section contient bien cette colonne si vous ne voyez pas la métrique recherchée dans le graphique en courbes.

Cette fonctionnalité prend en charge les données d&#39;historique pour les séries temporelles, mais elle ne permet pas d&#39;extraire différentes métriques de synthèse depuis différents projets. En d&#39;autres termes, vous ne pouvez pas créer un nuage de points à partir de colonnes enregistrées uniquement dans un autre projet.

Si vous devez comparer des runs de deux projets et que les colonnes ne fonctionnent pas, ajoutez un tag aux runs d&#39;un projet, puis déplacez ces runs vers l&#39;autre projet. Vous pouvez toujours filtrer uniquement les runs de chaque projet, mais le rapport inclut toutes les colonnes pour les deux ensembles de runs.

<div id="view-only-report-links">
  ## Liens de rapport en lecture seule
</div>

Partagez un lien en lecture seule vers un rapport dans un projet privé ou un projet d’équipe.

<Frame>
  <img src="/images/reports/magic-links.gif" alt="Liens de rapport en lecture seule" />
</Frame>

Les liens de rapport en lecture seule ajoutent un jeton d’accès secret à l’URL. Ainsi, toute personne qui ouvre le lien peut consulter la page. N’importe qui peut utiliser le lien magique pour voir le rapport sans se connecter au préalable. Pour les clients qui utilisent des installations en cloud privé [W&amp;B Local](/fr/platform/hosting/), ces liens restent derrière votre pare-feu. Seuls les membres de votre équipe disposant d’un accès à votre instance privée *et* au lien en lecture seule peuvent donc consulter le rapport.

En **mode lecture seule**, une personne non connectée peut voir les graphiques, les survoler pour afficher des infobulles avec les valeurs, zoomer et dézoomer sur les graphiques, et faire défiler les colonnes du tableau. Dans ce mode, elle ne peut pas créer de nouveaux graphiques ni de nouvelles requêtes de tableau pour explorer les données. Les visiteurs qui accèdent au rapport via un lien en lecture seule ne peuvent pas cliquer sur un run pour accéder à sa page. De plus, ils ne voient pas la fenêtre modale de partage, mais une infobulle au survol indiquant : `Sharing not available for view only access`.

<Info>
  Les liens magiques sont uniquement disponibles pour les projets « Private » et « Team ». Pour les projets « Public » (tout le monde peut voir) ou « Open » (tout le monde peut voir et contribuer des runs), il n’est pas possible d’activer ou de désactiver ces liens, car le projet est public et donc déjà disponible pour toute personne disposant du lien.
</Info>

<div id="send-a-graph-to-a-report">
  ## Envoyer un graphique vers un rapport
</div>

Envoyez un graphique de votre espace de travail vers un rapport afin de suivre votre progression. Cliquez sur le menu déroulant du graphique ou du panneau que vous souhaitez copier dans un rapport, puis sur **Ajouter au rapport** pour sélectionner le rapport de destination.
