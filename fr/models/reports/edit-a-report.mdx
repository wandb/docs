</Note>

Modifiez un rapport de manière interactive avec l&#39;interface de l&#39;application ou par programmation avec le SDK W&amp;B.

Les Reports se composent de *blocs*. Les blocs constituent le corps d&#39;un rapport. Dans ces blocs, vous pouvez ajouter du texte, des images, des visualisations intégrées, des graphiques issus d&#39;Experiments et de runs, ainsi que des grilles de panneaux.

Les *grilles de panneaux* sont un type spécifique de bloc qui contient des panneaux et des *ensembles de runs*. Les ensembles de runs sont des collections de runs enregistrés dans un projet W&amp;B. Les panneaux sont des visualisations des données d&#39;un ensemble de runs.

<Note>
  Consultez le [notebook sur les espaces de travail programmatiques](https://colab.research.google.com/github/wandb/wandb-workspaces/blob/Update-wandb-workspaces-tuturial/Workspace_tutorial.ipynb) pour obtenir un exemple étape par étape montrant comment créer et personnaliser une vue enregistrée d&#39;un espace de travail.
</Note>

<Note>
  Vérifiez que l&#39;API Report and Workspace W&amp;B `wandb-workspaces` est installée en plus du SDK Python W&amp;B si vous souhaitez modifier un rapport par programmation :

  ```pip
  pip install wandb wandb-workspaces
  ```
</Note>

<div id="add-plots">
  ## Ajouter des graphiques
</div>

Chaque grille de panneaux comporte un ensemble de run sets et un ensemble de panneaux. Les run sets au bas de la section contrôlent quelles données s’affichent sur les panneaux de la grille. Créez une nouvelle grille de panneaux si vous souhaitez ajouter des graphiques qui récupèrent des données à partir d’un ensemble différent de runs.

<Tabs>
  <Tab title="W&B App">
    Saisissez une barre oblique (`/`) dans le rapport pour afficher un menu déroulant. Sélectionnez **Ajouter un panneau** pour ajouter un panneau. Vous pouvez ajouter n’importe quel panneau pris en charge par W&amp;B, y compris un graphique en courbes, un nuage de points ou un graphique en coordonnées parallèles.

    <Frame>
      <img src="/images/reports/demo_report_add_panel_grid.gif" alt="Ajouter des graphiques à un rapport" />
    </Frame>
  </Tab>

  <Tab title="API Report and Workspace">
    Ajoutez des graphiques à un rapport de manière programmatique avec le SDK. Transmettez une liste d’un ou plusieurs objets de type graphique ou graphique au paramètre `panels` dans la classe `PanelGrid` de l’API publique. Créez un objet graphique avec sa classe Python associée.

    Les exemples suivants montrent comment créer un graphique en courbes et un nuage de points.

    ```python
    import wandb
    import wandb_workspaces.reports.v2 as wr

    report = wr.Report(
        project="report-editing",
        title="An amazing title",
        description="A descriptive description.",
    )

    blocks = [
        wr.PanelGrid(
            panels=[
                wr.LinePlot(x="time", y="velocity"),
                wr.ScatterPlot(x="time", y="acceleration"),
            ]
        )
    ]

    report.blocks = blocks
    report.save()
    ```

    Pour plus d’informations sur les graphiques disponibles que vous pouvez ajouter à un rapport de manière programmatique, voir `wr.panels`.
  </Tab>
</Tabs>

<div id="add-run-sets">
  ## Ajouter des ensembles de runs
</div>

Ajoutez des ensembles de runs à partir de projets, de manière interactive, avec l&#39;interface de l&#39;application ou le SDK W&amp;B.

<Tabs>
  <Tab title="W&B App">
    Saisissez une barre oblique (`/`) dans le rapport pour afficher un menu déroulant. Dans ce menu, choisissez **Panel Grid**. Cela importera automatiquement l&#39;ensemble de runs du projet à partir duquel le rapport a été créé.

    Si vous importez un panneau dans un rapport, les noms des runs sont hérités du projet. Dans le rapport, vous pouvez, si vous le souhaitez, [renommer un run](/fr/models/runs/#rename-a-run) pour donner davantage de contexte au lecteur. Le run est renommé uniquement dans le panneau concerné. Si vous clonez le panneau dans le même rapport, le run est également renommé dans le panneau cloné.

    1. Dans le rapport, cliquez sur l&#39;icône en forme de crayon pour ouvrir l&#39;éditeur du rapport.

    2. Dans l&#39;ensemble de runs, trouvez le run à renommer. Survolez le nom du rapport, puis cliquez sur les trois points verticaux. Sélectionnez l&#39;une des options suivantes, puis soumettez le formulaire.

       * **Rename run for project** : renomme le run dans tout le projet. Pour générer un nouveau nom aléatoire, laissez le champ vide.
       * **Rename run for panel grid** : renomme le run uniquement dans le rapport, tout en conservant le nom existant dans les autres contextes. La génération d&#39;un nouveau nom aléatoire n&#39;est pas prise en charge.

    3. Cliquez sur **Publish report**.
  </Tab>

  <Tab title="API Report and Workspace">
    Ajoutez des ensembles de runs à partir de projets avec les classes `wr.Runset()` et `wr.PanelGrid`. La procédure suivante explique comment ajouter un runset :

    1. Créez une instance d&#39;objet `wr.Runset()`. Fournissez le nom du projet qui contient les ensembles de runs pour le paramètre project, ainsi que l&#39;entité propriétaire du projet pour le paramètre entity.
    2. Créez une instance d&#39;objet `wr.PanelGrid()`. Passez une liste contenant un ou plusieurs objets runset au paramètre `run sets`.
    3. Stockez une ou plusieurs instances d&#39;objet `wr.PanelGrid()` dans une liste.
    4. Mettez à jour l&#39;attribut blocks de l&#39;instance du rapport avec la liste des instances de grille de panneaux.

    ```python
    import wandb
    import wandb_workspaces.reports.v2 as wr

    report = wr.Report(
        project="report-editing",
        title="An amazing title",
        description="A descriptive description.",
    )

    panel_grids = wr.PanelGrid(
        runsets=[wr.RunSet(project="<project-name>", entity="<entity-name>")]
    )

    report.blocks = [panel_grids]
    report.save()
    ```

    Vous pouvez également ajouter des runsets et des panneaux en un seul appel au SDK :

    ```python
    import wandb

    report = wr.Report(
        project="report-editing",
        title="An amazing title",
        description="A descriptive description.",
    )

    panel_grids = wr.PanelGrid(
        panels=[
            wr.LinePlot(
                title="line title",
                x="x",
                y=["y"],
                range_x=[0, 100],
                range_y=[0, 100],
                log_x=True,
                log_y=True,
                title_x="x axis title",
                title_y="y axis title",
                ignore_outliers=True,
                groupby="hyperparam1",
                groupby_aggfunc="mean",
                groupby_rangefunc="minmax",
                smoothing_factor=0.5,
                smoothing_type="gaussian",
                smoothing_show_original=True,
                max_runs_to_show=10,
                plot_type="stacked-area",
                font_size="large",
                legend_position="west",
            ),
            wr.ScatterPlot(
                title="scatter title",
                x="y",
                y="y",
                # z='x',
                range_x=[0, 0.0005],
                range_y=[0, 0.0005],
                # range_z=[0,1],
                log_x=False,
                log_y=False,
                # log_z=True,
                running_ymin=True,
                running_ymean=True,
                running_ymax=True,
                font_size="small",
                regression=True,
            ),
        ],
        runsets=[wr.RunSet(project="<project-name>", entity="<entity-name>")],
    )


    report.blocks = [panel_grids]
    report.save()
    ```
  </Tab>
</Tabs>

<div id="group-a-run-set-programmatically">
  ## Regrouper un ensemble de runs de manière programmatique
</div>

Regroupez les runs d&#39;un ensemble de runs de manière programmatique avec l&#39;[API Workspace and Reports](/fr/models/ref/wandb_workspaces/reports).

Vous pouvez regrouper les runs d&#39;un ensemble de runs selon les valeurs de configuration, les métadonnées du run ou les métriques de synthèse. Le tableau suivant répertorie les méthodes de regroupement disponibles ainsi que les clés disponibles pour chacune d&#39;elles :

| Méthode de regroupement  | Description                                           | Clés disponibles                                                |
| ------------------------ | ----------------------------------------------------- | --------------------------------------------------------------- |
| Valeurs de configuration | Regrouper les runs selon les valeurs de configuration | Valeurs spécifiées dans le paramètre `config` de `wandb.init()` |
| Métadonnées du run       | Regrouper les runs selon les métadonnées du run       | `État`, `Name`, `JobType`                                       |
| Métriques de synthèse    | Regrouper les runs selon les métriques de synthèse    | Valeurs que vous journalisez dans un run avec `wandb.Run.log()` |

{/* Principales différences entre le regroupement des runs dans un rapport et dans un Workspace :
  1. Chemins sous forme de chaînes de caractères vs objets : Reports utilise des chemins de chaîne de caractères comme `"config.group"` au lieu d'objets typés comme `ws.Config("group")`.
  2. Notation par points : utilisez des points pour accéder aux valeurs imbriquées : `"config.model_type"`.
  3. Champs prédéfinis : certains champs comme `"Name"`, `"Tags"` et `"State"` sont des champs spéciaux de métadonnées de run. */}

<div id="group-runs-by-config-values">
  ### Regrouper les runs par valeurs de configuration
</div>

Regroupez les runs par valeurs de configuration afin de comparer des runs ayant des configurations similaires. Les valeurs de configuration sont des paramètres que vous spécifiez dans la configuration de votre run `(wandb.init(config=))`. Pour regrouper des runs par valeurs de configuration, utilisez la syntaxe `config.<key>`, où `<key>` correspond au nom de la valeur de configuration sur laquelle vous souhaitez effectuer le regroupement.

Par exemple, l&#39;extrait de code suivant initialise d&#39;abord un run avec une valeur de configuration pour `group`, puis regroupe les runs dans un rapport en fonction de la valeur de configuration `group`. Remplacez les valeurs de `<entity>` et `<project>` par les noms de votre entité et de votre projet W&amp;B.

```python
import wandb
import wandb_workspaces.reports.v2 as wr

entity = "<entity>"
project = "<project>"

for group in ["control", "experiment_a", "experiment_b"]:
    for i in range(3):
        with wandb.init(entity=entity, project=project, group=group, config={"group": group, "run": i}, name=f"{group}_run_{i}") as run:
            # Simuler un entraînement
            for step in range(100):
                run.log({
                    "acc": 0.5 + (step / 100) * 0.3 + (i * 0.05),
                    "loss": 1.0 - (step / 100) * 0.5
                })
```

Dans votre script Python ou notebook, vous pouvez ensuite regrouper les runs selon la valeur de `config.group` :

```python
runset = wr.Runset(
  project=project,
  entity=entity,
  groupby=["config.group"]  # Grouper par la valeur de configuration "group"
)
```

Dans la continuité de l’exemple précédent, vous pouvez créer un rapport à partir de l’ensemble de runs regroupé :

```python
report = wr.Report(
  entity=entity,
  project=project,
  title="Grouped Runs Example",
)

report.blocks = [
  wr.PanelGrid(
      runsets=[runset],
          )
      ]

report.save()
```

<div id="group-runs-by-run-metadata">
  ### Regrouper les runs en fonction des métadonnées du run
</div>

Regroupez les runs en fonction du nom d’un run (`Name`), de son état (`State`) ou de son type de job (`JobType`).

En reprenant l’exemple précédent, vous pouvez regrouper vos runs selon leur nom à l’aide de l’extrait de code suivant :

```python
runset = wr.Runset(
  project=project,
  entity=entity,
  groupby=["Name"]  # Regrouper par nom des runs
)
```

<Note>
  Le nom du run est celui que vous indiquez dans le paramètre `wandb.init(name=)`. Si vous ne spécifiez pas de nom, W&amp;B génère un nom aléatoire pour le run.

  Vous pouvez trouver le nom du run sur la page **Aperçu** d’un run dans la W&amp;B App ou de manière programmatique avec `Api.runs().run.name`.
</Note>

<div id="group-runs-by-summary-metrics">
  ### Regrouper les Runs par métriques de synthèse
</div>

Les exemples suivants montrent comment regrouper les Runs par métriques de synthèse. Les métriques de synthèse sont les valeurs que vous consignez pour un run avec `wandb.Run.log()`. Après avoir consigné un run, vous pouvez trouver les noms de vos métriques de synthèse dans la W&amp;B App, dans la section **Summary** de la page **Aperçu** d’un run.

La syntaxe pour regrouper les Runs par métriques de synthèse est `summary.<key>`, où `<key>` est le nom de la métrique de synthèse selon laquelle vous souhaitez regrouper les Runs.

Par exemple, supposons que vous consigniez une métrique de synthèse appelée `acc` :

```python
import wandb
import wandb_workspaces.reports.v2 as wr

entity = "<entity>"
project = "<project>"

for group in ["control", "experiment_a", "experiment_b"]:
    for i in range(3):
        with wandb.init(entity=entity, project=project, group=group, config={"group": group, "run": i}, name=f"{group}_run_{i}") as run:
            # Simuler un entraînement
            for step in range(100):
                run.log({
                    "acc": 0.5 + (step / 100) * 0.3 + (i * 0.05),
                    "loss": 1.0 - (step / 100) * 0.5
                })

```

Vous pouvez ensuite regrouper les runs par la métrique de synthèse `summary.acc` :

```python
runset = wr.Runset(
  project=project,
  entity=entity,
  groupby=["summary.acc"]  # Regrouper par valeurs de synthèse 
)
```

<div id="filter-a-run-set-programmatically">
  ## Filtrer un ensemble de runs par programmation
</div>

Filtrez par programmation des ensembles de runs et ajoutez-les à un rapport avec l’[API Workspace and Reports](/fr/models/ref/wandb_workspaces/reports).

La syntaxe générale d’une expression de filtre est :

```text
Filter('key') operation <value>
```

Où `key` est le nom du filtre, `operation` est un opérateur de comparaison (par ex., `>`, `<`, `==`, `in`, `not in`, `or` et `and`), et `<value>` est la valeur à comparer. `Filter` est un espace réservé pour le type de filtre que vous souhaitez appliquer. Le tableau suivant répertorie les filtres disponibles et leurs descriptions :

| Filtre                 | Description                          | Clés disponibles                                                                        |
| ---------------------- | ------------------------------------ | --------------------------------------------------------------------------------------- |
| `Config('key')`        | Filtrer par valeurs de configuration | Valeurs spécifiées dans le paramètre `config` dans `wandb.init(config=)`.               |
| `SummaryMetric('key')` | Filtrer par métriques de synthèse    | Valeurs que vous consignez dans un run avec `wandb.Run.log()`.                          |
| `Tags('key')`          | Filtrer par tag                      | Valeurs de tag que vous ajoutez à votre run (par programmation ou avec la W&amp;B App). |
| `Metric('key')`        | Filtrer par propriétés du run        | `tags`, `state`, `displayName`, `jobType`                                               |

Une fois vos filtres définis, vous pouvez créer un rapport et transmettre les run sets filtrés à `wr.PanelGrid(runsets=)`. Voir les onglets **API Report and Workspace** sur cette page pour plus d’informations sur la façon d’ajouter divers éléments à un rapport par programmation.

Les exemples suivants montrent comment filtrer des run sets dans un rapport. Remplacez les valeurs entre `<>` par vos propres valeurs.

<div id="config-filters">
  ### Filtres de configuration
</div>

Filtrez un ensemble de runs par une ou plusieurs valeurs de configuration. Les valeurs de configuration sont des paramètres que vous définissez dans la configuration de votre run (`wandb.init(config=)`).

Par exemple, l’extrait de code suivant initialise d’abord un run avec une valeur de configuration pour `learning_rate` et `batch_size`, puis filtre des runs dans un rapport en fonction de la valeur de configuration `learning_rate`.

```python
import wandb

config = {
    "learning_rate": 0.01,
    "batch_size": 32,
}

with wandb.init(project="<project>", entity="<entity>", config=config) as run:
    # Votre code d'entraînement ici
    pass
```

Dans votre script Python ou votre notebook, vous pouvez ensuite filtrer de façon programmatique les runs dont le taux d’apprentissage est supérieur à `0.01`.

```python
import wandb_workspaces.reports.v2 as wr

runset = wr.Runset(
  entity="<entity>",
  project="<project>",
  filters="Config('learning_rate') > 0.01"
)
```

Vous pouvez également filtrer par plusieurs valeurs de configuration avec l’opérateur `and` :

```python
runset = wr.Runset(
  entity="<entity>",
  project="<project>",
  filters="Config('learning_rate') > 0.01 and Config('batch_size') == 32"
)
```

En reprenant l’exemple précédent, vous pouvez créer un rapport avec le runset filtré comme suit :

```python
report = wr.Report(
  entity="<entity>",
  project="<project>",
  title="My Report"
)

report.blocks = [
  wr.PanelGrid(
      runsets=[runset],
      panels=[
          wr.LinePlot(
              x="Step",
              y=["accuracy"],
          )
      ]
  )
]

report.save()
```

<div id="metric-filters">
  ### Filtres de métriques
</div>

Filtrez les ensembles de runs en fonction du tag (`tags`), de l’état (`state`), du nom de l’exécution (`displayName`) ou du type de job (`jobType`) du run.

<Note>
  Les filtres `Metric` utilisent une syntaxe différente. Transmettez une liste de valeurs sous forme de liste.

  ```text
  Metric('key') operation [<value>]
  ```
</Note>

Par exemple, considérez l’extrait de code Python suivant, qui crée trois runs et attribue à chacun un nom :

```python
import wandb

with wandb.init(project="<project>", entity="<entity>") as run:
    for i in range(3):
        run.name = f"run{i+1}"
        # Votre code d'entraînement ici
        pass
```

Lorsque vous créez votre rapport, vous pouvez filtrer les runs selon leur nom d’affichage. Par exemple, pour filtrer les runs nommés `run1`, `run2` et `run3`, vous pouvez utiliser le code suivant :

```python
runset = wr.Runset(
  entity="<entity>",
  project="<project>",
  filters="Metric('displayName') in ['run1', 'run2', 'run3']"
)
```

<Note>
  Vous pouvez trouver le nom du run sur la page **Aperçu** d’un run dans W&amp;B App ou de manière programmatique avec `Api.runs().run.name`.
</Note>

Les exemples suivants montrent comment filtrer un runset en fonction de l’état du run (`finished`, `crashed` ou `running`) :

```python
runset = wr.Runset(
  entity="<entity>",
  project="<project>",
  filters="Metric('state') in ['finished']"
)
```

```python
runset = wr.Runset(
  entity="<entity>",
  project="<project>",
  filters="Metric('state') not in ['crashed']"
)
```

<div id="summarymetric-filters">
  ### Filtres de SummaryMetric
</div>

Les exemples suivants montrent comment filtrer un ensemble de runs en fonction des métriques de synthèse. Les métriques de synthèse sont les valeurs que vous consignez dans un run avec `wandb.Run.log()`. Après avoir consigné un run, vous pouvez trouver les noms de vos métriques de synthèse dans la W&amp;B App, dans la section **Summary** de la page **Aperçu** d’un run.

```python
runset = wr.Runset(
  entity="<entity>",
  project="<project>",
  filters="SummaryMetric('accuracy') > 0.9"
)
```

```python
runset = wr.Runset(

)
```

L&#39;extrait de code suivant montre comment filtrer un ensemble de runs selon ses tags. Les tags sont des valeurs que vous ajoutez à un run (de manière programmatique ou avec la W&amp;B App).

runset = wr.Runset(

<div id="add-code-blocks">
  ## Ajouter des blocs de code
</div>

Ajoutez des blocs de code à votre rapport de manière interactive avec l&#39;interface de l&#39;application ou avec le SDK W&amp;B.

<Tabs>
  <Tab title="interface de l'application">
    Saisissez une barre oblique (`/`) dans le rapport pour afficher un menu déroulant. Dans le menu déroulant, choisissez **Code**.

    Sélectionnez le nom du langage de programmation à droite du bloc de code. Cela ouvrira un menu déroulant. Dans le menu déroulant, sélectionnez la syntaxe de votre langage de programmation. Vous pouvez choisir parmi Javascript, Python, CSS, JSON, HTML, Markdown et YAML.
  </Tab>

  <Tab title="API Report and Workspace">
    Utilisez la classe `wr.CodeBlock` pour créer un bloc de code par programmation. Fournissez le nom du langage ainsi que le code que vous souhaitez afficher pour les paramètres language et code, respectivement.

    Par exemple, l&#39;exemple suivant montre une liste dans un fichier YAML :

    ```python
    import wandb
    import wandb_workspaces.reports.v2 as wr

    report = wr.Report(project="report-editing")

    report.blocks = [
        wr.CodeBlock(
            code=["this:", "- is", "- a", "cool:", "- yaml", "- file"], language="yaml"
        )
    ]

    report.save()
    ```

    Cela produira le rendu d&#39;un bloc de code semblable à :

    ```yaml
    this:
    - is
    - a
    cool:
    - yaml
    - file
    ```

    L&#39;exemple suivant montre un bloc de code Python :

    ```python
    report = wr.Report(project="report-editing")


    report.blocks = [wr.CodeBlock(code=["Hello, World!"], language="python")]

    report.save()
    ```

    Cela produira le rendu d&#39;un bloc de code semblable à :

    ```md
    Hello, World!
    ```
  </Tab>

<div id="visualize-relationships-across-multiple-dimensions">
  ## Visualiser les relations entre plusieurs dimensions
</div>

Pour visualiser efficacement les relations entre plusieurs dimensions, utilisez un gradient de couleur pour représenter l&#39;une des variables. Cela améliore la lisibilité et facilite l&#39;interprétation des tendances.

1. Choisissez une variable à représenter avec un gradient de couleur (par exemple, les scores de pénalité, les taux d&#39;apprentissage, etc.). Cela permet de mieux comprendre comment la pénalité (couleur) interagit avec la récompense/les effets secondaires (axe Y) au fil de l&#39;entraînement (axe des x).
2. Mettez en évidence les principales tendances. Lorsque vous survolez un groupe précis de Runs, celui-ci est mis en évidence dans la visualisation.
