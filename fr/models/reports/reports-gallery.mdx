<div id="notes-add-a-visualization-with-a-quick-summary">
  ## Notes : ajoutez une visualisation avec un bref résumé
</div>

Capturez une observation importante, une idée pour de futurs travaux ou une étape clé franchie dans le développement d’un projet. Tous les Runs de votre rapport renverront vers leurs paramètres, métriques, Logs et code, afin que vous puissiez conserver tout le contexte de votre travail.

Consignez quelques lignes de texte et intégrez des graphiques pertinents pour illustrer votre idée.

Voir le W&B Report [What To Do When Inception-ResNet-V2 Is Too Slow](https://wandb.ai/stacey/estuary/reports/When-Inception-ResNet-V2-is-too-slow--Vmlldzo3MDcxMA) pour un exemple de partage de comparaisons du temps d’entraînement.

  <img src="/images/reports/notes_add_quick_summary.png" alt="Notes de bref résumé" max-width="90%" />
</Frame>

Enregistrez les meilleurs exemples issus d’une base de code complexe pour pouvoir les retrouver facilement et y revenir plus tard. Voir le W&B Report [LIDAR point clouds](https://wandb.ai/stacey/lyft/reports/LIDAR-Point-Clouds-of-Driving-Scenes--Vmlldzo2MzA5Mg) pour un exemple de visualisation de nuages de points LIDAR du jeu de données Lyft avec annotation de boîtes englobantes 3D.

<div id="collaboration-share-findings-with-your-colleagues">
  ## Collaboration : partagez vos conclusions avec vos collègues
</div>

Expliquez comment démarrer un projet, partagez ce que vous avez observé jusqu’à présent et synthétisez les dernières constatations. Vos collègues peuvent faire des suggestions ou discuter de détails à l’aide des commentaires sur n’importe quel panneau ou à la fin du rapport.

Incluez des paramètres dynamiques afin que vos collègues puissent explorer par eux-mêmes, obtenir des informations supplémentaires et mieux planifier leurs prochaines étapes. Dans cet exemple, trois types d’expériences peuvent être visualisés séparément, comparés ou moyennés.

Voir le W&amp;B Report [SafeLife benchmark experiments](https://wandb.ai/stacey/saferlife/reports/SafeLife-Benchmark-Experiments--Vmlldzo0NjE4MzM) pour un exemple de partage des premiers runs et des observations d’un benchmark.

<div id="work-log-track-what-youve-tried-and-plan-next-steps">
  ## Journal de travail : suivez ce que vous avez essayé et planifiez les prochaines étapes
</div>

Consignez vos réflexions sur vos expériences, vos conclusions, les éventuels pièges à éviter et les prochaines étapes au fur et à mesure que vous avancez dans un projet, en gardant tout organisé en un seul endroit. Cela vous permet de « documenter » tous les éléments importants au-delà de vos scripts. Voir le [Who Is Them? Text Disambiguation With Transformers](https://wandb.ai/stacey/winograd/reports/Who-is-Them-Text-Disambiguation-with-Transformers--VmlldzoxMDU1NTc) W&B Report pour un exemple de la manière dont vous pouvez présenter vos conclusions.

<Frame>
  <img src="/images/reports/intro_work_log_1.png" alt="Rapport de désambiguïsation de texte" />
</Frame>

Racontez l’histoire d’un projet, auquel vous et d’autres pourrez vous référer plus tard pour comprendre comment et pourquoi un modèle a été développé. Voir le [The View from the Driver&#39;s Seat](https://wandb.ai/stacey/deep-drive/reports/The-View-from-the-Driver-s-Seat--Vmlldzo1MTg5NQ) W&B Report pour découvrir comment vous pouvez présenter vos conclusions.

<Frame>
  <img src="/images/reports/intro_work_log_2.png" alt="Rapport de projet : vue depuis le siège du conducteur" />
</Frame>

Voir le [Learning Dexterity End-to-End Using W&amp;B Reports](https://bit.ly/wandb-learning-dexterity) pour un exemple de la façon dont les W&amp;B Reports ont été utilisés pour explorer la manière dont l’équipe OpenAI Robotics s’en est servie pour mener des projets d’apprentissage automatique à grande échelle.
