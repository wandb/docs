<div id="notes-add-a-visualization-with-a-quick-summary">
  ## Notes : ajoutez une visualisation avec un bref résumé
</div>

Capturez une observation importante, une idée pour de futurs travaux ou un jalon atteint dans le développement d’un projet. Tous les runs de votre rapport renverront vers leurs paramètres, métriques, journaux et code, afin que vous puissiez conserver tout le contexte de votre travail.

Notez quelques lignes de texte et intégrez des graphiques pertinents pour illustrer votre idée.

Voir le W&B Report [What To Do When Inception-ResNet-V2 Is Too Slow](https://wandb.ai/stacey/estuary/reports/When-Inception-ResNet-V2-is-too-slow--Vmlldzo3MDcxMA) pour un exemple de la manière de partager des comparaisons de temps d’entraînement.

<Frame>
  <img src="/images/reports/notes_add_quick_summary.png" alt="Notes de bref résumé" max-width="90%" />
</Frame>

Enregistrez les meilleurs exemples d’une base de code complexe pour pouvoir les consulter facilement et y revenir plus tard. Voir le W&B Report [LIDAR point clouds](https://wandb.ai/stacey/lyft/reports/LIDAR-Point-Clouds-of-Driving-Scenes--Vmlldzo2MzA5Mg) pour un exemple de visualisation de nuages de points LIDAR du jeu de données Lyft, avec annotation de boîtes englobantes 3D.

<div id="collaboration-share-findings-with-your-colleagues">
  ## Collaboration : partagez vos découvertes avec vos collègues
</div>

Expliquez comment démarrer un projet, partagez ce que vous avez observé jusqu’à présent et synthétisez les dernières conclusions. Vos collègues peuvent faire des suggestions ou discuter de détails à l’aide des commentaires sur n’importe quel panneau ou à la fin du rapport.

Incluez des paramètres dynamiques afin que vos collègues puissent explorer par eux-mêmes, obtenir des informations supplémentaires et mieux planifier leurs prochaines étapes. Dans cet exemple, trois types d’expériences peuvent être visualisés séparément, comparés ou moyennés.

Voir le W&amp;B Report [SafeLife benchmark experiments](https://wandb.ai/stacey/saferlife/reports/SafeLife-Benchmark-Experiments--Vmlldzo0NjE4MzM) pour un exemple de partage des premiers runs et des premières observations d’un benchmark.

<div id="work-log-track-what-youve-tried-and-plan-next-steps">
  ## Journal de travail : suivez ce que vous avez essayé et planifiez les prochaines étapes
</div>

Notez vos réflexions sur les expériences, vos conclusions, les pièges éventuels et les prochaines étapes tout au long d’un projet, en gardant tout organisé au même endroit. Cela vous permet de « documenter » tous les éléments importants au-delà de vos scripts. Voir le W&amp;B Report [Who Is Them? Text Disambiguation With Transformers](https://wandb.ai/stacey/winograd/reports/Who-is-Them-Text-Disambiguation-with-Transformers--VmlldzoxMDU1NTc) pour un exemple de la manière dont vous pouvez présenter vos conclusions.

<Frame>
  <img src="/images/reports/intro_work_log_1.png" alt="Rapport de désambiguïsation de texte" />
</Frame>

Racontez l’histoire d’un projet, afin que vous et d’autres puissiez vous y référer plus tard pour comprendre comment et pourquoi un modèle a été développé. Voir le W&amp;B Report [The View from the Driver&#39;s Seat](https://wandb.ai/stacey/deep-drive/reports/The-View-from-the-Driver-s-Seat--Vmlldzo1MTg5NQ) pour voir comment vous pouvez présenter vos conclusions.

<Frame>
  <img src="/images/reports/intro_work_log_2.png" alt="Rapport de projet vu depuis le siège du conducteur" />
</Frame>

Voir [Learning Dexterity End-to-End Using W&amp;B Reports](https://bit.ly/wandb-learning-dexterity) pour un exemple de la manière dont les W&amp;B Reports ont été utilisées pour explorer comment l’équipe OpenAI Robotics s’en est servie pour mener des projets de machine learning à grande échelle.
