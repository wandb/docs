<div id="initialize-a-run-and-track-hyperparameters">
  ## Initialiser un run et suivre les hyperparamètres
</div>

Dans votre script Python ou notebook, initialisez un objet run W&amp;B avec [`wandb.init()`](/fr/models/ref/python/experiments/run/). Utilisez un dictionnaire pour le paramètre `config`
afin de spécifier les noms et les valeurs des hyperparamètres. Dans le bloc `with`, vous pouvez consigner des métriques et d’autres informations dans W&amp;B.

```python
import wandb

wandb.login()

# Projet dans lequel le run est enregistré
project = "my-awesome-project"

# Dictionnaire des hyperparamètres
config = {
    'epochs' : 10,
    'lr' : 0.01
}

with wandb.init(project=project, config=config) as run:
    # Code d'entraînement ici
    # Enregistrer les valeurs dans W&B avec run.log()
    run.log({"accuracy": 0.9, "loss": 0.1})
```

Voir la section suivante pour un exemple complet qui simule un run d&#39;entraînement et enregistre les métriques de précision et de perte dans W&amp;B.

<Info>
  Un [run](/fr/models/runs/) est un élément fondamental de W&amp;B. Vous utilisez les runs pour [suivre les métriques](/fr/models/track/), [créer des journaux](/fr/models/track/log/), suivre les artefacts et bien plus encore.
</Info>

<div id="create-a-machine-learning-training-experiment">
  ## Créer une expérience d’entraînement en machine learning
</div>

Ce script d’entraînement fictif consigne des métriques simulées de précision et de perte dans W&amp;B. Copiez-collez le code suivant dans un script Python ou une cellule de notebook, puis exécutez-le :

```python
import wandb
import random

wandb.login()

# Projet dans lequel le run est enregistré
project = "my-awesome-project"

# dictionnaire des hyperparamètres
config = {
    'epochs' : 10,
    'lr' : 0.01
}

with wandb.init(project=project, config=config) as run:
    offset = random.random() / 5
    print(f"lr: {config['lr']}")
    
    # Simuler un run d'entraînement
    for epoch in range(2, config['epochs']):
        acc = 1 - 2**-config['epochs'] - random.random() / config['epochs'] - offset
        loss = 2**-config['epochs'] + random.random() / config['epochs'] + offset
        print(f"epoch={config['epochs']}, accuracy={acc}, loss={loss}")
        run.log({"accuracy": acc, "loss": loss})
```

Visitez [wandb.ai/home](https://wandb.ai/home) pour afficher les métriques enregistrées, comme la précision et la perte, et voir comment elles évoluent à chaque étape de l&#39;entraînement. L&#39;image suivante montre la perte et la précision suivies pour chaque run. Chaque objet run apparaît dans la colonne **Runs**, avec un nom généré automatiquement.

  ## Prochaines étapes

Découvrez d’autres fonctionnalités de l’écosystème W&amp;B :

1. Lisez les [tutoriels d’intégration W&amp;B](/fr/models/integrations), qui combinent W&amp;B avec des frameworks comme PyTorch, des bibliothèques comme Hugging Face et des services comme SageMaker.

4. Automatisez la recherche d’hyperparamètres et optimisez les modèles avec [W&amp;B Sweeps](/fr/models/sweeps).
5. Analysez les runs, visualisez les prédictions des modèles et partagez vos analyses sur un [tableau de bord central](/fr/models/tables).

7. Visitez [weave-docs.wandb.ai](/fr/weave) pour apprendre à suivre, expérimenter avec, évaluer, déployer et améliorer vos applications basées sur les LLM à l’aide de Weave.
