Bibliothèque Python pour utiliser l’API Workspace de W&amp;B par programmation.

```python
# Comment importer
import wandb_workspaces.workspaces as ws
import wandb_workspaces.reports.v2 as wr

# Exemple de création d'un Workspace
ws.Workspace(
     name="Example W&B Workspace",
     entity="entity", # entité propriétaire du Workspace
     project="project", # projet associé au Workspace
     sections=[
         ws.Section(
             name="Validation Metrics",
             panels=[
                 wr.LinePlot(x="Step", y=["val_loss"]),
                 wr.BarPlot(metrics=["val_accuracy"]),
                 wr.ScalarChart(metric="f1_score", groupby_aggfunc="mean"),
             ],
             is_open=True,
         ),
     ],
)
workspace.save()
```


## <kbd>class</kbd> `RunSettings`

Paramètres d’un run dans un runset (barre de gauche).

**Attributs :**

* `color` (str):  La couleur du run dans l’interface utilisateur. Peut être une valeur hexadécimale (#ff0000), une couleur CSS (red) ou une valeur RVB (rgb(255, 0, 0))
* `disabled` (bool):  Indique si le run est désactivé (œil fermé dans l’interface utilisateur). La valeur par défaut est `False`.

***

## <kbd>class</kbd> `RunsetSettings`

Paramètres du runset (la barre latérale gauche contenant les Runs) dans un Workspace.

**Attributs :**

* `query` (str):  Une requête pour filtrer le runset (peut être une expression regex, voir le paramètre suivant).
* `regex_query` (bool):  Indique si la requête (ci-dessus) est une expression regex. La valeur par défaut est `False`.
* `filters` (Union[str, LList[expr.FilterExpr]]):  Une liste de filtres à appliquer au runset, ou une expression sous forme de chaîne.
  * Sous forme de liste : les filtres sont combinés avec AND. Voir FilterExpr pour plus d&#39;informations sur la création de filtres.
  * Sous forme de chaîne : utilisez des expressions de type Python, par ex., &quot;Config(&#39;lr&#39;) = 0.001 and State = &#39;finished&#39;&quot;
* `Opérateurs pris en charge`:  `=`, `==`, `!=`, `<`, `>`, `<=`, `>=`, `in`, `not in`
* `groupby` (LList[expr.MetricType]):  Une liste de métriques selon lesquelles regrouper les éléments du runset. Définissez-la sur `Metric`, `Summary`, `Config`, `Tags` ou `KeysInfo`.
* `order` (LList[expr.Ordering]):  Une liste de métriques et de critères de tri à appliquer au runset.
* `run_settings` (Dict[str, RunSettings]):  Un dictionnaire de paramètres de run, où la clé est l&#39;ID du run et la valeur est un objet RunSettings.
* `pinned_columns` (LList[str]):  Liste des noms de colonnes à épingler.
* `Le format des noms de colonnes est`:  &quot;run:displayName&quot;, &quot;summary:metric&quot;, &quot;config:param&quot;.
* `run`: displayName est ajouté automatiquement s&#39;il n&#39;est pas présent.
* `Exemple`:  [&quot;summary:accuracy&quot;, &quot;summary:loss&quot;]

**Exemple :**

```python
    # Utilisation de filtres sous forme de chaîne (nouveau)
    RunsetSettings(
        filters="Config('learning_rate') = 0.001 and State = 'finished'",
        pinned_columns=["summary:accuracy", "summary:loss"],
    )

    # Utilisation d'une liste FilterExpr (méthode originale)
    RunsetSettings(
        filters=[expr.Config("learning_rate") == 0.001],
        pinned_columns=["summary:accuracy", "summary:loss"],
    )
```

## <kbd>class</kbd> `Section`

Représente une section dans un Workspace.

**Attributs :**

* `name` (str):  Nom/titre de la section.
* `panels` (LList[PanelTypes]):  Liste ordonnée des panneaux de la section. Par défaut, le premier se trouve en haut à gauche et le dernier en bas à droite.
* `is_open` (bool):  Indique si la section est ouverte ou fermée. Par défaut, elle est fermée.
* `pinned` (bool):  Indique si la section est épinglée. Les sections épinglées apparaissent en haut du Workspace. La valeur par défaut est `False`.
* `layout_settings` (SectionLayoutSettings):  Paramètres de mise en page des panneaux dans la section.
* `panel_settings`:  Paramètres au niveau du panneau appliqués à tous les panneaux de la section, similaires à `WorkspaceSettings` pour une `Section`.

***

## <kbd>class</kbd> `SectionLayoutSettings`

Paramètres de mise en page du panneau d’une section, généralement affichés en haut à droite de la section dans l’interface utilisateur Workspace de W&amp;B App.

**Attributs :**

* `columns` (int) :  Le nombre de colonnes de la mise en page. La valeur par défaut est 3.
* `rows` (int) :  Le nombre de lignes de la mise en page. La valeur par défaut est 2.

***

## <kbd>class</kbd> `SectionPanelSettings`

Paramètres du panneau d’une section, similaires à ceux de `WorkspaceSettings` pour une section.

Les paramètres appliqués ici peuvent être remplacés par des paramètres du panneau plus granulaires selon cette priorité : Section &lt; Panel.

**Attributs :**

* `x_axis` (str):  Paramètre du nom de la métrique pour l’axe des x. Par défaut, la valeur est définie sur &quot;Step&quot;.
* `x_min Optional[float]`:  Valeur minimale pour l’axe des x.
* `x_max Optional[float]`:  Valeur maximale pour l’axe des x.
* `smoothing_type` (Literal[&#39;exponentialTimeWeighted&#39;, &#39;exponential&#39;, &#39;gaussian&#39;, &#39;average&#39;, &#39;none&#39;]):  Type de lissage appliqué à tous les panneaux.
* `smoothing_weight` (int):  Poids du lissage appliqué à tous les panneaux.

***

## <kbd>class</kbd> `Workspace`

Représente un Workspace W&amp;B, y compris les sections, les paramètres et la configuration des ensembles de runs.

**Attributs :**

* `entity` (str):  L’entité sous laquelle ce Workspace sera enregistré (généralement le nom d’un utilisateur ou d’une équipe).
* `project` (str):  Le projet dans lequel ce Workspace sera enregistré.
* `name`:  Le nom du Workspace.
* `sections` (LList[Section]):  Une liste ordonnée de sections dans le Workspace.  La première section se trouve en haut du Workspace.
* `settings` (WorkspaceSettings):  Les paramètres du Workspace, généralement visibles en haut du Workspace dans l’interface utilisateur.
* `runset_settings` (RunsetSettings):  Les paramètres du runset  (la barre latérale gauche contenant les runs) dans un Workspace.
* `auto_generate_panels` (bool):  Indique s’il faut générer automatiquement des panneaux pour toutes les clés enregistrées dans ce projet.  Recommandé si vous souhaitez que toutes les données disponibles soient visualisées par défaut.  Cette option ne peut être définie qu’au moment de la création du Workspace et ne peut pas être modifiée par la suite.

## <kbd>class</kbd> `WorkspaceSettings`

Paramètres du Workspace, généralement visibles en haut du Workspace dans l&#39;interface utilisateur.

Cet objet inclut les paramètres de l&#39;axe des x, du lissage, des valeurs aberrantes, des panneaux, des info-bulles, des runs et de la barre de requête des panneaux.

Les paramètres appliqués ici peuvent être remplacés par des paramètres de Section et de Panel plus précis, selon l&#39;ordre de priorité suivant : Workspace &lt; Section &lt; Panel

**Attributs :**

* `x_axis` (str) :  Paramètre du nom de la métrique de l&#39;axe des x.
* `x_min` (Optional[float]) :  Valeur minimale de l&#39;axe des x.
* `x_max` (Optional[float]) :  Valeur maximale de l&#39;axe des x.
* `smoothing_type` (Literal[&#39;exponentialTimeWeighted&#39;, &#39;exponential&#39;, &#39;gaussian&#39;, &#39;average&#39;, &#39;none&#39;]) :  Type de lissage appliqué à tous les panneaux.
* `smoothing_weight` (int) :  Poids du lissage appliqué à tous les panneaux.
* `ignore_outliers` (bool) :  Ignorer les valeurs aberrantes dans tous les panneaux.
* `sort_panels_alphabetically` (bool) :  Trie les panneaux de toutes les sections par ordre alphabétique.
* `group_by_prefix` (Literal[&quot;first&quot;, &quot;last&quot;]) :  Regroupe les panneaux selon le premier préfixe ou jusqu&#39;au dernier préfixe (`first` ou `last`). La valeur par défaut est `last`.
* `remove_legends_from_panels` (bool) :  Supprime les légendes de tous les panneaux.
* `tooltip_number_of_runs` (Literal[&quot;default&quot;, &quot;all&quot;, &quot;none&quot;]) :  Nombre de runs à afficher dans l&#39;info-bulle.
* `tooltip_color_run_names` (bool) :  Indique s&#39;il faut colorer les noms de run dans l&#39;info-bulle pour qu&#39;ils correspondent au jeu de runs (`True`) ou non (`False`). La valeur par défaut est `True`.
* `max_runs` (int) :  Nombre maximal de runs à afficher par panneau (il s&#39;agira des 10 premiers runs du jeu de runs).
* `point_visualization_method` (Literal[&quot;line&quot;, &quot;point&quot;, &quot;line&#95;point&quot;]) :  Méthode de visualisation des points.
* `panel_search_query` (str) :  Requête de la barre de recherche des panneaux (peut être une expression régulière).
* `auto_expand_panel_search_results` (bool) :  Indique s&#39;il faut développer automatiquement les résultats de recherche des panneaux.
