Bibliothèque Python pour utiliser l’API Reports de W&amp;B par programmation.

```python
import wandb_workspaces.reports.v2 as wr

report = wr.Report(
     entity="entity",
     project="project",
     title="An amazing title",
     description="A descriptive description.",
)

blocks = [
     wr.PanelGrid(
         panels=[
             wr.LinePlot(x="time", y="velocity"),
             wr.ScatterPlot(x="time", y="acceleration"),
         ]
     )
]

report.blocks = blocks
report.save()
```


## <kbd>class</kbd> `BarPlot`

Un objet de panneau qui affiche un graphique à barres en 2D.

**Attributs :**

* `title` (Optional[str]) :  Le texte affiché en haut du graphique.
* `metrics` (LList[MetricType]) :  orientation Literal[&quot;v&quot;, &quot;h&quot;] : L’orientation du graphique à barres. Définissez-la sur verticale (&quot;v&quot;) ou horizontale (&quot;h&quot;). La valeur par défaut est l’horizontale (&quot;h&quot;).
* `range_x` (Tuple[float | None, float | None]) :  Tuple qui spécifie la plage de l’axe x.
* `title_x` (Optional[str]) :  Le libellé de l’axe x.
* `title_y` (Optional[str]) :  Le libellé de l’axe y.
* `groupby` (Optional[str]) :  Regroupe les runs en fonction d'une métrique enregistrée dans votre projet W&amp;B, à partir de laquelle le rapport extrait des informations.
* `groupby_aggfunc` (Optional[GroupAgg]) :  Agrège les runs à l’aide de la fonction spécifiée. Les options incluent &quot;mean&quot;, &quot;min&quot;, &quot;max&quot;, &quot;median&quot;, &quot;sum&quot;, &quot;samples&quot; ou `None`.
* `groupby_rangefunc` (Optional[GroupArea]) :   Regroupe les runs selon une plage. Les options incluent &quot;minmax&quot;, &quot;stddev&quot;, &quot;stderr&quot;, &quot;none&quot;, &quot;samples&quot; ou `None`.
* `max_runs_to_show` (Optional[int]) :  Le nombre maximal de runs à afficher sur le graphique.
* `max_bars_to_show` (Optional[int]) :  Le nombre maximal de barres à afficher sur le graphique à barres.
* `custom_expressions` (Optional[LList[str]]) :  Une liste d’expressions personnalisées à utiliser dans le graphique à barres.
* `legend_template` (Optional[str]) :  Le modèle de la légende.
* `font_size` ( Optional[FontSize]) :  La taille de police du graphique en courbes. Les options incluent &quot;small&quot;, &quot;medium&quot;, &quot;large&quot;, &quot;auto&quot; ou `None`.
* `line_titles` (Optional[dict]) :  Les titres des lignes. Les clés sont les noms des lignes et les valeurs sont les titres.
* `line_colors` (Optional[dict]) :  Les couleurs des lignes. Les clés sont les noms des lignes et les valeurs sont les couleurs.
* `aggregate` (Optional[bool]) :  Si défini sur `True`, agrège les données.

## <kbd>class</kbd> `LinePlot`

Un objet de panneau contenant des graphiques en courbes 2D.

**Attributs :**

* `title` (Optional[str]) :  Le texte affiché en haut du graphique.
* `x` (Optional[MetricType]) :  Le nom d&#39;une métrique enregistrée dans votre projet W&amp;B, à partir de laquelle le rapport extrait des informations. La métrique spécifiée est utilisée pour l&#39;axe x.
* `y` (LList[MetricType]) :  Une ou plusieurs métriques enregistrées dans votre projet W&amp;B, à partir desquelles le rapport extrait des informations. La métrique spécifiée est utilisée pour l&#39;axe y.
* `range_x` (Tuple[float | `None`, float | `None`]) :  Tuple spécifiant la plage de l&#39;axe x.
* `range_y` (Tuple[float | `None`, float | `None`]) :  Tuple spécifiant la plage de l&#39;axe y.
* `log_x` (Optional[bool]) :  Trace les coordonnées x sur une échelle logarithmique en base 10.
* `log_y` (Optional[bool]) :  Trace les coordonnées y sur une échelle logarithmique en base 10.
* `title_x` (Optional[str]) :  Le libellé de l&#39;axe x.
* `title_y` (Optional[str]) :  Le libellé de l&#39;axe y.
* `ignore_outliers` (Optional[bool]) :  Si défini sur `True`, n&#39;affiche pas les valeurs aberrantes.
* `groupby` (Optional[str]) :  Regroupe les runs en fonction d&#39;une métrique enregistrée dans votre projet W&amp;B, à partir de laquelle le rapport extrait des informations.
* `groupby_aggfunc` (Optional[GroupAgg]) :  Agrège les runs à l&#39;aide de la fonction spécifiée. Les options incluent &quot;mean&quot;, &quot;min&quot;, &quot;max&quot;, &quot;median&quot;, &quot;sum&quot;, &quot;samples&quot; ou `None`.
* `groupby_rangefunc` (Optional[GroupArea]) :   Regroupe les runs selon une plage. Les options incluent &quot;minmax&quot;, &quot;stddev&quot;, &quot;stderr&quot;, &quot;none&quot;, &quot;samples&quot; ou `None`.
* `smoothing_factor` (Optional[float]) :  Le facteur de lissage à appliquer au type de lissage. Les valeurs acceptées sont comprises entre 0 et 1.
* `smoothing_type Optional[SmoothingType]` :  Applique un filtre basé sur la distribution spécifiée. Les options incluent &quot;exponentialTimeWeighted&quot;, &quot;exponential&quot;,  &quot;gaussian&quot;, &quot;average&quot; ou &quot;none&quot;.
* `smoothing_show_original` (Optional[bool]) :    Si défini sur `True`, affiche les données d&#39;origine.
* `max_runs_to_show` (Optional[int]) :  Le nombre maximal de runs à afficher sur le graphique en courbes.
* `custom_expressions` (Optional[LList[str]]) :  Expressions personnalisées à appliquer aux données.
* `plot_type Optional[LinePlotStyle]` :  Le type de graphique en courbes à générer.  Les options incluent &quot;line&quot;, &quot;stacked-area&quot; ou &quot;pct-area&quot;.
* `font_size Optional[FontSize]` :  La taille de police du graphique en courbes.  Les options incluent &quot;small&quot;, &quot;medium&quot;, &quot;large&quot;, &quot;auto&quot; ou `None`.
* `legend_position Optional[LegendPosition]` :  Emplacement de la légende.  Les options incluent &quot;north&quot;, &quot;south&quot;, &quot;east&quot;, &quot;west&quot; ou `None`.
* `legend_template` (Optional[str]) :  Le modèle de la légende.
* `aggregate` (Optional[bool]) :  Si défini sur `True`, agrège les données.
* `xaxis_expression` (Optional[str]) :  L&#39;expression de l&#39;axe x.
* `xaxis_format` (Optional[str]) :  Le format de l&#39;axe x. Cette option apparaît si vous définissez une métrique personnalisée. Par exemple, vous pouvez spécifier &#39;datetime&#39; pour formater l&#39;axe x en date et heure.
* `legend_fields` (Optional[LList[str]]) :  Les champs à inclure dans la légende.
* `metric_regex` (Optional[str]) :  Motif d&#39;expression régulière permettant de faire correspondre les métriques de l&#39;axe y. Le backend utilisera ce motif pour sélectionner les métriques correspondantes.
* `point_visualization_method` (Optional[PointVizMethod]) :  La méthode utilisée pour agréger les points lorsqu&#39;ils sont trop nombreux pour être affichés. Les options incluent &quot;bucketing-gorilla&quot; (regroupe les points de données dans des buckets et affiche les valeurs min, max et avg par bucket afin de préserver les valeurs aberrantes et les pics) ou &quot;sampling&quot; (échantillonne aléatoirement des points pour un rendu plus rapide, mais peut manquer certaines valeurs aberrantes).

## <kbd>class</kbd> `Report`

Un objet qui représente un rapport W&amp;B. Utilisez l&#39;attribut `blocks` de l&#39;objet renvoyé pour personnaliser votre rapport. Les objets Report ne sont pas enregistrés automatiquement. Utilisez la méthode `save()` pour enregistrer les modifications.

**Attributs :**

* `project` (str):  Le nom du projet W&amp;B que vous souhaitez charger. Le projet spécifié apparaît dans l&#39;URL du rapport.
* `entity` (str):  L&#39;entité W&amp;B propriétaire du rapport. L&#39;entité apparaît dans l&#39;URL du rapport.
* `title` (str):  Le titre du rapport. Le titre apparaît en haut du rapport sous forme d&#39;en-tête H1.
* `description` (str):  Une description du rapport. La description apparaît sous le titre du rapport.
* `blocks` (LList[BlockTypes]):  Une liste d&#39;une ou plusieurs balises HTML, graphiques, grilles, ensembles de runs, etc.
* `width` (Literal[&#39;readable&#39;, &#39;fixed&#39;, &#39;fluid&#39;]):  La largeur du rapport. Les options incluent &#39;readable&#39;, &#39;fixed&#39; et &#39;fluid&#39;.

***

### <kbd>propriété</kbd> url

L’URL où le rapport est hébergé. L’URL du rapport est de la forme `https://wandb.ai/{entity}/{project_name}/reports/`, où `{entity}` et `{project_name}` correspondent respectivement à l’entité à laquelle appartient le rapport et au nom du projet.

* `color`:  Couleur d’affichage du run. Accepte le format hexadécimal (`#ff0000`), un nom de couleur CSS (red) ou une chaîne RGB (&quot;rgb(255,0,0)&quot;).
* `disabled`:  Si True, masque le run dans les graphiques. Cela équivaut à cliquer sur l’icône en forme d’œil fermé dans l’interface utilisateur.

## <kbd>class</kbd> `Runset`

Un ensemble de runs à afficher dans une grille de panneaux.

**Attributs :**

* `entity` (str):  Une entité qui possède le projet où les runs sont stockés, ou qui dispose des autorisations nécessaires pour y accéder.
* `project` (str):  Le nom du projet où les runs sont stockés.
* `name` (str):  Le nom de l’ensemble de runs. Défini sur `Run set` par défaut.
* `query` (str):  Une chaîne de requête pour filtrer les runs.
* `filters` (Union[str, LList[expr.FilterExpr]]):  Les filtres à appliquer aux runs. Peut être :
  * Une expression sous forme de chaîne : par exemple, &quot;Config(&#39;lr&#39;) = 0.001 and State = &#39;finished&#39;&quot;
* `Supports operators`:  `=` , `==` , `!=` , `<` , `>` , `<=` , `>=` , `in` , `not in`
  * Une liste d’objets FilterExpr : par exemple, [expr.Config(&#39;lr&#39;) == 0.001]
* `groupby` (LList[str]):  Une liste de noms de métriques à utiliser pour le regroupement. Les formats pris en charge sont :
  * &quot;group&quot; ou &quot;run.group&quot; pour regrouper par attribut de run
  * &quot;config.param&quot; pour regrouper par paramètre de configuration
  * &quot;summary.metric&quot; pour regrouper par métrique de résumé
* `order` (LList[OrderBy]):  Une liste d’objets `OrderBy` à utiliser pour le tri.
* `custom_run_colors` (dict):  Un dictionnaire qui associe les ID de run (ou les clés `RunsetGroup` pour les runs groupés) à des couleurs. Pour définir simplement une couleur par run, préférez `run_settings`.
* `run_settings` (Dict[str, RunSettings]):  Paramètres d’affichage par run, indexés par ID de run.  Utilisez-les pour définir des couleurs et/ou masquer des runs individuels. Voir `RunSettings`.

**Exemple :**

```python
    # Utilisation de filtres sous forme de chaîne
    wr.Runset(
         entity="my-entity",
         project="my-project",
         filters="Config('learning_rate') = 0.001 and State = 'finished'"
    )

    # Utilisation d'une liste FilterExpr
    wr.Runset(
         entity="my-entity",
         project="my-project",
         filters=[expr.Config("learning_rate") == 0.001]
    )
```

## <kbd>class</kbd> `WeavePanelSummaryTable`

Un panneau qui affiche un tableau W&amp;B, un dataframe pandas, un graphique ou toute autre valeur enregistrée dans W&amp;B. La requête se présente sous la forme de

```python
runs.summary['value']
```

Le terme &quot;Weave&quot; dans le nom de l’API ne fait pas référence à l’outil W&amp;B Weave utilisé pour le suivi et l’évaluation des LLM.

**Attributs :**

* `table_name` (str):  Le nom du tableau, du dataframe, du graphique ou de la valeur.
* `layout` (Layout):  La configuration de mise en page du panneau, y compris sa position et sa taille.  Héritée de la classe de base Panel. Permet d’ajuster la largeur (w) et la hauteur (h).

**Exemple :**
Créer un rapport avec un panneau de tableau de synthèse qui affiche un tableau avec des dimensions personnalisées :

```python
    import wandb_workspaces.reports as wr

    report = wr.Report(
         project="my-project",
         entity="my-entity",
         title="Summary Table Report"
    )

    report.blocks = [
         wr.PanelGrid(
             runsets=[wr.Runset(project="my-project")],
             panels=[
                 wr.WeavePanelSummaryTable(
                     table_name="my-table-name",
                     layout=wr.Layout(w=24, h=20)
                 )
             ]
         )
    ]

    report.save()
```

Les paramètres de mise en page contrôlent les dimensions du panneau :

* w (largeur) : largeur en unités de grille (par défaut : 8, max : 24)
  * h (hauteur) : hauteur en unités de grille (par défaut : 6)
  * x (position x) : position horizontale dans la grille (par défaut : 0)
  * y (position y) : position verticale dans la grille (par défaut : 0)
