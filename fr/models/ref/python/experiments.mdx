Ces classes constituent les principaux éléments de base pour suivre les expériences de machine learning, gérer les artefacts et configurer le comportement du SDK. Elles vous permettent de consigner des métriques, de stocker des points de contrôle du modèle, de versionner des jeux de données et de gérer les configurations d&#39;expérience avec des fonctionnalités complètes de reproductibilité et de collaboration.

> Pour plus de détails sur l&#39;utilisation de ces classes dans des expériences de ML, consultez la documentation [Experiments](/fr/models/ref/python/experiments/) et [Artifacts](/fr/models/artifacts/).

<div id="core-classes">
  ## Classes principales
</div>

| Classe                                                 | Description                                                                                                                                             |
| ------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [`Run`](/fr/models/ref/python/experiments/run/)           | L’unité de calcul principale enregistrée par W&amp;B, représentant une expérience de ML unique avec des métriques, des configurations et des résultats. |
| [`Artifact`](/fr/models/ref/python/experiments/artifact/) | Élément de base flexible et léger pour la gestion des versions des jeux de données et des modèles, avec déduplication automatique et traçabilité.             |
| [`Settings`](/fr/models/ref/python/experiments/settings/) | Gestion de la configuration du SDK W&amp;B, qui contrôle le comportement, de la journalisation aux interactions avec l’API.                             |

<div id="track-an-experiment">
  ### Suivre une expérience
</div>

Créer et suivre une expérience de machine learning avec journalisation des métriques :

```python
import wandb

# Initialiser un nouveau run
with wandb.init(project="my-experiments", config={"learning_rate": 0.001}) as run:
    # Accéder à la configuration
    config = run.config
    
    # Consigner des métriques pendant l'entraînement
    for epoch in range(10):
        metrics = train_one_epoch()  # Votre logique d'entraînement
        run.log({
            "loss": metrics["loss"],
            "accuracy": metrics["accuracy"],
            "epoch": epoch
        })
    
    # Consigner les métriques de synthèse
    run.summary["best_accuracy"] = max_accuracy
```

<div id="version-a-model-artifact">
  ### Versionner un artefact de modèle
</div>

Créez et journalisez un artefact de modèle versionné avec des métadonnées :

```python
import wandb

with wandb.init(project="my-models") as run:
    # Entraînez votre modèle
    model = train_model()
    
    # Créez un artefact pour le modèle
    model_artifact = wandb.Artifact(
        name="my-model",
        type="model",
        description="ResNet-50 trained on ImageNet subset",
        metadata={
            "architecture": "ResNet-50",
            "dataset": "ImageNet-1K",
            "accuracy": 0.95
        }
    )
    
    # Ajoutez les fichiers du modèle à l'artefact
    model_artifact.add_file("model.pt")
    model_artifact.add_dir("model_configs/")
    
    # Enregistrez l'artefact dans W&B
    run.log_artifact(model_artifact)
```

<div id="configure-sdk-settings">
  ### Configurer les paramètres du SDK
</div>

Personnalisez le comportement du SDK W&amp;B selon vos besoins spécifiques :

```python
import wandb

# Configurer les paramètres par programmation
wandb.Settings(
    project="production-runs",
    entity="my-team",
    mode="offline",  # Exécuter hors ligne, synchroniser plus tard
    save_code=True,   # Sauvegarder le code source
    quiet=True        # Réduire la sortie console
)

# Ou utiliser des variables d'environnement
# export WANDB_PROJECT=production-runs
# export WANDB_MODE=offline

# Initialiser avec des paramètres personnalisés
with wandb.init() as run:
    # Votre code d'expérience ici
    pass
```

<div id="link-artifacts-for-lineage-tracking">
  ### Lier des artefacts pour assurer la traçabilité
</div>

Suivez les relations entre les jeux de données, les modèles et les évaluations :

```python
import wandb

with wandb.init(project="ml-pipeline") as run:
    # Utiliser un artefact de jeu de données
    dataset = run.use_artifact("dataset:v1")
    dataset_dir = dataset.download()
    
    # Entraîner le modèle avec le jeu de données
    model = train_on_dataset(dataset_dir)
    
    # Créer un artefact de modèle avec la traçabilité du jeu de données
    model_artifact = wandb.Artifact(
        name="trained-model",
        type="model"
    )
    model_artifact.add_file("model.pt")
    
    # Journaliser avec suivi automatique de la traçabilité
    run.log_artifact(model_artifact)
```
