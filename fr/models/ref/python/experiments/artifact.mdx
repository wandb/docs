## <kbd>classe</kbd> `Artifact`

Élément de base flexible et léger pour la gestion des versions des jeux de données et des modèles.

Créez un artefact W&amp;B vide. Renseignez le contenu de l’artefact à l’aide de méthodes qui commencent par `add`. Une fois que l’artefact contient tous les fichiers souhaités, vous pouvez appeler `run.log_artifact()` pour le journaliser.

### <kbd>méthode</kbd> `Artifact.__init__`

```python
__init__(
    name: 'str',
    type: 'str',
    description: 'str | None' = None,
    metadata: 'dict[str, Any] | None' = None,
    incremental: 'bool' = False,
    use_as: 'str | None' = None,
    storage_region: 'str | None' = None
) → None
```

**Arguments :**

* `name` (str):  Nom explicite de l’artefact. Utilisez ce nom pour identifier un artefact spécifique dans l’interface utilisateur de l’application W&amp;B ou par programmation. Vous pouvez également référencer un artefact de manière interactive avec l’API publique `use_artifact`. Un nom peut contenir des lettres, des chiffres, des tirets bas, des traits d’union et des points. Le nom doit être unique à l’échelle d’un projet.
* `type` (str):  Le type de l’artefact. Utilisez le type d’un artefact à la fois pour organiser et distinguer les artefacts. Vous pouvez utiliser n’importe quelle chaîne contenant des lettres, des chiffres, des tirets bas, des traits d’union et des points. Les types courants incluent `dataset` ou `model`. Incluez `model` dans votre chaîne de type si vous souhaitez lier l’artefact au registre de modèles W&amp;B. Notez que certains types sont réservés à un usage interne et ne peuvent pas être définis par les utilisateurs. Ces types incluent `job` et les types qui commencent par `wandb-`.
* `description (str | None) = None`:  Description de l’artefact. Pour les Artifacts de modèle ou de jeu de données, ajoutez la documentation correspondant à votre fiche de modèle ou de jeu de données standardisée au sein de l’équipe. Consultez la description d’un artefact par programmation avec l’attribut `Artifact.description` ou dans l’interface utilisateur de l’application W&amp;B. W&amp;B affiche la description au format Markdown dans l’application W&amp;B.
* `metadata (dict[str, Any] | None) = None`:  Informations supplémentaires sur un artefact. Spécifiez les métadonnées sous la forme d’un dictionnaire de paires clé-valeur. Vous ne pouvez pas définir plus de 100 clés au total.
* `incremental`:  Utilisez plutôt la méthode `Artifact.new_draft()` pour modifier un artefact existant.
* `use_as`:  Obsolète.

**Retourne :**
Un objet `Artifact`.

### <kbd>propriété</kbd> Artifact.aliases

Liste d’une ou plusieurs références sémantiquement explicites, ou de « surnoms » d’identification, attribués à une version d’artefact.

Les alias sont des références modifiables que vous pouvez utiliser par programmation. Modifiez l’alias d’un artefact dans l’interface de l’application W&amp;B ou par programmation. Voir [Créer de nouvelles versions d’artefact](https://docs.wandb.ai/guides/artifacts/create-a-new-artifact-version) pour plus d’informations.

**Retourne :**

* `list[str]`: La valeur de la propriété aliases.

### <kbd>propriété</kbd> Artifact.collection

Collection depuis laquelle cet artefact est récupéré.

Une collection est un groupe ordonné de versions d’artefact. Si cet artefact est récupéré depuis une collection à laquelle il est lié, renvoie cette collection. Sinon, renvoie la collection dont est issue la version de l’artefact.

La collection dont un artefact est issu est appelée la séquence source.

**Retourne :**

* `ArtifactCollection` : La valeur de la propriété de la collection.

### <kbd>propriété</kbd> Artifact.ttl

La politique TTL d’un artefact.

Les artefacts sont supprimés peu après l’expiration de la durée définie par une politique TTL. Si cette valeur est définie sur `None`, l’artefact désactive les politiques TTL et ne sera pas programmé pour être supprimé, même s’il existe une politique TTL par défaut pour l’équipe. Un artefact hérite de la politique TTL par défaut de l’équipe si l’administrateur de l’équipe en définit une et qu’aucune politique personnalisée n’est définie sur l’artefact.

**Exceptions levées :**

* `ArtifactNotLoggedError`:  Impossible de récupérer la politique TTL héritée si l’artefact n’a pas été enregistré ou sauvegardé.

**Retourne :**

### <kbd>méthode</kbd> `Artifact.add`

```python
add(
    obj: 'WBValue',
    name: 'StrPath',
    overwrite: 'bool' = False
) → ArtifactManifestEntry
```

Ajoute l’objet wandb.WBValue `obj` à l’artefact.

**Arguments :**

* `obj` :  L’objet à ajouter. Prend actuellement en charge l’un des types suivants : Bokeh, JoinedTable,  PartitionedTable, Table, Classes, ImageMask, BoundingBoxes2D,  Audio, Image, Video, Html, Object3D
* `name` :  Le chemin au sein de l’artefact où ajouter l’objet.
* `overwrite` :  Si `True`, remplace les objets existants ayant le même chemin  de fichier, le cas échéant.

**Renvoie :**
L’entrée de manifeste ajoutée

**Exceptions levées :**

* `ArtifactFinalizedError` :  Vous ne pouvez pas modifier la version actuelle de l’artefact, car elle est finalisée. Journalisez plutôt une nouvelle version de l’artefact.

### <kbd>méthode</kbd> `Artifact.add_dir`

```python
add_dir(
    local_path: 'str',
    name: 'str | None' = None,
    skip_cache: 'bool | None' = False,
    policy: "Literal['mutable', 'immutable'] | None" = 'mutable',
    merge: 'bool' = False
) → None
```

Ajoute un répertoire local à l’artefact.

**Arguments :**

* `local_path` :  Le chemin du répertoire local.
* `name` :  Le nom du sous-répertoire dans un artefact. Le nom que vous  indiquez apparaît dans l’interface utilisateur de l’application W&amp;B, sous le `type` de l’artefact.  Par défaut, il s’agit de la racine de l’artefact.
* `skip_cache` :  Si défini sur `True`, W&amp;B ne copiera ni ne déplacera les fichiers vers  le cache pendant le téléversement
* `policy` :  Par défaut, &quot;mutable&quot;.
  * mutable: Créer une copie temporaire du fichier pour éviter  toute corruption pendant le téléversement.
  * immutable: Désactiver la protection et compter sur l’utilisateur pour ne pas supprimer  ni modifier le fichier.
* `merge` :  Si `False` (valeur par défaut), lève une ValueError si un fichier a déjà été ajouté  lors d’un précédent appel à add&#95;dir et que son contenu a changé. Si `True`,  remplace les fichiers existants dont le contenu a changé. Ajoute toujours les nouveaux fichiers  et ne supprime jamais de fichiers. Pour remplacer un répertoire entier, indiquez un nom  lors de l’ajout du répertoire avec `add_dir(local_path, name=my_prefix)`  puis appelez `remove(my_prefix)` pour supprimer le répertoire, puis ajoutez-le à nouveau.

**Exceptions levées :**

* `ArtifactFinalizedError` :  Vous ne pouvez pas apporter de modifications à la version actuelle de l’artefact  car elle est finalisée. Journalisez plutôt une nouvelle version  de l’artefact.
* `ValueError` :  La valeur de `policy` doit être &quot;mutable&quot; ou &quot;immutable&quot;

```python
add_file(

    name: 'str | None' = None,
    is_tmp: 'bool | None' = False,

```

Ajoutez un fichier local à l’artefact.

**Arguments :**

* `local_path`:  Le chemin du fichier à ajouter.
* `name`:  Le chemin à utiliser dans l’artefact pour le fichier ajouté.  Par défaut, il s’agit du nom de base du fichier.
* `is_tmp`:  Si la valeur est true, le fichier est renommé de manière déterministe afin d’éviter les collisions.
* `skip_cache`:  Si `True`, ne copiez pas les fichiers dans le cache après le téléversement.
* `policy`:  Par défaut, la valeur est &quot;mutable&quot;. Si la valeur est &quot;mutable&quot;,  créez une copie temporaire du fichier pour éviter toute corruption  pendant le téléversement. Si la valeur est &quot;immutable&quot;, désactivez  cette protection et comptez sur l’utilisateur pour ne pas supprimer ni modifier le  fichier.
* `overwrite`:  Si `True`, remplacez le fichier s’il existe déjà.

**Renvoie :**
L’entrée du manifeste ajoutée.

**Exceptions levées :**

* `ArtifactFinalizedError`:  Vous ne pouvez pas modifier la version actuelle de l’artefact, car elle est finalisée. Journalisez plutôt une nouvelle version de l’artefact.
* `ValueError`:  La politique doit être &quot;mutable&quot; ou &quot;immutable&quot;

### <kbd>méthode</kbd> `Artifact.add_reference`

```python
add_reference(
    uri: 'ArtifactManifestEntry | str',
    name: 'StrPath | None' = None,
    checksum: 'bool' = True,
    max_objects: 'int | None' = None
) → Sequence[ArtifactManifestEntry]
```

Ajoutez à l’artefact une référence désignée par un URI.

Contrairement aux fichiers ou répertoires que vous ajoutez à un artefact, les références ne sont pas téléversées vers W&amp;B. Pour plus d&#39;informations, voir [Suivre des fichiers externes](https://docs.wandb.ai/guides/artifacts/track-external-files).

Par défaut, les schémas suivants sont pris en charge :

* http(s) : la taille et l’empreinte du fichier seront déduites des en-têtes de réponse `Content-Length` et `ETag` renvoyés par le serveur.
* s3 : la somme de contrôle et la taille sont extraites des métadonnées de l’objet. Si la gestion des versions du bucket est activée, l’ID de version est également suivi.
* gs : la somme de contrôle et la taille sont extraites des métadonnées de l’objet. Si la gestion des versions du bucket est activée, l’ID de version est également suivi.
* https, domaine correspondant à `*.blob.core.windows.net`
* Azure : la somme de contrôle et la taille sont extraites des métadonnées du blob. Si la gestion des versions du compte de stockage est activée, l’ID de version est également suivi.
* file : la somme de contrôle et la taille sont extraites du système de fichiers. Ce schéma est utile si vous disposez d’un partage NFS ou d’un autre volume monté en externe contenant des fichiers que vous souhaitez suivre sans nécessairement les téléverser.

Pour tout autre schéma, l’empreinte est simplement un hachage de l’URI et la taille est laissée vide.

**Arguments :**

* `uri` : chemin URI de la référence à ajouter. Le chemin URI peut être un objet renvoyé par `Artifact.get_entry` pour stocker une référence vers l’entrée d’un autre artefact.
* `name` : le chemin à l’intérieur de l’artefact où placer le contenu de cette référence.
* `checksum` : indique s’il faut ou non calculer une somme de contrôle pour la ou les ressources situées à l’URI de référence. Le calcul d’une somme de contrôle est fortement recommandé, car il permet une validation automatique de l’intégrité. Désactiver ce calcul accélère la création de l’artefact, mais les répertoires de référence ne seront pas parcourus ; les objets qu’ils contiennent ne seront donc pas enregistrés dans l’artefact. Nous recommandons de définir `checksum=False` lors de l’ajout d’objets de référence. Dans ce cas, une nouvelle version ne sera créée que si l’URI de référence change.
* `max_objects` : le nombre maximal d’objets à prendre en compte lors de l’ajout d’une référence qui pointe vers un répertoire ou un préfixe de stockage dans un bucket. Par défaut, le nombre maximal d’objets autorisé pour Amazon S3, GCS, Azure et les fichiers locaux est de 10,000,000. Les autres schémas d’URI n’ont pas de maximum.

**Retourne :**
Les entrées de manifeste ajoutées.

**Exceptions levées :**

* `ArtifactFinalizedError` : vous ne pouvez pas modifier la version actuelle de l’artefact, car elle est finalisée. Consignez plutôt une nouvelle version de l’artefact.

### <kbd>méthode</kbd> `Artifact.download`

```python
download(
    root: 'StrPath | None' = None,
    allow_missing_references: 'bool' = False,
    skip_cache: 'bool | None' = None,
    path_prefix: 'StrPath | None' = None,
    multipart: 'bool | None' = None
) → FilePathStr
```

Téléchargez le contenu de l’artefact dans le répertoire racine spécifié.

Les fichiers existants dans `root` ne sont pas modifiés. Supprimez explicitement `root` avant d’appeler `download` si vous voulez que son contenu corresponde exactement à celui de l’artefact.

**Arguments :**

* `root`:  Le répertoire dans lequel W&amp;B stocke les fichiers de l’artefact.
* `allow_missing_references`:  Si défini sur `True`, tous les chemins de référence non valides seront ignorés lors du téléchargement des fichiers référencés.
* `skip_cache`:  Si défini sur `True`, le cache des artefacts sera ignoré pendant le téléchargement, et W&amp;B téléchargera chaque fichier dans le répertoire racine par défaut ou dans le répertoire de téléchargement spécifié.
* `path_prefix`:  Si spécifié, seuls les fichiers dont le chemin commence par le préfixe indiqué seront téléchargés. Utilise le format Unix (barres obliques).
* `multipart`:  Si défini sur `None` (par défaut), l’artefact sera téléchargé en parallèle à l’aide d’un téléchargement multipart si la taille d’un fichier individuel dépasse 2 Go. Si défini sur `True` ou `False`, l’artefact sera téléchargé en parallèle ou de manière séquentielle, quelle que soit la taille du fichier.

**Retourne :**
Le chemin d’accès au contenu téléchargé.

**Exceptions levées :**

* `ArtifactNotLoggedError`:  Si l’artefact n’est pas enregistré.

### <kbd>méthode</kbd> `Artifact.verify`

```python
verify(root: 'str | None' = None) → None
```

Vérifiez que le contenu d’un artefact correspond au manifeste.

La somme de contrôle de tous les fichiers du répertoire est calculée, puis comparée à celle indiquée dans le manifeste de l’artefact. Les références ne sont pas vérifiées.

**Arguments :**

* `root` :  Le répertoire à vérifier. Si la valeur est `None`, l’artefact sera téléchargé dans &#39;./artifacts/self.name/&#39;.

**Exceptions levées :**

* `ArtifactNotLoggedError` :  Si l’artefact n’est pas enregistré.
* `ValueError` :  Si la vérification échoue.
