## <kbd>fonction</kbd> wandb.sweep

```python
sweep: 'dict | Callable',
entity: 'str | None' = None,
project: 'str | None' = None,
prior_runs: 'list[str] | None' = None
```

Initialisez un sweep d&#39;hyperparamètres.

Recherchez les hyperparamètres qui optimisent une fonction de coût
d&#39;un modèle d&#39;apprentissage automatique en testant différentes combinaisons.

Notez bien l&#39;identifiant unique `sweep_id` renvoyé.
Lors d&#39;une étape ultérieure, fournissez le `sweep_id` à un agent de sweep.

Voir la [structure de configuration d&#39;un sweep](https://docs.wandb.ai/models/sweeps/define-sweep-configuration)
pour savoir comment définir votre sweep.

<div id="args">
  ## Arguments
</div>

* `sweep`: La configuration d&#39;une recherche d&#39;hyperparamètres (ou un générateur de configuration). Si vous fournissez un callable, assurez-vous qu&#39;il ne prend pas d&#39;arguments et qu&#39;il renvoie un dictionnaire conforme à la spécification de configuration des sweeps W&amp;B.
* `entity`: Le nom d&#39;utilisateur ou le nom de l&#39;équipe vers lequel vous souhaitez envoyer les runs W&amp;B créés par le sweep. Assurez-vous que l&#39;entité que vous spécifiez existe déjà. Si vous ne spécifiez pas d&#39;entité, le run sera envoyé à votre entité par défaut, qui correspond généralement à votre nom d&#39;utilisateur.
* `project`: Le nom du projet vers lequel sont envoyés les runs W&amp;B créés à partir du sweep. Si le projet n&#39;est pas spécifié, le run est envoyé à un projet intitulé &#39;Uncategorized&#39;.
* `prior_runs`: Les ID des runs existants à ajouter à ce sweep.

<div id="returns">
  ## Retourne
</div>

`str`: Un identifiant unique pour le sweep.
