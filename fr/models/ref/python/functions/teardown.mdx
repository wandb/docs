### <kbd>fonction</kbd> `teardown`

```python
teardown(exit_code: 'int | None' = None) → None
```

Attend que W&amp;B termine et libère les ressources.

Termine tous les runs qui n&#39;ont pas été explicitement terminés avec `run.finish()` et attend que toutes les données soient téléversées.

Il est recommandé d&#39;appeler cette fonction à la fin d&#39;une session ayant utilisé `wandb.setup()`. Elle est invoquée automatiquement via un hook `atexit`, mais ce mécanisme n&#39;est pas fiable dans certaines configurations, par exemple lors de l&#39;utilisation du module `multiprocessing` de Python.
