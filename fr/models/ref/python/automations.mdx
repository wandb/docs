L’API W&amp;B Automations permet de créer et de gérer par programmation des flux de travail automatisés qui réagissent aux événements de votre pipeline ML. Configurez des actions pour qu’elles se déclenchent lorsque des conditions spécifiques sont remplies, par exemple des seuils de performance du modèle ou la création d’un artefact.

<div id="core-classes">
  ### Classes principales
</div>

| Classe                                                           | Description                                                                 |
| ---------------------------------------------------------------- | --------------------------------------------------------------------------- |
| [`Automation`](/fr/models/ref/python/automations/automation/)       | Représente une instance d’automatisation enregistrée avec sa configuration. |
| [`NewAutomation`](/fr/models/ref/python/automations/newautomation/) | Classe utilitaire permettant de créer de nouvelles automatisations.         |

<div id="events-triggers">
  ### Événements (déclencheurs)
</div>

| Événement                                                                  | Description                                                                                     |
| -------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------- |
| [`OnRunMetric`](/fr/models/ref/python/automations/onrunmetric/)               | Se déclenche lorsqu’une métrique de run remplit une condition définie (seuil, variation, etc.). |
| [`OnCreateArtifact`](/fr/models/ref/python/automations/oncreateartifact/)     | Se déclenche lorsqu’un nouvel artefact est créé dans une collection.                            |
| [`OnLinkArtifact`](/fr/models/ref/python/automations/onlinkartifact/)         | Se déclenche lorsqu’un artefact est lié à un registre.                                          |
| [`OnAddArtifactAlias`](/fr/models/ref/python/automations/onaddartifactalias/) | Se déclenche lorsqu’un alias est ajouté à un artefact.                                          |

<div id="actions">
  ### Actions
</div>

| Action                                                                 | Description                                                     |
| ---------------------------------------------------------------------- | --------------------------------------------------------------- |
| [`SendNotification`](/fr/models/ref/python/automations/sendnotification/) | Envoie des notifications via Slack ou d’autres canaux intégrés. |
| [`SendWebhook`](/fr/models/ref/python/automations/sendwebhook/)           | Envoie des requêtes webhook HTTP vers des services externes.    |
| [`DoNothing`](/fr/models/ref/python/automations/donothing/)               | Action factice pour tester les configurations d’automatisation. |

<div id="common-use-cases">
  ## Cas d’usage courants
</div>

<div id="model-performance-monitoring">
  ### Surveillance des performances du modèle
</div>

* Déclenchez une alerte lorsque la précision du modèle passe sous un seuil
* Notifiez l’équipe lorsque la perte d’entraînement plafonne
* Déclenchez des pipelines de réentraînement basés sur des métriques de performance

<div id="artifact-management">
  ### Gestion des artefacts
</div>

* Envoyez des notifications lorsque de nouvelles versions de modèle sont créées
* Déclenchez des flux de travail de déploiement lorsque des artefacts sont tagués
* Automatisez les traitements en aval lorsque des jeux de données sont mis à jour

<div id="experiment-tracking">
  ### Suivi des expériences
</div>

* Alerte en cas d’échec ou d’interruption des runs
* Être notifié lorsque des expériences de longue durée se terminent
* Recevoir des résumés quotidiens des métriques des expériences

<div id="integration-workflows">
  ### Flux de travail d’intégration
</div>

* Mettre à jour les systèmes de suivi externes via des webhooks
* Synchroniser le registre de modèles avec les plateformes de déploiement
* Déclencher des pipelines CI/CD à partir d’événements W&amp;B

<div id="example-usage">
  ## Exemple d’utilisation
</div>

L’exemple suivant crée une automatisation qui envoie une notification Slack chaque fois qu’une métrique nommée `custom-metric` dépasse 10. `custom-metric` doit être enregistrée pendant l’entraînement à l’aide de `wandb.Run.log({"custom-metric": value })`.

```python
import wandb
from wandb.automations import OnRunMetric, RunEvent, SendNotification

api = wandb.Api()

project = api.project("<my-project>", entity="<my-team>")

# Utiliser la première intégration Slack pour l'équipe
slack_hook = next(api.slack_integrations(entity="<my-team>"))

# Créer un événement déclencheur
event = OnRunMetric(
     scope=project,
     filter=RunEvent.metric("custom-metric") > 10,
)

# Créer une action qui répond à l'événement
action = SendNotification.from_integration(slack_hook)

# Créer l'automatisation
automation = api.create_automation(
     event >> action,
     name="my-automation",
     description="Envoyer un message Slack chaque fois que 'custom-metric' dépasse 10.",
)
```

Pour en savoir plus sur l’utilisation de l’API Automations, voir le [guide Automations](/fr/models/automations/).
