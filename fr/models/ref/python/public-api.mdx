L’API publique W&amp;B fournit un accès programmatique pour effectuer des requêtes, exporter et mettre à jour les données stockées dans W&amp;B. Utilisez cette API pour l’analyse a posteriori, l’exportation de données et la gestion programmatique des runs, des artefacts et des balayages. Alors que le SDK principal gère la journalisation en temps réel pendant l’entraînement, l’API publique vous permet de récupérer les données historiques, de mettre à jour les métadonnées, de gérer les artefacts et d’effectuer des analyses sur les expériences terminées. La classe principale `Api` sert de point d’entrée à la plupart des fonctionnalités.

<div id="authentication">
  ## Authentification
</div>

L&#39;API publique utilise le même mécanisme d&#39;authentification que le SDK Python. Vous pouvez vous authentifier de plusieurs façons :

Utilisez la variable d&#39;environnement `WANDB_API_KEY` pour définir votre clé API :

```bash
export WANDB_API_KEY=your_api_key
```

Transmettez directement la clé API lors de l’initialisation de la classe `Api` :

```python
api = Api(api_key="your_api_key")
```

Ou utilisez `wandb.login()` pour authentifier la session actuelle :

```python
import wandb

wandb.login()
api = Api()
```

<div id="example-usage">
  ## Exemple d’utilisation
</div>

<div id="download-an-artifact-by-name-and-alias">
  ### Télécharger un artefact par son nom et son alias
</div>

L&#39;exemple suivant montre comment récupérer un artefact enregistré dans W&amp;B par son nom et son alias, puis télécharger son contenu.

```python
import wandb

api = wandb.Api()
artifact = api.artifact("entity/project/artifact:alias")
artifact.download()
```

<div id="download-an-artifact-from-a-registry">
  ### Télécharger un artefact depuis un registre
</div>

L&#39;exemple suivant montre comment récupérer un artefact lié à partir d’un W&amp;B Registry

```python
import wandb

REGISTRY = "<registry_name>"
COLLECTION = "<collection_name>"
VERSION = "<version>"

api = wandb.Api()
artifact_name = f"wandb-registry-{REGISTRY}/{COLLECTION}:{VERSION}"

# Récupérer l'artefact
fetched_artifact = api.artifact(name = artifact_name)

# Télécharger l'artefact. Renvoie le chemin vers le contenu téléchargé
downloaded_path = fetched_artifact.download()
```

<div id="query-wb-registry">
  ### Interroger W&amp;B Registry
</div>

Utilisez des filtres de type Mongo pour interroger W&amp;B Registries, les collections et les Artifacts. L&#39;exemple suivant montre comment filtrer les collections par leur nom à l&#39;aide d&#39;une expression régulière.

```python
import wandb

# Initialiser l'API wandb
api = wandb.Api()

# Filtrer toutes les collections, indépendamment du registre, qui
# contiennent la chaîne `yolo` dans le nom de la collection
collection_filters = {
    "name": {"$regex": "yolo"}
}

# Renvoie un itérable de toutes les collections correspondant aux filtres
collections = api.registries().collections(filter=collection_filters)
```

pour plus d’informations sur la manière d’interroger un registre, une collection ou un artefact, voir [Trouver des éléments du registre](/fr/models/registry/search_registry).
