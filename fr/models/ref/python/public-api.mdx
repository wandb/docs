L’API publique W&amp;B fournit un accès programmatique pour effectuer des requêtes, exporter et mettre à jour les données stockées dans W&amp;B. Utilisez cette API pour l’analyse a posteriori, l’export de données et la gestion programmatique des runs, des artefacts et des balayages. Alors que le SDK principal gère la journalisation en temps réel pendant l’entraînement, l’API publique vous permet de récupérer des données historiques, de mettre à jour les métadonnées, de gérer des artefacts et d’analyser des expériences terminées. La classe principale `Api` constitue le point d’entrée de la plupart des fonctionnalités.

<div id="authentication">
  ## Authentification
</div>

L’API publique utilise le même mécanisme d’authentification que le SDK Python. Vous pouvez vous authentifier de plusieurs manières :

Utilisez la variable d’environnement `WANDB_API_KEY` pour définir votre clé API :

```bash
export WANDB_API_KEY=your_api_key
```

Transmettez directement la clé API lors de l’initialisation de la classe `Api` :

```python
api = Api(api_key="your_api_key")
```

Ou utilisez `wandb.login()` pour vous authentifier pour la session en cours :

```python
import wandb

wandb.login()
api = Api()
```

<div id="example-usage">
  ## Exemple d’utilisation
</div>

<div id="download-an-artifact-by-name-and-alias">
  ### Télécharger un artefact par son nom et son alias
</div>

L&#39;exemple suivant montre comment récupérer un artefact enregistré sur W&amp;B à partir de son nom et de son alias, puis en télécharger le contenu.

```python
import wandb

api = wandb.Api()
artifact = api.artifact("entity/project/artifact:alias")
artifact.download()
```

<div id="download-an-artifact-from-a-registry">
  ### Télécharger un artefact depuis un registre
</div>

L’exemple suivant montre comment récupérer un artefact lié à partir d’un registre W&amp;B

```python
import wandb

REGISTRY = "<registry_name>"
COLLECTION = "<collection_name>"
VERSION = "<version>"

api = wandb.Api()
artifact_name = f"wandb-registry-{REGISTRY}/{COLLECTION}:{VERSION}"

# Récupérer l'artefact
fetched_artifact = api.artifact(name = artifact_name)

# Télécharger l'artefact. Renvoie le chemin vers le contenu téléchargé
downloaded_path = fetched_artifact.download()
```

<div id="query-wb-registry">
  ### Effectuer une requête sur le registre W&amp;B
</div>

Utilisez des filtres de type MongoDB pour effectuer des requêtes sur les registres, collections et Artifacts de W&amp;B. L&#39;exemple suivant montre comment filtrer des collections par nom à l&#39;aide d&#39;une expression régulière.

```python
import wandb

# Initialiser l'API wandb
api = wandb.Api()

# Filtrer toutes les collections, quel que soit le registre, qui
# contiennent la chaîne `yolo` dans le nom de la collection
collection_filters = {
    "name": {"$regex": "yolo"}
}

# Renvoie un itérable de toutes les collections correspondant aux filtres
collections = api.registries().collections(filter=collection_filters)
```

Pour en savoir plus sur la façon d’effectuer une requête sur un registre, une collection ou un artefact, voir [Trouver des éléments du registre](/fr/models/registry/search_registry).
