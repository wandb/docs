## <kbd>classe</kbd> `Project`

Un projet est un espace de noms pour les runs.

### <kbd>méthode</kbd> `Project.__init__`

```python
__init__(
    client: 'RetryingClient',
    entity: 'str',
    project: 'str',
    attrs: 'Mapping[str, Any]'
) → Project
```

**Arguments :**

* `client` :  Instance du client de l’API W&amp;B.
* `name` (str) :  Le nom du projet.
* `entity` (str) :  Le nom de l’entité propriétaire du projet.

Un projet unique associé à une entité.

**Arguments :**

* `client` :  Le client de l’API utilisé pour interroger W&amp;B.
* `entity` :  L’entité propriétaire du projet.
* `project` :  Le nom du projet à interroger.
* `attrs` :  Les attributs du projet.

### <kbd>méthode</kbd> `Project.collections`

```python
collections(
    filters: 'Mapping[str, Any] | None' = None,
    order: 'str | None' = None,
    per_page: 'int' = 50
) → public.ProjectArtifactCollections
```

Renvoie toutes les collections d’artefacts associées à ce projet.

**Arguments :**

* `filters` :  Mappage facultatif des filtres à appliquer à la requête.
* `order` :  Chaîne facultative permettant de spécifier l’ordre des résultats. Si vous faites précéder `order` d’un `+`, l’ordre est croissant (par défaut). Si vous faites précéder `order` d’un `-`, l’ordre est décroissant.
* `per_page` :  Nombre de collections d’artefacts à récupérer par page. La valeur par défaut est 50.

***


### <kbd>méthode</kbd> `Project.sweeps`

```python
sweeps(per_page: 'int' = 50) → Sweeps
```

Renvoie une collection paginée de sweeps dans ce projet.

**Arguments :**

* `per_page`:  Le nombre de sweeps à récupérer à chaque requête adressée à l’API.

**Retourne :**
Un objet `Sweeps`, qui est une collection itérable d’objets `Sweep`.
