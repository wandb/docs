## <kbd>class</kbd> `ArtifactType`

Un objet artefact qui correspond à une requête basée sur le type spécifié.

**Arguments :**

* `client`:  L&#39;instance client à utiliser pour interroger W&amp;B.
* `entity`:  L&#39;entité (utilisateur ou équipe) propriétaire du projet.
* `project`:  Le nom du projet dans lequel rechercher des types d&#39;artefact.
* `type_name`:  Le nom du type d&#39;artefact.
* `attrs`:  Attributs facultatifs pour initialiser l&#39;ArtifactType.  S&#39;ils sont omis, l&#39;objet chargera ses attributs depuis W&amp;B lors de l&#39;initialisation. 

### <kbd>propriété</kbd> ArtifactType.id

L’identifiant unique du type d’artefact.

**Retourne :**

* `str` : La valeur de la propriété `id`.

***

### <kbd>propriété</kbd> ArtifactType.name

Le nom du type d&#39;artefact.

**Retourne :**

* `str` : La valeur de la propriété nom.

***

### <kbd>méthode</kbd> `ArtifactType.collection`

```python
collection(name: 'str') → ArtifactCollection
```

Obtenir une collection d’artifacts spécifique par son nom.

**Arguments :**

* `name` (str):  Le nom de la collection d’artifacts à récupérer.

***


### <kbd>méthode</kbd> `ArtifactType.collections`

```python
collections(
    filters: 'Mapping[str, Any] | None' = None,
    order: 'str | None' = None,
    per_page: 'int' = 50
) → ArtifactCollections
```

Obtenir toutes les collections d’artifacts associées à ce type d’artefact.

**Arguments :**

* `filters` (dict):  Mappage facultatif des filtres à appliquer à la requête.
* `order` (str):  Chaîne facultative permettant de spécifier l’ordre des résultats. Si `order` est précédé d’un `+`, l’ordre est croissant (par défaut). Si `order` est précédé d’un `-`, l’ordre est décroissant. Par défaut, les résultats sont triés par ID de collection dans l’ordre décroissant.
* `per_page` (int):  Nombre de collections d’artifacts à récupérer par page. La valeur par défaut est 50.

***
