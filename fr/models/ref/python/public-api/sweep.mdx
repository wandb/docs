<div id="attributes">
  ## Attributs
</div>

* `runs` (`Runs`) : Liste des runs
* `id` (`str`) : ID du balayage
* `project` (`str`) : Nom du projet auquel le balayage appartient
* `config` (`dict`) : Dictionnaire contenant la configuration du balayage
* `state` (`str`) : État du balayage. Peut être &quot;Finished&quot;, &quot;Failed&quot;, &quot;Crashed&quot; ou &quot;Running&quot;.
* `expected_run_count` (`int`) : Nombre de runs attendus pour le balayage

<div id="properties">
  ## Propriétés
</div>

### <kbd>propriété</kbd> config

Configuration du balayage utilisée pour ce balayage.

### <kbd>propriété</kbd> entité

L’entité associée au balayage.

### <kbd>propriété</kbd> expected_run_count

Retourne le nombre de runs attendus dans le balayage, ou None si le nombre de runs est infini.

### <kbd>propriété</kbd> name

Le nom du balayage.

Renvoie le premier nom disponible selon l’ordre de priorité suivant :

1. Nom d’affichage modifié par l’utilisateur
2. Nom défini lors de la création
3. ID du balayage

### <kbd>propriété</kbd> order

Renvoie la clé d’ordre du balayage.

### <kbd>propriété</kbd> path

Renvoie le chemin du projet.

Le chemin est une liste qui contient l’entité, le nom du projet et l’ID du balayage.

### <kbd>propriété</kbd> url

L’URL du balayage.

L’URL du balayage est générée à partir de l’entité, du projet, du terme « sweeps » et de `ID du balayage.run_id`. Pour les utilisateurs SaaS, elle se présente sous la forme `https://wandb.ai/entity/project/sweeps/sweeps_ID`.

### <kbd>propriété</kbd> username

Obsolète. Utilisez `Sweep.entity` à la place.
