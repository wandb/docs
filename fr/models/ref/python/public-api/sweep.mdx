## <kbd>class</kbd> `Sweep`

Ensemble des runs associés au balayage.

**Attributs :**

* `runs` (Runs):  Liste des runs
* `id` (str):  ID du sweep
* `project` (str):  Nom du projet auquel appartient le balayage
* `config` (dict):  Dictionnaire contenant la configuration du balayage
* `state` (str):  État du balayage. Peut être « Finished », « Failed »,  « Planté » ou « Running ».
* `expected_run_count` (int):  Nombre de runs attendus pour le balayage

### <kbd>propriété</kbd> Sweep.config

La configuration utilisée pour ce balayage.

***

L’entité associée au balayage.

**Retourne :**

* `str` : La valeur de propriété d’entité.

***

### <kbd>propriété</kbd> Sweep.expected_run_count

Renvoie le nombre de runs attendus pour le balayage, ou `None` si le nombre de runs est infini.

**Retourne :**

* `int | None` : La valeur de propriété `expected_run_count`.

***

### <kbd>propriété</kbd> Sweep.name

Le nom du balayage.

Renvoie le premier nom disponible selon l’ordre de priorité suivant :

1. Display name modifié par l’utilisateur 2. Nom configuré à la date de création 3. ID du sweep

***

### <kbd>propriété</kbd> Sweep.order

Renvoie la clé d’ordre du balayage.

***

### <kbd>propriété</kbd> Sweep.path

Renvoie le chemin du projet.

Ce chemin est une liste contenant l’entité, le nom du projet et l’ID du sweep.

***

### <kbd>propriété</kbd> Sweep.url

L&#39;URL du balayage.

L&#39;URL du balayage est générée à partir de l&#39;entité, du projet, du terme « sweeps » et de l&#39;ID du sweep (`run_id`). Pour les utilisateurs SaaS, elle se présente sous la forme `https://wandb.ai/entity/project/sweeps/sweeps_ID`.

***

### <kbd>propriété</kbd> Sweep.username

Obsolète. Utilisez plutôt `Sweep.entity`.

**Retourne :**

* `str` : La valeur de la propriété `username`.
