## <kbd>classe</kbd> `Run`

Un seul run associé à une entité et à un projet.

### <kbd>méthode</kbd> `Run.__init__`

```python
__init__(
    client: 'RetryingClient',
    entity: 'str',
    project: 'str',
    run_id: 'str',
    attrs: 'Mapping | None' = None,
    include_sweeps: 'bool' = True,
    lazy: 'bool' = True,
    service_api: 'ServiceApi | None' = None
)
```

**Arguments :**

* `client`:  Le client d’API W&amp;B.
* `entity`:  L’entité associée au run.
* `project`:  Le projet associé au run.
* `run_id`:  L’identifiant unique du run.
* `attrs`:  Les attributs du run.
* `include_sweeps`:  Indique s’il faut inclure les Sweeps dans le run.

**Attributs :**

* `tags` ([str]):  une liste de tags associés au run
* `url` (str):  l’URL de ce run
* `id` (str):  identifiant unique du run (valeurs par défaut à huit caractères)
* `name` (str):  le nom du run
* `state` (str):  l’un des états suivants : running, finished, planté, killed, preempting, preempted
* `config` (dict):  un dict d’hyperparamètres associé au run
* `created_at` (str):  Horodatage ISO du démarrage du run
* `system_metrics` (dict):  les dernières métriques système enregistrées pour le run
* `summary` (dict):  Une propriété modifiable de type dict qui contient la synthèse actuelle. L’appel de update persistera toutes les modifications.
* `project` (str):  le projet associé au run
* `entity` (str):  le nom de l’entité associée au run
* `project_internal_id` (int):  l’ID interne du projet
* `user` (str):  le nom de l’utilisateur qui a créé le run
* `path` (str):  Identifiant unique [entity]/[project]/[run&#95;id]
* `notes` (str):  Notes sur le run
* `read_only` (boolean):  Indique si le run est modifiable
* `history_keys` (str):  Clés de métrique d’historique enregistrées avec `wandb.Run.log({"key": "value"})`
* `metadata` (str):  Métadonnées sur le run provenant de wandb-metadata.json

Initialisez un objet Run.

Run est toujours initialisé en appelant api.runs() où api est une instance de wandb.Api.

### <kbd>méthode de classe</kbd> `Run.create`

```python
create(
    api: 'public.Api',
    run_id: 'str | None' = None,
    project: 'str | None' = None,
    entity: 'str | None' = None,
    state: "Literal['running', 'pending']" = 'running'
) → Self
```

Créer un run pour le projet donné.

Dans la plupart des cas d’usage, utilisez `wandb.init()`. `wandb.init()` offre une logique plus robuste pour créer et mettre à jour des runs. `wandb.apis.public.Run.create` est destiné à des scénarios spécifiques, comme la création de runs à l’état « pending » pour des jobs qui risquent de ne pas pouvoir être planifiés (par exemple, dans un cluster Kubernetes avec un nombre insuffisant de GPU ou une forte contention). Ces runs en attente peuvent ensuite être repris et suivis par W&amp;B.

Les runs créés avec cette méthode offrent des fonctionnalités limitées. L’appel à `update()` sur un run créé de cette manière peut ne pas fonctionner comme prévu.

**Arguments :**

* `api` :  L’instance de l’API W&amp;B.
* `run_id` :  ID du run facultatif. S’il n’est pas fourni, un ID aléatoire sera généré.
* `project` :  Nom du projet facultatif. Par défaut, le projet défini dans les paramètres de l’API ou « uncategorized ».
* `entity` :  Nom de l’entité facultatif (utilisateur ou équipe).
* `state` :  État initial du run. Utilisez « pending » pour les runs qui seront repris plus tard, ou « running » pour une exécution immédiate.

**Retourne :**
Un objet Run représentant le run créé.

**Exemple :**
Création d’un run en attente pour une exécution ultérieure

```python
import wandb

api = wandb.Api()

run_name = "my-pending-run"

run = Run.create(
    api=api,
    project="project",
    entity="entity",
    state="pending",
    run_id=run_name,
)
```

### <kbd>méthode</kbd> `Run.files`

```python
files(
    names: 'list[str] | None' = None,
    pattern: 'str | None' = None,
    per_page: 'int' = 50

```

Renvoie un objet `Files` pour tous les fichiers du run qui correspondent aux critères indiqués.

Vous pouvez spécifier une liste de noms de fichiers exacts à rechercher, ou un motif de correspondance. Si les deux sont fournis, le motif sera ignoré.

Si `names` et `pattern` sont tous les deux fournis, une ValueError sera levée.
* `per_page` (int) :  nombre de résultats par page.

**Retourne :**
Un objet `Files`, qui est un itérateur sur des objets `File`.

### <kbd>méthode</kbd> `Run.logged_artifacts`

```python
logged_artifacts(per_page: 'int' = 100) → public.RunArtifacts
```

Récupère tous les artefacts enregistrés par ce run.

Récupère tous les artefacts de sortie qui ont été enregistrés pendant le run. Renvoie un résultat paginé qui peut être parcouru par itération ou rassemblé dans une seule liste.

**Arguments :**

* `per_page`:  Nombre d&#39;artefacts à récupérer par requête API.

**Retourne :**
Une collection itérable de tous les objets Artifact enregistrés comme sorties pendant ce run.

**Exemple :**

```python
import wandb
import tempfile

with tempfile.NamedTemporaryFile(mode="w", delete=False, suffix=".txt") as tmp:
    tmp.write("This is a test artifact")
    tmp_path = tmp.name
run = wandb.init(project="artifact-example")
artifact = wandb.Artifact("test_artifact", type="dataset")
artifact.add_file(tmp_path)
run.log_artifact(artifact)
run.finish()

api = wandb.Api()

finished_run = api.run(f"{run.entity}/{run.project}/{run.id}")

for logged_artifact in finished_run.logged_artifacts():
    print(logged_artifact.name)
```
