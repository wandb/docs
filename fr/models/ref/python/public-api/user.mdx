## <kbd>class</kbd> `User`

Un utilisateur sur une instance W&amp;B.

Cette classe permet de gérer les clés API d’un utilisateur et d’accéder à des informations telles que ses appartenances à des équipes. La méthode de classe `create` peut être utilisée pour créer un nouvel utilisateur.

**Arguments :**

* `client`:  Le client GraphQL à utiliser pour les opérations réseau.
* `attrs`:  Un sous-ensemble du type `User` dans le schéma GraphQL. 

### <kbd>propriété</kbd> User.api_keys

Noms des clés API de l’utilisateur.

Cette propriété renvoie les noms des clés API, et *non* le secret associé à la clé. Le nom de la clé ne peut pas être utilisé comme clé API.

La liste est vide si l’utilisateur n’a pas de clés API ou si les clés API n’ont pas été chargées.

**Retourne :**

* `list[str]`: La valeur de propriété api&#95;keys.

***

### <kbd>propriété</kbd> User.teams

Noms des équipes auxquelles l’utilisateur appartient.

Il s’agit d’une liste vide si l’utilisateur n’appartient à aucune équipe ou si les données des équipes n’ont pas été chargées.

**Retourne :**

* `list[str]`: La valeur de propriété teams.

***

### <kbd>propriété</kbd> User.user_api

Une instance de `wandb.Api` qui utilise les identifiants de l’utilisateur.

**Retourne :**

* `Api | None` : La valeur de propriété `user_api`.

***

### <kbd>méthode</kbd> `User.delete_api_key`

```python
delete_api_key(api_key: 'str') → bool
```

Supprime la clé API d’un utilisateur.

Seul le propriétaire de la clé ou un administrateur peut la supprimer.

**Arguments :**

* `api_key`:  Le nom de la clé API à supprimer. Utilisez l’un des noms renvoyés par la propriété `api_keys`.

**Retourne :**
True en cas de succès, false en cas d’échec.

***


### <kbd>méthode</kbd> `User.generate_api_key`

```python
generate_api_key(description: 'str | None' = None) → str | None
```

Génère une nouvelle clé API.

**Arguments :**

* `description` :  Une description de la nouvelle clé API. Elle peut être utilisée pour identifier l’usage de la clé API.

**Retourne :**
La clé API générée (le secret complet, et pas seulement son nom), ou `None` en cas d’échec.
