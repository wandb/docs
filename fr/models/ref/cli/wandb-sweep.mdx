Initialisez un balayage d’hyperparamètres. Recherchez les hyperparamètres qui permettent d’optimiser une fonction de coût d’un modèle de machine learning en testant différentes combinaisons.


<div id="usage">
  ## Utilisation
</div>

```bash
wandb sweep CONFIG_YAML_OR_SWEEP_ID [OPTIONS]
```


<div id="arguments">
  ## Arguments
</div>

| Argument                  | Description                   | Requis |
| :------------------------ | :---------------------------- | :----- |
| `CONFIG_YAML_OR_SWEEP_ID` | Aucune description disponible | Oui    |

<div id="options">
  ## Options
</div>

| Option              | Description                                                                                                                                                                                                                                                                                                        |
| :------------------ | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--project`, `-p`   | Le nom du projet auquel sont envoyés les runs W&amp;B créés à partir du balayage. Si aucun projet n&#39;est spécifié, le run est envoyé à un projet nommé Uncategorized.                                                                                                                                           |
| `--entity`, `-e`    | Le nom d&#39;utilisateur ou d&#39;équipe auquel vous souhaitez envoyer les runs W&amp;B créés par le balayage. Assurez-vous que l&#39;entité indiquée existe déjà. Si vous ne spécifiez pas d&#39;entité, le run sera envoyé à votre entité par défaut, qui correspond généralement à votre nom d&#39;utilisateur. |
| `--controller`      | Exécuter le contrôleur local (par défaut : False)                                                                                                                                                                                                                                                                  |
| `--verbose`         | Afficher une sortie détaillée (par défaut : False)                                                                                                                                                                                                                                                                 |
| `--name`            | Le nom du balayage. L&#39;ID du balayage est utilisé si aucun nom n&#39;est spécifié.                                                                                                                                                                                                                              |
| `--program`         | Définir le programme du balayage                                                                                                                                                                                                                                                                                   |
| `--update`          | Mettre à jour le balayage en attente                                                                                                                                                                                                                                                                               |
| `--stop`            | Terminer un balayage pour arrêter le lancement de nouveaux runs et laisser se terminer les runs en cours. (par défaut : False)                                                                                                                                                                                     |
| `--cancel`          | Annuler un balayage pour arrêter tous les runs en cours et empêcher le lancement de nouveaux runs. (par défaut : False)                                                                                                                                                                                            |
| `--pause`           | Mettre en pause un balayage pour arrêter temporairement le lancement de nouveaux runs. (par défaut : False)                                                                                                                                                                                                        |
| `--resume`          | Reprendre un balayage pour continuer à lancer de nouveaux runs. (par défaut : False)                                                                                                                                                                                                                               |
| `--prior_run`, `-R` | ID d&#39;un run existant à ajouter à ce balayage                                                                                                                                                                                                                                                                   |
