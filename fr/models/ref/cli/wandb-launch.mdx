Lancer ou mettre en file d&#39;attente un job W&amp;B. Voir https://wandb.me/launch


<div id="usage">
  ## Utilisation
</div>

```bash
wandb launch [OPTIONS]
```


<div id="options">
  ## Options
</div>

| Option                  | Description                                                                                                                                                                                                                                                                                                                                                                                                                                        |
| :---------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--uri`, `-u`           | Chemin local ou URI de dépôt git à lancer. Si cette option est fournie, cette commande crée un job à partir de l&#39;URI spécifiée.                                                                                                                                                                                                                                                                                                                |
| `--job`, `-j`           | Nom du job à lancer. Si cette option est fournie, Launch ne nécessite pas d&#39;URI.                                                                                                                                                                                                                                                                                                                                                               |
| `--entry-point`, `-E`   | Point d&#39;entrée dans le projet. [par défaut : main]. Si le point d&#39;entrée est introuvable, tente d&#39;exécuter comme script le fichier du projet portant le nom indiqué, en utilisant &#39;python&#39; pour les fichiers .py et le shell par défaut (spécifié par la variable d&#39;environnement $SHELL) pour les fichiers .sh. Si cette option est fournie, elle remplace la valeur entrypoint définie dans un fichier de configuration. |
| `--build-context`       | Chemin du contexte de build dans le code source. Par défaut, la racine du code source est utilisée. Compatible uniquement avec -u.                                                                                                                                                                                                                                                                                                                 |
| `--name`                | Nom du run sous lequel lancer le run. S&#39;il n&#39;est pas spécifié, un nom de run aléatoire sera utilisé. Si cette option est fournie, elle remplace le nom défini dans un fichier de configuration.                                                                                                                                                                                                                                            |
| `--entity`, `-e`        | Nom de l&#39;entité cible à laquelle le nouveau run sera envoyé. Par défaut, utilise l&#39;entité définie dans le dossier local wandb/settings. Si cette option est fournie, elle remplace la valeur d&#39;entité définie dans un fichier de configuration.                                                                                                                                                                                        |
| `--project`, `-p`       | Nom du projet cible auquel le nouveau run sera envoyé. Par défaut, utilise le nom du projet indiqué par l&#39;URI source ou, pour les Runs GitHub, le nom du dépôt git. Si cette option est fournie, elle remplace la valeur du projet définie dans un fichier de configuration.                                                                                                                                                                   |
| `--resource`, `-r`      | Ressource d&#39;exécution à utiliser pour le run. Valeurs prises en charge : &#39;local-process&#39;, &#39;local-container&#39;, &#39;kubernetes&#39;, &#39;sagemaker&#39;, &#39;gcp-vertex&#39;. Ce paramètre est désormais requis lors d&#39;un envoi vers une file d&#39;attente sans configuration de ressource. Si cette option est fournie, elle remplace la valeur de ressource définie dans un fichier de configuration.                   |
| `--docker-image`, `-d`  | Image Docker spécifique à utiliser. Au format nom:tag. Si cette option est fournie, elle remplace la valeur de l&#39;image Docker définie dans un fichier de configuration.                                                                                                                                                                                                                                                                        |
| `--base-image`, `-B`    | Image Docker dans laquelle exécuter le code du job. Incompatible avec --docker-image.                                                                                                                                                                                                                                                                                                                                                              |
| `--config`, `-c`        | Chemin vers un fichier JSON (doit se terminer par &#39;.json&#39;) ou chaîne JSON transmise comme configuration de Launch. Définit la manière dont le run lancé sera configuré.                                                                                                                                                                                                                                                                    |
| `--set-var`, `-v`       | Définit les valeurs des variables de modèle pour les files d&#39;attente avec liste d&#39;autorisation activée, sous forme de paires clé-valeur, par exemple `--set-var key1=value1 --set-var key2=value2`                                                                                                                                                                                                                                         |
| `--queue`, `-q`         | Nom de la file d&#39;attente de run vers laquelle envoyer. Si rien n&#39;est fourni, lance directement un run unique. Si l&#39;option est fournie sans argument (`--queue`), utilise par défaut la file d&#39;attente &#39;default&#39;. Sinon, si un nom est fourni, la file d&#39;attente de run indiquée doit exister dans le projet et l&#39;entité fournis.                                                                                   |
| `--async`               | Indicateur permettant d&#39;exécuter le job de manière asynchrone. La valeur par défaut est false, c&#39;est-à-dire que, sauf si --async est défini, wandb launch attend la fin du job. Cette option est incompatible avec --queue ; les options asynchrones lors d&#39;une exécution avec un agent doivent être définies sur wandb launch-agent. (par défaut : False)                                                                             |
| `--resource-args`, `-R` | Chemin vers un fichier JSON (doit se terminer par &#39;.json&#39;) ou chaîne JSON transmise comme arguments de ressource à la ressource de calcul. Le contenu exact à fournir varie selon le backend d&#39;exécution. Voir la documentation pour la structure de ce fichier.                                                                                                                                                                       |
| `--dockerfile`, `-D`    | Chemin vers le Dockerfile utilisé pour build le job, relatif à la racine du job                                                                                                                                                                                                                                                                                                                                                                    |
| `--priority`, `-P`      | Lorsque --queue est fourni, définit la priorité du job. Les jobs Launch avec une priorité plus élevée sont traités en premier. L&#39;ordre, de la priorité la plus élevée à la plus faible, est : critical, high, medium, low                                                                                                                                                                                                                      |
