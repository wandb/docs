<div id="usage">
  ## Utilisation
</div>

```bash
launch [OPTIONS]
```

<div id="description">
  ## Description
</div>

Launch ou mettre en file d&#39;attente un job W&amp;B. Voir [https://wandb.me/launch](https://wandb.me/launch)

<div id="options">
  ## Options
</div>

| Flag                  | Type      | Description                                                                                                                                                                                                                                                                                                                                                                                                                                                            |
| --------------------- | --------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--uri, -u`           | STR       | Chemin local ou URI d&#39;un dépôt Git à lancer. Si elle est fournie, cette commande crée un job à partir de l&#39;URI spécifiée. **Par défaut** : None                                                                                                                                                                                                                                                                                                                |
| `--job, -j`           | STR       | Nom du job à lancer. Si cet argument est fourni, Launch ne nécessite pas d&#39;URI. **Par défaut** : None                                                                                                                                                                                                                                                                                                                                                              |
| `--entry-point, -E`   | STR       | Point d&#39;entrée dans le projet. [default: main]. Si le point d&#39;entrée est introuvable, tente d&#39;exécuter comme script le fichier du projet portant le nom spécifié, en utilisant &#39;python&#39; pour les fichiers .py et le shell par défaut (spécifié par la variable d&#39;environnement $SHELL) pour les fichiers .sh. Si cet argument est fourni, il redéfinit la valeur `entrypoint` transmise via un fichier de configuration. **Par défaut** : None |
| `--build-context`     | STR       | Chemin vers le contexte de build dans le code source. Par défaut, la racine du code source est utilisée. Compatible uniquement avec -u. **Par défaut** : None                                                                                                                                                                                                                                                                                                          |
| `--name`              | STR       | Nom de l&#39;exécution sous lequel lancer l&#39;exécution. S&#39;il n&#39;est pas spécifié, un nom d&#39;exécution aléatoire est utilisé. Si cet argument est fourni, il redéfinit le nom transmis via un fichier de configuration. **Par défaut** : None                                                                                                                                                                                                              |
| `--entity, -e`        | STR       | Nom de l&#39;entité cible à laquelle la nouvelle exécution sera envoyée. Par défaut, utilise l&#39;entité définie dans le dossier local wandb/settings. Si cet argument est fourni, il redéfinit la valeur `entity` transmise via un fichier de configuration. **Par défaut** : None                                                                                                                                                                                   |
| `--project, -p`       | STR       | Nom du projet cible auquel la nouvelle exécution sera envoyée. Par défaut, utilise le nom du projet fourni par l&#39;URI source ou, pour les exécutions GitHub, le nom du dépôt Git. Si cet argument est fourni, il redéfinit la valeur `project` transmise via un fichier de configuration. **Par défaut** : None                                                                                                                                                     |
| `--resource, -r`      | STR       | Ressource d&#39;exécution à utiliser pour l&#39;exécution. Valeurs prises en charge : &#39;local-process&#39;, &#39;local-container&#39;, &#39;kubernetes&#39;, &#39;sagemaker&#39;, &#39;gcp-vertex&#39;. Ce paramètre est désormais requis lors d&#39;un envoi vers une file d&#39;attente sans configuration de ressource. Si cet argument est fourni, il redéfinit la valeur `resource` transmise via un fichier de configuration. **Par défaut** : None           |
| `--docker-image, -d`  | STR       | Image Docker spécifique à utiliser. Au format nom:tag. Si cet argument est fourni, il redéfinit la valeur de l&#39;image Docker transmise via un fichier de configuration. **Par défaut** : None                                                                                                                                                                                                                                                                       |
| `--base-image, -B`    | STR       | Image Docker dans laquelle exécuter le code du job. Incompatible avec --docker-image. **Par défaut** : None                                                                                                                                                                                                                                                                                                                                                            |
| `--config, -c`        | STR       | Chemin vers un fichier JSON (doit se terminer par &#39;.json&#39;) ou chaîne JSON qui sera transmis comme configuration Launch. Détermine comment l&#39;exécution lancée sera configurée. **Par défaut** : None                                                                                                                                                                                                                                                        |
| `--set-var, -v`       | STR       | Définissez les valeurs des variables de modèle pour les files d&#39;attente avec liste d&#39;autorisation activée, sous forme de paires clé-valeur, par exemple `--set-var key1=value1 --set-var key2=value2` **Par défaut** : None                                                                                                                                                                                                                                    |
| `--queue, -q`         | STR       | Nom de la file d&#39;attente d&#39;exécutions vers laquelle envoyer. Si aucune n&#39;est fournie, lance directement une exécution unique. Si elle est fournie sans argument (`--queue`), la file d&#39;attente &#39;default&#39; est utilisée par défaut. Sinon, si un nom est fourni, la file d&#39;attente spécifiée doit exister dans le projet et l&#39;entité fournis. **Par défaut** : None                                                                      |
| `--async`             | BOOL Flag | Indicateur permettant d&#39;exécuter le job de manière asynchrone. La valeur par défaut est false ; autrement dit, sauf si --async est défini, wandb launch attend la fin du job. Cette option est incompatible avec --queue ; les options asynchrones lors d&#39;une exécution avec un agent doivent être définies sur wandb launch-agent. **Par défaut** : False                                                                                                     |
| `--resource-args, -R` | STR       | Chemin vers un fichier JSON (doit se terminer par &#39;.json&#39;) ou chaîne JSON qui sera transmis comme arguments de ressource à la ressource de calcul. Le contenu exact à fournir diffère selon le backend d&#39;exécution. Voir la documentation pour connaître la structure de ce fichier. **Par défaut** : None                                                                                                                                                 |
| `--dockerfile, -D`    | STR       | Chemin vers le Dockerfile utilisé pour build le job, relatif à la racine du job. **Par défaut** : None                                                                                                                                                                                                                                                                                                                                                                 |
| `--priority, -P`      | Choice    | Lorsque --queue est fourni, définit la priorité du job. Les jobs Launch ayant la priorité la plus élevée sont traités en premier. L&#39;ordre, de la priorité la plus élevée à la plus basse, est : critical, high, medium, low. **Par défaut** : None                                                                                                                                                                                                                 |
