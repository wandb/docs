<div id="usage">
  ## Utilisation
</div>

```bash
sync [OPTIONS] [PATH]...
```

<div id="description">
  ## Description
</div>

Téléversez vers le cloud des données locales existantes d&#39;un run W&amp;B.

NOTE DE MIGRATION : cette commande est progressivement réacheminée vers
`wandb beta sync`. Certaines options sont autorisées uniquement en mode hérité, et d&#39;autres
uniquement en mode bêta. Voir les descriptions des options pour plus d&#39;informations.

Synchronisez des run hors ligne ou incomplets depuis le répertoire local `wandb` vers
le serveur W&amp;B. Si PATH est fourni, synchronisez les run à cet emplacement. Si aucun
chemin n&#39;est indiqué, recherchez un répertoire ./wandb, puis un sous-répertoire
wandb/.

Exécutez la commande sans argument pour afficher une synthèse des
run synchronisés et non synchronisés sans rien téléverser.

Lorsque vous synchronisez un chemin spécifique, les fichiers d&#39;événements TensorBoard
sont inclus par défaut. Lorsque vous utilisez `--sync-all`, TensorBoard est
désactivé par défaut (utilisez `--sync-tensorboard` pour l&#39;activer).

PATH est un fichier .wandb ou un répertoire de run qui contient un fichier .wandb.
Un chemin typique ressemble à :

./wandb/run-YYYYMMDD&#95;HHMMSS-RUN&#95;ID/run-RUN&#95;ID.wandb

où run-YYYYMMDD&#95;HHMMSS-RUN&#95;ID est le répertoire du run et
run-RUN&#95;ID.wandb est le fichier .wandb. YYYYMMDD&#95;HHMMSS correspond à l&#39;horodatage
de création du run et RUN&#95;ID est l&#39;ID unique
du run.

Par exemple, pour afficher une synthèse des run locaux et de leur statut de synchronisation :

```bash
wandb sync
```

Pour synchroniser l’ID du run abcd1234 enregistré localement dans
le répertoire ./wandb/run-20170617&#95;000000-abcd1234 :

```bash
wandb sync ./wandb/run-20170617_000000-abcd1234
```

Pour synchroniser le run abcd1234 à partir du chemin local de son fichier `.wandb`
(./wandb/run-20170617&#95;000000-abcd1234/run-abcd1234.wandb):

```bash
wandb sync ./wandb/run-20170617_000000-abcd1234/run-abcd1234.wandb
```

Pour synchroniser toutes les run pas encore synchronisées dans le répertoire local wandb :

```bash
wandb sync --sync-all
```

Pour supprimer les données locales des run déjà synchronisées :

```bash
wandb sync --clean
```

Pour supprimer les run synchronisées datant de plus de 48 heures sans demander de confirmation :

```bash
wandb sync --clean --clean-old-hours 48 --clean-force
```

<div id="arguments">
  ## Arguments
</div>

| Nom    | Par défaut | Type  |
| ------ | ---------- | ----- |
| `path` | PATH       | False |

<div id="options">
  ## Options
</div>

| Flag                                 | Type      | Description                                                                                                                                                   |
| ------------------------------------ | --------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--yes`                              | BOOL Flag | (bêta uniquement) Ne demande pas de confirmation. **Par défaut**: False                                                                                       |
| `--id`                               | STR       | Téléverse vers un ID du run existant. **Par défaut**: None                                                                                                    |
| `--project, -p`                      | STR       | Définit le projet vers lequel téléverser le run. **Par défaut**: None                                                                                         |
| `--entity, -e`                       | STR       | Définit l&#39;entité à laquelle associer le projet. **Par défaut**: None                                                                                      |
| `--job_type`                         | STR       | Définit le type de job pour regrouper les runs associés. **Par défaut**: None                                                                                 |
| `--sync-tensorboard`                 | BOOL Flag | (hérité uniquement) Synchronise les fichiers tfevent de TensorBoard. Activé par défaut pour certains chemins, désactivé avec --sync-all. **Par défaut**: None |
| `--include-globs`                    | STR       | (hérité uniquement) Inclut uniquement les runs correspondant à ces motifs glob (séparés par des virgules). **Par défaut**: None                               |
| `--exclude-globs`                    | STR       | (hérité uniquement) Exclut les runs correspondant à ces motifs glob (séparés par des virgules). **Par défaut**: None                                          |
| `--include-online, --no-skip-online` | BOOL Flag | Inclut les runs créés en mode en ligne. **Par défaut**: None                                                                                                  |
| `--include-offline`                  | BOOL Flag | (hérité uniquement) Inclut les runs créés en mode hors ligne. **Par défaut**: None                                                                            |
| `--include-synced, --no-skip-synced` | BOOL Flag | Inclut les runs déjà synchronisés. **Par défaut**: None                                                                                                       |
| `--mark-synced`                      | BOOL Flag | (hérité uniquement) Marque les runs comme synchronisés après le téléversement. **Par défaut**: True                                                           |
| `--sync-all`                         | BOOL Flag | (hérité uniquement) Synchronise tous les runs non synchronisés dans le répertoire local wandb. **Par défaut**: False                                          |
| `--clean`                            | BOOL Flag | (hérité uniquement) Supprime les données locales des runs déjà synchronisés. **Par défaut**: False                                                            |
| `--clean-old-hours`                  | INT       | Supprime uniquement les runs synchronisés datant de plus de ce nombre d&#39;heures (à utiliser avec --clean). **Par défaut**: 24                              |
| `--clean-force`                      | BOOL Flag | Ignore la demande de confirmation si --clean est spécifié. **Par défaut**: False                                                                              |
| `--show`                             | INT       | Définit le nombre de runs à afficher dans la synthèse. **Par défaut**: 5                                                                                      |
| `--append`                           | BOOL Flag | (hérité uniquement) Ajoute des données à un run existant au lieu de créer un nouveau run. **Par défaut**: False                                               |
| `--skip-console`                     | BOOL Flag | (hérité uniquement) Ignore le téléversement des journaux de console. **Par défaut**: False                                                                    |
| `--replace-tags`                     | STR       | Renomme les tags pendant la synchronisation. Utilisez des paires &#39;ancien=nouveau&#39; séparées par des virgules. **Par défaut**: None                     |
| `--legacy`                           | BOOL Flag | Force le comportement hérité au lieu de rediriger vers `wandb beta sync`. **Par défaut**: False                                                               |
