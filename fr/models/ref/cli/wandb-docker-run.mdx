<div id="usage">
  ## Utilisation
</div>

```bash
docker-run [OPTIONS] [DOCKER_RUN_ARGS]...
```

<div id="description">
  ## Description
</div>

Encapsule `docker run` et injecte automatiquement les variables d&#39;environnement W&amp;B.

Transmet tous les arguments à `docker run` tout en injectant :

* WANDB&#95;API&#95;KEY : injecte la clé API actuelle si vous êtes connecté, afin que le
  conteneur puisse s&#39;authentifier auprès de W&amp;B.

  * WANDB&#95;DOCKER : injecte l&#39;ID d&#39;image résolu si l&#39;image peut être détectée
    à partir des arguments, afin que W&amp;B puisse suivre quelle image Docker a produit l&#39;exécution.

Ajoute `--runtime nvidia` automatiquement si `nvidia-docker` est détecté
sur l&#39;hôte et que `--runtime` n&#39;est pas déjà défini.

Par exemple, pour exécuter le script Python train.py dans le conteneur &quot;my-image&quot; :

```bash
wandb docker-run my-image python train.py
```
