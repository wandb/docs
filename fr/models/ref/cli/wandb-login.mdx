<div id="usage">
  ## Utilisation
</div>

```bash
login [OPTIONS] [KEY]...
```

<div id="description">
  ## Description
</div>

Authentifiez votre machine auprès de W&amp;B.

Enregistrez localement une clé API pour vous authentifier auprès des services W&amp;B.
Par défaut, les identifiants sont stockés sans vérification côté serveur.

Si aucune clé API n’est fournie en argument, la commande recherche les
identifiants dans l’ordre suivant :

1. La variable d’environnement WANDB&#95;API&#95;KEY

   2. Le paramètre api&#95;key dans un fichier de paramètres système ou d’espace de travail (utilisez
      `wandb status` pour voir quel fichier de paramètres est utilisé)

   3. Le fichier .netrc (~/.netrc, ~/&#95;netrc ou le chemin indiqué par la variable d’environnement NETRC)

   4. Une invite interactive (si un TTY est disponible)

Pour les déploiements auto-hébergés ou sur Cloud dédié, spécifiez l’URL du serveur
avec `--host`, ou définissez la variable d’environnement WANDB&#95;BASE&#95;URL.

Par exemple, pour vous connecter de manière interactive (une invite vous demande la clé API) :

```bash
wandb login
```

Pour vous connecter à l’aide d’une clé API explicite (WANDB&#95;API&#95;KEY&#95;EXAMPLE) :

```bash
wandb login WANDB_API_KEY_EXAMPLE
```

Pour vous connecter et vérifier que la clé API est valide :

```bash
wandb login --verify
```

Pour vous connecter au cloud public de W&amp;B plutôt qu’à une instance auto-hébergée configurée :

```bash
wandb login --cloud
```

Pour vous connecter à une instance W&amp;B auto-hébergée :

```bash
wandb login --host https://my-wandb-server.example.com
```

Pour forcer une nouvelle demande de connexion, même si vous êtes déjà authentifié :

```bash
wandb login --relogin
```

<div id="options">
  ## Options
</div>

| Flag                 | Type      | Description                                                                                                                                                                                                |
| -------------------- | --------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--cloud`            | BOOL Flag | Se connecter au cloud public de W&amp;B ([https://api.wandb.ai](https://api.wandb.ai)). Ne peut pas être utilisé avec --host. **Par défaut** : False                                                       |
| `--host, --base-url` | STR       | Se connecter à une instance spécifique du serveur W&amp;B via son URL (par ex. [https://my-wandb.example.com](https://my-wandb.example.com)). Ne peut pas être utilisé avec --cloud. **Par défaut** : None |
| `--relogin`          | BOOL Flag | Forcer une nouvelle demande de connexion en ignorant les identifiants existants. **Par défaut** : None                                                                                                     |
| `--verify`           | BOOL Flag | Vérifier la clé API auprès de W&amp;B après l’avoir enregistrée. Si la vérification réussit, afficher l’origine des identifiants et l’équipe par défaut. **Par défaut** : False                            |
