Exécutez un balayage W&amp;B Launch (expérimental).


<div id="usage">
  ## Utilisation
</div>

```bash
wandb launch-sweep [CONFIG] [OPTIONS]
```


<div id="arguments">
  ## Arguments
</div>

| Argument | Description                   | Requis |
| :------- | :---------------------------- | :----- |
| `CONFIG` | Aucune description disponible | Non    |

<div id="options">
  ## Options
</div>

| Option              | Description                                                                                                                                         |
| :------------------ | :-------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--queue`, `-q`     | Le nom de la file d&#39;attente dans laquelle placer le balayage                                                                                    |
| `--project`, `-p`   | Nom du projet que l&#39;agent surveillera. Si cette option est fournie, elle remplace la valeur de projet transmise via un fichier de configuration |
| `--entity`, `-e`    | L&#39;entité à utiliser. Par défaut, il s&#39;agit de l&#39;utilisateur actuellement connecté                                                       |
| `--resume_id`, `-r` | Reprendre un balayage Launch en fournissant un identifiant de balayage de 8 caractères. File d&#39;attente requise                                  |
| `--prior_run`, `-R` | Identifiant d&#39;un run existant à ajouter à ce balayage                                                                                           |
