Exécuter un balayage Launch de W&amp;B (expérimental).


<div id="usage">
  ## Utilisation
</div>

```bash
wandb launch-sweep [CONFIG] [OPTIONS]
```


<div id="arguments">
  ## Arguments
</div>

| Argument | Description                         | Requis |
| :------- | :---------------------------------- | :----- |
| `CONFIG` | Aucune description n’est disponible | Non    |

<div id="options">
  ## Options
</div>

| Option              | Description                                                                                                                                           |
| :------------------ | :---------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--queue`, `-q`     | Le nom de la file d&#39;attente vers laquelle envoyer le balayage                                                                                     |
| `--project`, `-p`   | Nom du projet que l&#39;agent surveillera. Si cette option est fournie, elle remplacera la valeur de projet transmise via un fichier de configuration |
| `--entity`, `-e`    | L&#39;entité à utiliser. Par défaut, l&#39;utilisateur actuellement connecté est utilisé                                                              |
| `--resume_id`, `-r` | Reprendre un balayage Launch en fournissant un ID de balayage de 8 caractères. File d&#39;attente requise                                             |
| `--prior_run`, `-R` | ID d&#39;un run existant à ajouter à ce balayage                                                                                                      |
