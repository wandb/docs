<div id="usage">
  ## Utilisation
</div>

```bash
controller [OPTIONS] SWEEP_ID
```

<div id="description">
  ## Description
</div>

Démarrez un contrôleur de sweep local pour un sweep d&#39;hyperparamètres W&amp;B.

Démarrez un processus local qui orchestre le sweep spécifié. Lisez la
configuration du sweep depuis W&amp;B, sélectionnez des combinaisons d&#39;hyperparamètres selon
la stratégie de recherche configurée (grille, aléatoire, bayésienne, etc.),
et distribuez les runs aux agents de sweep.

Par défaut, W&amp;B exécute les contrôleurs de sweep sur son infrastructure gérée.
Utilisez cette commande pour exécuter le contrôleur localement à la place. Par exemple, vous
pouvez utiliser cette commande pour déboguer un comportement ou travailler dans des environnements où
la connectivité est limitée.

sweep&#95;id est affiché par `wandb sweep` lorsque vous créez un sweep. Il
se compose d&#39;un identifiant unique du sweep et peut inclure le
chemin de l&#39;entité et du projet (entity/project/sweep&#95;id).

Par exemple, pour démarrer un contrôleur de sweep local pour un sweep avec l&#39;ID de sweep wbyz9876 :

```bash
wandb controller wbyz9876
```
