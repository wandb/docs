Créer un job à partir d’une source, sans run wandb.

Les jobs peuvent être de trois types : git, code ou image.

git : une source git, avec un point d’entrée situé dans le chemin ou fourni explicitement, pointant vers l’exécutable Python principal. code : un chemin vers le code, contenant un fichier requirements.txt. image : une image Docker.


<div id="usage">
  ## Utilisation
</div>

```bash
wandb job create JOB_TYPE PATH [OPTIONS]
```


<div id="arguments">
  ## Arguments
</div>

| Argument   | Description                   | Requis |
| :--------- | :---------------------------- | :----- |
| `JOB_TYPE` | Aucune description disponible | Oui    |
| `PATH`     | Aucune description disponible | Oui    |

<div id="options">
  ## Options
</div>

| Option                  | Description                                                                                                                                                                                                                                        |
| :---------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `--project`, `-p`       | Le projet dont vous voulez lister les jobs.                                                                                                                                                                                                        |
| `--entity`, `-e`        | L’entité à laquelle appartiennent les jobs                                                                                                                                                                                                         |
| `--name`, `-n`          | Nom du job                                                                                                                                                                                                                                         |
| `--description`, `-d`   | Description du job                                                                                                                                                                                                                                 |
| `--alias`, `-a`         | Alias du job                                                                                                                                                                                                                                       |
| `--entry-point`, `-E`   | Point d’entrée du script, y compris un exécutable et un fichier de point d’entrée. Requis pour les jobs de code ou de dépôt. Si `--build-context` est fourni, les chemins dans la commande du point d’entrée seront relatifs au contexte de build. |
| `--git-hash`, `-g`      | Référence de commit à utiliser comme source pour les jobs Git                                                                                                                                                                                      |
| `--runtime`, `-r`       | Environnement d’exécution Python pour exécuter le job                                                                                                                                                                                              |
| `--build-context`, `-b` | Chemin vers le contexte de build à partir de la racine du code source du job. S’il est fourni, il sert de chemin de base pour le Dockerfile et le point d’entrée.                                                                                  |
| `--base-image`, `-B`    | Image de base à utiliser pour le job. Incompatible avec les jobs d’image.                                                                                                                                                                          |
| `--dockerfile`, `-D`    | Chemin vers le Dockerfile du job. Si `--build-context` est fourni, le chemin du Dockerfile sera relatif au contexte de build.                                                                                                                      |
