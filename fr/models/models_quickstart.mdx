Découvrez quand et comment utiliser W&amp;B pour suivre, partager et gérer les artifacts de modèle dans vos flux de travail de machine learning. Cette page explique comment enregistrer des expériences, générer des rapports et accéder aux données enregistrées à l’aide de l’API W&amp;B appropriée pour chaque tâche.

Ce tutoriel utilise les éléments suivants :

* [W&amp;B Python SDK](/fr/models/ref/python) (`wandb.sdk`) : pour enregistrer et surveiller les expériences pendant l’entraînement.
* [API publique W&amp;B](/fr/models/ref/python/public-api) (`wandb.apis.public`) : pour interroger et analyser les données d’expérience enregistrées.
* [API W&amp;B Reports and Workspaces](/fr/models/ref/wandb_workspaces) (`wandb.wandb-workspaces`) : pour créer un rapport afin de résumer les résultats.

<div id="train-a-model">
  ## Entraîner un modèle
</div>

Le code suivant simule un flux de travail de machine learning de base : entraîner un modèle, consigner des métriques et enregistrer le modèle en tant qu&#39;artifact.

Utilisez le SDK Python W&amp;B (`wandb.sdk`) pour interagir avec W&amp;B pendant l&#39;entraînement. Consignez la perte à l&#39;aide de [`wandb.Run.log()`](/fr/models/ref/python/experiments/run/#method-runlog), puis enregistrez le modèle entraîné en tant qu&#39;artifact avec [`wandb.Artifact`](/fr/models/ref/python/experiments/artifact), avant d&#39;ajouter le fichier du modèle avec [`Artifact.add_file`](/fr/models/ref/python/experiments/artifact#add_file).

Les points clés à retenir du bloc de code précédent sont :

* Utilisez `wandb.Run.log()` pour consigner des métriques pendant l&#39;entraînement.
* Utilisez `wandb.Artifact` pour enregistrer des modèles (jeux de données, etc.) en tant qu&#39;artifact dans votre projet W&amp;B.

Maintenant que vous avez entraîné un modèle et l&#39;avez enregistré en tant qu&#39;artifact, vous pouvez le publier dans un registre de W&amp;B. Utilisez [`wandb.Run.use_artifact()`](/fr/models/ref/python/experiments/run/#method-runuse_artifact) pour récupérer l&#39;artifact de votre projet et le préparer à être publié dans le registre de modèles. `wandb.Run.use_artifact()` remplit deux fonctions clés :

* Récupérer l&#39;objet artifact depuis votre projet.
* Marquer l&#39;artifact comme entrée du run, ce qui garantit la reproductibilité et la traçabilité. Voir [Créer et afficher la carte de traçabilité](/fr/models/registry/lineage) pour plus de détails.

<div id="publish-the-model-to-the-wb-registry">
  ## Publier le modèle dans le registre W&amp;B
</div>

Pour partager le modèle avec d&#39;autres personnes de votre organisation, publiez-le dans une [collection](/fr/models/registry/create_collection) à l&#39;aide de `wandb.Run.link_artifact()`. Le code suivant associe l&#39;artifact à un [registre](/fr/models/registry), ce qui le rend accessible à votre équipe.

Après avoir exécuté `wandb.Run.link_artifact()`, l&#39;artifact de modèle se trouvera dans la collection `DemoModels` de votre registre. Vous pouvez ensuite consulter des informations telles que l&#39;historique des versions, la [carte de traçabilité](/fr/models/registry/lineage) et d&#39;autres [métadonnées](/fr/models/registry/registry_cards).

Pour en savoir plus sur la façon de lier des artifacts à un registre, consultez [Lier des artifacts à un registre](/fr/models/registry/link_version).

<div id="retrieve-model-artifact-from-registry-for-inference">
  ## Récupérer un artifact de modèle du registre pour l’inférence
</div>

Pour utiliser un modèle pour l’inférence, utilisez `wandb.Run.use_artifact()` afin de récupérer l’artifact publié dans le registre. Cette méthode renvoie un objet artifact, que vous pouvez ensuite télécharger dans un fichier local à l’aide de [`wandb.Artifact.download()`](/fr/models/ref/python/experiments/artifact/#method-artifactdownload).

Pour en savoir plus sur la façon de récupérer des artifacts depuis un registre, voir [Télécharger un artifact depuis un registre](/fr/models/registry/download_use_artifact).

Selon votre framework de machine learning, vous devrez peut-être recréer l’architecture du modèle avant de charger les poids. Nous vous laissons cet exercice, car cela dépend du framework et du modèle que vous utilisez.
