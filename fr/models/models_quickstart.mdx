Découvrez quand et comment utiliser W&amp;B pour suivre, partager et gérer les artefacts de modèle dans vos flux de travail de machine learning. Cette page explique comment enregistrer des expériences, générer des rapports et accéder aux données enregistrées à l’aide de l’API W&amp;B appropriée pour chaque tâche.

Ce tutoriel s’appuie sur les éléments suivants :

* [W&amp;B Python SDK](/fr/models/ref/python) (`wandb.sdk`) : pour enregistrer et surveiller les expériences pendant l’entraînement.
* [W&amp;B Public API](/fr/models/ref/python/public-api) (`wandb.apis.public`) : pour interroger et analyser les données d’expérience enregistrées.
* [W&amp;B Reports and Workspaces API](/fr/models/ref/wandb_workspaces) (`wandb.wandb-workspaces`) : pour créer un rapport résumant les résultats.

<div id="sign-up-and-create-an-api-key">

Pour authentifier votre machine auprès de W&amp;B, vous devez d’abord générer une clé API sur [wandb.ai/settings](https://wandb.ai/settings). Copiez la clé API et conservez-la en lieu sûr.

<div id="install-and-import-packages">
  ## Installer et importer des paquets
</div>

Installez la bibliothèque W&amp;B ainsi que quelques autres paquets dont vous aurez besoin pour ce guide pas à pas.

Indiquez l’entité de votre équipe dans le bloc de code suivant :


<div id="train-a-model">
  ## Entraîner un modèle
</div>

Le code suivant simule un flux de travail de base de machine learning : entraînement d’un modèle, consignation des métriques et enregistrement du modèle en tant qu’artefact.

Utilisez le SDK Python de W&amp;B (`wandb.sdk`) pour interagir avec W&amp;B pendant l’entraînement. Journalisez la perte à l’aide de [`wandb.Run.log()`](/fr/models/ref/python/experiments/run/#method-runlog), puis enregistrez le modèle entraîné en tant qu’artefact avec [`wandb.Artifact`](/fr/models/ref/python/experiments/artifact), avant d’ajouter enfin le fichier du modèle avec [`Artifact.add_file`](/fr/models/ref/python/experiments/artifact#add_file).

```python
import random # Pour simuler des données

def model(training_data: int) -> int:
    """Simulation de modèle à des fins de démonstration."""
    return training_data * 2 + random.randint(-1, 1)  

# Simuler les poids et le bruit
weights = random.random() # Initialiser des poids aléatoires
noise = random.random() / 5  # Petit bruit aléatoire pour simuler le bruit

# Hyperparamètres et configuration
config = {
    "epochs": 10,  # Nombre d'époques d'entraînement
    "learning_rate": 0.01,  # Taux d'apprentissage pour l'optimiseur
}

# Utiliser le gestionnaire de contexte pour initialiser et fermer les runs W&B
with wandb.init(project=PROJECT, entity=TEAM_ENTITY, config=config) as run:    
    # Simuler la boucle d'entraînement
    for epoch in range(config["epochs"]):
        xb = weights + noise  # Données d'entraînement en entrée simulées
        yb = weights + noise * 2  # Sortie cible simulée (double du bruit en entrée)
        
        y_pred = model(xb)  # Prédiction du modèle
        loss = (yb - y_pred) ** 2  # Perte d'erreur quadratique moyenne

        print(f"epoch={epoch}, loss={loss}")
        # Enregistrer l'époque et la perte dans W&B
        run.log({
            "epoch": epoch,
            "loss": loss,
        })

    # Nom unique pour l'artefact du modèle,
    model_artifact_name = f"model-demo"  

    # Chemin local pour enregistrer le fichier de modèle simulé
    PATH = "model.txt" 

    # Enregistrer le modèle localement
    with open(PATH, "w") as f:
        f.write(str(weights)) # Enregistrement des poids du modèle dans un fichier

    # Créer un objet artefact
    # Ajouter le modèle enregistré localement à l'objet artefact
    artifact = wandb.Artifact(name=model_artifact_name, type="model", description="My trained model")
    artifact.add_file(local_path=PATH)
    artifact.save()
```

Les principaux points à retenir du bloc de code précédent sont :

* Utilisez `wandb.Run.log()` pour consigner des métriques pendant l&#39;entraînement.
* Utilisez `wandb.Artifact` pour enregistrer des modèles (jeux de données, etc.) en tant qu&#39;artefact dans votre projet W&amp;B.

Maintenant que vous avez entraîné un modèle et l&#39;avez enregistré en tant qu&#39;artefact, vous pouvez le publier dans un registre W&amp;B. Utilisez [`wandb.Run.use_artifact()`](/fr/models/ref/python/experiments/run/#method-runuse_artifact) pour récupérer l&#39;artefact depuis votre projet et le préparer en vue de sa publication dans le registre de modèles. `wandb.Run.use_artifact()` remplit deux fonctions principales :

* Récupère l&#39;objet artefact depuis votre projet.
* Marque l&#39;artefact comme entrée du run, ce qui garantit la reproductibilité et la traçabilité. Voir [Créer et consulter la carte de traçabilité](/fr/models/registry/lineage) pour plus de détails.

<div id="publish-the-model-to-the-wb-registry">
  ## Publier le modèle dans le registre W&amp;B
</div>

Pour partager le modèle avec d’autres membres de votre organisation, publiez-le dans une [collection](/fr/models/registry/create_collection) à l’aide de `wandb.Run.link_artifact()`. Le code suivant lie l’artefact à un [registre](/fr/models/registry), ce qui le rend accessible à votre équipe.

```python
# Le nom de l'artefact spécifie la version spécifique de l'artefact dans le projet de notre équipe
artifact_name = f'{TEAM_ENTITY}/{PROJECT}/{model_artifact_name}:v0'
print("Artifact name: ", artifact_name)

REGISTRY_NAME = "Model" # Nom du registre dans W&B
COLLECTION_NAME = "DemoModels"  # Nom de la collection dans le registre

# Créer un chemin cible pour notre artefact dans le registre
target_path = f"wandb-registry-{REGISTRY_NAME}/{COLLECTION_NAME}"
print("Target path: ", target_path)

with wandb.init(entity=TEAM_ENTITY, project=PROJECT) as run:
    model_artifact = run.use_artifact(artifact_or_name=artifact_name, type="model")
    run.link_artifact(artifact=model_artifact, target_path=target_path)
```

Après avoir exécuté `wandb.Run.link_artifact()`, l’artefact de modèle se trouvera dans la collection `DemoModels` de votre registre. Vous pourrez ensuite consulter des détails tels que l’historique des versions, la [carte de traçabilité](/fr/models/registry/lineage) et d’autres [métadonnées](/fr/models/registry/registry_cards).

Pour plus d’informations sur la façon de lier des artefacts à un registre, voir [Lier des artefacts à un registre](/fr/models/registry/link_version).

<div id="retrieve-model-artifact-from-registry-for-inference">
  ## Récupérer un artefact de modèle depuis le registre pour l’inférence
</div>

Pour utiliser un modèle pour l’inférence, utilisez `wandb.Run.use_artifact()` afin de récupérer l’artefact publié à partir du registre. Cette méthode renvoie un objet artefact, que vous pouvez ensuite télécharger dans un fichier local à l’aide de [`wandb.Artifact.download()`](/fr/models/ref/python/experiments/artifact/#method-artifactdownload).

```python
REGISTRY_NAME = "Model"  # Nom du registre dans W&B
COLLECTION_NAME = "DemoModels"  # Nom de la collection dans le registre
VERSION = 0 # Version de l'artefact à récupérer

model_artifact_name = f"wandb-registry-{REGISTRY_NAME}/{COLLECTION_NAME}:v{VERSION}"
print(f"Nom de l'artefact modèle : {model_artifact_name}")

with wandb.init(entity=TEAM_ENTITY, project=PROJECT) as run:
    registry_model = run.use_artifact(artifact_or_name=model_artifact_name)
    local_model_path = registry_model.download()
```

Pour plus d’informations sur la façon de récupérer des artefacts à partir d’un registre, voir [Télécharger un artefact à partir d’un registre](/fr/models/registry/download_use_artifact).

Selon votre framework de machine learning, vous devrez peut-être recréer l’architecture du modèle avant de charger les poids. Nous laissons cela en exercice au lecteur, car cela dépend du framework et du modèle que vous utilisez.

<div id="share-your-finds-with-a-report">
  ## Partagez vos découvertes avec un rapport
</div>

<Note>
  L&#39;API W&amp;B Report and Workspace est en préversion publique.
</Note>

Créez et partagez un [rapport](/fr/models/reports) pour résumer votre travail. Pour créer un rapport par code, utilisez l&#39;[API W&amp;B Report and Workspace](/fr/models/ref/wandb_workspaces/reports).

Commencez par installer l&#39;API W&amp;B Reports :

```python
pip install wandb wandb-workspaces -qqq
```

Le bloc de code suivant crée un rapport avec plusieurs blocs, notamment du markdown, des grilles de panneaux, entre autres. Vous pouvez personnaliser le rapport en ajoutant d’autres blocs ou en modifiant le contenu des blocs existants.

L’exécution du bloc de code affiche un lien vers l’URL du rapport créé. Vous pouvez ouvrir ce lien dans votre navigateur pour afficher le rapport.

```python
import wandb_workspaces.reports.v2 as wr

experiment_summary = """This is a summary of the experiment conducted to train a simple model using W&B."""
dataset_info = """The dataset used for training consists of synthetic data generated by a simple model."""
model_info = """The model is a simple linear regression model that predicts output based on input data with some noise."""

report = wr.Report(
    project=PROJECT,
    entity=TEAM_ENTITY,
    title="My Awesome Model Training Report",
    description=experiment_summary,
    blocks= [
        wr.TableOfContents(),
        wr.H2("Experiment Summary"),
        wr.MarkdownBlock(text=experiment_summary),
        wr.H2("Dataset Information"),
        wr.MarkdownBlock(text=dataset_info),
        wr.H2("Model Information"),
        wr.MarkdownBlock(text = model_info),
        wr.PanelGrid(
            panels=[
                wr.LinePlot(title="Train Loss", x="Step", y=["loss"], title_x="Step", title_y="Loss")
                ],
            ),  
    ]

)

# Enregistrer le rapport dans W&B
report.save()
```

Pour plus d’informations sur la création d’un rapport par programmation ou de manière interactive avec l’application W&amp;B, voir [Créer un rapport](/fr/models/reports/create-a-report) dans le guide développeur de la documentation W&amp;B.

<div id="query-the-registry">
  ## Interroger le registre
</div>

Utilisez les [API publiques de W&amp;B](/fr/models/ref/python/public-api) pour interroger, analyser et gérer les données historiques de W&amp;B. Cela peut être utile pour suivre la traçabilité des artefacts, comparer différentes versions et analyser les performances des modèles au fil du temps.

Le bloc de code suivant montre comment interroger le registre de modèles pour récupérer tous les artefacts d&#39;une collection donnée. Il récupère la collection et parcourt ses versions en affichant le nom et la version de chaque artefact.

```python
import wandb

# Initialiser l'API wandb
api = wandb.Api()

# Trouver toutes les versions d'artefact contenant la chaîne `model` et 
# ayant soit le tag `text-classification`, soit un alias `latest`
registry_filters = {
    "name": {"$regex": "model"}
}

# Utiliser l'opérateur logique $or pour filtrer les versions d'artefact
version_filters = {
    "$or": [
        {"tag": "text-classification"},
        {"alias": "latest"}
    ]
}

# Renvoie un itérable de toutes les versions d'artefact correspondant aux filtres
artifacts = api.registries(filter=registry_filters).collections().versions(filter=version_filters)

# Afficher le nom, la collection, les alias, les tags et la date created_at de chaque artefact trouvé
for art in artifacts:
    print(f"artifact name: {art.name}")
    print(f"collection artifact belongs to: { art.collection.name}")
    print(f"artifact aliases: {art.aliases}")
    print(f"tags attached to artifact: {art.tags}")
    print(f"artifact created at: {art.created_at}\n")
```

Pour plus d’informations sur l’interrogation du registre, voir [Interroger les éléments du registre](/fr/models/registry/search_registry/#query-registry-items-with-mongodb-style-queries).
