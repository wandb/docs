Visualisez et journalisez des données tabulaires avec W&amp;B Tables. Une W&amp;B Table est une grille de données bidimensionnelle dans laquelle chaque colonne contient un seul type de données. Chaque ligne représente un ou plusieurs points de données journalisés dans un [run](/fr/models/runs/). W&amp;B Tables prennent en charge les types primitifs et numériques, ainsi que les listes imbriquées, les dictionnaires et les types de médias enrichis.

Une W&amp;B Table est un [type de données](/fr/models/ref/python/data-types/) spécialisé dans W&amp;B, journalisé sous forme d’objet [artifact](/fr/models/artifacts/).

Vous [créez et journalisez des objets de type tableau](#create-and-log-a-new-table) à l’aide du W&amp;B Python SDK. Lorsque vous créez un objet tableau, vous spécifiez les colonnes et les données du tableau, ainsi qu’un [mode](#table-logging-modes). Le mode détermine comment le tableau est journalisé et mis à jour pendant vos expériences de machine learning.

<Note>
  Le mode `INCREMENTAL` est pris en charge sur W&amp;B Server v0.70.0 et versions ultérieures.
</Note>

<div id="create-and-log-a-table">
  ## Créer et journaliser un tableau
</div>

1. Initialisez un nouveau run avec `wandb.init()`.
2. Créez un objet Table avec la classe [`wandb.Table`](/fr/models/ref/python/data-types/table). Spécifiez respectivement les colonnes et les données du tableau à l’aide des paramètres `columns` et `data`. Il est recommandé de définir le paramètre facultatif `log_mode` sur l’un des trois modes suivants : `IMMUTABLE` (par défaut), `MUTABLE` ou `INCREMENTAL`. Voir [Modes de journalisation des tableaux](#logging-modes) dans la section suivante pour en savoir plus.
3. Journalisez le tableau dans W&amp;B avec `run.log()`.

<div id="logging-modes">
  ## Modes de journalisation
</div>

Le paramètre `log_mode` de [`wandb.Table`](/fr/models/ref/python/data-types/table) détermine comment un tableau est journalisé et mis à jour pendant vos expériences de ML. Le paramètre `log_mode` accepte l&#39;un des trois arguments suivants : `IMMUTABLE`, `MUTABLE` et `INCREMENTAL`. Chaque mode a des implications différentes sur la façon dont un tableau est journalisé, sur la manière dont il peut être modifié et sur son rendu dans la W&amp;B App.

Ce qui suit décrit les trois modes de journalisation, leurs principales différences et le cas d&#39;utilisation courant de chacun :

| Mode          | Définition                                                                                                | Cas d&#39;utilisation                                                                                                                                                            | Avantages                                                                                                                       |
| ------------- | --------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------- |
| `IMMUTABLE`   | Une fois qu&#39;un tableau est journalisé dans W&amp;B, vous ne pouvez plus le modifier.                    | - Stocker des données tabulaires générées à la fin d&#39;un run pour une analyse ultérieure                                                                                      | - Surcharge minimale lorsqu&#39;il est journalisé à la fin d&#39;un run<br />- Toutes les lignes sont affichées dans l&#39;UI   |
| `MUTABLE`     | Après avoir journalisé un tableau dans W&amp;B, vous pouvez remplacer le tableau existant par un nouveau. | - Ajouter des colonnes ou des lignes à des tableaux existants<br />- Enrichir les résultats avec de nouvelles informations                                                       | - Capturer les modifications apportées à la Table<br />- Toutes les lignes sont affichées dans l&#39;UI                         |
| `INCREMENTAL` | Ajouter des lots de nouvelles lignes à un tableau tout au long de l&#39;expérience de machine learning.   | - Ajouter des lignes aux tableaux par lots<br /> - Tâches d&#39;entraînement de longue durée<br />- Traiter de grands Datasets par lots<br />- Surveiller les résultats en cours | - Voir les mises à jour dans l&#39;UI pendant l&#39;entraînement<br />- Possibilité de parcourir les incréments étape par étape |

Les sections suivantes présentent des exemples de code snippets pour chaque mode, ainsi que des considérations sur le moment où utiliser chacun d&#39;eux.

<div id="mutable-mode">
  ### Mode `MUTABLE`
</div>

Le mode `MUTABLE` met à jour un tableau existant en le remplaçant par un nouveau. Il est utile lorsque vous souhaitez ajouter de nouvelles colonnes et lignes à un tableau existant dans le cadre d’un processus non itératif. Dans l’interface utilisateur, le tableau est affiché avec toutes les lignes et colonnes, y compris celles ajoutées après le journal initial.

<Note>En mode `MUTABLE`, l’objet tableau est remplacé chaque fois que vous journalisez le tableau. Remplacer un tableau par un nouveau est coûteux en calcul et peut être lent pour les tableaux volumineux.</Note>

L’exemple suivant montre comment créer un tableau en mode `MUTABLE`, le journaliser, puis y ajouter de nouvelles colonnes. L’objet tableau est journalisé trois fois : une fois avec les données initiales, une fois avec les scores de confiance et une fois avec les prédictions finales.

</Note>

Si vous souhaitez uniquement ajouter de nouveaux lots de lignes (sans colonnes), de façon incrémentielle, comme dans une boucle d’entraînement, utilisez plutôt le mode [`INCREMENTAL`](#INCREMENTAL-mode).

<div id="incremental-mode">
  ### INCREMENTAL mode
</div>

En mode incrémentiel, vous journalisez des lots de lignes dans un tableau au cours de l’expérience d’apprentissage automatique. C’est idéal pour surveiller des jobs de longue durée ou lorsque vous travaillez avec de grands tableaux qu’il serait inefficace de journaliser dans le run pour les mettre à jour. Dans l’interface utilisateur, le tableau est mis à jour avec de nouvelles lignes à mesure qu’elles sont journalisées, ce qui vous permet de consulter les données les plus récentes sans attendre la fin complète du run. Vous pouvez également parcourir les incréments pour afficher le tableau à différents moments.

<Note>Les Workspace de run dans la W&amp;B App sont limités à 100 incréments. Si vous journalisez plus de 100 incréments, seuls les 100 plus récents sont affichés dans le Workspace du run.</Note>

L’exemple suivant crée un tableau en mode `INCREMENTAL`, le journalise, puis y ajoute de nouvelles lignes. Notez que le tableau est journalisé une fois par étape d’entraînement (`step`).

<Note>
  L’exemple suivant utilise une fonction fictive `get_training_batch()` pour charger les données, une fonction fictive `train_model_on_batch()` pour entraîner le modèle et une fonction fictive
  `predict_on_batch()` pour effectuer des prédictions. Vous devrez les remplacer par vos propres fonctions de chargement des données, d’entraînement et de prédiction.
</Note>

La journalisation incrémentielle est généralement plus efficace en termes de calcul que la journalisation d’un nouveau tableau à chaque fois (`log_mode=MUTABLE`). Cependant, la W&amp;B App peut ne pas afficher toutes les lignes du tableau si vous journalisez un grand nombre d’incréments. Si votre objectif est de mettre à jour et de consulter les données de votre tableau pendant l’exécution de votre run, tout en rendant l’ensemble des données disponible pour l’analyse, envisagez d’utiliser deux tableaux : l’un avec le mode de journalisation `INCREMENTAL`, et l’autre avec le mode de journalisation `IMMUTABLE`.
