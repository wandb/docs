Personnalisez vos tableaux W&amp;B pour répondre à des questions sur les performances de votre modèle de machine learning, analyser vos données, et plus encore.

Explorez vos données de manière interactive pour :

* [Comparer deux tableaux W&amp;B](#table-comparison-options) enregistrées en tant que versions d&#39;artefact afin d&#39;analyser les changements dans vos données ou les performances du modèle.
* Comprendre des tendances générales dans vos données
* [Voir comment les valeurs que vous enregistrez dans un tableau évoluent au fil de vos runs](#visualize-how-values-change-throughout-your-runs).

<Note>
  Les tableaux W&amp;B présentent les comportements suivants :

  1. **Sans état dans un contexte d&#39;artefact** : tout tableau enregistré avec une version d&#39;artefact revient à son état par défaut après la fermeture de la fenêtre du navigateur
  2. **Avec état dans un contexte de Workspace ou de rapport** : toutes les modifications que vous apportez à un tableau dans le Workspace d&#39;un seul run, le Workspace du projet pour plusieurs runs ou un rapport sont conservées.

  Pour savoir comment enregistrer votre vue actuelle du tableau W&amp;B, voir [Enregistrer votre vue](#save-your-view).
</Note>

Depuis la vue fusionnée, vous pouvez

* **choisir la clé de jointure** : utilisez le menu déroulant en haut à gauche pour définir la colonne à utiliser comme clé de jointure pour les deux tableaux. Il s’agit généralement de l’identifiant unique de chaque ligne, par exemple le nom de fichier d’un exemple précis dans votre jeu de données ou un index incrémental pour vos échantillons générés. Notez qu’il est actuellement possible de sélectionner *n’importe quelle* colonne, ce qui peut produire des tableaux illisibles et ralentir les requêtes.
* **concaténer au lieu de joindre** : sélectionnez &quot;concaténer tous les tableaux&quot; dans ce menu déroulant pour *regrouper toutes les lignes* des deux tableaux dans un seul tableau plus grand, au lieu de faire une jointure sur leurs colonnes
* **référencer explicitement chaque tableau** : utilisez 0, 1 et * dans l’expression de filtre pour désigner explicitement une colonne dans l’une ou les deux instances du tableau
* **visualiser les différences numériques détaillées sous forme d’histogrammes** : comparez les valeurs de n’importe quelle cellule en un coup d’œil

<div id="visualize-how-values-change-throughout-your-runs">
  ## Visualisez comment les valeurs évoluent au fil de vos runs
</div>

Visualisez comment les valeurs que vous enregistrez dans un tableau évoluent au fil de vos runs à l’aide d’un curseur d’étape. Faites glisser le curseur d’étape pour afficher les valeurs enregistrées à différentes étapes. Par exemple, vous pouvez voir comment la perte, la précision ou d’autres métriques évoluent après chaque run.

Le curseur utilise une clé pour déterminer la valeur d’étape. La clé par défaut du curseur est `_step`, une clé spéciale que W&amp;B enregistre automatiquement pour vous. La clé `_step` est un entier qui s’incrémente de 1 chaque fois que vous appelez `wandb.Run.log()` dans votre code.

Pour ajouter un curseur d’étape à un tableau W&amp;B :

1. Accédez au Workspace de votre projet.
2. Cliquez sur **Add panel** dans le coin supérieur droit du Workspace.
3. Sélectionnez **Query panel**.
4. Dans l’éditeur d’expression de requête, sélectionnez `runs` et appuyez sur **Entrée** sur votre clavier.
5. Cliquez sur l’icône d’engrenage pour afficher les paramètres du panneau.
6. Réglez le sélecteur **Render As** sur **Stepper**.
7. Réglez **Stepper Key** sur `_step` ou sur la [clé à utiliser comme unité](#custom-step-keys) pour le curseur d’étape.

L’image suivante montre un panneau de requête avec trois runs W&amp;B et les valeurs qu’ils ont enregistrées à l’étape 295.

<Frame>
  <img src="/images/data_vis/stepper_key.png" alt="Fonctionnalité de curseur d’étape" />
</Frame>

Dans l’interface de l’application W&amp;B, vous pouvez remarquer des valeurs en double à plusieurs étapes. Cela peut se produire si plusieurs runs enregistrent la même valeur à des étapes différentes, ou si un run n’enregistre pas de valeurs à chaque étape. Si une valeur est manquante pour une étape donnée, W&amp;B utilise comme clé du curseur la dernière valeur enregistrée.

<div id="custom-step-key">
  ### Clé d’étape personnalisée
</div>

La clé d’étape peut être n’importe quelle métrique numérique que vous enregistrez dans vos runs en tant que clé d’étape, comme `epoch` ou `global_step`. Lorsque vous utilisez une clé d’étape personnalisée, W&amp;B associe chaque valeur de cette clé à une étape (`_step`) dans le run.

Ce tableau montre comment une clé d’étape personnalisée, `epoch`, est associée aux valeurs `_step` pour trois runs différents : `serene-sponge`, `lively-frog` et `vague-cloud`. Chaque ligne représente un appel à `wandb.Run.log()` à un `_step` donné dans un run. Les colonnes indiquent les valeurs d’`epoch` correspondantes, le cas échéant, qui ont été enregistrées à ces étapes. Certaines valeurs de `_step` sont omises pour gagner de la place.

La première fois que `wandb.Run.log()` a été appelé, aucun des runs n’a enregistré de valeur `epoch` ; le tableau affiche donc des cellules vides pour `epoch`.

| `_step` | vague-cloud (`epoch`) | lively-frog(`epoch`) | serene-sponge (`epoch`) |
| ------- | --------------------- | -------------------- | ----------------------- |
| 1       |                       |                      |                         |
| 2       |                       |                      | 1                       |
| 4       |                       | 1                    | 2                       |
| 5       | 1                     |                      |                         |
| 6       |                       |                      | 3                       |
| 8       |                       | 2                    | 4                       |
| 10      |                       |                      | 5                       |
| 12      |                       | 3                    | 6                       |
| 14      |                       |                      | 7                       |
| 15      | 2                     |                      |                         |
| 16      |                       | 4                    | 8                       |
| 18      |                       |                      | 9                       |
| 20      | 3                     | 5                    | 10                      |

Maintenant, si le curseur est réglé sur `epoch = 1`, voici ce qui se passe :

* `vague-cloud` trouve `epoch = 1` et renvoie la valeur enregistrée à `_step = 5`
* `lively-frog` trouve `epoch = 1` et renvoie la valeur enregistrée à `_step = 4`
* `serene-sponge` trouve `epoch = 1` et renvoie la valeur enregistrée à `_step = 2`

Si le curseur est réglé sur `epoch = 9` :

* `vague-cloud` n’enregistre pas non plus `epoch = 9`, donc W&amp;B utilise la dernière valeur antérieure, `epoch = 3`, et renvoie la valeur enregistrée à `_step = 20`
* `lively-frog` n’enregistre pas `epoch = 9`, mais la dernière valeur antérieure est `epoch = 5` ; il renvoie donc la valeur enregistrée à `_step = 20`
* `serene-sponge` trouve `epoch = 9` et renvoie la valeur enregistrée à `_step = 18`

<div id="save-your-view">
  ## Enregistrer votre vue
</div>

Les tableaux avec lesquels vous interagissez dans le Workspace du run, le Workspace du projet ou un rapport enregistrent automatiquement l’état de leur vue. Si vous appliquez des opérations à un tableau, puis fermez votre navigateur, le tableau conserve la dernière configuration consultée lorsque vous y revenez.

<Note>
  Les tableaux avec lesquels vous interagissez dans le contexte d’un artefact restent sans état.
</Note>

Pour enregistrer un tableau d’un Workspace dans un état donné, exportez-le vers un rapport W&amp;B. Pour exporter un tableau vers un rapport :

1. Sélectionnez l’icône à trois points (trois points verticaux) dans le coin supérieur droit du panneau de visualisation de votre Workspace.
2. Sélectionnez **Partager le panneau** ou **Ajouter au rapport**.
