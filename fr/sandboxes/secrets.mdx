Utilisez le [W&amp;B Secrets Manager](/fr/platform/secrets) avec le SDK Python W&amp;B pour accéder, dans un sandbox, à des informations sensibles telles que des clés API et des jetons.

Pour utiliser un secret dans un sandbox, ajoutez-le d&#39;abord au Secrets Manager de votre équipe. Faites ensuite référence à ce secret par son nom lorsque vous créez le sandbox. N&#39;incluez pas directement la valeur du secret dans votre code ni dans la configuration du sandbox.

W&amp;B injecte chaque secret demandé dans le sandbox sous forme de variable d&#39;environnement. Par défaut, le nom de la variable d&#39;environnement correspond au nom du secret. Vous pouvez personnaliser le nom de la variable d&#39;environnement avec le paramètre `env_var`. Pour les [secrets structurés](#extract-a-field-from-a-structured-secret), vous pouvez également utiliser le paramètre `field` pour exposer un champ spécifique de la valeur du secret.

<Note>
  Voir [Gérer les secrets](/fr/platform/secrets) pour plus d&#39;informations sur la création et la
  gestion des secrets dans votre compte W&amp;B.
</Note>

<div id="access-secrets-in-a-sandbox">
  ## Accéder aux secrets dans un sandbox
</div>

Utilisez `wandb.sandbox.Secret` pour demander l’accès aux secrets stockés dans le Secrets Manager de votre équipe. Passez un ou plusieurs objets `Secret` au paramètre `secrets` de `Sandbox.run()`.

Chaque objet `Secret` identifie un secret par son nom. W&amp;B injecte la valeur du secret dans le sandbox sous forme de variable d’environnement.

<Tip>
  Passez le nom du secret, et non sa valeur. Dans le sandbox, lisez le
  secret à partir de la variable d’environnement correspondante.
</Tip>

L’exemple suivant met deux secrets existants à disposition dans le sandbox : `HF_TOKEN` et `OPENAI_API_KEY`.

```python id="11o5yi"
from wandb.sandbox import Sandbox, Secret

with Sandbox.run(
    secrets=[
        Secret(name="HF_TOKEN"),
        Secret(name="OPENAI_API_KEY"),
    ],
) as sandbox:
    result = sandbox.exec([
        "python",
        "-c",
        """
import os
print(f"HF token: {os.environ['HF_TOKEN'][:8]}...")
print(f"OpenAI key: {os.environ['OPENAI_API_KEY'][:8]}...")
""",
    ]).result()
    print(result.stdout)
```

Dans le sandbox, accédez à chaque secret via sa variable d’environnement. Dans cet exemple, `HF_TOKEN` est disponible dans `os.environ["HF_TOKEN"]` et `OPENAI_API_KEY` dans `os.environ["OPENAI_API_KEY"]`.

Pour utiliser un autre nom de variable d’environnement, voir [Utiliser un nom de variable d’environnement personnalisé](#use-a-custom-environment-variable-name).

<div id="use-a-custom-environment-variable-name">
  ## Utiliser un nom personnalisé pour une variable d’environnement
</div>

Par défaut, le nom de la variable d’environnement d’un secret correspond au nom du secret. Pour utiliser un autre nom dans le sandbox, définissez le paramètre `env_var` de `Secret` (`Secret(env_var="CUSTOM_NAME")`).

L’exemple suivant rend le secret `HF_TOKEN` disponible sous le nom `HUGGINGFACE_TOKEN` dans le sandbox :

```python
from wandb.sandbox import Sandbox, Secret

with Sandbox.run(
    secrets=[
        Secret(name="HF_TOKEN", env_var="HUGGINGFACE_TOKEN")
    ],
) as sandbox:
    result = sandbox.exec([
        "python",
        "-c",
        """
import os
print(f"Hugging Face token: {os.environ['HUGGINGFACE_TOKEN'][:8]}...")
""",
    ]).result()
    print(result.stdout)
```
