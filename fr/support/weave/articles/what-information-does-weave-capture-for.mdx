---
title: "Quelles informations Weave capture-t-il pour une fonction ?"
keywords: ["Capture de données", "Capture de code", "Informations client", "Informations système"]
---

Une fonction peut être désignée comme une [Op](/fr/weave/guides/tracking/ops) Weave soit manuellement via un décorateur, soit automatiquement dans le cadre d’une intégration activée. Lorsqu’une Op s’exécute, Weave capture des informations détaillées pour faciliter votre analyse. Weave vous offre un contrôle précis sur ce qui est enregistré si vous souhaitez un comportement différent de celui par défaut ; voir ci-dessous pour des exemples de configuration.

* **Capture de code** - Weave capture une représentation du code source de l’Op. Cela inclut les commentaires en ligne ainsi que la capture récursive de la valeur des variables ou de la source des fonctions non-Op qui ont été appelées. La capture de code vous permet de voir ce que faisait votre fonction même si la modification n’a pas été enregistrée dans votre système de contrôle de source. La capture de code est utilisée dans le cadre de la gestion des versions des Op, ce qui vous permet de comprendre l’évaluation de votre code au fil du temps. Si la capture de code est désactivée, une valeur de hachage sera utilisée à la place.

* **Nom de la fonction, entrées et sorties** - Le nom de la fonction sera capturé mais peut être [remplacé](/fr/weave/guides/tracking/tracing/#call-display-name). Une représentation des entrées et des sorties basée sur JSON sera capturée. Pour les entrées, le nom de l’argument sera capturé en plus de la valeur. Weave vous permet de [personnaliser la journalisation](/fr/weave/guides/tracking/ops#customize-logged-inputs-and-outputs) des entrées et des sorties - vous pouvez spécifier une fonction pour ajouter/supprimer/modifier ce qui est enregistré.

* **Hiérarchie des appels d’Op** - Lorsqu’une Op est appelée dans le contexte d’exécution d’une autre Op, cette relation est capturée, même dans les cas
  où une fonction intermédiaire non-Op est en cours d’exécution. Cette relation entre les appels d’Op est utilisée pour fournir une &quot;arborescence de trace&quot;.

* **Statut d’exécution et exceptions** - Weave suit si une fonction est en cours d’exécution, terminée ou en erreur. Si une exception se produit pendant l’exécution, le message d’erreur et une trace de pile sont enregistrés.

* **Informations système** - Weave peut capturer des informations sur le système d’exploitation sur lequel le client s’exécute, y compris des informations détaillées sur la version.

* **Informations client** - Weave peut capturer des informations sur le client Weave lui-même, comme le langage de programmation utilisé et des informations détaillées sur la version de ce langage et de la bibliothèque cliente Weave.

* **Temps** - L’heure de début et de fin de l’exécution est capturée et utilisée également pour les calculs de latence.

* **Utilisation des jetons** - Dans certaines [intégrations](/fr/weave/guides/integrations/), les décomptes d’utilisation des jetons LLM peuvent être enregistrés automatiquement.

* **Contexte utilisateur et run** - La journalisation est associée à un compte utilisateur W&amp;B. Celui-ci sera capturé avec tout contexte de run wandb.

* **Informations dérivées** - Weave peut calculer des informations dérivées à partir des informations brutes enregistrées, par exemple une estimation du coût peut être calculée en fonction de l’utilisation des jetons et de la connaissance du modèle utilisé. Weave agrège également certaines informations sur plusieurs appels.

* **Informations supplémentaires de votre choix** - Vous pouvez choisir d’enregistrer des [métadonnées personnalisées avec `weave.attributes`](/fr/weave/guides/core-types/models#track-production-calls) dans le cadre de votre appel ou joindre un [retour](/fr/weave/guides/tracking/feedback#add-feedback-to-a-call) à un appel.
