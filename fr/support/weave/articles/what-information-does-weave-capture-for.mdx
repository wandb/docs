---
title: "Quelles informations Weave capture-t-il pour une fonction ?"
keywords: ["Capture de données", "Capture du code", "Informations client", "Informations système"]
---

Une fonction peut être définie comme une [Op](/fr/weave/guides/tracking/ops) Weave, soit manuellement à l&#39;aide d&#39;un décorateur, soit automatiquement dans le cadre d&#39;une intégration activée. Lorsqu&#39;une Op s&#39;exécute, Weave capture des informations détaillées pour faciliter votre analyse. Weave vous offre un contrôle précis sur ce qui est enregistré si vous souhaitez un comportement différent de celui par défaut ; voir ci-dessous pour des exemples de configuration.

* **Capture du code** - Weave capture une représentation du code source de l&#39;Op. Cela inclut les commentaires en ligne, ainsi que la capture récursive de la valeur des variables ou du code source des fonctions non-Op appelées. La capture du code vous permet de voir ce que faisait votre fonction, même si la modification n&#39;a pas été enregistrée dans votre système de contrôle de version. La capture du code est utilisée dans le cadre de la gestion des versions des Op, ce qui vous permet de comprendre comment votre code a été évalué au fil du temps. Si la capture du code est désactivée, une valeur de hachage sera utilisée à la place.

* **Nom de la fonction, entrées et sorties** - Le nom de la fonction sera capturé, mais peut être [redéfini](/fr/weave/guides/tracking/tracing/#call-display-name). Une représentation des entrées et des sorties basée sur JSON sera capturée. Pour les entrées, le nom de l&#39;argument sera capturé en plus de sa valeur. Weave vous permet de [personnaliser la journalisation](/fr/weave/guides/tracking/ops#customize-logged-inputs-and-outputs) des entrées et des sorties : vous pouvez spécifier une fonction pour ajouter, supprimer ou modifier ce qui est enregistré.

* **Hiérarchie des appels d&#39;Op** - Lorsqu&#39;une Op est appelée dans le contexte de l&#39;exécution d&#39;une autre Op, cette relation est capturée, même dans les cas
  où une fonction intermédiaire non-Op s&#39;exécute. Cette relation entre les appels d&#39;Op est utilisée pour fournir une &quot;arborescence de trace&quot;.

* **Statut d&#39;exécution et exceptions** - Weave suit si une fonction est en cours d&#39;exécution, terminée ou en erreur. Si une exception survient pendant l&#39;exécution, le message d&#39;erreur et une trace de pile sont enregistrés.

* **Informations système** - Weave peut capturer des informations sur le système d&#39;exploitation sur lequel le client s&#39;exécute, y compris des informations détaillées sur la version.

* **Informations client** - Weave peut capturer des informations sur le client Weave lui-même, telles que le langage de programmation utilisé et des informations détaillées sur la version de ce langage et de la bibliothèque cliente Weave.

* **Temps d&#39;exécution** - L&#39;heure de début et de fin d&#39;exécution est capturée et également utilisée pour les calculs de latence.

* **Utilisation des tokens** - Dans certaines [intégrations](/fr/weave/guides/integrations/), le nombre de tokens LLM utilisés peut être enregistré automatiquement.

* **Contexte utilisateur et run** - La journalisation est associée à un compte utilisateur W&amp;B. Celui-ci sera capturé avec tout contexte de run wandb.

* **Informations dérivées** - Weave peut calculer des informations dérivées à partir des informations brutes enregistrées ; par exemple, une estimation du coût peut être calculée en fonction de l&#39;utilisation des tokens et du modèle utilisé. Weave agrège également certaines informations sur l&#39;ensemble des appels.

* **Informations supplémentaires que vous choisissez** - Vous pouvez choisir d&#39;enregistrer des [métadonnées personnalisées avec `weave.attributes`](/fr/weave/guides/core-types/models#track-production-calls) dans le cadre de votre appel ou d&#39;associer un [feedback](/fr/weave/guides/tracking/feedback#add-feedback-to-a-call) à un appel.
