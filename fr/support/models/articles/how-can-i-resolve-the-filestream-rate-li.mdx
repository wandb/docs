---
title: "Comment résoudre l’erreur &quot;Filestream rate limit exceeded&quot; ?"
keywords: ["Connectivité", "Panne"]
---

L’erreur `Filestream rate limit exceeded` se produit lorsque W&amp;B reçoit trop de requêtes API sur une courte période, ce qui peut interrompre la journalisation pendant l’entraînement. Pour la résoudre, suivez les étapes suivantes afin de réduire le volume de requêtes, d’écarter un incident de service et de demander une limite plus élevée si nécessaire :

1. Optimisez la journalisation. Réduisez la fréquence de journalisation ou envoyez les journaux par lots afin de diminuer le nombre de requêtes API, et espacez les heures de démarrage des expériences pour éviter des requêtes API simultanées.
2. Vérifiez les pannes. Pour écarter un problème temporaire côté serveur, consultez la [page de statut de W&amp;B](https://status.wandb.com).
3. Contactez l’assistance. Contactez l’assistance W&amp;B (support@wandb.com) en fournissant les détails de votre configuration expérimentale afin de demander une augmentation des limites de débit.
