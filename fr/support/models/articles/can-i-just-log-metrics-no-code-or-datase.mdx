---
title: "Puis-je simplement consigner des métriques, sans code ni exemples de jeu de données ?"
keywords: ["Administrateur", "Gestion des équipes", "Métriques"]
---

Par défaut, W&amp;B ne consigne pas d’exemples de jeu de données. Par défaut, W&amp;B consigne le code et les métriques système.

Il existe deux façons de désactiver l’enregistrement du code à l’aide de variables d’environnement :

1. Définissez `WANDB_DISABLE_CODE` sur `true` pour désactiver tout le suivi du code. Cette action empêche la récupération du SHA git et du patch de diff.
2. Définissez `WANDB_IGNORE_GLOBS` sur `*.patch` pour arrêter la synchronisation du patch de diff vers les serveurs, tout en le conservant disponible localement pour pouvoir l’appliquer avec `wandb restore`.

Les administrateurs de l’organisation peuvent appliquer les paramètres de confidentialité pour toutes les équipes de leur organisation depuis la section **Confidentialité** de l’onglet **Paramètres** du tableau de bord du compte ou de l’organisation. Si les administrateurs de l’organisation appliquent un paramètre, les administrateurs d’équipe ne sont pas autorisés à le configurer dans leurs équipes respectives.

En tant qu’administrateur d’équipe, vous pouvez désactiver l’enregistrement du code par défaut pour les runs de votre équipe :

1. Accédez à `https://wandb.ai/<team>`. Remplacez `<team>` par le nom de votre équipe.
2. Sélectionnez **Team settings** dans la navigation de gauche.
3. Ouvrez la section **Confidentialité** et désactivez **Enable code saving by default**.
