---
title: "Comment résoudre les erreurs d’autorisation lors de l’enregistrement d’un run ?"
keywords: ["Runs", "Sécurité"]
---

Les erreurs d’autorisation lors de l’enregistrement d’un run indiquent généralement un décalage entre votre code, vos identifiants ou votre configuration de déploiement, et l’entité W&amp;B vers laquelle vous essayez d’enregistrer ce run. Cette page répertorie les causes les plus courantes et explique comment corriger chacune d’elles afin que votre run puisse écrire dans l’entité et le projet visés.

Pour résoudre les erreurs d’autorisation lors de l’enregistrement d’un run dans une entité W&amp;B, effectuez les vérifications suivantes :

* **Vérifier les noms de l’entité et du projet** : Assurez-vous que l’orthographe et la casse des noms de l’entité W&amp;B et du projet dans votre code sont correctes. Une faute de frappe ou une différence de casse peut amener le run à cibler un emplacement auquel vous n’avez pas accès.
* **Confirmer les autorisations** : Assurez-vous que l’administrateur vous a accordé les autorisations nécessaires pour l’entité et le projet cibles.
* **Vérifier les identifiants de connexion** : Vérifiez que vous êtes connecté au bon compte W&amp;B. Testez en créant un run avec le code suivant, en remplaçant `[ENTITY-NAME]` et `[PROJECT-NAME]` par votre entité et votre projet cibles :
  ```python
  import wandb

  run = wandb.init(entity="[ENTITY-NAME]", project="[PROJECT-NAME]")
  run.log({"example_metric": 1})
  run.finish()
  ```
* **Définir la clé API** : Définissez la variable d’environnement `WANDB_API_KEY` afin que le client s’authentifie avec le compte voulu. Remplacez `[API-KEY]` par votre clé API W&amp;B :
  ```bash
  export WANDB_API_KEY="[API-KEY]"
  ```
* **Confirmer les informations sur l’hôte** : Pour les déploiements personnalisés, pointez le client vers l’hôte W&amp;B approprié afin que le run soit envoyé au bon serveur. Remplacez `[HOST-URL]` par l’URL de votre hôte W&amp;B :
  ```bash
  wandb login --relogin --host=[HOST-URL]
  export WANDB_BASE_URL=[HOST-URL]
  ```

Après avoir effectué ces vérifications, vous pouvez initialiser et enregistrer un run dans l’entité et le projet cibles sans rencontrer d’erreurs d’autorisation.
