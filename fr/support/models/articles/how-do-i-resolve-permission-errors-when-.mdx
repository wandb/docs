---
title: "Comment puis-je résoudre les erreurs d’autorisation lors de l’enregistrement d’un run ?"
keywords: ["Runs", "Sécurité"]
---

Pour résoudre les erreurs d’autorisation lors de l’enregistrement d’un run dans une entité W&amp;B, suivez ces étapes :

* **Vérifier les noms de l’entité et du projet** : assurez-vous que l’orthographe des noms de l’entité W&amp;B et du projet est correcte, y compris la casse, dans votre code.
* **Confirmer les autorisations** : assurez-vous que les autorisations nécessaires ont bien été accordées par l’administrateur.
* **Vérifier les identifiants de connexion** : assurez-vous d’être connecté au bon compte W&amp;B. Testez en créant un run avec le code suivant :
  ```python
  import wandb

  run = wandb.init(entity="your_entity", project="your_project")
  run.log({'example_metric': 1})
  run.finish()
  ```
* **Définir la clé API** : utilisez la variable d’environnement `WANDB_API_KEY` :
  ```bash
  export WANDB_API_KEY='your_api_key'
  ```
* **Confirmer les informations sur l’hôte** : pour les déploiements personnalisés, définissez l’URL de l’hôte :
  ```bash
  wandb login --relogin --host=<host-url>
  export WANDB_BASE_URL=<host-url>
  ```
