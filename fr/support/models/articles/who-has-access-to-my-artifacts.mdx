keywords: ["Artifacts"]
---

Cette page explique comment W&amp;B détermine l’accès aux artifacts, afin que vous puissiez savoir qui, dans votre organisation, peut les lire ou les modifier. Les Artifacts héritent des autorisations d’accès de leur projet parent :

* Dans un projet privé, seuls les membres de l’équipe peuvent accéder aux Artifacts.
* Dans un projet public, tous les Users peuvent lire les Artifacts, mais seuls les membres de l’équipe peuvent les créer ou les modifier.
* Dans un projet ouvert, tous les Users peuvent lire et écrire des Artifacts.

<div id="artifacts-workflows">
  ## Flux de travail des Artifacts
</div>

De nombreux flux de travail pour gérer et modifier les artifacts utilisent [l’API W&amp;B](/fr/models/track/public-api-guide), qui fait partie de [la bibliothèque cliente Python W&amp;B](/fr/models/ref/python/), pour accéder aux données stockées dans W&amp;B.
