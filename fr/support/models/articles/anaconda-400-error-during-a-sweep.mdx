---
title: "Comment puis-je corriger une erreur `anaconda 400 error` lors d’un Sweep ?"
keywords: ["Sweeps", "Métriques"]
---

L’erreur suivante se produit généralement lorsque vous n’enregistrez pas la métrique que vous optimisez :

```
wandb: ERROR Error while calling W&B API: anaconda 400 error:
{"code": 400, "message": "TypeError: bad operand type for unary -: 'NoneType'"}
```

Pour corriger cela, assurez-vous de journaliser exactement la métrique spécifiée dans la configuration de votre sweep. Par exemple, si la configuration de votre sweep spécifie :

```yaml
metric:
  name: validation_loss
  goal: minimize
```

Votre code d’entraînement doit alors appeler :

```python
wandb.log({"validation_loss": value})
```

Le nom de la métrique doit être exactement le même dans votre configuration de sweep et dans votre appel à `wandb.log`.

Pour plus d&#39;informations, voir [Dépannage de Sweeps](/fr/models/sweeps/troubleshoot-sweeps/) et [Définir la configuration du sweep](/fr/models/sweeps/define-sweep-configuration/).
