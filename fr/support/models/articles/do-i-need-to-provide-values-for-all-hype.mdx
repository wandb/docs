---
title: "Dois-je fournir des valeurs pour tous les hyperparamètres dans un W&B Sweep ? Puis-je définir des valeurs par défaut ?"
keywords: ["Sweeps"]
---

Accédez aux noms et aux valeurs des hyperparamètres depuis la configuration du Sweep à l’aide de `(run.config())`, qui se comporte comme un dictionnaire.

Pour les runs en dehors d’un Sweep, définissez les valeurs de `wandb.Run.config()` en passant un dictionnaire à l’argument `config` dans `wandb.init()`. Dans un Sweep, toute configuration fournie à `wandb.init()` sert de valeur par défaut, que le Sweep peut remplacer.

Utilisez `rwandb.Run.config.setdefaults()` pour définir explicitement ce comportement. Les extraits de code suivants illustrent les deux méthodes :

<Tabs>
  <Tab title="wandb.init()">
    ```python
    # Définir des valeurs par défaut pour les hyperparamètres
    config_defaults = {"lr": 0.1, "batch_size": 256}

    # Démarrer un run et fournir des valeurs par défaut
    # qu’un Sweep peut remplacer
    with wandb.init(config=config_defaults) as run:
        # Ajoutez ici le code d’entraînement
        ...
    ```
  </Tab>

  <Tab title="config.setdefaults()">
    ```python
    # Définir des valeurs par défaut pour les hyperparamètres
    config_defaults = {"lr": 0.1, "batch_size": 256}

    # Démarrer un run
    with wandb.init() as run:
        # Mettre à jour les valeurs non définies par le Sweep
        run.config.setdefaults(config_defaults)

        # Ajoutez ici le code d’entraînement
    ```
  </Tab>
</Tabs>
