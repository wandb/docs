---
title: "Dois-je fournir des valeurs pour tous les hyperparamètres dans le cadre d'un W&B Sweep ? Puis-je définir des valeurs par défaut ?"
keywords: ["Sweeps"]
---

Accédez aux noms et aux valeurs des hyperparamètres à partir de la configuration du balayage à l&#39;aide de `(run.config())`, qui se comporte comme un dictionnaire.

Pour les runs en dehors d&#39;un balayage, définissez les valeurs de `wandb.Run.config()` en transmettant un dictionnaire à l&#39;argument `config` dans `wandb.init()`. Dans un balayage, toute configuration fournie à `wandb.init()` sert de valeur par défaut, que le balayage peut redéfinir.

Utilisez `rwandb.Run.config.setdefaults()` pour un comportement explicite. Les extraits de code suivants illustrent les deux méthodes :

<Tabs>
  <Tab title="wandb.init()">
    ```python
    # Définir des valeurs par défaut pour les hyperparamètres
    config_defaults = {"lr": 0.1, "batch_size": 256}

    # Démarrer un run et fournir des valeurs par défaut
    # qu'un balayage peut redéfinir
    with wandb.init(config=config_defaults) as run:
        # Ajoutez ici le code d'entraînement
        ...
    ```
  </Tab>

  <Tab title="config.setdefaults()">
    ```python
    # Définir des valeurs par défaut pour les hyperparamètres
    config_defaults = {"lr": 0.1, "batch_size": 256}

    # Démarrer un run
    with wandb.init() as run:
        # Mettre à jour toutes les valeurs non définies par le balayage
        run.config.setdefaults(config_defaults)

        # Ajoutez ici le code d'entraînement
    ```
  </Tab>
</Tabs>
