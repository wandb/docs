---
title: "Comment puis-je utiliser des commandes CLI personnalisées avec les Sweeps ?"
keywords: ["Sweeps"]
featured: true
---

Vous pouvez utiliser les Sweeps de W&amp;B avec des commandes CLI personnalisées si la configuration d&#39;entraînement transmet des arguments en ligne de commande.

Dans l&#39;exemple ci-dessous, l&#39;extrait de code montre un terminal bash dans lequel un utilisateur lance l&#39;entraînement d&#39;un script Python nommé `train.py` en fournissant des valeurs que le script interprète :

```bash
/usr/bin/env python train.py -b \
    your-training-config \
    --batchsize 8 \
    --lr 0.00001
```

Pour implémenter des commandes personnalisées, modifiez la clé `command` dans le fichier YAML. D’après l’exemple précédent, la configuration se présente comme suit :

```yaml
program:
  train.py
method: grid
parameters:
  batch_size:
    value: 8
  lr:
    value: 0.0001
command:
  - ${env}
  - python
  - ${program}
  - "-b"
  - your-training-config
  - ${args}
```

La clé `${args}` est remplacée par tous les paramètres de la configuration de balayage, mis en forme pour `argparse` sous la forme `--param1 value1 --param2 value2`.

Pour ajouter des arguments supplémentaires en dehors de `argparse`, implémentez ce qui suit :

```python
parser = argparse.ArgumentParser()
args, unknown = parser.parse_known_args()
```

<Note>
  Selon l’environnement, `python` peut désigner Python 2. Pour garantir l’utilisation de Python 3, utilisez `python3` dans la configuration de la commande :

  ```yaml
  program:
    script.py
  command:
    - ${env}
    - python3
    - ${program}
    - ${args}
  ```
</Note>
