---
title: "Quel espace de stockage utilise chaque version d'artifact ?"
keywords: ["Artifacts", "Stockage"]
---

Seuls les fichiers qui changent entre deux versions d&#39;artifact entraînent des coûts de stockage.

<Frame>
  <img src="/images/artifacts/artifacts-dedupe.PNG" alt="Déduplication d’un artifact" />
</Frame>

Prenons un artifact d’image nommé `animals` contenant deux fichiers image, `cat.png` et `dog.png` :

```text
images
|-- cat.png (2Mo) # Ajouts dans `v0`
|-- dog.png (1Mo) # Ajouts dans `v0`
```

Cet artifact se voit attribuer la version `v0`.

Lorsque vous ajoutez une nouvelle image, `rat.png`, W&amp;B crée une nouvelle version de l’artifact, `v1`, avec le contenu suivant :

```text
images
|-- cat.png (2Mo) # Ajouts dans `v0`
|-- dog.png (1Mo) # Ajouts dans `v0`
|-- rat.png (3Mo) # Ajouts dans `v1`
```

La version `v1` totalise 6 Mo, mais n&#39;occupe que 3 Mo d&#39;espace, car elle partage les 3 Mo restants avec `v0`. Si vous supprimez `v1`, vous récupérez les 3 Mo de stockage associés à `rat.png`. Si vous supprimez `v0`, les coûts de stockage de `cat.png` et `dog.png` sont reportés sur `v1`, ce qui porte sa taille de stockage à 6 Mo.
