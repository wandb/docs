---
title: "Combien d’espace de stockage utilise chaque version d’artefact ?"
keywords: ["Artifacts", "Stockage"]
---

Seuls les fichiers qui changent entre deux versions d’artefact entraînent des coûts de stockage.

<Frame>
  <img src="/images/artifacts/artifacts-dedupe.PNG" alt="Déduplication des artefacts" />
</Frame>

Considérez un artefact d’image nommé `animals`, qui contient deux fichiers image, `cat.png` et `dog.png` :

```
images
|-- cat.png (2MB) # Ajouté dans `v0`
|-- dog.png (1MB) # Ajouté dans `v0`
```

Cet artefact reçoit la version `v0`.

Lorsque vous ajoutez une nouvelle image, `rat.png`, une nouvelle version de l’artefact, `v1`, est créée avec le contenu suivant :

```
images
|-- cat.png (2MB) # Ajouté dans `v0`
|-- dog.png (1MB) # Ajouté dans `v0`
|-- rat.png (3MB) # Ajouté dans `v1`
```

La version `v1` totalise 6MB, mais n’occupe que 3MB d’espace puisqu’elle partage les 3MB restants avec `v0`. La suppression de `v1` libère les 3MB de stockage associés à `rat.png`. La suppression de `v0` transfère les coûts de stockage de `cat.png` et `dog.png` à `v1`, ce qui augmente sa taille de stockage à 6MB.
