---
title: "Code d’erreur API 404 - Modèle introuvable"
keywords: ["Erreurs du serveur"]
---

Une **réponse 404** de l’API W&amp;B Inférence signifie que le serveur n’a pas trouvé le modèle ou la ressource que vous avez demandé. Le corps de la réponse inclut souvent un message tel que **&quot;The requested resource was not found&quot;**, ce qui indique généralement que l’ID du modèle dans votre requête ne correspond à aucun modèle disponible via l’API.

<div id="why-this-happens">
  ## Pourquoi cela se produit
</div>

* **ID du modèle incorrect** : Le nom ou le chemin du modèle dans votre requête comporte une faute ou n’est pas correctement formaté.
* **Le modèle n’est pas disponible sur W&amp;B Inference** : Tous les modèles hébergés par d’autres fournisseurs ne sont pas disponibles via l’API W&amp;B Inférence. Il se peut que le modèle demandé n’ait pas encore été ajouté.
* **Utilisation de la mauvaise URL de base** : Si vous pointez vers le point de terminaison d’un autre fournisseur tout en utilisant une clé API W&amp;B, l’ID du modèle ne sera pas correctement résolu.
* **Le modèle a été supprimé** : Un modèle auparavant disponible peut avoir été rendu obsolète ou supprimé.

<div id="what-you-can-do">
  ## Ce que vous pouvez faire
</div>

1. **Vérifier l&#39;ID du modèle**
   * Vérifiez l&#39;identifiant exact du modèle dans [modèles disponibles](/fr/inference/models).
   * Les ID de modèle respectent la casse et utilisent généralement le format `provider/model-name` (par exemple, `meta-llama/Llama-3.3-70B-Instruct`).

2. **Confirmer l&#39;URL de base**
   * L&#39;URL de base de W&amp;B Inference est `https://api.inference.wandb.ai/v1`. Assurez-vous que votre client utilise cet hôte, et non l&#39;hôte d&#39;API du Cloud mutualisé (`https://api.wandb.ai`) ni le point de terminaison d&#39;un autre fournisseur. Pour le suivi de l&#39;utilisation, certains clients envoient également l&#39;en-tête facultatif `OpenAI-Project` défini sur `team/project`, comme indiqué dans la [référence de l&#39;API « list models »](/fr/inference/api-reference/list-models).

3. **Demander l&#39;accès à un nouveau modèle**
   * Si le modèle dont vous avez besoin n&#39;est pas actuellement pris en charge, contactez [l&#39;assistance W&amp;B](mailto:support@wandb.com) pour demander son ajout.

Pour plus d&#39;informations, voir [Serverless Inference](/fr/inference) et la [référence de l&#39;API « list models »](/fr/inference/api-reference/list-models).
