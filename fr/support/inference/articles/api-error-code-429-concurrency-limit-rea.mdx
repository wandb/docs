title: "Code d’erreur d’API 429 - Limite de concurrence atteinte pour les requêtes"
keywords: ["Quotas et limites de débit"]
---

Une erreur 429 accompagnée du message &quot;Concurrency limit reached for requests&quot; signifie que vous envoyez trop de requêtes simultanées à la W&amp;B Inference API.

<div id="why-this-happens">
  ## Pourquoi cela arrive
</div>

W&amp;B Inference applique des limites de concurrence afin de garantir une utilisation équitable et la stabilité du service. Lorsque le nombre de requêtes simultanées émises depuis votre compte dépasse la limite autorisée, les requêtes supplémentaires sont rejetées avec un code de statut 429.

<div id="what-you-can-do">
  ## Ce que vous pouvez faire
</div>

1. **Réduire les requêtes simultanées**
   * Mettez en place une file d&#39;attente des requêtes ou une limitation du débit dans votre application
   * Utilisez un backoff exponentiel lorsque vous réessayez des requêtes ayant échoué

2. **Augmenter vos limites**
   * Vérifiez les limites de concurrence de votre plan et passez à une offre supérieure si nécessaire

Pour plus d&#39;informations, voir [Informations sur l&#39;utilisation et les limites](/fr/inference/usage-limits/).
