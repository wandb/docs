---
title: "Code d’erreur 429 de l’API - Limite de concurrence atteinte pour les requêtes"
keywords: ["Quotas et limites de débit"]
---

Une erreur 429 accompagnée du message &quot;Concurrency limit reached for requests&quot; signifie que vous envoyez trop de requêtes simultanées à l’API W&amp;B Inférence.

<div id="why-this-happens">
  ## Pourquoi cela se produit
</div>

W&amp;B Inference applique des limites de concurrence afin de garantir une utilisation équitable et la stabilité du service. Lorsque le nombre de requêtes simultanées envoyées depuis votre compte dépasse la limite autorisée, les requêtes supplémentaires sont rejetées avec un code de statut 429.

<div id="what-you-can-do">
  ## Ce que vous pouvez faire
</div>

1. **Réduisez les requêtes simultanées**
   * Mettez en place une file d&#39;attente des requêtes ou une limitation du débit dans votre application
   * Utilisez une stratégie de backoff exponentiel lorsque vous réessayez des requêtes ayant échoué

2. **Augmentez vos limites**
   * Vérifiez les limites de concurrence de votre plan et passez à une offre supérieure si nécessaire

Pour plus d&#39;informations, voir [Informations d&#39;utilisation et limites](/fr/inference/usage-limits/).
