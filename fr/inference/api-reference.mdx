Cette référence décrit l’API REST Serverless Inference, qui vous permet d’appeler par programmation des modèles fondamentaux depuis vos propres applications. Utilisez-la pour intégrer l’inférence hébergée à des services, scripts ou notebooks, sans avoir à gérer l’infrastructure des modèles.

<div id="base-url">
  ## URL de base
</div>

Accédez au service d’Inférence à :

```plaintext
https://api.inference.wandb.ai/v1
```

<div id="prerequisites">
  ## Prérequis
</div>

Pour appeler l&#39;API d&#39;Inférence, il vous faut :

* Un compte W&amp;B avec des crédits d&#39;Inférence.
* Une clé API W&amp;B valide.

Si vous appartenez à plusieurs équipes, ou si vous souhaitez imputer votre utilisation à un projet, vous aurez également besoin des ID d&#39;équipe et de projet. Dans les exemples de code, ils apparaissent sous la forme `[YOUR-TEAM]/[YOUR-PROJECT]`. Si vous ne les spécifiez pas, W&amp;B utilise votre entité par défaut et le nom du projet `inference`.

<div id="available-methods">
  ## Méthodes disponibles
</div>

L’API d’Inférence fournit des endpoints compatibles avec OpenAI pour interagir avec des modèles fondamentaux. Les méthodes suivantes sont disponibles :

* **[Chat Completions](/fr/inference/api-reference/chat-completions)** : Créez des complétions de chat à l’aide de modèles fondamentaux.
* **[List Models](/fr/inference/api-reference/list-models)** : Obtenez tous les modèles disponibles et leurs ID.

<div id="authentication">
  ## Authentification
</div>

Toutes les requêtes API nécessitent une authentification à l’aide de votre clé API W&amp;B. Créez une clé API sur [wandb.ai/settings](https://wandb.ai/settings).

Incluez votre clé API dans les en-têtes de la requête :

* Pour le SDK d’OpenAI, définissez le paramètre `api_key`.
* Pour les appels API directs, utilisez `Authorization: Bearer [YOUR-API-KEY]`.
