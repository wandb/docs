LoRA (Low-Rank Adaptation) vous permet de personnaliser de grands modèles de langage en n&#39;entraînant et en ne stockant qu&#39;un « module complémentaire » léger au lieu d&#39;un nouveau modèle complet. Cela rend la personnalisation plus rapide, moins coûteuse et plus facile à déployer.

Vous pouvez entraîner ou téléverser un LoRA pour donner à un modèle de base de nouvelles capacités, par exemple le spécialiser pour l&#39;assistance client, l&#39;écriture créative ou un domaine technique particulier. Cela vous permet d&#39;adapter le comportement du modèle sans avoir à réentraîner ou redéployer l&#39;ensemble du modèle.

<div id="why-use-wb-inference-for-loras">
  ## Pourquoi utiliser W&amp;B Inference pour les LoRAs ?
</div>

* Téléversez une seule fois, déployez instantanément — aucun serveur à gérer.
* Suivez exactement quelle version est active grâce à la gestion des versions des artefacts.
* Mettez à jour les modèles en quelques secondes en remplaçant de petits fichiers LoRA plutôt que les poids complets du modèle.

<div id="workflow">
  ## Flux de travail
</div>

1. Téléversez les poids de votre LoRA en tant qu’artefact W&amp;B
2. Utilisez l’URI de l’artefact comme nom de votre modèle dans l’API
3. W&amp;B charge dynamiquement vos poids pour l’inférence

Voici un exemple d’appel de votre modèle LoRA personnalisé avec W&amp;B Inference :

```python
from openai import OpenAI

model_name = f"wandb-artifact:///{WB_TEAM}/{WB_PROJECT}/qwen_lora:latest"

client = OpenAI(
    base_url="https://api.inference.wandb.ai/v1",
    api_key=API_KEY,
    project=f"{WB_TEAM}/{WB_PROJECT}",
)

resp = client.chat.completions.create(
    model=model_name,
    messages=[{"role": "user", "content": "Say 'Hello World!'"}],
)
print(resp.choices[0].message.content)
```

Consultez ce [notebook de premiers pas](https://wandb.me/lora_nb) pour une démonstration interactive expliquant comment créer une LoRA et la téléverser sur W&amp;B en tant qu’artefact.

<div id="how-to-add-loras-and-use-them">
  ## Comment ajouter des LoRAs et les utiliser
</div>

Vous pouvez ajouter des LoRAs à votre compte W&amp;B et commencer à les utiliser de deux façons :

<Tabs>
  <Tab title="Téléverser un LoRA entraîné ailleurs">
    Téléversez votre propre répertoire LoRA personnalisé sous forme d&#39;artefact W&amp;B. C&#39;est idéal si vous avez entraîné votre LoRA ailleurs (environnement local, fournisseur de cloud ou service partenaire).

    Ce code Python téléverse vers W&amp;B les poids de votre LoRA stockés localement sous forme d&#39;artefact versionné. Il crée un artefact de type `lora` avec les métadonnées requises (modèle de base et région de stockage), ajoute vos fichiers LoRA depuis un répertoire local et le journalise dans votre projet W&amp;B pour l&#39;utiliser avec l&#39;Inférence.

    ```python
    import wandb

    run = wandb.init(entity=WB_TEAM, project=WB_PROJECT)

    artifact = wandb.Artifact(
        "qwen_lora",
        type="lora",
        metadata={"wandb.base_model": "OpenPipe/Qwen3-14B-Instruct"},
        storage_region="coreweave-us",
    )

    artifact.add_dir("<path-to-lora-weights>")
    run.log_artifact(artifact)
    ```

    ### Exigences clés

    Pour utiliser vos propres LoRAs avec l&#39;Inférence :

    * Le LoRA doit avoir été entraîné à l&#39;aide de l&#39;un des modèles listés dans la [section Modèles de base pris en charge](#supported-base-models).
    * Le LoRA doit être enregistré au format PEFT comme artefact de type `lora` dans votre compte W&amp;B.
    * Le LoRA doit être stocké dans `storage_region=&quot;coreweave-us&quot;` pour une faible latence.
    * Lors du téléversement, incluez le nom du modèle de base sur lequel vous l&#39;avez entraîné (par exemple, `meta-llama/Llama-3.1-8B-Instruct`). Cela garantit que W&amp;B peut le charger avec le bon modèle.
  </Tab>

  <Tab title="Entraîner un nouveau LoRA avec W&B">
    Entraînez un nouveau LoRA avec [W&amp;B Training (RL sans serveur)](/fr/training). Votre LoRA devient automatiquement un artefact W&amp;B que vous pouvez utiliser directement.

    Pour en savoir plus sur l&#39;entraînement de votre propre LoRA, voir le [Démarrage rapide ART d&#39;OpenPipe](https://art.openpipe.ai/getting-started/quick-start).

    Une fois l&#39;entraînement terminé, votre LoRA est automatiquement disponible en tant qu&#39;artefact.
  </Tab>
</Tabs>

Une fois votre LoRA ajouté à votre projet comme artefact, utilisez l&#39;URI de l&#39;artefact dans vos appels d&#39;inférence, comme ceci :

```python
# Une fois l'entraînement terminé, utilisez votre artefact directement
model_name = f"wandb-artifact:///{WB_TEAM}/{WB_PROJECT}/your_trained_lora:latest"
```

<div id="pricing">
  ## Tarification
</div>

L&#39;inférence LoRA sans serveur est simple et économique : vous payez uniquement pour le stockage et l&#39;inférence que vous utilisez réellement, plutôt que pour des serveurs qui tournent en permanence ou des instances GPU dédiées.

* [**Stockage**](https://wandb.ai/site/pricing/) - Le stockage des poids LoRA est peu coûteux, surtout par rapport à la gestion de votre propre infrastructure GPU.
* **Utilisation de l&#39;inférence** - Les appels utilisant des artefacts LoRA sont facturés aux mêmes tarifs que l&#39;[inférence de modèle standard](/fr/inference/usage-limits#account-tiers-and-default-usage-caps). Aucuns frais supplémentaires ne s&#39;appliquent pour le déploiement de LoRA personnalisés.
