title: "Créer un LoRA affiné"
description: >
  Découvrez comment créer un LoRA affiné à utiliser avec W&B Inference.
---

W&amp;B Inference vous permet d’utiliser un LoRA personnalisé avec certains modèles de base. Ce tutoriel montre comment créer un LoRA affiné par post-entraînement supervisé
avec la bibliothèque [TRL](https://huggingface.co/docs/trl/index). Cet exemple utilise un jeu de données de paires requête/réponse pour affiner un modèle afin qu’il réponde comme un cowboy.

Vous pouvez également créer un LoRA avec [W&amp;B Training](/fr/training), qui propose un apprentissage par renforcement sans serveur.

<div id="post-training-dataset">
  ## Jeu de données post-entraînement
</div>

Le jeu de données suivant contient 50 paires de requêtes et de réponses, présentées sous forme de liste de messages, comme :

> Utilisateur : « Quelle est votre couleur préférée ? »

> Assistant : « Eh bien, partenaire, ma couleur préférée, c&#39;est l&#39;orange flamboyant d&#39;un coucher de soleil dans le désert… »

Le fichier d&#39;exemple contient un objet JSON par ligne. Enregistrez les données suivantes dans votre répertoire de travail sous le nom `cowboy_examples.jsonl`.

<div id="post-training">
  ## post-entraînement
</div>

Ce script entraîne un adaptateur LoRA sur des exemples du fichier JSONL et le téléverse sur W&amp;B en tant qu&#39;artefact afin de l&#39;utiliser avec l&#39;API W&amp;B Inférence ou le Playground.

Dans les grandes lignes, ce script effectue les opérations suivantes :

1. Se connecte à W&amp;B. L&#39;[intégration Hugging Face Transformers](https://docs.wandb.ai/models/integrations/huggingface) de W&amp;B Models enregistre automatiquement la progression de l&#39;entraînement et les métriques.
2. Charge le modèle de base ([OpenPipe/Qwen3-14B-Instruct](https://huggingface.co/OpenPipe/Qwen3-14B-Instruct)) depuis Hugging Face.
3. Configure LoRA à l&#39;aide d&#39;hyperparamètres, comme le rang et l&#39;alpha définis comme constantes vers le haut du fichier.
4. Charge les exemples du fichier dans un jeu de données, puis exécute [SFTTrainer](https://huggingface.co/docs/trl/en/sft_trainer). Par défaut, le script utilise tous les exemples.
5. Enregistre LoRA et le téléverse sur W&amp;B en tant qu&#39;[artefact](/fr/models/artifacts) afin qu&#39;il puisse être utilisé avec W&amp;B Inference.

Lorsque le script se termine, ouvrez dans votre navigateur la dernière URL affichée pour voir l&#39;artefact enregistré. Elle ressemble à ceci :
`Artifact URL: https://wandb.ai/<yourentity>/create-lora-tutorial/artifacts/lora/OpenPipe_Qwen3-14B-Instruct_cowboy/v0`

Enregistrez le programme suivant sous `create_lora.py`, puis mettez à jour la valeur `ENTITY` avec votre entité W&amp;B. Pour simplifier l&#39;exécution, le script utilise des [métadonnées de script intégrées](https://docs.astral.sh/uv/guides/scripts/#declaring-script-dependencies) pour déclarer les dépendances.

```python title="create_lora.py" expandable lines highlight={37}
# /// script
# dependencies = [
#   "torch",
#   "transformers",

"""
Inférence LoRA avec W&B

Ce script montre comment créer un LoRA et le téléverser dans W&B en tant qu'artefact,
pour une utilisation avec l'API d'inférence ou le playground.

Vous pouvez exécuter ce script avec uv :
  uv run create_lora.py <examples_file.jsonl>
"""

import argparse
import json
import sys
import tempfile

import torch
import wandb
from transformers import AutoModelForCausalLM, AutoTokenizer, TrainingArguments
from peft import LoraConfig, get_peft_model
from trl import SFTTrainer
from datasets import Dataset

ENTITY = "<yourentity>"
PROJECT = "create-lora-tutorial"
if ENTITY == "<yourentity>":
    sys.exit("Please update the ENTITY variable")

# LoRA Details
BASE_MODEL = "OpenPipe/Qwen3-14B-Instruct"
RUN_NAME = BASE_MODEL.replace("/", "_") + "_cowboy"
LORA_RANK = 8
LORA_ALPHA = 32


def provision_lora(jsonl_path, max_examples=None):
    """
    Provisionnement du LoRA

    Dans un flux de travail en production, vous entraîneriez/mettriez à jour votre LoRA ici selon vos besoins.
    Pour cette démonstration d'inférence, nous créons un LoRA et l'entraînons sur les exemples fournis.

    Arguments :
        jsonl_path : Chemin vers le fichier JSONL contenant les exemples d'entraînement
        max_examples : Nombre maximum d'exemples d'entraînement à utiliser (None pour tous les exemples)

    Remarque : cela utilisera l'intégration W&B Hugging Face
    https://docs.wandb.ai/models/integrations/huggingface
    """
    # Créer un répertoire temporaire
    lora_dir = tempfile.mkdtemp(prefix="identity_lora_")
    print(f"Chargement du modèle : {BASE_MODEL}")

    # Détection du périphérique - utiliser MPS sur Apple Silicon, CUDA sur les GPU NVIDIA, ou CPU
    if torch.cuda.is_available():
        device_map = "auto"
        dtype = torch.bfloat16
    elif torch.backends.mps.is_available():
        device_map = "mps"
        dtype = torch.bfloat16
    else:
        device_map = "cpu"
        dtype = torch.float32

    # Charger le tokenizer
    tokenizer = AutoTokenizer.from_pretrained(BASE_MODEL)
    if tokenizer.pad_token is None:
        tokenizer.pad_token = tokenizer.eos_token

    # Charger le modèle
    model = AutoModelForCausalLM.from_pretrained(
        BASE_MODEL,
        dtype=dtype,
        device_map=device_map,
        trust_remote_code=True,
    )

    # Configurer le LoRA
    lora_config = LoraConfig(
        r=LORA_RANK,
        lora_alpha=LORA_ALPHA,
        target_modules=["q_proj", "k_proj", "v_proj", "o_proj",
                       "gate_proj", "up_proj", "down_proj"],
        lora_dropout=0,
        bias="none",
        task_type="CAUSAL_LM",
    )

    # Appliquer le LoRA au modèle
    model = get_peft_model(model, lora_config)

    # Activer le gradient checkpointing pour optimiser l'utilisation de la mémoire
    model.enable_input_require_grads()
    if hasattr(model, 'gradient_checkpointing_enable'):
        model.gradient_checkpointing_enable()

    # Charger les exemples depuis un fichier JSONL
    print(f"Chargement des exemples depuis {jsonl_path}")
    train_examples = []
    with open(jsonl_path, "r", encoding="utf-8") as f:
        for line in f:
            train_examples.append(json.loads(line))
            if max_examples is not None and len(train_examples) >= max_examples:
                break

    print(f"Utilisation de {len(train_examples)} exemples d'entraînement")

    # Convertir au format Dataset attendu par SFTTrainer
    # En supposant que le JSONL contient un champ 'text' ou similaire
    train_dataset = Dataset.from_list(train_examples)

    # Arguments d'entraînement
    training_args = TrainingArguments(
        output_dir=lora_dir,
        num_train_epochs=3,
        per_device_train_batch_size=1,
        gradient_accumulation_steps=4,
        learning_rate=2e-4,
        logging_steps=10,
        save_strategy="no",
    )

    # Créer le trainer
    trainer = SFTTrainer(
        model=model,
        train_dataset=train_dataset,
        args=training_args,
        processing_class=tokenizer,
    )
    trainer.train()

    # Sauvegarder le LoRA sur le disque avant de le téléverser pour l'inférence
    print(f"Sauvegarde dans {lora_dir}")
    model.save_pretrained(lora_dir)
    tokenizer.save_pretrained(lora_dir)

    return lora_dir


def upload_artifact(lora_dir, run):
    """
    Téléverser l'artefact

    Une fois votre LoRA préparé, vous pouvez le téléverser dans W&B en tant qu'artefact.
    Veillez à définir la région de stockage sur coreweave-us pour activer l'inférence LoRA !
    """

    artifact = wandb.Artifact(
        RUN_NAME,
        type="lora",
        metadata={"wandb.base_model": BASE_MODEL},
        storage_region="coreweave-us",
    )

    artifact.add_dir(lora_dir)
    run.log_artifact(artifact)
    artifact.wait()
    run.finish()

    return artifact


def parse_args():
    """Analyser les arguments de la ligne de commande"""
    parser = argparse.ArgumentParser(
        description="Entraîner un modèle LoRA et le téléverser dans W&B"
    )
    parser.add_argument(
        "examples_file",
        type=str,
        help="Chemin vers le fichier JSONL contenant les exemples d'entraînement"
    )
    parser.add_argument(
        "--max-examples",
        type=int,
        default=None,
        help="Nombre maximum d'exemples d'entraînement à utiliser (par défaut : tous les exemples)"
    )
    return parser.parse_args()


def main():
    """Fonction d'exécution principale"""
    args = parse_args()

    wandb.login()
    run = wandb.init(entity=ENTITY, project=PROJECT, name=RUN_NAME)

    lora_dir = provision_lora(args.examples_file, max_examples=args.max_examples)

    artifact = upload_artifact(lora_dir, run)

    print(f"URL de l'artefact : {artifact.url}")


if __name__ == "__main__":
    main()
```

Exécutez le script avec `uv` :

```bash
uv run create_lora.py cowboy_examples.jsonl
```

Le temps d’exécution dépend du matériel. Vous pouvez ajouter l’argument `--max-examples=10` pour accélérer l’entraînement, mais cela réduit la capacité du LLM à répondre en restant dans son personnage.
